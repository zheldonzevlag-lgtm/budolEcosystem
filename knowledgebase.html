Future Recommendations

Implement Webhook Retry Logic

Automated retry mechanism for failed payment webhooks to ensure cart cleanup and status updates.

Cart Item "Save for Later"

Allow users to move items out of the active cart without deleting them, improving conversion potential.

Advanced Stock Reservation

Reserve stock for a limited time during checkout to prevent out-of-stock errors during payment processing.

Centralized Environment Config Package

Move the environment detection logic into a shared package to avoid duplication and ReferenceErrors across services.

Automated Mobile Build Pipeline

Integrate Fastlane or GitHub Actions to automate the APK build process on version increment.

Biometric Fallback for OTP

Implement biometric authentication (FaceID/Fingerprint) as a fallback for OTP in the mobile app.

Enhanced Password Policies

Implement password complexity requirements (uppercase, symbols) to further align with PCI DSS standards.

Social Login Integration

Explore Google and Apple Sign-In as alternative login methods for improved user experience.

Database Migration for Legacy Phone Numbers

Execute the migrate-phone-numbers.mjs script to ensure all historical records match the new E.164 standard.

International Phone Support

Integrate libphonenumber-js to support global expansion beyond PH numbers.

Automated UI Regression Testing

Implement Playwright tests to automatically verify breadcrumb paths and UI alignment consistency.

Compliance Standards

Every release must adhere to the following regulatory and technical standards:

  • PCI DSS: Payment Card Industry Data Security Standard.
  • BSP: Bangko Sentral ng Pilipinas financial regulations.
  • BIR: Bureau of Internal Revenue tax compliance.
  • NPC: National Privacy Commission (Data Privacy Act of 2012).
  • DTI: Department of Trade and Industry consumer protection.
