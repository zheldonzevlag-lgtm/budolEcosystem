Release Summary: v474 (Dynamic Biometric UI)

This release addresses the static nature of the biometric login icons on the PIN entry screen. Previously, both Fingerprint and Face icons were shown if any biometric hardware was detected, leading to a suboptimal UX on devices supporting only one method. Version 474 introduces dynamic detection and rendering of biometric icons based on actual device capabilities.

Key Achievements

  • ✔ Enhanced BiometricService with specific type detection capabilities.
  • ✔ Implemented dynamic icon rendering in LoginScreen suffixIcon.
  • ✔ Maintained backward compatibility for custom TFLite face templates.
  • ✔ Improved visual feedback for enabled/disabled biometric states.
  • ✔ Verified layout stability across various device configurations.

Compliance Statement

This update remains compliant with BSP and NPC guidelines regarding secure authentication. No biometric data is stored within the application; the system continues to leverage OS-level secure enclaves and hardware-backed biometric verification.
