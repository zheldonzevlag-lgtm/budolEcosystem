Developer Manual - v474

Subject: Dynamic Biometric UI Implementation

1. Architecture Overview

The biometric UI logic has been moved from a static check to a dynamic list-based approach. The app now queries the OS for specific supported biometric types and renders the corresponding icons.

2. Service Layer Changes

Added getAvailableBiometricTypes() to BiometricService to wrapper the local_auth call.

File: biometric_service.dart


Future<List<BiometricType>> getAvailableBiometricTypes() async {
  try {
    return await _auth.getAvailableBiometrics();
  } catch (e) {
    if (kDebugMode) print('BiometricService: Error getting biometric types: $e');
    return [];
  }
}

3. UI Layer Changes

The LoginScreen state now tracks _availableBiometricTypes and uses a helper method to build the suffix icons for the PIN input field.

File: login_screen.dart


Widget? _buildBiometricSuffixIcons() {
  if (!_isBiometricAvailable && !_hasFaceTemplate) return null;

  final List<Widget> icons = [];

  // Fingerprint Icon
  if (_availableBiometricTypes.contains(BiometricType.fingerprint) || 
      _availableBiometricTypes.contains(BiometricType.strong)) {
    icons.add(IconButton(
      onPressed: _handleBiometricLogin,
      icon: Icon(Icons.fingerprint, color: _isBiometricEnabled ? Colors.white70 : Colors.white24, size: 28),
      tooltip: 'Login with Fingerprint',
    ));
  }

  // Face Icon (OS or Custom)
  if (_availableBiometricTypes.contains(BiometricType.face) || _hasFaceTemplate) {
    icons.add(IconButton(
      onPressed: _handleFaceRecognitionLogin,
      icon: Icon(Icons.face, color: (_isBiometricEnabled || _hasFaceTemplate) ? Colors.white70 : Colors.white24, size: 28),
      tooltip: 'Login with Face Recognition',
    ));
  }
  
  // ... (Row construction with fallback)
}

4. Data Contracts

  • BiometricType: Uses the local_auth enum values (fingerprint, face, iris, strong, weak).
  • _isBiometricEnabled: Boolean flag from secure storage that determines icon opacity and interactivity.
  • _hasFaceTemplate: Boolean flag indicating existence of custom TFLite face data.
