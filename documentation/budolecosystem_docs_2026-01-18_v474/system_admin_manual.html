System Admin Manual - v474

Release Objective: Optimize biometric login visibility and ensure hardware-specific icons are displayed on the mobile application.

1. Troubleshooting Biometric Icons

If a user reports that the biometric icon (Fingerprint or Face) is missing from the PIN input screen:

  1. Verify the user's device hardware supports the specific biometric method.
  2. Ensure the user has enrolled their biometrics in the device system settings.
  3. Confirm the app version is v474 or higher.
  4. Ask the user to log in once manually with their PIN to refresh the biometric availability cache.

2. Known Behaviors

  • Faint Icons: If the icons are visible but faint (low opacity), biometrics are supported but disabled in the app's internal Settings.
  • Fallback Icon: On devices where hardware is detected but the type cannot be precisely identified by the OS, the system will default to showing the Fingerprint icon as a generic biometric trigger.

3. Security Compliance

Biometric authentication remains a client-side operation. No biometric data (fingerprint hashes or facial templates) is sent to the central servers. The app only receives a "Success/Fail" boolean from the OS security layer.
