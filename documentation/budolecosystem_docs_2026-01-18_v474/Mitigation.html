Mitigation Plan - v474

R474-01: Empty Type Detection Fallback

Action: If the local_auth package fails to identify specific types but confirms biometric availability, the UI will display a generic Fingerprint icon to allow the user to attempt authentication.

R474-02: Unified Face Login Trigger

Action: Both OS-level face unlock and app-level custom face templates are mapped to the same Icons.face widget. Tapping this icon prioritizes the OS biometric prompt, which handles the secure PIN release.

R474-03: Layout Stability

Action: The biometric icons are encapsulated in a Row with MainAxisSize.min and placed within the suffixIcon of the InputDecoration. This ensures they remain inside the text box boundaries regardless of keyboard state or screen size.
