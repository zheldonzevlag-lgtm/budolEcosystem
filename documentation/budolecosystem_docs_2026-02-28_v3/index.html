v5.3.3 Release Notes

Security & Auth Standardization

Date: 2026-02-28

Executive Summary

This release addresses a critical authorization bug where administrators were receiving "Unauthorized" errors when attempting to create categories. The fix involves standardizing the Admin API to use database-level verification (requireAdmin) and improving JWT payload normalization.

Key Changes

  • Auth Normalization: Updated lib/auth.js to correctly derive isAdmin from JWT accountType.
  • API Standardization: Converted /api/admin/categories to use requireAdmin middleware.
  • Data Visibility: Administrators can now see hidden/inactive categories in the management portal.
