Issues & Fixes Log (v5.3.3)

1. "Unauthorized" Error on Category Management FIXED

Description: Authenticated administrators with valid "ADMIN" account types were blocked from creating or editing categories because the API only checked for a boolean isAdmin flag which was missing in some JWT contexts.

Resolution: Refactored auth normalization to treat accountType: "ADMIN" as functionally equivalent to isAdmin: true across the entire ecosystem.

2. Restricted Administrative Visibility FIXED

Description: The admin category management portal was only showing active categories (isActive: true), preventing admins from restoring or deleting hidden categories.

Resolution: Removed the where: { isActive: true } clause from the admin-specific categories route.
