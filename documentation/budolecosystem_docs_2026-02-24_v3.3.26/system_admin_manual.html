System Administration Manual (v3.3.25)

Database Schema Updates (v3.3.25)

Version 3.3.25 introduces changes to the `StoreAddress` table. Specifically:

  • Optional Fields: `district` and `zip` are now optional.
  • New Fields: `label` and `isDefault` have been verified for persistence.

Action Required: Run `npx prisma generate` to update the client. For production, ensure `npx prisma migrate deploy` is executed to apply schema changes if any migration files were created (none for this specific hotfix, but good practice to check).

Geocoding Configuration

The system now supports configurable geocoding providers.

Supported Providers

  • Nominatim (OpenStreetMap): Default provider. Free but subject to usage policies (1 request/sec). Requires a valid User-Agent (handled automatically by the system).
  • Geoapify: Commercial provider. Requires an API Key. Better for high-volume usage.

Configuration

To change the provider:

  1. Go to Admin Dashboard > Settings > Map Settings.
  2. Select the provider (OSM/Nominatim or Geoapify).
  3. If Geoapify is selected, enter your API Key.
  4. Save changes.

Troubleshooting

  • "Failed to fetch" errors: Check server logs for API connectivity issues. Ensure the server can reach nominatim.openstreetmap.org or api.geoapify.com.
  • Rate Limiting (429): If using Nominatim, ensure usage is within limits (1 req/sec). Consider switching to Geoapify for higher limits.
