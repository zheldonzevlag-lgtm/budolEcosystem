Developer Manual v3.3.0

1. Architecture Overview

This release integrates a centralized Audit Logging service (`lib/audit.js`) and a Real-time Event bus (`lib/realtime.js`) to ensure data integrity and immediate UI updates.

2. Core Components

2.1 Forensic Audit Logging

The audit logging system uses SHA-256 hashing to ensure log integrity. Every log entry includes a hash of its contents and the previous log's hash (simulating a blockchain-like structure).

// lib/audit.js

export async function createAuditLog(userId, action, request, options = {}) {
    // ... metadata collection ...
    const hashInput = `${userId}:${action}:${entityId}:${status}:${timestamp}:${JSON.stringify(metadata)}`;
    const integrityHash = crypto.createHash('sha256').update(hashInput).digest('hex');
    
    return await prisma.auditLog.create({
        data: {
            // ... fields ...
            metadata: { ...metadata, integrityHash }
        }
    });
}

3. Dependencies & Software Choices

Phase 8 Discussion:

  • Next.js (v14): Chosen for its server-side rendering capabilities, essential for SEO and secure handling of environment variables.
  • Prisma ORM: Provides type-safe database access and automated migrations, reducing SQL injection risks.
  • Jest: Selected for its robust testing framework, allowing mock injections (e.g., `global.fetch`) for reliable unit testing.
  • Crypto (Node.js built-in): Used for SHA-256 hashing. Native implementation ensures performance and security without external dependencies.
  • Pusher: Managed WebSocket service for real-time updates, minimizing infrastructure overhead for maintaining socket connections.
