Developer Manual - v111

Build Configuration Fixes

During the production build, we encountered several issues that required manual intervention in the Android project configuration.

1. Dependency Version Forcing

Some transitive dependencies (like androidx.browser and androidx.core) were pulling in versions that required Android Gradle Plugin 8.9.1, while the project uses 8.7.3. We forced stable versions in android/app/build.gradle.kts:

configurations.all {
    resolutionStrategy {
        force("androidx.browser:browser:1.8.0")
        force("androidx.core:core:1.15.0")
        force("androidx.core:core-ktx:1.15.0")
    }
}

App Branding Restoration

The app name and icon were updated to meet the production requirements.

  • AndroidManifest.xml: Updated android:label to "budolPay 1.0".
  • Launcher Icons: Re-generated using flutter_launcher_icons with assets/app_icon.png as the source.

Code Fixes

Resolved compilation errors that were blocking the release build:

  • home_screen.dart: Added missing material.dart import and fixed a trailing brace syntax error.
  • main.dart: Added missing api_service.dart import to provide the ApiService to the widget tree.

Build Command

The release APK was built using the following command to ensure all icons were included despite resource optimization flags:

flutter build apk --release --no-tree-shake-icons
