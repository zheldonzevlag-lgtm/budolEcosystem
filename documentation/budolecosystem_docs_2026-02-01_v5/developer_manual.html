Developer Manual - Version 2.1.2

1. Purpose & Rationale

This release focuses on finalizing the Shopee-style quick registration flow for budolPay and synchronizing the Android build version (v1.3.60). It also addresses critical SSO "Unauthorized Application" errors discovered during the release verification phase.

2. Technical Explanation

The mobile application versioning was synchronized across pubspec.yaml and api_service.dart. The build process utilizes obfuscation for enhanced security. On the backend, the SSO flow was hardened by ensuring proper ecosystem app registration and admin account provisioning.

3. Code References

  • Build Config: pubspec.yaml - Incremented version to 1.3.60+158.

  • Quick Registration: budolID index.js - Added /auth/register/quick endpoint.
