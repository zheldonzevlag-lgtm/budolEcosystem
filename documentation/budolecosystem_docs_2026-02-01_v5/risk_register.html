Risk Register & Mitigation Plan - v2.1.2

Risk Register

ID Risk Description Impact Probability Severity
R-001 SSO API Key exposure in public repositories High Low Medium
R-002 Database schema desynchronization during updates Critical Medium High
R-003 Mobile app version fragmentation Medium High Medium

Mitigation Plan

  • M-001 (API Keys): Use environment variables and .gitignore for sensitive keys. Transition to dynamic secrets rotation in v2.3.0.
  • M-002 (Schema): Always run npx prisma generate and prisma migrate in a staging environment before production release.
  • M-003 (Versioning): Enforce version checks in api_service.dart and prompt users for mandatory updates.
