Developer Manual - Universal Payment Modal

Rationale

Standardizing the payment interface ensures a consistent user experience across the marketplace (budolShap) and the payment gateway (budolPay). The "Image 2" format was selected for its superior branding, clear instructions, and intuitive timer display.

Technical Implementation

1. Checkout Web Page

The internal checkout page in the Payment Gateway Service was overhauled to use a responsive modal layout with centralized CSS variables for branding.

File: budolpay-0.1.0/services/payment-gateway-service/index.js

Changes

  • Updated QRCodeModal.jsx in budolShap to align with the universal modal structure.
  • Implemented a 10-minute (600s) countdown timer with a "warning" state (<60s) in both Web and React implementations.
  • Added a branding footer "budolShap." to all payment views.
  • Synchronized step-by-step "How to pay" instructions.
  • Added success state redirection with a 2-second delay for feedback.

Compliance

BSP Circular No. 808: All payment intent creations and completions are logged in the AuditLog table with financial transaction metadata.

PCI DSS: No sensitive card data is handled directly; the system uses secure payment intents and QR-based transfers.
