BudolEcosystem Documentation v204

PIN Setup Flow Fix: Resolving Authentication Loops

← Previous (v203) Main Index Next (v205) →

1. Purpose

This update fixes an authentication loop where users with missing PINs (e.g., after a system reset) were stuck on the PIN Login screen instead of being guided to the PIN Setup screen. This was caused by the mobile app misinterpreting the OTP_REQUIRED status as a request for more OTP verification instead of PIN creation.

2. Technical Explanation

The issue was dual-faceted, requiring changes in both the backend and the mobile client:

  • Auth Service: Updated verify-pin to return 403 PIN_SETUP_REQUIRED instead of 401 OTP_REQUIRED when a user exists but has no PIN. This clearly signals that the authentication method is invalid because the PIN hasn't been created yet.
  • Mobile App (Flutter): Added a new pinSetup step to the login flow. The app now monitors the verifyOtp response for the PIN_SETUP_REQUIRED status and automatically switches the UI to "Set your 6-digit PIN" instead of "Enter your 6-digit PIN".
  • API Service: Implemented the setupPin method in the mobile ApiService to communicate with the backend's /login/mobile/setup-pin endpoint.

3. Technical Details

Auth Service Change (verify-pin)

if (!user.pinHash) {
    return res.status(403).json({ 
        status: 'PIN_SETUP_REQUIRED',
        error: 'PIN not set for this account.',
        message: 'Your PIN needs to be set up. Redirecting to PIN setup...'
    });
}

Mobile App Change (Login Screen)

if (result['status'] == 'PIN_SETUP_REQUIRED') {
    setState(() {
        _currentStep = LoginStep.pinSetup;
    });
}

5. Risks & Mitigation

Risk Impact Mitigation
Missing Setup PIN Route Users can't set PIN Ensured setupPin is added to ApiService.
Old App Versions Loop persists Backend change to 403 provides a more descriptive error even on old apps.

6. Recommendations

  • Add a "Cancel PIN Setup" option that returns the user to the phone number screen safely.
  • Implement PIN complexity requirements (no repeated or sequential digits) in both client and server.
