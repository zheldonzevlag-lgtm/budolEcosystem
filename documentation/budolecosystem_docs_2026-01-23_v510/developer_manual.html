Developer Manual v510

Problem Identification

The ordersService.js was throwing a PrismaClientValidationError because it attempted to use a stock field in prisma.product.update(). Upon inspection of schema.prisma, it was confirmed that the Product model only contains an inStock boolean field, not a numerical stock field.

Technical Solution

The solution involved two main changes in ordersService.js:

1. Refactoring Availability Check

Replaced the numerical stock comparison with a simple boolean check.

// BEFORE
if (product.stock < item.quantity) {
    throw new Error(`Product ${product.name} is out of stock...`);
}

// AFTER
if (!product.inStock) {
    throw new Error(`Product ${product.name} is currently out of stock`);
}

2. Removing Stock Decrement

Removed the loop that attempted to decrement product stock during the transaction, as the database does not track individual stock units for products at this time.

// REMOVED SECTION
for (const item of storeData.items) {
    await tx.product.update({
        where: { id: item.productId },
        data: { stock: { decrement: item.quantity } }
    });
}
Note for Future Enhancement: If numerical stock tracking is required, a schema migration adding stock Int @default(0) to the Product model in schema.prisma will be necessary.
