Ecosystem Management

The Budol Ecosystem is managed via a central runner script that handles environment switching, database migrations, and service orchestration.

Startup Command

node start-ecosystem.js

Follow the prompts to select the target environment (LOCAL or PRODUCTION). This will automatically update all .env files across services.

Monitoring Logs

Admins can now monitor real-time API traffic in the service logs. Look for the following tags:

  • [Gateway Request]: Incoming traffic to the ecosystem.
  • [Auth Request]: Requests reaching the identity provider.
  • [Auth Body]: Request payloads (passwords redacted).

Database Settings

Ensure the following keys are present in the systemSetting table for real-time functionality:

  • REALTIME_PUSHER_APP_ID
  • REALTIME_PUSHER_KEY
  • REALTIME_PUSHER_SECRET
  • REALTIME_PUSHER_CLUSTER
