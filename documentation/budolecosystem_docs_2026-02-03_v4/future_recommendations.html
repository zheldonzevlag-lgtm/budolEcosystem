Future Recommendations

1. Async Notification Queue

Currently, the Auth Service waits for notification services (Email/SMS) before responding. Moving these to a background queue (e.g., BullMQ or RabbitMQ) would further reduce response times and prevent timeouts even on 10s limits.

2. Centralized Log Aggregator

While the new middleware provides visibility, logs are currently stored in individual service stdout. A centralized aggregator (e.g., ELK stack or Grafana Loki) is recommended for production monitoring.

3. Circuit Breakers

Implement circuit breakers in the API Gateway to handle service outages gracefully without hanging the entire ecosystem.

4. Monorepo Dependency Workspace

Adopt a formal monorepo workspace tool (e.g., Lerna, Nx, or pnpm workspaces) to manage dependencies across all services more reliably and avoid manual npm install in each directory.
