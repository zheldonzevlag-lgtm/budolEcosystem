Developer Manual - v3.0.5

Key Changes in v3.0.5 (v2026-02-03_v4)

1. System-wide Dependency Stabilization

Critical missing dependencies were identified and resolved across the ecosystem to ensure startup stability. This includes mDNS discovery and core service modules.

1.1 Bonjour Service for Discovery

Added bonjour-service to the root package.json. This is essential for the start-ecosystem.js script to broadcast and discover services on the local network.

File: package.json

1.2 API Gateway Dependencies

The API Gateway was missing socket.io, axios, and pusher, which caused startup failures. These have been installed and added to the gateway's dependency list.

File: api-gateway/package.json

1.3 Service-Specific Fixes

  • budolShap: Resolved missing next dependency.
  • budolID: Resolved missing express dependency.
  • budolAccounting: Fixed missing @budolpay/database local file reference in package.json.

2. ApiService Timeout Increase (Retained from v3.0.4)

All authentication-related methods in the mobile app's ApiService remain at 30 seconds to ensure reliability during registration and PIN setup.

File: api_service.dart

// Example change in register() method final response = await http.post( Uri.parse(url), headers: {'Content-Type': 'application/json'}, body: body, ).timeout(const Duration(seconds: 30));

2. Deep Observability Middleware

A custom middleware was added to the Auth Service and API Gateway to provide deep visibility into the request/response lifecycle. This includes logging request bodies (redacted for sensitive data), status codes, and execution duration.

File: auth-service/index.js

app.use((req, res, next) => { const start = Date.now(); console.log(`[Auth Request] ${req.method} ${req.url}`); res.on('finish', () => { const duration = Date.now() - start; console.log(`[Auth Response] ${req.method} ${req.url} ${res.statusCode} (${duration}ms)`); }); next(); });

3. Pusher Configuration Fix

The API Gateway was updated to correctly reference the REALTIME_PUSHER_* keys from the database SystemSetting table, resolving the "Configuration missing" error in the logs.

File: api-gateway/index.js
