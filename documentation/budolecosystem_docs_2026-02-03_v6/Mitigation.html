Risk Mitigations

Mitigation: Automated Fallback

Risk: Realtime provider failure.

Strategy: The system automatically detects failures in Pusher or Socket.io and falls back to SWR Polling to ensure continuity of data display.

Mitigation: PCI DSS Compliance (10.2.2)

Risk: Unauthorized access to audit logs.

Strategy: Every audit log access and update is logged itself. The use of realtime event streaming does not bypass authentication or role-based access control.

Mitigation: Code Review & Linting

Risk: Runtime errors due to typos.

Strategy: Implementing stricter TypeScript checks and ESLint rules to prevent undefined variable errors like the one fixed in v6.
