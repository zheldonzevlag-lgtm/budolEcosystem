Issues and Fixes

Issue: activeSMSProvider is not defined

Symptoms: Unhandled Runtime Error when accessing the Notification Settings page.

Cause: Typo in the variable name in page.tsx. The variable was defined as activeSmsProvider but used as activeSMSProvider in the JSX.

Fix: Standardized Variable Naming

Renamed all occurrences of activeSMSProvider to activeSmsProvider in app/settings/notifications/page.tsx to match the variable definition.

Issue: Realtime updates required manual refresh (from v5)

Symptoms: Forensic audit logs only updated when the page was manually reloaded.

Cause: Frontend was using static fetching without a live subscription or polling mechanism.

Fix: Provider-Agnostic Realtime Hook (from v5)

Implemented useRealtime hook that supports Pusher, Socket.io, and SWR Polling fallback. Integrated this into audit-logs/page.jsx.
