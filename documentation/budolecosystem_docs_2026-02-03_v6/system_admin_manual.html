System Admin Manual: Configuring Realtime Services

Overview

The Budol Ecosystem supports three types of realtime delivery. Admins can switch between them instantly via the Settings > Realtime menu.

1. SWR Polling (Default/Fallback)

Use Case: Low-traffic environments or when no external services are available.

Configuration: Simply set the "Polling Interval" (in milliseconds). Standard is 10,000ms (10 seconds).

Reliability: High Cost: Free

2. Pusher (Hosted Service)

Use Case: Production environments requiring high scalability and managed infrastructure.

Setup:

  1. Create an account at pusher.com.
  2. Create a new "Channels" app.
  3. Copy the App ID, Key, Secret, and Cluster.
  4. Paste them into the Pusher configuration card in Budol Admin.

Reliability: Very High Cost: Paid/Tiered

3. Socket.io (Self-Hosted)

Use Case: Internal networks or cost-sensitive high-traffic environments where you manage your own socket server.

Setup:

  1. Ensure your api-gateway or dedicated socket server is running.
  2. Provide the full URL (e.g., https://sockets.budolecosystem.com).

Reliability: High Cost: Server Resources

Troubleshooting

If the Forensic Audit Trail shows a Polling badge instead of your chosen provider, check the following:

  • Are all Pusher keys present?
  • Is the Socket.io URL reachable from the client browser?
  • Check the browser console for "Connection Error" logs.
