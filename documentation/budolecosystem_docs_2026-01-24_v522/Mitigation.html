Mitigation Plan

M-001: Customer Identification Fix

Risk: Inaccurate customer identification.

Mitigation: Implement a robust display logic in OrderItem.jsx that prioritizes the authenticated user's name (order.user.name) over the shipping address name.

M-002: Data Availability Safeguard

Risk: Broken UI due to null user objects.

Mitigation: Utilize optional chaining and fallback strings ("Unknown Customer") to ensure the UI remains functional even if data is missing.

M-003: Privacy and Compliance

Risk: Unauthorized data exposure.

Mitigation: Restrict order list visibility to authenticated store owners and admins via RBAC (Role-Based Access Control) as implemented in the API gateway.
