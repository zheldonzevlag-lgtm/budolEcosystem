Release Overview: Customer Name Display Fix

This release addresses a critical UI mismatch where the customer name in the order list was incorrectly displaying the shipping address name instead of the account holder's name.

Purpose

To ensure that store owners and administrators can correctly identify the purchasing customer at a glance, maintaining data integrity and improving the user experience for order management.

Rationale

The system previously prioritized the name field from the shipping address. In many cases, users might ship items to others (e.g., Jon Galvez) while the account belongs to someone else (e.g., Natasha Romanoff). This led to confusion in the administrative dashboard.
