Developer Manual

Technical details regarding the implementation of the Customer Name Display fix.

Technical Explanation

The system uses Prisma ORM to fetch order data. The `order` object contains a `userId` and an `addressId`. While the `user` relation contains the account details, the `address` relation contains the shipping details (which includes a `name` field). The frontend was incorrectly using `order.address.name` for the primary customer identification.

Changes Implementation

1. Frontend Component Modification

The OrderItem component was updated to prioritize order.user.name. If the user object is not available, it falls back to the address name, and finally to "Unknown Customer".

<td className="text-left max-md:hidden px-4 py-2"> <p className="font-medium text-slate-700">{order.user?.name || order.address?.name || 'Unknown Customer'}</p> <div className="text-slate-500 text-sm mt-1"> <p className="text-[10px] font-bold text-slate-400 uppercase mb-1">Shipping To:</p> <p className="font-medium text-slate-600">{order.address?.name}</p> <p>{order.address?.street},</p> <p>{order.address?.city}, {order.address?.state}, {order.address?.zip}, {order.address?.country}</p> <p>{order.address?.phone}</p> </div> </td>

2. Data Inclusion Verification

Verified that ordersService.js correctly includes the user relation in the Prisma query to ensure the data is available to the component.

const order = await prisma.order.findUnique({ where: { id: orderId }, include: { user: { select: { id: true, name: true, email: true, image: true } }, // ... other includes } });

Compliance

  • NPC (National Privacy Commission): Ensure that user names are handled securely and only displayed to authorized store owners.
  • Cybersecurity Law: Data integrity is maintained by correctly mapping orders to account holders.
