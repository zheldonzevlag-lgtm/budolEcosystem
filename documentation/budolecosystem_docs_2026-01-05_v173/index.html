Map Integration & Address Pinning

v173

Purpose & Rationale

To align BudolPay with ecosystem standards (matching BudolShap), we implemented a dynamic map integration system. This enables administrators to configure multiple map providers and allows users to precisely pin their physical address during the KYC verification process, improving data accuracy and user experience.

Technical Implementation Details

Admin Console

  • Created /settings/maps configuration interface.
  • Implemented dynamic provider switching (OSM, Geoapify, Google, Radar).
  • Added API credentials management for geospatial services.
  • Exposed settings via /api/system/settings.

Mobile App

  • Integrated flutter_map (Leaflet) and latlong2.
  • Implemented MapPicker widget with current location support.
  • Enabled dynamic tile URL generation based on admin settings.
  • Integrated picker into KYCVerificationScreen.

Risks & Mitigations

Risk: Dependency on external map tile servers (OSM/Geoapify).

Mitigation: Implemented multiple provider support and default fallback to OpenStreetMap (no API key required).

Future Recommendations

  • Implement reverse geocoding to automatically fill address text fields from coordinates.
  • Add Google Maps native SDK support for better performance on high-end devices.
  • Cache map tiles locally to improve offline performance.
