Planning Phase

Compliance-Grade Profile Management (v5.5.0)

Standardizing Admin-Staff workflows for BSP, NPC, and PCI DSS Compliance.

Mandatory Control: Maker-Checker (SoD)
To prevent Insider Threats and satisfy BSP Circular 808, administrative changes to sensitive PII (Phone/Email) will now require dual-approval (Four-Eyes Principle).

Phase 1: Architecture & Data Contracts

Maker-Checker Schema Design

Design a ChangeRequest state machine to track proposed vs. actual identity values.

Dual-Channel Verification Logic

Define verification protocols for both the Old and New contact channels during a change.

Phase 2: Backend Implementation

API Gating & Request Interception

Modify /api/employees to intercept direct writes and generate PENDING requests instead.

Approval Authorization Guard

Implement logic ensuring MakerId != CheckerId to enforce Segregation of Duties.

Phase 3: Frontend Implementation

Change Request Modal

Replace direct edit forms with "Submit Change Request" summaries for sensitive fields.

Administrative Approval Queue

A new dashboard for high-level admins to audit and finalize pending workforce changes.

Compliance Warning: Direct database modification of User table by administrative accounts is strictly audited and must be mitigated by this systemic control.
