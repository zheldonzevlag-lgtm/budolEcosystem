Release v5.5.0 Compliance Verified

Maker-Checker & Compliance Integration

Regulated workflow orchestration for high-security administrative profile management.

Release Summary

This release (v5.5.0) marks the transition of the BudolPay Internal Workforce Registry from an administrative convenience model to a Regulatory-Grade Compliance Model. In adherence to BSP Circular 808 and NPC Data Privacy requirements, we have implemented a strict Maker-Checker (Four-Eyes Principle) workflow for all sensitive profile modifications.

Key Objectives

  • Segregation of Duties (SoD) for staff edits
  • Forensic-ready Change Request tracking
  • Automated MFA-gated change initiation

Compliance Coverage

  • BSP Circular 808/943 (SoD)
  • PCI DSS v4.0 (Req 8.2.1, 10.2.2)
  • NPC DPA 2012 (Data Integrity)

Workflow Architecture

The Maker-Checker system decouples field modifications from the primary database state using a transit-state model.

[Maker] -> UPDATE_PROFILE -> [ChangeRequest Store (PENDING)]
[Checker] -> GET_CHANGE_REQUESTS -> [Review Queue]
[Checker] -> APPROVE -> [User Table Updated]
