System Admin Manual: Compliance Controls

Operational instructions for maintaining the Maker-Checker workflow and environment security.

1. Database Maintenance

The ChangeRequest table should be purged periodically (6 months default) after being archived for forensic review. Ensure backups include the latest schema migrations.

2. Monitoring & Auditing

Monitor the AuditLog for CHANGE_REQUEST_CREATED and CHANGE_REQUEST_APPROVED actions. Any mismatch in IP addresses or frequent rejections should trigger a security incident review.

3. Role Management

Only users with the ADMIN role are authorized to act as Checkers. STAFF roles are typically restricted to being Makers, though ADMIN can also act as a Maker for other Admin profiles (subject to SoD).

4. Disaster Recovery

In the event of a database corruption, follow the standard recovery procedure. The Maker-Checker state is preserved in the ChangeRequest table, which can be used to re-verify any pending changes after a restore.
