Issues & Fixes Log (v5.5.0)

Issue: Infinite Metadata Re-fetches in budolShap Shop

Observed: The client was re-mounting and re-fetching the entire product list every 5 seconds, causing performance degradation and potential API rate-limiting.

Fix: Identification of unstable SWR key in app/(public)/shop/page.jsx using Date.now(). Replaced with a static key for idempotent caching.

Issue: Prisma Schema Visibility Fix

Observed: The .gitignore rules were overly restrictive, preventing developer access and migration generation for schema.prisma.

Fix: Corrected the .gitignore glob pattern to allow specific exceptions for database schema files while maintaining repository hygiene.

Issue: Self-Approval of Role Escalation

Observed: Initial implementation allowed an Admin to approve their own Change Request after identity verification.

Fix: Added server-side validation logic in APPROVE_CHANGE_REQUEST action to explicitly block makerId === checkerId.
