Implementation Plan: GoTyme-Style Login

Objective

Implement an automatic, zero-tap biometric login trigger that activates as soon as the user is prompted for their PIN.

Technical Steps

1. State Management

Add a _hasAutoPrompted flag to login_screen.dart to track if the automatic prompt has already been shown in the current session.

2. Auto-Trigger Logic

Create a method _maybeTriggerAutoBiometrics() that checks if the user is on the PIN step and biometrics are enabled.


void _maybeTriggerAutoBiometrics() {
  if (_currentStep == LoginStep.pin && 
      _isBiometricEnabled && 
      !_hasAutoPrompted) {
    _hasAutoPrompted = true;
    Future.delayed(Duration(milliseconds: 500), () {
      _handleFaceRecognitionLogin();
    });
  }
}
3. Lifecycle Integration

Call the auto-trigger logic during step transitions (Identification, OTP verification) and after biometric availability checks are complete.

Rationale

This mimics the GoTyme experience by anticipating the user's intent. Since 90% of users who enable biometrics prefer using them over PIN, forcing a manual tap is an unnecessary hurdle.
