2. Admin Auth Header Standardization

To resolve data visibility issues (empty lists for users, stores, payouts, etc.), a standardized authentication pattern was implemented across all admin dashboard pages. This ensures that every fetch request includes the required JWT token and credentials.

Rationale: The backend requires explicit Authorization headers and cookie inclusion for session validation in production environments.

Implementation Pattern:

const token = typeof window !== 'undefined' ? localStorage.getItem('adminToken') : null;
const response = await fetch('/api/admin/resource', {
    credentials: 'include',
    headers: {
        'Authorization': `Bearer ${token}`,
        'Content-Type': 'application/json'
    }
});

Affected Files:

  • admin/users/page.jsx
  • admin/stores/page.jsx
  • admin/payouts/page.jsx
  • admin/analytics/page.jsx
  • admin/settings/environment/page.jsx
  • admin/error-tracking/page.jsx

7. Software Dependencies & Rationale (Phase 8)

The following software and libraries were chosen for this release based on security, performance, and compliance requirements:

  • Next.js (v16.0.7): Chosen for its robust routing, SSR capabilities, and seamless integration with our API gateway.
  • Prisma ORM (v6.19.0): Provides type-safe database access, critical for maintaining data integrity in KYC records.
  • Lucide React: Used for consistent, accessible iconography across the admin dashboard.
  • React Hot Toast: Provides immediate user feedback for critical actions like KYC approval/rejection.
  • Jest (v30.2.0): Industry standard for unit testing, ensuring new RBAC logic doesn't introduce regressions.

5. Compliance Verification

Every change in v1.5.0 has been audited against the following standards:

  • BSP Circular 808: Adherence to IT Risk Management standards for electronic banking services.
  • PCI DSS 4.0: Secure handling of user identification data.
  • NPC Data Privacy Act: Ensuring KYC documents are handled with strict confidentiality and audit logging.
