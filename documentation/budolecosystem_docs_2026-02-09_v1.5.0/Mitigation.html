Mitigation Plan - v1.5.0

R-1.5.0-01: Unauthorized KYC Approval

Mitigation: Implemented requireAdmin(request, PERMISSIONS.USERS_KYC_APPROVE) middleware on all KYC write endpoints. Updated verifyAdminAccess to strictly check for specific roles and permissions.

R-1.5.0-02: Document Rendering Issues

Mitigation: Used next/image with appropriate loader configurations and object-contain styling to ensure full visibility of submitted identity documents without cropping.

R-1.5.0-03: Audit Log Failure

Mitigation: Wrapped KYC status updates and audit log creation in a robust try-catch block. The system ensures that if the audit log creation fails, the transaction details are logged to server-side console for manual recovery.

R-1.5.0-04: Inconsistent Auth Headers

Mitigation: Enforced a standardized fetch pattern across all admin dashboard components. Added centralized JWT token retrieval and automated injection of Authorization and credentials headers in all API calls.
