Resolved Fixes - v1.5.0

1. Prisma Named Export Error

Issue: Build failure: @/lib/prisma does not contain a default export.

Fix: Updated app/api/user/kyc/route.js to use import { prisma } from '@/lib/prisma' instead of a default import.

2. Turbopack Duplicate Variable Definition

Issue: Build failure in OrderSummary.jsx: finalAmount defined multiple times.

Fix: Removed redundant finalAmount re-declarations; consolidated calculations using existing state variables.

3. Admin Sidebar Submenu Truncation

Issue: New KYC menu item was partially hidden due to max-height constraints.

Fix: Increased max-h-40 to max-h-60 in AdminSidebar.jsx to accommodate expanded membership menu.

4. Missing Authentication in Admin Dashboard

Issue: Empty data lists (users, stores, analytics) in admin dashboard due to missing JWT token in fetch calls.

Fix: Standardized Authorization headers and credentials: 'include' across all 12+ admin dashboard pages.

5. KYC API Role-Based Access

Issue: KYC visibility restricted to full ADMIN only; MANAGER and LEAD roles lacked access.

Fix: Extended requireAdmin middleware and rbac.js to grant USERS_KYC_APPROVE permission to MANAGER and LEAD roles.

6. Prisma Include Scalar Field Error

Issue: Error: Invalid scalar field kycDetails for include statements on model User.

Fix: Removed kycDetails: true and kycStatus: true from adminUserInclude in app/api/admin/users/utils.js. These are scalar fields and should not be in the include block.
