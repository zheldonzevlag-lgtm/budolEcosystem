1. Current Implementation Audit

Implemented

OS-Level Biometric Login

The app currently supports facial recognition via the device's native operating system (Face ID on iOS, Face Unlock on Android).

  • Service: biometric_service.dart
  • Method: authenticate() using the local_auth package.
  • Logic: The OS handles the face scan. If successful, the app retrieves a securely stored PIN to complete the login.
Implemented

KYC Face Detection

Basic face detection is used during the KYC (Know Your Customer) onboarding process to ensure a face is present in the selfie.

  • Screen: kyc_verification_screen.dart
  • Library: google_mlkit_face_detection
  • Function: _processImageML() verifies faces.isNotEmpty before allowing document upload.

2. Missing Features (Gap Analysis)

The following features mentioned in previous documentation (v338/v339) were found to be missing from the current codebase:

  • Liveness Detection: No code exists for blink, smile, or head movement detection to prevent photo spoofing.
  • Server-Side Matching: No implementation for generating face embeddings or performing server-side template matching.
  • Face Templates: The faceTemplate database synchronization logic is not yet integrated into the mobile client.

3. Implementation Roadmap (Strategy Plan)

To fully implement advanced Face Recognition as described in the documentation, we will follow this phase-based approach:

Phase 1: Enhanced Liveness Detection

Implement ML Kit classification to verify "liveness" during KYC and Face Login.

  • Enable FaceDetectorOptions(enableClassification: true).
  • Add logic to check face.smilingProbability and face.leftEyeOpenProbability.
  • Create a UI overlay with prompts: "Please Blink" and "Please Smile".

Phase 2: Face Embedding Generation

Convert captured face images into secure mathematical templates (embeddings).

  • Integrate tflite_flutter with a pre-trained FaceNet model.
  • Implement a service to crop the face and generate a 128-dimension vector.

Phase 3: Backend Integration & Compliance

Securely store and compare face templates on the server.

  • Create /api/v1/auth/face-enroll endpoint.
  • Implement Cosine Similarity matching in the backend.
  • Ensure templates are encrypted and compliant with NPC/DTI regulations.

Phase 4: Session Binding

Bind the authenticated session to the biometric identity for high-value transactions.

4. Risk Assessment

  • Risk: High false rejection rate in low light. Mitigation: Implement automatic screen brightness increase during capture.
  • Risk: Biometric data theft. Mitigation: Store only embeddings, never raw images. Use hardware-backed keystore for local tokens.
