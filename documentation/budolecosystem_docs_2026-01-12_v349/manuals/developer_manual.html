Developer Manual - v349

1. Splash Screen Compatibility Fix

On Web platforms, FlutterNativeSplash.preserve() was found to potentially hang the application before the first frame could be drawn. This version removes the preservation logic in favor of a manual removal trigger.

File: main.dart

void main() async {
  // Reverted to standard initialization to avoid Web hangs
  WidgetsFlutterBinding.ensureInitialized();
  // ...
}

2. Manual JavaScript Trigger

To ensure the HTML-based splash screen in index.html is removed, a manual call to the global removeSplashFromWeb function was added to both SplashScreen and LoginScreen.

File: splash_screen.dart

if (kIsWeb) {
  try {
    js.context.callMethod('removeSplashFromWeb');
  } catch (e) {
    debugPrint('Failed to call removeSplashFromWeb: $e');
  }
}

3. HTML Fallback Timeout

A safety timeout was added to index.html to ensure the splash screen is removed even if the Flutter application fails to initialize or call the removal function.

File: index.html

// Fallback: Remove splash after 10 seconds if Flutter doesn't
setTimeout(removeSplashFromWeb, 10000);
