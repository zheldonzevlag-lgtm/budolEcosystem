Forced Profile Refresh

This update ensures that the "Edit Profile" screen always displays the latest data from the server, addressing issues with stale cache or incomplete login data.

Implementation Details

  1. Exposed `fetchUserProfile`: Made `ApiService.fetchUserProfile` public to allow manual triggering.
  2. Force Refresh on Init: In `EditProfileScreen.initState`, we now call `fetchUserProfile` via `addPostFrameCallback`.
  3. Granular Field Updates: Improved `didChangeDependencies` to update fields individually as data arrives, while respecting user input (preventing overwrites if the user has started typing).
  4. PII Masking: Implemented `obscureText` toggle for First/Last Name fields to enhance privacy in public spaces. Defaults to `true` (masked).

Dirty Flag Logic for Data Overwrites

To balance between "fresh data" and "user input", we introduced dirty flags for each editable field.

  • Initialization: Flags start as false.
  • User Input: Typing in a field sets its flag to true.
  • Server Update: Incoming data from `fetchUserProfile` only updates a field if its flag is false.
  • Masked Data Correction: This ensures that if the initial state contained masked data (e.g. "M*****"), the fresh unmasked data from the server will correctly overwrite it, unless the user has already started editing.

Backend: Universal Sync & Unmasked Data

We consolidated the `/verify` endpoint logic to ensure robust user synchronization and data visibility.

Key Features

  • Universal User Lookup: Supports finding users by ID (standard) or Email (fallback).
  • Auto-Creation (Sync): If a valid SSO token is presented but the user is missing in BudolPay, the system automatically creates the user record.
  • Unmasked PII Return: The endpoint now explicitly returns unmasked First Name, Last Name, Phone, and Email to support the "Edit Profile" feature, replacing previous masked responses (e.g., "M******").
