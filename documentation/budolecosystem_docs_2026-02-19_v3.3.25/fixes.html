Issue ID Description Resolution
FIX-DATA-001 Edit Profile screen shows empty email even if it exists in DB (stale cache). Forced profile refresh on screen load to fetch latest data from server.
FIX-UX-003 Potential for overwriting user input if data loads late. Added checks to ensure fields are only auto-filled if they are empty and not yet initialized.
FIX-UX-005 Privacy concern: PII (Names) visible by default in public/shared environments. Implemented default masking (dots) for First/Last Name with a toggle to unmask.
FIX-DATA-002 Edit Profile screen retains masked data (e.g. "M*****") even after fetching unmasked data. Implemented dirty-flag tracking to allow overwriting non-edited fields with fresh unmasked data from server.
FIX-AUTH-003 Persistent Masking Issue (Asterisks) in Edit Profile due to duplicate endpoint and maskPII logic. Consolidated `/verify` endpoint to ensure unmasked data return and Universal Sync logic.
