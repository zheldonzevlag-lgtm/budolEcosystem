Admin Menu Fixes & RBAC Synchronization

Status: Completed & Verified

Version: v303.0.0 | Date: 2026-01-10

Purpose

Resolve critical data visibility issues in the Admin Dashboard and ensure secure, role-based access control (RBAC) synchronization across the BudolShap ecosystem. This update fixes broken API endpoints, resolves module resolution conflicts, and updates the frontend to use secure admin-specific routes.

Technical Details & Fixes

Component Issue Fix Applied
Module System ESM/CommonJS conflict in env-manager.js Renamed to env-manager.cjs and updated package.json.
API Routes Missing /api/admin/orders and /api/admin/stores routes Created comprehensive admin routes with requireAdmin middleware.
Dynamic Routing Slug name conflict (id vs orderId) Unified all order-related dynamic routes to use [orderId].
Frontend Pages Admin pages using non-admin API endpoints Refactored AdminStores, AdminOrders, AdminApprove, and AdminProducts to use secure routes.
Prisma Client Database provider mismatch warnings Updated prisma.js with stricter environment validation.

Code Implementation

Example of the new requireAdmin middleware enforcement in app/api/admin/orders/[orderId]/route.js:

export async function PATCH(request, { params }) {
    const { authorized, errorResponse } = await requireAdmin(request)
    if (!authorized) return errorResponse

    try {
        const { orderId } = await params
        const body = await request.json()
        const order = await updateOrderInService(orderId, body)
        return NextResponse.json(order)
    } catch (error) {
        return NextResponse.json({ error: 'Failed to update order' }, { status: 500 })
    }
}
