Version 3.3.18 - Forensic Audit Logging & Webhook History

Purpose & Rationale

This release implements a comprehensive forensic audit logging system and webhook history tracking. This ensures that all critical events (Orders, Payments, Returns, Shipping) are logged in a tamper-proof manner with granular metadata, satisfying compliance requirements for PCI DSS and internal audit standards.

Technical Summary

  • Forensic Audit Logging: Integrated createAuditLog utility across ordersService, returnsService, and all payment/shipping webhook handlers.
  • Webhook History: Implemented WebhookEvent logging for PayMongo, BudolPay, and Lalamove to track all incoming webhook payloads, statuses, and errors.
  • Realtime Visibility: Audit logs are triggered in realtime, enabling immediate admin oversight of critical transactions.
  • Metadata Enrichment: Logs now include detailed metadata (Payment IDs, Driver Status, Refund Amounts, etc.) for deeper forensic analysis.
