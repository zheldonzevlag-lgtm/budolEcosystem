Mitigation Plan: Audit Logging Implementation

R-AUDIT-01: Log Volume Overload

Mitigation Strategy: Implement data retention policy and archival.

Action Items:

  • Configure automated archival of logs older than 1 year to cold storage (e.g., S3 Glacier).
  • Monitor table size weekly and adjust database resources if needed.
  • Implement rate limiting on webhook endpoints to prevent DDoS-based log flooding.

R-AUDIT-02: Sensitive Data Leakage

Mitigation Strategy: Data sanitization and review.

Action Items:

  • Ensure `createAuditLog` and webhook handlers do not log full credit card numbers or passwords.
  • Review all `metadata` objects during code review to ensure PII is minimized (use IDs instead of names/phones where possible).
  • Implement automated scanning for PII patterns in logs (future enhancement).

R-WEBHOOK-01: Webhook Replay Attacks

Mitigation Strategy: Idempotency and signature verification.

Action Items:

  • Enforce strict signature verification for all webhook providers (PayMongo, Lalamove).
  • Use `idempotencyKey` or check existing transaction status before processing events (e.g., if Order is already PAID, ignore PAYMENT_SUCCESS).
  • Log all attempts in `WebhookEvent` for forensic analysis.
