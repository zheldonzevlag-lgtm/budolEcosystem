System Admin Manual: Monitoring & Audit Logs

Overview

The system now includes comprehensive forensic logging for all critical operations. Administrators can use these logs to investigate issues, verify transactions, and ensure compliance.

Accessing Logs

Audit logs are stored in the AuditLog table and can be accessed via the Admin Dashboard under Settings > Audit Logs.

Webhook history is stored in the WebhookEvent table and can be viewed under Settings > Webhooks.

Interpreting Logs

  • Entity: The object affected (e.g., Order, User, Payment).
  • Action: What happened (e.g., ORDER_CREATED, PAYMENT_SUCCESS).
  • Details: Human-readable description of the event.
  • Metadata: JSON object containing technical details like IDs, amounts, and previous states.
  • Hash: SHA-256 hash used to verify the log has not been tampered with.

Troubleshooting Webhooks

If a webhook fails (e.g., payment not reflected), check the Webhook History:

  1. Look for entries with status FAILED or PENDING.
  2. Check the error field for details.
  3. Verify the payload matches the expected format from the provider (PayMongo/Lalamove).
