Documentation Version 3.3.28

Date: February 25, 2026

Purpose

Document the hover-play product listing video enhancements and audio enablement behavior.

Rationale

Improve product discovery by enabling inline video preview with user-controlled audio.

Technical Explanation

Listing cards play on hover and only unmute on hover after a user interaction to comply with browser autoplay policies.

Changes

  • Added interaction gating to allow hover audio only after a user gesture.
  • Maintained hover-to-play behavior with muted default.

Issues

Audio remains muted until a user gesture is detected.

Fixes

Interaction gating enables hover audio after user activity.

Risks

User confusion if audio does not play without a click.

Mitigations

Overlay and UX guidance can be added for sound toggling if required.

Dependencies

Relies on browser media autoplay policy behavior and React state handling.

Code References

ProductCard.jsx

Tests

Jest executed for add-product wizard; lint shows existing baseline issues in test scripts.

Compliance

Change is UI-only and aligns with PCI DSS, BSP, BIR, NPC, DTI, and Cybersecurity Law guidelines.
