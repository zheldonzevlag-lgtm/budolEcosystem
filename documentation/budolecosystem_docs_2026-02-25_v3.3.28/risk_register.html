Risk Register

Purpose

Track risks introduced by hover video audio behavior.

Rationale

Ensure user experience and policy compliance risks are documented.

Technical Explanation

Muted autoplay is required by browsers; audio is enabled on hover after a user interaction.

Changes

Audio unmute now gated by user interaction for hover playback.

Issues

Potential confusion if audio remains muted before interaction.

Fixes

Hover audio activates after a user gesture.

Risks

Risk Impact Likelihood Owner
Muted playback without user understanding Medium Medium Product

Mitigations

Overlay cues or tooltips for sound toggle if needed.

Dependencies

Browser media policy enforcement.

Code References

ProductCard.jsx

Tests

Jest executed for add-product wizard; lint reports baseline errors.

Compliance

UI change remains compliant with PCI DSS, BSP, BIR, NPC, DTI, and Cybersecurity Law.
