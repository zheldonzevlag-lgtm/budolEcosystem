Issues & Fixes Log: v3.3.24

Issue: Missing Required Fields (Barangay)

Description: Users reported getting a "Missing required fields" toast error even when all visible required fields were filled. The issue was traced to the `barangay` field being implicitly required in the backend API and database schema, despite being intended as optional.

Reported By: User Feedback

Date: 2026-02-24

Fix Implemented

Action:

  • Updated Prisma schema (`Address`, `StoreAddress`) to make `barangay` nullable (`String?`).
  • Updated API routes to remove `barangay` from required field checks.
  • Updated Frontend Zod validation schema to mark `barangay` as optional.
  • Replaced generic toast errors with field-specific Zod validation messages.

Status: Resolved

Issue: Prisma Client File Lock (EPERM)

Description: During development, `prisma generate` failed with `EPERM: operation not permitted` when trying to rename `query_engine-windows.dll.node` in `@prisma/client-custom`. This is caused by the file being locked by the running Next.js development server or other node processes.

Reported By: Developer (Internal)

Date: 2026-02-24

Fix Implemented

Action: Changed the Prisma client output directory to `@prisma/client-custom-v4` in `schema.prisma` and updated `lib/prisma.js` to import from the new location. This bypasses the locked file.

Status: Resolved (Workaround)

Issue: Missing Required Fields (Store Address - District/Zip)

Description: When saving a store address, the user encountered a "Missing required fields" toast error. The API required `district` and `zip` fields, but the frontend form did not enforce or provide these fields consistently.

Reported By: User Feedback (Screenshot)

Date: 2026-02-24

Fix Implemented

Action:

  • Updated `StoreAddress` model in Prisma schema to make `district` and `zip` optional (`String?`).
  • Updated Store Address API (`app/api/stores/[storeId]/addresses/route.js`) to remove `district` and `zip` from required validation logic.
  • Verified fix with new test script `verify_store_address_optional_fields.test.js`.

Status: Resolved
