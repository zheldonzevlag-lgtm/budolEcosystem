Mitigation Plan: v3.3.24

Mitigation for Prisma Client File Locking

Risk: `EPERM: operation not permitted` when regenerating Prisma client while dev server is running.

Strategy:

  1. Immediate Action: Change `output` in `schema.prisma` to a new directory (e.g., `@prisma/client-custom-v2`).
  2. Code Update: Update `lib/prisma.js` to import from the new client path.
  3. Process: Developers should stop `npm run dev` before running `prisma generate` if they encounter this issue, or use the versioned path approach.

Mitigation for Missing Barangay Data

Risk: Legacy addresses might have null barangay, or new addresses might be created without it.

Strategy:

  • Frontend: Ensure UI components check for existence of barangay before displaying it to avoid "null" or "undefined" text.
  • Backend: API validation logic updated to strictly treat barangay as optional.
  • Database: Schema updated to allow NULL values.
