FIXED: Persistent Black Screen on Flutter Web Start

Status: FINAL RESOLUTION in v447

Issue: Despite previous timing adjustments, some web clients still experienced a black screen after the splash screen, specifically when the PageView controller failed to attach to the UI before the splash was removed.

Resolution Details:

  • Controller Guard: Explicitly set initialPage: 0 and wrapped timer initialization in addPostFrameCallback.
  • Timing Optimization: Increased safety buffer to 800ms to accommodate slower hardware/browsers.
  • Rendering Physics: Added ClampingScrollPhysics to the PageView to prevent overscroll glitches on web.
  • Constraint Fallbacks: Ensured the Marketing Ads container has default dimensions (393x780) if the browser provides invalid constraints during startup.
  • Syntax Fix: Resolved a missing closing parenthesis in the SizedBox wrapper within marketing_ads_screen.dart.
