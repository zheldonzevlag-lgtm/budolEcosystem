Final Web Rendering Stability Fixes

This version implements a robust state management and navigation sequence to ensure the Marketing Ads screen is always visible on Flutter Web, even with engine-level race conditions.

1. Enhanced PageView Controller Guard

Modified marketing_ads_screen.dart to explicitly initialize the PageController at page 0 and use a post-frame callback to ensure client availability before starting the auto-timer.

// marketing_ads_screen.dart
_pageController = PageController(initialPage: 0);
WidgetsBinding.instance.addPostFrameCallback((_) {
  if (mounted && _pageController.hasClients) {
    _startTimer();
  }
});

2. Optimized Splash Removal Buffer

Increased the web splash removal delay in splash_screen.dart to 800ms. This provides a safer margin for the browser's DOM to sync with the Flutter CanvasKit/HTML engine.

// splash_screen.dart
if (kIsWeb) {
  await Future.delayed(const Duration(milliseconds: 800));
  callJsMethod('removeSplashFromWeb');
  await Future.delayed(const Duration(milliseconds: 200));
}

3. Layout Constraint Fallbacks

Implemented explicit size fallbacks in the LayoutBuilder of the marketing screen to prevent 0x0 rendering when constraints are momentarily null during web engine stabilization.

// marketing_ads_screen.dart
final double width = constraints.maxWidth > 0 ? constraints.maxWidth : 393;
final double height = constraints.maxHeight > 0 ? constraints.maxHeight : 780;
return SizedBox(width: width, height: height, ...);
