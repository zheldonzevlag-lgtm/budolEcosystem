Developer Manual: Environment Isolation & SSO Fixes

1. Purpose

To ensure that all services in the ecosystem can connect to their respective databases without interference from the central runner's environment variables.

2. Technical Details & Code Logic

Database Password Synchronization

We identified a mismatch where some services expected postgres as the password, while the local environment actually requires r00t. We synchronized the following files:

  • budolID-0.1.0/.env.connection.local
  • budolAccounting-0.1.0/.env.connection.local
  • budolpay-0.1.0/.env.connection.local

Refined Environment Isolation

The start-ecosystem.js script now uses a "Clean Spawn" strategy:

// start-ecosystem.js
const cleanEnv = { ...process.env };
delete cleanEnv.DATABASE_URL;
delete cleanEnv.NODE_ENV;
delete cleanEnv.VERCEL;
delete cleanEnv.PORT;

const env = { 
    ...cleanEnv, 
    PORT: app.port.toString(),
    LOCAL_IP: localIP,
    // ... global service URLs
};

spawn(app.command, app.args, { cwd: app.cwd, shell: true, env });

3. Issues & Fixes
