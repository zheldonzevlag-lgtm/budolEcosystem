Documentation v141: SSO Login & Password Fixes

Purpose

This version addresses two critical issues discovered during end-to-end testing of the budolPay mobile login flow: environment variable leakage and database credential mismatches.

Technical Details

  • Issue 1: Prisma Authentication failed error in budolID.
    Cause: The local database password was set to postgres instead of r00t in several .env.connection.local files.
    Fix: Updated all local connection templates to use the correct password (r00t).
  • Issue 2: Prisma Protocol error in Mobile App.
    Cause: Environment variables from the Central Runner were polluting the child process environment, often resulting in empty strings for DATABASE_URL.
    Fix: Implemented "Clean Spawn" in start-ecosystem.js which strips sensitive env vars before launching services, forcing them to use their own .env files.
