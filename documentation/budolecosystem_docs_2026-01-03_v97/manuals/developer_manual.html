Developer Manual

Implementation Details for End-to-End Payment Flow

Overview

This phase focused on verifying the integration between budolShap (the marketplace) and budolPay (the payment gateway). We implemented a standardized payment intent creation and a simulation script for mobile payment confirmation.

Technical Implementation

1. Database Schema Updates

The Transaction model in budolPay was updated to include a metadata field to store context-specific information like orderId and app name.

// budolpay-0.1.0/packages/database/prisma/schema.prisma
model Transaction {
  id           String            @id @default(uuid())
  // ... other fields
  referenceId  String            @unique
  metadata     String?           // Added to store app-specific context
  fee          Decimal           @default(0.0) @db.Decimal(18, 2)
  // ...
}

Payment Simulation Scripts

Two key scripts were developed/refined to test the flow:

  • simulate-order-payment.js: Places an order in budolShap and requests a payment intent from budolPay.
  • simulate-mobile-scan.js: Simulates a user scanning the QRPH code with their mobile app, deducting balance, and triggering the webhook.
