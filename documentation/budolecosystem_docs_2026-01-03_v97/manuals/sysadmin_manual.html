SysAdmin Manual

Ecosystem Configuration and Maintenance

Database Synchronization

During this implementation, database schemas were updated to support metadata and ensure consistency across services. SysAdmins must ensure all databases are in sync with their respective Prisma schemas.

Critical: Always use npx prisma db push --accept-data-loss only in development environments. For production, use migrations.

Databases in Use:

  • budolpay_db: Central database for payments, wallets, and transactions.
  • budolshap_1db: Database for the Marketplace (Products, Orders, Users).
  • budolid_db: Central Identity/SSO database.

Service Ports and Health Checks

The following ports must be open and accessible within the ecosystem:

  • 8000: budolID SSO
  • 3001: budolShap Marketplace
  • 8004: budolPay Payment Gateway
  • 8002: budolPay Wallet Service
  • 8080: budolPay API Gateway

Common Issues & Fixes

1. EADDRINUSE (Port Conflict)

If a service fails to start because the port is in use, use PowerShell to find and stop the process:

Get-Process -Id (Get-NetTCPConnection -LocalPort 8004).OwningProcess | Stop-Process -Force

2. Prisma Client Initialization Error

Ensure the PostgreSQL service is running before starting the ecosystem.
