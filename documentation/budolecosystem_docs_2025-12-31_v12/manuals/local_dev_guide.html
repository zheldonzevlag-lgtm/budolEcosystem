Local Development Guide (Non-Docker)

This guide explains how to run the entire budol Ecosystem locally on Windows without using Docker.

1
Prerequisites

Ensure you have the following installed directly on your Windows machine:

  • Node.js (v18+)
  • PostgreSQL (Installed as a Windows Service)
  • Git Bash or PowerShell
2
Database Setup

Create separate databases in your local PostgreSQL for each app:

CREATE DATABASE budolpay_db; CREATE DATABASE budolshap_db; CREATE DATABASE budolid_db; CREATE DATABASE budoloan_db;
3
Environment Configuration

Update the .env files in each folder to point to your local Postgres instance:

# Example for budolPay DATABASE_URL="postgresql://postgres:password@localhost:5432/budolpay_db?schema=public" # Example for budolID DATABASE_URL="postgresql://postgres:password@localhost:5432/budolid_db?schema=public"
4
Running Multiple Services

Since we aren't using Docker Compose, you need to start each app in a separate terminal window, or use concurrently:

# In the root directory npm install -g concurrently concurrently "npm run dev --prefix budolpay-0.1.0" "npm run dev --prefix budolshap-0.1.0" "node budolID-0.1.0/index.js"
Pro Tip: Use the npm workspaces feature already configured in the repo to run npm install once from the root to link all dependencies across apps.

Port Mapping Summary

  • budolID (SSO): http://localhost:3000
  • budolPay (Admin/Core): http://localhost:3001
  • budolShap (Marketplace): http://localhost:3002
  • budoLoan (Future): http://localhost:3003
