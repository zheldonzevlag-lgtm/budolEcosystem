Authentication Strategy: Headless SSO

Date: 2025-12-31
Topic: Universal vs. App-Specific Authentication UI

Overview

One of the core architectural decisions in the BudolEcosystem is how users authenticate across different applications (budolPay, budolShap, etc.). The dilemma is between a Universal SSO Page (where all apps redirect to a single login site) versus App-Specific Login UIs (where each app has its own native login form).

Recommendation: The "Headless SSO" Approach
We recommend using Centralized Authentication (Backend) with Decentralized UI (Frontend). This means budolID acts as the "Identity Provider" via API, while each application renders its own native login/logout experience.

Comparison

Feature Universal SSO Page App-Specific UI (Headless)
User Experience Redirects user away from the app; generic feel. Native look and feel; stays within the app context.
Branding Consistent with Identity Provider (budolID). Consistent with the specific App (Pay vs. Shap).
Custom Fields Hard to implement app-specific fields. Easy to add fields like "Store ID" or "Dept Code".
Security Centralized and easy to audit. Centralized (via budolID API) and easy to audit.
Complexity Lower (one page for everyone). Medium (each app needs a login form).

Implementation Details

1. Centralized Identity (budolID)

budolID exposes a "Headless" API endpoint for authentication. It doesn't care who renders the form, as long as the apiKey and credentials are valid. 

// POST http://localhost:8000/auth/sso/login
{
  "email": "user@example.com",
  "password": "...",
  "apiKey": "app_specific_key"
}

2. Native App Integration

The application (e.g., budolPay Admin) hosts its own /login page. When the user submits the form, the application's backend acts as a proxy:

  1. App Frontend posts credentials to App Backend.
  2. App Backend calls budolID API.
  3. If successful, App Backend syncs the user profile locally and sets a secure HttpOnly cookie.

Conclusion

The "Headless SSO" strategy provides the flexibility required for a diverse ecosystem like Budol. It allows budolPay Admin to collect administrative metadata while ensuring budolShap remains a fast, consumer-focused shopping experience—all while sharing the same underlying identity account via budolID.
