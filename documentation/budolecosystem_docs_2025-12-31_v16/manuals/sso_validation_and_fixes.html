1. Developer Manual Updates

1.1 Issue Resolution: JWT Malformed Error

Problem: Intermittent "jwt malformed" errors were observed during rapid SSO login attempts (e.g., during automated testing).

Root Cause: budolID generates JWTs based on user data and timestamps (seconds). If two login requests occurred within the same second, the generated token was identical. The Session table in Prisma enforces a unique constraint on the token column, causing the second write to fail with a Unique Constraint Violation. This resulted in a 500 error, returning an undefined token to the client.

Fix Implementation: Added a unique JWT ID (jti) using crypto.randomUUID() to the token payload. This ensures every token is unique, even if generated for the same user at the exact same timestamp. 

// budolID-0.1.0/index.js
const token = jwt.sign(
    { 
        // ... other fields
        jti: require('crypto').randomUUID() // Added for uniqueness
    }, 
    JWT_SECRET, 
    { expiresIn: '7d' }
);

1.2 budolShap Robustness

Updated budolshap-0.1.0/app/api/auth/sso/callback/route.js to explicitly handle string literal "undefined" tokens, which could occur if the upstream provider fails gracefully.

1.3 Automated Validation Script

Created scripts/test_scripts/test-sso-flow.mjs to validate the end-to-end flow. 

node scripts/test_scripts/test-sso-flow.mjs

The script verifies:

  • Service availability (budolID, budolShap, budolPay Admin)
  • Token generation and verification
  • SSO Callback redirects and Session Cookie setting

2. System Admin Manual Updates

2.1 Service Dependencies

Ensure the following services are running for SSO to function:

  • budolID: Port 8000 (SSO Provider)
  • budolShap: Port 3001 (E-commerce)
  • budolPay Admin: Port 3000 (Admin Dashboard)

2.2 Troubleshooting Logs

If SSO fails, check budolID logs for:

  • Unique constraint failed on the fields: (`token`) - (Fixed in v16)
  • [Verify] Error: jwt malformed - Indicates the client is sending an invalid or undefined token.

3. User Manual Updates

No visible changes to the user interface. Login reliability has been improved, preventing occasional "Invalid Token" errors during sign-in.
