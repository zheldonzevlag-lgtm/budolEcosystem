Database & Service Requirements

Ecosystem Database Architecture

The Budol Ecosystem relies on a multi-service architecture where databases are shared or dedicated based on the service domain.

1. PostgreSQL (Relational Data)

ProviderVercel Postgres (Serverless)
UsageCore data for budolpay and budolshap.
ORMPrisma
Env VarsDATABASE_URL, POSTGRES_PRISMA_URL, POSTGRES_URL_NON_POOLING

2. Redis (Cache & Real-time State)

ProviderUpstash Redis / Vercel KV
UsageSystem settings cache, OTP rate limiting, session management.
Env VarsREDIS_URL, REDIS_PASSWORD

3. Vercel Blob (Object Storage)

ProviderVercel Blob
UsageKYC Documents, User Avatars, Verification Assets.
Env VarsBLOB_READ_WRITE_TOKEN

4. Third-Party Integrations

  • Cloudinary: Managed image hosting for budolshap products. (CLOUDINARY_URL)
  • Pusher: WebSockets for real-time UI updates. (PUSHER_APP_ID, PUSHER_KEY, PUSHER_SECRET)
  • Paymongo: Primary payment gateway for transactions. (PAYMONGO_SECRET_KEY)
  • Brevo/Zerix/iTextMo: Notification providers for SMS and Email.
