Release v439

Phase 3: Implementation - Branding Consistency

Ecosystem Standardization: Global budol-* Branding & Payment Return UX Fixes.

Executive Summary

This release completes the standardization of the budol-* ecosystem branding across all platforms (React & Flutter). By replacing hardcoded text and raw span tags with reusable branding components (BudolPayText and UIUtils.formatBudolPayText), we ensure consistent visual identity and color schemes for budolPay, budolShap, and budolCare. Additionally, critical UX improvements were made to the payment return flow and marketing onboarding screens.

Key Changes

  • ✔ Global Branding: Replaced all BUDOL_PAY and unbranded budol-* text with stylized components in 15+ files.
  • ✔ Payment Return UX: Standardized branding in succeeded, failed, and timeout states of the payment return page.
  • ✔ Marketing Ads Branding: Applied UIUtils.formatBudolPayText to slide titles and descriptions in the Flutter mobile app.
  • ✔ Component Stabilization: Fixed import paths and missing dependencies in React components (Footer, QR Page, Return Page).
