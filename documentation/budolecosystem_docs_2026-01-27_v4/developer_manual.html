Developer Manual - v540

Technical Overview

The core of the "Missing Orders" issue was located in the order retrieval logic within the ordersService.js. The service used a flag excludeAbandonedPayments which defaults to true.

Root Cause Analysis

In ordersService.js, the filter was structured as follows:

if (excludeAbandonedPayments) {
    where.AND.push({
        OR: [
            { paymentMethod: 'COD' },
            { isPaid: true },
            { paymentStatus: 'paid' },
            {
                AND: [
                    { paymentStatus: 'awaiting_payment' },
                    { createdAt: { gte: thirtyMinutesAgo } }
                ]
            }
        ]
    });
}

Orders that were paid but had their paymentStatus stuck at awaiting_payment (due to webhook delays or missing sync) and were older than 30 minutes were being filtered out from all UI views.

Implementation Details

1. Webhook Update

Modified budolpay/route.js to ensure paymentStatus is set to 'paid' when a payment is successful.

await updateOrderStatus(orderId, 'processing', {
    paymentStatus: 'paid',
    isPaid: true
});

2. Legacy Data Migration

Created and executed fix-paid-orders-status.mjs to fix 30+ legacy orders that were hidden.

3. Dependency Resolution

Fixed monorepo imports in BudolPay services. Services now correctly reference @budolpay/database and @budolpay/notifications packages.

Dependencies

  • Prisma ORM v6.2.1
  • Node.js v20+
  • Next.js v14.2.3

Compliance Verification

All financial status changes are logged in the AuditLog table, compliant with PCI DSS 10.2.2 and BSP Circular 808.
