Implementation Plan v207: Local Environment Configuration Fix

Date: 2026-01-08 | Status: COMPLETED

Purpose

The purpose of this fix was to resolve an issue where users in a local development environment were being redirected to Vercel-hosted services (e.g., budolid.vercel.app) during the login flow. This was caused by hardcoded production URLs in various .env files and outdated redirect URIs in the EcosystemApp table of the budolid database.

Technical Details

Environment Variable Updates

  • Updated budolshap-0.1.0/.env: Changed NEXT_PUBLIC_SSO_URL, AUTH_SERVICE_URL, and other service URLs from Vercel to localhost.
  • Updated budolpay-0.1.0/.env: Replaced Vercel-hosted service URLs with local service addresses.
  • Updated budolID-0.1.0/.env: Aligned service URLs with the local development ecosystem.

Database Configuration

  • Executed update-redirects-local.js to update the redirectUri for all ecosystem apps (budolPay, budolShap, budolExpress) in the budolid database to point to localhost instead of previous IP addresses or production URLs.

Issues & Fixes

Issue Fix Reason
Redirect to Vercel Login Updated NEXT_PUBLIC_SSO_URL to http://localhost:8000 Frontend was defaulting to production SSO URL.
Post-login Redirection Failure Updated EcosystemApp.redirectUri in DB SSO provider was using outdated or production redirect targets.

Future Recommendations

  • Environment-Specific .env Files: Implement .env.local and .env.production more consistently to avoid accidental overrides or hardcoding.
  • Dynamic Service Discovery: Further enhance start-ecosystem.js to automatically propagate service URLs to all child services, reducing the need for manual .env updates.
