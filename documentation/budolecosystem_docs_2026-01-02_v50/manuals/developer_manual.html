Developer Manual - v50

Release Date: 2026-01-02

Version: v50

Overview

This update resolves the issue where user name changes in the budolShap/budolPay admin panels were being reverted to their original values upon subsequent SSO login. This was caused by a lack of bi-directional synchronization between the application services and the budolID SSO provider.

Technical Changes

1. budolID SSO Provider Enhancements

A new profile update endpoint was added to allow applications to push name changes back to the central identity provider.

  • Endpoint: PATCH /users/:userId/profile
  • File: budolID-0.1.0/index.js
  • Functionality: Updates firstName, lastName, and avatarUrl in the Prisma database.

2. budolShap Client & API Updates

The budolShap application now proactively synchronizes name changes to the SSO provider.

  • Client Update: Added updateUserProfile to budolshap-0.1.0/lib/api/budolIdClient.js.
  • Admin API Update: Modified budolshap-0.1.0/app/api/admin/users/[userId]/route.js to call updateUserProfile when a name is edited.
  • SSO Callback Update: Updated budolshap-0.1.0/app/api/auth/sso/callback/route.js to store ssoUserId in the user's metadata, enabling the link between local and SSO identities.

3. budolPay Admin Sync Updates

Similar to budolShap, budolPay now synchronizes profile updates to budolID.

  • Client Created: budolpay-0.1.0/apps/admin/lib/budolIdClient.ts
  • Admin API Update: Added UPDATE_PROFILE action to budolpay-0.1.0/apps/admin/app/api/users/route.ts with SSO sync logic.
  • SSO Callback Update: Updated budolpay-0.1.0/apps/admin/app/api/auth/callback/route.ts to store ssoUserId in kycData.

4. Checkout & Payment Flow Fixes

Resolved critical errors in the checkout process for BudolPay and PayMongo integrations.

  • Case Sensitivity: Fixed OrderSummary.jsx to handle payment methods in a case-insensitive manner (e.g., "BUDOL_PAY" vs "budolpay").
  • Provider Routing: Ensured budolpay method correctly routes to the budolpay provider instead of defaulting to PayMongo.
  • Port Configuration: Standardized BudolPay Gateway communication on port 8004 to avoid conflicts with budolID on port 8000.

Impact Analysis

Name changes made in the admin panel are now persistent across sessions. The budolID service acts as the source of truth, and applications now have the capability to update this source of truth when administrative changes are made.
