Maintenance Procedures

Resetting Admin Credentials

When resetting admin credentials using the fix scripts, the names will no longer be hardcoded to "BudolShap Admin" or "Budol Admin". They will now default to "Admin User" if the user is being created for the first time.

# Run from budolID-0.1.0 directory
node fix-budolshap-admin.js
node fix-admin-login.js
Important: After running these scripts, the names can be customized via the budolShap Admin Panel (Users section), and these customizations will now persist across logins.

Configuration Requirements

Ensure that the NEXT_PUBLIC_SSO_URL environment variable is correctly set in all application services (budolShap, budolPay) to point to the budolID instance (default: http://localhost:8000).

BudolPay Gateway Port Change

The BudolPay Payment Gateway service has been moved from port 8000 to port 8004 to prevent port conflicts with the budolID SSO service on the same host.

  • New Gateway URL: http://localhost:8004
  • Environment Variable: BUDOLPAY_GATEWAY_URL in budolShap .env must be updated.
Verification: Run netstat -ano | findstr :8004 to confirm the payment gateway is running on the correct port.

Database Schema

This update utilizes the existing metadata JSON field in the budolShap User table and the kycData JSON field in the budolPay User table to store the ssoUserId. No migrations are required.
