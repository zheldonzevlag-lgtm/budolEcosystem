Dependencies & Software Rationale - v1.0.5

Prisma ORM

Use: Database access and schema management for budolID and budolShap.

Why Chosen: Type-safe database client generation and seamless handling of multi-schema environments (public vs budolid). Essential for the Admin ID synchronization script.

Flutter (Mobile)

Use: Cross-platform mobile application development for budolPay.

Why Chosen: High performance, native-like UI, and robust state management (using Provider/Bloc patterns). Allows for rapid UI updates like the Order # display fix.

Node.js / Express

Use: Backend microservices (wallet-service).

Why Chosen: Non-blocking I/O model is ideal for high-concurrency payment processing and real-time transaction updates.

PostgreSQL

Use: Relational database for all ecosystem data.

Why Chosen: Strong ACID compliance, support for complex relational data, and mature ecosystem. Critical for financial transaction integrity.
