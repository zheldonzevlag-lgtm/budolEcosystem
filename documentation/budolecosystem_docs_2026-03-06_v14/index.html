BudolEcosystem Rebuild Action Plan (Phase-by-Phase)

This plan outlines the reconstruction of the budolEcosystem infrastructure in the Singapore region following a total teardown. It implements the "Monolithic Independent Apps" strategy, migrating all apps to a microservices architecture with isolated databases and standardized APIs.

IMPORTANT: This plan adopts the Parallel Implementation Approach from version 2.0 of the project history. All core services (budolID, budolPay-0.1.0, budolPayMobile, budolAccounting, budolShap, budolLoan) will be provisioned and configured simultaneously to reduce timeline.
TIP: We will utilize the AWS Free Tier Deployment Plan (t3.micro, RDS db.t3.micro) to keep initial costs between $40-$60/month while maintaining professional standards.

Phase 1: Foundation & Infrastructure (Parallel)

Goal: Establish the infrastructure for all independent services.

  • NEW Networking: Provision a clean VPC with public/private subnets and tailored Security Groups.
  • NEW Databases: Provision 5 isolated RDS PostgreSQL (db.t3.micro) instances for budolID, budolPay, budolAccounting, budolShap, and budolLoan.
  • NEW Redis: Provision ElastiCache (cache.t3.micro) for each service suite.
  • NEW Secrets: Re-inject clean credentials and DATABASE_URL strings into AWS Secrets Manager.

Phase 2: Database Restoration & Independence

Goal: Restore data from local backups (2026-03-05) into the new isolated databases.

  • RESTORE RESTORE: Execute pg_restore from local backups into the 5 fresh RDS instances.
  • CONFIG Prisma: Update and generate Prisma clients for each service to point to their dedicated DBs.

Phase 3: API Integration Framework

Goal: Set up standardized communication between independently running apps.

  • NEW Gateway: Provision a central API Gateway to route traffic to individual service endpoints.
  • NEW DNS: Configure Route53 internal DNS (e.g., id.budol.internal, pay.budol.internal) for service discovery.

Phase 4: Service Refactoring & Deployment

Goal: Deploy the refactored services into the new environment.

  • NEW ECS: Create a fresh ECS Cluster and Task Definitions (v27+) for each service.
  • NEW ALB: Provision a new Application Load Balancer with independent Target Groups for each app.

Phase 5: Testing & Verification

Goal: Ensure 100% connectivity and data integrity.

  • TEST Connectivity: Verify each service can talk to its dedicated DB and the SSO provider (budolID).
  • TEST E2E: Validate product visibility on budolshap.duckdns.org and login functionality.

Phase 6: Documentation & Audit

Goal: Finalize the v14 documentation release.

  • DOC Manuals: Update Developer and System Admin manuals with the new isolated architecture.
  • DOC Walkthrough: Log the "Total Reset & Rebuild" success in the project knowledge base.

Verification Plan

Automated Tests

  • Run npm run test in each service directory to verify Prisma client generation and basic health checks.
  • Use aws elbv2 describe-target-health to ensure all 10 target groups show Healthy status.

Manual Verification

  • Access https://budolshap.duckdns.org and verify product categories and search functionality.
  • Perform a test login via budolID to confirm SSO integration with independent databases.
