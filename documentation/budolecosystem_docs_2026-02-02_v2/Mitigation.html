Mitigation Plan - v2.1.4

M-005: Compliance-Controlled User Management

PCI DSS/BSP Guard: Admins are restricted from viewing or setting user passwords directly. Only "Force Reset" with system-generated temporary credentials is allowed.

NPC Guard: All profile changes are logged with Old vs New values to prevent undetected administrative account takeover.

Data Integrity: Soft-delete (Deactivation) is enforced to preserve the integrity of the transaction ledger and audit logs.

M-007: Administrative Self-Action Restrictions NEW

Risk: Admin Account Takeover (Attacker uses admin session to reset own password and lock out legitimate owner).

Mitigation: Enforce server-side and client-side checks to prevent administrators from resetting their own passwords or deactivating their own accounts via the Administrative Interface.

Compliance: PCI DSS 8.1.2, BSP Dual Control Principles.

M-006: Zero-Knowledge Password Delivery NEW

Risk: Admin Account Takeover (Admin resets user password and logs in as user).

Mitigation: System sends temporary credentials directly to user via verified out-of-band channels (SMS/Email). The Administrative Interface is restricted from viewing the clear-text password after generation.

Compliance: BSP Circular 808 (Authentication Management), PCI DSS 8.2.4.

S-001: Automated SSO Health Checks

Implement a watchdog service that pings the /auth/sso/login endpoint every 5 minutes and alerts administrators if the API key validation fails.

S-002: Build Artifact Signing

Ensure all APKs in the android-installer directory are cryptographically signed and their SHA-256 hashes are published in the test_results.html.
