System Admin Manual - Transaction Infrastructure

Purpose

This manual covers the infrastructure requirements for the Transaction Service and the real-time notification gateway, including port assignments and database synchronization.

Infrastructure Setup

1. Service Port Mappings

Service Name Port Health Check Endpoint Status
SSO (budolID) 8000 /health ONLINE
API Gateway 8080 /health ONLINE
Transaction Service 8003 /health ONLINE
Wallet Service 8002 /health ONLINE

2. Database Configuration

The ecosystem uses two distinct PostgreSQL databases:

  • budolid: Stores master user credentials and profiles.
  • budolpay: Stores financial records, wallets, and transactions.
DATABASE_URL="postgresql://postgres:r00t@localhost:5432/budolpay?schema=public"

Deployment Procedures

Central Runner Usage

Admins should use the central runner to ensure all services start with the correct environment variables:

npm start

This command launches the start-ecosystem.js script which automatically detects the local IP and configures the microservices to communicate through the API Gateway.

Monitoring & Logs

Real-time transaction monitoring is available via the Admin Dashboard. If notifications fail to appear, verify that the API Gateway (8080) is accessible and that the Socket.io server is accepting connections.

Rollback Strategy

In case of transaction failures, the system implements a Double-Entry Ledger Rollback. If the credit to a receiver fails, the debit from the sender is automatically reversed in the wallet-service before the transaction status is set to FAILED.
