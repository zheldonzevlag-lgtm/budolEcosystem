Developer Manual - v284

Purpose

This version focuses on verifying the end-to-end SSO flow across the entire ecosystem in a LAN environment and ensuring consistent data synchronization between services.

SSO Flow Verification

Comprehensive testing was conducted to ensure that budolID correctly authenticates users and redirects them to the respective applications (budolShap and budolPay Admin) with valid session tokens.

Technical Details

  • JWT Consistency: Verified that JWT_SECRET is standardized across budolID, budolShap, and budolPay Admin.
  • Token Verification: Confirmed that the /auth/verify endpoint in budolID correctly validates tokens and includes the associated app and user data.
  • Cookie Management: Verified that session cookies (budolshap_token, budolpay_token) are correctly set with SameSite=Lax and appropriate expiration times for LAN accessibility.

Prisma Client Synchronization

Addressed issues where test scripts were using mismatched Prisma Client instances. Developers should always use the centralized client from lib/prisma.js to ensure schema consistency.
