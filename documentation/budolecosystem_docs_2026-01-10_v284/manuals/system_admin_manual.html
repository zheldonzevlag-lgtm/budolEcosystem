System Admin Manual - v284

Payment Gateway Configuration

Verified that the payment-gateway-service correctly uses the LOCAL_IP for internal checkout URLs, ensuring that users on the LAN can access the payment interface.

Internal Checkout URLs

The service generates checkout URLs based on the BASE_URL environment variable or defaults to http://${LOCAL_IP}:${PORT}. For LAN environments, ensure LOCAL_IP is set to the host machine's IP (e.g., 192.168.1.24).

SSO Application Seeding

Consolidated the seeding process for LAN environments. Admins should run the following command to ensure all applications have the correct redirect URIs:

$env:LOCAL_IP="192.168.1.24"; node packages/database/seed_lan.js

Service Connectivity Matrix

  • budolID (8000): Identity Provider & Token Issuer.
  • budolShap (3001): Consumer App (Buyer/Seller).
  • budolPay Admin (3000): Management Dashboard.
  • Payment Gateway (8004): Transaction Processor.

Troubleshooting

  • 401 Unauthorized: Ensure JWT_SECRET matches across all services and that LOCAL_IP is correctly set in .env files.
  • Connection Refused: Verify that services are listening on 0.0.0.0 and that firewall ports are open.
