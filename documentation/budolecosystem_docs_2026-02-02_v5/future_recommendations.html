Future Recommendations - v2.4.0

← Back to Overview

1. Immutability Layer (Blockchain/WORM)

Currently, audit logs are stored in a standard relational database (PostgreSQL). For high-security environments, we recommend implementing an immutability layer using a WORM (Write Once Read Many) storage or a private blockchain to prevent tampering by database administrators.

2. Offline Log Queue

Implement a background worker (e.g., BullMQ) to handle audit log creation. This will ensure that even if the primary database or SSO service is temporarily unreachable, logs are queued and eventually persisted, preventing gaps in the forensic trail.

3. AI-Driven Anomaly Detection

Integrate a machine learning model to analyze audit logs in real-time and flag suspicious patterns (e.g., multiple password resets followed by a large transaction) before they result in a security breach.

4. Automated BSP Report Generation

Build a dedicated module to automatically format forensic audit data into the specific templates required by the Bangko Sentral ng Pilipinas (BSP) for IT Risk reporting.
