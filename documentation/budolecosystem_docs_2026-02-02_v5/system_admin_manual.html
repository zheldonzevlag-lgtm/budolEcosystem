Forensic Monitoring & Audit Trail

As a System Admin, you are responsible for monitoring the forensic audit trail to ensure system integrity and compliance.

1. Accessing Audit Logs

Audit logs are available in the Security section of the Admin Dashboard. These logs provide a real-time view of all critical actions taken by users and administrators.

2. Interpreting Logs

Each log entry contains:

  • Action: The type of event (e.g., USER_LOGIN, KYC_STATUS_UPDATED).
  • User: The administrator or user who performed the action.
  • IP Address: The network source of the request.
  • Old/New Values: For data changes, the manual displays exactly what was modified.

3. Compliance Reports

The dashboard allows for the generation of compliance reports (e.g., BSP Reports) based on these audit logs. Ensure that all security events are reviewed daily.

Tip: Use the real-time notification feature to get instant alerts on high-risk actions like role changes or dispute resolutions.

4. Security Standards

This system adheres to:

  • PCI DSS 10.2: Automated audit trails for all system components.
  • BSP Circular 808: IT Risk Management for financial institutions.
