Forensic Audit Trail Implementation

The forensic audit trail is built using Prisma and is integrated across all critical API routes in the Admin Dashboard.

1. Audit Log Schema

The AuditLog model in schema.prisma defines the structure for forensic data.

model AuditLog {
  id        String   @id @default(uuid())
  userId    String?
  action    String
  metadata  Json?
  createdAt DateTime @default(now())
  entity    String?
  entityId  String?
  ipAddress String?
  newValue  Json?
  oldValue  Json?
  userAgent String?
}

2. Key Implementation Points

Authentication Events

  • Login: Logged in login/route.ts and callback/route.ts.
  • Logout: Logged in logout/route.ts.

User Management

  • KYC Status: Logged in users/route.ts under UPDATE_KYC_STATUS.
  • Document Rotation: Logged in users/route.ts under UPDATE_DOCUMENT_ROTATION.
  • Profile Updates: Logged in employees/route.ts.

Financial Operations

  • Dispute Resolution: Logged in resolve/route.ts.

3. Real-time Broadcasting

The RealtimeService in realtime.ts has been updated to auto-connect in the browser, enabling live audit log updates in the UI.

4. Compliance Logic

Each audit log entry includes metadata.compliance with references to pci_dss and bsp standards.
