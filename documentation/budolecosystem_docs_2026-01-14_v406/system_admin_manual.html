System Admin Manual: Forensic Audit & Compliance

1. Accessing Audit Logs

Audit logs are available via the Admin Dashboard under the "Security & Logs" section. Administrators can filter logs by User, Action Type, or Date Range.

2. Forensic Integrity Checks

As of v406, the system supports periodic forensic integrity checks. These checks verify that the database audit trail has not been tampered with and that all security-sensitive actions are being recorded.

Compliance Requirement: Per BSP Circular 808, audit logs must be retained for a minimum of 5 years. Ensure that database backups include the AuditLog table and are stored in a secure, off-site location.

3. Identifying Suspicious Activity

Look for the following patterns in the audit logs:

  • Multiple LOGIN_FAILED events for a single user in a short period.
  • ADMIN_USER_UPDATE events originating from unusual IP addresses.
  • Large numbers of FORENSIC_VERIFICATION events triggered outside of scheduled maintenance windows.
