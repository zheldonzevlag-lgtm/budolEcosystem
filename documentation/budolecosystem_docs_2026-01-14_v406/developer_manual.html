Developer Manual: Forensic Audit Trail Implementation

1. AuditLog Data Model

The AuditLog model is the core of our traceability system. It captures not just the action, but also the context (IP, Device, Metadata).

Code Reference: schema.prisma

model AuditLog {
  id        String   @id @default(uuid())
  userId    String
  action    String   // e.g., "LOGIN", "FORENSIC_VERIFICATION"
  ipAddress String?
  device    String?
  userAgent String?
  metadata  Json?    // Structured forensic evidence
  createdAt DateTime @default(now())
  user      User     @relation(fields: [userId], references: [id])
}

2. Forensic Verification Test

To verify the integrity of the audit trail, we use a specialized test script that simulates high-security events.

Test Script: test-forensic-audit-v406.js

const auditEntry = await prisma.auditLog.create({

        action: 'FORENSIC_VERIFICATION',

});

3. JSON Metadata Best Practices

When logging custom events, always include a compliance block in the metadata to map the action to specific regulatory requirements. This simplifies audit reporting for compliance officers.
