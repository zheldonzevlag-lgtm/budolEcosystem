Executive Summary

Version 418 addresses a critical database synchronization issue in the BudolShap application. The SystemSettings table was missing from the local PostgreSQL database, causing the application to fail during initialization. This fix involved force-resetting the database schema to match the Prisma model and verifying the automatic creation of default system settings.

Key Fixes

  • Schema Sync: Performed npx prisma db push --force-reset to synchronize the database with the current schema.
  • Table Restoration: Restored the SystemSettings table and other missing models.
  • Initialization Logic: Verified that getSystemSettings() correctly creates default settings if none exist.
  • Memory Management: Resolved RangeError: Array buffer allocation failed by stabilizing database connections.
