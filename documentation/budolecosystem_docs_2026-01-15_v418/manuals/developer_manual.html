Database Synchronization

The application uses Prisma as its ORM. When the database schema is out of sync with the schema.prisma file, the application may fail to fetch critical data like system settings.

Code References

  • prisma/schema.prisma - Database model definitions.
  • lib/settings.js - Logic for fetching and initializing system settings.

Initialization Logic

// lib/settings.js
export async function getSystemSettings(forceRefresh = false) {
    // ... cache logic ...
    try {
        let settings = await prisma.systemSettings.findUnique({
            where: { id: "default" }
        });

        if (!settings) {
            settings = await prisma.systemSettings.create({
                data: DEFAULT_SETTINGS
            });
        }
        return settings;
    } catch (error) {
        console.error("Failed to fetch system settings:", error);
        return { ...DEFAULT_SETTINGS };
    }
}

Troubleshooting

If you see The table 'public.SystemSettings' does not exist, run:

npx prisma db push --force-reset

This will synchronize the database with the schema. Note that this command will delete all data in the database.
