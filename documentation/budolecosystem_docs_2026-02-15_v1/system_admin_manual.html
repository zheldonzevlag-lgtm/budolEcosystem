System Admin Manual - Forensic Audit Trails (v1.0)

Date: 2026-02-15

1. Accessing the Audit Trails

To view the Forensic Audit Trails:

  1. Log in to the Admin Dashboard using your credentials.
  2. Navigate to Settings in the sidebar.
  3. Click on Forensic Audit Trails.

2. Using the Interface

2.1 Security Compliance Dashboard

At the top of the page, the Security & Compliance dashboard provides a real-time overview of the system's security posture.

  • Compliance Hardening: Verify that all security modules (AI Spam Guard, Encryption, etc.) are Active/Enabled.
  • Threat Mitigation Trend: Monitor the ratio of blocked attempts vs. actual threats over the last 7 days.
  • Vulnerability Scan: Check the "System Integrity" score. Any drop below 100% requires immediate attention.
  • AI Analyst Findings: Review automated insights for potential anomalies that haven't triggered alerts yet.

2.2 Filtering Logs

The interface provides powerful filtering capabilities:

  • Search: Enter keywords to find specific logs by detail, IP, or user email.
  • Status: Filter by SUCCESS, FAILURE, or WARNING to identify potential issues.
  • Date Range: Select start and end dates to investigate incidents within a specific timeframe.

2.2 Understanding Log Details

Each log entry contains:

  • Timestamp: Exact time of the event (Manila Time).
  • Action: The type of event (e.g., USER_LOGIN, ORDER_CREATED).
  • User/Actor: Who performed the action.
  • IP Address: The source IP of the request.
  • Entity: The object affected (e.g., Order #12345).
  • Status: Outcome of the action.

3. Incident Response

Critical Alert: If you see multiple FAILURE logs from the same IP address in a short period, investigate immediately as it may indicate a brute force attack.

Recommended actions for suspicious activity:

  1. Note the IP address and User ID.
  2. Check the Risk Register for predefined mitigation steps.
  3. If necessary, temporarily disable the user account via User Management.
  4. Report the incident to the cybersecurity team.

4. Data Retention

Audit logs are retained indefinitely by default. For compliance with data privacy laws, logs older than 5 years may be archived to cold storage (pending feature).
