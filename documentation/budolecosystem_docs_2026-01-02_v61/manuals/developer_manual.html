Developer Manual - v61

End-to-End Order Flow Validation (BudolPay + Lalamove)

1. Test Automation Script (test-complete-order-flow.js)

The primary test script was overhauled to support the modern ES module system and align with current API endpoints. Key improvements include:

  • ES Module Support: Replaced require with import and added polyfills for __filename and __dirname.
  • Endpoint Correction: Updated BASE_URL to http://localhost:3001 to match the authentication and payment API port.
  • Payload Alignment: Updated the Lalamove quote and booking payloads to include mandatory pickup and delivery contact details, as required by the shippingService.js.

2. BudolPay Adapter Fix (budolpay-adapter.js)

Identified that the BudolPay Gateway sometimes returns the payment identifier as reference instead of id or paymentIntentId. The adapter was updated to handle this fallback, ensuring the payment intent is correctly linked to the order.

// budolpay-adapter.js
paymentIntentId: data.id || data.paymentIntentId || data.reference

3. Lalamove Booking Logic Enhancement

Modified the booking request in the test flow to include the mandatory orderId in the request body, which is critical for tracking and reconciliation in the shippingService.js layer.

// test-complete-order-flow.js
const lalamoveBookPayload = {
orderId: order.id,
quotationId: quote.quoteId,
// ... contact details
};

4. Error Handling and Logging

Enhanced logging across the test flow to provide clearer insights into the Lalamove quote response and booking status, including driver assignment and tracking share links.

Recommendations for Future Enhancement

  • Centralized Test Suite: Create a reusable test library for common flows (Auth -> Cart -> Checkout -> Payment -> Shipping) to reduce boilerplate in individual test scripts.
  • Mock Webhook Server: Develop a dedicated mock server for simulating BudolPay and Lalamove webhooks more reliably during local development.
