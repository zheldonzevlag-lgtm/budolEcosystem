Developer Manual

Technical implementation details for Network Awareness in v480.

Architecture & Data Contracts

The core change involves using the LOCAL_IP environment variable as the source of truth for service discovery within a local network. If not provided, it fallbacks to localhost.

Key Code Reference:

Modified network-util.js to detect and propagate the local IP across all services.

const localIP = networkInterfaces[netName].find(net => net.family === 'IPv4' && !net.internal)?.address; // ... if (localIP) { process.env.LOCAL_IP = localIP; updateEnvFiles(localIP); }

Network Utility Implementation

The network-util.js script now handles:

  • Automatic detection of the primary network interface.
  • Updating .env files in all service subdirectories.
  • Updating the Flutter api_service.dart default value.
  • Updating MOBILE_CONFIG.txt for manual device linking.

Dynamic Test Scripts

Test scripts have been updated to use dynamic hostnames. Example from check-services.js:

const LOCAL_IP = process.env.LOCAL_IP || 'localhost'; const GATEWAY_URL = `http://${LOCAL_IP}:8080`;

Dependencies & Rationale

Node.js ecosystem
  • dotenv: Essential for standardizing environment variable injection across microservices.
  • axios: Chosen for its robust timeout handling and interceptors, critical for cross-network reliability.
  • bonjour-service: Used in the central runner for zero-config service advertising via mDNS.
Flutter ecosystem
  • http: Standard Dart package for API communication.
  • flutter_secure_storage: Used to persist session tokens securely on physical devices.
  • shared_preferences: Used for non-sensitive configuration like custom host overrides.

Pushing for LOCAL_IP reduces friction during team onboarding and cross-device testing, ensuring that developers can focus on features rather than network troubleshooting.
