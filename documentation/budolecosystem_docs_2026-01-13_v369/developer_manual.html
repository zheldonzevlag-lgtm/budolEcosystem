Robust TFLite Inference

This version focuses on the low-level stability of the FaceNet TFLite integration.

1. Input Reshaping

The interpreter's run method is sensitive to the input's shape. We now explicitly reshape the flat Float32List into the 4D tensor expected by FaceNet.

// Correcting shape to [Batch, Height, Width, Channels]
var inputReshaped = input.reshape([1, 160, 160, 3]);
_interpreter!.run(inputReshaped, output);

2. Typed Data Buffers

Using List<double> can sometimes cause issues with TFLite because Dart doubles are 64-bit. We've switched to Float32List for the output buffer to ensure binary compatibility with the model's output layer.

var output = Float32List(1 * 128).reshape([1, 128]);

3. Improved Diagnostics

The settings_screen.dart now includes explicit logs for the face detection and embedding stages, making it easier to identify if a failure is due to detection or inference.
