Developer Manual - Version 149

Purpose

To resolve reported issues with wallet balance not updating on the Android app by enhancing the real-time connectivity layer (Socket.io) and providing robust fallbacks for network-restricted environments.

Technical Implementation Details

1. Mobile Socket.io Enhancements

Modified SocketService.dart to allow polling as a fallback transport. This is critical for mobile devices on cellular networks or corporate Wi-Fi where pure WebSockets might be blocked.

_socket = IO.io(baseUrl, IO.OptionBuilder()
    .setTransports(['websocket', 'polling']) // Fallback enabled
    .enableAutoConnect()
    .setReconnectionAttempts(10)
    .setReconnectionDelay(2000)
    .build());

2. Connection Monitoring

Implemented connection event listeners to propagate connection status to the UI, allowing users and developers to diagnose connectivity issues in real-time.

_socket!.onConnect((_) {
    _isConnected = true;
    _socket!.emit('join', userId);
    notifyListeners();
});

3. UI Fallbacks

Added a RefreshIndicator to the HomeScreen to allow manual balance fetching. This ensures users can still get their balance even if the real-time stream is temporarily interrupted.

Issues & Fixes

  • Issue: Balance not updating in real-time on Android app.
    Root Cause: WebSocket transport occasionally failing in mobile network environments without fallback to polling.
    Fix: Enabled polling transport and increased reconnection attempts.
  • Issue: No visual feedback on connectivity.
    Fix: Added a status indicator dot in the App Bar.
