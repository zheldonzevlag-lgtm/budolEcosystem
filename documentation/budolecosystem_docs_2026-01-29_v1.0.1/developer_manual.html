Technical Implementation - v1.0.1

1. SSO LAN Configuration

The BudolID SSO service was updated to dynamically detect and log the LAN IP for cross-device access. It now listens on 0.0.0.0 to allow external connections while providing a clear LAN URL in the logs.

View: budolID-0.1.0/index.js 
app.listen(PORT, '0.0.0.0', () => {
    const localIP = process.env.LOCAL_IP || '192.168.1.2';
    console.log(`budolID SSO Service running on http://0.0.0.0:${PORT}`);
    console.log(`Local LAN access at http://${localIP}:${PORT}`);
});

Rationale: Hardcoded IPs in redirect URIs cause authentication failures when the server host IP changes. Using environment-aware seeding fixes this.

View: budolID-0.1.0/prisma/seed.js 
const localIP = process.env.LOCAL_IP || '192.168.1.2';
const apps = [
  { name: 'budolPay', apiKey: 'bp_key_2025', redirectUri: `http://${localIP}:3000/api/auth/callback` },
  ...
];

3. Automated Network Sync

The network-util.js utility was implemented to recursively scan all .env files in the ecosystem and update LOCAL_IP and related URL patterns automatically on startup.

Detects physical interfaces (Ethernet/Wi-Fi) and ignores virtual adapters (WSL/Docker).

4. Dependencies & Rationale

Core Stack

  • Node.js & Express: Chosen for its asynchronous nature and vast ecosystem, ideal for microservices.
  • Prisma ORM: Provides type-safe database access and automated migrations, critical for multi-service consistency.
  • PostgreSQL: Robust relational database with strong support for schemas, allowing us to isolate services (e.g., budolid schema) within a shared cluster.
  • JWT (JSON Web Tokens): Stateless authentication mechanism perfect for SSO across different domains/IPs.

Utilities

  • Bonjour Service: Planned for zero-config networking to resolve .local hostnames.
  • Dotenv: Standard for environment management, enabling easy switching between local and production.
