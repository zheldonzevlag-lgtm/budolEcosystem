Strategy Change: Fail-Safe Cart Cleanup

Based on user requirements, cart items are no longer removed immediately upon order placement for async payments. Instead, items are removed **only** when the payment process reaches a definitive end state, whether successful or unsuccessful.

1. Delayed Cleanup for Async Payments

In OrderSummary.jsx, we now explicitly skip the local cart cleanup for all asynchronous payment methods (GCash, Maya, etc.). This keeps the items in the cart until the payment result is known.

File: components/OrderSummary.jsx

2. Definitive Cleanup on Fail/Timeout

The app/payment/return/page.jsx has been updated to trigger handleCartCleanup for unsuccessful states. This ensures that even if the payment verification times out or fails at the gateway, the cart count is decreased as requested.

File: app/payment/return/page.jsx

3. Webhook Cleanup

Webhooks for PayMongo and BudolPay now include cart cleanup logic for payment.failed and payment_intent.payment_failed events. This provides a server-side guarantee that the cart will be cleared if the payment fails asynchronously.

Files: api/webhooks/paymongo/route.js, api/webhooks/budolpay/route.js

Verification

Inspection of the "Verification Timeout" screen logic confirmed that pollStatus now correctly identifies the order and clears the cart even when the polling limit is reached.
