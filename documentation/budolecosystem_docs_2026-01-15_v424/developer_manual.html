Developer Manual - v424

Technical Explanation

The fix addresses runtime crashes in the product detail pages caused by non-array data in the images and rating fields. This was achieved through defensive programming at both the API and UI layers.

Backend Changes

We updated the product API to sanitize image data during serialization. This ensures that even if the database contains malformed JSON or strings, the frontend receives a valid array.

  • API Route (Products): app/api/products/route.js

Frontend Changes

Components were updated to use safe access patterns (optional chaining and default values) when dealing with product images and ratings.

  • Product Details: components/ProductDetails.jsx
  • Product Description: components/ProductDescription.jsx

Code Snippets

1. API Data Sanitization

// app/api/products/route.js
let images = product.images;
if (typeof images === 'string') {
    try { images = JSON.parse(images); } 
    catch (e) { images = [images]; }
}
if (!Array.isArray(images)) {
    images = images ? [images] : [];
}

2. Safe UI Rendering

// components/ProductDetails.jsx
const ratings = Array.isArray(product?.rating) ? product.rating : [];
const averageRating = ratings.length > 0 
    ? ratings.reduce((acc, item) => acc + item.rating, 0) / ratings.length 
    : 0;
