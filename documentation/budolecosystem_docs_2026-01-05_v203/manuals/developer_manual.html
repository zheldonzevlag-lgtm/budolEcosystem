Developer Manual - v203

System Architecture: Notification Integration

1. @budolpay/notifications Package

Located at packages/notifications, this package provides a centralized interface for sending notifications.

import { sendNotification, sendOTP, sendKycSuccess } from '@budolpay/notifications';

// Example: Sending an OTP
await sendOTP('user@example.com', '123456', 'EMAIL');

2. Provider Selection Logic

The system dynamically fetches the active provider from the SystemSetting table. If no provider is configured, it defaults to SMTP/Gmail for email and Console logging for SMS.

3. Database Fields

Developers should use the following new fields in the User model for verification logic:

  • otpCode: The current active verification code.
  • otpExpiresAt: Expiration timestamp for the OTP.
  • emailVerified: Boolean flag for email status.
  • phoneVerified: Boolean flag for phone status.

4. Verification Endpoint

The auth-service now exposes POST /verify-otp:

Payload: {
  userId: string,
  otp: string,
  type: 'EMAIL' | 'SMS' | 'BOTH' | 'KYC'
}

Rollback Strategy

To roll back these changes:

  1. Revert schema.prisma and run npx prisma db push.
  2. Restore auth-service/index.js and admin/app/api/users/route.ts to previous versions.
  3. The @budolpay/notifications package can remain as it is backward compatible if not imported.
