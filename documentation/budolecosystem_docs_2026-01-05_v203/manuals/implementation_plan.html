Implementation Plan - v203

Subject: Dual-Channel Notification System & OTP Flow Integration

Version: v203 (2026-01-05)

Status: COMPLETED

1. Purpose

To enhance the budolPay ecosystem with a robust notification system that supports both Email and SMS channels for critical events like account creation, KYC approval, and OTP verification, ensuring users are informed and verified through their preferred active providers.

2. Explanation & Reason

Previously, notifications were either missing or limited to a single channel. This implementation centralizes notification logic in a shared package (@budolpay/notifications) and integrates it across microservices (Auth and Admin) to provide a seamless user experience and improve security via OTP verification.

3. Technical Details

  • New Shared Package: @budolpay/notifications handles delivery logic for various providers (Brevo, Gmail, Zerix, iTextMo, etc.).
  • Database Schema Update: Added otpCode, otpExpiresAt, emailVerified, and phoneVerified to the User model.
  • Auth Service Integration: Added OTP generation on registration and a new /verify-otp endpoint.
  • Admin Service Integration: Integrated notification triggers in the KYC approval flow.
