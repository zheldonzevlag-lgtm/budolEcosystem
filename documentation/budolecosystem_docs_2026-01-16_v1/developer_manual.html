BudolEcosystem | Flutter Web App Navigation Fix

Purpose

This manual documents the technical changes made to resolve the black screen issue in the Flutter Web application during the transition from the Splash Screen to the Marketing Ads Screen.

Technical Explanation

The "black screen" was caused by a layout issue where the PageView in MarketingAdsScreen and the main Navigator in main.dart's web builder did not have explicit sizing constraints within their respective Stack widgets. In Flutter Web, this often results in zero-size rendering, displaying only the background color (0xFF0F172A).

Key Changes

  • MarketingAdsScreen Layout: Wrapped PageView.builder in Positioned.fill to ensure it occupies the entire screen space.
  • Main App Builder: Wrapped the child (Navigator) in Positioned.fill within the web-specific Stack container.
  • Splash Screen Logic: Introduced small delays before and after calling removeSplashFromWeb to allow the browser and Flutter to synchronize the transition.

Compliance & Security

These changes are purely UI/UX focused and do not affect the security posture of the application. The navigation logic continues to respect authentication states and compliance requirements (BSP, DPA, AMLA).
