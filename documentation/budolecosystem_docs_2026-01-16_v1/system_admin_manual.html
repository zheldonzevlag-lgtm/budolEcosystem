Web Deployment & Troubleshooting

This version addresses rendering issues specific to the Web build of budolPay.

1. Web Container Constraints

The web application is rendered within a fixed-size container (393x780) to simulate a mobile experience. Admins should ensure that any new screens added to the application are wrapped in Positioned.fill if they use a Stack as the root widget.

2. Splash Screen Timing

The index.html contains a JavaScript function removeSplashFromWeb(). This is called by the Flutter engine. If the app appears to hang on the splash screen, check the browser console for logs from splash_screen.dart.

3. Cache Clearing

Since the navigation state is persisted in SharedPreferences (which uses localStorage on Web), users experiencing issues with the marketing ads not appearing can clear their browser's local storage or use the "Clear Data" option in the settings if accessible.
