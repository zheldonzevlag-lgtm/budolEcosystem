SSO Implementation Plan: budolPay Admin Integration

This document outlines the technical phases for integrating budolPay Admin into the centralized budolID Single Sign-On ecosystem.

Phase 1: UI & Branding Alignment Completed

Ensure the Admin interface reflects the ecosystem branding and provides entry points for SSO.

  • Sidebar Refinement: Adjusted vertical spacing and colors to match original design; implemented Rose Red branding (#f43f5e).
  • Login Page Update: Added "Sign in with budolID" button and removed unnecessary fields (e.g., Department Code) to streamline the experience.
Phase 2: Authentication Handshake Completed

Implement the backend logic for both Headless (Direct) and Redirect-based SSO flows.

  • Headless Login Route: /api/auth/login proxies credentials to budolID API for native form support.
  • SSO Callback Route: /api/auth/callback handles redirects from budolID, validates JWTs, and establishes local sessions.
  • Local User Sync: Automatic creation/update of local User records upon successful SSO verification.
Phase 3: Validation & Testing In Progress

Verify end-to-end flows and ensure security protocols are met.

  • Environment Configuration: Validate SSO_URL and apiKey settings in .env.
  • Flow Verification: Run test-sso-flow.mjs to simulate cross-app authentication.
  • Session Management: Verify HttpOnly cookie persistence and security flags.
Phase 4: Documentation & Handoff Pending

Complete versioned documentation for developers, sysadmins, and users.

  • Developer Manual: Detail the JWT verification logic and local sync strategy.
  • SysAdmin Manual: Document service ports (8000 vs 3000) and environment requirements.
  • User Manual: Update with login instructions.

Technical Specifications

API Mapping

Endpoint Method Description
http://localhost:8000/auth/sso/login POST JSON authentication (Headless)
http://localhost:8000/login?apiKey=... GET Interactive login page (Redirect)
http://localhost:3000/api/auth/callback GET Redirect target for token processing

Dependencies

Requires budolID-0.1.0 to be running on port 8000 with seeded EcosystemApp data.
