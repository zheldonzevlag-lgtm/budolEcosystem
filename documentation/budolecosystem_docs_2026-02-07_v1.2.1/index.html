Release v1.2.1

UI Stability & Accessibility Fixes

Overview

This release addresses a critical UI bug where input boxes would "disappear" or become unreachable during keyboard navigation (Tab button) in the Authentication Form. This was caused by the browser attempting to focus elements that were hidden but still present in the DOM for animation purposes.

Fixes Applied

  • Tab Navigation: Added tabIndex={-1} to all hidden input fields and buttons within the login toggle. This prevents the browser from focusing on elements that are currently off-screen or transparent.
  • Clipping Fix: Increased the minimum height (min-h-[85px]) of the animated input containers to ensure that labels and inputs are not clipped when overflow-hidden is active.
  • Autofill Support: Added standard autoComplete attributes (email, current-password, tel-national, one-time-code) to all authentication fields to improve browser compatibility and user experience.
  • Focus Management: Ensured that only the active login method (Email or Mobile) has focusable elements.

Files Modified

  • budolshap-0.1.0/components/auth/AuthForm.jsx

Verification

Tested Tab navigation through the entire login flow in both Email and Mobile modes. Focus now correctly cycles only through visible and interactive elements.
