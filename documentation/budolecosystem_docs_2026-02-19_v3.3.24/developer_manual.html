Profile Data Synchronization

This update addresses an issue where the "Edit Profile" screen would display empty fields if the user data was not fully loaded upon initialization.

Implementation Details

We introduced a didChangeDependencies override in EditProfileScreen to listen for updates from the ApiService. When the user data becomes available (and if it hasn't been loaded yet), the text controllers are populated.

Code Changes

File: edit_profile_screen.dart


  @override
  void didChangeDependencies() {
    super.didChangeDependencies();
    if (!_isDataLoaded) {
      final user = context.watch<ApiService>().user;
      if (user != null) {
        if (_firstNameController.text.isEmpty) _firstNameController.text = user['firstName'] ?? '';
        // ... populate other fields
        _isDataLoaded = true;
      }
    }
  }

This ensures that even if the network request completes after the screen is built, the fields will update automatically.
