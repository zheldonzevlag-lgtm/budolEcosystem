Android Build Process

The Android application was built for release using the following command:

flutter build apk --release --dart-define=API_HOST=192.168.1.17

Cross-Platform Compatibility Fix

To resolve the dart:js error on Android, a conditional import strategy was implemented. This allows the web-specific splash screen removal logic to coexist with mobile builds.

New Files Created:

  • js_helper.dart: The main export file with conditional logic.
  • js_helper_web.dart: Web implementation using dart:js.
  • js_helper_mobile.dart: Mobile stub (no-op).
  • js_helper_stub.dart: Base stub for other platforms.

Modified Screens:

  • login_screen.dart: Updated to use callJsMethod.
  • splash_screen.dart: Updated to use callJsMethod.
