Hybrid Accounting & budolshap Shipping Integration

Date: 2025-12-31

System Architecture Overview

In Phase 14, we bridged the gap between the budolshap marketplace, the budolpay gateway, and the budolAccounting observer. This fulfills the "Hybrid Accounting" model where each app maintains its local ledger while syncing critical business events to a centralized auditor.

1. Hybrid Accounting Implementation

We implemented a non-blocking sync utility in budolshap that records double-entry bookkeeping records in budolAccounting whenever a payment is credited.

Key Components:

  • Utility: lib/accounting.js - Standardized fetch client for the accounting service.
  • Entry Builder: Logic to map order totals to Debit (Cash) and Credit (Revenue + Liability) accounts.
  • Integration: lib/escrow.js - Automatically triggers a ledger sync upon creditPendingBalance().
// Example Ledger Entry Structure
[
  { accountCode: '1000', debit: 1000, credit: 0, desc: 'Order #123 Payment' },
  { accountCode: '4000', debit: 0, credit: 50, desc: 'Platform Fee' },
  { accountCode: '2000', debit: 0, credit: 950, desc: 'Seller Earnings' }
]

2. budolshap Shipping Logic

We enhanced the ordersService.js to explicitly handle the budolshap (Internal) provider, distinguishing it from external providers like Lalamove.

Business Rules:

  • Flat Rate: ₱50.00 for all standard deliveries.
  • Free Shipping: Automatically applied if the store order total is ₱1,000.00 or more.
  • Provider ID: budolshap (mapped from the cart UI).

3. Environment Configuration

To enable these integrations locally, ensure the following variables are set in your .env:

# Accounting Sync
BUDOLACCOUNTING_URL=http://localhost:8002

# budolpay Gateway
BUDOLPAY_GATEWAY_URL=http://localhost:8000
BUDOLPAY_API_KEY=bs_key_2025
