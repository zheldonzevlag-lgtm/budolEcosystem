Version 225

Null-Safety Fixes & Global Error Logging (APK v1.2.6)

Purpose

This release addresses the critical type 'Null' is not a subtype of type 'String' error reported in the mobile app. It strengthens null-safety across authentication, profile management, and registration flows.

Issues & Fixes

  • Issue: Unhandled null values in API responses (like userId, status, or error) were causing runtime crashes.
  • Fix: Implemented explicit ?.toString() conversions and null-safe defaults in:
    • login_screen.dart: Enhanced identification and OTP logic.
    • home_screen.dart: Protected user name display from null fields.
    • registration_screen.dart: Safe phone number initialization from route arguments.
    • api_service.dart: Improved JSON parsing for tokens and user objects.
  • Debugging Improvement: Added global error listeners (FlutterError.onError and PlatformDispatcher.onError) in main.dart to log stack traces in debug mode for future troubleshooting.

Technical Details

  • Version: Incremented pubspec.yaml to 1.2.6+1.
  • Build: Generated budolPay 1.2.6.apk with API_HOST=192.168.1.24.
  • Verification: flutter analyze passed with 0 issues.
