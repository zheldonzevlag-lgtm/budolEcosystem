System Admin Manual

Database Maintenance

To ensure all phone numbers remain standardized, the following maintenance script should be run after any large data import or if inconsistency is detected.

Standardization Script Location:
budolID-0.1.0/scripts/standardize-phones.js

Running the Migration

  1. Ensure environment variables are set (DATABASE_URL).
  2. Navigate to the scripts directory.
  3. Execute: node standardize-phones.js

Schema Management

The system operates across two main schemas:

  • budolid: Central identity and authentication.
  • public: Application-specific user data.

Admin must ensure that phoneNumber fields in both schemas have unique constraints and are kept in sync via the identity service.

Compliance Audit

Regularly check the VerificationCode table for expired codes. The system automatically handles expiry, but manual cleanup can be performed if storage exceeds limits.
