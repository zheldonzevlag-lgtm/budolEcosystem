Known Issues

I-001: Duplicate Prefixing

Entering '+63' in the input field would result in '+63+63...' in the database.

RESOLVED

I-002: Leading Zero Confusion

Users entering '0912...' with a static '+63' UI prefix caused invalid numbers like '+630912...'.

RESOLVED

Fixes Applied

  • Implemented normalizePhone utility to strip leading zeros and non-numeric characters.
  • Updated AuthForm.jsx to filter input and prevent non-numeric characters in phone field.
  • Enhanced budolID backend to search across multiple database schemas (budolid, public).
  • Standardized legacy database records using a migration script.
