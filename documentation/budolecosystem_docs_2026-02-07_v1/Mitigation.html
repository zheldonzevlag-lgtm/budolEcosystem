Mitigation Plan

M-001: Standardization Utility

Risk: Inconsistent phone number formatting.

Action: Implemented a ecosystem-wide normalizePhone utility that converts all variations (09..., 639..., 9...) to E.164 (+639...).

M-002: Fuzzy Matching in Identity Service

Risk: Users registered with non-standard formats are not found.

Action: Updated /auth/check-phone to perform fuzzy matching, checking for both normalized and raw variations across multiple database schemas.

M-003: UI Auto-formatting

Risk: User entering leading zero with +63 prefix.

Action: Updated AuthForm.jsx to automatically strip leading zeros in the phone input field and provide a clear 9XXXXXXXXX placeholder.
