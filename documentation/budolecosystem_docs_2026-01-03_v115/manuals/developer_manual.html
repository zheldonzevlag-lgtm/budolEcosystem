Developer Manual - Version 115

Problem Diagnosis

Modern Android versions (API 26+) use Adaptive Icons, which consist of separate background and foreground layers. The previous generation only updated the legacy ic_launcher.png, leaving the adaptive XML and its associated background color (previously red #F43F5E) unchanged.

Technical Solution

Modified pubspec.yaml to explicitly define the adaptive layers:

flutter_launcher_icons:
  adaptive_icon_background: "#0F172A"
  adaptive_icon_foreground: "assets/app_icon.png"

This forces the generation of ic_launcher_foreground.png and updates values/colors.xml with the correct background hex code.

Cleanup Command

A full flutter clean was required to purge the build/ directory and ensure the new resource IDs were correctly mapped during the Gradle build.
