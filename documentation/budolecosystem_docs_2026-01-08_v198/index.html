Release Summary: v198 (PIN Authentication Logic Refinement)

This version documents the fix for the "Invalid authentication request" error on the PIN screen. The issue occurred when the system prompted for a PIN from a trusted device, even if the user had not yet set a secure PIN (pinHash was null).

Key Improvements

  • Authentication Fallback: Modified the identification logic to trigger an OTP challenge if a user lacks a PIN, even if their device is already trusted.
  • Clearer Error Messaging: Updated the PIN verification endpoint to provide specific feedback when a PIN is not set or the user is not found.
  • Session Resilience: Ensured that users can still log in and verify their devices via OTP when PINs are unavailable.
