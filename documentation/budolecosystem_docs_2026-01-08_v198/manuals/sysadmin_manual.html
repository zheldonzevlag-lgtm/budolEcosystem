System Admin Manual Updates - v198

User Authentication Audit

Admins can now more easily identify users who are stuck in the PIN flow due to missing PINs. Use the following SQL query to audit your user base:

SELECT id, "phoneNumber", "pinHash", "trustedDevices"
FROM "User"
WHERE "pinHash" IS NULL;

Service Restart Requirements

The changes in v198 involve critical logic in the Auth Service. After applying these changes, ensure the service is restarted to clear any cached process states:

  • Local: Restart the npm run dev process for auth-service.
  • Production: Trigger a rolling redeploy of the auth-service container/pod.

Troubleshooting

If a user reports an "Invalid authentication request" error despite the v198 fixes, verify that their userId exists in the database. If it doesn't, they may need to re-register or clear their app cache to re-trigger the identification phase.
