Prisma Dependency Resolution

This manual covers the resolution of the Prisma MODULE_NOT_FOUND error in the monorepo.

Technical Explanation

In the budolpay-0.1.0 monorepo, dependencies are managed using npm workspaces. By default, npm hoists shared dependencies to the root node_modules folder to save space and ensure version consistency.

The error occurred because a workspace-specific node_modules directory existed in packages/database, which contained broken symlinks or incomplete installation of the prisma package. This caused the prisma generate command to look for build/index.js in the wrong location.

Resolution Steps

  1. Delete the workspace-specific node_modules:
    rm -rf packages/database/node_modules
  2. Reinstall from root with --ignore-scripts to avoid circular failures during postinstall:
    npm install --ignore-scripts
  3. Manually trigger Prisma generation:
    npm run db:generate

Code References

  • Root package.json - Defines workspace configuration and root dependencies.
  • Database package.json - Defines the generate script.
  • Prisma Schema - The source of truth for database client generation.
