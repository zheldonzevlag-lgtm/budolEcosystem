Developer Manual: LAN & mDNS

Overview

The ecosystem has been upgraded to support seamless LAN accessibility. This involves binding Express servers to 0.0.0.0 and advertising services via mDNS.

Binding to 0.0.0.0

All microservices must bind to 0.0.0.0 to be accessible from outside the local machine. Example:

app.listen(PORT, '0.0.0.0', () => {
    console.log(`Service running on http://0.0.0.0:${PORT}`);
});

mDNS Service Discovery

Service advertising is handled centrally in start-ecosystem.js using the bonjour-service library. This publishes the service name and IP to the local network.

const service = bonjour.publish({
    name: 'budolPay (Gateway API)',
    type: 'budolpay',
    protocol: 'tcp',
    port: 8080
});

Prisma Client Generation

The ecosystem uses multiple Prisma instances. To ensure all clients are synchronized and initialized, the start-ecosystem.js runner automatically executes npm run db:generate at startup. This covers:

  • budolPay microservices (@budolpay/database)
  • budolID
  • budolAccounting
  • budolShap

Environment Variable Sync

The sync-network.js script automatically updates .env files with the current LAN IP. Developers should run npm run sync-network whenever their IP changes.

Mobile App Integration

The mobile app uses multicast_dns to find the _budolpay._tcp.local service. This returns the host IP which is then used for all API calls.
