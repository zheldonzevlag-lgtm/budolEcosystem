v279: LAN Accessibility & mDNS Latest

Full implementation of cross-device LAN accessibility and mDNS (Bonjour) service discovery for the entire ecosystem.

Developer Manual System Admin Manual User Manual Recommendations

Purpose

This update resolves critical accessibility issues where ecosystem apps were only reachable via localhost. By implementing mDNS and binding services to 0.0.0.0, all budolEcosystem applications are now discoverable and accessible by any device on the same Local Area Network (LAN).

Key Changes

  • Service Binding: Updated all microservices (budolID, budolPay, budolAccounting) to bind to 0.0.0.0 instead of localhost.
  • mDNS Advertising: Integrated bonjour-service into the central runner (start-ecosystem.js) to advertise all 10 services via Multicast DNS.
  • Mobile Discovery: Verified and aligned the Flutter mobile app's DiscoveryService with the new mDNS types for seamless auto-discovery.
  • Prisma Optimization: Added automated prisma generate for all services in the central runner to prevent "client not initialized" errors.
  • Network Sync: Enhanced sync-network.js to propagate LAN IP (192.168.1.24) to all .env files and database redirect URIs.
  • Infrastructure: Added bonjour-service as a root dependency for the ecosystem.

Technical Details

Services are now advertised using the _budolpay._tcp.local type for the API Gateway and unique types for other services. This allows the mobile app to find the server IP automatically without manual configuration.
