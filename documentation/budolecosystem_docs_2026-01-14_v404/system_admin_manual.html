Configuring Realtime Providers

The Budol Ecosystem now supports multiple real-time data providers. Admins can configure the preferred method via the system_settings database table or the Admin Settings UI.

Configuration Keys

Key Possible Values Description
REALTIME_METHOD SWR, SOCKETIO, PUSHER The primary real-time communication protocol.
SOCKETIO_URL URL (e.g., http://192.168.1.17:8080) The endpoint for the Socket.io server (usually API Gateway).
PUSHER_KEY String Public key for Pusher integration.
PUSHER_CLUSTER String (e.g., ap1) Region cluster for Pusher.
Important: Switching the REALTIME_METHOD requires a dashboard refresh for all connected admins to take effect.

Monitoring & Logs

Admins can monitor notification traffic in the API Gateway logs. Look for entries starting with [Notification] to verify event broadcasting.

[Notification] Emitted new_user to admin room/channel
[Notification] Emitted new_transaction to admin room/channel
