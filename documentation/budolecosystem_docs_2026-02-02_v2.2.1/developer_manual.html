Technical implementation details for the Ecosystem Configuration Engine v2.2.1.

1. Specialized Input Engine

The settings UI now detects variable keys to render appropriate input types, ensuring data integrity and better UX.

Implementation Logic

// [page.tsx](file:///d:/IT%20Projects/budolEcosystem/budolpay-0.1.0/apps/admin/app/settings/page.tsx)
{setting.key === 'NOTIFICATION_EMAIL_PROVIDER' ? (
  <select name="value" defaultValue={setting.value} ...>
    <option value="resend">Resend</option>
    <option value="sendgrid">SendGrid</option>
    <option value="smtp">Custom SMTP</option>
  </select>
) : setting.key.includes('TIMEOUT') || setting.key.includes('DAYS') ? (
  <input type="number" name="value" defaultValue={setting.value} ... />
) : (
  <input type={setting.isSecret ? "password" : "text"} ... />
)}

2. Server-side Filtering

Filtering is performed on the server using Next.js searchParams, which maintains the application's state in the URL.

Filtering Logic

// Filter settings based on query
const filteredSettings = query 
  ? settings.filter(s => 
      s.key.toLowerCase().includes(query) || 
      (s.description?.toLowerCase().includes(query)) ||
      s.group?.toLowerCase().includes(query)
    )
  : settings;

3. Audit Logging

Every setting update is logged in the AuditLog table with the action UPDATE_SYSTEM_SETTING.

Audit Trace

await prisma.auditLog.create({
  data: {
    action: "UPDATE_SYSTEM_SETTING",
    entity: "SystemSetting",
    entityId: id,
    newValue: { value },
  }
});
