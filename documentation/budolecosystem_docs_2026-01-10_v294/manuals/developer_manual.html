Developer Manual - v294

SSO Redirect Logic Improvements

In this version, we improved how the redirect_uri is generated on the client side. Previously, it relied solely on window.location.origin, which caused issues when users accessed the site via different IP addresses or localhost.

Implementation Details

The login pages now prefer the NEXT_PUBLIC_APP_URL environment variable:

const appUrl = process.env.NEXT_PUBLIC_APP_URL || (typeof window !== 'undefined' ? window.location.origin : ''); const redirectUri = `${appUrl}/auth/callback`;

Database Synchronization

A new script scripts/update-db-ips.js was introduced to ensure that the EcosystemApp table in the database matches the current network environment. This is critical for budolID to permit redirects to the correct LAN IP.

Impact Analysis

  • SSO Flow: More resilient to network changes.
  • Configuration: .env files must be synced using npm run network:sync whenever the host IP changes.
