Managing Network Transitions

When moving the server between different networks (Home, Office, etc.), the LAN IP address usually changes. To ensure the ecosystem continues to function correctly, follow these steps:

  1. Connect to the new network.
  2. Open a terminal in the project root.
  3. Run the synchronization command:
    npm run network:sync
  4. This command will:
    • Detect your new LAN IP.
    • Update all .env files across all microservices.
    • Update the budolID database to allow redirects to the new IP.
  5. Restart the ecosystem:
    npm start
Important: If you do not run the sync command, users on other devices will not be able to log in, as the system will try to redirect them back to the old IP or 127.0.0.1.

Environment Variables

The following variables are dynamically updated:

  • NEXT_PUBLIC_APP_URL
  • NEXT_PUBLIC_SSO_URL
  • SSO_VERIFY_URL
