Mitigation Plan - v517

R-517-01: Time Sync Issues

Mitigation: Implement a server-side timestamp validation during the transaction finalization phase. The mobile app should prioritize NTP-synchronized time where available.

R-517-02: Hardcoded Store Names

Mitigation: Enhanced validation in the BudolPayAdapter to ensure the storeName property is mandatory in the QR payload. Fallback to a generic "Budol Merchant" only if strictly necessary, with a logged warning.

R-517-03: Navigation Stack Corruption

Mitigation: Used pushNamedAndRemoveUntil with a (route) => false predicate to explicitly clear all previous routes when returning to Home.
