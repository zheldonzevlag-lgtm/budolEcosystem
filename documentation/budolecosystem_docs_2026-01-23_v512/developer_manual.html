Developer Manual - v512

Technical Overview

This release focuses on data propagation and UI refinement for transaction history. We added store-related fields to the Transaction model and updated the mobile UI to display this information prominently.

Database Changes

The Transaction model in the Prisma schema was updated to include storeId and storeName.

model Transaction {
  // ...
  storeId      String?
  storeName    String?
  // ...
  @@index([storeId])
}

File: schema.prisma

Backend Implementation

The wallet-service was updated to populate these new fields during the QR payment process.

await prisma.transaction.update({
    where: { id: transaction.id },
    data: { 
        senderId: userId,
        status: 'COMPLETED',
        completedAt: getLegacyManilaDate(),
        storeId: qrData.storeId || null,
        storeName: qrData.storeName || null
    }
});

File: wallet-service/index.js

Frontend UI Updates

The TransactionDetailsScreen now displays the Store Name and uses a standardized 12px font for better density.

if (widget.transaction['storeName'] != null)
  _buildDetailRow('Store Name', widget.transaction['storeName']),
_buildDetailRow('Transaction ID', widget.transaction['id'] ?? 'N/A'),

File: transaction_details_screen.dart

Compliance

Ensured that store names are handled securely and no sensitive merchant data is exposed beyond what is required for user transaction history (NPC Compliance).
