System Admin Manual: Deployment & Sync

Operational procedures for environment variable management and Vercel synchronization.

1. Infrastructure Overview

The BudolEcosystem relies on a distributed architecture across multiple Vercel projects. Maintaining synchronization of environment variables (specifically public URLs) is critical for inter-service communication and authentication.

  • Admin Project: budol-pay-admin
  • Shap Project: budolshap-0.10
  • ID/SSO Project: budolid
  • Verification Project: budolpay-verification-service

2. Environment Variable Synchronization

We use a PowerShell script (sync-vercel-envs.ps1) to push local environment configurations to Vercel. This ensures that production environments are always aligned with the latest architectural changes.

Updated Sync Configuration

The budolpay-admin service has been added to the sync array in budolpay-0.1.0/scripts/sync-vercel-envs.ps1:

$services = @(
    # ... existing services ...
    @{ name = "budolpay-admin"; dir = "d:\IT Projects\budolEcosystem\budolpay-0.1.0\apps\admin" }
)
Important: Always verify that you are logged into the Vercel CLI (vercel login) before running the sync script.

3. Deployment Checklist

1
Update .env: Ensure the local .env file in the target project directory contains the correct production URLs.
2
Run Sync Script: Execute .\scripts\sync-vercel-envs.ps1 from the budolpay-0.1.0 root.
3
Redeploy on Vercel: Trigger a redeploy for the affected projects (Admin and Shap) to pick up the new environment variables.
4
Verify URLs: Use the Vercel dashboard to confirm that NEXT_PUBLIC_APP_URL and other critical variables match the deployment environment.

4. Troubleshooting Infinite Loops

If an infinite loop (508 error) persists after deployment:

  1. Clear Browser Cache: Redirection loops can sometimes be cached by the browser.
  2. Check Vercel Deployment Logs: Look for middleware execution logs.
  3. Validate Origins: Ensure that the URL in the browser matches NEXT_PUBLIC_APP_URL exactly (including protocol and subdomains).
  4. SSO Configuration: Verify that the NEXT_PUBLIC_SSO_URL is reachable and correctly configured in the Admin project.

5. Operational Runbook

Database Migrations: No database changes were required for this version. However, ensure that the Verification Service URL is updated in any database-stored configuration if applicable.

Rollback Procedure: To rollback, revert the changes in .env and .env.vercel files and re-run the sync script followed by a redeploy.
