Developer Manual: Infinite Loop Resolution

Technical implementation details for resolving production routing failures.

1. Purpose & Overview

The primary goal of this update was to resolve 508 INFINITE_LOOP errors occurring on the Vercel-deployed instances of the BudolPay Admin and BudolShap platforms. These errors prevented users from accessing the applications and were traced back to misconfigured environment variables that dictated authentication and redirection logic.

2. Technical Analysis

Root Cause: Environment Mismatch

In Next.js applications, especially those using middleware.ts for authentication, public URLs (like NEXT_PUBLIC_APP_URL) are often used to determine if a request is authenticated or needs redirection. If these URLs point to localhost or a local IP (e.g., 192.168.1.13) while running in a production environment (Vercel), the middleware can enter a recursive loop:

  1. User visits production URL (Vercel).
  2. Middleware checks authentication, sees mismatch between production URL and expected NEXT_PUBLIC_APP_URL (which was local).
  3. Middleware redirects to SSO/Auth page.
  4. SSO/Auth page redirects back to NEXT_PUBLIC_APP_URL (Local IP).
  5. Local IP is inaccessible or routes back to production, causing the loop.

Specific Issues Identified

Component Issue Impact
BudolPay Admin Local IPs (192.168.1.13) in .env Infinite loop during auth redirection on Vercel.
BudolShap Typo in .env.vercel (budulshap vs budolshap) Mismatched origins causing 508 errors.
Verification Service Incorrect URL endpoint in Admin config KYC/Verification features failing.

3. Code Details & Implementation

Admin Environment Alignment

Updated apps/admin/.env to use correct production endpoints:

# Before
NEXT_PUBLIC_SSO_URL="http://192.168.1.13:3003"
NEXT_PUBLIC_APP_URL="http://192.168.1.13:3004"
VERIFICATION_SERVICE_URL="https://budolpay-kyc.vercel.app"

# After
NEXT_PUBLIC_SSO_URL="https://budolid.vercel.app"
NEXT_PUBLIC_APP_URL="https://budolpay-admin.vercel.app"
VERIFICATION_SERVICE_URL="https://budolpay-verification-service.vercel.app"

BudolShap Typo Fix

Corrected the .env.vercel file in the budolshap-0.1.0 root:

# Fixed URLs
NEXT_PUBLIC_APP_URL="https://budolshap-0.10.vercel.app"
NEXT_PUBLIC_BASE_URL="https://budolshap-0.10.vercel.app"
NEXT_PUBLIC_SITE_URL="https://budolshap-0.10.vercel.app"

4. Risk & Mitigation

Risk Mitigation
Environment Variable Desync Implemented automated sync script (sync-vercel-envs.ps1) to push local configs to Vercel.
Middleware Versioning Verified middleware.ts logic in Admin to ensure it uses NEXT_PUBLIC_APP_URL correctly.
Local Development Breakage Developers must use .env.local for local development to override production defaults.

5. Test Results

Verified Configuration Validation: All environment variables now point to their respective production Vercel URLs.

Verified Sync Script: sync-vercel-envs.ps1 successfully includes budolpay-admin in the synchronization process.

Verified Ecosystem Alignment: Root .env.vercel now contains centralized SSO and App URL references.

6. Future Recommendations

  • Centralized Config Service: Move from file-based .env to a centralized secret management service (e.g., Vercel Secrets or AWS Secrets Manager).
  • Middleware Logging: Add server-side logging to middleware.ts to catch redirection loops early.
  • Automated URL Validation: Add a build-time script to verify that NEXT_PUBLIC_*_URL variables match the deployment target environment.
