Mitigation Strategy v483

M-483-01: Runtime Feature Detection

Instead of assuming all browsers support the Web Crypto API, we now perform runtime detection. If window.crypto.subtle is missing, we bypass encryption for the flutter_secure_storage package, preventing the TypeError: generateKey crash.

M-483-02: Payment State Synchronization

To prevent double payments, the "Pay Now" button is only rendered if the local order state matches the unpaid status from the backend. Additionally, the repayment logic includes a server-side check before initiating the transaction.
