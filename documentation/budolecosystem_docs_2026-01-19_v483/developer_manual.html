Developer Manual v483

Technical Explanation

Version v483 introduces logic to enable repayments for BudolPay orders and a safety layer for Web Crypto API dependencies in the mobile app's web build.

Core Changes

1. BudolPay Repayment UI

Enabled the "Pay Now" button in OrderItem.jsx for BUDOL_PAY payment method when status is unpaid.

// OrderItem.jsx
{!order.isPaid && order.paymentStatus !== 'cancelled' && order.paymentMethod !== 'COD' && order.status === 'ORDER_PLACED' && (
  <button
    onClick={(e) => { e.stopPropagation(); onRepay && onRepay(order); }}
    className="text-xs bg-green-500 text-white px-4 py-1.5 rounded mt-3 hover:bg-green-600 w-full"
  >
    Pay Now
  </button>
)}

2. Web Crypto API Safety Layer

Introduced isWebCryptoSupported() in js_helper_web.dart to detect crypto.subtle availability.

// js_helper_web.dart
bool isWebCryptoSupported() {
  try {
    return js.context.hasProperty('crypto') && 
           js.context['crypto'].hasProperty('subtle');
  } catch (e) {
    return false;
  }
}

This is used during FlutterSecureStorage initialization in api_service.dart to prevent crashes on non-secure origins.

// api_service.dart
_secureStorage = FlutterSecureStorage(
  webOptions: WebOptions(
    encrypt: isWebCryptoSupported(),
  ),
);
