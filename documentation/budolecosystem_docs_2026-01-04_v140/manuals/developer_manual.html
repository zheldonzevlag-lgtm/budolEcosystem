Developer Manual: Environment Isolation & SSO Fix

1. Purpose

The primary goal of this update was to fix the PrismaClientInitializationError encountered during budolPay mobile app login. This error was caused by improper environment variable handling in the ecosystem's central runner.

2. Technical Details & Code Logic

The Problem: Environment Pollution

In start-ecosystem.js, the pre-flight validation loop was doing the following:

process.env.DATABASE_URL = dbMatch[1]; // Polluted parent process
if (!validateEnvironment(app.name)) { ... }

Because process.env is shared, the DATABASE_URL of the last app in the loop remained in the runner's environment. When spawn() was called, it passed ...process.env to the child, overriding the child's own .env file settings.

The Fix: Proper Restoration & Cleanup

I implemented two layers of protection in start-ecosystem.js:

  1. Atomic Restoration: The validation loop now captures a full snapshot of process.env and restores it by explicitly deleting keys that weren't originally present.
  2. Spawn Isolation: Before spawning a child process, we create a cleanEnv object that explicitly deletes DATABASE_URL, NODE_ENV, and VERCEL. This forces the child service to load these values from its own .env file via dotenv.

3. Code Example (Cleaned Spawn)

const cleanEnv = { ...process.env };
delete cleanEnv.DATABASE_URL;
delete cleanEnv.NODE_ENV;
delete cleanEnv.VERCEL;

const env = { 
    ...cleanEnv, 
    PORT: app.port.toString(),
    LOCAL_IP: localIP,
    // ... URLs
};

spawn(app.command, app.args, { cwd: app.cwd, shell: true, env });

4. Test Results

  • Verified: Parent process.env remains clean after validation loop.
  • Verified: budolID correctly loads its own DATABASE_URL from .env.
  • Verified: SSO login no longer throws PrismaClientInitializationError.

5. Recommendation for Future Enhancement

  • Encapsulated Validator: Refactor scripts/env-validator.js to accept parameters instead of relying on process.env.
  • Env Monitoring: Add a /health/env endpoint to services to debug environment states in production safely.
