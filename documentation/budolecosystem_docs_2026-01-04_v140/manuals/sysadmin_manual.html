SysAdmin Manual: Environment & Deployment Stability

1. Purpose

Ensuring that the deployment runner does not introduce configuration drift into sub-services during startup.

2. Infrastructure Changes

No changes to physical infrastructure. The update focuses on process isolation within the Node.js runtime environment.

3. Deployment Procedures

When running npm start, the system now performs a "Clean Spawn" for each service. This means:

  • Sub-services are guaranteed to prioritize their local .env file.
  • Global environment variables set in the shell (e.g., SET DATABASE_URL=...) will still be passed, but the runner will specifically prevent internal leakage of database strings between apps.
Tip: If you need to override a database URL for ALL apps, set it in the shell environment before running npm start. However, the runner will now warn you if you try to use a Production DB while the apps are in Development mode.

4. Issues & Fixes

Issue Fix Impact
Prisma Protocol Error Cleaned spawn() environment. Fixed login failures in Mobile/App.
Parent Env Pollution Added originalEnv snapshot & restoration. Stable runner process.

5. Risks & Mitigation

  • Risk: Some apps might depend on environment variables passed from the parent that were unintentionally deleted.
  • Mitigation: Only specific keys (DATABASE_URL, NODE_ENV, VERCEL, PORT) are deleted. General system variables (PATH, etc.) are preserved.
