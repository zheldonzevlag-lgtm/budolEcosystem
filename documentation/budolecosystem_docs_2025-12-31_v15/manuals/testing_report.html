Issues & Fixes

Issue 1: ESM Alias Resolution in Test Runner

Problem: Node.js test runner failed to resolve @/lib imports in ESM mode, causing ERR_MODULE_NOT_FOUND when importing services.

Reason: Path aliases defined in Next.js/Webpack (jsconfig.json) are not natively understood by Node.js ESM loader without additional configuration (like ts-node or imports in package.json).

Issue 2: Branding Inconsistency (budolExpress)

Problem: The internal shipping provider was initially named budolExpress, which conflicted with the ecosystem branding of budolshap and budolpay.

Reason: Assumption based on generic delivery naming conventions instead of primary ecosystem pillars.

Fix 1: Logic-Level Unit Testing

Solution: Refactored the test script to test core logic functions by importing them via relative paths and isolating them from framework-specific aliases.

Fix 2: Ecosystem Rebranding

Solution: Renamed budolExpress / bud_express to budolshap across the shipping service, orders service, and documentation. Updated payment references to consistently use budolpay.

Verification: Re-ran Phase 16 branding tests; all assertions passed with correct provider names.
