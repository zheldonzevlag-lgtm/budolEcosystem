Implemented Fixes & Improvements

Details of technical resolutions for identified issues in v3.1.0.

F-310-1: Restored Login Method Toggle

Issue: Missing UI element to switch between Mobile and Email login.

Resolution: Implemented _buildMethodToggle widget in login_screen.dart. This widget uses a Container with two GestureDetector elements to provide an explicit, Lazada-style toggle.

File: login_screen.dart

F-310-2: Dynamic Password Field for Email Login

Issue: Email login required a password field which was absent in the mobile implementation.

Resolution: Added conditional rendering in _buildPhoneStep to display a TextField for password when _loginMethod == LoginMethod.email.

if (_loginMethod == LoginMethod.email) ...[
  const SizedBox(height: 16),
  TextField(
    controller: _passwordController,
    obscureText: _obscurePassword,
    // ... decoration ...
  ),
]

F-310-3: Backend Integration for Email Auth

Issue: Mobile backend lacked a dedicated route for direct Email/Password authentication.

Resolution: Implemented identifyEmail in api_service.dart which maps to the /login endpoint, handling JSON serialization and session persistence.

File: api_service.dart

F-310-4: State Collision Prevention

Issue: Switching between methods could leave stale validation errors or input values.

Resolution: Updated the toggle handler to clear controllers and reset validation states upon switching.

void _setLoginMethod(LoginMethod method) {
  setState(() {
    _loginMethod = method;
    _phoneController.clear();
    _emailController.clear();
    _passwordController.clear();
    _identifierError = null;
    _emailError = null;
    _passwordError = null;
  });
}
