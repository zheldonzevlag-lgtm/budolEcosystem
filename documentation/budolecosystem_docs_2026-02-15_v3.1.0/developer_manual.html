Technical Architecture

The login system uses a state-driven approach to switch between LoginMethod.mobile and LoginMethod.email. Switching methods resets validation states and clears relevant controllers.

1. Login Method State

Defined in login_screen.dart:

enum LoginMethod { mobile, email }

2. Email Authentication Flow

When Email login is selected, the UI dynamically adds a Password field. The _handleIdentify method calls the new identifyEmail API endpoint.

// api_service.dart

  final url = '$authUrl/login';

}

3. UI Toggle Implementation

The _buildMethodToggle widget provides an explicit UI for selection, ensuring users understand their chosen login method before entering credentials.
