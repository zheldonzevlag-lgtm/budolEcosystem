Future Recommendations

1. Unified Mobile Backend Identify Route

Rationale: Currently, the mobile app uses two different services for identity verification (checkEmail and identifyMobile). This increases frontend complexity.

Recommendation: Create a unified POST /auth/login/identify endpoint in the mobile backend that handles both email and phone numbers, similar to the web's implementation.

Priority: High

2. Shared Design System Library

Rationale: The login toggle UI was manually recreated in Flutter to match the React implementation. Any brand color change now requires updates in multiple repos.

Recommendation: Implement a shared Design Tokens repository (JSON/YAML) that exports brand colors and spacing to both CSS and Dart.

Priority: Medium
