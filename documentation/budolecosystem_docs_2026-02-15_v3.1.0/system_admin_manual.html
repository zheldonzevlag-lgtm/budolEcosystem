System Administrator Manual

Operational guide for managing the login systems and backend integration for v3.1.0.

1. Backend Configuration

Ensure the auth service is updated to support the /login endpoint for email/password authentication. The mobile app now directs email login requests to this unified endpoint.

Environment Variables:

  • AUTH_SERVICE_URL: Must point to the production authentication microservice.
  • DB_CONNECTION: Ensure the users table has indices on both email and phone_number for performant identifier lookups.

2. Monitoring Login Failures

Monitor logs for high rates of 401 (Unauthorized) or 404 (Not Found) responses on the /auth/login route. This may indicate a brute-force attempt or a sync issue between the mobile and web databases.

# Example log query (Kibana/CloudWatch)
status: 401 AND path: "/auth/login"

3. Security Auditing

In accordance with PCI DSS and BSP requirements, all password transmissions must be logged without the actual credential values. Verify that the api_service.dart logging implementation masks sensitive data or only logs request metadata.

4. Troubleshooting Session Issues

If users report being logged out unexpectedly when switching login methods, verify the FlutterSecureStorage implementation. The session token should be stored consistently regardless of the login method used.
