Developer Manual - Unified Environment Switching (v192)

Purpose

To enable developers to switch the entire ecosystem between Local, Production, and Vercel environments using a single command (npm start), while ensuring each service connects to its respective database.

Technical Implementation

The env-manager.js script was updated with Service-Aware Mapping. It now recognizes service-specific environment variables in the source files and maps them to the generic DATABASE_URL required by Prisma in each service.

Mapping Logic

if (key === 'BUDOLID_DATABASE_URL' && envPath.includes('budolID-0.1.0')) {
    envMap.set('DATABASE_URL', value);
} else if (key === 'BUDOLSHAP_DATABASE_URL' && envPath.includes('budolshap-0.1.0')) {
    envMap.set('DATABASE_URL', value);
} else if (key === 'BUDOLACCOUNTING_DATABASE_URL' && envPath.includes('budolAccounting-0.1.0')) {
    envMap.set('DATABASE_URL', value);
}

Test Results

✅ Verified: Running node env-manager.js local correctly maps BUDOLID_DATABASE_URL to DATABASE_URL inside budolID-0.1.0/.env.

✅ Verified: Running node env-manager.js prod correctly maps production strings for all 4 major services.

Recommendations

Move toward a centralized configuration service (Vault or similar) to avoid hardcoding production strings in connection files.
