v197 Developer Manual

Subject: Technical implementation of the collapsible sidebar submenu in budolPay Admin.

Architecture Overview

The sidebar uses React state and Tailwind CSS to manage navigation visibility. The refactor introduced a hierarchical structure for settings, aligning the file system structure with the navigation hierarchy in Next.js.

File System Changes

The settings module has been reorganized into subdirectories to support better modularity and cleaner routing:

  • /app/settings/environment/page.tsx: Previously /app/settings/page.tsx.
  • /app/settings/location/page.tsx: Previously /app/settings/maps/page.tsx.
  • /app/settings/page.tsx: Now serves as a redirect to /settings/environment.

Key Components

1. Sidebar State Management

The isSettingsOpen state is used to toggle the submenu visibility. It is initialized using pathname.startsWith('/settings') to ensure the menu is expanded if the user refreshes the page while on a settings sub-route.

const [isSettingsOpen, setIsSettingsOpen] = useState(pathname.startsWith('/settings'));

2. Routing & Redirect Logic

A simple redirect is implemented in the root settings page to ensure smooth navigation:

import { redirect } from "next/navigation";

export default function SettingsPage() {
  redirect("/settings/environment");
}

3. NavItem Enhancement

The NavItem component was extended to handle optional icons and sub-item specific styling.

function NavItem({ 
  href, 
  label, 
  active, 
  icon, 
  isSubItem 
}: { 
  href: string; 
  label: string; 
  active: boolean; 
  icon?: React.ReactNode;
  isSubItem?: boolean;
}) {
  return (
    <a 
      href={href} 
      className={`flex items-center gap-3 px-4 py-1.5 text-[13px] font-medium transition-all rounded-md group ${
        active ? 'text-white bg-white/5' : 'text-white hover:bg-white/5'
      } ${isSubItem ? 'pl-10' : ''}`}
    >
      {!isSubItem && (
        <span className={`w-1 h-1 rounded-full transition-all ${
          active ? 'bg-[#f43f5e]' : 'bg-slate-700 group-hover:bg-slate-500'
        }`}></span>
      )}
      {icon && <span className={active ? 'text-[#f43f5e]' : 'text-slate-500 group-hover:text-slate-400'}>{icon}</span>}
      <span className={isSubItem ? 'text-[12px]' : ''}>{label}</span>
    </a>
  );
}

Styling Conventions

  • Indentation: Sub-items are indented by pl-10 to align with the text of parent items.
  • Colors: Active items use #f43f5e (BudolPay Rose) for icons and indicator dots.
  • Transitions: Expand/collapse uses max-h-40 with transition-all duration-300 for smooth movement.

Impact Analysis

This change is purely UI-focused and does not affect the backend APIs or database schema. It improves the scalability of the sidebar as more settings modules are added in the future.
