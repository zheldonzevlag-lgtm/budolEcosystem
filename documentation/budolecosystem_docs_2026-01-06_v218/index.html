budolEcosystem v218

Mobile App Navigation Fix & Build Release

← Previous (v217) Main Index Next (v218) → Next →

Purpose

This release addresses user-reported navigation issues in the mobile app and updates the API connection configuration to match the current local development environment. It also provides a fresh build (APK) for validation.

Technical Specification & Fixes

1. Route Configuration Fix

Issue: The "Register here" link in the Login screen was attempting to navigate to /registration, but the main application route was defined as /register, causing the link to be non-functional.

Resolution: Updated lib/main.dart to align the route name with the screen's navigation logic.

// lib/main.dart
routes: {
    '/home': (context) => const HomeScreen(),
    '/login': (context) => const LoginScreen(),
    '/registration': (context) => const RegistrationScreen(), // Fixed from /register
},

2. API Host Configuration Update

Issue: The mobile app was hardcoded to connect to 192.168.1.24, which did not match the current local machine IP (192.168.1.3), resulting in "No route to host" errors.

Resolution: Updated lib/services/api_service.dart to use the correct default IP and ensured the build command explicitly defines the host.

// lib/services/api_service.dart
return const String.fromEnvironment(
    'API_HOST',
    defaultValue: '192.168.1.3',
);

Risks & Mitigations

Risk: Changing local IP addresses may break the build again in the future.

Mitigation: Always verify the current local IP using ipconfig before building and use the --dart-define flag to override the default host during the build process.

Recommendations for Future Enhancement

  • Dynamic Host Discovery: Implement a mechanism for the mobile app to automatically discover the gateway host on the local network.
  • Route Constants: Move all route strings to a central constants file to prevent future name mismatches.
  • Build Automation: Integrate the build process into a CI/CD pipeline that automatically detects the environment and sets the correct API host.
