Handling "Incorrect PIN" for Test Accounts

If a user is entering a PIN they believe is correct but the system rejects it, follow these steps:

  1. Verify the user's phone number.
  2. Run the reset_pin.js script to clear their current PIN.
  3. Instruct the user to re-identify. This ensures they set a fresh PIN that matches their current memory.

Support Runbook: Forced Recovery

Clearing the PIN is the primary method for "Forced Recovery" in the current implementation. It maintains the device trust (if already established) but requires a fresh OTP verification to re-secure the account with a new PIN.
