Issues & Fixes - v3.3.27

Resolved Issues

1. OTP Not Visible in Server Console

Issue: Developers reported that OTPs were no longer appearing in the server console during local development, hindering testing.

Root Cause: The `isLocal` flag was being evaluated before environment variables were loaded from `.env`, causing `process.env.NODE_ENV` to be undefined.

Fix: Moved `require('dotenv').config(...)` to the top of `services/auth-service/index.js`, before any variable definitions.

2. OTP Not Sending via Email/SMS

Issue: Users reported that OTPs were not being delivered.

Root Cause: The `notifications` package relies on environment variables for default provider credentials. Since `.env` failed to load in the `auth-service`, these credentials were missing.

Fix: Correcting the `dotenv` loading order resolved this issue as well, ensuring credentials are available to the notification service.

3. Unit Test Failures (quick-reg.test.js)

Issue: `npm test` reported failures in `quick-reg.test.js` with `TypeError: app.address is not a function`.

Root Cause: Incorrect import statement in the test file. It was importing the entire module object instead of destructuring the `app` instance.

Fix: Updated import to `const { app } = require('../index');`.
