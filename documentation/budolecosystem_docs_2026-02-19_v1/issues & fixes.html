1. OTP Not Sending / Not Visible in Logs

Issue: OTPs were not being generated or logged, and SMS/Email delivery was failing silently.

Root Cause:

  • Missing `notifications` package dependency in `auth-service`.
  • Log redirection or suppression prevented debug logs from showing.

Fix:

  • Added explicit dependency to `auth-service/package.json`.
  • Added debug logging in `notifications/index.js` to print OTP to console for dev/test environments.

2. Prisma Query Engine EPERM Error

Issue: Database operations failed with `EPERM: operation not permitted` on `query_engine-windows.dll.node`.

Root Cause: File locking by running Node.js processes prevented Prisma Client regeneration or access.

Fix:

  • Terminated all Node.js processes.
  • Renamed `.prisma/client` to `.prisma/client_old` to bypass the locked file.
  • Regenerated Prisma Client using `prisma generate`.
  • Deleted the old locked directory.

3. Home Screen Name Masking & Toggle

Issue: User requested to toggle full name visibility (Eye icon). Initially, toggling showed `*****` because the API returned masked data. Subsequent UI refinements were needed for layout and privacy.

Root Cause: `auth-service`'s `maskPII` function was applied to all responses. Long names caused UI overflow covering the eye icon.

Fix:

  • Modified `auth-service/index.js` to return unmasked `firstName` and `lastName` in authenticated responses.
  • Updated `home_screen.dart` to include a visibility state (`_isNameVisible`) and a toggle icon.
  • Refinement: Changed display to show First Name Only.
  • Refinement: Reduced font size to 14sp and added `Flexible` widget to fix overflow issues.
  • Refinement: Changed masked characters to dots (`........`) and set default visibility to masked (false).
  • Removed the exclamation mark from the welcome text.

// auth-service/index.js change
user: {
id: user.id,
phoneNumber: maskPII(user.phoneNumber),
firstName: user.firstName, // Unmasked
lastName: user.lastName // Unmasked
}
