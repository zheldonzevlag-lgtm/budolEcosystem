1. System Overview

The BudolPay Ecosystem consists of a Microservices backend (Node.js), a PostgreSQL database, and a Flutter Mobile Application. This manual guides system administrators in managing the infrastructure, deployments, and troubleshooting.

2. Service Management

Starting the Ecosystem

To start all services (API Gateway, Auth Service, Transaction Service, Notification Service, Mobile Web):

cd "d:\IT Projects\budolEcosystem"
npm start

Note: Ensure PostgreSQL is running before starting the services.

Environment Variables

Critical environment variables located in .env files of each service:

  • NODE_ENV: Set to development for verbose logging (including OTPs in console) or production for security.
  • DATABASE_URL: Connection string for PostgreSQL.
  • JWT_SECRET: Secret key for token generation.

3. Database Management

Backup Procedures

Database backups should be stored in \backup-db\db-date. Use pg_dump for creating backups.

Prisma Schema Updates

If the database schema is modified, run the following to update the client:

npx prisma generate

If you encounter EPERM errors, delete the node_modules/.prisma folder and retry.

4. Monitoring & Logs

OTP & Notification Logs

In Development Mode, OTPs are logged to the console for testing purposes. In Production, these logs are suppressed to comply with PCI DSS.

Check budolpay-0.1.0/packages/notifications/index.js for logging logic.

5. Compliance & Security

PII Handling: The system masks PII (Phone Number, First Name, Last Name) by default. Unmasking is strictly limited to authenticated endpoints (/verify-otp, /verify-pin, /setup-pin) to allow user profile display.

Ensure NODE_ENV is set to production in live environments to prevent data leakage in logs.
