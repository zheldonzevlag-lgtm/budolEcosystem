Future Recommendations

Enhancements and long-term improvements.

← Back to Index

1. Centralized Logging Service

Rationale: Currently, logs are printed to console. A centralized logging service (ELK Stack or similar) would improve auditability and troubleshooting in production.

2. Automated End-to-End Testing

Rationale: Implement Flutter driver tests or integration tests to automatically verify UI flows like the Name Visibility Toggle across different devices.

3. Dynamic Port Configuration

Rationale: To avoid port binding conflicts (like the Port 9000 issue), implement dynamic port assignment or a more robust configuration management for the Mobile Web service.

4. Enhanced PII Security

Rationale: Consider field-level encryption for PII in the database, in addition to the current masking at the API level.
