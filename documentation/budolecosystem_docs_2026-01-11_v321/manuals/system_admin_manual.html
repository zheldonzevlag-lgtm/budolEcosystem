System Admin Manual

Operational guidance for biometric prerequisites, troubleshooting, and support diagnostics.

Purpose

To provide a runbook for support and system administrators to validate biometric login readiness, diagnose biometric availability issues, and ensure compliance-aligned handling of authentication factors.

Prerequisites

  • Device security: Screen lock enabled (PIN/pattern/password).
  • OS enrollment: At least one fingerprint/face enrolled in OS settings.
  • Network: App can reach BudolPay gateway host (Settings → Gateway Host).
  • App state: Biometric login toggle enabled AND user has completed a manual PIN login at least once after enabling.

Explanation & Rationale

Biometric authentication is an OS-mediated factor. If the OS has no enrolled biometrics, the device may still “support biometrics,” but authentication prompts can fail or be unavailable. This distinction is a primary driver of user-reported inconsistencies.

For PCI DSS and BSP/NPC-aligned security posture, the app stores the user PIN only in OS secure storage and releases it only after OS biometric verification succeeds.

Troubleshooting

Case A: Biometric toggle not visible in Settings

  • Confirm device supports biometrics and has a secure screen lock set.
  • Confirm OS-level biometrics are enrolled.
  • On emulators, ensure biometric hardware is enabled in the emulator settings and at least one biometric is enrolled.

Case B: Biometric icon is visible but login fails

  • Confirm biometrics are enabled in Settings (toggle on).
  • Confirm user performed a manual PIN login after enabling (required to store PIN).
  • If user changed MPIN, instruct to login manually once to refresh stored PIN.
  • Confirm OS biometric prompt succeeds outside the app (device settings biometric test).

Case C: Works on one device but not another

  • Compare OS versions and OEM biometric implementations (some require stronger device security).
  • Confirm same account state and that stored PIN exists on the device (re-login manually once).

Test Protocol

Use the repository E2E checklist script as a standardized runbook:

cd \budolEcosystem
dart run scripts\test_scripts\test_biometric_login.dart

Install the APK from \budolEcosystem\android-installer\budolPay_1.2.47.apk and execute the checklist steps on a device/emulator.

Risk & Mitigation

  • Risk: Misdiagnosis leads to unnecessary app reinstall and user friction.
    Mitigation: Check OS enrollment and screen lock first; then verify toggle and manual PIN login prerequisite.
  • Risk: Users attempt biometric login without gateway connectivity.
    Mitigation: Validate gateway host reachability and correct IP/port configuration.
  • Risk: Regulatory compliance drift in authentication support processes.
    Mitigation: Keep support scripts and runbooks versioned; avoid collecting biometric data or PINs via support channels.

Test Results

  • ✅ Standard checks completed: flutter analyze, flutter test.
  • ✅ E2E checklist script executed and confirmed APK presence.

Recommendation for Future Enhancement

Add a dedicated “Biometric Diagnostics” panel in Settings that displays: device supported, biometrics enrolled, toggle enabled, stored PIN present, and last biometric auth attempt status.
