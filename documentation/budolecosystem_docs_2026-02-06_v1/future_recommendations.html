Future Enhancements & Recommendations

Strategies for further improving session management and security in the Budol Ecosystem.

Dynamic Timeout Configuration

Proposal: Move session timeout and grace period constants to the SystemSetting database table. Modify the mobile app to fetch these settings during initialization.

Rationale: Allows system administrators to adjust security policies globally without requiring an app store update.

Multi-Account Support on Lock Screen

Proposal: Enhance the lock screen to show a list of previously logged-in accounts for faster switching if the current session is locked.

Rationale: Improves UX for users with multiple accounts (e.g., Personal vs. Merchant).

Remote Session Termination

Proposal: Implement a feature in the Admin Dashboard to remotely terminate mobile sessions.

Rationale: Enhances security if a user reports their device as stolen.

Enhanced Forensic Logging

Proposal: Log "Session Locked" and "Session Unlocked" events to the forensic audit trail with device metadata.

Rationale: Provides a more detailed audit trail for security investigations (PCI DSS 10.2.2 compliance).
