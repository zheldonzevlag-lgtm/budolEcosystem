Risk Mitigation Plan

Strategies implemented to address risks identified in the 2026-02-06 v1 release.

M-001: Biometric Fallback

Risk: R-001 (Biometric failure)

Action: Implemented a "Log Out" button on the lock screen. This allows users who cannot authenticate via biometrics to log out and re-enter using their full credentials (Password/MPIN).

M-002: Global Interaction Listener

Risk: R-002 (Idle timer failure)

Action: Used a Listener widget at the root of the application (SessionOverlay) to capture all onPointerDown events, ensuring the timer resets on any screen interaction, regardless of the active page.

M-003: App Lifecycle Observation

Risk: R-002 (Background state issues)

Action: Implemented WidgetsBindingObserver in SessionService to accurately track background/foreground transitions and apply grace period logic consistently across Android and iOS.

M-004: Automated Unit Testing

Risk: R-004 (Performance/Logic bugs)

Action: Created test/session_test.dart to verify state transitions and ensure the timer behaves as expected without creating multiple overlapping timer instances.
