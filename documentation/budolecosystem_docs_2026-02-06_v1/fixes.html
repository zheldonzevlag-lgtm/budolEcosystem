Implemented Fixes

Details of the bugs resolved in this release.

F-001: Session Locked UI Not Appearing

Problem: The _isLocked flag in SessionService was never set to true. Instead, the app immediately logged the user out, bypassing the lock screen overlay.

Fix: Updated _handleTimeout and _handleResumed to set _isLocked = true and notify listeners. The hard logout call was commented out in favor of the lock screen UI.

F-002: Hardcoded Version Mismatch

Problem: The ApiService and pubspec.yaml had different version numbers (1.3.63 vs 1.3.64), causing confusion in system logs and splash screens.

Fix: Synchronized both files to version 1.3.65.

F-003: Inconsistent Background Locking

Problem: App switching did not consistently trigger the lock screen even if the user was away for a long time.

Fix: Refined the _handleResumed logic to correctly calculate the background duration and trigger the _isLocked state if it exceeds 180 seconds.
