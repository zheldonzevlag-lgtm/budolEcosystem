System Admin Manual

Version: 1.3.65 | Date: 2026-02-06

Deployment & Configuration

The session lock behavior is governed by hardcoded constants in the SessionService for this release. Admins should be aware of these values when troubleshooting session issues.

Parameter Default Value Description
_timeoutMinutes 3 Minutes Time of continuous inactivity before the session is locked.
_gracePeriodSeconds 180 Seconds (3m) Time the app can remain in the background before locking upon resume.
Info: These settings are currently optimized for mobile performance and security compliance. Future versions will allow these to be synced from the Admin System Settings API.

Troubleshooting

If the session lock screen does not appear:

  • Ensure the device has biometrics (Fingerprint/FaceID) enabled and enrolled.
  • Verify that the SessionOverlay is correctly wrapping the app in main.dart.
  • Check the debug logs for "SessionService: Idle timeout reached" messages.

If the user is logged out immediately instead of locked:

  • This occurs if the server-side session token has actually expired, regardless of the client-side idle timer.

Security Hardening

Requirement: To maintain PCI DSS compliance, the _timeoutMinutes must not exceed 15 minutes.

The app uses FLAG_SECURE on Android (via the flag_secure package) to prevent screenshots of sensitive data, though it is temporarily unset in certain development builds to allow for automated testing and manual forensic review.
