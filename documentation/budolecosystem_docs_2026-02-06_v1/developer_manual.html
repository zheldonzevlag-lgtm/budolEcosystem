Developer Manual

Version: 1.3.65 | Release Date: 2026-02-06

Purpose

To implement and fix the Session Locked UI in the budolPay mobile app. This feature ensures that the application interface is obscured and locked when the user is idle or the app is backgrounded beyond a certain grace period, requiring re-authentication via biometrics to resume.

Rationale

Mobile financial applications must protect sensitive data from unauthorized access if a device is left unattended. Hard logouts can be disruptive to the user flow; therefore, a "Session Lock" mechanism provides a balance between security and convenience by allowing quick re-entry via biometric verification while maintaining session integrity.

Task List

  • Identify session/idle timeout logic in the mobile app (Flutter)
  • Fix the session locked UI visibility by setting _isLocked = true in SessionService
  • Update _handleTimeout to trigger the lock screen instead of a hard logout
  • Update _handleResumed to trigger the lock screen when grace period is exceeded
  • Increment mobile app version to 1.3.65
  • Run flutter analyze and verify no errors
  • Create and run unit tests for session lock logic

Technical Implementation

Session Service Logic

The SessionService was modified to use the _isLocked state flag. Previously, the service would immediately call apiService.logout() upon timeout, which bypassed the lock screen UI. By setting _isLocked = true, the SessionOverlay widget (which wraps the entire app) detects the state change and displays the blur overlay.

// Updated _handleTimeout in session_service.dart
void _handleTimeout() {
  if (kDebugMode) print('SessionService: Idle timeout reached. Locking session.');
  _isLocked = true;
  notifyListeners();
}

Grace Period Handling

Similarly, when the app is resumed from the background, if the difference between current time and background time exceeds _gracePeriodSeconds, the app now locks instead of logging out.

Compliance Verification

  • PCI DSS 8.1.8: Compliant. Session re-authentication is required after 3 minutes of inactivity (well within the 15-minute maximum).
  • BSP Circular 808: Compliant. Implements multi-factor re-authentication (biometrics) for session resumption.
  • NPC (Data Privacy Act): Compliant. Prevents unauthorized viewing of sensitive financial data on unattended devices.
