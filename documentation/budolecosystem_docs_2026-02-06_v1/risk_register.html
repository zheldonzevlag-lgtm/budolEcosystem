Risk Register

Identification of risks associated with session management and the new lock UI implementation.

Risk ID Risk Description Impact Probability Severity Status
R-001 Biometric sensor failure prevents user from unlocking session. User cannot access funds/app features. Medium Medium Open
R-002 Idle timer fails to trigger on certain devices/background states. Unauthorized access to open session. Low High Mitigated
R-003 Session token expires on server but client UI remains in "Locked" state. Confusion when unlock fails due to invalid token. Medium Low Open
R-004 Memory leaks from continuous timer resets on user interaction. App performance degradation over time. Low Low Mitigated
