Developer Manual - v327

On-Screen Debug Logging System

To diagnose biometric issues that don't produce server-side logs (as they are device-local OS operations), an on-screen debug log has been implemented in the Login screen.

Implemented

Technical Implementation

  • Stateful Logging: Added List<String> _debugLogs to _LoginScreenState.
  • Log Collector: Implemented _addLog(String message) which prepends timestamps and keeps the last 50 entries.
  • UI Overlay: Added a scrollable ListView at the bottom of the login screen to display these logs in real-time.
  • File: login_screen.dart

Biometric Event Hooks

Hooks were added to the following biometric lifecycle events:

  • _checkBiometrics(): Logs hardware availability and app settings status.
  • _handleBiometricLogin(): Logs the icon tap event, enrollment checks, and the final OS authentication result.
  • authenticate(): Logs critical errors and prompt cancellations.

App Versioning

The app version has been incremented to 1.2.51+51.
