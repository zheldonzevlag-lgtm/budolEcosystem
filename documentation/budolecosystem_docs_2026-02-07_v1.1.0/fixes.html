Fixes Applied - v1.1.0

✅ User Migration to SSO Schema
Resolved "User Not Found" errors by syncing 5 missing users from the app-specific public schema to the centralized budolid SSO schema. This allows users registered in legacy systems to log in via the new BudolID SSO service.
✅ AuthForm "NOT FOUND" Flicker Fix
Fixed a UX issue where the "NOT FOUND" message would briefly appear while typing a valid phone number. The checkingPhone state is now triggered immediately upon reaching 10 digits, ensuring the user sees a "Checking..." status instead of a false error.
✅ Standardized Phone Normalization
Ensured that the frontend normalizePhone helper and the backend normalizePhilippinePhone utility are perfectly aligned to handle 0-prefixed, 63-prefixed, and raw 10-digit Philippine numbers.
✅ Database Standardization Script
Created and executed a script to standardize all existing phone numbers in the database to the E.164 (+639XXXXXXXXX) format, preventing future matching failures.
