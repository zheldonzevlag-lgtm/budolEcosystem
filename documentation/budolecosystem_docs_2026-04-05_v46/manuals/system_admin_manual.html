System Admin Manual: BudolID SSO Infrastructure

Version: 43.6 | Last Updated: 2026-04-05

1. Infrastructure Overview

The BudolID service serves as the central identity provider for the ecosystem. It is designed to run both in local development (port 8000) and in production (Render/Vercel).

Environment Public URL Default Port
Production https://budol-id-sso.onrender.com Dynamic (via PORT env)
Local / LAN http://localhost:8000 or http://[LOCAL_IP]:8000 8000

2. Production Deployment (Critical)

WARNING: Ensure NODE_ENV=production or VERCEL=1 is set in production. Failure to do so will cause the service to override production database URLs with local developer IPs (192.168.1.x) during startup.

Follow these steps to deploy or update BudolID:

1
Verify DATABASE_URL in environment variables. It must point to the production PostgreSQL instance.
2
Run prisma generate to update the client.
3
Execute npm start. The service will automatically verify core app registration without overwriting production URLs if NODE_ENV is correct.

4. Troubleshooting Redirection

If users are redirected to the wrong URL after login:

  • Symptom: Redirected to 192.168.x.x instead of vercel.app.
  • Root Cause: NODE_ENV not set to production, causing the seeding logic to reset URLs in the database.
  • Fix: Set NODE_ENV=production and manually update the redirectUri in the ecosystemApp table using SQL or Prisma Studio.
