Risk Register - BudolID SSO Infrastructure

Version: 43.6 | Last Updated: 2026-04-05
ID Description Category Impact Probability Severity Status Action Taken
SSO-001 Open Redirect Vulnerability: An attacker can redirect users to a malicious site after successful SSO login by tampering with the redirect_uri parameter. Security High Medium High Mitigated Implemented domain-based whitelist validation in budolID-0.1.0/index.js.
SSO-002 Production Configuration Reset: Seeding logic overriding production redirect URIs with local IP addresses (192.168.1.x) during service startup. Infrastructure High High High Mitigated Updated seeding logic to be non-destructive in production environments (via NODE_ENV check).
SSO-003 Session Fixation: Attacker hijacking an SSO session by pre-setting a session token. Security Medium Low Medium Active Sessions are currently short-lived (7 days) and bound to the user ID. Further CSRF protection implemented on login forms.
SSO-004 Information Leakage: PII (emails/phones) exposed in server logs during debugging. Privacy Medium Medium Medium Mitigated Implemented maskPII helper for all log outputs involving sensitive user data (NPC Compliance).
