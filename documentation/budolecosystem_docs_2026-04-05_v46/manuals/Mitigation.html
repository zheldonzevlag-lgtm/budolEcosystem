Security Mitigation Plan: BudolID SSO v46

Updated: 2026-04-05 | Compliance: PCI DSS, NPC, Cybersecurity Law

1. Open Redirect Protection

Risk: Malicious redirection after SSO login.
  • Cybersecurity Law Implemented a robust domain-based validation engine in the redirection handler.
  • Restricted dynamic redirects to subdomains of the primary registered application domain.
  • Maintained a whitelist for local development environments (localhost, 192.168.*).

2. Configuration Integrity

Risk: Automatic resets overriding production settings.
  • Modified the initialization sequence to perform a non-destructive check of existing core app registrations.
  • Enforced strict environment checks (NODE_ENV) before applying configuration updates.
  • Separated initial seeding from runtime health checks.

3. PII Masking & Privacy

Risk: Disclosure of sensitive user data in logs.
  • NPC Integrated a universal maskPII helper for all console outputs involving emails and phone numbers.
  • Redacted passwords and raw tokens from all audit logs.
  • Aligned logging practices with BSP Circular 808 for administrative operations.
