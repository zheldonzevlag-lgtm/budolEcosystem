Developer Manual: Dynamic SSO Redirection Fix

Version: v46 | Updated: 2026-04-05 | Author: Budol Ecosystem AI Orchestrator

1. Overview

This version addresses a critical issue in the SSO flow where the budolID service was overriding production redirect URIs with local IP addresses during the automatic seeding process. Furthermore, the SSO service did not respect the redirect_uri parameter provided by the calling application, leading to a hardcoded (and often incorrect) redirection after successful login.

2. Dynamic Redirection Implementation

The SSO service (budolID) now captures and utilizes the redirect_uri query parameter from the initial login request. This parameter is preserved through the login form submission and used as the primary target for the final redirection.

Implementation Details:

  • GET /login: Extracts redirect_uri from query params and injects it as a hidden field in the HTML form.
  • POST /auth/sso/login-form: Extracts redirect_uri from the form body and performs validation before redirecting.

3. Seeding Logic Changes

The automatic seeding logic in budolID-0.1.0/index.js was modified to prevent overwriting production configuration with local developer settings.

// Enhanced Seeding Logic
const isProd = process.env.NODE_ENV === 'production' || process.env.VERCEL === '1';

if (!existingApp) {
    // Create core app if missing
    await prisma.ecosystemApp.create({ ... });
} else if (!isProd) {
    // Only update redirectUri in non-prod environments
    await prisma.ecosystemApp.update({
        where: { apiKey: app.apiKey },
        data: { redirectUri: app.redirectUri }
    });
}

4. Security Validation PCI DSS Cybersecurity Law

To prevent Open Redirect vulnerabilities, the SSO service implements a domain-based whitelist validation. The requested redirect_uri must match the registered domain of the application or be a valid developer hostname (localhost, 192.168.*).

// Security Validation Logic
if (requestedUrl.hostname === allowedUrl.hostname || 
    requestedUrl.hostname.endsWith('.' + allowedUrl.hostname.split('.').slice(-2).join('.')) ||
    requestedUrl.hostname === 'localhost' ||
    requestedUrl.hostname.startsWith('192.168.')) {
    targetRedirectUri = redirect_uri;
}

7. Dependencies & Software

The following core technologies were utilized to implement this fix, chosen for their reliability and ecosystem compatibility:

  • Express.js: Used for the SSO server due to its flexible routing and middleware support, essential for handling dynamic redirections.
  • Prisma ORM: Chosen for type-safe database interactions, ensuring that configuration updates (like redirectUri) are handled with integrity.
  • JWT (jsonwebtoken): Utilized for secure identity token generation, providing a stateless and industry-standard way to pass user identities across the ecosystem.
  • Bcrypt.js: For secure password hashing, ensuring NPC and BSP compliance for credential storage.
  • Jest & Supertest: Selected for the testing framework to provide reliable E2E validation of the authentication and redirection flows.
