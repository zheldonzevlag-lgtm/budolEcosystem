Issues & Fixes: SSO Redirection & Configuration

Updated: 2026-04-05 | Release: v46
ISSUE #43.6-1 Critical Impact

SSO Callback Redirect Loop in Production

Symptoms: Users on budolpay.vercel.app were redirected to 192.168.1.2:3000 after successful SSO login, resulting in "Cannot GET /api/auth/callback" or connection errors.

Root Cause: The budolID service automatically seeded its database with local IP addresses on startup, overwriting production configuration. Additionally, it ignored the redirect_uri parameter from the calling app.

FIX IMPLEMENTED:

  • Updated budolID-0.1.0/index.js to prioritize the redirect_uri query parameter.
  • Restricted seeding logic to non-production environments only.
  • Implemented security validation for dynamic redirection.
ISSUE #43.6-2 Security Risk

Open Redirect Vulnerability

Symptoms: The SSO service could be manipulated to redirect users to arbitrary external domains after login.

Root Cause: Lack of validation on the redirect_uri parameter in the SSO handler.

FIX IMPLEMENTED:

  • Implemented a domain-based whitelist in budolID-0.1.0/index.js.
  • Only allowed redirections to the registered application domain or trusted local developer addresses.
