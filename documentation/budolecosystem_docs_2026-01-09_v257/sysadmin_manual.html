System Admin Manual v257

Deployment Procedures

  1. Admin App: Rebuild the Admin Next.js application to include the new RealtimeMethodSelector component.
    npm run build in apps/admin.
  2. Transaction Service: Restart the Transaction Service to apply the notification fixes.
    node index.js or pm2 restart transaction-service.
  3. Mobile App: Deploy the new APK (v1.2.22) from android-installer/budolPay_1.2.22.apk.

Configuration Changes

Real-time Settings

Ensure the following settings are correctly configured in the Admin Panel (Settings > Realtime):

  • REALTIME_METHOD: Choose between SWR, SOCKETIO, or PUSHER.
  • SOCKETIO_URL: Must be the publicly accessible URL of the Gateway (e.g., http://192.168.1.5:8080).
  • PUSHER_KEY / CLUSTER: Required if PUSHER is selected.

Monitoring & Alerting

Monitor Gateway logs for Socket.io connection attempts. Successful connections should show the user joining their respective room (user-[id]).
