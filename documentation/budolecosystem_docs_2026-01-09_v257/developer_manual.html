Developer Manual v257

Purpose

This version addresses the inability of the Admin Panel to switch real-time providers and ensures the mobile app correctly initializes real-time listeners using dynamic server configuration.

Technical Implementation Details

1. Admin Panel: Real-Time Method Selector

Issue: The real-time method selection in the Admin Panel was "stuck" because the radio buttons were not interactive within a Server Component.

Fix: Created RealtimeMethodSelector.tsx as a "use client" component to handle UI state and radio button selection. It properly propagates the selected value to the parent form.

// RealtimeMethodSelector.tsx
"use client";
import { useState } from "react";
// ...
export default function RealtimeMethodSelector({ initialMethod }) {
  const [selectedMethod, setSelectedMethod] = useState(initialMethod);
  // ... radio button implementation
}

2. Mobile App: Socket.io Dynamic Initialization

Issue: The RealtimeService was defaulting to the base API URL for Socket.io connections, ignoring the SOCKETIO_URL setting from the database.

Fix: Updated RealtimeService.dart to check the system settings for a custom Socket.io URL.

// RealtimeService.dart
final socketioConfig = settings?['realtime']?['socketio'];
final socketUrl = (socketioConfig != null && socketioConfig['url'] != null && socketioConfig['url'].isNotEmpty)
    ? socketioConfig['url']
    : _apiService.baseUrl;

_socketService.init(baseUrl: socketUrl, userId: userId);

3. Transaction Service: Cash-In Notifications

Issue: Users were not receiving real-time updates when a cash-in was completed.

Fix: Added notifyUpdate call to the /cash-in route in transaction-service/index.js.

// transaction-service/index.js
notifyUpdate(userId, 'transaction_update', { 
    message: `Successfully cashed in PHP ${amount}`, 
    transaction: completedTransaction 
});

Test Results

  • Admin Switching: Verified radio buttons are clickable and persist to database upon saving.
  • Mobile Real-time: Verified Socket.io connects to the URL specified in the Admin Panel.
  • Transaction Notification: Verified transaction_update event is received by the mobile app after a successful cash-in.

Impact Analysis

  • Admin: Improved UX for system configuration.
  • Mobile: Better reliability for real-time updates.
  • Backend: Consistent notification logic across transfer and cash-in operations.
