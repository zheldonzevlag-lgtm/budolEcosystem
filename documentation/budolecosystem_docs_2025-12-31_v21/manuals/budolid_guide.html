What is budolID?

budolID is the centralized identity provider (SSO) for the entire budol ecosystem. It allows users to maintain a single set of credentials (email and password) to access multiple services including budolPay, budolShap, and budolAccounting.

  • Single Sign-On (SSO): Log in once and stay authenticated across all ecosystem apps.
  • Centralized Security: Security policies and user data are managed in one secure location.
  • Simplified User Experience: No need to remember different passwords for different services.
System Admin Manual Updates

Troubleshooting "Internal Server Error" on Login

If you encounter an "Internal Server Error" when signing in, it is likely because the budolID service is offline. The admin dashboard relies on this service for authentication.

How to Start the SSO Service

Ensure the authentication server is running on port 8000. You can start it using the following commands:

cd d:\IT Projects\budolEcosystem\budolID-0.1.0 npm install npm run start

Note: Ensure your PostgreSQL database is running and accessible as configured in the .env file of budolID.

Developer Manual Updates

Authentication Flow

  1. User enters credentials in the Admin Login page.
  2. The /api/auth/login route in the Admin app makes a server-to-server POST request to budolID (port 8000).
  3. If successful, budolID returns a JWT token.
  4. The Admin app sets this token as an httpOnly cookie.

Error handling has been improved in apps/admin/app/api/auth/login/route.ts to provide a descriptive 503 error when the SSO service is unreachable, instead of a generic 500 error.
