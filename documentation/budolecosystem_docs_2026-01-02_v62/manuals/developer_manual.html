Developer Manual - Payment Integration

Purpose

This document details the multi-provider payment integration architecture in BudolShap, specifically focusing on the routing logic between PayMongo and BudolPay, and the differentiation between redirect-based and QR-based payment flows.

Architectural Decisions

  • Adapter Pattern: The system uses a BasePaymentAdapter to standardize interactions with different payment gateways (PayMongo, BudolPay).
  • Unified Checkout API: /api/payment/checkout acts as the single entry point for all non-COD payments, abstracting provider-specific logic.
  • State Persistence: Order and payment state are stored in localStorage (qrPaymentData) and passed via URL parameters to ensure recovery if the page is refreshed during the payment process.

Payment Method Routing

The routing is determined in lib/services/paymentService.js and components/OrderSummary.jsx:

  • BudolPay: Internal wallet system. Uses a redirect flow (checkoutUrl).
  • PayMongo - GCash: Uses a redirect flow (checkoutUrl) to the PayMongo-hosted GCash login page.
  • PayMongo - Maya, GrabPay, QRPh: Uses a QR-based flow (qrCode). The frontend displays the QR code and polls /api/paymongo/status for completion.

Code Implementation

Example of how the frontend handles different strategies in OrderSummary.jsx:

if (paymentData.qrCode) {
    // QR Strategy (Maya, GrabPay, QRPH)
    router.push(`/payment/qr?payment_intent_id=${qrData.paymentIntentId}...`);
} else if (paymentData.checkoutUrl) {
    // Redirect Strategy (GCash, BudolPay)
    window.location.replace(paymentData.checkoutUrl);
}

Issues & Fixes

  • Issue: QR modal unmounting during refresh. Fix: Implemented a dedicated /payment/qr page with URL-based state recovery.
  • Issue: Hostname mismatch in local development for BudolPay. Fix: Added logic in BudolPayAdapter to rewrite 'localhost' to the current browsing host.

Recommendations

  • Implement a webhook listener for PayMongo to handle asynchronous payment success/failure reliably.
  • Add a "Cancel Payment" button on the QR page to release order items back to inventory if applicable.
