System Admin Manual - Payment Configuration

Overview

This manual covers the environment variables and configuration required to maintain the payment ecosystem for PayMongo and BudolPay integrations.

Environment Variables

Variable Description Example
PAYMONGO_SECRET_KEY Secret key for PayMongo API (v1) sk_test_...
BUDOLPAY_API_KEY API key for BudolPay internal gateway bs_key_2025
BUDOLPAY_GATEWAY_URL URL of the BudolPay Gateway service http://localhost:8004
NEXT_PUBLIC_BASE_URL The public URL of the application (used for redirects) http://localhost:3000

Operational Runbooks

Payment Pending / Timeout

If a user reports a successful payment but the order is still "Pending":

  1. Check the PayMongo dashboard for the payment_intent_id.
  2. Verify if the webhook was sent to /api/webhooks/paymongo.
  3. If the webhook failed, manually update the order status in the database using the Admin panel.

BudolPay Connection Issues

If BudolPay returns a 404 or 500 error:

  1. Verify the BUDOLPAY_GATEWAY_URL is reachable from the application server.
  2. Ensure the BudolPay service is running and its database is healthy.

Future Enhancements

  • Implement automated reconciliation between PayMongo reports and BudolShap order history.
  • Enable 3D Secure (3DS) for credit card payments via PayMongo.
