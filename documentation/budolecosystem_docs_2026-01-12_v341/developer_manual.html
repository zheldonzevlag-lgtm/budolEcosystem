Developer Manual - v341

Purpose & Rationale

Version v341 marks the first successful production build of the BudolPay mobile experience as a web application. This allows users to access the mobile interface via standard web browsers on the 192.168.1.17 LAN network without requiring an APK installation.

Technical Explanation

1. Flutter Web Compilation

The application was compiled using the Flutter 3.38.5 stable channel. The build process uses the HTML and CanvasKit renderers to ensure high performance and consistency across mobile and desktop browsers.

Reference: pubspec.yaml

flutter build web --release

2. Asset Management

Icons from CupertinoIcons, MaterialIcons, and lucide_icons were tree-shaken to minimize the initial payload. The final build artifacts include main.dart.js, flutter.js, and the canvaskit directory.

Key Changes

  • Build Output: Artifacts generated in budolPayMobile/build/web.
  • Icon Optimization: Tree-shaking reduced font asset sizes by over 99%.
  • Renderer Support: Full support for CanvasKit and SKWasm for improved graphics rendering.
