Real-time Update Architecture

The Challenge

The Admin Dashboard in Next.js was initially implemented as a Server Component. While excellent for performance and SEO, Server Components cannot maintain stateful connections like Socket.io listeners.

The Solution

Extracted the "Transaction Pulse" section into a dedicated Client Component. This allows the dashboard to render immediately with server-fetched data while establishing a persistent socket connection for live updates.

Key Component: TransactionPulse

Located at TransactionPulse.tsx, this component handles the socket connection and state management.

useEffect(() => {
  const GATEWAY_URL = process.env.NEXT_PUBLIC_GATEWAY_URL || "http://192.168.1.17:8080";
  const socket = io(GATEWAY_URL);

  socket.on("new_transaction", (newTx: Transaction) => {
    setTransactions((prev) => [newTx, ...prev.slice(0, 4)]); // Keep top 5
  });

  return () => socket.disconnect();
}, []);

Dashboard Integration

Updated page.tsx to pass initial server-side data to the client component.

<TransactionPulse initialTransactions={recentTransactions.map(tx => ({
  ...tx,
  amount: Number(tx.amount),
  createdAt: tx.createdAt.toISOString()
}))} />
