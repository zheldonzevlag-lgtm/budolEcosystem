Purpose

The goal of this update was to improve the user experience on the Transaction Details screen and implement a scalable branding solution for the marketing carousel.

Code Details

1. Transaction Details UI Refactoring

In lib/screens/transaction_details_screen.dart, the recipient/sender name was moved above the Transaction ID. Additionally, the conditional button layout was replaced with a standardized Row containing two Expanded buttons.

// Reordering
if (widget.transaction['receiver'] != null && !isIncome)
  _buildDetailRow('Recipient Full Name', ...),
_buildDetailRow('Transaction ID', widget.transaction['id'] ?? 'N/A'),

// Standardized Button Row
Row(
  children: [
    Expanded(child: _buildPrimaryButton()),
    const SizedBox(width: 12),
    Expanded(child: _buildSecondaryButton()),
  ],
)

2. Dynamic Branding with RegExp

The _buildDescription helper in lib/screens/marketing_ads_screen.dart was refactored to use a RegExp for identifying and colorizing all budol ecosystem services.

final RegExp regExp = RegExp(r'(budol)(Pay|₱ay|Ecosystem|Shap|Loan|Express|ID)?', caseSensitive: false);
// ...
