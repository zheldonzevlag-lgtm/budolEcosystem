Developer Manual - v441

1. Purpose & Rationale

Version v441 addresses specific visual feedback regarding the consistency of the budol* branding across the ecosystem. Specifically, it resolves an issue where branded text in the mobile application appeared smaller than surrounding service labels and refines the budol prefix color for better visibility and professionalism.

2. Technical Changes

2.1 Flutter Branding Logic Update

Flutter Modified ui_utils.dart to use Text.rich instead of RichText. This change is critical because Text.rich automatically inherits the ambient TextScaler from the MediaQuery, ensuring that branded text scales identically to standard Text widgets when system font scaling is active.

The budol prefix color was also updated from Slate 400 (0xFF94A3B8) to Slate 600 (0xFF475569).

// Updated Flutter Logic
return Text.rich(
  TextSpan(
    style: baseStyle ?? const TextStyle(color: Colors.black),
    children: spans,
  ),
  textAlign: textAlign,
  maxLines: maxLines,
  overflow: overflow ?? (maxLines != null ? TextOverflow.ellipsis : TextOverflow.clip),
);

2.2 React Branding Logic Update

React Modified BudolPayText.jsx to align with the new color standard, changing the budol prefix from text-slate-400 to text-slate-600.

// Updated React Logic
<span key={match.index} className={baseClassName}>
    <span className="text-slate-600">budol</span>
    <span className={suffixColor}>{suffixPart}</span>
</span>

3. Dependencies & Software

  • Flutter SDK: 3.16+ (Utilizing TextScaler logic)
  • Tailwind CSS: For React UI styling (Slate 600 colors)
  • Dart RegExp: For dynamic branding extraction

4. Compliance

Ensures compliance with DTI Branding Standards and Accessibility Laws by maintaining legible font sizes across all system-wide scaling levels.
