BudolShap v0.1.0 Integration & Unified Ingress

Purpose

To ensure the correct version of the BudolShap marketplace (v0.1.0) is deployed and integrated into the central Budol Ecosystem routing. This version resolves deployment conflicts and standardizes the routing to mirror the local development environment.

Unified Ingress & Local Alignment

The ecosystem deployment has been aligned with the local development structure (Central Runner):

  • Root Domain (/): Proxies to the BudolPay Admin dashboard (Local Port 3000 equivalent).
  • /shap: Proxies to the BudolShap marketplace (Local Port 3001 equivalent).
  • /api: Proxies to the API Gateway, which manages microservice routing.

Standardized Prefix Handling

All backend services must handle the /api prefix to ensure compatibility with the consolidated Vercel ingress. The API Gateway has been updated to route these correctly.

BudolShap v0.1.0 Deployment

BudolShap v0.1.0 was deployed as a standalone Next.js project and is proxied via the root vercel.json. Conflicts between builds and functions/headers in Vercel configuration were resolved by moving headers to next.config.mjs and removing vercel.json from the sub-project.

Test Results

  • Marketplace (/shap): 200 OK
