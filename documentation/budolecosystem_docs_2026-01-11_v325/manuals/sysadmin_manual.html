Troubleshooting Prisma on Windows

If the system fails to start due to EPERM errors during Prisma engine generation, follow these steps:

Emergency Fix Script (PowerShell): 
# 1. Kill all node processes
Stop-Process -Name node -Force -ErrorAction SilentlyContinue

# 2. Remove locked Prisma engine directory
Remove-Item -Path "node_modules/@prisma" -Recurse -Force

# 3. Regenerate Prisma Client
cd packages/database
npx prisma generate

Network Configuration

The API Gateway automatically detects the local LAN IP (currently 192.168.1.17) and configures the mobile app fallback accordingly. Ensure that the firewall allows incoming traffic on port 8080 for mobile device connectivity.
