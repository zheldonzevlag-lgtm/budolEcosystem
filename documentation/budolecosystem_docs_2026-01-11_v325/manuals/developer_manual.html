Developer Manual - v325

Prisma EPERM Fix & Ecosystem Stability

This version focuses on resolving a critical EPERM (Operation Not Permitted) error encountered during the npm start process, specifically within the Prisma engine generation phase on Windows environments.

Fixed

Issue: Prisma Engine File Locking

On Windows, the Prisma engine (query_engine-windows.dll.node) can become locked by background Node.js processes or antivirus software during a rename operation. This prevents npx prisma generate from completing, which in turn blocks the database package and dependent services.

Resolution Strategy

  1. Process Cleanup: Forcefully terminating all node processes to ensure no active handles are held on the node_modules files.
  2. Manual Cache Purge: Deleting the @prisma folder within the root node_modules to clear corrupted or locked engine binaries.
  3. Re-generation: Manually triggering npx prisma generate in the packages/database directory to recreate the Prisma Client with fresh engines.

Technical Details

The fix was applied to the following package:

  • budolpay-0.1.0/packages/database
# Command executed to resolve:
Remove-Item -Path "D:\IT Projects\budolEcosystem\budolpay-0.1.0\node_modules\@prisma" -Recurse -Force
cd "D:\IT Projects\budolEcosystem\budolpay-0.1.0\packages\database"
npx prisma generate

Service Startup Verification

After resolving the Prisma issue, the ecosystem services were verified for successful startup.

  • API Gateway: Running on http://192.168.1.17:8080 (mDNS advertised).
  • Auth Service: Running on port 8001.
  • Wallet Service: Running on port 8002.
  • Transaction Service: Running on port 8003.

Note: The Admin app requires a production build (npm run build) before it can be started in production mode.

Impact Analysis

  • System Stability: High. Backend services are now operational and reachable by the mobile app.
  • Build Process: No changes to build scripts, but troubleshooting steps are now documented.
