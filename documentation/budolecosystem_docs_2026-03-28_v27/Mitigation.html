Mitigation Strategies

Cascading Schema Reset

To prevent duplicate records (e.g., duplicated Product IDs from previous failed pushes), we implemented a mandatory wipe.sql step that executes before the mirror import. This ensures a clean production state.

Environment Variable Sanitization

Vercel's environment synchronization often injects hidden \r (CRLF) characters. We mitigated this by adding a regex-based trim to lib/prisma.js, preventing "Invalid Connection String" errors in the production cluster.

Namespace Translation

Instead of mapping all tables to the public schema, we used schema-prefixed imports to maintain the logical separation of budolID and budolPay as inherited from the local multi-DB setup.
