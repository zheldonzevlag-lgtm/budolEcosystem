Developer Manual - v504

1. Orders Service Layer Restoration

The core orders service layer in ordersService.js has been restored to include missing business logic required for the API endpoints.

Implemented Functions:

  • createOrder: Handles order creation with stock validation and real-time triggers.
  • updateOrderStatus: Manages status transitions and COD payment logic.
  • getOrderById: Retrieves full order details with relations.
  • linkPaymentToOrder: Associates external payment intents with local orders.

Example COD Logic in updateOrderStatus:

if (status === 'DELIVERED' && isCod && !currentOrder.isPaid && !isPaid) {
    updatedFields.isPaid = true;
    updatedFields.paymentStatus = 'cod_paid';
    updatedFields.paidAt = new Date();
    await creditPendingBalance({
        orderId: currentOrder.id,
        amount: currentOrder.total,
        gateway: 'COD'
    });
}

2. Store Dashboard Search Fix

A ReferenceError in page.jsx was resolved by correctly defining the local search state.

State Implementation:

const [searchTerm, setSearchTerm] = useState('')

This state is now used to filter the orders array locally, matching against Order ID, Customer Name, and Product Items.

3. Real-time Integration

All order operations now broadcast events via realtime.js to ensure the UI stays in sync without manual refreshes.

await triggerRealtimeEvent(`order-${orderId}`, 'status-updated', {
    orderId,
    status: updatedOrder.status,
    isPaid: updatedOrder.isPaid
});
