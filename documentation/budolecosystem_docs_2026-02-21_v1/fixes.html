Issues & Fixes Log

FIX-20260221-01: Product Wizard Buttons Unresponsive

Issue: The "Publish Product" and "Save as Draft" buttons in the Add Product Wizard were clickable but performed no action or failed silently.

Root Cause:

  • The storeId was not being passed to the Wizard component.
  • The "Save as Draft" button had a placeholder function.
  • Missing validation allowed the form to attempt submission without a store context, leading to silent failures or errors.

Resolution:

  • Updated AddProductWizard to accept storeId.
  • Implemented handleSaveDraft using localStorage.
  • Added useEffect to restore drafts on mount.
  • Ensured StickyFooter receives the correct handler functions.

FIX-20260221-02: Knowledgebase Version History Missing

Issue: The Knowledgebase sidebar was missing previous version links.

Resolution: Created and ran a reconstruction script reconstruct_kb.py to parse index.html and restore the sidebar navigation.

FIX-20260221-03: AddProductWizard Syntax Error

Issue: Build error "x 'import', and 'export' cannot be used outside of module code" in AddProductWizard.jsx.

Root Cause: A misplaced import { useEffect } from 'react'; statement was found inside the component function body, along with a redundant useState block.

Resolution: Removed the misplaced import and redundant code. Verified useEffect is correctly imported at the top level.

FIX-20260221-04: DragDropImageUpload Unique Key Prop Error

Issue: "Each child in a list should have a unique 'key' prop" error when rendering the image upload component.

Root Cause: AddProductWizard was passing raw image strings (URLs) from initialData or saved drafts to DragDropImageUpload, which expects objects with unique id properties for sorting.

Resolution: Implemented a normalizeImages helper function in AddProductWizard.jsx to automatically convert image strings into objects with generated unique IDs before initializing the form or loading drafts.

FIX-20260221-05: VariationMatrixManager Infinite Update Loop

Issue: "Maximum update depth exceeded" error when using the Variation Matrix Manager in Add Product Wizard.

Root Cause: Circular dependency between parent component updates and child component state synchronization.

  1. VariationMatrixManager updates internal state and calls onUpdate.
  2. Parent (AddProductWizard) updates its form state and re-renders.
  3. Parent passes new state back to VariationMatrixManager as initialData prop.
  4. VariationMatrixManager's useEffect detects prop change and calls setState to sync.
  5. This triggers a re-render and another update cycle, causing an infinite loop.

Resolution: Implemented a useRef-based isSelfUpdate flag in VariationMatrixManager.jsx. When an update originates from the component itself, the flag is set to true. The synchronization useEffect checks this flag and skips the state update if it was self-initiated, breaking the loop while still allowing updates from external sources (like loading a draft).

FIX-20260221-06: DragDropImageUpload Unique Key Prop Error

Issue: "Each child in a list should have a unique 'key' prop" error prevents product images from showing in the Add/Edit Product Wizard.

Root Cause: The DragDropImageUpload component received image objects that were either missing id properties or had duplicate IDs (potentially from raw API data or inconsistent state updates). This caused React to fail rendering the list and dnd-kit to malfunction.

Resolution:

  • Updated DragDropImageUpload.jsx to internally normalize the images prop using useMemo.
  • Added fallback logic to generate stable IDs (fallback-id-{index}) if an image lacks an ID.
  • Ensured SortableContext and the rendering loop use these normalized IDs to maintain consistency for drag-and-drop operations.
  • Added debug logs in AddProductWizard.jsx to trace data flow.

FIX-20260221-07: Product Image Persistence and Display

Issue: Product images were not showing in the "Edit Product" wizard or the product list, and a 404 error occurred after saving a product.

Root Cause:

  • 404 Error: AddProductWizard.jsx redirected to /store/products, which does not exist. The correct path is /store/manage-product.
  • Image Saving: The API (api/products/route.js and [id]/route.js) used isValidImage filter which rejected image objects ({id, url}), discarding them during save.
  • Image Display: manage-product/page.jsx expected image strings in the src attribute, causing render errors when objects were present.

Resolution:

  • Updated AddProductWizard.jsx to redirect to /store/manage-product.
  • Modified lib/imageUtils.js to allow image objects with valid URLs in isValidImage.
  • Updated AddProductWizard.jsx's uploadImage to handle and return image objects, preserving IDs.
  • Updated manage-product/page.jsx to correctly extract URLs from image objects for display.

FIX-20260221-08: Shipping and Condition Data Persistence

Issue: Shipping dimensions (Weight, Length, Width, Height) and other product details (Condition, Pre-Order status) were not being saved or updated, resulting in values reverting to 0 or default.

Root Cause: The API endpoints (api/products/route.js and api/products/[productId]/route.js) were not extracting or updating these specific fields from the request body, causing them to be ignored during CREATE and UPDATE operations.

Resolution:

  • Updated POST handler in api/products/route.js to include weight, length, width, height, condition, and preOrder in the creation logic.
  • Updated PUT handler in api/products/[productId]/route.js to include these fields in the update logic.
  • Added preOrder field to Product model in prisma/schema.prisma to resolve "Unknown argument" error.
