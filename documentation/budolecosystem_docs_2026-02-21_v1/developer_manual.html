Store Management: Add Product Wizard

The AddProductWizard component has been updated to require a storeId prop for proper draft scoping and validation.

Component Signature

export default function AddProductWizard({ initialData, storeId }) {
    // ...
}

Draft Persistence

Drafts are now saved to localStorage using a key pattern: budol-product-draft-{storeId}. This ensures drafts are isolated per store.

Validation Changes

The onSubmit handler now explicitly checks for the presence of storeId before attempting to upload images or submit data to the API. This prevents "orphan" product creation attempts.
