Developer Manual v184

Purpose

Phase 2 focuses on deploying the central Identity provider (budolID) and ensuring all ecosystem services can communicate with it in a production environment.

Technical Details

1. URL Parameterization

All services now use parameterized environment variables for service discovery in production. This replaces local IP-based discovery with stable Vercel URLs.

# budolpay/.env.vercel
ID_SERVICE_URL="https://budolid.vercel.app"
GATEWAY_URL="https://budolpay-api-gateway.vercel.app"

# budolID/.env.vercel
GATEWAY_URL="https://budolpay-api-gateway.vercel.app"
BUDOLSHAP_URL="https://budolshap.vercel.app"

2. budolID Production Readiness

The budolID service has been audited for production readiness:

  • CORS: Enabled for all ecosystem domains.
  • JWT: Uses the ecosystem-wide shared secret standardized in Phase 1.
  • Vercel Compatibility: Includes vercel.json for serverless node execution.

Dependencies

  • Standardized JWT_SECRET from v183.
  • Active Vercel project for budolID.
