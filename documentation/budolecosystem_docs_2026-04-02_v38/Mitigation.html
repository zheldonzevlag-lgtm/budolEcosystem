Mitigation Strategies: v38 Profile Selfie

M1: Image Compression

Strategy: All profile selfies are resized to 1024x1024 and compressed to 85% quality before upload.

Outcome: Reduces server storage requirements and decreases latency for mobile users on low bandwidth.

M2: Secure Multipart Handling

Strategy: Verification service uses Multer with specific file size limits (5MB) and mime-type validation (images only).

Outcome: Prevents "bomb" attacks or malicious file injections via the profile upload endpoint.

M3: Liveness Pre-validation

Strategy: Reusing face detection logic to ensure a real face is present in the "oval" frame before capturing.

Outcome: Ensures profile pictures are actually of the user, preventing "black screen" or "object" profile pictures.
