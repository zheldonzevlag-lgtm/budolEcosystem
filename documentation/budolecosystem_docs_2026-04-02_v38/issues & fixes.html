Issues & Fixes: v38

Issue: Profile picture was not refreshing immediately after capture on some devices.

Fix: Implemented notifyListeners() in ApiService.uploadProfilePicture to ensure the UI rebuilds with the new avatarUrl.

Issue: KYCCaptureScreen was missing a clear "Face Capture" mode distinction from ID capture in the UI.

Fix: Added conditional title logic and specialized KYCOverlayPainter modes for oval vs rectangle.

Issue: Multipart upload was failing on slow connections due to 30s timeout.

Fix: Increased upload timeout to 60 seconds specifically for the uploadProfilePicture method.
