v38 Developer Manual: Profile Selfie Integration

Architecture & Data Flow

The profile selfie flow reuses the KYCCaptureScreen stabilized in v37. The flow is triggered from the EditProfileScreen, and the resulting image is uploaded to the verification-service.

1. Backend Extension

The verification-service now explicitly handles type: 'PROFILE_PICTURE'. This type ignores KYC logic and only updates the user's avatarUrl.

Code Reference: verification-service/index.js


// Update logic in verification-service
if (type === 'PROFILE_PICTURE') {
  updateData.avatarUrl = `/verification/uploads/${req.file.filename}`;
}

2. Mobile Implementation

The mobile app uses MultipartRequest via ApiService to send the file. The UI uses KYCCaptureType.face to trigger the circular overlay.

Code Reference: edit_profile_screen.dart


// Triggering the capture
final String? imagePath = await Navigator.push(
  context,
  MaterialPageRoute(
    builder: (context) => const KYCCaptureScreen(
      captureType: KYCCaptureType.face,
    ),
  ),
);

Dependencies

  • http: For multipart file uploads.
  • http_parser: For setting content types in multipart requests.
  • camera: For providing the low-level camera feed to KYCCaptureScreen.

Compliance Note

The circular frame ensures that face embeddings generated during KYC can be compared more accurately with the profile picture, as the head position is standardized.
