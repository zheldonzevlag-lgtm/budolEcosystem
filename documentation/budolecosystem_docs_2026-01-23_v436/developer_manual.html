Developer Manual - v436

Technical Summary

This version fixes a ReferenceError in the StoreOrders component. The searchTerm variable was being used in the JSX to bind the value of the search input, but it was not initialized as a state variable in the component's scope.

Code Changes

1. State Initialization

The following state was added to the StoreOrders function in app/store/orders/page.jsx:

const [searchTerm, setSearchTerm] = useState('')

2. Search Logic

The searchTerm state is used within the filteredOrders filter logic to match order IDs, customer details, and product names:

// Search term filter (customer name, email, order ID, product name)
if (searchTerm) {
    const searchLower = searchTerm.toLowerCase()
    const matchesOrderId = order.id.toLowerCase().includes(searchLower)
    const matchesCustomer = order.user?.name?.toLowerCase().includes(searchLower) ||
        order.user?.email?.toLowerCase().includes(searchLower)
    const matchesProduct = order.items?.some(item =>
        item.product?.name?.toLowerCase().includes(searchLower)
    )
    if (!matchesOrderId && !matchesCustomer && !matchesProduct) return false
}
