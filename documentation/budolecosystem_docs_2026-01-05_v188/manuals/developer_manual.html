Developer Manual - v188

Issue

Image rotation state was being lost when the document viewer modal was closed and reopened, despite being saved to the database.

Fix Implementation

1. Enhanced fetchUsers

Updated the fetchUsers function to support "silent" refreshes and automatically update the selectedUser state if a modal is open.

const fetchUsers = async (silent = false) => {
  if (!silent) setLoading(true);
  const data = await res.json();
  setUsers(data);
  if (selectedUser) {
    const updated = data.find(u => u.id === selectedUser.id);
    if (updated) setSelectedUser(updated);
  }
};

2. State Synchronization via useEffect

Implemented a useEffect hook to synchronize the rotations state whenever selectedUser changes. This ensures that the UI always reflects the latest persisted data from the database.

useEffect(() => {
  if (selectedUser?.verificationDocs) {
    const initialRotations = {};
    selectedUser.verificationDocs.forEach(doc => {
      initialRotations[doc.id] = doc.rotation || 0;
    });
    setRotations(prev => ({ ...prev, ...initialRotations }));
  }
}, [selectedUser]);

3. Post-Save Refresh

Added a silent data refresh immediately after a successful rotation API call to ensure the local state is in sync with the backend.

Impact

  • User Experience: Rotations are now reliably persisted across sessions and modal closures.
  • Data Integrity: UI state is now driven by the source of truth (database) while maintaining optimistic responsiveness.
