Developer Manual

Technical implementation of SSO restoration and Prisma local fixes.

Purpose

Restore the SSO flow between budolID and budolPay Admin in local development environments by resolving Prisma Client instantiation issues and environment variable misconfigurations.

Technical Details

1. Prisma Client Local Instantiation

The previous implementation was forcing Prisma Accelerate even in local development, leading to connection errors when the Accelerate URL was not available or incorrectly configured.

// Fixed instantiation in budolID and budolPay Admin
export const prisma = globalForPrisma.prisma || new PrismaClient({
    datasources: {
        db: {
            url: process.env.DATABASE_URL, // Explicitly use local DB URL
        },
    },
    log: process.env.NODE_ENV === 'development' ? ['error', 'warn'] : ['error'],
});

2. budolID /auth/verify Logging

Added detailed logging to the verification endpoint to facilitate debugging of token validation failures.
