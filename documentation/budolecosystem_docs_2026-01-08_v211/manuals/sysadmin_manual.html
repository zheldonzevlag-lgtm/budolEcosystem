SysAdmin Manual

Infrastructure and configuration updates for local SSO stability.

Environment Variable Modifications

budolPay Admin (.env)

Updated SSO URLs to point to the local budolID instance instead of Vercel.

Variable New Value
SSO_URL http://localhost:8000
NEXT_PUBLIC_SSO_URL http://localhost:8000
NEXT_PUBLIC_APP_URL http://localhost:3000

Database Configuration

Ensure that the local PostgreSQL instances are running for both services:

  • budolid: postgresql://postgres:r00t@localhost:5432/budolid
  • budolpay: postgresql://postgres:r00t@localhost:5432/budolpay

Deployment Procedures

After updating the Prisma configuration, you must regenerate the Prisma client in both services:

# In budolID-0.1.0
npm run prisma:generate

# In budolpay-0.1.0/packages/database
npm run generate
