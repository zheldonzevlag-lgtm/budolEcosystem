! Implementation Summary

Issue

SSO callback in budolPay Admin was failing with sso_failed and JSON parsing errors due to Prisma Accelerate conflicts and incorrect Vercel URLs in local development.

Fix

Modified Prisma Client instantiation to use local database in development. Updated budolPay Admin environment variables to point to local budolID.

Risk

Bypassing Prisma Accelerate might lead to different performance profiles between local and production environments.

Mitigation

Environment-specific configuration ensures Accelerate is still used in production while local development remains stable.
