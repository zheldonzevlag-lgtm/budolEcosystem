Fixing Interpreter Race Conditions

1. Completer Pattern

In face_embedding_service.dart, we now use a Completer to manage the asynchronous initialization of the TFLite model. This prevents multiple simultaneous loading attempts and ensures callers wait for the model to be ready.

Future<void> _initModel() async {
  if (_loadingCompleter != null) return _loadingCompleter!.future;
  
  _loadingCompleter = Completer<void>();
  try {
    _interpreter = await Interpreter.fromAsset('assets/models/facenet.tflite');
    _isModelLoaded = true;
    _loadingCompleter!.complete();
  } catch (e) {
    _loadingCompleter!.completeError(e);
    _loadingCompleter = null; 
  }
}

2. Synchronized Access

The generateEmbedding method now checks the _isModelLoaded flag and awaits _initModel() if necessary, guaranteeing that the _interpreter is not null when accessed.
