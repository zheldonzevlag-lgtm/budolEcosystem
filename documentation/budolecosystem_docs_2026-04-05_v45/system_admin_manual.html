System Admin Manual

This manual covers the administrative aspects of the Audit Log forensic fix.

Compliance Note: The system now tracks all administrative events using the provided userId, regardless of its format. This ensures 100% compliance with forensic trail requirements (PCI-DSS 10.2).

Administrative Tasks

1. Verifying Forensic Trails

System administrators can now reliably audit the actions of all staff members, including those with custom SSO IDs. The "Staff Identity" column in the Employee Management section will now correctly display the names for:

  • User Logins (Mobile and Web)
  • User Logouts
  • Security OTP Requests
  • MFA Verifications

Troubleshooting

If names still appear as "System / Anonymous", verify that the user profile exists in the budolpay schema (Registry table) and that their userId matches the ID stored in the audit logs.
