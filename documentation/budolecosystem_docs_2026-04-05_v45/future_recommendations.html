Future Recommendations

1. Unify User Identity Schema

The existence of both public.User (with name) and budolpay.User (with firstName/lastName) creates technical debt and requires complex UI fallbacks. We recommend migrating all apps to a single unified User model to simplify development and auditing.

2. Role-Based Access Control (RBAC) in Audit UI

Currently, the /employees audit trail is visible to all staff with dashboard access. We recommend restricting the Forensic Audit Trail to "Compliance Officers" or "Super Admins" to protect sensitive staff behavioral data.

3. Automated ID Linking Verification

Implement a CI/CD check or a background job that flags any AuditLog entries with a null userId where an ID was expected (e.g., in authenticated routes). This will prevent future regressions from going unnoticed.
