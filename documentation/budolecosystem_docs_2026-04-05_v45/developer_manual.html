Developer Manual

This manual covers the technical implementation of the Audit Log Staff Identity fix.

Technical Explanation

The createAuditLog helper in lib/audit.ts previously enforced a strict UUID format for the userId field. This was designed to prevent P2003 Foreign Key Violations by ensuring only valid-looking UUIDs reached the database. However, the system supports custom administrative IDs (e.g., admin-reynaldo-id), which failed this check and resulted in null user identities in logs.

Code Changes

1. Relaxed ID Validation

Location: apps/admin/lib/audit.ts

// v44.1/v45 Fix: Support custom and mixed ID formats
userId: params.userId || null,

2. Resilient Name Rendering (UI Fallback)

Location: apps/admin/app/employees/page.tsx

{log.user ? (
  (log.user.firstName || log.user.lastName) 
    ? `${log.user.firstName} ${log.user.lastName}` 
    : (log.user as any).name || log.user.email
) : (
  "System / Anonymous"
)}
