Service Implementation, Port Assignment & Code References

Microservice Port Configuration

All microservices in the Budol Ecosystem are designed to be environment-aware, defaulting to specific ports while allowing overrides via the PORT environment variable.

API Gateway Configuration

The central entry point routes traffic to downstream services. The default port is 8080.

// Defined in api-gateway/index.js
const PORT = process.env.PORT || 8080;

Core Services Mapping

  • Auth Service (8001): Manages security contexts.
    Reference: auth-service/index.js
  • Wallet Service (8002): Ledger and balance operations.
    Reference: wallet-service/index.js
  • Transaction Service (8003): Core payment processing.
    Reference: transaction-service/index.js
  • Payment Gateway (8004): External provider integration.
    Reference: payment-gateway-service/index.js

Realtime Infrastructure

The ecosystem uses a dedicated Socket.io server for event relaying, operating on port 4000.

// Defined in websocket-server/server.js
const PORT = process.env.PORT || 4000;

Service Implementation Details

Each microservice follows a standard Express.js template with integrated Prisma ORM and cross-service notification logic.

1. API Gateway (Port 8080)

The central hub for all traffic. It handles proxying, mDNS advertisement, and Pusher-to-Socket.io translation.

Code Reference: api-gateway/index.js


// Port Configuration in Gateway
const PORT = process.env.PORT || 8080;
server.listen(PORT, '0.0.0.0', () => {
    console.log(`[Gateway] Service running on http://${LOCAL_IP}:${PORT}`);
});

2. Socket.io Microservice (Port 4000)

A standalone service that listens for triggers from other microservices and broadcasts them to connected clients (Admin Panel, Mobile App).

Code Reference: websocket-server/server.js


// Dedicated Socket.io server logic
const PORT = process.env.PORT || 4000;
server.listen(PORT, () => {
    console.log(`🚀 Socket.io Microservice Running on PORT: ${PORT}`);
});

3. Transaction Service (Port 8003)

Handles the core business logic for orders and payments. It communicates with the Wallet and Accounting services.

Code Reference: transaction-service/index.js


// Default port for Transaction Service
const PORT = process.env.PORT || 8003;

Architecture Patterns

Service Discovery: The API Gateway uses mDNS (Bonjour) to advertise its presence on the network, allowing the BudolPay Mobile app to connect without manual IP entry.
