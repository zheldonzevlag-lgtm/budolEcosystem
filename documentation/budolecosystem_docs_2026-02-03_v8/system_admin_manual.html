The following table lists the default directory locations for each service within the Budol Ecosystem root.

Service Name Directory Location Default Port

Infrastructure & Port Inventory

The Budol Ecosystem relies on a distributed microservice architecture. All services must have their assigned ports open in the local firewall for inter-service communication.

Service Name Port Function Endpoint Example
BudolID (SSO) 8000 Identity Provider & SSO http://localhost:8000/api/auth/session
Auth Service 8001 Token Validation & RBAC http://localhost:8001/verify-token
Wallet Service 8002 Balance & Ledger Management http://localhost:8002/balance/:userId
Transaction Service 8003 Order & Payment Processing http://localhost:8003/api/tx/create
Payment Gateway 8004 External Provider Integration http://localhost:8004/api/payment-gw/process
BudolAccounting 8005 Double-Entry Bookkeeping http://localhost:8005/api/accounting/ledger/entry
Verification (KYC) 8006 Identity Verification (AI/OCR) http://localhost:8006/verify
Settlement Service 8007 Merchant Payouts & Settlement http://localhost:8007/settle
API Gateway 8080 Central Routing & mDNS http://localhost:8080/auth/login
Socket.io Server 4000 Real-time Event Broadcasting http://localhost:4000
Admin Panel 3000 Back-office Management UI http://localhost:3000
BudolShap 3001 E-commerce Marketplace UI http://localhost:3001

Connectivity Requirements

Network Awareness: All services rely on the LOCAL_IP environment variable to communicate over the LAN. Ensure the host firewall allows traffic on the ports listed above.
Port Conflict Warning: BudolShap (3001) and Socket.io (4000) are frequently misconfigured to use overlapping ports. Ensure they are explicitly assigned as per the table above.
