Developer Manual: Navigation Fix v1.2.67

Version: v364

This manual covers the navigation fix implemented in the Marketing Ads screen to ensure users are always directed to the Login UI.

1. Navigation Logic Change

The _proceedToApp method in marketing_ads_screen.dart was modified to explicitly direct users to Routes.login.

void _proceedToApp() async {
  final apiService = context.read<ApiService>();
  await apiService.setHasSeenAds(true);
  
  if (mounted) {
    Navigator.pushReplacementNamed(context, Routes.login);
  }
}

This ensures that even if there is stale authentication state, the user is forced to the login screen for verification before accessing the home screen.

  • api_service.dart - _appVersion = '1.2.67'
