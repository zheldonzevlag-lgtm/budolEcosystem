Operational Guide: Phase 3 Financial Services

1. Deploying the Microservices

Each service in budolpay-0.1.0/services has a vercel.json and should be deployed as a separate Vercel project.

# Example for Wallet Service
cd budolpay-0.1.0/services/wallet-service
vercel link --project budolpay-wallet
vercel env pull .env.vercel
vercel deploy --prod

2. Deploying the Admin Dashboard

The Admin app is a Next.js application located in apps/admin.

cd budolpay-0.1.0/apps/admin
vercel link --project budolpay-admin
vercel deploy --prod

3. Database Synchronization

Run the Prisma migration against the Vercel Postgres instance before first use:

cd budolpay-0.1.0
node scripts/env-manager.js vercel
npm run db:vercel

Risk & Mitigation

  • Risk: Cold starts in serverless functions impacting financial transaction latency.
  • Mitigation: Implement lightweight /health endpoints and use Vercel's "Warm Starts" (if available) or keep-alive pings.
