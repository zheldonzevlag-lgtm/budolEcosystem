Developer Manual v185

Purpose

Phase 3 focuses on the technical implementation of the budolPay financial services suite, ensuring scalable microservice communication and secure SSO integration for the administrative dashboard.

Technical Details

1. Flexible API Gateway

The api-gateway has been updated to support both consolidated (single-domain) and distributed (separate domain) deployments on Vercel. It prioritizes environment variables (e.g., AUTH_SERVICE_URL) over local or consolidated defaults.

// api-gateway/index.js
const isConsolidated = process.env.VERCEL === '1' && !process.env.AUTH_SERVICE_URL;
target: process.env.AUTH_SERVICE_URL || (isConsolidated ? '/api/auth' : `http://${LOCAL_IP}:8001`)

2. SSO Standardization

The Admin app (Next.js) now uses standardized environment variables for SSO communication, matching the ecosystem-wide budolID configuration.

// admin/app/api/auth/login/route.ts
const ssoUrl = process.env.SSO_URL || 'https://budolid.vercel.app';

3. Environment Management

The env-manager.js script was updated to include the apps/admin directory, ensuring centralized environment switching (Local ↔ Vercel) covers the administrative dashboard.

Dependencies

  • budolID (Phase 2) must be deployed for SSO to function.
  • Vercel Postgres (Neon) must be provisioned for the budolpay database.
