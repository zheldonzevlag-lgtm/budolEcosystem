budolEcosystem Documentation v125

Date: 2026-01-04

Focus: Session Race Condition Fix & UI Layout Cleanup

Purpose

To resolve a race condition where ApiService methods were called before the persistent session was fully loaded from storage. Additionally, refined the Payment Summary UI by removing redundant merchant info and prioritizing payment method visibility.

Technical Details

  • Initialization: Added _ensureInitialized() to ApiService to await session loading before processing payments or fetching balances.
  • UI Refinement: Removed "Store Name" from the TRANSACTION DETAILS list (relying on the header display) and moved "Payment Method" to the top of the list.
  • Build: Released as budolPay 1.0.7.apk.

Test Results

Automated tests confirmed that processPayment and getBalance now correctly wait for initialization, preventing the "User session expired" error on cold starts.
