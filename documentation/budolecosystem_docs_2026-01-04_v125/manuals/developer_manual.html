Developer Manual - Async Session Initialization

The Issue

A race condition occurred where SharedPreferences.getInstance() (async) was still running when processPayment was called, leading to a null user object and "session expired" errors.

The Fix

Implemented a Future-based initialization gate in ApiService:

Future<void>? _initFuture;

ApiService(...) {
  _initFuture = _loadSession();
}

Future<void> _ensureInitialized() async {
  if (_initFuture != null) await _initFuture;
}

Future<...> processPayment(...) async {
  await _ensureInitialized(); // Gatekeeper
  if (user == null) throw ...
}
