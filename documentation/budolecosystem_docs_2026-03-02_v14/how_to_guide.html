How-To Guide: ECS Fargate Deployment

Step-by-step guides for deploying and managing the budolEcosystem on AWS ECS Fargate.

HOW_TO_BUILD: Building Docker Images

Prerequisites: Docker installed and running on your machine.

1 Navigate to Service Directory

# Example for budolID service
cd budolID-0.1.0

2 Build the Docker Image

# Build with latest tag
docker build -t budolEcosystem/budolid:latest .

# Build with version tag
docker build -t budolEcosystem/budolid:v1.0.0 .

3 Tag for ECR

# Replace 123456789012 with your AWS account ID
docker tag budolEcosystem/budolid:latest 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com/budolEcosystem/budolid:latest

4 Push to ECR

# Get ECR login password
aws ecr get-login-password --region ap-southeast-1 | docker login --username AWS --password-stdin 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com

# Push the image
docker push 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com/budolEcosystem/budolid:latest

HOW_TO_DEPLOY: Deploying ECS Cluster

1 Deploy CloudFormation Stack

# Deploy the ECS cluster CloudFormation template
aws cloudformation deploy \
  --template-file ecs/cloudformation/budol-ecs-cluster.yaml \
  --stack-name budolEcosystem-ECS \
  --parameter-overrides \
    VpcId=vpc-xxxxxxxx \
    SubnetIds=subnet-xxxxx1,subnet-xxxxx2 \
    DatabaseHost=your-rds-endpoint.amazonaws.com \
    RedisHost=your-redis-endpoint.cache.amazonaws.com \
  --capabilities CAPABILITY_NAMED_IAM

2 Wait for Stack Creation

# Wait for stack to complete
aws cloudformation wait stack-create-complete --stack-name budolEcosystem-ECS

# Check stack status
aws cloudformation describe-stacks --stack-name budolEcosystem-ECS --query 'Stacks[0].StackStatus'

HOW_TO_REGISTER: Registering Task Definitions

1 Register Each Task Definition

# Register budolID task definition
aws ecs register-task-definition \
  --cli-input-json file://ecs/task-definitions/budolid.json

# Register budolAccounting task definition
aws ecs register-task-definition \
  --cli-input-json file://ecs/task-definitions/budolaccounting.json

# Register budolPay Gateway task definition
aws ecs register-task-definition \
  --cli-input-json file://ecs/task-definitions/budolpay-gateway.json

2 Verify Task Definitions

# List all task definitions
aws ecs list-task-definitions --family-prefix budol

HOW_TO_CREATE: Creating ECS Services

1 Create ECS Service

# Create budolID service
aws ecs create-service \
  --cluster budolEcosystem-production \
  --service-name budolid \
  --task-definition budolid:1 \
  --desired-count 1 \
  --launch-type FARGATE \
  --network-configuration "awsvpcConfiguration={subnets=[subnet-xxxxx1,subnet-xxxxx2],securityGroups=[sg-xxxxxxxx]}" \
  --load-balancers "[{targetGroupArn=arn:aws:elasticloadbalancing:ap-southeast-1:123456789012:targetgroup/budolid-tg/xxxxxxxx,containerName=budolid,containerPort=8000}]"

HOW_TO_UPDATE: Updating ECS Services

1 Update Service with New Task Definition

# Update service to use latest task definition
aws ecs update-service \
  --cluster budolEcosystem-production \
  --service-name budolid \
  --task-definition budolid:latest \
  --force-new-deployment

2 Wait for Deployment

# Wait for service to stabilize
aws ecs wait services-stable \
  --cluster budolEcosystem-production \
  --services budolid

HOW_TO_VERIFY: Verifying Deployment

1 Check Service Status

# Check service status
aws ecs describe-services \
  --cluster budolEcosystem-production \
  --services budolid \
  --query 'services[0]'

2 Check Task Status

# List running tasks
aws ecs list-tasks \
  --cluster budolEcosystem-production \
  --service-name budolid \
  --desired-status RUNNING

3 Check Target Group Health

# Describe target health
aws elbv2 describe-target-health \
  --target-group-arn arn:aws:elasticloadbalancing:ap-southeast-1:123456789012:targetgroup/budolid-tg/xxxxxxxx

HOW_TO_DEPLOY_ALL: Full Deployment Script

Using PowerShell (Windows)

# Full deployment
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action all

# Or individual steps
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action build
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action register
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action create
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action deploy

Using Bash (Linux/macOS)

# Full deployment
./scripts/ecs-deploy.sh production all

# Or individual steps
./scripts/ecs-deploy.sh production build
./scripts/ecs-deploy.sh production register
./scripts/ecs-deploy.sh production create
./scripts/ecs-deploy.sh production deploy

HOW_TO_TROUBLESHOOT: Troubleshooting Common Issues

View Container Logs

# Get log events from CloudWatch
aws logs get-log-events \
  --log-group-name /ecs/budolEcosystem-production/budolid \
  --log-stream-name ecs/budolid/xxxxxxxxx

Alternative: Using ECS Exec

# Execute command in container
aws ecs execute-command \
  --cluster budolEcosystem-production \
  --task  \
  --container budolid \
  --interactive \
  --command "/bin/sh"

Check Service Events

# Get service events
aws ecs describe-services \
  --cluster budolEcosystem-production \
  --services budolid \
  --query 'services[0].events'

Restart Service

# Force new deployment (restarts all tasks)
aws ecs update-service \
  --cluster budolEcosystem-production \
  --service-name budolid \
  --force-new-deployment

HOW_TO_ROLLBACK: Rolling Back Deployment

1 List Task Revisions

# List task definition revisions
aws ecs list-task-definitions --family-prefix budolid

2 Update to Previous Revision

# Update service to use previous task definition
aws ecs update-service \
  --cluster budolEcosystem-production \
  --service-name budolid \
  --task-definition budolid:5

HOW_TO_SCALE: Scaling Services

1 Manual Scaling

# Scale service to 3 tasks
aws ecs update-service \
  --cluster budolEcosystem-production \
  --service-name budolid \
  --desired-count 3

2 Configure Auto Scaling

# Register scalable target
aws application-autoscaling register-scalable-target \
  --service-namespace ecs \
  --resource-id service/budolEcosystem-production/budolid \
  --scalable-dimension ecs:service:DesiredCount \
  --min-capacity 1 \
  --max-capacity 10

# Create scaling policy
aws application-autoscaling put-scaling-policy \
  --policy-name budolid-cpu-scaling \
  --service-namespace ecs \
  --resource-id service/budolEcosystem-production/budolid \
  --scalable-dimension ecs:service:DesiredCount \
  --policy-type TargetTrackingScaling \
  --target-tracking-scaling-policy-configuration '{"TargetValue":70,"PredefinedMetricSpecification":{"PredefinedMetricType":"ECSServiceAverageCPUUtilization"}}'

HOW_TO_CLEANUP: Cleaning Up Resources

Warning: This will delete all ECS services and resources. Proceed with caution.

1 Delete ECS Services

# Delete each service
aws ecs delete-service \
  --cluster budolEcosystem-production \
  --service-name budolid \
  --force

aws ecs delete-service \
  --cluster budolEcosystem-production \
  --service-name budolaccounting \
  --force

# ... repeat for all services

2 Delete ECS Cluster

# Delete the cluster
aws ecs delete-cluster \
  --cluster budolEcosystem-production

3 Delete CloudFormation Stack

# Delete CloudFormation stack (will delete ALBs, Target Groups, etc.)
aws cloudformation delete-stack \
  --stack-name budolEcosystem-ECS

# Wait for deletion
aws cloudformation wait stack-delete-complete \
  --stack-name budolEcosystem-ECS

4 Delete ECR Repositories (Optional)

# Delete each ECR repository
aws ecr delete-repository \
  --repository-name budolEcosystem/budolid \
  --force
