Deployment Scripts

ecs-deploy.sh (Bash)

Main deployment script for Linux/macOS environments. Supports the following actions:

  • build - Build and push Docker images to ECR
  • register - Register ECS task definitions
  • create - Create ECS services (first time only)
  • deploy - Deploy/update services
  • status - Check service status
  • wait - Wait for deployment to complete
  • all - Run all deployment steps

Usage:

# Full deployment
./scripts/ecs-deploy.sh production all

# Individual steps
./scripts/ecs-deploy.sh production build
./scripts/ecs-deploy.sh production register
./scripts/ecs-deploy.sh production deploy

ecs-deploy-windows.ps1 (PowerShell)

Main deployment script for Windows environments. Supports the same actions as ecs-deploy.sh.

Usage:

# Full deployment
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action all

# Individual steps
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action build
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action register
.\scripts\ecs-deploy-windows.ps1 -Environment production -Action deploy

build-and-push.ps1

Builds and pushes Docker images to ECR repositories. This script handles:

  • Docker image build for all 9 microservices
  • ECR login authentication
  • Image tagging with version and latest tags
  • Push to ECR repositories

Usage:

.\scripts\build-and-push.ps1 -Service budolid -Tag v1.0.0

ecs-verify.sh

Verifies ECS service status and health. Checks:

  • Service running status
  • Task count and running tasks
  • Last deployment timestamp
  • Health check status

Usage:

./scripts/ecs-verify.sh production
./scripts/ecs-verify.sh production budolid

create-ecr-repositories.sh

Creates all required ECR repositories for the microservices.

Usage:

# Create all repositories
./scripts/create-ecr-repositories.sh

# List existing repositories
./scripts/create-ecr-repositories.sh list

Inter-Service Communication

Internal Communication (Container-to-Container)

Services communicate within the ECS cluster using service names:

# Example: budolPay Gateway calling budolPay Auth
http://budolpay-auth:8001/api/auth/verify

# Example: budolPay Gateway calling budolID
http://budolid:8000/api/sso/verify

External Communication (via Internal ALB)

Services can also be accessed via the Internal ALB using private DNS:

# Example: Accessing budolID via Internal ALB
http://budolid.budol.internal:8000/api/sso/verify

# Example: Accessing budolPay Gateway via Internal ALB
http://budolpay-gateway.budol.internal:8080/api/payment

Environment Variable Configuration

Each service should be configured with the following environment variables for inter-service communication:

INTERNAL_BUDOLID_URL=http://budolid.budol.internal:8000
INTERNAL_BUDOLACCOUNTING_URL=http://budolaccounting.budol.internal:8005
INTERNAL_BUDOLPAY_GATEWAY_URL=http://budolpay-gateway.budol.internal:8080

Docker Image Build Process

Each microservice has its own Dockerfile. The general process for building and deploying:

Build Image

# Navigate to service directory
cd budolID-0.1.0

# Build Docker image
docker build -t budolEcosystem/budolid:latest .

Tag Image

# Tag for ECR
docker tag budolid:latest 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com/budolecosystem/budolid:latest

Push to ECR

# Get ECR login token
aws ecr get-login-password --region ap-southeast-1 | docker login --username AWS --password-stdin 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com

# Push image
docker push 123456789012.dkr.ecr.ap-southeast-1.amazonaws.com/budolecosystem/budolid:latest

Troubleshooting

Docker Desktop - Unexpected WSL Error

If Docker fails to connect with "pipe/docker_engine missing" or a WSL error:

  1. Close Docker Desktop.
  2. Run wsl --shutdown in PowerShell.
  3. Restart Docker Desktop as Administrator.
  4. If the error persists, reboot the computer. This is often necessary when WSL distributions (like docker-desktop) disappear from the output of wsl --list.

Build Failures - Postinstall Scripts

If npm install fails during docker build due to db:generate or other postinstall steps:

  • Use the --ignore-scripts flag during build: RUN npm install --ignore-scripts.
  • Manually trigger required generation steps after copying the source code: RUN npx prisma generate.
