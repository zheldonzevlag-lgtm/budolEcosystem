budolEcosystem Documentation - v246

Version: v246

Date: 2026-01-09

Subject: Balance Update & Backend Communication Fix

Purpose

The primary purpose of this update was to set the balance for user "Jon Galvez" to 50,000,000 PHP as requested and to resolve the FormatException (HTML error) occurring during "Cash In" and "Send Money" operations in the mobile app.

Explanation

The FormatException was caused by the API Gateway proxying requests to Vercel URLs for the wallet and transaction services. These Vercel-hosted services were either returning 404/500 HTML pages or were not pointing to the same local database being used for testing. By switching the gateway to local microservices, we ensure data consistency and valid JSON responses.

Technical Details

  • Database Update: Performed a Prisma upsert on the Wallet model for user Jon Galvez (ID: 9e628dfa-d597-458a-ad6e-b75bc3ee41a3).
  • Gateway Configuration: Modified api-gateway/.env to comment out WALLET_SERVICE_URL, TRANSACTION_SERVICE_URL, etc., triggering the fallback to http://localhost:[PORT].
  • Service Management: Started local instances of api-gateway (8080), wallet-service (8002), and transaction-service (8003).
  • Mobile App: Updated version to 1.2.13 to reflect these environment changes.

Code Changes

Prisma Update Script (update-jon-balance.js)

const updatedWallet = await prisma.wallet.upsert({
  where: { userId: user.id },
  update: { balance: 50000000 },
  create: {
    userId: user.id,
    balance: 50000000,
    currency: 'PHP'
  }
});

Gateway Configuration (.env)

# WALLET_SERVICE_URL="https://budolpay-wallet-service.vercel.app"
# TRANSACTION_SERVICE_URL="https://budolpay-transaction-service.vercel.app"

Test Results

Database: Verified Jon Galvez's balance is exactly 50,000,000.00.

Gateway: Ran Invoke-RestMethod on /test-internal; confirmed connectivity to local services is properly configured.

Build: budolpay_1.2.13.apk generated successfully.

Risk & Mitigation

Risk: Local services (Gateway, Wallet, Transaction) must be manually running for the app to function during testing.

Mitigation: A consolidated startup script is recommended to launch all required microservices simultaneously.

Recommendations

  • Consolidate all microservice startup commands into a single npm run dev:all script in the root directory.
  • Implement automated health checks in the mobile app to warn the user if a specific microservice is unreachable.
