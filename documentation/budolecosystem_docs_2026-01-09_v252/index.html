v252: Realtime Update Settings (SWR, Pusher, Socket.io)

Date: 2026-01-09
Version: v252
Status: Completed
Author: Implementation Planner

Executive Summary

This update introduces a comprehensive Realtime Updates configuration page in the Admin System Settings. Admins can now choose between three synchronization methods: SWR (Polling), Socket.io (Self-hosted), and Pusher (Hosted). This allows for flexible infrastructure management based on scale and budget requirements.

Technical Changes

Admin Interface

  • New Settings Page: Created /settings/realtime/page.tsx with a modern, card-based configuration UI.
  • Sidebar Integration: Added "Realtime Updates" navigation item under System Settings with a dedicated Zap icon.
  • Method Switching: Implemented radio-based selection for SWR, Socket.io, and Pusher with visual feedback.

Backend & API

  • Database Integration: Configured upsert logic to store realtime settings in the SystemSetting table under the 'REALTIME' group.
  • API Gateway Update: Modified /api/system/settings route to expose the current realtime configuration to all ecosystem applications.
  • Configuration Keys: Added support for REALTIME_METHOD, PUSHER_*, SOCKETIO_URL, and SWR_REFRESH_INTERVAL.

Mobile Build (v1.2.17)

  • Version Increment: Bumped pubspec.yaml to 1.2.17+17.
  • APK Location: The installer is saved at /android-installer/budolpay 1.2.17.apk.

Verification Results

  • Settings Persistence: Verified that selecting a method and entering credentials successfully updates the SystemSetting table.
  • API Exposure: Verified that the /api/system/settings endpoint correctly returns the nested realtime configuration object.
  • UI/UX: Verified that the Sidebar correctly highlights the new active route and the settings page handles dynamic inputs.
