Encountered Issues

I-001: Tab-specific Sync Failure

Orders were not being removed from the "Cancelled" tab when their status changed to "Delivered" because the patchOrderCache logic did not account for isCancelledTab filters.

I-002: Missing Real-time Triggers

Operations like trackShipping and cancelShipping were updating the database but not broadcasting real-time events, leading to stale UI.

I-003: Incomplete Event Payloads

Real-time events lacked isPaid and paymentStatus fields, preventing the frontend from correctly placing orders into tabs like "To Ship".

I-004: Rapid-fire Event Duplication

Multiple event channels (order, store, user) were firing identical updates in rapid succession, causing unnecessary UI re-renders and flicker.
