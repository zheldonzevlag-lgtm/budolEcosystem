Developer Manual (v5.3.4)

Data Normalization Logic

The Category Admin UI sends parentId: "" when the user selects "None (Top Level)". In Prisma, a string field with a foreign key constraint must either contain a valid ID or be null. The empty string was causing a P2003 Foreign Key violation.

Fix Implementation

Modified app/api/admin/categories/route.js to normalize the input:

// In POST/PUT handlers
if (validatedData.parentId === '') {
    validatedData.parentId = null
}

// Auto-reset level if top-level
if (validatedData.parentId === null) {
    validatedData.level = 1
}

Zod Schema Update

The schema now explicitly permits null values for the parent identifier:

parentId: z.string().nullable().optional()
