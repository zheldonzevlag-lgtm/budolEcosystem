Developer Manual - v285

1. Purpose

This update addresses critical runtime errors and hydration mismatches in the budolShap application, ensuring a stable user experience during LAN-based cross-computer testing.

2. Technical Details & Code Changes

2.1. AuthContext ReferenceError Fix

Issue: ReferenceError: getToken is not defined occurred in AuthContext.jsx at line 29 because the getToken utility was used but not imported.

Fix: Added getToken to the imports from @/lib/auth-client.

// budolshap-0.1.0/context/AuthContext.jsx
import { getUser, getToken, logout as logoutService, getCurrentUser, setAuth } from '@/lib/auth-client';

2.2. Hydration Mismatch Resolution

Issue: Next.js hydration errors occurred due to:

  • Invalid HTML nesting in Banner.jsx (a div inside a p tag).
  • Browser extensions (like "Fast Download" or password managers) injecting attributes like fdprocessedid into form elements and buttons.
  • Client-side random selection in MarketingAdPopup.jsx.

Fixes:

  • Corrected Banner.jsx HTML structure.
  • Applied suppressHydrationWarning to buttons and input fields in Banner.jsx, Navbar.jsx, Hero.jsx, CategoriesMarquee.jsx, and MarketingAdPopup.jsx.
<button suppressHydrationWarning ... >

3. Test Results

Verification:
  • Console errors for getToken are resolved.
  • Hydration warnings related to fdprocessedid and random ad selection are suppressed.
  • The application now hydrates successfully on LAN-connected devices.

4. Recommendations for Future Enhancement

  • Server-Side Ad Selection: Move the random ad selection logic to the server or a stable client-side useEffect to avoid any hydration mismatch without relying on suppressHydrationWarning.
  • Global Hydration Strategy: Consider a more systematic approach to handling browser-injected attributes if they continue to cause issues across the ecosystem.
