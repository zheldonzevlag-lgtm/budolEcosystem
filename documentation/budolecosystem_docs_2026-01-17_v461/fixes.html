v461 Issue Fix Log

Issue: Persistent Blank Screen after Splash (Web)

Description: Even after DDC recovery, some environments show a blank screen after the HTML splash disappears. This is caused by the Flutter engine failing to initialize or mounting the glass pane too slowly, triggering the HTML timeout before the engine is ready.

Error Detected: Uncaught Library not defined: org-dartlang-app:/web_entrypoint.dart.

Fix: Startup Guard & Diagnostic UI

Solution: 1. Cache Busting: Added ?v=1.3.10 to flutter_bootstrap.js to force browser to fetch the latest engine loader.
2. Error Listener: Added a specialized error listener in index.html that detects DDC/Module errors and presents a "Hard Reload" button to the user.
3. Extended Timeout: Increased splash timeout from 10s to 20s to accommodate slower DDC initialization in debug environments.
4. Startup Logging: Added debugPrint at the very first line of main.dart to verify engine-to-app handoff.

Result: Users now receive immediate visual feedback if the engine fails to load, with a clear path to recovery via hard reload.
