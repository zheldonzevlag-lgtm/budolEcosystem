Developer Manual - v351

This version restores the Marketing Ads display flow and implements persistent tracking to ensure ads are shown appropriately.

  • Updated pubspec.yaml and api_service.dart to version 1.2.57.
  • Implemented persistent hasSeenAds flag in ApiService.

Technical Changes

1. Ads Status Persistence

Added _hasSeenAdsKey and associated getter/setter to ApiService to track whether the user has completed the marketing slide sequence.

Reference: api_service.dart

// api_service.dart
static const String _hasSeenAdsKey = 'budolpay_has_seen_ads';
bool _hasSeenAds = false;
bool get hasSeenAds => _hasSeenAds;

Future setHasSeenAds(bool value) async {
  _hasSeenAds = value;
  final prefs = await SharedPreferences.getInstance();
  await prefs.setBool(_hasSeenAdsKey, value);
  notifyListeners();
}

2. Splash Screen Navigation Flow

Updated SplashScreen to include Routes.marketing in the conditional navigation logic.

Reference: splash_screen.dart

// splash_screen.dart
if (apiService.isAuthenticated) {
  Navigator.pushReplacementNamed(context, Routes.home);
} else if (!apiService.hasSeenAds) {
  Navigator.pushReplacementNamed(context, Routes.marketing);
} else {
  Navigator.pushReplacementNamed(context, Routes.login);
}

3. Marketing Screen Optimization

Updated MarketingAdsScreen to trigger the persistent flag update when the user clicks "Proceed to App".

Reference: marketing_ads_screen.dart

// marketing_ads_screen.dart
void _proceedToApp() async {
  final apiService = context.read();
  await apiService.setHasSeenAds(true);
  // ... navigation logic
}

Risk Mitigation

Issue: Marketing Ads screen was bypassed, leading to loss of user engagement.

Fix: Restored navigation flow and added persistent tracking.

Compliance: User preferences for "seen" status are stored locally, adhering to NPC privacy standards for local data usage.
