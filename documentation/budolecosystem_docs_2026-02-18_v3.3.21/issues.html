Issues Encountered: Audit Logging Implementation

1. Missing Module Error in Tests

Description: Integration tests failed with `Could not locate module @/lib/cart`.

Impact: Blocked verification of audit logging in `ordersService`.

Root Cause: The test file was mocking a non-existent module `@/lib/cart` which was not used by the code under test.

Status: Resolved.

2. Realtime Event Ordering

Description: Inconsistent ordering of realtime events for Store vs User notifications in `returnsService`.

Impact: Potential race condition in frontend UI updates.

Root Cause: Sequential `await` calls instead of `Promise.all` or inconsistent grouping.

Status: Resolved.

3. Cart Items Persistence After Checkout

Description: Items remained in the cart after a successful checkout, causing user confusion and potential double-purchasing risk.

Impact: High - Affects user experience and financial integrity.

Root Cause: Inconsistency in `variationId` handling between `CartItem` (stored as empty string `''` or `null`) and `OrderItem` (sometimes `null`). The deletion query matched strictly on `null` but failed for empty strings.

Status: Resolved with robust query matching.

4. Prisma Invalid Invocation & Token Reference Error

Description: 1. Prisma `deleteMany` failed because `variationId: { in: [null, ''] }` is invalid for nullable fields. 2. `ReferenceError: token is not defined` in `OrderSummary.jsx` rollback logic.

Impact: Order creation failed, and automatic rollback also failed, leaving the system in an inconsistent state.

Root Cause: 1. Prisma does not support mixing `null` and strings in `in` filter for nullable fields. 2. `token` was defined inside `try` block but accessed in `catch` block.

Status: Resolved. Changed Prisma query to use `OR` condition and moved `token` declaration outside `try` block.
