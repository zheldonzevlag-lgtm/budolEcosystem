Developer Manual: Smart Crop & Data Extraction

This version implements advanced client-side image processing to enhance the KYC experience and OCR accuracy.

Technical Architecture

The system uses the Browser's Canvas API to perform real-time image analysis and cropping before the image is passed to the OCR engine.

1. Smart Crop Utility

Located in KYCWizard.jsx, the smartCrop function follows this pipeline:

  • Downsampling: Resizes the image to 400px width for fast pixel analysis.
  • Background Detection: Samples corner pixels to determine the likely background color.
  • Bounding Box Discovery: Scans for pixels that differ from the background by a specific threshold.
  • Safety Validation: Checks if enough "content" pixels were found to avoid cropping into empty space.
  • Final Transform: Performs the high-quality crop on the original image dimensions and returns a Base64 JPEG.
// Example of the safety check in smartCrop
if (detectedPoints < 50 || maxX <= minX || maxY <= minY) {
  console.warn('Smart Crop: No clear ID detected, using original image.');
  return img.src;
}

2. Refined OCR Extraction

The extraction logic now includes specific patterns for Philippine Government IDs:

  • UMID: \b\d{4}-\d{7}-\d{1}\b
  • Driver's License: \b[A-Z]\d{2}-\d{2}-\d{6}\b
  • Voter's ID: \b\d{4}-\d{5}-\d{9}-\d{1}\b
