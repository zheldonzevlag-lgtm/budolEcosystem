PIN Verification Integration

The Face Registration flow in settings_screen.dart now includes a fallback for missing secure PIN storage.

Technical Explanation

When _registerFace() is called, it first checks biometricService.getStoredPin(). If this returns null, the user is prompted with _showPinDialog(). The entered PIN is verified against the backend via apiService.verifyPin() before proceeding with the face scan.

// Verification Logic in SettingsScreen
if (storedPin == null) {
  final pin = await _showPinDialog();
  if (pin != null) {
    await apiService.verifyPin(userId: userId, pin: pin);
    await biometricService.storePin(pin);
    storedPin = pin;
  }
}

Compliance

Adheres to BSP Circular 808 for Multi-Factor Authentication (MFA) and Cybersecurity Law by ensuring that biometric enrollment is always backed by a valid primary credential (PIN).
