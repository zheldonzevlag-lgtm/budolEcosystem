Technical Explanation: Defensive API Handling in Disputes

The Disputes page was failing with TypeError: disputes.map is not a function when the backend service was unreachable or returned an error object. We implemented defensive checks to ensure the data is always treated as an array.

Code Implementation: Disputes Page

File: admin/disputes/page.tsx

// Handle error objects and ensure disputes is an array
if (disputesData.error) {
    console.error('Disputes API Error:', disputesData.error);
    setDisputes([]);
} else {
    setDisputes(Array.isArray(disputesData) ? disputesData : []);
}

Technical Explanation: Standardized Decimal Serialization

Financial data stored as Decimal in the database needs to be explicitly converted to Number for proper rendering in the Admin Dashboard. This version standardizes this conversion across the Disputes and Reconciliation modules.

Code Implementation: Financial Rendering

File: admin/disputes/page.tsx

// Standardized Number conversion for currency rendering
<p className="text-2xl font-black text-slate-900">
    ₱{Number(recon.metrics?.totalRevenue || 0).toLocaleString()}
</p>

<p className="text-sm font-bold text-slate-900">
    ₱{Number(dispute.transaction?.amount || 0).toLocaleString()}
</p>

Technical Explanation: Decimal Serialization

Next.js Server Components cannot pass raw Prisma Decimal objects to Client Components. We implemented explicit mapping to convert Decimal to number during data fetching.

Code Implementation: Admin Dashboard

File: admin/app/page.tsx

// Explicit Decimal to Number Mapping
const serializedTransactions = transactions.map(tx => ({
    ...tx,
    amount: Number(tx.amount)
}));
