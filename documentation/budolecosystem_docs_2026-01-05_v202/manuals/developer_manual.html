New Component: SubmitButton

A reusable client component that automatically handles loading states for forms using Next.js Server Actions.

// apps/admin/components/SubmitButton.tsx
"use client";

import { useFormStatus } from "react-dom";
import { Loader2 } from "lucide-react";

export function SubmitButton({ children, loadingText = "Saving...", ... }) {
  const { pending } = useFormStatus();
  return (
    <button disabled={pending}>
      {pending ? <><Loader2 />{loadingText}</> : children}
    </button>
  );
}

Implementation in Notification Settings

All forms in apps/admin/app/settings/notifications/page.tsx now use this component. The component must be used inside a <form> element to correctly access the form status via useFormStatus.

Code Details

The SubmitButton uses the pending state from useFormStatus to toggle between the default state and the loading state. This is highly efficient as it doesn't require manual state tracking for every form on the page.

Dependencies

  • react-dom: For the useFormStatus hook.
  • lucide-react: For the Loader2 spinner icon.
  • tailwindcss: For styling and animations (animate-spin).
