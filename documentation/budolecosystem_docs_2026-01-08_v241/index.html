v241: Profile Management & Regulatory Compliance

Implementing functional profile editing with built-in safeguards for Philippine financial regulations.

Purpose

This update activates the "Edit Profile" (pencil) button in the mobile app, allowing users to manage their personal information while maintaining strict adherence to BSP Circular 950/1011 and the Data Privacy Act (RA 10173).

Technical Implementation

1. Backend: Profile Update API

Backend Added a secure PATCH /profile endpoint in the auth-service.

Compliance Logic: The endpoint checks the user's kycTier. If the user is FULLY_VERIFIED, it rejects changes to firstName and lastName to preserve the integrity of the verified identity. 
// auth-service/index.js
app.patch('/profile', authenticate, async (req, res) => {
    const { firstName, lastName, email } = req.body;
    if (req.user.kycTier === 'FULLY_VERIFIED') {
        if (firstName !== req.user.firstName || lastName !== req.user.lastName) {
            return res.status(400).json({ error: 'Legal name cannot be changed for verified accounts.' });
        }
    }
    // Update logic...
});

2. Mobile: Edit Profile Screen

Mobile Created EditProfileScreen.dart and integrated it into the app routing.

  • Read-only Fields: Legal name fields are disabled for verified users.
  • Informative UI: Added a Blue Info Box explaining why certain fields are restricted under KYC regulations.
  • Dynamic Validation: Email validation and real-time feedback for field changes.

Regulatory Alignment

Regulation Requirement Implementation Detail
RA 10173 (DPA) Right to Rectification Users can update their contact email and display information at any time.
BSP Circular 950 KYC/CDD Integrity Legal names are locked once FULLY_VERIFIED to prevent unauthorized identity modification.
AMLA (RA 9160) Identity Traceability Ensures that the person performing transactions matches the identity verified during onboarding.

Test Results

  • Basic User Test: Successfully updated First Name, Last Name, and Email. Changes reflected immediately in Settings.
  • Verified User Test: Name fields correctly shown as disabled. Attempting to bypass via API (mocked) returns 400 Bad Request.
  • Validation Test: Invalid email format triggers "Please enter a valid email" error message.
  • Navigation Test: Pencil button in Settings correctly navigates to Edit Profile screen.

Issues & Mitigations

Issue Risk Mitigation
Verified user needs legal name change User frustration/Inability to use app UI provides a clear message to "Contact Support" for manual verification of legal documents (e.g., Marriage Certificate).
Unauthorized profile modification Account Takeover (ATO) Endpoint is protected by authenticate middleware requiring a valid JWT.

Recommendations for Future Enhancement

  • Audit Logging: Implement a UserAuditLog table to track who changed what and when (Critical for BSP audits).
  • OTP for Email Change: Require a one-time password sent to the new email address before finalizing the change.
  • Profile Picture Upload: Integrate image_picker and S3/Cloudinary for actual profile image management.
