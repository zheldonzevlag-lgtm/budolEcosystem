Infrastructure Setup

To deploy the ecosystem, admins must follow the sequence below to ensure secrets and dependencies are met.

1. Shared Environment Secret Management

All projects must share a set of core secrets for integration:

# Core Ecosystem Secrets ECOSYSTEM_SECRET_KEY="..." JWT_SECRET="..." INTERNAL_API_KEY="..."

2. Vercel Project Linking

Each workspace must be linked to its corresponding Vercel project:

# In root directory vercel link --repo budolecosystem # Deploying specific workspaces vercel deploy ./budolID-0.1.0 --name budolid vercel deploy ./budolpay-0.1.0 --name budolpay vercel deploy ./budolshap-0.1.0 --name budolshap

3. Domain & Routing Configuration

Use Vercel rewrites in the main vercel.json to route traffic to sub-services if using a single domain:

{ "rewrites": [ { "source": "/api/auth/:path*", "destination": "https://id.budol.com/:path*" }, { "source": "/api/pay/:path*", "destination": "https://pay.budol.com/:path*" } ] }

Post-Deployment Validation

  • Verify https://id.budol.com/health returns 200 OK.
  • Check Prisma Accelerate metrics for connection pool health.
  • Monitor Sentry for any cross-origin (CORS) or integration errors.
