Budol Ecosystem: Vercel Deployment Strategy v181

Purpose

To provide a structured, phased approach for deploying the entire Budol Ecosystem on Vercel, ensuring high availability, seamless integration, and centralized management.

Phase-Based Implementation

Day 1-2

Phase 1: Core Infrastructure & Shared Services

Establish the foundational data layer required by all applications.

  • Provisioning: Vercel Postgres (Relational), Upstash Redis (Session/Cache), Vercel Blob (Storage).
  • Security: Configure shared JWT_SECRET and encryption keys across all workspaces.
  • Success Metrics: All connection strings verified via prisma db pull from local environment.
Day 3-4

Phase 2: Identity Provider (budolID)

Deploy the central authentication authority.

  • Deployment: Deploy budolID-0.1.0 as a standalone Vercel project (e.g., id.budol.com).
  • Integration: Expose SSO endpoints for other services.
  • Success Metrics: Successful login and JWT issuance from the production URL.
Day 5-7

Phase 3: Financial Core (budolpay)

Deploy the transaction and wallet management system.

  • Architecture: Deploy budolpay-0.1.0 monorepo. Use Vercel's multi-zone or rewrites for sub-services (Auth, Wallet, TX).
  • Integration: Connect to budolID for admin authentication.
  • Success Metrics: Wallet balance queries and mock transaction processing working in production.
Day 8-10

Phase 4: Consumer Ecosystem (budolshap)

Deploy the flagship e-commerce application.

  • Deployment: Deploy budolshap-0.1.0. Configure NEXT_PUBLIC_API_URL to point to the unified gateway.
  • Integration: Finalize Paymongo and Lalamove production credentials.
  • Success Metrics: End-to-end flow from product selection to checkout with BudolPay.
Day 11-12

Phase 5: Supporting Services (Accounting & Loans)

Integrate backend financial tracking and lending.

  • Deployment: Deploy budolAccounting and budoloan.
  • Integration: Enable webhooks from budolpay to budolAccounting for automated ledger entries.
  • Success Metrics: Ledger entries automatically generated for every production transaction.

Risk & Mitigation

  • Risk: Cold start latency for Express apps on Vercel Functions.
  • Mitigation: Optimize bundle sizes and use Edge Runtime where possible for Next.js routes.
  • Risk: Database connection exhaustion.
  • Mitigation: Enforce Prisma Accelerate for all production database interactions.
