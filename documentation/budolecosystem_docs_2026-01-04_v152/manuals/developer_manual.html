Developer Manual - v152

Fixed

Prisma Enum Validation Fix

Purpose: Resolve unhandled runtime error in the budolPay Admin dashboard caused by invalid Prisma invocations.

Explanation: The application was attempting to query and count users with the role STAFF, which was not defined in the UserRole enum within the Prisma schema.

Reason: While database migrations (20251230142429_add_accounting_models) had added the STAFF value to the PostgreSQL type, the schema.prisma file was out of sync, leading to the Prisma Client rejecting STAFF as a valid input.

Technical Details

  • File Modified: budolpay-0.1.0/packages/database/prisma/schema.prisma
  • Action: Added STAFF to UserRole enum.
  • Command Executed: npx prisma generate in packages/database.

Code Changes

// packages/database/prisma/schema.prisma
enum UserRole {
  USER
  ADMIN
  STAFF
  MERCHANT
  DRIVER
}

Impact Analysis

Issues Resolved:

  • Fixed Invalid prisma.user.count() invocation in DashboardPage.
  • Fixed Invalid prisma.user.findMany() invocation in EmployeesPage.
  • Restored functionality of "Operations Staff" count on the dashboard.

Risks & Mitigation: Adding an enum value is a non-breaking change. No data migration was required as the database already supported the value.

Test Results

  • npx prisma generate: SUCCESS
  • Prisma Client updated to v6.2.1 with STAFF support.
  • End-to-End Query Test: PASSED
    • Total Users Count: Success (3 users)
    • Operations Staff Count (ADMIN/STAFF): Success (1 member)
    • Asset Ledger Query: Success
    • Recent Transactions: Success

Verified via budolpay-0.1.0/test_dashboard_queries.js

Recommendations

Ensure that whenever a new role or enum value is introduced in the database via raw SQL or migrations, the schema.prisma is updated immediately to maintain type safety and avoid runtime validation errors.
