Developer Manual - budolPay v1.2.49

Release Overview

This release focuses on improving the user experience for initial app setup by providing API Host Configuration directly on the Login screen, and consolidating UI logic for better maintainability.

Technical Implementation Details

1. Reusable UI Components Feature

Extracted the API Host Configuration dialog from settings_screen.dart into a reusable utility class to support multiple entry points (Settings and Login).

File: budolPayMobile/lib/utils/ui_utils.dart

Logic: Implemented UIUtils.showHostConfigDialog(BuildContext context) which encapsulates:

  • Manual host entry text field.
  • mDNS network scanning via ApiService.discoverAndSetHost().
  • Loading state management during network discovery.
  • Persistence via ApiService.setHost().

2. Login Screen Integration Feature

Added a settings icon to the Login Screen AppBar to allow users to configure the API host before attempting to identify their mobile number. This is critical for environments where the default host is not reachable.

File: budolPayMobile/lib/screens/login_screen.dart

Changes:

  • Added AppBar with Icons.settings action.
  • Set extendBodyBehindAppBar: true to maintain the full-screen background effect.
  • Integrated UIUtils.showHostConfigDialog call.

3. Version Synchronization Maintenance

Incremented version to 1.2.49+49 to track the new UI capability.

Files: pubspec.yaml, api_service.dart
