Ecosystem Consolidation Strategy

This manual details the architectural shift from 11 individual Vercel projects to a single consolidated budolecosystem project using path-based routing.

Consolidated

Path-Based Routing Map

The following table defines how the root budolecosystem domain routes to individual services:

Service Path Internal Destination Function
/api/gateway/* services/api-gateway/index.js Primary Entry Point & Socket Server
/api/auth/* services/auth-service/index.js Identity & Authentication
/api/wallet/* services/wallet-service/index.js Digital Wallet Management
/api/tx/* services/transaction-service/index.js Transaction Processing
/api/payment-gw/* services/payment-gateway-service/index.js External Payment Integration
/api/kyc/* services/verification-service/index.js User Verification (KYC)
/api/accounting/* budolAccounting-0.1.0/index.js Financial Ledger Service
/api/id/* budolID-0.1.0/index.js Budol Identity Service
/admin/* apps/admin/package.json Admin Management Portal (Next.js)
/beta/* budolshap-0.1.0/package.json BudolShap Beta (Next.js)

Deployment Instructions

To deploy the entire ecosystem, run the following command from the root directory:

vercel deploy --prod

This will rebuild all services and deploy them under a single production URL.
