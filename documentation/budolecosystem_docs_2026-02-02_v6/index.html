Release v2.5.0: Bootstrap Audit & Jest Verification

This release introduces forensic auditing for bootstrapping scripts and a comprehensive Jest-based verification suite for the Forensic Audit Trail, ensuring high-fidelity compliance monitoring.

Implementation Task List

  • Phase 0: Identify missing audit logs in bootstrapping scripts
  • Phase 2: Implement forensic audit logging in create-admin-account.js
  • Phase 10: Create Jest test suite for Forensic Audit Trail verification
  • Phase 5: Run Jest tests and verify all critical events are captured
  • Phase 6: Documentation & Audit (v2.5.0)

Core Objectives

  • Bootstrap Integrity: All administrative account creations via the CLI are now forensically logged.
  • Automated Verification: Integrated Jest tests to ensure the Forensic Audit Trail schema and data contracts remain compliant.
  • PCI DSS 10.2.2: Verified logging of all administrative actions, including those performed outside the UI.

Compliance Reference

  • PCI DSS 10.2.2: All actions taken by any individual with root or administrative privileges must be logged.
  • BSP Circular 808: Banks must maintain audit trails that are sufficient to enable the reconstruction of events.
