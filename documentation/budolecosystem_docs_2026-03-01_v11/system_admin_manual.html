System Admin Manual v5.4.0

Administering verified staff registry edits and security policies.

1. OTP Storage & Expiry

OTPs for staff edits are currently stored in a volatile in-memory store in the `apps/admin/app/api/employees/route.ts` instance. If the service restarts, existing OTPs will be cleared.

  • Expiry: 5 Minutes.
  • Cooldown: 60 Seconds before resend is enabled.

2. Forensic Audit Log Integration

All MFA-related actions (edit request, verification success/failure) are logged to the Forensic Audit Trail. These logs are stored permanently and cannot be tampered with by administrators.

Forensic Resource Table:

  • EDIT_MFA_REQUESTED: Occurs when the Admin clicks "Edit Profile".
  • EDIT_MFA_VERIFIED: Occurs after successful OTP entry.
  • USER_PROFILE_UPDATED: Occurs when "Update Profile" is clicked in the secondary modal.
