Developer Manual v5.4.0

Administrative Action Verification and MFA Implementation.

1. OTP Verification Flow

We've added a security gate (OTPVerificationModal) that must be passed before the EditUserModal is shown.

API Endpoints:

  • POST /api/employees { action: 'SEND_EDIT_OTP', userId: '...' }: Generates a 6-digit code and stores it in-memory. In development, it prints the OTP to the console.
  • POST /api/employees { action: 'VERIFY_EDIT_OTP', userId: '...', otp: '...' }: Validates the code.

Frontend Changes:

  • OTPVerificationModal.tsx: Interstitial modal for entering codes.
  • EmployeesPage.tsx: State logic to handle the sequence.

2. Phone Number Normalization

In EditUserModal.tsx, we've implemented normalizePhone() to convert Philippine formats to standard E.164 (+63...).

const normalizePhone = (phone: string) => {
    let digits = phone.replace(/\D/g, '');
    if (digits.startsWith('0') && digits.length === 11) {
        return '+63' + digits.substring(1);
    }
    // ...
};
