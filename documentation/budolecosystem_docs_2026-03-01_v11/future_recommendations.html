Future Recommendations v5.4.0

Recommendation 1: Move OTP to Redis Store

Rationale: Currently, OTPs are stored in memory within the `budolPay` admin app API route. While functional for v5.4.0, a Persistent Redis store would allow for multi-instance scaling and better reliability on server crashes.

Priority: High

Recommendation 2: Adaptive Authentication (Admin Risk)

Rationale: If the Admin is attempting multiple edits in a short time frame, we could implement a "Single-Authorization Window" (e.g., 10 minutes) instead of an OTP for every single user edit to improve administrative speed.

Priority: Medium

Recommendation 3: WebAuthn / Passkeys Support

Rationale: To fully satisfy "Hardware Security Keys enforced for Admins" policy (v2.1), we should integrate WebAuthn (Passkeys) for staff edits, replacing OTP with a physical touch-to-approve on the staff's device.

Priority: High

Recommendation 4: Global Normalization Service

Rationale: Rather than individual normalization in frontend components, we should centralize the `normalizePhone()` helper in a shared package or as a middleware in the API layer.

Priority: Medium
