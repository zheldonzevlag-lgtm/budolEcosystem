Issues & Fixes v5.4.0

1. UI Flow: Direct Edit Access without MFA Check

Description: Previously, any logged-in administrator could click the "Edit" button and immediately view and modify staff profiles. This violated the "MFA Enforcement" (v2.1) policy for sensitive actions.

FIX: Verification Interstitial Modal Implementation

Mechanism: Integrated OTPVerificationModal. The "Edit" click handler now triggers the SEND_EDIT_OTP action and opens the verification interstitial. The Edit User Profile modal is only opened upon onVerified() callback.

2. Data Quality: Non-standard Phone Formats

Description: Admin-entered phone numbers (e.g. 0917-123-4567) were stored "as-is" in the database, leading to synchronization failures between budolPay and budolID (which expects E.164).

FIX: International Normalization on Blur

Mechanism: Added normalizePhone() utility which strips non-digits and prepends +63. Applied to the phone number input's onBlur event to ensure all data is internationalized before submission.
