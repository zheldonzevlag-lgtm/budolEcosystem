SysAdmin Manual - v295

Environment Management

This version introduces better management of environment override files during deployment and local development setup.

Automated Synchronization

Run the following command to synchronize the current host's LAN IP across all services:

npm run network:sync

This command now performs the following actions:

  • IP Detection: Automatically identifies the primary LAN IPv4 address.
  • Multi-file Update: Updates .env, .env.local, and .env.development across all sub-folders.
  • Service URL Sync: Propagates the IP to SSO, App, and Gateway URL variables.
  • Mobile Build Skip: Skips the resource-intensive mobile build process by default.

JWT Configuration

Ensure that JWT_SECRET is defined in the root .env file. The synchronization script will propagate this value if configured, though it primarily focuses on IP-related variables. For security, ensure the same secret is used across all production nodes.

Database Requirements

After running npm run network:sync, it is recommended to run node scripts/update-db-ips.js (which is usually triggered automatically) to ensure the database's redirect_uri entries are up to date.
