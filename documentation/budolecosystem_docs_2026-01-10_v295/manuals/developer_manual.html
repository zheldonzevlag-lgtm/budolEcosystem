Developer Manual - v295

JWT Security Audit

A comprehensive audit was performed on all package.json start scripts and environment files to ensure consistent use of JWT_SECRET. The ecosystem now relies on a unified secret for cross-service authentication.

  • Verification: All services (budolID, budolPay, budolShap) correctly ingest JWT_SECRET from their respective .env files.
  • Discovery: Identified hardcoded secrets in certain test scripts. While non-critical for development, these are marked for future migration to environment variables.

Network Synchronization Enhancements

The scripts/network-util.js utility was refactored to be more robust and inclusive of modern Next.js environment patterns.

Implementation Details

The script now scans for a wider range of environment files:

const envFileNames = ['.env', '.env.local', '.env.development', '.env.development.local'];

Additional frontend-specific variables are now synchronized to maintain routing integrity on LAN:

  • NEXT_PUBLIC_BASE_URL
  • NEXT_PUBLIC_SITE_URL

Impact Analysis

  • Development Speed: Synchronization is now faster due to skipped mobile builds.
  • Local Overrides: .env.local files are now correctly updated, preventing "ghost" localhost redirects that were previously missed.
  • Auth Consistency: Guaranteed token validation across the entire ecosystem.
