Future Recommendations & Enhancements

While the v32 sync resolves immediate production blockers, the following architectural upgrades are recommended for future stabilization cycles.

1. GitHub Actions CI/CD HIGH

Automate the build_android_vercel.ps1 process on merge to main. This ensures APKs are always in sync with backend deployments.

2. Pusher App Key Sync

Add Pusher credentials to the Vercel env vars list to enable real-time notifications on mobile for the production environment.

3. Unified Local/Prod Config

Migrate all mobile authentication logic into a shared package (e.g., @budolpay/auth-bridge) to prevent logic duplication between Express and Next.js.

4. Play Store Alpha Track

Begin distributing the v1.3.73 APK via the Google Play internal testing track to leverage automated crash reporting and version management.

Consolidated Recommendations

All future enhancements from the v32 sync are logged in the central knowledgebase.html and should be prioritized in the next maintenance sprint.
