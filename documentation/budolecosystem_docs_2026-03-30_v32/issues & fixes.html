Issues & Fixes: v32 Release

FIXED

1. "Account not found" on Vercel App

PROBLEM: The mobile app was unable to log in when pointing to https://budolpay.vercel.app, even though the user existed in the database.

ROOT CAUSE: The check-phone endpoint was missing from the Next.js API router on Vercel. Additionally, the phone formats in the DB (09...) mismatched the app's normalized input (+63...).

FIX: Implemented apps/admin/app/api/auth/check-phone/route.ts with a multi-format search engine. Added search capability for `09`, `+63`, and bare `9` formats.

FIXED

2. Vercel Schema Mismatch

PROBLEM: Querying the search endpoint resulted in database errors or "exists: false" because it was defaulting to the public schema.

ROOT CAUSE: The DATABASE_URL provided to Vercel was missing the explicit &schema=budolpay parameter.

FIX: Updated apps/admin/push_env.cjs to include the mandatory schema parameter for all production database connections.

FIXED

3. Missing Auth Bridges (PIN/MFA)

PROBLEM: Mobile login identification and PIN verification were only functional in the local Express microservice.

FIX: Ported identify, verify-pin, and setup-pin routes to Next.js. Incorporated bcryptjs for secure hash comparison in serverless environments.
