Technical Walkthrough: Mobile Vercel Bridge

1. Problem Discovery: The "Account not found" Mystery

Users like Tony Stark were seeing "Account not found" on the mobile app when connected to Vercel, despite their records existing in the database.

Discrepancy Analysis
  • Mobile App: Normalizes inputs to +63....
  • Vercel DB: Stores many records in local 09... format.
  • Vercel API: Returns 404 Not Found because the specific check-phone route was missing from Next.js.

2. Solution: The Next.js API Bridge

We implemented a robust normalization layer in the new Next.js routes to ensure multi-format compatibility.

Smart Phone Search Logic

The new check-phone route now performs an OR search across three formats:

WHERE phoneNumber IN (input, normalized_09, normalized_+63)

3. Handshake Flow (Vercel Support)

The login flow now matches the local Express behavior precisely:

  1. Identify: Checks existence and device trust. Triggers OTP if new device.
  2. Verify PIN: Authenticates with hashed PIN and issues 30-day JWT.
  3. Setup PIN: Allows seamless onboarding for new mobile accounts.

Verification Proof

Routes have been verified in the admin app structure. Local development is protected because the mobile app continues to use port 8001 for local connections, while the Vercel path is now fully populated.
