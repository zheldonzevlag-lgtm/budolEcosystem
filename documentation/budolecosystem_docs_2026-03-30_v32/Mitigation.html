Mitigation Strategies: v32 Production Release

M1: Schema Drift Mitigation

ISSUE: Local dev uses different schemas than Vercel production by default.

MITIGATION: Standardized DATABASE_URL in push_env.cjs to include &schema=budolpay. Added verification scripts to confirm column existence before app logic execution.

M2: "Account not found" Prevention

ISSUE: Philippine phone number formats (+63 vs 09) vary between app input and DB storage.

MITIGATION: Implemented a "Smart Normalizer" in the check-phone API bridge. The query now performs an OR search across 4 formats: exact, 09x, +639x, and bare 9x.

M3: Local Dev Stability

ISSUE: Porting Vercel code might break local Express services.

MITIGATION: New API bridges are "Passive Additions" in the admin Next.js app. The mobile app continues to prioritize port 8001 for local connections, ensuring zero impact on development stability.
