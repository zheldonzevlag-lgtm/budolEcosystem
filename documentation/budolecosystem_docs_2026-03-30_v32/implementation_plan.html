Implementation Plan: v32 Mobile Vercel Sync

Goal: Port mobile authentication endpoints from Express (Local) to Next.js (Vercel) to fix "Account not found" and 404 errors.
Proposed Changes

Phase 1: API Discovery & Gap Analysis

Identified that /api/auth/check-phone and /login/mobile/* were missing from the Next.js App Router on Vercel.

Phase 2: Porting API Routes

Implement precisely matching logic in Next.js using NextResponse and prisma:

  • check-phone: Added multi-format phone search (09, +63, and exact).
  • identify: Added Device Trust (trustedDevices) and OTP triggering logic.
  • verify-pin: Added PIN comparison with bcryptjs and 30-day JWT issuance.
  • setup-pin: Added secure PIN hashing and initial setup flow.

Phase 3: Dependency Management

Added bcryptjs and jsonwebtoken to the admin app to support secure auth operations on Vercel.

Verification Plan

Each route has been manually verified against the Vercel schema constraints. Local dev stability is ensured by maintaining the original Express ports.

Critical Reminder: Ensure the JWT_SECRET env var is set correctly on Vercel to match the Express service secret if cross-compatibility is required.
