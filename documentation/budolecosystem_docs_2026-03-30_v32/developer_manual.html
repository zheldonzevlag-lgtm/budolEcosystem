Developer Manual: Vercel API Bridges

1. check-phone Endpoint

Path: apps/admin/app/api/auth/check-phone/route.ts

This endpoint checks if a phone number exists in the database. It normalizes numbers from +63 to 0 and vice versa to ensure compatibility with legacy records.

2. login/mobile/identify Endpoint

Path: apps/admin/app/api/auth/login/mobile/identify/route.ts

Handles the initial login step. Detects if the device is trusted. If not, triggers a dual-channel OTP (SMS/Email) via the @budolpay/notifications package.

3. login/mobile/verify-pin Endpoint

Path: apps/admin/app/api/auth/login/mobile/verify-pin/route.ts

Authenticates the user using a 6-digit PIN. If successful, issues a 30-day JWT token and updates the trustedDevices list.

4. login/mobile/setup-pin Endpoint

Path: apps/admin/app/api/auth/login/mobile/setup-pin/route.ts

Allows users to set up a new PIN after OTP verification. Hashes the PIN using bcryptjs with 10 salt rounds.
