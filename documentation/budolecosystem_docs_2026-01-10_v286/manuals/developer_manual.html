Developer Manual Update - v286

Purpose

To ensure that the primary test users (Jon, Tony, and Peter) are fully verified in the system and that Jon has sufficient funds for testing large transactions.

Technical Implementation Details

User Verification

The following fields were updated for users Jon, Tony, and Peter in the User model:

  • kycStatus: Set to VERIFIED
  • kycTier: Set to FULLY_VERIFIED
  • isFaceVerified: Set to true
  • emailVerified: Set to true
  • phoneVerified: Set to true

Wallet Funding

Jon Galvez's wallet balance was updated to 100,000,000.00 PHP using the Wallet model's upsert operation to ensure a wallet exists if it didn't previously.

Code Details

The update was performed via a custom script verify-and-fund.js using the Prisma Client:


// Verification logic
await prisma.user.update({
  where: { id: user.id },
  data: {
    kycStatus: 'VERIFIED',
    kycTier: 'FULLY_VERIFIED',
    isFaceVerified: true,
    emailVerified: true,
    phoneVerified: true
  }
});

// Funding logic
await prisma.wallet.upsert({
  where: { userId: user.id },
  update: { balance: 100000000 },
  create: {
    userId: user.id,
    balance: 100000000,
    currency: 'PHP'
  }
});
