Developer Manual - v391

Application Versioning Strategy

Version 391 standardizes the versioning for the next Android release (1.2.88). The version is tracked in two primary locations to ensure both build tools and runtime logic are synchronized.

1. Pubspec.yaml Configuration

The build version and build number are updated in the project manifest:


// [pubspec.yaml](file:///d:/IT%20Projects/budolEcosystem/budolPayMobile/pubspec.yaml#L4)
version: 1.2.88+88

2. ApiService Runtime Version

The ApiService maintains a runtime version string used for API headers and display logic:


// [api_service.dart](file:///d:/IT%20Projects/budolEcosystem/budolPayMobile/lib/services/api_service.dart#L22)
String _appVersion = '1.2.88'; // Android Build v1.2.88

Splash Screen Version Rendering

The version is dynamically retrieved from ApiService and displayed at the bottom of the splash screen:


// [splash_screen.dart](file:///d:/IT%20Projects/budolEcosystem/budolPayMobile/lib/screens/splash_screen.dart#L94-L100)
Text(
  'budolEcosystem $appVersion',
  style: const TextStyle(
    color: Colors.white70,
    fontSize: 14,
    fontWeight: FontWeight.w500,
    letterSpacing: 1.2,
  ),
)

Build Instructions

To generate the production APK, run the following command from the budolPayMobile directory:

flutter build apk --release

Once built, the APK should be moved to the android-installer directory and renamed to budolPay 1.2.88.apk.
