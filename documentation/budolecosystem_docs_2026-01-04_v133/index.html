Wallet Auto-Creation

Centralized Configuration

The SystemSetting model in BudolPay now serves as the single source of truth for ecosystem configuration. Variables can be scoped to specific applications (appId) or shared globally. 

// Fetching app-specific config via API Gateway
GET http://localhost:8080/config/:appId

// Response Format
{
  "DATABASE_URL": "...",
  "API_URL": "...",
  "VERCEL": "1"
}

BudolShap & BudolID

ReadOnly Config Visibility

Implemented read-only environment settings pages in both BudolShap and BudolID admins. This allows developers and admins to verify active configuration without needing direct server access.

CLI Environment Tooling

New scripts have been added to the root package.json to enable rapid switching between environments across all microservices simultaneously.

// Switch everything to Local DB

npm run db:local

// Switch everything to Production DB

npm run db:prod

// Switch everything to Vercel-ready config

npm run db:vercel

Note: These scripts update .env or .env.local files in BudolPay, BudolShap, and BudolID by merging .env.connection.* templates.
