Identified Issues

Issues discovered during the v496 system audit.

1. Missing Store Data in Session

Symptom: Store owners were not receiving realtime notifications for their orders, even when Pusher was active.

Root Cause: The /api/auth/me endpoint did not include the store.id in the returned user object, preventing RealtimeProvider.jsx from subscribing to the correct private channel (store-{id}).

2. Realtime Event Inconsistency

Symptom: Some frontend components were listening for order-completed while others expected order-updated, leading to inconsistent UI updates.

Root Cause: Lack of normalization in ordersService.js and returnsService.js regarding event naming for order status changes.

3. Verification Script Environment

Symptom: Verification scripts failing due to ReferenceError: jest is not defined or ESM module errors.

Root Cause: Scripts were being run as CommonJS in an ESM-first project, or were relying on testing frameworks (Jest) not configured for standalone CLI execution.

2. Admin Settings Cache Delay

Symptom: Changes to the realtime provider in the admin panel take up to 60 seconds to reflect for users.

Root Cause: settingsCache in lib/realtime.js and SWR dedupingInterval in RealtimeProvider.jsx are set to 60 seconds to optimize database and network performance.
