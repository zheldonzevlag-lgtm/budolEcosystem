Version 3.3.22 - Geocoding API & MapPicker Fix

Purpose & Rationale

This release addresses a "Failed to fetch" error in the MapPicker and AddressAutocomplete components. The issue was caused by client-side requests to Nominatim failing due to missing User-Agent headers (required by Nominatim) or CORS restrictions. A new server-side Geocoding API has been implemented to handle these requests securely and reliably.

Technical Summary

  • New API Endpoint: /api/system/geocode handles both forward and reverse geocoding.
  • Nominatim Compliance: The API sends a valid `User-Agent` header to comply with OpenStreetMap's usage policy.
  • Frontend Updates: MapPicker and AddressAutocomplete components now call the internal API instead of external services directly, solving CORS issues.
  • Provider Abstraction: The API handles switching between Nominatim and Geoapify based on system settings.
