Developer Technical Guide

1. Authentication Flow (Mobile Identify)

The identifyMobile endpoint is the first step in the login flow for the Flutter Mobile app (Web and Native). It checks if a user exists and if the device is trusted.

Code Reference: auth-service/index.js


// identifyMobile Endpoint
router.post('/login/mobile/identify', async (req, res) => {
    const { phoneNumber, deviceId } = req.body;
    // ... validation and normalization ...
    try {
        let user = await prisma.user.findFirst({
            where: {
                OR: [
                    { phoneNumber: phoneNumber },
                    { phoneNumber: normalizedPhone },
                    { phoneNumber: '+63' + normalizedPhone.substring(1) }
                ]
            }
        });
        // ... device trust logic ...
        if (!isDeviceTrusted || !user.pinHash) {
            // Trigger OTP flow
            await sendOTP(user.phoneNumber, otpCode, 'SMS');
            return res.json({ status: 'OTP_REQUIRED', ... });
        }
    } catch (error) {
        console.error('[Identify Error]', error);
        res.status(500).json({ error: error.message });
    }
});

2. Prisma Schema Synchronization

The SystemSetting model was updated to include appId. Developers must ensure the database is synced after schema changes.

Code Reference: packages/database/prisma/schema.prisma


model SystemSetting {
  id          String   @id @default(uuid())
  key         String   @unique
  value       String
  group       String?
  appId       String?  // New field added
  isSecret    Boolean  @default(false)
  isActive    Boolean  @default(true)
  description String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

3. Notification Service Integration

The sendOTP function retrieves settings from the SystemSetting table. If the database schema is not aligned, this function will fail, causing a 500 error in the Auth Service.

Code Reference: packages/notifications/index.ts


async function getSetting(key: string, defaultValue: string = ''): Promise {
  const setting = await prisma.systemSetting.findUnique({
    where: { key },
  });
  return setting?.value || defaultValue;
}

4. Flutter API Service

The Flutter web app uses the current window host to determine the Auth Service URL.

Code Reference: budolPayMobile/lib/services/api_service.dart


static String computeAuthUrlFromHost(String rawHost) {
  final hostOnly = _hostOnly(rawHost);
  return 'http://$hostOnly:8001';
}

5. Compliance and Data Privacy

  • NPC (Data Privacy Act): User phone numbers and personal details are handled securely. Error logs do not expose sensitive PII (only system-level errors).
  • Cybersecurity Law: Multi-factor authentication (OTP) is enforced for untrusted devices to prevent unauthorized access.
  • BSP/PCI DSS: Authentication flows follow standard security practices for financial applications.
