System Administration Tasks

1. Database Schema Synchronization

If the application returns 500 errors related to missing columns (e.g., SystemSetting.appId), the administrator must sync the database schema using Prisma.

cd budolpay-0.1.0/packages/database
npx prisma db push

2. Regenerating Prisma Client

After a schema change, the Prisma Client must be regenerated to update the application's internal models.

cd budolpay-0.1.0/services/auth-service
npm run generate

Note: Ensure no Node.js processes are locking the query engine files during this process.

3. Verifying Service Connectivity

Check if the Auth Service is reachable on the local network:

curl http://192.168.1.17:8001/health

4. Log Monitoring

Monitor the Auth Service logs for identification errors:

# Logs will show:
[Identify Error] PrismaClientKnownRequestError: ...
