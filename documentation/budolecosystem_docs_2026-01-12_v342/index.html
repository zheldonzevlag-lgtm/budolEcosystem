Release Overview - v342

Purpose: Resolve "Identification failed: 500" error during Flutter Web login flow.

Rationale: The login flow was failing due to a Prisma schema mismatch where the SystemSetting.appId column was missing from the physical database, causing notification service queries to crash. Additionally, the Prisma Client in the auth-service was outdated and missing the User model.

Technical Explanation

The identifyMobile endpoint in the Auth Service triggers an OTP generation flow. This flow requires querying SystemSetting to determine notification providers. The mismatch between the Prisma schema and the actual PostgreSQL database led to a 500 error. By syncing the schema (prisma db push) and regenerating the Prisma Client, the database-to-application layer was restored.
