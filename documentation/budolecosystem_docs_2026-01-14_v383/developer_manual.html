Developer Manual - Mobile Build v383

Technical Explanation

The Android build process utilizes the Flutter framework. Versioning is controlled via pubspec.yaml and reflected in the ApiService fallback for health checks. Additionally, the debug infrastructure was enhanced to provide full-screen logging visibility for troubleshooting biometric services.

Debug Window Enhancement

The Face Service Debug Window was redesigned to occupy the full screen for improved readability and diagnostics. Key improvements include:

  • Full-Screen Layout: Replaced Positioned with Positioned.fill to maximize viewing area.
  • Error Highlighting: Added logic to detect "error", "failed", or "exception" strings in logs and highlight them in Colors.redAccent.
  • Model Path Info: Added a footer showing the active TFLite model path (assets/models/facenet.tflite).
  • Safe Area Integration: Wrapped the window in a SafeArea to avoid interference with device notches and status bars.

Known Issues: TFLite Initialization

During the v383 build verification, a critical issue was identified with the facenet.tflite asset. The file currently exists as a 66-byte placeholder, causing the Interpreter.fromFile call to fail with Invalid argument(s): Unable to create model from file.

Impact: Face registration and verification are currently non-functional in this build.

Remedy: See Mitigation Plan (M-006) for model restoration steps.

Phase 8: Dependencies & Software Rationale

Flutter 3.10.4: Chosen for its stable rendering engine and mature support for TFLite via google_mlkit_face_detection. The specific version ensures compatibility with existing budolPay legacy code while allowing for modern null-safety.

Google ML Kit: Selected for its robust on-device face detection capabilities, minimizing latency and ensuring data privacy (NPC compliance) by avoiding cloud-based biometric processing.

Gradle 8.0+: Used for the Android build process to support modern Android SDK features and improved build performance for large Flutter projects.

Path Provider: Essential for the "Nuclear Option" (M-004) where TFLite models are extracted to temporary directories to bypass Android asset locking issues.
