Developer Manual - SSO & LAN Architecture

1. SSO Architecture & LAN Connectivity

Dynamic IP Resolution

To support LAN-based testing (e.g., mobile devices accessing the server), we use a dynamic LOCAL_IP environment variable. This ensures that redirect URIs seeded in the database match the current host IP.

// budolID-0.1.0/index.js app.listen(PORT, '0.0.0.0', () => { const localIP = process.env.LOCAL_IP || '192.168.1.2'; console.log(`Local LAN access at http://${localIP}:${PORT}`); });

Reference: budolID-0.1.0/index.js

2. Database Schema Isolation

PostgreSQL Schemas

BudolID now uses the budolid schema to prevent table name collisions with budolShap in the shared budolshap_1db database.

# .env for budolID DATABASE_URL="postgresql://user:pass@localhost:5432/budolshap_1db?schema=budolid&search_path=budolid"

Reference: budolID-0.1.0/.env

3. Orchestration & Environment Hygiene

Env Leakage Protection

The central orchestrator start-ecosystem.js now cleans up sensitive environment variables before spawning child processes. This prevents the parent process's DATABASE_URL or PORT from overriding the individual service configurations.

// start-ecosystem.js cleanup logic const cleanEnv = { ...process.env }; const varsToRemove = ['DATABASE_URL', 'NODE_ENV', 'PORT', 'JWT_SECRET', 'LOCAL_IP']; varsToRemove.forEach(v => delete cleanEnv[v]);

4. Dependencies & Rationale

  • Node.js & Express: Microservices framework.
  • Prisma: Type-safe database access with schema support.
  • BcryptJS: Secure password hashing.
  • JWT: Stateless authentication for SSO.
