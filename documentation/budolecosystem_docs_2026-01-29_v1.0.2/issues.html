Issues & Resolutions

Issue: Prisma Schema Conflict

Description: BudolID and BudolShap both used the public schema, causing table name collisions during prisma db push.

Fix: Configured BudolID to use a dedicated budolid schema in the connection string and Prisma schema.

Issue: Environment Variable Leakage

Description: The central orchestrator leaked its own DATABASE_URL and PORT to child processes, causing services to fail to load their own configs.

Fix: Modified start-ecosystem.js to sanitize the environment before spawning child processes.

Issue: SSO Redirect URI Hardcoding

Description: Seeded redirect URIs used a static IP (192.168.1.24) that didn't match the current host IP.

Fix: Implemented LOCAL_IP environment variable and updated seed.js to use it dynamically.
