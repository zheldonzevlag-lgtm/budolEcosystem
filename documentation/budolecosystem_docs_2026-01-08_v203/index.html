BudolEcosystem Documentation v203

Security & Session Enforcement: Forced PIN Setup Flow

← Previous (v202) Main Index Next (v204) →

1. Purpose

This update addresses a critical security bypass where users could access the dashboard and wallet features even without a PIN set. It ensures that the "PIN not set" state is strictly enforced across the API Gateway and Auth Service.

2. Technical Explanation

Previously, verify-otp issued a full 30-day access token upon successful OTP verification, regardless of whether a PIN existed. This allowed the mobile app to cache the token and bypass the PIN screen on subsequent app launches.

Changes Implemented:

  • Auth Service: Updated verify-otp to include a hasPin claim in the JWT. If hasPin is false, the token's lifetime is restricted to 10 minutes.
  • API Gateway: Implemented a verifyToken middleware that intercepts all proxied requests. If a mobile user attempts to access non-auth routes with a hasPin: false token, the gateway rejects the request with a 403 Forbidden status and a PIN_SETUP_REQUIRED code.
  • Auth Service: Updated setup-pin to issue a new, full-privilege 30-day token upon successful PIN creation.

3. Technical Details

JWT Payload Update

{
  "userId": "...",
  "role": "USER",
  "type": "MOBILE",
  "isVerified": true,
  "hasPin": false, // New claim
  "iat": 1704700000,
  "exp": 1704700600  // 10 minutes if hasPin is false
}

API Gateway Middleware

const verifyToken = (req, res, next) => {
    // ... extract token ...
    if (isMobile && !isAuthRoute && decoded.hasPin === false) {
        return res.status(403).json({
            status: 'PIN_SETUP_REQUIRED',
            error: 'PIN setup required'
        });
    }
    next();
};

5. Risks & Mitigation

Risk Impact Mitigation
Token Expiry (10m) User might time out during PIN setup Increased to 10 minutes; app should handle re-auth if needed.
Gateway Performance Latency in request processing Middleware is lightweight; JWT verification is fast.

6. Recommendations

  • Implement biometric setup immediately after PIN setup for better UX.
  • Add device-level encryption for the stored JWT on the mobile app.
  • Consolidate all security recommendations into the recommendations.html.
