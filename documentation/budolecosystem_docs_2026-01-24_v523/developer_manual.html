Developer Manual - Release v523

Technical Explanation

The issue was caused by the orders list UI defaulting to the address.name field when displaying customer information. In cases where the shipping recipient (address name) differs from the account holder (user name), the buyer would see an unfamiliar name in their own order list.

The fix involves explicitly prioritizing order.user.name in both the main table and the "Shipping To" sub-field.

Code Changes

1. components/OrderItem.jsx

Updated the customer name and shipping recipient logic to use the logged-in user's name.

<td className="text-left max-md:hidden px-4 py-2"> <p className="font-medium text-slate-700">{order.user?.name || 'Unknown Customer'}</p> <div className="text-slate-500 text-sm mt-1"> <p className="text-[10px] font-bold text-slate-400 uppercase mb-1">Shipping To:</p> <p className="font-medium text-slate-600">{order.user?.name || order.address?.name}</p> <p>{order.address?.street}</p> ... </div> </td>
