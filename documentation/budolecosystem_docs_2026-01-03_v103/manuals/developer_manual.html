Developer Manual: QR Checkout Auto-Redirect

Purpose

The checkout page was remaining static after a QR scan, requiring users to manually refresh or navigate away. This implementation introduces real-time transaction status polling to automatically redirect users upon payment completion.

Technical Specification

1. Status Polling Endpoint

Added GET /payments/status/:referenceId to the payment-gateway-service. This endpoint queries the database for the current status of a transaction.

app.get('/payments/status/:referenceId', async (req, res) => {
  const { referenceId } = req.params;
  const transaction = await prisma.transaction.findUnique({
    where: { referenceId }
  });
  res.json({ status: transaction?.status });
});

2. Frontend Polling Logic

The checkout page now includes a script that polls the status endpoint every 3 seconds.

const checkStatus = async () => {
  const response = await fetch(`/payments/status/${referenceId}`);
  const data = await response.json();
  if (data.status === 'COMPLETED') {
    window.location.href = redirectUri;
  }
};
setInterval(checkStatus, 3000);

Impact Analysis

  • Frontend: Improved UX with zero-click redirection.
  • Backend: Minor increase in traffic to the status endpoint during active checkout sessions.
  • Database: Optimized findUnique query on referenceId (indexed field).

Future Enhancements

  • Implement WebSockets or Server-Sent Events (SSE) for true real-time updates without polling overhead.
  • Add a "Transaction Expired" state handling in the polling logic.
