System Migration & Dependency Automation

Subject: Automating the setup of budolEcosystem on new development environments.

Version: v210 (2026-01-05)

1. Overview

To facilitate the transfer of the budolEcosystem to different machines, a unified PowerShell script setup-environment.ps1 has been developed. This script automates the detection and installation of system-level dependencies, package managers, and project-specific requirements.

How to Use

  1. Copy the budolEcosystem folder to the new computer.
  2. Open PowerShell as Administrator.
  3. Navigate to the root directory.
  4. Run the following command:
Set-ExecutionPolicy Bypass -Scope Process -Force; .\setup-environment.ps1

2. Automated Checks & Installations

Category Requirement Automation Tool
Package Manager Chocolatey Native PS Installer
Version Control Git Chocolatey
Runtime Node.js (LTS) Chocolatey
Database PostgreSQL 16 Chocolatey
ORM Prisma CLI npm (Global)

3. Post-Installation Procedures

After the script completes, the following actions are automatically performed:

  • Node Modules: Runs npm install at the monorepo root.
  • Prisma Client: Generates the type-safe database client.
  • Environment Configuration: Creates a template .env file if none exists.

Manual Steps Required

  • Configure the DATABASE_URL in .env to match your local PostgreSQL credentials.
  • Ensure the PostgreSQL service is running (handled by the installer, but verify via services.msc).

4. Technical Details

The script uses Chocolatey as the primary package manager for Windows to ensure idempotent installations. It also refreshes environment variables within the session to allow immediate use of newly installed binaries like node and git.
