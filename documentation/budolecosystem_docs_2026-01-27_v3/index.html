Purpose & Rationale

Purpose: To resolve the issue where paid orders were not appearing in the "My Orders" section of the BudolShap UI.

Rationale: A discrepancy between the payment status in the database and the UI filtering logic caused valid, paid orders to be hidden after 30 minutes. This release fixes the data flow from the payment gateway to the order service.

Technical Overview

This release fixes the BudolPay webhook to correctly update the paymentStatus field to 'paid' upon successful transaction. It also includes a database migration script to correct legacy records that were stuck in 'awaiting_payment' despite being paid.
