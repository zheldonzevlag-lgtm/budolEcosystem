Data Correction Procedures

In the event that paid orders are not showing in the user interface, administrators can run the verification and fix scripts to synchronize the database status.

1. Verify Database State

Run the following command to check for discrepancies between isPaid and paymentStatus:

node "scripts/test_scripts/budolShap/v2/verify-orders-db.mjs"

2. Apply Fix

If the verification script shows orders with Paid: YES but Payment Status: awaiting_payment, run the fix script:

node "scripts/test_scripts/budolShap/v2/fix-paid-orders-status.mjs"
Important: These scripts must be run from the root directory of the Budol Ecosystem. Ensure that the .env file in budolshap-0.1.0 contains a valid DATABASE_URL.

Monitoring

Monitor the BudolPay Webhook logs for any 500 errors, which might indicate issues with the updateOrderStatus service call.
