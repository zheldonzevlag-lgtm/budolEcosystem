Technical Explanation

The issue was caused by the getOrders service in ordersService.js which implements an automatic filter for abandoned payments.

Orders that are not Cash-on-Delivery (COD) and have paymentStatus: 'awaiting_payment' are automatically excluded from UI results if they are older than 30 minutes.

The BudolPay webhook was successfully marking orders as isPaid: true, but was failing to update the paymentStatus field, leaving them stuck in awaiting_payment.

Code Changes

1. Webhook Update

Modified budolpay/route.js to include the paymentStatus update:

// BEFORE
const updatedOrder = await updateOrderStatus(order.id, {
    isPaid: true,
    status: 'PROCESSING'
});

// AFTER
const updatedOrder = await updateOrderStatus(order.id, {
    isPaid: true,
    status: 'PROCESSING',
    paymentStatus: 'paid'
});

2. Database Migration Script

Created fix-paid-orders-status.mjs to correct existing data:

await prisma.order.update({
    where: { id: order.id },
    data: {
        paymentStatus: 'paid'
    }
});

Dependencies

  • Prisma ORM (v5.x)
  • BudolPay Gateway API
  • SWR (for frontend fetching)
