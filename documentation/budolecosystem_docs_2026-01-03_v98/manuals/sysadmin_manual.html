Transaction Database Maintenance

Database Source of Truth

All financial reporting for the budolPay Admin app is pulled from the budolpay_db. SysAdmins should monitor the Transaction table for any PENDING records that have not been updated for a long period, which might indicate webhook delivery failures.

Querying Transactions

To manually verify if a transaction reached the Admin app database, use the following SQL query:

SELECT * FROM "Transaction" WHERE "referenceId" = 'INTENT-XXXXX';

Connectivity Checklist

  • Admin App Connectivity: Ensure the Admin app has the correct DATABASE_URL pointing to budolpay_db.
  • Gateway Webhooks: Verify that the payment-gateway-service is reachable at port 8004 for incoming webhooks.
  • Prisma Client: If schema changes occur, run npx prisma generate in packages/database to update all services.
