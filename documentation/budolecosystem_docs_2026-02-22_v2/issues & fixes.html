Issues & Fixes

This document tracks known issues and the fixes applied in version 2.0.0.

1. Inconsistent Phone Number Format

Issue

Description: Users were able to register with varying phone number formats (e.g., 0917..., +639..., 639...), leading to lookup failures during login and profile updates.

Impact: Failed logins, duplicate accounts, and failed notifications.

Fix

Resolution: Implemented PhoneUtils.normalizePhoneNumber in the mobile app. All phone inputs are now converted to E.164 format (+639xxxxxxxxx) before API submission.

Verification: Tested with various input formats; all correctly normalized.

2. Email Login Failure (Local Password Sync)

Issue

Description: Users who registered or updated their password via BudolID (e.g., via OTP) could not log in to BudolShap using their email because the local password hash was missing or outdated.

Impact: User lockout, frustration, increased support tickets.

Fix

Resolution: Added a fallback mechanism in /api/auth/login. If local password verification fails, the system attempts to authenticate against BudolID. Upon success, the local password is updated to match.

Verification: Confirmed login success for users with mismatched local passwords.
