Mitigation Strategies

This document details the strategies employed to mitigate identified risks.

1. Data Inconsistency (R-001)

Mitigation Strategy: M-001

Approach:

  • Immediate: Apply normalization logic on all new inputs (Login, Registration, Profile Update).
  • Batch: Provide a SQL migration script to normalize existing database records during a maintenance window.
  • Fallback: (Future) Implement flexible lookup that tries normalized and raw inputs if the initial lookup fails.

Status: Immediate mitigation implemented in v2.

2. Authentication Service Dependency (R-002)

Mitigation Strategy: M-002

Approach:

  • Local Caching: Successful BudolID logins update the local password hash. Subsequent logins will succeed locally even if BudolID is down.
  • Retry Logic: Implement retries for transient network errors when contacting BudolID.
  • Monitoring: Alert administrators if BudolID fallback failure rates spike.

Status: Local Caching implemented in v2.

3. Security (Password Sync) (R-003)

Mitigation Strategy: M-003

Approach:

  • Priority: Treat BudolID as the Single Source of Truth (SSOT) for authentication credentials.
  • Verification: Only update local password if the BudolID verification succeeds. This ensures the new password is valid and current.

Status: Implemented in v2.
