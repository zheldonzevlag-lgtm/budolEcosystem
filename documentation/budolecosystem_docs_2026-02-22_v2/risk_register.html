Risk Register

This document identifies potential risks associated with the v2 implementation.

1. Data Inconsistency

Risk ID: R-001

Description: Existing database records may contain non-normalized phone numbers.

Impact: Lookup failures for existing users when normalized input is compared against non-normalized database entries.

Probability: Medium

Severity: High

2. Authentication Service Dependency

Risk ID: R-002

Description: Login fallback relies on the availability of the BudolID service.

Impact: If BudolID is down, users with outdated local passwords cannot log in via email.

Probability: Low

Severity: High

3. Security (Password Sync)

Risk ID: R-003

Description: Updating local password upon fallback success might overwrite a user's intentional local password change if sync logic is flawed.

Impact: User confusion if password reverts unexpectedly.

Probability: Low

Severity: Medium
