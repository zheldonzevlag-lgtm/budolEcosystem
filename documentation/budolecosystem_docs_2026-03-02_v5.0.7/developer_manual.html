Issue Description

Users reported that items remained in their cart even after a successful payment via PayMongo (specifically GCash/Card flows). This was caused by a mismatch in how variation IDs were compared between the order and the cart during the asynchronous webhook processing.

Technical Fixes

1. Robust Variation Matching

In updateOrderStatus, the logic now handles null and empty string ("") variation IDs interchangeably using an OR condition. This ensures that items with no variations are correctly targeted for removal from the cart regardless of how they were initialized in the database.

File: lib/services/ordersService.js

2. Async Payment List Expansion

The createOrder function was expanded to treat CARD and PAYMAYA as asynchronous payments. This prevents the system from clearing the cart prematurely before the payment is actually authorized by the user.

Verification

A test script test_webhook_cart_clear.mjs was executed, simulating a full lifecycle of an order with an empty-string variation. The script confirmed that the new robust logic successfully cleared the cart item upon the order status being updated to PAID.
