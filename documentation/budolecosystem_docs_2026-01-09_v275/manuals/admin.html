Security & Administration

System Requirements

  • Android: API Level 23 (Marshmallow) or higher for Fingerprint; API Level 29 (Android 10) for BiometricPrompt.
  • iOS: iOS 11.0 or higher for FaceID and TouchID.

Security Risks & Mitigations

Risk 1: Local Bypass
On rooted or jailbroken devices, local authentication checks can be bypassed using tools like Frida.
Mitigation: Implement root/jailbreak detection and disable biometric features if the device integrity is compromised.
Risk 2: Credential Theft
If the PIN is stored insecurely, it could be extracted by other malicious apps.
Mitigation: Use flutter_secure_storage which leverages Android Keystore and iOS Keychain with hardware-level encryption.

Administrative Configuration

Administrators can control biometric policies via the systemSettings API:

  • ALLOW_BIOMETRICS: Boolean to enable/disable the feature globally.
  • MAX_BIOMETRIC_ATTEMPTS: Number of failed attempts before forcing PIN entry (default: 3).
  • REQUIRE_PIN_EVERY_X_DAYS: Security policy to force full PIN verification periodically.

Dependencies

  • Device Hardware (Scanner/Camera).
  • OS-level Biometric Service.
  • Backend support for device-binding tokens (Phase 3).
