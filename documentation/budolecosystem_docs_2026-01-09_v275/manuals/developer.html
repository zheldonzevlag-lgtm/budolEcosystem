Technical Implementation Plan

Core Technology Stack

Flutter local_auth (Open-Source) flutter_secure_storage Biometric API

Architecture Strategy

The implementation will follow a "Secure Vault" pattern. Instead of storing the user's password/PIN directly, we store a cryptographically signed token or the user's PIN in the device's hardware-backed secure enclave (Keystore on Android, Keychain on iOS).

1. Dependency Integration

Add the following to pubspec.yaml:

dependencies:
  local_auth: ^2.1.6
  flutter_secure_storage: ^9.0.0

2. Implementation Logic

The biometric authentication flow consists of three main parts:

  • Availability Check: Verify if the device supports biometrics and has enrolled users.
  • Enrolment: When a user logs in successfully with a PIN, offer to enable biometrics. If accepted, store the PIN in FlutterSecureStorage.
  • Authentication: On the login screen, if biometrics are enabled, trigger the OS biometric prompt. Upon success, retrieve the PIN from secure storage and proceed with the standard verifyPin API call.

3. Code Prototype (Service Layer)

class BiometricService {
  final LocalAuthentication auth = LocalAuthentication();
  final storage = const FlutterSecureStorage();

  Future<bool> canCheckBiometrics() async {
    return await auth.canCheckBiometrics || await auth.isDeviceSupported();
  }

  Future<bool> authenticate() async {
    try {
      return await auth.authenticate(
        localizedReason: 'Please authenticate to log in to budolPay',
        options: const AuthenticationOptions(
          stickyAuth: true,
          biometricOnly: true,
        ),
      );
    } catch (e) {
      return false;
    }
  }
}

Build-Your-Own (BYO) Consideration

Feasibility: Building a "custom biometric reader" from scratch is not recommended for mobile apps as it would require low-level kernel/driver access, which is restricted by Android/iOS for security reasons. The "Custom" approach in this context means writing a MethodChannel implementation to interface directly with BiometricPrompt (Android) or LAContext (iOS) if the open-source plugins do not meet specific requirements.

Future Enhancement: Paid Solutions

For enterprise-grade requirements, we propose migrating to HYPR or Duo Security. These provide:

  • FIDO2-certified authentication.
  • Server-side biometric verification (preventing local bypass).
  • Centralized policy management.

Test Protocols

  • Unit test for the BiometricService mock.
  • Integration test on physical devices (Simulators do not support full biometric flows).
  • Security audit of FlutterSecureStorage data persistence.
