Developer Manual: Authentication & SSO Strategy

Version: 2026-01-01_v29 | Author: Implementation Planner

1. Architectural Overview

The BudolEcosystem uses a centralized identity provider model. budolID acts as the single source of truth for user identities across all platforms.

  • Centralized SSO (budolID): Port 8000. Manages global users and app API keys.
  • Application Clients: BudolShap (Next.js) and BudolPay (Express) consume the SSO service.

2. Authentication Flows

A. Global SSO Login

Apps redirect users to the budolID portal for authentication.

GET http://192.168.1.24:8000/login?apiKey={APP_API_KEY}

The portal verifies credentials against budolid_db and redirects back with a JWT token.

B. Token Verification

Apps verify the token using the budolID public API.

GET http://192.168.1.24:8000/auth/verify Authorization: Bearer {TOKEN}

3. Data Models & Syncing

Users exist in two contexts:

Context Database Table/Model Scope
Ecosystem budolid_db User Global (SSO)
BudolShap budolshap_1db User Local (Shop)

Note: Users must be synced to budolID to use SSO features.
