System Admin Manual: SSO & Database Operations

Version: 2026-01-01_v29

1. Database Management

The ecosystem uses two primary PostgreSQL databases on the same host:

  • BudolID DB: budolid_db (Centralized Users)
  • BudolShap DB: budolshap_1db (Local Shop Data)

Connection URL Configuration

Ensure the .env file in budolID-0.1.0 is configured correctly:

DATABASE_URL="postgresql://postgres:r00t@localhost:5432/budolid_db?schema=public" PORT=8000

2. Registering New Applications

To allow a new app to use SSO, register it via the API:

POST http://localhost:8000/apps/register { "name": "My New App", "redirectUri": "http://localhost:3000/auth/callback" }

Save the returned apiKey for use in the client application.

3. Manual User Sync

Warning: Currently, users created in BudolShap are not automatically synced to BudolID.

To manually sync a user, use the seed-tony-budolid.js script template to upsert users into the budolid_db.
