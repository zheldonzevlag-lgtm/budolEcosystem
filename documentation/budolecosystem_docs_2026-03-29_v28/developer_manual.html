Multi-Layer Fallback Architecture

To ensure real-time events are delivered even during Render "Cold Starts" or service disruptions, a robust fallback chain has been implemented across the ecosystem.

Fallback Sequence (Production Only)

  1. Socket.io: Primary real-time provider. 3.5s timeout.
  2. Pusher: Secondary fallback if Socket.io is unreachable or times out.
  3. Polling/SWR: Tertiary fallback. No-op on server; clients continue polling.

File References & Explanations

BudolShap: realtime.js

Implements `triggerRealtimeEvent` with environment isolation. Only triggers fallbacks when `process.env.NODE_ENV === 'production'`. 

// Production-safe fallback logic
if (response.ok) return { success: true };
if (isProduction) {
    provider = 'PUSHER'; // Attempt fallback
} else {
    throw new Error('Socket.io trigger failed');
}

BudolPay: realtime-server.ts

TypeScript implementation of the same fallback chain, utilizing `AbortSignal.timeout` for efficient disruption handling.

BudolID: realtime.js & index.js

Integrated WebSockets into SSO login and registration. Notifies the ecosystem on `ecosystem:sso` channel with `sso:login` and `sso:register` events.

Integration Guide

To trigger an ecosystem-wide event from a new service:

const { triggerRealtimeEvent } = require('./utils/realtime');

await triggerRealtimeEvent('channel-name', 'event-name', {
    key: 'value'
});

Security & Performance Architecture

Backend Verification (SSO)

All credential checks (Email, Phone, Password) are performed synchronously on the BudolID server using Prisma and Bcrypt. This ensures that sensitive data never leaves the secure backend environment.

Real-time Notification

Socket.io and Pusher are used asynchronously only after verification is complete. Their sole purpose is to provide instant UI updates and ecosystem synchronization. Passwords and full PII are never transmitted.

NPC & BSP Compliance Note: All real-time payloads are masked or minimized to prevent accidental exposure of PII during transit via external providers.
