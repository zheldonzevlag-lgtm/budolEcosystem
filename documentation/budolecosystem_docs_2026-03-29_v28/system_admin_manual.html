Infrastructure Setup (Render.com)

The WebSocket service is designed for deployment on Render.com's FREE Tier. Admins must manage the 15-minute inactivity spin-down.

Deployment Steps

  • Push the `websocket-server/` directory to a dedicated GitHub repository.
  • Connect the repo to Render.com as a Web Service.
  • Add the following Environment Variables in Render Dashboard:
    • PORT: 10000
    • NODE_ENV: production

Configuration via Admin UI

Administrators can switch the real-time provider via the BudolPay Admin Settings.

Required Settings

REALTIME_METHOD: Set to SOCKETIO to enable WebSockets.
REALTIME_SOCKETIO_URL: Point to the Render.com service URL (e.g., `https://budol-ws.onrender.com`).

Monitoring & Health Checks

The WebSocket server exposes a health check endpoint:

GET /health
