Mitigation Plan v28

Fallback Controls & Business Continuity

Strategies implemented to address the risks identified in the v28 Risk Register.

M-v28-001

Addressing Render Free Tier Inactivity

Primary Control: Multi-layer fallback implemented in the application layer.

Secondary Control: Automated Pusher activation if Socket.io trigger response exceeds 3.5s (Cold Start detection).

Tertiary Control: Silent failover to Client-Side Polling (SWR) for seamless user experience without real-time delays.

M-v28-003

Pusher Quota Management

Implementation: Pusher is only used as a fallback, minimizing daily message count. Normal operation uses zero quota on Pusher.
