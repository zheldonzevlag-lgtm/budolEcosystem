v196 Implementation Plan

Subject: SMS OTP Integration for Identity Verification

Date: 2026-01-05 | Version: v196

Purpose

To enhance the security and legitimacy of the budolEcosystem by integrating a Philippines-based SMS gateway. This will enable 2-Factor Authentication (2FA), phone number ownership verification (OTP), and real-time transaction alerts, bringing budolPay closer to GCash-level security standards.

API Selection Analysis

Provider Best For Pros Cons
Semaphore OTP & Verification Dead simple API; High reliability for PH carriers (Globe/Smart). Strict free tier limits.
ITexMo General Dev Generous free tier; Established PH presence. Signature tags on free messages.
engageSPARK 2-Way Messaging Powerful automation; Branded SMS. Complex for simple OTP needs.

Phase 1: Ecosystem SMS Service

1.1 Create Central SMS Microservice

Develop a new service budolEcosystem-SMS that abstracts the chosen gateway (e.g., Semaphore). This allows other services to send SMS via a simple internal API call.

1.2 OTP Generation & Storage

Implement logic to generate 6-digit codes, store them in Redis (with 5-minute TTL), and handle verification attempts.

Phase 2: Mobile App Integration (Step 1)

2.1 Registration Flow Update

Modify lib/screens/registration_screen.dart. After the user enters their phone number in Step 1, trigger an OTP request. Add a verification sub-step before allowing the user to proceed to Step 2.

Phase 3: Transaction Alerts

3.1 Transaction-Service Webhook

Update transaction-service to trigger an SMS notification to the recipient whenever a P2P transfer is successful.

Risks & Mitigations

  • Risk: SMS delivery failure due to network congestion.
  • Mitigation: Implement a "Resend OTP" button with a 60-second cooldown and a fallback to email OTP if SMS fails repeatedly.
  • Risk: API credit exhaustion.
  • Mitigation: Implement rate limiting per IP and phone number to prevent abuse.
