v196 System Admin Manual

Subject: SMS Gateway Configuration & Monitoring

Note: During development, use the SEMAPHORE_API_KEY provided by the Semaphore dashboard.

Environment Variables

Ensure the following variables are added to the sms-service configuration:

  • SEMAPHORE_API_KEY: Your unique API key from semaphore.co
  • SMS_SENDER_NAME: The approved sender name (default: BUDOLPAY)
  • OTP_EXPIRY_MINUTES: Default is 5

Monitoring Usage

Adminds can monitor SMS credits and delivery success rates via the Semaphore Dashboard. Locally, check service logs for delivery errors:

docker logs sms-service | grep "SMS Error"

Rate Limiting

To prevent SMS credit draining attacks, the service implements a rate limit of 3 OTPs per phone number per hour. This can be adjusted in the service configuration.
