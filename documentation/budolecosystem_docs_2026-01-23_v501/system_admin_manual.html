System Administration - Real-time Configuration

Overview

This version introduces an optimized caching mechanism for system settings. Admins no longer need to wait 1 minute for changes in Real-time Providers (Polling vs Pusher) to take effect.

Real-time Settings

Configure these settings in the Admin Dashboard under System Settings:

Setting Description Impact
Real-time Provider Switch between POLLING and PUSHER. Immediate update (v501).
Pusher Credentials App ID, Key, Secret, Cluster. Required for near-instant UI sync.

Troubleshooting Status Sync

If status updates are delayed:

  1. Verify Pusher credentials in System Settings.
  2. Check Lalamove Webhook logs in the developer portal.
  3. Ensure the "Real-time Provider" is set to PUSHER.
