Developer Manual - v501

Objective

Eliminate the 1-minute latency between status updates in external services (Lalamove, PayMongo) and the user interface.

Technical Implementation Details

1. Real-time Trigger Optimization

Modified realtime.js to improve connection efficiency and settings retrieval.

// Singleton pattern for Pusher to avoid re-initializing on every request
function getPusher(settings) {
    const configFingerprint = `${settings.pusherAppId}:${settings.pusherKey}:${settings.pusherCluster}`;
    if (pusherInstance && lastPusherConfig === configFingerprint) {
        return pusherInstance;
    }
    pusherInstance = new Pusher({...});
    return pusherInstance;
}

// Reduced cache TTL from 60s to 10s
if (settingsCache && (now - lastFetch < 10000)) {
    return settingsCache;
}

Dependencies

  • Pusher SDK: Used for WebSocket management. Chosen for reliability and global low-latency.
  • Prisma ORM: Used for database operations. Chosen for type safety and efficient querying.

Compliance

The real-time synchronization logic adheres to the following regulatory standards:

  • PCI DSS: Secure handling of payment status webhooks from PayMongo.
  • BSP (Bangko Sentral ng Pilipinas): Real-time reporting of transaction statuses for financial transparency.
  • BIR: Accurate timestamping of order fulfillment (Delivered status) for tax compliance.
  • NPC (Data Privacy Act): Masking of sensitive user data in real-time payloads.
  • DTI: Ensuring consumer protection through accurate delivery status tracking.
  • Cybersecurity Law: Implementation of secure WebSocket (WSS) and signature verification for all webhooks.
