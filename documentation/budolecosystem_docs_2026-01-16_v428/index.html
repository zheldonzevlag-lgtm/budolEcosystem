Version 428 - Order Progress & Payment Status Synchronization

Purpose & Rationale

This release resolves a UI synchronization issue where orders paid via the BudolPay Android app were correctly marked as "Paid" in the Order Items tab but remained as "Unpaid" or "Pending" in the Order Progress and Tracking History sections. This was caused by hardcoded logic that incorrectly categorized BudolPay as a deferred payment method (like COD).

Technical Summary

  • Logic Alignment: Removed BUDOL_PAY from the deferredPaymentMethods list in OrderProgressTracker.jsx and TrackingTimeline.jsx.
  • Payment Status Priority: Updated status calculation to prioritize the order.isPaid boolean from the database over deferred delivery checks for BudolPay transactions.
  • Timeline Description Fix: Updated the "Paid" event description in the tracking timeline to correctly reflect when a payment has been received and verified via BudolPay.
