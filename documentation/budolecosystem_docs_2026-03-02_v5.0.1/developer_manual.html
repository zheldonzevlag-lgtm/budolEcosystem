Purpose

This update addresses a critical issue in the checkout flow where the frontend would receive an empty response from the payment gateway due to internal service timeouts and unhandled request hangs in the audit logging system.

Technical Explanation

1. Audit Logging Timeout

The @budolpay/audit package was modified to include an AbortController with a 5-second timeout for all internal notifications to the API Gateway. This prevents the entire payment chain from hanging if the notification service is slow or unreachable.

File: audit/src/index.ts

2. BudolPay Adapter Robustness

The BudolPayAdapter in budolshap was enhanced to handle empty responses and non-JSON content types gracefully. It now provides specific error messages for timeout scenarios, allowing the frontend to inform the user accurately.

File: adapters/budolpay-adapter.js

Changes Summary

  • Implemented AbortController in notifyGateway function.
  • Added 5000ms timeout to audit log broadcasts.
  • Improved error parsing in BudolPayAdapter.
  • Added explicit check for empty response strings in fetch calls.

Compliance & Risk

Compliance: BSP Circular No. 808 (Financial Transaction Audit) - Maintained by ensuring logs are still attempted but don't block transactions.

Risk: Minor risk of missed real-time audit notifications if the Gateway API is under heavy load, but transaction integrity is prioritized.

Tests

Future Recommendations

  • Implement a retry queue for audit logs that fail due to timeout.
  • Add circuit breakers for inter-service communication.
