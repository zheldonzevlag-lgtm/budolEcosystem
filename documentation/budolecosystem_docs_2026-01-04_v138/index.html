Smart Network Prioritization

Refined network discovery logic to prioritize physical LAN/Wi-Fi adapters over virtual interfaces.

Purpose

In complex development environments, machines often have multiple network interfaces (VirtualBox, VMware, Docker, WSL, VPNs like Cloudflare WARP, and Hyper-V switches). Standard IP detection often picks the first available interface, which might be a virtual one unreachable by physical mobile devices.

This update introduces an Intelligent Prioritization Engine that scores interfaces based on their names and excludes known virtual/internal types.

Implementation Details

1. Scoring & Filtering Logic

Modified scripts/network-util.js to use a keyword-based selection process.

const IGNORE_KEYWORDS = [
  'virtual', 'warp', 'cloudflare', 'hyper-v', 'vethernet', 'switch', ...
];

const PREFER_KEYWORDS = [
  'wi-fi', 'wifi', 'ethernet', 'lan', ...
];

2. Candidate Analysis

The system now analyzes all available interfaces, assigns scores, and logs the analysis for transparency.

  • Preferred interfaces (Wi-Fi/LAN): Base score of 10.
  • Other interfaces: Base score of 1.
  • Ignored interfaces: Automatically excluded.
