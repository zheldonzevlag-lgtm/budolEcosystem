Release Summary: v197 (Database Schema Synchronization & Identification Fix)

This version documents the resolution of the "Identification failed: 500" error in the mobile app. The issue was caused by a database schema desynchronization where the pinHash column was missing from the local User table, causing Prisma queries to fail.

Key Improvements

  • Database Alignment: Synchronized the local PostgreSQL database with the latest schema.prisma definition.
  • Client Update: Regenerated the Prisma Client to ensure all services have the correct type definitions and query capabilities.
  • Authentication Flow: Restored the multi-phase mobile login flow (Identify & Challenge), enabling OTP and PIN-based authentication.
