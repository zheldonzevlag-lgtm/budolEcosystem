System Admin Manual Updates - v197

Database Management

This version required a schema update for the core budolpay database. Admins should ensure that all environments (Local, Staging, Production) are synchronized.

Synchronization Procedure

cd budolpay-0.1.0/packages/database
npx prisma db push

This command will compare the schema.prisma with the target database and apply missing columns or tables. Warning: Review the output for potential data loss warnings if columns are being dropped.

Prisma Client Generation

After a schema sync, the Prisma Client must be regenerated to be available for all microservices:

npx prisma generate
Note: If you encounter EPERM errors during generation on Windows, ensure all Node.js services are stopped, as they may be locking the generated client files in node_modules.

Health Monitoring

Post-deployment, monitor the Auth Service logs for 500 errors during login. A successful identification should result in a 200 OK or 201 Created response, even if an OTP is required.

Risk Mitigation

  • Risk: Database locking during db push.
  • Mitigation: Perform synchronization during maintenance windows or low-traffic periods. Ensure DATABASE_URL is correctly scoped to the target environment.
