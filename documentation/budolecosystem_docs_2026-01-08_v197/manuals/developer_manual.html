Developer Manual Updates - v197

Purpose

To resolve the 500 Internal Server Error during mobile app identification, ensuring the database schema matches the application's data models.

Issue: Schema Desynchronization

The Auth Service's /login/mobile/identify endpoint failed because the User.pinHash column was missing in the database. This led to the following error:

Invalid `prisma.user.findUnique()` invocation.
The column `User.pinHash` does not exist in the current database.

Fix: Schema Synchronization

Executed prisma db push to align the database with the schema and prisma generate to update the client.

# Commands executed in budolpay-0.1.0/packages/database
npx prisma db push
npx prisma generate

Technical Details

The User model in schema.prisma was updated to include pinHash, but the local PostgreSQL instance had an older version of the table. Using db push is preferred in development for quick synchronization without formal migration files.

Verification

The fix was verified by simulating a POST request to the Auth Service:

Invoke-RestMethod -Uri "http://localhost:8001/login/mobile/identify" \
    -Method Post \
    -Body '{"phoneNumber": "09484099388", "deviceId": "test-device"}' \
    -ContentType "application/json"

Result: Successfully returned status: OTP_REQUIRED, confirming the identification logic is now functional.

Future Recommendations

Implement a centralized "Schema Sync Check" during service startup to alert developers if the database is out of sync with the Prisma Client.
