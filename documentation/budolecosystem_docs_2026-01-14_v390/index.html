Release v390 Latest

Refined Face Login Registration Check & Logic

Purpose & Rationale

Version 390 refines the face login enablement logic. Previously, face login was strictly tied to the general biometric toggle. This version allows users with a registered face template to use face login even if the general biometric toggle is off, prioritizing the custom TFLite-based security profile. This ensures better accessibility for users who prefer face recognition over native biometrics.

Technical Highlights

  • Decoupled Logic: Face login now checks for _hasFaceTemplate independently of _isBiometricEnabled.
  • UI State: The face icon color now reflects availability based on either the general toggle OR a registered template.
  • Version Sync: App version synchronized to 1.2.87+87 in pubspec.yaml and api_service.dart.
  • Automated Validation: Verified with test_face_login_v390.dart, confirming logic works under toggle-off scenarios.
