Technical Implementation Details - v309

1. Ecosystem Script Restoration

The core ecosystem management scripts were identified as having been moved from the root directory to scripts/test_scripts/global/. This migration broke the relative paths used for service discovery and environment validation.

Restored Files:

  • package.json (Root)
  • package-lock.json (Root)
  • start-ecosystem.js (Root)

2. budolID Service Fixes

The budolID-0.1.0 service was failing to start due to missing configuration files that were located in scripts/test_scripts/budolID/.

Corrected Paths:

  • budolID-0.1.0/package.json (Restored)
  • budolID-0.1.0/package-lock.json (Restored)
  • budolID-0.1.0/index.js (Restored)

3. Environment Validation Logic

Updated start-ecosystem.js to correctly extract and validate essential environment variables before starting microservices.

const jwtMatch = envContent.match(/^JWT_SECRET=["']?(.+?)["']?$/m);
const localIpMatch = envContent.match(/^LOCAL_IP=["']?(.+?)["']?$/m);

// ... set process.env for validation
process.env.JWT_SECRET = jwtMatch ? jwtMatch[1] : undefined;
process.env.LOCAL_IP = localIpMatch ? localIpMatch[1] : undefined;
