Developer Manual - v108

1. UI Restoration & Branding

Following user feedback regarding UI discrepancies, we performed a deep dive into historical documentation (v65) and backup assets to restore the original brand identity.

1.1 Color Palette Restoration

The application theme has been reverted to the signature "Rose Red" palette:

  • Primary (Rose Red): #F43F5E - Used for buttons, headers, and highlights.
  • Secondary (Slate Gray): #94A3B8 - Used for brand text ("budol") and secondary info.
  • Background (Dark Slate): #0F172A - Used for the login screen background.
  • Surface (Light Slate): #F8FAFC - Used for the home screen background.

1.2 Logo Implementation

The "Philippine Peso (₱)" circular badge was re-implemented using Flutter widgets for high-fidelity rendering, serving as the primary brand mark on the login screen.

// Circular Peso Badge Implementation
Container(
  width: 120, height: 120,
  decoration: BoxDecoration(
    color: Color(0xFFF43F5E),
    shape: BoxShape.circle,
    boxShadow: [BoxShadow(color: Color(0xFFF43F5E).withOpacity(0.3), blurRadius: 20)],
  ),
  child: Center(child: Text('₱', style: TextStyle(fontSize: 70, color: Colors.white))),
)

2. Asset Recovery

App icons were recovered from intermediate build artifacts found in budolPayMobile_backup_v103 after source XML files were found to be missing or corrupted.

Recovery Path: build/app/intermediates/packaged_res/release/packageReleaseResources/mipmap-*-v4/ic_launcher.png

3. Backend Integration

The end-to-end payment flow remains integrated with the API Gateway (Port 8080). Balance fetching in home_screen.dart now correctly uses the wallet-service endpoint via the gateway.
