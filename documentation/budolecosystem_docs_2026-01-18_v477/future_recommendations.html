Future Enhancements & Recommendations - v477

Technical Recommendations

  • State Management Migration: Consider migrating biometric state to a global Provider or Bloc to avoid manual refreshes in multiple screens.
  • Biometric Lockout Handling: Implement logic to handle cases where the OS locks the user out of biometrics (too many failed attempts) and provide clear instructions to use PIN.
  • Enhanced Logging: Add remote error logging (e.g., Sentry) for biometric failures to identify device-specific issues in the field.

Security Recommendations

  • Dynamic Salt Rotation: Rotate encryption salts periodically for stored biometric templates.
  • App Attestation: Use Google Play Integrity API to ensure the app is running on a genuine, non-compromised device before allowing biometric login.
