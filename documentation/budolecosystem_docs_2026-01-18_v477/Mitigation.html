Mitigation Strategies - v477

Technical Mitigations

  • State Refresh: Implemented isEnabled() check on button tap in login_screen.dart to mitigate the issue where the login screen does not reflect changes made in the Settings screen.
  • Secure Storage Stability: Configured AndroidOptions with resetOnError: true to prevent app crashes if the hardware-backed keystore becomes inaccessible or corrupted.
  • Dependency Injection: Refactored BiometricService to accept a storage instance, enabling robust unit testing and reducing regression risks.

Compliance Mitigations

  • NPC Compliance: Ensured that biometric status flags are stored in encrypted SharedPreferences (hardware-backed), protecting user privacy settings.
  • BSP/PCI DSS: Maintained MPIN requirement before any biometric state change in Settings, ensuring multi-factor authorization for security-sensitive operations.
