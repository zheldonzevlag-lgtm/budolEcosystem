Database & Product Service Restoration (v16)

Situation Analysis

Issue: EcosystemApp table missing and "0 products found" on frontend.

Root Cause: Database corruption/missing tables in production instance and Prisma engine runtime failure due to missing libssl in Alpine container.

Implementation Steps

  • DB Restore: Created budol-db-restored from snapshot (2026-03-10).
  • Unified DB Connection: Pointed connection string to budolshap_1db which contains all combined schemas.
  • Container Patch: Switched Docker image to node:20-slim (Debian) to provide required OpenSSL libraries for Prisma.

Infrastructure Changes

RDS: budol-db-restored (Active)
ECR: budol-shap:v16 (libssl fix)
ECS: Redeployed services via force-deployment

Final Health Check

Verification script confirmed Product table rows are present in the restored database. ECS logs no longer show Prisma initialization errors.
