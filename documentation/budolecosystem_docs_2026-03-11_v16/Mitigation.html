Mitigation Plan - v15

R-15-02: NULL Constraint Failures

Action: Updated schema.prisma to make barangay and userId optional fields. This allows the database to accept legacy records that were missing these values during the migration phase.

R-15-03: Docker Build Errors

Action: Implemented a fallback to DOCKER_BUILDKIT=0 (Legacy Mode) for local builds. This resolves the INTERNAL_ERROR stream resets observed on the user's Docker Desktop environment.

R-15-04: Outdated Prisma Client

Action: Guaranteed npx prisma generate execution during the Docker build sequence. This ensures the categoryData field is recognized by the application logic in the production container.
