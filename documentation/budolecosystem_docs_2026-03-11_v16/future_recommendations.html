Strategic Recommendations - Post v15

1. Data Cleaning Script

While the schema now allows NULLs for 'barangay', we should execute a server-side cleanup script to populate these fields with 'Unspecified' or coordinate data to maintain high quality shipping info.

2. CI/CD Pipeline for Prisma

Integrate prisma generate and prisma db push into a GitHub Action or AWS CodeBuild to prevent future mismatches between the local development schema and production container images.

3. Postgres 17 Monitoring

Since the RDS instance was upgraded/restored to Postgres 17, monitor performance metrics on AWS CloudWatch for any specific query regressions compared to the previous version.
