Issues & Fixes Log — v16

All issues encountered during the v16 development cycle, including root cause analysis and applied fixes.

Critical Issues

Critical Cannot Login — SSO Callback Missing User Fields
Users could not log in via BudolID SSO. The login route in app/api/auth/login/route.js attempted to access user.phoneNumber and user.metadata but these fields did not exist in the Prisma schema, causing a runtime crash.
✅ Fix Applied
Added phoneNumber String? @unique and metadata Json? to the User model in budolshap-0.1.0/prisma/schema.prisma. Ran prisma migrate deploy to apply changes to the database.
Critical Failed to Fetch Settings from DB — prisma.systemSettings Typo
System settings were not loading. The code in lib/settings.js and lib/realtime.js referenced prisma.systemSettings (plural) instead of the actual Prisma model name prisma.systemSetting (singular). This caused an "Unknown model" runtime error.
✅ Fix Applied
Corrected all occurrences of prisma.systemSettings → prisma.systemSetting in lib/settings.js and lib/realtime.js.
Critical budolID Prisma Schema BOM Encoding Error
Running npm run generate in budolID-0.1.0 failed with 3 validation errors. The root cause was a UTF-8 BOM character (﻿) at the start of prisma/schema.prisma, which the Prisma parser could not interpret. This prevented Prisma Client from being generated for BudolID.
✅ Fix Applied
Rewrote the file from scratch without BOM encoding. prisma generate now succeeds and emits a valid Prisma Client.
Critical Docker Build Failed — exit code 1 on npm run build
The Docker image build for budol-shap:v16 failed at the npm run build step because DATABASE_URL was not set and Next.js tried to statically render pages that directly called Prisma at the top level.
✅ Fix Applied
Passed a dummy DATABASE_URL build arg to allow compilation. All dynamic DB pages (dashboard, accounting, settings) were marked with export const dynamic = 'force-dynamic' so Next.js skips them during static generation. Docker image budol-shap:v16 (467MB) built successfully.

Medium Issues

Medium budolpay Admin Build Fails — @budolpay/audit Not in Dependencies
The admin app in budolpay-0.1.0 imported @budolpay/audit via the re-export in lib/audit.ts, but @budolpay/audit was missing from the apps/admin/package.json dependencies, causing module resolution to fail.
✅ Fix Applied
Added "@budolpay/audit": "*" and "@budolpay/security": "*" to apps/admin/package.json. Also updated next.config.mjs to include both packages in transpilePackages.
Low Currency Symbol Showing $ Instead of ₱ on Some Pages
Some components used process.env.NEXT_PUBLIC_CURRENCY_SYMBOL || '$' as a fallback. Without the environment variable set, the dollar sign was displayed.
✅ Fix Applied
Set ARG NEXT_PUBLIC_CURRENCY_SYMBOL=₱ in Dockerfile as a build-time default. Components updated to use || '₱' as fallback.
