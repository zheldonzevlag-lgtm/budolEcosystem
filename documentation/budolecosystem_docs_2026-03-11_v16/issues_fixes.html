v16 Post-Mortem: Issues & Resolutions

Issue 16.1: Missing Products (Database Mismatch)

Symptom: BudolShap UI loaded but showed "0 products found".

Root Cause: The connection secret was pointing to the budolid database. Research via RDS screenshots confirmed that product data resides in the budolshap_1db database.

Resolution 16.1: Schema Alignment

Updated the budol/db-url secret to point explicitly to /budolshap_1db. This restored access to the public schema containing the Product table.

# Update Command
aws secretsmanager put-secret-value --secret-id budol/db-url \
--secret-string "postgresql://...:5432/budolshap_1db"

Issue 16.2: libssl.so.1.1 Failure (Prisma Engine)

Symptom: Container logs showed Error loading shared library libssl.so.1.1: No such file or directory.

Root Cause: The use of node:20-alpine (v3.19+) does not include OpenSSL 1.1 by default. The Prisma engine used in v5.7.0 specifically requires this version.

Resolution 16.2: Platform Optimization (Debian Switch)

Switched the base Docker image from Alpine to Debian (node:20-slim). Debian's apt package manager provides a more compatible environment for the Prisma binary engine.

# Dockerfile Change
- FROM node:20-alpine AS runner
+ FROM node:20-slim AS runner
+ RUN apt-get update && apt-get install -y openssl
