Developer Manual - v190

KYC Tier Database Fix

This release addresses a critical synchronization issue where the kycTier column was missing from the database despite being present in the Prisma schema and application code.

Issues Resolved

  • Database Drift: Resolved a scenario where the database schema had diverged from the migration history, preventing new migrations from applying correctly.
  • Missing Column: Fixed the PrismaClientKnownRequestError: The column 'User.kycTier' does not exist error by using prisma db push to force-sync the schema.
  • Type Safety: Cleaned up temporary as any type casts in the Admin API now that the Prisma client is correctly regenerated with the new schema.

Technical Details

  • Applied npx prisma db push --accept-data-loss to resolve the schema drift.
  • Successfully regenerated the Prisma client after stopping conflicting Node.js processes.
  • Verified the existence of the kycTier column using a custom verification script.
