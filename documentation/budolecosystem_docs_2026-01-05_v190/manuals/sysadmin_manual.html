System Admin Manual - v190

Critical Database Repair

This version includes a critical fix for the database schema drift that was causing the SSO login and Admin Panel to crash.

Manual Action Required: If the database is still showing "Column does not exist" errors, ensure that all services are stopped and run the recovery steps below.

Recovery Steps

  1. Stop all Node.js services to release locks on the Prisma client.
  2. Navigate to packages/database.
  3. Run npx prisma db push --accept-data-loss to force the schema into sync.
  4. Run npx prisma generate to update the application's database client.
  5. Restart all services using npm run dev from the root directory.

Post-Fix Verification

Admins should verify that the application no longer returns "SSO Callback Error" during login and that the User Management table correctly displays all columns.
