Purpose

This version addresses an issue in the Hero section where the "Starts from" price was displayed as ₱0.00 due to test or incomplete products in the database having a price of 0. The fix ensures that the "Starts from" price calculation ignores products with a price of 0, providing a more accurate and professional representation of available products.

Rational

Displaying a starting price of 0.00 when the actual products have costs can lead to user confusion and perceived system errors. By filtering for positive prices, the system correctly identifies the lowest actual entry price for the shop.

Implementation Task List

  • Phase 0: Baseline & Risk Analysis - Identified products with price 0 causing the issue.
  • Phase 1: Architecture & Data Contracts - Planned filtering logic in Hero component.
  • Phase 2: Backend & API Implementation - No backend changes required.
  • Phase 3: Frontend / App Implementation - Updated Hero.jsx to filter out 0 prices.
  • Phase 4: Integration & Security - Verified no regressions in product listing.
  • Phase 5: End-to-End Testing - Created Jest tests for the price logic.
  • Phase 6: Documentation & Audit - Generated manuals and risk registers.
  • Phase 7: Release & Monitoring - Final release documentation.
