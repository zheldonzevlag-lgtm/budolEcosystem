Technical Explanation

The Hero component uses Redux to fetch the list of products from the product.list state. The previous implementation calculated the starting price by taking the minimum value of all products in the list. However, products with a price of 0 (which are either intended to be free or are incomplete test data) resulted in a ₱0.00 display.

The fix involves filtering the products list to extract only products with a price greater than 0 before performing the Math.min() operation.

Modified File: Hero.jsx

Location: /components/Hero.jsx 

// Before Logic:
const min = Math.min(...products.map(product => product.price))

// After Logic:
const validPrices = products
    .map(product => Number(product.price))
    .filter(price => price > 0);

const min = validPrices.length > 0 ? Math.min(...validPrices) : 0;
setMinPrice(min);

Dependencies

  • React Redux: For accessing global product state.
  • Prisma: Used to query the database to verify the presence of zero-priced products in this task.
  • Jest: For verifying the component logic.

Compliance

This fix complies with NPC (National Privacy Commission) and BSP standards for data integrity and accurate representation of service/product pricing, ensuring users are not misled by incorrect system-generated values.
