Executive Summary

Version 412 addresses a critical UX issue where users (notably "Peter Parker") were logged out upon app restart. Additionally, it introduces a precision amount input formatter to ensure all financial transactions are entered with exactly 2 decimal places, fulfilling regulatory and UX requirements for financial accuracy.

Key Improvements

  • Immediate UI state restoration via notifyListeners() in ApiService.
  • Corrected /verify endpoint targeting in auth-service.
  • Precision Entry: New CurrencyInputFormatter ensures 2-decimal precision and comma grouping (e.g., 1,234.56) for all amount inputs.
  • Version increment to 1.2.96 for distribution tracking.
