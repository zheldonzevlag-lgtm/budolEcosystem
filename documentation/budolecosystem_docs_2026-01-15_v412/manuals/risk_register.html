Risk ID Description Impact Probability Status
R-412-01 Session mismatch due to cached SharedPreferences data on old builds. Medium Low Mitigated
R-412-02 Increased load on /verify endpoint during app mass-restart. Medium Low Monitoring
R-412-03 Token expiration (30d) causing unexpected logout for infrequent users. Medium Medium Mitigated
R-412-04 Sensitive data exposure if SharedPreferences is not cleared on logout. High Very Low Resolved

Risk Assessment Note

The primary risk for v412 is the potential for stale session data to cause confusion. However, the implementation of notifyListeners() combined with the corrected /verify background check ensures that the local cache is only used as a bridge until the server provides the authoritative state.
