System Admin Manual: Mobile Session Management & Deployment

Important: Version 412 requires an update to the budolPay Android APK (v1.2.96) to resolve user session persistence issues. Ensure the backend auth-service is running and reachable at the configured AUTH_URL.

1. Backend Configuration Verification

Administrators must verify that the auth-service correctly exposes the /verify endpoint. This endpoint is critical for mobile apps to validate existing JWT tokens upon restart.

Service Endpoint Function Status
Auth Service /verify Token Validation & User Sync Operational
Prisma DB User Table Persistent Storage Connected

2. APK Deployment & Distribution

Step 1: Build the APK

Run the Flutter build command to generate the release APK for version 1.2.96.

flutter build apk --release

Step 2: Version Verification

Confirm the splash screen displays v1.2.96. This build includes the fix for the /verify endpoint and immediate state notification.

Step 3: Distribution

Upload app-release.apk to the internal budolEcosystem/android-installer directory and update the distribution portal links.

3. Monitoring & Troubleshooting

Monitor the auth-service logs for any 401 Unauthorized or 404 Not Found errors during the /verify request. If users report being logged out, verify that their JWT tokens haven't expired (standard expiry is 30 days for mobile).
