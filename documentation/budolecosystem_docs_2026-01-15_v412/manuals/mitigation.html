Mitigation Strategy: Session Stability & Security

M-412-01: Corrective Endpoint Targeting

Risk: Session verification failing due to 404 on /auth/verify.

Mitigation: Updated ApiService to target the root /verify endpoint in auth-service. This aligns the mobile app with the backend's universal propagation logic, ensuring that tokens issued via SSO or Mobile Login are correctly validated.

M-412-02: State Notification Optimization

Risk: UI displaying "User" placeholder despite valid local session.

Mitigation: Introduced notifyListeners() immediately after loading session data from SharedPreferences in _initialize(). This ensures that the Provider system updates the HomeScreen (and other listeners) before the background verification network call completes.

M-412-03: Version Control & Forced Upgrade

Risk: Users remaining on buggy v1.2.95 build with session issues.

Mitigation: Incremented version to 1.2.96. System administrators are advised to promote this build as a "Stability Update" to ensure all users benefit from the session persistence fixes.

M-412-04: Token Lifecycle Management

Risk: Unauthorized access via stolen tokens.

Mitigation: The /verify endpoint performs a full jwt.verify check on every app restart. If a token is compromised or expired, the backend returns 401 Unauthorized, triggering an automatic logout and local cache clearance on the mobile device.
