Future Recommendations: Enhancing Mobile Experience

1. Implement Refresh Tokens

Currently, users are logged out after 30 days when their access token expires. Implementing a Refresh Token flow (per OAuth 2.0 standards) would allow the app to silently renew the session, providing an even more seamless experience for long-term users.

2. Secure Storage (Encrypted SharedPreferences)

While SharedPreferences is convenient, it is not encrypted by default. For higher security compliance (PCI DSS), sensitive data like JWT tokens should be stored using flutter_secure_storage (Keychain for iOS, Keystore for Android).

3. Offline Mode Capability

The app currently relies on the /verify endpoint for initial validation. Implementing a local "Last Verified" timestamp could allow the app to function in a limited offline mode for viewing balance and recent transactions when internet connectivity is intermittent.

4. Biometric Re-authentication for Sensitive Actions

Even with persistent sessions, critical actions like "Send Money" or "Pay Bill" should require a fresh biometric or PIN verification to prevent unauthorized use of a logged-in device.
