Developer Manual: Session Persistence & Endpoint Correction

1. Session Initialization & Listener Notification

To ensure that the UI reflects the user's session state immediately upon app startup, the _initialize() method in ApiService now calls notifyListeners() as soon as session data is retrieved from SharedPreferences. This prevents the "User" placeholder from appearing while the background verification process is running.

File: budolPayMobile/lib/services/api_service.dart
// Added notifyListeners() in _initialize() to trigger UI update with local session data
_hasSeenAds = prefs.getBool(_hasSeenAdsKey) ?? false;

// Notify listeners that we've loaded the session from SharedPreferences
notifyListeners();

// Non-blocking background initialization for network-dependent tasks
_backgroundInit();

2. Profile Verification Endpoint Correction

The _fetchUserProfile() method was incorrectly targeting /auth/verify, which resulted in 404 errors for mobile session validation. The backend (auth-service) exposes the verification logic at the root /verify endpoint for universal propagation.

File: budolPayMobile/lib/services/api_service.dart
// Fixed endpoint URL in _fetchUserProfile()
Future _fetchUserProfile() async {
  if (token == null) return;
  
  // The auth service has /verify at the root for mobile/universal sync
  final url = '$authUrl/verify';
  try {
    final response = await http.get(
      Uri.parse(url),
      headers: {
        'Authorization': 'Bearer $token',
        'Content-Type': 'application/json',
      },
    );
    // ...
  }
}

4. Precision Amount Input Formatter

To ensure financial accuracy and fulfill the requirement for 2-decimal precision during entry, the CurrencyInputFormatter was enhanced to treat all digit inputs as cents. This forces the UI to always display two decimal places and includes thousand separators (commas) for better readability.

File: budolPayMobile/lib/utils/formatters.dart
class CurrencyInputFormatter extends TextInputFormatter {
  @override
  TextEditingValue formatEditUpdate(
      TextEditingValue oldValue, TextEditingValue newValue) {
    if (newValue.text.isEmpty) return newValue.copyWith(text: '');

    // Handle backspace or empty - extract digits only
    String text = newValue.text.replaceAll(RegExp(r'[^\d]'), '');
    if (text.isEmpty) return const TextEditingValue(text: '', selection: TextSelection.collapsed(offset: 0));

    // Convert digits to double (treating as cents)
    double value = double.parse(text) / 100;
    
    // Format with commas and 2 decimals using NumberFormat
    final formatter = NumberFormat("#,##0.00", "en_US");
    String formatted = formatter.format(value);

    return TextEditingValue(
      text: formatted,
      selection: TextSelection.collapsed(offset: formatted.length),
    );
  }
}

5. Technical Rationale

The combination of immediate state notification (via notifyListeners) and corrected API targeting ensures a seamless "logged-in" experience for mobile users. Even if the network verification is still in progress, the UI will display the cached user profile from the previous session, fulfilling the requirement for persistent identity (e.g., Peter Parker).

6. Phase 8: Dependencies & Software Choices

Ecosystem Dependencies Analysis
  • Flutter (3.24.x): Chosen for its rapid development cycle and consistent UI rendering across Android and iOS. Its "Provider" pattern is utilized here for efficient state management of user sessions.
  • Node.js & Express: Powers the auth-service. Chosen for its asynchronous non-blocking I/O, which is ideal for handling concurrent authentication requests and session verifications.
  • Prisma ORM: Used for database interactions. Its type-safe client ensures that user and wallet data are handled consistently across the backend services, reducing runtime errors.
  • SharedPreferences: Selected for local session persistence on mobile. While simple, it provides the low-latency retrieval required to show the "Welcome back" message immediately on app startup.
  • JWT (JSON Web Tokens): Used for secure, stateless authentication. The 30-day expiry for mobile tokens balances user convenience with security best practices, aligning with PCI DSS recommendations for session management.
