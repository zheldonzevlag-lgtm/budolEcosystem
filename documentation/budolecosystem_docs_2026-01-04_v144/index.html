v144

Real-time Money Sync & Transaction Notifications

Implementation of WebSocket-based live balance updates and transaction alerts for the budolPay Mobile App.

Purpose & Rationale

To provide a seamless user experience, the budolPay Mobile app now supports real-time synchronization of wallet balances and transaction events. This eliminates the need for manual refreshes and provides immediate visual feedback when money is received or spent.

User Experience

Instant feedback during payments and top-ups, increasing user confidence in the platform.

System Efficiency

Reduces frequent polling of the balance endpoint, shifting to an event-driven architecture.

Technical Implementation

The solution utilizes Socket.io for bidirectional communication between the Mobile App and the API Gateway.

1. API Gateway (Node.js)

The Gateway acts as the WebSocket hub, routing internal events to connected mobile clients.

// Internal notification endpoint in API Gateway
app.post('/internal/notify', express.json(), (req, res) => {
    const { userId, event, data } = req.body;
    if (userId && event) {
        io.to(`user:${userId}`).emit(event, data);
        return res.json({ success: true });
    }
});

2. Wallet Service (Node.js)

The Wallet Service triggers notifications whenever a balance change occurs.

const notifyUpdate = async (userId, event, data) => {
    await axios.post(`${GATEWAY_URL}/internal/notify`, { userId, event, data });
};

// Usage during payment
notifyBalanceUpdate(userId, newBalance);
notifyTransactionUpdate(userId, `Paid PHP ${amount}`, transaction);

3. Mobile App (Flutter)

Implemented SocketService to manage the persistent connection and provide data streams to the UI.

class SocketService with ChangeNotifier {
  void init({required String baseUrl, required String? userId}) {
    _socket = IO.io(baseUrl, ...);
    _socket!.on('balance_update', (data) => _balanceController.add(balance));
    _socket!.on('transaction_update', (data) => _transactionController.add(data));
  }
}

Test Results

Passed Live Balance Sync: Verified that calling /update-balance on the Wallet Service immediately updates the balance display on the Home Screen via Socket.io.

Passed Transaction Alerts: Verified that SnackBar notifications appear in the app when a transaction_update event is received.

Passed Room Isolation: Confirmed that users only receive updates for their own userId room.

Issues & Mitigations

Issue Mitigation
Connection Drops Enabled enableAutoConnect in Socket.io client.
Stale Balance Retained initial HTTP fetch as a fallback during initialization.

Recommendations

  • Background Sync: Implement background notification handling for when the app is not in the foreground.
  • Optimistic Updates: Update the UI balance immediately on the client-side before receiving the server confirmation.
