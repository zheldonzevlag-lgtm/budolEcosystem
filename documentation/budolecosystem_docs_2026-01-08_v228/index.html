v228: Cross-Platform Compliance Standards

Implementation Complete

Date: 2026-01-08 | Compliance: PCI DSS 10.2.1.3, BSP Circular 808 (E-Banking Security)

Purpose

To unify compliance standards across the ecosystem by extending administrative audit logging to budolShap and ensuring synchronization with the budolID SSO service is fully auditable and traceable.

Technical Details

1. Schema Enhancement (budolShap)

The AuditLog model in budolShap was enhanced with a metadata field (JSON type) to support structured compliance data, matching the standards established in budolPay.

// prisma/schema.prisma
model AuditLog {
  ...
  metadata  Json?
  createdAt DateTime @default(now())
}

2. Audited Admin Actions

The simulate-admin-update.mjs script was refactored to include a mandatory compliance audit trail. This ensures that any administrative change to user privileges or profiles is recorded with "Before/After" state snapshots.

// scripts/simulate-admin-update.mjs
await prisma.auditLog.create({
  data: {
    userId: admin.id,
    action: 'ADMIN_USER_UPDATE',
    metadata: {
      changes: { name: { from: old, to: new } },
      sync: { service: 'budolID', status: 'SUCCESS' },
      compliance: { standard: 'PCI DSS 10.2.1.3' }
    }
  }
});

Rationale & Compliance

  • Distributed Auditing: Ensures that administrative actions in subsidiary apps (budolShap) are as traceable as those in the core financial app (budolPay).
  • Sync Transparency: Explicitly logs the status of SSO synchronization with budolID, preventing "Ghost Privileges" where a user is downgraded in one system but remains privileged in another.
  • PCI DSS 10.2.1.3: Specifically addresses the requirement to audit all changes, additions, or deletions to any individual with root or administrative privileges.

Issues & Fixes

  • Issue: budolShap AuditLog lacked structure for detailed changes.
  • Fix: Added metadata field and pushed schema to PostgreSQL.
  • Issue: User creation in scripts failed due to missing required fields (id, password, phoneNumber).
  • Fix: Refactored simulation script to provide mock compliance-ready data for administrative test actors.

Test Results

  • ✅ Database Sync: Local budolShap update successful.
  • ✅ SSO Sync: budolID profile update returned 200 OK.
  • ✅ Audit Recording: Structured JSON metadata successfully persisted in AuditLog.

Recommendations for Future Enhancement

  • Global Audit Dashboard: Create a unified view that aggregates AuditLog entries from both budolPay and budolShap.
  • Automated Reconciler: Implement a cron job that compares budolID roles with local application roles to detect and alert on sync drifts.
