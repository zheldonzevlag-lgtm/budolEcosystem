Single-Project Hub Architecture

This manual explains how to transition from 11 separate Vercel projects to a single, unified budolecosystem hub.

Hub Mode

Concept: One Card, Many Services

Instead of having 11 cards in your dashboard, we now use the budolecosystem project as the master container. All services live "under" this project and are accessed via sub-paths.

  • The Hub: budolecosystem.vercel.app
  • Service 1 (Auth): budolecosystem.vercel.app/api/auth
  • Service 2 (Wallet): budolecosystem.vercel.app/api/wallet
  • Service 3 (Admin): budolecosystem.vercel.app/admin

Dashboard Cleanup Guide

Action Required: To achieve the clean "Single Project" look, you must manually delete the following 11 project cards from your Vercel dashboard:

  1. budolpay-api-gateway
  2. budolpay-auth
  3. budolpay-wallet
  4. budolpay-tx
  5. budolpay-payment-gw
  6. budolpay-kyc
  7. budolpay-accounting
  8. budolid
  9. budolpay-admin
  10. budolpay
  11. budolshap-beta

Note: Deleting these cards will NOT delete your code. It only removes the redundant Vercel deployment configurations.

Benefits of the Hub Strategy

  • Zero Clutter: Only one project to monitor in the dashboard.
  • Unified Logs: View all service logs in one stream.
  • Atomic Deployments: Update the entire ecosystem with one vercel deploy.
  • Cost Efficiency: Stays well within Vercel's hobby/pro project limits.
