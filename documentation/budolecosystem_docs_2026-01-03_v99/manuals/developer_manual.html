Reference ID Prefix Customization

Change Description

The referenceId used for tracking transactions in the budolPay ecosystem has been updated to include a custom prefix.

Implementation Details

1. Gateway Service Update

The payment-gateway-service now uses the JON- prefix for all newly created payment intents.

// payment-gateway-service/index.js
const transaction = await prisma.transaction.create({
  data: {
    // ... other fields
    referenceId: `JON-${Date.now()}`,
    // ...
  }
});

Impact Analysis

  • Compatibility: Webhook processing logic remains unchanged as it relies on exact string matching for referenceId.
  • Simulation Scripts: Scripts that hardcode a referenceId (like simulate-mobile-scan.js) will need to be updated with the new ID generated by the gateway during a test run.
  • Admin Dashboard: The dashboard will automatically display the new prefix as it reads directly from the database.

Verification

✅ Reference ID generation verified with new JON- prefix.

✅ Database records correctly store the prefixed string.
