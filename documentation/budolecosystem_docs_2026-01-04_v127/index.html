budolEcosystem Documentation v127

Date: 2026-01-04

Focus: SSO Payload Alignment & Session Failsafes

Purpose

To resolve the "User session expired" error that occurred despite valid authentication. The investigation revealed a mismatch between the SSO login response and the mobile app's session requirements.

The Issue

The budolID SSO service was not returning the user profile object in its /auth/sso/login response. Consequently, the mobile app would show "Authenticated" (because the token existed) but have a null user object, causing failures during payment processing.

Technical Implementation

  • Backend Fix (budolID): Updated index.js to include the full user object (id, email, names, role) in the login response.
  • Mobile Failsafe (ApiService):
    • Implemented _fetchUserProfile() to retrieve user data from /auth/verify if missing.
    • Updated login() to automatically call this failsafe if the login response is incomplete.
    • Strengthened isAuthenticated to require both token AND user profile.
  • Diagnostics: Added granular logging to processPayment to track session state in real-time.
  • Build: Released as budolPay 1.0.9.apk.

Test Results

  • Unit Tests: api_service_test.dart passed.
  • End-to-End: Verified that login now correctly populates the user state and allows payment confirmation.
