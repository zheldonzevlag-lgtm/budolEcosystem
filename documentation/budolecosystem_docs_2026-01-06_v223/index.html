Version 223

OTP Verification UX Improvements & Mobile Auth Logic Fixes

Purpose

This release addresses critical UX issues in the mobile authentication flow, specifically focusing on the OTP (One-Time Password) and PIN verification steps. It also fixes a logical mismatch between the mobile app and the SSO backend that caused identification failures for new users.

Technical Details & Fixes

1. Mobile-Backend Logic Alignment

Fixed The mobile app previously expected an exists: false field to identify new users. However, the backend returns a 404 Not Found status for non-existent users. The logic in login_screen.dart has been updated to correctly interpret the backend's error response.

// Updated identification logic
if (result['error'] != null && result['error'].toString().contains('not found')) {
    Navigator.pushNamed(context, '/registration', ...);
}

2. Auto-Verification (UX Improvement)

Improved Users no longer need to click the "Verify" button manually. The app now automatically triggers verification as soon as the 6th digit of the OTP or PIN is entered.

onChanged: (value) {
    if (value.length == 6 && !_isLoading) {
        _handleVerifyOtp();
    }
}

3. Loading States & Visual Feedback

Improved To prevent multiple submissions and provide better feedback, the "Verify" buttons now show a centered loading spinner and are disabled during processing.

  • Disabled State: Button is grayed out and non-clickable during API calls.
  • Spinner: A white CircularProgressIndicator replaces the button text.
  • Success Feedback: A green SnackBar confirms successful OTP verification.

4. Session Resilience

Fixed Added checks for _userId == null in verification handlers. If a session expires or state is lost, the user is gracefully redirected to the initial login step with a clear error message.

Test Results

Verified the following flows in the mobile environment:

  • ✅ Identify: Correctly identifies existing vs. new users.
  • ✅ Auto-OTP: Verification triggers instantly on the 6th digit.
  • ✅ Auto-PIN: Login triggers instantly on the 6th digit.
  • ✅ Loading UI: Spinner appears and button disables as expected.
  • ✅ Error Handling: Invalid OTP/PIN displays appropriate error messages.

Recommendation for Future Enhancement

  • Biometric Integration: Enable biometric login after the first successful OTP/PIN verification for even faster access.
  • Resend OTP: Add a countdown timer and "Resend OTP" button for better user recovery.
  • Input Masking: Improve the OTP input field with separate boxes for each digit for a more modern look.
