v18: Docker Ecosystem Optimization & AWS Deployment

Release Date: 2026-03-12

Purpose

The objective of this release is to containerize all core applications within the Budol Ecosystem and deploy them to AWS ECS for improved scalability, reliability, and simplified management.

Task List

  • Phase 0: Baseline & Risk Analysis
  • Phase 1: Architecture & Data Contracts
  • Phase 2: Backend & API Implementation
  • Phase 3: Frontend / App Implementation
  • Phase 4: Integration & Security
  • Phase 5: End-to-End Testing
  • Phase 6: Documentation & Audit
  • Phase 7: Release & Monitoring

Architecture Overview

We are transitioning from a monolithic `Dockerfile.backend` to individual, optimized Dockerfiles for each service:

  • budolshap: Next.js Frontend (Standalone mode)
  • budolID: SSO & Identity Provider (Prisma supported)
  • budolaccounting: Financial Ledger Service
  • budolws: Real-time WebSocket Gateway
  • budolpay: Unified Payment Gateway Monorepo
