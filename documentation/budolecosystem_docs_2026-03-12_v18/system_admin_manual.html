System Admin Manual (v18)

AWS Infrastructure Requirements

This release targets AWS ECS (Elastic Container Service) using the Fargate launch type.

1. ECR Repositories

Ensure the following ECR repositories exist in `ap-southeast-1`:

  • `budolecosystem/budolshap`
  • `budolecosystem/budolid`
  • `budolecosystem/budolaccounting`
  • `budolecosystem/budolws`
  • `budolecosystem/budolpay-gateway`
  • `budolecosystem/budolpay-auth`

2. ECS Task Definitions

New task definitions are located in `ecs/task-definitions/`. These use dynamic variables for Account ID and Region.

3. Environment Variables & Secrets

Critical: All services require `DATABASE_URL` and `JWT_SECRET`. These must be stored in AWS Secrets Manager and referenced in the Task Definition.

Monitoring & Troubleshooting

All logs are streamed to CloudWatch Logs under the `/ecs/` log group prefix.

  • Health Checks: ECS services use HTTP-based health checks on ports 3000 (shap), 8000 (id), 8005 (accounting).
  • Scaling: Fargate tasks are configured with 0.25 vCPU and 0.5 GB RAM minimum.

Reversion Plan

In case of deployment failure, update the ECS Service to point back to the previous stable image tag (e.g., `v16`).
