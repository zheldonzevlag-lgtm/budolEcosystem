Developer Manual (v18)

Docker Image Architecture

In this release, we have decoupled the services into individual Dockerfiles to allow for independent scaling and deployment.

1. BudolShap (Frontend)

File: budolshap-0.1.0/Dockerfile

Uses Next.js standalone mode for minimal image size.

2. BudolID (Identity Provider)

File: budolID-0.1.0/Dockerfile

Includes Prisma client generation in the build stage.

3. BudolAccounting

File: budolAccounting-0.1.0/Dockerfile

Note: Requires copying the shared database package from `budolpay-0.1.0/packages/database` into the build context.

4. BudolWS (WebSocket)

File: websocket-server/Dockerfile

Lightweight Node.js alpine image for handling real-time socket connections.

5. BudolPay Monorepo

File: budolpay-0.1.0/Dockerfile

Utilizes multi-stage targets (`gateway`, `auth`, `wallet`, `transaction`) to build service-specific images from the same codebase.

Deployment Workflow

Use the provided PowerShell script to build and push all images to ECR:

Script: scripts/deploy_v18.ps1

Database Schema & Migrations

Ensure that all Prisma schemas are synchronized before building. The Dockerfiles automatically run `npx prisma generate`, but `npx prisma db push` should be run manually against the production database if schema changes are made.
