Mitigation Strategy (v18)

M18-001: Prisma Consistency

Risk: Client mismatch.

Mitigation: Perform `npx prisma generate` within the Docker build process for each specific service to ensure the client matches the bundled engines.

M18-002: Relative Dependency Resolution

Risk: `budolaccounting` local file dependency failing in Docker.

Mitigation: Use the root directory as Docker build context or use a multi-stage approach to copy the necessary `packages/database` directory into the image before installing dependencies.

M18-003: Environment Variable Security

Risk: Secrets leakage.

Mitigation: Utilize AWS Secrets Manager and ECS Parameter Store integration to inject sensitive environment variables at runtime, avoiding hardcoded values in images.
