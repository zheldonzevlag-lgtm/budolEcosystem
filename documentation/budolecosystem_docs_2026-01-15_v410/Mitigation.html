Mitigation Plan - Version 410

M-410-01: UX Guidance

Risk: User confusion over phone-only restriction.

Action: Updated hint text in SendMoneyScreen to explicitly show "e.g. 0912-345-6789" and changed the label to "Phone Number" instead of generic "Recipient".

M-410-02: Input Sanitization

Risk: Hyphens causing API failures.

Action: Implemented replaceAll('-', '') in all transaction handlers before sending data to the ApiService.

M-410-03: Precision Enforcement

Risk: Financial discrepancies due to rounding.

Action: Enforced toStringAsFixed(2) in all UI displays and success dialogs. Used CurrencyInputFormatter to block more than 2 decimal digits at the source.
