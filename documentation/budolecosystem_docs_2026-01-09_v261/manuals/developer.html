Developer Manual - v261

Technical Implementation Details

← Back to v261 Index

Purpose

To fix the "NotFound" error when users try to add recipients to favorites and ensure the favorites list correctly facilitates subsequent "Send Money" transactions.

Issues Identified

  • Incorrect API Prefix: The ApiService in the mobile app was calling /api/auth/favorites, but the API Gateway only proxies /auth to the Auth Service. This resulted in a 404 Not Found error.
  • Workflow Verification: Need to ensure that tapping a favorite recipient correctly populates the recipient field in the Send Money screen.

Technical Details & Code Changes

1. API URL Correction

Modified lib/services/api_service.dart to remove the redundant /api prefix for favorites endpoints.


// Before:
final url = '$baseUrl/api/auth/favorites';

// After:
final url = '$baseUrl/auth/favorites';

Impacted methods: getFavorites(), addFavorite(), and removeFavorite().

2. Navigation Logic Verification

Verified lib/screens/favorites_screen.dart correctly passes the recipient's email or phone number to the SendMoneyScreen.


onTap: () {
  Navigator.push(
    context,
    MaterialPageRoute(
      builder: (context) => SendMoneyScreen(
        initialRecipient: email.isNotEmpty ? email : phone,
      ),
    ),
  );
},

3. Version Increment

  • Updated pubspec.yaml to 1.2.25+25.
  • Updated lib/services/api_service.dart fallback version to 1.2.25.

Test Results

  • API Connectivity: Verified that removing /api prefix allows successful connection to Auth Service favorites endpoints through Gateway.
  • Navigation: Confirmed that selecting a favorite recipient correctly initializes the SendMoneyScreen with the recipient's details.
  • Build: APK build initiated for version 1.2.25.

Future Enhancements

  • Implement local caching for favorites to improve offline experience.
  • Add ability to categorize favorites (e.g., Family, Friends, Business).
  • Enhance error handling with more descriptive user messages for API failures.
