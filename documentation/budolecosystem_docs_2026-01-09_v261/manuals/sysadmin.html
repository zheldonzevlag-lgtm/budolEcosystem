Deployment Procedure

1. Build Mobile Application

Run the Flutter build command to generate the release APK:

flutter build apk --release

2. Version Verification

Ensure the built APK is version 1.2.25+25. The splash screen should display "budolEcosystem 1.2.25".

3. Distribution

Move the generated app-release.apk to budolEcosystem/android-installer/budolPay_1.2.25.apk.

Configuration Changes

No backend configuration changes (environment variables or database schemas) were required for this release. The fix was entirely client-side in the ApiService URL mapping.

Verification Checklist

  • [ ] Mobile App version 1.2.25 is visible on Splash Screen.
  • [ ] Tapping "Add to Favorites" in Transaction Details does not return a 404 error.
  • [ ] Favorites list loads successfully.
  • [ ] Selecting a favorite navigates to Send Money with the correct recipient pre-filled.

Rollback Strategy

If issues occur, revert to Mobile version 1.2.24 available in the android-installer folder (if archived) or re-build from the previous stable git commit.
