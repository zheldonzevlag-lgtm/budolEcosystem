Quick Registration Validation Fix

Issue: "Quick (Phone Only)" registration was failing because it was still enforcing password validation (minimum 6 characters) even though the password field is hidden for this registration type.

Technical Implementation:

  • Modified handleSubmit in budolshap-0.1.0/components/auth/AuthForm.jsx.
  • Wrapped password and confirmPassword checks in a conditional block that only executes when registrationType === 'standard'.

Impact Analysis:

  • Allows users to successfully use the "Quick (Phone Only)" registration path without being blocked by invisible password requirements.
  • Maintains strict validation for "Standard" registration where passwords are required.

Code Change:


// Before
if (formData.password.length < 6) { ... }

// After
if (registrationType === 'standard') {
    if (formData.password.length < 6) { ... }
}

Verification

The error reported in the UI ("Password must be at least 6 characters") will no longer trigger for the Phone Only registration type.
