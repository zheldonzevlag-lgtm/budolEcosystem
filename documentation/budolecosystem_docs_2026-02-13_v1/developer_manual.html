Developer Manual: Product Variation System

Technical Architecture

The variation system uses a "Tiered Matrix" approach. Products can have multiple tiers (e.g., Color, Size), and each combination is mapped to a row in the variation_matrix.

Data Contract

The Product model in schema.prisma was updated with:

model Product {
  ...
  parent_sku       String?
  tier_variations  Json?   @default("[]")
  variation_matrix Json?   @default("[]")
}

Implementation Details

1. Variation Matrix Manager

A new admin component VariationMatrixManager.jsx handles the dynamic generation of variation combinations.

// Logic for generating the matrix from tiers
const generateMatrix = () => {
    // ... combinations logic
    combinations.map(combo => ({
        sku: `${parentSku}-${options.join('-')}`,
        tier_index: combo,
        price: defaultPrice,
        stock: 0
    }));
}

2. API Integration

Updated POST/PUT routes in route.js and [productId]/route.js to handle the new fields.

3. Cart Management

The cartSlice.js now uses a composite key ${productId}_${variationId} to ensure each variation is treated as a unique item in the cart.

Dependencies

  • Prisma ORM for database modeling
  • Redux Toolkit for state management
  • Tailwind CSS for UI styling
