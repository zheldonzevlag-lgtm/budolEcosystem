budolEcosystem Documentation v130

Date: 2026-01-04

Focus: Database Constraint Resolution & UI Alignment

Purpose

To resolve Foreign Key constraint violations in the Wallet Service and fix horizontal alignment issues in the mobile app's service grid.

Issues Addressed

  • Prisma P2003 Error: The wallet.create() call was failing with a Foreign key constraint violated: Wallet_userId_fkey because the userId did not exist in the User table of the Wallet Service's database.
  • Zero Balance: The 1,000 PHP testing balance was not appearing because the wallet creation was failing silently due to the constraint error.
  • UI Misalignment: The budolLoan item (using a Text widget for ₱) was not horizontally aligned with the other icon-based service items.

Technical Implementation

  • Wallet Service (Failsafe Sync): Modified wallet-service/index.js to check if a User record exists before creating a Wallet. If missing, it creates a placeholder User record to satisfy the database relationship.
  • Mobile UI (Alignment Fix):
    • Wrapped the service item content in a fixed-size SizedBox(32x32) in home_screen.dart.
    • Added Center alignment to the custom Philippine Peso (₱) text widget.
    • Ensured all service items share the same vertical baseline via mainAxisAlignment: MainAxisAlignment.center in the item column.
  • Build: Bumped version to 1.1.2+1 and generated a new release APK.

Test Results

  • Constraint Fix: Confirmed that wallets are now successfully created for new users by satisfying the userId foreign key requirement.
  • Balance Display: Verified that the 1,000 PHP initial balance is correctly fetched and displayed on the Home Screen.
  • UI Layout: Confirmed budolLoan is now perfectly aligned with Load, Bills, and other services.
