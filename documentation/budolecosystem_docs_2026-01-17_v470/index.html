BudolEcosystem Documentation v470

Release Date: 2026-01-17

Status: Compliance Verified

Purpose

This version fixes a critical Auth Guard conflict where the application was redirecting to the Login screen prematurely, bypassing the Marketing Ads onboarding. This was caused by the global auth listener in main.dart not accounting for public onboarding routes.

Implementation Phases

  • Phase 0: Identified the conflict in _handleAuthChange within main.dart.
  • Phase 1: Modified _handleAuthChange to detect and ignore redirections when on Routes.splash or Routes.marketing.
  • Phase 2: Standardized SplashScreen navigation using RouteSettings to ensure the Auth Guard can identify the current route.
  • Phase 3: Increased navigation delay in SplashScreen to 1000ms for improved stability on slower web environments.
