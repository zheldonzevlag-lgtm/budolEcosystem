System Admin Manual - Version 478

Purpose

This manual provides instructions for system administrators to deploy and monitor the changes introduced in v478, specifically the Asia/Manila timezone standardization and mobile app biometric updates.

Deployment Instructions

Backend Services

The following services must be restarted after deploying the v478 changes:

  • transaction-service
  • settlement-service
  • wallet-service
Note: No database migrations are required for this release as the timezone enforcement is handled at the application level.

Mobile Application

Distribute the budolPay 1.3.17.apk from the android-installer directory to authorized testing and production channels.

Monitoring and Verification

Audit Log Verification

Verify that audit logs contain the metadata.timestamp field with the correct UTC+8 offset. Sample verification command:

docker logs transaction-service | grep "P2P_TRANSFER_COMPLETED"

Compliance Audit

Ensure that all generated financial reports for BSP (Bangko Sentral ng Pilipinas) use the standardized Manila time to avoid reporting discrepancies.

Technical Explanation

The timezone standardization uses a hardcoded offset of +8 hours. This was chosen over server-level timezone configuration to ensure consistent behavior across different cloud provider regions (e.g., AWS us-east-1 vs ap-southeast-1).
