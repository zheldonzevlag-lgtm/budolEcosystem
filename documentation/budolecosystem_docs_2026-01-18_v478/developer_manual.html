Backend: Asia/Manila Timezone Enforcement

All backend services have been updated to use the Asia/Manila timezone (UTC+8) for audit logs and error responses. This ensures consistency for regulatory reporting (BSP Circular 808).

Implementation Pattern

The following pattern was used to calculate the Asia/Manila timestamp:

new Date(new Date().getTime() + (8 * 60 * 60 * 1000)).toISOString()

  • Transaction Service: Updated audit logs for P2P transfers and Cash In/Out.

Mobile App: Biometric Status Refresh

Fixed a bug where the login screen would display a stale biometric status after setup or background changes.

Changes in login_screen.dart

Added WidgetsBindingObserver to detect app resumption and trigger a re-check of biometric capabilities.

@override void didChangeAppLifecycleState(AppLifecycleState state) { if (state == AppLifecycleState.resumed) { _checkBiometrics(); } }
login_screen.dart

Phase 8: Dependencies and Software Choice Rationale

Flutter & Dart

Chosen for cross-platform efficiency. The use of WidgetsBindingObserver is a native Flutter pattern for lifecycle management, ensuring we don't waste resources when the app is in the background.

Local Auth Package

Version 2.3.0 was used. It provides a standardized interface for biometric hardware, abstracting away Android/iOS specific implementations.

Node.js & Express

Used for backend services due to high throughput for JSON-based audit logs. The timezone enforcement was handled at the application level to avoid database-specific timezone quirks.

Build Process

The Android app was built using the following command:

flutter build apk --release
