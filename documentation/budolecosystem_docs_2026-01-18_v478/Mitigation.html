Mitigations - Version 478

M-478-01: Timezone Standardization

Risk: Compliance discrepancy due to timezone mismatch.

Mitigation: Enforced Asia/Manila (UTC+8) at the application level across all core services. Added automated audit log metadata that explicitly includes the offset.

M-478-02: Legacy Log Fragmentation

Risk: Difficulty in cross-referencing logs across release versions.

Mitigation: Centralized logging system (ELK/Splunk) configured to normalize all incoming logs to UTC+8 regardless of source timestamp, while preserving original source timestamps for forensic integrity.

M-478-03: Biometric UI Resilience

Risk: User confusion due to stale biometric status.

Mitigation: Implemented WidgetsBindingObserver in the mobile app to force a capabilities check every time the app regains focus (resumes from background).
