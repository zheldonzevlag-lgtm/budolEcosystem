Documentation v143: Mobile Balance Display Fix

Date: 2026-01-04 | Version: v143

Fixed

Purpose

To resolve the persistent spinning animation (loading state) on the budolPay Mobile home screen when fetching the available balance.

Issues Identified

  • Potential Network Hang: The balance fetch request could hang indefinitely if the gateway was unreachable, keeping the UI in a loading state.
  • Parsing Vulnerability: Direct assignment of JSON values to double variables without type checking could cause silent failures in the setState block.
  • Initialization Race Condition: The ApiService might not have fully loaded its session when getBalance() was first called.

Technical Implementation

1. Enhanced ApiService (api_service.dart)

Added robust initialization checks and HTTP timeouts to ensure requests either complete or fail within a reasonable timeframe (10 seconds).

// Added timeout and logging to getBalance
final response = await http.get(...).timeout(const Duration(seconds: 10));

2. Robust Home Screen State (home_screen.dart)

Modified _fetchBalance to use a comprehensive try-catch-finally pattern, ensuring _isLoading is set to false regardless of the outcome. Added type-safe parsing for the balance value (handling both num and String types).

// Robust parsing logic
if (balanceData is num) {
  balanceValue = balanceData.toDouble();
} else if (balanceData is String) {
  balanceValue = double.tryParse(balanceData) ?? 0.0;
}

Test Results

  • Gateway Connectivity: Verified Gateway responds correctly to balance requests via curl and test-gateway-balance.js.
  • Database Verification: Confirmed user wallet exists with a balance of 1000.00 PHP using check-wallet.js.
  • Service Stability: All ecosystem services are running and successfully proxying requests to the Wallet service.

Risk & Mitigation

  • Risk: Mobile device cannot reach the local IP (192.168.1.24) due to firewall or subnet differences.
  • Mitigation: Added detailed logging in api_service.dart to help diagnose network failures and implemented a 10-second timeout to prevent UI hangs.

Recommendations for Future Enhancement

  • WebSockets for Balance Updates: Move from polling to WebSocket-based updates to eliminate manual refresh needs.
  • Offline Cache: Implement local caching of the last known balance to show immediate data while fetching the latest from the server.
