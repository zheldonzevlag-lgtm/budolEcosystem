Deployment & Configuration

This version includes critical fixes for the Flutter Web deployment. Admins should ensure the following configurations are in place:

1. Flutter Web Renderer

The system now uses a hybrid renderer approach. While the command line flag --web-renderer html was deprecated in some CLI versions, we now enforce it via window.flutterConfiguration in the index.html.

2. Resource Pathing (Base Href)

Ensure that the base href in web/index.html is set to / for root deployments or the appropriate subdirectory. In v459, this is hardcoded to / to resolve MIME type errors (text/html served instead of application/javascript).

3. Cache Clearing

Due to changes in the Dart Dev Compiler (DDC) module structure, it is recommended to run a full clean before building:

flutter clean
flutter pub get
flutter build web --release

Monitoring

Check the browser console logs for MarketingAdsScreen: Layout determined to verify that the rendering guards are active and providing valid dimensions.
