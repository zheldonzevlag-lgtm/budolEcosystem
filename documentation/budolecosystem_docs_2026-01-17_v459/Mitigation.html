Mitigation Actions for v459

1. Rendering Guards (M-459-01)

Risk: Web rendering engine failure.

Action: Implemented LayoutBuilder with fallback dimensions in MarketingAdsScreen. This ensures that even if the engine fails to provide window sizes, the UI remains visible and interactive.

2. Diagnostic Overrides (M-459-02)

Risk: Navigation hangs.

Action: Added FORCE ADS and FORCE LOGIN buttons in debug builds to allow manual bypass of any automated navigation logic that might fail due to environmental timing issues.

3. Base Href Standardization (M-459-03)

Risk: MIME type errors.

Action: Explicitly set base href="/" in web/index.html to ensure the browser correctly resolves relative paths for JavaScript modules, preventing the server from serving index.html (text/html) instead of the requested script.
