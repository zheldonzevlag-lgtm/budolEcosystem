Developer Manual - Unified Commands & Real-time Sync

Purpose

This update focuses on streamlining the development workflow by unifying service management and ensuring robust real-time synchronization between the mobile app and the admin dashboard.

Technical Details

1. Unified Root Commands

Implemented concurrently in the root package.json to allow starting all core services and the admin dashboard with a single command.

// Root package.json
"scripts": {
  "start": "concurrently \"npm run start -w api-gateway\" ...",
  "dev": "concurrently -n \"GATEWAY,AUTH,WALLET,TX,ADMIN\" -c \"cyan,magenta,yellow,green,blue\" ..."
}

2. Socket.io Admin Room Integration

Enhanced the API Gateway to support a specialized admin room. Any socket connecting with the ID admin is automatically joined to this room to receive global transaction updates.

Implementation: api-gateway/index.js now handles socket.join('admin') and the /internal/notify endpoint supports an isAdmin flag to broadcast events to the admin room.

3. Real-time Transaction Recording

The Transaction Service and Wallet Service now proactively notify the Admin via the Gateway whenever a transaction state changes (e.g., Cash In completion, Transfer success).

Issues & Fixes

  • Issue: Admin required manual refresh to see new transactions.
  • Fix: Integrated Socket.io client in apps/admin/app/transactions/page.tsx to listen for new_transaction events and update the UI state in real-time.

Dependencies

  • concurrently: Root dependency for multi-service management.
  • socket.io-client: Added to Admin app for real-time monitoring.

Recommendations

Move to a dedicated message broker (like Redis or RabbitMQ) for inter-service communication instead of direct HTTP calls for notifications as the ecosystem scales.
