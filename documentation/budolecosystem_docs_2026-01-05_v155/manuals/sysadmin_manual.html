System Admin Manual - Infrastructure & Deployment

Overview

This version introduces a unified entry point for service orchestration, simplifying the deployment and monitoring of the budolPay microservices architecture.

Deployment Procedures

Starting the Full Stack

From the root of the budolpay-0.1.0 directory, you can now launch all required services and the admin dashboard using:

npm run dev

This command starts the following components:

  • API Gateway (Port 8080)
  • Auth Service (Port 8001)
  • Wallet Service (Port 8002)
  • Transaction Service (Port 8003)
  • Admin Dashboard (Port 3000)

Database Initialization

Ensure the database is synced before starting the services:

npm run db:local -w @budolpay/database

Configuration Updates

Environment Variables: Ensure NEXT_PUBLIC_GATEWAY_URL is set in the Admin app's environment to point to the API Gateway (default: http://localhost:8080).

Risk & Mitigation

Risk: High CPU/Memory usage when running all services concurrently on a single development machine.
Mitigation: Use Docker Compose in production environments to isolate resources and enable scaling.

Technical Details

The system now uses a shared admin room in Socket.io for centralized monitoring. SysAdmins can monitor the Gateway logs to see real-time connections and event emissions.
