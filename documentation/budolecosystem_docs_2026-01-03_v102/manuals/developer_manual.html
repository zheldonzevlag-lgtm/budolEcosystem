Developer Manual - v102

The Issue: Port Hijacking

When a user accesses budolShap via a LAN IP (e.g., 192.168.1.24), the BudolPayAdapter attempts to fix the checkoutUrl if the gateway returns localhost. However, the previous implementation used the host property, which includes the port.

Incorrect Logic (v101):

checkoutUrlObj.host = successUrlObj.host; // Resulted in 192.168.1.24:3001

Correct Logic (v102):

checkoutUrlObj.hostname = successUrlObj.hostname; // Results in 192.168.1.24:8004

Technical Specification

The URL interface in JavaScript distinguishes between host (hostname + port) and hostname (address only). By switching to hostname, we preserve the original port 8004 intended for the payment-gateway-service.

Impact Analysis

  • BudolShap: Successfully initiates payment and redirects to the correct service.
  • Payment Gateway: Receives traffic on port 8004 as expected.
  • Next.js: No longer returns a 404 because the request is no longer hitting the Next.js app server for checkout.
