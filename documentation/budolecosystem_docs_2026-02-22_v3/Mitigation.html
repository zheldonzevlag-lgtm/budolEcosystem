Mitigation Strategies - v3 (2026-02-22)

Risk Mitigations

R-001: Cloudinary API Downtime

  • Mitigation: Implement try-catch blocks in the API route to gracefully handle errors and inform the user.
  • Fallback: Users can still upload images without background removal if the AI service is down.

R-002: Exceeding Credit Limits

  • Mitigation: Monitor usage via Cloudinary Dashboard.
  • Action: Upgrade plan or implement a local caching mechanism if usage spikes.

R-003: Slow Internet/Timeouts

  • Mitigation: Use loading indicators (spinners) and disable buttons during processing to prevent multiple submissions.
  • Action: Set reasonable timeout limits on the client side fetch request.
