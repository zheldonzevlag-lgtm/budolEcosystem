Developer Manual v186

Purpose

Phase 4 integrates the commerce (budolShap) and accounting (budolAccounting) layers into the production ecosystem, ensuring unified authentication and automated financial tracking.

Technical Details

1. budolShap SSO Integration

The login page now uses NEXT_PUBLIC_SSO_URL for client-side redirection, ensuring the "Login with budolID" button points to the correct production environment.

// app/(public)/login/page.jsx
<a href={`${process.env.NEXT_PUBLIC_SSO_URL || '...'}/login?apiKey=bs_key_2025`}>
    Login with budolID
</a>

2. budolAccounting Vercel Readiness

The accounting service is configured as a serverless function on Vercel, with automatic DATABASE_URL resolution from Vercel Postgres.

// budolAccounting/index.js
if (isProd) {
    if (!process.env.DATABASE_URL && process.env.POSTGRES_PRISMA_URL) {
        process.env.DATABASE_URL = process.env.POSTGRES_PRISMA_URL;
    }
}

3. Standardized Shared Secrets

Both services now utilize the standardized JWT_SECRET for ecosystem-wide token verification, enabling seamless SSO across commerce and accounting functions.

Dependencies

  • budolID (Phase 2) must be operational for SSO.
  • budolpay (Phase 3) must be operational for ledger entries triggered by transactions.
