System Admin Manual - v58.0.0

Infrastructure Note: When testing the ecosystem across multiple devices (e.g., mobile phone accessing a laptop), ensure that all services are configured to trust the LAN IP of the host machine.

1. Cross-Service Communication

The budolPay gateway now includes basic IP detection logic. However, for production deployments, the BUDOLSHAP_URL environment variable should always be explicitly set to the public-facing URL of the application.

2. Email Configuration Verification

If order confirmation emails are not arriving:

  1. Check server logs for "ORDER PLACED (Email Not Configured)" - this indicates SMTP settings are missing in the database or environment.
  2. Ensure SMTP_USER and SMTP_PASS (or BREVO_API_KEY) are correctly set in the .env file of budolshap-0.1.0.
  3. Verify that the firewall allows outgoing connections on ports 587 or 465.

3. Rollback Procedures

If the host rewriting logic causes issues with custom domains:

  1. Revert lib/payment/adapters/budolpay-adapter.js to the previous version.
  2. Explicitly set NEXT_PUBLIC_BASE_URL to the desired domain in the environment variables.
