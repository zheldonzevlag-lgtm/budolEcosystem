Authentication Configuration

The system now supports multi-identifier authentication for BudolShap users. Administrators should be aware of how this impacts security and user management.

Setting Value Description
Allowed Identifiers Email, Phone (+63) Users can authenticate using either their registered email or phone number.
Phone Format E.164 (+639XXXXXXXXX) The backend automatically normalizes phone numbers to include the +63 prefix if missing.
Rate Limiting Global Identifier Key Rate limiting is applied based on the identifier provided (Email or Phone) + IP address.

Troubleshooting Login Issues

If a user reports being unable to log in via phone:

  • Verify that the user's phone number in the database is correctly formatted with the +63 prefix.
  • Ensure the user has completed their registration. Quick registration accounts may need to be verified before login is enabled depending on security settings.
  • Check the AuditLog for failed login attempts associated with the identifier.
