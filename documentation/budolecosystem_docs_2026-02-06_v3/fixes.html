Bug Fixes

Version: Login Toggle & Animation | Release Date: 2026-02-06

Backend Login Query (findFirst)

Fixed an issue where the login API only searched for users by email. The query now uses findFirst with an OR condition to include phoneNumber.

Phone Number Normalization

Added logic to automatically prepend the +63 country code and remove leading zeros from phone numbers submitted via the login form, ensuring they match the database format.

Input Required Validation

Fixed a bug where both email and phone fields were required simultaneously. Validation is now conditional based on the selected login method.
