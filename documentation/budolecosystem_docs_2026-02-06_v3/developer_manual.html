Developer Manual

Version: Login Toggle & Animation | Release Date: 2026-02-06

Purpose

To implement a togglable login interface that supports both Email and Mobile Number authentication. This enhancement improves user onboarding by providing flexible login options and a polished, animated UI that aligns with modern application standards.

Rationale

Users in the Philippine market frequently alternate between using email and mobile numbers for digital services. By providing a clear toggle and animated transition, we reduce cognitive load and clearly signal which identifier is being used. This approach also simplifies the backend logic by consolidating authentication through a single flexible endpoint.

Technical Implementation

State Management

A new state loginMethod was added to AuthForm.jsx to track whether the user is logging in via 'email' or 'phone'.

const [loginMethod, setLoginMethod] = useState('email') // 'email' or 'phone'

Animated Transitions

Animations were implemented using Tailwind CSS transition-all and transform classes. A sliding background pill was used for the toggle switch, and horizontal sliding transitions were applied to the input containers.

<div className={`transition-all duration-300 transform ${loginMethod === 'email' ? 'translate-x-0 opacity-100 relative' : '-translate-x-full opacity-0 absolute'}`}>
    {/* Email Input */}
</div>

Backend Multi-Identifier Support

The login route was updated to use Prisma's findFirst with an OR condition, allowing authentication by either email or phoneNumber.

const user = await prisma.user.findFirst({
    where: {
        OR: [
            { email: identifier.toLowerCase() },
            { phoneNumber: identifier }
        ]
    }
})

8. Dependencies & Software Choices

Tailwind CSS: Chosen for rapid UI development and utility-first styling. Its transition utilities (transition-all, duration-300) were instrumental in implementing the smooth login toggle animations.

Prisma ORM: Used for database interactions. The OR condition in findFirst allowed for seamless multi-identifier (email/phone) lookups without complex SQL queries.

Lucide React: Provides the iconography used in the mobile number prefix and toggle indicators, ensuring a consistent visual language.

Next.js (App Router): The framework's API routes allowed for a clean separation of backend logic, while client components enabled the rich interactivity required for the animated form.

9. Testing & Verification

Automated verification was performed using a custom Node.js script located at /scripts/test_scripts/budolShap/v3/verify-login-toggle.mjs. This script verifies both backend API support and frontend state/UI implementation.
