System Admin Manual v484

Web Deployment Security

Version v484 introduces a robust fallback mechanism for secure storage. However, system admins MUST prioritize HTTPS deployment. When the app detects a non-secure context (e.g., served over HTTP to a remote client), it will use SharedPreferences which stores sensitive data in plain text in the browser's localStorage.

Maintenance & Upgrades

When upgrading the mobile app, ensure that flutter_secure_storage version 9.2.4 or higher is used. The encrypt parameter in WebOptions has been removed in favor of the manual fallback logic implemented in ApiService and BiometricService.
