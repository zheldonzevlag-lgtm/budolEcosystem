Applied Fixes v484

F-484-01: Flutter Secure Storage 9.2.4 Compilation

Problem: Compilation failed after upgrading to flutter_secure_storage: 9.2.4 because the encrypt parameter was removed from WebOptions.

Fix: Removed the encrypt parameter and migrated the conditional encryption logic to a manual SharedPreferences fallback in ApiService and BiometricService.

F-484-02: Secure Context Detection

Problem: Previous Web Crypto detection only checked for crypto.subtle, which could still fail if the context was not secure.

Fix: Updated js_helper_web.dart to check isSecureContext before declaring Web Crypto support.

F-484-03: Flutter Deprecation & Code Quality

Problem: Multiple Flutter 3.x deprecation warnings (withOpacity, dart:js) and async context safety issues (use_build_context_synchronously).

Fix:

  • Migrated all withOpacity calls to withValues(alpha: ...) to prevent precision loss.
  • Replaced deprecated dart:js with dart:js_interop and dart:js_interop_unsafe in js_helper_web.dart.
  • Implemented if (!mounted) return; checks before accessing BuildContext in async gaps across multiple screens (Login, KYC, Settings).
  • Fixed unnecessary_brace_in_string_interps in marketing_ads_screen.dart.
  • Removed unused imports and variables in face_embedding_test.dart.

F-484-04: Asia/Manila Timezone Compliance

Problem: Transaction timestamps were displaying device-local time, leading to inconsistency for users traveling or using different system settings.

Fix: Standardized all transaction date displays to Asia/Manila (UTC+8) using a centralized DateUtils helper. Updated HomeScreen, TransactionHistoryScreen, and TransactionDetailsScreen.
