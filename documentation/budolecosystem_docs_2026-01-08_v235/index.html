v235: Ecosystem-Wide Telemetry

Unified Monitoring

Purpose

To synchronize the compliance monitoring capabilities across budolPay and budolShap, ensuring a consistent security posture and real-time visibility for administrators across the entire financial ecosystem.

Unified Features

  • Real-time Indicators: Both platforms now feature the "Real-time Telemetry" header with a pulsating emerald status dot.
  • Pulsating Shield Icon: The main shield icon now emits a "sonar" pulse when active, transitioning to a rose-colored alarm pulse during violations.
  • Proactive Telemetry API: Unified backend health checks for PCI DSS, BSP Circular 808, and DPA 2012 compliance.

Technical Changes (budolPay)

  • Component Created: apps/admin/components/ComplianceShield.tsx (Reactive TypeScript implementation).
  • API Endpoint Created: apps/admin/app/api/compliance/telemetry/route.ts (Real-time backend checks).
  • Dashboard Integration: Replaced static compliance block in apps/admin/app/page.tsx with the reactive component.

Compliance Logic (budolPay)

// Critical Health Checks implemented in budolPay:
- PCI DSS: Monitoring for Security Alerts in Audit Logs (24h window).
- BSP Logs: Verifying Financial Audit Log generation (12h window).
- BIR Sync: Monitoring high-volume e-invoice synchronization delays.
- NPC Privacy: Detecting Unauthorized Data Access attempts in logs.
