Test Script Consolidation & Reorganization

Status: Completed

Version: v306.0.0 | Date: 2026-01-10

Purpose

To centralize all ecosystem-wide and service-specific test scripts into a single, organized directory. This prevents root-level clutter and provides a standardized location for developers to find and execute diagnostic tools.

Changes Applied

  • Centralization: Moved 25+ test scripts from the project root and service subfolders (e.g., budolshap-0.1.0) to the unified /scripts/test_scripts/ directory.
  • Refactoring: Updated internal imports in critical scripts (like test-admin-api.mjs) to use correct relative paths after being moved.
  • Standardization: Renamed ES module scripts to .mjs to ensure compatibility with Node.js module resolution when running outside their original package context.

New Directory Structure

budolEcosystem/
├── scripts/
│   └── test_scripts/
│       ├── network-util.js (Network Detection)
│       ├── repair-env.js (Env Fixer)
│       ├── test-admin-api.mjs (Admin API Tests)
│       ├── test-auth-me.js (Auth Diagnostics)
│       └── ... (20+ other test scripts)
└── ...

Technical Details

Path Adjustments: Scripts that import local modules now use double-parent traversal (../../) to reach service-specific libraries:

import { generateToken } from '../../budolshap-0.1.0/lib/token.js';

Execution Policy: All future test scripts must be saved in /scripts/test_scripts/. If a script depends on service-specific node_modules, it should be run with proper path context or as a standalone utility.

Test Results

  • Move Verification: PASSED (Files successfully relocated from root and subfolders)
  • Script Execution: PASSED (Verified test-admin-api.mjs correctly imports modules from its new location)
  • Runner Integrity: PASSED (Main start-ecosystem.js correctly imports utilities from the new path)

Risk & Mitigation

Risk: Some legacy scripts might still have hardcoded relative paths that weren't caught during bulk move.

Mitigation: Developers should update paths on an as-needed basis when running older scripts. All primary diagnostic tools have been verified.
