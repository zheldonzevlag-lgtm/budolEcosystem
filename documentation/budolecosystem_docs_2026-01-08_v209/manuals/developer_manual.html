Developer Manual

Technical implementation details for the budolID SSO Service.

1. Purpose & Explanation

The budolID service serves as the Central Identity Provider (IdP) for the entire budolEcosystem. Its primary purpose is to provide a single, secure entry point for user authentication, enabling Single Sign-On (SSO) across multiple applications (budolShap, budolPay, etc.) without requiring users to log in separately to each service.

This update implements the core SSO routes that were previously missing, allowing applications to redirect unauthenticated users to a central login page and receive a secure JWT token upon successful authentication.

2. Issues & Fixes

Issue: Cannot GET /login

The /login route was undefined in the Express application, causing a 404 error when users attempted to log in via budolID.

Fix: Route Implementation

Implemented GET /login, POST /auth/sso/login, and POST /api/verify-token in api/index.js.

3. Technical Details

Code Architecture

The implementation uses Express.js for the API, Prisma ORM for database interactions (PostgreSQL), bcryptjs for password hashing, and jsonwebtoken (JWT) for secure session tokens.

Implementation Snippet (SSO Logic)

// Token generation and session creation
const token = jwt.sign(
    { sub: user.id, email: user.email, role: user.role, appId: ecosystemApp.id }, 
    JWT_SECRET, 
    { expiresIn: '24h' }
);

await prisma.session.create({
    data: {
        token,
        userId: user.id,
        appId: ecosystemApp.id,
        expiresAt: new Date(Date.now() + 24 * 60 * 60 * 1000)
    }
});

Dependencies

  • express: Web framework
  • @prisma/client: Database ORM
  • bcryptjs: Password security
  • jsonwebtoken: Session security
  • cors: Cross-Origin Resource Sharing

5. Risk & Mitigation

Risk: Token Theft

If a JWT is intercepted, an attacker could impersonate the user.

Mitigation: Use short-lived tokens and HTTPS (production).

Risk: Bruteforce

Attackers might try to guess passwords on the SSO page.

Mitigation: Implement rate limiting (future enhancement).

6. Recommendations for Future Enhancement

  • Implement 2FA/MFA for enhanced account security.
  • Add Rate Limiting to the login route to prevent brute-force attacks.
  • Support Social Logins (Google, GitHub) via budolID.
  • Implement Refresh Tokens to allow longer sessions without compromising security.
