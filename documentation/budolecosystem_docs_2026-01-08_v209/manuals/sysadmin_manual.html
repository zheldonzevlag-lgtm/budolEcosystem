System Admin Manual

Operational and deployment guide for the budolID Identity Service.

1. Infrastructure Changes

The budolID service has been upgraded from a skeleton API to a fully functional Identity Provider. This requires the service to be continuously running on port 8000.

Service Endpoint

http://localhost:8000

Database

PostgreSQL (budolid schema)

2. Environment Configuration

Ensure the following variables are set in the .env file:

Variable Description Value (Local)
DATABASE_URL Primary DB connection string postgresql://...
JWT_SECRET Secret key for signing tokens budolid-super-secret-key
PORT Port for the service to listen on 8000

3. Deployment Procedures

Step 1: Install Dependencies

npm install

Step 2: Database Synchronization

npx prisma generate
npx prisma db push

Step 3: Start Service

npm start

4. Operational Runbook

Health Check

Monitor the service health via GET http://localhost:8000/api/health. A status of "ok" indicates the service and database are reachable.

Issue: Port Conflict (EADDRINUSE)

If the service fails to start due to port 8000 being in use:
1. Run netstat -ano | findstr :8000
2. Identify the PID and run taskkill /F /PID [PID]

5. Rollback Procedures

In the event of critical failure, revert the api/index.js file to its previous skeleton state.

# Revert to skeleton
git checkout HEAD^ api/index.js
npm start
