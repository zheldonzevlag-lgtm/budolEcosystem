Technical Implementation Details

This update focuses on the transition of 7 microservices to Vercel Serverless and AWS EC2. Developers must align with the new index.js structure to support ephemeral execution.

Code Logic Explanations

Each service must now export its express app for Vercel, rather than calling app.listen() directly in the main thread.

// Before: Local Execution app.listen(PORT, () => { console.log(`Service running on ${PORT}`); }); // After: Vercel/Serverless Support module.exports = app;

Architectural Decisions

  • Statelessness: Services must not store state in-memory. Use Redis or the AWS RDS database for all session/transaction state.
  • Socket Handling: Only the api-gateway handles Socket.io. It must be deployed to AWS EC2 to maintain persistent connections.

Impact Analysis

Existing inter-service communication via localhost will break. All SERVICE_URL environment variables must be updated to their respective Vercel/AWS domains.
