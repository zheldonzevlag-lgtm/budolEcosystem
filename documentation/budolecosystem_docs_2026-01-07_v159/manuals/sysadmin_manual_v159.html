Infrastructure & Deployment Procedures

This version introduces the hybrid Vercel/AWS deployment model. Admins must manage two distinct cloud environments.

Environment Variable Modifications

All services require the following keys in Vercel/AWS Parameter Store:

  • DATABASE_URL: postgresql://budol_admin:[PASS]@budol-db.cluster-xxxx.rds.amazonaws.com:5432/budolpay
  • AWS_S3_ASSET_BUCKET: budolpay-assets-production

Database Schema Changes

No immediate breaking changes to the schema, but migration to RDS requires a pg_dump from local and pg_restore to the RDS endpoint.

Operational Runbooks

  • Scaling: Vercel services scale automatically. EC2 (Gateway) requires manual t3.micro to t3.small upgrade if CPU exceeds 70%.
  • Monitoring: Use Vercel Logs for Serverless functions and AWS CloudWatch for EC2/RDS metrics.
