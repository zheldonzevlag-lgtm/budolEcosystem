System Admin Manual - Version 2.2.0

1. Environment Configuration

The security hardening features rely on several system settings stored in the SystemSetting table. These can be adjusted via the Admin UI or directly in the database.

Setting Key Description Default Value
SECURITY_RATE_LIMIT_AUTH Max login attempts allowed within the window. 5
SECURITY_AUTH_MAX_ATTEMPTS Attempts before account lockout. 5
SECURITY_SESSION_IDLE_TIMEOUT Minutes of inactivity before logout. 15

2. Database Maintenance

The RateLimit table should be periodically cleaned of expired entries to maintain performance. While the system handles per-key cleanup during checks, a weekly truncation of expired records is recommended.

Maintenance Task: Run DELETE FROM "RateLimit" WHERE "expiresAt" < NOW(); weekly.

3. Troubleshooting

Rate Limit Lockouts

If an administrator is accidentally locked out due to rate limiting, the lockout can be cleared by deleting the corresponding key from the RateLimit table:

DELETE FROM "RateLimit" WHERE "key" = 'auth_login_[IP_ADDRESS]';
