User Manual (Admin) - Version 2.2.0

1. Security Hardening Dashboard

A new Security Hardening section has been added to the System Settings menu. This allows authorized administrators to manage global security policies.

1.1 Authentication Policies

  • Max Login Attempts: Set the number of failed attempts allowed before a user is locked out.
  • Lockout Duration: Define how long an account remains locked (in minutes).

1.2 Session Management

  • Idle Timeout: Automatically log out users after a period of inactivity.
  • Absolute Timeout: Force re-authentication after a set duration, regardless of activity.

1.3 Rate Limiting

  • Auth Rate Limit: Control the maximum number of login attempts allowed per IP address to prevent brute-force attacks.

2. Compliance Indicators

The security settings page includes real-time compliance indicators to ensure your configuration meets PCI DSS and BSP standards.
