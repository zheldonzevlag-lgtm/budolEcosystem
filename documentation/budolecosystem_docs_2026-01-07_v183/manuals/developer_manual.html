Developer Manual v183

Purpose

To implement the foundational layer of the Vercel deployment plan, specifically standardizing environment variables and service configurations for production readiness.

Technical Details

1. Shared JWT_SECRET Standardization

To enable seamless Single Sign-On (SSO) across the ecosystem, all services now share a common JWT_SECRET in their .env.vercel files.

JWT_SECRET="GJ7Lxn0/kdV/KuZJ5xJ7Ip0RvMerrGW5n0gf44mfHgc="

Affected Services: budolpay, budolID, budolshap, budolAccounting.

2. Vercel Configuration (vercel.json)

Implemented vercel.json in services that previously lacked them to ensure correct routing on the Vercel platform.

  • budolID: Configured to use @vercel/node for the Express.js server.
  • budolAccounting: Configured for Express.js deployment.
  • budolpay-gateway: Configured for serverless proxying (Note: Socket.io functionality may require dedicated infrastructure like Upstash).

Dependencies

  • Vercel CLI for deployment.
  • Provisioned Vercel Postgres databases for each service.

Test Results

Verified that npm run db:vercel correctly applies the new standardized .env.vercel content to the local .env files for testing.
