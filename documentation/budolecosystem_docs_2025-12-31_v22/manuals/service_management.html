Service Management Guide

The budolEcosystem relies on a centralized Single Sign-On (SSO) service called budolID. If this service is not running, authentication across all applications will fail with a "503 Service Unavailable" or "Internal Server Error".

1. Starting budolID SSO Service

The SSO service must be running on port 8000 for the Admin portal and other apps to authenticate users.

LAN Access Notice: To access the ecosystem from other devices, use the host IP address:
  • budolID: http://192.168.1.24:8000
  • budolShap: http://192.168.1.24:3001
  • budolPay Admin: http://192.168.1.24:3000
cd "d:\IT Projects\budolEcosystem\budolID-0.1.0"
npm run start

Verification: Visit http://localhost:8000/login in your browser. You should see the budolID login form.

2. Default Admin Credentials

Important: The admin user for the budolpay ecosystem has been reset to the following credentials:
  • Email: admin@budolpay.com
  • Password: admin123

3. Troubleshooting Login Failures

  • "SSO Service is offline": Ensure budolID is running on port 8000.
  • "Invalid Credentials": If admin123 fails, run the password reset script: 
    node verify-fix.js
  • "Unauthorized Application": Ensure the apiKey in apps/admin/app/api/auth/login/route.ts matches the seeded API key in the budolID database (default: bp_key_2025).

4. Technical Details

  • Port: 8000
  • Database: PostgreSQL (via Prisma)
  • Auth Type: JWT (RS256/HS256)
