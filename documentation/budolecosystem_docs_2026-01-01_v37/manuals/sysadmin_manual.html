System Admin Manual: OTP & SMS Configuration

Version: 2026-01-01_v37

Configuration Variables

To enable live SMS and Email delivery, ensure the following environment variables are configured in your .env file:

Email Configuration (SMTP)

VariableDescription
SMTP_HOSTSMTP server address (default: smtp.gmail.com)
SMTP_PORTSMTP port (default: 587)
SMTP_USERSMTP username/email
SMTP_PASSSMTP password or App Password

SMS Configuration (Placeholder)

The system is prepared for SMS integration. Currently, it logs to the console if these are missing:

VariableDescription
SMS_API_KEYAPI Key from your SMS provider
SMS_SECRETAPI Secret from your SMS provider

Operational Verification

Admins can verify OTP delivery by monitoring the server logs. Successful dual-channel triggers will show:

[Register] Dual-channel OTP sent to user@example.com and +639XXXXXXXXX

Database Schema

No schema changes were required. The User.emailVerifyToken field is used to store both standard tokens and numeric OTPs.
