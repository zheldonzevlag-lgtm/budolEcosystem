v3.4.6: Multi-Store Checkout Robustness & Inventory Hardening

Date: 2026-02-20 Version: v3.4.6 Status: Released

Implementation Summary

This release implements critical fixes identified in the Shopee vs BudolShap Comparative Analysis (Phase 1 & Phase 2). It introduces a master Checkout record to group multi-store orders and enforces atomic inventory deduction to prevent overselling.

  • Inventory Hardening (Phase 1): Implemented atomic stock deduction within the order creation transaction. Added stock field to Product model and UI.
  • Checkout Robustness (Phase 2): Created a Checkout model to group orders. Linked Payment Intents to the master Checkout ID instead of individual Order IDs.
  • Payment Flow: Updated OrderSummary.jsx, payment APIs, and webhooks to process payments via the master checkoutId, ensuring all linked orders are updated simultaneously.
