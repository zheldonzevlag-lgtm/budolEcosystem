Risk Register

Risk: Inventory Deadlocks (Medium)

Description: High concurrency on a single product during flash sales might cause database transaction timeouts due to row locking.

Mitigation:

  • Keep transactions short.
  • Consider Optimistic Concurrency Control (OCC) for future high-scale phases.
  • Current implementation uses Pessimistic Locking (via `update`), which is safer but slower.

Risk: Stale Checkout Records (Low)

Description: Abandoned checkout attempts will leave `PENDING` records in the `Checkout` table.

Mitigation:

  • Implement a scheduled cron job to mark `PENDING` checkouts older than 24 hours as `EXPIRED` (or delete them).

Risk: Webhook Metadata Loss (Low)

Description: If the payment gateway drops metadata, we lose the link to the `Checkout` ID.

Mitigation:

  • Implemented fallback search: `findFirst({ where: { paymentId: paymentIntentId } })`.
  • This ensures that even without metadata, we can find the record if the payment ID was saved during initiation.
