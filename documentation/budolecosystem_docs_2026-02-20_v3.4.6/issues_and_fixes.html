Issues & Fixes

Critical: Inventory Overselling

FIXED

Issue: Previously, the system checked for stock availability (if stock > qty) but did not atomically deduct the stock during the order creation process. This created a race condition where multiple users could purchase the last item simultaneously.

Fix: Implemented prisma.$transaction with atomic updates: 

data: { stock: { decrement: item.quantity } }
This ensures database-level locking and consistency.

Partial Payment Status for Multi-Store Orders

FIXED

Issue: Multi-store orders resulted in multiple Order records but only one Payment Intent. Webhooks only updated the first order.

Fix (v3.4.6): Introduced a master Checkout record. The Payment Intent is now linked to the Checkout ID. Webhooks update the Checkout status, which cascades to all linked Orders.
