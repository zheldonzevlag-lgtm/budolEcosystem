Developer Manual

This document details the critical changes made in v3.4.6 to support multi-store payments via a master Checkout record and atomic inventory management.

Purpose

To align with industry standards (e.g., Shopee), v3.4.6 introduces a master Checkout entity that groups multiple store-specific orders under a single transaction. It also enforces atomic stock deduction to prevent overselling.

Technical Implementation

1. Database Schema Changes (Prisma)

Added a Checkout model and a stock field to the Product model.

model Checkout {
  id              String   @id @default(uuid())
  userId          String
  user            User     @relation(fields: [userId], references: [id])
  orders          Order[]
  total           Float
  status          String   @default("PENDING") // PENDING, PAID, FAILED
  paymentId       String?
  paymentProvider String?
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

model Product {
  // ... existing fields
  stock           Int      @default(0)
}

2. Atomic Inventory Deduction

The createOrder function in ordersService.js now performs atomic stock deduction within a transaction.

// lib/services/ordersService.js

await prisma.$transaction(async (tx) => {
    // ... loop through items
    
    // 1. Lock and Get Product
    const product = await tx.product.findUnique({ where: { id: item.productId } });
    
    // 2. Check Stock
    if (product.stock < item.quantity) throw new Error("Out of stock");
    
    // 3. Decrement Stock (Atomic)
    await tx.product.update({
        where: { id: item.productId },
        data: { stock: { decrement: item.quantity } }
    });
    
    // 4. Create Order linked to Checkout
});

3. Multi-Store Checkout Flow

The payment flow now uses checkoutId as the primary key for linking payments to orders.

  • Order Creation: Returns { orders: [...], checkoutId: "..." }.
  • Payment Initiation: Sends checkoutId to the payment gateway metadata.
  • Webhooks: Listen for checkoutId in metadata. If present, updates the master Checkout record to PAID and cascades the update to all linked Order records.
