Developer Manual v467

Technical Overview

The Flutter Web engine initialization can sometimes hang or delay beyond the splash screen's timeout, leading to a blank screen. This release implements a three-layer fix:

  1. Engine Configuration: Removed explicit renderer flags in index.html to allow the engine to auto-select the most stable renderer.
  2. Fallback Timer: Added a 15-second JS fallback in index.html to force remove the splash screen if flt-glass-pane is not detected.
  3. Dart Signaling: The SplashScreen widget now explicitly calls removeSplashFromWeb via JS Interop once it's ready to navigate.

Code Changes

1. Web Index Optimization

File: web/index.html

// Added fallback timer
setTimeout(function() {
  if (!document.querySelector('flt-glass-pane')) {
     console.warn('BudolPay: Flutter engine still initializing or failed. Forcing splash removal.');
     removeSplashFromWeb();
  }
}, 15000);

2. JS Interop Safety

File: js_helper_web.dart

// Safe check before calling JS methods
if (js.context.hasProperty(method)) {
  js.context.callMethod(method, args);
}

3. Splash Signaling

File: splash_screen.dart

// Signal web engine to remove splash
if (kIsWeb) {
  callJsMethod('removeSplashFromWeb');
}

Dependencies

  • dart:js for Web Interop.
  • flutter_bootstrap.js for standard engine initialization.
