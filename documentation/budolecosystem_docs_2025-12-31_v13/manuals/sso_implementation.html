SSO Implementation (budolID & budolShap)

This guide documents the JWT-based Single Sign-On (SSO) flow between the identity provider (budolID) and the marketplace (budolShap).

[budolShap] --(Redirect with API Key)--> [budolID Login]
[budolID Login] --(Redirect with JWT Token)--> [budolShap Callback]
[budolShap Callback] --(Verify Token)--> [budolID Auth API]
[budolShap Callback] --(Set Local Cookie)--> [User Logged In]

Step 1: Application Registration

Every ecosystem app must be registered in budolID to obtain an API Key.

// budolID/prisma/seed.js { name: 'budolShap', redirectUri: 'http://localhost:3002/api/auth/sso/callback' }

Step 2: The Login Handshake

Users on budolShap click "Login with budolID", which initiates the redirect.

// URL Construction const loginUrl = `\${process.env.NEXT_PUBLIC_SSO_URL}/login?apiKey=\${process.env.BUDOLID_API_KEY}`;

Step 3: Secure Callback & Sync

After successful login, budolID redirects back to the app's callback route with a JWT token. The app then verifies this token and syncs the user locally.

// budolShap/app/api/auth/sso/callback/route.js const verifyResponse = await fetch(`\${ssoUrl}/auth/verify`, { headers: { 'Authorization': `Bearer \${token}` } });

Security Features

  • Stateless Authentication: Uses JWTs for fast, serverless-friendly sessions.
  • Cross-App Identity: Users share the same email across all apps (Lego blocks).
  • API Key Validation: Only authorized apps can initiate SSO logins.
