Budol Ecosystem v498 Release Overview

Version: 498 | Date: 2026-01-22 | Status: RELEASED | Compliance: BSP, PCI DSS, NPC

Purpose

This release focuses on standardizing timezone handling across the entire Budol Ecosystem, specifically aligning all services and frontend applications with the Asia/Manila (UTC+8) standard using a centralized utility library. This eliminates hardcoded 8-hour offsets and inconsistent toLocaleString calls.

Rationale

Hardcoded offsets (e.g., new Date().getTime() + 8*60*60*1000) are prone to errors and difficult to maintain. By centralizing timezone logic in dateUtils.js, we ensure consistency across backend audit logs (BSP Circular 808), transaction timestamps (PCI DSS), and frontend displays, reducing technical debt and improving reliability.
