Developer Manual: Timezone Standardization

Technical Explanation

The ecosystem now uses a centralized dateUtils.js library to handle all date and time operations related to the Asia/Manila timezone. This approach ensures that:

  • Backend logic always uses true UTC (getNowUTC()) for storage.
  • Frontend components use formatManilaTime() for display.
  • Legacy +8h manual offsets are deprecated and being phased out.

Central Utility: dateUtils.js

Location: budolshap-0.1.0/lib/dateUtils.js

Major Changes

1. Backend Services

Modified multiple services to replace hardcoded offsets with getNowUTC().

  • transaction-service/index.js: Health check timestamps.
  • budolID/api/index.js: SSO and registration timestamps.
  • budolAccounting/index.js: Ledger entry and health check timestamps.

2. Admin Dashboard Compliance

Updated compliance telemetry to use UTC-based checks for PCI DSS and BSP Circular 808.

File: compliance/telemetry/route.ts

3. Shipping Logic

Updated shipping contract and fallback provider to use standardized UTC time for delivery estimations and timeline events.

  • shipping/shippingContract.js
