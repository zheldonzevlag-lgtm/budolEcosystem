Future Recommendations

1. Database Level Constraints

Consider enforcing UTC storage at the Prisma schema level or database triggers to ensure no legacy code accidentally inserts offset timestamps.

2. Global Timezone Support

While Asia/Manila is the current standard, the dateUtils.js library should be expanded to support multiple timezones for future international expansion.

3. Automated Compliance Audits

Integrate the timezone compliance test into the CI/CD pipeline to prevent regression in future updates.

4. Mobile App Synchronization

Ensure that the BudolPay mobile app (Flutter) is also updated to use the same UTC-based timestamp logic for its splash screen and internal event tracking.
