Mitigation Plan

M-498-01: Legacy Data Strategy

Risk: Double-conversion of legacy +8h timestamps.

Action: Implement a heuristic in formatManilaTime to detect timestamps that are likely already offset (e.g., timestamps in the future relative to true UTC). Provide a getLegacyManilaISO() utility for temporary compatibility during the transition period.

M-498-02: Integration Verification

Risk: External provider failures.

Action: Updated Lalamove and Fallback providers to explicitly use standardized UTC. Added unit tests to verify that outgoing payloads contain correctly formatted ISO strings.

M-498-03: Real-time Sync Integrity

Risk: Missing events due to bucket shifts.

Action: Updated RealtimeProvider.jsx deduplication logic to use Date.now() (UTC epoch) for bucket calculation, ensuring consistent behavior regardless of server local time.
