Known Issues

  • CSV Filenames: Some legacy CSV export functions in budolshap-0.1.0 might still use a manual 8-hour offset for the filename timestamp if not explicitly updated in this phase.
  • Real-time Polling: While standardizing, some components using useSWR might still see a "flicker" if the backend returns a timestamp slightly different from the client's expectation during the transition.
