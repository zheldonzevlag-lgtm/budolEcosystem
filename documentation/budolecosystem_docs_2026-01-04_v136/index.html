Network-Aware Local Environment

v136 - 2026-01-04

Automated network discovery and cross-device connectivity system for the budolEcosystem.

Purpose

The manual configuration of IP addresses for local development, especially when testing with physical mobile devices, was error-prone and tedious. This update introduces an automated network discovery system that detects the machine's local IP and propagates it throughout the ecosystem.

Technical Implementation

1. Network Discovery Utility

A new script scripts/network-util.js uses the Node.js os module to find the primary IPv4 address of the host machine. It automatically updates the root .env file and generates a MOBILE_CONFIG.txt helper.

2. Dynamic Injection in start-ecosystem.js

The central runner now calls the network utility at startup and injects the detected IP into all services via environment variables:

LOCAL_IP=172.16.0.2
API_GATEWAY_URL=http://172.16.0.2:8080
AUTH_SERVICE_URL=http://172.16.0.2:8001
...

3. Standardized Service Binding

All microservices and Next.js applications now bind to 0.0.0.0 instead of localhost, allowing them to accept connections from other devices on the same network.

System Impact

Mobile Connectivity

Physical devices can now connect to the local server seamlessly by using the --dart-define=API_HOST=[IP] flag, which is documented in the auto-generated MOBILE_CONFIG.txt.

Zero Configuration

Developers no longer need to manually update IP addresses in multiple .env files when moving between different Wi-Fi networks (Home, Office, Cafe).

Verification & Testing

Test Scenario: Run ecosystem on a new network interface.

  • Action: Run npm start in root.
  • Observation: network-util.js correctly detected 172.16.0.2.
  • Observation: Root .env updated automatically.
  • Observation: MOBILE_CONFIG.txt generated with correct instructions.
  • Result: PASSED

Future Enhancements

  • mDNS / Bonjour Support: Implement service discovery so mobile apps can find the server without typing an IP.
  • QR Code Discovery: Generate a QR code in the terminal at startup that the mobile app can scan to auto-configure its API host.
  • Network Change Listener: Automatically restart services or update configs if the Wi-Fi network changes while the apps are running.
