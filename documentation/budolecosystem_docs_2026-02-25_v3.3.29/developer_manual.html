Developer Manual

Purpose

Explain how product video tiles are integrated into the product detail gallery, how sellers preview uploaded videos, and how Cloudinary folders are created per user before upload.

Rationale

Surface product videos alongside images, allow sellers to confirm uploads before publishing, and keep Cloudinary assets organized by user.

Technical Explanation

The gallery composes a unified media list for images and videos, the seller upload grid renders video tiles with a play button overlay, and the upload API creates folders under /assets/media_library/folder/budolshap/assets/profile_images/<full name>/<full name>-<date>-<time> before uploading.

Changes

  • Created a combined gallery list for images and videos.
  • Added video thumbnail tiles with play overlays.
  • Enabled the main viewer to render HTML5 video with overlay.
  • Added play button overlays to seller video previews in upload flow.
  • Created Cloudinary folder paths per user and timestamp before upload.

Issues

No regressions detected in the gallery layout or selection logic.

Fixes

Unified media selection state for mixed image/video galleries.

Risks

Video thumbnails can increase initial media payload on slow connections.

Mitigations

Use metadata preload and lightweight overlays to keep UI responsive.

Dependencies

React state hooks, Next.js Image, HTML5 video, Cloudinary API, Jest, ESLint.
