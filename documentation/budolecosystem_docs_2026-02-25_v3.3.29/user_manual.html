User Manual

Purpose

Explain how shoppers view gallery videos, how sellers preview uploaded videos, and how uploads are stored in user folders.

Rationale

Video previews help buyers understand products and help sellers validate uploads before publishing with organized storage.

Technical Explanation

The gallery includes video thumbnails with a play overlay, the seller upload grid renders videos with a play button for preview, and uploads are placed in per-user Cloudinary folders.

Changes

  • Video tiles appear in the thumbnail strip next to images.
  • Clicking a video thumbnail switches the main viewer to video playback.
  • Seller uploads show a play button overlay on each video preview.
  • Play overlays indicate video content in thumbnails and main viewer.
  • Uploads are organized into user folders before storage.

Issues

Videos may take slightly longer to load on slower connections.

Fixes

Metadata preload ensures the gallery remains responsive.

Risks

Users on limited data plans may experience increased usage.

Mitigations

Video loads on selection and uses metadata preload by default.

Dependencies

Browser HTML5 video support and Cloudinary storage.

Code References
