System Admin Manual

Purpose

Describe operational considerations for gallery video tiles, seller preview playback, and Cloudinary folder creation.

Rationale

Ensure operations teams understand that UI changes and Cloudinary folder organization do not require schema changes.

Technical Explanation

Gallery rendering and seller preview playback are handled in the client, while the upload API creates user-scoped Cloudinary folders before upload.

Changes

  • No schema migrations or new services required.
  • No environment variables or secrets added.
  • Seller video previews include play button overlays.
  • Uploads now create Cloudinary folders per user and timestamp.
  • Lint and Jest configurations updated for test stability.

Issues

Console logs appear during some tests but do not impact runtime behavior.

Fixes

Resolved lint/test failures by aligning test fixtures and ESLint rules.

Risks

Higher media payload for products with video thumbnails.

Mitigations

Use metadata preload to minimize bandwidth and render time.

Dependencies

Next.js runtime, React, Cloudinary API, Jest, ESLint.
