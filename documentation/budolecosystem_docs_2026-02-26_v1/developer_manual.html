Developer Manual - Release 2026-02-26 v1

Module: Marketplace & Mobile Core Version: 1.3.70 Created by: Budol AI Orchestrator

Purpose & Rationale

Purpose: To stabilize the marketplace core components and finalize the mobile app build v1.3.70. This release specifically targets issues with variation pricing logic, cart synchronization failures, and Prisma Client build inconsistencies on Windows environments.

Rationale: Marketplace sellers were unable to save products with variation prices correctly, and buyers were seeing incorrect ₱0 or ₱1 prices. Additionally, the mobile app needed to be updated to v1.3.70 to reflect the latest fixes and improvements.

Technical Explanation

Variation Price Priority: The UI has been updated to prioritize prices from the variation_matrix over the top-level Product.price field when variations are enabled. This ensures that the starting price displayed to buyers is accurate (the minimum variation price).

Cart Sync Logic: The /api/cart PUT route now performs a pre-sync validation. It filters out any productId that no longer exists in the database to prevent foreign key constraint violations (Prisma error P2003) during the createMany operation.

Prisma Client: Standardized the Prisma client to use the default @prisma/client instead of a custom output path. This prevents file-locking issues in the node_modules folder during development on Windows.

Compliance & Security

  • PCI DSS: All pricing calculations are performed using standardized rounding and currency formatting to ensure transaction integrity.
  • Cybersecurity Law: API endpoints now include enhanced validation to prevent database injection via orphaned product IDs.
  • Data Protection: All mobile app builds are signed and versioned to maintain source integrity.
