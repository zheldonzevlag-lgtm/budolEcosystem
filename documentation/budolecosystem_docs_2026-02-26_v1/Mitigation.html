Mitigation Plan - 2026-02-26 v1

Mitigation for R-01: Orphaned Cart Items

Action: Implemented server-side validation in /api/cart PUT route. Before saving the cart, the system now queries the Product table to verify all IDs in the payload exist.

Result: Prevents 500 errors and foreign key constraint violations if a user has a product in their local storage that was deleted by a seller.

Mitigation for R-02: Variation Price Logic

Action: Refactored ProductDetails and ManageProducts logic to check for the existence of variation_matrix. If variations exist, the UI uses the minimum price from the matrix as the default starting price.

Result: Eliminates misleading ₱0 or ₱1 prices on product pages and dashboard tables.

Mitigation for R-03: Prisma Sync Issues

Action: Removed custom output paths in schema.prisma and reverted to default @prisma/client. Added dynamic property access in API routes for newly added fields (like hidden_combos).

Result: Stabilizes the build process on Windows environments and allows the API to function even if the local client is slightly out of sync.
