Developer Manual - Version v500

Purpose & Rationale

The primary goal of this update was to resolve delays in order status updates from "In Transit" to "Delivered". The investigation revealed that the Lalamove webhook handler was using hardcoded status mapping that didn't align with the centralized statusMapper.js and was missing critical metadata updates required for order lifecycle completion.

Technical Explanation

The solution involved refactoring the webhook entry point to utilize the universal status normalization logic and explicitly handling the transition to DELIVERED status by setting deliveredAt and autoCompleteAt timestamps.

Key Code Changes

1. Lalamove Webhook Refactoring

Modified app/api/webhooks/lalamove/route.js to standardize status mapping and metadata updates.

// Before: Hardcoded switch-case
// After: Unified normalization and metadata handling
const newStatus = normalizeStatus(status, 'lalamove', false);

if (newStatus === UNIVERSAL_STATUS.DELIVERED && !order.deliveredAt) {
    updateData.deliveredAt = getNowUTC();
    const autoCompleteDate = getNowUTC();
    autoCompleteDate.setDate(autoCompleteDate.getDate() + 3);
    updateData.autoCompleteAt = autoCompleteDate;
}

2. Status Normalization Logic

The system now relies on lib/shipping/statusMapper.js for all courier-to-system status translations, ensuring consistency between polling and webhook events.

Tests

A specialized test script was developed to verify the webhook logic without requiring external Lalamove hits: scripts/test_scripts/budolShap/test_webhook_logic.js.

Phase 8: Dependencies and Software Choice

Why these tools?

  • Prisma ORM: Chosen for its type-safe database access and ease of managing complex relations like Order-Shipping-Return. It ensures that metadata updates are consistent across the database.
  • Pusher/Realtime: Integrated to provide immediate feedback to users when a webhook event occurs. This eliminates the need for manual page refreshes to see status updates.
  • statusMapper Utility: A centralized mapping pattern was chosen over inline switch statements to ensure that all courier status changes (polling or webhook) are handled identically, reducing bug surface area.
  • Lalamove API: Selected as the primary logistics partner for its robust webhook support and real-time rider tracking capabilities in the Philippine market.

Compliance

  • PCI DSS 10.2.2: Enhanced audit logging of order status transitions.
  • BSP Circular 808: Improved real-time monitoring of financial transactions (orders).
