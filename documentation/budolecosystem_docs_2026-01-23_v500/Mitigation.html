Risk Mitigations - v500

M500-1: Webhook & Polling Hybrid

The system uses both webhooks (real-time) and API polling (safety net) to ensure statuses are updated even if a webhook is missed.

M500-2: Centralized Status Mapping

By using statusMapper.js, all unknown statuses are logged for review, and the system defaults to "undefined" instead of crashing, allowing manual intervention.

M500-3: Database Constraints

Metadata updates (timestamps) are protected by conditional checks to prevent overwriting existing data or double-processing.
