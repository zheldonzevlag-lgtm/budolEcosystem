Rationale & Technical Explanation

Purpose: To prevent unauthorized concurrent access and account takeovers by ensuring a unique relationship between a physical device and a user account.

Rationale: Following industry leaders like GCash, budolPay recognizes that device binding is the most effective deterrent against automated fraud and remote account theft. By tying the account to a specific hardware ID (UUID/IMEI), we create a "Physical Token" requirement for every session.

Technical Explanation: The system utilizes a combination of SecureStorage for device tokens and a backend trustedDevices repository. When a new device is detected, the auth-service blocks the login and triggers the "Unlink Flow", requiring a cryptographically signed face embedding match before re-binding the account.

Compliance Verification

  • BSP Circular 958: Compliant via Strong Customer Authentication (SCA) and Device Fingerprinting.
  • PCI DSS: Compliant via secure credential storage and encrypted biometric transmission.
  • NPC (Data Privacy): Compliant via minimized data collection and local biometric processing.
