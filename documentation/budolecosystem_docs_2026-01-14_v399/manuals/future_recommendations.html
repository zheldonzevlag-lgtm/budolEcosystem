Future Recommendations (Phase 8)

Strategic directions for enhancing the budolPay security ecosystem beyond v399.

1. Passkey Integration (FIDO2)

High Priority

Transition from device-binding based on hardware IDs to FIDO2 Passkeys. This would allow users to link devices using cryptographically secure public-key pairs, reducing reliance on OTP and centralized biometric templates.

2. Behavioral Biometrics

Medium Priority

Implement background analysis of typing patterns, touch pressure, and device motion. This can detect if a device is being used by someone other than the owner, even if it is unlocked.

3. Zero-Trust Architecture

Low Priority

Move towards a model where every transaction is verified not just by the device ID, but by a contextual risk score (Location, IP, Transaction Velocity, Device Health).

4. Dependency Rationale (Phase 8)

Software Choices:

  • TFLite (Mobile): Chosen for on-device face embedding generation to ensure user privacy and offline capability.
  • Redis: Used for managing short-lived unlink sessions and OTP throttling.
  • PostgreSQL (Prisma): Provides the relational integrity required for 1:1 user-device mapping.
