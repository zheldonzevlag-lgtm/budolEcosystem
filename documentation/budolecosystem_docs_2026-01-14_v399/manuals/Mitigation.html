Mitigation Strategy (v399)

This document outlines the mitigation strategies for the risks identified in the "1 Device → 1 Account" policy implementation.

Risk Mitigation Matrix

Risk ID Risk Description Mitigation Action Owner
R-399-001 User locked out due to lost/broken device. Implement Multi-Factor Authentication (MFA) via email and Face Verification (AI-driven) to allow remote unlinking. Security Team
R-399-002 Device ID Collision (Spoofing). Use hardware-backed security (Android Keystore/iOS Keychain) and complex device fingerprinting (IMEI + Serial + MAC). Dev Team
R-399-003 Service Disruption during Migration. Phased rollout; allow a 24-hour "grace period" for existing multi-device users to select a primary device. Ops Team

Implementation of Mitigations

1. AI Face Verification for Unlinking

To prevent unauthorized unlinking, budolPay uses a 512-dimensional face embedding comparison. The user must provide a live selfie which is matched against the registered biometric profile.

2. Hardware-Level Binding

Instead of relying on software-generated IDs, we utilize the SecureElement or TEE (Trusted Execution Environment) to store a unique binding token that cannot be exported to another device.

Compliance Alignment

  • BSP Circular 958: Ensures strong authentication for electronic banking.
  • NPC (Data Privacy Act): Biometric data is encrypted and processed according to privacy standards.
