Purpose

This update implements the full navigation flow for transaction history and detailed views in the budolPay mobile application.

Technical Details

1. TransactionDetailsScreen Implementation

A new screen TransactionDetailsScreen was created to display comprehensive information for a single transaction. Key features include:

  • Safe Parsing: Handles both numeric and string representations of transaction amounts from the backend.
  • Dynamic UI: Changes color and prefix based on whether the transaction is income (green, '+') or expense (red, '-').
  • Counterparty Logic: Displays "From" for received funds and "To" for sent funds by comparing user IDs.

2. TransactionHistoryScreen Implementation

A dedicated TransactionHistoryScreen provides a full list of user transactions:

  • Infinite List: Uses ListView.separated for efficient rendering.
  • Refresh Support: Integrated with ApiService to fetch the latest transaction data.
  • Navigation: Tapping any item pushes the TransactionDetailsScreen onto the navigation stack.

3. HomeScreen Integration

The HomeScreen was updated to link these new screens:

  • See All Button: Navigates to the full history screen.
  • Recent Items: Wrapped in InkWell for direct access to detail views.
  • Bottom Navigation: Updated index 1 to navigate to the history screen for better accessibility.

Code Snippets

Example of safe amount parsing used across screens:

double amount = 0.0;
if (tx['amount'] != null) {
  if (tx['amount'] is num) {
    amount = (tx['amount'] as num).toDouble();
  } else {
    amount = double.tryParse(tx['amount'].toString()) ?? 0.0;
  }
}

Test Results

Verified navigation from Home to History, Home to Details, and History to Details. All data fields populated correctly with valid mock and real transaction data.
