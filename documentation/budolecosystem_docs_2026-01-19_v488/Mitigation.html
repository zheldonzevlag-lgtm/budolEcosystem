M-488-01 (API Fallback)

Risk: QR Generation API failure.

Mitigation: Implement a secondary QR generation service or local generation library if primary API fails.

M-488-02 (Polling Optimization)

Risk: Webhook delay.

Mitigation: QRCodeModal implements a 3-second polling interval to check payment status directly from the database/service.

M-488-03 (UI Robustness)

Risk: JS errors in modal.

Mitigation: Added try-catch blocks in handleRepay and stopPropagation on buttons to ensure UI stability.
