Technical Explanation

The core improvement in v488 is the normalization of QR data between different providers (budolPay vs PayMongo) to allow a single component, QRCodeModal, to handle all payment displays.

QR Data Normalization

When an order is initiated, the backend now returns either a raw QR string (for budolPay) or a structured object (for PayMongo). The frontend normalizes these into a consistent object structure containing an imageUrl.

    // Priority 2: Fallback to Redirection

Compliance

BSP Circular 808: All payment intents are now audited with metadata linking to the specific order and customer ID. Cybersecurity Law: Payment data is transmitted over secure internal channels with HMAC verification.
