Future Recommendations - v352

Roadmap for System Enhancement

Index Future Recommendations

1. Complete Migration to WebSocket

While the SWR (polling) method is reliable, it consumes more resources. Migrating fully to RealtimeService (WebSockets) for all balance and transaction updates will improve performance and reduce backend load.

2. Address Deprecated API Warnings

The 12 info warnings in flutter analyze regarding withOpacity and dart:js should be addressed once the Flutter web ecosystem stabilizes further. Currently, withValues causes stalling issues on certain browsers, so the rollback to withOpacity is temporary.

3. Centralized Data Refresh Logic

Instead of manual _fetchData() calls after navigation, implement a reactive architecture where screens automatically refresh their state based on authentication and connectivity changes provided by a central store (e.g., Bloc or Riverpod).
