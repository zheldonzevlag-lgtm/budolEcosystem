Deployment Overview

This release is a stability patch focused on client-side state management. There are no backend schema changes or gateway configurations required.

Version Control

  • Current Version: 1.2.58
  • Previous Version: 1.2.57
  • Compatibility: Fully compatible with all existing backend services.

Monitoring & Logs

Admins should monitor for 401 Unauthorized errors in the backend wallet-service logs. A significant reduction in 401 errors during logout events is expected with this release.

Check for Session Errors in Logs

tail -f logs/wallet-service.log | grep "401"

Risk Mitigation

If users report that their balance is not updating after returning from a transaction, verify that the isAuthenticated state is correctly maintained in the client app. This can be checked in the debug logs of the Flutter application.
