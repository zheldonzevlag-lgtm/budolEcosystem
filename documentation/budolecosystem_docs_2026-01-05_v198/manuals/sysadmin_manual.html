v198 System Admin Manual

Subject: Infrastructure & Configuration for SMS API Management.

Overview

The SMS API Selection module allows system administrators to switch between different SMS gateways without changing environment variables or restarting the server. All configurations are stored in the database.

Database Configuration

The following keys are added/managed in the SystemSetting table:

Key Group Description
SMS_PROVIDER SMS Currently active provider (TWILIO, INFOBIP, SEMAPHORE, MOVEX)
TWILIO_ACCOUNT_SID SMS Account SID for Twilio API
TWILIO_AUTH_TOKEN SMS Authentication Token for Twilio API
TWILIO_PHONE_NUMBER SMS Sender phone number for Twilio
SEMAPHORE_API_KEY SMS API Key for Semaphore (Philippines)
INFOBIP_API_KEY SMS API Key for Infobip
INFOBIP_BASE_URL SMS Base URL for Infobip API instance

Operational Procedures

1. Switching SMS Providers

Navigate to System Settings > SMS API Selection in the Admin Panel. Select the desired provider and provide the necessary credentials. Click "Save SMS Settings" to apply changes immediately.

2. Verifying Configuration

After saving, the system will revalidate the cache. You can verify the current setting directly in the database using:

SELECT * FROM "SystemSetting" WHERE "group" = 'SMS';

Risks & Mitigation

  • Risk: Database unavailability prevents SMS sending.
  • Mitigation: Ensure database high availability. The system defaults to SEMAPHORE if no provider is found in settings.
  • Risk: Invalid credentials.
  • Mitigation: Admins should perform a test SMS send (to be implemented in v199) after updating credentials.

Test Results

Database migrations and seed data for SMS settings were verified. The UI correctly handles empty or missing values by displaying placeholders.

Recommendation for Future Enhancement

Implement a "Test SMS" button within the admin interface to validate credentials before saving.
