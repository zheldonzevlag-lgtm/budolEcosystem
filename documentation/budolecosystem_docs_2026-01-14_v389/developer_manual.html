Developer Manual - v389

1. Technical Explanation: Face Login UX Refinement

The core change in v389 is the introduction of a confirmation dialog within _handleFaceRecognitionLogin when a custom face template is missing. This prevents unexpected redirects to native Face ID and aligns with user expectations.

Key Logic Change

File: login_screen.dart

// v389 Refined Logic with Confirmation Dialog
Future<void> _handleFaceRecognitionLogin() async {
  // 1. Check for stored custom template first
  final storedTemplate = await biometricService.getStoredFaceTemplate();
  
  // 2. If no custom template, handle fallback with user choice
  if (storedTemplate == null) {
    final hasNativeFace = await biometricService.hasEnrolledFace();
    if (hasNativeFace) {
      if (mounted) {
        final useNative = await showDialog<bool>(
          context: context,
          builder: (context) => AlertDialog(
            title: const Text('Face Login'),
            content: const Text('Your custom face profile was not found. Would you like to use your device\'s native Face ID instead?'),
            actions: [
              TextButton(
                onPressed: () => Navigator.pop(context, false),
                child: const Text('Cancel'),
              ),
              ElevatedButton(
                onPressed: () => Navigator.pop(context, true),
                child: const Text('Use Native Face ID'),
              ),
            ],
          ),
        );
        
        if (useNative == true) {
          return _handleBiometricLogin();
        }
        return;
      }
    }
    _showError('No face profile found. Please re-enable face login in Settings.');
    return;
  }
}

3. Dependencies

  • google_mlkit_face_detection: Used for liveness and bounding box detection.
  • FaceEmbeddingService: Custom service for TFLite inference.
  • BiometricService: Handles secure storage of templates and native auth fallbacks.

3. Compliance & Security

  • NPC Data Privacy Act: Biometric templates are stored locally in Encrypted Shared Preferences (Android) / Keychain (iOS) via flutter_secure_storage. No biometric data is sent to the server.
  • BSP Circular 808: Implementation of strong authentication for mobile payments.
  • Liveness Detection: Uses ML Kit to check eye-opening and smile probabilities to prevent spoofing with photos.

4. Testing Procedures

Verification script: \budolEcosystem\scripts\test_scripts\verify_face_login_flow.ps1

  • Test Case 1: Custom face registered -> Tap Face icon -> Camera should open.
  • Test Case 2: No custom face, native Face ID enrolled -> Tap Face icon -> Native biometric prompt should show.
  • Test Case 3: No face at all -> Tap Face icon -> Error "No face profile found" should show.
