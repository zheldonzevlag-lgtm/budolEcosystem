Release v389 Latest

Custom Face Login Prioritization & UX Refinement

Purpose & Rationale

Version 389 addresses user experience gaps in the face login flow. Tapping the face icon now triggers a confirmation dialog if a custom face profile is missing, allowing users to choose between native Face ID or cancellation. This prevents unexpected redirects and ensures version consistency across the ecosystem.

Technical Highlights

  • UX Refinement: Added confirmation dialog in login_screen.dart for missing face templates.
  • On-Device Recognition: Prioritizes 512-dimension embeddings via FaceNet TFLite model.
  • Native Fallback: User-approved redirection to device-native Face ID.
  • Version Sync: App version synchronized to 1.2.86+86 in pubspec.yaml and api_service.dart.
