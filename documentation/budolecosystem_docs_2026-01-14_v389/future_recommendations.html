Future Recommendations - v389

1. Hybrid Biometric Enrollment

Currently, native and custom face enrollments are separate. Future versions should offer a "Unified Enrollment" where the app registers the face with the OS *and* creates a local template simultaneously for maximum compatibility.

2. Advanced Liveness (3D/Depth)

Explore using ARCore/ARKit for depth-based liveness detection to further mitigate high-resolution photo spoofing risks.

3. Auto-Redirection Settings

Allow users to set a "Default Biometric Method" in settings (e.g., Always use Fingerprint, Always use Face, or Ask Every Time).
