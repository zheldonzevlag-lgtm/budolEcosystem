Developer Manual (v5.3.2)

Technical Architecture

The system utilizes a centralized icon mapping strategy located in resolveIconByName within CategoryIcons.jsx. The administrative portal components were updated to leverage this centralized object instead of local legacy arrays.

Core Changes

  • File Path: app/admin/categories/page.jsx
  • Change: Replaced PROFESSIONAL_ICONS reference with Object.entries(PROFESSIONAL_ICON_MAP).
  • Optimization: Connected the iconSearch state to both the grid and list icon pickers to enable real-time filtering among 50+ professional icons.

Code Reference

// Correct resolution of icon map to iterable entries
{Object.entries(PROFESSIONAL_ICON_MAP)
    .filter(([name]) => name.toLowerCase().includes(iconSearch.toLowerCase()))
    .map(([name, Icon]) => (
        <button key={name} ...>

}
