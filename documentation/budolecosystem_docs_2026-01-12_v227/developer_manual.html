Developer Manual - v1.2.60

Release Date: 2026-01-12

Biometric Icon & Layout Fix

Purpose: To resolve issues with biometric icons (fingerprint and face) being misplaced outside the PIN input textbox and ensure face recognition icons are displayed when supported by the device.

Changes

  • Added getAvailableTypes() to BiometricService to detect all enrolled biometric methods.
  • Modified LoginScreen to fetch available biometric types on initialization.
  • Updated _inputDecoration to support suffixIcon within the TextField.
  • Moved biometric icons from a separate Row into the TextField's suffixIcon for better layout compliance.
  • Implemented _buildBiometricIcons() to dynamically display all available biometric icons (Face, Fingerprint, Iris).

Code References

1. Biometric Service Enhancement:
Added support for detecting specific biometric types.
biometric_service.dart 


Future<List<BiometricType>> getAvailableTypes() async {
  try {
    return await _auth.getAvailableBiometrics();
  } catch (e) {
    if (kDebugMode) print('BiometricService: Error getting available types: $e');
    return [];
  }
}

2. Login Screen UI Fix:
Integrating biometric icons into the PIN input field.
login_screen.dart 


Widget _buildBiometricIcons() {
  List<IconData> icons = [];
  if (_availableBiometricTypes.contains(BiometricType.face)) {
    icons.add(Icons.face);
  }
  if (_availableBiometricTypes.contains(BiometricType.fingerprint)) {
    icons.add(Icons.fingerprint);
  }
  // ... iris and fallback ...
  return Row(
    mainAxisSize: MainAxisSize.min,
    children: icons.map((icon) => Icon(icon, size: 24)).toList(),
  );
}
