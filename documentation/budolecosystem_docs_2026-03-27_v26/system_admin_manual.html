Security Configuration

The `MathCaptcha` component is entirely client-side and requires no backend configuration. It primarily serves as a rate-limiting and anti-automation barrier at the UI layer.

Anti-Spam Maintenance

Admins should monitor login failure logs in the Identity Service. A high volume of failures before captcha completion may indicate a coordinated bot attempt. The `Shield Challenge` is currently hardcoded for a 1-digit sum/difference, which is sufficient for current threat levels.
