Budol Ecosystem v26 - Developer Manual

1. Budol Shield (Unified MathCaptcha)

We implemented a math-based bot protection mechanism that gates all authentication entry points. This prevents bots from attempting brute-force logins or automated registrations.

Components

  • MathCaptcha.jsx: Located at budolshap-0.1.0/components/auth/MathCaptcha.jsx.
    • Generates simple arithmetic problems (e.g., 5 + 8).
    • Uses a verifyAnswer() function instead of a formal HTML <form> to avoid nested form issues.
    • Triggers onSolve() callback upon successful verification.

Implementation in AuthForm

The AuthForm.jsx component now utilizes a gatekeeper pattern:


{!isCaptchaSolved ? (
    <MathCaptcha 
        onSolve={() => setIsCaptchaSolved(true)} 
        primaryColor={isLogin ? 'blue' : 'rose'} 
    />
) : (
    <formFields />
)}

2. Nested Form Resolution

Issue: Browsers do not allow nesting <form> elements. Since AuthForm is a form, rendering another form inside it for the captcha caused validation errors.

Fix: Refactored MathCaptcha.jsx to use a <div> wrapper and manual event handlers (Enter key and Button Click) to trigger verification.

3. Production Sync

The NEXT_PUBLIC_SSO_URL in Vercel should be kept as https://budol-id-sso.onrender.com to ensure all environments point to the centralized identity service.
