Issues & Fixes

Resolved Issues

  • Issue: MapPicker was tightly coupled to Leaflet/OSM.
  • Fix: Refactored MapPicker to support conditional rendering of map providers. Introduced GoogleMapView component.
  • Issue: MapPicker always defaulted to OSM even when Google Maps was selected in settings.
  • Fix: Corrected useMapSettings hook to properly parse the API response (removed incorrect optional chaining on data.settings).
  • Issue: Admin Settings saved Google Maps provider as 'GOOGLE' but components expected 'GOOGLE_MAPS'.
  • Fix: Updated app/admin/settings/maps/page.jsx to use 'GOOGLE_MAPS' ID, aligning with MapPicker and API logic.
  • Issue: Existing database records with 'GOOGLE' map provider caused UI mismatch (showing "Enable Provider" instead of active state) and fallback to OSM.
  • Fix: Added backward compatibility normalization in lib/settings.js to automatically map 'GOOGLE' to 'GOOGLE_MAPS' when reading settings.

Pending Issues

  • Google Maps integration relies on the places library which adds to the bundle size. (Optimization pending)
  • Markers on Google Maps are currently default; custom pin styling (Shopee-style) is handled by overlay, which works but might have slight parallax during drag on slower devices.
