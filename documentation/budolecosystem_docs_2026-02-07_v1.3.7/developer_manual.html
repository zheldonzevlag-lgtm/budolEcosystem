Developer Manual: KYC Standards Alignment

This document details the technical implementation of changes in v1.3.7, focusing on aligning the BudolShap KYC process with industry standards (Shopee model).

1. Database Schema Updates (Tiered KYC)

To align with the Shopee model, the User and Store models have been updated to support differentiated verification types (Buyer vs. Individual Seller vs. Business Seller).

2. Shopee-Style Buyer KYC Flow

Following the ShopeePay model, the KYCWizard now supports a "Standard Buyer" verification flow. This focuses on unlocking financial capabilities within the Budol Ecosystem.

  • Standard Buyer: Unlocks ₱100k budolPay limit, bank withdrawals, and P2P transfers. Requires Valid ID + Liveness Check.
  • Biometric Verification: Integrated a real-time biometric liveness check using the navigator.mediaDevices.getUserMedia API.
  • Security Requirement: Access to the camera requires a Secure Context (HTTPS). The application will block biometric capture on insecure connections to protect user data.
  • Photo Preview: Upon successful liveness verification, the captured selfie is displayed to the user within a circular preview, overlaid with a verification badge to confirm the session's authenticity.
  • Real OCR Engine: Integrated Tesseract.js for client-side Optical Character Recognition.
  • OCR Accuracy Hardening (Phase 2):
    • Line-by-Line Interactive Mapping: Added a UI section in the verification step that displays all raw OCR-extracted lines, allowing users to manually map specific lines to Name, ID, or Business fields via interactive buttons.
    • Sequence-Based Extraction: Implemented layout-aware extraction that follows the visual sequence of data on common PH IDs (e.g., Header -> ID Number -> Name for Voter IDs).
    • Image Pre-processing: Automated grayscale conversion, ultra-high contrast boosting (2.2x), and 1200px resolution normalization.
    • Deep Name Refinement: Implemented heuristic cleanup for trailing OCR artifacts (1-2 character noise) and common PH name misread corrections (e.g., 0/1 to O/I mapping).
    • PH ID Optimization: Advanced regex patterns for Voter ID (XXXX-XXXXX-XXXXXXXXX), Driver's Licenses, UMID, and Passport.
    • Fuzzy Label Matching: Implemented label detection for "VOTER ID", "LICENSE NO", and "PASSPORT" to find ID numbers even when layout varies.
    • Noise Filtering: Expanded blacklist including government departments (DOTr, LTO, BIR) to prevent false-positive name extraction.
  • Document Storage & Compliance:
    • Persistence: Uploaded documents and biometric captures are now saved as base64 in the kycDetails field for audit and manual review.
    • AMLA/BSP Compliance: Retention policy set to 5 years (calculated via retentionExpiry in the backend).
    • NPC Compliance: Documents are stored solely for verification purposes with encrypted transmission (HTTPS).
  • Dynamic Input Binding: Extracted data is automatically bound to the verification form, allowing users to review and correct any misread characters before submission.
  • Secure Capture: Frames are processed locally via a hidden canvas before being certified by the AI liveness engine.

2. KYC Data Contract

The kycDocuments JSON field in the Store model is now expected to follow a structured format based on the sellerType:

Individual Seller

{ "id_type": "Passport", "id_number": "P1234567A", "id_front_url": "...", "selfie_url": "..." }

Business Seller

{ "business_type": "Corporation", "dti_sec_url": "...", "bir_2303_url": "...", "representative_id_url": "...", "representative_selfie_url": "..." }

3. Registration Flow Persistence (OTP Visibility)

To fix the issue where the OTP input was hidden immediately after phone registration (v1.3.6), we introduced justRegisteredRef in the AuthForm component. This ref prevents the auth state from being reset to "login" when the modal closes or re-renders during the registration-to-OTP transition.

// AuthForm.jsx snippet const justRegisteredRef = useRef(false); const handleRegisterSuccess = () => { justRegisteredRef.current = true; setMode('otp'); // Force transition to OTP };

4. UI Optimization: Navbar Identifier Filtering

To improve user clarity, internal system identifiers (@quick.budolpay.com) are now filtered out of the Navbar dropdown, showing only user-friendly names or phone numbers.

5. Universal Shipping Status Mapping

To ensure courier-agnostic shipping status tracking, we updated the statusMapper.js logic to handle both forward and return logistics with universal labels (e.g., "To Pick Up" for returns in PROCESSING state). Corresponding integration tests in integration.test.js were updated to validate these mappings.

6. Real-Time OCR Integration (KYC)

To automate data extraction and reduce manual entry errors during the KYC process, we integrated google_mlkit_text_recognition. This allows the system to scan ID cards and business documents in real-time, extracting key fields like ID number, registration date, and legal name for automatic validation against user-provided data.

// Conceptual OCR implementation import { TextRecognition } from 'google_mlkit_text_recognition'; async function processKYCDocument(imageUri) { const result = await TextRecognition.recognize(imageUri); const extractedData = parseExtractedText(result.text); return extractedData; }

8. Dependencies and Software Rationale

The following software and libraries were chosen for the KYC implementation to ensure compliance, performance, and developer productivity:

  • Next.js (App Router): Chosen for its server-side rendering (SSR) capabilities which are essential for SEO and initial load performance, as well as its robust API routing for backend logic.
  • Prisma ORM: Provides type-safe database access, ensuring that KYC data contracts are strictly enforced at the database level.
  • Tailwind CSS: Allows for rapid UI development with a consistent design system, ensuring that compliance shields and status indicators are visually clear across all devices.
  • Lucide React: Used for lightweight, consistent iconography (e.g., Shield, CheckCircle, AlertTriangle) to enhance UX during the verification flow.
  • Jest: Selected as the primary testing framework to validate complex state transitions and audit logging logic, ensuring system reliability before deployment.
  • Google ML Kit (OCR): Chosen for its on-device processing capabilities, ensuring that sensitive ID information is extracted locally whenever possible to enhance user privacy and reduce server load.
