System Admin Manual: Version 1.3.7

System Administrators should be aware of the following changes in v1.3.7 related to Seller KYC and Tiered Architecture.

1. Seller Tier Management

The system now differentiates between Individual and Business sellers. This is managed via the sellerType field in the Store database. Admins must verify documents based on the tier chosen by the seller.

2. KYC Verification Workflow

Admins should monitor the verificationStatus of stores. Stores in PENDING status are restricted from public product activation and payout requests.

3. OTP Flow Continuity (v1.3.6)

Users who register via phone are now automatically redirected to the OTP verification screen. Admin support should ensure that OTP services (SMS/Email) are operational to prevent login blockers.

4. OCR-Assisted Verification

To streamline the review process, the system now uses google_mlkit_text_recognition to automatically extract data from uploaded IDs and business documents. Admins will see the "System Extracted" data side-by-side with the user-provided information in the admin dashboard, highlighting any discrepancies for faster auditing.
