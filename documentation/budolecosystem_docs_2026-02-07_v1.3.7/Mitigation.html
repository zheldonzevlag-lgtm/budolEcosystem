Mitigation Plan: v1.3.7

M-016: KYC Document Validation

Risk: Sellers submitting invalid or blurry documents.

Mitigation: Implementation of client-side file validation (size/type) and integration of real-time OCR using google_mlkit_text_recognition to verify data against user input during submission.

M-017: Multi-Channel OTP

Risk: SMS delivery failure.

Mitigation: Provide fallback options for verification code delivery via email or authorized messenger apps (Viber/WhatsApp) to ensure continuity in the registration flow.

M-018: TIN/Business Registration Verification

Risk: Submission of fake tax identifiers.

Mitigation: Cross-referencing submitted TIN and Business Registration numbers with BIR/DTI databases during the manual admin approval phase. Future API integration with government portals is planned.

M-019: Operational Feature Compliance (Gap Resolution)

Risk: Users bypassing KYC by using internal features like refunds or P2P transfers without withdrawing.

Mitigation: Implementing "Operational KYC Triggers" where the system enforces verification as a prerequisite for activating high-risk financial features, resolving the identified compliance gap between BudolShap and industry standards.

M-020: Proactive KYC UI Visibility

Risk: User friction and abandonment due to intrusive or unexpected KYC requirements during checkout or financial operations.

Mitigation: Integrate KYC status visibility and submission links directly into the Profile/Navbar menu. This proactive approach ensures users are aware of their verification status and can complete KYC at their convenience before it becomes an operational blocker.
