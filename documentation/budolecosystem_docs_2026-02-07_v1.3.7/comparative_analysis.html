KYC Comparative Analysis: BudolShap vs. Shopee

This document provides a detailed comparison between the current BudolShap KYC implementation and the industry-standard Shopee KYC model, followed by a roadmap for alignment.

1. Buyer KYC Comparison

Aspect BudolShap (Current) Shopee (Reference)
Registration Email/Phone + OTP. No KYC for browsing. Mobile/Email/Social. No KYC for browsing.
Triggers P2P Send, Cash Out, Wallet Limits. ShopeePay Activation, Refunds, SPayLater, Withdrawals.
Submission Manual ID + Selfie + Face Match. OCR-enhanced ID Capture + Selfie + Auto-fill.
Limits ₱5k (Basic) vs. ₱500k (Verified). Verified vs. Partially Verified (Limited Withdrawals).
Key Gap: Shopee uses KYC triggers for operational features like "Refunds to Wallet," which ensures compliance even before the user actively tries to withdraw funds.

2. Seller KYC Comparison

Aspect BudolShap (Current) Shopee (Reference)
Mandatory? Yes (Pending store status). Yes (Mandatory to list and withdraw).
Entity Types Unified (Store model). Individual vs. Business / Company.
Requirements ID + Selfie (Basic). DTI/SEC/BIR + Bank Matching + Authorized Rep ID.
Enforcement Manual Review. Real-time OCR + Fraud Scoring + Bank API validation.
Key Gap: BudolShap lacks a formal "Business Seller" track requiring BIR and DTI documents, which is essential for legal compliance (E-Commerce Law & Tax).

3. Implementation Roadmap for BudolShap (Gap Resolution)

Phase 1: Architecture & Data Contracts Planned

Primary Goal: Resolve the "Business Seller" gap by establishing legal entity data structures.

  • Introduce SellerType (Individual/Business) in the database schema to support legal entity differentiation.
  • Define JSON schema for kycDocuments to support mandatory BIR 2303, DTI, and SEC uploads for business tracks.
  • Update VerificationStatus to include PARTIALLY_VERIFIED for phased onboarding.

Phase 2: Seller KYC Flow Optimization Planned

Primary Goal: Automate compliance checks to match industry-standard enforcement.

  • Implement multi-step Seller Registration wizard with tier-specific requirements.
  • Compliance-First UI: Integrate KYC status visibility and submission links directly into the Profile/Navbar menu.
  • Integrate real-time OCR using google_mlkit_text_recognition to resolve manual review delays and extraction errors.
  • Mandatory bank account linking with name-matching logic to prevent fraud and ensure BSP compliance.

Phase 3: Compliance & Monitoring Pending

Primary Goal: Resolve the "Operational KYC Triggers" gap and ensure NPC/AML alignment.

  • Feature-Based Triggers: Implement mandatory KYC for operational features (e.g., Wallet Refunds, P2P Transfers) to ensure compliance before withdrawal attempts.
  • Real-time AML watchlist screening against global sanction lists.
  • Audit logging for all KYC status transitions to satisfy National Privacy Commission (NPC) requirements.
