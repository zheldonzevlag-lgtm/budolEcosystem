Architectural Decision: Standalone budolAccounting

Following the Umbrella Architecture, we have decoupled the financial record-keeping into a standalone service: budolAccounting.

(Pay, Shap, etc.)
The "Bank-as-a-Service" Strategy: budolAccounting provides a unified ledger for the entire ecosystem. It doesn't care about "Products" or "Wallets"—it only cares about "Debits" and "Credits" in balanced entries.

Core Features:

  • Double-Entry Ledger: Implements professional accounting standards for all ecosystem apps.
  • Atomic API: /ledger/entry ensures that transactions are balanced and immutable.
  • Ecosystem Transparency: Any app (budolPay, budolShap, budoLoan) can report financial events to the central ledger.

Technical Details:

Located at /budolAccounting-0.1.0, this service uses its own isolated database and Prisma schema focused purely on the ChartOfAccount and LedgerEntry models.
