Future Recommendations – v25

Next Steps (v26 Candidates)

1. Deploy budolID SSO to a Public URL HIGH PRIORITY

Currently, budolID runs only on localhost/LAN. In production (Vercel), the phone availability check gracefully degrades to local DB only. A public budolID deployment would enable full cross-service verification and OTP flows.

2. OTP Verification for Quick Registration HIGH PRIORITY

After Quick Registration, send a one-time SMS/OTP to the registered phone number to verify ownership. This is required for BSP compliance and prevents fraudulent registrations.

3. Jest E2E Tests for Registration Flows MEDIUM PRIORITY

Automate TC-001 through TC-007 as Jest integration tests stored in /scripts/test_scripts/. Include mocking for budolID responses. This prevents future regressions like BUG-001 from reaching production silently.

4. PM2 Process Manager for Local Dev MEDIUM PRIORITY

Replace the manual concurrently start script with PM2 to avoid port conflicts, get auto-restart on crash, and retrieve structured logs per service without terminal hunting.

5. KYC / Profile Completion Prompt MEDIUM PRIORITY

After Quick Registration login, display a non-blocking modal prompting users to complete their profile (full name, email, KYC document upload). This aligns with BSP KYC requirements for e-wallet services.

6. Upgrade Vercel CLI LOW PRIORITY

Current CLI is v50.1.3; latest is v50.37.1. Run yarn global add vercel@latest or npm i -g vercel@latest to stay current and avoid deprecation warnings.
