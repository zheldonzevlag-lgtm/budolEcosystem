Implementation Plan: CAPTCHA Shield Challenge

Integrated Registration Gatekeeper for Budol Ecosystem

Phase: GO-LIVE

Version: 25.0.0

Core Rationale

To mitigate automated registration attacks (bots) without introducing third-party API dependencies or user friction (like complex alphanumeric CAPTCHAs), we are implementing a "Shield Challenge"—a randomized, single-digit addition/subtraction problem that acts as a mandatory pre-form gatekeeper.

Proposed Changes by Component

budolID SSO Central Gateway Registration

File: index.js

  • • Injected captcha-container HTML template before the registration form.
  • • Applied hidden class to the main registration form by default.
  • • Implemented randomized math generation (0-9) via vanilla JS.
  • • Added smooth transition effects for form revelation.

budolShap Storefront Authentication

Component: MathCaptcha.jsx

Modular React component providing a self-contained security challenge.

Container: LoginModal.jsx

  • • Integrated conditional rendering logic based on isCaptchaSolved.
  • • Added state management to reset challenges when switching modes.

Risk Mitigation & Compliance

PCI DSS & NPC Safeguards

By blocking access to the registration form until verified, we reduce the exposure of PII fields (Email, Phone) to automated scrapers. No sensitive data is transmitted during the challenge phase.

User Experience (UX)

Single-digit arithmetic provides minimal cognitive load for legitimate users while being highly effective against simple bot scripts.
