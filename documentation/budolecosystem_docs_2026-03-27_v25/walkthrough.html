Shield Challenge Walkthrough

Phased Implementation of Math-Based CAPTCHA Gatekeeper

Status: Verified

Released: Mar 27, 2026

Objective & Outcome

To prevent automated account creation (bot registrations), we have implemented a "Shield Challenge" gatekeeper across the entire Budol Ecosystem. This gatekeeper intercepts the registration flow, requiring users to solve a simple math problem before accessing PII input fields.

  • "Zero-cost protection without external API dependencies."
  • "Consistent security across SSO, Storefront, and Admin apps."

Verification Evidence (budolID SSO)

1. Initial Security Gatekeeper

The registration form is completely hidden. Only the "Shield Challenge" is visible.

2. Failure Handling

Incorrect answers trigger immediate feedback and block access.

3. Success Revelation

Correct answers reveal the full BudolID registration form.

Technical Implementation Summary

Frontend (budolShap)

  • • New MathCaptcha.jsx reusable component
  • • Integrated into LoginModal.jsx (Next.js)
  • • State-driven gatekeeper: isCaptchaSolved

SSO (budolID)

  • • Injected logic in Express /register route
  • • DOM Manipulation to swap challenge/form containers
  • • Random single-digit generator (Add/Sub)
