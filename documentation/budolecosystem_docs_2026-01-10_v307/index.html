Network Awareness & Stability Fixes v307

Date: 2026-01-10 | Scope: Ecosystem-wide Network Update & Bug Fixes

Purpose & Explanation

The primary objective of this update was to restore full functionality to the budolEcosystem following a transition to a new local network (192.168.1.17). This involved a systematic update of all hardcoded IP references, environment variables, and cross-service communication adapters to ensure the mobile app, web apps, and microservices can communicate seamlessly.

Issues & Fixes

1. Admin API 500 Error

Issue: /api/auth/admin/check failed with a 500 error because Prisma was queried with an undefined user ID when no session was present.

Fix: Added a null check in route.js to return a 401 Unauthorized response instead of crashing.

FIXED

2. Payment Integration "NotFound"

Issue: budolShap could not initiate payments via budolPay because the adapter was hitting the wrong port/path on the API Gateway.

Fix: Updated budolpay-adapter.js to correctly handle port 8080 (Gateway) with the /payments prefix and port 8004 (Direct).

FIXED

3. Mobile App Connectivity

Issue: The mobile app was hardcoded to 192.168.1.24, preventing login and transaction scanning.

Fix: Updated API_HOST to 192.168.1.17, incremented version to 1.2.41, and generated a new release APK.

FIXED

4. Test Script Fragmentation

Issue: 100+ test scripts were scattered across sub-directories, making it difficult to maintain and run ecosystem-wide tests.

Fix: Centralized all scripts into scripts/test_scripts with sub-folders for each service.

FIXED

Technical Details

Network Detection Logic

Implemented a dynamic IP detection utility in network-util.js that scans all network interfaces and identifies the primary LAN IPv4 address.

const interfaces = os.networkInterfaces();
const localIP = Object.values(interfaces)
    .flat()
    .find(iface => iface.family === 'IPv4' && !iface.internal)?.address;

Payment Adapter Logic Update

// lib/payment/adapters/budolpay-adapter.js
this.gatewayUrl = process.env.PAYMENT_GATEWAY_URL || process.env.GATEWAY_URL;
if (this.gatewayUrl.includes(':8080') && !this.gatewayUrl.includes('/payments')) {
    this.gatewayUrl += '/payments';
}

Risk & Mitigation

Risk: Subsequent network changes will break connectivity again if IP addresses change (DHCP lease expiry).

Mitigation: Use the npm run sync-network command regularly. The mobile app now supports mDNS discovery (budolpay.local) to avoid IP-dependency in the future.
