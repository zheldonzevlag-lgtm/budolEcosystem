SysAdmin Manual: Operations & Auditing

Database Audit Procedures

With the transition to the JON-TIMESTAMP-RANDOM format, sysadmins should update their auditing queries to support wildcard searches.

Querying Transactions by Reference Prefix

-- Search for all JON- prefixed transactions SELECT * FROM "Transaction" WHERE "referenceId" LIKE 'JON-%' ORDER BY "createdAt" DESC;

Extracting Timestamp from Reference

Since the first part of the ID after 'JON-' is still a Unix timestamp, you can convert it to a readable date in PostgreSQL:

-- Example for PostgreSQL (v12+) SELECT "referenceId", to_timestamp(split_part("referenceId", '-', 2)::bigint / 1000.0) as generated_at FROM "Transaction" WHERE "referenceId" LIKE 'JON-%';

Monitoring for Collisions

While statistically improbable, we recommend monitoring for UniqueConstraint violations on the referenceId column in the application logs.

# Example grep command for logs grep "Unique constraint failed on the fields: (`referenceId`)" /var/log/budolpay-service.log
