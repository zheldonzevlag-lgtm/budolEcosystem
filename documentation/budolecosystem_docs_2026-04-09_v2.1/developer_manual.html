Developer Manual: v1.1.0 (Stabilization)

Overview

This release addresses critical path stabilization identified during Phase 0 testing. Key focus areas include health monitoring for SSO services and refining the interaction between BudolPay Admin and BudolID.

Code Changes Reference

Below are the specific files modified in this release and their purpose:

Service File & Link Rationale
BudolID index.js Added /api/health endpoint for ecosystem monitoring.
BudolPay Admin auth/callback/route.ts Fixed callback handling for SSO redirects.
BudolPay Admin lib/notifications.ts Stabilized notification dispatching logic.
BudolShap shipping/quote/route.js Corrected API routing for logistics quotes.

SSO Monitoring Implementation

To comply with Phase 10 health checks, a new route was added to the budolID service. This allows the master health checker to verify service availability.

app.get('/api/health', (req, res) => {
    res.status(200).json({ status: 'UP', service: 'budolID', timestamp: new Date().toISOString() });
});

BudolPay Admin Logic Updates

Modified the OTPVerificationModal.tsx to handle state transitions more gracefully, ensuring NPC-compliant PII masking during OTP entry.
