🛡️ Mitigation Strategies (v19.0.0)

Issue: Prisma Schema Desynchronization

Impact: Critical failure of Admin UI when saving settings.

Mitigation 1 (Software): Implemented a Raw SQL Fallback bridge in lib/settings.js. This bypasses the Prisma Client validation layer for specific columns, allowing the system to work even if the client binary is outdated.

Mitigation 2 (Infrastructure): Updated the central schema.prisma to include all production columns and triggered a full rebuild (v19.0.2) to synchronize the Prisma client binary with the live RDS instance.

Mitigation 3 (Identity): Added environment variable fallbacks (SMTP_USER, SMTP_PASS) to ECS Task Definitions as a secondary source of truth if the database remains unconfigured.
