⚙️ System Administration Manual - v19.0.0

Backend Infrastructure Update

This release includes Revision 11 (Task Definition) which introduces environment variable overrides for critical SMTP settings (SMTP_USER, SMTP_PASS, etc.). These values are now used as fallbacks if the database is unpopulated.

Database Integrity

The system now uses a hybrid approach for the SystemSettings table. While standard fields are managed via Prisma, new columns added in production migrations (up to v19) are handled via a dynamic SQL proxy layer. This ensures no service downtime when the database schema evolves ahead of the code binary.

ECS Monitoring

Admins should monitor CloudWatch Log Group /ecs/budol-shap-task for [Email Fallback] markers to confirm OTP generation even if the SMTP provider is down.
