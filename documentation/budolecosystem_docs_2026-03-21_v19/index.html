Purpose & Rationale

This release fixes a critical System Settings Storage Crash where the Admin UI would fail to save email and SMS settings due to a hard-coded Prisma client limitation in the production image. The fix introduces a dynamic Raw SQL Fallback layer to handle system settings without requiring a full Docker image rebuild for ogni schema change.

Refactored settings.js logic

The system now splits settings into "Known Fields" (saved via Prisma for performance) and "Raw Fields" (saved via $executeRawUnsafe to avoid schema validation crashes).

// Partial Code Reference (lib/settings.js)
const RAW_ONLY_FIELDS = [
    'emailProvider', 'smtpHost', 'smtpPort', 'smtpUser', 'smtpPass',
    'smtpFrom', 'brevoApiKey', 'gmassApiKey', ...
];

for (const [col, val] of Object.entries(rawData)) {
    // RAW SQL BYPASS to avoid Prisma validation
    await prisma.$executeRawUnsafe(
        `UPDATE "SystemSettings" SET "${col}" = $1 WHERE "id" = 'default'`, 
        val
    );
}
