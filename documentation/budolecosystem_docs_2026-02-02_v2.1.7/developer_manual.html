Developer Manual: Dual-Channel OTP & SSO Fix

1. Dual-Channel OTP Delivery

The /auth/forgot-password endpoint in index.js has been updated to simulate delivery to both SMS and Email channels.

// Dual-channel delivery simulation
console.log(`[EMAIL] To: ${email} | Body: Your OTP is ${otp}`);
console.log(`[SMS] To: ${user.phoneNumber} | Body: Your OTP is ${otp}`);

2. SSO API Key Fallback

To prevent the "Unauthorized Application" error when users access /login directly or via SSPR without an apiKey, a default fallback to bp_key_2025 (budolPay) has been implemented.

// Fallback logic in /login and /auth/sso/login-form
const activeApiKey = apiKey || 'bp_key_2025';
const ecosystemApp = await prisma.ecosystemApp.findUnique({ where: { apiKey: activeApiKey } });

3. SSPR Context Persistence

The /forgot-password and /reset-password routes now accept and propagate the apiKey parameter to ensure the "Back to Login" link returns the user to their originating application context.
