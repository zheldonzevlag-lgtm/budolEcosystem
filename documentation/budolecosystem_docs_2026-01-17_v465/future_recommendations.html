Future Enhancements & Recommendations (v465)

Strategic roadmap for further hardening BudolPay security.

Security Roadmap

1. Device Binding / Fingerprinting (HIGH PRIORITY)

Recommendation: Bind the user session to the specific hardware ID of the device.

Rationale: Prevents session hijacking even if a token is somehow cloned to another device.

2. Dynamic Jailbreak/Root Detection

Recommendation: Implement real-time detection for compromised operating systems.

Rationale: BSP and banking standards require apps to refuse operation on rooted devices to prevent advanced memory scraping.

3. Certificate Pinning

Recommendation: Hardcode the server's SSL certificate within the app.

Rationale: Mitigates Man-in-the-Middle (MitM) attacks by ensuring the app only talks to the official BudolPay backend.

4. Adaptive Session Timeouts

Recommendation: Dynamically adjust timeout duration based on transaction risk (e.g., shorter timeout if viewing large balances).
