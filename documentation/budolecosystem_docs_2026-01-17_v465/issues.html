Security Vulnerabilities & Issues (v465)

Pre-implementation analysis and identified gaps in session management.

Identified Security Gaps

1. Insecure Token Storage CRITICAL

Description: Authentication tokens were stored using SharedPreferences, which stores data in plain text on the device filesystem. This is vulnerable to extraction on rooted/jailbroken devices.

Impact: Unauthorized account takeover if the device is compromised.

2. Lack of Inactivity Logout HIGH

Description: The app maintained an active session indefinitely as long as the token was valid, even if the user was inactive.

Impact: If a user leaves their phone unlocked, any person can access their financial data and perform transactions.

3. Vulnerable App Resume HIGH

Description: Switching back to the app from the background did not require re-authentication, exposing the last viewed screen immediately.

Impact: Sensitive balance or transaction history could be seen by unauthorized users during multitasking.

4. Screen Capture Exposure MEDIUM

Description: No protection against screenshots or screen recording was present in the mobile application.

Impact: Malicious apps or unauthorized users could capture sensitive financial information via screen recording.

5. Android Build Compatibility (V1 vs V2 Embedding) HIGH

Description: The flutter_windowmanager plugin, used for screen security, relies on deprecated Android V1 embedding. This caused build failures on modern Android configurations used by BudolPay.

Impact: Prevented the generation of release APKs for distribution.

6. Inactivity Timer Synchronization MEDIUM

Description: The inactivity timer logic did not correctly detect all user interactions across the entire widget tree, leading to premature logouts even while the user was interacting with specific UI components.

Impact: Poor user experience due to unexpected session termination during active use.
