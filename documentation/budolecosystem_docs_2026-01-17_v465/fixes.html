Security Implementations & Fixes (v465)

Technical solutions deployed to resolve identified session management vulnerabilities.

Implemented Security Fixes

1. Hardware-Backed Secure Storage RESOLVED

Fix: Replaced SharedPreferences with FlutterSecureStorage in ApiService.

Technical Detail: Uses AES encryption for Android (KeyStore) and Keychain for iOS. This ensures tokens are never stored in plain text.

Code: final FlutterSecureStorage _secureStorage = const FlutterSecureStorage();

2. 5-Minute Idle Timeout RESOLVED

Fix: Implemented a SessionService with an inactivity timer and global interaction detection.

Technical Detail: Every user interaction (tap, scroll) detected via HitTestBehavior.translucent in SessionOverlay resets a 5-minute timer. This ensures active users are never logged out unexpectedly.

Code: _inactivityTimer = Timer(const Duration(minutes: 5), _handleTimeout);

3. Background Grace Period & Biometric Lock RESOLVED

Fix: Monitored app lifecycle states (resumed/paused).

Technical Detail: If the app is in the background for more than 30 seconds, it blurs the screen and requires biometric (FaceID/Fingerprint) or PIN to resume. This aligns with Philippine banking standards.

4. Anti-Screen Capture (flag_secure) RESOLVED

Fix: Migrated from flutter_windowmanager to flag_secure for modern Android V2 embedding support.

Technical Detail: Enabled FLAG_SECURE via FlagSecure.set(), which prevents screenshots and blacks out the app preview in the multitasking switcher. This migration resolved critical build failures.
