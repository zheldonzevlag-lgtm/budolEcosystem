v465 Mitigation Report

1. Encrypted Storage (PCI DSS Requirement 3)

Risk: Unauthorized access to JWT tokens from device backups or physical access.

Mitigation: Implementation of FlutterSecureStorage. On Android, this uses the KeyStore system with AES-256 encryption. On iOS, it uses the Keychain. This ensures tokens are never stored in plaintext.

2. Session Locking (BSP Circular 808)

Risk: Session exposure during app switching or backgrounding.

Mitigation: SessionService monitors AppLifecycleState. Any transition to background for >30 seconds triggers an isLocked state, requiring biometric verification to resume.

3. Anti-Screen Capture (NPC Compliance)

Risk: Sensitive financial data capture via malicious screen recording apps.

Mitigation: Programmatic addition of FLAG_SECURE to the window layout. This is enforced globally via the SessionService during initialization.

4. Inactivity Cleanup

Risk: Long-lived sessions on shared or unattended devices.

Mitigation: Global Timer in SessionService, reset by a Listener in SessionOverlay that wraps the entire app. 5 minutes of no touch interaction triggers apiService.logout().
