v465 Secure Session Implementation

Aligning BudolPay with Banking Standards (GoTyme, BPI, GCash)

Purpose & Rationale

To enforce robust security measures for mobile session management, mitigating risks of unauthorized access, session hijacking, and sensitive data leakage.

  • Idle Timeout: Prevents session exposure if the device is left unattended.
  • Background Lock: Ensures re-authentication when resuming the app from the background.
  • Secure Storage: Uses hardware-backed encryption for tokens (PCI DSS & BSP Compliance).
  • Screen Security: Prevents screenshots and screen recording of sensitive financial data.

Technical Architecture

Introduced a SessionService that orchestrates the security lifecycle of the application.

// lib/services/session_service.dart
// - Idle Timeout: 5 minutes
// - Background Grace Period: 30 seconds
// - Screenshot Protection: FLAG_SECURE
Key Files:
  • session_service.dart - Core security logic.
  • session_overlay.dart - UI Blur & Biometric Prompt.
  • api_service.dart - Migrated to FlutterSecureStorage.

Compliance Checklist

  • ✔ PCI DSS: Encrypted token storage and session timeouts.
  • ✔ BSP: Biometric re-authentication on app resume.
  • ✔ Cybersecurity Law: Prevention of screen scraping and unauthorized data capture.
