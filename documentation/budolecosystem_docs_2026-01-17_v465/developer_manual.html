Developer Manual - v465 Secure Session Management

1. Session Persistence Migration

The ApiService has been upgraded from SharedPreferences to FlutterSecureStorage for sensitive credentials. This ensures that tokens are encrypted at rest using the device's secure enclave (KeyChain on iOS, KeyStore on Android).

// api_service.dart migration final FlutterSecureStorage _secureStorage = const FlutterSecureStorage( aOptions: AndroidOptions(encryptedSharedPreferences: true), ); Future<void> _saveSession() async { if (token != null) await _secureStorage.write(key: _tokenKey, value: token!); if (user != null) await _secureStorage.write(key: _userKey, value: json.encode(user)); }

Reference: api_service.dart

2. Session Lifecycle Orchestration

A new SessionService manages the application's security state. It observes the WidgetsBinding lifecycle and handles transitions between resumed and paused states.

Idle Timeout Logic

A 5-minute inactivity timer is reset on every touch interaction via the SessionOverlay's Listener. If the timer expires, the user is logged out automatically.

Background Locking

If the app is backgrounded for more than 30 seconds (grace period), the session is locked. Upon return, the SessionOverlay displays a blurred UI and requests biometric authentication.

Reference: session_service.dart

3. Screen Security

To prevent sensitive information from being leaked via screenshots or screen recordings (as required by PCI DSS), we have enabled FLAG_SECURE on Android.

Future<void> _enableScreenSecurity() async { await FlutterWindowManager.addFlags(FlutterWindowManager.FLAG_SECURE); }

4. Global Integration

The SessionOverlay is injected into the MaterialApp.builder, ensuring that the security layer sits above all navigation routes and captures all user interactions.

Reference: main.dart

5. Dependencies & Software Choice Rationale

The following libraries were selected to meet banking-grade security standards:

  • flutter_secure_storage (v9.0.0): Rationale: Standard SharedPreferences stores data in XML files accessible to root users. SecureStorage uses Android KeyStore and iOS KeyChain, providing hardware-level encryption (AES/RSA) compliant with PCI DSS Requirement 3.
  • local_auth (v2.1.6): Rationale: Essential for biometric re-authentication (FaceID/Fingerprint). It provides a native experience for session resumption, aligning with BSP Circular 808 requirements for multi-factor authentication.
  • flag_secure (v2.0.2): Rationale: Modern replacement for flutter_windowmanager that supports Android V2 embedding. It interfaces with native Android window flags to enforce FLAG_SECURE, blocking screenshots and screen recording.
  • provider (v6.1.1): Rationale: Used for reactive state management. It allows the SessionService to notify the UI globally when a lock state changes without complex callback chains.
