System Administrator Manual

Version: v465 | Project: Budol Ecosystem

Overview

v465 introduces banking-grade session security. System administrators must ensure that environment variables and backend services are configured to support these new security features.

1. Environment Configuration

The mobile app now relies on FlutterSecureStorage. For Android devices, this requires EncryptedSharedPreferences which is handled automatically by the app, but ensures compatibility with API Level 23 (Android 6.0) and above.

# Ensure the following ports are accessible for auth verification AUTH_SERVICE_PORT=8001 GATEWAY_PORT=8080

2. Session Management Policy

Administrators should be aware of the following hardcoded security policies in v465:

  • Idle Timeout: 5 Minutes (Automatic Logout)
  • Background Grace Period: 30 Seconds (Lock Screen Trigger)
  • Biometric Enforcement: Required for unlocking without full logout.
Security Note: These values are aligned with PCI DSS and BSP regulations for mobile financial applications.

3. Troubleshooting

If users report "Session Locked" immediately after opening the app:

  1. Check if the device clock is synchronized. Significant drift can affect session validity.
  2. Ensure the auth-service is responding to /verify requests.
  3. If biometrics fail, the user can use their device PIN/Pattern as a fallback, or choose "Log Out" to re-authenticate with credentials.

4. Compliance Monitoring

Ensure that backend logs capture SECURITY_SESSION_LOCK and SECURITY_SESSION_UNLOCK events for auditing purposes. These are triggered by the mobile app during lifecycle transitions.
