SysAdmin Manual: KYC Compliance & Auditing

1. Regulatory Context (BSP & AMLA)

BudolPay's tiered KYC system is designed to meet Bangko Sentral ng Pilipinas (BSP) Circular 950 and the Anti-Money Laundering Act (AMLA) requirements for Electronic Money Issuers (EMI).

Critical: All "Fully Verified" accounts must have a valid government-issued ID and a real-time liveness photo stored for a minimum of 5 years after account closure.

2. Audit Trail Requirements

Every change in kycLevel or kycStatus must be logged in the AuditLog table with the following metadata:

  • reviewerId: The ID of the admin who approved/rejected.
  • previousStatus & newStatus.
  • reason: For rejections (e.g., "Blurry ID").
  • timestamp: Server-side generated.

3. Database Migration Steps

Deploying the new tiered levels requires a schema update:

-- Add KYCLevel enum and field
CREATE TYPE "KYCLevel" AS ENUM ('BASIC', 'FULLY_VERIFIED', 'PLUS');
ALTER TABLE "User" ADD COLUMN "kycLevel" "KYCLevel" DEFAULT 'BASIC';

-- Initialize levels based on existing status
UPDATE "User" SET "kycLevel" = 'FULLY_VERIFIED' WHERE "kycStatus" = 'VERIFIED';

4. Risk Management

  • Risk: Unauthorized access to sensitive ID documents. Mitigation: Ensure the Admin panel uses role-based access control (RBAC) and documents are served via signed URLs or secure proxy.
  • Risk: Fraudulent re-submission. Mitigation: Implement a cooldown period after a rejection and log all historical attempts.
