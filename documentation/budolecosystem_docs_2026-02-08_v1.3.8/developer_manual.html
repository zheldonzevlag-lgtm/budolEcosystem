Developer Manual: KYC Wizard Refactor (3-Step Flow)

This manual covers the technical implementation of the 3-step KYC flow and the OCR transparency panel in KYCWizard.jsx.

1. Architecture Overview

The KYC flow has been structured into 3 distinct steps. The internal state step now ranges from 1 to 3.

  • Step 1: Tier Selection (Individual, Business, Buyer).
  • Step 2: Document Upload, Biometric Liveness, and AI Extraction Preview.
  • Step 3: Final Review and Submission.

2. Step 3: Review & Submit Implementation

Step 3 provides a summary of all data collected in Step 2. It is implemented in the renderStep3 function, which displays the selected tier, extracted data, and document previews before final submission.

2. OCR Data Contract

The onComplete callback now returns an object containing raw OCR lines for administrative reference.


{
  tier: 'INDIVIDUAL',
  details: {

4. UI/UX Enhancements

To improve user confidence and data accuracy, several UI enhancements were implemented:

  • Larger Document Previews: The preview container in Step 2 was increased from 140px to 280px in height, with the image preview using h-64 (256px). This allows users to verify that their ID is clear and readable before processing.
  • Enhanced Review Step: In Step 3, document icons were increased from w-12 to w-20 for better visibility during final review.

5. Testing

Unit tests are located in KYCWizard.test.jsx. Use npm test components/KYCWizard.test.jsx to run the tests.
