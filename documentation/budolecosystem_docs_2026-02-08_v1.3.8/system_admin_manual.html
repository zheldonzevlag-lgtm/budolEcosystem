System Admin Manual: KYC Review Process

As of v1.3.8, all KYC verification and data correction responsibilities have been shifted to administrative staff. This ensures higher data quality and reduces user friction.

1. Reviewing Submissions

When a user submits their KYC documents, the admin dashboard will display:

  • Uploaded Documents: High-resolution photos of ID cards and business permits.
  • Biometric Photo: The captured selfie from the liveness check.
  • AI Extracted Data: The name and ID number detected by the AI.
  • Raw OCR Lines: Every line of text detected by the AI (used for manual cross-referencing).
Admin Responsibility: You must verify that the AI-extracted data matches the text on the uploaded ID photo. If there is a discrepancy, you must manually correct the fields in the admin interface before approving the verification.

2. Handling OCR Discrepancies

OCR is not 100% accurate. Common issues include:

  • Character Substitution: '0' for 'O', '1' for 'I', etc.
  • Noise: Random artifacts from the background being interpreted as text.
  • Misalignment: Text being read from the wrong section of the ID.

Always use the Raw OCR Lines as a secondary reference to understand how the AI interpreted the document.
