Developer Manual - Session & Payment Fixes (v57)

Purpose: To resolve unintended user logouts during the checkout/payment flow and ensure consistent cart clearance across all payment methods.

1. Session Persistence Fixes

Modified AuthContext.jsx to implement a "soft-fail" mechanism during session synchronization. Previously, if the server returned a 401 Unauthorized during a transient state (like a redirect), the system would immediately clear the local session.

Change: The system now checks if the user is in a payment flow (determined by URL or referrer). If so, it preserves the localStorage session and attempts to re-sync cookies instead of logging out.

2. Cart Clearance Optimization

In OrderSummary.jsx, the cart clearance logic was fragmented and inconsistent across different payment methods (COD vs QR vs BudolPay).

Change:

  • Cart is now cleared immediately upon successful order creation for all payment methods.
  • Session state is explicitly synced using setAuth before any external redirects (e.g., to GCash or BudolPay) to prevent session loss on return.
  • Server-side cart clearance is now handled asynchronously with error catching to prevent blocking the UI.

3. Impact Analysis

  • AuthContext: Reduced risk of session loss during redirects.
  • OrderSummary: More reliable cart state; items are removed from cart as soon as they are "converted" into an order.
