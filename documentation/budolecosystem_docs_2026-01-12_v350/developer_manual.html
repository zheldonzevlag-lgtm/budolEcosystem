Developer Manual - v350

This version addresses critical compatibility issues that prevented the application from proceeding past the splash or login screens on certain platforms.

  • Updated pubspec.yaml and api_service.dart to version 1.2.56 to reflect release v350.
  • Implemented multi-layered splash screen removal for Web.

Technical Changes

1. Color API Restoration

Reverted withValues(alpha: ...) to withOpacity(...) across the codebase to support legacy Flutter versions and ensure consistent rendering.

Reference: login_screen.dart

// Example Change in login_screen.dart
fillColor: Colors.white.withOpacity(0.1), // Reverted from withValues

2. Web Splash Removal Interop

Implemented dart:js interop to manually trigger splash screen removal on web platforms. This resolves the white screen issue where Flutter fails to automatically dismiss the native splash.

Reference: splash_screen.dart

if (kIsWeb) {
  try {
    js.context.callMethod('removeSplashFromWeb');
  } catch (e) {
    debugPrint('Failed to call removeSplashFromWeb: $e');
  }
}

3. Web Fallback Mechanism

Added a 10-second fallback in index.html to ensure the splash screen is removed even if the Flutter engine encounters a critical error during startup.

Reference: index.html

Risk Mitigation

Issue: Application stalling on web ("not proceeding").

Fix: Manual JS splash removal + syntax corrections in initState.

Compliance: Verified that these UI changes do not impact the security of PIN/OTP entry.
