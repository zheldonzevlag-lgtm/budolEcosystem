System Admin Manual: Mobile Auth Auditing & Device Traceability

1. Monitoring Mobile Authentication

Administrators can now track mobile-specific authentication events through the Forensic Audit Trail. These events are critical for identifying unauthorized device access or brute-force attempts on mobile PINs.

New Audit Actions (v407):
  • MOBILE_LOGIN_PIN: Recorded when a user successfully logs in via 6-digit PIN.
  • MOBILE_LOGIN_BIOMETRIC: Recorded for successful Face Recognition or Fingerprint logins.

2. Investigating Device Activity

Each mobile login entry in the audit log includes a deviceId. This ID is unique to the user's handset and should be used during forensic investigations to confirm if a transaction originated from a "Trusted Device".

Field Description Forensic Use Case
deviceId Unique hardware-linked identifier Confirming "1 Device → 1 Account" policy compliance.
ipAddress Network origin of the login request Identifying geo-location anomalies or VPN usage.
method PIN vs. BIOMETRIC Analyzing which auth method is being targeted in credential stuffing attacks.

3. Compliance Reporting

When generating reports for BSP or PCI DSS auditors, ensure you include the MOBILE_LOGIN_* actions. These provide evidence that the institution is monitoring all entry points into the payment ecosystem, including native mobile applications.

Pro-Tip: Use the "Security" filter in the Admin Dashboard Audit View to quickly isolate these authentication events for daily review.
