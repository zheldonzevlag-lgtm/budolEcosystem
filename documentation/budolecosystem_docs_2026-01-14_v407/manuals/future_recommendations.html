Future Recommendations (v407)

Building on the mobile authentication audit foundation, the following enhancements are recommended for future phases:

1. Mobile Device Attestation HIGH PRIORITY

Integrate Google Play Integrity API (Android) and App Attest (iOS) to cryptographically verify that the login request is coming from a legitimate, un-tampered version of the BudolPay app running on a secure device. This data should be included in the AuditLog metadata.

2. "Impossible Travel" Detection

Implement a backend worker that analyzes MOBILE_LOGIN audit logs to detect logins from geographically distant locations in a time period that makes travel impossible (e.g., login from Manila followed by login from New York 1 hour later). This should trigger an automatic account lock and CRITICAL audit alert.

3. Failed Biometric Attempt Thresholds

Enhance the audit system to track *failed* biometric attempts per device. If a device exceeds 5 failed attempts within 10 minutes, the AuditLog should record a SUSPICIOUS_BIOMETRIC_ACTIVITY event and temporarily disable biometric login for that device.

4. User-Facing Security Log

Expose a read-only "Login History" view within the mobile app itself, allowing users to see their own AuditLog entries for MOBILE_LOGIN actions. This increases user trust and allows them to report any unrecognized device activity early.

5. Device Fingerprint Enrichment

Enrich the deviceId in the audit logs with additional non-sensitive device metadata (e.g., Device Model, OS Version, App Version) to provide a more comprehensive forensic picture of the authentication context.
