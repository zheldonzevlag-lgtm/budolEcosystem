2. Technical Explanation

We have enhanced the KYCVerificationScreen to include active liveness detection. This prevents "presentation attacks" (e.g., holding up a photo) by requiring the user to perform specific actions.

Changes

  • State Management: Added _hasBlinked, _hasSmiled, and _livenessPrompt to kyc_verification_screen.dart.
  • ML Kit Integration: Updated _processImageML to extract leftEyeOpenProbability and smilingProbability from the Face object.
  • Validation Logic: Selfies are now only accepted if both a blink and a smile have been detected during the session.
  • UI Feedback: A dynamic status badge prompts the user for the next required action.

Code Reference

// Liveness Logic in kyc_verification_screen.dart
if (face.leftEyeOpenProbability! < 0.2 || face.rightEyeOpenProbability! < 0.2) {
    blinkDetected = true;
}
if (face.smilingProbability! > 0.7) {
    smileDetected = true;
}

3. Rationale & Compliance

Active liveness detection is a requirement for BSP (Bangko Sentral ng Pilipinas) and NPC (National Privacy Commission) compliance regarding remote identity verification. It ensures that the person being verified is physically present.

4. Future Recommendations

  • Head Rotation: Add detection for left/right head movement for higher security.
  • Brightness Boost: Automatically increase screen brightness if low light is detected.
