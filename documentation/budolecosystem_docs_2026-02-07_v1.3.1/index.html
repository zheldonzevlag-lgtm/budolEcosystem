Lazada Technical Auth Flow Analysis

1. Identifier Processing Logic

Lazada employs a high-conversion "One-Field" or "Smart-Toggle" approach. When a user inputs data, the system performs real-time classification:

  • Format Validation: Regex checks to distinguish Email from Mobile.
  • Account Discovery: If the mobile number exists but the user is on the "Sign Up" page, Lazada triggers a "Reclaim Account" or "Login Instead" prompt.
  • Regional Handling: For PH, Lazada automatically handles the +63 / 09 prefix normalization.

2. Security Verification Hierarchy

To balance UX and security (PCI DSS / BSP Compliance), Lazada uses tiered verification:

  1. Silent Verification: Device fingerprinting and IP reputation checks.
  2. Active Verification (Challenge): Slide-to-verify (Alibaba Security Technology) or standard CAPTCHA.
  3. Possession Factor: SMS/Email OTP (Multi-channel fallback).

3. Mapping to Budol Ecosystem

Current implementation in AuthForm.jsx already mirrors several Lazada patterns:
