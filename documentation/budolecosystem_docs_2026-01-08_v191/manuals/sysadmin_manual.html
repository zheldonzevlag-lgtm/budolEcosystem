SysAdmin Manual - Service Management & Configuration

Environment Configuration

The budolEcosystem uses a centralized .env file in the root directory. Key variables include:

Variable Purpose Recommended Value (Local)
NODE_ENV Environment mode production (for local-to-cloud testing)
VERCEL Vercel compatibility flag 1
LOCAL_IP Automatically detected IP Auto-generated by start-ecosystem.js

Health Check Verification

After starting the ecosystem, verify all services are healthy using curl or a browser:

Service Health Check Endpoint
Core Services GET /health
Accounting Service GET /health (Note: Fixed from /api/health)

Database Schema & Migrations

No schema changes were introduced in v191. Ensure PRISMA_DATABASE_URL is correctly set in .env for persistence.

Operational Runbook

Starting the Ecosystem

To start all services concurrently with automatic IP detection:

npm start

Select option 3 (SKIP) when prompted if you want to keep existing database configurations.

Emergency Shutdown

If ports are blocked (EADDRINUSE), run the following command in PowerShell to clear node processes:

taskkill /F /IM node.exe
