Product Upload Fix & Error Handling

Documentation Version: v276 | Status: Implemented
Developer Manual SysAdmin Manual User Manual ← Previous (v275) Next (v277) →

Executive Summary

This update addresses the "Failed to fetch" error encountered during product image uploads in the BudolShap platform. The fix involved enhancing both client-side and server-side error handling, increasing request timeouts, and explicitly configuring payload limits.

  • Issue: Intermittent "Failed to fetch" errors during image upload (TypeError).
  • Root Cause: Likely payload size limits and network timeouts for large base64-encoded images.
  • Solution: Enhanced /api/upload route with detailed logging, increased timeout, and improved client-side diagnostics.

Key Changes

  1. Server-Side: Added maxDuration = 60 and detailed logging to /api/upload/route.js.
  2. Client-Side: Refactored onSubmitHandler in add-product/page.jsx for robust error capture.
  3. Configuration: Created next.config.js with 10MB body size limits for server actions and API routes.
