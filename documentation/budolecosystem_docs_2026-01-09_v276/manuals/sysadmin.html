System Admin Manual Updates - v276

Infrastructure Changes

Next.js Configuration

A new next.config.js file has been added to the budolshap-0.1.0 project to explicitly handle larger request bodies.

  • Server Actions Limit: 10MB
  • API Route Body Parser: 10MB

Environment Variables

Ensure the following Cloudinary variables are correctly set in the production environment (Vercel/Local):

  • NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME
  • CLOUDINARY_API_KEY
  • CLOUDINARY_API_SECRET

Operational Runbook

Monitoring Upload Failures

Check the server logs for "Error in upload route" or "Cloudinary upload successful". If 413 (Payload Too Large) persists, verify that the WAF or ingress controller (if any) is not overriding the 10MB limit.

Verification Steps

  1. Log in as a Store Owner.
  2. Navigate to "Add Product".
  3. Upload a high-resolution image (>5MB).
  4. Verify the upload succeeds and the image URL is returned from Cloudinary.
