Developer Manual v5.3.10

Last Updated: 2026-02-28 | Focus: Unified Identity Sync & Accurate Pricing

Introduction

This release focuses on two critical areas of the Budol Ecosystem: Identity Unification and Pricing Integrity. Previously, users registered in the common identity pool (budolID) were not always correctly located by downstream services like budolPay mobile app if they had not "bridged" their accounts yet. We have implemented a "Universal Identity Sync" mechanism that discovers and syncs users from the pool into the specific service databases as they attempt to login.

1. budolID: Auth Enhanced Search

We've improved the /auth/check-phone endpoint to return rich detail, including first name, last name, and ecosystem-wide User ID. This enables services like budolPay to auto-sync user profiles with matching IDs.

📁 budolID-0.1.0/index.js // [budolID] check-phone logic enhanced to return identity metadata res.json({ exists: !!user, id: user ? user.id : null, normalizedPhone: normalizedPhone, foundAs: user ? user.phoneNumber : null, email: user ? user.email : null, name: user ? user.name : null, firstName: user ? user.firstName : null, lastName: user ? user.lastName : null, schema: foundSchema, message: user ? `Phone number registered in ${foundSchema}` : 'Phone number is available' });

Rationale

Previously, downstream services only received a Boolean "exists". Adding identity metadata allows for "Sync-on-Demand", which reduces friction for existing ecosystem users.

2. budolPay: Auth proxying & Sync

The auth-service in budolPay now acts as a bridge. If a user is not found in the local database, it queries the budolID service. If found there, it creates a local shadowed record with the SAME id.

📁 budolpay-0.1.0/services/auth-service/index.js // [budolPay Auth Sync Logic] if (!user) { // Cross-check with budolID for Mobile Identification const budolIdRes = await axios.get(`${BUDOL_ID_URL}/auth/check-phone?phone=${encodeURIComponent(phoneNumber)}`, { timeout: 3000 }); if (budolIdRes.data && budolIdRes.data.exists) { // Create user in budolPay with matching ID and profile user = await prisma.user.create({ data: { id: budolIdRes.data.id, email: budolIdRes.data.email, phoneNumber: normalizedPhone, firstName: budolIdRes.data.firstName || 'User', lastName: budolIdRes.data.lastName || '', passwordHash: 'SSO_MANAGED', // Authenticate via SSO/OTP wallet: { create: { balance: 0.0, currency: 'PHP' } } } }); } }

Compliance Note

Sensitive data (Phone, Email) is localized to the service to ensure PCI DSS compliance for wallet-related operations, while identity is centralized in budolID to satisfy BSP Circular 808 (E-Money & Identity management).

3. budolShap: Pricing Fix

Corrected the product card pricing logic which was showing ₱1.00 instead of the actual lowest price (₱0.00 for promotional items).

📁 budolshap-0.1.0/components/Hero.jsx // [Old Logic] Incorrectly excluded 0.00 prices const validPrices = products .map(product => Number(product.price)) .filter(price => price > 0); // <-- ERROR: items for 0.00 were ignored // [New Logic] Corrected for 0.00 (e.g., Free samples / Promo items) const validPrices = products .map(product => Number(product.price)) .filter(price => !isNaN(price)); // <-- FIXED

Technical Explanation

By using price > 0, items with price: 0 (promotional) were filtered out, causing the next lowest price (e.g. ₱1.00) to be displayed as the minimum. This led to customer confusion regarding promotional offers.
