Mitigation Strategies v5.3.10

Mitigation [R-001]: Identity Collisions

Action: Implemented "Sync-by-ID" logic. The Auth service (budolPay) now explicitly adopts the id from budolID during sync, ensuring a single identity per user across all ecosystem tables.

Mitigation [R-002]: Proxy Latency

Action: Introduced a 3000ms timeout for cross-service calls and backgrounded non-blocking sync operations where possible. Future plan includes identity caching.

Mitigation [R-003]: Price Manipulation

Action: Reverted frontend mapping to honor RAW database values. Any price modification must now pass backend pricing engine validation before transaction commit.

Mitigation [R-004]: PII Protection

Action: All metadata returned in identity checks (budolID check-phone) is MASKED on egress if the user is not authenticated. Internal cross-service calls use trusted service tokens.
