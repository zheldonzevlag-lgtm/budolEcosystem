System Admin Manual v5.3.10

Service Management Guide

System Overview

Administers the unified ecosystem synchronization between budolID (SSO) and downstream services (budolPay, budolShap). Key changes in this release involve the introduction of a proxy authentication layer in the budolPay Auth service.

1. Auth Proxy Configuration

The budolPay Auth service (Port 8001) now requires awareness of the identity service (budolID). This is managed via environment variables.

Variable Default Value Purpose
BUDOL_ID_URL http://localhost:8000 Endpoint for identity verification and sync.
LOCAL_IP Auto-detected Used for service linking if no explicit URL is set.
Important: If budolID is running on a different server, the BUDOL_ID_URL MUST be updated in the .env file of budolpay-0.1.0.

2. DTD & Service Discovery

The services utilize the Dart Tooling Daemon (DTD) for mobile-to-backend linkage. Ensure all services are registered and discoverable on the local network if dev mode is enabled.

_block # Checking Port status for key services # [8000] - budolID (Identity/SSO) # [8001] - budolPay Auth (Bridge) # [8002] - budolPay Wallet (Shadow Table) netstat -ano | findstr 8000 8001 8002

3. Troubleshooting Sync Issues

If a user exists in the ecosystem but is not showing up in the mobile app, follow these steps:

  1. Verify budolID is ACTIVE: Check if http://localhost:8000/auth/check-phone?phone=... returns a valid JSON response.
  2. Check Service Logs: Monitor budolPay logs for [Auth Proxy] or [Auth Sync] tags.
  3. Database Integrity: Ensure the phoneNumber in budolID matches the format expected (normalized to E.164).
