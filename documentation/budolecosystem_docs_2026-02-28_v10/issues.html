Issues & Fixes v5.3.10

1. Mobile App: Account Not Found for Ecosystem Users

Description: Existing users in budolID were not recognized by budolPay mobile app during login, showing an "Account not found" error if they hadn't previously registered specifically for the Pay service.

FIX: Identity Proxy & Sync Implementation

Mechanism: The budolPay Auth service now queries budolID if a phone number search fails locally. If found, a local profile is automatically created (Synced) using the same Identity UID.

// budolPay Auth Sync if (budolIdUserExists) { autoSyncToPayLocal(); proceed(); }
2. Homepage: Minimum Pricing Displayed as ₱1.00 for Free Items

Description: Product cards on the homepage were incorrectly skipping ₱0.00 items in the minimum price calculation calculation, resulting in ₱1.00 being shown as the "Starts from" price.

FIX: Inclusive Price Range Logic

Mechanism: Corrected the pricing filter in Hero.jsx to allow 0.0 values while excluding invalid (NaN) entries. Verified that promotional items now display from ₱0.00.

const validPrices = products.map(p => Number(p.price)).filter(price => !isNaN(price));
