JWT Synchronization & SSO Fix

Purpose

To resolve the persistent "Session Expired" loop in budolShap. This issue was identified as a signature verification failed error caused by inconsistent JWT_SECRET values between the signing service (budolID) and the verifying service (budolShap).

Technical Details

Root Cause Analysis

  • budolID was signing tokens with "budolid-super-secret-key" (fallback).
  • budolShap was verifying tokens with "GJ7Lxn0/kdV/KuZJ5xJ7Ip0RvMerrGW5n0gf44mfHgc=".
  • The mismatch caused an invalid signature error in jsonwebtoken and jose.

Fix Implementation

Aligned all JWT_SECRET definitions and hardcoded fallbacks to the standardized Base64 secret:

GJ7Lxn0/kdV/KuZJ5xJ7Ip0RvMerrGW5n0gf44mfHgc=

Impact Analysis

Resolved

SSO Login Loop fixed. Users can now maintain active sessions across services.

Risk

Existing sessions signed with the old secret will be invalidated, forcing a one-time re-login.

Future Enhancement

Implement a centralized Secret Management Service (e.g., HashiCorp Vault) to manage environment variables dynamically and prevent future drift between microservices.
