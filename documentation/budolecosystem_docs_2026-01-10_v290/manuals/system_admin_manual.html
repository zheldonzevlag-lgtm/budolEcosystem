Service Synchronization Protocol

Critical Update: JWT Secrets

All services in the budolEcosystem must use the same JWT_SECRET to allow seamless cross-service authentication. v290 enforces a new standard secret across all .env files.

Required Action

Verify the following variable in all microservice .env files:

JWT_SECRET="GJ7Lxn0/kdV/KuZJ5xJ7Ip0RvMerrGW5n0gf44mfHgc="

Restart Sequence

To apply changes, restart services in the following order:

  1. 1

    budolID Service (Port 8000)

    Primary token issuer. Must be updated first.

  2. 2

    budolShap Service (Port 3001)

    Service Provider. Verifies tokens issued by budolID.

  3. 3

    budolPay Services (Ports 8001-8080)

    Gateway and Auth services must be aligned.

Monitoring

Watch the logs for [Token] or [TokenEdge] tags. Successful authentication will show "Successfully verified token for user: ...". Mismatches will show "signature verification failed".
