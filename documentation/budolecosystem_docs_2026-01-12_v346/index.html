Release Overview - v346

Purpose: Resolve "stuck in loading" issue in budolPay Mobile Web App and optimize initialization flow.

Rationale: Users reported the app getting stuck on the splash screen during loading. This was caused by blocking network calls and duplicate initialization during the startup sequence. By making network-dependent tasks non-blocking and improving splash screen navigation logic, the app now loads reliably even under poor network conditions.

Technical Explanation

Key architectural changes include:

  • Non-Blocking Initialization: The ApiService._initialize method was refactored to perform network-dependent tasks (fetching system settings and user profiles) in a background _backgroundInit method. This prevents the app's main initialization from hanging due to network delays.
  • Splash Screen Logic: Removed redundant apiService.init() call in splash_screen.dart and added explicit authentication-based navigation. Authenticated users are now routed directly to the Home screen, bypassing unnecessary steps.
  • Host Normalization: Improved host detection logic for web environments to ensure consistent connectivity across different LAN configurations.
