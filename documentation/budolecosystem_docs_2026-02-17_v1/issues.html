Known Issues (v3.4.1)

Status: Active

Open

Issue: Checkout Flow Interruption

Description: If a user navigates back during the PayMongo redirect, the order might be created in PENDING state but the cart remains full.

Workaround: Users must manually cancel the pending order or complete the payment via the "My Orders" page (if retry is available).

Resolved

Issue: Cart Persistence Race Condition

Description: Cart items were disappearing after logout/login due to a race condition in the sync logic.

Resolution: Fixed in v3.4.0 by reordering the logout and clear cart actions.

Resolved

Issue: Async Payment Cart Cleanup

Description: Items remained in cart after successful GCash payment.

Resolution: Fixed in v3.4.1 by adding server-side fallback cleanup in ordersService.js.
