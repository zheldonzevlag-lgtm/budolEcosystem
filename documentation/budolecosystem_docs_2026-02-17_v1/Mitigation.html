Mitigation Plan (v3.4.1)

Status: Active

Risk: Webhook Latency (R-2026-004)
Mitigation Strategy:
  • Implemented redundant cart cleanup logic. It runs on the client side (return/page.jsx) for immediate feedback AND on the server side (ordersService.js) as a fallback.
  • If the webhook is delayed, the client-side cleanup will likely handle it first. If the user closes the tab, the server-side logic catches it eventually.
  • Contingency: If both fail, the user can manually remove items. This is a UX annoyance but not a critical data failure.
Risk: Backup Script Failure (R-2026-005)
Mitigation Strategy:
  • The script uses try-catch blocks for each table export to prevent one failure from stopping the entire process.
  • Added logging to clearly indicate which tables succeeded or failed.
  • Contingency: Keep the previous day's backup available. Run backups during low-traffic periods.
Risk: Variation Deletion Mismatch (R-2026-006)
Mitigation Strategy:
  • The cleanup logic explicitly handles null variation IDs. variationId: item.variationId || null ensures that products without variations are also correctly targeted.
  • The query uses a precise AND condition (cart user + specific product + specific variation) to avoid accidental deletion of other items.
