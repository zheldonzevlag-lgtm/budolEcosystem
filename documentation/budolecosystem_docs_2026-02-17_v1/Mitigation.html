Strategies

M-001: Transactional Integrity for Escrow

Addresses Risk: R-001 (Escrow Lock Failure)

Strategy: All return operations (Approve, Refund, Reject) are wrapped in prisma.$transaction. This ensures that status updates and fund unlock/transfer happen atomically. If one fails, the entire operation rolls back.

M-002: Cryptographic Hashing for Audit Logs

Addresses Risk: R-002 (Audit Log Tampering)

Strategy: Implemented SHA-256 hashing of critical audit log fields (User ID, Action, Entity ID, Timestamp, Metadata). The hash is stored in the metadata. Any alteration to the log entry will result in a hash mismatch during verification.

M-003: Robust Cart Cleanup Logic

Addresses Risk: R-003 (Cart Persistence)

Strategy: Enhanced ordersService.js to handle products without variations explicitly by forcing variationId: null. Added logging to trace cleanup operations.

M-004: Strict Authorization Checks

Addresses Risk: R-004 (Unauthorized Return Response)

Strategy: In returnsService.js, every function (e.g., buyerRespondToProposal, respondToReturn) begins with a strict check verifying that the userId matches the Order's User (for buyers) or the Store's Owner (for sellers).
