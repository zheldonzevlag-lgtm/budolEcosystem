System Admin Manual (v3.4.1)

Topic: Maintenance & Backup Procedures

Critical Update: The database backup script has been updated to include Cart and CartItem tables. Ensure you are using the latest version of the script.

1. Database Backup Procedures

Regular backups are essential for data integrity. The system uses a Node.js script to export all tables to JSON format.

Running a Backup

To perform a manual backup of the production database:

  1. Open a terminal in the project root directory (d:\IT Projects\budolEcosystem\budolshap-0.1.0).
  2. Execute the following command: 
    node scripts/backup_db.js
  3. Verify the output: 
    Starting backup to .../backup-db/db-YYYY-MM-DD...
✅ Backed up user
✅ Backed up product
✅ Backed up order
...
✅ Backed up cart (NEW)
✅ Backed up cartItem (NEW)
Backup completed.

Backup Location

Backups are stored locally in:

d:\IT Projects\budolEcosystem\budolshap-0.1.0\backup-db\db-[DATE]\

2. Cart Cleanup Maintenance

The system now includes automated cart cleanup logic for async payments. However, manual cleanup may occasionally be required if webhooks fail catastrophically.

Monitoring Orders

Check the application logs for the following entry to confirm automatic cleanup is working:

[OrdersService] Removed X items from cart for User Y

Manual Intervention

If a user reports items sticking in the cart after payment:

  1. Verify the order status is PAID in the admin dashboard.
  2. If PAID, you can manually trigger the status update logic by toggling the status to PENDING and back to PAID (if the UI allows), or by running a script.

3. Deployment Checklist

Before deploying this version to production:

  • [ ] Database: Ensure prisma db push has been run to update the schema (added variationId to CartItem).
  • [ ] Environment: Check PAYMONGO_SECRET_KEY is valid for webhook processing.
  • [ ] Backup: Run a full database backup using the new script.
