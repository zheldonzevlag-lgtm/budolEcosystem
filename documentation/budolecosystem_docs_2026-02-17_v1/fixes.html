Version 2026-02-17 v1 Fixes

Issue: Cart items remain after successful GCash/Maya payment (Async Payments).
Fix: Implemented server-side fallback cart cleanup logic in updateOrderStatus.
  • Problem: Async payments (PayMongo) rely on webhooks or status checks. The initial order creation might fail to clear the cart if the payment status isn't immediately finalized, or if the client-side cleanup in return/page.jsx is skipped (e.g., user closes tab).
  • Solution: Added a check in ordersService.js. When an order transitions to PAID status, the system now explicitly deletes the corresponding CartItem records for the user.
  • Detail: The deletion is precise, targeting only the productId and variationId present in the paid order, preserving other items in the cart.
Issue: Database Backup Script missing Cart tables.
Fix: Updated scripts/backup_db.js to include Cart, CartItem, and other missing models (Store, Wallet, Transaction, etc.) in the backup process.
