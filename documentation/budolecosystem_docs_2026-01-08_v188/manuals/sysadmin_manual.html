System Admin Manual - v188

Deployment Procedures

To synchronize environment variables and redeploy the ecosystem on Vercel:

  1. Ensure .env.vercel in budolpay-0.1.0 contains the latest credentials.
  2. Run the environment manager locally:
    cd budolpay-0.1.0
    npm run db:vercel
  3. Run the Vercel synchronization script:
    powershell -ExecutionPolicy Bypass -File "scripts/sync-vercel-envs.ps1"
  4. Redeploy services (Note: Vercel has a 100-deployment daily limit):
    cd services/[service-name]
    vercel --prod --yes

Infrastructure Changes

Database: Prisma Accelerate

The system has transitioned to using Prisma Accelerate as the primary connection layer. This improves reliability and performance in serverless environments.

DATABASE_URL format: prisma+postgres://accelerate.prisma-data.net/?api_key=...

Operational Runbook

Issue: 500 Internal Server Error (Connectivity)

Diagnosis: Check if target URLs in API Gateway logs contain \r\n or are using old "friendly" domains.

Fix: Run sync-vercel-envs.ps1 and redeploy the API Gateway.

Issue: 401 Unauthorized (Vercel Protection)

Diagnosis: Inter-service communication is blocked by Vercel's deployment protection.

Fix: Ensure VERCEL_BYPASS_TOKEN is set in all services and matches the one in API Gateway.

Environment Variables

Key variables now required in all service deployments:

  • DATABASE_URL: Prisma Accelerate URL
  • JWT_SECRET: Shared secret for token validation
  • VERCEL_BYPASS_TOKEN: Shared secret for inter-service bypass
  • AUTH_SERVICE_URL, WALLET_SERVICE_URL, etc.: Fully qualified HTTPS URLs of peer services.
