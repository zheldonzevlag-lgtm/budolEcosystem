Developer Manual - v188

Version: v188 (2026-01-08)

Purpose: Documentation of the Vercel Connectivity Strategy and Environment Synchronization.

Technical Implementation Details

1. Environment Synchronization Strategy

The ecosystem now follows a "Centralized Config" strategy where .env.vercel in the root of budolpay-0.1.0 acts as the source of truth for all Vercel deployments. This is managed by the updated env-manager.js script.

2. Code Logic Explanations

  • env-manager.js: Modified to support automatic creation of .env files for services if they don't exist. This ensures that even fresh service clones are correctly configured for the target environment (Local/Prod/Vercel).
  • API Gateway (index.js): Implemented sanitizeUrl function using regex replace(/\s/g, '') to strip all whitespace characters (including \r\n) from environment-sourced URLs. This ensures robust proxy routing even if Vercel environment variables contain hidden characters.
  • sync-vercel-envs.ps1: A new PowerShell utility that automates the linking of sub-services to Vercel projects and synchronizes their environment variables from the central .env.vercel file.

3. Architectural Decisions

  • Prisma Accelerate: Switched from direct Vercel Postgres/Neon connections to Prisma Accelerate. Reason: Direct connections frequently hit connection limits and authentication issues in serverless environments. Accelerate provides a connection pooler that is highly compatible with Vercel's scaling model.
  • Unique Deployment URLs: Moved away from "friendly" URLs (e.g., budolpay-auth.vercel.app) to unique deployment URLs (e.g., budolpay-auth-service.vercel.app) to resolve 508 Loop Detected and 404 errors caused by misrouted proxy requests.

Impact Analysis

System Impact Mitigation
API Gateway High: Proxy routing logic changed. Added /test-internal endpoint for real-time validation.
Database High: Credentials changed for all services. Verified connectivity via check-db.js before rollout.
Deployments Medium: New environment variables required. Automated sync script created to prevent manual errors.
