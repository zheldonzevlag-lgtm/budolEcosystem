Umbrella Architecture

The budol Ecosystem is designed as a Lego Block System. Each application is a self-contained "block" that snaps into a central "base plate" (budolID).

Core Lego Blocks (Phase 1)

Base Plate

budolID

The foundation. Handles Identity, SSO, and App Permissions. Every other block snaps into this for authentication.

Payment Block

budolPay

The financial engine. Handles wallets, ledgers, and transactions. Provides "Pay with budolPay" services to other blocks.

E-commerce Block

budolShap

The shopping engine. Handles products, orders, and stores. Consumes payment services from the Payment Block.

Snap-in Integration Strategy

Integration between budolPay and budolShap follows these rules:

  • Loose Coupling: budolShap doesn't need to know how budolPay's ledger works; it only needs to call a "Charge" API.
  • Shared Identity: A user's unique budolID is the thread that connects their wallet to their shopping cart.
  • Independent Scaling: The shopping block can be updated or scaled without affecting the payment block.

Future Blocks (Roadmap) COMING SOON

These blocks are designed to snap into the same base plate when ready:

  • budoLoan: Credit and lending services.
  • budolExpress: Logistics and delivery tracking.
  • budolCare: Health and insurance services.
