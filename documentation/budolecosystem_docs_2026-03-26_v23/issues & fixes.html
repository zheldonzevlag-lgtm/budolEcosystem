Historical Issues & Emergency Fixes

ISSUE: Storefront displaying "0 products" on Vercel deployment.
FIX: Restored production data from local PostgreSQL backups into Vercel's 'public' schema using the custom manual-data-sync.js script.
ISSUE: User authentication (SSO) failing on login attempt.
FIX: Restored 'User' records to the 'budolid' schema and synchronized the JWT_SECRET environment variable across all SSO-dependent services.
ISSUE: Vercel Postgres connection timeout (10s proxy limit) during bulk restoration.
FIX: Replaced monolithic psql restores with a partitioned Node.js sync script that performs row-by-row insertion with ON CONFLICT clauses.
ISSUE: Prisma Interactive Transaction Timeout (5841ms > 5000ms) during order creation.
FIX: Increased interactive transaction timeout to 15s and optimized stock updates by using a bulk findMany call inside the transaction to reduce database round-trips.
ISSUE: ERR_CONNECTION_REFUSED to 192.168.1.10:4000 in production console.
FIX: Modified lib/realtime.js to ignore LOCAL_IP in production and automatically fall back to POLLING mode if a localhost Socket.io URL is detected.
