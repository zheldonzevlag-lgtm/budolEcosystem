Developer Manual: Data Sync & Automation

Manual Sync Architecture

To bypass Vercel's connection timeouts, we use a row-by-row synchronization strategy. The Primary tool for this is scripts/manual-data-sync.js.

Core Sync Script

This script parses the SQL output of a pg_dump and performs targeted upserts.

// Location: budolshap-0.1.0/scripts/manual-data-sync.js // WHAT: Parses COPY blocks and inserts data into Vercel Postgres schemas. // WHY: Cloud database proxy limits (10s) prevent large monolithic restores.

Database Schema Mapping

Developers must ensure that all Prisma queries targeting non-public schemas use the specific schema names in their connection strings:

  • Identity: postgresql://USER:PASS@HOST:PORT/postgres?schema=budolid
  • Payments: postgresql://USER:PASS@HOST:PORT/postgres?schema=budolpay
  • Accounting: postgresql://USER:PASS@HOST:PORT/postgres?schema=budolaccounting

Restoring New Tables

To restore a new table, follow these steps:

  1. Convert the .PGDMP to .sql_txt.
  2. Identify the COPY table_name (...) FROM stdin; block.
  3. Add the table name to tablesToSync in manual-data-sync.js.
  4. Run node scripts/manual-data-sync.js.

Compliance Note

All data restoration must maintain PCI DSS compliance. No plain-text passwords or card details should be logged during the sync process.
