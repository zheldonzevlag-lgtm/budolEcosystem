System Admin Manual: Vercel Infrastructure

Vercel Postgres Configuration

The Budol Ecosystem is deployed on a single Vercel Postgres instance. All 13 services MUST connect to this cluster using Schema Isolation.

Environment Variable Audit (v23)

Service Variable Name Value Pattern
BudolShap DATABASE_URL postgresql://.../postgres?schema=public
BudolID DATABASE_URL postgresql://.../postgres?schema=budolid
BudolPay DATABASE_URL postgresql://.../postgres?schema=budolpay
BudolAccounting DATABASE_URL postgresql://.../postgres?schema=budolaccounting
Auth System JWT_SECRET RESTORED_SECURE_KEY

Prisma Acceleration

All microservices are configured to use Prisma Accelerate to handle serverless connection pooling on Vercel. Ensure PRISMA_DATABASE_URL is set correctly in Vercel settings for each project.

Security & Backup Protection

  • Automated Backups: Enabled via Vercel Storage settings (daily snapshots).
  • Manual Syncs: Should only be performed using the manual-data-sync.js script to avoid downtime.
  • Credential Management: All environment variables are stored in Vercel Encrypted Secrets. DO NOT EXPOSE DATABASE_URL in code.

Restoration Points

In the event of a cluster failure, the PGDMP files in G:\IT\database-backup\ are the authoritative source for reconstruction.
