Developer Manual - v429 Fixes

Issue Description

A TypeError: Cannot read properties of undefined (reading 'slice') occurred in OrderHeader.jsx when order.id was not present. This typically happened when an API error (e.g., 403 Forbidden) was returned as a JSON object but not caught as an error state.

Technical Fixes

1. Defensive Programming in UI

File: OrderHeader.jsx

// Before
Order #{order.id.slice(-12)}

// After
Order #{order?.id ? order.id.slice(-12) : '---'}

2. SWR Fetcher Enhancement

File: useRealtimeOrderDetail.js

const fetcher = async (url) => {
    const res = await fetch(url);
    if (!res.ok) {
        const error = new Error('An error occurred');
        error.info = await res.json();
        error.status = res.status;
        throw error;
    }
    return res.json();
};

3. Error Propagation

File: page.jsx

useEffect(() => {
    if (swrError) {
        setError(swrError.info?.error || swrError.message);
    } else if (order?.error) {
        setError(order.error);
    }
}, [swrError, order]);

Verification

  • Simulated 403 error from /api/orders/[orderId].
  • Verified page displays "Forbidden" error UI instead of crashing.
  • Verified valid orders still render correctly.
