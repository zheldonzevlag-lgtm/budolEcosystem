Release Notes v429

Purpose & Rationale

This release addresses a critical Runtime TypeError in the Order Details page where order.id was undefined, causing the application to crash when trying to render the Order Header.

Key Changes

1. Order Header Resilience

Modified OrderHeader.jsx to include optional chaining and defensive guards for order and its properties (id, createdAt, status, etc.).

2. API Error Handling

Updated the SWR fetcher in useRealtimeOrderDetail.js to throw structured errors on non-OK API responses.

3. Error UI Integration

Modified page.jsx to capture SWR errors and display them using the existing error UI instead of allowing the page to crash.

Compliance

  • Cybersecurity Law: Improved application stability and error handling prevents information leakage through stack traces.
  • UX Best Practices: Replaced blank screen/crash with informative error messages.
