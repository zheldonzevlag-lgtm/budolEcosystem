Managing Admin Access

With the implementation of SSPR, administrators no longer need to manually update the database if a password is forgotten. However, certain security parameters should be monitored.

Monitoring SSPR Activity

SSPR attempts are logged in the budolID service logs. Admins should monitor for:

  • Frequent POST /auth/forgot-password requests for the same email (Potential enumeration/harassment).
  • Repeated POST /auth/verify-otp failures (Potential brute force).

Configuring OTP Expiration

Currently, the OTP expiration is hardcoded to 5 minutes in budolID/index.js. To change this, locate the otpExpires calculation:

const otpExpires = new Date(Date.now() + 5 * 60 * 1000);
Security Note: Always ensure the JWT_SECRET in the .env file of budolID is kept secure, as it is used to sign session tokens after a successful password reset login.
