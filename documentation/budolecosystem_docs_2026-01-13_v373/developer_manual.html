File-Based TFLite Loading

To resolve the persistent initialization failures on Android, the FaceEmbeddingService now uses a 3-step extraction process:

  1. Extraction: rootBundle.load() retrieves the binary data from the APK assets.
  2. Persistence: Data is written to getTemporaryDirectory() using standard I/O.
  3. Initialization: Interpreter.fromFile() loads the model from the filesystem path.

Key Code Reference

// lib/services/face_embedding_service.dart
final tempDir = await getTemporaryDirectory();
final modelFile = File('${tempDir.path}/facenet.tflite');
await modelFile.writeAsBytes(bytes, flush: true);
_interpreter = await Interpreter.fromFile(modelFile, options: options);
