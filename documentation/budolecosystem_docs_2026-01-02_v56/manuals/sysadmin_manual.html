SysAdmin Manual - Version 56

Session Synchronization & Stability

Version 56 focuses on ensuring session stability across multi-service environments, specifically when testing via Local IP or LAN addresses.

1. Cross-Layer Session Persistence

Authentication now relies on a dual-mechanism:

  • Client-Side: Tokens are stored in both localStorage and cookies (budolshap_token).
  • Request-Side: API calls now include the Authorization: Bearer header, providing a fallback when cookies are blocked or stripped by proxies/local network settings.

2. Verification Steps

To verify session stability in your environment:

  1. Perform a checkout using a local IP address (e.g., http://192.168.x.x:3000).
  2. Monitor the browser console for [AuthContext] logs.
  3. Verify that the /api/auth/me call is successful.
  4. Observe that a temporary network disconnection or a server 500 error does NOT trigger a logout.
Note for LAN Testing: Ensure your .env file does not have SESSION_SECURE=true if you are testing on non-HTTPS local IP addresses, as this will prevent cookies from being sent.

3. Operational Runbook

  • Issue: User is redirected to login during checkout.
  • Check: Verify if the token exists in localStorage. Check server logs for 401 status on /api/orders.
  • Resolution: Ensure JWT_SECRET matches across all services (budolShap and budolPay).
