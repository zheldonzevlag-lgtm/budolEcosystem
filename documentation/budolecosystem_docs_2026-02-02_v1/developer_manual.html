Purpose & Rationale

This document details the technical implementation of the administrative role elevation for the Budol Ecosystem. The rationale is to ensure that authorized administrators have consistent access across all integrated services (BudolPay, BudolID, BudolShap).

Technical Explanation

The role elevation involves updating the role column in the User table across multiple PostgreSQL databases. Due to the distributed nature of the ecosystem, synchronization is handled at the database level using raw SQL queries where Prisma schema synchronization is not yet fully propagated.

Database Promotion Script (BudolPay)

This script updates the user role in the primary BudolPay database.

promote_to_admin_v2.js
const result = await prisma.$executeRaw`
  UPDATE "User" SET role = 'ADMIN' WHERE email = ${email};
`;

Database Promotion Script (BudolID Schema)

This script updates the user role in the BudolID schema within the shared database.

update-budolid-role.js
const result = await prisma.$executeRaw`
  UPDATE "budolid"."User" SET role = 'ADMIN' WHERE email = ${email}
