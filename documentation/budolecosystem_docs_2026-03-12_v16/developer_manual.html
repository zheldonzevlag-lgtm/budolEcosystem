Developer Manual - v16

Technical Explanation

The core issue was a database schema mismatch between the Prisma model and the actual RDS table naming. The database used pluralized SystemSettings while the application code and initial schema used singular SystemSetting.

Prisma Schema Mapping

We used the @@map directive to bridge the gap without renaming the model in the codebase, although we eventually pluralized the model for full consistency.

model SystemSettings { id String @id @default("default") ... @@map("SystemSettings") }

View schema.prisma

Code Changes

Updated all references from prisma.systemSetting to prisma.systemSettings across the following files:

  • lib/realtime.js: Updated settings fetch logic.
  • lib/settings.js: Updated global settings provider.
  • lib/rate-limit.js: Fixed dependency on system settings.

Prisma Client Generation

The build process now includes a forced npx prisma generate with the updated schema to ensure type safety for the pluralized model.
