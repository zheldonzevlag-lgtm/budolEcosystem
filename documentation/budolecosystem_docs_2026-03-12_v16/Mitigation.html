Mitigation Plan - v16

Strategy for Database Consistency

Strategy: Architectural Mapping & Environment Pinning
Execution: We have pinned the DATABASE_URL in Secrets Manager to explicitly point to budolshap_1db and modified the Prisma schema to use @@map("SystemSettings") to align with the production database table names.

Mitigation of Deployment Failures

  • Pre-deployment: Used npx prisma generate inside the Docker build stage to ensure the client is built against the fixed schema.
  • Rollback Plan: Maintain the budolshap legacy database as a cold backup.
  • Monitoring: Post-deployment log inspection via CloudWatch is mandatory for the first 4 hours of production traffic.
