Issues & Fixes - v16

Issue #1: "The table public.SystemSetting does not exist"

Root Cause: The application was looking for a singular SystemSetting table while the database had SystemSettings (plural).

Fix: Renamed Prisma model to SystemSettings and updated all prisma.systemSetting calls in lib/realtime.js and lib/settings.js to use the pluralized client property.

Issue #2: "Internal Server Error" on Mobile Login

Root Cause: The database URL was pointing to the budolshap DB which lacked critical system configuration records.

Fix: Corrected AWS Secrets Manager budol/db-url to point to budolshap_1db.

Issue #3: Missing libssl in Docker Image

Root Cause: Debian Bookworm (default in node:20-slim) does not include libssl1.1 required by Prisma 5.7.0.

Fix: Switched Docker base image to node:20-bullseye-slim.
