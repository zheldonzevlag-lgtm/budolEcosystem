Developer Technical Guide

1. API Gateway CORS Configuration

The API Gateway was updated to move security and CORS middleware to the very top of the app. This ensures that preflight OPTIONS requests from browsers (like Flutter Web) are handled before they hit any proxy routes.

Code Reference: api-gateway/index.js


// 1. GLOBAL MIDDLEWARE (Security, CORS, Logging) - MUST BE FIRST
app.use(cors({
    origin: '*', // Allow all for development
    methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS', 'PATCH'],
    allowedHeaders: ['Content-Type', 'Authorization', 'X-Requested-With'],
    credentials: true
}));
app.use(helmet({
    contentSecurityPolicy: false, // Disable for development
    crossOriginResourcePolicy: { policy: "cross-origin" }
}));
app.use(morgan('dev'));

2. Microservice Port Synchronization

The Gateway proxy routes for Accounting and ID services were synchronized with the local start-ecosystem.js configuration to avoid 404 and ECONNREFUSED errors in dev mode.

Code Reference: api-gateway/index.js


const services = [
    // ... other services ...
    {
        route: '/accounting',
        target: sanitizeUrl(isDev ? `http://${LOCAL_IP}:8005` : (process.env.ACCOUNTING_SERVICE_URL || (isConsolidated ? '/api/accounting' : `http://${LOCAL_IP}:8005`)))
    },
    {
        route: '/id',
        target: sanitizeUrl(isDev ? `http://${LOCAL_IP}:8000` : (isConsolidated ? '/api/id' : (process.env.ID_SERVICE_URL || `http://${LOCAL_IP}:8000`)))
    }
];

4. Transaction History Fetching

The Transaction service provides a history endpoint used by the Flutter Web app. This service must be running for the dashboard to display transaction data.

Code Reference: transaction-service/index.js


router.get('/history/:userId', async (req, res) => {
    const { userId } = req.params;
    try {
        const transactions = await prisma.transaction.findMany({
            where: {
                OR: [
                    { senderId: userId },
                    { receiverId: userId }
                ]
            },
            orderBy: {
                createdAt: 'desc'
            },
            take: 50
        });
        res.json(transactions);
    } catch (error) {
        console.error('[Transaction] History Error:', error.message);
        res.status(500).json({ error: 'Failed to fetch transaction history' });
    }
});

5. Compliance and Data Privacy

  • BSP Circular 808: System availability and transaction logging are maintained via microservice health checks and internal gateway routing.
  • PCI DSS: Secure headers (Helmet) and CORS policies are enforced at the API Gateway level.
  • NPC (Data Privacy Act): Transaction history and balance data are scoped to specific User IDs and protected by the Gateway's auth middleware.
