Release Overview - v343

Purpose: Resolve "Failed to fetch balance" and empty Transaction History in Flutter Web App.

Rationale: Post-login functionality was impaired due to incorrect middleware ordering in the API Gateway (causing CORS failures) and port mismatches between the Gateway and microservices (Accounting and ID services). Additionally, services needed a coordinated restart via the central runner.

Technical Explanation

The API Gateway's CORS middleware must be positioned before any routes or proxy definitions to correctly handle preflight OPTIONS requests. Furthermore, the LOCAL_IP based proxy targets in dev mode were out of sync with the start-ecosystem.js configuration for Accounting (8007 vs 8005) and ID (8008 vs 8000) services. Synchronizing these ports and ensuring a global origin: '*' policy resolved the connectivity issues.
