Risk Mitigation Actions

M-343-01: Global CORS Reconfiguration

Action: Moved cors() and helmet() middleware to the very beginning of the Express app in the API Gateway. Set origin: '*' and credentials: true for development flexibility.

M-343-02: Port Sync and Verification

Action: Updated the Gateway's service definition array to match the actual ports used by budolID (8000) and budolAccounting (8005). Verified connectivity via /test-internal.

M-343-03: Service Default Port Correction

Action: Hardcoded the default fallback port for budolAccounting to 8005 in its index.js to prevent accidental conflicts with the Wallet service on 8002.

M-343-04: Orchestrated Startup

Action: Used start-ecosystem.js with ENV_CHOICE=3 to ensure all dependent microservices (Auth, Wallet, Transaction, Payment, Verification, Accounting, ID) are started in a coordinated manner.
