System Administration Guide

1. Ecosystem Startup

To start all microservices, use the central ecosystem runner with the SKIP option to maintain local configurations.

$env:ENV_CHOICE="3"; node start-ecosystem.js

2. Verifying Service Health

The API Gateway provides a centralized endpoint to check the connectivity and health of all microservices.

Invoke-RestMethod -Uri "http://localhost:8080/test-internal"

3. Troubleshooting Connectivity

  • 404 on /id: Check if budolID is running on port 8000 and has the /api/health endpoint.
  • ECONNREFUSED: Ensure the specific microservice is running. Check ports 8001-8008.
  • CORS Errors: Verify that api-gateway/index.js has the cors() middleware defined before any routes.

4. Network Configuration

The system automatically detects the local LAN IP (e.g., 192.168.1.17) and updates .env files. Ensure your firewall allows traffic on ports 8000-8080 and 9000 (Flutter Web).

5. Compliance Monitoring

Audit logs are generated by morgan in each service. Monitor the API Gateway logs for suspicious preflight failures or unauthorized access attempts (401/403).
