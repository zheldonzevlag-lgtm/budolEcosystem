Developer Manual: Unified KYC Capture

Technical Architecture

The KYCCaptureScreen utilizes the camera package and a custom CustomPainter to provide real-time UI guidance. It replaces ImagePicker to allow for fine-grained control over orientation and resolution.

Core Component: KYCCaptureScreen

Link: kyc_capture_screen.dart


enum KYCCaptureType { face, idCard }

// Implementation uses CustomPainter for overlays
if (captureType == KYCCaptureType.face) {
    innerPath = Path()..addOval(innerRect);
} else {
    innerPath = Path()..addRRect(RRect.fromRectAndRadius(innerRect, Radius.circular(12)));
}

Orientation & Stability Fix

To prevent sideways orientation on emulators, capture orientation is explicitly locked:

await _controller!.lockCaptureOrientation(DeviceOrientation.portraitUp);

Phase 8 - Software & Dependencies

This release introduces significant changes to the mobile camera handling layer. The following dependencies were chosen based on specific technical requirements:

Software/Package Version/Status Rationale for Choice
camera Latest (Stable) Chosen over ImagePicker to allow for custom overlays and lower-level control of capture parameters (resolution, orientation locking).
flutter_lints ^3.0.0 Ensures code quality and adherence to Budol documentation standards.
Android SDK 35 (Updated) Optimized for performance on newer devices while maintaining legacy permission support.
