Issues & Applied Fixes

Issue: NaN Amount in Payment Gateway

Symptom: Users could theoretically send non-numeric strings in the amount field, causing Prisma errors or 0-value transactions.

FIXED Added isNaN check in payment-gateway-service/index.js.

Issue: Duplicate Orders for Same orderId

Symptom: Multiple "Pending" transactions could be created for the same order if the user clicked "Pay" multiple times.

FIXED Implemented findFirst check with OR logic for metadata orderId search.

Issue: QR Amount Tampering Risk

Symptom: A malicious user could scan a QR but modify the amount in the client-side request before it reaches the wallet service.

FIXED Added cross-reference check in wallet-service/index.js against the database transaction amount.
