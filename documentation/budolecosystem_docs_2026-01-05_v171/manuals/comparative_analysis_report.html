Phone Management Analysis & Recommendations

A deep dive into GCash protocols, BudolPay's implementation, and industry-standard roadmaps.

1. GCash Official Process Review

GCash treats the mobile number as the primary account identifier. Because of this, a direct "edit number" function does not exist for verified accounts.

The "Migration" Workflow:
  1. User must register a new GCash account with the new number.
  2. The new account must undergo Full Verification (KYC).
  3. User submits a Transfer of Funds request via the Help Center.
  4. Old account balance is moved; however, Transaction History, GScore, and GForest data are LOST.

3. Industry Standards & Future Recommendations

To align with global fintech standards (FIDO2, Biometric Security), we recommend the following evolution for BudolPay:

🚀 Recommendation A: Automated Face-Match (AFM)

Standard: NIST Digital Identity Guidelines.

Technical Detail: Compare the user's "Phone Change Selfie" against the original "KYC ID" using AWS Rekognition or ML Kit Face Embeddings.

Benefit: Reduces admin workload by 80% and prevents "Social Engineering" attacks where attackers might upload a fake ID.

🔐 Recommendation B: FIDO2 / Passkey Implementation

Standard: W3C WebAuthn / FIDO Alliance.

Technical Detail: Decouple the account identity from the phone number. Use device-bound Passkeys (Biometrics/Hardware) as the primary credential.

Benefit: If a user changes their number, their Passkey remains valid on their device, allowing them to authorize the number change instantly without a manual support ticket.

📱 Recommendation C: Dual-OTP Handshake

Process: Require OTP verification from both the old and new numbers simultaneously.

Use Case: For users who still have access to their old SIM. This provides an "Express Path" for number changes that skips manual review.

4. Technical Implementation Roadmap

  • Phase 1 (Q1 2026): Implement Recommendation C (Dual-OTP) to handle 60% of requests automatically.
  • Phase 2 (Q2 2026): Integrate ML-based Face Matching for "Lost SIM" scenarios.
  • Phase 3 (Q4 2026): Transition to FIDO2 Passkeys as the root of identity.
