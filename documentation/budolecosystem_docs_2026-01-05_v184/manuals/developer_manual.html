Developer Manual: Image Rotation Persistence

Purpose

This implementation allows administrators to rotate KYC documents in the Admin Panel and have those rotations saved to the database. This ensures that once a document is correctly oriented, it remains so for all future reviews.

Technical Details

1. Prisma Schema Update

A new rotation field was added to the VerificationDocument model to store the rotation angle (0, 90, 180, or 270 degrees).

model VerificationDocument {
  id       String @id @default(uuid())
  // ... existing fields
  rotation Int    @default(0)
  // ...
}

2. API Implementation

The /api/users POST endpoint was updated with a new action: UPDATE_DOCUMENT_ROTATION.

if (action === "UPDATE_DOCUMENT_ROTATION") {
  const updatedDoc = await prisma.verificationDocument.update({
    where: { id: documentId },
    data: { rotation: rotation },
  });
  return NextResponse.json({ success: true, document: updatedDoc });
}

3. Frontend Integration

In apps/admin/app/users/page.tsx, the handleRotate function now performs an optimistic update of the UI state and then persists the new rotation to the database via the API.

const handleRotate = async (docId: string) => {
  const newRotation = ((rotations[docId] || 0) + 90) % 360;
  setRotations(prev => ({ ...prev, [docId]: newRotation }));
  await fetch("/api/users", {
    method: "POST",
    body: JSON.stringify({ action: "UPDATE_DOCUMENT_ROTATION", documentId: docId, rotation: newRotation }),
    headers: { "Content-Type": "application/json" }
  });
};

Issues & Fixes

  • Issue: Rotation state lost on page refresh.
  • Fix: Added database persistence and initialization logic when opening the document viewer.
  • Issue: Multiple rotations triggering many API calls.
  • Fix: Implemented optimistic UI updates to ensure responsiveness while the backend processes the change.

Dependencies

  • Prisma ORM for database operations.
  • Next.js App Router for API and UI.
  • Lucide-React for the rotation icon.
