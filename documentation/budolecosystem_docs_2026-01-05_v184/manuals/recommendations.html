Recommendations for Future Enhancements

Priority: Medium

1. Debounced API Calls

Reason: Rapid clicking of the rotate button currently sends an API request for every 90-degree turn.

Fix: Implement a debounce (e.g., 500ms) so that only the final rotation state is sent to the server.

Priority: High

2. Automatic Orientation Detection

Reason: Users often upload photos without checking orientation, leading to manual work for admins.

Enhancement: Use AI/ML models (e.g., Google Vision API or a custom TFLite model) to detect the correct orientation of ID cards and automatically set the initial rotation value.

Priority: Medium

3. Image Quality Scoring

Reason: Blurry or poorly lit photos often cause OCR failures.

Enhancement: Implement a quality score check on the mobile app before submission to ensure images are sharp and readable, reducing the need for re-submissions.

Priority: High

4. Rejection Reason & Notifications

Reason: As identified in v183, users are not notified when their KYC is rejected.

Fix: Add a rejectionReason field and integrate Firebase Cloud Messaging to notify users immediately with actionable feedback.
