SysAdmin Manual: Image Rotation Persistence

Deployment Procedures

1. Database Migration

The VerificationDocument table requires a new rotation column. Run the following command in the budolpay-0.1.0 directory:

npx prisma db push --schema=packages/database/prisma/schema.prisma

Or if using migrations:

npx prisma migrate dev --name add_document_rotation

2. Prisma Client Regeneration

After updating the schema, regenerate the Prisma client to include the new field in TypeScript types:

npx prisma generate --schema=packages/database/prisma/schema.prisma
Note: Ensure no Node.js processes (like the dev server) are running during regeneration to avoid EPERM errors on Windows.

Risk & Mitigation

  • Risk: Database migration failure due to existing data.
  • Mitigation: The rotation field has a default value of 0, ensuring compatibility with existing records.
  • Risk: High frequency of API calls if users click rotate rapidly.
  • Mitigation: Current implementation uses optimistic UI. Future enhancement could include debouncing the API call.

Test Results

  • Database Sync: Verified. Column rotation (Int, default 0) successfully added to VerificationDocument.
  • API Action: Verified. UPDATE_DOCUMENT_ROTATION successfully updates the database.
  • UI State: Verified. Image rotation persists after closing and reopening the modal.
