v195 System Admin Manual

Subject: Ecosystem Synchronization & Security Alignment

CRITICAL: All services in the budolEcosystem must now share the same JWT_SECRET to ensure cross-app authentication works correctly.

Environment Configuration

1. JWT Secret Alignment

Ensure the following services have the identical JWT_SECRET in their .env files or docker configurations:

  • budolID SSO Service
  • budolPay Auth Service

Active Secret: budolid-super-secret-key

2. budolID Configuration

The budolID service now requires PORT=8000 to be accessible by budolPay services for availability checks and user lookups.

3. Database Migrations

The budolID database schema has been updated to include phoneNumber in the User model. Ensure the latest Prisma schema is pushed to the database:

npx prisma db push

Operational Runbook

Checking Service Sync

To verify if the universal propagation is working, monitor the logs of the budolPay-Auth container:

docker logs budolpay-auth | grep "[Universal Sync]"

You should see entries like: [Universal Sync] Creating missing user in budolPay: user@example.com when a new budolID user logs in for the first time.

Troubleshooting Authentication

If users report "Invalid Token" errors across apps:

  1. Verify JWT_SECRET matches in all .env files.
  2. Restart services to apply environment changes.
  3. Check if the iss (issuer) in the token is set to budolID.
