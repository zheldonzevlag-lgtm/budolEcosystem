v195 Developer Manual

Subject: Universal Identity & Registration Architecture

Technical Architecture

The ecosystem uses budolID as the source of truth for all user identities. When a user registers via the budolPay mobile app, the account is created in the budolID database. Synchronization to budolPay is handled "lazily" via the /verify endpoint.

1. budolID: Centralized Registration

The /auth/register endpoint now accepts phoneNumber. This allows the ecosystem to support phone-based identity, which is critical for mobile payment platforms like GCash.

// budolID/index.js
app.post('/auth/register', async (req, res) => {
    const { email, password, firstName, lastName, phoneNumber } = req.body;
    // ... validation and hashing ...
    const user = await prisma.user.create({
        data: { email, passwordHash, firstName, lastName, phoneNumber }
    });
});

2. budolPay: Lazy Sync Implementation

Instead of complex real-time webhooks, budolPay synchronizes users the first time they interact with the service. The /verify endpoint in budolpay/services/auth-service/index.js checks if the user exists locally; if not, it clones the profile from the budolID token and initializes a wallet.

// budolpay/services/auth-service/index.js
if (!user && decoded.email) {
    user = await prisma.user.create({
        data: {
            id: decoded.sub,
            email: decoded.email,
            phoneNumber: decoded.phoneNumber,
            wallet: { create: { balance: 0.0, currency: 'PHP' } }
        }
    });
}

3. Flutter: 3-Step Registration Flow

The RegistrationScreen uses a PageView to handle the multi-step flow. Each step validates availability before proceeding to the next.

  • Step 1: checkAvailability(phoneNumber)
  • Step 2: checkAvailability(email)
  • Step 3: register(...)

Impact Analysis

  • Database: New phoneNumber field added to budolID User schema.
  • Security: All apps must now use the aligned JWT_SECRET to verify tokens across the ecosystem.
  • User Experience: Reduced friction by verifying availability in real-time before form submission.
