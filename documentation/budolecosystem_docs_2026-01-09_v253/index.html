v253: Pusher Integration & Realtime Provider Agnosticism

Date: 2026-01-09 Version: v253 Author: BudolEcosystem Team
Completed

Purpose

To eliminate the need for manual refreshes in the BudolPay mobile app by implementing a provider-agnostic realtime synchronization layer. This version introduces official Pusher support and allows the system to dynamically switch between SWR (Polling), Socket.io, and Pusher based on administrative settings.

Technical Implementation

1. Backend: Multi-Provider Event Emission

The API Gateway was updated to support dual-provider emission. It now triggers events via both Socket.io and Pusher simultaneously, ensuring all clients receive updates regardless of their chosen transport.

// api-gateway/index.js
if (isAdmin) {
    io.to('admin').emit(event, data);
    if (pusher) pusher.trigger('admin', event, data);
} else if (userId && event) {
    io.to(`user:${userId}`).emit(event, data);
    if (pusher) pusher.trigger(`user-${userId}`, event, data);
}

2. Mobile: Realtime Service Abstraction

Created a unified RealtimeService that abstracts the underlying provider logic. It automatically selects the best available provider (SWR, Socket.io, or Pusher) based on system settings fetched from the API.

  • PusherService: Handles Pusher channel subscriptions and event mapping.
  • SocketService: Maintains WebSocket connections.
  • SWR Fallback: Provides periodic polling when realtime services are unavailable.

Testing & Validation

Test Scenarios

  • Realtime Balance Update: Verified that performing a Cash In or Transfer on the web/another device instantly updates the mobile app balance.
  • Provider Switching: Confirmed that changing the realtime.method in the database (SWR, SOCKETIO, PUSHER) correctly updates the app behavior without a rebuild.
  • Error Resilience: Verified that the app handles connection drops gracefully and attempts reconnection.

Impact Analysis

  • UX: Users no longer need to manually refresh to see their current balance.
  • Scalability: Pusher integration allows the app to scale without putting additional load on the self-hosted Socket.io server.
  • Flexibility: The agnostic approach allows switching providers instantly if one experiences downtime or cost issues.

Recommendations for Future Enhancement

  • Push Notifications: Integrate Firebase Cloud Messaging (FCM) alongside Pusher for background notifications.
  • Latency Monitoring: Add telemetry to measure latency differences between Socket.io and Pusher.
  • Encrypted Channels: Implement Pusher Private Channels for enhanced security of sensitive transaction data.
