⚙️ System Admin Manual v494

Overview

Version v494 introduces "Filter-Aware Realtime Sync". As an administrator, you should be aware of how this impacts server-to-client communication.

Realtime Configuration

This update relies on the Admin-Configured Realtime Provider (Pusher, Socket.IO, or Polling) established in v493. No additional configuration is required in the Admin Panel.

Admin Note: The "Tab Sync" logic is client-side. It processes the standard order-updated events. If users report that tabs are not syncing, verify that the order-updated events are being fired by the backend webhooks.

Monitoring & Debugging

  • Browser Console: Look for [Realtime] Removing order ... from cache ... logs to verify that tab removals are working as intended.
  • SWR Cache: The system now forces revalidation (revalidate: true) after every realtime patch to ensure that the UI eventually settles to the exact database state, even if a socket event was slightly out of sync.
