🐞 Issues Log v494

I-494-01: Order Tab Sync Lag

RESOLVED

Description: When an order status was updated (e.g., from PAID to SHIPPED), the order would remain in the "To Ship" tab until a manual refresh, even though the status code changed.

Detected: Internal Testing / User Feedback.

I-494-02: Cryptic Notification Labels

RESOLVED

Description: Realtime notifications were showing technical status codes (e.g., "Order is now PROCESSING") instead of user-friendly labels (e.g., "Order is now being prepared").

Detected: UX Review.

I-494-03: Missing Return Lifecycle Notifications

RESOLVED

Description: Events for return-updated and dispute-resolved were not being listened to in RealtimeProvider.jsx, causing gaps in the return/refund communication flow.

Detected: Feature Gap Analysis.

I-494-04: Branding Mismatch in Notifications

RESOLVED

Description: Order status notifications were using the BudolPayText component for Order IDs, which clashed with the clean, professional design required for status updates. Additionally, toast styling was inconsistent with the brand's minimalist aesthetic.

Detected: Design Review (Image 2 Alignment).
