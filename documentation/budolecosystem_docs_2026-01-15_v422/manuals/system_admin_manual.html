Database Maintenance

To ensure the BudolShap ecosystem remains functional, the following administrative procedures should be followed after any major schema update or environment migration.

1. Verify Admin Status

Run the diagnostic script to check user roles and product counts:

node scripts/test_scripts/budolShap/diagnose_db.cjs

2. Restore Admin & Seed Data

If admin access is lost or products are missing, execute the restoration script:

node scripts/fix_admin_and_seed.cjs

3. Synchronization

Ensure Prisma client is generated after any changes:

npx prisma generate

Compliance Monitoring

Admin activities are logged for auditability (NPC/Cybersecurity Law compliance). Ensure all admin-level changes are performed through approved scripts or the Admin Dashboard.
