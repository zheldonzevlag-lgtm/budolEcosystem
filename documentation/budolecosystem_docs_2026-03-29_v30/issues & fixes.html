Issues & Fixes v30

Resolved Engineering Challenges

Issue: PII Address Leakage on Logout

When a user logged out, the browser retained authentication cookies and Redux state. If a new, unauthenticated session attempted to add an address, the `AddressModal` would pre-fill with the previous user's Personal Identifiable Information (PII).

Fix: Zero-Trust Form & Session Purging

Updated `clearAuth()` to forcefully expire auth cookies (`max-age=0`). Enforced `EMPTY_CONTACT` initial state for new addresses, and added `clearAddresses` in Redux to strictly isolate user environments.

Issue: Cart Page Infinite Loading Hang

Unauthenticated users (or sessions with cleared site data) experienced an infinite loading spinner on the Cart page because the Redux store initialized `isLoading: true` and never toggled it off if there was no active profile to fetch.

Fix: Lazy Loading Initialization

Changed initial `isLoading` state to `false` and modified `layout.jsx` to only initiate foreground loading spinners when an authenticated fetch sequence is actually triggered.

Issue: Vercel Deployments Failing to Build Prisma client

`payment-gateway-service` failed to deploy on Vercel with a `file or directory not found` error for `schema.prisma`. The root cause was the global `.gitignore` file hiding all `prisma/` directories from Git.

Fix: Gitignore Rule Removal

Removed the global `prisma/` ignore rule to track service schemas and migrations properly. Replaced it with rules specifically targeting `*.db` and `*.sqlite` files instead.
