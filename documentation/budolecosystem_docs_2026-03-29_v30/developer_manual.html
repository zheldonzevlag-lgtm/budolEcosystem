v30 Security & PII Isolation Architecture

Release v30 hardens the checkout process and enforces strict zero-trust data isolation across browser sessions to prevent PII leakage when users log out or clear site data.

Isolation Enforcements

  • Authentication Expiration: `clearAuth()` now explicitly sets authentication cookies (budolshap_token, budolshap_user) to `max-age=0` to prevent stale session hydration.
  • Redux State Purge: Added `clearAddresses` reducer to flush shipping data from application state upon logout via the `AuthContext`.
  • Form State Zero-Trust: Enforced `EMPTY_CONTACT` initial state for all new address creations to strictly prevent pre-filling PII from previous sessions.

File References & Explanations

BudolShap: auth-client.js

Implements `clearAuth()` to aggressively expire cookies alongside localStorage to guarantee isolated sessions upon logout. 

// Aggressive Cookie Expiration
const cookieOptions = "expires=Thu, 01 Jan 1970 00:00:00 GMT; max-age=0; path=/; SameSite=Lax";
document.cookie = `budolshap_token=; ${cookieOptions}`;
document.cookie = `budolshap_user=; ${cookieOptions}`;

BudolShap: addressSlice.js

Added `clearAddresses()` reducer triggered in `AuthContext` on logout to prevent state hydration leakage.

BudolShap: AddressModal.jsx

Enforces `EMPTY_CONTACT` for `mode === 'address'` to prevent the form from pre-filling cached PII for unauthenticated new session users.

await triggerRealtimeEvent('channel-name', 'event-name', {
    key: 'value'
});

Security & Performance Architecture

Backend Verification (SSO)

All credential checks (Email, Phone, Password) are performed synchronously on the BudolID server using Prisma and Bcrypt. This ensures that sensitive data never leaves the secure backend environment.

Real-time Notification

Socket.io and Pusher are used asynchronously only after verification is complete. Their sole purpose is to provide instant UI updates and ecosystem synchronization. Passwords and full PII are never transmitted.

NPC & BSP Compliance Note: All real-time payloads are masked or minimized to prevent accidental exposure of PII during transit via external providers.
