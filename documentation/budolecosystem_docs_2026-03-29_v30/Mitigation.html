Mitigation Plan v30

Zero-Trust Isolation & Build Controls

Strategies implemented to address the PII and unauthenticated access risks identified in the v30 Risk Register.

M-v30-001

PII Leakage Prevention

Primary Control: Aggressive cookie expiration upon logout (`max-age=0`) to obliterate session cache recovery potential.

Secondary Control: Synchronous `clearAddresses` and `clearCart` Redux dispatch to instantly purge the client-side state memory.

Tertiary Control: Form-level zero-trust via `EMPTY_CONTACT`, guaranteeing the `AddressModal` starts with an empty form for 'address' modes regardless of prior session states.

M-v30-003 & M-v30-004

State & Infrastructure Hardening

Cart Hang Mitigation: Implemented deferred state loading in Redux (`isLoading: false` by default). Active user check gating added to `layout.jsx` to prevent phantom fetches.

Vercel Build Protection: `.gitignore` audited and modified to prioritize `*_db` exclusion instead of blanket `prisma/` exclusion, ensuring robust database schema propagation to CI/CD pipelines.
