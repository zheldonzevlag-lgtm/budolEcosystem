Developer Manual Update v160

Purpose

This update documents the technical transition to a Vercel-Only cloud architecture, specifically removing all AWS dependencies (RDS/S3) and ensuring 100% compatibility with local development environments.

Technical Details

1. Vercel Serverless Refactoring

All microservices have been refactored to support Vercel's serverless function model while maintaining local Express server functionality.

// Pattern used in all services:
if (process.env.NODE_ENV !== 'production' || !process.env.VERCEL) {
    app.listen(PORT, () => {
        console.log(`Service running on port ${PORT}`);
    });
}
module.exports = app;

2. Database Strategy

The @budolpay/database package has been streamlined for Vercel Postgres:

  • Automatic Configuration: Detects POSTGRES_PRISMA_URL provided by Vercel.
  • AWS Removal: Removed RDS endpoint logic and AWS SDK dependencies.
  • Local Safety: Retains safety checks to prevent local development from accidentally connecting to production databases.

3. Verification Service (Blob Storage)

The Verification Service has been refactored to store documents directly in the PostgreSQL database as BYTEA blobs, removing the dependency on AWS S3.

  • Optimized Uploads: Images are resized and optimized using sharp before DB storage to keep database size manageable.
  • Vercel Compatibility: Fully supports multipart/form-data uploads within Vercel's 4.5MB payload limit.

Environment Variables

Local .env files remain untouched. For Vercel deployment, use the values defined in budolpay-0.1.0/.env.vercel.

Testing Results

Verified auth-service and verification-service locally to ensure the conditional app.listen logic works correctly without breaking the dev workflow.

Recommendations

  • Monitor Database Size: Since verification documents are stored in DB, monitor PostgreSQL storage limits.
  • Consider Vercel Blob: For future scaling, evaluate Vercel Blob as a native S3 alternative for document storage.
