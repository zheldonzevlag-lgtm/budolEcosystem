System Admin Manual Update v160

Infrastructure Overview

The system has been consolidated to run entirely on the Vercel Cloud Platform. All AWS services (EC2, RDS, S3) have been decommissioned from the current migration plan to simplify operations and reduce cross-cloud latency.

Deployment Configuration

1. Vercel Project Structure

Each service in budolpay-0.1.0/services/ is configured with a vercel.json file. When deploying, connect each service folder as a separate Vercel Project.

2. Environment Variables

Ensure the following variables are set in the Vercel Dashboard for each service:

POSTGRES_PRISMA_URL="[Vercel Postgres URL]" POSTGRES_URL_NON_POOLING="[Vercel Postgres Direct URL]" JWT_SECRET="[Your Secret]" AUTH_SERVICE_URL="https://[auth-project].vercel.app" WALLET_SERVICE_URL="https://[wallet-project].vercel.app" TRANSACTION_SERVICE_URL="https://[transaction-project].vercel.app" PAYMENT_GATEWAY_URL="https://[payment-project].vercel.app" VERIFICATION_SERVICE_URL="https://[verification-project].vercel.app"

Risks & Mitigations

⚠️ Database Growth

Issue: Verification documents (images) are now stored as BLOBS in the main database.

Risk: Fast database storage consumption leading to cost increases or performance degradation.

Mitigation: Sharp optimization is applied to all uploads. Admins should regularly archive old verification records or migrate to Vercel Blob if storage exceeds 1GB.

⚠️ Sockets & Serverless

Issue: Vercel Serverless Functions do not support persistent WebSockets.

Risk: Real-time notifications via api-gateway will not work on Vercel.

Mitigation: The system will fall back to HTTP polling. For full socket support, a single persistent instance (e.g., Vercel's upcoming persistent primitives or a small VPS) would be required for the Gateway only.

Operational Runbook

  1. Provisioning: Create a Vercel Postgres database and link it to all services.
  2. Build: Vercel automatically runs npm install and npm run build.
  3. Prisma: The postinstall script in package.json automatically runs prisma generate.
  4. Verification: Use the /health endpoint on each service to verify deployment success.
