Issues & Fixes: Phase 4 Implementation

Identified Technical Roadblocks

Issue 4.1: AWS Secrets Resolution Error

Symptom: ECS Task Events reported ResourceNotFoundException: Secrets Manager can't find the specified secret despite the secret existence in the account.

Root Cause: The task definition used short names (e.g., budol/db-url) which AWS could not resolve without the specific hash suffix (e.g., -NxLd8V) during the runtime execution role's secret retrieval phase.

Issue 4.2: Next.js Standalone Hostname Loop

Symptom: Budolshap logs showed ENOTFOUND or infinite redirect loops when starting in a Fargate container.

Root Cause: The environment was not specifying a binding address, causing Next.js to try to resolve the internal AWS Task ID as its hostname, which failed to map to a valid network interface.

Issue 4.3: Missing Database Schema

Symptom: Website loaded but showed "0 products found". Database queries confirmed connection but failed with The table public.Product does not exist.

Root Cause: The RDS instance was provisioned but the Prisma schema had not been pushed or migrated to the production endpoint.

Verification & Fix Implementation

Fix 4.1: Full ARN Standardization

Resolution: Updated the infrastructure automation script (register_tasks.py) to replace all secret references with their **Full Amazon Resource Names (ARNs)**.

# register_tasks.py snippet
if secret['name'] == 'DATABASE_URL':
    secret['valueFrom'] = 'arn:aws:secretsmanager:ap-southeast-1:194442925745:secret:budol/db-url-NxLd8V'

Fix 4.2: Global Interface Binding (0.0.0.0)

Resolution: Injected the HOSTNAME=0.0.0.0 environment variable via the ECS Task Definition to allow the service to listen on all IPv4 addresses within the container networking stack.

# Task Def Update
"environment": [
    {"name": "HOSTNAME", "value": "0.0.0.0"},
    {"name": "REDIS_URL", "value": "redis://127.0.0.1:6379"}
]

Fix 4.3: Manual Schema Push & Seed

Resolution: Executed a one-off ECS task to run npx prisma db push and node scripts/seed-test-data.cjs to initialize the production database tables and sample records.

# One-off task command override
sh -c "npx prisma db push --skip-generate && node scripts/seed-test-data.cjs"
