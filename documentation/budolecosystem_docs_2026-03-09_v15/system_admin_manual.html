System Admin Manual: Infrastructure Phase 4

ALB & Port 80-443 Redirects

To enforce PCI DSS and BSP security compliance, all insecure (Port 80) traffic is automatically redirected to Port 443.

Redirect Configuration

aws elbv2 create-rule \
    --listener-arn arn... \
    --priority 1 \
    --conditions Field=path-pattern,Values='/*' \
    --actions Type=redirect,RedirectConfig={Protocol=HTTPS,Port=443,StatusCode=HTTP_301}

ACM Certificate Management

SSL Certificates from DuckDNS are imported into ACM for full domain verification.

Admin Tip: Always renew DuckDNS certificates before expiration (every 90 days) and re-import them to ACM to maintain service continuity.

ECS Service Health Monitoring

Services are monitored via their Target Group status. A HEALTHY status indicates that the task has successfully completed its Node.js/Next.js startup and the ALB can start routing production traffic to it.

# Check Target Health
aws elbv2 describe-target-health --target-group-arn arn...

Troubleshooting Crashing Tasks

If a task is stuck in a PENDING → STOPPED loop:

  1. Check CloudWatch Logs immediately: aws logs tail /ecs/budol-shap --follow
  2. Look for ResourceNotFoundException in service events: aws ecs describe-services --cluster budol-ecosystem-cluster --services ...
  3. Verify Secrects Manager ARNs: Ensure the task definition matches the full ARN with suffixes.
Common Fix: If db-url or jwt-secret are not found, update the task definition to use the Full ARN from the Secrets Manager console.
