Future Recommendations: Infrastructure Stability

Short-Term Action Items (Phase 5)

Provision Redis Managed Instance (ElastiCache)

The current bypass (pointing to 127.0.0.1) is a temporary fix for service startup stability. Phase 5 should include the deployment of an Amazon ElastiCache (Redis) instance in the same private subnet as the backend services to restore session management and fast caching capabilities.

Standardized Environment Manager

Integrate env-manager.cjs into the ECS entrypoint to dynamically pull all secrets upon container boot, rather than relying solely on hardcoded task definition secret mapping.

Automated SSL Renewal Monitor

Implement a Lambda function to check DuckDNS SSL certificate expiration dates and auto-import new certificates into ACM every 60 days to prevent downtime.

Long-Term Enhancements

Production Domain & Route 53

Migrate from *.duckdns.org to a professional registered domain (e.g., budol-ecosystem.com) with Route 53 management for advanced health checking and failover routing.

WAF Integration (PCI DSS Compliance)

Attach an AWS WAF (Web Application Firewall) to the ALB to protect against SQL injection, cross-site scripting (XSS), and other common OWASP Top 10 web vulnerabilities.
