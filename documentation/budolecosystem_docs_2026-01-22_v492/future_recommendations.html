Proposed Enhancements

REC-492-001: WebSocket-Only Mode

Once the infrastructure is stable on AWS, consider disabling POLLING entirely to further reduce server load and latency. This requires ensuring 100% WebSocket availability.

REC-492-002: Edge Caching for Realtime Events

Implement a layer at the edge (e.g., Cloudflare Workers or Vercel Edge Functions) to filter and batch realtime events before they reach the client, further reducing bandwidth.

REC-492-003: Batch Realtime Events

For high-frequency events (like typing indicators or rapid status changes), implement server-side batching to send multiple events in a single WebSocket frame every 500ms.

REC-492-004: Adaptive Polling Intervals

Dynamically adjust polling intervals based on user activity levels (e.g., mouse movement, scroll position) rather than just tab visibility, to further save costs for idle users.
