Problems Identified During Realtime Optimization

Identified Issues

ISSUE-492-001: UI Flickering in "My Orders"

The use of router.refresh() in the realtime listener caused the entire page to re-render and show loading states, leading to a poor user experience.

ISSUE-492-002: Excessive Chat Polling

The Chat system was using a fixed 3-second interval for fetching messages, which consumes significant bandwidth even when no new messages are sent.

ISSUE-492-003: Stale Dashboard Stats

The Seller Dashboard stats (revenue, order counts) did not update in realtime when an order was placed or updated, requiring a manual page refresh.

ISSUE-492-004: Cloud Hosting Cost Risk

High-frequency polling (every 3-5s) across multiple tabs could lead to thousands of API calls per hour per user, increasing AWS/Vercel costs.
