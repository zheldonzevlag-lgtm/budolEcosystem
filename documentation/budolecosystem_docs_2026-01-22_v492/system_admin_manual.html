1. Realtime Provider Configuration

Budol Ecosystem supports three realtime modes. Admins can configure the active provider via the Admin UI Settings or Environment Variables.

Provider Description Best For
PUSHER Cloud-hosted managed WebSocket service. Scalability, Low latency, High reliability.
SOCKET_IO Self-hosted WebSocket server. Zero external service costs, Data privacy.
POLLING Fallback mechanism using HTTP requests. Low-traffic sites, simple environments.
Cost Notice: When using POLLING on cloud providers like AWS or Vercel, bandwidth usage is higher. Version 492 implements "Visibility-Aware Polling" which reduces costs by slowing down requests when the user is not actively viewing the tab.

2. Visibility-Aware Polling

To optimize bandwidth, the system automatically adjusts polling intervals based on browser tab visibility:

  • Active Tab: 5 seconds interval.
  • Hidden Tab: 30 seconds interval (83% bandwidth reduction).

This is handled globally by the RealtimeProvider component.

3. Environment Variables

Ensure the following variables are correctly set for Pusher integration:

NEXT_PUBLIC_PUSHER_KEY=your_key
NEXT_PUBLIC_PUSHER_CLUSTER=your_cluster
PUSHER_APP_ID=your_id
PUSHER_SECRET=your_secret
