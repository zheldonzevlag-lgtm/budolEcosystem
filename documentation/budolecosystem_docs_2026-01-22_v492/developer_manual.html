1. SWR Cache Patching

To eliminate UI flickering (caused by router.refresh()), we implemented targeted cache patching. Instead of re-fetching the entire page, we manually update the SWR cache when a realtime event is received.

Core Implementation

File: RealtimeProvider.jsx

const patchOrderCache = (data, forceRevalidate = false) => {
    const orderId = data.orderId || data.id;
    const newStatus = data.status;

    if (!orderId || !newStatus) return;

    mutate(
        (key) => typeof key === 'string' && (key.startsWith('/api/orders') || key.startsWith('/api/dashboard/seller/orders')),
        (currentData) => {
            if (!currentData || !currentData.orders) return currentData;

            const updatedOrders = currentData.orders.map(order => {
                if (order.id === orderId) {
                    return { ...order, status: newStatus };
                }
                return order;
            });

            return { ...currentData, orders: updatedOrders };
        },
        { revalidate: forceRevalidate }
    );
};

2. Event-Driven Chat Updates

We replaced the 3-second message polling with a realtime event-driven architecture. The backend now triggers a message-received event upon message creation.

Backend Trigger

File: route.js

await triggerRealtimeEvent(`chat-${id}`, 'message-received', {
    messageId: message.id,
    chatId: id,
    senderId: user.id,
    content: message.content,
    sender: message.sender
});

Frontend Listener

The RealtimeProvider.jsx listens for these events and mutates the specific chat cache:

if (event === 'message-received') {
    if (data.chatId) {
        mutate(`/api/chats/${data.chatId}/messages`);
    }
}

3. Cloud Cost & Bandwidth Optimization

For hosting on AWS/Vercel, reducing API calls is critical. We implemented several strategies:

  • Tab Visibility Guard: Polling intervals increase from 5s to 30s when the browser tab is hidden.
  • SWR Deduplication: Dashboard stats have a 10s deduplication interval to prevent redundant fetches.
  • Minimal Payloads: Realtime events carry only the necessary data (IDs and status) to keep bandwidth low.
  • Revalidation Control: mutate is called with revalidate: false by default when the payload is sufficient.

File: RealtimeProvider.jsx (Visibility Logic)

const handleVisibilityChange = () => {
    const isHidden = document.visibilityState === 'hidden';
    if (provider === 'POLLING') {
        setSwrOptions(prev => ({ 
            ...prev, 
            refreshInterval: isHidden ? 30000 : 5000 
        }));
    }
};
