Mitigation Strategies

M-492-001: Bandwidth Optimization (for R-492-001)

Implemented visibility-aware polling (30s interval for hidden tabs) and SWR deduplication (10s window). This significantly reduces API calls on AWS/Vercel.

M-492-002: SWR Hybrid Strategy (for R-492-002 & R-492-004)

Realtime events trigger a targeted SWR mutate(). If the event payload is missing data, SWR is configured to revalidate (re-fetch) in the background to ensure consistency.

M-492-003: Multi-Provider Fallback (for R-492-003)

The system allows admins to switch between Pusher (Cloud), Socket.IO (Self-hosted), and Polling (Stateless) without code changes, providing an exit strategy for service limits.

M-492-004: Event Deduplication

The RealtimeProvider implements a 5-second rapid-fire suppression window to prevent redundant UI updates from the same event source.
