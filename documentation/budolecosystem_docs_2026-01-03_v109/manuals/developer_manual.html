Branding Implementation

We used the following tools to automate the branding process:

  • flutter_launcher_icons: For generating multi-resolution app icons from a single high-res source.
  • flutter_native_splash: For creating a seamless native splash screen experience on Android and iOS.

Configuration (pubspec.yaml)

flutter_launcher_icons:
  android: true
  ios: true
  image_path: "assets/icon/app_icon.png"
  min_sdk_android: 21

flutter_native_splash:
  color: "#0F172A"
  image: "assets/icon/splash_icon.png"
  android: true
  ios: true
  fullscreen: true

Release Build Process

To generate the production APK, we used the following command to avoid resource-intensive icon tree shaking which was causing OOM (Out of Memory) errors:

flutter build apk --release --no-tree-shake-icons

Assets Information

The original assets were sourced from C:\Users\Administrator\Pictures\budolPay_app_icons. The 512x512 resolution was used as the master source for both launcher icons and the splash screen.
