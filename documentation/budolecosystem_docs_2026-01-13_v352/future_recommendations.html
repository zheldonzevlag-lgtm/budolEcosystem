Future Recommendations - v352

Strategic Improvements for Onboarding and Marketing

← Back to Index

1. Version-Based Ad Reset

Instead of a simple boolean hasSeenAds, consider storing the last version seen (e.g., lastSeenAdsVersion = "1.2.61"). This would allow marketing to re-trigger ads automatically when major new features are introduced in future versions.

2. Dynamic Marketing Content

Currently, the ads are hardcoded in MarketingAdsScreen. Recommending a migration to a remote configuration service (e.g., Firebase Remote Config or a custom API) so that marketing slides can be updated without requiring a new app store submission.

3. A/B Testing

Implement A/B testing on the "Skip" vs "Get Started" buttons to optimize user conversion and retention during the onboarding phase.
