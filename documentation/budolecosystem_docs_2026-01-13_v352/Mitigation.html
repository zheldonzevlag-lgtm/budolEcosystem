Mitigation Strategy - v352

Proactive Measures for Identified Risks

← Back to Index

Mitigation for R-352-01 (UX Friction)

The "GET STARTED" button is prominently displayed and allows users to bypass the ads with a single tap, returning them immediately to their dashboard if authenticated.

Mitigation for R-352-02 (Storage Failure)

ApiService includes error handling for SharedPreferences initialization. If it fails, the app defaults to showing ads to ensure marketing compliance, which is preferred over missing them.

Mitigation for R-352-03 (Navigation Loop)

The _proceedToApp method in MarketingAdsScreen uses Navigator.pushReplacementNamed, which clears the current screen from the stack. Even if the persistent flag fails to write, the user can still proceed to the app during the current session.
