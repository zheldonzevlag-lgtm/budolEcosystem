1. Navigation Logic Refactoring

The core issue was the prioritization of the authentication check over the marketing ads check in the SplashScreen. This caused authenticated users to skip the marketing ads entirely.

File: splash_screen.dart

The _navigateToNext method was updated to check hasSeenAds first.


// PREVIOUS LOGIC (v351 and earlier)
if (isAuthenticated) {
  Navigator.pushReplacementNamed(context, Routes.home);
} else if (!hasSeenAds) {
  Navigator.pushReplacementNamed(context, Routes.marketing);
} else {
  Navigator.pushReplacementNamed(context, Routes.login);
}

// UPDATED LOGIC (v352)
if (!hasSeenAds) {
  // Priority: Ensure all users see marketing ads once
  Navigator.pushReplacementNamed(context, Routes.marketing);
} else if (isAuthenticated) {
  Navigator.pushReplacementNamed(context, Routes.home);
} else {
  Navigator.pushReplacementNamed(context, Routes.login);
}

3. Web Splash Removal

On Flutter Web, the native splash screen is removed manually via JavaScript to ensure a smooth transition to the Flutter UI.

File: index.html


function removeSplashFromWeb() {
  document.getElementById("splash")?.remove();
  document.getElementById("splash-branding")?.remove();
  document.body.style.backgroundColor = "#0F172A";
}
