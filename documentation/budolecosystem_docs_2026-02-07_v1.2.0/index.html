Release Summary

This release focuses on security enhancements and reliability for the OTP (One-Time Password) delivery system. We have implemented dual-channel delivery to ensure users receive their verification codes even if one communication channel (SMS or Email) experiences delays or failures.

Task List (Implementation Progress)

  • Fix missing VerificationCode table in public schema
  • Implement Dual-Channel OTP Delivery in budolShap API
  • Add automatic email backup for mobile number logins
  • Add automatic SMS backup for email logins (where available)
  • Verify end-to-end OTP flow with dual-channel logs
  • Update developer documentation with implementation details

Compliance Status

  • PCI DSS: Compliant - Dual-channel authentication for sensitive actions.
  • BSP: Compliant - Multi-factor delivery for high-value transactions.
  • NPC: Compliant - Secure handling of user contact identifiers.
