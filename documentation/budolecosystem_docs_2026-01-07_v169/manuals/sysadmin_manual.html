Unified Dashboard Management

This manual details how to achieve the "One Card" vision in the Vercel dashboard by treating the entire ecosystem as a single monorepo.

Monorepo Mode

Dashboard Goal: One Project Card

Instead of 11 separate cards, your dashboard will only show budolecosystem. When you click it, you are managing the entire ecosystem at once.

  • Current View: 11 cards (Confusing and cluttered)
  • Target View: 1 card (Clean and professional)

Technical Implementation

We have updated the following files to signal to Vercel that this is a single unified monorepo:

  • Root package.json: Added "workspaces" field to link all service folders.
  • Root vercel.json: Added "framework": null to ensure Vercel treats the root as the primary entry point.

Dashboard Cleanup Steps

To finalize the "One Card" look, please manually delete the 11 redundant projects from your Vercel dashboard. Keep only the budolecosystem project.

Once deleted, the only project you will see is the one that actually contains all your code and routing logic.
