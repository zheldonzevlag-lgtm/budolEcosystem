SSO Button Redundancy Resolution

This manual covers the resolution of the duplicate "Login with budolID" buttons on the BudolShap login page.

Technical Explanation

The redundancy was caused by a design overlap between the AuthForm component and the LoginPage. Both files contained logic and UI for the SSO button.

Changes

  1. AuthForm Update: The button in AuthForm.jsx was converted from an <a> tag to a <button> to handle dynamic redirect_uri construction in an onClick handler.
  2. LoginPage Cleanup: The hardcoded SSO section was removed from page.jsx, relying solely on the AuthForm component.

Code Implementation

// AuthForm construction of redirect URI const ssoUrl = process.env.NEXT_PUBLIC_SSO_URL || (typeof window !== 'undefined' ? `http://${window.location.hostname}:8000` : 'http://localhost:8000'); const appUrl = process.env.NEXT_PUBLIC_APP_URL || (typeof window !== 'undefined' ? window.location.origin : ''); const redirectUri = `${appUrl}/auth/callback`; const apiKey = process.env.NEXT_PUBLIC_BUDOLID_API_KEY || 'bs_key_2025'; window.location.href = `${ssoUrl}/login?apiKey=${apiKey}&redirect_uri=${encodeURIComponent(redirectUri)}`;
