SysAdmin Manual (v18 - Recovery Ops)

CRITICAL: Ensure all RDS snapshots are enabled before deleting any production instance in the future.

Managing the RDS Production DB

1. RDS Identifier

Instance: budol-db-production (PostgreSQL 17.4)

2. Database Endpoints

Host: budol-db-production.cnu6imkq689x.ap-southeast-1.rds.amazonaws.com

Port: 5432

3. Connection Management

Database credentials are stored in **AWS Secrets Manager**:

  • budol/db-url (BudolShap)
  • budol/id-db-url (BudolID, BudolPay Services)

Operational Scripts

Forced Redeployment

aws ecs update-service --cluster budol-ecosystem-cluster --service budol-shap-service --force-new-deployment --region ap-southeast-1

Status Verification

aws rds describe-db-instances --db-instance-identifier budol-db-production --query "DBInstances[0].DBInstanceStatus"

Database Backups

Backups are stored in **S3**: s3://budol-ecosystem-backups/db-backup/YYYY-MM-DD/

Restoration must be done via the budol-mgmt-host using psql.
