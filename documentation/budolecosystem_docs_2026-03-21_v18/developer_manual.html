Developer Manual (v18 - Database Recovery)

Infrastructure Changes

The system now uses a freshly provisioned RDS PostgreSQL 17.4. All DATABASE_URL environmental variables are managed via AWS Secrets Manager. Developers must not hardcode database endpoints.

How to Restore DB (Dev/Stage)

To restore databases locally for development, use the following bash script:

# Restore to Local PostgreSQL Container cat budolshap_1db.sql | docker exec -i postgres_container psql -U user -d budolshap_1db

View Recovery Script

Schema Verification

For BudolShap (v17+), the schema now strictly uses SystemSettings (plural) for the main configuration table, whereas v16 used SystemSetting. Always use the latest generated Prisma Client.

// Example Query const settings = await prisma.systemSettings.findFirst();

  • budol/id-db-url (BudolID, BudolAuth, BudolPay Services)
