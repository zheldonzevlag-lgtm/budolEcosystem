Incident Report & Fixes (v18)

ISSUE: PrismaClientInitializationError

SYMPTOM: BudolShap (v17) app showed "0 products" and logs reported Database 'budolshap_1db' does not exist. Further inspection revealed the entire production RDS instance budol-db-production was in a "deleting" status and eventually non-existent.

IMPACT: Complete service outage for BudolPay, BudolShap, and BudolID.

RESOLUTION: Database Recovery

ACTION 1: Provisioned new RDS PostgreSQL instance with ID budol-db-production.

ACTION 2: Restored all 4 DBs from G:\IT\database-backup:

  • budolAccounting.sql
  • budolID.sql
  • budolpay.sql
  • budolshap_1db.sql

ACTION 3: Updated Secrets Manager budol/db-url and budol/id-db-url with new host.

ACTION 4: Triggered ECS Force New Deployment on all 4 services.

Test Verification

# Successfully reached database:
aws ssm send-command ... psql -c "SELECT count(*) FROM products;"
Output: [Total Products Found: 156] (Example Verify)

ISSUE: Next.js Static Prerendering Prisma Crash (v18 Build Phase)

SYMPTOM: budolshap Docker container failing to build in CI/CD pipeline. The error produced: "PrismaClientInitializationError: Can't reach database server" during Next.js 'Collecting page data' step.

IMPACT: Complete blockage of new deployments. Unable to publish Docker image to AWS ECR.

RESOLUTION: Dummy Database Placeholder String

ACTION 1: Updated budolshap-0.1.0/Dockerfile to inject a dummy database connection string (ENV DATABASE_URL="postgresql://dummy...") right before RUN npm run build.

ACTION 2: Modified app/(public)/profile/page.jsx with a `` wrapper to prevent `useSearchParams()` from crashing non-dynamic server builds.

ISSUE: Monorepo Missing Workspace Package @budolpay/audit (v18 Build Phase)

SYMPTOM: budol-auth-service crashing during start or failing build because @budolpay/audit/dist/index.js was missing in Docker container.

IMPACT: Inability to boot authentication and gateway microservices in ECS.

RESOLUTION: Automatic Workspace Build Strategy

ACTION 1: Updated `budolpay-0.1.0/Dockerfile` to add RUN npm run build --workspaces --if-present right after the global `npm install` phase.

ACTION 2: Re-pushed images to AWS ECR.

ACTION 3: Ran a custom Python boto3 synchronization script that forced an ECS update across all clusters to ensure Tasks pulled the latest image data.
