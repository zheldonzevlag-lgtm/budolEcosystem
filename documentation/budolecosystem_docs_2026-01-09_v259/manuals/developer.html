Developer Manual Updates - v259

1. Purpose & Reason

The primary goal was to fix the broken real-time notification loop in the budolPay ecosystem. Microservices were failing to communicate with the Gateway due to environment variable collisions (Vercel URLs overriding local settings), and Pusher events were not being triggered correctly for balance updates.

2. Technical Implementation Details

Environment Variable Override

We discovered that dotenv was not overriding existing environment variables injected by the OS or previous runs. We enforced the override globally:

require('dotenv').config({ 
    path: path.resolve(__dirname, '../../.env'), 
    override: true 
});

Pusher Event Flow

The Gateway now correctly initializes Pusher with fallbacks from the database or environment. The /internal/notify endpoint triggers both Socket.io and Pusher events:

pusher.trigger(`user-${userId}`, event, data);

3. Code Details

  • Gateway: Modified index.js to handle asynchronous Pusher initialization and environment overrides.
  • Wallet Service: Ensured notifyBalanceUpdate sends correctly formatted payloads to the Gateway.
  • Mobile App: Updated PusherService to handle balance_update events and stream them to the UI.
