Developer Manual: Dynamic Notification Providers

Version: 2026-01-01_v38

Overview

The system now supports dynamic switching between multiple Email and SMS providers through the Admin Dashboard. This replaces hardcoded environment variable dependencies with database-driven configuration.

1. Database Schema Updates

The SystemSettings model in schema.prisma has been extended with provider-specific fields:

// Email
emailProvider: String (GOOGLE, BREVO)
smtpHost, smtpPort, smtpUser, smtpPass, smtpFrom
brevoApiKey

// SMS
smsProvider: String (CONSOLE, ZERIX, ITEXTMO, VIBER)
zerixApiKey, itextmoApiKey, itextmoClientCode, viberApiKey

2. Email Service Logic

Modified lib/email.js to use getSystemSettings(). The createTransporter function now dynamically constructs the nodemailer instance based on the active provider.

  • GOOGLE/SMTP: Uses standard SMTP configuration.
  • BREVO: Uses smtp-relay.brevo.com with the provided API key as the password.

3. SMS Service Logic

Modified lib/sms.js to support provider routing. The sendOTPSMS function checks the smsProvider setting and executes the corresponding provider logic (Zerix, iTextMo, Viber, or Console fallback).

4. Admin Settings Integration

A new settings page has been created at /admin/settings/notifications to allow admins to:

  • Switch active providers in real-time.
  • Configure API keys and credentials.
  • Preview provider-specific descriptions and requirements.

Implementation Note

Providers like Zerix and iTextMo are currently implemented with placeholders in lib/sms.js. Developers should add the specific HTTP fetch calls once API documentation for these providers is finalized.
