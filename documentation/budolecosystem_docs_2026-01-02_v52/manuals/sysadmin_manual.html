System Admin Manual - v52

Release Date: 2026-01-02

Payment Gateway Configuration

Local Mock Fallback

The Payment Gateway service (port 8004) is now configured to automatically fall back to a local mock payment flow if the PayMongo API is unreachable or returns an error during development.

  • Requirement: Ensure budolShap is running on http://localhost:3001.
  • Behavior: Users will be redirected to http://localhost:3001/payment/qr instead of receiving a 404 error from PayMongo.
Security Note: The mock fallback is currently enabled for all environments where NODE_ENV is not production. For production deployments, ensure PAYMONGO_SECRET_KEY is correctly set to disable this fallback.

SSO Synchronization Validation

budolPay Admin Sync

Bi-directional sync is now active for budolPay Admin. To verify the link between a budolPay user and their budolID identity:

# Check kycData field for ssoUserId
# Connect to budolPay database
SELECT id, email, "kycData" FROM "User" WHERE email = 'admin@budolpay.com';

The kycData field should contain a JSON object with an ssoUserId key. This key is used to push profile updates back to budolID.

Maintenance

Service Verification

If payment method errors persist ("invalid payment_method"), ensure that the budolshap-0.1.0 service has been restarted to pick up the normalization logic in OrderSummary.jsx.

Port Audit: Confirm that port 8004 is not being blocked by a firewall, as it is the primary communication channel for all payment intents.
