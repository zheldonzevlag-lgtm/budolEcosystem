Mitigation Plan - v497

Technical Mitigations

SWR Revalidation Fallback: To mitigate R-497-01 (WebSocket failure), the system uses SWR's refreshInterval as a background polling fallback if the socket disconnects.
Prisma Optimistic Locking: To address R-497-02 (Race conditions), database updates use version-based optimistic locking to prevent overlapping status changes.

Financial Mitigations

Atomic Transactions: To prevent R-497-03 (Escrow lock failure), the return request and fund lock are wrapped in a single database transaction. If the lock fails, the return request is rolled back.
