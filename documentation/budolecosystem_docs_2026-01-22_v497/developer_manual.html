1. Backend Architecture

The return request lifecycle is managed via Prisma transactions to ensure atomicity between order status updates and escrow fund locking.

Key File: returnsService.js

// Transactional Return Creation
export async function createReturnRequest({ orderId, userId, reason, type, refundAmount, images }) {
    return await prisma.$transaction(async (tx) => {
        const order = await tx.order.findUnique({ where: { id: orderId } });
        // 1. Create Return Record
        const returnRequest = await tx.orderReturn.create({ data: { ... } });
        // 2. Update Order Status
        await tx.order.update({
            where: { id: orderId },
            data: { status: 'RETURN_REQUESTED' }
        });
        // 3. Trigger Realtime Event
        await triggerRealtimeEvent('order-status-update', { orderId, status: 'RETURN_REQUESTED' });
        return returnRequest;
    });
}

2. Frontend Realtime Engine

The RealtimeProvider handles cache patching for SWR. It intelligently routes orders between dashboard tabs (To Ship, Shipped, Return) based on the incoming status update.

Key File: RealtimeProvider.jsx

// Intelligent Cache Patching
function patchOrderCache(currentData, updatedOrder) {
    if (!currentData) return currentData;
    
    const { id, status: newStatus } = updatedOrder;
    const url = new URL(key, 'http://dummy.com');
    const statusFilter = url.searchParams.get('status');
    
    if (statusFilter) {
        const allowedStatuses = statusFilter.split(',');
        const shouldBeInList = allowedStatuses.includes(newStatus);
        
        if (!shouldBeInList) {
            // Remove from current tab if status no longer matches
            return currentData.filter(o => o.id !== id);
        }
    }
    // Update existing item or trigger revalidation
    return currentData.map(o => o.id === id ? { ...o, ...updatedOrder } : o);
}

4. Dependency Analysis & Rationale (Phase 8)

The following software and libraries were selected for the v497 release based on their reliability and ecosystem compatibility.

Dependency Choice Rationale
Next.js (App Router) Enables efficient server-side rendering and API route management for the seller dashboard.
SWR (Stale-While-Revalidate) Chosen for its powerful cache-patching capabilities, allowing for instant UI updates during return status transitions.
Prisma ORM Provides robust transactional support ($transaction) critical for atomic return and refund operations.
Pusher / Socket.io Selected as the primary and fallback realtime engines to ensure high availability of status updates across diverse network conditions.
