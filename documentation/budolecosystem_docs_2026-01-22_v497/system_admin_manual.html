System Admin Manual - v497

Technical configuration and monitoring guide for the realtime infrastructure.

1. Infrastructure Overview

The system uses a hybrid realtime strategy: Pusher for primary delivery and Socket.io as a self-hosted fallback for high-traffic events.

2. Environment Configuration

Ensure the following environment variables are correctly set in the production environment:

PUSHER_APP_ID="your_app_id"
PUSHER_KEY="your_key"
PUSHER_SECRET="your_secret"
PUSHER_CLUSTER="ap1"
SOCKET_IO_URL="https://socket.budolecosystem.com"

3. Monitoring & Troubleshooting

Realtime Health Check: Admins can monitor event delivery latency via the /admin/realtime-health dashboard.

Common Issues:

  • Issue: Dashboard not updating in realtime.
  • Fix: Check if the browser tab is active. The system reduces polling frequency or disconnects idle sockets after 5 minutes of inactivity to save resources.
  • Issue: Duplicate toasts for the same event.
  • Fix: The system has built-in deduplication based on event fingerprint. Check if multiple providers (Pusher & Socket.io) are active simultaneously in settings.
