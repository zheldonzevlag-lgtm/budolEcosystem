Developer Manual - BudolEcosystem v5.0.4

Release Date: 2026-03-02

Status: Alignment & Regression Fix

v5.0.4 Bug Fixes Task List

✔ Fix PrismaClientValidationError: Unknown arguments failedAt, paidAt, expiredAt in checkoutService.js
✔ Re-apply budolpay schema isolation (?schema=budolpay) to all .env files
✔ Move application-specific timestamps (failedAt, paidAt, etc) to metadata JSON

Purpose

This release corrects several regressions where environment variables were reverted and fixes a schema-model mismatch in the checkout process.

Technical Explanation

1. Schema Alignment (Checkout Model)

The Checkout model in the budolShap schema does not natively support failedAt, paidAt, or expiredAt fields. Attempting to update these directly caused PrismaClientValidationError.

Solution: All dynamic timestamps are now moved into the metadata JSON field, maintaining compatibility with the existing schema while preserving the data.

File: lib/services/checkoutService.js

2. Database Schema Isolation (budolPay)

Regression fix for budolPay microservices. All services were pointing back to the public schema, which lacks the Transaction.status field required by the Gateway.

Fix: Re-synchronized 6+ microservices to use the budolpay PostgreSQL schema.

Changes Summary

  • Refactored markCheckoutAsPaid, expireCheckoutSession, and markCheckoutAsFailed to use metadata for custom timestamps.
  • Corrected DATABASE_URL in root, database, and service .env files.

Compliance & Risk

Compliance: BSP Circular No. 808 - Ensured financial transaction data integrity by correctly isolating the budolPay schema.

Risk: Low. Re-deployment of services is required to pick up ENV changes.
