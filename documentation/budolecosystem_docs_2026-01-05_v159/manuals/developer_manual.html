Developer Manual - v159

Topic: Connectivity Debugging & Mobile Resilience

1. Purpose

To resolve intermittent connectivity issues where some mobile devices experience timeouts while others work fine on the same network. This update adds diagnostic tools to the app to help identify the root cause (Firewall, AP Isolation, or Cleartext restrictions).

2. Implementation Details

A. Connectivity Test Tool (ApiService.dart)

Added checkConnectivity() method to probe both port 8000 (Auth/SSO) and port 8080 (Gateway). This uses a short 3-second timeout to quickly provide feedback.

Future<Map<String, bool>> checkConnectivity() async {
    // Probes both ports using http.get
}

B. Android Cleartext Support

Modified AndroidManifest.xml to explicitly allow cleartext traffic. While usually handled by default in debug builds, explicit configuration ensures compatibility across more Android versions when using local IP addresses (HTTP).

<application ... android:usesCleartextTraffic="true">

C. Network Settings UI

Updated the Network Settings dialog in login_screen.dart to include a "Test Connection" button. This allows users to verify server reachability before attempting login.

3. Troubleshooting Connectivity Issues

Common Root Causes:
  • Windows Firewall: Ensure node.exe is allowed to receive incoming connections on Private networks.
  • AP Isolation: Some routers block device-to-device communication on Guest Wi-Fi networks.
  • VPN/DNS: Active VPNs or Private DNS (like AdGuard) can block local IP routing.
