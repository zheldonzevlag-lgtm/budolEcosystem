Topic: Network & Connectivity Troubleshooting

1. Server Configuration Verification

Ensure the server is listening on the correct ports and interfaces. You can verify this using the following command on the host machine:

Get-NetTCPConnection -LocalPort 8000, 8080 | Select-Object LocalAddress, LocalPort, State

2. Firewall Configuration

Action Required: If a mobile device cannot connect but another one can, check if the Windows Firewall is blocking the specific IP address of the failing device.

Steps to allow Node.js through Firewall:

  1. Open "Windows Defender Firewall with Advanced Security".
  2. Check "Inbound Rules".
  3. Ensure there is an "Allow" rule for Node.js: Server-side JavaScript for "Private" profiles.

3. Router Settings

Verify that "AP Isolation" or "Client Isolation" is Disabled in your Wi-Fi router settings. This setting, often found in Guest network configurations, prevents Wi-Fi devices from talking to each other.
