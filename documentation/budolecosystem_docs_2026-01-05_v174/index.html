KYC Enhancements & Release 1.2.0

v174

Purpose & Rationale

This phase focused on enhancing the KYC verification process in BudolPay by adding robust DOB selection, address pinning with map integration, and GCash-compliant phone number change procedures. It also included the preparation for the 1.2.0 release to ensure the codebase meets quality standards.

Build Notice: The automated APK generation encountered a JVM memory allocation failure (OOM) due to environment resource constraints. However, all code changes have been successfully implemented, verified with flutter analyze, and are ready for compilation in a standard build environment.

Key Implementation Details

KYC & Mobile

  • DOB Format: Switched to MM-dd-yyyy with hyphen separators and integrated showDatePicker.
  • Address Pinning: Integrated MapPicker (from v173) into the KYC flow for precise coordinate selection.
  • Phone Migration: Implemented GCash-compliant "Change Phone Number" screen with migration guidance.
  • Versioning: Bumped BudolPay to 1.2.0+1.

Admin Console

  • KYC Review: Enhanced admin interface to display submitted user documents (ID photos, selfies) for verification.
  • Map Alignment: Ensured admin map settings are consumed by the mobile app's MapPicker.
  • API Consistency: Updated Prisma schemas and routes to support extended KYC data.

GCash Phone Change Analysis

Process: GCash requires a migration-based approach where the user "migrates" their account to a new number via the app settings, which involves OTP verification on both old and new numbers (if accessible) or identity re-verification.

BudolPay Strategy: We've implemented a similar secure migration screen that guides the user through the identity verification required before allowing a phone number update, ensuring parity with industry security standards.

Testing & Validation

PASSED Static Analysis: flutter analyze returned zero errors for BudolPay 1.2.0.

PASSED Date Selection: Verified intl date formatting and calendar picker interaction.

PASSED Map Integration: Verified that mobile MapPicker successfully retrieves admin settings.

BLOCKED Release Build: JVM Crash (OOM) during assembleRelease. Code is clean but build failed due to local RAM limits.

Risks & Mitigation

Risk: Users without technical knowledge might find the coordinate-based address pinning confusing.

Mitigation: Added clear SnackBar feedback and maintained the text-based address field as a fallback/display.

Future Recommendations

  • Biometric Auth: Implement FIDO2/WebAuthn for passwordless login as per industry trends.
  • OCR Integration: Automate data extraction from ID cards using ML Kit to reduce manual entry.
  • CI/CD: Move builds to a cloud environment (e.g., GitHub Actions) to avoid local memory constraints.
