Purpose

This update modifies the navigation logic from the Favorites screen to the Send Money screen to prioritize the recipient's phone number over their email address when populating the input field.

Technical Details

The change was implemented in favorites_screen.dart. The initialRecipient parameter passed to the SendMoneyScreen constructor was updated to check for a phone number first.

Code Changes

File: lib/screens/favorites_screen.dart

// Old logic
initialRecipient: email.isNotEmpty ? email : phone,

// New logic (v262)
initialRecipient: phone.isNotEmpty ? phone : email,

Reasoning

Users often prefer identifying recipients by phone number in mobile P2P applications. Prioritizing the phone number aligns with user expectations for mobile-first payment platforms.

Impact Analysis

  • Favorites Screen: Tapping a favorite or the send icon now passes the phone number if available.
  • Send Money Screen: The recipient field will now show the phone number instead of the email if both are present in the favorite's record.

Test Results

COMPLETED
