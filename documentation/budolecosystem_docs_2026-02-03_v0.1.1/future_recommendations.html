Future Recommendations & Roadmap

1. Webhook Signatures for Realtime Events

Implement HMAC signing for all events triggered via /internal/notify to ensure that only authorized services can broadcast events to the frontend.

2. Realtime Health Dashboard

Add a dedicated status page in the Admin Dashboard showing connection counts, event throughput, and latency for each provider.

3. Multi-Region Pusher Support

Allow configuring different clusters for different regions to reduce latency for global users.

4. Client-side Event Filtering

Implement a more granular event filtering system in useRealtime.js to reduce noise in high-volume applications.
