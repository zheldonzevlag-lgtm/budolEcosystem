Mitigation Plan: Realtime System Stability

Risk ID: RR-011-01 (Connection Drops)

Mitigation Strategy: Implement exponential backoff reconnection in useRealtime.js and provide a manual "Refresh" button as a fallback.

Risk ID: RR-011-02 (Server Load)

Mitigation Strategy: Enforce a minimum polling interval of 5000ms in the Admin UI and implement server-side caching of settings in realtime.js (already implemented with 10s TTL).

Risk ID: RR-011-03 (Unauthorized Notification)

Mitigation Strategy: Enforce x-internal-key validation in the API Gateway for all /internal/notify requests. Ensure the key is rotated periodically.
