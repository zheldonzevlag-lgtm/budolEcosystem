System Admin Manual: Realtime Configuration

1. Accessing Realtime Settings

Navigate to Settings > Realtime Updates in the Admin Dashboard.

2. Provider Selection

Pusher (Hosted)

Recommended for production environments with high traffic. Requires valid Pusher credentials.

  • App ID, Key, Secret, Cluster

Socket.io (Self-hosted)

Uses the internal API Gateway. Best for cost optimization in development or high-security internal networks. The Socket.io server runs directly on the API Gateway instance and serves as the **central communications hub for all apps** in the Budol Ecosystem (BudolPay, BudolShap, BudolID).

  • Gateway URL: Typically http://localhost:8080 or the production LB URL. This URL is used by the frontend to establish the WebSocket connection.
  • Cross-App Communication: By centralizing Socket.io on the Gateway, events from any service are unified into a single stream for the Admin Dashboard and Mobile App.

SWR Polling (Client-side)

Fallback mode. Clients will poll the API at a defined interval.

  • Polling Interval: Default 10,000ms (10 seconds).

3. Troubleshooting

  • Audit Logs not updating: Check the GATEWAY_URL in .env and ensure BUDOLPAY_API_KEY matches between services.
  • Socket.io connection failed: Ensure CORS settings in API Gateway allow the dashboard domain.
