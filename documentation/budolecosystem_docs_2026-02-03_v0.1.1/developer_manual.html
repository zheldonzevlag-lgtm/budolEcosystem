Developer Manual: Realtime System & Audit Logs

1. Architecture Overview

The realtime system now supports a provider-agnostic architecture (Pusher, Socket.io, SWR Polling) with dynamic configuration. The core components are:

  • Backend Trigger: lib/realtime.js handles event dispatching.
  • Socket.io Server (API Gateway): The Socket.io server is hosted within the BudolPay API Gateway service. It manages active connections and channel-based room management for the entire Budol Ecosystem.
  • API Gateway (Central Relay): Acts as the single Socket.io relay and handles /internal/notify for event triggering. All apps (BudolShap, BudolID, BudolPay Mobile) route their events through this central hub to ensure a unified real-time stream.
  • Frontend Hook: useRealtime.js manages subscriptions and fallbacks.

2. Ecosystem-Wide Event Flow

The Budol Ecosystem uses a hub-and-spoke model for real-time events:

  1. Producer: Any microservice (e.g., BudolShap) triggers an event using triggerRealtimeEvent.
  2. Transport: The event is sent via an internal HTTP POST to the API Gateway's /internal/notify endpoint.
  3. Relay: The API Gateway receives the event and broadcasts it via Socket.io to the appropriate room.
  4. Consumer: Any connected client (Admin Dashboard, Mobile App, etc.) receives the update instantly.

API Gateway Socket Subscription

Added explicit subscribe event handler to support client-side room joining.

File: services/api-gateway/index.js Open File 
socket.on('subscribe', (channel) => {
    if (channel === 'admin') {
        socket.join('admin');
        console.log(`[Socket] Client subscribed to admin channel`);
    } else if (channel && channel.startsWith('user:')) {
        socket.join(channel);
    }
});

Frontend Realtime Hook

Optimized useRealtime to prevent re-renders and handle connection errors gracefully.

File: hooks/useRealtime.js Open File 
useEffect(() => {
    if (provider === 'SOCKET_IO') {
        socket = io(config.socketUrl, { ... });
        
        socket.on('connect', () => setIsConnected(true));
        socket.on(event, (data) => {
            if (onData) onData(data);
        });
        
        // Explicit subscription required for rooms
        socket.emit('subscribe', channel);
    }
}, [enabled, provider, config, channel, event, onData]);

3. Adding New Realtime Events

  1. Define the event constant in lib/constants.js (if applicable).
  2. Call triggerRealtimeEvent(channel, eventName, data) in your backend logic.
  3. Consume in frontend using useRealtime(channel, eventName, callback).
