System Admin Manual - BudolEcosystem v63

Date: 2026-01-03

Version: v63

1. SMTP Configuration Audit

Issue

Emails were failing with 535 5.7.8 BadCredentials.

Audit Tool

A new script check-settings.cjs is available to verify current settings.

node check-settings.cjs

Output includes:

  • Current DB Settings (masked passwords)
  • Environment Variable Fallbacks (SMTP_USER, SMTP_PASS)

Corrective Actions

If the DB password is incorrect (e.g., outdated or revoked):

  1. Log in to the Admin Panel (/admin/settings) and update the SMTP password with a valid Gmail App Password.
  2. Alternatively, update the .env file with SMTP_USER and SMTP_PASS, and clear the DB setting (set to null) to force fallback.

Note: The system now automatically removes spaces from the password, so copy-pasting App Passwords with spaces is safe.

2. Service Management

Payment Gateway Service

The Payment Gateway Service runs on port 8004. It is critical for BudolPay transactions.

Status Check:

netstat -ano | findstr 8004

Manual Start:

node "d:\IT Projects\budolEcosystem\budolpay-0.1.0\services\payment-gateway-service\index.js"

Mock Gateway

For testing, a Mock Gateway runs on port 8005 via scripts/mock-budolpay-gateway.cjs. This is used when the "BudolPay" provider is selected in "Mock" mode.

3. Deployment Notes

  • API Gateway: The application is currently configured to bypass the API Gateway (port 8080) for payment intent creation due to local proxy issues. It connects directly to the Payment Service (port 8004).
  • Firewall: Ensure port 8004 is allowed if running on a different machine (though currently bound to 127.0.0.1).
