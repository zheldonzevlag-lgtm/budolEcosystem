Developer Manual - BudolEcosystem v63

Date: 2026-01-03

Version: v63

1. BudolPay QR Payment Flow Fixes

Purpose

Resolved issue where BudolPay payments were redirecting to /orders instead of the dedicated QR page (/payment/qr), causing the QR code to be missed by the user.

Technical Details

  • File Modified: lib/payment/adapters/budolpay-adapter.js
    • Fix: Updated to generate qrCode object directly within the adapter, using api.qrserver.com for generation.
    • Fix: Corrected Gateway URL to point directly to http://127.0.0.1:8004 to bypass API Gateway proxy issues on Windows.
    • Fix: Ensured amount conversion from centavos to decimal is handled correctly.
  • File Modified: scripts/mock-budolpay-gateway.cjs
    • Fix: Changed the hardcoded redirect from /orders to /payment/return.
    • Rationale: The Mock Gateway was simulating a "Success" redirect that prematurely sent the user to the order list, skipping the QR code display. Now it redirects to the return handler, which (in a real scenario) would validate the payment. For QR flow, the adapter now returns the QR data before any redirect happens.
  • File Modified: components/OrderSummary.jsx
    • Logic: Verified that the component prioritizes qrCode presence. If qrCode is returned by the adapter, it saves data to localStorage and redirects to /payment/qr.

Code Example (Adapter)

// lib/payment/adapters/budolpay-adapter.js
// Direct connection to service on port 8004
this.gatewayUrl = 'http://127.0.0.1:8004';

// QR Code Generation
qrCode: {
    imageUrl: `https://api.qrserver.com/v1/create-qr-code/?...`,
    amount: amount,
    label: 'budolPay'
}

2. SMTP Authentication Fixes

Purpose

Addressed SMTP 535 5.7.8 BadCredentials error preventing order confirmation emails.

Technical Details

  • File Modified: lib/email.js
    • Fix: Added sanitization for smtpPass to remove spaces (.replace(/\s+/g, '')). This handles cases where App Passwords are copied with spaces.
    • Fix: Enhanced error logging to distinguish between connection errors and authentication errors.
  • Tool Created: check-settings.cjs
    • Purpose: Audits current System Settings in the database vs Environment Variables.
    • Usage: node check-settings.cjs

3. API Gateway Debugging

Issue

API Gateway (port 8080) was returning 504 Gateway Timeout when proxying to Payment Service (port 8004).

Resolution

Bypassed API Gateway in budolpay-adapter.js by connecting directly to http://127.0.0.1:8004. This ensures stability for local development on Windows.

4. Recommendations

  • Ensure Payment Gateway Service (port 8004) is running. If it crashes, restart it manually or via start-ecosystem.js.
  • Update System Settings with a valid Gmail App Password if SMTP errors persist.
