Dependencies & Software Rationale

Pusher (pusher-js / pusher-http-node)

Role: Managed Realtime Event Transport

Rationale: Chosen for its reliability and ease of scaling. In a fintech ecosystem, manual WebSocket management (Socket.io) can be complex when scaling across multiple regions or instances. Pusher provides a managed infrastructure that handles reconnections and horizontal scaling out-of-the-box.

Socket.io

Role: Internal Microservice Realtime Communication

Rationale: Used as a cost-effective alternative for internal service-to-service notifications within the same network/VPC where managed services like Pusher might introduce unnecessary latency or cost.

Prisma ORM

Role: Database Abstraction & Type Safety

Rationale: Essential for forensic audit logging. Prisma's type safety ensures that audit log structures are consistent across the ecosystem, and its include capabilities make it easy to fetch associated user metadata (Name/Email) in a single query.

UA-Parser-JS

Role: Forensic Device Identification

Rationale: Required for PCI DSS 10.2.2 compliance. It allows the system to identify the specific device and browser version used during a security event, providing critical forensic data for incident investigations.
