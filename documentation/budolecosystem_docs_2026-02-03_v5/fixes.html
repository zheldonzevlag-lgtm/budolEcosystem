Issues and Fixes

Issue: Realtime updates required manual refresh

Symptoms: Forensic audit logs only updated when the page was manually reloaded.

Cause: Frontend was using static fetching without a live subscription or polling mechanism.

Fix: Provider-Agnostic Realtime Hook

Implemented useRealtime hook that supports Pusher, Socket.io, and SWR Polling fallback. Integrated this into audit-logs/page.jsx.

Issue: Pusher/Socket.io misconfiguration leads to broken UI

Symptoms: If Pusher keys were missing or Socket server was down, the UI stopped updating entirely.

Cause: Lack of automated fallback logic to standard polling.

Fix: Automated Fallback Logic

Added validation in realtime.js (backend) and useRealtime.js (frontend) to automatically revert to POLLING if credentials or connectivity fails.

Issue: Inflexible Realtime Settings

Symptoms: Admin could only configure Pusher, not Socket.io or SWR Polling intervals.

Cause: Hardcoded UI and missing database fields in SystemSettings.

Fix: Comprehensive Realtime Admin UI

Updated SystemSettings model with swrPollingInterval and socketUrl. Revamped realtime/page.jsx with a card-based provider selection and dynamic configuration forms for all three methods.
