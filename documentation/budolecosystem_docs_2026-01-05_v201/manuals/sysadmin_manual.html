SysAdmin Manual: Notification Settings Layout Refactor

Version: v201 | Date: 2026-01-05

1. Overview

This document provides operational details for managing the updated Notification Settings layout in the Admin Panel.

2. Configuration Management

The layout refactor does not change how settings are stored in the database. All configurations continue to be stored in the SystemSetting table under the SMS and EMAIL groups.

2.1. Key Settings

  • EMAIL_PROVIDER: Determines which email gateway is active (GOOGLE, BREVO, GMASS).
  • SMS_PROVIDER: Determines which SMS gateway is active (VIBER, ITEXMO, BREVO_SMS, SEMAPHORE, ENGAGESPARK).

3. Deployment Procedures

No special deployment steps are required beyond the standard deployment process. Ensure that the Prisma client is regenerated after any schema changes (though none were introduced in this version).

npm run build
npx prisma generate

4. Monitoring & Troubleshooting

If the settings page fails to load or settings are not being saved:

  • Check the SystemSetting table in the database to ensure it's accessible.
  • Verify that the server has network access to the configured SMTP hosts and API endpoints.
  • Review server logs for any errors during revalidatePath("/settings/notifications").

5. Operational Impacts

The new layout provides a clearer view of active vs. inactive providers. Admins should ensure that only one provider is marked as "Active" for each category to avoid confusion, although the UI handles this via server actions.
