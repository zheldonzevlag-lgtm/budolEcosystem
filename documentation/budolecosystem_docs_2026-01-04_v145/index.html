v145

Network Awareness & Dynamic Host Configuration

Enabling budolPay Mobile to connect across varying local network environments without code changes.

Purpose & Rationale

As a local ecosystem, budolPay relies on local IP addresses that change depending on the physical location (home, office, internet cafe). This update introduces manual network configuration, allowing users to point the app to the current local server IP address dynamically.

Portability

Use the same APK in any environment by simply updating the Server IP in settings.

Persistence

Configurations are saved locally, so you only need to set it once per network environment.

Technical Implementation

The implementation focuses on the ApiService as the single source of truth for host resolution.

1. ApiService (Host Resolution)

Added persistent storage for custom host and dynamic resolution logic.

// Dynamic host resolution in ApiService
String get host {
  if (kIsWeb) return Uri.base.host;
  if (_customHost != null) return _customHost!;
  return const String.fromEnvironment('API_HOST', defaultValue: '192.168.1.24');
}

Future<void> setCustomHost(String newHost) async {
  _customHost = newHost;
  await _prefs?.setString(_hostKey, newHost);
  notifyListeners();
}

2. Login Screen (User Interface)

Added a Network Settings dialog accessible via a gear icon on the login screen.

// Network Settings Dialog Logic
void _showNetworkSettings() {
  showDialog(
    builder: (context) => AlertDialog(
      title: Text('Network Settings'),
      content: TextField(controller: _hostController),
      actions: [
        TextButton(onPressed: _resetHost, child: Text('Reset to Default')),
        ElevatedButton(onPressed: _saveHost, child: Text('Save')),
      ],
    ),
  );
}

Test Results

Passed Persistence: Verified that custom IP addresses survive app restarts using SharedPreferences.

Passed Host Fallback: Verified that resetting the host correctly reverts to the default IP (192.168.1.24).

Passed Unit Tests: Added and passed automated tests for host resolution logic in api_service_test.dart.

Passed UI Consistency: Fixed deprecated ElevatedButton.fromPlatform to ensure consistent look and feel.

Issues & Mitigations

Issue Mitigation
Invalid IP Input Added basic validation and a "Reset to Default" option to prevent permanent disconnection.
Deprecated UI Widgets Replaced fromPlatform with styleFrom to comply with latest Flutter standards.

Future Recommendations

  • Auto-Discovery: Implement mDNS/ZeroConf to automatically find the budolPay server on the local network.
  • Connectivity Check: Add a "Test Connection" button in settings to verify the IP before saving.
