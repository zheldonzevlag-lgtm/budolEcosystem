Developer Manual: Modal Keyboard Responsiveness

1. Implementation Overview

This update addresses the issue where the system keyboard overlaps modal dialogs (specifically the MPIN verification dialog in SettingsScreen), preventing users from clicking "Verify" or "Cancel" buttons. The fix involves wrapping the dialog content in a SingleChildScrollView and applying dynamic bottom padding based on MediaQuery.of(context).viewInsets.bottom.

2. UI Code Changes

File: budolPayMobile/lib/screens/settings_screen.dart
// Updated _showPinDialog to handle keyboard insets
builder: (context) => Dialog(
  shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(20)),
  child: SingleChildScrollView(
    child: Padding(
      padding: EdgeInsets.only(
        top: 24,
        left: 24,
        right: 24,
        bottom: 24 + MediaQuery.of(context).viewInsets.bottom,
      ),
      child: Column(
        mainAxisSize: MainAxisSize.min,
        children: [
          // ... dialog content ...
        ],
      ),
    ),
  ),
),

3. Key Technical Changes

  • Dynamic Insets: Utilized MediaQuery.of(context).viewInsets.bottom to calculate the space occupied by the keyboard and adjust padding accordingly.
  • Scrollable Content: Wrapped the dialog body in a SingleChildScrollView to ensure that if the content (including the keyboard offset) exceeds the screen height, the user can still scroll to access the action buttons.
  • Constraint Management: Used mainAxisSize: MainAxisSize.min in the Column to maintain the compact look of the dialog when the keyboard is hidden.

5. Phase 8: Dependencies and Software Choice

The following software and dependencies were utilized in this version:

  • Flutter SDK: Chosen for its robust layout system and MediaQuery API which simplifies handling device-specific UI states like keyboard visibility.
  • SingleChildScrollView: A critical widget for handling overflow in small viewports or when the effective viewport is reduced by the system keyboard.
  • Dialog Widget: Used as the base for custom MPIN verification to maintain a consistent look and feel across the platform.

6. Compliance Verification

This implementation adheres to the following standards:

  • Cybersecurity Law: Ensures that secure MPIN verification is accessible and usable, preventing security bypasses due to UI failures.
  • NPC (National Privacy Commission): Maintains the integrity of the MPIN input process by ensuring the UI remains stable during sensitive data entry.
  • PCI DSS: The MPIN input field remains masked and secure, even when the layout shifts for keyboard visibility.
