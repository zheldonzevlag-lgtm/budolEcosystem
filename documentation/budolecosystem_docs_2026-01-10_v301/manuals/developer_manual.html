Developer Manual: RBAC & Permissions

Overview

The RBAC (Role-Based Access Control) system in BudolShap is designed to be granular, centralized, and compliant with security standards like PCI DSS v4.0. It uses a combination of role (String) and permissions (Json) fields in the User model.

Core Components

1. lib/rbac.js

This is the source of truth for all roles and permissions. It defines:

  • PERMISSIONS: A constant object containing all available granular permissions (e.g., products:view, users:manage_roles).
  • ROLES: Standard roles available (ADMIN, MANAGER, STAFF, SELLER, BUYER).
  • ROLE_PERMISSIONS: A mapping of roles to their default permission sets.

2. lib/adminAuth.js

The requireAdmin helper has been updated to support granular permission checks. When a route calls requireAdmin(request, PERMISSIONS.USERS_EDIT), it verifies:

  1. The user is an admin (isAdmin: true or accountType: 'ADMIN').
  2. The user has the specific permission in their permissions field OR their role has the permission in lib/rbac.js.

API Endpoints

GET /api/admin/users/roles

Returns the current RBAC configuration. Used by the PermissionsMatrix component to visualize the access policies.
Location: budolshap-0.1.0/app/api/admin/users/roles/route.js

POST /api/admin/users/sync-rbac

A maintenance endpoint that synchronizes all users' roles and permissions based on the current standards defined in lib/rbac.js. It ensures that isAdmin flags are correctly set for administrative roles.
Location: budolshap-0.1.0/app/api/admin/users/sync-rbac/route.js

Database Connection

In the current local development environment, we do not use Docker. The application connects to a local PostgreSQL instance.
Service Name: postgresql-x64-18
Port: 5432
Credentials: Defined in .env (postgres:r00t)

UI Implementation

PermissionsMatrix.jsx

A "read-only" visualization component located in the Admin Settings. It fetches configuration from the roles API and displays a grid of permissions for each role. It also includes a "Sync All Users" button to trigger the synchronization process.

Security Considerations

  • Least Privilege: Users are assigned the minimum permissions necessary for their role.
  • Audit Logging: All RBAC changes should be accompanied by an audit log entry (to be enhanced in future versions).
  • Compliance: The system aligns with PCI DSS Requirement 7 (Restricting access to system components and cardholder data).
