SysAdmin Manual: RBAC & User Access Management

Warning: RBAC changes affect system-wide security. Always perform a backup of the User table before running synchronization scripts.

Managing User Roles

Administrators can manage user roles through the Admin Dashboard under Settings > RBAC & Permissions.

1. Reviewing Access Policies

Use the Permissions Matrix to visualize what each role (ADMIN, SELLER, etc.) can do. This matrix is the authoritative guide for access control in the ecosystem.

2. Synchronizing User Roles

If you notice users have incorrect access levels or if lib/rbac.js has been updated with new permissions, use the Sync All Users button in the Permissions Matrix UI. This will:

  • Update role and permissions fields based on accountType.
  • Ensure isAdmin flags are correctly synchronized with the database.
  • Enforce the "Least Privilege" principle across all accounts.

3. Manual CLI Synchronization

In cases where the UI is inaccessible, you can run the synchronization script manually via Node.js:

node sync_rbac_roles.mjs

Elevating Admin Users

To promote a standard user to an Admin, use the provided update_admin.mjs script. This script ensures that all necessary flags (isAdmin, accountType, role, permissions) are set correctly and optionally resets the password.

node update_admin.mjs

Follow the on-screen prompts to enter the user's email and desired password.

Database Service Troubleshooting (No Docker)

If the application reports "Can't reach database server at localhost:5432", it is likely that the PostgreSQL Windows service is stopped.

Checking Service Status

Run the following command in an administrative PowerShell terminal:

Get-Service -Name *postgresql*

Starting the Service

If the status is Stopped, start it with:

Start-Service -Name "postgresql-x64-18"
Note: If you are using a different version of PostgreSQL, the service name may vary (e.g., postgresql-x64-16). Adjust the command accordingly.

Compliance Auditing

The system is designed to meet PCI DSS v4.0 Requirement 7. SysAdmins should periodically:

  • Review the list of Admin users for accuracy.
  • Verify that no "orphan" accounts have administrative privileges.
  • Check audit logs for any unauthorized role modifications.
