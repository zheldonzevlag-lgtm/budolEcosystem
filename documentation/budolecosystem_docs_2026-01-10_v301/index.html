RBAC Audit & Role Assignment

Standardizing role-based access control and ensuring permission consistency across all ecosystem users.

Status: Implemented & Verified

Purpose

The primary goal of this update was to review the existing Role-Based Access Control (RBAC) model, audit all current users, and assign standardized roles and permissions based on the lib/rbac.js configuration. This ensures that every user has the correct level of access required for their account type (ADMIN, SELLER, BUYER).

Implementation Overview

  • RBAC Audit: Created and ran audit_rbac.mjs to identify any inconsistencies between accountType, isAdmin, and assigned role.
  • Role Standardization: Developed sync_rbac_roles.mjs to automatically map users to their correct roles and permissions based on the ecosystem standard.
  • Permission Mapping: Ensured that ADMINs receive all 23 defined permissions, while SELLERs and BUYERs receive their respective subsets.
  • Infrastructure Management: Resolved "Database Connection Refused" errors by identifying and starting the local PostgreSQL service (postgresql-x64-18) for non-Docker environments.
  • Verification: Validated the logic using scripts/test-rbac.mjs, confirming both static logic and database integration work as expected.
