Issues & Fixes Log (v48)

Vercel Build Failure: Node.js Engine Mismatch CRITICAL

The payment-gateway-service had a hard-coded engine requirement for Node 20.x, which conflicted with Vercel's newer default environments ($NODE_VERSION=24).

✅ Resolution

Updated package.json engine constraint to >=20.x to allow flexibility across local and VPC/Vercel environments.

Test Suite Instability: External Service Dependencies CRITICAL

Integration tests for Anti-Spam and Payment Security were failing because they attempted to connect to live Auth and Gateway services not present in the monorepo test runner.

✅ Resolution

Implemented robust axios mocks using jest.mock factory functions in spam_prevention.test.js and payment-security.test.js. Decoupled test logic from environment state.

Database Diagnostic Hang in Tests HIGH

A self-invoking diagnostic function in api/index.js was attempting database connections during test imports, causing Jest to hang or fail initialization.

✅ Resolution

Wrapped startup diagnostics in a process.env.NODE_ENV !== 'test' check to ensure silent imports for unit testing.

Notification Type/Method Error MEDIUM

Mismatched model name (systemSetting vs SystemSetting) and method (findFirst vs findMany) in the notifications test mock.

✅ Resolution

Aligned the mock with the actual Prisma schema and repository implementation.
