The Ecosystem Glue

Our applications are "glued" together through three primary mechanisms: Shared Identify, Event-Driven Communication, and Unified Data Auditing.

[User] --(Login)--> [budolID SSO]
        |
        +--(Token/PII)--> [budolPay Services]
        |                    |
        |                    +--(Event)--> [Payment Gateway]
        |                    |                |
        +--(Audit)--> [Postgres Shared Schema] <--(Broadcast)--+

Identity Management (budolID)

BudolID acts as the Central Identity Provider (IdP). It manages EcosystemApp records and issues scoped JWTs. Services verify these tokens using a shared JWT_SECRET.

budolID/index.js // Cross-app initialization
const apps = ['budolPay', 'budolID', 'budolShap'];
for (const app of apps) {

}

Real-time Event Propagation

The @budolpay/audit package contains the triggerRealtimeEvent utility which bridges backend actions to the Admin Dashboard via the Socket.io Gateway.

packages/audit/src/index.ts export const triggerRealtimeEvent = async (event: RealtimeEvent) => {
  const gatewayUrl = process.env.PAYMENT_GATEWAY_URL;
  await axios.post(`${gatewayUrl}/api/notify`, event);
};
