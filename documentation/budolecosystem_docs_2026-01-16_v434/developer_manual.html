Developer Manual - v434

Technical Overview

This release involves changes in both the budolShap payment adapter and the budolPay mobile app (Flutter Web).

1. Payment Adapter Update

The merchant owner name for BudolPay payments was hardcoded in the adapter. This has been updated to reflect the correct merchant owner.

File: budolpay-adapter.js


// lib/payment/adapters/budolpay-adapter.js:129
qrCode: {
    imageUrl: `https://api.qrserver.com/v1/create-qr-code/?size=300x300&data=${encodeURIComponent(JSON.stringify({
        type: 'budolpay_payment',
        orderId: options.orderId,
        amount: amount / 100,
        merchant: 'budolShap Merchant',
        paymentIntentId: paymentIntentId
    }))}`,
    amount: amount,
    label: 'Jon Galvez' // Updated from 'budolPay'
},

2. Flutter Web Layout Fix

The Flutter Web app experienced a black screen issue due to nested layout constraints in the MaterialApp.builder and MarketingAdsScreen.

File: main.dart

Simplified the web builder to avoid redundant Stack and Positioned.fill which could interfere with Navigator rendering.


// lib/main.dart:193
child: Material(
    color: const Color(0xFF0F172A),
    child: DefaultTextStyle(
      style: const TextStyle(
        fontFamily: 'Inter',
        color: Colors.white,
        decoration: TextDecoration.none,
      ),
      child: child ?? const SplashScreen(), // Simplified from Stack/Positioned.fill
    ),
  ),

File: marketing_ads_screen.dart

Removed SafeArea and added explicit dimensions to PageView items to ensure they render correctly in the constrained web container.


// lib/screens/marketing_ads_screen.dart:127
return Scaffold(
  backgroundColor: const Color(0xFF0F172A),
  body: Stack(
    children: [
      Positioned.fill(
        child: PageView.builder(
          // ...
        ),
      ),
      // ...
    ],
  ),
);
