v249: Favorite Recipients Feature & Mobile Build v1.2.14

Date: 2026-01-09
Version: v249
Status: Completed
Author: Implementation Planner

Purpose

To implement the "Favorite Recipients" feature, allowing users to bookmark frequent transaction partners for quick access. This includes backend schema updates, REST API endpoints, and a new mobile UI workflow.

Explanation & Reason

User feedback indicated that manually entering phone numbers or emails for frequent transfers was tedious. By allowing users to mark recipients as "Favorites" directly from their transaction history, we reduce friction and improve the speed of P2P transfers.

Issues Identified

  • Transfer Friction: No way to save frequent recipients; users had to re-enter details every time.
  • Incomplete Workflow: The favorites list needed to integrate with the SendMoneyScreen to pre-fill data.
  • Parameter Mismatch: Initial implementation of FavoritesScreen failed flutter analyze due to missing constructor parameters in SendMoneyScreen.

Fixes & Enhancements Implemented

  • Prisma Schema Update: Added FavoriteRecipient model with unique constraints on userId and recipientId.
  • Auth Service CRUD: Implemented GET /favorites, POST /favorites, and DELETE /favorites/:recipientId in the Auth Service.
  • Mobile Favorites Screen: Created a dedicated screen to list, remove, and quickly send money to favorites.
  • Transaction Details Integration: Added a "Add to Favorites" button in the transaction details view for outgoing P2P transfers.
  • SendMoneyScreen Upgrade: Modified the screen to accept an initialRecipient parameter for pre-filling the recipient field.

Technical Details

Database Schema

model FavoriteRecipient {
  id          String   @id @default(uuid())
  userId      String
  recipientId String
  alias       String?
  createdAt   DateTime @default(now())
  user        User     @relation("UserFavorites", fields: [userId], references: [id])
  recipient   User     @relation("RecipientFavorites", fields: [recipientId], references: [id])
  @@unique([userId, recipientId])
}

API Integration (Auth Service)

app.post('/favorites', authenticate, async (req, res) => {
    const { recipientId, alias } = req.body;
    const favorite = await prisma.favoriteRecipient.upsert({
        where: { userId_recipientId: { userId: req.user.id, recipientId } },
        update: { alias },
        create: { userId: req.user.id, recipientId, alias }
    });
    res.json(favorite);
});

Test Results

Success: End-to-end flow verified!
  • Analysis: flutter analyze passed with 0 issues after fixing constructor parameters.
  • UI Verification: "Favorites" button added to Home Screen action row; navigation verified.
  • Functional: Adding a favorite from transaction details and initiating a transfer from the favorites list verified.

Risks & Mitigations

  • Risk: Potential data inconsistency if a user is deleted but remains in someone's favorites.
  • Mitigation: Relying on Prisma's referential integrity (foreign keys) to prevent orphaned records.
  • Risk: Overcrowded Home Screen action row.
  • Mitigation: Adjusted spacing (16px) to accommodate 4 actions (Cash In, Send, Favorites, Scan).

Dependencies

  • Database: New FavoriteRecipient table (Prisma).
  • Auth Service: New endpoints for CRUD operations.
  • Mobile: Updated ApiService and SendMoneyScreen.
