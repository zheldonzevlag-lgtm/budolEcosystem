Unified Ecosystem RBAC Standards v229

Date: 2026-01-08

Purpose

To establish a unified Role-Based Access Control (RBAC) standard across the BudolEcosystem (BudolPay and BudolShap). This ensures consistent security protocols, simplified auditing, and compliance with PCI DSS, BSP Circular 808, and NPC DPA 2012.

Technical Details

1. Schema Enhancements (BudolShap)

The User model in BudolShap was upgraded to support granular permissions, matching BudolPay's architecture.

model User {
  // ...
  role        String   @default("USER")
  permissions Json?
  // ...
}

2. Standardized Roles

Role Description Access Level
ADMIN System Owner / Full Access All Permissions
MANAGER Operational Supervisor Approval & View rights
STAFF Daily Operations Transactional View rights
SELLER Store Owner Store/Product Management

Code Implementation

A new lib/rbac.js was created to centralize permission logic, and lib/adminAuth.js was refactored to enforce these permissions in API routes.

// Usage in API Routes
import { requireAdmin } from '@/lib/adminAuth'
import { PERMISSIONS } from '@/lib/rbac'

export async function POST(req) {
    const { authorized, user, errorResponse } = await requireAdmin(req, PERMISSIONS.SETTINGS_EDIT)
    if (!authorized) return errorResponse;
    
    // Proceed with secure action
}

Test Results

PASSED
  • Static Logic: Verified that STAFF roles cannot perform ADMIN-only actions.
  • Database Storage: Confirmed that roles and permission overrides are correctly persisted in Prisma.
  • Middleware Enforcement: Validated that requireAdmin correctly blocks unauthorized requests with a 403 Forbidden status.

Risk & Mitigation

Risk Mitigation
Permission Creep Regular access reviews (PCI DSS 7.1.1) and automated audit logging of role changes.
Legacy Bypass Refactored all admin middleware to use the new permission-based verifyAdminAccess.

Recommendations

  • Implement a UI-based Permissions Matrix in BudolShap Admin (similar to BudolPay).
  • Centralize RBAC definitions into a shared ecosystem library to prevent drift.
  • Add MFA requirements for roles with SETTINGS_EDIT or PAYOUTS_APPROVE permissions.
