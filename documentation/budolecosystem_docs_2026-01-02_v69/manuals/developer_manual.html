Developer Manual - v69

Purpose

This update addresses critical connectivity issues when running the budolPay mobile app on physical Android devices. It also fixes a Dart compilation error related to const field declarations in the ApiService.

Technical Implementation

1. ApiService Connectivity Fix

The _host getter in lib/services/api_service.dart was updated to point to the host machine's Wi-Fi IP address (192.168.1.24) instead of localhost. This allows physical devices on the same network to reach the local backend services.

static String get _host {
  if (kIsWeb) {
    return Uri.base.host;
  }
  // Updated for physical device testing
  return '192.168.1.24';
}

2. Dart Compilation Fix

The apiKey field was changed from const String to static const String to comply with Dart's requirement that only static fields can be declared as const within a class.

static const String apiKey = String.fromEnvironment(
  'BUDOLPAY_API_KEY',
  defaultValue: 'bp_key_2025',
);

Issues & Fixes

  • Issue: Connection Refused error when logging in on a physical device.
    Fix: Changed host from localhost to 192.168.1.24.
  • Issue: "Only static fields can be declared as const" error.
    Fix: Added static keyword to the apiKey declaration.

Recommendations

For production, ensure the GATEWAY_URL and AUTH_URL environment variables are properly set during the build process to avoid hardcoding IP addresses.
