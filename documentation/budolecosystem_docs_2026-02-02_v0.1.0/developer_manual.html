Technical Architecture: Forensic Audit Trail

1. Data Model (Prisma)

The audit trail is built on a robust relational model designed for immutability and compliance.

model AuditLog { id String @id @default(uuid()) userId String? action String metadata Json? // Stores compliance tags (PCI DSS/BSP) createdAt DateTime @default(now()) entity String? entityId String? ipAddress String? user User? @relation(fields: [userId], references: [id]) }

Source: schema.prisma

2. Backend: Audit Log Creation

The createAuditLog helper in Auth Service ensures all security events are logged with IP addresses and compliance metadata.

const createAuditLog = async (req, userId, action, metadata = {}, entity = 'Security') => { const log = await prisma.auditLog.create({ data: { userId, action, entity, ipAddress: req.ip, metadata: { ...metadata, compliance: { pci_dss: '10.2.2', bsp: 'Circular 808' } } } }); await notifyAdmin('AUDIT_LOG_CREATED', log); };

Source: auth-service/index.js

3. Real-time Broadcast (API Gateway)

The Gateway proxies internal notifications to Socket.io and Pusher channels for immediate UI updates.

router.post('/internal/notify', async (req, res) => { const { event, data, isAdmin } = req.body; if (isAdmin) { io.to('admin').emit(event, data); if (pusher) pusher.trigger('admin', event, data); } });

Source: api-gateway/index.js

4. Frontend: Real-time Subscription

Admin pages use a singleton realtime service to listen for AUDIT_LOG_CREATED events.

realtime.on("AUDIT_LOG_CREATED", (newLog) => { setAuditLogs(prev => [newLog, ...prev.slice(0, 14)]); });

Source: employees/page.tsx

5. Manual Admin Account Setup

In scenarios requiring database resets, the primary admin account is re-established using an environment-driven provisioning script. This follows PCI DSS 8.2 by ensuring no cleartext credentials are hardcoded in source control.

// credentials are pulled from .env file const adminEmail = process.env.ADMIN_EMAIL; const adminPassword = process.env.ADMIN_PASSWORD; const hashedPassword = await bcrypt.hash(adminPassword, 10); await prisma.user.upsert({ where: { email: adminEmail }, update: { passwordHash: hashedPassword, role: 'ADMIN' }, create: { email: adminEmail, passwordHash: hashedPassword, role: 'ADMIN', phoneNumber: process.env.ADMIN_PHONE, department: 'System Administration' } });
