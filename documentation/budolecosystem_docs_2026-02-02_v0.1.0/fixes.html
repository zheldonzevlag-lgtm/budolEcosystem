Dashboard Issues Tracking Fixes Applied

Fixes & Resolutions

Fix for ISS-001: Centralized Prisma Access

Root Cause: Scripts were attempting to instantiate their own Prisma clients, leading to path errors in the monorepo structure.

Resolution: Modified scripts to use the centralized prisma client from packages/database using absolute path references.

Fix for ISS-002: Schema Compliance in Tests

Root Cause: Test scripts were using outdated user model fields (password instead of passwordHash) and missing phoneNumber.

Resolution: Updated all seed and verification scripts to align with the latest schema.prisma definition, including random unique phone numbers for test isolation.

Fix for ISS-003: Jest Configuration Bridge

Root Cause: Jest's default configuration in the monorepo was optimized for workspace-specific runs, making root-level TypeScript test execution difficult.

Resolution: Created a simplified JavaScript test runner (audit-trail-simple.test.js) that uses ts-jest capabilities while avoiding complex module resolution issues during the verification phase.

Fix for ISS-004: User Profile Update Persistence

Root Cause: 1) Double request.json() call in the API handler caused runtime crashes. 2) The department field was missing from the database schema despite being present in the UI and update logic.

Resolution: Refactored the employee API route to parse the request body once and added the department field to the User model in schema.prisma, followed by a database sync.
