Dashboard Risk Register Mitigation Plan

Risk Mitigation Strategies

M-001: Log Volume Management (Mitigates R-001)

Implementation of a multi-tier storage strategy for audit logs.

  • Hot Storage (PostgreSQL): Last 12 months for immediate retrieval.
  • Cold Storage (S3/Glacier): Logs older than 12 months, archived with integrity hashes.
  • Indexing: Specialized indexing on createdAt and userId to maintain query performance.

M-002: Real-time Resilience (Mitigates R-002)

Dual-channel notification system for administrative alerts.

  • Primary: Socket.io for low-latency browser updates.
  • Fallback: Pusher for guaranteed delivery via managed infrastructure if local socket fails.
  • Buffering: Client-side buffering to re-sync missed events during short disconnects.

M-003: Log Integrity (Mitigates R-003)

Ensuring immutability of the forensic trail.

  • RBAC: Database-level permissions preventing DELETE or UPDATE on the AuditLog table.
  • WORM: (Future) Migration to Write-Once-Read-Many storage for production compliance.
  • Audit of the Audit: Logging of all attempts to access the AuditLog table itself.
