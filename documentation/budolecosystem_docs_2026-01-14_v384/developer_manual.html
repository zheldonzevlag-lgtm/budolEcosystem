Developer Manual: Face Recognition Subsystem

Overview

The Face Recognition subsystem uses on-device TFLite models to generate 128-dimensional face embeddings. These embeddings are compared using Cosine Similarity to verify user identity.

Technical Implementation

The core logic for similarity calculation is implemented in face_embedding_service.dart and verified in the test suite.

Cosine Similarity Logic

double calculateSimilarity(List<double> emb1, List<double> emb2) {
  double dotProduct = 0.0;
  double normA = 0.0;
  double normB = 0.0;
  for (int i = 0; i < emb1.length; i++) {
    dotProduct += emb1[i] * emb2[i];
    normA += pow(emb1[i], 2);
    normB += pow(emb2[i], 2);
  }
  return dotProduct / (sqrt(normA) * sqrt(normB));
}

Test Scripts

Two main test scripts are used for verification:

  • test_face_logic_v362.dart: A Flutter test script that validates the mathematical accuracy of the embedding comparison.
  • test-face-verification-v362.js: A backend script that verifies the Prisma schema has the necessary faceTemplate fields.

Known Issues & Resolution

The facenet.tflite model is currently a placeholder (66 bytes). To resolve this, the official 93.9MB model must be downloaded and placed in budolPayMobile/assets/models/.
