Intelligent Ecosystem Runner

Unified pre-flight validation and full service integration for the budolEcosystem Central Runner.

Purpose

The primary goal of this update was to make the npm start command (Central Ecosystem Runner) "intelligent" by ensuring it validates the environment before launching any services. This prevents the common developer error of accidentally connecting to a production database while running in a local development environment.

Safety First

Automated checks ensure local code never touches production data without explicit intent.

Full Integration

Includes the new budolAccounting service in the core ecosystem startup sequence.

Technical Implementation Details

1. Unified Environment Validator

Converted scripts/env-validator.js to CommonJS to support the root ecosystem runner and individual services.

module.exports = {
  validateEnvironment: (appId) => {
    // Detects production DB keywords in local dev mode
    // Returns false if safety check fails
  }
};

2. Pre-flight Check Logic

The runner now scans each app's .env file before starting to verify database connection strings.

  • Extracts DATABASE_URL from app-specific environment files.
  • Triggers a 5-second warning delay if safety violations are found.
  • Continues with high-visibility warnings to alert the developer.

3. Service Expansion

Added budolAccounting to the apps registry and injected its URL into the global environment.

{
  name: 'budolAccounting',
  cwd: path.join(__dirname, 'budolAccounting-0.1.0'),
  command: 'npm',
  args: ['run', 'start'],
  port: 8005
}

Verification & Results

Local Start Success

Verified that npm start successfully launches all 8 services simultaneously on detected local IP.

Safety Barrier Test

Manually injected a production DB URL into a local .env; runner correctly identified the risk and displayed a high-visibility error message.

Accounting Integration

Confirmed ACCOUNTING_SERVICE_URL is available to all spawned apps.
