Developer Manual: Universal Identity & Trust

Overview

The "Eco-Sync" system ensures that every user in the Budol Ecosystem has a single, verifiable identity managed by budolID. This implementation covers registration synchronization, anti-fraud scoring, and universal KYC/KYB state management.

Eco-Sync Registration Flow

When a user registers in budolShap, the following synchronous handshake occurs:

  1. Data Capture: budolShap captures email, password, phoneNumber, and an optional profile picture.
  2. budolID Handshake: budolShap calls budolID API to create the master identity.
  3. Fraud Analysis: budolID performs velocity, phone duplication, and reputation checks. Providing a profile picture during registration reduces the initial fraud score.
  4. Local Sync: If budolID succeeds and the risk is acceptable, budolShap creates a local shadow record with the same ID.

Face Verification Infrastructure

A new field isFaceVerified has been added to the User model. This serves as the foundation for high-trust apps:

  • budolPay: Requires isFaceVerified = true for wallet withdrawals and high-value transfers.
  • budoloan: Mandatory face verification before loan application processing.
  • budolExpress: Drivers must complete a liveness check (photo + ID) to activate their account.

Universal Identity Requirements

To maintain ecosystem integrity, all registration endpoints must enforce the following:

  • Email: Must be unique and not from a disposable domain.
  • Phone Number: Must be unique across the entire ecosystem. Duplicates are blocked at the budolID layer.

Universal KYC/KYB

Trust status is now "Universal". If a user is verified as a merchant in budolShap, their status is pushed to budolID, making them instantly eligible for merchant features in budolPay and budoloan.

// Example Trust Update Call
await updateTrustStatus(userId, {
    kybStatus: 'APPROVED'
});

Anti-Fraud Measures

  • Registration Velocity: Limits multiple accounts from the same IP within 24 hours.
  • Reputation Scoring: Detects disposable email domains and suspicious device patterns.
  • Risk Gates: "CRITICAL" risk accounts are automatically blacklisted and prevented from registering.

budolShap Frontend Registration Update

The AuthForm component in budolShap has been updated to align with the new ecosystem trust requirements:

  • Dual Registration Modes: Users can now choose between "Standard" (Email, Name, Phone, Password) and "Quick" (Phone Only).
  • Phone-Only Flow: In Quick mode, budolID automatically generates a placeholder email and temporary password to maintain system compatibility.
  • KYC Roadmap: Users registered via the Quick flow are flagged for mandatory profile completion and KYC before they can access high-trust features (Pay, Loan, Driver activation).
  • Phone Number Field: Always mandatory as it serves as the primary identity anchor for both flows.

SSO Login & Authentication Fix (v34)

A critical issue was resolved regarding budolPay admin login where "Invalid credentials" errors were encountered. The resolution involved:

  • Phone Number Sync: Ensuring phoneNumber is a mandatory field during user creation/seeding in budolID.
  • Credential Alignment: Resetting the admin@budolpay.com password to admin123 in the central identity provider.
  • Token Payload: Confirming that /auth/verify returns the full user profile including phoneNumber to allow local database sync in consuming apps.
