System Admin Manual: Infrastructure & API

Prisma Schema Updates (budolID)

The User model in budolID has been expanded with universal trust fields:

Field Type Description
kycLevel Enum NONE, BASIC, VERIFIED, PREMIUM
kybStatus Enum NONE, PENDING, APPROVED, REJECTED
fraudScore Int 0-100 (Risk probability)
riskLevel Enum LOW, MEDIUM, HIGH, CRITICAL
isFaceVerified Boolean Infrastructure for biometric/ID photo verification status.

Admin Account Recovery & Seeding

If the budolPay admin cannot log in, use the provided seed script to reset credentials in budolID:

node prisma/seed-user.js

This script ensures the admin@budolpay.com user has the required phoneNumber field and a known password (admin123).

New API Endpoints

budolID Service

  • POST /auth/register: Enhanced with anti-fraud metadata support and optional profile picture.
  • POST /auth/sso/login: Validates credentials against master identity and returns ecosystem-wide token.
  • GET /auth/verify: Decodes JWT and returns full user profile including phone number.
  • PATCH /users/:userId/trust: Universal endpoint to update KYC/KYB status.

Environment Configuration

Ensure budolShap can reach budolID:

BUDOL_ID_URL=http://localhost:8000

Monitoring Fraud

Check the budolID logs for [Trust Update] or [Registration Error] to monitor for suspicious activity. Accounts with riskLevel: CRITICAL are automatically blocked.
