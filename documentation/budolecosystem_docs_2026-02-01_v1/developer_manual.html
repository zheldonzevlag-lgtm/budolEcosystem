Developer Manual: v1.3.58

1. Backend Implementation (Auth Service)

The backend was updated to support cross-identifier searching in the /login/mobile/identify endpoint and new endpoints for real-time uniqueness checks.

Key Changes in: auth-service/index.js

1.1. Identification Logic

// Detect if input is an email
const isEmail = phoneNumber.includes('@');

if (isEmail) {
  user = await prisma.user.findFirst({
    where: { email: phoneNumber.toLowerCase().trim() }
  });
} else {
  user = await prisma.user.findFirst({
    where: {
      OR: [
        { phoneNumber: phoneNumber },
        { phoneNumber: normalizedPhone },
        { phoneNumber: '+63' + normalizedPhone.substring(1) }
      ]
    }
  });
}

1.2. Uniqueness Endpoints

Added /check-email and /check-phone to provide real-time validation for frontends.

2. Mobile Implementation (Flutter)

Updated the Login and Registration screens with 800ms debounced validation.

Key Changes in: login_screen.dart

void _onIdentifierChanged() {
  if (_debounceTimer?.isActive ?? false) _debounceTimer!.cancel();
  _debounceTimer = Timer(const Duration(milliseconds: 800), () {
    _checkIdentifier(_phoneController.text);
  });
}

3. Web Implementation (React)

Modified AuthForm.jsx to handle the identifier field dynamically.

Key Changes in: AuthForm.jsx

4. Compliance & Security

  • PCI DSS 10.2.2: All identification attempts are logged via createAuditLog.
  • BSP Circular 808: Phone normalization ensures consistent user mapping across the ecosystem.
  • NPC (Data Privacy): Email addresses are trimmed and lowercased before comparison to prevent duplication.
