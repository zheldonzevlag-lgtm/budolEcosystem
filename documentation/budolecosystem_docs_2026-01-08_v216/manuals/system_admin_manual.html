System Admin Manual: SSO User Sync

Operational Procedure

When a user exists in an application database but cannot log in via SSO, follow these steps to sync their account to budolID.

1. Verification

Run a query against the budolID database to check if the user exists:

node -e "const { PrismaClient } = require('@prisma/client'); const prisma = new PrismaClient(); prisma.user.findUnique({ where: { email: 'user@example.com' } }).then(console.log)"

2. Manual Sync

Use the provided sync utility in the budolID-0.1.0/scripts/ directory:

node scripts/sync-galvez-to-budolid.js
Security Note: Always ensure the password provided to the sync script is handled securely and not logged to permanent files.

Test Results

  • Pre-Sync: User galvezjon59@gmail.com was NOT found in budolID.
  • Post-Sync: User successfully created with ADMIN role.
  • SSO Test: Credentials validated against budolID authentication endpoint.
