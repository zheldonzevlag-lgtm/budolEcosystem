Developer Manual: SSO User Sync

Technical Details

The synchronization process involves mapping user data from the application database (budolShap) to the central identity provider (budolID). In this ecosystem, budolID acts as the SSO gateway.

Database Schema Mapping

  • budolShap: Uses password field (hashed).
  • budolID: Uses passwordHash field.

Sync Script (Node.js/Prisma)

The script uses upsert to ensure the user is either created or updated with the correct hash.

const user = await prisma.user.upsert({
    where: { email: 'galvezjon59@gmail.com' },
    update: { passwordHash: hashedPassword },
    create: {
        email: 'galvezjon59@gmail.com',
        passwordHash: hashedPassword,
        role: 'ADMIN',
        isVerified: true
    }
});

Dependencies

  • @prisma/client: For database operations.
  • bcryptjs: For hashing passwords (10 salt rounds).
  • dotenv: For environment configuration.
