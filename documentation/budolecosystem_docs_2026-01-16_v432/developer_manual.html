Developer Manual - v432

Technical Overview

This release introduces a branding update that replaces all instances of "BUDOL_PAY" with "budolPay", where "Pay" is colored rose red (#F43F5E). This is achieved via a reusable utility function that handles dynamic text splitting and styling.

Core Implementation

1. Branding Utility Function

A static method formatBudolPayText was added to UIUtils. It uses RichText to apply different styles to different parts of the string.

File: ui_utils.dart

static Widget formatBudolPayText(String text, {TextStyle? baseStyle, bool useColors = true}) {
  if (!text.contains('BUDOL_PAY')) {
    return Text(text, style: baseStyle);
  }

  final List spans = [];
  final parts = text.split('BUDOL_PAY');

  for (int i = 0; i < parts.length; i++) {
    if (parts[i].isNotEmpty) {
      spans.add(TextSpan(text: parts[i], style: baseStyle));
    }
    
    if (i < parts.length - 1) {
      spans.add(TextSpan(
        children: [
          TextSpan(
            text: 'budol',
            style: (baseStyle ?? const TextStyle()).copyWith(fontWeight: FontWeight.bold),
          ),
          TextSpan(
            text: 'Pay',
            style: (baseStyle ?? const TextStyle()).copyWith(
              color: useColors ? const Color(0xFFF43F5E) : null,
              fontWeight: FontWeight.bold,
            ),
          ),
        ],
      ));
    }
  }

  return RichText(
    text: TextSpan(
      style: baseStyle ?? const TextStyle(color: Colors.black),
      children: spans,
    ),
  );
}

Dependencies & Software Choice Rationale

1. Flutter (Cross-Platform Framework)

Choice: Flutter was chosen for its high-performance rendering engine and single-codebase approach for both Mobile and Web. It allows for rapid UI development and consistent branding across platforms.

2. RichText & TextSpan (Branding Implementation)

Choice: Using Flutter's native RichText and TextSpan was preferred over creating multiple Text widgets in a Row. This approach is more efficient, ensures better text alignment, and allows for precise control over individual word styling within a single block of text.

3. Shared Preferences (Persistence)

Choice: shared_preferences is used for simple key-value storage of flags like hasSeenAds. It is chosen for its ease of use and cross-platform reliability for small amounts of persistent state.

4. Web-JavaScript Interop

Choice: To handle the native browser splash screen, we use dart:js. This is necessary because Flutter's lifecycle does not automatically manage external DOM elements created by the index.html bootstrapper.
