Comparative Analysis: GCash vs. budolPay

This analysis compares the industry-standard tiered KYC model used by GCash with the current and target state of budolPay.

Feature GCash (Standard) budolPay (Target v193+) Rationale
Basic Registration Mobile + OTP Only Mobile + OTP Only Minimal friction for onboarding.
Basic Wallet Limit ₱1k - ₱5k ₱5,000 (Soft Cap) BSP-compliant for unverified users.
P2P Transfers Blocked or Very Low Blocked (Basic) Anti-Money Laundering (AML) safety.
Merchant Payments Small amounts allowed Allowed up to ₱1,000 Enables marketplace testing.
Withdrawal/Cash-out Full KYC Required Full KYC Required Prevents illicit funds exit.

Phase 2: Backend Logic Enforcement

Implement middleware or service-level checks in packages/api to enforce limits based on kycTier.

  • Rule 1: if (user.kycTier === 'BASIC' && transaction.amount > 5000) throw Error("Limit Exceeded");

Phase 3: Frontend UI Adaptations

Modify the Mobile App and Admin Panel to display tier-specific messaging and restricted buttons.

  • Admin: Visual indicators for users nearing their tier limits.

Compliance & Security

By following the Low Identity Confidence + Low Amount = Limited Access logic, budolPay satisfies BSP requirements for tiered KYC while maintaining a smooth user growth funnel.
