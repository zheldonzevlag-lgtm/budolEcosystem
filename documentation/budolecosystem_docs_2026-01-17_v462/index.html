v462 Deep Diagnostic Release

Target: Web Bootstrap Failure Resolution

Diagnostic Strategy

The previous release (v461) failed to initialize the Flutter engine in the DDC (debug) environment, despite scripts loading successfully. v462 implements deep tracing to identify why the glass pane isn't mounting.

Changes

  • Global Error Trap: Added a catch-all listener in index.html that displays a red dialog for any JavaScript error, not just resource failures.
  • Sequential Loading: Removed async from the bootstrap script to ensure window.flutterConfiguration is fully processed before the engine starts.
  • Frame Tracing: Added flutter-first-frame listener to pinpoint the exact moment rendering begins (or fails).
  • Base Href Reset: Reverted to / for debug stability, ensuring consistent path resolution across all environments.
