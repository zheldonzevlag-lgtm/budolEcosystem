System Admin Manual - v187

Infrastructure Changes

This update introduces a database schema change. A new field kycTier has been added to the User table.

Database Migration

CRITICAL: Before running migrations or generating the Prisma client, ensure all Node.js processes (dev servers, services) are stopped to avoid EPERM errors.

Steps to Apply Changes:

  1. Stop all running services (Admin Panel, Verification Service, etc.).
  2. Navigate to packages/database.
  3. Run the following commands: 
    npx prisma migrate dev --name add_kyc_tier
npx prisma generate
  4. Restart all services.

Operational Impact

  • Verification Service: Now automatically upgrades users to SEMI_VERIFIED upon successful document upload.
  • Admin API: Automatically upgrades users to FULLY_VERIFIED when their KYC status is set to VERIFIED by an administrator.

Rollback Procedure

To roll back these changes:

  1. Revert the schema.prisma file to the previous version.
  2. Run npx prisma migrate dev to sync the database (note: this may result in data loss for the kycTier column).
  3. Revert the code changes in apps/admin and services/verification-service.
