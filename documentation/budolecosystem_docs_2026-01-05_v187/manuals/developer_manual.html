Developer Manual - v187

Purpose

Implement Tiered KYC logic based on comparative analysis with GCash. This allows for granular user verification states: Basic, Semi-Verified, and Fully Verified.

Technical Details

1. Prisma Schema Changes

Added KYCTier enum and updated User model to include kycTier field.

enum KYCTier {
  BASIC
  SEMI_VERIFIED
  FULLY_VERIFIED
}

model User {
  // ...
  kycTier KYCTier @default(BASIC)
}

2. Verification Service Update

Updated /upload endpoint in services/verification-service/index.js to set tier to SEMI_VERIFIED upon document submission.

await prisma.user.update({
    where: { id: userId },
    data: { 
        kycStatus: 'PENDING',
        kycTier: 'SEMI_VERIFIED'
    }
});

3. Admin API Update

Updated UPDATE_KYC_STATUS action in apps/admin/app/api/users/route.ts to automatically progress tiers.

if (status === "VERIFIED") {
  kycTier = "FULLY_VERIFIED";
} else if (status === "PENDING") {
  kycTier = "SEMI_VERIFIED";
}

Impact Analysis

  • Database: Requires schema migration.
  • Backend: Verification service now manages user tiers.
  • Admin Panel: Displays user tier in table and detail view.
