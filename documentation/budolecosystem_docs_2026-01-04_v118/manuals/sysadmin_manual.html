System Admin Manual: Secure Reference ID Compliance

Overview

This manual covers the system-level changes and compliance alignment resulting from the new Secure Reference ID generation logic implemented on 2026-01-04.

Compliance Standards

Security Compliance Aligned

The system now uses Cryptographically Secure Pseudo-Random Number Generators (CSPRNG) provided by the Node.js crypto module. This prevents predictability attacks where malicious actors could guess future transaction IDs.

Auditability Compliance Aligned

Every Reference ID now includes a granular timestamp (YYYYMMDDHHMMSS), allowing system administrators to correlate transactions with system logs and external provider timestamps (e.g., PayMongo) more efficiently.

Database Integrity

The Transaction table in the PostgreSQL database enforces uniqueness via a unique index on the referenceId column. The new format significantly reduces the mathematical probability of collisions compared to the previous Math.random() implementation.

Operational Procedures

Identifying Transactions

Administrators searching for transactions in the database or logs should look for the JON- prefix. Example: JON-20260104123045-A1B2C3D4.

Troubleshooting

If a "Duplicate Reference ID" error occurs (highly unlikely), check the system clock synchronization (NTP) to ensure timestamps are advancing correctly across all service nodes.
