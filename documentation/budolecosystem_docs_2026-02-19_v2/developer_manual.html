Developer Manual

Version: v1 | Release Date: 2026-02-19

Purpose & Rationale

This release focuses on enhancing the mobile application's user profile management and enforcing stricter security compliance across the ecosystem. Specifically, it addresses user data visibility issues in the Edit Profile screen and implements comprehensive audit logging and password security standards compliant with BSP Circular 808 and PCI DSS.

Technical Explanation

Mobile App: Profile Data Synchronization

The EditProfileScreen in the mobile app was modified to listen for asynchronous user data updates. Previously, if the user data wasn't fully loaded when the screen initialized, the fields would remain empty. The new implementation uses a listener on the ApiService to populate the text controllers as soon as data becomes available, ensuring a seamless user experience.

Security: Centralized Audit Logging

A centralized audit logging mechanism was implemented in the authentication service. This ensures that all critical actions (login, logout, profile updates, role changes) are logged with timestamps, user IDs, and IP addresses, satisfying regulatory requirements.

    if (user == null) return;

        _firstNameController.text = user['firstName'];
