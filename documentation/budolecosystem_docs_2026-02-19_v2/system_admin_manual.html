Audit Logging

A new centralized audit logging system has been implemented across the Budol Ecosystem to comply with BSP Circular 808 and PCI DSS requirements.

All critical actions, including user authentication (login/logout), profile updates, and administrative role changes, are now logged with:

  • Timestamp
  • User ID
  • Event Type
  • IP Address
  • Success/Failure Status

Logs are stored locally (for now) but can be configured to stream to a centralized logging service (e.g., Elasticsearch, CloudWatch) via the updated logging configuration.

Password Security

All password hashing has been standardized to use `bcryptjs` with a cost factor of 10. This ensures consistency and security across all authentication services (BudolPay, BudolID, BudolShap).
