Mitigation Plan

Risk: Inconsistent Password Hashing

Strategy: Standardized on `bcryptjs` with salt rounds 10 for all password storage.

Action: Updated authentication services in `budolPayMobile`, `budolID`, and `budolShap` to use the standardized hashing function.

Risk: Missing Audit Logs

Strategy: Implement centralized logging for all compliance-sensitive actions.

Action: Integrated `winston`-based logger in the backend to capture user login, logout, profile updates, and role changes with required metadata (IP, user ID, timestamp).

Risk: User Profile Visibility Issues

Strategy: Improve data synchronization in the mobile app.

Action: Implemented a listener in `EditProfileScreen` to reactively update UI fields when user data is fetched asynchronously, preventing empty fields and confusion.
