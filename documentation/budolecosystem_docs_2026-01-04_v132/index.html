Unified Environment Control

Implementation of a centralized environment configuration system and a comprehensive multi-phase plan for migrating the BudolEcosystem to Vercel production infrastructure.

Vercel Migration Plan

01 Phase 1: Environment Variable Centralization

Centralize all secrets and configurations into the `SystemSetting` model with app-specific scoping.

  • Updated `SystemSetting` schema with `appId` and `isActive` fields.
  • Implemented proxy endpoint `/config/:appId` for secure cross-service retrieval.
  • Added Vercel-ready `env-manager.js` for local/remote environment switching.
Completed Duration: 1 Day

02 Phase 2: Remote Database Orchestration

Provisioning and connecting production-grade databases for the Vercel deployment.

  • Setup Vercel Postgres or compatible managed database (Supabase/Neon).
  • Configure `db:vercel` scripts for all core packages.
  • Execute data migration and schema synchronization.
Next Step Estimated: 2 Days

03 Phase 3: Project Deployment & Domain Mapping

Linking codebases to Vercel projects and configuring production domains.

Pending Estimated: 3 Days

Technical Implementation

Database Schema (Prisma) Updated 
model SystemSetting {
  id          String   @id @default(uuid())
  key         String   @unique
  value       String
  appId       String?  // Scoping: 'budolshap', 'budolpay', etc.
  isActive    Boolean  @default(true)
  isSecret    Boolean  @default(false)
  // ... other fields
}
Config Proxy Logic New

Implemented in `wallet-service` to provide scoped configuration to all ecosystem services.

app.get('/config/:appId', async (req, res) => {
  const settings = await prisma.systemSetting.findMany({
    where: { OR: [{ appId: appId }, { appId: null }], isActive: true }
  });
  // Returns mapped key-value pairs
});
