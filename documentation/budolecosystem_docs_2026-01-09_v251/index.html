v251: Continuous Marketing Ad Loop & Mobile Build v1.2.16

Date: 2026-01-09
Version: v251
Status: Completed
Author: Implementation Planner

Executive Summary

This update implements a continuous looping mechanism for the marketing advertisements shown after the splash screen. Additionally, a secondary "GET STARTED" button was added to provide a clear path for users to proceed into the application while the ads loop.

Technical Changes

Marketing Ads Looping

  • Timer Implementation: Added a Timer.periodic (5-second interval) to MarketingAdsScreen for automatic slide transitions.
  • Looping Logic: Modified _onNext and the timer callback to animate back to the first slide (index 0) after reaching the final slide.
  • UX Enhancement: Split the action buttons into "NEXT" (loops through ads) and "GET STARTED" (proceeds to login/home).

Mobile Build (v1.2.16)

  • Version Increment: Bumped pubspec.yaml to 1.2.16+16.
  • Build Stability: Resolved a transient Flutter engine version detection issue with flutter clean.
  • APK Location: The installer is saved at /android-installer/budolpay 1.2.16.apk.

Verification Results

  • Ad Loop: Verified that slides automatically transition every 5 seconds and wrap around from the last to the first slide.
  • Navigation: Verified that the "GET STARTED" button correctly redirects authenticated users to the Home screen and unauthenticated users to Login.
  • Release Build: flutter build apk --release successfully generated the v1.2.16 installer.
