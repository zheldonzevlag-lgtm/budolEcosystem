Strategic Recommendations

Critical

1. Implement Centralized Error Logging (Sentry/Firebase)

Purpose: To proactively detect crashes on user devices without waiting for manual reports.

Mitigation: Currently, we rely on a local Error Boundary. While this prevents white screens, we still don't know *why* a user saw an error unless they report it. Adding Sentry would automate this.

Medium

2. Dynamic API Host Discovery

Purpose: Reduce friction for users on different local networks.

Explanation: Instead of manually entering the IP, the app could use mDNS (Bonjour) or a central registry to find the budolPay server on the local network.

Medium

3. Paged Transaction History

Purpose: Improve performance for long-term users.

Reason: Currently, we fetch the last 50 transactions. As history grows, we should implement infinite scrolling (pagination) to keep the app snappy.

Dependencies & Risks

  • Dependencies: Implementing Sentry requires adding the sentry_flutter package and setting up a DSN.
  • Risk: Over-logging can affect app performance. Mitigation: Use sampled logging for non-critical events.

Consolidated Future Roadmap

  • Biometric Authentication (FaceID/Fingerprint) for Transfers.
  • Offline Mode (Cache last known balance and transactions).
  • Push Notifications for real-time transaction alerts (replacing snackbars).
