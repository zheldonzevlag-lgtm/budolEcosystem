Initialization & Timeouts

In v164, the mobile app now includes a 5-second hard timeout for initial session loading. This prevents the app from being stuck on a splash/white screen if the local storage (SharedPreferences) or network initialization hangs.

Session Verification Policy

The ApiService now performs a more robust session check. If the /auth/verify endpoint returns a 401 Unauthorized or indicates an invalid session, the app will automatically clear local storage and redirect to the login screen. This prevents "Ghost Sessions" where the app thinks it's logged in but the backend rejects all tokens.

Network Requirements

Admins should ensure the following ports are open for mobile connectivity:

  • Port 8000: Auth Service (SSO, Login, Verify)
  • Port 8080: Gateway API (Transactions, Wallet, Socket.io)

Operational Runbook: Resolving persistent white screens

  1. If a user reports a white screen, ask them to check their internet connection.
  2. With the new v164 update, they should see an "Oops" screen instead of a blank white one.
  3. Instruct the user to tap "Reset App & Logout" on the error screen. This clears the local cache and allows a fresh login.
  4. Verify the Server IP in the Login Screen settings (Gear icon) matches the current backend IP.
