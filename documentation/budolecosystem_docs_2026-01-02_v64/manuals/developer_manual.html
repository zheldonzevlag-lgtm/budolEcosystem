Developer Manual - QR Scanning Integration

Purpose

To enable seamless QR code-based payments between budolShap and budolPay Mobile, ensuring a frictionless checkout experience for users.

QR Data Specification

The system uses a structured JSON format encoded within the QR code. This allows the mobile app to validate and display payment details before processing.

{
  "type": "budolpay_payment",
  "orderId": "ORD-12345",
  "amount": 250.00,
  "merchant": "budolShap Merchant",
  "paymentIntentId": "pi_abc123..."
}

Implementation Details

1. budolShap QR Generation

Updated budolshap-0.1.0/app/payment/qr/page.jsx to generate QR codes with the structured payload:

const qrData = {
  type: 'budolpay_payment',
  orderId: orderId,
  amount: amount / 100,
  merchant: 'budolShap',
  paymentIntentId: pi
};
const qrUrl = `https://api.qrserver.com/v1/create-qr-code/?data=${encodeURIComponent(JSON.stringify(qrData))}`;

2. budolPay Mobile Scanner

Implemented QRScannerScreen.dart using the mobile_scanner package. Key logic includes:

  • Validation: Checks if type == 'budolpay_payment'.
  • Parsing: Decodes JSON string into a Dart Map.
  • Navigation: Routes to a PaymentSummaryScreen for user confirmation.

Issues & Fixes

Issue: Web build failure due to mobile_scanner Wasm incompatibilities.
Fix: Used standard JS compilation (flutter build web) instead of Wasm, and handled dart:html imports for web-specific functionality.
Issue: Scope error in PaymentSummaryScreen where variables were inaccessible in async handlers.
Fix: Refactored _handlePayment to extract values from paymentData map at the start of the method.

Recommendations

  • Security: Implement digital signatures for QR data to prevent tampering.
  • UX: Add vibration feedback on successful scan.
  • Deep Linking: Support budolpay://pay?data=... for direct browser-to-app transitions.
