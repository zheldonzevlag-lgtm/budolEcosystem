Issues & Fixes - v452

Issue: Marketing Ads Screen Skipped

Symptoms: The application skips the Marketing Ads screen and goes directly to the Login UI after the Splash screen.

Root Cause: The hasSeenAds flag in SharedPreferences was likely set to true from a previous session, causing the SplashScreen to skip the onboarding flow.

Fix: Navigation Tracing & Reset Tool

  • State Tracing: Added debugPrint statements to log the values of hasSeenAds and isAuthenticated before navigation.
  • Debug Reset: Added a DEBUG: RESET ADS button to the SplashScreen (visible only in debug mode) to allow developers to clear the ads status.
  • Branding Sync: Fixed the budolEcosystem text color in MarketingAdsScreen to match the official Slate 500 branding.
