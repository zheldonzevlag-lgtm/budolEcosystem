System Admin Manual Updates - v196

Infrastructure Changes

No infrastructure changes were required. The fix was purely a software configuration correction.

Deployment Procedures

To apply the fix in a local environment:

  1. Pull the latest changes from the main branch.
  2. Ensure no other service is manually running on port 8006.
  3. Start the service using the ecosystem runner:
    npm run dev --prefix budolpay-0.1.0/services/verification-service

Operational Runbook

Handling Port Conflicts

If you encounter EADDRINUSE errors on port 8006 despite this fix, use the following commands to identify and terminate the zombie process:

Windows:

netstat -ano | findstr :8006
taskkill /F /PID [PROCESS_ID]

Linux/macOS:

lsof -i :8006
kill -9 [PROCESS_ID]

Risk & Mitigation

  • Risk: Service failing to start in production.
  • Mitigation: The !process.env.VERCEL check ensures that on Vercel (Production), the app.listen is skipped, allowing the Vercel handler to manage the request lifecycle correctly.
