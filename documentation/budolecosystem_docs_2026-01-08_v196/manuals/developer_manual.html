Developer Manual Updates - v196

Purpose

To resolve the Error: listen EADDRINUSE: address already in use :::8006 encountered when starting the Verification Service.

Technical Details

The issue was located in budolpay-0.1.0/services/verification-service/index.js. The service contained two consecutive app.listen calls:

Code Logic Explanation

Previously, the code attempted to start the server twice:

// Call 1 (Conditional)
if (process.env.NODE_ENV !== 'production' || !process.env.VERCEL) {
    app.listen(PORT, () => { ... });
}

// Call 2 (Unconditional)
app.listen(PORT, () => { ... });

In a local development environment, both conditions would evaluate to true, causing the second call to fail because the port was already claimed by the first call.

Issues & Fixes

  • Issue: Redundant server listener causing startup crash.
  • Fix: Removed the second app.listen and refined the conditional logic for the first one.
  • Code Changes: 
    // Refined Logic
if (process.env.NODE_ENV !== 'production' && !process.env.VERCEL) {
    app.listen(PORT, () => {
        console.log(`[budolPay (KYC)] Service running on port ${PORT}`);
    });
}

Impact Analysis

This change has zero impact on the functional logic of the KYC service. It only affects the startup sequence, ensuring the service remains stable in local and serverless environments.

Test Results

  • Port Conflict Test: Verified port 8006 is free after fix.
  • Service Startup: Service starts successfully with nodemon.
  • Health Check: GET /health returns 200 OK.
