budolEcosystem Documentation v128

Date: 2026-01-04

Focus: API Gateway Optimization & UI Refinement

Purpose

To resolve persistent 404/BadRequest errors in the API Gateway and fulfill user requests for Payment Summary UI improvements. This version also introduces session recovery mechanisms in the mobile app.

Issues Addressed

  • API Gateway Proxy Failure: The gateway was using express.json() before proxying, which consumed the request body stream, leading to BadRequestError or 404 on proxied routes (e.g., /wallet/balance).
  • UI Mismatch: The Payment Summary screen prioritized "Store Name" over "Payment Method", which didn't align with user preference for transaction clarity.
  • Transient Session Loss: Occasional "User session expired" errors despite valid tokens suggested a need for more robust session state recovery.

Technical Implementation

  • API Gateway (Refactoring): Moved app.use(express.json()) after the proxy middleware registrations in api-gateway/index.js. This ensures the raw request stream reaches the target services (Wallet, Auth, etc.) intact.
  • Mobile UI (qr_scanner_screen.dart):
    • Replaced the "Pay to [Store Name]" header with the "Payment Method" (e.g., budolPay Wallet).
    • Demoted the Store Name to a subtitle: "Payment for [Merchant]".
    • Updated iconography to reflect wallet-based payments.
  • Session Recovery (ApiService): Added a proactive _loadSession() call within processPayment as a failsafe if the in-memory user object is null but a token exists.
  • Testing: Updated payment_summary_test.dart to validate the new UI hierarchy and ensures all 8 critical transaction data points are displayed correctly.

Test Results

  • Unit Tests: payment_summary_test.dart PASSED (0.0:11).
  • Gateway Connectivity: Verified that proxied routes no longer suffer from stream consumption issues.
  • Build: Successfully built budolPay 1.1.0.apk with the fix for icon tree-shaking.
