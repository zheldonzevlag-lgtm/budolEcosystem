Developer Manual: PIN Recovery & Mobile Transaction Bridge

Purpose & Rationale

This release addresses the synchronization gap between the Flutter application client and the Vercel (Next.js) backend monorepo. Specifically, it resolves the legacy parameter mismatch between the mobile app's userId / pin and the backend's expected phoneNumber / tempToken format.

Key Code References

1. Mobile API Target Fixed (Flutter)

Transitioned from localhost to production-first resolution logic across all mobile environments.

📂 lib/services/api_service.dart 
return const String.fromEnvironment(
  'API_HOST',
  defaultValue: 'budolpay.vercel.app',
);

2. PIN Reset Sequence (Vercel Backend)

The backend now correctly clears internal verification flags to re-enter the setup state after OTP success.

📂 app/api/auth/login/mobile/reset-pin/route.ts 
await prisma.user.update({
  where: { id: user.id },
  data: {
    pinHash: null,
    otpCode: null
  }
});

3. P2P Transfer Security & KYC Tier Check

Ensures that users are properly verified according to the risk-based BASIC vs FULLY_VERIFIED tiers (BSP Circular No. 920).

📂 app/api/transactions/transfer/route.ts 
if (sender.kycTier === 'BASIC') {
    throw new Error("Verification required: BASIC accounts cannot send money P2P.");
}
