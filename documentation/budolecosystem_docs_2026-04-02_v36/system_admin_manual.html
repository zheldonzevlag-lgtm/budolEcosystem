System Admin Manual: Vercel & Mobile Deployment

Production Endpoint Synchronization

This release confirms the full migration of the mobile authentication bridge to Vercel production. All administrative functions and mobile API endpoints are consolidated onto budolpay.vercel.app.

Deployment Branch: main
Root Dir: budolpay-0.1.0/apps/admin
Log Monitoring: Vercel Dashboard → Logs → Filter for /api/auth/login/mobile/reset-pin

Manual KYC Verification Overrides

To enable testing on new accounts like 09123456789 or developer test numbers on NoxPlayer, administrators may bypass the selfie verification stage via the prisma command below. Never run this in Production for non-test accounts.

// Upgrade target testing number to FULLY_VERIFIED
await prisma.user.update({
  where: { phoneNumber: "09484099400" },
  data: { kycTier: 'FULLY_VERIFIED' }
});

Security Audit Tasks

  • PIN Reset Audits: Filter the AuditLog table for MOBILE_PIN_RESET_REQUESTED to verify reset requests align with OTP patterns.
  • MFA Failure Lockouts: Monitor for VERIFY_PIN_FAILED events. If a device exceeds 5 failures, it should be manually reviewed for suspicious pattern behavior according to BSP/Cybersecurity law standards.
