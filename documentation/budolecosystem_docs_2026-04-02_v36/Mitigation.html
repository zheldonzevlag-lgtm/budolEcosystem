Mitigation Strategies: Compliance & Security

BSP Circular No. 808 (E-Banking Standards)

MFA Verification Implementation:

Our PIN reset flow (Build 1.3.77) enforces Multi-Factor Authentication (MFA) via SMS OTP. The system restricts any PIN modification unless a 6-digit code tied to the account's confirmed primary mobile number is successfully verified by the Vercel backend.

BSP Circular No. 920 (Anti-Money Laundering)

KYC Tier Enforcement:

Unverified (BASIC) accounts are restricted to low-risk deposit actions (Cash-In) and bill payments only. High-risk P2P money transfers are programmatically blocked until the system receives a FULLY_VERIFIED status after manual/automated ID verification.

PCI DSS 10.2.1 Audit Traceability

Security Auditing:

The AuditLog table records all sensitive authorization attempts (MOBILE_PIN_RESET_REQUESTED, VERIFY_PIN_FAILED). This gives investigators clear traceability of credential tampering attempts in the production database.
