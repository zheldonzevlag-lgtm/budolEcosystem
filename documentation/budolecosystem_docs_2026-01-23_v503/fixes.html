Fixes Log

Solutions implemented to resolve issues in v503.

F503-01: Enhanced SWR Cache Mutation

Description: Refactored RealtimeProvider to include isPaid, paymentStatus, and shipping metadata in Pusher events. This allows SWR to match orders against tab filters (e.g., isCancelledTab) and update the UI correctly without a reload.

Impact: Instant UI updates across all order filter tabs.

F503-04: Dynamic Page-Specific Search Implementation Plan

Description: Implemented a unified search system that adapts to the current page context. This was achieved by migrating local search states to a global SearchContext and updating backend API handlers to support standardized search parameters.

Implementation Steps:

  • Infrastructure: Created SearchContext.jsx to hold global searchQuery and placeholder state.
  • Dynamic Routing: Integrated usePathname in SearchContext to reset search and update placeholders automatically on navigation.
  • Global Layout: Wrapped the application in SearchProvider and updated Navbar to use the global state.
  • Backend Standardization: Refactored ordersService and various API routes (Products, Users, Stores, Returns, Coupons, Payouts, Memberships) to support case-insensitive Prisma contains filtering via a common search parameter.
  • Frontend Migration: Updated 10+ frontend pages to consume searchQuery from context and trigger re-fetches on change.

Impact: A seamless, intuitive search experience that works across all administrative and customer-facing lists without state conflicts.

F503-02: Dynamic Pathname-Based Search Context

Description: Updated SearchContext to use Next.js usePathname. The search query is now reset on navigation, and placeholders are dynamically updated based on the current route.

Impact: Prevents search term "leakage" between different sections of the app.

F503-03: Webhook Environment Isolation

Description: Corrected the signature verification logic to handle local development environment differences and ensured that the BASE_URL in test scripts matches the running instance (3000 vs 3001).

Impact: Reliable testing and production security.
