System Admin Manual

Administration and maintenance guide for v503 update.

1. Real-time Provider Configuration

Ensure that Pusher or Socket.io is correctly configured in the System Settings. Admin-configured providers are now prioritized over SWR polling to reduce server load and improve latency.

Critical: If using Pusher, ensure the Cluster and Key are correct. The system will automatically fallback to SWR polling (every 10 seconds) if the provider fails to connect.

2. Webhook Monitoring

Lalamove webhooks are now mapped to the UNIVERSAL_STATUS set. If an order status does not update, check the logs for "Webhook signature missing" or "Invalid signature" errors.

  • Endpoint: /api/webhooks/lalamove
  • Required Secret: LALAMOVE_WEBHOOK_SECRET in .env

3. Cache Management

The system settings cache has been optimized with a reduced TTL of 10 seconds (down from 60s) to ensure that administrative changes (like switching real-time providers) take effect almost immediately across all server instances.

4. Compliance Verification

Standard Compliance Action in v503
PCI DSS Search logic excludes sensitive payment data (tokens, full CVVs) from indexing and query results.
BSP & NPC Customer data in search results is filtered based on administrative permissions (RBAC).
BIR Real-time sync ensures that order statuses for tax-relevant transactions (Paid/Delivered) are accurately recorded in the audit log.
Cybersecurity Law Mandatory signature verification for all incoming Lalamove webhooks to prevent spoofing.
