Biometric Risk Mitigation Plan

Version: v362

Mitigation Strategies

M-BIO-01: Liveness Enforcement (Anti-Spoofing)

Action: Implemented active liveness (Blink/Smile) in Phase 1. Further enhancement with head rotation in Phase 3.

Target Risk: R-BIO-01

M-BIO-02: Template Encryption

Action: Use only mathematical embeddings (128D vectors). Ensure backend storage uses AES-256 encryption at rest.

Target Risk: R-BIO-02

M-BIO-03: Diverse Model Training

Action: Select TFLite models trained on diverse datasets (e.g., FaceNet, MobileFaceNet) to minimize demographic bias.

Target Risk: R-BIO-03

M-BIO-04: Integrity Verification

Action: Implement checksum verification for the facenet.tflite asset at app startup to detect tampering.

Target Risk: R-BIO-04
