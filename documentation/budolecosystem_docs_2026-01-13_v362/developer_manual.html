Developer Manual: Face Recognition Implementation

Version: v362

Purpose: Technical guide for the full end-to-end implementation of the Face Recognition system (Phases 1-4).

1. Architecture Overview

The system uses a hybrid approach for biometric security:

  • Liveness Detection: On-device ML Kit (Blink/Smile detection) to prevent spoofing.
  • Face Embedding: FaceNet TFLite model generates a 128-dimension mathematical template.
  • Backend Storage: Templates are stored in PostgreSQL via Prisma for server-side verification and cross-device synchronization.
  • Biometric Login: Local storage of templates for fast, secure authentication without repeated server calls.

Data Flow

  1. Enrollment (KYC): Captured face is converted to embedding and sent to backend.
  2. Storage: Backend updates User and VerificationDocument models.
  3. Local Link: On successful login, the app offers to "Register Face Login", storing the template in FlutterSecureStorage.
  4. Verification (Login): Captured selfie is compared locally against the stored template using Cosine Similarity.

Example: Cosine Similarity Calculation

// In face_embedding_service.dart
double compare(List<double> emb1, List<double> emb2) {
  double dotProduct = 0.0;
  double normA = 0.0;
  double normB = 0.0;
  for (int i = 0; i < emb1.length; i++) {
    dotProduct += emb1[i] * emb2[i];
    normA += pow(emb1[i], 2);
    normB += pow(emb2[i], 2);
  }
  return dotProduct / (sqrt(normA) * sqrt(normB));
}

4. Compliance

NPC / GDPR: We store only mathematical embeddings, never raw facial images for matching purposes. This adheres to privacy-by-design principles.
