KYC UI Improvements & Regulatory Compliance Roadmap

v240 (2026-01-08)
← Previous Version (v239) Main Index Next Version (v241) →

Purpose

This phase focused on enhancing the mobile application's User Experience (UX) for verified users by implementing conditional UI logic for KYC-related elements. Additionally, a comprehensive roadmap for Philippine regulatory compliance (BSP, NPC, AMLC) was established to ensure the platform's long-term legality and security.

UI Implementation Details

1. Conditional Visibility for "Verify Account"

Modified SettingsScreen to hide the "Verify Account" tile once a user achieves FULLY_VERIFIED status or has a VERIFIED KYC status. This follows the industry-standard UX pattern observed in leading Philippine digital banks like GoTyme.

if (!isFullyVerified)
  _buildSettingsTile(Icons.verified_user, 'Verify Account', 
    kycStatus == 'PENDING' ? 'Verification in progress' : 'Get higher limits', () {
    // Navigation logic...
  }),

2. Dynamic User Tier Labeling

Updated the profile header to display the user's current verification tier with appropriate color coding:

  • Basic User (Amber): No verification or rejected.
  • Under Review (Blue): Verification documents submitted and pending approval.
  • Fully Verified (Green): KYC approved and tier upgraded.

Philippine Regulatory Compliance Roadmap

BSP Circular 808/982

IT Risk Management: Implementation of Multi-Factor Authentication (MFA), secure API gateways with rate limiting, and real-time transaction monitoring for fraud detection.

Data Privacy Act (RA 10173)

NPC Compliance: Explicit consent management, data minimization, and encryption of PII at rest and in transit. Appointment of a Data Protection Officer (DPO) is recommended.

AMLA (RA 9160)

KYC/AML: Tiered KYC limits, sanctions screening against AMLC/OFAC lists, and suspicious transaction reporting (STR) capabilities.

Consumer Protection

BSP Circular 1160: Transparent fee disclosure, standardized receipting, and a robust redress mechanism (Help Center/Dispute management).

Verification & Test Results

  • Code Integrity: Verified that isFullyVerified logic correctly handles both kycTier and kycStatus fields from the backend.
  • UI Consistency: Confirmed the "Verify Account" tile disappears only when the verification is complete.
  • Static Analysis: Ran flutter analyze to ensure no regressions.

Recommendations for Future Enhancement

  • Biometric Integration: Implement FIDO2/WebAuthn for passwordless, secure authentication (Requirement for higher trust tiers).
  • Real-time Document OCR: Integrate an OCR provider (e.g., Regula or Jumio) for instant ID verification in the mobile app.
  • In-App Notifications: Add push notifications for KYC status updates (Approved/Rejected/Pending).
