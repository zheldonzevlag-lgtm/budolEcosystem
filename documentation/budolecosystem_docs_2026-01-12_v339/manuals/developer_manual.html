Developer Manual: Admin Build Stability & LAN Compliance

Purpose & Rationale

Objective: Restore the ecosystem to a functional state with stable Admin Service builds, LAN IP compliance (192.168.1.17), and RBAC integration.

Rationale: The Admin service required build optimizations for Prisma and dynamic rendering to prevent prerendering errors. LAN compliance is critical for cross-device connectivity (mobile app to backend) in a local network environment. RBAC provides the foundation for secure administrative access.

Technical Implementation

1. Admin Service Build Optimization

To resolve build-time errors related to Prisma paths and static generation in Next.js 14, the following changes were implemented:

File: next.config.mjs

import path from 'path';
import { fileURLToPath } from 'url';

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

/** @type {import('next').NextConfig} */
const nextConfig = {
  transpilePackages: ['@budolpay/database'],
  experimental: {
    serverComponentsExternalPackages: ['@prisma/client'],
  },
  env: {
    DATABASE_URL: process.env.DATABASE_URL || `file:${path.join(__dirname, '../../packages/database/prisma/dev.db')}`,
  },
  output: 'standalone',
};

export default nextConfig;

Forced dynamic rendering for Prisma-dependent pages:

File: page.tsx

export const dynamic = 'force-dynamic';

export default async function RealtimeSettingsPage() {
  const allSettings = await prisma.systemSetting.findMany({ ... });
  // ...
}

2. LAN IP Compliance (192.168.1.17)

Standardized all microservices and frontend applications to use the LAN IP 192.168.1.17 instead of 127.0.0.1 or localhost to enable mobile app connectivity.

Services Updated:

  • auth-service/index.js
  • transaction-service/index.js
  • api-gateway/index.js
  • wallet-service/index.js

3. SSO & Auth Route Restoration

Fixed SSO callback and login routes in the Admin app to correctly handle the LAN IP for authentication redirects.

File: callback/route.ts

4. RBAC Phase 1 Integration

Integrated basic Role-Based Access Control middleware and configuration in the @budolpay/database package.
