System Administration: v339

This manual covers the deployment and maintenance of the BudolPay v339 ecosystem, focusing on the optimized Admin Service and LAN-compliant networking.

1. Networking & IP Compliance

As of v339, all services are standardized to communicate via the LAN IP 192.168.1.17. This is a critical change from loopback (127.0.0.1) to support cross-device testing and mobile application integration.

Important: Ensure your server's static IP is set to 192.168.1.17. If your LAN uses a different subnet, update the LOCAL_IP environment variable in the root .env file.

2. Admin Service Deployment

The Admin Service (Next.js) is now configured for standalone output. This significantly reduces the deployment footprint by only including necessary node_modules.

To build the admin service with optimized Prisma path resolution:

# From the admin workspace
cd budolpay-0.1.0/apps/admin
npm run build

The build output will be located in .next/standalone.

3. RBAC Configuration

Role-Based Access Control is managed in packages/database/rbac-config.js. Administrators can define permission sets for different roles:

const ROLES = {
  ADMIN: 'ADMIN',
  STAFF: 'STAFF',
  USER: 'USER'
};

const PERMISSIONS = {
  TRANSACTION_READ_ALL: 'transaction:read:all',
  SYSTEM_SETTINGS_WRITE: 'system:settings:write',
  // ...
};

Ensure that the JWT_SECRET is synchronized across all microservices to maintain consistent auth verification.

4. Monitoring & Logging

The API Gateway (port 8080) serves as the primary entry point and logs all RBAC-blocked attempts for compliance with BSP Circular 808 standards.

  • Gateway Logs: Monitor for [RBAC] Access Denied entries.
  • Health Checks: http://192.168.1.17:8080/health
  • Admin Build: Verify standalone output stability post-deployment.
