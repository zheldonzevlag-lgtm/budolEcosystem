Risk Register

This document identifies potential risks associated with the implementation of phone number normalization and email login fallback.

Risk ID Description Likelihood Impact Severity Status Mitigation Strategy
R-001 BudolID Service Downtime: If BudolID is unavailable, email login fallback will fail, potentially locking out users with stale local passwords. Low High High Open Implement caching of last successful BudolID validation or allow temporary grace login if recent activity exists. Monitor BudolID uptime.
R-002 Incorrect Phone Normalization: Edge cases in phone number formats (e.g., landlines entered as mobile) might be incorrectly processed or rejected. Medium Medium Medium Monitored Comprehensive unit testing of PhoneUtils with various valid/invalid inputs. Log normalization failures for review.
R-003 Data Inconsistency (Legacy Data): Existing user records with non-normalized phone numbers may cause lookup failures or duplicates. High Medium Medium Active Run a database migration script to normalize existing phone numbers. Update all lookup queries to handle both formats temporarily.
R-004 Security - Brute Force on Fallback: Attackers might exploit the fallback mechanism to bypass rate limits on the primary authentication flow. Low High High Mitigated Ensure rate limiting applies to the entire login route, regardless of the authentication path taken (local vs. BudolID).
