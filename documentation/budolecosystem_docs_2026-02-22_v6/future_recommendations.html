Future Recommendations

Suggestions for future improvements based on the current implementation.

1. Transition to Exclusive BudolID Auth

Priority: High

Currently, we maintain a hybrid state (local password + BudolID fallback). To simplify architecture and security, we should aim to remove local password storage entirely and rely solely on BudolID OAuth/OIDC tokens for authentication.

2. Two-Factor Authentication (2FA)

Priority: High

Implement SMS or App-based 2FA for all logins, especially when a fallback login occurs or when a new device is detected.

3. Real-time Phone Validation

Priority: Medium

Integrate an external API (e.g., Twilio or local telco API) to verify if the phone number is active and reachable via SMS before allowing registration.

4. Automated Data Migration

Priority: Medium

Automate the cleanup of legacy non-normalized phone numbers via a background job instead of manual admin scripts.
