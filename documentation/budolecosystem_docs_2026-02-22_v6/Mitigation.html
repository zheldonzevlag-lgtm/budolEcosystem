Mitigation Strategies

This document details the strategies employed to address the risks identified in the Risk Register.

R-001: BudolID Service Downtime

Strategy: Graceful Degradation & User Notification

  • User Notification: If BudolID is unreachable, inform the user with a specific message: "Authentication service temporarily unavailable. Please try again later or use your local password."
  • Timeout Handling: Set a strict timeout (e.g., 5 seconds) for BudolID calls to prevent hanging the login process.
  • Caching (Future): Consider caching successful BudolID validation tokens locally with a short expiration to allow offline login capabilities in extreme cases.

R-002: Incorrect Phone Normalization

Strategy: Validation & Unit Testing

  • Strict Regex: Use a strict regex for phone validation that only accepts Philippine mobile prefixes (09, 639, +639).
  • Unit Tests: Implement a comprehensive suite of unit tests covering all known edge cases (e.g., spaces, dashes, parentheses, landlines).
  • Client-Side Validation: Provide real-time feedback in the UI as the user types, ensuring they correct invalid formats before submission.

R-003: Data Inconsistency (Legacy Data)

Strategy: Data Migration & Dual Lookup

  • Migration Script: Develop and execute a script to normalize all existing phone numbers in the database to E.164 format.
  • Dual Lookup (Transitional): Update authentication queries to search for both the normalized and raw input formats until the migration is confirmed complete.
  • Audit Log: Log every instance where a legacy format is successfully used for login to track migration progress.

R-004: Security - Brute Force on Fallback

Strategy: Rate Limiting & Monitoring

  • Global Rate Limit: Apply rate limiting to the /api/auth/login endpoint based on IP address and user identifier, regardless of the backend authentication method.
  • Account Lockout: Temporarily lock accounts after 5 failed attempts within 1 minute.
  • Monitoring: Alert administrators if a sudden spike in fallback authentication attempts is detected.
