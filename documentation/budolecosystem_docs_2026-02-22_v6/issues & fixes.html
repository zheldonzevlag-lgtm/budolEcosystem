Issues & Fixes Log

This document records the issues encountered during development and testing, along with their resolutions.

Issue ID Description Status Fix / Resolution Date
I-001 Login Failure via Email: Users who updated their password on BudolID could not log in to BudolShap using their email because the local password hash was outdated. Resolved Implemented BudolID fallback authentication in the login route. Upon successful fallback, the local password hash is automatically updated to match the input password. 2026-02-22
I-002 Inconsistent Phone Formats: Mobile app allowed various phone number formats (e.g., 09xx, +639xx), causing backend validation errors or duplicate accounts. Resolved Created PhoneUtils class to normalize all inputs to E.164 (+639...) format. Updated API service to use this utility before sending data. 2026-02-22
I-003 Documentation Creation Error: mkdir command failed due to syntax error in PowerShell environment. Resolved Switched to using New-Item or direct file writing which handles directory creation implicitly/explicitly correctly. 2026-02-22
I-004 BudolID Connection Failure (Port Mismatch): Fallback authentication failed because budolshap was defaulting to port 8000 (SSO/Gateway) while the Auth Service runs on port 8001. This caused "Invalid email or password" errors even with valid credentials. Resolved Updated budolIdClient.js to prioritize AUTH_SERVICE_URL (port 8001) and adjust the endpoint path (removing /auth prefix for direct service calls). Added robust logging to diagnose connection issues. 2026-02-23
I-005 Legacy Password Format (Plain Text): Some users had passwords stored as plain text in the database (e.g., from initial testing or migration). The application expected bcrypt hashes, causing verifyPassword to fail even with correct credentials, and the BudolID fallback also failed due to user sync issues. Resolved Updated verifyPassword in lib/auth.js to support legacy plain text passwords as a fallback. Modified login route to automatically upgrade plain text passwords to secure bcrypt hashes upon successful login. 2026-02-23
