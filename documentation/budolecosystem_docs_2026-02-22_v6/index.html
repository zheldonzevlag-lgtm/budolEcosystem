Budol Ecosystem Documentation v6

Date: 2026-02-22

Version: 6.0.0

Overview

This release focuses on improving authentication reliability and data consistency across the Budol Ecosystem.

  • Phone Number Normalization: Implemented E.164 standard formatting (+639xxxxxxxxx) for all phone number inputs in the BudolPay mobile app to ensure consistency with backend systems.
  • Email Login Fallback: Enhanced the BudolShap web application login process to automatically fallback to BudolID verification if local password authentication fails. This resolves login issues for users who updated credentials externally.

Change Log

Component Change Type Description
BudolPay Mobile Enhancement Added PhoneUtils for centralized phone number normalization. Updated API service to normalize inputs.
BudolShap Web Bug Fix Implemented BudolID fallback authentication in /api/auth/login route.
BudolShap Web Enhancement Automatic local password synchronization upon successful BudolID fallback login.
