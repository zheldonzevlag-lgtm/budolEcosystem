Developer Manual

This document details the technical implementation of phone number normalization and email login fallback.

1. Phone Number Normalization (BudolPay Mobile)

Purpose: To ensure all phone numbers stored and processed by the system adhere to the E.164 standard for the Philippines (+639xxxxxxxxx).

Implementation Details

A centralized utility class PhoneUtils was created to handle all phone number parsing and formatting.

d:\IT Projects\budolEcosystem\budolPayMobile\lib\utils\phone_utils.dart 
class PhoneUtils {
  static String? normalizePhoneNumber(String? phone) {
    if (phone == null || phone.isEmpty) return null;
    String cleaned = phone.replaceAll(RegExp(r'[^0-9+]'), '');
    if (cleaned.startsWith('+63') && cleaned.length == 13) return cleaned;
    else if (cleaned.startsWith('63') && cleaned.length == 12) return '+$cleaned';
    else if (cleaned.startsWith('09') && cleaned.length == 11) return '+63${cleaned.substring(1)}';
    else if (cleaned.startsWith('9') && cleaned.length == 10) return '+63$cleaned';
    return cleaned;
  }
}

Integration in API Service

The ApiService was updated to use PhoneUtils.normalizePhoneNumber before making any requests involving phone numbers.

d:\IT Projects\budolEcosystem\budolPayMobile\lib\services\api_service.dart 
Future<Map<String, dynamic>> identifyMobile(String phoneNumber) async {
  final normalizedPhone = PhoneUtils.normalizePhoneNumber(phoneNumber) ?? phoneNumber;
  final url = '$authUrl/login/mobile/identify';
  // ... sends normalizedPhone to backend
}

2. Email Login Fallback (BudolShap Web)

Purpose: To allow users to login via email even if their local password hash is missing or outdated (e.g., if they reset their password via BudolID).

Implementation Details

The login route now attempts a fallback to BudolID authentication if the local password check fails. Upon success, it synchronizes the local password.

d:\IT Projects\budolEcosystem\budolshap-0.1.0\app\api\auth\login\route.js 
// Verify password
isValidPassword = await verifyPassword(password, user.password)

if (!isValidPassword) {
    // Fallback: Try to authenticate against BudolID directly
    try {
        console.log(`[Login] Local password failed. Attempting fallback.`);
        const budolLogin = await loginWithBudolId(searchIdentifier, password);
        
        if (budolLogin && budolLogin.user) {
            isValidPassword = true;
            
            // Update local password to match BudolID
            const hashedPassword = await hashPassword(password);
            await prisma.user.update({
                where: { id: user.id },
                data: { password: hashedPassword }
            });
        }
    } catch (e) {
        console.warn(`[Login] BudolID fallback failed: ${e.message}`);
    }
}
