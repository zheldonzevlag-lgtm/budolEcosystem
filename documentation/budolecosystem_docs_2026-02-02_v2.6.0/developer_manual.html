Developer Manual: Audit Trail Identity Attribution

1. Technical Rationale

To meet PCI DSS 10.2.2 requirements, every administrative action must be attributed to a specific user. Previously, server actions in the system settings modules were creating audit logs without a userId, resulting in "System" appearing as the actor in the Forensic Audit Trail UI.

2. Implementation Architecture

The solution involved implementing a standardized getCurrentUser() helper function in server components to retrieve the authenticated staff identity from the SSO service via the budolpay_token cookie.

SSO User Retrieval Helper & Foreign Key Safety

Ensures consistent identity resolution while preventing foreign key constraint violations if the user does not exist in the local database.

async function getCurrentUser() { const cookieStore = cookies(); const token = cookieStore.get('budolpay_token')?.value; if (!token) return null; try { const LOCAL_IP = process.env.LOCAL_IP || 'localhost'; const ssoUrl = process.env.SSO_URL || `http://${LOCAL_IP}:8000`; const response = await fetch(`${ssoUrl}/auth/verify`, { headers: { 'Authorization': `Bearer ${token}` } }); const data = await response.json(); if (!data.valid || !data.user) return null; // Verify user exists in local database to avoid foreign key violations (PCI DSS 10.2.2) const localUser = await prisma.user.findFirst({ where: { OR: [ { id: data.user.id }, { email: data.user.email } ] } }); if (localUser) { return localUser; } // Return SSO user data but ensure ID is null if not found locally to prevent FK crash return { ...data.user, id: null, ssoId: data.user.id }; } catch (e) { return null; } }

3. Code References & Changes

The following files were updated to include identity attribution and compliance metadata:

  • Environment Settings: environment/page.tsx
    Added userId to CHANGE_ENV_VAR logs.
  • Realtime Settings: realtime/page.tsx
    Added UPDATE_REALTIME_CONFIG logs.
  • Notification Settings: notifications/page.tsx
    Added granular logs for SMTP, Twilio, and Pusher configurations.
  • Security Policy Settings: security/page.tsx
    Added UPDATE_SECURITY_POLICY logs.
  • Location/Map Settings: location/page.tsx
    Added UPDATE_MAP_CONFIG logs.
  • Security API Filter: security/route.ts
    Updated "System" filter to include SystemSetting entity and new action patterns.

4. Compliance Metadata

All audit logs now include a metadata.compliance block for easier auditing:

metadata: { compliance: { pci_dss: "10.2.2", bsp: "Circular 808" } }

5. Verification Testing

Two specialized test scripts were developed to verify identity attribution and system stability:

  • Identity Attribution Test: test-audit-identity.js - Verifies staff identity appears in logs.
  • FK Safety Test: test-audit-fk-safety.js - Verifies no crashes occur if user is not local.
