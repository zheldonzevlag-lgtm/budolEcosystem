Resolved Fixes - v2.6.0

Staff Identity Attribution

Resolved the issue where system configuration changes were logged without a userId. All new logs now correctly attribute the action to the authenticated administrator.

Audit Log Filter Visibility

Updated the Security API filter logic to ensure SystemSetting entity logs are correctly displayed when the "System" filter is selected in the dashboard.

Configuration Coverage Gap

Extended forensic audit logging to include Realtime, Notification, Security Policy, and Location settings, which were previously un-audited.

Foreign Key Safety (FK Violation Prevention)

Implemented a safety check in getCurrentUser() to verify if an SSO user exists in the local database before attempting to link an audit log. This prevents database crashes (FK constraint violations) for users who are not yet synced locally.

Syntax & Build Error Remediation

Resolved "Expected '(', got 'saveSecuritySettings'" and other syntax errors across settings pages (Security, Notifications, Realtime, Location) caused by incomplete code blocks during the audit logging migration.
