System Admin Manual: Enhanced Forensic Auditing

Overview

In version 2.6.0, the Forensic Audit Trail has been enhanced to provide full accountability for system configuration changes. Administrators can now see exactly who changed what setting and when.

Viewing the Audit Trail

To access the Forensic Audit Trail:

  1. Log in to the BudolPay Admin Dashboard.
  2. Navigate to the Security & Compliance section in the sidebar.
  3. Locate the Forensic Audit Trail card.
  4. Use the "System" filter to view configuration changes.

Identity in Audit Logs

Each entry now displays the staff member's name and email. Entries that were previously marked as "System" will now correctly show the acting administrator for all new changes.

Supported Audit Modules

The following configuration modules are now fully audited with staff identity:

  • Environment Variables: Changes to API keys, URLs, and system flags.
  • Realtime Engine: Updates to Socket.io or Pusher configurations.
  • Notifications: SMTP, Twilio, and Pusher provider settings.
  • Security Policies: Session timeouts, password requirements, and 2FA settings.
  • Map Services: Google Maps or Mapbox provider updates.
Pro Tip: All audit logs are immutable and stored in the secure database. They cannot be modified or deleted, ensuring compliance with BSP Circular 808.
