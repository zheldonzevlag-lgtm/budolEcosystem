BudolEcosystem Implementation Plan v190

Mobile Dynamic Host Discovery & Route Centralization

← Previous Version (v189) Main Documentation Index Next Version →

Purpose

The goal of this implementation is to eliminate the fragility of hardcoded IP addresses and route strings in the BudolPay mobile app. By implementing mDNS (Multicast DNS) discovery, the mobile app can automatically find the API Gateway on the local network, significantly improving the developer and user experience in local environments.

Technical Details

1. mDNS Advertisement

The API Gateway now advertises itself on the local network using the bonjour-service package. It broadcasts as _budolpay._tcp.local.

  • Service Name: budolpay-gateway
  • Service Type: budolpay
  • Protocol: tcp

2. Mobile Discovery

The Flutter app uses the multicast_dns package to scan the network for the gateway service during initialization or via manual trigger.

  • Package: multicast_dns: ^0.3.2+2
  • Service: DiscoveryService

3. Route Centralization

All route strings are now centralized in a dedicated constants class to prevent typos and simplify navigation management.

  • File: lib/constants/routes.dart
  • Usage: Navigator.pushNamed(context, Routes.home)

Implementation Summary

  • Gateway Update: Installed bonjour-service and added advertisement logic to api-gateway/index.js.
  • Flutter Dependencies: Added multicast_dns to pubspec.yaml.
  • Route Constants: Created Routes class and refactored main.dart and all screen files.
  • Discovery Logic: Implemented DiscoveryService and integrated it into ApiService.
  • Settings UI: Added "Gateway Host" configuration with manual override and network scanning capabilities.

Risks & Mitigations

Risk Mitigation
Firewall Blocking mDNS Provided manual IP override in Settings screen as a fallback.
Multiple Gateways on Network Service picks the first discovered; manual override allows selection if needed.
Network Latency during Scan Implemented async scanning with UI loading indicators.

Future Enhancements

  • Implement service health checks before switching to a discovered host.
  • Add support for QR code scanning of host configuration for even easier setup.
  • Implement encrypted mDNS for improved security in shared networks.
