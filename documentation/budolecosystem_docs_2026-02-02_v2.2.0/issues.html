Identified Issues - v2.2.0

1. Missing Navigation for Realtime/Notifications

Description: Admin users had no direct way to configure Realtime (Pusher) or Notification (Email/SMS) settings through the sidebar UI, requiring manual database updates or deep-linking.

Impact: High - Hindered administrative efficiency and system configuration flexibility.

2. Flat Settings List Scalability

Description: All system settings were displayed in a single, long list without functional grouping, making it difficult to find specific parameters as the system grew.

Impact: Medium - Poor UX and increased risk of misconfiguration.

3. Sidebar Active State Desync

Description: Navigating between settings sections via anchor links (#realtime, #notification) did not update the active state of the sidebar menu items.

Impact: Low - Minor navigation confusion for administrators.

4. Missing Security Configuration

Description: Parameters like session timeout and login attempt limits were hardcoded or missing from the database entirely.

Impact: Medium - Security compliance risk (PCI DSS/BSP).
