Mitigation Plan: v2.2.0 Settings & Navigation

M-220-01: Audit Logging for Configuration Changes

Risk: Unauthorized modification of security parameters.

Mitigation: Every change to the SystemSetting table triggers an AuditLog entry, recording the old value, new value, and the administrative user responsible. This ensures full traceability for BSP audits.

M-220-02: Secret Masking in UI

Risk: Exposure of sensitive API keys (Pusher/Twilio/Resend) in the admin dashboard.

Mitigation: Settings marked as isSecret: true are rendered as password-type inputs in the UI, preventing casual over-the-shoulder viewing while allowing admins to update the values when necessary.

M-220-03: Redundant Notification Fallback

Risk: Notification failure due to invalid API keys.

Mitigation: The notification engine is designed to support multiple providers. Admins can quickly switch providers via the new "Notification Settings" panel if one service fails.
