Applied Fixes: Navigation & UI Resiliency

FIX-220-001

Missing Realtime & Notification Settings Navigation

Issue: Administrative users were unable to access or configure Realtime (Pusher) and Notification (Email/SMS) parameters from the Sidebar, requiring direct database intervention.

Fix: Implemented a grouped settings engine that automatically generates UI sections and sidebar sub-links based on the group field in SystemSetting.

FIX-220-002

Sidebar Active State Desync

Issue: Navigating to anchor-based settings sections (e.g., /settings#realtime) did not update the active state of the sidebar items, leading to UX confusion.

Fix: Added a hashchange event listener to the Sidebar component to track window location changes in real-time.

FIX-220-003

Flat Settings List Scalability

Issue: As the number of environment variables grew, the flat table in the settings page became difficult to manage and navigate.

Fix: Refactored the Settings page to use grouped sections with distinct headers and anchor points, improving scannability.
