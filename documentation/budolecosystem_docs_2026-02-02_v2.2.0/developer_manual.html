Developer Manual: Settings Grouping & Sidebar Submenus

Technical Rationale

The transition from a flat settings table to a grouped interface was necessary to support the growing complexity of the Budol Ecosystem. By introducing a group field in the SystemSetting model, we can logically partition configurations without modifying the underlying database schema structure.

Data Model Changes

The SystemSetting model in schema.prisma was utilized to its full potential by populating the group field:

model SystemSetting {
  id          String   @id @default(uuid())
  key         String   @unique
  value       String
  group       String?  // Now used for UI partitioning
  ...
}

Backend Implementation

The settings page page.tsx now performs a runtime grouping of all settings retrieved from the database:

const groupedSettings = settings.reduce((acc, setting) => {
  const group = setting.group || 'OTHER';
  if (!acc[group]) acc[group] = [];
  acc[group].push(setting);
  return acc;
}, {} as Record);

Sidebar Submenu Integration

The Sidebar.tsx component was updated with a hash state listener to track anchor changes for active-state styling:

useEffect(() => {
  const handleHashChange = () => setHash(window.location.hash);
  window.addEventListener('hashchange', handleHashChange);
  return () => window.removeEventListener('hashchange', handleHashChange);
}, []);

Tests

Tests were conducted by seeding new groups and verifying that the UI automatically generates the corresponding sections and that sidebar sub-links navigate correctly to the anchor IDs.
