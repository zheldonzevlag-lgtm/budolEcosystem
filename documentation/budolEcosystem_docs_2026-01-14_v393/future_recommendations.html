System Admin Notes

The transition to native biometrics reduces the reliance on custom embeddings stored in FlutterSecureStorage. Admins should note that native biometric data never leaves the device's hardware, enhancing privacy compliance (NPC/GDPR).

Future Recommendations

  • Passive Liveness: Explore background liveness detection for the custom flow to remove the manual "blink/smile" requirement.
  • Biometric Refresh: Prompt users to re-enroll if the device's native biometric database changes (e.g., new face added to system settings).
  • Multi-factor Fallback: Integrate 2FA for large transactions even when logged in via biometrics.
