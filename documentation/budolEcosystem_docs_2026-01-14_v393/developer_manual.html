Technical Rationale: Native Biometric Prioritization

Version: v393

Context: Previously, users who enrolled in "BudolPay Face Login" (Custom TFLite flow) were forced to manually capture a photo via ImagePicker every time they logged in, even if their device supported native Face ID or Android Face Unlock. This resulted in significant UX friction.

1. Logic Improvement

We have updated the login flow to prioritize Native OS Biometrics. If the device supports and has enrolled native face recognition, the app will now trigger the system-level authentication dialog instead of the custom camera capture flow.

Updated Flow in login_screen.dart

// Prioritize Native OS Biometrics (Face ID/Android Face Unlock)
final hasNativeFace = await biometricService.hasEnrolledFace();
if (hasNativeFace && _isBiometricEnabled) {
  debugPrint('[FaceLogin] Prioritizing native OS face biometric flow');
  return _handleBiometricLogin(); // Seamless OS-level authentication
}

// Fallback to Custom TFLite Flow (Manual Photo Capture)
// Only if native is not available but a custom template exists
if (storedTemplate != null) {
  // ... custom photo capture logic ...
}

2. Key Code References

  • Entry Point: login_screen.dart: _handleFaceRecognitionLogin
  • Native Service: biometric_service.dart
  • Custom Inference: face_embedding_service.dart

3. Benefits

  • UX: Zero-click or single-click authentication on modern devices.
  • Security: Leverages hardware-backed Trusted Execution Environments (TEE) for biometric matching where available.
  • Reliability: Reduces failures caused by poor lighting or manual camera handling.
