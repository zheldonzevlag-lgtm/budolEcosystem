Issues & Fixes Register (v2)

Fixed

1. User Synchronization Failure (BudolID vs Local)

Issue: Users logged in via BudolID SSO (OTP/Password) were receiving "User not found" or "Account not found" errors when accessing protected routes because their record existed in BudolID but not in the local BudolShap Postgres database.

Fix: Implemented Just-in-Time (JIT) provisioning in /api/auth/me, /api/auth/login, and /api/auth/otp. If a valid BudolID user is detected but missing locally, the system now automatically creates a local shadow account.

Files Changed: me/route.js, login/route.js

Fixed

2. Product Image Validation Error

Issue: API routes for creating/editing products were throwing ReferenceError: isValidImage is not defined.

Fix: Created isValidImage utility function in lib/imageUtils.js and imported it correctly in the product routes.

Files Changed: imageUtils.js, products/route.js

Fixed

3. Database Backup Script Compatibility

Issue: The backup script scripts/backup-db.js was using require but the project is configured as ES Modules ("type": "module" in package.json), causing runtime errors.

Fix: Renamed script to backup_db.cjs to force CommonJS treatment and updated the execution path.

Fixed

4. Registration Desync

Issue: Users who already had a BudolID account but tried to "Register" again on BudolShap were causing unique constraint violations or confusing states.

Fix: Updated /api/auth/register to check for existing BudolID accounts first. If found, it returns a 409 Conflict with a message instructing the user to login instead.

Fixed

5. OTP Email Sending Failure for BudolID Users

Issue: Verification codes were only being sent via SMS for users who existed in BudolID but not locally, as the email sending logic depended on the local user object which was null.

Fix: Updated /api/auth/otp/route.js to capture the email address from the BudolID check response and use it as a fallback for sending the OTP email when the local user record is missing. Required an update to the BudolID API (see below).

Files Changed: otp/route.js

Fixed

6. BudolID API Missing Email in Phone Check

Issue: The BudolID internal API /auth/check-phone was returning { exists: true } but not the email address, causing the BudolShap OTP flow to fail sending emails to BudolID users.

Fix: Updated budolID/index.js to include email and name in the JSON response when a user is found.

Files Changed: budolID/index.js

Fixed

7. Login API Redeclaration Error

Issue: The isOtp variable was declared twice in /api/auth/login/route.js, causing a "Identifier 'isOtp' has already been declared" build error.

Fix: Removed the duplicate declaration. The variable is now declared once at the top of the function scope.

Files Changed: login/route.js
