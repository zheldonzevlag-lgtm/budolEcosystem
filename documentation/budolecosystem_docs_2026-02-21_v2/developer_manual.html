1. Architecture & Data Contracts

1.1 Authentication & User Sync

The authentication system has been enhanced to support seamless synchronization between the local BudolShap database and the central BudolID SSO service.

  • Strategy: Just-in-Time (JIT) Provisioning.
  • Trigger Points: Login (Password/OTP), Session Validation (/auth/me), and Registration.
  • Data Flow: When a user authenticates via BudolID but does not exist locally, the system fetches their profile from BudolID and creates a local `User` record with `accountType: 'BUYER'` and a placeholder password.

2. Backend & API Implementation

2.1 Image Handling & Compression

Purpose: To optimize storage and ensure reliability when Cloudinary uploads fail.

Implementation:

  • Frontend: Images are compressed to WebP format (quality 0.8) using HTML5 Canvas before upload.
  • Backend: The `isValidImage` utility validates inputs. If Cloudinary fails, the base64 string is stored directly in the database (Postgres JSONB/Text) as a fallback.
lib/imageUtils.js
export const isValidImage = (image) => {
    if (!image || typeof image !== 'string') return false;
    // Allow HTTP/HTTPS URLs
    if (image.startsWith('http://') || image.startsWith('https://')) return true;
    // Allow Base64 Data URIs
    if (image.startsWith('data:image/')) return true;
    // Allow Relative Paths
    if (image.startsWith('/')) return true;
    return false;
};

3. Maintenance & Utilities

3.1 Database Bloat Monitoring

Script: npm run db:check-bloat (runs scripts/check-db-bloat.cjs)

Scans the `Product` table for images stored as base64 strings instead of URLs. It reports the count and total size of these images to help identify database bloat.

3.2 Cloudinary Retry Mechanism

Script: npm run db:retry-uploads (runs scripts/cloudinary-retry.cjs)

A background job that finds products with base64 images and attempts to upload them to Cloudinary. If successful, it updates the product record with the new Cloudinary URL, reducing database size.

4. Dependencies

  • Prisma ORM: Database access and schema management. Chosen for its type safety and ease of migration with Postgres.
  • Cloudinary: Primary image storage. Chosen for its robust CDN and transformation capabilities.
  • Next.js App Router: API routes and frontend pages. Chosen for its server-side rendering and unified full-stack capabilities.
  • BudolID Client: Internal library for communicating with SSO. Essential for the ecosystem's centralized identity management.
  • browser-image-compression: Client-side image compression. Chosen to reduce payload size before upload, improving user experience on mobile networks and reducing server load.
