Shopee vs BudolShap UX Analysis & Implementation Plan

Date: 2026-02-21

Version: 1.0

Objective: Transform the BudolShap "Add Product" experience to match Shopee's seller-centric, guided, and efficient UX.

1. Comparative Analysis

Feature Area Shopee (Target State) BudolShap (Current State) Gap Analysis
Form Structure Step-based, Guided, Logical Grouping (Basic, Sales, Shipping, Others). Single long scrolling page. Sections are stacked without clear navigation. CRITICAL: Current linear layout causes fatigue. Needs distinct steps/tabs.
Variation System Tier-based matrix (Color, Size) with Batch Edit (set all prices at once). Limits to 2 tiers. Matrix exists (VariationMatrixManager) but lacks Batch Edit tools. Manual entry for each row is tedious. HIGH: Missing batch tools makes adding 50+ variants painful.
Validation Real-time, inline validation (Red borders, helper text). "Incomplete Section" indicators. Submit-time validation only. Uses Toast notifications for errors. HIGH: User only knows errors after clicking submit.
Image Management Drag-and-Drop, 1:1 Crop Guide, Grid View, "Main Image" badge. Basic file input slots (1-4). No drag-and-drop reordering. MEDIUM: Lack of DnD makes reordering images impossible without re-uploading.
Shipping & Logistics Explicit Weight & Dimensions (LxWxH) for shipping fee calculation. Missing Weight and Dimension fields in Product schema. MEDIUM: Cannot calculate accurate shipping fees without these fields.
User Experience Sticky Save Footer, Right-side Progress Panel, Auto-save draft. Standard scrolling page. No auto-save. LOW: "Nice to have" for polish and safety.

2. Implementation Plan

Phase 1: Foundation & Dependencies (Day 1-2)

Goal: Set up the technical stack for the new form architecture.

  • Dependencies to Install:
    • react-hook-form: For performant form state management.
    • zod: For schema-based validation.
    • @hookform/resolvers: To bridge Zod and React Hook Form.
    • @dnd-kit/core & @dnd-kit/sortable: For drag-and-drop images.
    • framer-motion: For smooth step transitions (optional but recommended).
    • clsx, tailwind-merge: For dynamic class handling.
  • Tasks:
    1. Install packages.
    2. Define productSchema.js using Zod (including new fields like weight/dimensions).
    3. Create reusable UI components: FormSection, Stepper, StickyFooter.

Phase 2: Database Upgrade (Day 3)

Goal: Update schema to support logistics and robust variations.

  • Tasks:
    1. Modify Product model in schema.prisma: 
      model Product {
  ...
  weight      Float?    @default(0) // in kg
  length      Float?    @default(0) // in cm
  width       Float?    @default(0) // in cm
  height      Float?    @default(0) // in cm
  condition   String?   @default("New") // New/Used
  ...
}
    2. Run npx prisma migrate dev --name add_product_logistics.
    3. Update productSlice.js (Redux) to accommodate new fields.

Phase 3: Core Form Reconstruction (Day 4-6)

Goal: Build the multi-step wizard framework.

  • Tasks:
    1. Create AddProductWizard.jsx container.
    2. Step 1: Basic Information
      • Product Name (w/ char counter).
      • Category (Dropdown).
      • Description (Keep CKEditor).
      • New: Image Gallery with Drag-and-Drop using dnd-kit.
    3. Step 2: Sales Information
      • Refactor VariationMatrixManager to accept/return data compatible with react-hook-form.
      • Add "Batch Edit" modal (Price/Stock/SKU).
    4. Step 3: Shipping
      • Add inputs for Weight, Length, Width, Height.
      • Display estimated shipping fee (mock calculation based on weight).

Phase 4: Smart Validation & Logic (Day 7)

Goal: Prevent user errors before submission.

  • Tasks:
    1. Implement Zod resolvers in AddProductWizard.
    2. Add visual error states (Red borders, "Required" text).
    3. Create "Progress Sidebar" that checks step completion status (e.g., ✔ Basic Info, ⚠ Shipping).
    4. Implement Sticky Footer with "Save & Publish" and "Save as Draft".

Phase 5: Testing & Integration (Day 8)

Goal: Ensure stability and data integrity.

  • Tasks:
    1. Verify API endpoints accept new fields.
    2. Test Edit Product flow (ensure useEffect pre-fills the wizard correctly).
    3. Test Drag-and-Drop image reordering (ensure index updates).
    4. Test Batch Edit variations (ensure all rows update).
