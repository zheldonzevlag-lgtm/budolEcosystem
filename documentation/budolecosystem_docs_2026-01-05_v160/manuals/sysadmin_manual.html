System Admin Manual - Recipient Validation

Operational Overview

The v160 update introduces client-side validation for phone numbers and emails. This reduces unnecessary API hits for clearly malformed identifiers.

Configuration Requirements

Important: While the client now validates format, the backend services (budolID and budolPay Auth) must still correctly handle the lookups as established in v157.

Ensure that the budolID service is running and accessible on port 8000, as the mobile app will attempt to lookup these validated identifiers during the transfer process.

Issue Mitigation

If users report "Invalid phone number" errors despite entering a correct number, verify if they are using the supported formats:

  • Domestic: 09XXXXXXXXX
  • International (PH): +639XXXXXXXXX
