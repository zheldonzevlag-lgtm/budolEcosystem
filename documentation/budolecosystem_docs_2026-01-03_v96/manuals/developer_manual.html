Developer Manual - v96

Ecosystem Expansion & Service Isolation

In this update, the central runner start-ecosystem.js has been expanded to support the full budolPay service stack. Previously, only the Admin frontend was managed, leading to ECONNREFUSED errors when budolShap attempted to communicate with the payment gateway.

Orchestration Updates

The apps array in start-ecosystem.js now includes the following service boundaries:

{
    name: 'budolPay (Gateway)',
    cwd: path.join(__dirname, 'budolpay-0.1.0', 'services', 'payment-gateway-service'),
    command: 'node',
    args: ['index.js'],
    port: 8004,
    color: '\x1b[36m' // Cyan
},
{
    name: 'budolPay (Wallet)',
    cwd: path.join(__dirname, 'budolpay-0.1.0', 'services', 'wallet-service'),
    command: 'node',
    args: ['index.js'],
    port: 8002,
    color: '\x1b[94m' // Light Blue
}

Environment Variable Propagation

Key Fix: We ensured that every child process receives its own PORT environment variable. This prevents the monorepo root from forcing a single port on all its workspaces. 
const env = { ...process.env, PORT: app.port.toString() };

const childProcess = spawn(app.command, app.args, { 
    cwd: app.cwd, 
    shell: true,
    env: env
});

Monorepo Startup Optimization

The budolPay (Admin) startup was refined to use workspace-specific commands. This ensures that npm run dev only triggers the admin app development server, while other services are managed independently by the central runner.

{
    name: 'budolPay (Admin)',
    cwd: path.join(__dirname, 'budolpay-0.1.0'),
    command: 'npm',
    args: ['run', 'dev', '-w', 'admin'],
    port: 3000,
    color: '\x1b[34m'
}

Recommendations for Future Enhancement

  • Health Check Dependencies: Implement a retry mechanism in budolShap when the gateway is not yet ready.
  • Service Discovery: Transition from hardcoded localhost URLs to a shared environment registry for service ports.
  • Dockerization: Consider moving the service-heavy budolPay monorepo into a Docker Compose setup for better resource isolation.
