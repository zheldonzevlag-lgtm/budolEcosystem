SysAdmin Manual - v96

Infrastructure Port Mapping

The budolEcosystem now manages 5 distinct services. Ensure the following ports are open and not occupied by other applications (e.g., XAMPP, Docker, or other Node.js instances).

Service Name Port Type Responsibility
budolID (SSO) 8000 Next.js/Express Authentication & User Registry
budolShap (App) 3001 Next.js Main E-commerce Storefront
budolPay (Admin) 3000 Next.js Payment & Transaction Dashboard
budolPay (Gateway) 8004 Express Payment Intent Processing
budolPay (Wallet) 8002 Express Wallet Balance & Transactions

Operational Runbook

1. Verifying Service Health

To verify that all services are correctly listening on their ports, use the following PowerShell command:

Get-NetTCPConnection -State Listen | Where-Object { $_.LocalPort -match "8000|3001|3000|8004|8002" }

2. Handling Port Conflicts

If a service fails to start with EADDRINUSE, identify and terminate the process using the port:

Stop-Process -Id (Get-NetTCPConnection -LocalPort [PORT_NUMBER]).OwningProcess -Force

3. Monorepo Service Management

Note: The budolPay services (Gateway, Wallet) are started directly via node index.js from their respective subdirectories. Do not attempt to start them via npm start from the root monorepo folder as it may cause port conflicts with the Admin frontend.

Recovery Procedures

  • Gateway Offline: If budolShap shows a 500 error during checkout, check the logs of budolPay (Gateway). Ensure port 8004 is active.
  • Database Connection: Ensure PostgreSQL is running (see v95 manual) as all services except the Admin frontend rely on it for persistence.
