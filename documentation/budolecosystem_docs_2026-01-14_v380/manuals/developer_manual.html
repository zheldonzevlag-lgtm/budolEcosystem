Developer Manual: Debug Visibility Implementation

Purpose & Rationale

This update addresses the "Face Registration Error" reported by users where the underlying cause of TFLite initialization failures was invisible in the production UI. We implemented a real-time log streaming mechanism to expose internal service events directly to the UI for troubleshooting.

Technical Explanation

1. Log Streaming Service

The FaceEmbeddingService.dart was enhanced with a broadcast StreamController. This allows UI components to subscribe to internal logs without tight coupling.

final StreamController<String> _logController = StreamController<String>.broadcast();
Stream<String> get logStream => _logController.stream;

void _log(String message) {
  debugPrint('FaceEmbeddingService: $message');
  _logController.add('${DateTime.now().toString().split(' ').last.substring(0, 8)}: $message');
}

All critical operations (asset loading, file writing, interpreter creation) now invoke _log().

2. UI Debug Overlay

The SettingsScreen.dart implements a Stack-based overlay that listens to the log stream and displays entries in a terminal-like window.

Compliance

  • NPC (Data Privacy): Logs are ephemeral and stored only in memory. No PII (Personal Identifiable Information) or face embeddings are logged.
  • Cybersecurity Law: Debug features are restricted to the settings menu and do not expose sensitive system paths.

Dependencies

  • provider: For service injection.
  • dart:async: For Stream management.
  • google_mlkit_face_detection: Upstream dependency for face processing.
