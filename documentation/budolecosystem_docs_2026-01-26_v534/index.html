v1.3.39+136

Release Documentation - 2026-01-26

This release addresses a critical UI bug where the "Verifying Payment" status was being skipped or not rendered correctly during the payment process in budolPay.

Task Progress

Investigate and identify race condition in payment status navigation.
Implement `onVerify` callback architecture in `PaymentStatusScreen`.
Add 1.5s minimum display duration for verifying state to improve UX.
Eliminate redundant `pushReplacement` navigation in `QRScannerScreen`.
Verify fix with `flutter analyze`.
Increment app version to v1.3.39+136.

Purpose & Rationale

User feedback indicated that the "Verifying Payment" state was no longer visible, leading to uncertainty during transaction processing. This fix restores the expected UI flow, ensuring users receive immediate feedback that their payment is being processed, which is a requirement for positive UX and compliance with NPC and BSP standards for clear transaction status communication.
