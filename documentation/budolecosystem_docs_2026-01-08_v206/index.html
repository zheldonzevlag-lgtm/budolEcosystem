Implementation Plan v206: User Name Display Fix

Date: 2026-01-08 | Status: COMPLETED

Purpose

The purpose of this task was to resolve an issue where the mobile application displayed a generic "User" name instead of the actual registered name (e.g., "Jon Galvez") after a successful login. This was caused by the backend authentication service not including name fields in the mobile login response and the mobile API service not correctly caching available user metadata during the identification phase.

Technical Details

Backend Changes (auth-service)

  • Modified /login/mobile/identify endpoint to include firstName and lastName in the user object for both OTP_REQUIRED and AUTH_REQUIRED statuses.
  • Modified /login/mobile/verify-pin to include both firstName and lastName in the success response.
  • Modified /login/mobile/setup-pin to include firstName and lastName in the success response.
  • Modified /verify-otp to include firstName and lastName in the success response when a device is being verified.

Mobile App Changes (budolPayMobile)

  • Updated ApiService.identifyMobile in lib/services/api_service.dart to capture and notify listeners of user metadata received during the identification phase.
  • Updated ApiService.verifyOtp to ensure user metadata is cached even if a full session token is not yet generated (e.g., when moving to PIN setup).

Testing & Validation

  • Database Verification: Confirmed user 09484099388 has firstName: "Jon" and lastName: "Galvez".
  • API Audit: Verified that backend responses now include the full name fields.
  • UI Logic: Confirmed HomeScreen logic correctly uses apiService.user['firstName'] and apiService.user['lastName'] to construct the welcome message.

Future Recommendations

  • Profile Synchronization: Implement a background sync in the mobile app to periodically refresh user profile data from the user-service.
  • Caching Strategy: Consider implementing a more robust local database (like Hive or Sqflite) for user metadata to ensure immediate display on app launch before API calls complete.
