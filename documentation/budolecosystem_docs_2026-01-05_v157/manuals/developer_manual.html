Developer Manual - v157

Topic: Phone Number Support for P2P Transactions

1. Purpose

The primary purpose of this update is to allow users to send money or cash in using a recipient's phone number, in addition to their email address. This enhances usability, especially for mobile users who prefer phone-based identification.

2. Technical Details & Implementation

A. budolID SSO Service (index.js)

Updated the /auth/user/find endpoint to accept an optional phone query parameter. It now performs a lookup using either email or phone number.

// Modified lookup logic in budolID
const { email, phone } = req.query;
const where = email ? { email } : { phoneNumber: phone };
const user = await prisma.user.findUnique({ where, select: { ... } });

B. budolPay Auth Service (index.js)

Updated the internal /user/find endpoint to handle phone number queries and synchronize user data from budolID if not found locally.

  • Added support for phone query parameter.
  • Enhanced SSO synchronization to include phoneNumber field.

C. budolPay Mobile (Flutter)

Modified ApiService and SendMoneyScreen to handle dual-mode identification.

  • ApiService.findUser(identifier): Automatically detects if the identifier is an email (contains '@') or a phone number.
  • SendMoneyScreen: Updated input labels and controllers to support "Email or Phone Number".

3. Code Changes Impact Analysis
