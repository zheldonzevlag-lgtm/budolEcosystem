System Admin Manual - v157

1. Environment Configuration

No new environment variables were introduced in this version. However, ensure that budolID and budolPay Auth services are restarted to apply the endpoint changes.

2. Database Schema

No schema changes (DDL) were required as phoneNumber field already existed in both budolID and budolPay databases. Ensure Prisma client is regenerated if any discrepancies occur.

npm run prisma:generate --prefix budolID-0.1.0
npm run prisma:generate --prefix budolpay-0.1.0

3. Operational Procedures

A. User Synchronization

The system now automatically syncs phone numbers from budolID SSO to budolPay when a user is looked up for the first time by phone number.

B. Troubleshooting

If a phone-based transfer fails with 404, verify that the phone number exists in the budolID database. You can use the provided script:

node budolID-0.1.0/check-users-budolid.js

4. Risk & Mitigation

  • Risk: Duplicate phone numbers in the system.
    Mitigation: phoneNumber is marked as @unique in the Prisma schema.
  • Risk: Phone number format mismatch (e.g., +63 vs 09).
    Mitigation: Recommendation for future enhancement includes adding a phone normalization library.
