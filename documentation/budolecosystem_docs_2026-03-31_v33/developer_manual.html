Developer Manual - Monorepo & Security Fixes

1. Vercel Monorepo Workspace Resolution

To resolve the "Module not found" error for `@budolpay/audit` on Vercel, we explicitly configured the internal workspaces to point to their source files instead of pre-compiled `dist` folders, enabling Next.js to handle transpilation naturally.

Next.js Config Update

transpilePackages: ['@budolpay/database', '@budolpay/audit', '@budolpay/security', '@budolpay/notifications'],
View File: next.config.mjs

Workspace Package Mapping

Updated individual package settings to export TypeScript source index directly:

"main": "src/index.ts",
"types": "src/index.ts",
View File: audit/package.json
