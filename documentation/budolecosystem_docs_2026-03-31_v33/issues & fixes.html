Vercel Production Issues & Resoultions - v33

A detailed record of critical production issues encountered and the subsequent fixes applied.

Module not found (Internal Workspaces) FIXED

Issue: Next.js Webpack failed to resolve `@budolpay/audit` because the package mapping was pointing to `/dist` folders which were not part of the production environment's working directory on Vercel.

Fix Implemented:

Updated `package.json` entry points for all internal workspace modules to point directly to their TypeScript source files (`src/index.ts`).

Missing Security Challenge FIXED

Issue: The Admin Login Dashboard was missing the anti-bot Shield Challenge (MathCaptcha), which was found during the v33 audit to be out of parity with the Budol Ecosystem Security Standard.

Fix Implemented:

Ported the high-fidelity MathCaptcha Shield Challenge from the ecosystem repository and integrated it as a mandatory verification gate before credential exposure.

Prisma Engine ENOENT Error FIXED

Issue: Production build on Vercel failed with an `ENOENT: no such file or directory, copyfile` error when attempting to copy the Prisma binary to the default global location.

Fix Implemented:

Configured `PRISMA_ENGINES_CACHE_DIR` to the writable `/tmp` path and transitioned the environment to a stable **Prisma 5.14.0** to ensure engine binary compatibility.

Auth Bridge 404 Failure FIXED

Issue: Mobile authentication calls were hitting 404 because the API routes had not yet been fully migrated to the Next.js production backend for BudolPay Admin.

Fix Implemented:

Ported the legacy Express endpoints for `check-phone`, `login`, and `register` to Next.js API Routes at `/api/auth/`. Updated mobile client base URLs to target the new bridge.
