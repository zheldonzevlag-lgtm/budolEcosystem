Stabilization Walkthrough - v33

Step-by-step review of the fixes for Vercel monorepo deployment and Admin Dashboard security.

1

Identifying Build Failures

Analyzed the Vercel logs and confirmed the `Module not found: Can't resolve '@budolpay/audit'` error occurred during the compilation of `lib/audit.ts`. This was due to internal workspace packages failing to expose their TypeScript source correctly for Webpack bundling.

2

Correcting Workspace Mapping

Modified the `package.json` files for `@budolpay/audit`, `@budolpay/security`, and other internal packages. We changed the `main` and `types` entry points to point directly to their TypeScript source files (`src/index.ts`).

3

Updating Next.js Transpilation

Updated `apps/admin/next.config.mjs` to include the specific workspace packages in the `transpilePackages` array. This ensures that the code from outside the `admin` app directory is correctly processed and bundled in production.

4

Restoring the Shield Challenge

Rebuilt the `MathCaptcha` bot-shiled component in the `admin` components directory and integrated it into the login flow. The credential fields are now programmatically locked until the math challenge is successfully solved.

5

Production Re-deployment

Committed and pushed the changes to the `feature/auth-bridge-v32` branch. Triggered a re-deployment on Vercel with a cleared build cache to ensure all new package mappings are picked up.
