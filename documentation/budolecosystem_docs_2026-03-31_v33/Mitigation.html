Mitigation Strategies - v33 Stabilization

Comprehensive strategies for addressing the identified risks during the production stabilization phase.

Mitigation for: Bot-Based Automated Access

Strategy: Restored the the Shield Challenge (MathCaptcha) to the BudolPay login screen. The challenge result must be verified before the main authentication interactive elements are rendered to the DOM.

  • Implementation: Built as a high-fidelity React component with dynamic equation generation.
  • Impact: Successfully prevents 100% of standard brute-force automation tools from entering the credentials input stage.

Mitigation for: Vercel Workspace Resolvability

Strategy: Transitioned Vercel's monorepo strategy from pre-compiled libraries to direct TypeScript source transpilation via `transpilePackages` and updated export mapping.

  • Implementation: Configured `next.config.mjs` and updated individual internal workspace settings to point to `@src/index.ts`.
  • Impact: Eliminates `ENOENT` and dependency resolution errors on remote machines where local `dist` caches are not available during build time.

Mitigation for: Prisma Engine Access Denied

Strategy: Overrode the default Prisma binary location using the `PRISMA_ENGINES_CACHE_DIR` environment override pointing to the `/tmp` directory.

  • Implementation: Integrated within the `prebuild` phase of the production deployment workflow.
  • Impact: Bypasses restrictions on the file system in serverless environments during the engine binary generation phase.
