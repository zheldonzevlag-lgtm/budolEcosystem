🛡️ AI Anti-Spam Engine

The Budol Ecosystem now includes a neural-inspired Anti-Spam Engine that analyzes registration patterns in real-time. This system is designed to prevent bot-driven account creation without impacting legitimate users.

Rationale

As a fintech platform, we are targets for automated account creation for money laundering or phishing. v1.7.0 introduces pattern-based risk scoring to block high-probability spam accounts at the registration gateway.

Risk Scoring Algorithm

Each registration request is passed through the aiAntiSpamEngine which calculates a risk score based on several patterns:

  • Email Randomness (+30-50 pts): Checks for high digit-to-letter ratios and sequential numbers.
  • Name Consistency (+15-40 pts): Detects single-character names or identical First/Last names.
  • Disposable Domains (+50 pts): Instant high-risk flags for known temporary email providers.
View Implementation 
const aiAntiSpamEngine = {
    score: async (data) => {
        let riskScore = 0;
        const { email, firstName, lastName, phoneNumber } = data;

        // Pattern 1: Email Randomness (Spam indicator)
        const emailLocal = email ? email.split('@')[0] : '';
        if (emailLocal && /[0-9]{4,}/.test(emailLocal)) riskScore += 30; 
        
        // Pattern 2: Name Consistency
        if (firstName && lastName && firstName.toLowerCase() === lastName.toLowerCase()) riskScore += 25;
        if (firstName && firstName.length < 2) riskScore += 15;

        // Pattern 3: Common Spam Domains (Temporary/Disposable)
        const disposableDomains = ['tempmail.com', '10minutemail.com', 'guerrillamail.com'];
        if (email && disposableDomains.some(d => email.endsWith(d))) riskScore += 50;

        return {
            riskScore,
            isHighRisk: riskScore >= 70,
            analysis: riskScore >= 70 ? 'AI detected high-probability spam pattern.' : 'Low risk pattern detected.'
        };
    }
};

🛡️ Phase 12: Compliance Audit Scheduling Basis

The Next Audit date displayed in the Compliance Shield is dynamically calculated to ensure the ecosystem remains in a state of continuous audit-readiness. The scheduling logic is derived from the following institutional and regulatory frameworks:

Framework Requirement Cycle
BSP Circular 808 IT Risk Management & External Audit for EMIs Semi-Annual (180 Days)
PCI DSS v4.0 Internal/External Vulnerability Scanning Quarterly (90 Days) / Annual
NPC (DPA 2012) Privacy Impact Assessment & Renewal Annual

Scheduling Algorithm: The system enforces a semi-annual audit cycle with fixed synchronization points on January 15 and July 15. This ensures that the technical team has predictable windows for deep-system forensic reviews and compliance artifact generation.

Implementation: The logic is centralized in the getNextAuditDate() utility function within the core library, ensuring consistency across both BudolPay and BudolShap admin dashboards.

📦 Phase 8: Dependencies and Software Selection Rationale

The AI Anti-Spam Engine leverages the following core technologies within the BudolPay ecosystem. Each choice was made to balance security, performance, and regulatory compliance.

Dependency Role Rationale & Selection Criteria
Node.js & Express 5.x Runtime & API Framework Chosen for its asynchronous, non-blocking I/O model. This is critical for real-time risk scoring, ensuring that security checks do not increase registration latency for legitimate users. Express 5.x provides improved error handling and routing for security-sensitive endpoints.
Bcryptjs Security & Hashing Selected for its industry-standard salt-and-hash implementation. This ensures PCI DSS compliance by protecting user credentials against brute-force attacks, even if the database is compromised.
JWT (JsonWebToken) Authentication Provides stateless session management. This architecture was chosen to support horizontal scaling across multiple microservices, allowing for distributed spam detection and session validation without a central bottleneck.
Jest & Supertest Testing Framework Adopted as the primary testing suite due to its robust support for integration and E2E testing. This allows for rigorous validation of security patterns, ensuring that no release occurs without passing mandatory risk-based test cases.
Tailwind CSS UI Components Used in the Security Dashboard to implement zero-JavaScript interactions. This reduces the attack surface for the administrative interface and ensures that the dashboard remains performant and accessible on all devices.
