System Administration Guide: AI Security Watchtower

🛡️ AI Spam Guard Monitoring

In v1.7.0, the Security Dashboard includes real-time metrics for blocked spam attempts. Administrators must monitor these patterns to ensure legitimate users are not being inadvertently blocked.

1. Viewing Spam Block Metrics

Navigate to the Security page in the Admin Dashboard. The Compliance Overview now includes an AI Spam Guard status card showing the total number of blocked registration attempts.

2. Analyzing Block Trends

The Threat Mitigation Trend chart now includes data for SPAM_BLOCK events. Hover over the daily bars to see specific counts. A sudden spike in blocks may indicate a coordinated bot attack.

Policy Reminder: If a legitimate user reports being blocked, review the Audit Log for entries with action SECURITY_SPAM_REGISTRATION_BLOCKED to see the specific risk score and reason.

⚙️ System Configuration

The AI Spam Guard operates on a dynamic risk threshold. While currently set to 70/100, this can be adjusted in the auth-service configuration for higher or lower sensitivity.

Dynamic Risk Thresholds

  • 70 (Default): Balanced protection against bots and disposable emails.
  • 50 (Aggressive): Blocks more aggressively; may impact users with numerical email addresses.
  • 90 (Conservative): Only blocks the most obvious bots and disposable domains.

📜 Regulatory Compliance Reporting

All blocked attempts are logged with forensic detail to meet BSP Circular 808 requirements for non-repudiation and fraud detection.

  • Log Action: SECURITY_SPAM_REGISTRATION_BLOCKED
  • Data Captured: Email, Risk Score, AI Analysis Reason, IP/Device Info (where available).
