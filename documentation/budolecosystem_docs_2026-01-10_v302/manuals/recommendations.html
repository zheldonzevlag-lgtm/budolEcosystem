Recommendations for Future Enhancement

Based on the LAN configuration and current ecosystem state, the following enhancements are recommended for future phases:

1. Security & Access Control

High Priority

Standardized RBAC (Role-Based Access Control)

Purpose: Unify authentication and authorization across all microservices and web apps.

Explanation: Currently, some services have independent user checks. Implementing a centralized RBAC service or middleware within the API Gateway will ensure consistent security policies.

Dependencies: Auth Service, API Gateway.

Medium Priority

HTTPS/TLS for LAN Traffic

Purpose: Encrypt data in transit within the local network.

Explanation: While LAN is safer than the public internet, internal traffic is still vulnerable to sniffing. Implementing self-signed certificates or a local CA will secure cross-service communication.

Risks: Certificate management complexity on mobile devices.

2. Infrastructure & Scalability

Medium Priority

Service Mesh (e.g., Consul or Istio)

Purpose: Advanced service discovery, load balancing, and observability.

Explanation: As the number of microservices grows, manual IP management becomes brittle. A service mesh can handle dynamic service discovery and health monitoring more robustly than mDNS.

Mitigation: Start with simple mDNS as implemented in v302 before transitioning to a full service mesh.

Low Priority

Docker Compose for Production-like LAN Testing

Purpose: Simplify environment setup and ensure consistency across developer machines.

Explanation: Providing a Docker Compose file that automatically handles network bridging and environment variable injection will reduce onboarding time.

3. Monitoring & Reliability

High Priority

Centralized Logging & Distributed Tracing

Purpose: Debug cross-service requests (e.g., Wallet -> Transaction -> Auth).

Explanation: Use tools like ELK Stack or Jaeger to trace requests as they move through the gateway to various microservices. This is critical for resolving 504 Gateway Timeout issues.

Issues Fixed: Faster identification of slow services or network bottlenecks.
