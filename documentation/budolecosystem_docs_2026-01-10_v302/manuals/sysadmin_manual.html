System Admin Manual: LAN Infrastructure & Configuration

1. Environment Variable Modifications

The following environment variables in the root .env file are critical for LAN operation:

  • LOCAL_IP: Set this to the host's LAN IP (e.g., 192.168.1.24). This is used by all services to communicate with each other.
  • AUTH_SERVICE_URL: Should use LOCAL_IP (e.g., http://192.168.1.24:8001).
  • WALLET_SERVICE_URL: Should use LOCAL_IP (e.g., http://192.168.1.24:8002).
  • GATEWAY_URL: The central entry point (e.g., http://192.168.1.24:8080).

2. Infrastructure Changes

2.1. Network Firewall Configuration

Admins must ensure that the following ports are open on the host machine's firewall to allow LAN traffic:

  • 8080: API Gateway (Primary Access)
  • 3000: Admin Dashboard
  • 3001: BudolShap Marketplace
  • 8000-8009: Microservices (Internal & External Access)

2.2. Database Schema Changes

No direct database schema changes were required for this update. However, ensure the SystemSetting table contains the correct LOCAL_IP if used by the mobile app for dynamic configuration fetching.

3. Deployment Procedures

  1. Run node start-ecosystem.js and choose Option 1 (LOCAL). This will automatically detect the LAN IP and update all .env files.
  2. Verify that the console output shows services starting on http://0.0.0.0:[PORT].
  3. Confirm LAN accessibility by visiting http://[LOCAL_IP]:8080/health from another device on the network.

4. Operational Runbook

If services are not accessible on the LAN:

  • Check if the LOCAL_IP in .env matches the current IP in ipconfig.
  • Restart the start-ecosystem.js script to refresh IP detection.
  • Verify that the machine is on a "Private" network profile in Windows settings (Public profiles often block incoming connections).
