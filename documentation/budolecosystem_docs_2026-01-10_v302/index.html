LAN Network Configuration & API Gateway Accessibility

Enabling cross-device ecosystem access via dynamic IP binding and 0.0.0.0 server listeners.

Status: Implemented & Verified

Purpose

This update addresses the need for the BudolEcosystem to be accessible across a Local Area Network (LAN). By transitioning from hardcoded localhost/127.0.0.1 references to environment-driven dynamic IP binding and 0.0.0.0 server listeners, the API Gateway and all dependent services can now be reached by mobile devices and other workstations on the same network.

Implementation Overview

  • Dynamic IP Binding: Replaced hardcoded IP references in all services with process.env.LOCAL_IP, allowing the ecosystem to adapt to the host's current LAN IP.
  • 0.0.0.0 Listeners: Configured all Express servers to listen on 0.0.0.0, enabling connections from any network interface.
  • API Gateway Proxying: Updated the API Gateway to dynamically route requests to services using the detected LAN IP, ensuring seamless cross-service communication.
  • mDNS Service Discovery: Verified and stabilized Bonjour/mDNS advertisements for automatic service discovery by the mobile app.
  • End-to-End Testing: Successfully performed health checks and proxy route verification using the host's LAN IP (192.168.1.24).
