🛠️ Developer Manual - v3.0.3

1. Purpose & Rationale

This version addresses two critical operational bottlenecks: Request Timeouts during registration and Zero Visibility into mobile API failures. By shifting non-critical external calls (OTP, Admin Notifications) to asynchronous patterns, we guarantee response times within the 10s mobile timeout window. The Debug Console provides on-device log inspection for faster field troubleshooting.

2. Technical Explanation: Non-Blocking Architecture

In the auth-service, operations like sending SMS or notifying the admin dashboard were previously awaited, causing the user's registration request to hang if these third-party or internal services were slow.

Implementation Detail:

We replaced blocking await calls with fire-and-forget patterns using .then().catch() and self-invoking async functions.

File Reference: auth-service/index.js

// Non-blocking Admin Notification
prisma.user.count().then(totalUsers => {
    notifyAdmin('new_user', { count: totalUsers, user: { ... } });
}).catch(err => console.error(err));

// Non-blocking OTP/Email Notification
(async () => {
    try {
        await sendAccountCreationSuccess(...);
        await sendOTP(...);
    } catch (err) {
        console.error(err);
    }
})();

3. Mobile Debug Console Architecture

The DebugConsoleScreen is a dedicated Flutter screen that displays a rolling buffer of ApiLog objects captured by the ApiService.

Data Contract: ApiLog

File Reference: api_service.dart

class ApiLog {
  final LogType type; // request, response, error, info
  final String url;
  final String? method;
  final int? statusCode;
  final String? requestBody;
  final String? responseBody;
  final String? error;
  final DateTime timestamp;
}

UI Component: DebugConsoleScreen

File Reference: debug_console_screen.dart

This screen uses a ListView.separated to render logs with color-coding based on the LogType. It includes a toggle for system information and a clear button.

4. Security & Hidden Gestures

To prevent accidental access in production, the console is accessed via a long-press (500ms+) on specific UI elements:

  • Login Screen: Long-press on the BudolPay logo. (login_screen.dart)
  • Registration Screen: Long-press on the "Create Account" title. (registration_screen.dart)

5. Dependencies

  • Express.js (v5.2.1): Backend routing.
  • Axios: Internal service notifications.
  • Flutter/Dart: Mobile UI and State management.
  • Prisma: Audit logging and database operations.

6. Compliance

This implementation adheres to PCI DSS 10.2.2 (Audit logs for all security events) and BSP Circular 808 (Ensuring system availability and response time guarantees).
