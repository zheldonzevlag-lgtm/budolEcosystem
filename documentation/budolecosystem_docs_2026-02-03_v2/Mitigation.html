🛡️ Mitigation Strategies - v3.0.3

M-DEBUG-001: PII Redaction in Logs

Risk: R-DEBUG-001 (PII Exposure)

Action: The DebugConsoleScreen displays raw data for developer use, but production builds should implement a LogMasker utility to redact PINs, passwords, and full card numbers before they are added to the ApiService log buffer.

M-PERF-001: Rolling Log Buffer

Risk: R-PERF-001 (Memory Bloat)

Action: The ApiService implements a maximum log limit of 50 entries. When the 51st log arrives, the oldest log is removed. This ensures the app memory footprint remains constant regardless of usage duration.

M-AUTH-001: Retry Mechanism for Async Calls

Risk: R-AUTH-001 (Notification Failure)

Action: Asynchronous calls in auth-service are wrapped in .catch() blocks. Future iterations will include a Dead Letter Queue (DLQ) or a retry worker to ensure that failed background tasks are eventually completed.
