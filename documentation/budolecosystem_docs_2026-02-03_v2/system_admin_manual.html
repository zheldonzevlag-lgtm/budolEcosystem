🖥️ System Admin Manual - v3.0.3

1. Monitoring Asynchronous Operations

With the shift to non-blocking calls in the auth-service, standard HTTP response codes (e.g., 201 Created) no longer guarantee that downstream notifications (SMS/Admin Dashboard) have succeeded. Admins must monitor the Forensic Audit Trail for failure events.

Important: Check the Gateway logs or the Admin Dashboard for AUDIT_LOG_CREATED events to verify successful background processing.

2. Mobile Debugging Support

When a user reports a "Timeout" or "Unexpected Error" on the mobile app, instruct them to perform the Hidden Gesture (Long-press on Logo) to open the Debug Console. They can then take a screenshot of the API logs for your analysis.

Log Interpretation:

  • Red Logs (ERROR): Indicates a network failure or a 5xx server error.
  • Yellow Logs (REQUEST): Shows the outgoing payload and endpoint.
  • Green Logs (RESPONSE): Shows the server's JSON response and status code.

3. Service Health Checks

Ensure the auth-service (Port 8001) and api-gateway (Port 8080) are running. If registration timeouts persist even with async calls, check the database connection pool in the prisma service.
