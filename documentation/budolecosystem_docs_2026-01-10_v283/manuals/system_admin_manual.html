System Admin Manual - v283

Environment Configuration

To support LAN testing, ensure the following environment variables are correctly set on the host machine.

1. IP Configuration

The host machine IP is assumed to be 192.168.1.24. All services must be configured to bind to 0.0.0.0 to be accessible from other computers.

2. Environment Variables

Update budolshap-0.1.0/.env.local:

NEXT_PUBLIC_SITE_URL=http://192.168.1.24:3001
NEXT_PUBLIC_SSO_URL=http://192.168.1.24:8000

Update root .env for the ecosystem:

LOCAL_IP=192.168.1.24

Database Seeding

After any database reset or schema update, run the LAN seeding script to update the redirectUri for all SSO-enabled applications.

$env:LOCAL_IP="192.168.1.24"; node packages/database/seed_lan.js

Firewall Requirements

Ensure that the following ports are open in the Windows Firewall for the Private Network:

  • 3000 (budolPay Admin)
  • 3001 (budolShap App)
  • 8000 (budolID SSO)
  • 8080 (budolPay Gateway)

Browser Security Workaround

When testing on a LAN computer, browsers will treat http://192.168.1.24 as an "Insecure Origin". This blocks access to the camera and microphone. To bypass this for testing:

  1. Open Chrome and navigate to chrome://flags/#unsafely-treat-insecure-origin-as-secure
  2. Enable the flag.
  3. Add http://192.168.1.24:3001 and http://192.168.1.24:3000 to the text area.
  4. Relaunch Chrome.
