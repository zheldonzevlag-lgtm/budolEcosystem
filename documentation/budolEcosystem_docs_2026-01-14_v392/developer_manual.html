Developer Manual - v392

Application Versioning Correction

Version 392 corrects the application version to 1.2.85. This ensures consistency with the target release and prevents version drift in the production environment.

1. Pubspec.yaml Correction

The version and build number have been reverted to 1.2.85:


// [pubspec.yaml](file:///d:/IT%20Projects/budolEcosystem/budolPayMobile/pubspec.yaml#L4)
version: 1.2.85+85

2. ApiService Runtime Sync

The runtime version string in ApiService is updated to match:


// [api_service.dart](file:///d:/IT%20Projects/budolEcosystem/budolPayMobile/lib/services/api_service.dart#L22)
String _appVersion = '1.2.85'; // Android Build v1.2.85

Verification

A dedicated test script was used to verify the version correction:


// [version_test.dart](file:///d:/IT%20Projects/budolEcosystem/budolPayMobile/test/version_test.dart)
test('ApiService version should be 1.2.85', () {
