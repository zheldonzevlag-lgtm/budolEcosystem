Developer Manual

This document details the changes made in v3.4.4 regarding Cart Clearance and Multi-Store Logic.

Purpose

To ensure that items are reliably removed from the user's cart after an order is successfully created, regardless of the payment method (Synchronous or Asynchronous). Additionally, to enforce shipping restrictions for multi-store orders.

Rationale

Previously, asynchronous payments (GCash, Maya, etc.) were excluded from immediate cart clearance. This was intended to preserve the cart in case of payment failure. However, it led to duplicate orders if users navigated back and re-submitted the order. Clearing the cart upon order creation is a safer default to prevent duplicates.

Technical Explanation

The logic in OrderSummary.jsx was modified to remove the conditional check !isAsyncPayment. Now, the deleteItemFromCart Redux action is dispatched for all successful API responses from /api/orders.

Code Changes

File: components/OrderSummary.jsx

// Before:
const isAsyncPayment = ['GCASH', 'MAYA', 'GRAB_PAY', 'QRPH', 'BUDOL_PAY'].includes(paymentMethod);
if (response.ok && !isAsyncPayment) { ... }

// After:
if (response.ok) {
    // Clear cart immediately
    orderItems.forEach(item => {
        dispatch(deleteItemFromCart({ ... }));
    });
}

Compliance

Ensures that order records are distinct and reduces the risk of accidental double-charging or inventory allocation errors (PCI DSS considerations for transaction integrity).

Dependencies

  • react-redux: For dispatching cart actions.
  • cartSlice.js: Reducer handling deletion logic.

Tests

Unit tests for cartSlice confirm that deleteItemFromCart correctly removes items based on productId and variationId.
