System Admin Manual: Vercel Postgres & Storage Nexus

This manual covers the configuration of the production database on Vercel for the BudolPay ecosystem (Admin Dashboard & APIs).

1. Database Configuration (Vercel Storage)

1

Storage Connection

Ensure the budolshap-db-v3 Vercel Postgres database is connected to the budolpay-0.1.0 project under the Storage tab.

2

Environment Variable Sync

Admin must ensure the following environment variables are correctly mapped for the Production environment:

Key Mapped To (Vercel) Component
DATABASE_URL POSTGRES_PRISMA_URL Prisma Pooled Connection
DIRECT_URL POSTGRES_URL_NON_POOLING Prisma Migration Direct Access
POSTGRES_URL POSTGRES_URL Raw Node.js / SQL Access (Optional)
⚠️ CRITICAL: DATABASE_URL must not point to db.prisma.io. This release transitions to native Vercel Postgres to resolve latency and schema mapping issues during high-volume admin transactions.

2. Build Optimizations

The system now detects build-time environments. If you see a warning in the build log saying Prisma skipping initialization for build environment, this is normal behavior for static generation of the dashboard shell.

3. Troubleshooting

  • Build Prerender Error: Check if a new page was added that fetches data without export const dynamic = 'force-dynamic';.
  • DB Connection Timeout: Verify the Vercel Postgres database status in the Vercel console.
