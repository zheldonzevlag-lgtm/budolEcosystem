1. Admin API Authentication Fix

Problem

The admin API endpoints were failing when accessed with SSO tokens because the token payload used sub for the user ID instead of userId or id, which the admin authentication middleware expected. Additionally, it was only checking cookies and not the Authorization header.

Fix

Updated lib/adminAuth.js to use the centralized getAuthFromRequest helper. This ensures support for both cookies and Authorization headers, and correctly handles userId, id, or sub fields in the JWT payload.

// lib/adminAuth.js
const auth = await import('./auth').then(m => m.getAuthFromRequest(request))
const userId = decoded?.userId || decoded?.id || decoded?.sub

2. IP Synchronization

Updated all .env files across the ecosystem to use 192.168.1.17 as the primary local IP for service URLs. This ensures seamless communication between microservices (budolshap, budolpay, budolid) and the mobile app on the local network.
