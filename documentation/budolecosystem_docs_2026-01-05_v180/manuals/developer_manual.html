Developer Manual: KYC Document Rendering Fix

Problem Overview

Prisma Bytes fields are serialized into JSON as objects with a type: 'Buffer' and a data array. Standard img src attributes cannot consume this format directly.

Implementation Logic

The getDocUrl helper function was rewritten to perform the following steps:

  1. Extraction: Identifies the raw byte array from blobData.
  2. Conversion: Converts bytes to a binary string using String.fromCharCode.
  3. Encoding: Encodes the binary string to Base64 using window.btoa.
  4. Data URI Construction: Prepends data:image/jpeg;base64, to create a valid source for <img> tags.

Code Example

const uint8Array = new Uint8Array(bytes);
let binary = '';
const chunkSize = 8192;
for (let i = 0; i < uint8Array.length; i += chunkSize) {
    const chunk = uint8Array.subarray(i, Math.min(i + chunkSize, uint8Array.length));
    binary += String.fromCharCode.apply(null, Array.from(chunk));
}
return `data:image/jpeg;base64,${window.btoa(binary)}`;

Error Handling

If conversion fails, a debug overlay is shown in the UI containing the typeof and Object.keys of the data, which assists in identifying unexpected data structures from the database.
