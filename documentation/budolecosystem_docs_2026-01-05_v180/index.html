v180 Implementation: Admin KYC Image Fix

Date: 2026-01-05

Status: Completed

Purpose

This implementation addresses the issue where KYC documents uploaded by users were not viewable in the admin panel, showing "Stored as BYTEA" placeholders instead of images.

Technical Details

1. Robust Image Conversion Logic

Modified apps/admin/app/users/page.tsx to include a comprehensive getDocUrl function that handles various binary data formats returned by Prisma and serialized via JSON.

// Support for:
// - Prisma Buffer JSON: { type: 'Buffer', data: [...] }
// - Direct numeric arrays
// - Uint8Array instances
// - Hex strings (with and without 0x prefix)
// - Base64 strings

2. Chunked Base64 Processing

Implemented 8KB chunking for String.fromCharCode.apply to ensure compatibility with large images and prevent stack overflow errors in the browser.

3. UI Troubleshooting Metadata

Added a debug overlay in the document viewer that displays the data type and object keys if an image fails to render, enabling faster diagnosis of edge cases.

Impact Analysis

  • Admin Panel: KYC review process is now fully functional with visual document verification.
  • Database: No changes required; existing BYTEA data is now correctly interpreted.
  • Performance: Conversion happens on-demand in the browser, minimizing server overhead.
