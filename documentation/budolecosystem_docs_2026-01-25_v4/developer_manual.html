Developer Manual

Technical Architecture

The system uses a React-based frontend (Next.js) and a Flutter-based mobile application. The core payment flow involves generating a QR code on the web which is then scanned or viewed by the user.

Frontend: QR Modal ID Reordering

The requirement was to display the Order ID before the Reference ID in the QR payment modal. This ensures that users can immediately identify their purchase context.

1. QRCodeModal Component Update

File: QRCodeModal.jsx

Changes: Added orderId prop and reordered the JSX elements.

<div className="flex justify-between text-sm"> <span className="text-gray-600">Order ID:</span> <span className="font-semibold text-gray-800 truncate ml-2"> <BudolPayText text={orderId || "N/A"} /> </span> </div> <div className="flex justify-between text-sm pt-1"> <span className="text-gray-600 mb-0.5">Reference ID:</span> <span className="text-[12px] text-gray-500 break-all leading-tight bg-gray-100 p-1"> {paymentIntentId} </span> </div>

2. Prop Drilling Updates

The orderId was passed down from the following parent components:

  • OrderSummary.jsx
  • app/payment/qr/page.jsx
  • app/(public)/orders/page.jsx

Mobile: Android App Build

The Android application was built using Flutter. The version was incremented to 1.3.34+131 in pubspec.yaml.

File: pubspec.yaml
