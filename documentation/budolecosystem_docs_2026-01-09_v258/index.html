v258: System Settings Restoration

API Gateway Route Fix & Real-Time Functionality Recovery

Status: Fully Implemented

Executive Summary

This update restores the missing /system/settings route in the API Gateway, which was identified as the root cause for real-time transaction updates failing in the mobile application. By exposing the correct Pusher/Socket.io configurations, the mobile app can now dynamically initialize the appropriate real-time provider instead of defaulting to SWR.

Key Improvements

  • Gateway Fix: Re-implemented the GET /system/settings endpoint.
  • Data Integrity: Ensured system settings are correctly mapped from the database to the mobile-expected format.
  • Real-Time Restoration: Verified end-to-end event emission from Transaction Service through Gateway to Pusher.
