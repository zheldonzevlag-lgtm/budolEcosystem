v212

Auth Service Validation & Technical Fixes

Date: 2026-01-06

Purpose

To validate the "GoTyme-Aligned" mobile-first authentication service, resolve database schema inconsistencies, and ensure successful end-to-end flow execution including device trust and biometric registration.

Technical Details & Implementation

  • Database Schema Sync: Resolved "pinHash column missing" error by correcting the DATABASE_URL in .env to point to budolpay_db.
  • Debug Endpoint: Implemented /debug/db-columns in auth-service/index.js for real-time validation of Prisma-to-PostgreSQL sync.
  • E2E Test Hardening:
    • Automated test user cleanup with cascading deletion (Wallet, Sessions, AuditLogs).
    • Dynamic OTP retrieval from database for verification testing.
    • Corrected biometric registration payload (added credentialId and base64 publicKey).
