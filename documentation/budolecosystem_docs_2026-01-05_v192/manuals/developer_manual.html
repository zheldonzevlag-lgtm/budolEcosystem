Developer Manual - v192

Real-time Admin Updates for KYC

This release implements an automated polling mechanism in the Admin Panel to ensure that new KYC submissions from the mobile app are reflected immediately without manual page refreshes.

Technical Implementation

1. Automated Polling

Modified the useEffect hook in [users/page.tsx](file:///d:/IT Projects/budolpay-0.1.0/apps/admin/app/users/page.tsx) to establish a background polling interval.

useEffect(() => {
  fetchUsers();
  
  // Auto-refresh every 5 seconds
  const interval = setInterval(() => {
    fetchUsers(true); // Silent refresh
  }, 5000);

  return () => clearInterval(interval);
}, [filter]);

2. Silent Refresh Mechanism

The fetchUsers function now supports a silent mode, which updates the state without triggering a global loading spinner. This ensures a smooth user experience while browsing the table or reviewing documents.

3. Modal Synchronization

The polling mechanism also updates the selectedUser state if a document review modal is open, allowing administrators to see new documents or status changes in real-time while they are in the review process.
