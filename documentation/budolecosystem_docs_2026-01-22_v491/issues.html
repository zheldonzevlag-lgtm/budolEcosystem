Issues Log - v491

Identified Challenges during Implementation

ISS-491-01: Missing Buyer-Side Realtime Subscriptions.

Description: Initially, the RealtimeProvider only subscribed to store channels, causing buyers to miss status updates unless they manually refreshed.

ISS-491-02: Webhook-Realtime Desync.

Description: Payment webhooks were updating the database but not triggering the realtime broadcast, leading to a delay in UI updates despite successful payment.

ISS-491-03: Confusion over "Needs Booking" Status.

Description: Sellers reported confusion regarding the "Needs Booking" label for paid orders, as it didn't explicitly state the order was ready for shipment.

ISS-491-04: Buyer Remaining on "Unpaid" Tab Post-Payment.

Description: Even with realtime updates, the buyer's view remained on the "Unpaid" tab after payment, requiring manual navigation to "To Ship".
