Developer Manual - Version 112

1. Login 404 Resolution

Issue: The mobile app was unable to log in when built in release mode, returning a 404 error.

Root Cause: The ApiService class in lib/services/api_service.dart was using Vercel URLs for kReleaseMode by default. Since the user is testing the release build on a local LAN, the app could not reach the internet-facing Vercel endpoint, or the endpoint was incorrect for the local SSO service.

Fix: Modified the baseUrl and authUrl logic to default to the local IP 192.168.1.24 even in release mode unless overridden by environment variables.

// Modified in api_service.dart
final String authUrl = String.fromEnvironment(
  'AUTH_URL',
  defaultValue: kReleaseMode 
    ? 'http://$_host:8000' // Local SSO for release testing
    : 'http://$_host:8000',
);
