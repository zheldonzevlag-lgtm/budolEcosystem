GoTyme vs budolPay: Comparative Analysis & Integration Strategy

1. Purpose

The purpose of this document is to provide a comprehensive technical roadmap for upgrading budolPay's authentication security to "Bank-Grade" standards, using GoTyme Philippines as the primary benchmark. This transition moves budolPay from a traditional mobile wallet security model to a stricter, biometric-first digital bank model.

2. Explanation & Reason

Explanation: GoTyme Philippines utilizes a highly secure, mobile-centric authentication flow where biometrics are the primary login method and a 6-digit PIN serves as a fallback. Unlike typical wallets that use 4-digit MPINs and conditional OTPs, GoTyme enforces strong device binding and mandatory OTP verification for risk-based sessions.

Reason: As budolPay expands into more complex financial ecosystems (including budolLoan), it must meet BSP (Bangko Sentral ng Pilipinas) digital banking security standards. Upgrading to a 6-digit PIN and WebAuthn-based biometrics significantly reduces phishing risks and SIM-swapping vulnerabilities.

4. Issues, Risks & Mitigations

Issues Identified: Current 4-digit MPIN is susceptible to brute-force and social engineering. Lack of hardware-backed biometric verification allows for easier session hijacking.

Risk Mitigation
User friction due to mandatory OTP Implement risk-based authentication; only trigger OTP when device fingerprint or location changes significantly.
Device incompatibility Provide the 6-digit PIN as a robust and equally secure fallback for devices without secure enclaves.
Production Costs (SMS) Implement strict rate limiting and explore WhatsApp/Email fallbacks for non-critical alerts.

5. Dependencies & Technical Details

  • Biometrics: SimpleWebAuthn (FIDO2/WebAuthn library).
  • PIN Hashing: Argon2id for superior resistance against GPU-based cracking.
  • State Management: Redis for high-speed OTP and session challenge storage.
  • Database: PostgreSQL with Prisma ORM for managing device-to-user mappings.
Open App ↓ Enter Mobile Number (+63) ↓ Check Device Trust (via Device Fingerprint + WebAuthn) ├─ [NOT TRUSTED] → Send OTP (SMS) → Verify OTP → Register Biometrics (WebAuthn) └─ [TRUSTED] → Proceed ↓ Biometric Prompt (Face ID / Fingerprint) ├─ [SUCCESS] → Issue Session Token → Access Wallet └─ [FAIL / NOT SUPPORTED] → Fallback to 6-digit PIN ↓ Enter 6-digit PIN ├─ [SUCCESS] → Access Wallet └─ [FAIL] → Lock Account (after 3 attempts) → Requires OTP Reset
