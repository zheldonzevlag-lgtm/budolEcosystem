Developer Manual: Domain-Driven Development (v162)

Inter-Service Communication

In the Pure Vercel environment, microservices communicate over HTTPS using their assigned domains. Use the SERVICE_URLS environment variable to manage these endpoints.

Local Development Tip

To test cross-service calls locally, you can use localhost ports as defined in the .env file. The code automatically switches to Vercel domains when process.env.VERCEL is present.

const AUTH_URL = process.env.VERCEL
  ? 'https://budolpay-auth.vercel.app'
  : 'http://localhost:8001';

Deployment Verification

After running vercel deploy, verify that your service is reachable at its production domain specified in the Domain Architecture Manual.

Updating Domains

If a service domain changes, you must update the SERVICE_URLS in the API Gateway project settings on Vercel and trigger a redeploy of the Gateway.
