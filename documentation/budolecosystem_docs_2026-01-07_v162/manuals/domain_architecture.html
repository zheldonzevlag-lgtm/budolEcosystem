Consolidated Domain Architecture (v162)

Overview

This document consolidates the domain architecture requirements from v158 and maps them to the current Pure Vercel microservice deployment. It defines the public-facing and internal communication endpoints for the budolEcosystem.

Central Entry Point: API Gateway

The budolpay-api-gateway.vercel.app domain serves as the primary ingress for all client applications (Mobile & Web). It performs request routing to the specific microservice domains listed above.

  • Protocol: HTTPS only.
  • CORS Policy: Restricted to budolpay.vercel.app and budolid.vercel.app.

Communication Flow

1. Client → Request → api-gateway.vercel.app
2. Gateway → Routing → [service].vercel.app
3. Service → Query → Vercel Postgres
4. Service → Response → Gateway → Client

Risk Mitigation

Latency: Cross-domain communication in a serverless environment adds overhead. Mitigation includes using Vercel's regional deployment (e.g., sin1 for Singapore) to keep services close to each other and the database.
