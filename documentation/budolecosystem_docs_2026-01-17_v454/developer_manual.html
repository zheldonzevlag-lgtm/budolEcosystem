Developer Manual - v454

Forced Ads Navigation

To bypass the hasSeenAds check during development, a FORCE ADS button has been added to the SplashScreen. This uses direct navigation to Routes.marketing regardless of the persistent state.

splash_screen.dart 
TextButton.icon(
  onPressed: () {
    Navigator.pushReplacementNamed(context, Routes.marketing);
  },
  icon: const Icon(Icons.play_arrow, color: Colors.green, size: 16),
  label: const Text('FORCE ADS', ...),
)

Enhanced Rendering Diagnostics

Added diagnostic logging to MarketingAdsScreen to trace the lifecycle from initState to the first frame rendering. This helps identify if the screen is being skipped or failing to render on specific platforms.

marketing_ads_screen.dart 
debugPrint('MarketingAdsScreen: initState - Initializing controller and timer');
WidgetsBinding.instance.addPostFrameCallback((_) {
  debugPrint('MarketingAdsScreen: postFrameCallback - Checking clients');
  ...
});

Files Impacted

  • splash_screen.dart: Added DEBUG TOOLS panel with RESET and FORCE buttons.
  • marketing_ads_screen.dart: Added detailed lifecycle logging and rendering stability checks.
