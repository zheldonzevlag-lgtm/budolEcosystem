Developer Manual - v328

Android OS Integration Fix (FragmentActivity)

Analysis of debug logs revealed that the Android OS was immediately rejecting biometric prompt requests with result: false. This was due to a mismatch between the Flutter Activity type and the requirements of the local_auth plugin.

Fixed

Technical Implementation

  • Manifest Update: Changed android:name from io.flutter.embedding.android.FlutterActivity to .MainActivity in AndroidManifest.xml.
  • Kotlin Activity Update: Verified MainActivity.kt extends FlutterFragmentActivity (required for biometric prompts on Android).
  • Error Capturing: Enhanced BiometricService.dart to catch PlatformException and log specific Android error codes to the on-screen debug log.
  • File: AndroidManifest.xml
  • File: biometric_service.dart

App Versioning

The app version has been incremented to 1.2.52+52.
