Developer Manual: Category Icon Implementation

Technical Explanation

The system now centralizes category icon resolution in components/CategoryIcons.jsx. Instead of returning strings (emojis), the functions now return Lucide React components which can be instantiated in JSX with custom props (size, color, etc.).

Architecture Changes

1. Icon Mapping Logic

New mapping structure in components/CategoryIcons.jsx:

  • SLUG_ICON_MAP: Maps category slugs to Lucide components.
  • getCategoryLucideIcon(slug, name): Resolves the component with fallback to Package.
  • renderCategoryIcon(slug, name, className): Helper for immediate rendering.

Core Component Updates

Homepage Tiles

File: components/CategoriesSection.jsx

Replaced {icon} (emoji string) with <IconComponent className="w-6 h-6" />.

Shop Sidebar & Breadcrumbs

File: app/(public)/shop/page.jsx

Modified to use ActiveCatIcon components in both the sidebar and the active category chip filter.

Admin & Seller UI

  • components/store/add-product/CategorySelector.jsx
  • app/store/manage-product/page.jsx
