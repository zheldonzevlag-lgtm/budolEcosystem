System Admin Manual - Version v255

System Configuration

Real-time behavior is controlled via the /system/settings endpoint on the Gateway. Admins can toggle between providers by updating the JSON configuration:

{ "realtime": { "method": "SOCKETIO", // Options: SOCKETIO, PUSHER, SWR "pusher": { "key": "your_key", "cluster": "ap1" } } }

Deployment Procedures

Mobile App Distribution

  1. The latest APK is available at \budolEcosystem\android-installer\budol_pay_mobile_1.2.20.apk.
  2. Ensure the Gateway is running on version 1.2.x to support the new system settings payload.

Rollback Strategy

If the dynamic switching causes stability issues, roll back the mobile app to v1.2.19. The backend settings are backward compatible, but v1.2.19 will require an app restart to pick up changes in the real-time method.

Dependencies

  • Socket.io Gateway: Must support user-id headers or explicit join events.
  • Pusher Channels: Requires valid API keys and cluster configuration in the system settings.
