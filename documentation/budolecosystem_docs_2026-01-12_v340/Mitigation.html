Mitigation Plan - v340

M-340-01: Schema Synchronization Enforcement

Risk: Prisma schema mismatch during build.

Action: Use the --schema flag in all build scripts to ensure the source of truth is the shared database package. Run npx prisma generate as part of the prebuild step.

M-340-02: Dynamic Rendering Strategy

Risk: Next.js build-time database connection errors.

Action: Apply export const dynamic = 'force-dynamic'; to all server-side pages that fetch data from Prisma. This bypasses static site generation (SSG) for these routes, ensuring they only run at request time when the database is guaranteed to be available.

M-340-03: Network Isolation Mitigation

Risk: Localhost hardcoding in production builds.

Action: Standardize on LOCAL_IP environment variables. Use 192.168.1.17 instead of localhost in all .env files and API route calls.
