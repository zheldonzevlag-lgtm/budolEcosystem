Developer Manual - v340

Purpose & Rationale

Version v340 focuses on achieving stable production builds for the BudolEcosystem web applications (Admin and Consumer). This required standardizing the Prisma schema, fixing TypeScript type errors, and ensuring that Next.js 14 prerendering does not fail due to environment or database missing during build time.

Technical Explanation

1. Prisma Schema Standardization

We updated the shared database schema to include missing models and fields required by the Admin dashboard. This includes the Settlement model for merchant payouts and enhanced AuditLog fields for PCI DSS compliance.

Reference: schema.prisma

model AuditLog {
  id        String   @id @default(uuid())
  userId    String?
  action    String
  entity    String?
  entityId  String?
  newValue  Json?
  ipAddress String?
  // ...
}

Key Changes

  • Prisma Schema: Added Settlement model and updated AuditLog and SystemSetting fields.
  • Type Safety: Created rbac-config.d.ts to provide types for RBAC configuration.
  • Build Optimization: Configured force-dynamic in Next.js pages to prevent build-time database dependency.
  • Bug Fixes: Resolved syntax errors in packages/database/index.ts and corrected NextRequest imports.
