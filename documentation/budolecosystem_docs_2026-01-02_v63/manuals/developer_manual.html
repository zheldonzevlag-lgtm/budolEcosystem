Technical Architecture

  • Framework: Flutter (Dart)
  • State Management: Provider
  • Branding: budolPay Rose Red (#F43F5E) for ecosystem consistency.
  • API Communication: HTTP package with JWT interceptors.
  • Local Storage: Shared Preferences for settings; Secure Storage for tokens.

Code Logic Explanations

Authentication Flow

The app utilizes budolID's SSO flow. The mobile application opens a secure web view or calls the API directly to obtain a JWT token, which is then stored securely on the device.

Wallet Service Integration

Communication is handled via the budolPay API Gateway (default port 8080). The gateway proxies requests to the underlying Wallet Microservice.

// Example API Service call
Future<Map<String, dynamic>> getBalance(String userId) async {
  final response = await http.get(
    Uri.parse('$baseUrl/wallet/balance/$userId'),
    headers: { 'Authorization': 'Bearer $token' },
  );
  return json.decode(response.body);
}

Impact Analysis

The introduction of the mobile app has zero breaking changes on existing services. However, it requires:

  • New API endpoints in budolPay for mobile-specific features like push notification registration.
  • CORS updates to allow requests from the mobile app's origin if applicable.
