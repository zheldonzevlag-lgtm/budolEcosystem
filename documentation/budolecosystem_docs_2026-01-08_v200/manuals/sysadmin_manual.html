System Admin Manual Updates - v200

Post-Deployment Verification

Admins should perform the following checks after the v200 rollout:

  1. Verify that the Auth Service logs show successful setup-pin events for users previously stuck in the login loop.
  2. Monitor the Gateway logs for any 401 errors originating from /login/mobile/verify-pin.
  3. Ensure that OTP codes are being generated and logged (in local mode) or sent (in production mode).

Support Runbook: "PIN not set" Error

If a user reports the "PIN not set" error message on their screen:

  • Instruct the user to tap "Back to Phone Number".
  • Ask them to re-enter their phone number to trigger the fresh identification flow.
  • This will correctly route them to the OTP and then the PIN setup screen.
