Developer Manual Updates - v200

Final Authentication State Machine

The mobile authentication flow is now a robust multi-phase state machine implemented across the Auth Service and API Gateway.

Phase 1: Identification

Checks phone existence, device trust, and PIN availability. Returns OTP_REQUIRED or AUTH_REQUIRED.

Phase 2: Challenge (OTP or PIN)

If OTP_REQUIRED, the client must call /verify-otp. If AUTH_REQUIRED, the client must call /login/mobile/verify-pin.

Phase 3: Finalization & Setup

If /verify-otp returns needsPinSetup: true, the client must call /login/mobile/setup-pin before entering the dashboard.

Test Coverage

Automated integration tests now cover the following edge cases:

  • Trusted device with missing PIN.
  • Untrusted device (first login).
  • Invalid PIN attempt on trusted device.
  • OTP expiry and invalid code handling.
