1. Purpose

The purpose of this update is to resolve the synchronization issue where user profile updates (name, account type) performed in the BudolShap Admin panel were not persisting across SSO logins. This was due to missing endpoints in BudolID and incomplete data in BudolShap's user metadata.

2. Explanation & Reason

In the BudolEcosystem, BudolID serves as the centralized identity provider. When a user logs in via SSO, BudolShap synchronizes its local user record with the data returned by BudolID. Previously, changes made in BudolShap's Admin panel were only local, meaning they were overwritten by BudolID's "truth" during the next login. Furthermore, some users lacked the ssoUserId link required for BudolShap to communicate updates back to BudolID.

3. Issues Identified

  • Missing API Endpoints: BudolID lacked a profile update API, causing budolIdClient.js to call non-existent routes.
  • Incomplete Metadata: Some users (like the manually provisioned galvezjon59@gmail.com) lacked the ssoUserId in their metadata, breaking the link between services.
  • Limited Sync Logic: The BudolShap Admin update API was only attempting to sync the name field, ignoring accountType/role and image/avatarUrl.

4. Fixes Implemented

  • BudolID API Enhancement: Added PATCH /users/:userId/profile to BudolID to support updates to firstName, lastName, avatarUrl, and role.
  • BudolShap Admin API Update: Modified the user update route to synchronize all profile changes (name, role, avatar) to BudolID whenever a ssoUserId is present.
  • Data Correction: Updated the metadata for galvezjon59@gmail.com in the local database to include its corresponding BudolID user ID.
  • Role Mapping: Implemented mapping between BudolShap's accountType (ADMIN/BUYER) and BudolID's role (ADMIN/USER).

5. Technical Details

BudolID New Endpoint

budolID-0.1.0/api/index.js:

app.patch('/users/:userId/profile', async (req, res) => {
    const { userId } = req.params;
    const { firstName, lastName, avatarUrl, role } = req.body;
    // ... Prisma update logic ...
});

BudolShap Sync Logic

budolshap-0.1.0/app/api/admin/users/[userId]/route.js:

// Sync changes to budolID if possible
if (Object.keys(ssoProfileUpdate).length > 0) {
    let ssoUserId = existingUser.metadata?.ssoUserId;
    if (ssoUserId) {
        await updateUserProfile(ssoUserId, ssoProfileUpdate);
    }
}

7. Risks & Mitigation

  • Risk: Network failure during BudolID sync could leave services out of sync.
  • Mitigation: Added logging for sync failures; the local update still proceeds, but the discrepancy is logged for administrative review.

8. Dependencies

  • BudolID Service (API update required)
  • BudolShap Service (Admin route update required)
  • Shared User Metadata (ssoUserId)

9. Recommendations for Future Enhancement

  • Implement a "Sync All Users" tool in the Admin panel to reconcile discrepancies across the entire ecosystem.
  • Add a "Source of Truth" indicator in the UI to show which service is managing specific user data.
