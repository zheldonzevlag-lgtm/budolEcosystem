Risk Mitigation - v348

R-348-01: Non-PH Formats

The current implementation strictly enforces the 09XX-XXX-XXXX format. This is intentional to prevent invalid registrations in the primary Philippine market. If international expansion occurs, the validation logic will be moved to a service that detects country codes.

R-348-02: Backend Parsing

The _handleIdentify method in login_screen.dart continues to send the raw string from the controller. The backend identifyMobile API has been verified to handle both hyphenated and non-hyphenated strings by stripping non-numeric characters before database lookup.
