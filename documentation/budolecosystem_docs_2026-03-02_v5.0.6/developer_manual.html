Technical Fixes & Rationale

1. Code-Level Schema Enforcement

To prevent regressions from .env files reverting to schema=public, I implemented a wrapper in the Prisma initialization logic of @budolpay/database. This wrapper automatically appends ?schema=budolpay to the connection string if it's missing.

File: budolpay-0.1.0/packages/database/index.js

2. Resilient Checkout Metadata

Application-specific fields (failedAt, paidAt, failureReason) are now consistently mapped to the metadata JSON field in the Checkout model, ensuring zero schema collisions between the app and the gateway.

File: budolshap-0.1.0/lib/services/checkoutService.js
