Issues & Fixes

Partial Payment Status for Multi-Store Orders

FIXED

Issue: When a user placed an order spanning multiple stores, the system created multiple Order records (one per store). However, the single payment transaction was only linked to the first Order ID returned by the API. This resulted in the first order being marked as PAID, while subsequent orders remained UNPAID despite the payment covering the total amount.

Root Cause: Frontend only captured the first ID from the response array (data[0].id) and sent it to the payment service. The payment service and webhooks were unaware of the other orders.

Fix:

  • Updated OrderSummary.jsx to send all orderIds.
  • Updated Payment API to link all IDs to the payment intent.
  • Updated Webhooks to iterate through all IDs and update statuses.
