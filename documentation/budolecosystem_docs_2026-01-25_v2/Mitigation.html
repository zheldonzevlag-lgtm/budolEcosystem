Mitigation Strategies

Risk: Mismatch between selected shipping and displayed shipping

Mitigation: The createOrder service now strictly maps the shipping object from the request to the database. Frontend display logic in OrderItem.jsx has been updated to use the provider field from the shipping object instead of a default string.

Risk: Loss of shipping provider metadata

Mitigation: The Prisma schema uses a Json type for the shipping field, allowing for flexible yet structured storage of provider-specific metadata (e.g., quoteId, serviceType). Validation has been added in the backend to ensure this field is populated if provided.

Risk: Lalamove API downtime

Mitigation: A fallback mechanism is in place (fallbackProvider.js) that provides "Standard Delivery" if external APIs are unavailable or if the user chooses the default option. This ensures the checkout flow is never completely blocked by external dependencies.
