Developer Manual

Technical Rationale

The previous implementation of the order creation flow in ordersService.js was hardcoding the shipping cost and failing to persist the user-selected shipping provider data. This led to a mismatch between what the user selected during checkout and what was displayed in the order history.

Data Contract Changes

The createOrder function now explicitly accepts shipping and shippingCost from the request payload.

Code Changes

1. Backend: ordersService.js

Modified the createOrder function to handle incoming shipping data.

export async function createOrder(orderData) {
    const { userId, addressId, orderItems, paymentMethod, couponCode, shippingCost: providedShippingCost, shipping } = orderData;
    // ...
    const shippingCost = providedShippingCost !== undefined ? providedShippingCost : 50;
    // ...
    const order = await tx.order.create({
        data: {
            // ...
            shippingCost,
            shipping: shipping || { provider: 'standard' },
            // ...
        }
    });
}

Dependencies & Software Rationale

  • Next.js: Chosen for its robust SSR/ISR capabilities, essential for SEO and fast loading of product pages.
  • Prisma: Selected as the ORM for its type-safety and ease of use, ensuring data integrity.
  • Tailwind CSS: Used for rapid, utility-first UI development, ensuring responsive designs.
  • Lucide React: Lightweight and consistent icon set for a modern UI.
  • Lalamove API: Integrated for reliable, real-time logistics solutions in the Philippines.
