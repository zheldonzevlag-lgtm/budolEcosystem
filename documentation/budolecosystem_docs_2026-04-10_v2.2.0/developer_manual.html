Developer Manual: Local SSO Stabilization

Overview

This document details the configuration changes required to ensure BudolID SSO works correctly in a local developer environment using the LAN IP 192.168.1.2.

Core Configuration Changes

The primary fix involved updating the .env.connection.local template which acts as the source for all ecosystem services.

1. Template Update

File: budolshap-0.1.0/.env.connection.local

# Added consistent LAN IP URLs
NEXT_PUBLIC_SSO_URL=http://192.168.1.2:8000
BUDOLACCOUNTING_URL=http://192.168.1.2:8005
NEXT_PUBLIC_APP_URL=http://192.168.1.2:3000
NEXT_PUBLIC_BASE_URL=http://192.168.1.2:3000
LOCAL_IP=192.168.1.2

2. Synchronization Script

The env-manager.cjs script was used to propagate these values across all 11 relevant .env files in the ecosystem.

node scripts/env-manager.cjs local

Code References

  • Redirection Logic: budolID-0.1.0/index.js (L1310-1314) - Validates the dynamic redirect_uri.
  • Environment Management: env-manager.cjs - Handles the secure propagation of connection strings.
Important: After updating .env files, you must restart all services (BudolID, BudolShap, etc.) for the changes to take effect in the process environment.
