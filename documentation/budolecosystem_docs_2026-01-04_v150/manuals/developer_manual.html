Developer Manual - Version 150

Purpose

To improve the visual presentation of the wallet balance by replacing the 'PHP' text prefix with the official Philippine Peso symbol (₱) and ensuring large numbers are formatted with standard comma separators (e.g., ₱20,000,000.00 instead of PHP 20000000.00).

Technical Implementation Details

1. Currency Formatting Logic

Integrated the intl package to handle locale-aware number formatting. Created a helper method _formatBalance in HomeScreen.dart:

String _formatBalance(double value) {
  final formatter = NumberFormat.currency(
    locale: 'en_PH',
    symbol: '₱',
    decimalDigits: 2,
  );
  return formatter.format(value);
}

2. UI Integration

Updated all balance-related setState calls and the initial state to use the new formatter. The balance display in the Text widget was simplified as the symbol is now part of the formatted string.

// Before
Text('PHP $_balance', ...)

// After
Text(_balance, ...)

Issues & Fixes

  • Issue: Balance display lacked readability for large amounts (no commas).
    Fix: Implemented NumberFormat.currency with en_PH locale.
  • Issue: Branded 'PHP' text felt inconsistent with modern mobile app UI.
    Fix: Replaced with ₱ symbol integrated into the formatted string.
