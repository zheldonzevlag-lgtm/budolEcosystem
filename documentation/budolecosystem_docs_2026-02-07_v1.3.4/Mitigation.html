Mitigation Plan (v1.3.4)

M-134-01: UI Fallback Strategy

To prevent "empty" UI states, all frontend components (Navbar, Profile, Orders) must implement a fallback logic: user.name || user.phoneNumber || 'Guest'.

M-134-02: Privacy & Masking

For future releases, we recommend masking the phone number (e.g., +639****5678) in non-essential UI views to enhance user privacy while maintaining the phone-primary identity.

M-134-03: User Education

The registration success screen should explicitly mention that the user can set a custom display name in their Profile to replace the default phone number display.
