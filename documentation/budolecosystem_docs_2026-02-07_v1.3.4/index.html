v1.3.4

Release Overview: Phone-Primary Identity

Released: 2026-02-07 | Security Status: Verified

Version 1.3.4 implements a streamlined identity model inspired by Lazada/Shopee process flows. The "User" prefix has been removed from phone-only registrations to provide a cleaner, more professional user experience where the phone number acts as the primary identifier.

Rationale

Modern e-commerce platforms (Lazada, Shopee) prioritize the phone number as the root of identity. Prepending "User " to the phone number creates unnecessary visual clutter and doesn't align with local market expectations for "Quick Registration" flows.

Key Changes

  • Removed "User " prefix in budolshap registration API.
  • Updated budolID SSO to use phone number as default first name for quick reg.
  • Enhanced Navbar and Profile UI to gracefully fallback to phone number if name is unset.
  • Updated dual-channel OTP notifications to use phone number in greetings when name is missing.

Process Flow (Lazada Aligned)

The system now follows the 📱 Lazada Process Flow (Phone Number–Based):

  1. Account Registration / Login: Primary identifier is the Phone Number (+63).
  2. OTP Verification: SMS-first verification with email backup.
  3. Profile Setup: Phone number is automatically verified and used for recovery.
