Developer Manual - v485

1. KYC Verification Architecture

The KYC process involves the mobile app capturing a selfie and ID scan, which are then optimized on-device and uploaded to the Verification Service via the API Gateway.

2. KYC Enum Fix

The 500 Internal Server Error was caused by a mismatch between the backend service logic and the Prisma schema enums. The backend was attempting to set kycStatus to 'APPROVED', which does not exist in the KYCStatus enum (valid values: NONE, PENDING, VERIFIED, REJECTED).

// d:/IT Projects/budolEcosystem/budolpay-0.1.0/services/verification-service/index.js

// FIXED LOGIC:
if (type === 'SELFIE' && faceTemplate) {
    updateData.faceTemplate = faceTemplate;
    updateData.isFaceVerified = true;
    updateData.kycStatus = 'VERIFIED'; // Changed from 'APPROVED'
    updateData.kycTier = 'FULLY_VERIFIED';
}
