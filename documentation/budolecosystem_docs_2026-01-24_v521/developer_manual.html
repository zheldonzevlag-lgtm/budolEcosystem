Purpose

This manual documents the implementation of the Order Rollback mechanism, QR Code Modal refinements, Admin Search fixes, and Customer Name verification. These changes ensure a more robust checkout experience, accurate data display, and a bug-free administrative interface.

Technical Explanation

1. Order Rollback Mechanism

The system now follows a "Pending Order" pattern. When a user clicks "Place Order", an order is created with a pending status. If the payment process (QR Code) is interrupted, the frontend triggers a cancellation request to the backend.

The centralized cancellation logic is implemented in ordersService.js. It uses a Prisma transaction to ensure atomicity:

export async function cancelOrder(orderId, restoreCart = true) {
  return await prisma.$transaction(async (tx) => {
    const order = await tx.order.findUnique({
      where: { id: orderId },
      include: { orderItems: true }
    });
    // ... validation ...
    const updatedOrder = await tx.order.update({
      where: { id: orderId },
      data: { status: 'CANCELLED', paymentStatus: 'cancelled' }
    });
    // ... restore cart items ...
    return updatedOrder;
  });
}

2. QR Code Modal Refinements

In QRCodeModal.jsx, we refined the display to show the store owner and store name separately. The "Store Owner" is now hardcoded to "Jon Galvez" for the demo store, and the Payment ID is displayed in full using break-all for responsiveness.

3. Admin Search Context Fixes

Administrative pages (Users, Products, Orders, Returns) were updated to correctly use the searchQuery and updateSearchQuery from the SearchContext.jsx. This resolved "search is not defined" runtime errors caused by accessing undefined variables.

4. Customer Name Standardization

To ensure consistent testing, "Natasha Romanoff" has been set as the default buyer name in seed-test-data.js and as a placeholder in AddressFormManager.jsx and AuthForm.jsx.

Data Contracts

Endpoint: POST /api/orders/[orderId]/cancel

Response: Returns the updated order object with status: 'CANCELLED'.

Compliance

This implementation aligns with DTI and NPC guidelines regarding fair trade and data accuracy, ensuring users are not billed for incomplete transactions and inventory is correctly managed.
