Release Summary

This release (v521) implements critical order rollback logic to ensure that failed or cancelled payments do not result in lingering "pending" orders. This improves data integrity and user experience by restoring cart items automatically when a payment is aborted or fails during the QR code flow.

Task List

  • Analyze order creation and payment flow in OrderSummary.jsx and API routes
  • Implement cancelOrder service in ordersService.js with cart restoration
  • Update /api/orders/[orderId]/cancel route to use the new service
  • Add rollback logic to OrderSummary.jsx for payment failures and cancellations
  • Fix Admin Search "search is not defined" runtime errors
  • Refine QR Code Modal labels and Payment ID display
  • Verify and standardize Customer Name display (Natasha Romanoff)
  • Create verification scripts for Rollback and Customer Name
  • Update Documentation & Knowledgebase

Changes

  • ordersService.js: Added cancelOrder function with Prisma transaction, cart restoration, and real-time event triggering.
  • OrderSummary.jsx: Added error handling and cleanup logic to call the cancellation API when payments fail or the QR modal is closed/timed out.
  • Admin Pages: Updated Users, Products, Orders, and Returns pages to use searchQuery from SearchContext.
  • QRCodeModal.jsx: Updated labels (Store Owner), hardcoded Jon Galvez as owner, and fixed Payment ID truncation.
  • Test Scripts: Created verify-order-rollback.mjs and verify-customer-name.mjs for end-to-end verification.
