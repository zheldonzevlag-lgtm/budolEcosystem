Developer Manual - v191

KYC Data Cleanup (Tony Stark & Jon Galvez)

This release documents the targeted cleanup of KYC documents and user status resets for specific test accounts.

Technical Implementation

A custom maintenance script [delete-kyc.js](file:///d:/IT Projects/budolpay-0.1.0/packages/database/delete-kyc.js) was executed to perform the following operations:

1. Target Identification

Users were identified using case-insensitive name matching for "Tony Stark" and "Jon Galvez".

OR: [
  { firstName: { contains: 'Tony', mode: 'insensitive' }, lastName: { contains: 'Stark', mode: 'insensitive' } },
  { firstName: { contains: 'Jon', mode: 'insensitive' }, lastName: { contains: 'Galvez', mode: 'insensitive' } }
]

2. Document Deletion

All records in the VerificationDocument table associated with these user IDs were deleted.

3. Status Reset

The users' KYC status was reset to NONE and their KYC Tier was reset to BASIC, allowing them to restart the verification process from scratch.

Results

  • Tony Stark: Deleted 4 documents, reset to BASIC.
  • Jon Galvez: Deleted 6 documents, reset to BASIC.
