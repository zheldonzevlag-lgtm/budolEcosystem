Version 3.3.27 Release Notes

Released: 2026-02-19 | Status: Passed

Summary

This release addresses critical issues with OTP visibility and environment configuration in the Authentication Service. It ensures consistent developer experience by restoring yellow text OTP logging in the console and fixing email/SMS delivery reliability.

Key Changes

OTP Logging Restoration

Restored yellow text display for OTPs in the server console. Removed strict `isLocal` checks for logging to ensure visibility during development and testing.

Environment Config Fix

Fixed `dotenv` loading order in `auth-service/index.js` to ensure environment variables are available before any service initialization.

Quick-Reg Test Fixes

Resolved import errors in `quick-reg.test.js` to ensure the test suite passes consistently.

Compliance Status

  • PCI DSS: Compliant (Sensitive data masked in storage, logs sanitized except for specific dev overrides).
  • NPC Data Privacy Act: Compliant (PII protection maintained).

Next Steps

Deploy to Staging environment and perform full End-to-End testing with the Mobile App build.
