Developer Manual - v3.3.27

Technical Guide for Backend & Test Implementations

1. Environment Configuration Fix

Rationale: The dotenv package was being loaded too late in the execution flow, causing environment variables (like email credentials and `NODE_ENV`) to be undefined during initial service setup. This led to failures in email/SMS sending and incorrect `isLocal` flag evaluation.

Implementation: Moved `dotenv` loading to the very top of `services/auth-service/index.js`.

const path = require('path'); const envPath = path.resolve(__dirname, '../../.env'); // Load .env immediately before any other imports require('dotenv').config({ path: envPath, override: true }); const express = require('express'); ...

Reference: services/auth-service/index.js

2. OTP Logging & Visibility

Rationale: Developers require immediate visibility of OTP codes in the console for testing and debugging without checking databases or email/SMS logs. Previous logic suppressed these logs or hid them based on environment flags inconsistent with local development needs.

Implementation: Removed strict `isLocal` conditional checks for the console log statement and enforced ANSI yellow color code for visibility.

// Always log OTP for visibility during dev/test console.log(`[LOCAL] Login OTP for ${maskPII(recipient)}: \x1b[33m${otpCode}\x1b[0m`);

Reference: services/auth-service/index.js

3. Test Suite Corrections

Rationale: The `quick-reg.test.js` file was failing because it was importing the entire module exports object as `app`, instead of destructuring the `app` property from the exports.

Implementation: Updated the import statement in the test file.

const { app } = require('../index'); // Destructured import const request = require('supertest');

Reference: services/auth-service/tests/quick-reg.test.js

4. Dependencies

  • dotenv: Version ^16.x. Used for environment variable management. Critical to load this first.
  • supertest: Version ^6.x. Used for HTTP assertions in tests.
  • jest: Test runner.
