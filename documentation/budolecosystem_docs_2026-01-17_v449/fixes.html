Issues & Fixes - v449

Issue: Flutter Web Black Screen Post-Splash

Symptoms: After the splash screen logo disappears, users see a blank black screen. The app appears frozen, but logs show it has navigated to the Marketing Ads screen.

Root Cause: The removeSplashFromWeb JS function was called in the SplashScreen before the next screen was fully painted. Additionally, RepaintBoundary on web sometimes fails to initialize its layer quickly enough, resulting in a black frame.

Fix: Multi-Layered Rendering Guard

  • Deferred Splash Removal: Moved the removal call to the PostFrameCallback of the target screens (Marketing, Login, Home) with an additional 100-200ms delay to ensure the canvas is active.
  • Constraint Optimization: Updated LayoutBuilder to handle non-finite constraints on web by providing sensible defaults (393x780).
  • Layer Cleanup: Removed RepaintBoundary from the MarketingAdsScreen to simplify the initial render path.

Issue: SharedPreferences Timeout on Web

Symptoms: App hangs on splash screen indefinitely on some browsers.

Fix: Added 2-second timeout to PackageInfo and improved ApiService.init() error handling to prevent blocking the entire app initialization.
