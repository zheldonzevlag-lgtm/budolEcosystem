Developer Manual - v449

Technical Architecture: Web Splash Optimization

The core problem addressed in v449 was the race condition between the Flutter engine's initial frame paint and the removal of the HTML splash screen (defined in index.html). On slower web connections or hardware, the HTML splash would be removed before the Flutter MarketingAdsScreen was ready to render, resulting in a "black screen" (which is actually the dark background of the Scaffold).

Core Changes

1. Marketing Ads Screen Guards

We removed the RepaintBoundary which was suspected of causing layer initialization delays on web. We also introduced a PostFrameCallback to ensure the splash is only removed after the first frame is painted.

marketing_ads_screen.dart 
// Optimization: Moved splash removal to the target screen
WidgetsBinding.instance.addPostFrameCallback((_) {
  if (mounted) {
    if (kIsWeb) {
      // Give the engine 100ms buffer after first frame
      Future.delayed(const Duration(milliseconds: 100), () {
        if (mounted) {
          callJsMethod('removeSplashFromWeb');
        }
      });
    }
  }
});

2. Splash Screen Navigation Logic

The SplashScreen now focuses solely on navigation state (auth check, ads seen check) and no longer prematurely removes the web splash. This ensures the "hand-off" between screens is seamless.

splash_screen.dart 
// Removed premature callJsMethod('removeSplashFromWeb')
if (mounted) {
  if (!hasSeenAds) {
    Navigator.pushReplacementNamed(context, Routes.marketing);
  } else if (isAuthenticated) {
    Navigator.pushReplacementNamed(context, Routes.home);
  } else {
    Navigator.pushReplacementNamed(context, Routes.login);
  }
}

3. Direct Navigation Fallbacks

Added splash removal guards to LoginScreen and HomeScreen for cases where the user is already authenticated or has seen ads, bypassing the marketing screen.

login_screen.dart 
if (kIsWeb) {
  WidgetsBinding.instance.addPostFrameCallback((_) {
    Future.delayed(const Duration(milliseconds: 200), () {
      if (mounted) callJsMethod('removeSplashFromWeb');
    });
  });
}

Dependencies

  • SharedPreferences: Used for hasSeenAds state persistence.
  • js_helper.dart: Provides the bridge to browser-side JavaScript.
  • LayoutBuilder: Critical for detecting container constraints on web.
