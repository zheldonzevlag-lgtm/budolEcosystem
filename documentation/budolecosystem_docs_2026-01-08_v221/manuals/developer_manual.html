Developer Manual: v221 Database Sanitation

1. Selective Record Deletion Logic

Purpose: To clear the BudolPay database of all transactional and session data while maintaining key testing identities.

Technical Detail: The cleanup script uses Prisma's deleteMany and updateMany within a specific order to respect foreign key constraints.

Order of Deletion:

  1. Dispute (Child of Transaction)
  2. LedgerEntry (Child of Transaction/Account)
  3. Transaction
  4. Session (Child of User)
  5. AuditLog (Child of User)
  6. VerificationDocument (Child of User)
  7. Wallet (Child of non-essential User)
  8. User (Non-essential only)
