Consolidated Recommendations for Database Management

1. Automated Daily Sandbox Reset

Purpose: Schedule the cleanup-db.mjs script to run automatically every 24 hours.

Reason: Keeps the testing environment clean and prevents the accumulation of stale test data.

2. Data Seeding Integration

Purpose: Integrate the cleanup script with a seeding script that repopulates standard test scenarios (e.g., pre-defined balances, specific transaction states).

Reason: Allows for consistent, repeatable test results after every reset.

3. Soft-Delete Implementation

Purpose: Transition from hard-deletes to a soft-delete mechanism using a deletedAt timestamp.

Reason: Provides better auditability and allows for recovery of accidentally deleted records while still hiding them from the active UI.
