SysAdmin Manual: Database Sanitation & Sandbox Resets

1. Executing the Sandbox Reset

To reset the BudolPay database to its core state, execute the following command in the budolpay-0.1.0 directory:

node cleanup-db.mjs

Warning: This operation is destructive and cannot be undone. It will permanently delete all transaction history.

2. User Retention Policy

The cleanup script is configured to retain only authorized test users. If new test accounts need to be whitelisted, modify the keepEmails array in cleanup-db.mjs before execution.

3. Wallet Balance Integrity

Post-cleanup, all retained wallets are reset to 0.00. If specific starting balances are required for testing, they must be manually updated via the Wallet table or a separate initialization script.

4. Post-Cleanup Verification

After running the cleanup, ensure the API Gateway and microservices are restarted to clear any cached session tokens that may still point to deleted users.
