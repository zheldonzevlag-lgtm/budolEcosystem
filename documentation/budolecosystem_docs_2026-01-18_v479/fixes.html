Fixes Applied - Version 479

Fix for ISS-479-001: Forensic Audit Logging Compliance

Rationale: BSP Circular 808 & PCI DSS 10.2.2 Compliance.

Technical Explanation: Implemented a standardized createAuditLog helper across Auth, Wallet, and Transaction services. This helper captures IP address, user agent, and compliant Manila timestamps (UTC+8) for every critical action.

Files: auth-service/index.js, wallet-service/index.js, transaction-service/index.js

Fix for ISS-479-002: Real-time Admin Notification Sync

Rationale: Operational visibility and security monitoring.

Technical Explanation: Integrated notifyAdmin calls into the audit logging flow. Critical events are now sent to the API Gateway via internal POST, which then broadcasts to the Admin App via Socket.io/Pusher.

Files: api-gateway/index.js, auth-service/index.js

Fix for ISS-479-003: Port Conflict Resolution

Rationale: Service availability and system stability.

Technical Explanation: Resolved a port conflict where multiple services were attempting to use the same port. Settlement Service was moved to port 8007, and the ecosystem runner (start-ecosystem.js) was updated to manage it.

Files: settlement-service/index.js, start-ecosystem.js
