Developer Manual - Version 479

1. Asia/Manila (UTC+8) Timezone Enforcement

To comply with BSP Circular 808 and PCI DSS 10.2.2, all timestamps generated by the system must reflect Asia/Manila (UTC+8). This ensures forensic accuracy and regulatory alignment.

Backend Pattern (Node.js)

The standardized implementation for all JavaScript/Node.js microservices is:

new Date(new Date().getTime() + (8 * 60 * 60 * 1000))

This pattern was applied to over 20+ critical endpoints across the ecosystem.

Mobile Pattern (Flutter)

A centralized TimezoneUtils class was created to handle Manila time:

static DateTime getManilaNow() { return DateTime.now().toUtc().add(const Duration(hours: 8)); }

3. Forensic Audit Logging (PCI DSS 10.2.2 & BSP Circular 808)

A standardized createAuditLog helper was implemented across critical services (Auth, Wallet, Transaction) to ensure forensic integrity.

const createAuditLog = async (req, userId, action, metadata = {}, entity = 'Financial', entityId = null) => { const log = await prisma.auditLog.create({ data: { userId, action, entity, entityId: entityId || userId, ipAddress: req.headers['x-forwarded-for'] || req.socket.remoteAddress, userAgent: req.headers['user-agent'], metadata: { ...metadata, compliance: { pci_dss: '10.2.2', bsp: 'Circular 808' }, timestamp: new Date(new Date().getTime() + (8 * 60 * 60 * 1000)).toISOString() } } }); await notifyAdmin('AUDIT_LOG_CREATED', log); };

This implementation ensures that every critical action is logged with IP, User Agent, and Manila Time, and immediately broadcasted to the Admin Dashboard.

5. Dependencies and Software Choice (PCI/BSP/NPC)

The selection of technologies for v479 was driven by regulatory compliance and forensic requirements:

  • Node.js & Prisma: Chosen for rapid development and type-safe database access, crucial for maintaining data integrity in audit logs (PCI DSS 10.2).
  • Axios: Used for reliable inter-service communication (API Gateway to microservices), enabling real-time compliance notifications.
  • UTC+8 Offset (Manual): While libraries like moment-timezone exist, a manual 8-hour offset was chosen for lightweight execution in high-performance microservices, ensuring BSP compliance without adding unnecessary dependency overhead.
  • X-Forwarded-For: Standard headers were used to capture original client IPs, satisfying PCI DSS requirements for source tracking in audit trails.
