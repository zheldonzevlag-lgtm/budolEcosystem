Developer Manual - BudolEcosystem v5.0.5

Release Date: 2026-03-02

Status: Schema Deep Dive & Fix

v5.0.5 Bug Fixes Task List

✔ Deep dive into database schemas (public vs budolpay) to verify Transaction table structure
✔ Re-apply schema=budolpay to all budolPay .env files (Fixing regressions)
✔ Corrected Checkout model field usage (moved failedAt, paidAt to metadata JSON)

Purpose

This release provides a definitive fix for the persistent database schema conflicts and model validation errors by logically isolating budolPay data and aligning the service layer with the existing schema.

Database Deep Dive Results

1. Transaction Table Discrepancy

A deep dive revealed that the Transaction table exists in both public (budolShap) and budolpay schemas but with different structures:

  • public.Transaction: Lacks the status column. Used for internal wallet logging.
  • budolpay.Transaction: Contains the status column. Used for payment gateway operations.

The error occurred because budolPay services were defaulting to the public schema.

2. Checkout Table Alignment

The Checkout table in the public schema does not contain failedAt, paidAt, or expiredAt columns. Prisma validation errors were triggered by attempts to update these non-existent fields.

Technical Fixes

1. Schema Isolation Re-enforcement

Re-applied the ?schema=budolpay suffix to the DATABASE_URL in all budolPay microservices. This ensures they correctly target the budolpay.Transaction table.

Affected Files: All .env files in budolpay-0.1.0/services/* and budolpay-0.1.0/packages/database/.

2. Metadata Mapping for Timestamps

In checkoutService.js, application-specific timestamps (failedAt, paidAt, expiredAt) and failure reasons are now correctly mapped to the metadata JSON field instead of direct column updates.

File: lib/services/checkoutService.js

Compliance & Risk

Compliance: BSP Circular No. 808 - Financial Transaction Audit integrity is maintained through schema-level isolation.

Risk: Minimal. Services must be restarted to pick up the updated ENV configurations.
