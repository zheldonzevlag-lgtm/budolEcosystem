v5.0.5: Schema Deep Dive & Definitive Fix

Date: 2026-03-02 Version: v5.0.5 Status: Released

Implementation Summary

This release follows a comprehensive database deep dive to resolve the persistent Transaction.status error and PrismaClientValidationError. It enforces logical data separation via PostgreSQL schemas and aligns the service layer with the existing data model.

  • Schema Isolation: Re-applied ?schema=budolpay to all microservices to correctly target the payment gateway's Transaction table.
  • Model Stabilization: Mapped custom timestamps (failedAt, paidAt, expiredAt) to the metadata field in the Checkout model.
  • Deep Dive: Verified discrepancies between public and budolpay schemas to confirm the root cause of service crashes.
