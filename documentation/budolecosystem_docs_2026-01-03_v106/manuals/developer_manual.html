Developer Manual - v106

1. MainActivity Transition (Kotlin to Java)

Due to unresolved references in the Kotlin implementation of MainActivity, the project has been transitioned to a Java-based MainActivity. This ensures better compatibility with the current build environment and simplifies dependency management.

// Location: android/app/src/main/java/com/example/budol_pay_mobile/MainActivity.java
package com.example.budol_pay_mobile;
import io.flutter.embedding.android.FlutterActivity;
public class MainActivity extends FlutterActivity {}

The old MainActivity.kt was removed to prevent compilation conflicts.

2. Plugin Registration Fixes

Identified issues in GeneratedPluginRegistrant.java where mobile_scanner and url_launcher plugins were failing to resolve. These were addressed by ensuring flutter pub get is synchronized and the Android source sets correctly include the Java directories.

3. Dependency Synchronization

Successfully ran flutter pub get to align all package versions, specifically ensuring mobile_scanner (3.5.7) and url_launcher (6.1.11) are correctly linked for the Android build.

4. Impact Analysis

  • Build Stability: Improved by removing conflicting Kotlin source files.
  • Redirection Logic: Post-scan redirection in qr_scanner_screen.dart remains intact, awaiting APK verification.
