SysAdmin Manual - v106

1. Android Build Environment Optimization

The build environment was optimized to handle JVM memory constraints. The gradle.properties file was updated to allocate sufficient heap space for the Kotlin and Java compilers.

org.gradle.jvmargs=-Xmx2048m -XX:MaxMetaspaceSize=512m
org.gradle.daemon=true

2. SDK and NDK Requirements

The build process now requires:

  • Android SDK Platform: 34
  • NDK (Side by side): 27.0.12077973

These are automatically installed by the Flutter build tool if missing, provided the licenses are accepted.

3. Troubleshooting Compilation Failures

If the build fails with "Unresolved reference: io", follow these steps:

  1. Run flutter clean to clear build artifacts.
  2. Run flutter pub get to refresh dependencies.
  3. Ensure local.properties correctly points to the Flutter and Android SDK paths.
  4. Verify that only one MainActivity exists (either Java or Kotlin).

4. APK Generation Command

Run the following command to generate the debug APK while skipping dependency validation (useful in environments with pre-installed SDKs):

flutter build apk --debug --android-skip-build-dependency-validation
