System Admin Manual - v65

1. Web Deployment & Preview Configuration

The web preview is served via http-server. The container dimensions in main.dart control how the app is rendered on the web.

1.1 Container Dimensions

To ensure a consistent "mobile-first" preview experience on desktop, the BudolPayApp builder wraps the app in a constrained container when running on the web:

// Updated in main.dart
width: 393,  // Standard Pro width
height: 852, // Standard Pro height

1.2 Building for Web

After any UI changes, the web build must be refreshed:

cd budolPayMobile
flutter build web --release --base-href /

1.3 Serving the App

The app is served from the build/web directory:

npx http-server build/web -p 8081 -c-1

2. Branding Assets

The app no longer relies on external image assets for the logo, using a native Flutter Text widget with the ₱ sign. This reduces bundle size and improves load times for the web preview.
