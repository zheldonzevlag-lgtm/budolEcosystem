Documentation v89: Address Invalid Fix & Connection Stability

Published: 2026-01-02 | Version: 89

Issue Analysis

Error: ERR_ADDRESS_INVALID when navigating to http://0.0.0.0:8080/ and AppConnectionException in debug console.

Diagnosis: 0.0.0.0 is a non-routable binding address, not a valid browsable URL. Additionally, binding the Flutter debug service to 0.0.0.0 often causes websocket handshake failures (DWDS).

Resolution: Reverted to explicit IP binding (192.168.1.24) to ensure both browser accessibility and stable debugging connections.

Technical Fix

flutter run -d chrome --web-port=8080 --web-hostname=192.168.1.24 --dart-define=API_HOST=192.168.1.24
