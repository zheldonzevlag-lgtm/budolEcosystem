v5.3.7 Release Notes

Shop Visibility & Categorization Fix

Date: 2026-02-28

Change Summary

This release addresses the issue where "Socks" products were not appearing in the Shop page. We identified two root causes: incorrect categorization during product creation and aggressive API caching.

  • ✅ Leaf-Node Selection Enforcement: Updated CategorySelector to prevent products from being accidentally assigned to parent categories (e.g., "Men's Fashion").
  • ✅ Shop Cache Busting: Implemented real-time synchronization by forcing the Products API to be dynamic and adding client-side timestamping to SWR fetches.
  • ✅ Data Repair: Manually corrected the categorization of existing "Socks" products in the database.
