Developer Manual (v5.3.7)

1. Shop Visibility Fix

Modified app/api/products/route.js to include export const dynamic = 'force-dynamic'. This prevents Next.js from serving a cached version of the product list, allowing new items to appear instantly.

2. Cache Busting in SWR

In app/(public)/shop/page.jsx, updated the SWR key to include a timestamp:

const { data, error, isLoading } = useSWR(`/api/products?t=${Date.now()}`, { ... })

3. Categorization Guardrails

Updated CategorySelector.jsx to verify if a category has children before allowing it to be "selected" as the form value. This ensures products are assigned to leaf nodes (like "Socks") rather than parents (like "Men's Fashion").

4. Database Script

Ran the following fix script to correct existing mis-categorized items:


const socksCategoryId = "cmm57agk70001gpyop1ppxgl3"; 
await prisma.product.update({
    where: { id: "cmm58aus50003gpyo72a83si9" },
    data: { categoryId: socksCategoryId, category: "Socks" }
});
