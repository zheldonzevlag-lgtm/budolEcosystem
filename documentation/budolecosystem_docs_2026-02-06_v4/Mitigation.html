Risk Mitigation Plan: OTP Login

M-OTP-01: Multi-Provider Redundancy

Risk: SMS Delivery Failure (R-OTP-01)

Mitigation: Implement multiple SMS providers (Zerix, iTextMo, Viber) with automatic failover logic. Fallback to Email delivery if all SMS channels fail.

M-OTP-02: Rate Limiting & Account Lockout

Risk: OTP Brute Force Attack (R-OTP-02)

Mitigation: Enforce a strict rate limit of 5 OTP requests per 10 minutes per IP/Identifier. Implement a 30-minute lockout after 10 failed verification attempts.

M-OTP-03: Code Expiry & One-Time Usage

Risk: SIM Swapping / Interception (R-OTP-03)

Mitigation: OTP codes are valid for only 10 minutes and are immediately deleted from the database upon successful verification or expiry.

M-OTP-04: Secure Transport

Risk: Interception

Mitigation: All OTP-related API calls must be made over HTTPS. Codes are never logged in plaintext in server logs.
