Developer Manual: OTP Login System

1. Architecture Overview

The OTP login system uses a VerificationCode model in Prisma to store temporary codes. The frontend handles the toggle between email/password and mobile/OTP.

2. Code References

  • Frontend Form: AuthForm.jsx

    Handles the login method state and conditional rendering of the OTP button/input.

  • OTP API: otp/route.js

    Generates codes, applies rate limiting, and triggers delivery via SMS/Email.

  • Login API: login/route.js

    Updated to verify 6-digit codes against the VerificationCode table.

  • Database Schema: schema.prisma

    Added VerificationCode model for OTP storage.

3. OTP Flow Logic


// 1. User enters mobile number
// 2. Client calls /api/auth/otp { identifier: '+63...', action: 'send' }
// 3. Server generates 6-digit code, saves to DB with 10min expiry
// 4. Server sends SMS via Zerix/iTextMo
// 5. User enters code in AuthForm
// 6. Client calls /api/auth/login { email: '+63...', password: 'code' }
// 7. Server detects 6-digit 'password', verifies against DB
