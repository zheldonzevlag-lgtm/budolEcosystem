Developer Manual v440

Purpose & Rationale

This version focuses on standardizing the "budolPay" branding across all platforms (Flutter and React). The primary goal is to ensure that any variation of the brand name (e.g., BUDOL_PAY, budol_pay, budolpay) is automatically normalized to budolPay with the "Pay" suffix styled in rose red.

Branding Normalization Logic

The logic follows these rules:

  • Detect patterns matching "budol" followed by an optional underscore and a suffix.
  • Extract the suffix and normalize its casing (e.g., "pay" becomes "Pay").
  • Apply specific colors based on the suffix:
    • Pay: Rose Red (0xFFF43F5E)
    • Shap: Emerald (0xFF10B981)
    • Care: Amber (0xFFF59E0B)
    • Express: Blue (0xFF3B82F6)
  • Style the "budol" prefix with a neutral Slate 400 color.

This utility is used across transaction lists, history screens, and wallet displays to ensure consistent branding.

Regex Patterns

The regex (budol)(_?)([a-zA-Z\₱]+) is used to capture:

  1. The "budol" prefix (case-insensitive).
  2. An optional underscore (to handle "budol_pay" format).
  3. The suffix (e.g., "pay", "shap", or even "₱ay" with currency symbols).
