System Admin Manual v440

Infrastructure Branding Consistency

System administrators should ensure that all environment-level branding configurations are aligned with the new normalization logic. While the normalization is handled at the application level (Flutter and React), any external systems or database entries that use "BUDOL_PAY" will now be rendered as "budolPay" in the user interface.

Note: Database migrations are not required for this update as the normalization is performed dynamically during rendering.

Monitoring & Logs

Logs may still reflect the underlying system identifiers (e.g., BUDOL_PAY). This is intentional to maintain audit trails with legacy data while providing a polished experience for the end-user.
