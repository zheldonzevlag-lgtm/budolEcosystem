Developer Manual: Authentication & Mobile Login Fixes

Technical Detail Version 120 | 2026-01-04

Purpose

To resolve reported login failures in the budolPay mobile app caused by rigid authentication error handling and incorrect database seeding.

Technical Details & Code Details

1. Mobile Service Error Handling

Modified budolPayMobile/lib/services/api_service.dart to handle both message and error fields in the JSON response from the auth server.

// Updated error parsing logic
final errorData = json.decode(response.body);
final errorMessage = errorData['message'] ?? errorData['error'] ?? 'Failed to login';
throw Exception(errorMessage);

2. budolID API Compatibility

Updated the /auth/sso/login endpoint in budolID-0.1.0/index.js to return a message field alongside the error field for better mobile app compatibility.

3. Prisma Seeding Logic

Fixed seed-tony-budolid.js by adding the missing phoneNumber field required by the Prisma client validation during the upsert operation.

Issues & Fixes

  • Issue: User reported "Auth server error (401)" instead of "Invalid credentials".
    Fix: Updated api_service.dart to parse the server's error response correctly.
  • Issue: User typo in email (budolashap.com) prevented login.
    Fix: Implemented a typo-alias user in the database via seed-tony-typo.js.

Test Results

Verified login via curl/node-scripts for both tony.stark@budolshap.com and tony.stark@budolashap.com. Both now return 200 OK with valid tokens.

Dependencies

Requires bcryptjs and @prisma/client in budolID-0.1.0.

Recommendation for Future Enhancement

Implement a "fuzzy match" login option for internal test users or common domain typos.
