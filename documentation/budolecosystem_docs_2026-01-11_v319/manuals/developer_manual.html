Developer Manual

Technical implementation of functional biometric authentication for budolPay mobile.

Purpose

The primary goal of this update is to transform the previously static biometric icon on the login screen into a fully functional authentication system. This enhances user convenience and security by allowing biometric-backed PIN auto-fill.

Explanation & Rationale

Biometric authentication (fingerprint/face) provides a seamless user experience. By leveraging hardware-backed secure storage, we can securely store a user's PIN and only release it upon successful biometric verification. This maintains the security of the PIN while removing the friction of manual entry.

Issues & Fixes

  • Issue: Biometric icon was unresponsive on the Login Screen.
    Fix: Integrated BiometricService and linked the IconButton to _handleBiometricLogin.
  • Issue: No way to enable/disable biometrics in the app.
    Fix: Converted SettingsScreen to a StatefulWidget and added a functional biometric toggle.
  • Issue: BuildContext async gap warnings in LoginScreen.
    Fix: Refactored async methods to capture service instances before await calls and check mounted status.
  • Issue: Deprecated activeColor in Switch widget.
    Fix: Replaced with activeThumbColor.

Technical Details

Dependencies

  • local_auth: ^2.1.6 - Handles the platform-specific biometric prompts.
  • flutter_secure_storage: ^9.0.0 - Encrypts and stores the user PIN in Keystore/Keychain.

BiometricService Implementation

Located at lib/services/biometric_service.dart, this service encapsulates all logic for:

  • Checking hardware availability and enrollment status.
  • Triggering the platform biometric dialog.
  • Securely storing, retrieving, and clearing the user's PIN.

Code Details

Settings Toggle Logic


// lib/screens/settings_screen.dart
Future<void> _toggleBiometrics(bool value) async {
  final biometricService = context.read<BiometricService>();
  if (value) {
    // Show dialog to confirm enabling
    _showEnableBiometricDialog();
  } else {
    await biometricService.setEnabled(false);
    await biometricService.clearStoredPin();
    setState(() => _isBiometricEnabled = false);
  }
}

Login Authentication Flow


// lib/screens/login_screen.dart
Future<void> _handleBiometricLogin() async {
  final biometricService = context.read<BiometricService>();
  final authenticated = await biometricService.authenticate();
  if (authenticated) {
    final storedPin = await biometricService.getStoredPin();
    if (storedPin != null) {
      _pinController.text = storedPin;
      _handleVerifyPin(); // Auto-login
    }
  }
}

Risk & Mitigation

  • Risk: Storage of plain-text PIN.
    Mitigation: PIN is stored using flutter_secure_storage which uses AES encryption backed by hardware security modules (HSM).
  • Risk: Biometric bypass.
    Mitigation: The system relies on the OS-level biometric prompt which is highly secure. No biometric data ever leaves the device.

Test Results

Tested manually on Android Emulator with simulated fingerprint sensor:

  • ✅ Biometric toggle correctly saves preference.
  • ✅ PIN is successfully stored upon first login after enabling.
  • ✅ Biometric icon on login screen triggers sensor and fills PIN.
  • ✅ Disabling biometrics in settings wipes the stored PIN from secure storage.

Recommendation for Future Enhancement

Implement "Biometric Timeout" - require re-authentication if the app has been in the background for more than 5 minutes, even if the session is still active.
