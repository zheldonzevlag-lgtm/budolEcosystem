System Admin Manual

Infrastructure and deployment configuration for biometric authentication.

Infrastructure Requirements

The biometric implementation relies on hardware features of the mobile device and specific OS-level configurations.

  • Android: Requires devices with Fingerprint sensor or Face recognition support. Minimum API level 23 (Android 6.0) for fingerprint, API level 28+ for biometric prompt.
  • iOS: Requires devices with Touch ID or Face ID.

Permissions Configuration

Android (AndroidManifest.xml)

The following permissions must be present in the AndroidManifest.xml file:

<uses-permission android:name="android.permission.USE_BIOMETRIC" />
<uses-permission android:name="android.permission.USE_FINGERPRINT" />

Android (MainActivity.kt)

To support the system biometric dialog, the MainActivity must inherit from FlutterFragmentActivity instead of FlutterActivity.

Deployment Procedures

  1. Dependency Audit: Verify that local_auth and flutter_secure_storage are listed in the pubspec.yaml of the build environment.
  2. Version Increment: Ensure the app version is incremented (currently 1.2.46+46).
  3. Build Command: flutter build apk --release or flutter build appbundle --release.
  4. Artifact Distribution: The generated APK should be placed in the android-installer directory with the correct version naming.

Environment Variables & Configuration

No backend environment variables are required for this update. All configuration is handled client-side via secure local storage.

Database & Storage

  • Server-side: No changes.
  • Client-side: Uses hardware-backed secure storage (Android Keystore / iOS Keychain). The data is isolated per app and is not included in cloud backups by default.
