System Admin Manual: Security Operations

1. Certificate Management

Administrators are responsible for tracking certificate expiration dates. When a new certificate is generated (via AWS ACM or Let's Encrypt), the SHA-256 hash must be communicated to the mobile development team.

CRITICAL: Do not update server-side certificates until the corresponding hash has been integrated into the mobile app pinning list and deployed to at least 95% of the active user base.

2. Monitoring Pinning Failures

Monitor API Gateway logs for an increase in SSL handshake failures. A sudden spike may indicate:

  • A certificate mismatch (incorrectly deployed certificate).
  • A widespread Man-in-the-Middle (MITM) attack.
  • Users on extremely old app versions after a mandatory certificate swap.

3. Emergency Procedures

In the event of a total app lockout due to pinning, refer to the Emergency Mitigation section in the Risk Register for instructions on using the remote configuration override.
