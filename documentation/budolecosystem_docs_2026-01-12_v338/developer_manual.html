Developer Manual: SSL/TLS Certificate Pinning

This manual covers the implementation of SSL/TLS certificate pinning in the BudolPay Flutter application, ensuring secure communication with the backend services.

1. Implementation Details

SSL pinning is implemented in the ApiService class using a custom HttpClient that verifies the server's certificate hash against a list of expected SHA-256 hashes.

1.1 Service Configuration

The ApiService now supports multiple hashes to allow for certificate rotation during cloud migration (e.g., moving to AWS or Vercel).

// Certificate Pinning Hashes (SHA-256)
static const List<String> _expectedCertHashes = [
  'SHA256_HASH_OF_PRIMARY_CERTIFICATE',
  'SHA256_HASH_OF_BACKUP_CERTIFICATE', // Backup for rotation
];

1.2 Verification Logic

The verification occurs during the badCertificateCallback or through manual verification of the certificate chain. We extract the DER encoding of the certificate and calculate its SHA-256 hash.

bool _verifyCertificatePin(X509Certificate cert) {
  final List<int> certBytes = cert.der;
  final hash = sha256.convert(certBytes).toString();
  
  if (kDebugMode) {
    print('ApiService: Server Certificate Hash: $hash');
  }

  if (_expectedCertHashes.contains('SHA256_HASH_OF_PRIMARY_CERTIFICATE')) {
    return true; // Allow in dev if not configured
  }

  return _expectedCertHashes.contains(hash);
}

2. Integration in Screens

All network requests must use the pinned client exposed by ApiService. For example, in KYCVerificationScreen, the MultipartRequest was updated to use the pinned client's send method.

final apiService = Provider.of<ApiService>(context, listen: false);
// ...
final response = await apiService.client.send(request);
View kyc_verification_screen.dart
Security Warning: Never hardcode production hashes in the source code without a rotation strategy. Use the _expectedCertHashes list to include both current and upcoming certificates.

3. Cloud Migration Roadmap (Phase 0)

When migrating to AWS (ACM) or Vercel (Managed Certificates), ensure that the public key hashes are extracted and added to the pinning list before switching the DNS traffic.

  • AWS ACM: Extract hash from the ACM certificate public key.
  • Vercel: Extract hash from the Let's Encrypt or custom certificate provided by Vercel.
