Mitigation Strategies

R-SSL-01 / R-CLD-01 Zero-Downtime Certificate Rotation

Strategy: Implement support for multiple pinned hashes. Before a certificate expires or a migration occurs, the new certificate's hash is added to the app's pinning list in a previous version. Only after the majority of users have updated is the server-side certificate swapped.

R-SSL-01 Hard-Fail Override (Emergency Only)

Strategy: In extreme cases where pinning causes a total lockout, a secure, signed remote configuration flag can be used to temporarily disable pinning or update hashes without a full app store release (e.g., via Firebase Remote Config or a dedicated secure endpoint).

R-CLD-02 Regional Compliance Enforcement

Strategy: Configure AWS/Vercel to use specific regions (e.g., AWS ap-southeast-1 in Singapore) that align with Philippine data privacy requirements and latency needs.
