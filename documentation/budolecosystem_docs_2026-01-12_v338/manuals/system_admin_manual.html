System Admin Manual: SSL Pinning & Certificate Management

Infrastructure Security Overview

With the introduction of SSL Pinning in v337, System Administrators must coordinate certificate renewals with the mobile development team to prevent service outages.

Certificate Rotation Protocol

When a backend certificate is scheduled for renewal (e.g., on AWS or Vercel), the following steps must be followed:

  1. Generate the new certificate but do not deploy it yet.
  2. Calculate the SHA-256 fingerprint of the new certificate.
  3. Provide the fingerprint to the mobile team to include as a "Backup Hash" in api_service.dart.
  4. Release a new mobile app version containing the new hash.
  5. Wait for at least 7 days (or until 90% of active users have updated).
  6. Deploy the new certificate to the backend servers.
CRITICAL: Deploying a new certificate without updating the mobile app's pinned hashes will cause all network requests to fail for users on older app versions.

Calculating Certificate Hashes

To calculate the SHA-256 hash of a certificate file (DER or PEM format):

openssl x509 -in certificate.crt -outform der | openssl dgst -sha256 -hex

Cloud Migration Readiness (AWS/Vercel) - HOT PRIORITY

Note: While Cloud Migration is deferred for the immediate release, it is marked as a HOT PRIORITY for the next infrastructure sprint. Administrators should prepare the following prerequisites:

  • ACM (AWS Certificate Manager): Planning for managed certificates for the API Gateway.
  • Vercel Managed SSL: Preparing for automatic SSL for web-facing components.
  • Route 53: Designing managed DNS with health checks.
