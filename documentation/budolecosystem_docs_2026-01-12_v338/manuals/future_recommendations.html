Future Recommendations - v337

🔥 HOT PRIORITY

1. AWS / Vercel Cloud Migration (DEFERRED)

Rationale: Transitioning from local hosting to AWS/Vercel will provide better scalability, reliability, and security. This is currently DEFERRED to focus on app-level security stabilization but remains the top priority for the next major infrastructure phase.

Status: Hot-marked for future implementation.

Strategic Roadmap:

  • Set up AWS Route 53 for DNS management.
  • Provision AWS ACM certificates for the API Gateway.
  • Integrate Vercel for the Admin Portal and Marketing site.

2. Certificate Pinning Automation

Rationale: Manually updating certificate hashes in the app is error-prone. We should explore automated ways to fetch and verify pinning configuration.

Next Steps:

  • Implement a "Dynamic Pinning" service where the app fetches valid hashes from a highly-secure secondary source upon startup.
  • Automate the CI/CD pipeline to extract hashes from new certificates and update api_service.dart automatically.

3. Biometric Session Binding

Rationale: Further enhance security by binding the authenticated session to the device's biometric identity.

Next Steps:

  • Require biometric re-authentication for high-value transactions.
  • Store the session token in the device's Secure Enclave/Keystore with biometric protection.
