Dependencies & Software Choices: Version 3.3.22

Core Dependencies

  • Next.js (API Routes): Chosen for its seamless integration of backend logic within the frontend application. API Routes allow us to create serverless endpoints (/api/system/geocode) without managing a separate backend server.
  • SWR (Stale-While-Revalidate): Used in useMapSettings.js for fetching system configuration. It provides automatic caching, revalidation, and focus tracking, ensuring the map settings are always up-to-date without manual refresh logic.

Geocoding Services

  • Nominatim (OpenStreetMap): Selected as the primary geocoding provider due to its open-source nature and cost-effectiveness (free for reasonable usage). However, it requires strict adherence to User-Agent policies, which we addressed in this update.
  • Geoapify: Chosen as the secondary/fallback provider for higher reliability and commercial-grade SLA. It requires an API key but offers higher rate limits.
  • Google Maps Platform: Added for enterprise-grade geocoding accuracy and familiarity. Requires API key and billing account. Selected for its superior coverage in complex urban environments.
  • Radar: Integrated as a developer-friendly alternative with generous free tier and robust geofencing capabilities.

Testing

  • Node-fetch: Used in our test scripts to simulate HTTP requests to the API endpoints.
