Mitigation Plan: Geocoding System

1. External API Reliability

Risk: Nominatim (OSM) service disruption.

Mitigation: Implement a fallback mechanism to Geoapify if Nominatim fails.

Action: Backend logic updated to check provider availability. Currently manual switch is required, future work includes auto-failover.

2. Rate Limiting

Risk: Exceeding Nominatim's 1 request/second limit.

Mitigation: Added a 1.5-second debounce delay on the client-side (MapPicker) to prevent rapid firing of requests during dragging.

Action: Verified in components/address/MapPicker.jsx.

3. Security

Risk: API Key abuse.

Mitigation: Server-side API key handling ensures keys are never exposed to the client.
