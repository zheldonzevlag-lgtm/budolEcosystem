Version 3.3.23 - Expanded Geocoding Providers (Google Maps, Radar)

Purpose & Rationale

This release expands the Geocoding API to support additional commercial providers: Google Maps and Radar. This ensures the system is truly provider-agnostic and can switch seamlessly between OpenStreetMap (Nominatim), Geoapify, Google Maps, and Radar based on administrative settings.

Technical Summary

  • Google Maps Integration: Added support for Google Maps Geocoding API and tile layers.
  • Radar Integration: Added support for Radar Geocoding and Maps.
  • Provider Agnosticism: The /api/system/geocode endpoint now normalizes responses from all 4 providers into a standard format for the frontend.
  • Frontend Updates: MapPicker component updated to handle multiple provider keys and tile layers dynamically.
  • Rate Limiting: Improved rate limiting logic to apply only to free/community providers (Nominatim), allowing higher throughput for commercial keys.
