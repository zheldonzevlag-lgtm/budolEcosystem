Developer Manual v435

This manual details the technical changes implemented in v435 to resolve Flutter Web rendering issues and update merchant metadata.

1. Merchant Identity Update

The "budolShap Owner" metadata was updated in the payment adapter to ensure the correct merchant name ("Jon Galvez") is displayed in QR payment modals.

File: budolpay-adapter.js


// Line 129
qrCode: {
    label: "Jon Galvez", // Changed from "budolPay"
    amount: amount,
    // ...
}

Rationale: Business requirement to brand budolShap transactions under the owner's legal identity for compliance and trust.

2. Flutter Web Layout Stabilization

The persistent black screen issue was caused by a combination of missing layout constraints in the app builder and unpositioned children in a Stack widget within the Marketing Ads screen.

2.1 App Builder Optimization

File: main.dart


// Updated builder for Web environment
builder: (context, child) {
    if (!kIsWeb) return child ?? const SizedBox.shrink();
    return Container(
        color: const Color(0xFF0F172A),
        child: Center(
            child: Container(
                width: 393,
                height: 780,
                // ... decoration ...
                child: ClipRRect(
                    borderRadius: BorderRadius.circular(32),
                    child: SizedBox.expand( // Added to ensure child fills the container
                        child: Material( // Added to provide Material context for Web widgets
                            color: const Color(0xFF0F172A),
                            child: child ?? const SplashScreen(),
                        ),
                    ),
                ),
            ),
        ),
    );
}

2.2 Marketing Ads Screen Rendering

File: marketing_ads_screen.dart


// Ensuring PageView fills the Stack
Positioned.fill(
    child: PageView.builder(
        controller: _pageController,
        // ...
    ),
),

Technical Explanation: In Flutter Web, children of a Stack without explicit positioning or constraints can sometimes fail to render (zero size). Using Positioned.fill and SizedBox.expand ensures the rendering engine allocates the necessary space.
