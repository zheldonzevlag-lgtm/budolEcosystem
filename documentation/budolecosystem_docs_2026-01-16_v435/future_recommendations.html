Future Recommendations v435

1. Centralized Merchant CMS

Moving merchant names and branding from hardcoded adapter strings to a database-driven CMS will reduce the need for code deployments for simple branding updates.

2. Advanced Web Rendering

Explore using SKIA_WEB or CanvasKit as the primary rendering engine for the web build to avoid HTML-related layout inconsistencies.

3. Unit Testing for Navigation

Implement widget tests for the SplashScreen and MarketingAdsScreen to automatically detect navigation or rendering failures in future builds.
