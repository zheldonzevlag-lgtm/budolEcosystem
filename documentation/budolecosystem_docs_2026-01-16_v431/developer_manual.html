Developer Manual - v431

Technical Overview

This release fixes a critical rendering issue in the Flutter Web environment. On web, the Navigator and PageView widgets sometimes fail to calculate their constraints correctly, resulting in zero-size rendering (black screen).

Core Fixes

1. Main Navigator Wrapper

In main.dart, the Navigator was wrapped in a Stack with Positioned.fill to ensure it occupies the entire viewport on web.

File: main.dart

// main.dart logic
return Stack(
  children: [
    Positioned.fill(
      child: Navigator(
        key: navigatorKey,
        onGenerateRoute: Routes.generateRoute,
        initialRoute: Routes.splash,
      ),
    ),
  ],
);

2. Marketing Ads PageView

The PageView.builder in marketing_ads_screen.dart was wrapped in Positioned.fill to prevent zero-height rendering in the web Stack.

File: marketing_ads_screen.dart

// marketing_ads_screen.dart logic
Positioned.fill(
  child: PageView.builder(
    controller: _pageController,
    itemCount: _slides.length,
    onPageChanged: (int page) {
      setState(() => _currentPage = page);
    },
    itemBuilder: (context, index) {
      return _buildSlide(_slides[index]);
    },
  ),
),

3. Web Splash Removal Delays

Added Future.delayed calls in splash_screen.dart to mitigate race conditions between the browser's native splash removal and Flutter's navigation.

File: splash_screen.dart

// splash_screen.dart logic
await Future.delayed(const Duration(milliseconds: 100));
removeSplashFromWeb();
await Future.delayed(const Duration(milliseconds: 50));

Dependencies

  • Flutter SDK (Web support enabled)
  • shared_preferences for hasSeenAds flag.
