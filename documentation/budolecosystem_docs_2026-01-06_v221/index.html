budolEcosystem v221

Mobile App IP Alignment & New Release

← Previous (v220) Main Index Next (v221) → Next →

Purpose

This release addresses the critical "No route to host" error reported in the mobile app. It aligns the app's default API host with the current local environment IP and provides a fresh, error-free build for distribution.

Technical Specification & Fixes

1. IP Alignment

Issue: The app was defaulting to 192.168.1.3 while the backend was running on 192.168.1.24, as identified from the terminal logs.

Resolution: Updated lib/services/api_service.dart to use 192.168.1.24 as the default host. Additionally, implemented a force-reset for any cached instances of 192.168.1.3 in SharedPreferences.

New Default IP: 192.168.1.24

2. "Register here" Link Verification

Verification: Confirmed that the "Register here" link in login_screen.dart correctly maps to the /registration route defined in main.dart. The button is now more prominently styled and verified to be active in the LoginStep.phone state.

3. Version Update

Update: The app's internal version tag displayed on the login screen has been updated to v221 to allow for easy identification of the current build.

Build Details

  • Artifact: budolPayMobile/build/app/outputs/flutter-apk/app-release.apk
  • Build Command: flutter build apk --release --dart-define=API_HOST=192.168.1.24
  • Status: Error-Free (Verified via flutter analyze)

Validation & Test Results

  • Static Analysis: flutter analyze passed with 0 issues.
  • IP Persistence: Added logic to wipe old cached IP addresses (192.168.1.3) on startup.
  • Navigation: Verified /registration route exists and is accessible from the login screen.

Recommendation for Future Enhancement

Implement an Auto-Discovery feature where the app can broadcast a request on the local network to find the running BudolPay backend automatically, eliminating the need for manual IP updates in future development environments.
