v293 System Admin Manual

Infrastructure Changes

In v293, the primary infrastructure change is the requirement for all services to bind to the 0.0.0.0 interface. This is necessary to allow external devices (like mobile phones or other PCs) to connect to the server.

Deployment Procedures

  1. Environment Setup: Ensure the host machine has a static LAN IP if possible.
  2. Network Discovery: Run the discovery script before starting the services:
    node scripts/network-util.js
  3. Port Forwarding: Ensure ports 8000 (budolID), 3000 (budolPay Admin), and 3001 (budolShap) are open in the local firewall.

Environment Variables

The following variables are dynamically managed by the system in .env files:

  • LOCAL_IP: The detected LAN IP address.
  • SSO_URL: Base URL for authentication.
  • NEXT_PUBLIC_APP_URL: The public-facing URL for Next.js applications.
Security Note: Ensure that 0.0.0.0 is only used in trusted local networks. For public production environments, use specific firewall rules or a reverse proxy.

Operational Runbook

Issue: Services not reachable from other devices

  • Check if the device is on the same Wi-Fi/Subnet.
  • Verify firewall settings on the host machine.
  • Re-run node scripts/network-util.js to ensure the IP hasn't changed.
