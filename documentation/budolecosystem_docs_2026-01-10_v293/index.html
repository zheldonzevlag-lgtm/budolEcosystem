v293: LAN Access & Network Discovery Implementation

Date: 2026-01-10 | Version: v293

COMPLETED

Purpose & Scope

This implementation ensures the entire budolEcosystem is accessible from other computers on the same Local Area Network (LAN). It automates the detection of the server's LAN IP and propagates it across all service configurations, enabling seamless cross-device SSO and service communication.

Key Changes

  • Automated LAN IP Propagation: Enhanced scripts/network-util.js to automatically update SSO_URL, REDIRECT_URI, and other network-dependent variables in all .env files with the detected LAN IP (e.g., 192.168.1.24).
  • Service Binding: Verified that all services (budolID, budolPay, budolShap) bind to 0.0.0.0, allowing them to accept connections from any network interface.
  • Dynamic SSO Redirects: Removed hardcoded 127.0.0.1 overrides in the client-side login components, allowing browsers to correctly redirect to the server's LAN IP.
  • Network Discovery: Generated a MOBILE_CONFIG.txt helper to assist in connecting physical mobile devices to the local ecosystem.

Technical Details

The network utility now performs a recursive update of environment files, replacing local loopback addresses with the physical LAN IP for external-facing variables while preserving localhost for internal database connections.

// network-util.js Logic const varsToUpdate = ['SSO_URL', 'NEXT_PUBLIC_SSO_URL', 'REDIRECT_URI', ...]; envContent = envContent.replace(regex, (match) => { return match.replace(/127\.0\.0\.1/g, detectedIP).replace(/localhost/g, detectedIP); });

Test Results

End-to-end SSO flow validation confirmed that redirects now point to the LAN IP 192.168.1.24, making the system fully accessible to other devices on the network.
