Future Recommendations & Roadmap

1. Native SSO SDK Integration

Transition from simulated SSO authentication to actual Google and Facebook SDKs for Flutter. This will provide a more secure and seamless user experience, including native login dialogs.

2. Biometric Account Linking

Allow users to link their biometric profile (FaceID/Fingerprint) directly to their SSO identity. This would enable one-touch login without even needing to tap an SSO button after the first link.

3. SSO Unlinking Feature

Implement a management screen in the User Profile where users can view linked social accounts and unlink them if desired, providing more control over their data and privacy.

4. Automated Email Verification

For SSO accounts, automatically mark the email as "Verified" in the Budol system, as Google/Facebook have already performed this verification, reducing friction for the user.

5. Risk-Based Auth (RBA)

Implement RBA to detect unusual SSO login locations or devices, triggering an additional OTP challenge even if the SSO link is valid.
