System Admin Manual: SSO & OTP Configuration

1. SSO Provider Configuration

The system now supports automated account linking for Google and Facebook SSO. Admins must ensure the following environment variables are correctly set in the auth-service:

  • GOOGLE_CLIENT_ID: Client ID from Google Cloud Console.
  • FACEBOOK_APP_ID: App ID from Facebook Developers Portal.
  • SSO_AUTO_LINK_ENABLED: Set to true to enable automated linking based on email/phone match.

2. OTP-Only Login Monitoring

OTP-only login bypasses traditional password checks but requires a verified mobile number. Admins can monitor OTP requests via the audit logs:

Log Filter: SECURITY_OTP_LOGIN_REQUEST and SECURITY_OTP_LOGIN_SUCCESS.

Excessive OTP requests from a single IP or phone number should be flagged for potential brute-force attempts.

3. Automated Linking Logs

When a user links their SSO account, the system creates an audit log entry with the type SECURITY_SSO_LINK_SUCCESS or SECURITY_SSO_REG_SUCCESS.

The linkedBy field indicates whether the match was found via EMAIL or PHONE.

4. Troubleshooting

  • SSO Linking Fails: Ensure the user's email or phone number in the SSO provider matches exactly with the record in the Budol database.
  • Prisma Unique Constraint Errors: Check if a user with the same phone number already exists under a different identity. The system handles this by generating a unique placeholder for new SSO registrations if needed.
