Mitigation Strategies

M-001: Centralized Identity (SSO)

Risk: Fragmented user data and multiple login points.

Mitigation: All apps use budolID as the single source of truth for authentication. Local sync is only for profile metadata.

M-002: Secure Hashing

Risk: Plaintext password exposure.

Mitigation: Mandatory bcryptjs hashing with a salt factor of 10 for all passwords stored in the database.

M-003: App Verification

Risk: Unauthorized applications requesting tokens.

Mitigation: Every SSO request must provide a valid apiKey that exists in the EcosystemApp table.
