Developer Manual: v1.3.59

This manual covers the implementation of Automated SSO Linking and OTP-Only Login for the Budol Ecosystem.

Backend Implementation (Auth Service)

New endpoints were added to auth-service/index.js to support quick registration and SSO linking.

  • /register/quick: Handles phone-only registration with BSP-compliant normalization.
  • /sso/link: Automates account linking if email or phone matches an existing profile.

Code Reference: auth-service/index.js

Frontend Implementation (BudolPay Mobile)

The Flutter application was updated to support the new flows.

  • Quick Registration Toggle: Added to registration_screen.dart.
  • SSO Buttons & OTP Login: Added to login_screen.dart.
  • API Service: Updated api_service.dart with linkSso and quickRegister methods.

Code References:

  • registration_screen.dart
  • login_screen.dart
  • api_service.dart

Phase 8: Dependencies & Software Choices

The following core technologies were utilized in this release:

  • Express.js: Lightweight backend framework for rapid API development.
  • Prisma ORM: Type-safe database management for complex user relations.
  • Flutter: Cross-platform mobile framework for a consistent UX.
  • Jest: Comprehensive testing suite for backend validation.
  • JWT: Secure, stateless authentication for OTP and SSO flows.

Rationale: These technologies were chosen to balance speed of development with the high security standards required by BSP and PCI DSS.
