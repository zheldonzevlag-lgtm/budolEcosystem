Developer Manual - SSO & Admin Provisioning

1. Purpose & Rationale

To ensure seamless administrative access across the ecosystem, we've implemented auto-seeding for core application keys and provisioned primary admin accounts in the budolID identity provider.

2. Technical Explanation

2.1 Auto-Seeding Logic

The budolID service now verifies core ecosystem apps (budolPay, budolShap) on every startup. If the API keys are missing, it automatically creates them to prevent 403 errors in client applications.

File Reference: budolID/index.js 
// Auto-seed core apps if missing
const coreApps = [
    { name: 'budolPay', apiKey: 'bp_key_2025', redirectUri: `http://${localIP}:3000/api/auth/callback` },
    { name: 'budolShap', apiKey: 'bs_key_2025', redirectUri: `http://${localIP}:3001/api/auth/sso/callback` }
];

for (const app of coreApps) {
    await prisma.ecosystemApp.upsert({
        where: { apiKey: app.apiKey },
        update: { redirectUri: app.redirectUri },
        create: { ... }
    });
}
