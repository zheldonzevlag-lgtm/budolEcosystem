SSO Linking & OTP Login Implementation

Purpose & Rationale

To further streamline the user experience, BudolPay has implemented automated SSO account linking. If a user logs in via Google or Facebook and their email or phone number matches an existing account, the systems automatically link them. Additionally, an "OTP-Only" login option has been added for users who prefer SMS-based authentication over passwords/PINs.

Key Benefits:

  • UX: Zero-friction transition from social login to existing profiles.
  • Security: Secure linking based on verified SSO identifiers.
  • Flexibility: Multiple login options (Password, PIN, Biometric, or OTP).

Architecture

The system uses the auth-service as the central authority for linking and OTP generation.

  1. SSO Linking: Backend endpoint /sso/link performs a lookup by email/phone and updates the user record.
  2. OTP Login: identifyMobile API accepts a mode: 'OTP' parameter to force OTP generation even for existing users.
  3. Audit Logs: Every linking event and OTP login is logged for security compliance.

Implementation Details

Backend: SSO Linking

The logic in auth-service/index.js handles both linking to existing accounts and creating new accounts if no match is found.

// SSO Linking logic snippet
let user = await prisma.user.findFirst({
    where: {
        OR: [
            email ? { email: email.toLowerCase().trim() } : null,
            phoneNumber ? { phoneNumber: phoneNumber.replace(/\D/g, '') } : null
        ].filter(Boolean)
    }
});

Frontend: Flutter SSO & OTP

The login_screen.dart was updated with SSO buttons and a "Login with OTP" option.

// SSO Login Trigger
final result = await apiService.linkSso(
  email: simulatedEmail,
  provider: provider,
  providerId: simulatedProviderId,
);

Testing Strategy

Automated tests were added to the Jest suite to verify linking and creation scenarios.

  • SSO Linking Test: Verifies that an existing user is linked by email.
  • SSO Creation Test: Verifies that a new user is created via SSO if no match exists.
  • OTP Login Test: Verifies that the mode: 'OTP' parameter correctly triggers an OTP.

Test file: quick-reg.test.js
