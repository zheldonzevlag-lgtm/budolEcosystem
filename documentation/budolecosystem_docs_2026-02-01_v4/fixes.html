Fixes Applied: v2.2.0

F-2.2.1: Prisma Unique Constraint Error in SSO

Issue: SSO registration would fail if a phone number placeholder (SSO_...) conflicted with an existing one.

Fix: Implemented dynamic phone number generation for SSO users who haven't provided a number yet, using SSO_${Date.now()}_${Math.floor(Math.random() * 1000)}.

F-2.2.2: OTP Navigation Arguments

Issue: Quick registration flow would lose context when navigating to the OTP verification step.

Fix: Updated Routes.login to accept and handle registrationData and initialStep arguments, ensuring the OTP screen knows it's verifying a registration, not a login.

F-2.2.3: KYCTier Enum Mismatch

Issue: Prisma validation error when using 'UNVERIFIED' for new quick registration users.

Fix: Standardized on 'BASIC' tier for new accounts, aligning with existing database schema.
