User Manual: Using the Ecosystem Runner

Getting Started

When you run npm start in the root of the budolEcosystem, you will now see an interactive menu.

Choice 1: LOCAL

Use this for normal development. It will configure all apps to use your local PostgreSQL database and local network settings.

Choice 2: PRODUCTION / VERCEL

Use this when testing cloud connectivity. It will switch your .env files to point to Supabase/Prisma cloud databases.

Choice 3: SKIP

Use this if you have manually modified your .env files and don't want the system to overwrite them.

What happens if I make a mistake?

Don't worry! The system has a Safety Block. If you choose "Local" but your settings still point to a production database, the system will stop and tell you exactly what is wrong. It won't let you accidentally delete or modify production data while working locally.

Workflow Changes

You no longer need to run npm run db:local separately before starting. Just run npm start and follow the prompt.
