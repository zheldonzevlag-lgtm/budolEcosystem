System Admin Manual: Environment & Safety Control

Infrastructure Impact

The new selector centralizes the control of .env file states. It relies on the presence of .env.connection.local and .env.vercel files in each sub-project directory.

Safety Barriers

Critical: The system now prevents running the ecosystem if it detects a "Production" database URL while the environment is not explicitly set to production mode.

The scripts/env-validator.js uses a keyword-based approach to identify production databases (e.g., supabase.co, db.prisma.io). If detected in a local context, the startup is aborted.

Configuration Procedures

Adding New Production Keywords

To update the safety filter, edit scripts/env-validator.js and add the new hostname to the PRODUCTION_DB_KEYWORDS array.

Rollback Procedure

If the interactive selector causes issues in automated CI/CD environments, developers can use choice 3 (SKIP) or manually run node start-ecosystem.js with a piped input (though not recommended).

Risks and Mitigations

  • Risk: User selects wrong environment by accident.
  • Mitigation: The pre-flight validation double-checks the actual DATABASE_URL against the selected mode.
  • Risk: execSync fails due to permissions.
  • Mitigation: The script catches errors and displays a clear "Failed to switch" message, allowing the user to troubleshoot.
