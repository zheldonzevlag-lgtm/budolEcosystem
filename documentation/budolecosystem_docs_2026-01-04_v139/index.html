Interactive Environment Selector

Introduction of a user-friendly CLI prompt to switch between Local and Vercel/Production environments during ecosystem startup.

← Back to Index

Summary of Changes

This update enhances the npm start experience by adding an interactive menu that allows developers to choose their target environment before the ecosystem services are launched. This eliminates the need for manual npm run db:* commands and reduces the risk of environment misconfiguration.

Developer Manual

Technical details on the readline implementation and environment switching logic.

Configuration management, safety barriers, and environment variable synchronization.

User Manual

Guide on using the new interactive prompt and understanding the environment choices.

Key Features

  • Interactive Prompt: 3-choice menu (Local, Production/Vercel, Skip).
  • Automated DB Switching: Executes the appropriate db:local or db:vercel scripts based on user selection.
  • Enhanced Pre-flight Validation: Blocks startup if a mismatch is detected between the chosen environment and the .env configuration.
  • Network-Aware Integration: Maintains compatibility with existing LAN orchestration and IP discovery systems.
