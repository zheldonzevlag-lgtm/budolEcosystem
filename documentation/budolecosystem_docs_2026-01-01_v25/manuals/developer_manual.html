Developer Manual Update (v25)

Topic: Network Portability & Auto-Sync IP

Problem: Dynamic Local IP

In a LAN environment, your machine's local IP address (e.g., 192.168.1.24) is usually assigned dynamically by the router. If you switch to a different Wi-Fi network or the router restarts, your IP might change to 192.168.1.50, breaking all hardcoded SSO redirects and environment variables.

Solution: Automated Network Sync

We've implemented a sync-network.js script in the root directory that automates the reconfiguration of the entire ecosystem.

1. IP Detection Logic

The script uses Node's os.networkInterfaces() to scan for physical network adapters (Wi-Fi, Ethernet) while ignoring virtual ones (vEthernet, VMware). It prioritizes the standard home network range (192.168.x.x).

2. Dynamic Login Pages

Frontend code in budolPay and budolShap has been updated to use window.location.hostname. This allows the login pages to work regardless of whether you access them via localhost or a specific IP address.

// Example of dynamic redirection in page.tsx
href={`http://\${typeof window !== 'undefined' ? window.location.hostname : 'localhost'}:8000/login?apiKey=...`}

3. Database & Env Sync

The script automatically performs the following:

  • Updates budolID database redirectUri entries for all registered apps.
  • Updates .env files in budolshap-0.1.0 with the new IP.
