1. Standardized Manila Timezone Offset

To comply with BSP Circular 808 and PCI DSS 10.2.2, all timestamps generated by the system must reflect Asia/Manila (UTC+8). The standardized implementation for JavaScript/Node.js is:

new Date(new Date().getTime() + (8 * 60 * 60 * 1000))

Software Rationale & Dependencies

The following software stack was selected for the Budol Ecosystem to ensure compliance, scalability, and developer efficiency:

  • Node.js & Express: Chosen for its asynchronous nature and rapid development cycle, ideal for microservices architecture.
  • Prisma ORM: Provides type-safe database access and automated migrations, ensuring data integrity across the ecosystem.
  • Socket.io: Enables low-latency, real-time communication for admin dashboards and user notifications.
  • Pusher: Utilized as a high-availability fallback for critical alerts, ensuring redundancy in notification delivery.
  • Flutter: Cross-platform mobile development framework that allows for a single codebase for Android and iOS, with native-level performance.
  • mDNS (Bonjour): Implemented for local service discovery, allowing microservices to locate each other without hardcoded IP addresses in local environments.

Real-time Admin Notifications

Implemented a standardized real-time notification flow for admin users via Socket.io and Pusher. Critical events such as new transactions and audit logs are broadcasted to the admin room/channel.

// api-gateway/index.js
router.post(['/internal/notify', '/'], express.json(), async (req, res) => {
    const { userId, event, data, isAdmin } = req.body;
    if (isAdmin) {
        io.to('admin').emit(event, data);
        if (pusher) pusher.trigger('admin', event, data);
        return res.json({ success: true });
    }
});
