Risk Mitigations - v467

M-467-01: Global Timezone Audit

Risk: Inconsistent timestamps across legacy services.

Mitigation: Continuous automated testing via test-timezone-compliance.js. All new services must pass this test before deployment.

M-467-02: Server-Side Timestamp Authority

Risk: Client-side clock manipulation.

Mitigation: Critical transaction timestamps are always generated server-side. Mobile timestamps are only used for UI and non-critical logging.

M-467-03: Standardized Offset Logic

Risk: Database storage mismatch.

Mitigation: Explicit +8 hour offset logic applied at the application layer before database insertion, ensuring the database reflects Manila time regardless of host OS settings.

M-467-04: Hybrid Notification System

Risk: Real-time notification delay or failure.

Mitigation: Implemented a dual-layer notification system using Socket.io for immediate browser updates and Pusher as a high-availability fallback for critical admin alerts.
