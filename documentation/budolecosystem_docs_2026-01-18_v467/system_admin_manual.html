Compliance Monitoring

Administrators must ensure that all system logs reflect the correct Manila Time (UTC+8). This is critical for forensic audits and regulatory compliance (BSP/PCI DSS).

Forensic Audit Tip: When reviewing logs in Prisma Studio or direct database queries, always account for the 8-hour offset if the database stores in UTC. However, application-level logs in this release are pre-offset to Manila time.

Service Health Verification

To verify compliance across services, run the following command from the ecosystem root:

node scripts/test_scripts/test-timezone-compliance.js

The output will show green checkmarks for all compliant services.
