Developer Manual - v60

QR Flow Stabilization & Webhook Enhancements

1. Webhook Metadata Integration

Modified the PayMongo adapter to inject orderId into the metadata of payment intents. This allows the webhook handler to match payments directly to orders without relying solely on provider-generated IDs.

// paymongo-adapter.js
metadata: { orderId: order.id }

2. Robust QR State Recovery

Enhanced OrderSummary.jsx to pass amount and provider as URL parameters to the /payment/qr page. This enables QRPaymentPage.jsx to recover the payment state even if localStorage is cleared.

// OrderSummary.jsx
const redirectUrl = `/payment/qr?payment_intent_id=${qrData.paymentIntentId}&orderId=${orderId}&amount=${normalizedQrCode.amount}&provider=${paymentMethod === 'BUDOL_PAY' ? 'budolpay' : 'paymongo'}`;

3. Unified DB-First Status Check

Optimized the /api/paymongo/status endpoint to check the local database first for a "paid" status. This provides a faster response and unifies status checking for both PayMongo and BudolPay.

// status/route.js
const order = await prisma.order.findFirst({ where: { paymentId: intent_id } });
if (order && (order.isPaid || order.paymentStatus === 'paid')) return NextResponse.json({ status: 'succeeded' });

4. Cart Persistence

Introduced isProcessing state in cart/page.jsx to keep the checkout UI mounted until the redirect to the payment page is fully completed, preventing race conditions during cart clearance.

5. Webhook Prioritization

Updated handlePaymentPaid in the PayMongo webhook route to prioritize metadata.orderId for definitive order matching, ensuring 100% accuracy in fulfillment.
