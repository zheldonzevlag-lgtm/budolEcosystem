System Admin Manual - v60

Infrastructure & Webhook Strategy

Metadata-First Matching

The system now prioritizes internal order IDs embedded in payment provider metadata. This provides a more resilient link between external payment events and internal order states, surviving provider-side ID changes or retries.

Unified Status Polling

The /api/paymongo/status endpoint has been optimized to check the local database first. This reduces the number of external API calls to PayMongo and allows for a unified polling strategy for multiple payment providers.

QR Flow Resilience

The QR payment page (/payment/qr) now supports state recovery via URL parameters. This ensures that even if a user's local storage is cleared mid-transaction, the payment interface can be reconstructed using the payment_intent_id, amount, and provider params.
