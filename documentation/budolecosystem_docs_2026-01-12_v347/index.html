Release Overview - v347

Purpose: Resolve "Page Not Responding" error in Flutter Web and fix UI thread blocking during app initialization.

Rationale: The Flutter web app was experiencing a complete UI hang during the splash screen, leading browsers to suggest killing the page. This was primarily caused by sequential blocking operations and incompatible imports (dart:io) that stalled the JavaScript event loop. By parallelizing the initialization process and ensuring full web compatibility, the app now starts instantly and remains responsive.

Technical Explanation

Key technical improvements include:

  • Parallel Initialization: Replaced sequential apiService.init() and branding delay with Future.wait in splash_screen.dart. This allows background tasks and UI animations to run concurrently.
  • Route Redundancy Fix: Resolved a Flutter assertion error in main.dart where both home and a / route were defined. Changed home to initialRoute for standard compliance.
  • Stack Overflow Mitigation: Replaced Scaffold with Material in the global ErrorWidget.builder to prevent recursive rendering loops (Stack Overflow) during critical startup failures.
  • Web Compatibility Fix: Removed dart:io from api_service.dart, which is known to cause issues in Flutter Web environments.
  • Main Thread Optimization: Commented out the blocking init().timeout() call in main.dart, moving the responsibility for initialization to the SplashScreen, which provides visual feedback during the process.
  • Version Increment: Updated app version to 1.2.55 to reflect the fix.
