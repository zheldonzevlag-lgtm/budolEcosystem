Developer Manual Update (v26)

Topic: Fixing SSO Redirect Loop

Problem: Redirected to Login After SSO

Users were experiencing a loop where signing in via budolID would redirect them back to the application's login page instead of the dashboard.

Root Causes

  • Secure Cookie Flag: The secure: true flag was active even in local development/LAN environments. Since these environments use http instead of https, browsers rejected the session cookie.
  • Middleware Static Path Exclusion: Middleware was intercepting internal Next.js static files (/_next/...), causing unnecessary authentication checks and potential race conditions during page load.

Solutions Implemented

1. Environment-Aware Cookies

Updated the SSO callback routes in both budolPay and budolShap to use the process.env.NODE_ENV check. This ensures that secure: true is used in production (requiring HTTPS), while secure: false is used during local development and LAN testing (allowing HTTP).

// Environment-aware cookie security
response.cookies.set('token', token, {
    httpOnly: true,
    secure: process.env.NODE_ENV === 'production', 
    sameSite: 'lax',
    path: '/',
    maxAge: 60 * 60 * 24 * 7
});

2. Middleware Hardening

Enhanced the middleware.ts in budolPay to ignore static assets and public files, preventing the "No token found" redirect for non-page requests.

const isPublicPath = pathname === '/login' || 
                     pathname.startsWith('/api/auth') || 
                     pathname.startsWith('/_next') || 
                     pathname.includes('.'); // Ignore images, icons, etc.

3. Enhanced Debugging

Added console.log statements to the middleware and callback routes to trace session creation and authentication checks in real-time.
