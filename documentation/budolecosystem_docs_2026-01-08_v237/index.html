v237: Stability & Type Safety Implementation

Resolving critical runtime exceptions and hardening cross-service communication.

2026-01-08
Version 237.0.0

Summary of Fixes

Version 237 addresses two critical issues identified during cash-in workflow testing: a missing backend environment variable and a strict type-casting failure in the Flutter mobile application.

Backend Fix

1. Transaction Service Environment Variable

Issue: The transaction-service was attempting to communicate with the wallet-service using WALLET_SERVICE_URL, which was not defined in the code, causing cash-in transactions to fail with a 500 error.

Status: RESOLVED - Variable defined with fallback logic.
// services/transaction-service/index.js
const WALLET_SERVICE_URL = process.env.WALLET_SERVICE_URL || `http://${LOCAL_IP}:8002`;
Mobile App Fix

2. Flutter Type Casting Safety

Issue: json.decode() in Dart returns a dynamic map which, in recent Flutter versions, is often _Map<dynamic, dynamic>. Attempting to cast this directly to Map<String, dynamic> caused a runtime crash.

Status: RESOLVED - Implemented safe casting using Map.from().
// budolPayMobile/lib/services/api_service.dart
final decoded = json.decode(response.body);
return Map<String, dynamic>.from(decoded);
