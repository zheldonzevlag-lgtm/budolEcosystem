Purpose

Resolved a critical logic error in FaceEmbeddingService where TFLite initialization failures were swallowed, causing the system to attempt inference with a null interpreter. This fix ensures that any failure during the initialization phase (including retries) is correctly propagated to the UI layer with descriptive error messages.

Technical Changes

  • face_embedding_service.dart
    • Modified _initModel() to explicitly throw an Exception when all initialization attempts are exhausted.
    • Enhanced generateEmbedding() with a try-catch wrapper around _initModel() to provide context-rich error messages (e.g., "AI Inference failed to initialize").
    • Added a secondary null-check with a specific error message for cases where initialization returns normally but the interpreter remains null.
    • Improved debug logging to capture the exact failure point in the initialization lifecycle.

Compliance Impact

Strengthens compliance with Cybersecurity Law and NPC requirements by ensuring that biometric system failures are handled gracefully and logged accurately, preventing undefined behavior during identity verification.
