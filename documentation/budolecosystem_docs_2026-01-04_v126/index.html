budolEcosystem Documentation v126

Date: 2026-01-04

Focus: Robust Session Lifecycle & Initial Route Management

Purpose

To eliminate persistent "User session expired" errors by moving session initialization to the app startup phase (before runApp) and preventing race conditions during login where old storage data could overwrite fresh session data.

Technical Details

  • Early Initialization: Modified main.dart to await apiService.init() before launching the UI.
  • Route Logic: BudolPayApp now dynamically selects its home screen (HomeScreen vs LoginScreen) based on the pre-loaded authentication state.
  • Login Protection: ApiService._loadSession now explicitly checks if a session is already active before loading from SharedPreferences, preventing state corruption.
  • Build: Released as budolPay 1.0.8.apk.

Test Results

Automated tests in api_service_test.dart confirm that the initialization gate correctly waits for session data and that the isAuthenticated flag accurately reflects the storage state on boot.
