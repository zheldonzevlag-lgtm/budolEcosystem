Executive Summary

Version 417 fixes a race condition where the home screen "Welcome back" message would show "User" instead of the logged-in user's name upon app restart. This was caused by the splash screen navigating to the home screen before the background user profile fetch was completed. The implementation synchronizes the splash screen with the user profile verification process.

Key Improvements

  • Session Synchronization: Added waitForFullInit() to ApiService to allow waiting for background tasks.
  • Splash Screen Guard: SplashScreen now awaits profile verification before navigating if the user is authenticated.
  • Data Merging: Profile fetch now merges new data with existing session data to ensure field persistence.
  • Version increment to 1.3.1 for profile persistence fix.
