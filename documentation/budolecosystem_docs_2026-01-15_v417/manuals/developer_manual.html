Developer Manual - Version 417

Technical Explanation: Splash Synchronization

The core fix involves ensuring that the ApiService background initialization (which fetches the fresh user profile) is awaited by the SplashScreen before it proceeds to the HomeScreen.

ApiService Changes

A new waitForFullInit() method was added to allow awaiting the _backgroundInitFuture.

api_service.dart 
Future waitForFullInit() async {
  await _ensureInitialized();
  if (_backgroundInitFuture != null) {
    await _backgroundInitFuture;
  }
}

SplashScreen Changes

The navigation logic now checks if the user is authenticated and waits for the full initialization if so.

splash_screen.dart 
if (apiService.isAuthenticated) {
  debugPrint('SplashScreen: Authenticated, waiting for full initialization...');
  await apiService.waitForFullInit().timeout(const Duration(seconds: 5));
}
