Mitigation Plan: v1.3.5

M-013: Temporary Email Filtering

Risk: Legit emails might be hidden if they contain the quick registration domain string.

Mitigation: The filter is strictly bound to the specific domain @quick.budolpay.com, which is exclusively used for internal auto-generated identities. Regular user emails on standard providers (Gmail, Yahoo, etc.) will not be affected.

M-014: Form Data Clearing

Risk: Users might lose progress when switching methods.

Mitigation: This is intentional design to ensure data integrity. Users switching between Phone and Email usually intend to use a completely different set of credentials. Standardized placeholders guide them through the fresh entry.

M-015: Auto-complete Disablement

Risk: Increased manual typing.

Mitigation: While autoComplete="off" is applied to the form, modern browsers may still provide suggestions for certain fields. The primary goal is to prevent the browser from automatically populating old registration data into a new signup flow, which was a reported point of confusion.
