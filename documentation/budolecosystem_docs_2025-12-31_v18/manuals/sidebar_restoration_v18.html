Documentation Update v18

Sidebar Restoration & Layout Redesign

Context

The admin sidebar was previously removed to resolve runtime errors caused by undefined Tailwind color classes (budolshap-primary). This update restores the sidebar using native Tailwind colors and a responsive, conditional layout system.

Technical Implementation

  • Componentization: Moved the sidebar logic to a dedicated Sidebar component and created a MainContent wrapper.
  • Conditional Rendering: Implemented a usePathname check to hide the sidebar and remove margins on the /login page.
  • Safe Styling: Replaced all custom color references with green-500 (primary) and slate-900 (sidebar background).
  • Fixed Navigation: The sidebar is now fixed to ensure it remains visible while scrolling the main content.

Files Modified

  • apps/admin/app/layout.tsx: Integrated new layout components.
  • apps/admin/components/Sidebar.tsx: Core navigation component.
  • apps/admin/components/MainContent.tsx: Conditional margin wrapper.

Usage

The sidebar will automatically appear for all routes except /login. It includes navigation for:

  • Dashboard
  • User Management
  • Transactions
  • Disputes
  • Accounting
  • Security
  • Settings
