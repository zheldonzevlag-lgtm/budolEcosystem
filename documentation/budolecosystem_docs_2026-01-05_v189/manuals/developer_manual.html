Developer Manual - v189

Finalization of Tiered KYC

This release finalizes the Tiered KYC implementation by applying the necessary database migrations and regenerating the Prisma client to support the new KYCTier field.

Technical Details

1. Prisma Migration

Applied the add_kyc_tier migration to the PostgreSQL database. This migration adds the KYCTier enum and the kycTier field to the User table.

enum KYCTier {
  BASIC
  SEMI_VERIFIED
  FULLY_VERIFIED
}

model User {
  // ...
  kycTier KYCTier @default(BASIC)
}

2. Prisma Client Regeneration

Regenerated the Prisma client to include the new types and enums. This allows for type-safe interactions with the kycTier field in the application code.

3. Admin API Enhancements

Updated [route.ts](file:///d:/IT Projects/budolEcosystem/budolpay-0.1.0/apps/admin/app/api/users/route.ts) to correctly handle the kycTier update during KYC status changes, ensuring that the user's tier progresses automatically based on admin approval.

Dependencies

  • Prisma CLI (v6.2.1)
  • PostgreSQL Database
