System Admin Manual - v189

Infrastructure Updates

This update finalizes the deployment of the Tiered KYC system. It requires a database schema update and client regeneration.

Important: All Node.js processes must be stopped before running the migration and generation commands to avoid file locking errors.

Deployment Steps

  1. Stop all running services (Admin Panel, Verification Service, API Gateway).
  2. Navigate to packages/database.
  3. Run: npx prisma migrate deploy to apply the new schema.
  4. Run: npx prisma generate to update the client.
  5. Restart all services.

Verification

  • Check the User table in the database for the new kycTier column.
  • Verify that the Admin Panel correctly displays user tiers in the management table.
  • Test a KYC approval to ensure the tier automatically updates to FULLY_VERIFIED.
