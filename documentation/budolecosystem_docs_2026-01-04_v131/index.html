budolEcosystem Documentation v131

Date: 2026-01-04

Focus: Wallet Robustness & Status Propagation Fixes

Purpose

To ensure consistent balance synchronization between the backend and mobile app, resolve remaining "wallet not found" errors, and guarantee payment status propagation from the mobile app to the budolShap desktop UI.

Issues Addressed

  • 0.00 Balance Display: The mobile app displayed 0.00 even after successful payment because it failed to parse the balance data type (Decimal/String vs double).
  • Remaining "Wallet Not Found": The /update-balance endpoint in Wallet Service still returned 404 for missing wallets, causing errors during certain transaction types.
  • Status Propagation Gap: Payment success was visible on the phone but not on the desktop UI because the gateway service was skipping webhook triggers for already-completed transactions.

Technical Implementation

  • Mobile App (Balance Parsing): Updated home_screen.dart with robust type checking for num and String balance values to prevent parsing failures.
  • Wallet Service (Failsafe Update): Added auto-creation logic to the /update-balance endpoint in wallet-service/index.js. It now ensures both User and Wallet records exist before attempting an update.
  • Gateway Service (Webhook Robustness): Modified payment-gateway-service/index.js to re-trigger downstream webhooks (e.g., to budolShap) even if the transaction is already marked as COMPLETED. This ensures the desktop UI receives the status update even if the internal webhook was called first.
  • Data Consistency: Standardized balance return as a Number in all Wallet Service responses to avoid cross-language parsing issues.

Risk & Mitigation

  • Risk: Potential for duplicate balance deductions if webhooks are retried aggressively.
  • Mitigation: The Gateway Service now performs an idempotency check on the transaction status. If COMPLETED, it only re-sends the notification and skips all ledger/balance modification logic.

Test Results

  • Balance Sync: Verified via debug logs that the mobile app correctly parses both integer and decimal balance strings.
  • Wallet Creation: Confirmed that /update-balance no longer returns 404 for missing users, instead creating the necessary records on-the-fly.
  • Status Propagation: Simulated a cross-service payment flow and verified that the budolShap webhook is triggered even when the transaction was previously finalized by the wallet service.

Recommendations

See the Consolidated Recommendations for future architectural improvements.
