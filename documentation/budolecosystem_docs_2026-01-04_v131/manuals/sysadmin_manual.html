SysAdmin Manual Updates - v131

Service Endpoints & Webhooks

Ensure the following internal network routes are accessible for status propagation:

Gateway to budolShap Webhook: http://localhost:3001/api/webhooks/budolpay

The gateway service now attempts to re-send success notifications to this endpoint if a merchant payment is detected as already completed.

Database Integrity

The Wallet Service now handles its own user synchronization. No manual database intervention is required when new users are created in the system; the first balance check or update will automatically provision the necessary records in the wallet database.

Monitoring

Watch for the following log patterns in the payment-gateway-service to verify status propagation:

  • [Gateway] Transaction [ID] already COMPLETED. Checking if webhook needs to be triggered anyway.
  • [Gateway] Re-triggering webhook to budolShap for Order: [ORD-ID]
  • [Gateway] Webhook re-sent successfully to budolShap
