Developer Manual Updates - v131

Balance Data Type Handling

The mobile app now uses a robust parsing logic in home_screen.dart to handle balance data returned from the API. This addresses issues where the balance might be returned as a String (from Decimal fields) or a num.

// home_screen.dart
var balanceData = result['balance'];
double balanceValue = 0.0;

if (balanceData is num) {
  balanceValue = balanceData.toDouble();
} else if (balanceData is String) {
  balanceValue = double.tryParse(balanceData) ?? 0.0;
}

Wallet Service Failsafe Logic

The /update-balance endpoint in wallet-service/index.js has been enhanced to automatically create User and Wallet records if they are missing during a balance update request. This prevents 404 errors during cross-service operations where a user might exist in one service but not yet in the wallet service.

Webhook Re-triggering Logic

To support status propagation in complex multi-service flows, the payment-gateway-service now re-triggers downstream webhooks even if a transaction is already COMPLETED. This is critical when the wallet-service finalizes a transaction internally before the gateway's own webhook is processed.

// payment-gateway-service/index.js
if (existingTx.status === 'COMPLETED') {
  if (existingTx.type === 'MERCHANT_PAYMENT') {
    const metadata = JSON.parse(existingTx.metadata || '{}');
    if (metadata.app === 'budolShap') {
      // Re-trigger webhook to budolShap
      await axios.post('http://localhost:3001/api/webhooks/budolpay', { ... });
    }
  }
}

Status Translation

Verified that budolShap API (route.js) correctly translates the COMPLETED status from BudolPay into succeeded for frontend compatibility.
