Developer Manual: End-to-End Verification & Database Seeding

Purpose

This update verifies the complete SSO login flow and ensures all ecosystem services have synchronized databases with test data for local development.

Implementation Details

1. Database Seeding

Created and updated seed scripts across services to ensure a functional local environment:

  • budolID: Added test@example.com with password password123 to the seed script.
  • budolAccounting: Created a new prisma/seed.js to initialize the Chart of Accounts (Cash, Bank, Wallets, Revenue, Expenses).
  • budolShap: Fixed a schema validation error by adding the missing phoneNumber field to the admin user creation logic.

2. Prisma Client Synchronization

Resolved a PrismaClientInitializationError where the client was looking for a non-existent isFaceVerified column. This was fixed by running npx prisma generate after ensuring no active node processes were holding the DLL files.

Test Results

Performed a successful simulation of the budolPay mobile app login flow:

  • Endpoint: POST /auth/sso/login
  • Status: 200 OK
  • Payload: Correctly returned a signed JWT, user details, and the application redirect URI.

Future Recommendations

Implement a centralized seeding orchestrator in the root start-ecosystem.js to allow one-click database reset and initialization for the entire ecosystem.
