System Admin Manual: Service Orchestration & Port Management

Infrastructure Changes

Standardized the startup procedure for all services to prevent port conflicts and environment pollution.

Port Management: Port 8000 is reserved for budolID. If an EADDRINUSE error occurs, ensure no orphaned node.exe processes are running.

Deployment Procedures

To initialize a fresh local environment:

  1. Run npm run db:local in the root.
  2. Run npm start and select choice 1 (LOCAL).
  3. For each service, run npx prisma db push and npm run prisma:seed.

Risk & Mitigation

Risk: Database schema drift between local and production.
Mitigation: Always run npx prisma db pull before generating the client to ensure the schema matches the active environment.
