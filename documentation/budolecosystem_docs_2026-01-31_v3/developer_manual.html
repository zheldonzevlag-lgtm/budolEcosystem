Developer Manual - Security & Session Management

1. Web Auth Form Fix

Resolved the ReferenceError: Cannot access 'formData' before initialization in the React registration form. This was caused by hook ordering where useEffect hooks were referencing formData before its state declaration.

File: budolshap-0.1.0/components/auth/AuthForm.jsx

View AuthForm.jsx 
// Corrected Order:
const [formData, setFormData] = useState({ ... });

useEffect(() => {
    // Debounced email check now safely accesses formData
    if (!formData.email) return;
    // ... logic
}, [formData.email]);

2. Session Timeout Implementation (Mobile)

The SessionService manages inactivity timers. Upon timeout, it triggers a lock screen. This release ensures that manual "Log Out" from this state correctly purges all authentication tokens.

File: budolPayMobile/lib/services/session_service.dart

View SessionService.dart 
void forceLogout() {
  _isLocked = false;
  apiService.logout(); // Triggers global state clearing
  notifyListeners();
}

3. Global Auth Navigation

The main.dart file acts as the orchestrator for authentication state changes. When apiService.logout() is called, the token becomes null, triggering an automatic redirect to the login screen.

File: budolPayMobile/lib/main.dart

View main.dart 
void _handleAuthChange() {
  if (apiService.token == null) {
    navigatorKey.currentState?.pushNamedAndRemoveUntil(
      Routes.login,
      (route) => false,
    );
  }
}
