Mitigation Plan

M-545-1: Hard Inactivity Timeout

Risk: Session remains active on abandoned devices.

Mitigation: Implemented a 3-minute hard inactivity timer in SessionService. Upon expiration, the app automatically locks and requires biometric re-authentication or manual logout.

M-545-2: Background App Security

Risk: Lock screen bypass via app backgrounding.

Mitigation: App lifecycle listeners ensure that the lock state is preserved even if the app is moved to the background and brought back. The SessionOverlay is a global widget that persists across route changes.

M-545-3: State Initialization Guard

Risk: Web form crashes during registration.

Mitigation: Refactored AuthForm.jsx to declare formData state at the top of the component, before any useEffect or secondary hooks that depend on it. This ensures the variable is always defined when accessed.
