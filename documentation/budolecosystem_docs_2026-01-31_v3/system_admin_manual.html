System Admin Manual - Session Configuration

Session Timeout Policy

The system is configured with a 3-minute inactivity timeout for mobile users. This is a hard-coded security policy to comply with BSP and NPC standards for financial applications.

Note: Admins do not need to manually configure timeout durations as they are standardized across the ecosystem for security consistency.

User Session Termination

If a user reports a lost device or compromised account, the standard procedure is to trigger a remote logout by invalidating the user's tokens in the backend database. The app's new session management will automatically detect this token invalidation and redirect the user to the login screen upon the next interaction.
