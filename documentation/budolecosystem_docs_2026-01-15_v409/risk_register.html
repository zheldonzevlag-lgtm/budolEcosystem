Risk Register: v409 Release

Identification and assessment of risks associated with real-time audit logs and provider-agnostic infrastructure.

Identified Risks

Mobile App Version Disparity (High)

Description: Users on versions prior to v1.2.93 will not trigger the backend logout audit log, creating a compliance gap for forensic tracking.

Impact: Incomplete audit trails for security events, potentially violating BSP Circular 808 and Cybersecurity Law requirements.

Real-time Sync Latency (High)

Description: In poor network conditions, real-time events (Socket.io/Pusher) may fail to reach the client, causing data inconsistency between UI and database.

Impact: User might see stale balance or missing transactions, leading to duplicate transaction attempts.

Provider Lock-in / Failure (Medium)

Description: Reliance on a single real-time provider (e.g., Pusher) could lead to system-wide outages if that provider's API goes down.

Impact: Loss of real-time functionality across all mobile and admin interfaces.

Audit Log Volume (Medium)

Description: High-frequency logging (including all logouts and every transfer) may lead to rapid database growth and performance degradation.

Impact: Slower query performance for audit searches and increased storage costs.

Forensic Integrity (Low)

Description: Risk of audit logs being modified or deleted by compromised admin accounts.

Impact: Loss of forensic evidence for regulatory compliance (BSP/BIR).
