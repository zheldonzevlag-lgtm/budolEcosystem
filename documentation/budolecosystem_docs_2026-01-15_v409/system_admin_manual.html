System Admin Manual: Real-time & Compliance

Guide for configuring and monitoring the v409 real-time forensic audit infrastructure.

1. Real-time Provider Configuration

Admin can now dynamically switch the real-time delivery method via the SystemSettings table in the database. This affects all clients globally.

Setting Key: REALTIME_METHOD
Available Values: SOCKETIO, PUSHER, SWR
Default: SWR
Provider Configuration Keys:
  • PUSHER_KEY / PUSHER_CLUSTER: Required for Pusher delivery.
  • SOCKETIO_URL: Endpoint for Socket.io connections (usually the API Gateway).
  • SWR_REFRESH_INTERVAL: Polling frequency in milliseconds (default 5000ms).

2. Forensic Audit Monitoring

Access the Security & Audit page in the Admin Dashboard to view real-time logs. The dashboard automatically joins the admin real-time room to receive all system-wide events.

Admin Dashboard URL: /security
Features: Live event stream, IP tracking, User-Agent analysis, compliance tagging.

3. Security Policy & Logout Audits

v409 enforces strict logout auditing. Every time a user logs out (or their session is terminated), a SECURITY_LOGOUT event is captured with the user's IP and device info.

This is critical for investigating session hijacking or unauthorized access attempts.
