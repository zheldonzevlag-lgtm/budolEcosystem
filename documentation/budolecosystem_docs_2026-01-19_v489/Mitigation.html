M-489-01: Status Inclusive Queries

Risk: Legacy 'PAID' orders being excluded.

Mitigation: All critical queries and filters have been updated to use IN ['PAID', 'PROCESSING'] instead of a single string match.

M-489-02: Targeted Realtime Broadcasting

Risk: Event overload.

Mitigation: Realtime events are strictly scoped to user-{userId} and store-{storeId} channels, preventing global broadcast overhead.
