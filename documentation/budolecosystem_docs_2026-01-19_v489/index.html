Budol Ecosystem Release v489

Order Status Consistency & Tab Visibility Optimization

Index Developer Manual SysAdmin Manual User Manual Test Results Previous (v488)

Version: 489

Release Date: 2026-01-19

Focus: Resolving "Paid Orders Missing from Tabs" issue through status standardization and realtime event synchronization.

Purpose & Rationale

The primary objective of this release is to ensure that orders successfully paid via budolPay or PayMongo are immediately and correctly reflected in the "To Ship" tab for both buyers and sellers. Previously, inconsistent status updates (mixing 'PAID' and 'PROCESSING') and lack of buyer-side realtime triggers caused synchronization issues.

Task List

  • Completed Standardize Webhook Status: Updated budolPay and PayMongo webhooks to set status to 'PROCESSING' instead of 'PAID'.
  • Completed Escrow Service Sync: Modified escrow.js to use 'PROCESSING' status when crediting pending balance.
  • Completed Frontend Filter Update: Updated orders/page.jsx to include both 'PAID' and 'PROCESSING' in the 'To Ship' tab.
  • Completed OrderItem UI Labels: Updated OrderItem.jsx to distinguish between 'Pending Payment' (ORDER_PLACED) and 'Processing' (PAID/PROCESSING).
  • Completed Automated Transitions: Updated ordersService.js to auto-transition ORDER_PLACED orders to PROCESSING upon payment.
  • Completed Realtime Sync: Added user-specific realtime events to webhooks for immediate UI state transitions.
  • Completed Validation: Created and ran test-order-visibility-v489.mjs to verify order visibility across all tabs.

Key Changes

  • Webhook Standardization: All successful payments now transition orders to PROCESSING status, aligning with the universal status mapping.
  • Realtime Triggers: Added user-{userId} channel notifications to webhooks, ensuring the buyer's "My Orders" page refreshes automatically upon payment confirmation.
  • Robust Filtering: Frontend tabs now explicitly handle both PAID and PROCESSING statuses for the "To Ship" category.

Compliance

This update maintains compliance with BSP Circular 808 by ensuring the transaction lifecycle is accurately reflected in the audit trail (escrow and ledger entries).
