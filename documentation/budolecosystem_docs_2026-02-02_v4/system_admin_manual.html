System Admin Manual - Profile Management & Auditing

Configuration Overview

Admins can now manage employee profiles with guaranteed persistence. This module handles first names, last names, phone numbers, roles, and departments.

Key Update: The "Contact & Dept" column is now visible in the main Employee Management table, allowing for quick verification of profile data.

Audit Log Inspection

Every profile change is logged for compliance (PCI DSS). Admins should monitor the AuditLog table for the following action:

  • Action: UPDATE_USER_PROFILE
  • Module: ADMIN_MANAGEMENT
  • Details: Contains JSON blob of updatedFields and previousValues.

Troubleshooting

Database Locks: During system updates (e.g., running npx prisma generate), ensure no active Node processes are locking the Prisma query engine. Use taskkill /f /im node.exe if EPERM errors occur.

If profile values do not update in the UI immediately, ensure the fetchData() trigger is properly firing on modal success.
