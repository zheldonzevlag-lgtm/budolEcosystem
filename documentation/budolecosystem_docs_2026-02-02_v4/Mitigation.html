Mitigation Plan

M-PROF-001: Atomic Conditional Updates

Risk: Partial data loss during profile updates.

Strategy: Use explicit null/undefined checks in Prisma update payloads to ensure only targeted fields are modified. This prevents overwriting valid data with empty strings or nulls from the frontend.

M-PROF-002: Real-time UI Refresh

Risk: User confusion due to stale data display.

Strategy: Implemented onSuccess callback in the EditUserModal which triggers a fresh fetchData() call on the parent page, ensuring the UI reflects the DB state immediately after a successful save.

M-PROF-003: Mandatory Auditing

Risk: Non-compliance with PCI DSS 8.2.

Strategy: Every successful update operation now includes a secondary database write to the AuditLog table, capturing the who, what, and when of the change.
