Developer Manual - Admin Profile Persistence

Purpose & Rationale

This release addresses a critical bug where admin profile updates (firstName, phoneNumber, department) were not persisting correctly in the database or reflecting in the UI. The fix ensures data integrity and real-time synchronization.

Technical Explanation

The core issue was in the API route's update logic, which occasionally failed to map optional fields correctly. We implemented conditional updates using Prisma to ensure that only provided fields are modified, while preserving existing values for undefined fields.

Backend: API Logic

File: apps/admin/app/api/employees/route.ts

const updatedUser = await prisma.user.update({
  where: { id: userId },
  data: { 
    firstName: firstName !== undefined ? firstName : oldUser.firstName, 
    lastName: lastName !== undefined ? lastName : oldUser.lastName, 
    email: email !== undefined ? email : oldUser.email, 
    phoneNumber: phoneNumber !== undefined ? phoneNumber : oldUser.phoneNumber, 
    role: role !== undefined ? role : oldUser.role,
    department: department !== undefined ? department : oldUser.department
  }
});

Frontend: UI Sync

File: apps/admin/app/employees/page.tsx

Added a dedicated "Contact & Dept" column to display phoneNumber and department in the employee table, ensuring visibility of these previously hidden/non-updating fields.
