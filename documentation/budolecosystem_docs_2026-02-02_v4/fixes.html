Fixes & Resolutions

FIX-PROF-001: Conditional Data Persistence

Modified the API route to use defensive mapping. This ensures that if a field (like phoneNumber) is missing from the payload, it doesn't overwrite the existing value with null.

// Before: Potential data loss if fields missing
// After: Conditional assignment
data: { 
  firstName: firstName !== undefined ? firstName : oldUser.firstName,
  phoneNumber: phoneNumber !== undefined ? phoneNumber : oldUser.phoneNumber,
  department: department !== undefined ? department : oldUser.department
}
FIX-PROF-002: Real-time Table Visibility

Added phoneNumber and department display logic to the main employees table to provide immediate feedback after an update.

FIX-PROF-003: Modal State Reset

Fixed a bug where EditUserModal would retain data from the previous user. Added a state reset in the onClose handler.
