System Admin Manual: KYC Approver Guidelines

This manual explains how the new OCR visibility restrictions impact KYC administrators and system auditors.

Admin View Access

KYC Administrators (users with accountType: 'ADMIN' or role: 'ADMIN') still have full access to view AI-extracted data in the KYC Wizard and via the Admin Dashboard.

  • Full Transparency: Admins see the raw text lines, detected names, and ID numbers.
  • Manual Override: If the AI incorrectly extracts data, admins can see exactly what was misread to assist in manual verification.

Security Hardening for Users

Regular users (Buyers/Sellers) can no longer see their own extracted data in raw form. This is a security feature to prevent sensitive data from being cached or exposed in the frontend for unauthorized users.

Compliance Monitoring

All KYC submissions are still logged in the AuditLog table with full metadata, including the IP address and user agent of the submitter. The raw extraction results are stored securely in the kycDetails field of the User table, but are now masked when returned via the standard user API.

Troubleshooting

If an admin reports that they cannot see the AI results, verify that their account has the correct accountType or isAdmin flag set in the database.
