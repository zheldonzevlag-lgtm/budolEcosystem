Future Recommendations - v1.4.0

1. Granular RBAC

Implement more granular roles such as KYC_OFFICER and COMPLIANCE_AUDITOR instead of a generic ADMIN flag to further restrict access based on specific business functions.

2. Dynamic Masking Configuration

Move the masking logic from hardcoded fields to a database-driven configuration. This would allow compliance officers to update which fields are masked without requiring code deployments.

3. End-to-End Encryption (E2EE)

Consider encrypting the kycDetails JSON blob in the database using a key stored in a secure HSM (Hardware Security Module) or KMS (Key Management Service) to provide at-rest protection for sensitive PII.

4. Automated Compliance Audits

Develop a tool that periodically scans the User table to ensure that retentionExpiry dates are being honored and that expired KYC data is automatically purged or anonymized.
