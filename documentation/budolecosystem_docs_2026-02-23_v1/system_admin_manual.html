System Admin Manual: Map Provider Configuration

1. Supported Providers

The system supports the following map providers. The active provider is determined by the System Settings.

  • OpenStreetMap (Nominatim): Default, free, no API key required (but rate limited).
  • Geoapify: Requires API Key. Good coverage for PH.
  • Google Maps: Requires API Key. Best coverage, but costly.
  • Radar: Requires API Key. Good alternative.

2. Configuration

To configure the map provider, navigate to the System Settings page in the Admin Dashboard.

2.1 API Keys

Ensure valid API keys are entered for the selected provider.

  • Geoapify: Enter in Geoapify API Key field.
  • Google Maps: Enter in Google Maps API Key field. Enable Geocoding API and Maps JavaScript API.
  • Radar: Enter in Radar API Key field.
Important: If changing providers, ensure to clear any cached configurations if applicable. The geocoding service switches immediately based on the saved setting.

3. Troubleshooting

If map addresses are not syncing:

  1. Check if the API key is valid and has quota remaining.
  2. Verify the internet connection of the server (for outgoing API requests).
  3. Check the browser console for any client-side errors in MapPicker or AddressAutocomplete.
  4. Check the server logs for errors in /api/system/geocode.
