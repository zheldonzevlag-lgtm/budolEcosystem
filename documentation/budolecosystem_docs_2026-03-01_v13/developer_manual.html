Developer Manual: AWS Integration

S3 Storage Integration

We use the @aws-sdk/client-s3 package to interact with AWS S3. The lib/cloudinary.js has been deprecated in favor of a centralized S3 utility.

Configuration

const s3Client = new S3Client({
    region: process.env.AWS_REGION,
    credentials: {
        accessKeyId: process.env.AWS_ACCESS_KEY_ID,
        secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY
    }
});

Lambda Background Removal

The background removal process is handled by a Python Lambda function. Developers should not run background removal locally to save resources.

Lambda Interface

Trigger: S3:ObjectCreated:* on uploads/raw/

Destination: uploads/processed/

Real-time Layer (Socket.io)

The WebSocket server acts as a real-time event broadcaster. Services trigger events via HTTP POST to the /trigger endpoint, which then emits the event to clients subscribed to specific channels.

Trigger API

To trigger an event from a backend service, send a JSON POST request to the internal microservice. Always use the triggerRealtimeEvent helper to ensure compatibility with both Local and AWS environments. 

POST http://ws.budol.internal:4000/trigger
Content-Type: application/json

{
    "channel": "order-123",
    "event": "status-updated",
    "data": { "status": "DELIVERED" }
}

Environment Variables:

  • INTERNAL_WS_URL: Set to http://ws.budol.internal:4000 in production to bypass external proxies for backend-to-backend calls.
  • NEXT_PUBLIC_SOCKET_URL: Set to https://budolws.duckdns.org for client-side connections.

Client Subscription

Clients should connect to https://budolws.duckdns.org and use the subscribe event to join a room.

const socket = io('https://budolws.duckdns.org');
socket.emit('subscribe', 'order-123');

socket.on('status-updated', (data) => {
    console.log('Order update:', data);
});

Internal Service Integration

To ensure smooth communication between services (e.g., BudolPay -> BudolID), use internal DNS aliases.

Service Connectivity Matrix

Target Service Internal DNS Port
Database (RDS) db.budol.internal 5432
Redis Cache redis.budol.internal 6379
Websocket (WS) ws.budol.internal 4000

Prisma & DB Deployment

Deploy schema changes without downtime using prisma migrate deploy. This command applies pending migrations without reset.

# Inside application container
npx prisma migrate deploy

Warning: Always ensure the DATABASE_URL includes the schema name (e.g., ?schema=budolpay) to maintain logical isolation on the shared RDS instance.

Local Development

Even with cloud services, developers should continue using .env.local for credentials. Never commit .env files containing AWS Secret Keys.

Local SSL (Optional)

If you need to test HTTPS features locally (e.g., secure cookies), we recommend using mkcert: 

# Install mkcert (Windows/Choco)
choco install mkcert
mkcert -install

# Generate certs for localhost
mkcert localhost 127.0.0.1 ::1

Update env-manager.cjs or your .env.local to use https:// if active.
