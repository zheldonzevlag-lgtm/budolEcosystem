Mitigation Plan: AWS Cloud Migration

M001: Billing Control (Mitigates R001)

Action: Configure AWS Budgets and set a limit of $0.05. Enable SNS notifications to alert admin via email if actual or forecasted costs exceed the threshold. Weekly manual audits of the Billing Dashboard.

M002: S3 Access Control (Mitigates R002)

Action: Use S3 Block Public Access at the account level. Implement Origin Access Control (OAC) with CloudFront to serve private content. Audit bucket policies using IAM Access Analyzer.

M003: Lambda Reliability (Mitigates R003)

Action: Implement SQS-based retry logic for image processing failed events. Increase Lambda memory allocation to 1024MB to ensure enough CPU for image processing (Rembg).

M005: DNS High-Availability (Mitigates R005)

Action: Register fallback subdomains on a secondary free DNS provider (e.g., No-IP). Maintain a manual "Production Host" emergency list in the System Admin Manual for immediate cutover.

M006: Redundant IP Sync (Mitigates R006)

Action: Deploy the sync script to multiple EC2 nodes (Websocket and Redis). Each node will independently heartbeat to DuckDNS, ensuring synchronization even if the primary node fails.

M007: Hybrid DNS Strategy (Mitigates R007)

Action: Proactively distribute subdomains across multiple free providers (DuckDNS and FreeDNS). This bypasses individual provider limits and adds institutional redundancy to the ecosystem access layer.
