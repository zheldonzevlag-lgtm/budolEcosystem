System Architecture Deep Dive

Comprehensive analysis of microservices, data models, and service choreography.

Executive Summary

The BudolEcosystem is a high-performance, microservices-driven financial ecosystem designed for secure QR payments and centralized accounting. It utilizes a hybrid deployment model:

  • Static Frontends: AWS Amplify (Managed SSL, CI/CD)
  • Containerized Microservices: AWS ECS Fargate (Scalable, Secure VPC)
  • Real-time Gateway: Dedicated WebSocket Server (wss:// via Nginx)

2. Data Architecture

The BudolEcosystem utilizes a distributed data strategy with focused schemas for each core domain.

2.1. Identity & SSO (budolID)

  • User: Essential profile, credentials, and user roles.
  • EcosystemApp: Authorized applications (e.g., BudolShap, BudolAdmin).
  • Session: Cross-application session state.

2.2. Financial Core (budolpay)

  • User: Extended profile with KYC, biometrics, and security settings.
  • Wallet: Real-time balance (Decimal 18,2) and currency (PHP).
  • Transaction: Immutable records of Cash-In, Cash-Out, P2P, and Merchant payments.
  • Settlement: Batching merchant payments for periodic payouts.
  • AuditLog: PCI DSS & BSP Circular 808 compliant forensic logs.

2.3. Unified Ledger (budolAccounting)

  • ChartOfAccount: Standardized structure (Asset, Liability, Equity, Revenue, Expense).
  • LedgerEntry: Double-entry records mapped to business transactions.

3. Service Interactions

3.1. Payment Flow (QR/Merchant)

  1. Mobile App: Initiates /api/wallet/process-qr.
  2. API Gateway: Verifies JWT, checks RBAC, and proxies to Wallet Service.
  3. Wallet Service:
    • Validates QR data and transaction status.
    • Performs balance check and deduction from Wallet.
    • Updates Transaction status to COMPLETED.
    • Notifies real-time gateway for instant feedback.
  4. Gateway: Pushes balance_update and transaction_update to clients.

4. Security & Compliance

  • VPC Isolation: Databases restricted to internal VPC traffic.
  • Encryption: 100% HTTPS/WSS coverage via Let's Encrypt and ACM.
  • Auditability: Centralized forensic logging (PCI DSS 10.2.2).
  • RBAC: Granular permission model enforced at the Gateway.
