Version 323

Developer Manual

Technical implementation of bug fixes for host handling, login flow, and release 1.2.48.

Purpose

This version addresses critical runtime bugs that prevented the app from communicating correctly with the backend when custom ports were used, and fixed issues in the login identification flow. It also marks the formal release of version 1.2.48.

Explanation & Rationale

Reliable connectivity and a smooth login experience are fundamental to the app's operation. By hardening the URL derivation logic and making user identification more resilient to backend response variations, we ensure the app works across diverse network environments and backend configurations.

Issues & Fixes

  • Issue: Invalid URL construction in api_service.dart when host included a port.
    Fix: Implemented robust host normalization and URL derivation logic (computeBaseUrlFromHost, computeAuthUrlFromHost).
  • Issue: userId extraction failed in login_screen.dart depending on backend response structure.
    Fix: Added multiple fallback mechanisms to extract userId from the identify API response.
  • Issue: BuildContext async gap warnings in LoginScreen.
    Fix: Added mounted checks after asynchronous calls before using BuildContext.
  • Issue: Hardcoded version in test scripts.
    Fix: Updated test_biometric_login.dart to dynamically read the version from pubspec.yaml.

Technical Details

Host Normalization

The ApiService now uses a private _normalizeHost method to strip protocols and trailing slashes from the host input, ensuring a clean string for URL construction.


// lib/services/api_service.dart
static String _normalizeHost(String host) {
  String normalized = host.trim();
  if (normalized.startsWith('http://')) normalized = normalized.substring(7);
  if (normalized.startsWith('https://')) normalized = normalized.substring(8);
  if (normalized.endsWith('/')) normalized = normalized.substring(0, normalized.length - 1);
  return normalized;
}

Robust User Identification

The login flow now checks multiple potential locations for the user ID in the API response, making it compatible with various backend versions.


// lib/screens/login_screen.dart
final apiUserId = apiService.currentUser?['id']?.toString();
final resultUserId = (result['user'] is Map)
    ? (result['user'] as Map)['id']?.toString()
    : null;
final String? userId = result['userId']?.toString() ?? resultUserId ?? apiUserId;
