Version 323

System Admin Manual

Deployment procedures and operational updates for budolPay release 1.2.48.

Infrastructure Changes

No major infrastructure changes were required for this update. The primary focus was on client-side bug fixes and build artifact generation.

Deployment Procedure

  1. Artifact Location: The new release APK is available at android-installer/budolPay_1.2.48.apk.
  2. Verification: Ensure the file size is approximately 128 MB and the version code is 48.
  3. Distribution: This APK can be distributed to testers or uploaded to the internal app distribution platform.

Environment Configuration

The app now handles host inputs more robustly. Admins can provide host strings in the following formats:

  • 192.168.1.24 (Default port 8080 used)
  • 192.168.1.24:9090 (Custom port used correctly)
  • http://192.168.1.24:8080 (Protocol stripped and handled)

Operational Runbooks

Troubleshooting API Connectivity

If the app fails to connect to the backend:

  1. Verify the backend is reachable from the mobile device's network.
  2. Check the host string in the app's settings.
  3. Inspect backend logs for incoming requests from the mobile app's IP.

Rollback Plan

If critical issues are discovered in 1.2.48, roll back to version 1.2.47 (budolPay_1.2.47.apk) located in the android-installer directory.
