v292: SSO Flow Robustness & 127.0.0.1 Standardization

Date: 2026-01-10 | Version: v292

COMPLETED

Purpose & Scope

This implementation focused on resolving the "Invalid Token" and redirect loop errors reported by the user during SSO login in the development environment. The primary cause was identified as IPv6 resolution issues (localhost vs 127.0.0.1) and inconsistent SSO URL configurations across services.

Key Changes

  • Standardized SSO URLs: Updated all .env files to use 127.0.0.1 instead of localhost for cross-service communication.
  • Robust Callback Handling: Updated budolPay and budolShap SSO callback routes with fallback logic for 127.0.0.1 and detailed error logging.
  • JWT Secret Sync: Verified and synchronized JWT_SECRET across 21 environment files in the ecosystem.
  • User Synchronization: Created seed-galvez.js to ensure the primary test user reynaldomgalvez@gmail.com exists in the budolID database.

Technical Details

Modified budolshap-0.1.0/app/auth/callback/route.js and budolpay-0.1.0/apps/admin/app/api/auth/callback/route.ts to explicitly replace localhost with 127.0.0.1 in the SSO URL before making the verification fetch request.

// Robust URL Resolution let ssoUrl = process.env.NEXT_PUBLIC_SSO_URL || 'http://127.0.0.1:8000'; if (ssoUrl.includes('localhost')) { ssoUrl = ssoUrl.replace('localhost', '127.0.0.1'); }

Test Results

End-to-end SSO flow validation was performed using test-sso-full.js. The test successfully simulated a login at budolID and subsequent verification at budolPay and budolShap.
