2. Database Schema Updates

Ensure the Session table in the budolid database is properly indexed on the token column to facilitate fast verification during the SSO handshake.

CREATE UNIQUE INDEX IF NOT EXISTS "Session_token_key" ON "public"."Session"("token");

3. Deployment Procedures

  1. Stop all running services in the ecosystem.
  2. Update .env files in budolID, budolPay, and budolShap with the standardized JWT_SECRET.
  3. Run the synchronization seed script: node budolID-0.1.0/seed-galvez.js.
  4. Restart services using node auto-start.js.
Security Risk: The standardized JWT_SECRET provided in this manual is for DEVELOPMENT USE ONLY. Ensure a unique, high-entropy secret is generated and stored in a secure vault (e.g., AWS Secrets Manager, Vercel Secrets) for production deployments.

Mitigation Strategies

  • Risk: Unauthorized access if JWT_SECRET is compromised.
    Mitigation: Implement automated secret rotation every 30 days and monitor for unusual login patterns in budolID logs.
  • Risk: Service downtime during .env updates.
    Mitigation: Use blue-green deployment strategies to update environment variables without interrupting active user sessions.
