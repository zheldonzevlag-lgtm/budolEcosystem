System Admin Manual - Version 260

Deployment Procedures

1. Backend Update

Ensure the transaction-service is restarted to apply the new Prisma selection logic. No database migrations are required as the fields firstName and lastName already exist in the User model.

# Restart transaction service
cd services/transaction-service
npm start

2. Mobile App Deployment

The new mobile application version 1.2.24+24 has been built and is available in the installer directory.

  • File: android-installer/budolPayMobile_1.2.24.apk
  • Version: 1.2.24+24
Note: Ensure users uninstall previous versions before installing the new APK to avoid cache conflicts with the splash screen display.

Operational Runbook

Monitor the transaction-service logs for any Prisma errors related to user field selection. If the firstName or lastName fields are missing in the database, the system will gracefully fallback to using the email address.
