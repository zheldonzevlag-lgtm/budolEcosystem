Developer Manual - Version 260

Purpose

The goal of this update was to improve user experience by displaying the recipient's full name (First Name + Last Name) instead of just their email address in the mobile application's transfer success notifications.

Technical Details

1. Backend: Transaction Service

The transaction-service/index.js was modified to include firstName and lastName in the Prisma select statement when creating or updating transactions.

// services/transaction-service/index.js
const completedTransaction = await prisma.transaction.update({
    where: { id: transaction.id },
    data: { status: 'COMPLETED' },
    include: {
        sender: { select: { email: true, firstName: true, lastName: true } },
        receiver: { select: { email: true, firstName: true, lastName: true } }
    }
});

2. Frontend: Mobile Application

The lib/screens/send_money_screen.dart was updated to parse the new firstName and lastName fields from the transaction response.

// lib/screens/send_money_screen.dart
final tx = response['transaction'];
String displayName = recipient;
if (tx != null && tx['receiver'] != null) {
    final firstName = tx['receiver']['firstName']?.toString() ?? '';
    final lastName = tx['receiver']['lastName']?.toString() ?? '';
    final fullName = '$firstName $lastName'.trim();
    if (fullName.isNotEmpty) {
        displayName = fullName;
    }
}

Test Results

Verified using test-p2p-transfer.js. The backend successfully returns:

"receiver": {
    "email": "receiver@example.com",
    "firstName": "John",
    "lastName": "Doe"
}

The mobile UI correctly falls back to email if names are null or empty.
