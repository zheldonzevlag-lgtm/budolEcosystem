Vercel Deployment & API Configuration

The mobile app is designed to be deployed to Vercel. To ensure it connects to the correct backend services in production, use the following build variables:

  • GATEWAY_URL: The production URL of the budolPay Gateway.
  • AUTH_URL: The production URL of the budolID Auth Service.
  • BUDOLPAY_API_KEY: The API Key for budolPay authentication (Default: bp_key_2025).

Build Command Example:

flutter build web --release \
  --dart-define=GATEWAY_URL=https://api.budolpay.com \
  --dart-define=AUTH_URL=https://id.budolpay.com \
  --dart-define=BUDOLPAY_API_KEY=your_production_key
Pro Tip: If no build variables are provided, the app defaults to local discovery mode, automatically connecting to the host IP on ports 8004 and 8000.

Android SDK Setup

To build native Android apps, you must have the Android SDK installed. You have two options:

Method A: Android Studio (Easiest)

  1. Download and install Android Studio.
  2. During installation, ensure "Android SDK" and "Android Virtual Device" are selected.
  3. Once installed, open Android Studio and go to Settings > Languages & Frameworks > Android SDK to install the latest platform tools.

Method B: Automated CLI Setup (Advanced)

I have successfully configured the Android SDK at d:\android and built the native app. If you encounter memory issues during build, follow these optimizations:

  1. Memory Optimization: The android/gradle.properties has been tuned to work on systems with limited paging files: 
    org.gradle.jvmargs=-Xmx1024m -XX:MaxMetaspaceSize=256m
org.gradle.daemon=false
  2. Environment Setup: Ensure JAVA_HOME is set to the Android Studio JBR: 
    $env:JAVA_HOME = "d:\android\Android Studio\jbr"
  3. Final Installation: To reinstall or update the app on your connected phone: 
    flutter run -d RR8M60AYXKF --debug
  4. Custom Icons: The app icons have been updated using the files from C:\Users\Administrator\Pictures\budolPay_app_icons. The 512x512 Android icon was used as the base.

Mobile Installation Guide

Option 1: Progressive Web App (PWA) - Recommended

Since the project is currently optimized for Web, users can install the app directly from their mobile browser without needing an APK.

  1. Open Chrome on your Android phone.
  2. Navigate to http://92.168.1.24:8081 (or your current server IP).
  3. Tap the three dots (⋮) in the top right corner.
  4. Select "Add to Home screen" or "Install app".
  5. The app will now appear on your phone's home screen with the budolPay icon.

Option 2: Native Android APK (Installer)

A release APK has been built and is ready for installation. This version is optimized and avoids many of the issues found in debug mode (like the "black screen" during startup).

  • APK Location: D:\IT Projects\budolEcosystem\budolPayMobile\build\app\outputs\flutter-apk\app-release.apk
  • How to Install:
    1. Copy the app-release.apk to your phone.
    2. Open the file on your phone and tap Install (you may need to allow "Install from unknown sources").
    3. Alternatively, use ADB to install it from your PC: 
      & "d:\android\platform-tools\adb.exe" install -r "D:\IT Projects\budolEcosystem\budolPayMobile\build\app\outputs\flutter-apk\app-release.apk"
Troubleshooting: If you see a black screen, it is likely due to the app's dark theme (#0F172A) combined with a slow initial load. The release APK significantly improves this performance.
