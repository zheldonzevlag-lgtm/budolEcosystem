Developer Manual - v326

Biometric UI Responsiveness & Enrollment Checks

This version addresses issues where the biometric login icon was non-responsive on certain devices due to missing enrollment verification and outdated app versions.

Fixed

Changes in BiometricService

Added hasEnrolledBiometrics() to check if the user actually has fingerprints or face data registered in their OS settings. This prevents the app from attempting to launch the OS prompt when no biometrics are available, which can cause hangs on some Android versions.

  • File: biometric_service.dart

Changes in LoginScreen

Updated _handleBiometricLogin to provide explicit feedback for different failure states:

  • Not Supported: Shows error if hardware is missing.
  • Not Enrolled: Shows error if no biometrics are registered in OS.
  • Disabled: Shows error if biometric login is not enabled in app settings.
  • File: login_screen.dart

App Versioning

The app version has been incremented to 1.2.50+50. This is critical as older versions (e.g., 1.2.45) contained incomplete biometric logic.
