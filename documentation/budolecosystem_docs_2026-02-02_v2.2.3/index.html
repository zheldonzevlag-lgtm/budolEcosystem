Configuration Guide

The definitive guide to configuring Email, SMS, and Payment Gateway providers within the Budol Ecosystem.

1. Email Provider Setup

Configure how the system sends transactional emails, OTPs, and alerts.

Step 1: Select Provider

Go to System Settings > Notification Settings and set NOTIFICATION_EMAIL_PROVIDER.

Step 2: Provide Credentials

  • Gmail: Enter your email in NOTIFICATION_EMAIL_USER and App Password in NOTIFICATION_EMAIL_PASSWORD.
  • Resend: Provide your API key in RESEND_API_KEY.
  • SMTP: Configure SMTP_HOST, SMTP_PORT, SMTP_USER, and SMTP_PASS.

2. SMS Provider Setup

Manage mobile verification and SMS alerts via third-party gateways.

Step 1: Select Provider

Set NOTIFICATION_SMS_PROVIDER to your chosen gateway (e.g., Twilio).

Step 2: Credentials

For Twilio, configure TWILIO_ACCOUNT_SID and TWILIO_AUTH_TOKEN. Ensure the SID is correct to avoid 401 errors.

3. Payment Gateway Setup

Connect your merchant accounts to process global and local transactions.

Step 1: Choose Active Provider

In Payment Gateways, set ACTIVE_PAYMENT_PROVIDER.

Step 2: API Integration

Input your Public and Secret keys in the corresponding provider fields (e.g., PAYMONGO_SECRET_KEY). All secret keys are masked for security.
