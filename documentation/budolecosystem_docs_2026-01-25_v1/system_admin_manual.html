System Admin Manual - Payment Integration

Monitoring Payment Status

Admins can monitor payment verification logs in the application server logs. Look for [Unified Checkout] and [PaymentStatusAPI] tags.

Common Statuses:

  • succeeded: Payment confirmed by provider.
  • pending: Awaiting user action or provider confirmation.
  • failed: Payment rejected or cancelled.
  • timeout: Verification took longer than 60 seconds.

Configuration

Ensure the following environment variables are correctly set for budolPay integration:

  • BUDOLPAY_API_KEY: Required for status verification.
  • NEXT_PUBLIC_BASE_URL: Used to construct return URLs.
