Mitigation Plan

M-001: Polling Timeout Mitigation

Strategy: Progressive Polling & Fallback.

  • Implementation of exponential backoff in status polling (3s, 5s, 10s intervals).
  • Provision of a "Check Order Status" button on timeout to manually trigger verification.
  • Background webhook listener to update order status even if the user closes the browser.

M-002: Status API Failure Mitigation

Strategy: Redundancy & Manual Audit Trail.

  • Logging of all failed status requests with full payload for manual reconciliation.
  • Automatic retry logic for transient 5xx errors.
  • Direct database status check before calling external provider APIs.
