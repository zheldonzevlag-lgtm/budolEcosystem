budolEcosystem v219

Critical Persistence Fix & Navigation UI Update

← Previous (v218) Main Index Next (v219) → Next →

Purpose

This emergency release addresses the persistence of the old IP address (192.168.1.24) which was being loaded from SharedPreferences, overriding the new default. It also adds a version/API indicator to the Login UI for easier verification.

Technical Specification & Fixes

1. IP Persistence Reset

Issue: Even after updating the code to use 192.168.1.3, some devices were still hitting the old IP because it was saved in the device's local storage (SharedPreferences).

Resolution: Added a safety check in lib/services/api_service.dart during initialization to explicitly clear the problematic IP if detected.

// lib/services/api_service.dart
if (_customHost == '192.168.1.24') {
    _customHost = null;
    await prefs.remove(_hostKey);
}

2. Login UI Debug Indicator

Improvement: Added a small, non-intrusive text at the bottom of the Login screen showing the current APK version and the API host being used.

Resolution: Modified lib/screens/login_screen.dart to display v219 - API: [host].

Validation & Test Results

  • Persistence Cleared: Verified that the app now correctly falls back to 192.168.1.3 even if the old IP was previously saved.
  • Navigation Verified: Confirmed main.dart routes and LoginScreen navigation logic match (/registration).
  • Build Artifact: budolPay_v219_fix.apk generated successfully.

Recommendation

User Action Required: To ensure all old settings are cleared, it is highly recommended to UNINSTALL the previous version of budolPay from the mobile device before installing the new budolPay_v219_fix.apk.
