BudolEcosystem Documentation v178

Admin Document Viewing Fix & Blob-to-Image Conversion

← Previous (v177) Main Index Next (v178) → Next (Latest) →

Purpose

This version documents the fix for the Admin User Verification screen, where KYC documents were previously shown as placeholders with "Stored as BYTEA" text. The update enables actual image rendering by converting database blob data into viewable base64 strings.

Problem Analysis

  • Issue: The Admin panel received document data from Prisma as Bytes, which serializes to a Buffer object in JSON.
  • Constraint: Browsers cannot directly render Buffer objects as <img> sources.
  • Impact: Admins could not review submitted IDs or selfies, blocking the KYC approval workflow.

Technical Implementation

1. Blob-to-URL Conversion Utility

A new helper function getDocUrl was added to the Admin UsersPage component to handle both remote URLs and local database blobs:

const getDocUrl = (doc: any) => {
  if (doc.remoteUrl) return doc.remoteUrl;
  if (doc.blobData && doc.blobData.data) {
    const uint8Array = new Uint8Array(doc.blobData.data);
    let binary = '';
    for (let i = 0; i < uint8Array.byteLength; i++) {
      binary += String.fromCharCode(uint8Array[i]);
    }
    return `data:image/jpeg;base64,${window.btoa(binary)}`;
  }
  return null;
};

2. UI Rendering Logic

The VerificationDocument modal was updated to conditionally render an <img> tag if getDocUrl returns a valid string, replacing the static placeholder.

Verification Results

  • Data Conversion: Successfully transforms Prisma Buffer objects to Base64.
  • Image Display: Images are rendered correctly with object-cover styling and hover animations.
  • Fallback: Maintains the previous placeholder UI if no image data is available.

Dependencies

  • Prisma: Provides the VerificationDocument model with blobData.
  • Lucide React: Used for icons in the fallback UI.

Future Enhancements

  • Server-side Image Serving: Implement an API route to serve images directly (e.g., /api/docs/[id]) to reduce frontend memory usage for large datasets.
  • Zoom/Lightbox: Add a fullscreen viewer for detailed document inspection.
