System Admin Manual

Procedures for recovering and maintaining the Budolshap production environment.

1 Database Restoration

To restore the database from an S3 backup, use the run_db_restore.py script. This script launches an ECS Fargate task with the postgres:17 image to execute pg_restore against the RDS instance.

Command: python run_db_restore.py

2 Schema Synchronization

After a restore, ensure the Prisma schema matches the database state. If NULL constraint errors occur, use run_db_fix.py to patch legacy records before running prisma db push.

3 Container Deployment

When the schema changes, the frontend container must be rebuilt to update the Prisma Client. Use Docker with DOCKER_BUILDKIT=0 (Legacy Mode) if stream errors occur during build.

Registry: 194442925745.dkr.ecr.ap-southeast-1.amazonaws.com/budol-shap

Image Tag: v15

4 ECS Service Update

Force a new deployment on the budol-shap-service in the budol-ecosystem-cluster to pull the latest image.

Command: aws ecs update-service --cluster budol-ecosystem-cluster --service budol-shap-service --force-new-deployment
