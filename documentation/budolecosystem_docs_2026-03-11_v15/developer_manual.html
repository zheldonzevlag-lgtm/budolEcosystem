Database Schema Changes

To resolve the "Missing Products" issue, the Prisma schema was updated to accommodate legacy data with NULL fields in Address and AuditLog models.

Model: Address

model Address { id String @id @default(cuid()) userId String name String email String houseNumber String? street String barangay String? // Reverted to optional to handle legacy nulls subdivision String? ... }

Code Reference: schema.prisma

Product Query Fix

The categoryData relation was causing 500 errors because the generated Prisma Client in the container was outdated. Rebuilding the image with npx prisma generate resolved this.
