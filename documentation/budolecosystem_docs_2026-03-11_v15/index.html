Release v15 Overview

Database Consistency & Product Display Recovery for Budolshap E-commerce site.

Purpose

This release addresses the critical "Product Missing" issue on budolshap.duckdns.org. The root cause was a combination of an outdated database version in the restore script, schema mismatches regarding categoryData, and NULL constraint violations in legacy data.

Implementation Task List

  • DONE Phase 0: Baseline & Risk Analysis - Identified data mismatch in categoryData and Address fields.
  • DONE Phase 1: Architecture & Data Contracts - Updated schema.prisma to make barangay and userId optional.
  • DONE Phase 2: Backend & API Implementation - Restored production database to RDS Postgres 17.
  • DONE Phase 3: Frontend / App Implementation - Rebuilt Docker image (v15) with updated Prisma types.
  • DONE Phase 4: Integration & Security - Deployed to ECS with forced new deployment.
  • DONE Phase 5: End-to-End Testing - Verified product display on budolshap.duckdns.org.
  • PROGRESS Phase 6: Documentation & Audit - Generating versioned documentation.
  • TODO Phase 7: Release & Monitoring - Final release validation.
  • DONE Phase 8: Dependencies & Software - RDS (Postgres 17), ECS (Fargate), Docker, Next.js, Prisma.
  • TODO Phase 9: Update Knowledgebase - Update knowledgebase.html.
  • PROGRESS Phase 10: Test the code - Running Jest validation.
  • TODO Phase 11: Push/Commit - Create new branch and push to repository.
