Consolidated Recommendations for Future Enhancement

1. Automated Profile Sync Webhooks

Purpose: Replace the current manual fetch-based sync with a webhook system.

Reason: Improves reliability and reduces latency when updating user data across multiple microservices.

Risk: Delivery failure. Mitigation: Implement a retry queue (e.g., BullMQ or RabbitMQ).

2. Centralized Middleware Library

Purpose: Create a shared NPM package for common middleware (CORS, JSON parsing, Global Error Handling).

Reason: Prevents repetitive configuration errors (like the middleware ordering issue) across services.

Dependencies: All microservices must adopt the shared library.

3. Real-time KYC Verification OCR

Purpose: Implement server-side OCR (e.g., Tesseract.js or AWS Textract) for the Verification Service.

Reason: Automates the approval of ID documents, removing the need for manual SysAdmin database overrides.

Technical Details: Integrate with the current VerificationDocument model ocrData field.

4. API Gateway Request Validation

Purpose: Enforce strict JSON schema validation at the Gateway level.

Reason: Catches "Bad Request" errors before they reach downstream services like transaction-service.
