Developer Manual: v220 Technical Updates

1. BudolShap-BudolID Synchronization

Purpose: Ensure Admin profile changes in BudolShap persist through SSO logins.

Implementation:

  • Added PATCH /users/:userId/profile to budolID API to allow partial updates.
  • Modified BudolShap Admin API (/api/admin/users/[userId]) to call BudolID whenever a profile is updated locally.
  • Mapped BudolShap accountType (BUYER/ADMIN) to BudolID role (USER/ADMIN).

Code Snippet (BudolShap Sync Logic):

const syncResponse = await fetch(`${BUDOLID_API_URL}/users/${ssoUserId}/profile`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        firstName: firstName,
        lastName: lastName,
        role: accountType === 'ADMIN' ? 'ADMIN' : 'USER'
    })
});

2. Express Middleware Ordering (BudolPay)

Issue: req.body was undefined in transaction-service and wallet-service because express.json() was placed after route definitions.

Fix: Reordered middleware to ensure body parsers run before any route handlers.

// Correct Order
app.use(cors());
app.use(express.json());
app.use('/api/transactions', router);

3. Global JSON Error Handling

Purpose: Prevent services from leaking HTML error pages (e.g., 404 or 500) to the mobile app.

Fix: Added a global error handler to every Express app in the BudolPay ecosystem.

app.use((err, req, res, next) => {
    res.status(err.status || 500).json({
        error: err.name || 'InternalServerError',
        message: err.message || 'An unexpected error occurred',
        path: req.path
    });
});

4. Face Verification Route Fix

Issue: Mobile app was hitting 404 on Face Verification because api-gateway was routing to a non-existent /face-verify endpoint on the verification-service.

Fix: Added router.post('/face-verify', ...) to verification-service/index.js with logic to update user KYC status to FULLY_VERIFIED upon success.
