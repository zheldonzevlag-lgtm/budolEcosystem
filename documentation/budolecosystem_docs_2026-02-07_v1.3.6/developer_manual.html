Developer Manual: OTP Flow & State Persistence

This document details the technical implementation of changes in v1.3.6, focusing on optimizing the registration-to-login transition for phone users.

1. OTP Flow Continuity

After a successful registration using the "Quick (Phone Only)" method, the system must immediately present the OTP verification field. This is achieved by manipulating the component state before toggling the UI mode.

AuthForm.jsx
// AuthForm.jsx: Implementation of OTP trigger after registration if (registrationType === 'phone_only') { toast.success('Account created! A verification code (OTP) has been sent...'); // Set state to show OTP input in login mode immediately justRegisteredRef.current = true; setLoginMethod('phone'); setIsOtpSent(true); if (onToggleMode) { onToggleMode() // Switch to login } }

2. State Preservation Mechanism

The useEffect hook responsible for resetting the form on mode change was updated to check for the justRegisteredRef flag. This prevents the registered phone number from being cleared when the component transitions to login mode.

// AuthForm.jsx: Effect hook for state management useEffect(() => { if (justRegisteredRef.current) { // Skip full reset if we just registered justRegisteredRef.current = false; setFormData(prev => ({ ...prev, password: '', // Clear password/OTP field // Keep phoneNumber... })); return; } // Standard reset logic for other cases... }, [loginMethod, mode]);

3. Compliance & Security

This implementation ensures compliance with NPC guidelines by providing a seamless and secure verification path immediately after data collection. It also maintains security by clearing the password/OTP buffer while preserving the identifier.
