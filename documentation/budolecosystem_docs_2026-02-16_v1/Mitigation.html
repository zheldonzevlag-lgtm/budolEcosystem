Mitigation Strategies (v3.4.0)

Ref: R-001 (Data Loss during Schema Migration)

Database Backup & Validation

  • Action: Executed `backup_db_compliant.js` before applying any Prisma schema changes.
  • Action: Verified backup integrity by checking the generated SQL dump file size and content headers.
  • Outcome: Schema migration was successful; `variationId` added as nullable first to support existing data, though currently we treat it as part of the unique constraint.
Ref: R-002 (Inventory Mismatch)

Variation-Aware Order Processing

  • Action: Updated `OrderSummary.jsx` to explicitly pass `variationId` to the order creation API.
  • Action: Implemented backend validation in `api/orders` (future phase) to check stock levels of the specific variation, not just the parent product.
  • Verification: Tested with `test_cart_variations.mjs` ensuring distinct variation IDs are preserved.
Ref: R-005 (Duplicate Cart Entries)

Composite Unique Constraints & Atomic Transactions

  • Action: Added `@@unique([cartId, productId, variationId])` in `schema.prisma`. This database-level constraint prevents duplicate rows for the same variation.
  • Action: Used `prisma.$transaction` in `api/cart/route.js` to ensure cart updates are atomic (delete old -> insert new) or use upsert logic where applicable.
Ref: R-003 (Guest Cart Loss)

Hybrid Persistence Strategy

  • Action: Implemented LocalStorage for guest users.
  • Action: Created a "Merge Cart" mechanism. When a guest logs in, their local cart items are sent to the server and merged with any existing account cart.
  • User Education: Added a note in the User Manual advising users to log in to save their cart across devices.
