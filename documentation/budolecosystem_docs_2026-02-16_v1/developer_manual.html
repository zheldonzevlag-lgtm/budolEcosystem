Developer Manual - v3.3.3

1. Collapsible Admin Sidebar Implementation

The AdminSidebar component has been updated to support a collapsible state, improving the dashboard's layout efficiency.

Key Changes:

  • State Management: Added isSidebarOpen state (default false).
  • Responsive Width: Sidebar transitions between w-64 (open) and w-20 (closed).
  • Tooltip Component: Created a reusable SidebarTooltip component for displaying menu names on hover when collapsed.
  • Profile Display:
    • Open: Shows User Avatar and "Hi, [Name]".
    • Closed: Shows User Avatar (smaller) and User Initials below it.
  • Menu Items: Text labels are hidden when collapsed, and tooltips are enabled.

Code Implementation Details

The SidebarTooltip component handles the hover visibility and positioning using React state and fixed positioning to avoid being clipped by the sidebar's scrollable area.

    const [tooltipData, setTooltipData] = useState({ visible: false, text: '', top: 0, left: 0 })

    const handleMouseEnter = (e, text) => {
        if (isSidebarOpen) return
        const rect = e.currentTarget.getBoundingClientRect()
        setTooltipData({
            visible: true,
            text,
            top: rect.top + rect.height / 2,
            left: rect.right
        })
    }

    const SidebarTooltip = () => {
        if (!tooltipData.visible || isSidebarOpen) return null
        return (
            <div 
                className="fixed z-[100] px-3 py-1.5 bg-slate-800 text-white text-xs font-medium rounded shadow-lg whitespace-nowrap pointer-events-none transition-opacity duration-200"
                style={{ 
                    top: tooltipData.top, 
                    left: tooltipData.left + 12, 
                    transform: 'translateY(-50%)'
                }}
            >
                {tooltipData.text}
                <div className="absolute right-full top-1/2 -translate-y-1/2 border-4 border-transparent border-r-slate-800"></div>
            </div>
        )
    }

The sidebar width and layout transition uses Tailwind CSS classes:

<div className={`inline-flex h-full flex-col gap-5 border-r border-slate-200 transition-all duration-300 ease-in-out ${isSidebarOpen ? 'w-64' : 'w-20'} justify-between bg-white relative`}>

2. Dependencies

No new external dependencies were added. The implementation relies on:

  • lucide-react for icons.

  • next/link for navigation.
  • tailwindcss for styling and transitions.

3. Security Dashboard Update

The SecurityComplianceDashboard component has been updated to improve the visualization of the Threat Mitigation Trend chart.

  • Empty State: Days with no threat data (0 threats, 0 blocked) now display a slate background placeholder instead of an empty space.
  • Icon Update: The chart header icon has been updated to the History icon to better represent the trend data.
  • Chart Height: The chart height has been increased significantly to improve readability.
    • Container height increased from h-24 to h-80.
    • Bar wrapper height increased from h-20 to h-72.
    • Margins between the title and chart were reduced to maximize vertical space.
