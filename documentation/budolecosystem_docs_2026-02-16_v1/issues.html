Known Issues (v3.4.0)

1. Mobile Cart UI Flicker Open

Description: On some mobile devices, rapidly changing quantities in the cart might cause a brief flicker as the state resyncs with the server.

Workaround: Wait a moment between rapid clicks. The data is saved correctly; it is purely a visual artifact.
2. Variation Stock Display Lag Investigating

Description: When a variation goes out of stock while a user is viewing the page, the "Add to Cart" button might not disable immediately until the page is refreshed.

Workaround: The checkout process will catch this and prevent the purchase of out-of-stock items.
3. Old Cart Items Compatibility Investigating

Description: Users who had items in their cart before this update (v3.4.0) will see them without specific variation details (defaults to parent product). This is expected behavior for legacy data.

Workaround: Users may need to remove and re-add items if they want to select a specific new variation.
