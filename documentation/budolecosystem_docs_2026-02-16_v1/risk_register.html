Risk Register (v3.4.0)

Date: 2026-02-16

Scope: Cart Variation Implementation & Persistence

ID Risk Description Impact Area Probability Severity Risk Level Status
R-001 Data Loss during Schema Migration: Existing cart items without `variationId` might be incompatible with new schema constraints. Database Integrity Low High Medium Closed
R-002 Inventory Mismatch: Order processing might deduct stock from the main product instead of the specific variation if `variationId` is missing. Operations / Inventory Medium High High Mitigated
R-003 Guest Cart Loss: Local storage might be cleared by browser policies or user action, leading to lost sales opportunities. User Experience / Revenue High Medium Medium Accepted
R-004 Performance Degradation: Real-time syncing of cart badges on every quantity change might overload the client-side state management on low-end devices. Performance Low Low Low Monitored
R-005 Duplicate Cart Entries: Race conditions during rapid "Add to Cart" clicks for the same variation might create duplicate entries instead of updating quantity. Data Consistency Medium Medium Medium Mitigated
