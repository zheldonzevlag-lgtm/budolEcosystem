System Admin Manual

This guide explains how to manage and monitor the Budol Ecosystem using the updated Admin Dashboard.

1. Accessing Forensic Audit Trails

To view the audit logs, navigate to Settings > Forensic Audit Trails in the admin sidebar.

Note: Audit logs are immutable and provide a complete history of user actions within the system.

Using the Audit Log Table

  • User Identification: Each log entry shows the user's name and avatar. If the avatar is missing, their initials will be displayed.
  • Action Types: Actions are categorized (e.g., CREATE, UPDATE, DELETE).
  • Timestamp: Logs are sorted by date, with the most recent actions at the top.

2. Monitoring Security Compliance

The Security Compliance Dashboard provides a real-time overview of system health and security status.

Dashboard Components

  • Vulnerability Scan: Shows the number of open vulnerabilities and their severity.
  • AI Findings: Displays potential security threats detected by the AI monitoring system.
  • Threat Mitigation Trend: A visual representation of mitigated threats over time.

3. Troubleshooting

Missing User Avatars

If user avatars are not displaying, the system will automatically fall back to showing the user's initials. This ensures the layout remains consistent even if external image sources fail.

Data Discrepancies

If the audit logs appear incomplete, verify that the backend services are running and that the database connection is stable. Check the server logs for any connection errors.
