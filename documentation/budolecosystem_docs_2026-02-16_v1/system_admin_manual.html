System Admin Manual (v3.4.0)

1. Database Maintenance

Backup Procedures

Before any deployment or schema update, run the compliant backup script. This script creates a timestamped dump of the `budolshap_1db` and `budolpay` databases.

node scripts/backup_db_compliant.js

Location: Backups are stored in `\backup-db\db-YYYY-MM-DD\`.

Schema Updates

The `CartItem` table now includes a `variationId`. If you need to manually inspect cart data:

npx prisma studio

Navigate to the `CartItem` model. You will see columns: `id`, `cartId`, `productId`, `variationId`, `quantity`.

Warning: Do not manually delete `CartItem` rows directly in the database unless necessary. This may cause sync issues with the frontend Redux state if the user is currently active.

2. Deployment Steps

  1. Stop the Application: `npm stop` or Ctrl+C in terminal.
  2. Backup Database: Run the backup script.
  3. Pull Latest Code: `git pull origin main` (or relevant branch).
  4. Install Dependencies: `npm install`
  5. Update Database Schema: `npx prisma migrate deploy`
  6. Build Application: `npm run build`
  7. Start Application: `npm start`

3. Monitoring

Monitor the application logs for any `PrismaClientKnownRequestError` related to unique constraints on `CartItem`. This would indicate a race condition in the cart logic that needs investigation.
