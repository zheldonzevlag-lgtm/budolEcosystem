Version 2026-02-16 v1 Fixes

Issue: Cart items with different variations were being merged or overwritten.
Fix: Updated the data model and logic to support variation-aware cart items.
  • Schema: Added variationId to CartItem model in Prisma schema.
  • Frontend: Updated cartSlice.js to use composite keys (productId_variationId) for state management.
  • Frontend: Updated ProductDetails.jsx to generate composite keys when adding to cart.
  • Frontend: Updated Cart/page.jsx to parse composite keys and display correct variation details (price, stock, image).
  • Backend: Updated api/cart/route.js to handle atomic transactions for cart updates, correctly splitting and saving variation IDs.
Issue: Order creation was removing ALL variations of a product from the cart, even if only one was purchased.
Fix: Updated lib/services/ordersService.js to specifically target cart items by both productId and variationId during the cleanup phase of order creation.
Issue: Order cancellation restored items without variation details, potentially merging them incorrectly.
Fix: Updated lib/services/ordersService.js to include variationId when checking for existing items and when creating new cart items during restoration.
Issue: Cart data was not persisting reliably across sessions or page reloads for guests.
Fix: Enhanced persistence logic.
  • Implemented local storage persistence for guest users.
  • Improved merge logic in app/(public)/layout.jsx to combine local guest cart with server cart upon login.
  • Added logic in Navbar.jsx to force-sync cart data to the server before logout.
Issue: Order creation payload was missing variation details.
Fix: Updated OrderSummary.jsx to include variationId in the orderItems payload sent to the /api/orders endpoint.
Issue: Database safety during schema updates.
Fix: Created and executed a compliant database backup script scripts/backup_db_compliant.js to secure data before applying schema changes.
Issue: Cart items disappearing after Logout/Login (Race Condition).
Fix: Resolved a race condition in the logout process.
  • Problem: The clearCart() action was dispatched in Navbar.jsx before the user session was fully terminated. This triggered the PublicLayout.jsx auto-sync effect to overwrite the server-side cart with an empty state just before logout completed.
  • Solution: Moved the clearCart() dispatch and local storage cleanup to occur strictly after the logout() process completes and the user ID is nullified. This ensures the PublicLayout sync effect aborts (due to missing user ID) and preserves the server-side cart data.
