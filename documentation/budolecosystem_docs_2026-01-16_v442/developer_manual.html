Developer Manual - v442

Version: v442 | Date: 2026-01-16 | Author: Budol Ecosystem Master Orchestrator

Purpose

To enforce consistent branding for "budolPay" across all platforms (React and Flutter) specifically when displayed within order numbers, payment descriptions, and instructional text.

Rationale

Users reported seeing inconsistent casing (e.g., BUDOL_PAY) in order summaries and transaction details. Normalizing these to the branded "budolPay" (with rose red "Pay") strengthens brand identity and improves visual professionalism.

Technical Explanation

The solution leverages existing cross-platform branding utilities:

  • React: The BudolPayText component uses a regex /(budol)(_?)([a-zA-Z₱]+)/gi to identify and style "budol" variations in real-time.
  • Flutter: The UIUtils.formatBudolPayText method uses a similar regex to return a Text.rich widget with appropriate styling.

Changes

1. React (budolShap)

Updated multiple components to wrap dynamic order numbers and payment methods in BudolPayText.

// Example: OrderHeader.jsx
<h1 className="text-lg md:text-xl font-bold mb-1">
    Order #<BudolPayText text={order?.id ? order.id.slice(-12) : '---'} />
</h1>

2. Flutter (budolPayMobile)

Audited screens to ensure formatBudolPayText is applied to transaction titles and descriptions.
