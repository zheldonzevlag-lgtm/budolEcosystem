Mitigation Plan - v442

Version: v442 | Date: 2026-01-16

M-442-01: Performance Optimization

Risk: Regex performance on large lists.

Mitigation: Implement const pattern outside of loops and use React.memo or Flutter's const constructors where applicable to prevent unnecessary re-renders.

M-442-02: Regex Refinement

Risk: False positives in branding.

Mitigation: Ensure the regex /(budol)(_?)([a-zA-Z₱]+)/gi is specific enough. Currently, it requires the "budol" prefix, which is unique to our ecosystem.

M-442-03: Email Branding Audit

Risk: Inconsistent branding in legacy emails.

Mitigation: Plan a Phase 2 update to migrate all email templates to a centralized branding utility that handles text normalization before sending.
