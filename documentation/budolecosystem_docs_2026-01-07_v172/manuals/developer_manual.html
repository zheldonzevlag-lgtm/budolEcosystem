Vercel Prefix Compatibility Implementation

Unified Ingress & Local Alignment

The ecosystem deployment has been aligned with the local development structure (Central Runner):

  • Root Domain (/): Proxies to the BudolPay Admin dashboard (Local Port 3000 equivalent).
  • /shap: Proxies to the BudolShap marketplace (Local Port 3001 equivalent).
  • /api: Proxies to the API Gateway, which manages microservice routing.

Standardized Prefix Handling

All backend services must handle the /api prefix to ensure compatibility with the consolidated Vercel ingress:


const router = express.Router();
app.use('/api/service-name', router);
app.use('/api', router); // Standardized consolidated prefix
app.use('/', router);

Affected Services

  • Auth Service: Handled /api/auth
  • Wallet Service: Handled /api/wallet
  • Transaction Service: Handled /api/tx
  • Payment Gateway: Handled /api/payment-gw
  • KYC/Verification: Handled /api/kyc
  • Settlement Service: Handled /api/settlement
  • Accounting Service: Handled /api/accounting
  • ID Service: Handled /api/id
  • API Gateway: Handled /api/gateway and /api/notify

Prisma Generation

Added postinstall scripts to budolAccounting and budolID package.json files to ensure the Prisma client is generated in the serverless environment:

"postinstall": "prisma generate"
