Purpose

This release (v243) focuses on hardening the mobile application's financial service layer through robust type safety, refactoring core transaction methods for better developer ergonomics, and resolving runtime parsing issues that affected the user's wallet visibility.

Issues & Fixes

Issues Identified

  • Type Safety Risk: getBalance() returned a raw Map, forcing every UI component to implement manual parsing logic, leading to frequent _InternalLinkedHashMap' is not a subtype of type 'double' errors.
  • Developer Ergonomics: Transaction methods (cashIn, transfer) used positional Map parameters which lacked compile-time verification for required fields like amount or recipient.
  • Null Safety: Potential crash in CashInScreen when _selectedProvider was null during rapid UI interactions.

Fixes Implemented

  • Service Layer Refactoring: Updated ApiService.getBalance() to return a double directly, centralizing error handling and parsing logic.
  • Named Parameters: Refactored cashIn(), transfer(), and processPayment() to use Dart named parameters ({required double amount, ...}), enabling compile-time checks and better IDE autocompletion.
  • UI Stabilization: Simplified balance handling in HomeScreen and WalletScreen, removing redundant parsing logic.
  • Safeguard Integration: Added null-coalescing defaults in CashInScreen to prevent runtime crashes.
  • Branding Accuracy: Fixed double-versioning prefix (v v1.2.10) in Splash and Login screens for better professional presentation.

Test Results

  • Build Verification: Successfully generated production APK budolPay_v1.2.10.apk.
  • API Verification: All refactored methods verified against local gateway.
  • UI Verification: Balance display and transaction flows tested on emulator/device.
