Purpose

This update focuses on standardizing administrative authentication across all API endpoints, securing sensitive system settings, and ensuring full menu access for the elevated admin account (reynaldomgalvez@gmail.com).

Explanation

Previous audits revealed several unprotected administrative API endpoints and inconsistent authentication logic. This version centralizes authentication using the requireAdmin helper and isUserAdmin utility, ensuring a consistent security posture across the entire admin dashboard.

Technical Details

Centralized Auth Logic

Updated isUserAdmin in lib/adminAccess.js to prioritize the database isAdmin flag, followed by account type and email checks.

export function isUserAdmin(user, adminEmailSet) {
    if (!user) return false;
    
    // Priority 1: Database isAdmin flag (Primary source of truth)
    if (user.isAdmin === true) return true;
    
    // Priority 2: Account Type
    if (isAccountTypeAdmin(user.accountType)) return true;
    
    // Priority 3: Email list
    if (isEmailConfiguredAsAdmin(user.email, adminEmailSet)) return true;
    
    return false;
}

API Route Protection

The following categories of routes were secured:

  • System Settings: /api/system/settings, /api/system/cache, /api/system/error-tracking, /api/system/realtime
  • Order Management: /api/orders, /api/orders/[orderId]
  • Admin Features: Coupons, Store Approvals, Memberships, Payouts, etc.
