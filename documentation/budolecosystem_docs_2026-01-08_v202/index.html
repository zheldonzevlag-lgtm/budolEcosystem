Release Summary: v202 (Device Trust Reset & Forced Recovery)

This update resolves the issue where users were stuck on the PIN screen after a system-level PIN reset. By clearing device trust and adding proactive error handling, we ensure the mobile app correctly redirects users to the OTP verification and PIN setup flow.

Key Achievements

  • Device Trust Reset: Cleared trustedDevices for the test account to force a fresh identification flow.
  • Proactive Error Handling: Updated verify-pin route to return OTP_REQUIRED status when no PIN is set, aiding mobile app navigation.
  • State Consistency: Synchronized database state (null PIN + null trusted devices) to trigger the most reliable recovery path.
