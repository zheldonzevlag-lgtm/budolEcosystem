Developer Manual: v385 Android Build

Build Process

The build was generated using the standard Flutter release workflow:

flutter pub get
flutter build apk --release

Version Synchronization

To ensure consistent version reporting across the ecosystem, the version was incremented in two key locations:

  • pubspec.yaml: version: 1.2.81+81
  • api_service.dart: _appVersion = '1.2.81'

Artifact Management

The final APK is stored in the centralized installer repository:

\budolEcosystem\android-installer\budolPay 1.2.81.apk

Phase 8: Dependencies & Software Choice

The following key dependencies were chosen for this release:

  • Flutter (v3.0.0+): Chosen for its high-performance UI rendering and unified codebase for Android/iOS.
  • google_mlkit_face_detection: Used for robust on-device face bounding box detection.
  • tflite_flutter: Enables high-speed on-device inference for custom models (FaceNet), essential for offline biometric verification.
  • package_info_plus: Ensures accurate version reporting in the splash screen by reading native build metadata.
