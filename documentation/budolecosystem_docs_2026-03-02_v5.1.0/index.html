v5.1.0: Lazy Cart Clearing Strategy

Date: 2026-03-02 Version: v5.1.0 Status: Implementation Verified

Release Summary

This release implements a user-requested change to the cart clearing behavior. Items now persist in the cart during the payment phase for asynchronous methods, ensuring a better user experience if the payment fails or is cancelled.

  • Deferred Removal: Cart items are only removed from the database and frontend state AFTER a successful payment is confirmed.
  • Fail-Safe UX: If a GCash or Maya payment is cancelled at the gateway, the user's cart remains intact.
  • Realtime Sync: Uses the cart-updated event to trigger the final UI refresh once payment is confirmed.
