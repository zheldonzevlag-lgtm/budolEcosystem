Strategy Change: Lazy Cart Clearing

Based on user feedback, the cart items are no longer removed immediately upon placing an order for Asynchronous Payments (GCash, Maya, Cards, etc.). This ensures that if the user cancels or fails the payment at the gateway, they can easily return to their cart and try again.

1. Frontend Delay (Redux)

The OrderSummary.jsx component now detects the payment method. If it's an async method, it skips the local cart clearing. The items stay visible in the cart header and page until the payment is confirmed.

File: components/OrderSummary.jsx

2. Backend Delay (Database)

The ordersService.createOrder function now persists the cart items for async payments. The removal is deferred to the updateOrderStatus function, which is only called when the payment gateway sends a successful webhook.

File: lib/services/ordersService.js

3. Realtime Synchronization

When the payment is finally successful, the backend triggers a cart-updated event. The Navbar.jsx component listens for this and refreshes the cart count instantly, providing a seamless "Paid -> Cart Cleared" transition.

Verification

A verification script verify_lazy_cart_v510.mjs confirmed that for a MAYA order:

  • Cart items exist after order placement.
  • Cart items are removed after payment success.
