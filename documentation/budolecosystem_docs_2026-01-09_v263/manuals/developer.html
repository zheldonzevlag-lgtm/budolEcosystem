Purpose

This update replaces the synchronous AlertDialog used for transfer success feedback with an asynchronous SnackBar. This improves user flow by allowing immediate navigation back to the previous screen while still providing persistent feedback about the transaction details.

Technical Details

The changes were made in lib/screens/send_money_screen.dart within the _handleTransfer method.

Code Implementation

File: lib/screens/send_money_screen.dart

// Success logic in v263
final reference = tx != null ? (tx['reference'] ?? tx['id']?.toString().substring(0, 8).toUpperCase() ?? 'N/A') : 'N/A';

ScaffoldMessenger.of(context).showSnackBar(
  SnackBar(
    content: Column(
      mainAxisSize: MainAxisSize.min,
      crossAxisAlignment: CrossAxisAlignment.start,
      children: [
        const Text('Transfer Successful', style: TextStyle(fontWeight: FontWeight.bold, color: Colors.white)),
        Text('Sent ₱${amount.toStringAsFixed(2)} to $displayName', style: const TextStyle(color: Colors.white)),
        Text('Reference: $reference', style: const TextStyle(fontSize: 12, color: Colors.white70)),
      ],
    ),
    backgroundColor: Colors.green.shade700,
    behavior: SnackBarBehavior.floating,
    shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(10)),
    margin: const EdgeInsets.all(10),
    duration: const Duration(seconds: 4),
  ),
);
Navigator.pop(context);

Key Changes

  • Feedback Type: Switched from AlertDialog to SnackBar.
  • Navigation: Navigator.pop(context) is now called immediately after triggering the Snackbar, rather than waiting for user interaction with a dialog button.
  • Data Display: Added explicit "Reference Number" display derived from the transaction object.
  • Styling: Used SnackBarBehavior.floating and custom margins/shapes for a modern look.
