Developer Manual - v433

Technical Explanation

The QR payment system uses an adapter pattern to handle different payment providers. The BudolPay provider was hardcoding the merchant label as "budolPay". This update modifies the budolpay-adapter.js to return "Jon Galvez" as the label for QR payments.

Changes

Modified budolpay-adapter.js to update the QR label.

return { checkoutUrl: checkoutUrl, qrCode: { imageUrl: `https://api.qrserver.com/v1/create-qr-code/?...`, amount: amount, // Changed from 'budolPay' to 'Jon Galvez' label: 'Jon Galvez' }, paymentIntentId: paymentIntentId, status: data.status || 'pending', originalResponse: data };

Rationale

The "budolShap Owner" field in the frontend modal displays this label. To ensure the correct owner name is shown to the customer, the source of this data in the payment adapter needed to be updated.

Data Flow

  1. Checkout API calls paymentService.initiatePayment.
  2. paymentService uses BudolPayAdapter.
  3. Adapter returns QR metadata including the label "Jon Galvez".
  4. Frontend OrderSummary.jsx receives this and passes it to QRCodeModal.jsx.
  5. QRCodeModal.jsx displays the label in the "budolShap Owner" field.

Compliance

This change adheres to transparency requirements for merchant identification under DTI and NPC regulations, ensuring the customer knows who they are paying.
