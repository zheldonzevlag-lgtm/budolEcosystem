Developer Manual Updates - v199

Purpose

To enable a seamless recovery path for users who reach the authentication phase but have not yet configured a security PIN.

Endpoint: /login/mobile/setup-pin (POST)

A new endpoint dedicated to setting the pinHash for an existing user account.

// Request Body
{
    "userId": "uuid",
    "pin": "123456"
}

// Logic
const pinHash = await bcrypt.hash(pin, 10);
await prisma.user.update({
    where: { id: userId },
    data: { pinHash }
});

Logic Change: /verify-otp Response

The OTP verification response now includes a needsPinSetup boolean flag. This informs the mobile client if it should navigate to the home screen or a PIN setup screen.

return res.json({ 
    message: 'Device verified and login successful', 
    token,
    user: { id: updatedUser.id, phoneNumber: updatedUser.phoneNumber },
    needsPinSetup: !updatedUser.pinHash
});

Mobile Implementation (Inferred)

Mobile developers should update the OTP verification handler to check for needsPinSetup. If true, the app should navigate to a PinCreationScreen instead of the HomeScreen.
