System Admin Manual Updates - v199

PIN Adoption Monitoring

Admins can track how many users have successfully set up their PINs using the new flow:

SELECT count(*) as "UsersWithoutPIN"
FROM "User"
WHERE "pinHash" IS NULL;

Security Considerations

The /setup-pin endpoint is critical for account security. It should ideally only be called within a session that has just been verified via OTP. While the current implementation relies on the userId, future versions will require the temporary JWT issued during OTP verification to prevent unauthorized PIN resets.

Database Integrity

Ensure that the pinHash column in the User table allows nulls (which it does by default in the current Prisma schema) to support users who have registered but not yet completed PIN setup.
