SSO Integration: budolPay Admin & budolID

Overview

The Single Sign-On (SSO) integration for budolPay Admin is now complete, supporting both Headless API and Interactive Redirect flows.

1. The "Sign in with budolID" Button

A new entry point has been added to the login page (apps/admin/app/login/page.tsx). This button initiates a redirect to the central budolID login portal.

<a href="http://localhost:8000/login?apiKey=bp_key_2025">
  Sign in with budolID
</a>

2. Callback Logic (The Handshake)

The callback route /api/auth/callback/route.ts handles the return from budolID. It performs the following steps:

  1. Extracts the token from the URL.
  2. Calls budolID/auth/verify to validate the JWT.
  3. Syncs the user locally in the budolPay database.
  4. Sets a secure, HttpOnly cookie named token.

3. Headless Fallback

The standard login form remains functional but now acts as a proxy to the budolID SSO API. This allows for a native "in-app" login experience while maintaining centralized identity.

4. UI Branding Updates

The login page and sidebar have been updated with Rose Red (#f43f5e) accents to align with the ecosystem's visual identity.
