Comparative Analysis: Phone Number Login

Date: 2026-02-01

Version: v2.0.0 (Research & Planning)

1. Purpose & Rationale

To evaluate the user experience and security of phone number login flows between the Budol Ecosystem and Shopee. The goal is to identify best practices for "Phone-First" authentication to improve conversion rates and simplify onboarding for the Philippine market.

2. Comparative Analysis Table

Feature Shopee Process Budol Ecosystem (Current) Recommendation
Primary Identifier Phone number is the primary key. Email is optional. Dual-identifier (Email/Phone) are equal. Adopt Phone-First for Mobile; maintain dual for Web.
Onboarding Flow OTP-only registration. Password set later. Full form registration (Name, Email, Phone, Password). Implement "Quick Registration" (OTP + Name only).
Login Verification OTP by default, can fallback to Password. Password by default, OTP for new devices. Allow OTP-only login as a secondary option.
Real-time Validation Checks existence after inputting number. Debounced uniqueness check (v1.3.58). Current Budol logic is on par with Shopee.
Account Linking Seamless linking with Google/Facebook via phone. Manual linking via profile settings. Automate linking if phone/email matches SSO provider.

3. Technical Explanation (Shopee Flow)

Shopee uses a Stateless OTP-First approach. When a user enters a phone number:

  1. Backend checks if user exists.
  2. If YES: Send OTP -> Verify -> Issue JWT.
  3. If NO: Redirect to "Quick Register" -> Send OTP -> Verify -> Create User -> Issue JWT.

4. Implementation Plan for Budol

Phase 1: Quick Registration (Mobile)

  • Modify registration_screen.dart to support "Quick" mode.
  • Input: Phone Number + Display Name.
  • Action: Send OTP via auth-service.
  • Post-Verification: Create temporary password/profile.

Phase 2: OTP-First Login

  • Update login_screen.dart to offer "Login via OTP" button.
  • Backend: New /auth/otp-login endpoint in auth-service.

Phase 3: Ecosystem-wide Sync

  • Ensure budolID, budolPay, and budolShap all recognize the same Quick-Register profile.

5. Risks & Mitigations

Risk Mitigation
Account Takeover (SIM Swapping) Require PIN or Biometrics for financial transactions (BudolPay).
OTP Cost/Spam Implement strict rate limiting and Firebase App Check for Mobile.
