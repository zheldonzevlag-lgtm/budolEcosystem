Developer Manual - Version v509

Problem Statement

The createOrder function in ordersService.js was failing with a PrismaClientValidationError because it attempted to pass a subtotal field to the Prisma order.create method. The Order model in schema.prisma does not define a subtotal field, only total and shippingCost.

Technical Solution

The subtotal field was removed from the data object passed to Prisma. The value is still calculated locally to derive the total, but it is no longer persisted as a separate database column.

Code Changes

File: ordersService.js


// BEFORE (Lines 294-298)
status: 'ORDER_PLACED',
paymentMethod,
- subtotal: storeData.subtotal,
shippingCost,
total,

// AFTER
status: 'ORDER_PLACED',
paymentMethod,
shippingCost,
total,

Rationale

The schema design intended for total to represent the final amount. subtotal can be derived if needed by subtracting shippingCost from total, or by summing orderItems. Removing the redundant field prevents runtime validation errors.

Dependencies

  • Prisma Client v6.19.1
  • Next.js 15+

Tests

Verification involved checking the Prisma schema against the service layer logic. No database migration was required as the fix aligned the code with the existing schema.
