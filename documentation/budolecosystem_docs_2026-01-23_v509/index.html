Overview

Version v509 focuses on fixing a critical Prisma validation error in the order creation process. The issue was caused by an unknown argument subtotal being passed to the Prisma order.create() method, which was not defined in the schema.

Key Objectives

  • Resolve PrismaClientValidationError during order placement.
  • Align ordersService.js with the actual Prisma schema.
  • Ensure transaction integrity during multi-store order creation.

Compliance Verification

  • NPC (Data Privacy): Order data handling remains compliant.
  • Cybersecurity Law: Error logging and validation practices maintained.
  • DTI: Order processing logic remains consistent with consumer protection standards.
