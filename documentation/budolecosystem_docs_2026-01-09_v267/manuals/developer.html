Transaction Details UI Overhaul

This update focused on improving the readability and usability of the TransactionDetailsScreen.

Recipient Name Integration

The recipient's full name (or sender's name for incoming transactions) was moved to a more prominent position at the top of the details list, immediately before the Transaction ID.


// lib/screens/transaction_details_screen.dart

if (widget.transaction['receiver'] != null && !isIncome)
  _buildDetailRow('Recipient', '${widget.transaction['receiver']['firstName'] ?? ''} ${widget.transaction['receiver']['lastName'] ?? ''}'.trim()),
if (widget.transaction['sender'] != null && isIncome)
  _buildDetailRow('Sender', '${widget.transaction['sender']['firstName'] ?? ''} ${widget.transaction['sender']['lastName'] ?? ''}'.trim()),
_buildDetailRow('Transaction ID', widget.transaction['id'] ?? 'N/A'),

Button Layout Optimization

The action buttons ("Favorite" and "Receipt") were moved from a vertical stack to a horizontal Row. Expanded widgets were used to ensure both buttons share equal width, and Flexible with TextOverflow.ellipsis was applied to labels to prevent overflow on smaller screens.


Row(
  children: [
    if (canFavorite) ...[
      Expanded(
        child: ElevatedButton(
          // ... Favorite button implementation
        ),
      ),
      const SizedBox(width: 12),
    ],
    Expanded(
      child: OutlinedButton(
        // ... Receipt button implementation
      ),
    ),
  ],
),
