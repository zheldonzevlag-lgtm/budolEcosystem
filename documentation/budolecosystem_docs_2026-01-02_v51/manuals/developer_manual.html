Developer Manual - v51

Order Flow & Port Conflict Fixes

Purpose

This update resolves the critical "Unexpected token '<', \"

Technical Implementation Details

1. Port Re-assignment

The budolPay Gateway was incorrectly configured to use port 8000, which is reserved for the budolID SSO service. This caused budolShap to receive HTML 404 pages from the SSO service when attempting to call payment APIs.

  • Moved budolPay Gateway to port 8004.
  • Updated budolshap-0.1.0/.env and budolpay-adapter.js to reflect this change.

2. Payment Gateway API Fixes

The Gateway service's create-intent endpoint was failing due to Prisma schema mismatches:

  • Enum Fix: Changed TransactionType from PAYMENT_INTENT (invalid) to MERCHANT_PAYMENT (valid).
  • Schema Alignment: Removed the metadata field from the transaction creation call as it was not present in the current Transaction model.

3. Response Validation

Added a safety layer in budolpay-adapter.js to check for application/json content types before parsing, providing clear error messages if HTML is returned.

Code Logic Explanations

// Enhanced validation in budolpay-adapter.js
const contentType = response.headers.get('content-type');
if (!contentType || !contentType.includes('application/json')) {
    const text = await response.text();
    throw new Error(`Invalid response from BudolPay Gateway: Expected JSON but received ${contentType}`);
}

Impact Analysis

  • Order Flow: Order placement is now functional across the ecosystem.
  • Stability: Reduced service conflicts by isolating port assignments.
  • Debugging: Improved error visibility for gateway communication failures.

Recommendations for Future Enhancement

  • Implement a centralized service registry to manage port assignments dynamically.
  • Update the Prisma schema to include a JSON metadata field in the Transaction model for better auditing.
