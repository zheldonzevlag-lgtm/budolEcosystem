System Admin Manual - v51

Infrastructure & Port Configuration Updates

Infrastructure Changes

This version updates the service port mapping to prevent collisions between the SSO and Payment Gateway services.

Service Old Port New Port Status
budolID (SSO) 8000 8000 Unchanged
budolPay Gateway 8000 8004 Updated
budolShap (App) 3001 3001 Unchanged
budolPay Admin 3000 3000 Unchanged

Environment Variable Updates

budolShap-0.1.0/.env

# Corrected Gateway URL
BUDOLPAY_GATEWAY_URL="http://localhost:8004"

Deployment Procedures

  1. Update the .env file in the budolshap-0.1.0 directory.
  2. Ensure no processes are hanging on port 8000 before starting (SSO will take this port).
  3. Restart the ecosystem using the central runner: node start-ecosystem.js.
Important: If you encounter EADDRINUSE errors, use Get-Process -Id (Get-NetTCPConnection -LocalPort 8000).OwningProcess | Stop-Process -Force in PowerShell to clear the port.

Database Schema Changes

No schema migrations were required for this version, but internal service logic was adjusted to match the existing TransactionType enum values.

Operational Runbooks

Issue: "Unexpected token <" in Checkout

This usually indicates the BUDOLPAY_GATEWAY_URL is pointing to a service that returns HTML (like a 404 page) instead of the Gateway API. Verify the port in .env matches the running Gateway service port (8004).
