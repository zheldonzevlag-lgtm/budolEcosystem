Developer Manual: Dynamic Metrics Implementation

1. Why was Transaction Volume 0?

The "PHP 0" transaction volume was addressed by identifying and fixing two primary causes:

  • Logic Constraint: Previously, the dashboard only retrieved the first account of type ASSET. If that specific account had no entries, the total reflected 0 even if other asset accounts were active.
  • Database State: A [Database Audit](test_results.html) confirmed that while users exist, no LedgerEntry records had been created for the ASSET accounts. The new logic sums all accounts of type ASSET to provide a unified ecosystem valuation.

Code Implementation: page.tsx

// Corrected logic to sum all asset accounts
const assetAccounts = await prisma.chartOfAccount.findMany({
  where: { type: 'ASSET' },
  include: { ledgerEntries: true }
});

const totalBalance = assetAccounts.reduce((acc, account) => {
  const accountBalance = account.ledgerEntries.reduce((sum, entry) => {
    return sum + (Number(entry.debit) - Number(entry.credit));
  }, 0);
  return acc + accountBalance;
}, 0);

2. Dynamic System Uptime Metrics

The static 99.9% placeholder was replaced with a Live Service Health Monitor. The system now pings the /health endpoints of core microservices to calculate real-time availability.

const services = [
  { name: 'Auth', url: process.env.AUTH_SERVICE_URL },
  { name: 'Wallet', url: process.env.WALLET_SERVICE_URL },
  { name: 'Transactions', url: process.env.TRANSACTION_SERVICE_URL },
  { name: 'Gateway', url: process.env.GATEWAY_URL },
];

let healthyCount = 0;
for (const service of services) {
  try {
    const res = await fetch(`${service.url}/health`, { next: { revalidate: 30 } });
    if (res.ok) healthyCount++;
  } catch (e) {
    console.error(`Service ${service.name} is down`);
  }
}
const uptimePercentage = ((healthyCount / services.length) * 100).toFixed(1) + "%";

3. Compliance & Technical Rationale

This implementation ensures compliance with BSP Circular 808 by providing accurate, real-time monitoring of financial positions and system availability. The use of double-entry bookkeeping at the ledger level remains the source of truth for all dashboard metrics.
