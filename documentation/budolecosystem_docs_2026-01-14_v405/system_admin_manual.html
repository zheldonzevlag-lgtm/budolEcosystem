System Admin Manual: Health Monitoring & Dashboard Management

1. Live Service Monitoring

As of v405, the Admin Dashboard's "System Uptime" metric is now dynamic. The system performs automated health checks on the following services:

Service Name Monitoring Endpoint Environment Variable
Auth Service /health AUTH_SERVICE_URL
Wallet Service /health WALLET_SERVICE_URL
Transactions Service /health TRANSACTION_SERVICE_URL
API Gateway /health GATEWAY_URL
Admin Action: If the System Uptime drops below 100%, check the service logs for the specific endpoint listed above. Ensure that all microservices are running and reachable from the Admin App.

2. Financial Accuracy & Ledger Audit

The "Transaction Volume" metric is derived from the **Chart of Accounts**. To ensure accuracy:

  • Verify that all relevant bank and cash accounts are categorized as ASSET.
  • Ensure that the LedgerEntry table is being populated correctly by transaction services.
  • Manual adjustments to the volume should be made via ledger entries rather than direct database edits to maintain the audit trail.

3. Configuration

All service URLs are managed via the .env file. Changes to service locations require a restart of the Admin application to reflect in the health monitoring loop.
