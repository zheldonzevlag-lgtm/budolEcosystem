Technical Implementation Details

Affected Component: budolShap (budolshap-0.1.0)

File Modified: .env.local

Code Logic Explanation

The email verification service in lib/email.js relies on the NEXT_PUBLIC_APP_URL environment variable to construct the absolute URL for the verification link.

// Construction in lib/email.js
const verifyUrl = `${process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'}/verify-email?token=${encodedToken}`

The typo "budulshap" in the configuration caused all generated links to point to the wrong domain.

Impact Analysis

  • Onboarding: Users could not verify their accounts via email links.
  • SSO: No direct impact on SSO, but general user trust and flow were compromised.

Test Results

Verified that .env.local now contains:

NEXT_PUBLIC_APP_URL="https://budolshap.vercel.app"
NEXT_PUBLIC_BASE_URL="https://budolshap.vercel.app"
NEXT_PUBLIC_SITE_URL="https://budolshap.vercel.app"
