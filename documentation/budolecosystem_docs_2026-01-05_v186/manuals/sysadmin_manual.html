SysAdmin Manual: Image Rotation Persistence

Deployment & Maintenance

1. Database State

The VerificationDocument table has been updated with the rotation column. This was verified using prisma db push.

Column: rotation
Type: Integer
Default: 0
Constraints: NOT NULL

2. API Stability

The /api/users endpoint now handles rotation updates via the UPDATE_DOCUMENT_ROTATION action. The implementation includes debouncing on the client side to prevent API abuse and reduce database write load.

3. Troubleshooting

Note: If the Prisma client regeneration fails with EPERM on Windows, ensure that all Node.js processes (dev servers, build tasks) are stopped before running npx prisma generate.

Operational Risks

  • Data Integrity: Rotation values are restricted to 0, 90, 180, and 270 via the UI logic. Invalid values sent to the API will still be stored as integers but may not render correctly in the UI.
  • Performance: Write operations are debounced, making the system resilient to rapid user input.
