Developer Manual: Image Rotation Persistence (Enhanced)

Purpose

This implementation enhances the document rotation feature in the Admin Panel by adding debounced automatic saving and loading indicators. It ensures that orientation adjustments are persisted to the database efficiently without overwhelming the API.

Technical Details

1. Debounced Saving Logic

In apps/admin/app/users/page.tsx, the handleRotate function now uses a 1-second debounce timer. This allows users to click the rotate button multiple times quickly, but only the final orientation is sent to the server.

const handleRotate = (docId: string) => {
  const newRotation = ((rotations[docId] || 0) + 90) % 360;
  setRotations(prev => ({ ...prev, [docId]: newRotation }));

  if (rotationTimeouts[docId]) clearTimeout(rotationTimeouts[docId]);

  const timeout = setTimeout(async () => {
    setIsSavingRotation(prev => ({ ...prev, [docId]: true }));
    try {
      await fetch("/api/users", {
        method: "POST",
        body: JSON.stringify({ action: "UPDATE_DOCUMENT_ROTATION", documentId: docId, rotation: newRotation }),
        headers: { "Content-Type": "application/json" }
      });
    } finally {
      setIsSavingRotation(prev => ({ ...prev, [docId]: false }));
    }
  }, 1000);

  setRotationTimeouts(prev => ({ ...prev, [docId]: timeout }));
};

2. Loading Indicators

A Loader2 spinner from lucide-react is shown inside the rotation button while the persistence request is in flight, providing visual feedback to the administrator.

3. Database Schema

The VerificationDocument model in packages/database/prisma/schema.prisma includes the rotation field (Int, default 0), which is now fully synced and used by the UI.

Verification Results

  • Optimistic UI: Rotation is instantaneous in the browser.
  • Debouncing: Multiple clicks only trigger one API call after 1 second.
  • Persistence: Refreshing the page or reopening the modal preserves the rotation.
  • Feedback: Spinner correctly appears during the save operation.
