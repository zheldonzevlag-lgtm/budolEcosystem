System Admin Manual - v183 Updates

Database Integrity

Admin status updates directly modify the User table in the primary PostgreSQL database. Admins should be aware that:

  • Audit Trails: Current rejection actions are logged at the API level but do not have a dedicated audit table for "Rejection History".
  • Data Retention: Document images are retained in the Document table (linked to the user) even after rejection.

Infrastructure Considerations

The KYC rejection flow does not trigger any high-resource tasks (like OCR or image processing) since it is a purely metadata update. However, re-submissions from users will re-trigger OCR processes, impacting engine costs.

Issues & Troubleshooting

Warning: If a user remains in "VERIFIED" status on their mobile app after an admin rejection, ensure the mobile app's session has been refreshed. The user may need to log out and log back in to force a profile sync if push notifications are not active.

Recommendations for Monitoring

  • Monitor the frequency of "REJECTED" statuses to identify potential bot attacks or systemic issues with the OCR engine.
  • Verify that OCR data is being correctly associated with the source engine in the Document table.
