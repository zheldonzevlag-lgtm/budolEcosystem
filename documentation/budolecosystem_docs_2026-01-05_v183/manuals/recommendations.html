Strategic Recommendations - v183

Enhancing the KYC Rejection Workflow

Based on the analysis of the current flow, the following enhancements are recommended to improve security and user experience.

Priority: High

1. Automated User Notifications

Purpose: To immediately inform users when their KYC is rejected so they can take action.

Implementation: Trigger a Firebase Cloud Message (FCM) or email via a post-update hook in the backend.

Priority: High

2. Rejection Reason Persistence

Purpose: To tell the user *why* they were rejected (e.g., "Blurry Image").

Implementation: Add a rejectionReason column to the User table and an input field in the Admin UI.

Priority: Medium

3. KYC History & Versioning

Purpose: To prevent fraud by tracking users who repeatedly submit different documents.

Implementation: Create a KycAttempt table to archive previous submissions instead of overwriting the Document table.

Priority: Medium

4. Status Badges in Mobile UI

Purpose: To provide visual confirmation of the "Rejected" status in the app settings.

Implementation: Update settings_screen.dart to show a red "Rejected" badge next to the verification tile.
