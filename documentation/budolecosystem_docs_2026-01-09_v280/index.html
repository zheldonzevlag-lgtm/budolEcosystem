v280: Ecosystem-wide LAN & mDNS Verification Latest

Verification of LAN accessibility, mDNS service discovery, and official release of budolPayMobile v1.2.36.

Developer Manual System Admin Manual User Manual Recommendations

Purpose

The primary goal of this update was to verify and finalize the LAN-wide accessibility of the budolEcosystem. This included testing the mDNS (Multicast DNS) advertising logic, ensuring all services are reachable via the LAN IP (192.168.1.24), and preparing the mobile app for a physical device deployment with auto-discovery enabled.

Key Achievements

  • Verified LAN Accessibility: All core services (Gateway, Auth, ID, Accounting) confirmed reachable via 192.168.1.24 using custom verification scripts. Verified
  • mDNS Success: Confirmed mDNS advertisements are broadcasting correctly. Services like budolpay and budolid are now discoverable by network browsers. Verified
  • Mobile App v1.2.36: Released budolPayMobile v1.2.36 with auto-discovery integrated. The app now detects the API Gateway automatically upon launch. Released
  • Prisma Synchronization: Fixed versioning and binary issues in the shared @budolpay/database package, ensuring all services use a consistent Prisma client. Fixed
  • Next.js LAN Binding: Verified budolShap binding to 0.0.0.0 using next dev -H 0.0.0.0.

Validation Summary

Verification scripts verify-lan.js and verify-mdns.js were executed to confirm the state of the network. The ecosystem is now ready for cross-device interaction on the local network.
