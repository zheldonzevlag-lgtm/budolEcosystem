Developer Manual Updates - v280

Shared Database Package Fixes

A critical issue was identified where api-gateway was using a local prisma client instead of the shared @budolpay/database package. This caused synchronization issues when the database schema was updated in the shared package but not in the gateway.

Fix: Updated api-gateway/package.json to depend on @budolpay/database and updated its index.js to import the shared prisma instance:

// api-gateway/index.js
const { prisma } = require('@budolpay/database');

Prisma Versioning & Binaries

An error (0 , pq.ensureNeededBinariesExist) is not a function occurred during Prisma generation due to a version mismatch between Prisma CLI and the client. This was resolved by pinning the shared database package to Prisma 6.2.1.

npm install prisma@6.2.1 @prisma/client@6.2.1 --save-dev

mDNS Advertising Logic

The start-ecosystem.js script now categorizes services for mDNS advertising to help clients (like the mobile app) identify specific endpoints:

if (app.port === 8080) {
    mdnsType = 'budolpay'; // Main Gateway
} else if (app.name.toLowerCase().includes('id')) {
    mdnsType = 'budolid'; // Identity Provider
}

Verification Tools

Two new verification tools were added to the root directory for developers to test network health:

  • verify-lan.js: Checks if all core service ports are open and responding on the LAN IP.
  • verify-mdns.js: Browses the network for mDNS advertisements to ensure services are discoverable.
