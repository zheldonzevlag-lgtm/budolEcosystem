System Admin Manual: Verification & Maintenance

Verification Tools

Two new scripts have been added to the root directory for system administrators to verify the ecosystem's health on the network:

Command: node verify-lan.js
Purpose: Pings all core services using the detected LAN IP to ensure they are responding.
Command: node verify-mdns.js
Purpose: Browses the network for 10 seconds to find all advertised budolEcosystem services.

Prisma Client Maintenance

The start-ecosystem.js runner now automatically handles Prisma client generation. If you encounter "Prisma client not initialized" errors, you can manually run:

npm run db:generate

This will generate clients for all 4 database-driven services (Pay, ID, Accounting, Shap).

Firewall Configuration

Ensure that the following ports are open for inbound traffic on the server:

  • 3000-3001: Web Applications (Admin, Shap)
  • 8000-8080: Microservices and Gateway
  • 5353 (UDP): mDNS / Bonjour
