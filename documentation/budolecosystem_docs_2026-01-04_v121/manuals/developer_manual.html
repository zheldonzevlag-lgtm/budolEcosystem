Developer Manual: Payment Summary UI & Session Fixes

Technical Detail Version 121 | 2026-01-04

Purpose

To enhance the Payment Summary user experience by providing more transaction details and resolving a confusing "User not logged in" error during the payment confirmation flow.

Technical Details & Code Details

1. Session Validation Improvement

Modified budolPayMobile/lib/services/api_service.dart to provide a more descriptive error message when the user session is missing or expired before processing a payment.

// Enhanced session check in api_service.dart
if (user == null) {
  throw Exception('User session expired. Please log in again.');
}

2. Payment Summary UI Refactoring

Refactored PaymentSummaryScreen in budolPayMobile/lib/screens/qr_scanner_screen.dart to include:

  • SafeArea: Prevents UI elements from being cut off by device navigation bars.
  • Detail Rows: New _buildDetailRow helper to display Reference ID, Payment Method, and Store Name.
  • Button Styling: Replaced TextButton for Cancel with a structured OutlinedButton.
// New UI Structure
SafeArea(
  child: Column(
    children: [
      _buildDetailRow('Reference ID', referenceId),
      _buildDetailRow('Payment Method', paymentMethod),
      // ...
    ]
  )
)

Issues & Fixes

  • Issue: User received a generic "User not logged in" error despite having logged in.
    Fix: Improved client-side check in ApiService.processPayment with debug logging and user-friendly error messages.
  • Issue: "Cancel" text was cut off on some devices.
    Fix: Added SafeArea and converted the text link to a full OutlinedButton with proper vertical padding.
  • Issue: Missing transaction details (Store Name, Reference ID).
    Fix: Extracted additional fields from paymentData and added them to the summary view.

Test Results

Verified that the Payment Summary screen now correctly displays all four key pieces of information (Amount, Reference, Method, Store). Confirmed that clicking "Cancel" returns the user safely to the scanner without UI clipping.

Automated Tests: Created and passed test/api_service_test.dart and test/payment_summary_test.dart to verify session validation and UI structure.

00:05 +4: All tests passed!
  • ApiService Session Management: Verified that processPayment correctly identifies missing sessions.
  • PaymentSummaryScreen UI: Verified existence of SafeArea, SingleChildScrollView, OutlinedButton, and transaction details.

Risk & Mitigation

  • Risk: Session state loss on app reload in web/mobile environments.
    Mitigation: Added descriptive error messages to guide the user back to login rather than failing silently.
  • Risk: UI Overflow on small screens due to additional transaction details.
    Mitigation: Wrapped the summary content in a SingleChildScrollView and used Expanded/Spacer strategically.

Dependencies

Relies on provider package for ApiService access and material for UI components.

Recommendation for Future Enhancement

Implement local persistence (e.g., shared_preferences) for the user session to prevent "User session expired" errors after app restarts or minor network disruptions.
