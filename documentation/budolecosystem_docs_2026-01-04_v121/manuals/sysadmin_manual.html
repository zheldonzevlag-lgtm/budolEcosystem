System Admin Manual: Payment Flow Stability

Operations Version 121 | 2026-01-04

Purpose

This version addresses mobile-side payment stability and UI layout issues to ensure consistent operation across various device screen sizes.

Infrastructure Changes

No backend infrastructure changes were required for this version. The updates are strictly within the budolPayMobile application layer.

Environment Variables

Ensure the following variables are correctly passed during the mobile build process if targeting production:

  • GATEWAY_URL: Should point to the wallet-service or gateway endpoint.
  • AUTH_URL: Should point to the budolID service.

Deployment Procedures

  1. Navigate to budolPayMobile directory.
  2. Run flutter clean and flutter pub get to ensure fresh dependencies.
  3. Build the application: flutter build apk --release.
  4. The generated installer is located at android-installer/budolPay 1.0.5.apk.

Risk & Mitigation

  • Risk: Session timeout during the scanning process.
    Mitigation: Implemented clearer "Session expired" messaging to guide the user back to login rather than showing generic errors.
  • Risk: UI clipping on smaller devices.
    Mitigation: Added SafeArea and converted critical text links into high-visibility buttons.

Recommendation for Future Enhancement

Enable server-side session heartbeats or longer-lived JWT tokens for internal testing to reduce the frequency of re-logins during manual QA cycles.
