Troubleshooting Manual: Connection & Server Errors

Version: 2026-01-01_v39

Common Error: ERR_CONNECTION_REFUSED

This error typically occurs when the browser tries to connect to a service (like budolID on port 8000) that is not currently running or is blocked.

Scenario 1: Ecosystem Services Stopped

The central runner (start-ecosystem.js) may have crashed or was not started after a system reboot.

Fix: Restart the Ecosystem

Run the following command in your terminal from the root directory:

node start-ecosystem.js

This will automatically start budolID (8000), budolShap (3001), and budolPay (3000).

Scenario 2: Port Conflict (EADDRINUSE)

Another process is already using the required port, preventing the service from starting.

Fix: Identify and Kill Conflicting Processes

1. Find the process using the port (e.g., 8000):

netstat -ano | findstr :8000

2. Kill the process using its PID (found in the last column of netstat output):

taskkill /F /PID <PID>

3. Restart the ecosystem.

LAN IP Access

All services are configured to bind to 0.0.0.0, allowing access via your local IP (e.g., 192.168.1.24). If access fails but localhost works, check your firewall settings.
