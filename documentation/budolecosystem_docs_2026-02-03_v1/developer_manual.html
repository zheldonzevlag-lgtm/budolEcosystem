Architecture Overview

The BudolPay Ecosystem uses a microservices architecture. The KYC flow involves the Mobile App, API Gateway, and the Verification Service.

Rationale: Decoupling verification logic allows for independent scaling and easier compliance auditing for PII (Personally Identifiable Information).

KYC Verification Flow Integration

The mobile app submits documents to the /verification/upload endpoint. We aliased the backend /verify endpoint to support this.

Backend Alignment (Verification Service)

Modified the verification service to accept both /verify and /upload routes and handle multi-part file uploads.

Open verification-service/index.js 
app.post(['/verify', '/upload'], upload.single('document'), async (req, res) => {
  // Logic to handle document upload and user status update

  // req.file contains the uploaded image

Admin Panel Document Resolution

Updated the Admin Dashboard to resolve relative URLs for documents stored on the verification service via the API Gateway.

Open apps/admin/app/users/page.tsx 
const getDocUrl = (doc: any) => {
  if (doc.remoteUrl) {
    const GATEWAY_URL = process.env.NEXT_PUBLIC_GATEWAY_URL || "http://localhost:8080";
    return `${GATEWAY_URL}${doc.remoteUrl.startsWith('/') ? '' : '/'}${doc.remoteUrl}`;
  }
  // Fallback to blobData...
};

API Contracts

POST /verification/upload

  • Request: Multipart Form Data
  • Fields: userId, type (SELFIE/ID_DOCUMENT), documentType, faceTemplate (optional)
  • File: document (JPEG/PNG)
  • Response: 201 Created with { success: true, status: 'VERIFIED'|'PENDING' }

Security & Compliance

All KYC status changes are logged in the AuditLog table with PCI DSS 10.2.2 and BSP Circular 808 metadata.
