Database Schema Updates

The User and VerificationDocument models were updated to include faceTemplate fields to support biometric verification.

File: schema.prisma

model User { ... faceTemplate String? ... } model VerificationDocument { ... faceTemplate String? ... }

Technical Explanation: These fields are used to store hashed biometric data for AI-driven face verification during KYC. The columns were added manually via SQL to avoid data loss from a full migration reset.

API Gateway Implementation

The API Gateway serves as the central entry point and logging hub.

File: api-gateway/index.js

// Proxy configuration for microservices const proxies = [ { path: '/auth', target: process.env.AUTH_SERVICE_URL }, { path: '/wallet', target: process.env.WALLET_SERVICE_URL }, ... ];

Rationale: Centralizing authentication and RBAC at the gateway level ensures compliance with BSP Circular 808 standards for unauthorized access logging.
