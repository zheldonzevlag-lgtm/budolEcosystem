LAN Orchestration & Central Management

This document details the recent updates to the budolEcosystem designed to simplify local development, enable multi-device LAN testing, and improve the SSO user experience across applications.

Key Update: You can now start all ecosystem services with a single command: npm start from the root directory.

1. Developer Manual: Centralized Orchestration

1.1 Central Runner Script

A new script start-ecosystem.js has been added to the project root. This script uses child_process to spawn and manage all core services simultaneously with explicit port assignments to prevent conflicts.

  • budolID: Port 8000 (SSO Provider)
  • budolShap: Port 3001 (Main Application)
  • budolPay: Port 3000 (Admin Portal)

The runner script automatically sets the PORT environment variable for each application, ensuring they start on their designated ports even if default configurations differ.

1.2 LAN Communication Fixes

To support LAN access, internal service-to-service communication has been updated. Previously, apps like budolPay would attempt to verify tokens by calling localhost:8000. This fails when accessed from another device because "localhost" refers to the device browsing the app, not the server.

Code Change in budolPay (callback/route.ts):

// Updated to use Host IP for service-to-service verification
const SSO_VERIFY_URL = 'http://192.168.1.24:8000/api/verify-token';

2. System Admin Manual: LAN Access & Configuration

2.1 Binding to 0.0.0.0

All services are now configured to bind to 0.0.0.0. This allows them to accept connections from any IP address on the network, not just the local loopback (127.0.0.1).

  • budolID: Updated index.js to app.listen(PORT, '0.0.0.0', ...).
  • budolShap: Updated package.json scripts to include -H 0.0.0.0.
Browser Redirection Tip: While the servers bind to 0.0.0.0, you should access them using the host IP (192.168.1.24) or localhost. Navigating directly to http://0.0.0.0:3001 in some browsers may cause redirection issues.

2.2 Environment Variable Requirements

For full LAN functionality, ensure your .env files use the host IP for public-facing URLs:

NEXT_PUBLIC_SSO_URL=http://192.168.1.24:8000
NEXT_PUBLIC_APP_URL=http://192.168.1.24:3001

3. User Manual: Improved SSO Experience

3.1 Cross-App Login

Users can now use their budolPay credentials (e.g., admin@budolpay.com) to log into budolShap. This is powered by the centralized budolID service.

3.2 Enhanced Login Interface

The budolShap login modal now includes a prominent "Login with budolID" button. To guide users:

  • If a user enters an ecosystem email (ending in @budolpay.com or @budolid.com) in the standard login form, the system will detect it.
  • An error message will appear: "Ecosystem account detected. Please use the 'Login with budolID' button below."
  • The SSO button will automatically highlight to direct the user's attention.
Administrator Access: Use admin@budolpay.com and admin123 for full access across all ecosystem platforms.
