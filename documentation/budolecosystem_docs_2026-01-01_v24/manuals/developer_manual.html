Developer Manual Update (v24)

Topic: Reverting SSO Redirection to Localhost

Technical Implementation Details

This update reverts the ecosystem from LAN IP-based authentication to Localhost-based authentication to ensure seamless development on the host machine.

1. budolPay Admin Callback Fix

Modified budolpay-0.1.0/apps/admin/app/api/auth/callback/route.ts to use localhost as the default fallback for SSO_URL.

// Before
const ssoUrl = process.env.SSO_URL || 'http://192.168.1.24:8000';

// After
const ssoUrl = process.env.SSO_URL || 'http://localhost:8000';

2. Login Page Redirection Logic

Hardcoded URLs were reverted from dynamic IP detection back to explicit localhost to prevent jumping between hostnames during the SSO handshake.

  • budolPay Admin: apps/admin/app/login/page.tsx
  • budolShap: app/(public)/login/page.jsx

3. Architectural Decisions

The decision to revert to localhost was made to simplify the developer experience on the primary machine, as the previous LAN IP configuration introduced unexpected redirects to 192.168.1.24 even when accessing via localhost.

Impact Analysis

  • SSO Flow: Restored to 100% localhost-compatible flow.
  • Cross-Origin: CORS issues related to IP vs Localhost hostname mismatch are resolved.
