System Administration Guide

KYC Dashboard Management

The KYC Dashboard is the primary interface for processing user identities. In v1.6.0, the default view has been optimized.

1. Accessing All Applications

By default, the dashboard now shows All Applications. This allows admins to see the full funnel of user onboarding at a glance.

2. Filtering by Status

Use the dropdown menu to filter between Pending, Verified, and Rejected. The backend [route.js](file:///d:/IT Projects/budolEcosystem/budolshap-0.1.0/app/api/admin/kyc/route.js) handles these filters dynamically.

🤖 AI Security Analyst

The AI Security Analyst is an autonomous layer that monitors the Forensic Audit Trail. It is designed to assist admins in identifying complex security patterns.

Using AI Insights

Click the AI Security Insights button on the Security page. The system will analyze the last 50 actions and provide a summary of the current security posture.

Trend Visualization & Interaction

The Threat Mitigation Trend chart provides a daily view of security activity. Hover your mouse over any bar in the chart to see the exact number of Blocked and Threat events for that day.

Why we use AI here:

  • Error Reduction: Humans often miss subtle patterns in long log files. AI provides a consistent second pair of eyes.
  • Regulatory Speed: BSP often requires rapid responses to security incidents. AI summarizes the situation in seconds, allowing admins to act faster.

Compliance Monitoring

Admins must monitor the Compliance Shield daily to ensure regulatory health.

  • SYNCING Status: If BIR shows "SYNCING", it means transactions from the last 10 minutes are being anchored. No action required unless it persists for >30 mins.
  • WARNING Status: If BSP shows "WARNING", it means no admin actions were logged in 24h. Verify if system maintenance occurred.
