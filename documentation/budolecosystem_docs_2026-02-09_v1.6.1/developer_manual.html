📡 Real-time Telemetry System

The Budol Ecosystem employs a proactive "Watchtower" telemetry system. Unlike traditional static dashboards, our data is pulled directly from service heartbeats and database integrity checks.

Data Rationale

To meet institutional standards (BSP, BIR, NPC), we track metrics that reflect the active security posture of the platform rather than just system uptime.

🛡️ Compliance Metrics Strategy

Each metric in the Compliance Shield corresponds to a specific regulatory requirement in the Philippines.

Compliance Standard Technical Metric Implementation Strategy
PCI DSS v4.0 Gateway Heartbeat Real-time fetch to BudolPay API (Port 8080) to verify encrypted tunnels.
BSP Circular 808 Audit Trail Density Database count of AuditLog entries in the last 24h to ensure non-repudiation.
BIR Tax Integration Sync Queue Latency Checks for COMPLETED orders in the last 10m to trigger CAS/EIS e-invoicing.
NPC Data Privacy TLS/SSL Integrity Inspection of DATABASE_URL and environment headers for transport-layer security.

🤖 AI Security Orchestration

In v1.6.0, we introduced the AI Security Analyst. This layer sits above the raw telemetry to provide generative insights and anomaly detection.

Why AI for Compliance?

  • Cognitive Offloading: Philippine fintech regulations (BSP/BIR) generate massive volumes of logs. AI reduces "alert fatigue" by summarizing normal behavior and highlighting outliers.
  • Proactive Hardening: While traditional code is reactive, AI models can predict potential attack vectors based on the velocity and variety of audit logs.
  • Semantic Auditing: AI understands the context of an action (e.g., distinguishing a legitimate bulk KYC approval from a database injection attempt).

Implementation Architecture

The AI Analyst uses the runAiAnalysis function to process live database snapshots.

Open File 
const runAiAnalysis = async () => {
    // 1. Snapshot live AuditLogs
    const logSummary = logs.slice(0, 50).map(l => l.action);
    // 2. Process via Security v4.0 Model
    const analysis = await aiEngine.analyze(logSummary);
    // 3. Output human-readable mitigation steps
    setAiSummary(analysis);
};

KYC Approval Infrastructure

The KYC system uses RBAC (Role-Based Access Control) to ensure only authorized leads can approve sensitive identity documents.

Open File 
export async function GET(request) {
    const { authorized } = await requireAdmin(request, PERMISSIONS.USERS_KYC_APPROVE);
    if (!authorized) return errorResponse;

    const status = searchParams.get('status') || 'ALL';
    const users = await prisma.user.findMany({
        where: { kycStatus: status === 'ALL' ? undefined : status }
    });
    return NextResponse.json(users);
}
