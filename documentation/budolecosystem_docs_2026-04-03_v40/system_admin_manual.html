System Admin Manual: KYC Infrastructure on Vercel

This document details the configuration and maintenance steps for the verification-service deployed on Vercel.

Deployment Configuration

MetricConfiguration Value
PlatformVercel Serverless (Node.js 20.x)
Production URLverification-service-six.vercel.app
Proxy Entrybudolpay.vercel.app/api/verification/*
DatabaseVercel Postgres (Prisma Accelerate Connection)
Runtime Schemabudolpay Isolated

Environment Variables

The service requires the following production secrets:

  • DATABASE_URL: postgres://...&schema=budolpay (Prisma Postgres Platform)
  • DIRECT_URL: ...&schema=budolpay (Direct Driver Access)
  • PRISMA_ENGINES_CACHE_DIR: /tmp/prisma_cache (Necessary for Vercel deployment stability)

Maintenance Procedures

Redeployment

To apply schema changes, run:

vercel --prod --yes

Health Checks

Monitor the /api/verification/upload endpoint. A 404 indicates a failure in the Admin app proxy routing. A 500 with "Prisma" in the logs indicate a database connection schema mismatch.
