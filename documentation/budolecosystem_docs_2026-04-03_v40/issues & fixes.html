Issues & Fixes Log (v40)

Issue: 404 Upload Error

Mobile app requests were hitting the Next.js admin app root which lacked the /api/verification endpoint, resulting in a 404.

Fix: Server-Side Proxy

Created a Next.js catch-all API route at apps/admin/app/api/verification/[...path] to proxy all traffic to the verification microservice.

Issue: Prisma Client P1001

Prisma could not initialize the engine in serverless environment due to missing cache or incorrect direct driver settings.

Fix: Engine Caching & Version Sync

Aligned Prisma to v5.14.0 and added PRISMA_ENGINES_CACHE_DIR=/tmp/prisma_cache to the Vercel build script.

Issue: VerificationDocument Table Not Found

Prisma defaulted to public schema, while data resided in the budolpay schema.

Fix: Schema Isolation Logic

Updated application code to force schema=budolpay parameter in the connection string at client instantiation.
