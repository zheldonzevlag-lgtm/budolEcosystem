Mitigation Plan (v40)

Documenting strategies implemented to ensure production stability on Vercel.

Database Schema Conflict Mitigation

Action: Implemented runtime schema isolation. The Prisma client now explicitly appends ?schema=budolpay to connection strings, preventing shared-database collisions with budolShap.

Filesystem Read-Only Mitigation

Action: Reconfigured multer to use /tmp/ as the storage directory for all incoming KYC document uploads, bypassing the read-only restrictions of the Vercel serverless environment.

Networking (404/405) Mitigation

Action: Implemented a server-side proxy route in the Next.js admin app to correctly forward legacy mobile app traffic to the new verification-service deployment alias.
