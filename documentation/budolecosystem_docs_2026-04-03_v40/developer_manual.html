Developer Manual: KYC Stabilization & Vercel Migration

Purpose: To document the technical architecture and implementation of the self-contained KYC verification service deployed on Vercel.

Architecture Overview

The KYC extraction logic was decoupled from the monorepo to allow for serverless deployment on Vercel. This bypassed workspace dependency resolution issues and enabled high-availability document processing.

Key Components

  • Serverless Function: Entry point at api/index.js.
  • Prisma Client: Direct integration with Vercel Postgres using the budolpay schema.
  • Multer Storage: Utilizes /tmp for transient multipart storage.

Code Implementation Details

Schema Isolation Logic

To prevent targeting the public schema, a custom getIsolatingUrl() function was implemented to force schema=budolpay at runtime.


// Ref: services/verification-service/index.js
const getIsolatingUrl = () => {
  const url = process.env.DATABASE_URL;
  const params = new URLSearchParams(url.split('?')[1] || '');
  params.set('schema', 'budolpay');
  return `${url.split('?')[0]}?${params.toString()}`;
};

API Reference

POST /api/verification/upload

Accepts multipart form data for KYC documents.

  • userId: Valid production UUID from budolpay.User table.
  • type: e.g., ID_DOCUMENT.
  • documentType: e.g., PASSPORT, DRIVERS_LICENSE.
  • document: Binary file.
Compliance: This implementation adheres to NPC Data Privacy standards by ensuring all PII processed in /tmp is immediately ingested into the encrypted database and purged from serverless memory.
