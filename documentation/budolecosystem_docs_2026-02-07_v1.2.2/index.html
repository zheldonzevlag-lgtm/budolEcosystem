Release v1.2.2

UX Enhancement: Visual Feedback for Selfie Capture

Overview

This release introduces visual feedback during the selfie capture process. Previously, users were not informed that the system was processing the image upload after clicking the capture button, leading to potential confusion or double-clicks.

Changes Applied

  • Capture Button Spinner: The capture button now transforms into a loading spinner (RefreshCw) when the upload is in progress.
  • Asynchronous Flow Optimization: Modified captureSelfie to maintain the camera view and capture button visibility until the upload is successfully completed.
  • Button State Management: The capture button is now disabled during the upload process to prevent duplicate requests.
  • Improved User Guidance: Updated button titles to provide context-aware tooltips (e.g., "Processing..." during upload).

Task List

  • ✅ Add loading spinner to capture button during selfie upload
  • ✅ Ensure camera view stays active until upload finishes

  • ✅ Update documentation to v1.2.2

Technical Implementation

Modified the captureSelfie function in AuthForm.jsx to delay stopCamera() and setSelfieCaptured(true) until the fetch request to /api/upload returns successfully.


// Simplified logic
const captureSelfie = async () => {
    // ... capture image ...
    setIsUploading(true);
    try {
        const response = await fetch('/api/upload', { ... });
        if (response.ok) {
            setSelfieCaptured(true);
            stopCamera();
        }
    } finally {
        setIsUploading(false);
    }
}

Compliance

This enhancement aligns with UX Best Practices and Accessibility Standards by providing clear, real-time feedback for long-running operations.
