System Admin Manual: Infrastructure & Ops

1. Infrastructure Changes

The real-time synchronization layer has been upgraded. API Gateway now handles admin-specific socket rooms for global monitoring.

Service URLs & Ports

  • API Gateway: http://localhost:8080 (WebSocket Entry Point)
  • Transaction Service: http://localhost:8003 (Business Logic)
  • Wallet Service: http://localhost:8002 (Balance Updates)

2. Environment Variables

Ensure the following environment variables are set for the Admin Dashboard and Services:

# Admin App (.env.local)
NEXT_PUBLIC_GATEWAY_URL=http://localhost:8080

# Transaction Service (.env)
GATEWAY_URL=http://localhost:8080
WALLET_SERVICE_URL=http://localhost:8002

3. Database Schema Impact

No new migrations were required, but the following tables are now actively populated with ledger-compliant data:

  • Transaction: Records all Cash In, Transfers, and status changes.
  • LedgerEntry: Double-entry accounting records for every financial move.
  • Wallet: Real-time balance updates for users.

4. Operational Runbook

Monitoring Transactions

  1. Access the Admin Dashboard at /transactions.
  2. Verify that the "Connected" status appears in console logs (Socket.io).
  3. Perform a test Cash In from the mobile app to verify real-time arrival.

Troubleshooting Real-time Sync

  • If transactions don't appear: Check if api-gateway is running and reachable on port 8080.
  • Check transaction-service logs for [Transaction] Admin notification sent to Gateway.
