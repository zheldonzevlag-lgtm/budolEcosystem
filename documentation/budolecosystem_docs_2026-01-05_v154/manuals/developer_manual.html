Developer Manual: Transaction & Real-time Sync

1. Purpose

This implementation ensures that financial transactions (Cash In and P2P Transfers) are fully functional, recorded in the ledger, and synced in real-time across the Android Mobile App and the Admin Dashboard.

2. Technical Implementation Details

Backend: Transaction Service

Modified services/transaction-service/index.js

  • Added /cash-in endpoint to handle external fund deposits.
  • Integrated Ledger Entry creation for both Cash In and P2P Transfers to ensure financial compliance.
  • Added notifyAdmin helper to emit new_transaction events to the Admin Dashboard via the API Gateway.

Backend: API Gateway

Modified services/api-gateway/index.js

  • Updated Socket.io logic to support an admin room for global monitoring.
  • Enhanced /internal/notify endpoint to support isAdmin flag for targeted broadcasting.

Mobile: Flutter API Service

Modified lib/services/api_service.dart

  • Implemented cashIn(amount, provider) method.
  • Implemented transfer(recipientEmail, amount, description) with built-in recipient lookup.

Admin: Transactions Page

Modified apps/admin/app/transactions/page.tsx

  • Integrated socket.io-client for real-time transaction monitoring.
  • Added listeners for new_transaction to update the UI instantly without refresh.

3. Code Examples

Admin Real-time listener

useEffect(() => {
    const socket = io(GATEWAY_URL);
    socket.on("connect", () => socket.emit("join", "admin"));
    socket.on("new_transaction", (newTx) => {
        setTransactions((prev) => [newTx, ...prev]);
    });
    return () => socket.disconnect();
}, []);

4. Issues & Fixes

  • Issue: Recipient lookup was missing for transfers in mobile app.
    Fix: Created /api/auth/user/find endpoint in budolshap to fetch user details by email.
  • Issue: Admin dashboard required manual refresh to see new transactions.
    Fix: Implemented Socket.io room admin and real-time event emission from backend services.

5. Risk & Mitigation

  • Risk: High volume of real-time events could overwhelm the Admin UI.
    Mitigation: Implement virtualization for the transaction list and throttling for socket events if needed.
