Issues & Fixes - v455

Issue: Marketing Ads Bypassed in Sequence

Symptoms: The app transitions from Splash (Image 2) directly to Login (Image 3), skipping Marketing Ads.

Potential Root Cause: hasSeenAds flag is likely set to true in the user's local storage from a previous session, or there is a build synchronization issue where the latest diagnostic tools are not yet visible.

Fix: Navigation Transparency & Build Verification

  • Status Indicators: Added a real-time status message to the Splash Screen (e.g., "Navigating to Ads...") to confirm the intended target.
  • Version Increment (v1.3.7): Updated the internal and displayed version number to verify the user is running the latest build.
  • State Logging: Added explicit print statements to verify the value of hasSeenAds upon app startup.
  • Web Resource Diagnostics: Added an error listener to index.html to catch and display failed resource loads (MIME type errors).
  • Booting Fallback: Added a "Booting BudolPay Engine..." UI in main.dart to distinguish between engine initialization and widget rendering.
