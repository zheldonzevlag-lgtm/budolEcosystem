Developer Manual - BudolEcosystem v5.0.3

Release Date: 2026-03-02

Status: Validation & ENV Fix

v5.0.3 Bug Fixes Task List

✔ Fix PrismaClientValidationError: Unknown argument failureReason in checkoutService.js
✔ Re-apply budolpay schema migration (?schema=budolpay) to all .env files
✔ Corrected Checkout model field usage (moved failureReason to metadata)

Purpose

This hotfix addresses a PrismaClientValidationError encountered during checkout failure handling and ensures environment variable consistency for the budolpay schema isolation.

Technical Explanation

1. Prisma Schema Validation Fix

The Checkout model in budolShap does not have a native failureReason field. Attempting to update this field directly caused a Prisma validation error. This release moves the failure reason into the metadata JSON field.

File: lib/services/checkoutService.js

2. Environment Consistency

Re-enforced the logical data separation by ensuring all budolPay services point to the budolpay schema. This prevents model collisions with the budolShap public schema.

Affected Files: All .env files in budolpay-0.1.0/services/* and budolpay-0.1.0/packages/database/.

Changes Summary

  • Updated markCheckoutAsFailed in checkoutService.js to store reason in metadata.
  • Synchronized 6+ .env files to use ?schema=budolpay.

Compliance & Risk

Compliance: PCI DSS Requirement 3 (Protect stored cardholder data) - Logical separation via schemas ensures financial data is isolated from application data.

Risk: Minimal. Data integrity is preserved.
