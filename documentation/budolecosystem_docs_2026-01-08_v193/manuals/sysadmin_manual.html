SysAdmin Manual: Release Operations

1. APK Versioning & Artifacts

This release generates version 1.2.6+1 of the budolPay Mobile application.

Release Artifact: android-installer/budolPay 1.2.6.apk
Build Type: Release (Signed with debug keys for testing)
Size: ~126.5 MB

2. Build Procedure

To reproduce the build from source, ensure you are in the budolPayMobile directory and run:

flutter build apk --release --android-skip-build-dependency-validation

This command bypasses full dependency validation to speed up the assembly process on limited-resource environments while still producing a valid production binary.

3. Verification Protocol

Before distribution, administrators must verify the following:

  • Hash Check: Verify the APK size matches the expected ~126MB.
  • Installation: Install on a physical device or emulator to ensure the splash screen and login flow load correctly.
  • Network: Ensure the app can connect to the local API Gateway (default: 192.168.1.24:8080) or auto-discover the gateway if configured.

4. Rollback Strategy

If issues are encountered with v1.2.6, revert to the previous stable release:

android-installer/budolPay 1.2.5.apk
