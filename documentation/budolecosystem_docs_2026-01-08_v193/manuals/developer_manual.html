Developer Manual: Mobile Discovery & Testing

1. mDNS Discovery Refactor

Refactor

The DiscoveryService was refactored to handle stream-based lookups more reliably. Previously, the await for loop could hang indefinitely if the mDNS client didn't receive a response, especially in environments where mDNS is restricted (like CI or certain Windows configurations).

Key Changes:

  • Implemented Completer and Timer for explicit discovery timeouts.
  • Replaced await for with stream listen to allow for manual cancellation of subscriptions.
  • Added a 2-second timeout window for gateway discovery.
// New Stream-based implementation
final subscription = ptrStream.listen((ptr) async {
  // ... nested lookups ...
  if (!completer.isCompleted) {
    completer.complete(discoveredHost);
  }
});

Timer(const Duration(seconds: 2), () {
  if (!completer.isCompleted) completer.complete(null);
});

2. Test Environment Isolation

Testing

To prevent unit tests from hanging due to network discovery attempts, ApiService now detects the test environment and bypasses auto-discovery.

// lib/services/api_service.dart
if (_customHost == null && !kIsWeb && !Platform.environment.containsKey('FLUTTER_TEST')) {
  // Only attempt discovery if NOT in a test environment
  final discoveredHost = await _discoveryService.discoverGateway();
}

4. Impact Analysis

These changes have zero impact on existing features but significantly improve the reliability of the development workflow and automated testing pipelines. The mobile app now fails gracefully and quickly when the backend is unreachable.
