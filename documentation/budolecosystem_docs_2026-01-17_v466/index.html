Release v466: Web Blank Screen Fix LATEST

Purpose

This release addresses a critical UI bug where the Flutter Web application would display a blank screen after the splash screen loads, preventing users from seeing marketing ads or progressing to the login screen.

Task List

  • Analyze Marketing Ads implementation and Splash Screen transition logic
  • Identify why Web app is stuck on blank screen (Layout constraints)
  • Fix the blank screen issue (Positioned.fill in SessionOverlay)
  • Add debug logging for Web layout verification
  • Verify the fix with automated test script
  • Create comprehensive documentation (v466)
  • Update Knowledge Base and version history

Technical Summary

The issue was caused by the SessionOverlay widget using a Listener that didn't enforce full-screen constraints on its child in certain Web rendering modes. By wrapping the Listener in a Positioned.fill, we ensure that the application content occupies the entire viewport, preventing zero-size rendering that leads to a blank screen.

Compliance

This fix ensures availability and usability of the platform, adhering to NPC (National Privacy Commission) accessibility guidelines and Cybersecurity Law regarding system reliability.
