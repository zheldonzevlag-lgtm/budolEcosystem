Layout Constraint Fix in SessionOverlay

The primary fix involves ensuring the application content (child) of the SessionOverlay is properly sized on Web. We use Positioned.fill within a Stack to force the Listener to take up all available space.

File: session_overlay.dart

Positioned.fill(
  child: Listener(
    behavior: HitTestBehavior.translucent,
    onPointerDown: (_) {
      if (kDebugMode) print('SessionOverlay: User interaction detected. Resetting timer.');
      sessionService.resetInactivityTimer();
    },
    child: child,
  ),
),

Rationale: On Flutter Web, specifically with certain CanvasKit/HTML renderers, widgets inside a Stack without explicit positioning may collapse to zero size if their parent doesn't provide strict constraints.
