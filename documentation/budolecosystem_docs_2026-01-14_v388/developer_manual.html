Developer Manual - v388

TFLite Shape Alignment Fix

The core issue in v387 was a dimension mismatch between the facenet.tflite model and the FaceEmbeddingService output buffer. The model returned a 512-dimensional vector, but the app was expecting 128 dimensions, causing an Invalid argument(s): Output object shape mismatch error.

Modified Code Reference

The output buffer in face_embedding_service.dart was updated to allocate 512 floats:

// lib/services/face_embedding_service.dart

// Before:
// var output = List<List<double>>.generate(1, (i) => List<double>.filled(128, 0.0));

// After:
var output = List<List<double>>.generate(1, (i) => List<double>.filled(512, 0.0));

// The inference now runs successfully:
_interpreter!.run(inputReshaped, output);

Data Persistence Compatibility

The biometric_service.dart handles embedding storage using a comma-separated string format in FlutterSecureStorage. This implementation is inherently flexible and supports the transition from 128 to 512 dimensions without schema changes.

// lib/services/biometric_service.dart
Future<void> storeFaceTemplate(List<double> template) async {
  await _storage.write(key: _faceTemplateKey, value: template.join(','));
}

Compliance Verification

  • PCI DSS: Biometric templates remain encrypted in Secure Storage.
  • Cybersecurity Law: Model integrity is maintained via SHA256 validation before execution.
