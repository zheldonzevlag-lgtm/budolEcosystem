Future Recommendations

Database Migration for Legacy Phone Numbers

Rationale: While the new normalization logic handles user input, legacy data in the database remains in various formats.

Action: Run a one-time migration script to normalize all existing phone fields in the User and Store tables to E.164 format.

International Phone Support

Rationale: The current logic is strictly optimized for Philippine (+63) numbers.

Action: Integrate a library like libphonenumber-js to support global phone normalization as the ecosystem expands.

Automated UI Regression Testing

Rationale: Manual verification of breadcrumbs and alignment is prone to human error.

Action: Implement Playwright or Cypress tests to automatically verify breadcrumb paths and UI alignment across different screen sizes.
