Recommendations for Verification Synchronization

1. Centralized Identity Webhooks

Purpose: Implement webhooks in BudolID that notify other services (BudolPay, BudolShap) whenever a user's isVerified status changes.

Reason: Eliminates the need for manual script-based synchronization and ensures real-time updates across the ecosystem.

2. Unified KYC Tier Mapping

Purpose: Define a standard set of KYC tiers (e.g., BASIC, SEMI, FULL) in BudolID and have all services inherit these tiers.

Reason: Simplifies business logic and ensures that a "Fully Verified" user in one app is recognized as such in all others.
