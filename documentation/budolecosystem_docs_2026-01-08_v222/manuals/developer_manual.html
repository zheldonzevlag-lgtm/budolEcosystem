Developer Manual: Identity Verification Implementation

1. BudolPay Upgrade Logic

The verification status in BudolPay is governed by three key fields in the User model:

  • kycStatus: Set to VERIFIED
  • kycTier: Set to FULLY_VERIFIED
  • isFaceVerified: Set to true
// upgrade-jon-pay.js
const updatedUser = await prisma.user.update({
  where: { email: 'reynaldomgalvez@gmail.com' },
  data: {
    kycStatus: 'VERIFIED',
    kycTier: 'FULLY_VERIFIED',
    isFaceVerified: true
  }
});

2. BudolID Sync

To ensure consistent SSO experience, the isVerified flag in the budolID database must match the KYC state in budolPay.

// verify-jon-id.js
const updatedUser = await prisma.user.update({
  where: { email: 'reynaldomgalvez@gmail.com' },
  data: { isVerified: true }
});
