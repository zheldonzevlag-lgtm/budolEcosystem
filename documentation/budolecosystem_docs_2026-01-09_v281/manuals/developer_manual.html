Developer Manual - v281

1. Purpose & Overview

The primary goal of this update was to transition the ecosystem from a localhost-only development environment to a fully LAN-accessible platform with automated service discovery. This involved significant refactoring of networking logic and database client management.

2. mDNS / Bonjour Implementation

To enable service discovery without manual IP entry, we integrated bonjour-service into the central ecosystem runner (start-ecosystem.js).

Code Implementation:

const bonjour = require('bonjour-service').default();

// Inside service start logic
const service = bonjour.publish({
    name: app.name,
    type: 'http',
    protocol: 'tcp',
    port: app.port,
    txt: { 
        version: '1.0.0', 
        ip: localIP
    }
});

This allows other devices on the network to discover the services via their network names (e.g., budolID.local).

3. Prisma Database Centralization

To reduce code duplication and ensure schema consistency, the Prisma client was centralized into the @budolpay/database package.

Key Changes:

  • Moved prisma-client initialization to budolpay-0.1.0/packages/database.
  • Created a root db:generate script to update all client instances.
  • Automated prisma generate in start-ecosystem.js before services boot.
Issue: Services failing because Prisma client was not generated in their local context.
Fix: Added npm run db:generate to the startup sequence.
