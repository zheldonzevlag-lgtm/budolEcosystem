System Admin Manual - v281

1. LAN Configuration

The ecosystem is now configured to listen on all available network interfaces (0.0.0.0). This allows any computer on the same local network to access the services.

Action Required: Ensure that your host machine's firewall allows incoming connections on ports 3000-3010, 8000-8010, and 8080.

2. Network Synchronization

A new utility script sync-network.js has been enhanced to automatically detect the host's LAN IP and propagate it to all .env files across the ecosystem.

Usage:

node sync-network.js

This script updates LOCAL_IP in all service directories, ensuring that the mobile app and other services know where to find each other.

3. Database Operations

Prisma migrations and client generation are now centralized. To sync the database schema across the ecosystem, use the following root command:

npm run db:generate

This will update the Prisma client in the shared @budolpay/database package, which is used by all services.

4. mDNS Monitoring

You can verify that services are being advertised on the network using standard mDNS discovery tools like Bonjour Browser or command-line tools:

dns-sd -B _http._tcp .

You should see entries for budolID, budolPay, budolAccounting, etc.

5. Risk & Mitigation

  • Risk: Open ports on LAN might expose services to unauthorized local users.
    Mitigation: Ensure the network is trusted (private Wi-Fi/Ethernet) and use strong passwords for the database.
  • Risk: IP address changes when switching networks.
    Mitigation: Run node sync-network.js whenever the host moves to a different network.
