v22 System Admin Manual: Vercel Mirror Configuration

Vercel Environment Setup

To mirror the local environment, the following Environment Variables must be set in the Vercel Dashboard for budolShap:

1. Database Connection (13-Service Shared PG)

Use the Vercel Postgres connection string with the schema parameter:

DATABASE_URL="postgres://postgres:password@host/db?schema=public&sslmode=require" DIRECT_URL="postgres://postgres:password@host/db?sslmode=require"

2. Service URL Mapping (DuckDNS Mirrors)

Critical: All internal URLs must point to the public mirrors verified on 2026-03-24.
VariableProduction Mirror
SSO_URLhttps://budolid.duckdns.org
GATEWAY_URLhttps://budolpay.duckdns.org
INTERNAL_WS_URLhttps://budolws.duckdns.org
NEXT_PUBLIC_SOCKET_URLhttps://budolws.duckdns.org

Deployment Automation

Admins can trigger a synchronized mirror deployment using the following script:

.\scripts\test_scripts\deploy-vercel.ps1

This script performs a pre-flight check, pulls the Vercel env, and pushes any local Prisma schema changes via prisma db push before triggering the vercel build.
