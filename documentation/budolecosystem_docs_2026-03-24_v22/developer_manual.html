v22 Developer Manual: Vercel Mirroring Strategy

Architecture: 13-Service Shared Shared Backbone

The system refactors the local mDNS discovery into a public DNS discovery model compatible with Vercel Serverless. Each service is mapped to its public mirror.

Database Consolidation (Mirroring budolshap_1db)

Local dev uses a consolidated database budolshap_1db with multiple schemas. On Vercel, we mirror this exactly using a single PostgreSQL instance:

ServiceLogical SchemaVercel Connection
budolIDbudolid?schema=budolid
budolShappublic?schema=public
budolPaybudolpay?schema=budolpay
budolAccountingbudolaccounting?schema=budolaccounting

Code Modifications

1. Prisma Schema Updates

All schemas were updated to include directUrl for Vercel infrastructure compatibility:

datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")
  directUrl = env("DIRECT_URL")
}

2. Environment Management

The env-manager.cjs script was refactored to propagate DIRECT_URL across the entire 13-service ecosystem during npm run db:local or npm run db:vercel commands.

3. Vercel Configuration

Synchronized .env.vercel with the following verified DuckDNS endpoints:

  • SSO_URL=https://budolid.duckdns.org
  • GATEWAY_URL=https://budolpay.duckdns.org
  • INTERNAL_WS_URL=https://budolws.duckdns.org
