Technical Explanation: Forensic Audit Logs

The forensic audit trail utilizes the SECURITY_ prefix for all authentication-related events. This allows the Admin Dashboard to filter and present immutable logs specifically for security audits.

Code Implementation: Auth Service

File: auth-service/index.js

Added logic to capture identification and PIN verification attempts:

// Audit: Mobile Identification (Phase 1)
await createAuditLog(req, user.id, 'SECURITY_MOBILE_IDENTIFY_SUCCESS', {
    deviceId,
    isDeviceTrusted
}, 'Security', user.id);

// Audit: Mobile Identification Failure
await createAuditLog(req, null, 'SECURITY_MOBILE_IDENTIFY_FAILED', {
    phoneNumber,
    deviceId,
    reason: 'USER_NOT_FOUND'
}, 'Security');

// Audit: Mobile Logout (New)
app.post('/logout', async (req, res) => {
    const { userId } = req.body;
    if (userId) {
        await createAuditLog(req, userId, 'SECURITY_LOGOUT', {
            status: 'SUCCESS',
            timestamp: new Date().toISOString()
        }, 'Security', userId);
    }
    res.json({ message: 'Logged out successfully' });
});

Technical Explanation: Real-time Forensic Audit Trail

The system now supports real-time, provider-agnostic forensic audit trail updates. When a createAuditLog event occurs in the auth-service, it triggers a notification through the api-gateway, which broadcasts the event to the Admin Dashboard via either Socket.io or Pusher, depending on the system configuration.

Code Implementation: Provider Agnostic Notification

File: auth-service/index.js

// Notify Admin in Real-time (Provider Agnostic via Gateway)
await notifyAdmin('AUDIT_LOG_CREATED', log);

Code Implementation: Admin Dashboard Listener

File: admin/security/page.tsx

useEffect(() => {
    const initRealtime = async () => {
        await realtime.init();
        realtime.on("AUDIT_LOG_CREATED", (newLog) => {
            console.log("[Realtime] New audit log received:", newLog);
            setLogs(prevLogs => [newLog, ...prevLogs.slice(0, 49)]);
        });
    };
    initRealtime();
    return () => realtime.disconnect();
}, []);

Technical Explanation: Decimal Serialization

Next.js Server Components cannot pass raw Prisma Decimal objects to Client Components. We implemented explicit mapping to convert Decimal to number during data fetching.

Code Implementation: Admin Dashboard

File: admin/app/page.tsx

// Explicit Decimal to Number Mapping
const serializedTransactions = transactions.map(tx => ({
    ...tx,
    amount: Number(tx.amount)
}));
