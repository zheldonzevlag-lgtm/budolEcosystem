Mitigation Plan v407

R-407-01: Forensic Log Gaps

Mitigation Strategy: Implementation of mandatory audit hooks in the auth-service. Every authentication phase (Identify, PIN, Biometric) now triggers a createAuditLog call with the SECURITY_ prefix.

R-407-02: Decimal Serialization

Mitigation Strategy: Standardized data fetching patterns in Next.js Server Components. All Decimal fields are explicitly converted to number before being passed to client-side hooks.

R-407-03: Unauthorized Access

Mitigation Strategy: Multi-factor authentication (MFA) via OTP for all new/untrusted devices. Trusted device verification is required before allowing PIN or Biometric login.

R-407-05: Real-time Visibility Delay

Mitigation Strategy: Implemented provider-agnostic real-time event broadcasting using the API Gateway. Audit logs are now pushed to the Admin Dashboard immediately upon creation via Socket.io/Pusher.

R-407-06: Missing Logout Events

Mitigation Strategy: Added a dedicated /logout endpoint to the auth-service that records a SECURITY_LOGOUT audit log. The mobile app now explicitly notifies the backend on user logout.
