Developer Manual - v55

Purpose

To improve the user experience by ensuring that users are redirected to the login page when attempting to place an order or apply a coupon without being authenticated, and then redirected back to the cart upon successful login.

Implementation Details

Modified OrderSummary.jsx to use router.push('/login?redirect=/cart') instead of simply redirecting to the home page or showing a toast message without redirection.

Code Changes

In budolshap-0.1.0/components/OrderSummary.jsx:

// Before
if (!user) {
    toast.error("Please login first");
    router.push('/');
    return;
}

// After
if (!user) {
    toast.error("Please login first");
    router.push('/login?redirect=/cart');
    return;
}

Impact

  • Ensures users are prompted to login at the exact moment authentication is required.
  • Maintains session context by using the redirect query parameter.
  • Improves conversion rates by reducing friction in the checkout flow.

Recommendations

Consider implementing a global authentication guard for the checkout process to handle multiple entry points consistently.
