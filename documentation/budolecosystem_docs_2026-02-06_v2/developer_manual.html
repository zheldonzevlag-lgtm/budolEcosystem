Developer Manual

Version: BudolShap UI Optimization | Release Date: 2026-02-06

Purpose

To optimize the registration modal in BudolShap for maximum visibility and responsiveness across all screen sizes. This fix ensures that long registration forms, including the selfie capture and preview sections, are fully scrollable and correctly contained within the viewport, preventing UI breakage on mobile and small-screen devices.

Rationale

User registration is a critical conversion point. If the form is partially invisible or unscrollable on smaller screens, users cannot complete the sign-up process. Furthermore, for KYC (Know Your Customer) compliance, the selfie capture UI must be clear and accessible. Implementing responsive constraints and custom scrollbars ensures a seamless onboarding experience while maintaining brand consistency.

Technical Implementation

Modal Container Optimization

The LoginModal container was updated with Tailwind classes to handle small viewports. We added p-4 to the outer fixed container to ensure spacing around the modal on mobile, and max-h-[90vh] overflow-y-auto to the modal body itself.

Responsive Media Previews

In AuthForm.jsx, the camera video feed and selfie image preview were previously using fixed max-w values. These were updated to use responsive sm: prefixes to allow slightly larger previews on desktop while remaining compact on mobile.
