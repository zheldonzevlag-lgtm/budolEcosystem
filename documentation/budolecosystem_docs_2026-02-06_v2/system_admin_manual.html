Version: BudolShap UI Optimization | Date: 2026-02-06

UI Configuration & Performance

This release introduces responsive constraints for the BudolShap registration flow. System administrators should monitor user onboarding metrics to verify that these changes improve registration completion rates on mobile devices.

UI Component Constraint Description
Registration Modal max-h-[90vh] Ensures the modal never exceeds 90% of the viewport height, forcing scrollability for long forms.
Registration Modal overflow-y-auto Enables vertical scrolling within the modal container.
Selfie Preview sm:max-w-[320px] Responsive scaling for camera feed to ensure visibility on all screen widths.
Maintenance Note: The custom-scrollbar utility is defined in globals.css. Any changes to the scrollbar appearance should be made globally there to maintain ecosystem-wide UI consistency.

Onboarding Monitoring

Admins can verify the effectiveness of the UI fixes by monitoring:

  • Registration Drop-off Rate: Specifically looking for reductions in abandonment at the "Selfie Upload" step on mobile devices.
  • UI Rendering Logs: Check for any CSS injection errors or layout shifts in browser console logs during standard user testing.

Compliance Audit Requirements

Requirement: To maintain NPC (National Privacy Commission) compliance, ensure that all registration checkboxes (Terms of Service, Privacy Policy) are fully visible and clickable without obstruction from the new scrollable container.
