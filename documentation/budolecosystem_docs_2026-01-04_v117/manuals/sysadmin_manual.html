System Administrator Manual

Version: 1.0.1 (v63)
Date: 2026-01-04

1. Payment Gateway Service Configuration

The BudolPay integration relies on the Payment Gateway Service running locally or on a designated server.

Service Details

  • Port: 8004
  • Path: budolpay-0.1.0/services/payment-gateway-service
  • Command: node index.js
  • Health Check: GET http://localhost:8004/health

2. Environment Variables

Ensure the following environment variables are set in the main budolshap-0.1.0/.env file:

  • BUDOLPAY_API_KEY: Default is bs_key_2025. Must match the key configured in the gateway service.
  • PAYMONGO_SECRET_KEY: Required for PayMongo integrations.
  • SMTP_PASSWORD: Required for email notifications. Ensure this is valid to avoid "535 5.7.8 BadCredentials" errors.

3. Troubleshooting

Issue: Payment Status Not Updating

Symptoms: User scans QR code, pays, but the "Waiting for payment" modal never updates.

Fix:

  1. Check if the Payment Gateway Service is running on port 8004.
  2. Verify that the BudolPayAdapter is connecting to http://127.0.0.1:8004 (not localhost if IPv6 issues exist).
  3. Check the browser console for 404 errors on /api/paymongo/status.

Issue: SMTP Bad Credentials

Symptoms: "535 5.7.8 BadCredentials" in server logs.

Fix: Run the check-settings.cjs script to validate SMTP configuration. Note that for Gmail, you must use an App Password, not your regular password.
