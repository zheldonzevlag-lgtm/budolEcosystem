Mitigation Plan

M-001: ID Visibility

Risk: Reconciliation errors due to truncated IDs.
Mitigation: Remove all substring and truncate logic for critical identifiers (Transaction ID, Reference ID, User ID) across all platforms.

M-002: Version Control

Risk: Mobile app mismatch.
Mitigation: Implement a mandatory update check in the mobile app and display the version number clearly on the splash screen.

M-003: Data Privacy

Risk: Unnecessary exposure of internal IDs.
Mitigation: Ensure that full IDs are only visible to authenticated and authorized administrators. User-facing IDs should remain limited to Reference IDs.
