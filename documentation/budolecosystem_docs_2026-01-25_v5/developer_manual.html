Developer Manual

Technical Explanation

This release addresses UI truncation issues in both React (Next.js) and Flutter environments. The core issue was the use of .substring(0, 8)... for ID display, which hindered transaction reconciliation. In Flutter, the softWrap property was missing in the Text widget within the Expanded layout of the payment status screen.

Changes & Code References

1. budolPay Admin - Transactions Page

Removed truncation logic for Reference ID and Internal Transaction ID.

transactions/page.tsx 
// Before
<div className="text-[10px] text-slate-400 font-mono mt-0.5">{tx.id.substring(0, 8)}...</div>

// After
<div className="text-[10px] text-slate-400 font-mono mt-0.5">{tx.id}</div>

2. budolPay Admin - Users Page

Removed truncation logic for User IDs to ensure full traceability.

users/page.tsx 
// Before
<div className="text-xs text-slate-500 font-mono">{user.id.substring(0, 8)}...</div>

// After
<div className="text-xs text-slate-500 font-mono">{user.id}</div>

3. budolPay Mobile - Payment Status Screen

Enabled softWrap for transaction detail rows to handle long IDs.

payment_status_screen.dart 
// Updated _buildDetailRow to include softWrap: true
Expanded(
  child: Text(
    value,
    textAlign: TextAlign.right,
    softWrap: true, // Added to prevent truncation
    style: TextStyle(...)
  ),
)

4. budolShap - QR Modal Reordering

Reordered display to show Order ID before Reference ID for better user flow.

QRCodeModal.jsx 
<div className="flex justify-between text-sm">
    <span className="text-gray-600">Order ID:</span>
    <span className="font-semibold text-gray-800 truncate ml-2">
        <BudolPayText text={orderId || "N/A"} />
    </span>
</div>
<div className="flex justify-between text-sm pt-1">
    <span className="text-gray-600 mb-0.5">Reference ID:</span>
    <span className="text-[12px] text-gray-500 break-all leading-tight bg-gray-100 p-1">
        {paymentIntentId}
    </span>
</div>

Dependencies

  • Next.js 14+: Used for the Admin and Merchant portals.
  • Flutter 3.x: Used for the mobile application.
  • Tailwind CSS: Used for consistent styling in React.
