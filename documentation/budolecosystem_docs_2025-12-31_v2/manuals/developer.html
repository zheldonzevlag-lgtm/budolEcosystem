Developer Manual v2.0

Purpose & Explanation

The budolPay Ecosystem is a unified financial platform designed for high-concurrency transactions and multi-tenant application support. It provides a shared database layer and common security protocols for apps like budolPay, budolShap, and budolExpress.

Technical Details

  • Architecture: Monorepo (npm Workspaces)
  • Backend: Next.js 14 (App Router), Node.js Microservices
  • Database: PostgreSQL with Prisma ORM
  • Authentication: Shared SSO via JWT and API Keys
  • Frontend: React 18, Tailwind CSS, Lucide Icons

    Common Issues & Fixes

    1. Module Resolution Error (@budolpay/database)

    Reason: Workspace packages not linked or transpiled.

    Fix: Run npm install at root and add transpilePackages: ['@budolpay/database'] to next.config.mjs.

    2. Missing Database Tables

    Reason: Schema out of sync with local DB.

    Fix: Run npx prisma db push followed by npx prisma db seed.

    Risks & Mitigations

    RiskSeverityMitigation
    Database Connection LeakHighUse a global Prisma singleton pattern (lib/prisma.ts).
    Schema DriftMediumStrict CI/CD checks for migration compatibility.
    Sensitive Data ExposureHighUse isSecret flag in SystemSettings and encrypted ENV variables.

    Developer Recommendations

    • Type Safety: Always use npx tsc --noEmit before committing to catch JSX/TypeScript mismatches (e.g., colSpan numbers).
    • Schema Changes: Use prisma migrate dev for local development to maintain a clean history.
    • Service Isolation: Keep microservice logic independent; only share schemas through the @budolpay/database package.
