System Administrator Manual

System Purpose

To provide a stable, secure, and monitored environment for budolPay financial services. Admins are responsible for database integrity, service availability, and ecosystem security.

Deployment & Infrastructure

  • Database: PostgreSQL 15+ (Self-hosted or RDS)
  • App Hosting: Dockerized microservices on Kubernetes or Vercel for Next.js
  • Environment Variables: Managed via .env files or vault systems.

Critical Maintenance Tasks

1. Database Synchronization

Ensure the production database matches the current schema version:

npx prisma migrate deploy

2. Seeding Initial Environment

For new installations, setup the admin account and system constants:

npx prisma db seed

System Dependencies

  • Node.js: v18.0.0 or higher
  • PostgreSQL: v14.0 or higher
  • Network: Access to port 5432 (DB) and application ports (3000-3005)
Security Warning: Never expose the DATABASE_URL in public logs or client-side code. Ensure isSecret settings in the database are only accessible via the Admin Dashboard.

Troubleshooting for Admins

SymptomLikely CauseAction
"Connection Refused"DB Service DownCheck PostgreSQL status on port 5432.
"Invalid Invocation"Prisma Client MismatchRun npm run db:generate.
High LatencyMissing IndexesCheck schema.prisma for @@index declarations.

Recommendations

  • Backups: Implement daily automated PostgreSQL dumps (pg_dump).
  • Monitoring: Use Prometheus/Grafana (configs available in /monitoring).
  • Logs: Rotate build_log.txt and application logs weekly.
