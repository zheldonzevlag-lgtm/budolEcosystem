Developer Manual - v49

This version focuses on resolving critical deployment bottlenecks in the Payment Gateway Service and stabilizing the Vercel infrastructure configuration.

1. The NOT_FOUND Issue

The "The page could not be found NOT_FOUND" error in production was traced to two primary factors:

  • Incorrect Export Logic: The api/index.js file was exporting an object containing the app, rather than the app instance itself. Vercel's Node.js runtime requires the entry point to export the request handler directly.
  • Mismatched Root Directory: The Vercel project configuration was clearing the rootDirectory to null, causing Vercel to look for api/index.js in the repository root instead of the service subdirectory.

2. Technical Changes

2.1 Fix: Payment Gateway Export

Modified budolpay-0.1.0/services/payment-gateway-service/api/index.js:

// OLD: module.exports = { app, generateSecureReferenceId };
// NEW:
module.exports = app;

2.2 Fix: Vercel Infrastructure Pathing

Updated fix-vercel-root.js to use the explicit relative path for the monorepo service:

const body = JSON.stringify({ 
    rootDirectory: "budolpay-0.1.0/services/payment-gateway-service" 
});

3. Data Contracts & Routing

The service remains compatible with the following routes mounted in api/index.js:

  • POST /create-intent - Initiates a payment.
  • GET /checkout/:referenceId - Renders the payment UI.
  • POST /webhooks/:provider - Processes external payment callbacks.

4. Security & Compliance

All changes adhere to PCI DSS requirements by ensuring that sensitive transaction logic is contained within the secure budolpay-0.1.0 boundary and that Vercel routing is explicitly restricted to the necessary subdirectories.

5. Dependencies

No new dependencies were added. The fix utilizes existing express and prisma configurations.
