Developer Manual Update

Implementation of Mobile App v1.2.39 & Build Pipeline

Technical Details

1. Version Increment

The mobile app version was incremented from 1.2.38 to 1.2.39 to reflect recent fixes and ensure unique build identification.

  • Modified pubspec.yaml: version: 1.2.39+39
  • Modified api_service.dart: _appVersion = '1.2.39'

2. API Host Injection

The build pipeline uses --dart-define to inject the LAN IP (192.168.1.24) directly into the app at compile time.

flutter build apk --release --no-tree-shake-icons --dart-define=API_HOST=192.168.1.24

3. Splash Screen Logic

The splash screen dynamically fetches the version from ApiService, which now correctly reflects the fallback version even if package_info_plus fails during initialization.

Code Details

// api_service.dart

String _appVersion = '1.2.39'; // Updated Fallback

String get host => const String.fromEnvironment('API_HOST', defaultValue: '192.168.1.24');
