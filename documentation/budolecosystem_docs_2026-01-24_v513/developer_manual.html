Developer Manual v513

Technical Overview

This release focuses on the end-to-end data flow for merchant/store information and UI modernization of the transaction details screen.

1. Store Name Propagation

Implemented a consistent data contract to pass the store name from the e-commerce frontend to the wallet transaction record.

OrderSummary.jsx

Calculates the display store name from cart items and passes it to the checkout API.

const storeNames = items ? [...new Set(items.map(item => item.store?.name || item.storeName || 'budolShap Store').filter(Boolean))] : [];
const displayStoreName = storeNames.length > 1 ? `${storeNames[0]} +${storeNames.length - 1}` : (storeNames[0] || 'budolShap Store');
wallet-service/index.js

Updates the transaction record with the store information during QR processing.

await prisma.transaction.update({
    where: { id: transaction.id },
    data: { 
        senderId: userId,
        status: 'COMPLETED',
        completedAt: getLegacyManilaDate(),
        storeId: qrData.storeId || null,
        storeName: qrData.storeName || null
    }
});
