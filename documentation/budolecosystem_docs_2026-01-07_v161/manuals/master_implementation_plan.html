Technical Details

Each service was linked using the following non-interactive command:

vercel link --yes --project [project-name]

This command automatically generates the project.json file within the .vercel directory, mapping the local folder to the Vercel project ID and Organization ID.

Risks & Mitigations

  • Risk: Monorepo dependency conflicts. Mitigation: Each project uses its own vercel.json and points to the specific service root.
  • Risk: Environment variable mismatch. Mitigation: Centralized .env.vercel template used for all 7 projects.

Recommendations

  • Implement Vercel GitHub Integration for automatic deployments on push.
  • Set up Vercel Checks for automated E2E testing post-deployment.
