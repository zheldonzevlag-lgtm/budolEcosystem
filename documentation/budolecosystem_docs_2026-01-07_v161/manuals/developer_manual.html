Developer Manual: Vercel CLI Workflow (v161)

Workflow Overview

With projects now linked via the Vercel CLI, developers can manage deployments directly from the terminal for each individual microservice.

Deploying a Specific Service

To deploy changes for a specific service (e.g., auth-service):

cd budolpay-0.1.0/services/auth-service
vercel deploy --prod

Managing Environment Variables

Environment variables can be pulled or pushed using the CLI to ensure local parity with the cloud:

# Pull environment variables from Vercel to .env.local
vercel env pull .env.local

# Add a new variable to Vercel
vercel env add DATABASE_URL production

Local Development with Vercel

To run a service locally while simulating the Vercel environment (including serverless function behavior):

vercel dev

Linked Project Structure

Each service now contains a .vercel directory. Do not delete this directory as it contains the project.json file required for CLI operations. It is already added to .gitignore.

Common Troubleshooting

  • "Project not linked": Run vercel link --yes --project budolecosystem-[name] in the service directory.
  • "Build failed": Check the vercel.json routing and ensure index.js exports the app instance.
