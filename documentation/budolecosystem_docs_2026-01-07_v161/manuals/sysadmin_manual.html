SysAdmin Manual: Project Administration (v161)

Vercel Project Portfolio

The budolEcosystem now consists of 7 managed projects on Vercel under the account galvezjon59-3555.

Project Monitoring

Administrators should monitor the following project dashboards for build status and runtime errors:

  • budolecosystem-api-gateway: Entry point for all mobile traffic.
  • budolecosystem-auth-service: User authentication and session management.
  • budolecosystem-wallet-service: Balance and ledger operations.
  • budolecosystem-transaction-service: P2P and merchant transaction logic.
  • budolecosystem-payment-gateway: External payment integrations.
  • budolecosystem-settlement-service: Daily reconciliation and batch processing.
  • budolecosystem-verification-service: KYC and document processing.

Database Connection Management

All projects share a single Vercel Postgres instance. Administrators must ensure that the POSTGRES_URL and POSTGRES_PRISMA_URL environment variables are correctly inherited across all 7 projects to prevent connectivity issues.

Scaling and Limits

Note: On the Hobby plan, there are limits on the number of serverless functions and concurrent builds. Monitor the "Usage" tab in the Vercel Dashboard to ensure the microservice architecture does not exceed these limits.

Deployment Rollbacks

In case of a critical failure, use the Vercel Dashboard to roll back to a previous successful deployment for the specific affected service. The projects are independent, so a rollback on one does not affect the others.

Security Configuration

Ensure that "Deployment Protection" (Vercel Authentication) is enabled for preview deployments to prevent unauthorized access to internal microservice APIs during the testing phase.
