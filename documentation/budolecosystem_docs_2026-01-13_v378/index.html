budolPay v378 – Face Recognition Interpreter Reliability Release

Date: 2026-01-13
Version: 1.2.78+78
Previous: v377
Next: v379

Purpose

Eliminate the "TFLite Interpreter is unavailable (null)" runtime failure during face registration by adding robust retry logic, interpreter health validation, and defensive file handling.

Technical Changes

  • face_embedding_service.dart
    • Wrapped initialization in 2-attempt retry loop with 200 ms back-off.
    • Added getInputTensors() health check immediately after interpreter creation to detect native-side null edge-cases.
    • Unique temp file per attempt (facenet_attempt_N.tflite) to avoid stale file locks.
    • Removed unused dart:typed_data import to satisfy linter.
  • pubspec.yaml & api_service.dart
    • Version bumped to 1.2.78+78 for release tracking.

Compliance Impact

No new data flows; strengthens biometric reliability (BSP, NPC) and error resilience (Cybersecurity Law).
