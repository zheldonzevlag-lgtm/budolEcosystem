Mitigation Controls – v378 Face Recognition Reliability

R-378-01 Interpreter still null after 2 attempts

  • Retry loop with 200 ms back-off.
  • Interpreter health check via getInputTensors() immediately after creation.
  • Clear error message propagated to UI; user can re-tap register.

R-378-02 Temp directory full / write failure

  • Verify file existence and exact byte length match after write.
  • Unique filename per attempt to avoid locks.
  • Exception raised to caller; logged for diagnostics.

R-378-03 APK size increase

  • Minimal extra code (~30 lines); no new assets.
  • ProGuard/R8 minification already enabled; impact < 5 KB.
