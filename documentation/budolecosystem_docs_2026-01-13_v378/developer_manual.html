Developer Manual – Face Recognition Interpreter Reliability (v378)

1. Problem Statement

Intermittent Exception: TFLite Interpreter is unavailable (null) after initialization on Android devices caused by native-side race conditions and asset-manager limitations.

2. Root Cause

  • Interpreter.fromAsset() sometimes returns an invalid handle on Android API 30+.
  • Single-attempt loading without health validation.
  • No retry or fallback path when temp file is locked.

3. Solution Overview

File-based extraction + retry loop + interpreter health check.

4. Code Changes

File: face_embedding_service.dart

4.1 Retry Loop

const maxRetries = 2;
for (int attempt = 1; attempt <= maxRetries; attempt++) {
  try {
    ...
    return; // success
  } catch (e, stack) {
    if (attempt == maxRetries) {
      _loadingCompleter!.completeError(e);
    } else {
      await Future.delayed(const Duration(milliseconds: 200));
    }
  }
}

4.2 Unique Temp File per Attempt

final modelFile = File('${tempDir.path}/facenet_attempt_$attempt.tflite');

Avoids stale file locks between retries.

4.3 Interpreter Health Check

final newInterpreter = Interpreter.fromFile(modelFile, options: options);
try {
  newInterpreter.getInputTensors(); // native call
} catch (e) {
  throw Exception('Interpreter health check failed: $e');
}

Immediately validates the native handle before assignment.

4.4 File Verification

if (!await modelFile.exists() || await modelFile.length() != bytes.length) {
  throw Exception('Failed to write or verify model temporary file');
}

5. Usage

No API change; consumers still call:

final embedding = await faceService.generateEmbedding(imageFile);

Internally the service now self-heals or throws a descriptive exception after 2 attempts.

6. Testing

  • flutter analyze lib/services/face_embedding_service.dart – 0 issues.
  • Manual stress-test: rapid successive registrations on Android 12/13 devices.

7. Dependencies

  • path_provider – already in pubspec; used for temp directory.
  • tflite_flutter – unchanged.

8. Compliance Notes

Biometric reliability improvement satisfies BSP Circular 1108 and NPC advisory on accuracy. No new PII flows.

9. Future Work

Consider migrating to tflite_flutter_helper’s Interpreter.fromBuffer() once stable to skip disk I/O.
