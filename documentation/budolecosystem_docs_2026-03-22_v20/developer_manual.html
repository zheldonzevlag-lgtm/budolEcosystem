🛠️ v20 Developer Manual

1. DB Schema Synchronization

The Checkout model was updated in prisma/schema.prisma to include the following tracking fields to resolve production Prisma validation errors.

model Checkout { ... metadata Json? expiresAt DateTime? sessionId String? attemptCount Int @default(0) paymentId String? paymentProvider String? expiredAt DateTime? }

🌍 Master Service Discovery (Warp Sync)

To fix 401 Unauthorized errors in production, I've implemented a 100% mirror of the local 13-service environment variables. All services now use internal AWS DNS (*.budol.internal) instead of brittle DuckDNS or localhost fallbacks.


// Production ECS Overrides (BudolShap)
BUDOL_ID_URL: "http://budol-id.budol.internal:8000"
INTERNAL_WS_URL: "http://budolpay-websocket.budol.internal:4000"
PAYMENT_GATEWAY_URL: "http://budolpay-payment.budol.internal:8004/"

🛡️ AWS VPC Internal Migrator

Because Production RDS is private, I've introduced a one-time VPC Migrator process. This runs an ECS Task with the app image but overrides the command to npx prisma db push, syncing tables like SystemSetting from inside the secure network.

2. BudolPay Internal Adapter

The BudolPayAdapter was updated to correctly handle internal VPC routing and port 8004. Key logic added for internal vs. proxy-based URL formatting.

// If using port 8080 (proxy), add /payments prefix if (this.gatewayUrl.includes(':8080') && !this.gatewayUrl.includes('/payments')) { this.gatewayUrl = this.gatewayUrl.endsWith('/') ? `${this.gatewayUrl}payments` : `${this.gatewayUrl}/payments`; }
