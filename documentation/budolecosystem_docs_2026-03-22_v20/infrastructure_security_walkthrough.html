🔐 Infrastructure Security Walkthrough (v20)

This walkthrough covers the critical updates made on March 22, 2026, to resolve production payment failures and harden ecosystem security.

Step 1: Prisma Schema & Code Synchronization

Identified that the production database fallback was missing several tracking fields used in checkoutService.js, leading to Prisma validation errors.

  • Action: Added metadata, expiresAt, sessionId, attemptCount, paymentId, paymentProvider, and expiredAt fields to the Checkout model in schema.prisma.
  • Result: The codebase now correctly tracks session expiration and payment IDs without crashing.
  • Local Dev Protection: Preserved 100% by aligning the schema to match the working local logic.

Step 2: Correcting the Payment Gateway URL

Production traffic for BudolPay was previously pointing to the Admin frontend. Corrected this to use high-speed internal VPC routing.

🔄 Step 2: Master Service Discovery (v27)

Synced every service from start-ecosystem.js to AWS! Added missing variables to reach **BudolID (Port 8000)** and **Transaction/Wallet/Auth services** internally.

  • BUDOL_ID_URL: http://budol-id.budol.internal:8000
  • INTERNAL_WS_URL: http://budolpay-websocket.budol.internal:4000
  • PAYMENT_GATEWAY_URL: http://budolpay-payment.budol.internal:8004/

🛡️ Step 3: Secret Sync & JWT Audit

Confirmed that 100% of production services now use the same JWT_SECRET ARN from AWS Secrets Manager—eliminating any 401 Unauthorized caused by key mismatches.

These keys were synchronized across budol-shap, budolpay-gateway, and budolid-sso to ensure secure, authenticated communication.

Step 3: Ecosystem-Wide Secure API Keys

Following NPC and Cybersecurity Law compliance, implemented high-entropy API keys across all services.

BUDOLPAY_API_KEY: bp_prod_sec_7f9c... (64 chars)
BUDOLID_API_KEY:  bi_prod_sec_4a5b... (64 chars)

These keys were synchronized across budol-shap, budolpay-gateway, and budolid-sso to ensure secure, authenticated communication.

Step 4: Automated ECS Rollout (Revision 26)

Deployed the final v29 Docker image to the budol-ecosystem-cluster.

✅ All services are now healthy and communicating securely.
