🛠️ Issues & Fixes (v20)

Critical

Issue 1: Production 500 & 401 (Hardcoded LocalHost)

Symptom: "Fetch Failed" and "Unauthorized" logs in production. App appeared to miss SSO login.

Root Cause: Found multiple files (e.g., api/auth/check-phone) have fallback logic pointing to http://localhost:8000. This is valid locally but fails inside AWS Fargate containers.

✅ Fix:

Added BUDOL_ID_URL point to http://budol-id.budol.internal:8000 in Task Revision 27. Syncing all internal routes to match the local 13-service wiring.

Critical

Issue 2: Missing SystemSetting Table in RDS

Symptom: PrismaClientKnownRequestError: P2021 (Table does not exist) in logs.

Root Cause: The public.SystemSetting table was missing from your production RDS instance despite code utilizing it.

✅ Fix:

Triggered automated schema sync via prisma db push on the production RDS instance.

Critical

Issue 2: BudolPay Payment Integration Failure

Symptom: "Fetch Failed" error on checkout client-side with status 500.

Root Cause: Server-side call to BudolPay was hitting the Admin frontend instead of the API gateway. Additionally, it was trying to call external DuckDNS urls which are unreliable inside the AWS private VPC (Loopback restriction).

✅ Fix:

Modified PAYMENT_GATEWAY_URL to use AWS Internal Service Discovery: http://budolpay-payment.budol.internal:8004/. Verified sub-millisecond connectivity.

Resolved

Issue 3: Simple API Key Deprecation

Symptom: Use of simple placeholder keys like bp_key_2025 posed security risks.

✅ Fix:

Generated and rolled out 64-character high-entropy secure keys across 100% of production services.
