Continuous Improvement Roadmap

Strategic

1. Automated Build Pipeline (GitHub Actions)

Purpose: To ensure every release is built in a clean, reproducible environment.

Reason: Currently, builds are manual. Automating this would include automatic static analysis, unit testing, and APK generation on every tag.

Security

2. App Signing & Play Store Preparation

Purpose: To move beyond debug/adhoc distribution.

Mitigation: Implement official Keystore signing and prepare the app's metadata for potential Play Store or internal enterprise store distribution.

Stability

3. Crashlytics Integration

Purpose: Real-time monitoring of production crashes.

Explanation: While the v164 error boundary catches crashes, we still need a way to receive those logs automatically. Firebase Crashlytics is the recommended next step.

Summary of Consolidated Recommendations

  • Infrastructure: Move to a dedicated build server to avoid local environment discrepancies.
  • Features: Begin research for Version 1.2.0, focusing on biometric security and deep-linking for payments.
  • Quality: Increase unit test coverage for the ApiService and TransactionModel.
