Operational Runbook: Database & Service Recovery

If the ecosystem fails to start due to database connectivity or port conflicts, follow these steps:

1. Verify PostgreSQL Service

Check if the PostgreSQL service is running using PowerShell:

Get-Service -Name postgresql*

If stopped, start it:

Start-Service -Name postgresql-x64-18

2. Resolve Port Conflicts

The ecosystem requires ports 3000, 3001, and 8000. To clear these ports, run:

Stop-Process -Id (Get-NetTCPConnection -LocalPort 8000).OwningProcess -Force Stop-Process -Id (Get-NetTCPConnection -LocalPort 3001).OwningProcess -Force Stop-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess -Force

3. Restart Ecosystem Central Runner

Navigate to the root directory and run:

npm start

Infrastructure Configuration

  • DB Host: localhost
  • DB Port: 5432
  • OS: Windows (PowerShell environment)
  • PostgreSQL Version: 18 (x64)
Important: Ensure that no other local applications (like XAMPP or separate Docker containers) are competing for port 5432 or the application ports.
