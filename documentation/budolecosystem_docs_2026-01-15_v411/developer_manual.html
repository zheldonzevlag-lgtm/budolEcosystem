Developer Manual - Version 411

This manual details the technical changes for the Android version increment and release preparation.

Version Increment Logic

The versioning in BudolPay Mobile is tracked in two primary locations to ensure both build tools and the UI have access to the correct version string.

1. pubspec.yaml

Updated the version field to 1.2.95+95. This controls the versionName and versionCode in the Android build.

version: 1.2.95+95

Link: pubspec.yaml

2. ApiService.dart

The _appVersion constant is used for UI display in the Splash Screen and for potential version-based logic in API calls.

String _appVersion = '1.2.95'; // Android Build v1.2.95

Link: api_service.dart

UI Implementation

The Splash Screen dynamically fetches the version from the ApiService.

final appVersion = context.watch<ApiService>().appVersion;
...
Text('budolEcosystem $appVersion')

Link: splash_screen.dart

Dependencies

  • Flutter SDK: 3.0.0+
  • package_info_plus: ^4.2.0
