Detailed Future Recommendations - v330

[SECURITY] Flag Secure Implementation

Rationale: Compliance with financial security standards (PCI DSS/BSP) requires protecting user data from unauthorized screen capture.

Technical Strategy: Implement a service in the mobile app that toggles WindowManager.LayoutParams.FLAG_SECURE based on the current route. This is critical for the "View Balance" and "Transaction Details" screens.

[DEVOPS] Remote Logging Integration

Rationale: Maintenance of a stable production app (v1.2.53+) requires visibility into runtime errors that no longer appear on-screen.

Technical Strategy: Deploy Sentry.io integration to capture stack traces, device state, and user actions leading to failures. This will replace the manual "debug window" approach used in development.

[UX] Biometric Grace Period

Rationale: High-frequency users find constant biometric prompts intrusive when multitasking between the banking app and other tools.

Technical Strategy: Create a SessionManager that tracks the last successful biometric auth timestamp. If the app is resumed within 300 seconds, bypass the prompt while keeping the secure token valid in memory.
