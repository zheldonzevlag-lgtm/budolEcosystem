Developer Manual: SSO Login Button Restoration

Technical details for restoring the ecosystem-wide Single Sign-On (SSO) entry point in budolShap.

Version: 2026-01-01_v31
Focus: Restoring the "Login with budolID" button and ensuring LAN connectivity.

Implementation Details

The "Login with budolID" button was restored and repositioned within the AuthForm component. This change ensures that the ecosystem-wide login option is prominently displayed and remains visible even on smaller viewports or within constrained modal windows.

Key Changes in AuthForm.jsx

  • Repositioning: Moved the SSO section above the primary "Sign In" button. This prevents it from being pushed out of view in long forms.
  • Visual Hierarchy: Added a "Quick Access" separator to clearly distinguish between local login and ecosystem login.
  • Dynamic Hostname Resolution: Improved the href attribute to use window.location.hostname when process.env.NEXT_PUBLIC_SSO_URL is missing, ensuring LAN compatibility.
  • UI Enhancements: Increased the size of the "B" icon (w-5 h-5) and improved typography for better accessibility.
// Location: budolshap-0.1.0/components/auth/AuthForm.jsx

<a
    href={`${process.env.NEXT_PUBLIC_SSO_URL || `http://${typeof window !== 'undefined' ? window.location.hostname : 'localhost'}:8000`}/login?apiKey=${process.env.NEXT_PUBLIC_BUDOLID_API_KEY || 'bs_key_2025'}`}
    className="..."
>
    <div className="w-5 h-5 bg-indigo-600 rounded flex items-center justify-center text-[10px] text-white font-bold">B</div>
    <span>Login with budolID</span>
</a>

Impact Analysis

The changes re-establish the primary entry point for the budolEcosystem SSO flow within the budolShap application, affecting both the login modal and the standalone login page.
