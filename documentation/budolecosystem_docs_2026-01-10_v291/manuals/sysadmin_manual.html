SysAdmin Manual - v291 Updates

1. Environment Updates

The JWT_SECRET must be identical across all services for SSO to function. This includes Docker containers and local development environments. Verify the following files:

  • budolID-0.1.0/.env
  • budolshap-0.1.0/.env
  • budolpay-0.1.0/.env
  • budolpay-0.1.0/docker-compose.yml (auth-service)
  • .env (Root)

2. Deployment Procedures

After updating the JWT_SECRET, you MUST restart the following services. If using Docker, rebuild the containers to pick up the new environment variables.

Action Required:

  1. Stop budolID (Port 8000)
  2. Stop budolShap (Port 3001)
  3. Stop budolPay services (8001-8080) or run docker-compose down
  4. Restart all services or run docker-compose up -d --build

3. Risk & Mitigation

Risk: Session Invalidation

Changing the JWT_SECRET will invalidate all currently active sessions across the entire ecosystem. Users will be required to log in again.

Mitigation: Coordinated Rollout

Always update the secret during maintenance windows and ensure all services are updated simultaneously to prevent partial SSO failures.

4. Operational Runbook

To verify secret synchronization without manual inspection:

node verify-final-sync.js

This script (located in the root) checks the JWT_SECRET in all critical .env files and the docker-compose.yml file.
