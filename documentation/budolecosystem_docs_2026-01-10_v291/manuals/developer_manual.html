Developer Manual - v291 Updates

1. Purpose

Fix the persistent "Signature Verification Error" during SSO login between budolID and budolShap, and ensure consistency of JWT_SECRET across Docker and npm start scripts. This release also addresses investigation into hidden environment variables in startup scripts.

2. Technical Implementation

Secret Synchronization

The ecosystem now strictly enforces a standardized JWT_SECRET across all services, including Docker configurations. The investigation confirmed that npm start scripts do not override these secrets.

// Updated budolID-0.1.0/.env & budolpay-0.1.0/docker-compose.yml
JWT_SECRET="GJ7Lxn0/kdV/KuZJ5xJ7Ip0RvMerrGW5n0gf44mfHgc="

npm Script Investigation

A comprehensive audit of all package.json files across the monorepo confirmed that no npm start or dev scripts are injecting JWT_SECRET. All secrets are managed via .env files or container environment variables.

Docker Configuration Update

The auth-service in budolpay-0.1.0/docker-compose.yml was identified as having an outdated secret. This has been synchronized with the rest of the ecosystem.

3. Test Results

  • Cross-runtime Compatibility Test: PASS
  • budolID Secret Update: VERIFIED
  • Signature Verification: PASS

4. Issues & Fixes

Issue Fix
Signature Verification Failed Updated budolID .env with the correct ecosystem JWT_SECRET.
SSO Loop/Sign Modal Redirect Standardized secrets to allow budolShap to trust budolID tokens.

5. Recommendations

Implement a centralized secret management service or a shared configuration loader to prevent future synchronization issues across submodules.
