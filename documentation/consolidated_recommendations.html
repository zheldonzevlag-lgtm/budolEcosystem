Software Choices & Rationale (Phase 8)

Software / Library Choice Rationale
Next.js (React) Used for the frontend to provide server-side rendering (SSR) for SEO and fast initial page loads, along with a robust routing system.
Flutter (Dart) Chosen for budolPayMobile to maintain a single codebase for both Android and iOS while delivering native performance.
Tailwind CSS Provides utility-first styling for rapid UI development and consistent design across the web application.
Prisma (ORM) Ensures type-safe database access and easy schema migrations for the backend services.
Pusher / WebSockets Enables real-time updates for payment status and order tracking across the ecosystem.

Consolidated Future Recommendations

  • Hard Inactivity Timeout Policies: Standardize 3-minute inactivity locks across all financial interaction points to meet BSP security standards.
  • Dynamic Timeout Configuration: Implement a backend-driven timeout policy that allows admins to adjust the inactivity duration without requiring an app update.
  • Automated Compliance Audits: Integrate tools to automatically scan for PCI DSS and NPC compliance violations in code and database schemas.
  • Microservices Decoupling: Gradually move towards a fully decoupled microservices architecture to improve scalability and fault tolerance.
  • AI-Driven Fraud Detection: Utilize the existing face recognition models to implement behavioral biometrics for fraud prevention.
  • Global Timezone Compliance: Continue the transition to full UTC compliance for all timestamp-related data.
  • MFA Implementation: Implement OTP-based Multi-Factor Authentication for Admin logins to enhance security (v2.1.2 Recommendation).
  • Automated CI/CD: Integrate GitHub Actions for automated Flutter builds and deployment (v2.1.2 Recommendation).
  • Real-time Monitoring: Enhance budolPay Admin with real-time fraud pattern detection (v2.1.2 Recommendation).
