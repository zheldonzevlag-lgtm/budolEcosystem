Developer Technical Guide

1. Flutter Web App Integration

The Flutter Web App is now part of the central ecosystem. It is configured to run on all network interfaces (0.0.0.0) to allow external access from mobile devices on the same LAN.

Code Reference: start-ecosystem.js


{
    name: 'budolPay (Mobile Web)',
    cwd: path.join(__dirname, 'budolPayMobile'),
    command: 'flutter',
    args: ['run', '-d', 'web-server', '--web-hostname', '0.0.0.0', '--web-port', '9000'],
    port: 9000,
    color: '\x1b[38;5;208m' // Orange
}

2. LAN Access URL

To access the app from another device, use the machine's Local IP address followed by the port 9000. The central runner automatically detects and displays this IP on startup.

Example: http://192.168.1.17:9000

3. Service Discovery (mDNS)

The app continues to use mDNS to discover backend services. The budolPay (Mobile Web) service is also advertised via mDNS for easy discovery by other tools in the ecosystem.

4. Compliance and Security

  • NPC (Data Privacy): Access is restricted to the local network for development. In production, HTTPS and origin validation are required.
  • DTI/BSP: System availability is tracked via the central runner logs, ensuring the full frontend-to-backend stack is observable.
