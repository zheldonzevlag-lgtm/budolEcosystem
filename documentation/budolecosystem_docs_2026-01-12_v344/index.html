Release Overview - v344

Purpose: Integrate Flutter Web App into the central ecosystem runner for LAN-accessible development.

Rationale: Developers and testers need to access the mobile application from physical devices (phones/tablets) or other computers on the same network. By integrating the Flutter web runner with --web-hostname 0.0.0.0, the app becomes accessible via the machine's Local IP.

Technical Explanation

The central runner start-ecosystem.js was updated to include the budolPay (Mobile Web) service. The command uses flutter run -d web-server with specific flags to bind to all network interfaces and port 9000. This complements the existing mDNS discovery and Local IP detection logic used by the microservices.
