Mitigation Plan

Strategies & Controls

Dual-Channel OTP Delivery

To mitigate SMS delivery failures (R-001), the system implements a provider-agnostic notification package that can switch between Zerix, iTextMo, and SMTP for email delivery.

Environment Parity Controls

The fix for R-003 involved centralizing environment detection in auth-service/index.js, ensuring that production logic is never bypassed in non-local environments unless explicitly running automated tests.

Rate Limiting & Audit Logging

Enforced 60-second cooldown for OTP resends using the otpUpdatedAt database field. Every registration attempt is logged in the AuditLog table with IP address and metadata for compliance monitoring.
