Deployment Procedures

The update involves changes to the transaction-service. Follow these steps to apply the changes:

  1. Stop the current ecosystem process (start-ecosystem.js).
  2. Pull the latest changes for services/transaction-service/index.js.
  3. Restart the ecosystem using node start-ecosystem.js.
  4. Verify the api-gateway correctly proxies the /transactions/history route.

Endpoint Configuration

Service Internal Port External Route (via Gateway) Status
Transaction Service 8003 /transactions/history/:userId ACTIVE

Operational Verification

SysAdmins can verify the history endpoint using the following command from the host machine:

curl -i http://localhost:8080/transactions/history/[VALID_USER_ID]

A successful response should return a 200 OK status and a JSON array of transaction objects.

Rollback Strategy

If the new endpoint causes performance degradation in the database:

  1. Revert services/transaction-service/index.js to the previous version.
  2. Restart the transaction service.
  3. Note that mobile users will revert to seeing "No recent transactions".
