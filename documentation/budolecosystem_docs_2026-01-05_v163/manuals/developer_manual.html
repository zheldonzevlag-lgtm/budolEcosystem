Developer Manual v163

Transaction History Implementation & Parsing Fixes

← Back to v163 Index

Purpose

To enable the "Recent Transactions" feature on the budolPay mobile app by implementing the missing backend history endpoint and fixing frontend data parsing errors.

Issues Identified

  • 404 Not Found: The mobile app was calling GET /transactions/history/:userId, which was not defined in the transaction-service.
  • Type Casting Error: The Flutter HomeScreen was attempting to cast tx['amount'] directly to num, which failed when the API returned a string (common with Prisma Decimal fields).

Technical Implementation

1. Backend: Transaction Service History Endpoint

Added a new route in services/transaction-service/index.js to handle user-specific history requests. The query uses an OR condition to include transactions where the user is either the sender or the receiver.

app.get('/history/:userId', async (req, res) => {
    const { userId } = req.params;
    try {
        const transactions = await prisma.transaction.findMany({
            where: {
                OR: [
                    { senderId: userId },
                    { receiverId: userId }
                ]
            },
            orderBy: { createdAt: 'desc' },
            take: 50
        });
        res.json(transactions);
    } catch (error) {
        res.status(500).json({ error: 'Failed to fetch transaction history' });
    }
});

2. Frontend: Safe Amount Parsing

Modified lib/screens/home_screen.dart to safely parse the transaction amount from the API response.

// Before
final double amount = (tx['amount'] as num).toDouble();

// After
final double amount = double.tryParse(tx['amount'].toString()) ?? 0.0;

Test Results

Verified using a dedicated test script test-transaction-history-v163.js which performs a P2P transfer and immediately verifies its presence in the history endpoint.

Risks & Mitigations

  • Risk: Performance issues with large transaction tables.
  • Mitigation: Implemented take: 50 limit and ensured createdAt indexing in the database.
