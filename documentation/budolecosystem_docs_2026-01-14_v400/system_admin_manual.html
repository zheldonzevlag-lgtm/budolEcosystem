System Admin Manual

Administrative procedures for ecosystem maintenance and recovery.

Database Maintenance

To ensure system stability, the budolID database must remain synchronized with the master Prisma schema. In the event of schema mismatch errors (e.g., missing columns), the following procedure should be followed:

# Synchronization Command

npx prisma db push --schema="D:\IT Projects\budolEcosystem\budolpay-0.1.0\packages\database\prisma\schema.prisma"

Note: Ensure the DATABASE_URL environment variable points to the budolid database before execution.

Health Monitoring

Administrators can verify the health of the Identity Provider via the health check endpoint:

GET http://localhost:8000/api/health

Expected Response: {"status":"ok", "message":"budolID..."}

ACTIVE

Identity Recovery

If a master admin account is compromised or lost, use the recovery script:

node prisma_old/reset-admin.js
