Orchestration Layer Updates

The root orchestration logic (start-ecosystem.js) relies on delegated scripts defined in the root package.json. These scripts forward commands to the appropriate sub-projects.

Root Scripts Configuration

The following scripts have been added to d:\IT Projects\budolEcosystem\package.json to support the central runner:

"scripts": {
  "start": "node start-ecosystem.js",
  "db:local": "npm run db:local --prefix budolshap-0.1.0",
  "db:vercel": "npm run db:vercel --prefix budolshap-0.1.0",
  "db:generate": "npm run prisma:generate --prefix budolshap-0.1.0 && npm run db:generate --prefix budolpay-0.1.0 && npm run prisma:generate --prefix budolID-0.1.0 && npm run prisma:generate --prefix budolAccounting-0.1.0",
  "test:integration": "node scripts/test_scripts/test-phase16-integration.mjs"
}

Architecture & Data Contracts

The Budol Ecosystem uses a distributed architecture where BudolShap handles order management and BudolPay handles financial transactions. Persistence is achieved via PostgreSQL databases managed by Prisma ORM.

Order Persistence Flow

  1. Order Creation: Initialized in budolshap-0.1.0 via ordersService.js.
  2. Payment Processing: Handled by budolpay-0.1.0.
  3. Status Update: Webhooks from BudolPay trigger status updates in BudolShap via budolpay/route.js.
