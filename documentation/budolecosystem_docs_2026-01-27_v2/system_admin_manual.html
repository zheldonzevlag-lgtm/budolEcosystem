System Admin Manual

System Startup Procedures

To start the entire ecosystem, use the central runner from the root directory:

npm start

Environment Management

The system supports multiple environments. Switching is handled via the root orchestration scripts:

  • npm run db:local - Switches all services to local database configurations.
  • npm run db:vercel - Switches all services to Vercel/Production database configurations.

Prisma Client Generation

If you encounter "Prisma client not found" errors, regenerate clients for all services using:

npm run db:generate
