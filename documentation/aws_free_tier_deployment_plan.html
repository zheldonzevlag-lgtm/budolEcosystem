1. Executive Summary
📌 Overview
This document outlines a comprehensive plan to deploy the budolEcosystem on AWS Free Tier. The plan leverages EC2 t3.micro for container orchestration and RDS PostgreSQL for database, while using managed services to minimize costs.

Key Benefits

  • 12 months free - EC2 t3.micro + RDS db.t3.micro
  • Always free - Lambda, S3, CloudFront, API Gateway
  • Full control - Run all microservices on EC2
  • Production-ready - Scalable architecture

Deployment Strategy: Single EC2 + Docker

Instead of using ECS (which requires paid tier for Fargate), we'll use a single t3.micro EC2 instance running Docker Compose to host all microservices. This is the fastest and most cost-effective approach.

5. Phase 1: AWS Account Setup
1 Phase 1: AWS Account Setup ⏱️ Day 1
1 Create AWS Account
  1. Go to aws.amazon.com
  2. Create new account with email
  3. Complete identity verification
  4. Add payment method (required for free tier)
2 Enable Free Tier Alerts
  1. Go to AWS Billing Console
  2. Create budget alert at 80% usage
  3. Set up CloudWatch billing alarms
3 Create IAM User for Deployment 
# Create IAM user with necessary permissions
aws iam create-user --user-name budol-deploy
aws iam attach-user-policy --user-name budol-deploy --policy-arn arn:aws:iam::aws:policy/AdministratorAccess
4 Install AWS CLI 
# Install AWS CLI
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Configure
aws configure
# Enter Access Key ID
# Enter Secret Access Key
# Enter region: ap-southeast-1
# Enter output format: json
✅ Phase 1 Deliverables
  • AWS account created
  • IAM user configured
  • AWS CLI installed and configured
6. Phase 2: Core Infrastructure
2 Phase 2: Core Infrastructure ⏱️ Days 1-2
1 Create Key Pair 
# Create EC2 key pair
aws ec2 create-key-pair --key-name budol-ecosystem --query 'KeyMaterial' --output text > budol-ecosystem.pem
chmod 400 budol-ecosystem.pem
2 Create Security Group 
# Create security group
aws ec2 create-security-group --group-name budol-sg --description "Security group for budolEcosystem"

# Add rules
aws ec2 authorize-security-group-ingress --group-name budol-sg --protocol tcp --port 22 --cidr 0.0.0.0/0  # SSH
aws ec2 authorize-security-group-ingress --group-name budol-sg --protocol tcp --port 80 --cidr 0.0.0.0/0  # HTTP
aws ec2 authorize-security-group-ingress --group-name budol-sg --protocol tcp --port 443 --cidr 0.0.0.0/0  # HTTPS
aws ec2 authorize-security-group-ingress --group-name budol-sg --protocol tcp --port 3000 --cidr 0.0.0.0/0  # Next.js
aws ec2 authorize-security-group-ingress --group-name budol-sg --protocol tcp --port 3001 --cidr 0.0.0.0/0  # budolShap
3 Launch EC2 Instance (t3.micro) 
# Get latest Amazon Linux 2 AMI
aws ec2 describe-images --owners amazon --filters "Name=name,Values=amzn2-ami-hvm-2.0.*-x86_64-gp2" --query 'Images[0].ImageId'

# Launch instance
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.micro \
  --key-name budol-ecosystem \
  --security-group-ids sg-xxxxxxxx \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=budolEcosystem}]' \
  --user-data file://user-data.sh
4 Create RDS PostgreSQL 
# Create RDS subnet group
aws rds create-db-subnet-group \
  --db-subnet-group-name budol-subnet-group \
  --subnet-ids subnet-xxx subnet-yyy \
  --description "Subnet group for budolEcosystem"

# Create RDS instance
aws rds create-db-instance \
  --db-instance-identifier budol-postgres \
  --db-instance-class db.t3.micro \
  --engine postgres \
  --engine-version 15.3 \
  --allocated-storage 20 \
  --master-username budoladmin \
  --master-user-password YourSecurePassword123! \
  --db-name budolshap \
  --vpc-security-group-ids sg-xxxxxxxx \
  --db-subnet-group-name budol-subnet-group \
  --backup-retention-period 7
5 Create S3 Bucket 
# Create S3 bucket
aws s3 mb s3://budol-ecosystem-media --region ap-southeast-1

# Enable versioning
aws s3api put-bucket-versioning \
  --bucket budol-ecosystem-media \
  --versioning-configuration Status=Enabled

# Set bucket policy for public read (if needed for images)
aws s3api put-bucket-policy \
  --bucket budol-ecosystem-media \
  --policy file://s3-policy.json
✅ Phase 2 Deliverables
  • EC2 t3.micro instance running
  • RDS PostgreSQL database created
  • S3 bucket for media storage
  • Security groups configured
7. Phase 3: Microservices Deployment
3 Phase 3: Microservices Deployment ⏱️ Days 2-3
1 SSH into EC2 and Install Docker 
# SSH into instance
ssh -i budol-ecosystem.pem ec2-user@your-ec2-ip

# Install Docker
sudo yum update -y
sudo amazon-linux-extras install docker
sudo service docker start
sudo usermod -a -G docker ec2-user

# Install Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.23.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# Log out and log back in
2 Clone Repository and Configure 
# Clone repository
git clone https://github.com/your-repo/budolEcosystem.git
cd budolEcosystem

# Create .env.production file
cat > .env.production << 'EOF'
# Database URLs (use RDS endpoint)
BUDOLSHAP_DATABASE_URL=postgresql://budoladmin:password@budol-postgres.xxxx.ap-southeast-1.rds.amazonaws.com:5432/budolshap
BUDOLID_DATABASE_URL=postgresql://budoladmin:password@budol-postgres.xxxx.ap-southeast-1.rds.amazonaws.com:5432/budolid
BUDOLPAY_DATABASE_URL=postgresql://budoladmin:password@budol-postgres.xxxx.ap-southeast-1.rds.amazonaws.com:5432/budolpay
BUDOLACCOUNTING_DATABASE_URL=postgresql://budoladmin:password@budol-postgres.xxxx.ap-southeast-1.rds.amazonaws.com:5432/budolaccounting

# Redis (use ElastiCache or local)
REDIS_URL=redis://localhost:6379

# S3 Configuration
AWS_ACCESS_KEY_ID=your-key
AWS_SECRET_ACCESS_KEY=your-secret
AWS_REGION=ap-southeast-1
AWS_S3_BUCKET=budol-ecosystem-media

# JWT Secrets
JWT_SECRET=your-jwt-secret
SSO_JWT_SECRET=your-sso-jwt-secret
EOF
3 Deploy with Docker Compose 
# Build and start all services
docker-compose -f docker-compose.prod.yml up -d

# Check status
docker-compose -f docker-compose.prod.yml ps

# View logs
docker-compose -f docker-compose.prod.yml logs -f
4 Set Up Application Load Balancer (Optional) 
# Create target group
aws elbv2 create-target-group \
  --name budol-target-group \
  --protocol HTTP \
  --port 80 \
  --vpc-id vpc-xxxxx

# Register instance
aws elbv2 register-targets \
  --target-group-arn arn:aws:elasticloadbalancing:ap-southeast-1:xxxxx:targetgroup/budol-target-group/xxxxx \
  --targets Id=i-xxxxx

# Create ALB
aws elbv2 create-load-balancer \
  --name budol-alb \
  --scheme internet-facing \
  --type application \
  --subnets subnet-xxx subnet-yyy \
  --security-group sg-xxxxx
✅ Phase 3 Deliverables
  • Docker and Docker Compose installed
  • All microservices running on EC2
  • Database connected to RDS
  • S3 integration configured
8. Phase 4: Frontend & Integration
4 Phase 4: Frontend & Integration ⏱️ Days 3-4
1 Configure DNS with Route 53 
# Create hosted zone
aws route53 create-hosted-zone \
  --name budolyourdomain.com \
  --caller-reference "budol-$(date +%s)"

# Create record sets
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXXXXX \
  --change-batch file://dns-records.json
2 Request SSL Certificate (ACM) 
# Request certificate (must validate email or DNS)
aws acm request-certificate \
  --domain-name budolyourdomain.com \
  --subject-alternative-names "*.budolyourdomain.com" \
  --validation-method DNS \
  --region us-east-1
3 Configure CloudFront CDN 
# Create CloudFront distribution
aws cloudfront create-distribution \
  --origin-domain-name budol-ecosystem-media.s3.amazonaws.com \
  --default-root-object index.html
4 Update Environment Variables 
# Update .env.production with production URLs
NEXT_PUBLIC_API_URL=https://api.budolyourdomain.com
NEXT_PUBLIC_SOCKET_URL=wss://ws.budolyourdomain.com
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your-cloud-name
NEXT_PUBLIC_CLOUDINARY_API_KEY=your-api-key
✅ Phase 4 Deliverables
  • Custom domain configured
  • SSL certificate active
  • CloudFront CDN for static assets
  • Production URLs configured

3 Set Up Monitoring 
# Create CloudWatch dashboard
aws cloudwatch put-dashboard --dashboard-name budol-ecosystem \
  --dashboard-body file://cloudwatch-dashboard.json

# Create alarms
aws cloudwatch put-metric-alarm \
  --alarm-name budol-cpu-high \
  --metric-name CPUUtilization \
  --namespace AWS/EC2 \
  --statistic Average \
  --period 300 \
  --threshold 80 \
  --comparison-operator GreaterThanThreshold \
  --evaluation-periods 2
