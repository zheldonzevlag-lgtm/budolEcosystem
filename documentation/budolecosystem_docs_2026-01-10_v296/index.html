Admin Navigation & Security Validation

Comprehensive fix for admin dashboard navigation responsiveness and verification of the underlying authentication layout security.

1. Purpose & Overview

Purpose

This update stabilizes the administrative user interface by decoupling navigation logic from manual state management. It also validates the AdminLayout security layer to ensure that unauthorized access is blocked at the component level, not just the link level.

Explanation

The Navbar.jsx was previously prone to navigation interruptions due to onClick handlers interfering with Next.js route transitions. By moving to a usePathname listener, the UI now reacts to successful routing rather than attempting to anticipate it.

Reason

Administrators reported "dead clicks" in the mobile and desktop user menus. Ensuring a seamless management experience is critical for platform operational efficiency.

2. Developer Manual Updates

State Management Standardization

Moved isMobileMenuOpen to the top-level state block in Navbar.jsx to prevent initialization race conditions when the component mounts or updates.

Admin Auth Logic Validation

The AdminLayout.jsx component has been verified to correctly implement a three-tier check:

  • Local Presence: Immediate redirect if no user is found in client storage.
  • Server Verification: Explicit /api/auth/admin/check call to verify server-side session and role.
  • Role Fallback: Local state check against accountType === 'ADMIN' for offline/intermittent connectivity scenarios.

3. System Admin Manual Updates

API Endpoint Stability

The /api/auth/admin/check endpoint is the source of truth for administrative privileges. Admins must ensure that the ADMIN_EMAILS environment variable is correctly populated in the .env file for bootstrap admin accounts.

Session Timeout Parameters

The AdminLayout now pulls session timeout settings from /api/system/settings. Admins can configure sessionTimeout (default 15 mins) and sessionWarning (default 1 min) to control the idle logout behavior.

4. User Manual Updates

Mobile Menu Behavior

The mobile menu drawer now automatically closes upon selecting any navigation item. This includes the Admin Dashboard, My Orders, and Become a Seller links.

Admin Badge

The red "Admin" badge in the navbar and user profile header confirms active administrative status. If this badge disappears, the session may have expired or roles were updated.

5. Issues & Technical Fixes

Fix Summary

  • Navbar: Removed all onClick={() => setShowUserMenu(false)} from Link tags.
  • Routing: Added useEffect(() => { ... }, [pathname]) to handle all overlay closures.
  • Security: Validated AdminLayout logic to ensure isAdmin state is set only after server-side verification.
  • Cleanup: Standardized isMobileMenuOpen declaration position.

6. Risk Assessment & Mitigation

Potential Risk: Redirect Loops

If the admin check API fails due to network issues, the user might get stuck in a redirect loop between the dashboard and login.

Mitigation: Graceful Fallback

Implemented a fallback check against the locally stored accountType in AdminLayout.jsx when the API returns a non-200 status, preventing loops during minor network interruptions.

  • Centralized Permission Hook: Create a usePermissions() hook to standardize access checks across the entire ecosystem.
  • Admin Audit Logs: Implement real-time audit logging for all navigation within the admin dashboard.
  • Pre-fetching: Use Next.js prefetch intelligently for admin sub-routes to improve perceived performance.
