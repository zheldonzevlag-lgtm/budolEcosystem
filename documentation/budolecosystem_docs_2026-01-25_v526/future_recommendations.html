Future Enhancements & Recommendations

While v526 significantly improves synchronization, the following enhancements are recommended for future versions.

WebSocket-first Sync

Medium Priority

Transition from Pusher (HTTP-based polling internally) to a native WebSocket implementation using Socket.io for even lower latency and reduced infrastructure costs.

Biometric Transaction Signing

High Priority

Integrate FaceID/Fingerprint signing for the process-qr endpoint on mobile to further enhance security and compliance with upcoming NPC guidelines.

Edge-based Event Routing

Medium Priority

Use edge functions (e.g., Vercel Edge or Cloudflare Workers) to handle webhook processing closer to the user, reducing the global latency for international transactions.
