System Admin Operations: v526

As of v526, the admin dashboard features enhanced real-time visibility into all ecosystem transactions, powered by the TransactionPulse component.

Real-Time Transaction Monitoring

TransactionPulse Dashboard

The TransactionPulse component in the admin dashboard listens for new_transaction events across the entire ecosystem. Admins can now see payments from budolShap appearing instantly as they are processed by budolPay.

  • Channel: admin-transactions
  • Event: new_transaction
  • Data Included: Reference ID, Amount, Status, and Source (budolShap/Mobile).

Sync Troubleshooting

Pro Tip: If a user reports a "Payment Successful" screen on mobile but "Pending" on budolShap, check the payment-gateway-service logs for webhook delivery status to the /api/webhooks/budolpay endpoint.

Common Checks:

  • Pusher Connectivity: Ensure the Pusher credentials in .env are consistent across wallet-service, payment-gateway-service, and budolshap.
  • Webhook Latency: Monitor the response time of the budolShap webhook endpoint. It must return a 200 OK within 5 seconds to avoid retries.
  • Redis Cache: The ordersService.js uses SWR for caching. If status doesn't update, verify that the order-updated event successfully triggered a cache revalidation on the frontend.

Audit Compliance

Administrators can export transaction logs that include the realtime_delivered_at flag, which confirms that the user was notified of the payment status within the 2-second SLA required by internal compliance standards.
