Fixes & Improvements: v526

This document details the code-level fixes implemented to resolve the synchronization issues.

Fix: ISS-526-01

Event Name Alignment

Modified ordersService.js to use the standardized order-updated event name. This ensures that the RealtimeProvider.jsx on the frontend can correctly identify and process the event.

Fix: ISS-526-02

Real-time Mobile Listener

Implemented a StreamSubscription in the Flutter mobile app's PaymentStatusScreen. The app now listens for transaction_update events on the user's private channel, allowing for instantaneous UI updates when the wallet-service completes a payment.

Fix: ISS-526-03

Test Authentication Helper

Created generate_test_token.mjs to programmatically generate valid JWT tokens for automated testing. This resolved the 401 Unauthorized errors in the end-to-end sync validation script.

Improvement: UI Consistency

Unified Payment UI

Updated QRCodeModal.jsx to ensure that budolPay transactions show the same "Verifying" and "Success" states as GCash, providing a consistent brand experience across all payment methods.
