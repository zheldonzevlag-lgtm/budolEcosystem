Technical Architecture: Cross-Platform Sync

Version v526 introduces a robust, multi-layered synchronization strategy to ensure that payment events are propagated instantly across all user touchpoints.

Cross-Platform Sync Flow

[budolPay Mobile App] <--- (transaction_update) --- [Wallet Service] | ^ (QR Processed) | (Webhook) | | v v [Payment Gateway] --- (order-updated) ---> [budolShap Web] | ^ |--- (new_transaction) ---> [budolPay Admin] --|

Backend Implementation

1. Real-time Event Harmonization

We resolved a critical event name mismatch in the ordersService.js which prevented the budolShap frontend from receiving real-time status updates.

budolshap-0.1.0/lib/services/ordersService.js
// Unified event name to 'order-updated' to match RealtimeProvider.jsx pusher.trigger(`store-${storeId}`, 'order-updated', { orderId, status }); pusher.trigger(`user-${userId}`, 'order-updated', { orderId, status });

See code: ordersService.js

2. Wallet-to-Mobile Direct Sync

The wallet-service now broadcasts transaction_update events directly to the mobile app immediately after a QR payment is processed, eliminating the need for aggressive polling.

budolpay-0.1.0/services/wallet-service/index.js
const notifyTransactionUpdate = (userId, transaction) => { pusher.trigger(`user-${userId}`, 'transaction_update', { id: transaction.id, referenceId: transaction.referenceId, status: transaction.status, amount: transaction.amount, type: transaction.type, updatedAt: transaction.updatedAt }); };

See code: wallet-service/index.js

Frontend Real-time Integration

budolShap's QRCodeModal.jsx was updated to support provider-agnostic status polling and real-time event listening.

// Polling fallback with provider support const checkStatus = async () => { const res = await fetch(`/api/status/${referenceId}?provider=${provider}`); const data = await res.json(); if (data.status === 'succeeded' || data.status === 'COMPLETED') { onSuccess(data); } };

See code: QRCodeModal.jsx

Mobile Stream Synchronization

The Flutter mobile app now uses a StreamSubscription in PaymentStatusScreen to listen for real-time updates from the wallet service.

budolpayMobile/lib/screens/payment_status_screen.dart
_subscription = PusherService.instance.eventStream.listen((event) { if (event.channelName == 'user-${widget.userId}' && event.eventName == 'transaction_update') { final data = jsonDecode(event.data); if (data['referenceId'] == widget.referenceId) { setState(() { _status = data['status']; }); } } });

See code: payment_status_screen.dart

Compliance Note (BSP Circular 808)

All real-time events are logged with high-resolution timestamps in the audit_logs table to ensure forensic traceability of transaction state transitions, meeting BSP requirements for electronic payment systems.
