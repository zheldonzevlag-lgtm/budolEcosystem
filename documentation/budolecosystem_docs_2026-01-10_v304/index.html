Environment Manager Path Fixes

Status: Resolved & Verified

Version: v304.0.0 | Date: 2026-01-10

Purpose

Resolve the ENOENT error during environment switching (npm run db:local) by implementing robust path validation in the environment management scripts. This ensures that the ecosystem can synchronize .env files across different services even if they have varying directory structures.

Issue Description

The env-manager.cjs and env-manager.js scripts were hardcoded with paths specific to the BudolPay service layout. When executed from BudolShap (which has a different structure), the scripts attempted to write to non-existent directories (e.g., packages/database/), resulting in a fatal ENOENT error.

Fixes Applied

  • Directory Existence Check: Added fs.existsSync(path.dirname(envPath)) checks before any write operations.
  • Graceful Skipping: Scripts now automatically skip paths that don't exist in the local project structure, while still synchronizing ecosystem-wide files (BudolID, BudolAccounting, etc.).
  • Cross-Service Synchronization: Verified that both budolpay-0.1.0 and budolshap-0.1.0 can now successfully switch environments without errors.

Code Implementation

envPaths.forEach(envPath => {
    // Check if the directory exists before attempting to write
    const dir = path.dirname(envPath);
    if (!fs.existsSync(dir)) {
        // If it's a service/app/package that doesn't exist in this project structure, skip it
        return;
    }

    // Read Target (.env) if exists, or start empty
    let targetContent = '';
    if (fs.existsSync(envPath)) {
        targetContent = fs.readFileSync(envPath, 'utf8');
    }
    // ... rest of the sync logic
});

Test Results

  • BudolShap (db:local): PASSED (Successfully updated local and ecosystem .env files)
  • BudolPay (db:local): PASSED (Successfully updated all 14 target .env files)
  • Ecosystem Root (.env): PASSED (Correctly synchronized with source configuration)

Risk & Mitigation

Risk: Some services might miss updates if their directories are renamed or moved.

Mitigation: Maintain a standard directory layout and update the envPaths array in the scripts if structural changes occur.
