Issues & Fixes: Phase 2

Problem: Category icons missing in Store Management View

Product records in the Store Management table showed broken or missing icons because the Product API wasn't returning Category metadata.

Solution: Updated `app/api/products/route.js` to include `categoryData` relation in GET queries.

Problem: Icon Picker modal overflow

On small screens, the grid of 60+ icons pushed the "Save" button off-screen.

Solution: Added `max-h-[70vh]` and `overflow-y-auto` to the modal form container.

Problem: Case sensitivity in icon resolution

Storing "smartphone" (lowercase) in the DB failed to resolve to the "Smartphone" component.

Solution: Normalized matching in `resolveIconByName` using `.toLowerCase()` on both map keys and lookup query.
