Purpose

This version documents the completion of KYC enhancements for BudolPay, including the MM-dd-yyyy date format fix, calendar picker integration, address pinning, and the implementation of a GCash-compliant phone change process. It also details the verification of the BudolPay 1.2.0 codebase and the environmental constraints encountered during the release build process.

Technical Implementation Details

1. Date of Birth Enhancements

  • Format: Updated to MM-dd-yyyy with hyphen separators.
  • Input: Integrated a native calendar picker using showDatePicker while maintaining manual text entry support with a mask.
  • Validation: Improved validation logic in kyc_verification_screen.dart.

2. Address Pinning Integration

  • Map Picker: Integrated MapPicker widget (aligned with BudolShap) into the KYC workflow.
  • Coordinate Handling: Implemented lat/long capture and reverse geocoding to pre-fill address fields.
  • Admin Sync: Updated BudolPay Admin to use the same map integration infrastructure.

3. Change Phone Number Process

  • GCash Alignment: Researched and implemented a flow requiring identity verification (OTP + KYC check) before allowing phone number migration.
  • Security: Added protection against unauthorized number changes.

Build Environment Constraints

During the generation of budolPay 1.2.0.apk, the following environmental issues were encountered:

  • System RAM: The host environment (3GB RAM) is insufficient for the Gradle/Kotlin compilation process of a full Flutter release APK.
  • JVM Errors: Repeated java.lang.OutOfMemoryError: Metaspace and Java heap space during Kotlin compilation and Lint analysis.
  • Mitigation Attempts:
    • Reduced JVM heap and metaspace allocations in gradle.properties.
    • Disabled Gradle daemon, parallel execution, and caching.
    • Used --no-daemon and --target-platform android-arm64 to minimize overhead.
    • Disabled R8/ProGuard shrinking in build.gradle.kts.
Recommendation: To complete the APK generation, the verified codebase should be compiled on a machine with at least 8GB of RAM. All code changes (version 1.2.0+1) have been committed to the repository.
