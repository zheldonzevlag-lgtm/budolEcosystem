Developer Manual - v244 Updates

Scan to Pay Implementation

The QRScannerScreen has been updated with a Cancel button. The button is implemented as an OutlinedButton inside a Positioned stack element to ensure it overlays correctly on top of the MobileScanner view.

// lib/screens/qr_scanner_screen.dart
Positioned(
  bottom: 40,
  left: 0,
  right: 0,
  child: Column(
    children: [
      // ... helper text
      OutlinedButton(
        onPressed: () => Navigator.pop(context),
        style: OutlinedButton.styleFrom(
          side: const BorderSide(color: Colors.white70, width: 1.5),
          foregroundColor: Colors.white,
          // ...
        ),
        child: const Text('CANCEL'),
      ),
    ],
  ),
)

Test Suite Fixes

Unit tests in api_service_test.dart were updated to match the named parameter signatures introduced in v243. Specifically, processPayment calls now require the qrData: keyword.

// test/api_service_test.dart
expect(
  () => apiService.processPayment(qrData: {'amount': 100}),
  throwsA(isA()),
);

Flutter 3.27 Deprecation Fixes

All instances of withOpacity(double) have been migrated to withValues(alpha: double) to align with the latest Flutter engine standards and prevent precision loss in color rendering.
