System Admin Manual - v487

1. Configuration

The in-page modal update relies on existing environment variables. No new variables are introduced in v487.

  • NEXT_PUBLIC_PAYMENT_GATEWAY_URL: Endpoint for payment status polling.
  • PAYMONGO_SECRET_KEY: Used by the backend to verify transaction status.

2. Monitoring & Logging

Admins should monitor the following endpoints for performance and error rates:

  • /api/payment/checkout: Monitor for 500 errors during modal initiation.
  • /api/paymongo/status: Monitor for polling timeouts or API rate limiting.
Note: Forensic audit logs now capture modal_triggered events to differentiate between checkout initiation and actual payment confirmation.

3. Troubleshooting

Common Issues

  • Modal doesn't appear: Check browser console for JS errors in OrderSummary.jsx. Usually caused by invalid QR data structure from the gateway.
  • Timer stuck at 10:00: Ensure the client system clock is synchronized. The timer relies on local intervals but should ideally be synced with server time in future versions.
