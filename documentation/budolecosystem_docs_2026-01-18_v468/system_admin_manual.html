System Admin Manual - v468

Monitoring Timezone Compliance

Administrators can now rely on standardized timestamps in the Admin Dashboard. All telemetry, audit logs, and security events are recorded in Asia/Manila (GMT+8).

Audit Requirement: When providing logs to regulators (BSP, BIR), ensure that the timestamps are interpreted as UTC+8. The system converts these to ISO strings with a shifted baseline for consistency.

Compliance Verification

A new automated compliance test script is available for periodic verification:

node "scripts/test_scripts/test-timezone-compliance.js"

Security Logs

Login attempts and rate-limit triggers are now synchronized with Manila time, preventing "future-dated" or "expired" security tokens due to server clock drift from UTC.
