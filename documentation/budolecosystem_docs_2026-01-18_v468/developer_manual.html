Developer Manual - Timezone Compliance

Rationale

Under BSP Circular 808 and PCI DSS requirements, all financial and operational logs must be timestamped accurately. Given our primary operation is in the Philippines, all timestamps generated by the system must reflect the Asia/Manila (UTC+8) timezone to avoid discrepancies during forensic audits.

Technical Implementation

We implemented a standard offset for all new Date() and Date.now() calls that generate persistent timestamps or relative time windows.

Core Pattern

// Compliant Manila Timestamp
const manilaNow = new Date(new Date().getTime() + (8 * 60 * 60 * 1000));
const manilaISO = manilaNow.toISOString();

Key Changes

1. Lalamove Webhook Handler

Updated to ensure all processing timestamps, return deadlines, and status updates use Manila time.

File: route.js

// Example: Return Deadline Calculation (2 days)
returnDataUpdate.deadline = new Date(new Date().getTime() + (8 * 60 * 60 * 1000) + (2 * 24 * 60 * 60 * 1000));

2. Fallback Shipping Provider

Ensures quotes and estimated delivery times are calculated based on local time.

File: fallbackProvider.js

// Example: Quote Expiry (30 minutes)
expiresAt: new Date(new Date().getTime() + (8 * 60 * 60 * 1000) + 30 * 60000).toISOString();

3. Rate Limiting Logic

Prevents timezone drift from affecting security throttling.

File: rate-limit.js

const now = new Date().getTime() + (8 * 60 * 60 * 1000);
