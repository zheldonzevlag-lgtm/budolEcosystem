Version 320

System Admin Manual

Deployment and configuration guide for budolPay Mobile 1.2.47.

Purpose

This document defines the release operational procedures for budolPay Mobile version 1.2.47 and the associated Android APK artifact management in the budolEcosystem installer directory.

Explanation & Rationale

Versioned artifacts and consistent deployment steps reduce operational risk, support traceability for NPC/BSP/BIR audits, and ensure a repeatable rollout process. This release also includes functional biometric login, so admin validation must include device capability checks and secure storage behavior validation.

Issues & Fixes

  • Issue: Release artifacts were not centralized.
    Fix: Release APK is stored in \budolEcosystem\android-installer using a consistent name (budolPay_1.2.47.apk).
  • Issue: Operational verification lacked a repeatable checklist.
    Fix: Added an E2E checklist script at scripts\test_scripts\test_biometric_login.dart.

Dependencies

  • Build tooling: Flutter SDK installed on the build machine.
  • Android signing: Release signing configuration must be available in the build environment.
  • Device capabilities: Android 6.0+ for fingerprint support; biometrics must be enrolled on the device.

Technical Details

Artifact Location

Release APK:

d:\IT Projects\budolEcosystem\android-installer\budolPay_1.2.47.apk

Build Command (Reference)

cd budolPayMobile
flutter build apk --release

Biometric Permissions (Maintained from v319)

<uses-permission android:name="android.permission.USE_BIOMETRIC" />
<uses-permission android:name="android.permission.USE_FINGERPRINT" />

Code Details

Operationally relevant version sources (used for display/API headers):

  • budolPayMobile/pubspec.yaml version: 1.2.47+47
  • budolPayMobile/lib/services/api_service.dart fallback app version: 1.2.47

Risk & Mitigation

  • Risk: Installing the wrong APK version on test devices.
    Mitigation: Validate file name and file timestamp in android-installer; validate in-app splash version display.
  • Risk: Biometric login fails due to missing enrollment or device policy restrictions.
    Mitigation: Confirm biometrics are enrolled and verify the OS biometric prompt appears on login.
  • Risk: Sensitive data exposure.
    Mitigation: PIN is stored in OS-protected secure storage; no biometric data is stored or transmitted.

Test Results

  • Build: flutter build apk --release succeeded.
  • Static analysis: flutter analyze returned no issues.
  • Automated tests: flutter test completed successfully.
  • E2E checklist: dart run scripts\test_scripts\test_biometric_login.dart validated APK presence and provided manual steps.

Recommendation for Future Enhancement

Introduce signed release provenance and checksum publishing (SHA-256) for every artifact in android-installer to strengthen integrity controls and simplify incident response verification.
