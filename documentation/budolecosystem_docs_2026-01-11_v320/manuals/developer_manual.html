Version 320

Developer Manual

Technical implementation details for Mobile App Build v1.2.47 and Version Synchronization.

Purpose

The purpose of this update is to formalize the release of budolPay Mobile version 1.2.47, ensuring all configuration files and build artifacts are synchronized and correctly stored in the centralized ecosystem installer directory.

Explanation & Rationale

Consistent versioning across pubspec.yaml and internal services (like api_service.dart) is critical for debugging and API gateway compatibility. Centralizing build artifacts in \budolEcosystem\android-installer allows for easier distribution and version tracking by system administrators.

Issues & Fixes

  • Issue: App version was out of sync (1.2.46) with the latest biometric features.
    Fix: Incremented version to 1.2.47 in pubspec.yaml and api_service.dart.
  • Issue: Build artifacts were scattered in default Flutter build directories.
    Fix: Automated the movement of the release APK to the centralized android-installer folder with a standardized naming convention.

Technical Details

Version Configuration

The following files were updated to reflect version 1.2.47:

  • budolPayMobile/pubspec.yaml: version: 1.2.47+47
  • budolPayMobile/lib/services/api_service.dart: String _appVersion = '1.2.47';

Build Procedure


cd budolPayMobile
flutter build apk --release
Move-Item -Path build\app\outputs\flutter-apk\app-release.apk -Destination ..\android-installer\budolPay_1.2.47.apk -Force

Code Details

Version Synchronization


// lib/services/api_service.dart
class ApiService {
  // ...
  String _appVersion = '1.2.47'; // Fallback for display and headers
  // ...
}

Risk & Mitigation

  • Risk: Version mismatch between APK and API Gateway.
    Mitigation: Manual verification of the _appVersion string in the service layer during the build process.
  • Risk: Corrupted build artifact.
    Mitigation: Use of Flutter's built-in integrity checks during flutter build apk.

Test Results

Build integrity and version verification:

  • ✅ flutter build apk --release completed successfully.
  • ✅ APK moved to \budolEcosystem\android-installer\budolPay_1.2.47.apk.
  • ✅ Verified version 1.2.47 is displayed on the Splash Screen.
  • ✅ Verified version 1.2.47 is correctly reported in API headers.

Recommendation for Future Enhancement

Implement an automated CI/CD pipeline that automatically increments the version and moves the build artifact upon every merge to the main branch. Integrate ProGuard for further APK obfuscation and size reduction.
