Implementation Plan - v199: Unified Notification Settings

1. Route & Navigation Update

Renamed the /settings/sms directory to /settings/notifications to reflect the broader scope. Updated Sidebar.tsx to change the submenu title from "SMS API Selection" to "Notification".

2. Notification Dashboard Redesign

Implemented a dual-column layout for Email and SMS providers based on the provided technical specifications. Each provider is now represented as an individual card with dedicated configuration fields.

3. Provider Expansion

Added support for new providers: Email (Google/SMTP, Brevo, GMass) and SMS (Viber, ITexMo, Brevo SMS, Zerix), in addition to maintaining existing configurations.

4. Server-Side Logic Integration

Refactored the updateSetting server action to handle dynamic configuration updates for both Email and SMS groups, ensuring data persistence in the SystemSetting model.
