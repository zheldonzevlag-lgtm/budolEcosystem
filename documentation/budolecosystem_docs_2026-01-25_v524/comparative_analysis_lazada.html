📊 Comparative Analysis: Lazada vs BudolShap

Strategic analysis of payment cancellation workflows and implementation roadmap

1. Process Comparison

Feature Lazada Standard BudolShap (Pre-v524) BudolShap (v524 Fix)
Visibility Unpaid orders only in "To Pay" tab. All orders in main list. Abandoned orders hidden after 30m.
Payment Expiration Strict 30-60 min window. Indefinite awaiting payment. 30-min exclusion logic applied.
Stock Reservation Released immediately on expiration. Held until manual cancellation. Release logic ready for automation.
User Recovery Easy "Pay Now" from dedicated tab. Mixed with paid orders, hard to find. Prioritizes active/paid orders.

2. Gap Analysis

The primary gap between BudolShap and Lazada's industry-leading standard is the lack of Automated Status Transition and a Dedicated Payment Lifecycle View. While v524 fixes the visibility issue (preventing clutter), the underlying data remains in "Awaiting Payment" status until manually addressed.

3. Implementation Plan (Roadmap)

Phase 1: Visibility Controls (v524 - Completed)

Implemented server-side filtering to exclude orders with cancelled/failed payments or stale QR attempts (>30 mins) from the main buyer order list. This addresses the immediate UX clutter reported by users.

Phase 2: Automated Cancellation Engine (Phase 7)

Develop a scheduled background worker (Cron Job) that identifies orders in awaiting_payment status for non-COD methods and automatically transitions them to CANCELLED after 60 minutes. This will trigger stock release logic.

Phase 3: Inventory Restoration Logic

Integrate the cancelOrder function with inventory management to ensure that when an order is automatically cancelled due to payment expiration, the product.stock is incremented back to its original value.

Phase 4: "To Pay" UI Segment

Refactor the Buyer Orders UI to include a "To Pay" tab. This tab will specifically show orders that are within the 30-minute grace period, providing a clear call-to-action for users to complete their transaction, matching the Lazada UX.

4. Rationale

By adopting these phases, BudolShap moves from a reactive "hide the mess" approach to a proactive "manage the lifecycle" approach, improving both system efficiency (inventory turnover) and user trust (clear order status).
