🐛 Known Issues - v524

Documented limitations and issues in the current release

1. User Interface & Experience

Issue I-524-001: Missing "Cancelled Orders" View

Description: Currently, cancelled or failed orders are completely hidden from the user. There is no way for a user to view their history of failed transactions if they want to retry or check what happened.

Workaround: Users must contact support or check their email for payment failure notifications (if configured).

Status: Scheduled for Phase 7 implementation.

2. Technical Limitations

Issue I-524-002: Hardcoded 30-Minute Grace Period

Description: The 30-minute window for unpaid non-COD orders is hardcoded in ordersService.js. This makes it difficult to adjust without a full deployment.

Workaround: Manual code update and deployment required to change the duration.

Status: To be moved to environment variables or database configuration in v525.

Issue I-524-003: Real-time Delay on Slow Connections

Description: On very slow mobile connections, the Pusher/Socket.io event might take several seconds to update the order list after a payment failure, leading to a temporary "stale" state.

Workaround: Refreshing the page manually will fetch the correct state from the server.

Status: Investigating optimistic UI updates for future versions.
