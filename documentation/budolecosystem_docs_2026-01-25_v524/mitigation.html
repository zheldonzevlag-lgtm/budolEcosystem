🛡️ Mitigation Strategies - v524

Detailed plans to address identified risks

1. Addressing Risk R-524-001 (User Confusion)

Strategy: Clear UI Feedback & Education

Implementation:

  • Updated User Manual to explicitly explain why orders disappear.
  • Future Tech Fix: Implement a "Cancelled Orders" tab in the UI (Phase 7 Recommendation) so users can toggle visibility instead of hard hiding.
  • Immediate Action: Real-time status updates via Pusher ensure that if a user is staring at the screen when an order expires/fails, it updates dynamically.

2. Addressing Risk R-524-003 (Delayed Webhooks)

Strategy: Robust Status Checking & Webhook Reliability

Implementation:

  • The exclusion logic strictly checks for awaiting_payment. It does NOT exclude processing, paid, or succeeded.
  • Fail-Safe: If a webhook arrives late (e.g., 45 mins later) and updates status to paid, the order will immediately reappear in the list because the exclusion condition paymentStatus: 'awaiting_payment' will no longer be met.
  • Monitoring: System Admins should monitor the failed payment logs to identify if legitimate payments are failing.

3. Addressing Risk R-524-004 (Name Mismatch)

Strategy: Explicit Labeling

Implementation:

  • We modified OrderItem.jsx to separate the identities.
  • Primary Header: Uses order.user.name (The Account Holder).
  • Secondary Label: "Shipping To: order.address.name" (The Recipient).
  • This dual-display eliminates ambiguity while ensuring the user knows which account placed the order.

4. Addressing Risk R-524-005 (Mobile/Backend Desync)

Strategy: Versioned API & Defensive Coding

Implementation:

  • Backend: Updated /process-qr to include the transaction object. The change is backward-compatible as it only adds new fields.
  • Frontend (Mobile): Implemented defensive null-checks in PaymentStatusScreen.dart. If transactionData is missing fields, it falls back to N/A or Merchant instead of crashing.
  • Deployment Order: Wallet Service update is prioritized in the deployment pipeline before the Mobile App update.
