🔧 Fixes & Patches - v524

Bugs resolved and improvements implemented in this version

1. Critical Bug Fixes

Fix F-524-001: Customer Name Mismatch

Issue: The order list was displaying the Shipping Name (e.g., "Jon Galvez") as the primary customer name, even when another user was logged in.

Resolution: Updated OrderItem.jsx to prioritize order.user.name. Added a separate "Shipping To" field to show the recipient's name clearly.

Files: components/OrderItem.jsx

Fix F-524-002: Abandoned Orders in My Orders List

Issue: Users were seeing cancelled or failed payment orders in their "My Orders" list, causing confusion and clutter.

Resolution: Implemented backend filtering in ordersService.js to exclude cancelled, failed, and expired orders, as well as stale unpaid non-COD orders (>30 mins).

Note on "Fail/Expire" Tests: When a payment is failed or expired via a test gateway, it will be hidden from the main list as soon as the status is updated in the database. During the transition (while still awaiting_payment), it may appear for up to 30 minutes to allow the user to retry.

Files: lib/services/ordersService.js, app/api/orders/route.js

2. Improvements

Fix F-524-003: Real-time Order Synchronization

Issue: The order list did not always update immediately when an order status changed via a webhook.

Resolution: Added order-updated event binding to the useRealtimeBuyerOrders hook, ensuring UI updates instantly when the server sends a Pusher/Socket notification.

Files: hooks/useRealtimeBuyerOrders.js
