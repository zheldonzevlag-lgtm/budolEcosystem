1. Abandoned & Failed Payment Handling

We implemented a robust system to manage order visibility based on payment status. Failed and expired payments are now explicitly handled to prevent user confusion.

Logic Refinement

File: budolshap-0.1.0/lib/services/ordersService.js

The getOrders function was updated with the following rules:

  • Active Lists (All/To Pay): Orders with failed or expired payment status are immediately excluded.
  • Cancelled Tab: Orders with failed or expired payment status are now included alongside cancelled orders.

3. Customer Name Display Fix

Users reported seeing incorrect names (e.g., "Jon Galvez") instead of the logged-in user's name. This was due to the UI prioritizing the shipping address name over the user profile name.

Frontend Component

File: budolshap-0.1.0/components/OrderItem.jsx

The logic was updated to prioritize order.user?.name. We also added a specific "Shipping To:" label for clarity.

// Prioritize user's account name, fallback to address name
const customerName = order.user?.name || order.address?.name || 'Customer';

// Display in UI
<div className="font-medium text-gray-900">{customerName}</div>
<div className="text-sm text-gray-500">
    Shipping To: {order.address?.name}
</div>

3. Realtime Updates

File: budolshap-0.1.0/hooks/useRealtimeBuyerOrders.js

The useRealtimeBuyerOrders hook was updated to listen for order-updated events via Pusher/Socket.io to ensure the order list reflects status changes (e.g., payment success or cancellation) immediately.

4. Cross-Platform UI Consistency (budol₱ay Mobile)

We aligned the budol₱ay Mobile App with the budolShap Web payment experience by replacing standard bottom sheets and alert dialogs with full-page feedback screens.

New Component: PaymentStatusScreen

File: budolPayMobile/lib/screens/payment_status_screen.dart

A unified screen that handles Verifying, Success, and Failed states with distinct animations and transaction breakdowns. Key features include:

  • Elastic Animations: Uses CurvedAnimation(curve: Curves.elasticOut) for status icons.
  • Transaction Breakdown: Displays Order ID, Amount, Status, Merchant, and Date/Time in a clean card layout.
  • Responsive Actions: Dynamic buttons ("VIEW MY ORDERS", "CONTINUE SHOPPING") based on transaction outcome.

Mobile Build: budolPay 1.3.33

The mobile app was successfully built and archived in /android-installer/budolPay 1.3.33.apk. The version was incremented from 1.3.32 to 1.3.33 to reflect the major UI alignment with the web ecosystem.
