📅 Implementation Plan: Future Recommendations

Strategic Roadmap for v524 Recommendations (Q1 - Q4 2026)

Strategic Roadmap Overview

This plan outlines the execution strategy for the five key recommendations identified during the v524 release. The roadmap is divided into three distinct phases to ensure stability and compliance.

Recommendation ID Feature Name Phase Est. Timeframe Complexity
Rec-524-001 History/Archive View Phase 1 2 Weeks Medium
Rec-524-005 Audit Log for Exclusions Phase 1 2 Weeks Low
Rec-524-002 Dynamic Grace Period Phase 2 3 Weeks Medium
Rec-524-004 Inventory Reservation Phase 2 4 Weeks High
Rec-524-006 Cross-Platform UI Consistency Phase 2 3 Weeks Medium
Rec-524-003 Payment Intent System Phase 3 8 Weeks Critical

Phase 1: Visibility & Compliance (Q1 2026)

Focuses on immediate user experience improvements and fulfilling basic compliance requirements.

Rec-524-001: History/Archive View

  • Design "Archive" tab in the "My Orders" interface.
  • Implement toggle/filter in OrderItem.jsx to show hidden orders.
  • Update api/orders/route.js to handle includeArchived=true query parameter.
  • Performance testing for large order histories.

Rec-524-005: Audit Log for Exclusions

  • Create OrderExclusionAudit table in schema.prisma.
  • Update ordersService.js to trigger an audit entry whenever an order meets exclusion criteria.
  • Develop internal admin tool to view audit logs for support troubleshooting.

Phase 2: Intelligent Automation (Q2-Q3 2026)

Enhances the system with dynamic logic and inventory protection.

Rec-524-002: Dynamic Grace Period

  • Develop Admin Dashboard settings for global and per-merchant grace periods.
  • Refactor ordersService.js to fetch configuration from DB instead of using hardcoded 30-minute constants.
  • Implement notification system to warn users when their grace period is expiring.

Rec-524-004: Inventory Reservation

  • Develop a Redis-based soft reservation lock for SKU quantities.
  • Implement a 30-minute expiration for locks.
  • Add a cron job to reconcile locked stock with actual database inventory.
  • Security audit of the locking mechanism.

Rec-524-006: Cross-Platform UI Consistency

  • Design full-page "Verifying" and "Success" screens in Flutter for budolPay.
  • Implement a transaction breakdown card matching the web design.
  • Add Lottie animations for the success/failure states.
  • Update the budolPay payment flow to redirect to the new screens post-transaction.

Phase 3: Architectural Excellence (Q4 2026)

A deep overhaul of the payment lifecycle to align with international standards (Stripe-like flow).

Rec-524-003: Payment Intent System

  • Architect a new "Payment Intent" state machine.
  • Separate DB record creation into PaymentIntent (pre-payment) and Order (post-payment).
  • Migration script for historical data to fit the new schema.
  • Full security penetration testing of the new payment flow.
⚠️ Compliance Reminder: All Phase 3 changes must undergo a mandatory review by the Compliance Officer to ensure continued alignment with BSP and PCI DSS requirements.
