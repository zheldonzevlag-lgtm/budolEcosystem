🔧 System Admin Manual - v524

Operational guide for managing the new Payment Safety features

1. Monitoring Abandoned Payments

With the new exclusion logic, "abandoned" payments (cancelled/failed) are hidden from the user's main view but remain in the database for audit purposes.

Database Queries

To view excluded orders directly in the database (Prisma Studio or SQL), query for:

  • paymentStatus: 'cancelled', 'failed', 'expired'
  • paymentStatus: 'awaiting_payment' AND createdAt < (Now - 30 mins) AND paymentMethod != 'COD'

2. Cron Jobs & Automation

Unpaid Order Cleanup Every 1 hour Cancel orders > 24 hours old Release inventory held by stale orders.

3. Troubleshooting User Reports

If a user claims their order "disappeared":

  1. Check Payment Status: Was the payment failed or cancelled? If so, it is correctly hidden.
  2. Check Time Elapsed: If they selected a non-COD payment method but didn't pay within 30 minutes, it will be hidden.
  3. Verify "Shipping To" Name: Confirm if the user is confused by the "Shipping To" name vs. their account name. The new UI clarifies this.

4. Configuration

The 30-minute grace period is currently hardcoded in ordersService.js. To change this value, a code deployment is required.

  • File: lib/services/ordersService.js
  • Variable: 30 * 60 * 1000 (30 minutes in milliseconds)
