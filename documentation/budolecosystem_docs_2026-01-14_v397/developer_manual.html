Refined Biometric Flow

In version 1.2.88, we simplified the _handleFaceRecognitionLogin in login_screen.dart to prioritize seamless OS-level biometrics.


Future<void> _handleFaceRecognitionLogin() async {
  final biometricService = context.read<BiometricService>();
  
  // App-level face registration
  final hasAppFace = _hasFaceTemplate;
  
  // OS-level face registration
  final hasNativeFace = await biometricService.hasEnrolledFace();
  
  // Generic biometric enrollment
  final hasAnyBiometric = await biometricService.hasEnrolledBiometrics();

  if (hasAppFace || hasNativeFace) {
    // Zero-Action OS Prompt
    return _handleBiometricLogin();
  }

  if (hasAnyBiometric && _isBiometricEnabled) {
    // Fallback to Fingerprint
    return _handleBiometricLogin();
  }

  _showError('No face profile or biometric login enabled.');
}

Rationale

The manual TFLite flow (camera capture -> crop -> embedding -> compare) was found to be too high-friction for daily login. By leveraging the local_auth OS prompt, we provide a secure, system-native experience while keeping the custom face template as a valid "opt-in" trigger for this flow.
