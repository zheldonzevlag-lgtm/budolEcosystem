Issues & Fixes Log

1. Jest Configuration Conflict

Issue: Running tests failed with "Implicit config resolution does not allow multiple configuration files".

Cause: Jest detected multiple potential config files or conflicts in the project root.

Fix: Explicitly specified the configuration file in the test command.

npm test ... -- --config=jest.config.cjs

2. Accessibility / Test Selector Failure

Issue: screen.getByLabelText(/Product Name/i) failed in tests.

Cause: Inputs were not associated with labels via id/for attributes, nor nested inside labels.

Fix: Added htmlFor="name" to label and id="name" to input in AddProductWizard.jsx.

3. Image Sorting Instability

Issue: Drag-and-drop sorting in DragDropImageUpload was glitchy when using base64 strings as keys.

Fix: Refactored to use object-based state { id, url } where id is a unique string, ensuring stable keys for dnd-kit.

4. Product Data Persistence Failure (Shipping & Condition)

Issue: User reported "data values are not save" and "Failed to update product". Specifically, shipping dimensions (weight, length, width, height), condition, and pre-order status were not persisting to the database.

Cause: The API endpoints (POST /api/products and PUT /api/products/[id]) were not extracting these fields from the request body, and the Prisma schema was missing the preOrder field definition.

Fix:

  • Updated prisma/schema.prisma to include preOrder Boolean @default(false).
  • Modified API route handlers to destructure and save all shipping/condition fields.
  • Fixed BigInt serialization error in database backup script to enable safe backups before schema changes.

// prisma/schema.prisma model Product { ... preOrder Boolean @default(false) ... }

5. Infinite Loop on Cart Page

Issue: User reported infinite loop with repeated GET requests to /api/products?ids=....

Cause: The useEffect hook in Cart component was repeatedly fetching missing product IDs that were never successfully resolved (e.g., deleted products), causing state updates that re-triggered the effect.

Fix: Introduced attemptedIdsRef (useRef) to track IDs that have already been fetched (success or fail) and prevent re-fetching them within the same component lifecycle.

// app/(public)/cart/page.jsx const attemptedIdsRef = useRef(new Set()); // ... const missingIds = productIds.filter(id => !productsMap[id] && !attemptedIdsRef.current.has(id));

6. Prisma Client Generation Error

Issue: Build error ENOENT: no such file or directory, open '.../@prisma/client-custom/index.js'.

Cause: The custom Prisma client output directory was not generated after package installation.

Fix: Ran npx prisma generate to rebuild the client and added "postinstall": "prisma generate" to package.json to ensure automatic generation in the future.
