System Admin Manual - v5.0

1. Database Schema Changes

No new tables were added in this version. However, ensure `Product` table columns `storeId`, `variation_matrix`, and `tier_variations` are correctly indexed if performance issues arise with large catalogs.

2. Environment Configuration

Ensure `CLOUDINARY_URL` is set in the `.env` file for image uploads to work. The system now supports client-side compression to reduce bandwidth usage, but the backend upload endpoint remains the same.

3. Troubleshooting Image Uploads

If users report "Upload failed", check:

  • Cloudinary quota and API status.
  • Network connectivity for client-side uploads.
  • Server logs for `POST /api/upload` failures.

4. Maintenance

Regularly prune temporary files if local storage fallback is enabled (currently defaulting to Cloudinary or base64 fallback in database - Warning: Base64 in DB can bloat backups; monitor `Product` table size).
