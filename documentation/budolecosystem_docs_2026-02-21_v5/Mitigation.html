Mitigation Plan - v5.0

R-001 Database Bloat (Base64 Fallback)

Strategy: Monitor usage. If Base64 storage exceeds 1GB, implement a cron job to upload these to Cloudinary asynchronously and update the DB records with URLs.

R-002 Image Quality

Strategy: Enforce minimum resolution checks in `DragDropImageUpload` before compression. Current setting is 0.8 quality which is standard.

R-003 UI Lag (Variations)

Strategy: The `VariationMatrixManager` uses pagination or virtualization if the matrix exceeds 50 rows. Currently optimized for standard use cases (max ~20 variants).

R-004 Missing Store ID

Strategy: Implemented. The `page.jsx` now fetches the `storeId` before rendering the wizard, and the API enforces `storeId` presence. If fetch fails, user is redirected/alerted.
