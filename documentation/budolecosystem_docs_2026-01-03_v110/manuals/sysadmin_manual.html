SysAdmin Manual - v110

Infrastructure Configuration

The mobile app relies on the API Gateway being accessible. Ensure the following environment variables are correctly set during the build or via --dart-define.

Environment Variables

  • GATEWAY_URL: The base URL for the API Gateway (e.g., https://budolpay-api-gateway.vercel.app for production or http://192.168.1.24:8080 for LAN).
  • API_HOST: Used as a fallback for LAN development.

Service Dependencies

For the mobile app to function correctly, the following services must be healthy:

  • API Gateway: Proxies all requests to internal services.
  • Auth Service: Handles SSO login and token validation.
  • Wallet Service: Manages balances and processes QR payments.
  • Payment Gateway Service: Generates the QR codes that the app scans.

Troubleshooting Integration

If payments fail, check the following:

  1. Connectivity: Ensure the mobile device can reach the GATEWAY_URL.
  2. CORS/Proxy: Verify the API Gateway correctly forwards POST bodies for /wallet/process-qr.
  3. Logs: Check the Wallet Service logs for "Processing QR Payment" entries to see the incoming payload from the mobile app.
