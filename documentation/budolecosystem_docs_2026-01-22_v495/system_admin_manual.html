System Admin Manual: Realtime Configuration

Instructions for managing the ecosystem's realtime update engine.

Configuring the Realtime Provider

1. Access Settings: Log in to the Admin Dashboard and navigate to System Settings > Realtime.
2. Choose Provider:
  • Polling: Best for low-traffic sites. No external dependencies. (Recommended for Dev/Staging).
  • Pusher: Industry standard. Requires Pusher.com account. Provides the lowest latency.
  • Socket.io: Self-hosted alternative. Requires a running Budol Socket Microservice.
3. Enter Credentials: Provide the necessary keys/URLs for your chosen provider.
4. Save & Propagate: Click "Save Settings". The system will update globally within 60 seconds due to cache TTL.

Troubleshooting Provider Connection

If realtime updates stop working:

  1. Switch provider back to Polling to verify core system functionality.
  2. Check browser console for Pusher or Socket.io connection errors.
  3. Verify that the configured socketUrl or Pusher keys are correct and accessible from the client's network.
