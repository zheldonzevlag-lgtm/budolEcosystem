System Admin Manual

Local Startup Procedure

  1. Database Backup: Always backup your databases before making schema changes. Use node scripts/backup-db-standard.js to backup to \backup-db\db-YYYY-MM-DD.
  2. Ecosystem Startup: Run npm start and select 1 for LOCAL environment. This will automatically switch environments and start all services.
  3. Network Configuration: The ecosystem automatically detects your local IP and updates the .env files to ensure network-aware communication between services.

Database Management

The Budol Ecosystem uses multiple databases locally:

  • budolid: SSO and Identity Management
  • budolpay: Financial services (uses the budolpay schema)
  • budolshap: Main application data
  • budolaccounting: Financial accounting and ledger entries

Prisma Commands

If you make changes to the database schema, run the following commands in the relevant sub-project:

  • npx prisma generate: Update the Prisma client after schema changes.
  • npx prisma db push: Sync the local database with your Prisma schema (useful for development).
