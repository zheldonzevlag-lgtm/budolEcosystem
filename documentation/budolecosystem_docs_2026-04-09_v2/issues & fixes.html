Issues & Fixes

Issue: Rendered fewer hooks than expected (RealtimeProvider)

Cause: A conditional return statement was placed before React hooks in RealtimeProvider.tsx, violating the "Rules of Hooks" when navigating between the login page and the dashboard.

Fix: Corrected Hook Order

Action: Moved the conditional pathname === '/login' check to occur after all hook definitions (useState, useRef, useEffect).

Issue: Internal Server Error (500) during login or system events

Cause: The AuditLog table had a foreign key constraint to the User table. Logging events for non-existent users (like 'SYSTEM' or failed login attempts) triggered a P2003 Prisma error.

Fix: Graceful Audit Log Fallback

Action: Updated lib/audit.ts to catch foreign key errors and store logs with userId: null while preserving the identity in metadata (denormalization).

Issue: Cannot find module '@budolpay/database'

Cause: Services were run using node index.js in their subdirectories without first linking the monorepo dependencies.

Fix: Linked Monorepo Dependencies

Action: Ran npm install at the root of budolpay-0.1.0 to ensure symlinks were created in node_modules.

Issue: npm error No workspaces found: --workspace=@budolpay/admin

Cause: The workspace name in apps/admin/package.json was admin, but the runner script used @budolpay/admin.

Fix: Standardized Workspace Naming

Action: Renamed the workspace in apps/admin/package.json to @budolpay/admin to match ecosystem standards.

Issue: Prisma schema validation error in api-gateway

Cause: The datasource db block was missing the url parameter.

Fix: Updated Prisma Schema

Action: Added url = env("DATABASE_URL") and corrected provider to prisma-client-js.

Issue: Database Connection Failed in payment-gateway-service

Cause: Incorrect .env path reference (../.env instead of ../../../.env) and local database tables were in public instead of budolpay schema.

Fix: Corrected Path and Schema Isolation

Action: Updated api/index.js path reference and forced local environment to use schema=budolpay via .env.connection.local.
