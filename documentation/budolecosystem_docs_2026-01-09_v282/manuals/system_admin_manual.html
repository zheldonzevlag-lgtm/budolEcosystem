System Admin Manual - v282

Operational Procedures

This manual covers the critical steps for resetting the ecosystem databases and verifying network connectivity.

Database Management

Warning: Using --force-reset will delete all existing data in the target database. Use with caution in production.

Syncing Databases

To ensure all databases are in sync with the latest Prisma schema, run the following commands in PowerShell:

# Sync budolID database
$env:DATABASE_URL="postgresql://postgres:r00t@localhost:5432/budolid?schema=public"
npx prisma db push --schema=budolpay-0.1.0/packages/database/prisma/schema.prisma --force-reset

# Sync budolPay database
$env:DATABASE_URL="postgresql://postgres:r00t@localhost:5432/budolpay?schema=public"
npx prisma db push --schema=budolpay-0.1.0/packages/database/prisma/schema.prisma --force-reset

Seeding Initial Data (LAN)

After a reset, use the seed_lan.js script to restore both the test user and the ecosystem application registrations with correct LAN redirect URIs:

# Detect and set LOCAL_IP, then seed both databases
$env:LOCAL_IP="192.168.1.24"
$env:DATABASE_URL="postgresql://.../budolid"; node budolpay-0.1.0/packages/database/seed_lan.js
$env:DATABASE_URL="postgresql://.../budolpay"; node budolpay-0.1.0/packages/database/seed_lan.js

Troubleshooting

Error: EADDRINUSE

If you see an "address already in use" error when starting the ecosystem, it means previous instances of the services are still running. You can kill them using the following PowerShell command:

# Kill processes on common ecosystem ports
Get-Process -Id (Get-NetTCPConnection -LocalPort 3001,3000,8000,8080,8005).OwningProcess | Stop-Process -Force

Network Verification

The verify-lan.js tool automates the detection of your LAN IP and tests if all core services are reachable from other computers.

node verify-lan.js

Expected output should show all services as REACHABLE on the detected LAN IP (e.g., 192.168.1.24).

Troubleshooting

  • ECONNABORTED: Usually occurs if a service (like Next.js) is still compiling. Wait 30 seconds and run verification again.
  • Invalid API Key: Ensure the EcosystemApp table is seeded in the database.
