Developer Manual - v3.3.26

This document details the technical implementation of fixes for PII masking and data integrity.

1. Backend: /verify Endpoint

The /verify endpoint in auth-service is responsible for returning fresh user data, including unmasked PII for authorized users (e.g., when editing their profile).

Location: budolpay-0.1.0/services/auth-service/index.js


// Verify Token & Get Profile (Mobile App Sync) - Universal Entry Point
app.get('/verify', async (req, res) => {
    // ... authentication logic ...
    
    // Universal User Lookup (Support SSO & Local)
    // ...
    
    res.json({
        valid: true,
        user: {
            id: user.id,
            email: user.email,
            phoneNumber: user.phoneNumber, // Unmasked for profile edit
            firstName: user.firstName,     // Unmasked for profile edit
            lastName: user.lastName,       // Unmasked for profile edit
            // ...
        }
    });
});

2. Frontend: EditProfileScreen Safety Check

A critical safety check was added to EditProfileScreen.dart to prevent saving masked data (e.g., "M*****") back to the database, which would permanently corrupt the user's name.

Location: budolPayMobile/lib/screens/edit_profile_screen.dart

CRITICAL: Do not remove this check unless you implement a robust way to distinguish between user input and masked data. 

Future<void> _handleUpdate() async {
    if (!_formKey.currentState!.validate()) return;

    // Safety Check: Prevent saving if data appears to be masked (contains asterisks)
    // This prevents overwriting valid DB data with masked "M*****" values
    if (_firstNameController.text.contains('*') || _lastNameController.text.contains('*')) {
      if (mounted) {
        ScaffoldMessenger.of(context).showSnackBar(
          const SnackBar(
            content: Text('Cannot save masked data. Please wait for profile to fully load.'),
            backgroundColor: Colors.orange,
            duration: Duration(seconds: 4),
          ),
        );
        // Try to fetch fresh data again
        context.read<ApiService>().fetchUserProfile();
      }
      return;
    }

    setState(() => _isLoading = true);
    // ... proceed with update ...
}

3. Testing Scripts

Two scripts were created/modified to verify these fixes:

  • budolpay-0.1.0/test_mobile_flow.js: Simulates the mobile app flow, including identify (masked) and verify (unmasked).
  • budolpay-0.1.0/check_user_by_phone.js: Inspects the database directly to verify user data integrity (unmasked).
