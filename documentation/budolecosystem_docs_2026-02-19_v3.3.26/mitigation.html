Mitigation Plan

Version: v3.3.26

Date: 2026-02-19

Risk R-2026-001: Data Overwrite with Masked Values

Mitigation Strategy: Client-Side Validation

Implementation:

  • Added a check in EditProfileScreen._handleUpdate().
  • Code scans the text fields for asterisk characters (*).
  • If found, the save operation is blocked, and a user-friendly SnackBar is displayed ("Cannot save masked data...").
  • The app automatically triggers a retry of fetchUserProfile() to attempt to get clean data.

Risk R-2026-002: PII Exposure in Logs

Mitigation Strategy: Conditional Logging

Implementation:

  • Used kDebugMode in Flutter to ensure logs are only printed in debug builds.
  • Backend logs should be configured to production level (INFO/WARN) where payload details are omitted, unless debugging a specific issue in a controlled environment.

Risk R-2026-003: Endpoint Confusion

Mitigation Strategy: Documentation & Naming

Implementation:

  • Clearly documented in the Developer Manual that /verify is the "Universal Entry Point" for getting the user's own profile data unmasked.
  • Other endpoints (e.g., search/find) continue to return masked data by default (Security by Design).

Risk R-2026-004: Network Latency on /verify

Mitigation Strategy: Timeout Adjustment

Implementation:

  • Increased the timeout in api_service.dart from 5 seconds to 30 seconds.
  • This accounts for potentially slower network conditions or database queries, preventing premature "failure" states in the UI.
