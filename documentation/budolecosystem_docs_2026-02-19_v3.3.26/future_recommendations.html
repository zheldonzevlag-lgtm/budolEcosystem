Future Recommendations

Version: v3.3.26

Date: 2026-02-19

1. Unified User Service

Observation: Currently, user data is split and synced between budolID and budolPay databases. This leads to potential sync gaps (as seen with the marijoy user).

Recommendation: Move towards a single "User Service" or "Identity Provider" (IdP) that serves as the single source of truth for user profile data across the ecosystem, reducing the need for dual-write or sync logic.

2. GraphQL Implementation

Observation: The mobile app often needs specific subsets of data (sometimes masked, sometimes unmasked). REST endpoints are becoming cluttered with flags and query parameters.

Recommendation: Implement GraphQL. This would allow the client to request exactly what it needs (e.g., query { user { firstName(masked: false) } }), handling the visibility logic more elegantly.

3. End-to-End Encryption (E2EE) for PII

Observation: Unmasked data travels over HTTPS, which is secure in transit, but visible to the application layer.

Recommendation: For highly sensitive fields (like future tax IDs or biometrics), consider application-level encryption where the backend decrypts only when absolutely necessary, or uses a dedicated vault service.

4. Automated UI Testing

Observation: UI issues like the "Edit Profile" empty state require manual verification.

Recommendation: Integrate Flutter Integration Tests (using integration_test package) to automatically spin up the app, log in, navigate to the profile, and assert that fields are populated and editable.
