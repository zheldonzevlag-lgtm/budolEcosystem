System Administrator Manual

Version: v3.3.26

Date: 2026-02-19

1. Deployment Instructions

Backend Services (Auth Service)

The auth-service has been updated to support unmasked data retrieval for the "Edit Profile" feature via the /verify endpoint. It also includes logic to consolidate user data retrieval.

Steps:

  1. Navigate to the auth-service directory:
    cd budolpay-0.1.0/services/auth-service
  2. Stop any running instances of the auth-service (Port 8001).
  3. Install dependencies (if not already up to date):
    npm install
  4. Start the service:
    npm start

Mobile Application (BudolPay)

The mobile application requires a rebuild to include the PII masking fixes and data integrity safety checks.

Steps:

  1. Navigate to the mobile app directory:
    cd budolPayMobile
  2. Clean the build:
    flutter clean
  3. Get dependencies:
    flutter pub get
  4. Build the APK:
    flutter build apk --release

2. Configuration Changes

Environment Variables

No new environment variables are required for this release. Ensure the existing JWT_SECRET and database connection strings are correctly configured.

Database Schema

No schema changes were made in this release. The User table structure remains compatible.

3. Monitoring & Maintenance

Logs

Monitor server.log for the auth-service. Look for entries related to /verify endpoint access.

  • Success: "User verified: [email]"
  • Error: "Token verification failed" or "User not found"

Troubleshooting

Critical: If users report "Masked data detected" errors in the mobile app, ensure the backend is running the latest version with the updated /verify endpoint.
Issue Check Resolution
Profile data shows asterisks (*) and cannot be saved. Check if the backend is returning masked data. Restart auth-service. Verify /verify endpoint is accessible.
"Edit Profile" screen is empty. Check network connectivity and logs. Ensure the mobile device can reach the auth-service host.

4. Backup & Recovery

Regular database backups should be maintained. This release does not require a specific pre-deployment backup as it involves logic changes only, but standard procedures apply.
