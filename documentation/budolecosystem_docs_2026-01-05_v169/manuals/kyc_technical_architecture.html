KYC Technical Architecture: Deep Dive

Detailed technical specifications for implementing a BSP-compliant Tiered KYC system.

1. Frontend Implementation (Flutter)

The mobile client handles the capture, preprocessing, and submission of sensitive identity data. We use a multi-step state machine to manage the verification wizard.

ID Capture & OCR

Uses the device camera to capture high-resolution images. On-device OCR extracts text to verify against user input.

  • google_mlkit_text_recognition ^0.11.0
  • camera ^0.10.5

Liveness Detection

Ensures the user is a real person by requesting random actions (blink, smile, head turn) during the selfie step.

  • google_mlkit_face_detection ^0.11.0

Core Flutter Dependencies

2. Backend Infrastructure (Microservices)

A dedicated verification-service will manage the lifecycle of a KYC application, including storage, manual review queues, and 3rd party API integrations.

Development Storage (Local)

In the development environment, images are stored in the Local PostgreSQL database using the BYTEA data type. This simplifies local setup without needing external cloud storage.

Production Storage (Vercel)

In production, we utilize Vercel Blob (Hobby Plan). It provides a high-performance, edge-optimized object storage that integrates seamlessly with our Vercel-hosted services.

Storage Tech Stack

  • Dev Database PostgreSQL (BYTEA)
  • Prod Object Store Vercel Blob
  • Image Processing Sharp (Node.js)
  • Vercel SDK @vercel/blob

3. API Specifications

The following endpoints are required for the verification-service:

POST /api/v1/kyc/start
- Initiates a verification session

POST /api/v1/kyc/upload-id
- Multipart/form-data: id_front, id_back (optional), id_type

POST /api/v1/kyc/selfie
- Multipart/form-data: selfie_image, liveness_score

GET /api/v1/kyc/status
- Returns: PENDING, APPROVED, REJECTED, or NEEDS_RECAPTURE

4. Security & Compliance (BSP Standards)

  • Data Masking: Sensitive data (e.g., full ID number) should be masked in logs and administrative dashboards.
  • Encryption in Transit: Mandatory TLS 1.3 for all mobile-to-gateway communication.
  • Audit Trails: Every change in KYC status must be logged with the reviewer's ID and timestamp.

5. Cost Analysis & Licensing

A major advantage of this architecture is the utilization of open-source and on-device technologies to minimize operational costs.

On-Device ML (Free)

Google ML Kit (Text Recognition & Face Detection) is 100% free as it runs locally on the user's device. It does not require a Google Cloud subscription for these specific APIs.

Image Processing (Free)

Sharp is an open-source library (Apache 2.0 license) for Node.js. It is free for commercial use and is significantly faster than alternative libraries like JIMP.
