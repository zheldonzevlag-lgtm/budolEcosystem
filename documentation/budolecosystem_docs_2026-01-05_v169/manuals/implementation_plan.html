Data Management Strategy

To ensure cost-efficiency and performance, we utilize a hybrid storage approach:

  • Development: Identity documents are stored as BYTEA blobs in the local PostgreSQL database. This enables offline development and simplified schema migrations.
  • Production: Documents are offloaded to Vercel Blob (Hobby Plan). This provides edge-optimized storage and reduces database size, staying within Vercel's free tier limits.

Verification Engine Architecture

The system follows a hybrid verification model to balance speed and security:

  1. Automated Check (L1): On-device ML Kit extracts data and performs basic liveness checks.
  2. Service Validation (L2): Backend service verifies extracted data against internal blacklists and formatting rules.
  3. Manual Review (L3): Flagged or high-value accounts enter a queue for human administrator approval via the BudolID admin dashboard.
