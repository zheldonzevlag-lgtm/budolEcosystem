System Admin Manual: Wallet & KYC Operations

Infrastructure management, deployment procedures, and operational runbooks for v169.

1. Purpose & Operational Overview

Purpose: To ensure high availability and security of the identity verification pipeline and financial transaction simulators.

Explanation: This manual details the setup of the Verification Service and the required infrastructure changes to support large binary data (ID images) and ML Kit integration.

Reason: Proper configuration of the gateway and internal services is critical to prevent timeouts during image processing and to ensure data integrity during KYC submissions.

2. Infrastructure Components

The v169 ecosystem adds the following internal components:

  • Verification Service: Node.js service running on port 8006.
  • API Gateway Proxy: New route /verification pointing to localhost:8006.
  • Prisma Client: Updated to include VerificationDocument model.
Critical Dependency: The sharp library requires libvips on the host OS. Ensure it is installed before running npm install.

3. Environment Configuration

Ensure the following variables are set in the root .env file:

# Service Ports
VERIFICATION_SERVICE_PORT=8006

# Gateway Routes
VERIFICATION_SERVICE_URL=http://localhost:8006

# Database
DATABASE_URL="postgresql://user:pass@localhost:5432/budolpay"

# Storage Configuration (Phase 4 Ready)
STORAGE_PROVIDER=local # Options: local, s3
UPLOAD_LIMIT=5mb

4. Deployment Runbook

Step 1: Database Migration

Run npx prisma db push to apply the new VerificationDocument table without losing existing user data.

Step 2: Service Initialization

Navigate to services/verification-service and run npm install. Verify that sharp compiles correctly.

Step 3: Ecosystem Start

The root start-ecosystem.js has been updated to automatically launch the verification service. Run node start-ecosystem.js.

Step 4: Health Verification

Curl http://localhost:8080/verification/health to confirm the gateway is correctly proxying requests.

5. Risks & Mitigations (Operations)

  • Risk: Database Bloat (Image storage). Mitigation: Implement automated cleanup of rejected documents and migrate to S3 in v170.
  • Risk: Service Port Conflict. Mitigation: Verification service was moved from 8005 to 8006 to avoid collision with budolAccounting.
  • Risk: OCR Latency. Mitigation: ML Kit processing is handled on-device (client-side) to reduce server load.

6. Technical Issues & Fixes

Issue Reason Fix
Upload Timeout Gateway default timeout was too low for large images. Increased proxyTimeout to 30s in api-gateway/index.js.
P2003 Foreign Key Attempted to upload document for non-existent user. Added userId validation in verification-service/index.js.

7. Operational Dependencies

  • libvips: Native OS library required for sharp image optimization.
  • PostgreSQL 14+: Required for BYTEA and JSONB column support.
  • Node.js 18+ (LTS): Required for the microservices runtime.

  • Implement automated database backups before running prisma db push.
