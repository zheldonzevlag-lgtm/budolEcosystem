Registration Endpoint Implementation

Purpose

The purpose of this update is to restore the universal registration flow within the budolEcosystem. By implementing a centralized /auth/register endpoint in budolID, we ensure that all client applications (like budolShap and budolPay) can register users in a single identity database, facilitating seamless SSO across the ecosystem.

Technical Details

Endpoint Specification

Route: POST /auth/register

Request Body:

{
  "email": "user@example.com",
  "password": "securepassword",
  "firstName": "John",
  "lastName": "Doe",
  "phoneNumber": "09123456789",
  "registrationIp": "127.0.0.1",
  "deviceFingerprint": "browser-fingerprint-hash"
}

Code Logic

The implementation uses bcryptjs for secure password hashing and Prisma for database operations. It includes a check for existing users to prevent duplicate registrations.

// Create user in budolID database
const user = await prisma.user.create({
    data: {
        email,
        passwordHash,
        firstName,
        lastName,
        phoneNumber,
        role: 'USER'
    }
});

Test Results

The endpoint was verified using Invoke-RestMethod. Successful registration returns a 201 Created status with the user ID and fraud analysis flags.

Status: Success

Response: { "message": "User registered successfully", "userId": "...", "fraudAnalysis": { "risk": "LOW" } }

Dependencies

  • express: Web framework
  • bcryptjs: Password hashing
  • @prisma/client: Database ORM
