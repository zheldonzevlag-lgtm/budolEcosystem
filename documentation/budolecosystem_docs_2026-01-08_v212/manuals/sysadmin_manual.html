Service Management & Verification

Service Restart

After implementing the registration endpoint, the budolID service must be restarted to apply changes. In development environments using nodemon, this happens automatically. For manual restarts:

# In budolID-0.1.0 directory
npm run dev

Verification Procedures

Health Check

Ensure the service is active by calling the health endpoint:

curl http://localhost:8000/api/health

Endpoint Validation

Test the registration endpoint with a mock payload:

# PowerShell example
$body = @{ email="admin@example.com"; password="password" } | ConvertTo-Json
Invoke-RestMethod -Uri "http://localhost:8000/auth/register" -Method Post -Body $body -ContentType "application/json"

Troubleshooting

Issue: Unexpected token '<' in JSON response.
Cause: Service is returning an HTML error page (usually 404).
Fix: Ensure the /auth/register route is correctly defined in api/index.js and the service has been restarted.
