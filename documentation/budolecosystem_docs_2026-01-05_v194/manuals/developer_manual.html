Developer Manual - v194

Purpose

This version documents the technical implementation of Tiered KYC Enforcement within the transaction-service. The primary goal is to restrict financial activities for unverified users to comply with BSP (Bangko Sentral ng Pilipinas) anti-money laundering regulations.

Technical Implementation

1. P2P Transfer Blocking

Modified budolpay-0.1.0/services/transaction-service/index.js

We introduced a pre-transaction check in the /transfer endpoint. The service now fetches the sender's kycTier from the database before proceeding with ledger entries.

// Enforcement Logic in /transfer
const sender = await prisma.user.findUnique({ where: { id: senderId } });
if (sender.kycTier === 'BASIC') {
    throw new Error('Verification required: BASIC accounts cannot send money P2P.');
}

2. Wallet Balance Hard Caps

Modified budolpay-0.1.0/services/transaction-service/index.js

The /cash-in endpoint was updated to include a cumulative balance check. If a BASIC user attempts to deposit funds that would push their total balance above ₱5,000, the transaction is rejected.

// Enforcement Logic in /cash-in
if (user.kycTier === 'BASIC') {
    const currentBalance = parseFloat(user.wallet.balance);
    if (currentBalance + amount > 5000) {
        throw new Error(`Limit Exceeded: Max wallet balance for BASIC is ₱5,000.`);
    }
}

Issues & Resolutions

  • Issue: User model in transaction-service was not including the wallet relation by default.
  • Fix: Updated prisma.user.findUnique to include: { wallet: true } for accurate balance calculations during Cash-In checks.

Future Recommendations

  • Daily Velocity Limits: Implement a rolling 24-hour transaction volume limit (e.g., ₱100,000 for Fully Verified).
  • Tier Upgrade Hooks: Automatically trigger a balance limit expansion when a user's KYC status is updated to APPROVED by an admin.
