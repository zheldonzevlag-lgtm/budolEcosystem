System Admin Manual - v194

Compliance Enforcement Rules

As of v194, the system automatically enforces the following compliance rules at the API level. Administrators no longer need to manually block transactions for unverified users.

Rule ID: KYC-001 - P2P Restriction

Condition: user.kycTier == 'BASIC'

Action: Blocks /transfer requests.

Error Message: "Verification required: BASIC accounts cannot send money P2P. Please upgrade to FULLY VERIFIED."

Rule ID: KYC-002 - Wallet Ceiling

Condition: user.kycTier == 'BASIC' && (wallet.balance + amount) > 5000

Action: Blocks /cash-in requests.

Error Message: "Limit Exceeded: BASIC accounts have a maximum wallet balance of ₱5,000."

Audit Logs

Failed transactions due to KYC limits are logged in the transaction-service console with the prefix [Transaction] Limit Enforcement:. These do not create FAILED records in the database to prevent ledger bloat from invalid attempts.
