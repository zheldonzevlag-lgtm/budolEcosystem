Developer Manual - Version v430

Technical Overview

The core issue was a mismatch between client-side and server-side authentication state. While the client had a valid token in localStorage, the browser was failing to send the corresponding cookie to the /api/auth/admin/check endpoint, resulting in a 401 Unauthorized response.

Backend Fix: API Route Enhancement

The /api/auth/admin/check route was modified to use the requireAdmin middleware, which is capable of extracting the token from both cookies and the Authorization: Bearer header. This makes the endpoint much more robust.

File: app/api/auth/admin/check/route.js

// Enhanced logic to support both cookies and headers
const { authorized, user: authUser, errorResponse } = await requireAdmin(request)

if (authorized && authUser) {
    // Return detailed admin info...
} else {
    return errorResponse || NextResponse.json({ isAdmin: false, error: 'Not authenticated' }, { status: 401 });
}

Frontend Fix: Token Header Injection

The AdminLayout.jsx component was updated to explicitly retrieve the token from auth-client.js and include it in the Authorization header when performing the admin check. This ensures that even if cookies are missing or blocked, the server can still authenticate the request.

File: components/admin/AdminLayout.jsx

// Get token for Authorization header
const { getToken } = await import("@/lib/auth-client")
const token = getToken()

const response = await fetch('/api/auth/admin/check', {
    headers: token ? {
        'Authorization': `Bearer ${token}`
    } : {}
})

Rationale

  • Reliability: localStorage is often more reliable than cookies in multi-origin or complex LAN environments.
  • Consistency: Using the Authorization header is the standard practice for modern API-driven applications.
  • Security: The token is still verified on the server using the same JWT_SECRET, maintaining high security standards.

Compliance

This fix adheres to Cybersecurity Law and NPC requirements by ensuring that only authorized administrators can access sensitive management functions, while providing a seamless user experience for legitimate users.
