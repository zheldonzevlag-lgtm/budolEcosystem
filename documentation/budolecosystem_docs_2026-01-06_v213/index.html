Mobile App Auth Integration & Environment Fixes

← Previous (v212) Main Index Next (v213) → Next →

Purpose

This update focuses on integrating the GoTyme-aligned mobile authentication service into the Flutter mobile application and fixing critical environment configuration issues that prevented database connectivity.

Technical Implementation

1. Mobile App: GoTyme Auth Integration

The ApiService and UI screens were redesigned to support the multi-step mobile auth flow (Identification → OTP/PIN → Login).

  • ApiService Updates: Added identifyMobile, verifyOtp, verifyPin, and register (with PIN) methods. Implemented automatic Device ID generation and persistence.
  • Login Screen: Redesigned as a state machine with three steps:
    • phone: Identification via mobile number.
    • otp: Mandatory for untrusted devices.
    • pin: 6-digit PIN verification for all logins.
  • Registration Screen: Added a dedicated step for setting the mandatory 6-digit PIN and aligned the payload with the new auth service.

2. Environment Fixes

Corrected the DATABASE_URL in setup-environment.ps1 and .env files to point to budolpay_db instead of budolpay, resolving "column missing" errors during Prisma operations.

Risk & Mitigation

Risk Mitigation
Device ID mismatch on app reinstall Stored Device ID in SharedPreferences; future enhancement will use device_info_plus for hardware-bound IDs.
Insecure PIN storage in transit PIN is hashed on the server; future update will implement RSA encryption for PIN payloads.

Future Enhancements

  • Hardware Device Binding: Integrate device_info_plus to replace generated UUIDs with hardware serials.
  • Biometric UI: Add Biometric login button to the PIN step in LoginScreen.
  • Automatic OTP Retrieval: Implement SMS listener for automatic OTP filling in Flutter.
