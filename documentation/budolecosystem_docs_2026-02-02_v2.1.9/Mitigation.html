Risk Mitigation Plan

Strategies implemented to address identified risks in version 2.1.9.

M1 Submenu State Persistence

Risk: Users might lose their place when navigating between settings pages if the menu collapses automatically.

Mitigation: Implemented useEffect hook to detect settings pathnames and force the submenu to remain open, ensuring visual consistency across transitions.

M2 UI Clutter Reduction

Risk: Adding submenus might make the sidebar feel cramped on smaller screens.

Mitigation: Used a smaller font size (text-[12px]) and removed bullet indicators for sub-items, creating a clear visual hierarchy that differentiates main links from sub-links.

M3 Access Control Visibility

Risk: Unauthorized users might see links to sensitive configuration areas.

Mitigation: While the UI links are restored, the underlying API routes and page access remain protected by server-side authentication checks, adhering to NPC and PCI DSS "Least Privilege" principles.
