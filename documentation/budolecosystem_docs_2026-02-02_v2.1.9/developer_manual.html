Technical Implementation Details

Architectural breakdown of the sidebar submenu implementation and state persistence logic.

Rationale

As the budolPay Admin console expands, a flat navigation structure becomes cluttered. The System Settings module contains multiple distinct configuration areas (Environment Variables, Service Endpoints, Security Config) that warrant a dedicated submenu to maintain clean UX and logical grouping.

Code Implementation

1. Submenu State Management

Added state to track the visibility of the System Settings submenu, initialized based on the current pathname to ensure the menu stays open when navigating within settings.

// Sidebar.tsx:L18
const [showSettingsSubmenu, setShowSettingsSubmenu] = useState(pathname.startsWith('/settings'));

useEffect(() => {
  if (pathname.startsWith('/settings')) {
    setShowSettingsSubmenu(true);
  }
}, [pathname]);

2. Enhanced NavItem Component

Updated the NavItem component to support an isSubmenu prop, which adjusts font size and removes the bullet indicator for nested items.

// Sidebar.tsx:L199
function NavItem({ href, label, active, isSubmenu = false }: NavItemProps) {
  return (
    <a href={href} className={`... ${isSubmenu ? 'text-[12px]' : 'text-[13px] font-medium'}`}>
      {!isSubmenu && <span className="..."></span>}
      {label}
    </a>
  );
}

Dependencies

  • Lucide React: Used for ChevronDown, ChevronRight, and Settings icons.
  • Next.js Navigation: usePathname hook for active state detection.
  • Tailwind CSS: For responsive and consistent UI styling.
