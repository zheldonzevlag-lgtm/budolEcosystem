Card-Based Settings UI Refactor

RELEASED Last updated: 2026-02-02

Overview

Version 2.2.5 marks a significant shift in how system settings are presented. We have moved away from the traditional tabular view to a modern, card-based interface. Each setting now occupies its own card, providing better focus and readability.

Rationale: Tabular data becomes hard to manage as the number of configuration keys grows. A card-based layout allows for better organization, clearer descriptions, and more focused interaction for each parameter.

UI Enhancements

  • Individual Setting Cards: Each configuration parameter is isolated in a responsive card with its own title, description, and action button.
  • Responsive Grid Layout: Sections use a CSS Grid that scales from 1 to 3 columns depending on screen size, maximizing real estate.
  • Enhanced Readability: Descriptions are now more prominent, and mono-spaced keys are easier to scan.
  • Categorical Grouping: Cards are still logically grouped by functional domains (SYSTEM, PAYMENT, etc.) for easy navigation.
