Local DNS & Service Discovery Feasibility

Technical analysis for implementing automated server discovery in the budolEcosystem.

Overview Feasibility Report

This document explores the possibility and technical difficulty of implementing a local DNS or mDNS (Multicast DNS) system to allow the mobile app and other ecosystem services to discover each other automatically without manual IP configuration.

Option A: mDNS (ZeroConf)

Fully automatic "plug-and-play" discovery using Bonjour/Avahi protocols. The server broadcasts its presence, and the app finds it.

Difficulty: Moderate

Option B: Local DNS Server

Configuring a static DNS entry (e.g., budol.local) on a router or dedicated DNS server (like Pi-hole).

Difficulty: Low (Infrastructure based)

Technical Specification

  • Protocol: mDNS (Multicast DNS) / DNS-SD
  • Service Type: _http._tcp
  • Service Name: budol-ecosystem
  • Proposed Library (Flutter): nsd (v4.0.3)
  • Proposed Library (Node.js): bonjour-service
