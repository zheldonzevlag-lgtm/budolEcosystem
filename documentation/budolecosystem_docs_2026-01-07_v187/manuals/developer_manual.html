Developer Manual v187

Purpose

This manual documents the technical implementation details for the final deployment of the Budol Ecosystem services (v0.1.0) on Vercel, focusing on cross-service compatibility and deployment protection handling.

Technical Details

Service Architecture

The ecosystem consists of multiple microservices and frontend applications:

  • budolID-foundation: Identity provider (Express)
  • budolpay-api-gateway: Central routing and proxy (Express/Socket.io)
  • budolpay-auth-service: Authentication logic (Express)
  • budolpay-wallet-service: Wallet management (Express)
  • budolpay-transaction-service: Transaction processing (Express)
  • budolpay-admin: Admin dashboard (Next.js)
  • budolAccounting: Ledger management (Express)
  • budolshap: Marketplace (Next.js)

Vercel Compatibility Fixes

To ensure Express services run correctly on Vercel's serverless environment, the following changes were implemented in index.js across all services:

// Conditional server startup
if (process.env.NODE_ENV !== 'production' && !process.env.VERCEL) {
    app.listen(PORT, () => {
        console.log(`Service running on port ${PORT}`);
    });
}

// Export app for Vercel
module.exports = app;

Issues & Fixes

  • Issue: Infinite loop (508) on Vercel aliased domains.
    Fix: Identified as an edge network routing issue. Developers should use direct deployment URLs for testing until aliases are resolved.
  • Issue: 401 Unauthorized on deployment URLs.
    Fix: Confirmed as Vercel Deployment Protection (SSO). Access requires logging in through the Vercel deployment portal.

Dependencies
