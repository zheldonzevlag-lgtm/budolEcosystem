System Admin Manual v187

Purpose

This manual provides operational instructions for managing the deployed Budol Ecosystem services and addresses current infrastructure constraints.

Deployment Procedures

Vercel Deployment

All services are deployed using the Vercel CLI. The standard command used is:

vercel deploy --prod --yes --cwd [service-directory]

Infrastructure Changes

  • Database: Prisma clients are configured to automatically use Vercel Postgres environment variables (POSTGRES_PRISMA_URL).
  • Routing: Each service has its own Vercel project card, ensuring isolation and granular scaling.

Test Results

Health checks were performed using curl.exe -I on all deployment URLs.

  • budolID: 401 Unauthorized (SSO Active)
  • budolpay-api-gateway: 401 Unauthorized (SSO Active)

Risk & Mitigation

  • Risk: Unauthorized access to internal APIs.
    Mitigation: Vercel Deployment Protection (SSO) is enabled, requiring Vercel authentication for all direct URLs.
  • Risk: Service downtime during redeployment.
    Mitigation: Vercel's zero-downtime deployment ensures the old version remains active until the new build is ready.

Recommendations

  • Disable Vercel Deployment Protection for public-facing services (Shap, Admin) if external access is required.
  • Configure custom domains to resolve the infinite loop issues seen on .vercel.app subdomains.
