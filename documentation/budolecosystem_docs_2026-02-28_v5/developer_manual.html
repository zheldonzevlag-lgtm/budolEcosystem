Cache Invalidation Strategy

The CategorySelector component is a client-side component that fetches data from a public API route. To prevent stale data, we've implemented the following:

1. Server-Side De-caching

In app/api/categories/route.js:

export const dynamic = 'force-dynamic'
export const revalidate = 0

2. Client-Side Cache Busting

In CategorySelector.jsx, the fetch call now includes headers and a timestamp to bypass browser cache:

const res = await fetch(`/api/categories?flat=true&t=${Date.now()}`, {
    cache: 'no-store',
    headers: {
        'Cache-Control': 'no-cache'
    }
})

Slug Pattern for Non-Unique Names

Users reported "Socks" not being found despite existing in Admin. Our database analysis showed that if "Socks" exists in multiple sub-paths (e.g., Men vs Women), the second entry's slug is automatically suffixed with a timestamp. The CategorySelector searching functionality is name-based and slug-based, so both remain findable.
