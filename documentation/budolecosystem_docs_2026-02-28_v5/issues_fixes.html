Issues & Fixes Log (v5.3.5)

1. New Categories Not Showing in Product Picker FIXED

Description: Categories successfully created in the Admin panel (confirmed in Image 1) failed to appear in the Storefront product creation categories list (Image 2).

Cause: The browser was caching the /api/categories GET response, causing the "Add Product" page to show a stale version of the category hierarchy from before the transition/addition.

Fix: Implemented double-ended cache invalidation (Server: force-dynamic, Client: no-store + timestamp) to ensure immediate availability of schema changes.
