BudolEcosystem Docs v214 (2026-01-08)

Subject: SMTP Authentication and Domain Typo Fixes

Purpose

Address the issue where users were not receiving password reset emails and the verification links contained a domain typo (budulshap instead of budolshap).

Explanation & Reason

The email delivery failure was caused by two main factors:

  • SMTP Credentials Format: The SMTP password in .env.local and the database SystemSettings contained quotes and spaces, which nodemailer could not process correctly.
  • Invalid App Password: After sanitizing the password, the connection still failed with 535 5.7.8 Username and Password not accepted, indicating the Gmail App Password is likely invalid or revoked.
  • Domain Typo: The NEXT_PUBLIC_APP_URL variable had a typo (budulshap), leading to invalid reset links.

Technical Details

Modified files and database settings:

  • .env.local: Removed quotes and spaces from SMTP_PASS and corrected NEXT_PUBLIC_APP_URL.
  • Database (Prisma): Updated SystemSettings table to sanitize SMTP fields.
  • Test Script: Created test-smtp.js to verify connection independently.

Test Results

❌ SMTP connection failed:
Invalid login: 535-5.7.8 Username and Password not accepted.
Suggestion: Generate a new App Password at https://myaccount.google.com/apppasswords

Issues & Risks

  • Risk: Email functionality remains disabled until a valid App Password is provided.
  • Mitigation: Provide clear instructions to the user on how to generate a new Gmail App Password.

Recommendation for Future Enhancement

  • Implement a "Test Connection" button in the Admin System Settings UI to validate SMTP credentials before saving.
  • Add environment variable validation on startup to catch formatting issues like quotes or spaces.
