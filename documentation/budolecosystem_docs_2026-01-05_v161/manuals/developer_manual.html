Developer Manual - Real-time Home Transactions

Implemented

Purpose

Replace static dummy transaction data in the mobile app Home screen with live data from the backend to provide users with an accurate view of their financial activity.

Technical Details

1. API Integration

A new getTransactions() method was added to the ApiService to fetch history from the Gateway's transaction endpoint.


Future<List<dynamic>> getTransactions() async {
  final userId = user!['id'];
  final url = '$baseUrl/transactions/history/$userId';
  final response = await http.get(
    Uri.parse(url),
    headers: {
      'Authorization': 'Bearer $token',
      'Content-Type': 'application/json',
    },
  );
  // ... parsing logic
}

2. UI State Management

The HomeScreen was updated to maintain a _transactions list and a _fetchData() method that aggregates balance and transaction fetches.


Future<void> _fetchData() async {
  await Future.wait([
    _fetchBalance(),
    _fetchTransactions(),
  ]);
}

3. Real-time Refresh

The Socket.io listener now triggers a re-fetch of transactions whenever a transaction event is received, ensuring the list stays current without manual refreshing.

Test Results

  • Data Integrity: Transactions correctly show income (+) in green and expenses (-) in red.
  • Formatting: Amounts are formatted using PHP #,##0.00 standards.
  • Empty State: "No recent transactions" message displays correctly for new users.
  • Socket Sync: Successfully refreshed list upon receiving transaction event via socket.
