Mitigation Strategy (v14.0.0)

M14-01: Cost & Quota Management

Risk: Exceeding Free Tier or reaching RDS quotas.

Action: Consolidated 5 planned RDS instances into 1 powerful t4g.micro instance using logical database separation. This maintains isolation at the schema level while staying within the single-instance quota.

M14-02: Zero Trust Networking

Risk: Unauthorized access to production data.

Action: Implemented VPC with Zero-Public RDS. All access is controlled via Security Group peering (ALB -> ECS -> RDS) or authenticated SSM sessions via a temporary Management Bridge.

M14-03: Version Control & Bridge Recovery

Risk: Incompatibility between legacy backups and new RDS targets.

Action: Provisioned PG17-capable management bridge to handle modern headers. Automated verification via Lambda to ensure row counts match legacy audits.
