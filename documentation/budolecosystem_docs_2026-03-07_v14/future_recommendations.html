Future Recommendations (v14.0.0)

1. Automated Backup Rotation

Rationale: Currently, backups are manual (`F:\db-backup`).

Recommendation: Implement AWS Backup with a cross-region copy to a secondary region (e.g., Tokyo) for Disaster Recovery (DR) compliance.

2. IAM Database Authentication

Rationale: Currently using static passwords (`r00tPassword2026!`).

Recommendation: Transition to IAM Database Authentication to eliminate long-lived credentials and integrate with AWS Secrets Manager rotate policy.

3. RDS Proxy Implementation

Rationale: As we scale to more Lambda functions and Fargate tasks, connection pooling will become a bottleneck.

Recommendation: Deploy AWS RDS Proxy to manage connection pooling and reduce database load during traffic spikes.
