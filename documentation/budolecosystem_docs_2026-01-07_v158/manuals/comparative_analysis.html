Cloudinary vs. AWS S3 Analysis

A key component of the plan is migrating from Cloudinary to AWS S3. Here is why:

  • Unified Security: Cloudinary uses public URLs with transformations. S3 allows us to keep KYC documents in a private bucket accessible only via time-limited Signed URLs generated by our backend.
  • Integration: Using the AWS SDK for both compute (Lambda) and storage (S3) reduces the number of third-party dependencies and simplifies environment variable management.
  • Cost Control: Cloudinary's "credits" system can be unpredictable. AWS S3 charges strictly by GB stored and transferred, which is easier to audit for financial applications.

Conclusion & Recommendation

The Proposed Free Tier Plan is the optimal next step for most services. However, for stable Socket.io connections and zero cold starts, the Non-Serverless (Docker on EC2) approach is superior. We recommend a hybrid approach: host the static frontend on Vercel and the core microservices on a persistent EC2 instance.
