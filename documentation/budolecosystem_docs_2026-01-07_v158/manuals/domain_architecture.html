Technical Explanation

The Role of the API Gateway

The budolpay-api-gateway.vercel.app is the most critical domain. It acts as a Reverse Proxy. All requests from the budolpay.vercel.app frontend are sent to the API Gateway, which then forwards them to the specific backend domains based on the URL path (e.g., /auth -> budolpay-auth.vercel.app).

Why Two Gateways?

  • API Gateway: Manages internal traffic, security, and real-time socket connections.
  • Payment Gateway: Manages financial handshakes with external banks and e-wallets.

Risks & Mitigations

Risk: Vercel Hobby plan execution limits (10 seconds) may cause timeouts for complex transactions.

Mitigation: Optimize Prisma queries and use connection pooling (Prisma Accelerate) to minimize latency.

Risk: Serverless functions do not support persistent Sockets.

Mitigation: Migrate real-time logic to a persistent host (Railway/Render) or use Pusher/Ably.
