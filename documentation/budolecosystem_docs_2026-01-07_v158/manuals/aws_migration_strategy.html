AWS Free Tier Migration Strategy v158

Purpose

To provide a roadmap for migrating the budolEcosystem from Vercel/Local to AWS Free Tier. This strategy focuses on maintaining persistent connections (Sockets) and professional-grade database management while staying within the $0 cost threshold for the first 12 months.

AWS Service Mapping

Local/Vercel Component AWS Free Tier Service Why?
API Gateway (Socket.io) EC2 (t2.micro / t3.micro) Requires persistent connection for Sockets (750 hours/month free).
PostgreSQL Database RDS (db.t2.micro / db.t3.micro) Managed PostgreSQL with automated backups (750 hours/month free).
Microservices (Auth, TX, etc.) Lambda + API Gateway Serverless execution (1 Million requests/month free).
Image Repository (Cloudinary) S3 (Simple Storage Service) Unified storage for KYC docs, profile pics, and assets (5GB free).
Static Assets / Mobile APKs S3 Object storage for installers and media (5GB free).
Frontend (budolpay.vercel.app) Amplify / S3 + CloudFront Global CDN for web delivery.

Migration Phases

Phase 1: Database Migration (RDS)

1. Provision an RDS instance using the db.t3.micro (Free Tier eligible).
2. Update DATABASE_URL in all services.
3. Run npx prisma db push to sync the schema.

Phase 2: Persistent Gateway (EC2)

Critical for Sockets: Unlike Vercel, an EC2 instance can run 24/7, allowing our Socket.io server in the api-gateway to maintain active rooms for real-time transaction updates.

1. Launch a Linux EC2 instance.
2. Install Node.js and PM2.
3. Deploy the api-gateway service to this instance.

Phase 3: Serverless Microservices (Lambda)

1. Wrap core services (Auth, Wallet, Transaction) as Lambda functions.
2. Use AWS API Gateway to route traffic to these functions.
3. This mirrors the Vercel architecture but keeps everything within the AWS ecosystem.

Phase 4: Image Repository Migration (S3)

From Cloudinary to S3: Consolidating images into S3 allows for tighter security control via IAM policies, ensuring KYC documents are only accessible via signed URLs.

1. Export existing assets from Cloudinary.
2. Upload to a private S3 bucket.
3. Update services to use the @aws-sdk/client-s3 instead of the Cloudinary SDK.

Risks & Mitigations

Risk: Unexpected costs after the 12-month free tier period.

Mitigation: Set up AWS Budgets to alert at $0.01 spending. Monitor data transfer out (DTO) which is limited to 100GB/month.

Risk: Complexity of managing EC2 security groups.

Mitigation: Only allow traffic on port 80 (HTTP) and 443 (HTTPS). Use IAM roles instead of hardcoded access keys.

Recommendations

  • Use Infrastructure as Code (IaC): Use AWS CDK or Terraform to ensure the environment can be recreated instantly.
  • CloudWatch Logs: Centralize all microservice logs for easier debugging compared to distributed Vercel logs.
  • Elastic IP: Assign a static IP to the EC2 instance to prevent domain DNS updates if the instance restarts.
