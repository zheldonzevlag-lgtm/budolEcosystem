Master Implementation Plan v158

Subject: Ecosystem Migration to Vercel & AWS (Hobby/Free Tier)

Agent Instruction: This document serves as the primary technical specification for executing the budolEcosystem migration. Follow each phase sequentially.

Phase 0: Environment Alignment & Prerequisites

Objective: Prepare local codebase and cloud environment variables.
  • Create vercel.json in each microservice directory (Auth, Wallet, TX, etc.).
  • Update .gitignore to ensure .env files are never committed, while .env.example is up-to-date.
  • Verify all Vercel domains are active (as per domain_architecture.html).

Command for Agent:

# Check all .env files for cloud-incompatible values (e.g., localhost)
grep -r "localhost" ./services

Phase 1: Database Migration (AWS RDS)

Objective: Move from local PostgreSQL to Managed AWS RDS.
  • Provision db.t3.micro on AWS RDS (PostgreSQL 15+).
  • Configure Security Group to allow inbound traffic on 5432 from Vercel IPs (or 0.0.0.0/0 for testing).
  • Run Prisma migrations to build the schema in the cloud.

Command for Agent:

# From /packages/database
set DATABASE_URL="postgresql://user:pass@aws-rds-endpoint:5432/budolpay"
npx prisma db push

Phase 2: Persistent Gateway Deployment (AWS EC2)

Objective: Host the Socket.io Gateway on a persistent server.
  • Launch t3.micro EC2 instance with Ubuntu.
  • Install Node.js 18+ and PM2.
  • Clone api-gateway and configure .env to point to other Vercel domains.
# Sample PM2 start command
pm2 start index.js --name "budol-gateway"

Phase 3: Image & Asset Migration (AWS S3)

Objective: Transition from Cloudinary to AWS S3.
  • Create private S3 bucket budolpay-assets.
  • Implement S3 upload logic in wallet-service and kyc-service.
  • Configure CORS to allow requests from budolpay.vercel.app.

Code Change Snippet:

// Update s3-helper.js
const { S3Client } = require("@aws-sdk/client-s3");
const s3 = new S3Client({ region: process.env.AWS_REGION });

Phase 4: Serverless Service Deployment (Vercel)

Objective: Deploy core logic as serverless functions.
  • Deploy each microservice using vercel deploy.
  • Mirror all local .env variables into the Vercel Dashboard for each project.
  • Set GATEWAY_URL to the new EC2 public DNS/IP.

Phase 5: Validation & End-to-End Testing

Objective: Verify full system integrity in the cloud.
  • Auth Test: Register a new user via budolpay-auth.vercel.app.
  • P2P Test: Perform a transfer and verify Socket.io notification via EC2 Gateway.
  • Image Test: Upload a KYC document and verify it is stored in S3 (Private).

Phase 6: Scaling for Growth (Paid Tier)

Objective: Upgrade to Production-grade infrastructure.
  • Vercel Pro: Enable Pro tier for 300s timeouts and 1TB bandwidth.
  • AWS RDS Multi-AZ: Modify RDS instance to "Multi-AZ" for high availability.
  • AWS ALB: Provision Application Load Balancer for the EC2 Gateway cluster.

Command for Database Upgrade:

# AWS CLI command to upgrade RDS to Multi-AZ
aws rds modify-db-instance --db-instance-identifier budol-db --multi-az --apply-immediately

Rollback Procedures

If Database Sync Fails: Immediately revert DATABASE_URL to local/staging and check RDS Security Group logs.
If Socket Connection Drops: Verify the EC2 instance status and ensure the api-gateway process is running via pm2 status.
