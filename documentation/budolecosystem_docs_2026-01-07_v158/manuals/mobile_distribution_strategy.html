Mobile App Distribution Strategy v158

Unlike web apps, Flutter mobile apps for iOS and Android are not "hosted" on a server. Instead, they are compiled into binaries and distributed through specific platforms. This manual outlines the strategy for the budolEcosystem mobile apps.

1. Production Distribution

The final binaries (.ipa for iOS, .aab for Android) are "hosted" by the official app stores:

  • Google Play Store: For Android devices.
  • Apple App Store: For iOS devices.

Requires a one-time $25 fee for Google and a $99/year subscription for Apple.

2. Beta/Testing Distribution

Before public release, apps are hosted on testing platforms for the internal team:

  • Firebase App Distribution: Fastest way to get builds to Android/iOS testers.
  • TestFlight (Apple): Required for official iOS beta testing.

Technical Implementation: CI/CD Pipeline

To automate the "hosting" of new versions, we recommend a CI/CD pipeline using GitHub Actions or Codemagic.

Stage Tool Outcome
Build GitHub Actions Compiles Flutter code into APK/IPA binaries.
Test Flutter Test Runs unit and widget tests automatically.
Deploy (Beta) Firebase App Distribution Testers receive an email with the new version instantly.
Deploy (Store) Fastlane Automates the upload to App Store Connect and Play Console.

Recommendation for budolEcosystem

For the budolEcosystem mobile apps, we recommend the following setup:

  1. Backend Connection: The mobile apps should point to the Non-Serverless API Gateway (e.g., https://api.budolpay.com) hosted on our VPS. This ensures stable WebSocket connections for real-time transaction alerts in the app.
  2. CI/CD: Use GitHub Actions (since the code is already on GitHub). It's free for public repos and has a generous free tier for private ones.
  3. Instant Updates: Use Shorebird. Shorebird allows you to push small code fixes to your Flutter app instantly without waiting for App Store review.
  4. Hosting Assets: Any dynamic images or KYC documents uploaded from the mobile app should be hosted on AWS S3 (as planned in our storage strategy).

Costing for Mobile Strategy

  • Google Play Console: $25 (One-time).
  • Apple Developer Program: $99 / year.
  • Firebase App Distribution: Free.
  • GitHub Actions: Free (within tier limits).
  • Shorebird (Optional): Free tier available; paid for high volume.
