Paid Tier Scaling Strategy v158

This document outlines the transition from Free/Hobby tiers to Vercel Pro and AWS Production levels. This is recommended when the ecosystem reaches >1,000 active users or requires strict SLA compliance.

Vercel Pro Migration (App & Serverless)

Why Upgrade to Vercel Pro?

  • Timeout Extension: Serverless function timeout increases from 10s to 300s (critical for complex accounting tasks).
  • Concurrency: Higher concurrent execution limits for peak traffic (e.g., payday transfers).
  • Deployment Protection: Password protection for staging environments.
  • Analytics: Real-time performance monitoring and Core Web Vitals.

AWS Production Tier (Core Infrastructure)

Transitioning from t3.micro to production-grade instances ensures high availability and data durability.

1. Managed RDS (Multi-AZ)

Upgrade RDS to a Multi-AZ deployment (e.g., db.t3.medium).

  • High Availability: Automatic failover to a standby instance in a different Availability Zone.
  • Performance: Increased RAM and IOPS for faster ledger queries.

2. EC2 Auto-Scaling (Gateway)

Move the api-gateway behind an Application Load Balancer (ALB).

  • Auto-Scaling: Automatically add/remove EC2 instances based on CPU/Socket load.
  • Zero Downtime: Deploy new gateway versions without dropping active socket connections.

AWS S3 Production Storage

  • S3 Intelligent-Tiering: Automatically move older KYC documents to cheaper storage classes to save costs.
  • CloudFront CDN: Add CloudFront in front of S3 for global asset delivery with sub-50ms latency.
  • Object Lock: Enable for regulatory audit logs to prevent accidental deletion (WORM compliance).

Implementation Roadmap

Phase 1: Performance Benchmarking

Use Vercel Analytics and AWS CloudWatch to identify bottlenecks in the Free Tier.

Phase 2: Database Vertical Scaling

Upgrade RDS instance size. This can be done with minimal downtime (< 60s) during maintenance windows.

Phase 3: Load Balancing & High Availability

Provision the ALB and setup Target Groups for the EC2 Gateway cluster.
