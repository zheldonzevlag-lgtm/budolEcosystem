Migration Configuration

To transition from local development to Vercel, developers must update service URLs in the api-gateway.

Service URL Mapping (Production)


// services/api-gateway/index.js
const services = [
    { route: '/auth', target: 'https://budolpay-auth.vercel.app' },
    { route: '/wallet', target: 'https://budolpay-wallet.vercel.app' },
    { route: '/transactions', target: 'https://budolpay-tx.vercel.app' },
    { route: '/payments', target: 'https://budolpay-payment-gw.vercel.app' },
    { route: '/verification', target: 'https://budolpay-kyc.vercel.app' }
];

Environment Variables

Each service must have its .env file mirrored in the Vercel Dashboard. Key variables include:

  • DATABASE_URL: Must use a publicly accessible PostgreSQL instance with pooling.
  • JWT_SECRET: Must be identical across all services.
  • GATEWAY_URL: Points to https://budolpay-api-gateway.vercel.app.
  • AWS_S3_BUCKET: The name of your S3 bucket for images/assets.
  • AWS_REGION: e.g., ap-southeast-1.

S3 Image Repository (AWS SDK)

When migrating from Cloudinary, use the following pattern for file uploads:


import { S3Client, PutObjectCommand } from "@aws-sdk/client-s3";

const s3Client = new S3Client({ region: process.env.AWS_REGION });

export const uploadImage = async (file, key) => {
    const command = new PutObjectCommand({
        Bucket: process.env.AWS_S3_BUCKET,
        Key: key,
        Body: file.buffer,
        ContentType: file.mimetype
    });
    return await s3Client.send(command);
};

Dependencies

  • vercel-cli: Required for deployment and environment management.
  • prisma: Required for database synchronization.
