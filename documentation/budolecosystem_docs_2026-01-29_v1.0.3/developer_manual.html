Developer Manual - SSO & LAN Architecture (v1.0.3)

1. Request Logging in budolID

Middleware Implementation

To improve observability, a simple request logging middleware has been added to the budolID service. This logs the timestamp, HTTP method, and URL for every incoming request.

// budolID-0.1.0/index.js app.use((req, res, next) => { console.log(`${new Date().toISOString()} ${req.method} ${req.url}`); next(); });

Reference: budolID-0.1.0/index.js

2. SSO Login Test Flow

Automated Login Test

A new test script test-login-flow.js has been implemented to verify the SSO login flow programmatically. It performs the following steps:

  1. Fetches a valid App API Key and User Email from the database.
  2. Sends a POST request to /auth/sso/login.
  3. Verifies that a JWT token and Redirect URI are returned.
  4. Checks the database to ensure the session was persisted.
  5. Calls the /auth/verify endpoint with the returned token.

Reference: test-login-flow.js

3. Test Data Seeding

Tony Stark Test User

A dedicated seed script seed-tony-budolid.js ensures that a consistent test user exists in the budolid schema with a known password (budolshap).

Reference: seed-tony-budolid.js

4. SSO Architecture & LAN Connectivity

Dynamic IP Resolution

To support LAN-based testing, we use a dynamic LOCAL_IP environment variable. This ensures that redirect URIs seeded in the database match the current host IP.
