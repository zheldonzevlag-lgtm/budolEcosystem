Risk Register & Mitigation Plan (v1.0.3)

Active Risks

RiskImpactProbabilityMitigation Strategy
Database Schema Collision High Low (Fixed) Implemented isolated budolid schema and search paths.
Environment Variable Leakage Medium Medium (Fixed) Added sanitization logic in start-ecosystem.js to clean env before spawning services.
LAN Accessibility / IP Changes Medium High Use LOCAL_IP environment variable and dynamic redirect URI seeding.
SSO Token Expiration Low Medium Tokens set to 7-day expiration; sessions stored in Session table for audit.

Compliance Verification

  • Cybersecurity Law: Data isolation via schemas ensures logical separation of identity data.
  • NPC (Privacy): All passwords hashed with Bcrypt (10 rounds); PII stored in dedicated schema.
  • DTI/BIR: Audit logs for all SSO login attempts maintained in database.
