Developer Manual: Wallet Balance Sync Fix

Version: v148 | Date: 2026-01-04

Purpose

To diagnose and fix issues preventing real-time wallet balance updates on the Android application, ensuring consistent user data across the ecosystem.

Technical Details & Findings

1. User ID Mismatch (Root Cause)

The primary issue was a mismatch between the userId generated by budolID (SSO) and the userId used in budolPay (Wallet Service). Even though the admin email was identical, the UUIDs were different, leading to "Wallet not found" or empty balance responses when the mobile app requested balance using its SSO-provided token.

2. SSO Schema Out of Sync

The budolID service was missing the apiSecret column in the EcosystemApp table, causing login failures during the authentication flow. This was resolved by resetting the SSO database and re-seeding the data.

3. Real-time Synchronization (WebSocket)

Validated the API Gateway's WebSocket implementation. The gateway uses Socket.io to proxy real-time events. The wallet-service notifies the gateway via an internal endpoint (/internal/notify), which then emits events to specific user rooms (user:{userId}).

Implementation Details

Code Changes

  • budolID: Reset database schema and seeded with consistent Master Admin credentials.
  • budolPay: Updated scripts/reset-admin.js to use the exact UUID (9e628dfa-d597-458a-ad6e-b75bc3ee41a3) from budolID.

Verification Results

- SSO Login: SUCCESS (Token generated)
- Balance Retrieval (via Gateway): SUCCESS (Returned 20,000,000 PHP)
- Real-time WebSocket: SUCCESS (Received balance_update event)

Recommendations for Future Enhancement

  • Centralized User Event Bus: Implement a message queue (like RabbitMQ or Redis Pub/Sub) for user creation events to ensure all microservices are automatically in sync without manual ID alignment.
  • Health Monitoring: Add WebSocket health checks to the Android app to auto-reconnect if the connection drops.
