Developer Manual: Profile & Compliance Enhancements

Technical implementation of KYC UI improvements, Phone Change Security protocols, and Admin Review systems.

1. Purpose & Architectural Overview

Purpose: To streamline the KYC data entry process and implement secure account management features (Phone Number Change) following industry standards (GCash/BSP compliance).

Explanation: v170 focuses on "User Experience Compliance" — ensuring that data entry is accurate (via calendar pickers) and account modifications are secure (via request-based phone changes).

Reason: Manual typing of DOB often leads to formatting errors. Phone number changes are high-risk operations that require secondary ID verification rather than direct database updates.

Flutter 3.x Intl (Date Formatting) Next.js 14 Prisma ORM Lucide Icons

2. Technical Details: Mobile (Flutter)

DOB Calendar Picker Implementation

Implemented showDatePicker with a custom theme to match BudolPay branding. The format is strictly enforced as MM-DD-YYYY using the intl package.

// MM-DD-YYYY Implementation
Future<void> _selectDate(BuildContext context) async {
  final DateTime? picked = await showDatePicker(
    context: context,
    initialDate: DateTime(2000),
    firstDate: DateTime(1900),
    lastDate: DateTime.now(),
  );
  if (picked != null) {
    setState(() {
      _dobController.text = DateFormat('MM-dd-yyyy').format(picked);
    });
  }
}

Address Pinning Simulation

Added _pinAddress to kyc_verification_screen.dart to allow users to "pin" their current location, reducing address entry friction.

3. Security Protocol: Phone Number Change

Based on GCash research, direct phone number updates are disabled. v170 implements a Request-Based Flow:

  • Validation: User must provide the new number and a reason.
  • Identity Proof: User MUST capture and upload a photo of their ID specifically for the change request.
  • Manual Review: The request is queued for admin approval before the database update occurs.
// Security Check in ChangePhoneNumberScreen
if (_newPhoneController.text.isEmpty || _idImage == null) {
  ScaffoldMessenger.of(context).showSnackBar(
    const SnackBar(content: Text('ID Photo Required for Security'))
  );
  return;
}

5. Issues & Fixes (v170 Development)

Issue Technical Root Cause Fix Implementation
DOB Formatting Errors Manual text input allowed "MM/DD/YY" or "YYYY-MM-DD". Replaced text input with showDatePicker and forced MM-dd-yyyy format.

7. Impact Analysis & Risks

  • Risk: Increased database size due to additional ID images for phone changes. Mitigation: Implement aggressive image compression using sharp on the backend.
  • Risk: User frustration with manual review of phone changes. Mitigation: Clear UI messaging about the 24-48 hour security window.
  • Impact: Higher KYC success rate due to reduced manual entry errors in the DOB field.
