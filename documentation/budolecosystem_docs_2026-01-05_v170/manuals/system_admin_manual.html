System Admin Manual: Compliance & Review Infrastructure

Operational guide for managing KYC submissions and user identity modifications in v170.

1. Overview of Changes

v170 introduces enhanced visibility for administrators into the KYC verification process. Admins can now view raw document uploads and ML-extracted data directly from the Admin Dashboard.

2. Operational Runbook: KYC Document Review

Task: Verifying User Identity

  1. Log into the Admin Console (apps/admin).
  2. Navigate to the Users section.
  3. Locate users with a PENDING status.
  4. Click the Eye Icon (View Documents) to open the Review Modal.
  5. Compare the uploaded ID photo with the OCR Analysis Result.
  6. Approve or Reject based on visual clarity and data consistency.

3. Managing Phone Number Change Requests

Phone number changes are now a two-step process to prevent account takeovers:

  • Step 1: User submits a request with an ID photo proof (Mobile).
  • Step 2: Admin reviews the request in the audit log (Infrastructure update pending).

Note: Currently, admins must manually verify the ID provided in the phone change request against the user's primary KYC document.

4. Database & Storage Considerations

With the addition of identity proof for phone changes, storage usage for the VerificationDocument table will increase.

-- SQL to monitor storage usage for identity documents
SELECT 
    type, 
    COUNT(*) as doc_count, 
    pg_size_pretty(SUM(octet_length(blobData))) as total_size 
FROM "VerificationDocument" 
GROUP BY type;

5. Risks & Mitigations

  • Risk: High resolution images slowing down Admin UI. Mitigation: v170 uses CSS-level lazy loading and fixed aspect ratios for document previews.
  • Risk: Inconsistent DOB formats in legacy data. Mitigation: The new calendar picker enforces MM-dd-yyyy, but legacy data should be audited using a cleanup script.
