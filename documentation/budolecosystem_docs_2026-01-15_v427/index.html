Version 427 - Order Visibility & Microservice Connectivity Fix

Purpose & Rationale

This release addresses a critical issue where users were unable to view their orders after successful payment. The investigation revealed an authentication object mismatch in the API layer and several connectivity issues between microservices (Accounting and BudolPay).

Technical Summary

  • Auth Normalization: Standardized the object returned by getAuthFromCookies to include id and accountType at the top level, preventing 403 Forbidden errors in RBAC checks.
  • Microservice Sync Fix: Corrected the BudolAccounting service URL from port 8002 (Wallet) to 8005 (Accounting).
  • Ecosystem Robustness: Added fallback identification (ID/Sub) in BudolPay's token verification to handle tokens without email fields.
