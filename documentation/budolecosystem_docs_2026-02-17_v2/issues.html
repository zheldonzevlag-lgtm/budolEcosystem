Known Issues

This document tracks issues identified and addressed in this version.

BUG-2026-02-17-01: Unpurchased Cart Items Deleted After Partial Payment

Severity: High

Description: When a user has multiple items in their cart and proceeds to checkout for only a subset of them (partial purchase), the remaining unpurchased items are incorrectly deleted from the cart after the order is placed.

Impact: Poor user experience, lost sales, and user frustration as they have to re-add items to their cart.

Reproduction Steps:

  1. Add two different products (e.g., Product A and Product B) to the cart.
  2. Select only Product A for checkout.
  3. Complete the purchase (via COD or BudolPay).
  4. Return to the cart page.
  5. Observe that Product B is missing.

Root Cause:

  • Synchronous Payments (COD): OrderSummary.jsx contained logic that unconditionally cleared the entire cart (both local Redux state and server-side database cart) upon successful order creation, regardless of whether it was a partial or full purchase.
  • Asynchronous Payments (BudolPay): The webhook handler (app/api/webhooks/budolpay/route.js) contained logic to delete cart items based on productId only. This meant that if a user purchased one variation of a product, all other variations of the same product in the cart would be deleted because they share the same productId.

BUG-2026-02-17-02: Reoccurrence of Unpurchased Cart Items Deleted (PayMongo/GCash)

Severity: High

Description: After the initial fix for synchronous payments, users reported that the issue persisted when using PayMongo/GCash payment methods. Unpurchased items, specifically variations of purchased products, were still being deleted.

Impact: Continued data loss for users paying via GCash/PayMongo.

Reproduction Steps:

  1. Add two variations of the same product (e.g., Size M and Size L) to the cart.
  2. Checkout only Size M using GCash (via PayMongo).
  3. Complete payment.
  4. Observe that Size L is also removed from the cart.

Root Cause:

  • The PayMongo webhook handler (app/api/webhooks/paymongo/route.js) contained a redundant cart cleanup block.
  • This block deleted cart items using productId matching, ignoring variationId.
  • Consequently, purchasing any variation of a product triggered the deletion of ALL variations of that product from the cart.
