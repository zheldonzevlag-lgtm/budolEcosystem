Mitigation Strategies

Detailed mitigation plans for identified risks.

RISK-001: Cart Data Loss on Checkout

Description

Risk of losing unpurchased items when a user checks out only a subset of their cart.

Mitigation Plan

  1. Update frontend logic (OrderSummary.jsx) to selectively delete purchased items from the local cart state instead of clearing the entire cart.
  2. Refactor backend webhook logic (budolpay/route.js) to remove redundant and dangerous bulk deletion of cart items based on product ID.
  3. Verify the fix with test cases covering partial purchases, multi-variation purchases, and full cart checkout.

Status

Implemented & Verified

RISK-003: Webhook Idempotency Failure

Description

Risk of processing the same payment success event multiple times, potentially leading to duplicate order status updates or credits.

Mitigation Plan

  1. Check the `isPaid` status of the order before processing the payment success event.
  2. If `isPaid` is already true, log the duplicate event and skip processing.

Status

Implemented
