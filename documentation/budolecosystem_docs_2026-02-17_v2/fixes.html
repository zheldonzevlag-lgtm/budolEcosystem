Implemented Fixes

This document details the solutions applied to resolve known issues in this version.

FIX-2026-02-17-01: Targeted Cart Item Deletion (Partial Payment Fix)

Resolved Issue: BUG-2026-02-17-01 (Unpurchased Cart Items Deleted)

Affected Components:

  • components/OrderSummary.jsx
  • app/api/webhooks/budolpay/route.js

Implementation Details:

1. Synchronous Payments (COD) - OrderSummary.jsx:

  • Removed the logic that unconditionally cleared the entire cart (clearCart() action and /api/cart PUT request).
  • Replaced it with logic that iterates through the purchased items and dispatches deleteItemFromCart for each item, specifying both productId and variationId.
  • This ensures that only the items involved in the order are removed from the local cart state, preserving unpurchased items.
  • The server-side cart update is handled by the backend createOrder service, so explicit frontend server sync was removed to prevent double-handling and potential race conditions.

2. Asynchronous Payments (BudolPay) - app/api/webhooks/budolpay/route.js:

  • Commented out the logic that attempted to delete cart items based on productId.
  • This logic was redundant because the createOrder service already handles cart item removal upon order creation.
  • It was also dangerous because it deleted all items with a matching productId, which would incorrectly remove other variations of the same product if they were also in the cart.

Verification:

  • Confirmed that partial purchases (e.g., buying 1 item out of 3) correctly remove only the purchased item from the cart.
  • Confirmed that unpurchased items remain in the cart after successful payment.
  • Confirmed that purchasing a specific variation (e.g., Size M) leaves other variations (e.g., Size L) of the same product intact in the cart.

FIX-2026-02-17-02: PayMongo Webhook Cart Deletion Logic Removal

Resolved Issue: BUG-2026-02-17-02 (Reoccurrence of Unpurchased Cart Items Deleted)

Affected Components:

  • app/api/webhooks/paymongo/route.js

Implementation Details:

Reason for Change:

  • The PayMongo webhook contained logic to delete cart items based solely on productId.
  • This logic was flawed because it did not account for variationId, causing all variations of a purchased product to be deleted from the cart.
  • This logic was also redundant as the createOrder service already handles cart cleanup atomically.

Action Taken:

  • Commented out the problematic cart deletion block in the webhook handler.
  • This ensures that only the initial order creation logic dictates cart item removal, preserving data integrity for variations and unpurchased items.

Verification:

  • Verified that payments processed via PayMongo no longer trigger unintended cart item deletions.
