Developer Manual

This manual provides technical details on the Budol Ecosystem implementation, focusing on recent changes and critical components.

Cart Management Strategy (Partial Payments)

A critical update was made to how cart items are handled during checkout, specifically to support partial payments where users select only a subset of items in their cart for purchase.

Problem

Previously, the system would unconditionally clear the entire cart upon successful order creation for synchronous payments (like COD). This resulted in unpurchased items being lost if the user had multiple items in their cart but only checked out some of them.

Solution

The cart clearing logic was refactored to be targeted. Instead of wiping the entire cart, we now iterate through the specific items included in the order and remove only those from the local cart state. The server-side cart update is handled by the backend createOrder service, which ensures consistency.

Implementation Details

1. OrderSummary Component (Frontend)

The OrderSummary component handles the checkout process. For synchronous payments (COD), it now updates the local Redux store by removing only the purchased items.

components/OrderSummary.jsx 
// Updated logic for synchronous payments (lines ~324-343)
const isAsyncPayment = ['GCASH', 'MAYA', 'GRAB_PAY', 'QRPH', 'BUDOL_PAY'].includes(paymentMethod);

if (response.ok && !isAsyncPayment) {
    console.log('🛒 [OrderSummary] Order created successfully. Updating local cart for synchronous payment.');
    
    // 1. Sync session
    if (typeof window !== 'undefined' && token && user) {
        setAuth(token, user);
    }

    // 2. Update Local Cart (Remove purchased items only)
    // Note: Server cart is already updated by the createOrder backend logic
    if (orderItems && orderItems.length > 0) {
        orderItems.forEach(item => {
            dispatch(deleteItemFromCart({
                productId: item.productId,
                variationId: item.variationId
            }));
        });
        console.log('🛒 [OrderSummary] Local cart updated (purchased items removed).');
    }
}

2. BudolPay Webhook (Backend)

The BudolPay webhook handler was updated to remove redundant and dangerous cart clearing logic. The createOrder service already handles the removal of purchased items from the database cart. The webhook's previous logic attempted to delete items by productId, which risked deleting unpurchased variations of the same product.

app/api/webhooks/budolpay/route.js 
// 2. Clear Cart Items
// NOTE: This is handled by createOrder() in ordersService.js during order creation.
// We do NOT want to clear items here because:
// a) They are likely already cleared.
// b) The logic below (productId IN [...]) is dangerous as it deletes ALL variations of a product,
//    even if only one variation was purchased.
// Reference: ordersService.js lines 380-391
Important: Always use deleteItemFromCart with both productId and variationId to ensure precise removal of items from the cart.
