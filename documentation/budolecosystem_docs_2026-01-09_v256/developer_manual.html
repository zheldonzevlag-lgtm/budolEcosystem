Developer Manual - Version v256

Issue

Users reported that the TransactionHistoryScreen did not update automatically when a new transaction was received. It required a manual refresh or a screen reload.

Solution

We implemented a real-time listener using RealtimeService's transactionStream. This stream aggregates events from either Socket.io or Pusher depending on the current system configuration.

Technical Implementation

1. Real-time Listener in TransactionHistoryScreen

Added a StreamSubscription to listen for transaction_update events. When an event is received, the new transaction is inserted at the beginning of the _transactions list.

// lib/screens/transaction_history_screen.dart
void _setupRealtimeListener() {
  Future.microtask(() {
    if (!mounted) return;
    final realtimeService = Provider.of<RealtimeService>(context, listen: false);
    _transactionSubscription = realtimeService.transactionStream.listen((newTx) {
      if (mounted) {
        setState(() {
          final alreadyExists = _transactions.any((tx) => tx['id'] == newTx['id']);
          if (!alreadyExists) {
            _transactions.insert(0, newTx);
          }
        });
      }
    });
  });
}

2. State Synchronization

The implementation checks for duplicate IDs before insertion to handle cases where an event might be delivered multiple times or if a manual refresh happens simultaneously with a real-time update.

Test Results

  • Socket.io: ✅ New transactions appear instantly in history list.
  • Pusher: ✅ New transactions appear instantly in history list.
  • Memory Leak Test: ✅ Subscription is correctly cancelled in dispose().
