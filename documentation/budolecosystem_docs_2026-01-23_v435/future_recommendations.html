Future Recommendations - v435

Technical Enhancements

  • Distributed Locking: Implement Redis-based distributed locking for stock management to handle extreme traffic spikes more safely than standard database transactions.
  • Background Jobs: Move side effects like email sending and complex escrow calculations to a background job queue (e.g., BullMQ) to improve API response times.
  • Enhanced Logging: Integrate a centralized logging service (e.g., Winston with ELK stack) for better observability of service-layer operations.

Feature Requests

  • Automated Refunds: Implement a service function to handle automated refunds for cancelled orders via the PayMongo API.
  • Store-Specific Commission: Allow dynamic commission rates per store instead of the fixed 5% platform fee.

Compliance

  • Audit Logs: Implement a dedicated AuditLog table in the database to track every status change, including the actor who performed the action.
