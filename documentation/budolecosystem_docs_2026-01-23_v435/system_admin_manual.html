System Admin Manual - v435

Service Management

The Orders Service is part of the core backend. It relies on the following infrastructure:

  • Database: PostgreSQL (via Prisma). Ensure connection pool is monitored.
  • Real-time: Pusher. Verify credentials in .env.
  • Payments: PayMongo. Ensure webhooks are correctly registered to /api/webhooks/paymongo.

Monitoring Orders

Admins can monitor order flow via the Admin Dashboard. The system logs critical transitions in the server console with the prefix [OrdersService].

Troubleshooting

1. Real-time notifications not firing

Check the triggerRealtimeEvent logs. Ensure the PUSHER_APP_ID and PUSHER_KEY are correct in the environment settings.

2. COD payment not marked automatically

Verify that the order status was updated to DELIVERED. The logic only triggers on this specific transition. Check lib/services/ordersService.js around line 380.

Database Maintenance

Run npx prisma studio to manually inspect or correct order records if needed. Use with caution in production.
