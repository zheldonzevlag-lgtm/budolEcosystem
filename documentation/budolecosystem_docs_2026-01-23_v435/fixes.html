Fixes & Updates - v435

Service Layer Implementation

Restored the following functions to lib/services/ordersService.js:

  • getOrderById(orderId): Fetches a single order with full relations (user, store, items, products).
  • createOrder(orderData): Handles multi-store checkout, stock decrement, and cart clearing.
  • updateOrderStatus(orderId, updateData): Manages status transitions and COD automated payments.
  • linkPaymentToOrder(orderId, paymentData): Updates payment details from external gateways.
  • findOrderByPaymentIntent(paymentIntentId): Lookups orders by gateway transaction IDs.

Automated COD Payments

Implemented logic to automatically mark COD orders as paid and credit the seller's wallet when the order status is set to DELIVERED. This uses the escrow service to ensure correct fee calculation.

Real-time Integration

Added triggerRealtimeEvent calls to createOrder and updateOrderStatus to ensure both buyers and sellers are notified of critical events immediately.
