Technical Architecture

The system follows a service-layer architecture where the API routes (Public API) delegate business logic to specialized service modules (Service Layer). This ensures consistency and reusability across different entry points (Web, Webhooks, Internal APIs).

Escrow Integration

File: escrow.js

The updateOrderStatus function calls creditPendingBalance when a COD order is delivered. This automatically calculates the 5% platform fee and credits the net amount to the seller's wallet.

Real-time Notifications

File: realtime.js

Events triggered:

  • store-{storeId}: new-order
  • order-{orderId}: status-updated
  • user-{userId}: order-update

Compliance & Security

  • Data Integrity: All database changes are wrapped in prisma.$transaction.
  • Security: Input validation is performed before any database operation.
  • Audit: Status changes are logged for tracking.
