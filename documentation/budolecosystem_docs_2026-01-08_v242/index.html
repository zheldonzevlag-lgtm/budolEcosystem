Purpose

This release (v242) focuses on enhancing the mobile application's splash screen with dynamic versioning, resolving critical API service issues that were causing build failures, and generating a production-ready APK (v1.2.10) for distribution.

Explanation & Reason

As the budolEcosystem matures, maintaining clear versioning within the app UI becomes essential for support and compliance. A dynamic splash screen ensures the user always knows which version they are running without manual hardcoding in multiple UI components. Furthermore, recent accidental code deletions in the ApiService necessitated a comprehensive restoration and stabilization phase to ensure all wallet and transaction features remain functional.

Issues Identified

  • Build Failure: Duplicate updateMpin method in api_service.dart.
  • Build Failure: Invalid EdgeInsets.bottom(24) constructor in edit_profile_screen.dart.
  • Critical Regression: Accidental deletion of core wallet methods (getBalance, transfer, cashIn, etc.) during manual editing.
  • API Inconsistency: ApiService methods had inconsistent parameter signatures (named vs. positional) compared to their call sites in screens.
  • Static Versioning: The splash screen was using a hardcoded version string instead of retrieving it from package_info_plus.

Fixes Implemented

  • ApiService Stabilization: Restored all 210+ lines of missing wallet and transaction methods.
  • Method Signature Alignment: Updated transfer() and cashIn() to accept Map, matching the calls in SendMoneyScreen and CashInScreen.
  • Layout Correction: Fixed the EdgeInsets constructor in edit_profile_screen.dart to use EdgeInsets.only(bottom: 24).
  • Host Management: Re-implemented missing setHost(), resetHost(), and discoverAndSetHost() methods in ApiService.
  • Dynamic Versioning: Integrated package_info_plus in splash_screen.dart to display the real app version from pubspec.yaml.

Technical Details

Mobile App Changes

  • File: lib/services/api_service.dart - Reconstructed core logic and aligned API signatures.
  • File: lib/screens/splash_screen.dart - Implemented PackageInfo.fromPlatform().
  • File: pubspec.yaml - Incremented version to 1.2.10+1.
  • Build: Successfully generated budolPay_v1.2.10.apk.

Code Details

Dynamic Versioning in Splash Screen

// lib/screens/splash_screen.dart
String _version = 'v1.0.0';

@override
void initState() {
  super.initState();
  _loadVersion();
}

Future<void> _loadVersion() async {
  final info = await PackageInfo.fromPlatform();
  setState(() {
    _version = 'v${info.version}';
  });
}
