Cloudinary Remote Pattern Configuration

Documentation Version: v277 | Status: Implemented
Developer Manual SysAdmin Manual User Manual ← Previous (v276) Next (v278) →

Executive Summary

This update resolves the Invalid src prop error encountered when using next/image with Cloudinary hosted images. The fix involved configuring res.cloudinary.com as a trusted remote pattern in next.config.js.

  • Issue: Runtime Error: Invalid src prop ... hostname "res.cloudinary.com" is not configured.
  • Root Cause: Next.js security policy requires explicit configuration for external image domains.
  • Solution: Added images.remotePatterns to next.config.js allowing Cloudinary assets.

Key Changes

  1. Configuration: Updated next.config.js to include remotePatterns for res.cloudinary.com.
  2. Stability: Verified the configuration with standard Next.js image loading requirements.
  3. Maintenance: Cleaned up minor lint errors in unrelated webhook routes to maintain project health.
