Cloudinary Image Optimization

To resolve the Invalid src prop error on next/image, the next.config.js was updated to whitelist Cloudinary as a remote image source.

Code Implementation

The following configuration was added to next.config.js:


images: {
  remotePatterns: [
    {
      protocol: 'https',
      hostname: 'res.cloudinary.com',
      pathname: '/**',
    },
  ],
},

Technical Details

  • Purpose: Allow Next.js to optimize images hosted on Cloudinary.
  • Reason: Next.js Image component requires explicit domain whitelisting for security.
  • Fixes: Runtime Error in StoreSidebar.jsx where store logos failed to load.
  • Dependencies: Requires next/image usage in the frontend.

Test Results

Verification was performed by running npm run lint and ensuring the next.config.js syntax is valid. End-to-end testing confirms that images from res.cloudinary.com now render without the hostname error.
