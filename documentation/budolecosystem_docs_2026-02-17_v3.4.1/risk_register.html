Risk Register (v3.4.1)

Identified risks associated with the implementation of the new webhook-based cart clearing mechanism.

Risk ID Description Probability Impact Risk Level Owner Status
R-WEB-001 Webhook Delivery Failure: PayMongo fails to send the webhook or the server is temporarily down. Low High (Cart not cleared, order status delayed) Medium Backend Team Mitigated
R-CART-002 Partial Cart Deletion: If a user has multiple items but only one matches the variation logic due to data inconsistency. Very Low Low (Minor user annoyance) Low QA Team Accepted
R-PERF-003 Database Lock: High concurrency of webhook requests causing database locks on the CartItem table. Low Medium (API Latency) Low DevOps Monitored
R-DATA-004 Variation Mismatch: Frontend sends variationId but backend records it as null, causing deletion failure. Medium Medium (Cart persistence bug reoccurs) Medium Frontend Team Addressed
