System Admin Manual (v3.4.1)

This document outlines the administrative procedures for deploying, maintaining, and troubleshooting the Budol Ecosystem v3.4.1 release.

1. Deployment Instructions

1 Backup Database: Before applying any updates, run the full backup script. 
node scripts/backup_db.js
Note: The backup script captures Cart and CartItem tables to ensure user shopping sessions are preserved during maintenance.
2 Pull Latest Code: Ensure you are on the v3.4.1 branch. 
git pull origin v3.4.1
3 Install Dependencies: 
npm install
4 Restart Services: 
pm2 restart all

2. Monitoring & Troubleshooting

Webhook Monitoring

The new cart clearing logic relies on successful webhook processing. Monitor the application logs for the following signature:

[Webhook] Clearing cart for user [userId] based on order [orderId]

Cart Persistence Issues

If users report items remaining in the cart after payment:

  1. Check PayMongo dashboard to confirm the webhook was sent and received a 200 OK response.
  2. Check server logs for "Failed to clear cart in webhook" errors.
  3. Verify if the user has multiple variations of the same product. The new logic is strict about matching variationId.

3. Database Maintenance

Periodic cleanup of abandoned carts is recommended but not yet automated. Ensure the database storage is monitored.

4. Monitoring

Admins should regularly check the system health:

  • Dashboard: /admin - Overview of sales and user activity.
  • Webhooks: /admin/webhooks - Monitor payment gateway events. Look for:
    • Failed Signatures: Indicates potential tampering or configuration issues.
    • Pending Events: Should process within seconds. If stuck, check server logs.
  • Logs: Check server logs for "[Webhook] Clearing cart for user..." messages to confirm successful cart cleanup.

5. Troubleshooting

  • Payments not reflecting: Check the "Webhooks" tab for failed events. Use the "Retry" button if needed.
  • Cart not clearing: Ensure the user didn't have multiple sessions open. Check logs for the specific Order ID.

6. Dependencies & Requirements

  • Node.js Environment: v18+ required for the Next.js server.
  • PostgreSQL: Primary database for storing orders and webhook logs.
  • PayMongo Account: Active account with Webhooks enabled pointing to /api/webhooks/paymongo.

7. Compliance & Security

  • Audit Logs: All admin actions and webhook events are logged for compliance.
  • Access Control: Only authorized admins can access the Webhook logs.
