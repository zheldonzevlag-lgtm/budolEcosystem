Mitigation Plan (v3.4.1)

Strategies implemented to address the risks identified in the Risk Register.

R-WEB-001: Webhook Delivery Failure

Strategy: Redundancy & Idempotency

  • Implemented idempotent logic in the webhook handler so retries (from PayMongo's side) are safe.
  • Future Recommendation: Implement a scheduled job to poll PayMongo for "paid" intents that are still "pending" in our system.
Implemented
R-DATA-004: Variation Mismatch

Strategy: Strict Matching & Fallback

  • The webhook logic now strictly matches variationId to ensure only purchased items are removed.
  • If the initial createOrder service fails to clean up due to a race condition, the webhook serves as the definitive source of truth for "paid" items.
Implemented
R-PERF-003: Database Lock

Strategy: Parallel Execution

  • Used Promise.all for parallel deletions to minimize transaction time.
  • Database indexing on CartItem(cartId, productId, variationId) ensures fast lookups.
Implemented
