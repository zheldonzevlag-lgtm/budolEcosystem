Developer Manual Update v157

Purpose

To resolve conflicting success/failure messages during P2P transfers and ensure recipients can be identified by email or phone number.

Technical Details

Recipient Resolution Logic

The transaction-service was updated to resolve a recipient (email or phone number) to a receiverId before processing the transaction. This prevents the service from failing with a 500 error after the initial wallet balance update, which was the cause of the conflicting messages.


// transaction-service/index.js
let resolvedReceiverId = receiverId;
if (!resolvedReceiverId && recipient) {
    const recipientUser = await prisma.user.findFirst({
        where: {
            OR: [
                { email: recipient },
                { phoneNumber: recipient }
            ]
        }
    });
    if (!recipientUser) {
        throw new Error(`Recipient not found: ${recipient}`);
    }
    resolvedReceiverId = recipientUser.id;
}
