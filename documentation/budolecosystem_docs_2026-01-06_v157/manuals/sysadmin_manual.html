System Admin Manual Update v157

Ecosystem Management

After updating core logic in any service, it is recommended to restart the entire ecosystem to ensure all services are using the latest code and environment configurations.

Restart Procedure

  1. Kill existing node processes: Get-Process node | Stop-Process -Force
  2. Start the ecosystem: node start-ecosystem.js
  3. Choose option 3 (SKIP) to preserve existing environment variables.

Verification

Check the console logs of the start-ecosystem.js runner to ensure all services are listening on their respective ports:

  • Auth Service: 8001
  • Wallet Service: 8002
  • Transaction Service: 8003
  • Gateway API: 8080

Risk & Mitigation

Risk: Database lock contention during high-volume P2P transfers.

Mitigation: Ensure the database (PostgreSQL) is properly tuned and has sufficient connection pool size for the microservices.
