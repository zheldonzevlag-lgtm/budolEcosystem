🚚 Lalamove Tracker Accuracy Fix

Driver Information Display & Status Synchronization

December 6, 2025

📋 Overview

This document details the comprehensive fixes applied to ensure accurate Lalamove order tracking and dynamic driver information display in Budolshap V3.

🔴 Issues Identified

  1. Mock Driver Data: LalamoveTracking component was displaying hardcoded mock driver information instead of actual data from the order
  2. Incomplete Driver Info: Webhook handler and trackOrder function weren't fetching/storing complete driver details (rating, vehicle type, photo)
  3. Inaccurate Status Mapping: Order progress tracker wasn't correctly mapping Lalamove-specific statuses (ASSIGNING_DRIVER, ON_GOING, PICKED_UP, COMPLETED)
  4. Missing Sync Mechanism: No way to manually refresh Lalamove data when needed

📁 Files Modified

1. LalamoveTracking Component

components/LalamoveTracking.jsx
Changes:
  • Removed mock driver data fallback
  • Now uses actual order.shipping.driverInfo from database
  • Shows "Waiting for Driver Assignment" message when driver not yet assigned
  • Conditionally renders driver details (rating, vehicle type, plate number) only when available
  • Added proper null checks for all driver fields

2. Lalamove Service

services/lalamove.js
Enhanced trackOrder() method:
  • Returns complete driver information: name, phone, plateNumber, photo, rating, vehicleType
  • Returns location data with lat/lng coordinates
  • Returns shareLink for tracking map
  • Returns estimated pickup and delivery times
  • Added detailed logging for debugging

3. Webhook Handler

app/api/webhooks/lalamove/route.js
Critical Enhancement:
  • When driver is assigned (ON_GOING event), webhook now calls Lalamove API to fetch complete driver details
  • Stores comprehensive driver info in shipping.driverInfo field
  • Updates location data from API response
  • Updates shareLink for live tracking
  • Falls back to webhook data if API call fails
  • Timeline events now include full driver information

4. Order Progress Tracker

components/OrderProgressTracker.jsx
Accurate Status Mapping:
  • Added Lalamove-specific status mapping for each stage
  • Checks both order.status and order.shipping.status
  • Correctly maps: ASSIGNING_DRIVER → Payment Confirmed, ON_GOING → Driver Assigned, PICKED_UP → Picked Up, COMPLETED → Delivered

5. Sync API Endpoint (NEW)

app/api/orders/[orderId]/sync-lalamove/route.js
Manual Sync Feature:
  • POST endpoint to manually sync Lalamove order data
  • Fetches latest driver info, location, and status from Lalamove API
  • Updates database with fresh data
  • Returns updated order object
  • Useful for testing and troubleshooting

🔔 Webhook Processing Flow

1. Webhook Received

Lalamove sends webhook with event type and basic data

2. Signature Verification

Verify webhook authenticity using HMAC signature

3. Find Order

Locate order in database using Lalamove booking ID

4. Fetch Complete Data

If driver assigned, call Lalamove API to get full driver details

5. Update Database

Store driver info, location, status in order.shipping

6. Update Order Status

Change order status based on event (PICKED_UP → SHIPPED, COMPLETED → DELIVERED)

7. Send Email

Notify customer of status change

8. Return 200 OK

Acknowledge webhook receipt to Lalamove

⚙️ Technical Notes

API Response Mapping

The Lalamove API returns driver data in result.data.driver, not result.data.driverInfo. The trackOrder function has been updated to use the correct field.

Webhook Data Limitations

Lalamove webhooks only send driverId, not complete driver information. The webhook handler now makes an additional API call to fetch full details when needed.

Database Schema

The order.shipping JSON field stores all Lalamove-related data:

  • provider: "lalamove"
  • bookingId: Lalamove order ID
  • status: Current Lalamove status
  • driverInfo: Complete driver details object
  • location: Current GPS coordinates
  • shareLink: Tracking map URL
  • timeline: Array of status change events

Auto-Refresh

The order details page automatically refreshes every 30 seconds for active orders, ensuring users see the latest driver information and location updates.

🔧 Troubleshooting

Driver Information Not Showing

  • Check if webhook was received (check Vercel logs)
  • Verify Lalamove API credentials are correct
  • Use sync endpoint to manually fetch data
  • Check if order status is ON_GOING or later

Status Not Updating

  • Verify webhook endpoint is accessible
  • Check webhook signature verification
  • Ensure LALAMOVE_WEBHOOK_SECRET is set correctly
  • Check Lalamove webhook configuration in dashboard

Map Not Displaying

  • Verify shareLink is present in order.shipping
  • Check browser console for iframe errors
  • Ensure order has been booked with Lalamove
