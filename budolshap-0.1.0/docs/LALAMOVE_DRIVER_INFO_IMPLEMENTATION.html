⚠️ 3. Issues Encountered

Issue #1: Driver Information Not Displaying

Problem: Driver information card was not visible on the order details page despite driver being assigned.

Symptoms:

  • Order page showed tracking map but no driver details
  • Database had driver data but UI didn't display it
  • Console showed no errors

Root Cause: Component rendering logic was checking for driver data incorrectly.

Issue #2: Generic Driver Name Instead of Real Name

Problem: System was showing "Driver #80557" instead of "TestDriver 34567" (the actual driver name from Lalamove).

Symptoms:

  • Lalamove Partner Portal showed "TestDriver 34567"
  • Application showed "Driver #80557" or "Lalamove Driver"
  • Driver ID was being used as fallback name

Root Cause: Lalamove Sandbox API returns only driverId without full driver details. The tracking API doesn't include driver name, phone, or plate number.

Issue #3: Webhook Data Being Overwritten

Problem: Driver information from webhooks was being replaced with fallback data during sync operations.

Symptoms:

  • Driver name changed from "TestDriver 34567" to "Lalamove Driver"
  • Phone number changed to "Available in tracking map"
  • Data loss after auto-sync runs

Root Cause: The trackOrder() function was creating fallback driver objects when API returned only driverId, overwriting existing webhook data.

Issue #4: Missing Driver Rating

Problem: Driver rating was not being displayed even though the UI component supported it.

Symptoms:

  • Rating field was null in database
  • UI showed driver card without rating stars

Root Cause: Rating field was not being captured from webhook payload or was not included in test data.

Issue #5: Map Loading Only in Incognito Mode

Problem: Lalamove tracking map loaded correctly in incognito mode but failed in regular browser sessions.

Symptoms:

  • Map iframe showed blank or error in normal mode
  • Worked perfectly in incognito/private browsing
  • Hard refresh sometimes fixed the issue temporarily

Root Cause: Aggressive browser caching of JavaScript and CSS files. Incognito mode worked because it started with no cache.

✅ 4. Solutions Implemented

Solution #1: Fixed Component Rendering Logic

Fix: Updated LalamoveTracking.jsx to properly check for driver data and render the driver information card.

Changes Made:

// Before: Incorrect check if (!driver) return null; // After: Proper conditional rendering if (!driver) { // Show map with info message return ( <div> <Map /> <InfoMessage>Driver details displayed during active deliveries</InfoMessage> </div> ); } // Show full driver card when data available return ( <div> <Map /> <DriverCard driver={driver} /> </div> );

Result: ✅ Driver card now displays correctly

Solution #2: Preserved Webhook Data

Fix: Modified services/lalamove.js to return null instead of creating fallback data when only driverId is available.

Changes Made:

// File: services/lalamove.js // Lines 336-340 async trackOrder(orderId) { const result = await this.apiRequest('GET', `/v3/orders/${orderId}`); let driverInfo = null; if (result.data.driverInfo) { // Full driver details available driverInfo = { ...result.data.driverInfo }; } else if (result.data.driver) { // Driver object available driverInfo = { ...result.data.driver }; } else if (result.data.driverId) { // ❌ OLD: Create fallback data // driverInfo = { name: 'Driver #' + driverId, ... } // ✅ NEW: Return null to preserve webhook data driverInfo = null; } return { status: result.data.status, driver: driverInfo, // Will be null if only driverId location: result.data.location, shareLink: result.data.shareLink }; }

Result: ✅ Webhook data preserved during sync operations

Solution #3: Enhanced Sync Endpoint

Fix: Updated /api/orders/[orderId]/sync-lalamove to preserve existing driver data when API returns null.

Changes Made:

// File: app/api/orders/[orderId]/sync-lalamove/route.js // Lines 54-59 // Add driver info if available from API, otherwise keep existing if (trackingData.driver) { updatedShipping.driver = trackingData.driver; } else if (freshOrder.shipping?.driver) { // ✅ Preserve existing driver info if API returns null updatedShipping.driver = freshOrder.shipping.driver; }

Result: ✅ Driver information persists across sync operations

Solution #4: Added Driver Rating

Fix: Updated webhook handler and database to capture and store driver rating.

Changes Made:

// File: app/api/webhooks/lalamove/route.js // Line 156 if (driver) { updatedShipping.driver = { name: driver.name, phone: driver.phone, plateNumber: driver.plateNumber, photo: driver.photo, vehicleType: driver.vehicleType, // ✅ Capture driver rating rating: driver.rating || driver.averageRating || null, driverId: driver.driverId || driver.id || null }; }

Result: ✅ Driver rating now displays with star icon (⭐ 5.0)

Solution #5: Fixed Browser Caching Issues

Fix: Implemented cache-busting strategies and proper cache headers.

Changes Made:

Key Learnings:

Important Insights:
  • Webhooks vs API: Lalamove webhooks provide complete driver details, while the tracking API only returns limited data (driverId only in sandbox)
  • Data Preservation: Always preserve webhook data instead of overwriting with API fallbacks
  • Cache Management: Implement proper cache-busting strategies for dynamic content
  • Auto-Refresh: Combine webhooks (push) with polling (pull) for best real-time experience
  • Testing: Create comprehensive test scripts for database verification
