# Ngrok & Local Development Setup for Budolshap

## Overview
This guide walks you through the complete steps to:
1. **Configure Ngrok** with your authentication token.
2. **Run the Next.js development server** on port `3000`.
3. **Expose the local server** to the internet using Ngrok.
4. **Verify the public URL** and troubleshoot common issues.

The instructions are written for a Windows environment (PowerShell) and assume the project lives at `C:\wamp64\www\budolshap`.

## 2. Add Ngrok Auth Token
Ngrok requires an auth token to create tunnels.

```powershell
# Replace <YOUR_TOKEN> with the token you received from https://dashboard.ngrok.com/get-started/your-authtoken
c:\laragon\bin\ngrok\ngrok.exe authtoken <YOUR_TOKEN>
```

> **Note**: The token is stored in `~/.ngrok2/ngrok.yml` and will be reused for all future tunnels.

### What we did
- Ran the command with token `35sssFR77x8M1hvEcJj4xsBB2Nj_Zg24C18JHDFr2Asfooi5`.
- The command completed with **no output**, indicating success.

## 3. Start the Next.js Development Server
From the project root (`C:\wamp64\www\budolshap`):

```powershell
npm install   # (only needed the first time)
npm run dev   # starts Next.js on http://localhost:3000
```

You should see output similar to:
```
▲ Next.js 15.3.5 (Turbopack)
- Local:        http://localhost:3000
- Network:      http://<your‑ip>:3000
- Environments: .env

✓ Starting...
✓ Compiled middleware in 358ms
✓ Ready in 2.7s
```
The server must stay running while you create the Ngrok tunnel.

## 4. Launch Ngrok Tunnel
Open **a second PowerShell window** (so the dev server stays alive) and run:

```powershell
c:\laragon\bin\ngrok\ngrok.exe http 3000
```

**Do not** wrap the path in extra quotes; PowerShell will interpret them as part of the command and cause errors.

### Expected output
```
Session Status                online
Version                       3.1.x
Region                        Asia
Forwarding                    https://<random>.ngrok-free.dev -> http://localhost:3000
```
The **public URL** (`https://<random>.ngrok-free.dev`) is now forwarding traffic to your local app.

## 5. Verify the Tunnel
1. Open a browser and navigate to the public URL.
2. You should see the same UI you get when visiting `http://localhost:3000`.
3. For debugging, open Ngrok’s web UI at <http://127.0.0.1:4040>. It shows:
   - Active tunnels
   - Request logs
   - Replay button
   - JSON endpoint (`/api/tunnels`) for programmatic access

## 6. Common Issues & Fixes
| Symptom | Cause | Fix |
|---------|-------|-----|
| Terminal closes instantly after `ngrok http 3000` | Command was quoted incorrectly or run from a shortcut that exits when the process ends. | Run the command **directly** in an interactive PowerShell window as shown above. |
| `Unexpected token 'http'` error | PowerShell interpreted part of the command as a string because of stray quotes. | Remove the surrounding quotes around the executable path. |
| No public URL appears | Ngrok cannot find a service listening on port 3000. | Ensure `npm run dev` is running and listening (check the console or `netstat -ano | findstr :3000`). |
| `ERR_NGROK_0001: auth token not configured` | Token not set or stored in a different user profile. | Re‑run `ngrok authtoken <YOUR_TOKEN>`.
| Tunnel disconnects after a short time | Free tier idle timeout (30 seconds) or network firewall. | Keep the tunnel active (send a request periodically) or upgrade to a paid plan for persistent tunnels. |
