Phase 2: UI Updates

This phase involved replacing all hardcoded text references to the escrow period with dynamic values fetched from the APIs updated in Phase 1.

Seller Dashboard Updates

The "In Escrow" card hint now correctly reflects the protection period:


// Before:
hint: 'Funds held in escrow (released 7 days after delivery)'

// After:
hint: `Funds held in escrow (released ${dashboardData.protectionWindowDays} days after delivery)`

Public Order Page Updates

The "BudolShap Guarantee" banner now shows the correct dynamic window to buyers for all order statuses:


// Before:
"Get the items you ordered or get your money back. Protected by escrow."
"Funds will be released to the seller in 3 days unless you request a Return/Refund."

// After:
`Get the items you ordered or get your money back. Protected by escrow for ${order.protectionWindowDays || 7} days after delivery.`
`Funds will be released to the seller in ${order.protectionWindowDays || 7} days unless you request a Return/Refund.`

Admin Escrow Settings Updates

The settings page now explicitly shows the currently active window in its descriptive text, reducing confusion during configuration.


<p>The current protection window is <strong>{protectionDays} days</strong>. Industry standard is 7 days...</p>

Other UI Touchpoints

  • Seller Wallet: Updated the pending balance explanation text.
  • Admin Dashboard: Updated the escrow stats card hint to match the configured period.
