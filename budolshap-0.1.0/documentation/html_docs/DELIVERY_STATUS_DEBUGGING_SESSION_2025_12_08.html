Delivery Status Debugging Session

Date: December 8, 2025

Status: RESOLVED

✅ Resolution: Missing Provider Field

Root Cause: The API response for successful booking did not return provider: 'lalamove'. The frontend logic explicitly checks order.shipping.provider === 'lalamove'. Because the optimistic UI update relied solely on the API response, the provider check failed immediately after booking.

Fix Applied:

  • Updated app/api/shipping/lalamove/book/route.js to include provider: 'lalamove' in the response.
  • Updated app/store/orders/page.jsx to optimistically set provider: 'lalamove' and update order status to PROCESSING.

🚨 Issue: Booked Orders Not Visible in "Booked" Tab

Description: After successfully booking a Lalamove delivery for an order, the order does not appear in the "Booked" filter tab in the Store Orders dashboard.

Impact: Sellers cannot easily track which orders have been booked, leading to confusion and potential double-booking or missed shipments.

Hypothesis & Investigation Plan

1. Frontend Delivery Status Logic

The "Booked" tab relies on the getDeliveryStatus helper function in the frontend. This function likely checks for a bookingId within the order's shipping details.

Check: Does the order object received by the frontend actually contain the shipping object with a valid bookingId?

2. Backend Booking Persistence

When the /api/shipping/lalamove/book endpoint is called, it should save the Lalamove orderRef or orderId into our database as the bookingId.

Check: Verify app/api/shipping/lalamove/book/route.js is correctly saving this ID to the database.

3. Order Fetching Logic

When valid orders are fetched via /api/orders, the associated shipping information must be included in the response.

Check: Verify app/api/orders/route.js includes the shipping relation in its Prisma query.

Implementation Plan

  1. Inspect app/api/shipping/lalamove/book/route.js to confirm database write.
  2. Inspect app/api/orders/route.js to confirm data retrieval.
  3. Debug app/store/orders/page.jsx to ensure frontend logic matches backend data structure.
  4. Apply fixes and deploy.
