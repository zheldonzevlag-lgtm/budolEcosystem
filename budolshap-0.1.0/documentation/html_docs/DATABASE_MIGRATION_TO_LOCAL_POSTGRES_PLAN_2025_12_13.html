Migration Plan: Vercel Postgres to Local Postgres

Goal: Replicate the production database to a local PostgreSQL instance.

Date: December 13, 2025

Strategy: Parallel Migration. We will create a new local database named budolshap_local and restore the backup to it, without disrupting the current application connection.

Prerequisites

  • Local PostgreSQL Server: Running on `localhost:5432`.
  • Credentials: Default `postgres` / `admin` (or user-supplied).
  • Backup Data: Existing backup at backups/2025-12-13_12-19/.

Execution Plan

Step 1: Provision Local Database

We do not need a special schema file because the provider is already postgresql. We will use the standard schema but override the connection string to point to localhost.

# Command to create database structure
$env:DATABASE_URL="postgresql://postgres:admin@localhost:5432/budolshap_local?schema=public"
npx prisma db push

This will create the database budolshap_local if it doesn't exist and apply the current schema.

Step 2: Restore Data

We will use the scripts/restore-database.js script with the same local connection string. 

# Command to restore data
$env:DATABASE_URL="postgresql://postgres:admin@localhost:5432/budolshap_local?schema=public"
node scripts/restore-database.js 2025-12-13_12-19
Step 3: Verification

We will verify the data restoration using the script output logs. Since the target is also PostgreSQL, we expect full compatibility, including the Order data that failed in MySQL.

Important Notes

Credential Check: Ensure your local PostgreSQL password is `admin`. If it is different, please inform us before execution so we can adjust the connection string.
