Plan: Dynamic Map Synchronization

Objective

Ensure the map display accurately reflects the real-time status from Lalamove, keeping it in sync with the order status in Budolshap.

Technical Constraint

Important: The map is displayed via an iframe loading a URL provided by Lalamove. We cannot control the content inside that iframe via code (Cross-Origin Policy). If Lalamove's link shows "Drop-off Point" even after completion, that is a behavior of their tracking page.

Strategy

1. Real-time Status Refresh

We will implement a polling or refresh mechanism on the Order Details page.

  • When the user views the page, we will check the status against the Lalamove API (via our backend).
  • If the API says "COMPLETED", we ensure the frontend updates immediately.

2. Handling the Map Display

If the order is DELIVERED, the Lalamove tracking link should eventually show "Completed" or expire. If it continues to show "Drop-off", it might be a caching issue or a sandbox behavior.

Proposed Logic:

  • We cannot change the map pixels.
  • However, we can add a visual indicator above the map saying "Status: Delivered" to confirm our system has received the completion signal, even if the driver hasn't closed the app on their end or the map is lagging.
