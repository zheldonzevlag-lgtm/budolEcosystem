Session Timeout UX/UI Improvements

Implementation Walkthrough • December 13, 2025

Overview

This update improves the user experience when a session expires due to inactivity. Instead of a generic login screen or a race condition where specific error pages flash, the system now gracefully handles the timeout and subsequent re-login.

Changes Implemented

Feature Dynamic Session Timeout

  • Settings: The application now correctly respects the Idle Session Timeout settings configured in the Admin Panel (Security Settings).
  • Warning: A modal appears 1 minute before timeout (e.g., at 14 minutes if set to 15).
  • Logout: At 0 seconds, the user is automatically logged out.

UI Update Improved "Session Expired" UI

  • New Header: Login modal now displays "Session Expired" instead of "Welcome Back" when triggered by inactivity.
  • Clear Context: Subtext updated to "Please sign in again to continue working."
  • Consistency: This state is triggered by both automatic timeout and the "Log Out" button in the warning modal.

Bug Fix Unauthorized Flash Prevention

  • The Issue: Previously, logging in after a timeout would briefly show the "You are not authorized" page before reloading.
  • The Fix: The login modal now transitions to a "Redirecting..." loading state upon success.
  • Result: Smooth experience with no jarring error screens during the reload process.

Verification Results

  • Confirmed conditional rendering in LoginModal.

Manual User Verification

  • Auto Logout: Confirmed "Session Expired" login modal appears after timer ends.
  • Re-login: Confirmed smooth transition with "Redirecting..." spinner, effectively hiding the "Unauthorized" page.
