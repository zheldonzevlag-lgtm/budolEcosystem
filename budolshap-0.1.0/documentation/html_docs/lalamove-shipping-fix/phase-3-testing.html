Lalamove Shipping Fee Discrepancy Fix

Implementation Plan - Phase 3: Testing Protocol & Validation

Summary Phase 1: Investigation Phase 2: Implementation Phase 3: Testing

Testing Strategy

The testing phase ensures that the fixes implemented in Phase 2 are robust and correctly handle all Lalamove pricing scenarios, specifically targeting the identified discrepancy root causes.

Manual Test Cases

TC-01: Price Consistency (Standard Flow)

Scenario: Place an order with a standard address during non-peak hours.

Steps: Select delivery address -> View quote in widget -> Proceed to checkout -> Complete order.

Expected Outcome: The price shown in the Widget (e.g., 73.00) matches the price in the Order Confirmation and the final Lalamove dashboard booking exactly.

TC-02: Peak Hour Surcharges

Scenario: Place an order during a high-demand period (e.g., heavy rain or rush hour).

Steps: Trigger a quote when surcharges are active -> Verify "Total Include Tax" is displayed.

Expected Outcome: The UI explicitly includes the surcharge in the total shipping fee (e.g., 102.00 instead of 73.00).

TC-03: Quote Expiration & Refresh

Scenario: Simulate a long checkout session (>30 minutes).

Steps: Get a quote -> Wait 31 minutes -> Click "Place Order".

Expected Outcome: Backend successfully refreshes the quote; if the price has changed significantly, the system should either update the order or notify the user before finalizing payment.

TC-04: Coordinate Precision

Scenario: Verify coordinates stored in the database.

Steps: Inspect the Order record in Prisma Studio after placement.

Expected Outcome: Coordinates are stored with 6 decimal places and match the coordinates sent in the initial quote request.

Automated Validation

Run the existing test scripts to ensure no regressions in the Lalamove API communication:

  • node test-quote.js
  • node test-quote-stops.js

Rollback Procedures

In case of critical failure (e.g., 500 errors on all bookings):

  1. Revert changes to lib/services/shippingService.js.
  2. Revert services/lalamove.js to the previous price extraction logic.
  3. Flush the system settings cache to restore default behavior.
