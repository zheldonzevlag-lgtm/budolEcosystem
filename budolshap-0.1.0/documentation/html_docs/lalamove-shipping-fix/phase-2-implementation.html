Lalamove Shipping Fee Discrepancy Fix

Implementation Plan - Phase 2: Implementation of Fixes

Summary Phase 1: Investigation Phase 2: Implementation

Overview

This phase details the technical changes required to ensure accurate shipping quotes and eliminate the 73 vs 102 discrepancy. The focus is on price standardization, coordinate precision, and data integrity.

Technical Tasks

1. Standardize Price Extraction

Task: Update services/lalamove.js to prioritize totalIncludeTax.

File: services/lalamove.js

Modify getQuote return structure to explicitly include tax and surcharges in the amount displayed to the user.

2. Coordinate Normalization

Task: Implement coordinate rounding to 6 decimal places across the stack.

Ensure that both the LalamoveWidget.jsx (frontend) and shippingService.js (backend) use the exact same precision for latitude and longitude.

// Example helper to be used
const normalizeCoord = (coord) => parseFloat(parseFloat(coord).toFixed(6));

3. Enhanced Order Shipping Data

Task: Expand the shipping JSON field in the Order model.

Store the full priceBreakdown from Lalamove to provide transparency in the admin and store owner dashboards.

// Enhanced shipping object structure
{
  "provider": "lalamove",
  "cost": 102.00,
  "breakdown": {
    "base": 60.00,
    "distance": 13.00,
    "surcharge": 29.00
  },
  "coordinates": { "lat": 14.123456, "lng": 121.123456 }
}

4. Quote Validation Middleware

Task: Add a validation check in lib/services/ordersService.js.

Before finalizing an order, compare the requested shipping cost with a fresh backend-calculated cost. If the difference exceeds a threshold (e.g., 5%), flag the order for review or prompt the user.

Dependencies

  • Lalamove v3 API access (Production/Sandbox)
  • Prisma Client (for database updates)
  • Nominatim API (for continued geocoding fallback)

Risk Mitigation

  • Backward Compatibility: Ensure existing orders with the old shipping JSON structure still load correctly in the UI.
  • API Limits: Be mindful of Lalamove API rate limits when implementing additional validation calls.
