Phase 1: Root Cause Analysis

The investigation revealed several key issues in the Lalamove shipping flow that prevented correct fee calculation and order booking.

Identified Issues

  • 1. Data Loss in Quote Response

    The shippingService.js was fetching quotes from Lalamove but only returning the price and basic ETA info to the frontend. It was discarding the stops array which contains critical stopIds required for order creation.

  • 2. Order Creation Payload Mismatch

    The frontend OrderSummary.jsx was receiving the quote but didn't have the stops data to send back to the /api/orders endpoint. This resulted in the Order record in the database missing necessary stop data in its shipping JSON field.

  • 3. Calculation Path Disconnect

    While the frontend calculated the total price correctly for display, the backend ordersService.js needed verification to ensure it properly extracted the shipping.cost from the Lalamove-provided data instead of falling back to default store shipping profiles.

Technical Audit

The Lalamove API (v3) requires stopId from a previous quotation for both pickup and delivery stops when creating an order. Without these IDs, the order creation call in services/lalamove.js would fail with validation errors.
