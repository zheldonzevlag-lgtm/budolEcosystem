🔄 Complete Production Order Flow

End-to-End Delivery Process

1 Order Creation & Checkout
Customer Budolshap System
Customer completes checkout process and selects Lalamove as the delivery method. The system validates the order and prepares for delivery booking.
System Actions:
  • Validate customer delivery address
  • Calculate total order amount
  • Store order in database with status: PENDING
  • Prepare pickup and delivery coordinates
2 Request Delivery Quote
Budolshap System Lalamove API
System requests a delivery quote from Lalamove API to get pricing and estimated delivery time.
API Call: POST /v3/quotations
  • Send pickup location (seller's store address)
  • Send delivery location (customer's address)
  • Specify service type (motorcycle, sedan, van, etc.)
  • Receive quote with pricing and ETA
3 Book Delivery Order
Budolshap System Lalamove API
After payment confirmation, the system books the delivery order with Lalamove using the quotation.
API Call: POST /v3/orders
  • Submit quotation ID from step 2
  • Include pickup and delivery stop details
  • Provide contact information for both parties
  • Receive order ID and initial status: ASSIGNING
  • Store lalamoveOrderId in database
4 Driver Assignment (Automated)
Lalamove Platform Driver
Lalamove's platform automatically notifies nearby available drivers. A driver accepts the delivery request through their Lalamove driver app.
Lalamove Actions:
  • Broadcast delivery request to nearby drivers
  • Driver reviews pickup/delivery locations
  • Driver accepts the order
  • Status changes to: ON_GOING
  • Webhook sent to Budolshap 🔔
5 Webhook: Driver Accepted
Lalamove API Budolshap System
Lalamove sends a webhook notification to Budolshap when a driver accepts the order.
Webhook Endpoint: POST /api/webhooks/lalamove
  • Receive driver information (name, phone, vehicle)
  • Update order status in database
  • Send notification to customer: "Driver assigned!"
  • Send notification to seller: "Driver on the way to pickup"
  • Enable real-time tracking
6 Driver En Route to Pickup
Driver Seller
Driver navigates to the seller's store to pick up the order. Customer and seller can track driver's location in real-time.
Available Features:
  • Real-time GPS tracking of driver
  • Estimated time of arrival (ETA) to pickup location
  • Driver contact information available
  • Status remains: ON_GOING
7 Order Pickup at Store
Driver Seller
Driver arrives at the store, collects the package from the seller, and confirms pickup in the Lalamove driver app.
Pickup Process:
  • Driver verifies order details with seller
  • Seller hands over the package
  • Driver confirms pickup in app
  • Status changes to: PICKED_UP
  • Webhook sent to Budolshap 🔔
8 Webhook: Order Picked Up
Lalamove API Budolshap System
Lalamove sends a webhook notification confirming the order has been picked up.
System Actions:
  • Update order status to "Picked Up"
  • Record pickup timestamp
  • Send notification to customer: "Your order is on the way!"
  • Update tracking information
  • Calculate estimated delivery time
9 Driver En Route to Customer
Driver Customer
Driver navigates to the customer's delivery address. Customer can track driver's real-time location.
Delivery Phase:
  • Real-time GPS tracking to delivery location
  • Updated ETA to customer's address
  • Customer can contact driver if needed
  • Status remains: PICKED_UP
10 Delivery Completed
Driver Customer
Driver arrives at customer's location, hands over the package, and confirms delivery completion in the app.
Completion Process:
  • Driver verifies delivery address
  • Customer receives the package
  • Driver may take proof of delivery photo
  • Driver confirms completion in app
  • Status changes to: COMPLETED
  • Webhook sent to Budolshap 🔔
11 Webhook: Delivery Completed
Lalamove API Budolshap System
Lalamove sends final webhook notification confirming successful delivery completion.
Final Actions:
  • Update order status to "Completed"
  • Record delivery timestamp
  • Store proof of delivery (if available)
  • Send notification to customer: "Delivery completed!"
  • Notify seller of successful delivery
  • Trigger payment release to seller (if applicable)
  • Request customer review/rating
12 Post-Delivery Processing
Budolshap System Customer
System finalizes the order and handles post-delivery activities.
Post-Delivery Tasks:
  • Archive delivery tracking data
  • Generate delivery receipt
  • Update order history
  • Enable customer review/feedback
  • Calculate delivery performance metrics
  • Close order ticket

🔔 Webhook Integration Details

How Webhooks Work

1. Webhook Configuration

Webhook URL must be configured in Lalamove Developer Portal:

Production Webhook URL: https://budolshap.vercel.app/api/webhooks/lalamove Webhook Secret: [Configured in environment variables]

2. Webhook Events

Lalamove sends webhooks for these events:

  • ORDER_STATUS_CHANGED: When order status updates
  • DRIVER_ASSIGNED: When driver accepts the order
  • DRIVER_LOCATION_UPDATED: Real-time location updates
  • ORDER_PICKED_UP: When driver picks up package
  • ORDER_COMPLETED: When delivery is completed
  • ORDER_CANCELED: When order is cancelled

3. Webhook Payload Example

{ "event": "ORDER_STATUS_CHANGED", "orderId": "PH-123456789", "status": "ON_GOING", "timestamp": "2025-11-27T15:30:00Z", "driver": { "name": "Juan Dela Cruz", "phone": "+639171234567", "plateNumber": "ABC1234", "location": { "lat": 14.5995, "lng": 120.9842 } }, "stops": [ { "stopId": "stop_pickup_001", "status": "COMPLETED", "arrivedAt": "2025-11-27T15:25:00Z" }, { "stopId": "stop_delivery_001", "status": "PENDING", "eta": "2025-11-27T15:45:00Z" } ] }

4. Webhook Security

All webhooks are verified using HMAC SHA256 signature:

// Webhook verification in /api/webhooks/lalamove/route.js const signature = request.headers.get('x-lalamove-signature'); const payload = await request.text(); const expectedSignature = crypto .createHmac('sha256', LALAMOVE_WEBHOOK_SECRET) .update(payload) .digest('hex'); if (signature !== expectedSignature) { return new Response('Invalid signature', { status: 401 }); }

5. Webhook Processing

System processes webhooks and updates database:

  • Verify webhook signature
  • Parse webhook payload
  • Update order status in database
  • Store driver information
  • Send notifications to customer/seller
  • Update real-time tracking data
  • Log webhook event for audit trail
  • Return 200 OK to Lalamove

⚠️ Error Handling & Edge Cases

Common Scenarios

❌ No Driver Available

Scenario: No drivers accept the order within timeout period

System Response:

  • Lalamove automatically cancels the order
  • Webhook sent with CANCELED status
  • Customer notified: "Unable to find driver"
  • Offer alternative: Retry booking or choose different delivery method
  • Refund delivery fee if already charged

⚠️ Driver Cancellation

Scenario: Driver cancels after accepting

System Response:

  • Order status reverts to ASSIGNING
  • Lalamove searches for new driver
  • Customer notified of the change
  • New driver assigned automatically

ℹ️ Customer Not Available

Scenario: Customer not at delivery location

Driver Actions:

  • Driver attempts to contact customer via phone
  • Wait at location for specified time (usually 10-15 minutes)
  • If no response, driver may return package to seller
  • Additional fees may apply for failed delivery

❌ Webhook Delivery Failure

Scenario: Webhook fails to reach Budolshap server

Fallback Mechanism:

  • Lalamove retries webhook delivery (up to 3 times)
  • Budolshap implements polling as backup (every 2 minutes)
  • Manual status sync via GET /v3/orders/{orderId}
  • Admin dashboard shows webhook delivery status

✅ Production Best Practices

1. Order Creation

  • Always validate addresses before booking
  • Provide accurate contact information for both parties
  • Include clear pickup and delivery instructions
  • Verify quote before creating order
  • Implement idempotency to prevent duplicate orders

2. Webhook Handling

  • Always verify webhook signatures
  • Process webhooks asynchronously to avoid timeouts
  • Return 200 OK immediately after receiving webhook
  • Implement retry logic for failed webhook processing
  • Log all webhook events for debugging
  • Handle duplicate webhooks gracefully (idempotent processing)

3. Error Handling

  • Implement exponential backoff for API retries
  • Provide clear error messages to users
  • Have fallback mechanisms for critical failures
  • Monitor API rate limits and quotas
  • Set up alerts for repeated failures

4. User Experience

  • Show real-time updates without requiring page refresh
  • Provide clear status indicators and progress bars
  • Enable easy communication between customer and driver
  • Send proactive notifications at key milestones
  • Offer self-service tracking and support

5. Monitoring & Analytics

  • Track delivery success rates
  • Monitor average delivery times
  • Analyze cancellation reasons
  • Measure customer satisfaction scores
  • Set up alerts for anomalies (e.g., high cancellation rate)
