Phase 2: API & Hook Integration

Synchronizing events across seller and buyer channels.

Implementation Details

This phase focused on ensuring that every status update event is broadcasted to both the Store channel (for sellers) and the User channel (for buyers).

1. Hook Update: hooks/useRealtimeBuyerOrders.js

Added the order-updated event listener to the buyer-side hook. This ensures that the buyer's order list refreshes immediately when a status changes.

channel.bind('order-updated', (data) => {
    console.log('⚡ [Buyer] Order Updated (Pusher):', data);
    mutate(); // Triggers SWR refresh
});
2. API Update: app/api/orders/[orderId]/sync-lalamove/route.js

Updated the Lalamove synchronization endpoint to trigger events for both channels simultaneously using Promise.all.

await Promise.all([
    triggerRealtimeEvent(`store-${order.storeId}`, 'order-updated', eventData),
    triggerRealtimeEvent(`user-${order.userId}`, 'order-updated', eventData)
]);
3. Webhook Update: app/api/webhooks/lalamove/route.js

The webhook handler now broadcasts updates to the buyer as well, ensuring that background updates from Lalamove are reflected in the buyer's UI in real-time.

4. General Sync Update: app/api/orders/[orderId]/sync/route.js

Applied the same dual-channel triggering logic to the general synchronization endpoint.

Status

All core synchronization logic is now dual-channel. The next phase will focus on refining the Buyer's Order Detail page to ensure it reacts to these real-time events without relying solely on polling.
