Phase 1: Root Cause Analysis

Identifying why "booked" status updates fail for buyers.

Executive Summary

The investigation revealed a discrepancy in how real-time events are broadcasted and consumed between the Seller and Buyer sides of the application. While the seller's dashboard listens to store-specific channels, the buyer's order page listens to user-specific channels which are currently not receiving the 'order-updated' event.

Key Findings

  • Channel Misalignment: API endpoints trigger events to store-{storeId} but ignore user-{userId}.
  • Missing Hook Binding: The useRealtimeBuyerOrders hook only listens for rating-created events.
  • Polling Fallback: The buyer side relies on a 10-second polling mechanism which masks the real-time failure for most statuses but feels delayed for critical transitions like "booked".

Technical Analysis

1. Seller-Side Hook (Working)

The useRealtimeOrders.js hook correctly binds to the order-updated event on the store-{storeId} channel.

// hooks/useRealtimeOrders.js
channel.bind('order-updated', (data) => {
    console.log('⚡ Order Updated (Pusher):', data);
    mutate(); // Refresh SWR data
});

2. Buyer-Side Hook (Incomplete)

The useRealtimeBuyerOrders.js hook is missing the binding for order updates.

// hooks/useRealtimeBuyerOrders.js
// CURRENT STATE
channel.bind('rating-created', (data) => {
    console.log('⚡ [Buyer] Rating Created (Pusher):', data);
    mutate();
});
// MISSING: order-updated binding

3. API Trigger Logic (One-Sided)

The Lalamove sync endpoint only targets the store channel.

// app/api/orders/[orderId]/sync-lalamove/route.js
await triggerRealtimeEvent(`store-${order.storeId}`, 'order-updated', {
    orderId: order.id,
    status: newOrderStatus,
    shippingStatus: lalamoveStatus
});
// MISSING: triggerRealtimeEvent(`user-${order.userId}`, ...)

Conclusion

To fix the issue, we must ensure that every order status update triggers Pusher events for both the Store (Seller) and the User (Buyer), and that the buyer-side hook is updated to listen for these events.
