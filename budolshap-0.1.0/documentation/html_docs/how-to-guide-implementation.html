How to Guide Implementation Documentation

Date: December 2, 2025

Description: Implementation of a "How to Guide" feature (FAQ) accessible from the "Under Construction" page and the website Footer.

Summary of Changes

1. New Component: HowToGuide

Created a new reusable modal component that displays Frequently Asked Questions (FAQs) in an accordion format.

  • File: components/HowToGuide.jsx
  • Features:
    • Accordion-style Q&A for better UX.
    • Categorized sections: Getting Started, Store Management, Shopping, Orders & Payment, Others.
    • Custom entrance animations (fade-in, scale-in).
    • Responsive design with a clean UI.

2. Under Construction Page Update

Integrated the "How to Guide" into the main landing area of the Under Construction page.

  • File: app/under-construction/page.jsx
  • Changes:
    • Added a new card labeled "How to Guide" positioned beside "New Features".
    • Implemented state logic to open the HowToGuide modal upon clicking the card.
    • Fixed minor typos in the page text ("use's" -> "uses", "will slows" -> "will slow").

3. Global Styles Update

Added custom animation utilities to support the modal's entrance effects without external libraries.

  • File: app/globals.css
  • Additions:
    • .animate-fadeIn: Smooth opacity transition.
    • .animate-scaleIn: Zoom-in effect for modals.
    • .animate-blob: Background blob animation (restored/enhanced).

4. Footer Integration

Added a link to the "How to Guide" in the website footer for global accessibility.

  • File: components/Footer.jsx
  • Changes:
    • Converted the component to a Client Component ('use client') to support state.
    • Added "How to Guide" link under the "WEBSITE?" section, positioned below "Create Your Store".
    • Updated the link rendering logic to support button actions (opening the modal) alongside standard navigation links.
    • Integrated the HowToGuide modal.

Technical Implementation Details

Modal Logic

The modal uses a simple boolean state (isOpen) passed from the parent component (Page or Footer). It renders conditionally and uses a fixed-position overlay with a backdrop blur effect.

Footer Link Handling

// Logic to handle both Links and Actions in the Footer loop
{link.action ? (
    <button onClick={link.action} className="...">
        {link.text}
    </button>
