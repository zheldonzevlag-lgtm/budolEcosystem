Sync Order Status Verification

The changes to synchronize the Buyer's "Order Status" with the Seller's view have been applied. Follow these steps to verify the fix.

What Changed

  • API Caching Disabled: The /api/orders endpoint now forces dynamic rendering, ensuring the buyer always fetches the latest status from the database.
  • UI Synchronization: The Buyer's order list (Desktop and Mobile) now uses the exact same status text and color coding as the Seller's view.

Verification Steps

1. Setup

Open two separate browser windows or tabs. You may need to use Incognito/Private mode for one if you need to log in as two different users simultaneously.

2. Buyer View (Before Update)

Log in as a Buyer and go to "My Orders". Note the status of a specific order (e.g., it might say "Paid" in blue/green).

3. Seller Action

In the other window, log in as the Store Owner (Seller).

Go to "Store Orders" and find the same order.

Change the status (e.g., click "Book Lalamove" or use the dropdown to change status to "Shipped").

4. Buyer View (Verification)

Switch back to the Buyer window.

Refresh the page.

Success Criteria: The "Order Status" column should now match the Seller's view:

  • If Seller sees IN TRANSIT, Buyer should also see IN TRANSIT.
  • If Seller sees PROCESSING, Buyer should also see PROCESSING.
