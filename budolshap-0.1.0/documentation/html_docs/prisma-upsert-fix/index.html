Implementation Plan: Prisma Upsert Error Fix

This plan documents the resolution for the Invalid prisma.systemSettings.upsert() invocation error in the System Settings management.

Phase 1: Analysis & Diagnosis

Completed

Identified that the upsert operation in lib/settings.js was missing required fields in its create block, leading to Prisma validation failures when the record didn't exist.

Phase 2: Refactoring & Schema Sync

Completed

Consolidated default settings into a central constant and synced multiple Prisma schema files (MySQL and Local) to ensure consistency across environments.
