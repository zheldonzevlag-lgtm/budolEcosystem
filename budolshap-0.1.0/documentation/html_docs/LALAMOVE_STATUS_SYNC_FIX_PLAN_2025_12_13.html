Lalamove Order Status Synchronization Fix

Date: December 13, 2025

Status: Planned

Problem: Orders cancelled by Lalamove drivers appear as "Processing" or "Driver Assigned" in the Seller and Buyer dashboards, causing confusion.

1. Issue Analysis

Based on the investigation and user reports (Images 1, 2, 3), the following discrepancy exists:

Platform Current Status Display Expected Status Display
Lalamove Portal CANCELLED CANCELLED
Seller Dashboard PROCESSING CANCELLED / FAILED
Buyer Tracking Driver Assigned Delivery Cancelled

Root Cause

When Lalamove sends a CANCELLED or EXPIRED webhook, the backend (app/api/webhooks/lalamove/route.js) intentionally reverts the order status to PROCESSING. This is designed to allow the seller to "Rebook" the order. However, the UI does not distinguish between a "normal" processing order and a "failed/reverted" processing order, leading to the confusing "Driver Assigned" display.

2. Implementation Plan

We will fix this by updating the UI Logic to detect the failure flags set by the backend, without removing the ability to rebook.

Strategy: We will check for order.shipping.failureReason. If present, the UI will treat the order as "Cancelled/Failed" even if the database status is "PROCESSING".

A. Seller Dashboard Updates

File: app/store/orders/page.jsx

  • Update getDeliveryStatus(order) helper function.
  • Logic: If status === 'PROCESSING' AND shipping.failureReason exists → Return 'cancelled'.
  • This will move the order card to the Cancelled tab automatically.
  • The "Rebook Delivery" button will remain accessible (or be added to the cancelled view).

B. Buyer Progress Tracker Updates

File: components/OrderProgressTracker.jsx

  • Add check for order.shipping.failureReason.
  • If failed, override the active step label to "Delivery Cancelled" (Red).
  • Hide the "Driver Assigned" icon/text in this state.

C. Tracking Timeline Updates

File: components/TrackingTimeline.jsx

  • Add a new timeline event block.
  • If shipping.failureReason exists, append a "Delivery Failed/Cancelled" event to the history.
  • Display the reason (e.g., "Driver Cancelled", "Expired") if available.

3. Verification Steps

  1. Simulate Failure: Use existing test scripts to create an order and push a CANCELLED webhook.
  2. Seller Check: Verify the order moves to the "Cancelled" tab in the Seller Dashboard.
  3. Buyer Check: Verify the order tracking page shows "Delivery Cancelled" instead of "Driver Assigned".
  4. Rebook Check: Ensure the seller can still click "Rebook" to restart the process.
