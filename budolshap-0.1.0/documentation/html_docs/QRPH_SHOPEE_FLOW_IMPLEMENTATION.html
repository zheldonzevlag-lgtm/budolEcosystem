🛒 QRPh Payment Flow - Shopee-like Implementation

Date: December 18, 2024

Project: Budolshap v3

Author: Development Team

📋 Executive Summary

This document details the implementation of a Shopee-like QRPh payment flow in Budolshap, including the fixes applied to display QR codes immediately after order placement and align the user experience with industry-standard e-commerce platforms.

Key Achievements

  • ✅ QR code displays immediately after order placement
  • ✅ Redirect to "To Pay" tab on modal close (Shopee-like behavior)
  • ✅ Cart persistence until payment success
  • ✅ Screenshot upload instruction added
  • ✅ Simplified user flow with removed redundant modals

🔍 Problem Statement

Initial Issues

  1. QR Modal Not Displaying: After placing a QRPh order, the QR code modal failed to appear
  2. Loading Overlay Blocking: Fullscreen loading component covered the QR modal
  3. Premature Cart Clearing: Cart was cleared before QR modal could render, causing component unmounting
  4. Non-standard UX: Flow didn't match familiar Shopee experience

🛠️ Technical Implementation

1. QR Modal Display Fix

File: components/OrderSummary.jsx

Issue: Loading overlay was covering the QR modal

Solution: Added condition to prevent Loading component when QR modal is visible

{isPlacingOrder && !showQRModal && <Loading />}

2. Cart Clearing Timing

File: components/OrderSummary.jsx:260-279

Issue: Cart cleared immediately after order creation, unmounting OrderSummary before QR modal could render

Solution: Delayed cart clearing for QR-based payments until modal closes or payment succeeds

const isQRPayment = ['MAYA', 'GRAB_PAY', 'QRPH'].includes(paymentMethod); if (response.ok && !isQRPayment) { // Clear cart immediately for COD, BudolPay dispatch(clearCart()); // ... }

3. Shopee-like Redirect Flow

File: components/OrderSummary.jsx:675-684

Change: Redirect to orders page on QR modal close instead of showing order info modal

onClose={() => { console.log('📱 [OrderSummary] QR Modal closed by user - redirecting to orders'); setShowQRModal(false); router.push('/orders?tab=TO_PAY'); }}

4. Enhanced QR Instructions

File: components/payment/QRCodeModal.jsx:174

Addition: Screenshot upload instruction for same-device payments

<li>Or take a screenshot and upload it to your payment app</li>

📊 User Flow Comparison

Step Before After (Shopee-like)
1. Place Order ✅ QRPh selected ✅ QRPh selected
2. QR Display ❌ Modal didn't appear ✅ Modal appears immediately
3. Close Modal Cart cleared → Order Info modal → Stay on cart Redirect to /orders?tab=TO_PAY
4. Cart State Cleared on modal close Persists until payment success
5. Retry Payment Manual navigation needed Direct access from "To Pay" tab

🎯 Current vs Shopee Behavior

✅ Implemented (Matches Shopee)

1. QR code appears immediately after order placement
2. Order persists in database
3. "To Pay" tab shows unpaid orders
4. Redirect to "To Pay" on modal close
5. Cart persists until payment success
6. Repay functionality from orders page

⚠️ Not Yet Implemented (Shopee Features)

1. Change payment method for existing orders
2. Automatic order cancellation after 24 hours
3. QR code reuse (currently generates new QR on retry)
4. Payment deadline countdown on order details

📝 Implementation Recommendations

1. Database Schema Updates

// Add to Order model model Order { // ... existing fields paymentDeadline DateTime? paymentIntentId String? paymentIntentExpiry DateTime? allowPaymentChange Boolean @default(true) }

2. Cron Job for Order Cancellation

// vercel.json { "crons": [{ "path": "/api/cron/cancel-expired-orders", "schedule": "0 * * * *" // Every hour }] } // app/api/cron/cancel-expired-orders/route.js export async function GET(request) { const expiredOrders = await prisma.order.findMany({ where: { isPaid: false, paymentDeadline: { lt: new Date() }, status: 'ORDER_PLACED' } }); for (const order of expiredOrders) { await prisma.order.update({ where: { id: order.id }, data: { status: 'CANCELLED', cancelReason: 'Payment deadline exceeded' } }); } return NextResponse.json({ cancelled: expiredOrders.length }); }

3. Change Payment Method API

// app/api/orders/[orderId]/change-payment/route.js export async function POST(request, { params }) { const { newPaymentMethod } = await request.json(); const { orderId } = params; // Invalidate old payment intent if (order.paymentIntentId) { await invalidatePaymentIntent(order.paymentIntentId); } // Update order await prisma.order.update({ where: { id: orderId }, data: { paymentMethod: newPaymentMethod, paymentIntentId: null, paymentIntentExpiry: null } }); return NextResponse.json({ success: true }); }
