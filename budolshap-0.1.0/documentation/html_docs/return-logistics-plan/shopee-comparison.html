Shopee-Aligned Return System

Comparing BudolShap's implementation with Shopee's industry-standard workflow.

BudolShap's implementation is heavily inspired by Shopee's Return & Refund 2.0 flow, focusing on escrow protection and reason-based logistics.

🟠 Shopee Standard

  • Types: Refund Only vs Return & Refund.
  • Evidence: Mandatory photos/videos for disputes.
  • Partial Refund: Negotiation loop between buyer and seller.
  • Waybill: Auto-generated QR/Barcode for drop-off.
  • RSF: Paid by Shopee, Seller, or Buyer depending on fault.

🔵 BudolShap Implementation

  • Types: MATCHED Identical classification.
  • Evidence: MATCHED Up to 5 photos supported.
  • Partial Refund: MATCHED Supported in backend (Seller can propose amount).
  • Waybill: GAP Manual booking via Lalamove currently.
  • RSF: MATCHED Logic assigns payer based on reason.

Key Alignments

1. Escrow Lock: Just like Shopee Guarantee, funds are held in a lockedBalance state during the return process. They are only released once the buyer receives a refund or the return is rejected.

2. RSF Calculation: We use calculateRSF(reason) to automatically determine if the Seller should be debited for the return shipping (Seller Fault) or if it's deducted from the Buyer's refund (Buyer Fault).

Planned Enhancements (Shopee Parity)

  • Interactive Partial Refund: Allow sellers to offer a "Keep Item + Partial Refund" deal.
  • Admin Dispute Center: A dedicated UI for BudolShap admins to rule on DISPUTED returns.
  • Automatic Waybills: Integrating Lalamove/J&T APIs to generate return labels instantly.
