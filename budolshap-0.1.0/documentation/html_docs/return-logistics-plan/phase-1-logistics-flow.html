📦 Return Logistics & Booking Plan

Technical specification for handling the physical return of items in "Return & Refund" requests.

1. Concept: The "Return Leg" Booking

When a buyer requests a Return & Refund, a physical movement of goods is required from the Buyer back to the Seller. This is the "Return Leg" of the transaction.

Question: "Should the return & refund be book again?"
Answer: Yes. A new courier booking must be created because the original booking only covered the Seller → Buyer path. The Return Leg requires a new pickup from the Buyer's location.

2. "Return & Refund" vs "Refund Only"

Feature Refund Only Return & Refund
Item Status Buyer keeps item (or item never arrived) Buyer must ship item back
New Booking? No Yes (Return Leg)
Refund Timing Immediately upon Seller Approval After Seller receives & inspects item
Use Case Missing items, "Item Not Received", Low-value damage Wrong item, High-value damage, Change of mind

3. Implementation Steps

3.1 Database Updates

Add returnShipping to the Return model to track the return parcel separately from the original order shipping.

model Return {
  // ... existing fields
  returnShipping Json?  // Stores { bookingId, provider, trackingUrl, cost, etc }
}

3.2 Buyer: Arrange Shipment Action

Once the seller clicks "Request Return" (or approves the return request), the buyer is notified to arrange shipment.

  • Endpoint: /api/shipping/lalamove/book-return
  • Logic: Automatically swaps pickup (Buyer) and delivery (Seller) locations.
  • Tracking: Generates a new Lalamove orderRef stored in the Return record.

3.3 Seller: Receive and Inspect

The seller tracks the return parcel. Once delivered, they must click "Confirm Receipt & Refund".

  • Status: RETURN_APPROVED → SHIPPED (Return) → RECEIVED → REFUNDED.

4. Success Criteria

  • Buyer can successfully book a Lalamove rider for the return leg.
  • Shipping fee (RSF) is correctly handled (deducted from refund if buyer fault, or charged to seller if seller fault).
  • Seller can see the live tracking of the returning parcel.

5. Phase 3: Buyer & Seller UI Actions

Integrated UI components for managing the return process with clearer terminology.

Implemented
  • Buyer Dashboard: "Return / Refund" button opens a simplified selection modal.
  • Clearer Terminology:
    • "Refund Only" (No return needed) - For missing items.
    • "Return Item" (Ship back to seller) - For damaged/wrong items.
  • Booking: "Book Return Courier" button appears for approved returns.
  • Negotiation: Accept/Reject buttons for partial refund offers.
