Automated Order Lifecycle

Implementation Process

To prevent stock hoarding and manual order cleanup, we implemented a CRON-driven automation engine. This allows the system to autonomously cancel unpaid orders outside the user-defined payment threshold.

Code Touch: Created
app/api/cron/cancel-unpaid-orders/route.js

Workflow:

  • System checks SystemSettings for orderCancellationHours.
  • Query finds orders with status ORDER_PLACED created before the threshold.
  • Batch updates status to CANCELLED.
  • Logs actions to system audit trails.

Identified Issue

Early implementation used a static 48-hour window, which didn't allow flexibility for seasonal sales or high-demand periods.

Technical Fix

Decoupled logic from hardcoded values. Moved settings to Prisma DB, enabling on-the-fly updates via Admin UI.

Shopee-Aligned Return & Refund

Developed a comprehensive escrow-locked return system. This phase introduced the ability for buyers to dispute deliveries while ensuring seller wallets reflect refund deductions accurately.

B

Buyer-Side Logic

  • Shopee Guarantee: A persistent banner ensures buyers that funds are held in escrow.
  • Conditional Controls: "Order Received" vs "Return/Refund" logic prevents finalized orders from being disputed after the protection window.
  • Request Modal: ReturnRequestModal.jsx collects Reason, Refund Amount, and Evidence Images.

Seller-Side Management

  • Returns Dashboard: Created app/store/returns/page.jsx with deep tab filtering (Pending, Approved, Refunded).
  • Escrow Refund Engine: Integrated returnsService.js which automatically debits the seller wallet and creates a DEBIT transaction record.
  • Real-time Sync: Integrated realtime.js to notify sellers instantly when a return is requested.
