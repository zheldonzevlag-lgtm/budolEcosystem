Phase 3: Admin Mediation Hub

Estimated Timeframe: 2-3 Days
Primary Goal: Build a dedicated arbitration environment for BudolShap Admins to resolve RETURN_DISPUTED cases.

1. Technical Specifications

  • Evidence Hub UI: A side-by-side comparison interface for Buyer Evidence (photos/vids) and Seller Packing Proof.
  • Admin Override API: A protected endpoint /api/admin/returns/[id]/mediate that can force a REFUNDED or REJECTED status regardless of party consent.
  • Audit Logging: Every admin verdict must be logged in the AuditLog model with the admin's ID and reasoning.

2. Backup & Safety Protocol

Before proceeding, the following files must be backed up to /backups/phase3/:

  • lib/services/adminService.js
  • app/admin/orders/page.jsx
  • lib/adminAuth.js

3. Implementation Steps

  • Task 1 Dispute Queue Creation

    Build the Admin UI page /admin/returns that lists all orders with RETURN_DISPUTED status.

  • Task 2 Evidence Viewer

    Implement a modal that pulls images from both the Return record and the original order's paymentProof/shipping labels.

  • Task 3 Arbitration Engine

    Develop the backend mediation function that handles complex wallet re-balancing if an admin overturns a seller's rejection.

4. Testing Protocol

Test Case 1: Force Refund
As Admin, overturn a "Rejected" return. Verify that the buyer receives the refund and the seller's wallet is debited correctly.

Test Case 2: Evidence Integrity
Verify that an Admin cannot delete or edit the evidence uploaded by either party, maintaining the audit trail.
