Phase 1: Foundation & Core Lifecycle

Estimated Timeframe: 1-2 Days
Primary Goal: Establish the database schema and backend logic for Shopee-style escrow and performance tracking.

1. Technical Specifications

  • Database Update: Add buyerReturnRate, codUnpaidCount to User model. Add nonFulfilmentRate, penaltyPoints to Store model.
  • Escrow Logic: Refactor returnsService.js to ensure funds are locked in PENDING status during the 72-hour seller response window.
  • Performance Calculation: Implement utility functions to recalculate BRR/NFR after every completed return.

2. Backup & Safety Protocol

3. Implementation Steps

  • Task 1 Schema Enhancement

    Update prisma/schema.prisma and run npx prisma generate.

  • Task 2 Refine returnsService.js

    Implement the 72-hour response deadline and escrow locking logic in createReturnRequest.

    Add logic to receiveReturn to increment NFR for sellers and BRR for buyers upon resolution.

Test Case 2: Deadline Enforcement
Simulate a 72-hour lapse. Verify that the system auto-approves the refund and debits the seller wallet correctly.
