Phase 2: Logistics & Exception Handling

Estimated Timeframe: 1-2 Days
Primary Goal: Handle complex return scenarios including COD failures, return shipping costs, and Shopee Guarantee extensions.

1. Technical Specifications

  • COD Unpaid Logic: Track failed COD deliveries and automatically increment the user's codUnpaidCount.
  • RSF (Return Shipping Fee): Implement a rule engine to decide if the Buyer or Seller pays for the return shipping based on the return reason.
  • Guarantee Extension: Add an autoCompleteAt field update logic when a buyer requests an extension.

2. Backup & Safety Protocol

Before proceeding, the following files must be backed up to /backups/phase2/:

  • lib/services/shippingService.js
  • lib/services/ordersService.js
  • app/api/orders/[id]/extend-guarantee/route.js (to be created/edited)

3. Implementation Steps

  • Task 1 COD Failure Tracking

    Modify ordersService.js to capture COD delivery failures and update buyer performance.

  • Task 2 Return Shipping Calculation

    Implement calculateRSF(reason) in returnsService.js to handle pro-rata shipping deductions.

  • Task 3 Extension UI & Backend

    Add the "Extend Shopee Guarantee" button to the Order Detail page and implement the 3-day extension logic.

4. Testing Protocol

Test Case 1: RSF Deduction
Process a return for "Change of Mind". Verify that the return shipping cost is deducted from the buyer's refund amount.

Test Case 2: COD Restriction
Trigger 2 COD failures for a test user. Verify that the COD payment option is hidden in the checkout UI for their next order.
