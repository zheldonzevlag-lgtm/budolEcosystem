Phase 4: Risk & Performance Analytics

Estimated Timeframe: 1-2 Days
Primary Goal: Implement automated performance scoring and risk-based restrictions for both Buyers and Sellers.
Safety Protocol: Backup Phase 4

Before implementing automated restrictions, backup the restriction logic and user models:

  • prisma/schema.prisma (Latest version from Phase 1)
  • lib/services/userService.js
  • lib/services/storeService.js

Technical Tasks

  • Task 1 Performance Aggregator Service

    Create lib/services/analyticsService.js to calculate BRR and NFR asynchronously using cron jobs or on-demand triggers.

  • Task 2 Automated Sanction Engine

    Implement logic to restrict payment methods (e.g., disable COD for high-risk buyers) and apply penalty points to sellers.

  • Task 3 Admin Analytics Dashboard

    Develop the /admin/analytics view to visualize platform-wide return trends and flag high-risk accounts.

Testing & Validation Gates

Gate 4: Sanction Accuracy & UI Fidelity

Must pass all criteria before project sign-off:

  • Scenario A: Verify a buyer with >20% BRR has the COD option disabled at checkout.
  • Scenario B: Verify a seller with high NFR receives a "Penalty Warning" in their store dashboard.
  • Scenario C: Verify admin analytics correctly aggregate data from AuditLog and ReturnRequest tables.

Future Roadmap Suggestions

  • AI Fraud Detection: Integrate machine learning to predict fraudulent return requests based on image metadata.
  • Loyalty Integration: Reward buyers with low BRR with "Trusted Buyer" badges and faster refund processing.
