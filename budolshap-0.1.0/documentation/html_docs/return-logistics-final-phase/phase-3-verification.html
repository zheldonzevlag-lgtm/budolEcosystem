Phase 3: Shipping Verification & Waybills

Enhancing the "Integrated Shipping" feel by providing formal documentation and automated status updates.

In Progress

Technical Specification

This phase focuses on the post-booking experience, ensuring that both parties can track the physical movement of the return item.

Goals

  • Waybill Generation: Generate a PDF waybill for the buyer to print/attach to the return parcel.
  • Webhook Integration: Handle Lalamove order status updates (PICKED_UP, COMPLETED) to automate the escrow release.
  • Dispute Trigger: If the return is not booked within 3-5 days, the request should auto-expire/cancel.

Planned Tasks

  • PDF Waybill Generation: Implement a route to generate a standard shipping label with the swapped address details.
  • Lalamove Webhook Handler: Extend /api/webhooks/shipping to support return order references.
  • Auto-Expiry Logic: Background job to cancel return requests that remain in APPROVED but not SHIPPED.
