Phase 1: Core Logistics Architecture

Establish the technical foundation for address swapping and return shipping bookings.

Completed

Technical Specification

The core of return logistics is the Address Swap. Unlike a standard delivery (Store → Buyer), a return requires a (Buyer → Store) flow.

Key Components

  • Service Layer: shippingService.js handles the translation between Order addresses and Shipping Provider (Lalamove) stops.
  • API Endpoint: /api/shipping/lalamove/book-return triggers the booking logic.
  • State Management: Order status moves to RETURN_APPROVED then IN_TRANSIT upon booking.
