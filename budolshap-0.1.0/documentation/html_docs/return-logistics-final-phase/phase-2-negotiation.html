Phase 2: Negotiation Loop (Partial Refunds)

Implementation of the Shopee-style negotiation flow where sellers can offer partial refunds to avoid physical returns.

Completed

Technical Specification

The negotiation loop involves a state machine in returnsService.js that manages OFFER_PARTIAL actions.

Workflows

  • Seller Side: Seller uses OFFER_PARTIAL with a partialAmount.
  • Buyer Side: Buyer can ACCEPT or REJECT the offer.
  • Finalization: Accepting triggers an immediate escrow release of the partial amount to the buyer and the remainder to the seller.

Implementation Details

// returnsService.js - respondToReturn
if (action === 'OFFER_PARTIAL') {
    finalRefundAmount = partialAmount;
    newStatus = 'PENDING'; // Buyer must now respond
    sellerAction = 'OFFER_PARTIAL';
}
