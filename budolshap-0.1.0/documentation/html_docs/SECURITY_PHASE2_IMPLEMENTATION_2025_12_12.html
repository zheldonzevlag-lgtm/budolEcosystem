COMPLETED

Focus: Automatic Idle Logout for Admins & Sellers

Implementation Summary

We have successfully implemented an automatic session timeout mechanism to protect sensitive Admin and Seller accounts. If a user leaves their dashboard unattended for 15 minutes, the system will automatically log them out.

Key Components:

  • useIdleTimeout Hook: A custom React hook that tracks user activity (mouse, keyboard, etc.) and manages inactivity timers.
  • SessionTimeoutModal: A warning dialog that appears 1 minute before the actual logout to give the user a chance to stay connected.
  • Layout Integration: Applied the protection logic to both `AdminLayout` and `StoreLayout`.

How to Verify (Manual Test)

Since this involves waiting for 15 minutes, you can verify it faster by temporarily modifying the timeout values in the code:

  1. Open components/admin/AdminLayout.jsx.
  2. Change timeoutMs to 10000 (10 seconds) and warningMs to 5000 (5 seconds).
  3. Log in as an Admin.
  4. Do not move your mouse for 5 seconds.
  5. Expected Result: The "Session Expiring" modal should appear.
  6. Wait another 5 seconds.
  7. Expected Result: You should be automatically logged out and redirected to the login page.

Note: Revert changes after testing!
