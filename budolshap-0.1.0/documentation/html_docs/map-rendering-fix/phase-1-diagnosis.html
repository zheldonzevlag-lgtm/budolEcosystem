Phase 1: Diagnosis & Root Cause Analysis

The Issue

Symptom: The interactive map in the Address Form was appearing as a white/empty box with only the border visible. No map tiles or center pins were rendered, although the "Coordinates Verified" overlay was visible.

Root Cause Analysis

After examining the codebase and the rendered UI, the following causes were identified:

  • Height Collapse: The MapPicker component was receiving a height="100%" prop. In CSS, percentage heights only work if the immediate parent has an explicit height. The wrapper div in MapPicker.jsx didn't have an explicit height, causing the map container to collapse to 0 pixels.
  • CSS Specificity: The Tailwind classes were not sufficient to force the Leaflet container to expand when nested inside several relative/absolute containers without explicit dimensions.
  • Theme Inconsistency: The map markers and UI elements were still using the old orange branding (#ea580c), which conflicted with the newly reverted green theme (#10b981).

Technical Specifications

  • Component: MapPicker.jsx
  • Parent Component: AddressFormManager.jsx
  • Styling Framework: Tailwind CSS + Inline Styles for dynamic height
  • Map Library: Leaflet.js / react-leaflet
