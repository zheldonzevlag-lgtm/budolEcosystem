Phase 1: Order Progress UI Refinement

Overview

This phase focused on improving the Order Progress tracker's visual consistency, responsiveness, and usability. The goal was to align it with the "Shopee-style" design language while optimizing for mobile views.

Key Implementation Details

1. Collapsibility

Logic Added state management to allow users to toggle the visibility of the order progress section.

const [isCollapsed, setIsCollapsed] = useState(true);

The section is now closed by default to save vertical space on initial load.

2. Size Optimization

UI Reduced the dimensions of status circles and icons to create a more compact and professional look.

  • Desktop Circles: Reduced from 40px (w-10 h-10) to 32px (w-8 h-8).
  • Mobile Circles: Reduced from 32px (w-8 h-8) to 24px (w-6 h-6).
  • Icons: Scaled down proportionally using text-base for desktop and text-sm for mobile.
  • Lines: Progress lines thinned to 0.5px (h-0.5) and realigned to match smaller circles.

3. Styling Consistency

UI Standardized the card appearance to match other sections in the order details modal.

  • Shadow: Switched to shadow-sm for a subtle depth effect.
  • Border: Updated to border-slate-200 to match standard card borders.
  • Padding: Normalized padding to p-4.
  • Title: Matched "Order Progress" font size and weight with "Customer Details" (font-semibold text-slate-800).

Validation Steps

  1. Verify "Order Progress" starts collapsed in the Order Details modal.
  2. Toggle the section and check for smooth expansion.
  3. Inspect mobile view (375px width) to ensure circles and labels don't overflow.
  4. Compare card width and shadow with the "Customer Details" section below it.

✓ All criteria met and verified in OrderProgressTracker.jsx and app/store/orders/page.jsx.
