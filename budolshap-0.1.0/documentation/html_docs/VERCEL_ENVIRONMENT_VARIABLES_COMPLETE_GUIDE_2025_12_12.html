🔴 Critical Variables (Application Won't Work Without These)

JWT_SECRET CRITICAL
📝 Purpose:
Secret key used to sign and verify JWT (JSON Web Tokens) for user authentication. This is the backbone of your authentication system.
⚠️ Impact if Missing:
SEVERE: Users cannot stay logged in. After successful login, they will be immediately redirected back to the login page because the server cannot verify their authentication token.
❌ Impact if Wrong Value:
All existing user sessions become invalid. Users will be logged out and need to login again. Tokens signed with the old secret cannot be verified with the new secret.
✅ How to Set:
# Generate a secure random value (32+ characters): node -e "console.log(require('crypto').randomBytes(32).toString('hex'))" # Example output: 0f8ea2f62aec76393597dc8acad695e74273adb405ae9a64122cf6d75d625af # Add to Vercel: Name: JWT_SECRET Value: [paste the generated value] Environments: Production, Preview, Development
🔄 Migration to New Account:
Option 1 (Recommended): Generate a NEW secret for the new deployment. All users will need to login again.
Option 2: Copy the exact same value from the old account if you want to preserve existing sessions (not recommended for security).
DATABASE_URL AUTO-SET
📝 Purpose:
Connection string to your PostgreSQL database. This is where all your data (users, products, orders) is stored.
⚠️ Impact if Missing:
FATAL: Application crashes on startup. Cannot read or write any data. All pages will show errors.
❌ Impact if Wrong Value:
Database connection fails. Application shows "Cannot connect to database" errors. No data can be accessed.
✅ How to Set:
Automatically set by Vercel when you:
  1. Go to Storage tab in Vercel project
  2. Create → Postgres Database
  3. Connect to your project
Vercel automatically adds: DATABASE_URL, POSTGRES_URL, POSTGRES_PRISMA_URL, etc.
🔄 Migration to New Account:
You MUST create a new Postgres database in the new Vercel account. Then either:
  • Option 1: Start fresh (empty database, run migrations)
  • Option 2: Migrate data from old database using pg_dump and pg_restore
NEXT_PUBLIC_APP_URL CRITICAL
📝 Purpose:
The public URL of your deployed application. Used for generating absolute URLs in emails, webhooks, OAuth callbacks, and social sharing.
⚠️ Impact if Missing:
Email verification links break (point to localhost). Webhook callbacks fail. Social sharing shows incorrect URLs. OAuth redirects fail.
❌ Impact if Wrong Value:
Links in emails redirect to wrong domain. Webhooks sent to wrong URL. Users cannot verify email or reset password.
✅ How to Set:
# For your current deployment: Name: NEXT_PUBLIC_APP_URL Value: https://budolshap-v3-j81ea1fen-jon-galvezs-projects-f5372ed2.vercel.app Environments: Production # For custom domain (if you have one): Value: https://budolshap.com
🔄 Migration to New Account:
MUST CHANGE: Update to the new Vercel deployment URL or your custom domain.
# New account will have different URL: https://budolshap-v3-[new-random-id]-[new-username].vercel.app

🟡 Important Variables (Required for Key Features)

LALAMOVE_CLIENT_ID IMPORTANT
📝 Purpose:
API client ID for Lalamove delivery service integration. Required for booking deliveries and getting shipping quotes.
⚠️ Impact if Missing:
Delivery booking fails. "Book Lalamove Delivery" button shows errors. Sellers cannot arrange shipping for orders.
❌ Impact if Wrong Value:
API authentication fails. Error: "Invalid API credentials". All Lalamove features non-functional.
✅ How to Get:
  1. Register at Lalamove Business
  2. Login to Lalamove Dashboard
  3. Go to Settings → API Integration
  4. Copy your Client ID
Name: LALAMOVE_CLIENT_ID Value: [your-client-id-from-lalamove] Environments: Production, Preview, Development
🔄 Migration to New Account:
SAME VALUE: Use the same Lalamove credentials. The API keys are tied to your Lalamove business account, not Vercel.
LALAMOVE_CLIENT_SECRET IMPORTANT
📝 Purpose:
API secret key for Lalamove. Used together with CLIENT_ID to authenticate API requests.
⚠️ Impact if Missing:
Same as LALAMOVE_CLIENT_ID - all delivery features fail.
❌ Impact if Wrong Value:
HMAC signature verification fails. API returns 401 Unauthorized. Cannot book deliveries.
✅ How to Get:
From the same location as CLIENT_ID in Lalamove Dashboard → Settings → API Integration
🔄 Migration to New Account:
SAME VALUE: Copy from old deployment or get from Lalamove Dashboard.
LALAMOVE_ENV IMPORTANT
📝 Purpose:
Specifies which Lalamove environment to use: sandbox (testing) or production (real deliveries).
⚠️ Impact if Missing:
Defaults to sandbox. Not critical but you might be testing when you think you're in production.
❌ Impact if Wrong Value:
DANGEROUS: If set to production in testing, you'll create REAL delivery orders and be charged! If set to sandbox in production, deliveries won't actually happen.
✅ How to Set:
# For testing/development: Name: LALAMOVE_ENV Value: sandbox # For production (real deliveries): Value: production
🔄 Migration to New Account:
DECIDE: Choose based on whether new deployment is for testing or production use.
PAYMONGO_SECRET_KEY IMPORTANT
📝 Purpose:
Secret API key for PayMongo payment processing. Required for accepting GCash and card payments.
⚠️ Impact if Missing:
Payment processing fails. Users cannot pay for orders. "Pay with GCash" button shows errors. Revenue stream blocked.
❌ Impact if Wrong Value:
API authentication fails. Payments rejected. Error: "Invalid API key". No transactions can be processed.
✅ How to Get:
  1. Register at PayMongo Dashboard
  2. Complete KYC verification
  3. Go to Developers → API Keys
  4. Copy Secret Key (starts with sk_test_ or sk_live_)
Name: PAYMONGO_SECRET_KEY Value: sk_test_... (for testing) or sk_live_... (for production)
🔄 Migration to New Account:
SAME VALUE: PayMongo keys are tied to your business account, not Vercel. Use same credentials.
CLOUDINARY_API_KEY & CLOUDINARY_API_SECRET IMPORTANT
📝 Purpose:
Credentials for Cloudinary image hosting service. Used for uploading and storing product images, user avatars, and store logos.
⚠️ Impact if Missing:
Image uploads fail. Sellers cannot add product photos. Store logo upload broken. Users see default placeholder images.
❌ Impact if Wrong Value:
Upload API fails with 401 Unauthorized. Images not saved. Product creation fails if images required.
✅ How to Get:
  1. Register at Cloudinary
  2. Go to Dashboard
  3. Copy: Cloud Name, API Key, API Secret
Name: NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME Value: [your-cloud-name] Name: CLOUDINARY_API_KEY Value: [your-api-key] Name: CLOUDINARY_API_SECRET Value: [your-api-secret]
🔄 Migration to New Account:
SAME VALUE: Use same Cloudinary account. OR create new Cloudinary account if you want separate image storage.

🔵 Optional Variables (Feature Enhancements)

SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS OPTIONAL
📝 Purpose:
Email server configuration for sending transactional emails (order confirmations, password resets, delivery notifications - Phase 2 feature).
⚠️ Impact if Missing:
GRACEFUL DEGRADATION: Emails are logged to console instead of being sent. Users won't receive email notifications but app still works. Phase 2 delivery failure notifications won't reach users.
❌ Impact if Wrong Value:
Email sending fails silently. Users don't receive verification emails, password resets, or order notifications. Check server logs for SMTP errors.
✅ How to Set (Gmail Example):
# Using Gmail: 1. Enable 2-Step Verification in Google Account 2. Generate App Password: Google Account → Security → App Passwords 3. Add to Vercel: SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=your-email@gmail.com SMTP_PASS=[16-character app password] SMTP_FROM=noreply@budolshap.com
Alternatives: SendGrid, AWS SES, Mailgun, Postmark
🔄 Migration to New Account:
SAME VALUE: Email credentials are independent of Vercel. Copy same values or use different email service.
LALAMOVE_WEBHOOK_SECRET OPTIONAL
📝 Purpose:
Secret for verifying Lalamove webhook signatures. Ensures webhook requests actually come from Lalamove and haven't been tampered with.
⚠️ Impact if Missing:
SECURITY RISK: Webhook signature verification is currently disabled in code (marked "TEMPORARILY DISABLED"). App works but vulnerable to fake webhook attacks. Real-time order updates still work.
❌ Impact if Wrong Value:
If verification is enabled, all webhooks will be rejected. Order status won't update automatically. Deliveries appear stuck.
✅ How to Get:
From Lalamove Dashboard → Webhooks → Webhook Secret
🔄 Migration to New Account:
SAME VALUE: Use same Lalamove webhook secret.
CRON_SECRET OPTIONAL
📝 Purpose:
Secret for authenticating cron job requests. Used by Phase 2 auto-polling feature (/api/cron/sync-orders) to prevent unauthorized access.
⚠️ Impact if Missing:
Cron endpoint is publicly accessible (security risk). Auto-polling still works but anyone can trigger it.
❌ Impact if Wrong Value:
Vercel Cron Jobs will fail authentication. Auto-sync won't run. Manual sync button still works.
✅ How to Set:
# Generate random secret: node -e "console.log(require('crypto').randomBytes(16).toString('hex'))" Name: CRON_SECRET Value: [generated value]
🔄 Migration to New Account:
GENERATE NEW: Create a new random value for security.

🔄 Migration Guide: Deploying to New Vercel Account

Scenario: Moving BudolShap to a Different Vercel Account

Step 1: Prepare New Vercel Account

  • Create/login to new Vercel account
  • Install Vercel CLI: npm i -g vercel
  • Login: vercel login

Step 2: Create New Postgres Database

  • In Vercel dashboard → Storage → Create Database → Postgres
  • Name it (e.g., "budolshap-db")
  • Select region (choose closest to your users)
  • This auto-creates: DATABASE_URL, POSTGRES_URL, etc.
⚠️ Data Migration Required: New database is empty. You must either:
  • Option A: Start fresh (run migrations, seed data)
  • Option B: Export from old DB and import to new DB

Step 3: Set Environment Variables

Variables That MUST Change:
Variable Action How to Get New Value
JWT_SECRET Generate NEW node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
NEXT_PUBLIC_APP_URL Update to new URL Will be https://budolshap-v3-[random]-[new-username].vercel.app
DATABASE_URL Auto-set Automatically set when you create Postgres database
Variables That Can Stay Same:
Variable Action Notes
LALAMOVE_CLIENT_ID Copy same value Tied to your Lalamove business account
LALAMOVE_CLIENT_SECRET Copy same value Tied to your Lalamove business account
LALAMOVE_ENV Copy or decide sandbox for testing, production for live
PAYMONGO_SECRET_KEY Copy same value Tied to your PayMongo account
CLOUDINARY_* Copy same value Tied to your Cloudinary account
SMTP_* Copy same value Tied to your email provider

Step 4: Deploy Application

# From project directory: vercel # Follow prompts: # - Link to existing project? No # - Project name: budolshap-v3 # - Directory: ./ # - Override settings? No # Deploy to production: vercel --prod

Step 5: Run Database Migrations

# Connect to new database and run migrations: npx prisma migrate deploy # Or reset and seed: npx prisma migrate reset --force node scripts/seed-users-prod.js

Step 6: Update Webhook URLs

  • Login to Lalamove Dashboard
  • Update webhook URL to: https://[new-url]/api/webhooks/lalamove
  • Login to PayMongo Dashboard
  • Update webhook URL to: https://[new-url]/api/webhooks/paymongo

Step 7: Test Everything

  • Test user registration and login
  • Test product creation with image upload
  • Test order placement
  • Test payment (use test mode first)
  • Test Lalamove delivery booking
  • Test email sending (if configured)

🔧 Troubleshooting Common Issues

Issue: "Redirected to login after successful login"

Cause: JWT_SECRET is missing or incorrect
Solution: Add/verify JWT_SECRET in Vercel environment variables, then redeploy

Issue: "Cannot connect to database"

Cause: DATABASE_URL missing or database not created
Solution: Create Postgres database in Vercel Storage tab and connect to project

Issue: "Lalamove booking fails with 401 Unauthorized"

Cause: Wrong LALAMOVE_CLIENT_ID or LALAMOVE_CLIENT_SECRET
Solution: Verify credentials in Lalamove Dashboard → API Integration

Issue: "Email verification links point to localhost"

Cause: NEXT_PUBLIC_APP_URL not set or set to wrong value
Solution: Set to your actual Vercel deployment URL or custom domain

Issue: "Image upload fails"

Cause: Cloudinary credentials missing or incorrect
Solution: Verify CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET, and NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME

Issue: "Emails not being sent"

Cause: SMTP not configured or credentials wrong
Solution: Check server logs. Emails are logged to console if SMTP not configured. Verify SMTP credentials if configured.

📋 Quick Reference Checklist

Minimum Required for Basic Functionality:

  • JWT_SECRET - Authentication
  • DATABASE_URL - Data storage (auto-set)
  • NEXT_PUBLIC_APP_URL - App URL

Add These for Full E-commerce Features:

  • LALAMOVE_CLIENT_ID - Delivery booking
  • LALAMOVE_CLIENT_SECRET - Delivery booking
  • LALAMOVE_ENV - Sandbox or production
  • PAYMONGO_SECRET_KEY - Payment processing
  • CLOUDINARY_API_KEY - Image uploads
  • CLOUDINARY_API_SECRET - Image uploads
  • NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME - Image uploads

Optional but Recommended:

  • SMTP_* - Email notifications
  • LALAMOVE_WEBHOOK_SECRET - Webhook security
  • CRON_SECRET - Cron job security

✅ Final Checklist Before Going Live:

  1. All critical variables set (JWT_SECRET, DATABASE_URL, NEXT_PUBLIC_APP_URL)
  2. All API credentials verified (Lalamove, PayMongo, Cloudinary)
  3. LALAMOVE_ENV set to production (not sandbox)
  4. PAYMONGO_SECRET_KEY using sk_live_ (not sk_test_)
  5. Database migrated and seeded
  6. Webhook URLs updated in external services
  7. Test all critical flows (login, order, payment, delivery)
  8. SMTP configured for production emails
