Phase 2: Implementation

1. Z-Index and Scrolling Behavior

Modified AddressFormManager.jsx to change the sticky footer to a standard scrolling footer and resolve z-index issues.

// Before: Sticky and low z-index <div className="sticky bottom-0 -mx-4 md:-mx-6 -mb-4 md:-mb-6 p-4 md:p-6 bg-white/90 backdrop-blur-md border-t border-slate-100 flex gap-3 lg:col-span-2 z-10"> // After: Scrollable with form <div className="flex gap-3 lg:col-span-2 mt-4 pb-2">

Rationale: Removing sticky allows the buttons to follow the form flow. Removing z-10 and negative margins cleans up the layout for better responsiveness.

2. Map Mouse Wheel Zoom

Enabled interactive zooming in MapPicker.jsx.

<MapContainer center={center} zoom={15} style={{ height: '100%', width: '100%' }} zoomControl={false} scrollWheelZoom={true} // Enabled here ref={setMap} >

Rationale: Enhances user experience by allowing intuitive map navigation.

3. Autocomplete Dropdown Visibility

Ensured the address suggestions are always visible by increasing their z-index in AddressAutocomplete.jsx.

<div className="absolute z-[4000] w-full mt-2 bg-white rounded-xl shadow-xl ...">

Rationale: z-[4000] ensures the dropdown stays above all map elements (max ~2000) and other UI components.

4. Balanced Button Layout

Updated the action buttons in AddressFormManager.jsx to have equal width for a more balanced UI.

// Both buttons now use flex-1 <button className="flex-1 ...">Cancel</button> <button className="flex-1 ...">Save Address</button>

Rationale: Provides a more consistent and professional appearance in the form footer.

5. Form Validation & Button State

Implemented real-time validation to disable the "Save Address" button until all required fields are correctly filled.

// Real-time validation logic const isFormValid = () => { return ( formData.name?.trim() !== '' && numberPart.length === 10 && formData.city?.trim() !== '' && formData.barangay?.trim() !== '' && formData.detailedAddress?.trim() !== '' ); }; // Button disabled state <button disabled={isLoading || !isFormValid()} className="... disabled:bg-green-300 disabled:shadow-none ..." >

Rationale: Prevents accidental submissions of incomplete data and improves user guidance by providing visual feedback (green 300 background) when the form is invalid.

6. Real-time Field Validation

Added useEffect hooks to monitor form state changes and provide immediate visual feedback (error messages and red borders) as the user types.

// Real-time validation effect useEffect(() => { const newErrors = { ...errors }; // Example: Phone validation const code = COUNTRY_CODES.find(c => formData.phone.startsWith(c.code))?.code || '+63'; const numberPart = formData.phone.slice(code.length).replace(/\D/g, ''); if (numberPart.length > 0) { if (numberPart.length !== 10) { newErrors.phone = "Phone number must be exactly 10 digits"; } else { delete newErrors.phone; } } setErrors(newErrors); }, [formData]);

Rationale: Improves UX by allowing users to correct mistakes immediately rather than waiting for form submission.

7. Mobile-First Collapsible Sections

Optimized the mobile user experience by ensuring all form sections are collapsed by default on smaller screens (below 1024px).

// Handle mobile default state for sections useEffect(() => { const checkMobile = () => { if (window.innerWidth < 1024) { setOpenSections({ contact: false, map: false, address: false, settings: false }); } }; checkMobile(); }, []);

Rationale: Reduces vertical scrolling on mobile devices and allows users to focus on specific sections of the form one at a time.
