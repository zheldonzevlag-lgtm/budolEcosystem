Phase 1: Analysis & Diagnosis

This phase focuses on identifying and analyzing the UX issues reported in the Address Modal.

Issue 1

Map Overlapping Action Buttons

Observation: The Leaflet map container and its controls (zoom, navigation) had higher z-indices than the sticky footer containing the "Save Address" and "Cancel" buttons.

Impact: Users were unable to click the save button when it was covered by map tiles or navigation buttons.

Root Cause: Leaflet default z-indices range from 1000 to 2000, while the action buttons were set to z-10.

Issue 2

Sticky vs. Scrollable Footer

Observation: The action buttons were fixed to the bottom of the modal (sticky), which sometimes obscured content or felt disconnected from the form flow on smaller screens.

Requirement: The user requested that buttons should scroll naturally with the form content.

Issue 3

Map Interaction Restrictions

Observation: The map was configured with scrollWheelZoom={false} by default, requiring users to use the plus/minus buttons to zoom.

Requirement: Users expected intuitive mouse wheel/trackpad zoom interaction.

Technical Scope

  • Component: AddressFormManager.jsx
  • Component: MapPicker.jsx
  • Component: AddressAutocomplete.jsx
