Phase 2: Logistics & Exception Handling

Status: Completed

Executive Summary

Phase 2 addressed the operational complexities of returns, specifically focusing on Return Shipping Fees (RSF), COD failure tracking for high-risk profiles, and the "Shopee Guarantee" extension mechanism.

Key Implementation Details

1. Return Shipping Fee (RSF) Logic

Implemented a dynamic RSF calculator in lib/services/returnsService.js. The logic determines the payer (Buyer or Seller) based on the return reason:

  • Seller Fault: (e.g., Wrong Item, Damaged) → Seller pays RSF.
  • Buyer Fault/Change of Mind: → RSF is deducted from the buyer's refund amount.

2. COD Failure Tracking

Modified the Lalamove webhook handler (app/api/webhooks/lalamove/route.js) to detect failed COD deliveries. When an order is 'CANCELLED', 'REJECTED', or 'EXPIRED', the system increments the user's codUnpaidCount, enabling future restrictions for high-risk buyers.

3. Shopee Guarantee Extension

Added extendShopeeGuarantee to lib/services/ordersService.js. This allows buyers to request a one-time 3-day extension of their confirmation window, providing more time to inspect the parcel before funds are released to the seller.

// Guarantee Extension Logic
const newDeadline = new Date(currentDeadline);
newDeadline.setDate(newDeadline.getDate() + 3);
await tx.order.update({
    data: { autoCompleteAt: newDeadline, isGuaranteeExtended: true }
});
