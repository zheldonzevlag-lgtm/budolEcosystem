Phase 3: Admin Mediation Hub

Status: Completed

Implemented Features

  • getDisputedReturns(): Fetches all returns currently in DISPUTED status for admin review.
  • resolveDispute(): Core mediation engine allowing admins to rule in favor of either party.
  • Resolution - REFUND_BUYER: Triggers refundFromLocked, updates order status, and increments seller NFR.
  • Resolution - REJECT_RETURN: Triggers releaseFromLocked, returning funds to seller's pending balance.
  • Real-time Notifications: Automated updates to both buyer and seller via WebSockets upon resolution.

Technical Implementation

Admin Resolution Logic

// resolveDispute logic implemented in returnsService.js
if (resolution === 'REFUND_BUYER') {
    await refundFromLocked({ orderId, amount });
    await updatePerformanceMetrics(buyerId, storeId);
} else if (resolution === 'REJECT_RETURN') {
    await releaseFromLocked({ orderId, amount });
}
