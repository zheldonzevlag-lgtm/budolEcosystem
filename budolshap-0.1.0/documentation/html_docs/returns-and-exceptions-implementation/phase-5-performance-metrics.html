Phase 5: Marketplace Performance Metrics

Implementation of safeguards and quality tracking for buyers and sellers.

Status: Completed
Objective: Track NFR, BRR, and penalty points to maintain marketplace integrity.

Technical Specifications

Non-Fulfilment Rate (NFR)

Calculated for stores based on a rolling 30-day window. Includes seller-cancelled orders and returns due to seller fault (e.g., wrong item, damaged item).

Formula: (Seller-Cancelled + Seller-Fault Returns) / Total Orders (excluding buyer cancellations)
Buyer Return Rate (BRR)

Calculated for users to identify high-risk return patterns. Used for fraud prevention and logistics optimization.

Formula: (Refunded Orders) / Total Orders
Penalty Points System

Automatic point assignment for stores with high NFR to incentivize quality fulfillment.

  • Threshold: 10% NFR
  • Scaling: 1 point for every 5% NFR above the threshold.

Implementation Details

The performanceService.js handles the heavy lifting of calculating these metrics using efficient Prisma queries.

Key Integration Points:

  • Order Service: Triggers metric updates on order completion, cancellation, or refund.
  • Returns Service: Triggers metric updates when return disputes are resolved or returns are processed.
  • Lalamove Webhook: Tracks COD delivery failures (codUnpaidCount).
// Example Performance Calculation
export async function updateStorePerformance(storeId) {
    const thirtyDaysAgo = new Date();
    thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);

    const nfr = (nonFulfilledOrders / totalOrders) * 100;
    let penaltyPoints = nfr > 10 ? Math.floor((nfr - 10) / 5) : 0;

    return await prisma.store.update({
        where: { id: storeId },
        data: { nonFulfilmentRate: nfr, penaltyPoints }
    });
}

Success Criteria

  • NFR and BRR correctly reflect the last 30 days of activity.
  • Penalty points are automatically calculated and stored.
  • Performance metrics are updated in real-time following status transitions.
