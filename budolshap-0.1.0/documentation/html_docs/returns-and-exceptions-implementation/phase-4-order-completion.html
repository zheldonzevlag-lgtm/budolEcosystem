Phase 4: Order Completion & Automation

This phase focuses on the final transition of funds from the escrow (pending) to the seller's available balance, either through manual buyer confirmation or automated expiration of the Shopee Guarantee.

1. Order Completion Logic

Implemented completeOrder in ordersService.js to handle the final state transition and fund release.

  • Platform Fee: Automatically deducts a 5% marketplace fee from the total order value.
  • Escrow Release: Triggers releaseFunds to move the net amount to the seller's available balance.
  • Status Update: Moves order to COMPLETED and records the completion timestamp.

2. Auto-Completion Trigger

Implemented autoCompleteOrders to handle orders where the buyer fails to confirm receipt within the Shopee Guarantee period.

// Example Auto-Completion Query
const ordersToComplete = await prisma.order.findMany({
    where: {
        status: { in: ['DELIVERED', 'SHIPPED'] },
        autoCompleteAt: { lte: now }
    }
});

3. Wallet Integration

The updateOrderStatus function was refactored to ensure that for COD (Cash on Delivery) orders, the funds are first credited to the pending balance upon delivery, before being released upon completion.

Success Criteria

  • Orders correctly transition to COMPLETED status.
  • Sellers receive 95% of the order total in their available balance.
  • Platform wallet (if implemented) or records track the 5% fee correctly.
  • Expired Shopee Guarantee orders are automatically finalized.
