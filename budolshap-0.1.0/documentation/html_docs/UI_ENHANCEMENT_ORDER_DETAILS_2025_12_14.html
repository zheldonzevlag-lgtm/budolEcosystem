UI Enhancement: Order Details & Price Formatting

Date: December 14, 2025

Author: Antigravity (AI Assistant)

Status: Deployed to Production

1. Implementation Process & Changes

The objective was to improve the readability and information density of the Order UI for buyers. This involved a series of iterative updates across multiple components.

Core Enhancements

  1. Price Formatting (Comma Separation):
    • Goal: Improve readability of large monetary values (e.g., P6,000.00 instead of P6000).
    • Implemented .toLocaleString() styling across:
      • components/OrderItem.jsx (Order List)
      • components/OrderItemsList.jsx (Order Details)
      • components/OrderSummary.jsx (Checkout/Cart Summary)
      • app/(public)/cart/page.jsx (Cart)
      • app/store/orders/page.jsx (Seller Dashboard)
  2. Booking ID Display:
    • Goal: Surface the Lalamove Booking ID to the buyer for transparency.
    • Added Booking ID display in three key locations:
      • Order List (OrderItem.jsx): Displayed below the status badge (Desktop & Mobile status logic).
      • Order Details Header (OrderHeader.jsx): Displayed prominently in the green status banner below the Order ID.
      • Delivery Details Tab (OrderAddressInfo.jsx): Added a dedicated row for Booking ID in the Order Information section.
  3. Store & Contact Information:
    • Goal: Allow buyers to easily identify and contact the seller from the order details page.
    • Updates:
      • Modified API app/api/orders/[orderId]/route.js to include the store's contact field in the Prisma query.
      • Updated OrderItemsList.jsx to display:
        • "Sold by: [Store Name]"
        • Seller Contact Number (with phone icon)
        • "Chat the Seller" button (placeholder for future functionality).
  4. Payment Display Improvements:
    • Enhanced OrderItem.jsx payment column:
      • Added "via" prefix (e.g., "via GCASH").
      • Styled method name to be Bold and Green for visibility.
      • Displayed Payment Method directly below Payment Status.

Description: An initial request to reorder the Order ID and Booking ID in `OrderItem.jsx` (Order ID first, then Booking ID) was implemented but was subsequently requested to be undone to match a specific UI design preference.

Fix: Resolved Reverted the layout changes in `OrderItem.jsx` to the previous approved state while implementing the new Order ID/Booking ID layout specifically in the `OrderHeader.jsx` component instead.

🌏 Currency Localization

We are currently using `Number(val).toLocaleString()` which defaults to the browser's locale.

  • Recommendation: Standardize this to `toLocaleString('en-PH', { style: 'currency', currency: 'PHP' })` globally via a utility helper function (e.g., `formatCurrency(amount)`) to ensure consistent "₱" symbol and decimal formatting across all browsers/regions, rather than relying on manual `process.env.CURRENCY_SYMBOL` + `toLocaleString` concatenation.
