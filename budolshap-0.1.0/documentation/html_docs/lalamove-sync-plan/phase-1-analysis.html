Phase 1: Analysis & Overview

Comprehensive review of the existing Lalamove integration and synchronization architecture.

System Architecture

The Lalamove integration follows a modular architecture consisting of three primary components:

  • Service Layer: services/lalamove.js - Handles direct communication with Lalamove API (v3), HMAC authentication, and payload verification.
  • Webhook Endpoint: app/api/webhooks/lalamove/route.js - Receives asynchronous events from Lalamove (Driver Assigned, Status Changed, etc.).
  • Manual Sync Endpoint: app/api/orders/[orderId]/sync-lalamove/route.js - Provides a polling fallback to fetch latest data on-demand.

Current Implementation State

Analysis of existing code reveals a robust implementation with several key features:

  • HMAC-SHA256 Authentication for all API requests and webhook verification.
  • State Machine Logic that maps complex Lalamove status codes to simplified Budolshap order statuses.
  • Real-time Triggers using Pusher/Socket.io to notify both Seller and Buyer UIs immediately upon status changes.
  • Robust Error Handling with webhook logging in the database for troubleshooting.

Key Files & Responsibilities

/services/lalamove.js
- class Lalamove implements ShippingProvider
- trackOrder(orderId): Fetches status and driver info
- verifyWebhookSignature(body, signature, timestamp)

/app/api/webhooks/lalamove/route.js
- POST handler for Lalamove events
- Logs events to WebhookEvent table
- Updates Order status and triggers real-time events

/app/api/orders/[orderId]/sync-lalamove/route.js
- Manual refresh triggered by the UI
- Ensures status is up-to-date even if webhooks are delayed

Technical Constraints & Risks

  • Webhook Reliability: Lalamove webhooks may occasionally fail or arrive out of order. The system handles this via manual sync polling.
  • Sandbox Limitations: In the sandbox environment, driver location data may be mocked (0,0), which the system correctly ignores.
  • Rate Limiting: Aggressive polling (every 10s) in the Buyer UI should be monitored for API rate limit issues.
