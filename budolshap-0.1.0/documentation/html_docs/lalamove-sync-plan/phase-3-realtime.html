Phase 3: Real-time Synchronization

How the system keeps the Seller and Buyer UIs updated in real-time.

Synchronization Multi-Strategy

Budolshap employs a triple-layered synchronization strategy to ensure that order status changes are reflected instantly across all interfaces.

1. Webhook Push

Lalamove pushes events to our webhook. The server updates the DB and immediately broadcasts a message via Pusher or Socket.io.

2. Real-time Subscription

UIs (Seller & Buyer) subscribe to their respective channels (store-{id} and user-{id}). Upon receiving an update, they refresh their data.

3. Smart Polling Fallback

The Buyer UI detects active Lalamove orders and triggers a POST /api/orders/[id]/sync-lalamove every 10 seconds as a safety net.

4. SWR Cache Management

Using useSWR, the system manages data fetching with background revalidation and instant UI updates (optimistic UI patterns where applicable).

Real-time Channel Logic

Channels are secured and scoped to specific users/stores:

  • Seller Channel: store-${storeId} - Receives order-updated events for any order belonging to the store.
  • Buyer Channel: user-${userId} - Receives order-updated events for orders the user has placed.

Event Payload Example:

{
  "orderId": "ord_12345",
  "status": "IN_TRANSIT",
  "shippingStatus": "PICKED_UP"
}

UI Integration Details

  • Seller UI: Uses useRealtimeOrders hook which abstracts the Pusher/Socket.io connection and auto-mutates the SWR cache.
  • Buyer UI: Uses a dedicated useEffect in the Order Detail page to listen for specific order updates and show toast notifications.
