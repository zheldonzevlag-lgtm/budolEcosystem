Phase 1: Centralized Configuration Update

The first step was to update the centralized shipping configuration to support the new terminology.

Changes in lib/shipping/config.js

Modified PROVIDER_STAGES and PROVIDER_TIMELINE_EVENTS to align with seller UI labels.

Updated Stage Labels:

  • Placed (Order Placed)
  • Paid (Payment Received)
  • Needs Booking (Awaiting shipment booking)
  • Booked (Driver assigned/Shipment booked)
  • In Transit (Package on the way)
  • Delivered (Package received by buyer)

Key Decisions

  • Label Consolidation: Replaced "To Receive" with "In Transit" to match the more technical status used in seller UIs.
  • Granular Booking: Split the processing stage into "Needs Booking" and "Booked" specifically for providers like Lalamove.
