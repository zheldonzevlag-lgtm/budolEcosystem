Audit Trail Implementation

PHASE 1 & 2 COMPLETE

Date: 2025-12-12
Scope: Backend Foundation, Database Schema, and Authentication Integration.

Process Overview

The objective was to implement a secure, database-backed logging system to track user Login and Logout events. This foundational work enables future auditing capabilities for administrators.

  • Phase 1: Database Schema modifications to store logs.
  • Phase 2: Backend logic to capture and persist log data automatically.

Technical Details

1. Database Schema

Prisma Added a new AuditLog model with a relation to the User table.

model AuditLog {
    id        String   @id @default(cuid())
    userId    String
    action    String   // "LOGIN" | "LOGOUT"
    ipAddress String?
    country   String?
    device    String?
    userAgent String?
    createdAt DateTime @default(now())

    user      User     @relation(fields: [userId], references: [id])
}

2. Logic Implementation

lib/audit.js Created a reusable utility function createAuditLog.

  • Extracts IP Address from headers (supporting proxies).
  • Parses User-Agent string using ua-parser-js to identify OS and Browser.
  • Captures Geolocation from Vercel headers (x-vercel-ip-country).

3. Integration Points

  • Login Route: app/api/auth/login/route.js - Logs success immediately after password verification.
  • Logout Route: app/api/auth/logout/route.js - Decodes the JWT token to identify the user before clearing the cookie.

Issues & Fixes

Issue 1: Build Failure (Module Import)

Problem: The initial import of ua-parser-js used a default import, which caused a build error because the library exports named exports in its newer version.

Fix: Changed import to import { UAParser } from 'ua-parser-js';.

Issue 2: Verification Script Compatibility

Problem: The test script verify-audit-logs.js failed because it used ES Module syntax (import) in a project configured mostly for CJS scripts or implicitly handled by Next.js.

Fix: Rewrote the script to use CommonJS require() syntax for standalone execution.

Test Results

A manual test script scripts/test-audit-db.js was created and executed.

  • User Identification: ✅ Successfully identified an existing test user.
  • Data Persistence: ✅ Successfully wrote both LOGIN and LOGOUT records to the database.
  • Field Integrity: ✅ IP, Country, and Device strings were correctly stored.
