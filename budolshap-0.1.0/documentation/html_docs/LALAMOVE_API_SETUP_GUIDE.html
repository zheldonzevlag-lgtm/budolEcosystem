🚚 Lalamove API Setup Guide

Complete step-by-step guide to get your Lalamove API credentials

1

Visit Lalamove Developer Portal

Go to the Lalamove Developer Portal to create your account and application.

🔗 Open Lalamove Developer Portal

Alternative URL: https://www.lalamove.com/en-ph/developers

2

Create a Lalamove Account

If you don't have a Lalamove account yet:

  1. Click on "Sign Up" or "Register"
  2. Choose "Business Account" (required for API access)
  3. Fill in your business details:
    • Business name
    • Email address
    • Phone number
    • Business address
  4. Verify your email address by clicking the link sent to your inbox
  5. Complete your business profile
⚠️ Important: You need a Business Account to access the API. Personal accounts cannot use the developer portal.
3

Access the Developer Dashboard

Once logged in:

  1. Navigate to "Developer" or "API" section in the menu
  2. You may need to accept the Developer Terms of Service
  3. Click on "Create Application" or "New App"
📸 Screenshot: Developer Dashboard showing "Create Application" button
4

Create a Sandbox Application

Fill in the application details:

  • Application Name: "Budolshap Shipping" (or your preferred name)
  • Environment: Select "Sandbox" (for testing)
  • Description: "E-commerce shipping integration for Budolshap marketplace"
  • Webhook URL: https://budolshap.vercel.app/api/webhooks/lalamove
  • Redirect URL: Leave blank (not needed for server-to-server)
💡 Tip: Start with Sandbox environment for testing. You can create a separate Production app later when you're ready to go live.
5

Get Your API Credentials

After creating the application, you'll receive three important credentials:

Credential Description Environment Variable
Client ID Public identifier for your application LALAMOVE_CLIENT_ID
Client Secret Private key for authentication (keep secret!) LALAMOVE_CLIENT_SECRET
Webhook Secret Used to verify webhook signatures LALAMOVE_WEBHOOK_SECRET
🔒 Security Warning: Never share your Client Secret or commit it to public repositories. Store it securely in environment variables only.
6

Add Credentials to Your Project

Copy your credentials and add them to your .env or .env.local file:

# Lalamove API Credentials (Sandbox) LALAMOVE_CLIENT_ID="your_client_id_from_dashboard" LALAMOVE_CLIENT_SECRET="your_client_secret_from_dashboard" LALAMOVE_WEBHOOK_SECRET="your_webhook_secret_from_dashboard" # Environment: 'sandbox' for testing, 'production' for live LALAMOVE_ENV=sandbox # Feature flag (optional) ENABLE_LALAMOVE=true

Steps:

  1. Open your .env.local file in the project root
  2. Paste the above configuration at the end of the file
  3. Replace the placeholder values with your actual credentials
  4. Save the file
7

Add Credentials to Vercel (Production)

To use Lalamove in production, add the environment variables to Vercel:

  1. Go to Vercel Dashboard
  2. Select your budolshap project
  3. Go to Settings → Environment Variables
  4. Add each variable:
    • Key: LALAMOVE_CLIENT_ID, Value: (your client ID)
    • Key: LALAMOVE_CLIENT_SECRET, Value: (your secret)
    • Key: LALAMOVE_WEBHOOK_SECRET, Value: (your webhook secret)
    • Key: LALAMOVE_ENV, Value: sandbox (or production)
    • Key: ENABLE_LALAMOVE, Value: true
  5. Select environment: Production, Preview, Development (all three)
  6. Click "Save"
💡 Note: After adding environment variables, you need to redeploy your application for the changes to take effect.
8

Test Your API Connection

Verify that your credentials work by testing the authentication:

// Test file: test-lalamove.js const Lalamove = require('./services/lalamove'); async function testAuth() { const lalamove = new Lalamove(); try { const token = await lalamove.getAccessToken(); console.log('✅ Authentication successful!'); console.log('Token:', token.substring(0, 20) + '...'); } catch (error) { console.error('❌ Authentication failed:', error.message); } } testAuth();

Run the test:

node test-lalamove.js
9

Configure Webhook in Lalamove Dashboard

Set up webhooks to receive delivery status updates:

  1. In the Lalamove Developer Dashboard, go to your application settings
  2. Find the "Webhooks" section
  3. Add webhook URL: https://budolshap.vercel.app/api/webhooks/lalamove
  4. Select events to subscribe to:
    • ✅ Order Created
    • ✅ Driver Assigned
    • ✅ Picked Up
    • ✅ Delivered
    • ✅ Cancelled
  5. Save the webhook configuration
💡 Note: The webhook endpoint will be created in Phase 2 of the integration. For now, just configure it in the dashboard.
10

When to Switch to Production

After testing in sandbox, create a production application:

  1. Complete all testing in sandbox environment
  2. Verify all features work correctly (quote, booking, tracking, webhooks)
  3. In Lalamove Dashboard, create a new application with Environment: Production
  4. Get new production credentials (different from sandbox)
  5. Update your environment variables:
    • Replace sandbox credentials with production credentials
    • Change LALAMOVE_ENV=production
  6. Redeploy to Vercel
  7. Perform a test order with real delivery
⚠️ Important: Production orders create real deliveries and charge real money. Always test thoroughly in sandbox first!
🎉 Congratulations! You now have your Lalamove API credentials set up and ready to use. You can proceed to Phase 2: Backend API Endpoints.

Troubleshooting Common Issues

Issue Solution
Can't find Developer Portal Make sure you have a Business Account. Personal accounts don't have API access.
Authentication fails Double-check your Client ID and Secret. Make sure there are no extra spaces or quotes.
Webhook not receiving events Verify the webhook URL is publicly accessible and uses HTTPS. Check Vercel deployment logs.
"Invalid credentials" error Ensure you're using the correct environment (sandbox vs production) credentials.
Rate limit exceeded Sandbox has lower rate limits. Implement request queuing or upgrade to production.
