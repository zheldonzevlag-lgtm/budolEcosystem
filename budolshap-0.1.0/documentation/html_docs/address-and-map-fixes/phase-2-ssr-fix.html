Phase 2: SSR Fix (AddressFormManager.jsx)

Problem Description

A window is not defined error occurred because Leaflet requires a browser environment, but Next.js was attempting to render the MapPicker component on the server.

Solution: Dynamic Import

Used Next.js next/dynamic with ssr: false to ensure the component only loads on the client side.

Key Code Changes

// In AddressFormManager.jsx
const MapPicker = dynamic(() => import('./MapPicker'), { 
    ssr: false,
    loading: () => <div className="h-[300px] w-full bg-gray-100 flex items-center justify-center rounded-lg">
        <p className="text-gray-500">Loading Map...</p>
    </div>
});

Affected Files

  • components/address/AddressFormManager.jsx

Validation

  • Verified that the page loads without "window is not defined" error.
  • Confirmed the loading state appears briefly before the map initializes.
