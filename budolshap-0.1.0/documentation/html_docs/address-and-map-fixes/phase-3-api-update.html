Phase 3: API Update (latitude/longitude saving)

Problem Description

While the frontend was capturing coordinates, the backend API was not persisting them to the database, causing coordinates to be lost on page refresh.

Solution: Schema and API Alignment

Updated the address API endpoints to include latitude and longitude fields in both POST and PUT requests.

Key Code Changes

// In app/api/addresses/route.js (POST)
const { ..., latitude, longitude } = body;
const address = await prisma.address.create({
    data: {
        ...,
        latitude: latitude ? parseFloat(latitude) : null,
        longitude: longitude ? parseFloat(longitude) : null
    }
});

Affected Files

  • app/api/addresses/route.js
  • app/api/addresses/[addressId]/route.js
  • app/api/stores/[storeId]/route.js
  • app/api/stores/[storeId]/addresses/route.js
  • app/api/stores/[storeId]/addresses/[addressId]/route.js

Validation

  • Verified that new addresses save with coordinates.
  • Confirmed that updating an existing address preserves/updates coordinates.
  • Checked database entries to ensure Float types are correctly stored.
