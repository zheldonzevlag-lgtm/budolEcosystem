Phase 1: Infinite Loop Fix (MapPicker.jsx)

Problem Description

The MapPicker component was triggering a Maximum update depth exceeded error. This was caused by a circular dependency between map movements and state updates:

  • Map moves → handleMoveEnd triggers setCenter
  • setCenter triggers useEffect → map.setView
  • map.setView triggers map move → loop starts again

Solution: Threshold-based Updates

Implemented a 1-meter threshold check using Leaflet's distanceTo method. Updates only occur if the coordinate change is significant.

Key Code Changes

// In MapCenterUpdater.jsx
const currentCenter = map.getCenter();
const newCenter = L.latLng(center[0], center[1]);

if (currentCenter.distanceTo(newCenter) > 1) {
    map.setView(newCenter);
}
