  • JWT_SECRET
  • NEXT_PUBLIC_APP_URL
  • NEXT_PUBLIC_SITE_URL
  • NEXT_PUBLIC_BASE_URL
  • NEXT_PUBLIC_CURRENCY_SYMBOL

3. Code Changes

🚀 Deployment Steps

Option A: Automated Script (Recommended)

Run the included PowerShell script to handle everything automatically.

.\deploy-vercel.ps1

This script will:

  1. Check Vercel login status
  2. Link the project
  3. Pull environment variables
  4. Run database migrations
  5. Deploy to production

Option B: Manual Steps

# 1. Login to Vercel
vercel login

# 2. Link project
vercel link

# 3. Pull environment variables
vercel env pull .env.production

# 4. Run migration
npx dotenv -e .env.production -- npx prisma db push

# 5. Deploy
vercel --prod

🔧 Troubleshooting

Migration Fails

Check if DATABASE_URL and DIRECT_URL are correctly set in Vercel. Try running migration manually.

Build Fails

Verify all required environment variables are set and the Prisma schema is valid. Check deployment logs.

🎉 Success Criteria

  • ✅ Build completes without errors
  • ✅ Database migrations run successfully
  • ✅ Application loads at your Vercel URL
  • ✅ You can register/login
  • ✅ Products display correctly
