✅ Delivered Features (Checklist)

  • Auto-Revert to "To Ship" (PROCESSING)
    When Lalamove sends a CANCELLED, EXPIRED, or REJECTED webhook, the order status is now automatically reverted to PROCESSING (instead of canceling the whole order).
  • Failure Tracking & History
    We now store the specific failureReason (e.g., "EXPIRED") and archive the failed attempt in a new shipping.previousAttempts array.
  • Visual "Failed" Indicator
    Added a red "Failed" badge in the Store Orders table for immediate visibility of problem orders.
  • "Rebook Delivery" Workflow
    A dedicated button allowing one-click re-booking of failed orders.
  • Manual Cancellation Recovery
    If a seller manually cancels a delivery via the API/Dashboard, it also triggers the rebookable state.

🧪 User Guide: How to Rebook

  1. Identify Failed Orders: Look for the ❌ Failed badge in your "Store Orders" list.
  2. View Details: Click the "Rebook Delivery" button (or the row).
  3. Review History: In the modal, you will see a red box explaining why it failed (e.g., "Delivery expired"). You can also see a list of "Previous Attempts".
  4. Action: Click the "🔄 Rebook Delivery" button inside the modal to instantly request a new driver.
