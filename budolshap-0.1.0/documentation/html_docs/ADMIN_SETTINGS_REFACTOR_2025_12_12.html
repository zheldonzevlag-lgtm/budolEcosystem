Admin Settings Menu Refactor

DEPLOYED

Date: 2025-12-12
Objective: Reorganize the Admin Settings into logical submenus for better usability and scalability.

Overview

The monolithic "System Settings" page has been refactored into distinct functional areas. This improves navigability and allows for modular management of Security, Rate Limiting, and Realtime configurations.

1. Sidebar Navigation Update

The Admin Sidebar has been updated to include a collapsible "Settings" menu with three distinct options.

  • Added: "Security Settings" (Idle Timeout)
  • Added: "Rate Limiting" (Login/Register Protection)

2. Security Settings Page

Dedicated page for configuring Idle Session limits.

c:\wamp64\www\budolshap-v3\app\admin\settings\security\page.jsx

Key Features:

  • Idle Session Timeout: Configurable in minutes (Default: 15).
  • Warning Threshold: Time before logout to show warning (Default: 1).

Backend Integration:

Fetches and Updates via /api/system/settings using sessionTimeout and sessionWarning keys.

3. Rate Limiting Settings Page

Dedicated page for configuring API Abuse Protection thresholds.

c:\wamp64\www\budolshap-v3\app\admin\settings\rate-limiting\page.jsx

Key Features:

  • Login Limit: Max attempts per 15 mins (Default: 10).
  • Register Limit: Max attempts per 60 mins (Default: 5).

Backend Integration:

Fetches and Updates via /api/system/settings using loginLimit and registerLimit keys.
