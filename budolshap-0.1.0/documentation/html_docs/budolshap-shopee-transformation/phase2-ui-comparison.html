5. Technical Recommendations

Data Integrity

  • Status Safety: Do not change order.status enum. The existing webhook handler relies on these exact strings.
  • Driver Info: Use the shipping.driverInfo object populated by the webhook for the new driver card.
  • Real-time: Reuse the existing order-updated event channel to refresh UI components.

UI Integration

  • Conditional Rendering: Only show Lalamove-specific components if order.shipping.provider === 'lalamove'.
  • Timeline Hooks: Update generateTimeline() in TrackingTimeline.jsx to return more granular events when driver info is available.

5. Technical Implementation Guide

Status Logic & Sync Safety

  • Source of Truth: The Lalamove Webhook remains the ONLY source of truth for status updates.
  • Sync Mechanism: We utilize the existing triggerRealtimeEvent in route.js which broadcasts to both Seller and Buyer channels.
  • UI Interpretation: Components like OrderProgressTracker will interpret Lalamove's ASSIGNING_DRIVER as "Looking for Driver" and ON_GOING as "Driver Assigned".

UI Components

  • OrderItem Refactor: Replace the desktop-only tooltip with a mobile-friendly collapsible breakdown in OrderItem.jsx.
  • Progress Tracker: Update the tracker to show the 5 Shopee milestones (Placed, Paid, To Ship, To Receive, Completed).
