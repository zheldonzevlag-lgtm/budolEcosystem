Phase 2: Map Rendering & Location Pinning

Problem Analysis

Users reported three major issues with the map implementation:

  • Grey Area: Occurs when the Leaflet map container is resized or initially hidden.
  • Missing Pin: The central location pin was either hidden or not properly positioned.
  • UI Overlap: Map controls and overlays were clashing with the map container.

Technical Solutions

1. Resolving the Grey Area

Implemented a robust invalidateSize strategy in the MapCenterUpdater component. The map now triggers multiple invalidation checks to ensure tiles are correctly loaded when the section expands.

2. Enhanced Location Pin

Redesigned the central pin overlay using high-visibility SVG icons with glow effects and shadows. The pin is now fixed at the container's center with a higher z-index (2000).

3. UI Controls Optimization

Relocated and restyled the zoom and geolocation controls. They now feature a modern, rounded design with better contrast and higher z-indexing to prevent overlap issues.

Data Flow

User Moves Map → onMoveEnd Triggered → reverseGeocode API Call → Update FormData → Coordinates Verified Badge Shown

Map Specifications

  • Container Height: Increased from 200px to 350px.
  • Z-Index Strategy: Controls and Pin at 2000, Overlays at 1001, Map at 0.
  • Rate Limiting: 1.5s delay for Nominatim geocoding to prevent API blocks.
