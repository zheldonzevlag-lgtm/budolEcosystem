Phase 1: UI Structure & Collapsible Sections

Overview

The primary goal of this phase was to transform the static address form into a more manageable, sectioned interface. This improves usability on mobile devices and reduces cognitive load by hiding non-essential information initially.

Technical Specifications

Feature Implementation Detail
State Management React useState hook for section visibility toggling.
Components AddressFormManager.jsx
Styling Tailwind CSS with transition effects.
Input Fields Conversion of Detailed Address from textarea to input.

Code Implementation

Example of the collapsible section logic implemented:


const [openSections, setOpenSections] = useState({
    contact: true,
    location: true,
    settings: true
});

const toggleSection = (section) => {
    setOpenSections(prev => ({
        ...prev,
        [section]: !prev[section]
    }));
};

UI Adjustments & Responsive Layout

  • Four-Section Refactoring: The form has been split into four distinct collapsible sections:
    • Contact Information: Name and Phone.
    • Address Location: City, Barangay, and Street fields.
    • Settings: Labeling and Default Address toggle.
    • Map: Address Search and Interactive Pin Location.
  • Desktop Split Layout:
    • Left Column: Contact Information, Address Location (Fields), and Settings.
    • Right Column: Map (Search + Visual Pin) which occupies the full vertical height.
  • Auto-Adjusting Elements:
    • The modal container width has been increased to max-w-6xl (approx. 1152px) on desktop to support the side-by-side layout.
    • The map height dynamically expands or contracts to fill available space in the right column on desktop.
