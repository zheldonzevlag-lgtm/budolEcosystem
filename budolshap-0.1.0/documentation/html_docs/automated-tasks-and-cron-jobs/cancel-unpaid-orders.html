Cancel Unpaid Orders

This job automatically voids orders that have remained in ORDER_PLACED status without payment beyond the allowed threshold.

Configuration Details

  • Endpoint: GET /api/cron/cancel-unpaid-orders
  • Recommended Schedule: Every 12 Hours
  • Threshold: 48 Hours (Default, configurable in Admin Settings)
  • Source: /app/api/cron/cancel-unpaid-orders/route.js
  • Service: /lib/services/ordersService.js

Process Flow

  1. Verification: Checks if orderCancellationEnabled is set to true in system settings.
  2. Identification: Finds orders with status ORDER_PLACED, isPaid: false, and paymentMethod: NOT 'COD' created before the cutoff date.
  3. Cancellation: Updates order status to CANCELLED and payment status to cancelled.
  4. Notifications: Triggers real-time events to notify stores that an unpaid order has been voided by the system.

Security & Safeguards

  • CRON_SECRET: Endpoint is secured via the authorization header to prevent unauthorized triggers.
  • Selective Targeting: Specifically excludes COD orders to ensure only online payment abandoned carts are cancelled.

Admin Controls

Admins can configure the cancellation threshold (in hours) and toggle the entire automation on/off via the Admin Settings Dashboard.
