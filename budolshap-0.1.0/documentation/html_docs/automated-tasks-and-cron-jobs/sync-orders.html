Shipping Status Sync

← Back to Overview

Overview

The Shipping Status Sync task is responsible for periodically fetching real-time delivery updates from shipping providers (e.g., Lalamove) for all active orders. This ensures that the marketplace remains in sync with actual physical delivery events, especially when webhooks might be delayed or missed.

Process Flow

1
Identify Active Orders: Queries the database for orders with status SHIPPED or DELIVERING that have an associated Lalamove tracking ID.
2
Fetch External Status: Calls the Lalamove API (or provider wrapper) to get the current status for each identified order.
3
Compare and Map: Maps the provider's status (e.g., COMPLETED, PICKED_UP) to the marketplace's internal order statuses.
4
Update Database: If a status change is detected, updates the order status and creates an entry in the order history/activity log.
5
Trigger Side Effects: If an order is marked as DELIVERED, it starts the Shopee Guarantee timer for the buyer.

Technical Details

Endpoint: /api/cron/sync-orders

Service Function: syncLalamoveStatus() in lib/services/shippingService.js

Configuration

Setting Value Description
Recommended Schedule Every 1-2 Hours Balanced frequency for real-time updates without hitting API rate limits.
Order Status Filter SHIPPED, DELIVERING Only syncs orders that are currently in transit.
Timeout 60 seconds Vercel serverless timeout for background tasks.

Security

Like all cron jobs in the system, this endpoint is protected by a CRON_SECRET environment variable check. Only requests with the correct Authorization: Bearer [SECRET] header will be executed.

// Authorization Check
const authHeader = request.headers.get('authorization');
if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
}
Pro Tip: This task acts as a safety net for the Lalamove Webhook. While the webhook provides instant updates, the sync job ensures no order gets "stuck" if the webhook fails to reach our server.
