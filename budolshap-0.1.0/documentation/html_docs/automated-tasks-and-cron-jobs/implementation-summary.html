Cron Job Engine

The marketplace utilizes Vercel Cron Jobs to manage all background processes. This serverless approach ensures high availability and scalability without the need for a dedicated long-running server.

Scheduling

Managed via vercel.json. All jobs are configured with standard cron syntax to run at specific intervals.

Centralized Config

Authentication

Endpoints are secured via CRON_SECRET environment variable to prevent unauthorized external triggers.

Secure Access

Error Handling

Each job implements robust try-catch blocks with detailed logging to catch and report failures.

Resilient Logic

Current Active Jobs

  • Auto-Complete Orders: Runs daily to finalize orders past the protection window.
  • Cancel Unpaid Orders: Runs every 12 hours to release stock from abandoned orders.
  • Shipping Status Sync: Runs every 2 hours to ensure logistics data is up-to-date.

Manual Overrides

While the jobs are automated, each endpoint supports a manual POST trigger for administrative use. This is useful for troubleshooting or forcing a sync outside the scheduled window.

Admin Note: Manual triggers still require the Authorization: Bearer [SECRET] header for security.

Best Practices

  • Idempotency: All cron functions are designed to be idempotent; running them multiple times will not cause duplicate state changes.
  • Timeouts: Vercel serverless functions have a 60-second limit for cron jobs (standard plan). Long-running tasks are split into batches if necessary.
  • Monitoring: Execution logs are available in the Vercel Dashboard under the "Logs" tab.
