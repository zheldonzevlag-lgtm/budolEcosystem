Auto-Complete Orders

This job automatically transitions orders from DELIVERED to COMPLETED status after the buyer confirmation period expires.

Configuration Details

  • Endpoint: GET /api/cron/auto-complete-orders
  • Schedule: 0 16 * * * (Daily at 4 PM UTC)
  • Source: /app/api/cron/auto-complete-orders/route.js
  • Utility: /lib/orderAutoComplete.js

Process Flow

  1. Identification: Scans for orders with status DELIVERED and autoCompleteAt date in the past.
  2. Escrow Release: For each order, it calculates the net earnings (Total - 5% platform fee) and moves funds from pendingBalance to availableBalance in the seller's wallet.
  3. Status Update: Updates the order status to COMPLETED and sets the completedAt timestamp.
  4. Notifications: Sends a completion email to the customer and triggers real-time updates for the store dashboard.

Manual Trigger (Admin Only)

Admins can manually trigger this process via the dashboard or a POST request for testing purposes.

POST /api/cron/auto-complete-orders
{
  "days": 7
}

Technical Implementation

The core logic is implemented in processAutoCompletions() inside /lib/orderAutoComplete.js. It handles self-healing for legacy orders that might be missing initial escrow records.
