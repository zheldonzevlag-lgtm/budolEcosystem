Project Overview

The Lalamove Return Tracking Fix successfully addressed a critical flaw in the return logistics pipeline. By isolating return shipping data and implementing consistent status propagation via webhooks and manual syncs, we have ensured that "BOOKED" returns now progress correctly to "SHIPPED" and "RECEIVED" statuses.

Key Achievements

  • Tracking ID Resolution: Implemented support for Lalamove's booked ID as the primary tracking identifier for returns.
  • Unified Sync Logic: Standardized isReturnSync detection across background services and API endpoints.
  • Realtime UI Updates: Added return-updated events to eliminate the need for manual page refreshes.
  • Failure Resilience: Implemented automatic status resets for cancelled or expired return bookings.

Technical Debt & Recommendations

While the immediate issue is resolved, we recommend the following future optimizations:

  • Log Centralization: Implement a dedicated logging service for shipping webhooks to facilitate easier troubleshooting of Lalamove API changes.
  • Schema Evolution: Consider promoting returnShipping from a Json field to a dedicated relational model for better query performance and type safety.
  • Multi-Carrier Support: Generalize the isReturnSync pattern to support other carriers (e.g., Flash Express, J&T) in the future.

Final Conclusion

The system is now robustly handling return status updates. The "flawed return process" reported by the user has been systematically analyzed, fixed, and verified across all tracking channels.
