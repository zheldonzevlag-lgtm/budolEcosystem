Phase 1: Problem Analysis

Lalamove Return Tracking System Optimization

← Back to Index Phase 1: Analysis Phase 2: Implementation Phase 3: Testing Summary

Executive Summary

The current return process in Budolshap encountered significant issues where return shipments (booked via Lalamove) failed to update their statuses in the UI. While the original delivery tracking worked correctly, the return tracking logic was decoupled from the main order's shipping updates, leading to a "stuck" state in the user and store dashboards.

Problem Identification

Through detailed code analysis and log review, the following core issues were identified:

  • Data Decoupling: The return tracking used Return.returnShipping while the system primarily monitored Order.shipping.
  • Webhook Routing: Lalamove webhooks for returns were being processed but failed to update the specific Return record because the metadata was not consistently utilized to differentiate between a standard delivery and a return.
  • Realtime Event Gaps: Even when data updated in the database, return-updated events were not consistently triggered, preventing UI updates without page refreshes.
  • Sync Logic Flaws: Background services and manual sync routes only looked at the main order's trackingNumber, ignoring the secondary booked ID generated for returns.

Technical Constraints & Dependencies

Component Requirement Impact
Prisma Schema Return model with returnShipping Json field Storage for return-specific Lalamove data.
Lalamove API Webhook payloads with metadata Used to identify return-type bookings.
Realtime Service Pusher/Socket integration Required for instant UI status propagation.

Risk Assessment

  • Medium Risk: Accidental updates to main order status when a return fails.
  • Medium Risk: Race conditions between webhook updates and manual syncs.
  • High Risk: Incorrect status mapping (e.g., Lalamove COMPLETED marking a return as CANCELLED).

Next Steps (Phase 2)

The implementation will focus on unifying the tracking logic across three main entry points: Webhooks, Manual Sync, and Background Services, ensuring isReturnSync logic is applied consistently.
