Phase 1: Component Analysis

Initial assessment of address-related components and branding requirements.

1. Component Inventory

The following components were identified as part of the address management ecosystem requiring branding updates:

AddressFormManager.jsx Core Form
AddressAutocomplete.jsx Search Engine
MapPicker.jsx Map Interface
AddressModal.jsx UI Wrapper
StoreAddressModal.jsx UI Wrapper

2. Visual Audit (Legacy Orange)

Identified legacy branding patterns across components:

  • Primary Actions

    Save buttons and form submissions using bg-orange-500.

  • Focus States

    Input fields highlighting with focus:border-orange-500.

  • Loading Indicators

    Spinners and progress bars using text-orange-500 or border-t-orange-500.

  • Map Markers

    Custom Leaflet divIcons hardcoded with orange SVG paths.

3. Technical Dependencies

Libraries

  • Tailwind CSS (Styling)
  • Lucide React (Icons)
  • Leaflet (Map Rendering)
  • Framer Motion (Animations)

Shared Logic

  • Geoapify API (Geocoding)
  • React Context (Address State)
  • Formik/Custom Validation
