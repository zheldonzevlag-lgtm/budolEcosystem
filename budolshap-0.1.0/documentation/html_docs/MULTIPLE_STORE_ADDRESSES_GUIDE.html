📍 Multiple Store Addresses

Implementation Guide & Documentation

BudolShap E-Commerce Platform

🎯 Overview

This feature allows stores to manage up to 3 addresses with the ability to set a default address. This is essential for stores with multiple branches or locations.

Key Features

  • ✓ Support for up to 3 addresses per store
  • ✓ Default address selection
  • ✓ Add, edit, and delete addresses
  • ✓ Structured address input with validation
  • ✓ Geocoding support for delivery optimization

🗄️ Database Schema

StoreAddress Model

model StoreAddress { id String @id @default(cuid()) storeId String // Address fields phone String district String // Region province String? city String barangay String detailedAddress String @db.Text zip String country String @default("Philippines") notes String? @db.Text // Geocoding cache latitude Float? longitude Float? // Default address flag isDefault Boolean @default(false) createdAt DateTime @default(now()) updatedAt DateTime @updatedAt // Relations store Store @relation(fields: [storeId], references: [id], onDelete: Cascade) @@index([storeId]) }

Store Model Update

Added relation to StoreAddress:

model Store { // ... existing fields addresses StoreAddress[] // ... rest of fields }

🔌 API Endpoints

Method Endpoint Description
GET /api/stores/[storeId]/addresses Get all addresses for a store
POST /api/stores/[storeId]/addresses Create a new address (max 3)
GET /api/stores/[storeId]/addresses/[addressId] Get a specific address
PUT /api/stores/[storeId]/addresses/[addressId] Update an address
DELETE /api/stores/[storeId]/addresses/[addressId] Delete an address

API Features

  • Automatic Default Management: First address is automatically set as default
  • Default Address Switching: When setting a new default, previous default is automatically unset
  • Smart Deletion: When deleting default address, another address is automatically promoted
  • 3-Address Limit: API enforces maximum of 3 addresses per store

Default Address Logic

  • First Address: Automatically set as default
  • Setting New Default: Previous default is automatically unset
  • Deleting Default: Next available address becomes default
  • Only One Default: System ensures only one default address exists

🚀 Migration Guide

Database Migration

File: scripts/migrations/add-store-addresses.sql

⚠️ Important: Run this migration before deploying to production!

Migration Steps:

  1. Creates StoreAddress table
  2. Adds foreign key constraint to Store table
  3. Creates index on storeId for performance
  4. Migrates existing store addresses to new table

Running the Migration:

# Connect to your database and run: psql $DATABASE_URL -f scripts/migrations/add-store-addresses.sql
Note: The migration creates a default address for each existing store using their current address and contact information. Store owners should update these addresses through the UI for accuracy.
