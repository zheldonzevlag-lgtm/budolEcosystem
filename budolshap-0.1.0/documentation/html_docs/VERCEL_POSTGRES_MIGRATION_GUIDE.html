We need the Vercel CLI to connect your local environment to the cloud database.

npm install -g vercel
2

Link Your Project

Run the following command in your project root (`c:\wamp64\www\budolshap`) to link it to your Vercel project.

vercel link

Follow the prompts:

  • Log in to Vercel (if asked).
  • Set up "c:\wamp64\www\budolshap"? Y
  • Which scope? (Select your account)
  • Link to existing project? Y
  • What's the name? (Enter budolshap or whatever you named it in Vercel)
3

Pull Environment Variables

This downloads the database connection strings (`POSTGRES_PRISMA_URL`, etc.) from Vercel to a local file.

vercel env pull .env.local

This creates a `.env.local` file with your production database credentials.

4

Generate & Apply Migration

Now we will generate the SQL migration file based on your updated schema and apply it to the Vercel Postgres database.

Note: We use `dotenv -e .env.local` to force Prisma to use the Vercel DB credentials we just downloaded, instead of your local `.env` file. 

npx dotenv -e .env.local -- npx prisma migrate dev --name init_postgres

What this does:

  • Connects to your Vercel Postgres database.
  • Creates a `migrations/` folder in your project.
  • Generates the SQL to create all your tables (User, Product, Order, etc.).
  • Applies the SQL to the Vercel database immediately.
  • Generates the Prisma Client.
5

Push Changes

The migration files (`migrations/*.sql`) are now part of your codebase. You must commit them to GitHub so Vercel knows about the database structure.

git add .
git commit -m "Add postgres migrations"
git push origin main
6

Verify Deployment

Go to your Vercel Dashboard. The new deployment should trigger automatically. Once finished:

  1. Open your live app URL.
  2. Since the DB is empty, you need to create a new account.
  3. To make yourself an admin, you can use Prisma Studio connected to the remote DB: 
npx dotenv -e .env.local -- npx prisma studio
