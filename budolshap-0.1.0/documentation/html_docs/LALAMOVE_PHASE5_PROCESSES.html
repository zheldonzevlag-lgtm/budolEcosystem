📖 Lalamove Phase 5

Step-by-Step Implementation Guide
Creating Real Orders in Partner Portal

🎯 Overview

This guide provides detailed, step-by-step instructions for implementing Phase 5 of the Lalamove integration. By following these steps, you will enable actual order creation that appears in your Lalamove Partner Portal.

⚠️ Critical Understanding

The Key Issue: The current implementation uses hardcoded stopIds ('0' and '1') when creating orders. However, Lalamove requires you to use the actual stopIds returned in the quote response.

Why This Matters: Each quote generates unique stopIds for each location. These IDs link the order back to the original quote and ensure proper routing and pricing.

6.1 Create Webhook Endpoint IMPORTANT

File: app/api/shipping/lalamove/webhook/route.js (Create new)

// app/api/shipping/lalamove/webhook/route.js import { NextResponse } from 'next/server'; import crypto from 'crypto'; import prisma from '@/lib/prisma'; /** * POST /api/shipping/lalamove/webhook * Receive status updates from Lalamove */ export async function POST(request) { try { // Get raw body for signature verification const body = await request.text(); const signature = request.headers.get('X-Lalamove-Signature'); console.log('[Webhook] Received request'); // Verify webhook signature const webhookSecret = process.env.LALAMOVE_WEBHOOK_SECRET; const hmac = crypto.createHmac('sha256', webhookSecret); const expectedSignature = hmac.update(body).digest('hex'); if (signature !== expectedSignature) { console.error('[Webhook] Invalid signature'); return NextResponse.json( { error: 'Invalid signature' }, { status: 401 } ); } // Parse payload const payload = JSON.parse(body); const { updateType, data } = payload; console.log('[Webhook] Update type:', updateType); console.log('[Webhook] Data:', JSON.stringify(data, null, 2)); // Handle different update types switch (updateType) { case 'ORDER_STATUS_CHANGED': await handleStatusChange(data); break; case 'DRIVER_ASSIGNED': await handleDriverAssigned(data); break; case 'ORDER_AMOUNT_CHANGED': await handleAmountChange(data); break; default: console.log('[Webhook] Unknown update type:', updateType); } // Always return 200 to acknowledge receipt return NextResponse.json({ received: true }); } catch (error) { console.error('[Webhook] Error:', error); // Still return 200 to prevent retries return NextResponse.json({ received: true }); } } // Handle status change async function handleStatusChange(data) { const { orderId, status } = data; console.log('[Webhook] Status change:', orderId, '->', status); // Find order by Lalamove orderId in shipping.booking.orderId const orders = await prisma.order.findMany({ where: { shipping: { path: ['booking', 'orderId'], equals: orderId } } }); if (orders.length > 0) { const order = orders[0]; // Update order status await prisma.order.update({ where: { id: order.id }, data: { shipping: { ...order.shipping, booking: { ...order.shipping.booking, status: status, lastStatusUpdate: new Date().toISOString() } } } }); console.log('[Webhook] Updated order:', order.id); } else { console.log('[Webhook] Order not found:', orderId); } } // Handle driver assigned async function handleDriverAssigned(data) { const { orderId, driverInfo } = data; console.log('[Webhook] Driver assigned:', orderId); const orders = await prisma.order.findMany({ where: { shipping: { path: ['booking', 'orderId'], equals: orderId } } }); if (orders.length > 0) { const order = orders[0]; await prisma.order.update({ where: { id: order.id }, data: { shipping: { ...order.shipping, booking: { ...order.shipping.booking, driverInfo: driverInfo, driverAssignedAt: new Date().toISOString() } } } }); console.log('[Webhook] Driver info updated'); } } // Handle amount change async function handleAmountChange(data) { const { orderId, priceBreakdown } = data; console.log('[Webhook] Amount changed:', orderId); const orders = await prisma.order.findMany({ where: { shipping: { path: ['booking', 'orderId'], equals: orderId } } }); if (orders.length > 0) { const order = orders[0]; await prisma.order.update({ where: { id: order.id }, data: { shipping: { ...order.shipping, booking: { ...order.shipping.booking, priceBreakdown: priceBreakdown, priceUpdatedAt: new Date().toISOString() } } } }); console.log('[Webhook] Price updated'); } }

🔧 Troubleshooting Guide

Problem: ERR_INVALID_STOP_ID

Symptom: Order creation fails with invalid stopId error.

Cause: Using hardcoded stopIds instead of values from quote.

Solution:

  1. Verify getQuote returns stops array
  2. Check stops are stored in database
  3. Confirm createOrder receives stops parameter
  4. Verify stopIds are extracted correctly: stops[0].stopId

Problem: Order Not Appearing in Partner Portal

Possible Causes:

  • Using wrong environment (sandbox vs production)
  • API credentials mismatch
  • Order creation actually failed but error not caught
  • Wrong market code

Solution:

  1. Check console logs for actual API response
  2. Verify LALAMOVE_ENV matches portal you're checking
  3. Confirm API credentials in .env.local
  4. Check market code is 'PH' not 'PH_MNL'

Problem: ERR_INSUFFICIENT_CREDIT

Symptom: HTTP 402 error when creating order.

Cause: Lalamove wallet balance too low.

Solution:

  1. Log in to Partner Portal
  2. Check wallet balance
  3. Top up wallet if needed
  4. Retry order creation

Problem: Quote Expired

Symptom: ERR_INVALID_QUOTE when creating order.

Cause: More than 15 minutes passed since quote was created.

Solution:

  1. Check quote.expiresAt before creating order
  2. If expired, fetch new quote
  3. Implement auto-refresh logic in checkout flow
  4. Notify user if price changed
