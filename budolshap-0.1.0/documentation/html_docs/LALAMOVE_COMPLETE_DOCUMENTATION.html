2. Issues Encountered

Issue #1: Missing Store Coordinates

CRITICAL Problem

All Lalamove quotes were using hardcoded default coordinates (Glorietta 4, Makati) instead of actual store locations.

Impact

  • Inaccurate delivery quotes
  • Wrong pickup locations for drivers
  • Potential delivery failures

Root Cause

Store model lacked latitude and longitude fields. No geocoding system in place.

Issue #2: Invalid Phone Numbers

CRITICAL Problem

Error: ERR_INVALID_PHONE_NUMBER (initially masked as "Shipping service error")

Details

Field Value Issue
Store Contact 01234567899 Dummy number, invalid format
Formatted Result +6301234567899 Invalid PH phone number
Expected Format +639xxxxxxxxx PH mobile standard

Why Quote Succeeded but Booking Failed

  • Quote API: Only requires coordinates and address (no contact validation)
  • Booking API: Requires valid contact names and phone numbers

Issue #3: Quote Expiration & Schedule Time

CRITICAL Problem

Error Sequence:

  1. ERR_INVALID_SCHEDULE_TIME - Misleading initial error
  2. ERR_UNKNOWN_FIELD - After attempting to add scheduleAt

Timeline

  • 2025-11-29 17:23: Quote generated
  • 2025-11-30 01:52: Booking attempted (next day)
  • Result: Quote schedule time was in the past

Discovery Process

  1. Initial error suggested missing scheduleAt field
  2. Added scheduleAt to order payload
  3. Lalamove rejected: "additionalProperties 'scheduleAt' not allowed"
  4. Conclusion: Lalamove API v3 doesn't accept scheduleAt in order creation
  5. Solution: Must refresh quote instead of overriding schedule time

Geocoding API Endpoint

File: app/api/stores/[storeId]/geocode/route.js

export async function PATCH(request, { params }) {
  const { storeId } = params;
  const store = await prisma.store.findUnique({ where: { id: storeId } });
  
  // Check if already geocoded
  if (store.latitude && store.longitude) {
    return NextResponse.json({ 
      success: true, 
      cached: true,
      coordinates: { lat: store.latitude, lng: store.longitude }
    });
  }
  
  // Geocode using Nominatim
  const query = encodeURIComponent(store.address + ', Philippines');
  const response = await fetch(
    `https://nominatim.openstreetmap.org/search?format=json&q=${query}`,
    { headers: { 'User-Agent': 'BudolShap/1.0' } }
  );
  
  const data = await response.json();
  const coordinates = { lat: parseFloat(data[0].lat), lng: parseFloat(data[0].lon) };
  
  // Cache coordinates
  await prisma.store.update({
    where: { id: storeId },
    data: { latitude: coordinates.lat, longitude: coordinates.lng }
  });
  
  return NextResponse.json({ success: true, cached: false, coordinates });
}

Widget Integration

File: components/LalamoveWidget.jsx

// Fetch store details and use cached coordinates
const storeRes = await fetch(`/api/stores/${storeId}`);
const storeData = await storeRes.json();

if (storeData.latitude && storeData.longitude) {
  pickup = {
    coordinates: { lat: storeData.latitude, lng: storeData.longitude },
    address: storeData.address
  };
} else {
  // Geocode and cache
  const geocodeRes = await fetch(`/api/stores/${storeId}/geocode`, { method: 'PATCH' });
  const geocodeData = await geocodeRes.json();
  
  if (geocodeData.success) {
    pickup = {
      coordinates: geocodeData.coordinates,
      address: storeData.address
    };
  }
}

Fix #2: Phone Number Validation

IMPLEMENTED Solution

File: app/api/shipping/lalamove/book/route.js

const formatPhone = (phone, fallback = '+639170000000') => {
  if (!phone) return fallback;
  
  let cleaned = phone.replace(/[\s\-\(\)]/g, '');
  let formatted;
  
  if (cleaned.startsWith('09')) {
    formatted = '+63' + cleaned.substring(1);
  } else if (cleaned.startsWith('0') && !cleaned.startsWith('09')) {
    // Invalid dummy numbers like "01234567899"
    console.warn(`[Phone Format] Invalid phone: ${phone}, using fallback`);
    return fallback;
  }
  // ... other format handling ...
  
  // Validate final format
  const mobilePattern = /^\+639\d{9}$/;
  const landlinePattern = /^\+632\d{7,8}$/;
  
  if (!mobilePattern.test(formatted) && !landlinePattern.test(formatted)) {
    console.warn(`[Phone Format] Invalid format: ${formatted}, using fallback`);
    return fallback;
  }
  
  return formatted;
};
