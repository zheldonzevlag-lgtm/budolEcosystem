Implementation Summary

Executive Overview

The Address Phone Refactor project successfully consolidated phone number management into a single, robust component. By introducing dynamic country codes and strict 10-digit validation, we have improved data quality and user experience across both customer and merchant interfaces.

Key Outcomes

100%
Code Centralization
-150
Lines of Redundant Code
  • Dynamic Country Codes: Support for PH, US, UK, SG, JP, CN, AU, and AE prefixes.
  • Improved UX: Users no longer need to manually type country codes; validation provides immediate feedback.
  • Backend Consistency: All addresses now submit phone numbers in standardized E.164 format.

Risk Assessment & Mitigation

  • Risk: Existing phone numbers in different formats.
    Mitigation: The parsing logic in AddressFormManager handles various prefixes and defaults to +63 if unrecognized.
  • Risk: Different validation rules for international numbers.
    Mitigation: Currently enforced 10-digit rule is optimized for the primary market (PH). Future phases can introduce country-specific regex validation.

Future Enhancements

  • Dynamic Validation: Adjust digit length requirements based on the selected country code.
  • Searchable Dropdown: Implement a searchable country picker for a better experience as the list of codes grows.
  • SMS Verification: Integrate OTP verification directly into the centralized form logic.
