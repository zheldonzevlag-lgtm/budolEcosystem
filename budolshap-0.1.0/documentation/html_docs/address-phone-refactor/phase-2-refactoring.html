Phase 2: Component Refactoring

Objective

Streamline parent components (AddressModal and StoreAddressModal) by removing redundant state management and logic that has been moved to AddressFormManager.

Refactoring Details

Refactor AddressModal.jsx

Removed heavy lifting of address state. The component now acts as a pure wrapper for the user profile loading and the AddressFormManager.

  • Deleted addresses array state.
  • Deleted errors object state.
  • Removed handlePhonePartChange, handleAddressChange, handleAddForm, and handleRemoveForm.
  • Cleaned up unused imports: PlusIcon, MapPin, Navigation, useMemo.
Refactor StoreAddressModal.jsx

Simplified initial data mapping and removed hardcoded string manipulations.

  • Removed manual +63 replacement logic for phone numbers.
  • Standardized initialData prop passing to AddressFormManager.

Technical Debt Addressed

  • Code Duplication: Phone validation and formatting were previously implemented in three different places.
  • State Complexity: Parent components are no longer responsible for validating internal form fields of the child component.
  • Bundle Size: Removed several unused Lucide icons and React hooks.
