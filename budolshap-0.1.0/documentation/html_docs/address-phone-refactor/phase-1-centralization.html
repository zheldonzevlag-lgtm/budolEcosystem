Phase 1: Logic Centralization

Objective

Consolidate phone number handling logic into a single source of truth (AddressFormManager.jsx) to ensure consistency across the application, specifically supporting dynamic country codes and standardized validation.

Technical Specifications

  • Component: AddressFormManager.jsx
  • Key Feature: Separation of Country Code and Phone Number parts in the UI.
  • Storage Format: E.164 (e.g., +639123456789).
  • Validation: Exact 10-digit requirement for the number part.

New COUNTRY_CODES Configuration

const COUNTRY_CODES = [
    { code: '+63', country: 'PH' },
    { code: '+1', country: 'US' },
    { code: '+44', country: 'UK' },
    // ... additional codes
]

Implementation Details

The handlePhonePartChange function was introduced to manage the state synchronization between the dropdown and the input field:

const handlePhonePartChange = (part, value) => {
    const currentCode = COUNTRY_CODES.find(c => formData.phone.startsWith(c.code))?.code || '+63';
    const currentNumber = formData.phone.startsWith(currentCode) ? formData.phone.slice(currentCode.length) : formData.phone;
    
    let newPhone = '';
    if (part === 'code') {
        newPhone = value + currentNumber;
    } else {
        const digits = value.replace(/\D/g, '');
        newPhone = currentCode + digits;
    }
    
    setFormData(prev => ({ ...prev, phone: newPhone }));
    // ... error clearing logic
};

Success Criteria

  • Dropdown correctly switches between country codes.
  • Input field only accepts digits.
  • Validation triggers if the number part is not exactly 10 digits.
  • Initial data with existing phone numbers is correctly parsed into the split UI.
