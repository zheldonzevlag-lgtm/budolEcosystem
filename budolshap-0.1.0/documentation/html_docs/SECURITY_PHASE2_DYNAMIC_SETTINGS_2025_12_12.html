Security Implementation - Phase 2.1

ENHANCEMENT COMPLETED

Focus: Dynamic Configuration for Idle Timeout

Feature Overview

Based on feedback, we have upgraded the Idle Timeout mechanism to be fully configurable via the Admin Dashboard. This removes the need for code changes to adjust security policies.

New Capabilities:

  • Database Storage: Settings are stored in the SystemSettings table.
  • Admin Interface: A new "Security Settings" section in /admin/settings.
  • Live Updates: Dashboards automatically fetch the latest timeout rules upon page load/refresh.

How to Configure

  1. Navigate to Settings in the Admin Dashboard.
  2. Locate the Security Settings section.
  3. Adjust the Idle Session Timeout (Default: 15 mins).
  4. Adjust the Warning Before Timeout (Default: 1 min).
  5. Click Save Configuration.

Note: Changes will take effect for users the next time they refresh their page or log in.

Technical Implementation

  • Schema: Added sessionTimeout and sessionWarning to SystemSettings model.
  • API: Created /api/system/settings endpoint for unified settings management.
  • Frontend: Updated AdminLayout and StoreLayout to consume dynamic values.
