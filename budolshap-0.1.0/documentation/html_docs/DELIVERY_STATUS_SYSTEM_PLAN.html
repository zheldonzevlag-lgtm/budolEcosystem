📊 Proposed Order Status Stages for Budolshap

Complete Order Lifecycle

1. Order Placed

PENDING

Description: Order has been created but payment is not yet confirmed.

Customer Action Required: Complete payment

System Action: Hold order for 24 hours, then auto-cancel if unpaid

Notifications:
  • Email: Order confirmation with payment instructions
  • SMS: Payment reminder after 12 hours

2. Payment Confirmed

TO SHIP

Description: Payment received and verified. Waiting for seller to prepare and ship.

Seller Action Required: Prepare order and arrange shipping within 2-3 business days

System Action: Notify seller, start countdown timer

Notifications:
  • Buyer: "Payment confirmed! Seller is preparing your order."
  • Seller: "New order received. Please ship within 2 days."
⚠️ Auto-Cancel Policy: If seller doesn't ship within 3 business days, order is automatically cancelled and buyer receives full refund.

3. Order Processed

READY TO SHIP

Description: Seller has arranged shipping and obtained tracking number.

Next Step: Waiting for courier pickup

Notifications:
  • Buyer: "Your order is ready! Waiting for courier pickup."
  • Display tracking number (if available)

4. Picked Up by Courier

SHIPPED

Description: Courier has collected the package from seller.

EDD Displayed: Show estimated delivery date prominently

Notifications:
  • Email + SMS: "Your order is on the way! Track it here: [link]"
  • Push notification: "Package picked up. EDD: [date]"

5. In Transit

TO RECEIVE

Description: Package is moving through logistics network.

Sub-statuses:

  • At sorting facility
  • Departed from origin city
  • Arrived at destination city
  • At local delivery hub
Notifications:
  • Update every major milestone (via webhook from Lalamove)
  • Daily EDD reminder if delivery is tomorrow
ℹ️ Note to Customer: "Tracking updates may take 24-48 hours to reflect. Please check back later if no updates are shown."

6. Out for Delivery

OUT FOR DELIVERY

Description: Package is with the driver and will be delivered today.

Real-time Tracking: Show driver location on map (Lalamove integration)

Notifications:
  • SMS: "Your order is out for delivery! Driver: [Name], ETA: [time]"
  • Push: "Driver is 15 minutes away!"
  • Display driver contact information

7. Delivered

COMPLETED

Description: Package successfully delivered to customer.

Proof of Delivery: Photo, signature, or confirmation code

Notifications:
  • Email + SMS: "Your order has been delivered! Please rate your experience."
  • Request review after 24 hours
✓ Next Steps: Customer can now rate the product and seller. Order moves to "To Rate" category.

8. Cancelled / Returned

CANCELLED

Reasons:

  • Customer cancelled before shipping
  • Seller didn't ship within timeframe (auto-cancel)
  • Delivery failed multiple times (Return to Sender)
  • Customer rejected delivery
Notifications:
  • Email: Cancellation confirmation with refund details
  • Refund processed within 5-7 business days

⏰ Handling Delayed Deliveries - Action Plan

Scenario 1: Order Still Within EDD

Customer Actions Available:

1️⃣ Check Order Status

Direct customer to Order Details page showing:

  • Current status with visual progress bar
  • EDD prominently displayed in green bar
  • Last tracking update timestamp
  • Expected next update timeframe

2️⃣ Track Delivery

Provide comprehensive tracking information:

  • Real-time location (if out for delivery)
  • Tracking history with timestamps
  • Courier contact information
  • Estimated delivery time window
ℹ️ Set Expectations: Display message: "Tracking updates may take 24-48 hours to reflect. Delays may occur during holidays or severe weather."

3️⃣ Contact Seller

Enable direct communication:

  • "Contact Seller" button on Order Details page
  • In-app messaging system
  • Seller must respond within 24 hours
  • Escalate to admin if no response

Scenario 2: Order Past EDD (Delayed)

Escalated Customer Actions:

1️⃣ Follow Up via Support Chat

Implement Shopee-style follow-up form:

  • Dedicated "Report Delayed Delivery" button appears after EDD
  • Pre-filled form with order details
  • Customer describes issue
  • System creates support ticket automatically
  • Email update sent within 24-48 hours
// Auto-display delayed delivery options if (currentDate > estimatedDeliveryDate) { showDelayedDeliveryOptions(); // Options: Follow Up, Request Refund, Contact Support }

2️⃣ Request Return/Refund

Enable self-service refund request:

  • "Return/Refund" button appears on Order Details after EDD + 2 days
  • Customer selects reason: "Item not received"
  • System investigates with seller and courier
  • Auto-approve refund if no proof of delivery after 7 days

3️⃣ Request Expedited Delivery

For packages stuck at hub:

  • Eligible if package at last hub > 24 hours
  • Eligible if order past EDD
  • Customer clicks "Request Priority Delivery"
  • System contacts courier to prioritize
  • Update sent within 24 hours

4️⃣ Contact Courier Directly

Provide courier contact information:

  • Display Lalamove support hotline
  • Provide order tracking number
  • Include courier reference number
  • Show business hours

Scenario 3: Special Cases

⚠️ COD Orders Delayed

Cash on Delivery orders require special handling:

  • Customer must follow up via support chat
  • Provide order number in inquiry
  • Verify customer identity before sharing details
  • Coordinate with courier on cash handling

❌ Return to Sender (RTS)

When delivery fails multiple times:

  • Notify customer after 2 failed delivery attempts
  • Offer alternative: pickup at depot, reschedule, change address
  • If no response after 3 attempts, package returns to seller
  • Refund issued minus restocking fee (if applicable)
  • Customer notified of RTS status

🚨 Fake Delivery Attempts

Handling fraudulent delivery claims:

  • Customer can report "Fake Delivery" if marked delivered but not received
  • System requests proof of delivery from courier
  • If no valid proof, refund customer immediately
  • Report courier to Lalamove
  • Flag courier for future orders

📱 Notification Best Practices

  • Consolidate: Don't spam customers. Group related updates.
  • Timing: Send during business hours (8 AM - 8 PM) unless urgent.
  • Personalize: Use customer's name and order details.
  • Actionable: Every notification should have a clear next step.
  • Opt-out: Allow customers to manage notification preferences.
