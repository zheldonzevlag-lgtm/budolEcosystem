Phase 1: Lalamove Return Booking Bug Fix

Issue Overview

The system was failing to book return shipments with Lalamove, resulting in a 500 Internal Server Error and the UI error message: lalamove.placeOrder is not a function.

Component Status Details
Lalamove Provider MISMATCH The provider implemented createOrder() but the service called placeOrder().
Payload Structure INVALID The payload structure sent to the provider did not match the required schema for Lalamove V3.
Phone Formatting MISSING Return contacts were not being formatted to E.164, causing Lalamove API rejection.

Implemented Fixes

1. Standardized Provider Method Call

Updated shippingService.js to use the correct method createOrder() instead of the non-existent placeOrder().

2. Payload Restructuring

Restructured the booking payload to match the Lalamove provider's expected abstraction:

const bookingPayload = {
  quotationId: quoteId,
  stops,
  pickupContact: { name: pickup.name, phone: pickupPhone },
  deliveryContact: { name: delivery.name, phone: deliveryPhone },
  packageDetails: { remarks: `Return for Order ${order.id}` }
};

3. Centralized Phone Formatting

Extracted formatPhone into a top-level utility within shippingService.js to ensure consistent E.164 formatting for both forward and return logistics.

4. Database Mapping & Schema Fix

Identified that the Return model in schema.prisma was missing the returnShipping field. Added the field to the schema and synchronized the database.

model Return {
  ...
  returnShipping Json?
  ...
}

Corrected the field mapping when updating the Return record, ensuring the complex shipping object is stored in the new returnShipping JSON field.

Troubleshooting: EPERM Errors

If you encounter an Unknown argument returnShipping error after these changes, it means the Prisma Client needs to be regenerated. On Windows, this may fail with an EPERM error if the dev server is running.

Solution:

  1. Stop your Next.js dev server (Ctrl+C).
  2. Run npx prisma generate in your terminal.
  3. Restart your dev server with npm run dev.

Verification Steps

  • Verified lalamove.js implements createOrder and correctly handles the restructured payload.
