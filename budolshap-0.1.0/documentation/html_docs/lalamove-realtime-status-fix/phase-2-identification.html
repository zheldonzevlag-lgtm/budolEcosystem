Phase 2: Identification & Fix Implementation

In Progress

Root Cause Identification

The primary reason for the failure of real-time updates via webhooks was identified as an inconsistency in status data passed to the triggerRealtimeEvent function.

  • Manual Sync: Always passed the mapped BudolShap OrderStatus.
  • Webhooks: Often passed raw Lalamove status strings (e.g., "PICKED_UP") when newStatus was null.

Proposed Implementation

1. Normalize Event Data

Modify the webhook handler to ensure the status field in the Pusher event always reflects the internal Order status.

// app/api/webhooks/lalamove/route.js

- status: newStatus || status,
+ status: newStatus || order.status,

2. Robust Settings Retrieval

Enhance lib/realtime.js to handle transient database failures by implementing a secondary cache or increasing the retry logic for fetching Pusher credentials.

Technical Specifications

  • Event Name: order-updated
  • Channel Pattern: store-{storeId}
  • Payload: 
    {
    orderId: string,
    status: OrderStatus,
    shippingStatus: string
}
