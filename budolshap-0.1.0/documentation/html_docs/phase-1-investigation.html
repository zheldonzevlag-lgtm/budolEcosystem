Executive Summary

This phase focused on analyzing user-reported issues with shipping fee display and computation in the Budolshap application. The primary discrepancy identified was the shipping fee showing as ₱0.00 or "FREE" instead of the actual Lalamove quote (e.g., ₱73) in cart, payment success, and order details pages. Through codebase searches, file readings, and logic reviews, the root cause was determined to be a missing 'cost' field in the shipping object sent to the backend during order creation for Lalamove providers. This phase sets the foundation for subsequent fixes, ensuring accurate persistence and display of shipping costs.

Technical Details

Initial user requests included including shipping fees in order totals, displaying them in payment modals and pages, and resolving display discrepancies. Key activities:

  • Verified backend logic in ordersService.js, confirming shippingCost is included in order.total but only saved if shipping.provider === 'lalamove' and shipping.cost is provided.
  • Searched codebase for LalamoveWidget usage and onQuoteReceived handlers.
  • Read and analyzed files: OrderSummary.jsx (lines 1-697), LalamoveWidget.jsx, ordersService.js.
  • Identified gap: LalamoveWidget fetches quotes but OrderSummary.jsx does not include 'cost' in the POST body for Lalamove shipping.

Key code snippet from ordersService.js:

let shippingCost = 0;
if (shipping && shipping.provider === 'lalamove') {
    shippingCost = shipping.cost || 0;
}

Key code snippet from OrderSummary.jsx (pre-fix):

shipping: shippingMethod === 'LALAMOVE' && lalamoveQuote ? {

    quotationId: lalamoveQuote.quoteId,
    quoteId: lalamoveQuote.quoteId,
