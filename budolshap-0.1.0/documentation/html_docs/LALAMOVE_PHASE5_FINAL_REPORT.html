1. Processes Implemented

The following core processes were implemented to enable accurate store location tracking for deliveries:

PROCESS Store Geocoding Workflow

Goal: Replace hardcoded pickup coordinates with actual store locations.

  1. Schema Update: Added latitude and longitude columns to the Store table.
  2. Coordinate Retrieval: When a user requests a quote, the system checks if the store has cached coordinates.
  3. On-Demand Geocoding: If no coordinates exist, the system calls the Nominatim API via /api/stores/[storeId]/geocode.
  4. Caching: The result is saved to the database to prevent future API calls.
  5. Quote Generation: The Lalamove API is called with the specific store coordinates as the pickup point.
PROCESS Production Deployment & Migration

Goal: Apply changes to the live Vercel/Postgres environment.

  1. Code Commit: Changes committed to local git repository.
  2. Database Migration: SQL commands executed directly against the production database
  3. ! Existing Data: 3/3 stores failed geocoding due to bad addresses.
  4. ✓ New Data: Any new store created with a valid address will be geocoded automatically.
