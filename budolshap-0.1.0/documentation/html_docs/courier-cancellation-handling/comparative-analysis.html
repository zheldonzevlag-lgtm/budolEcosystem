🚫 Scenario: Courier Agent Cancels Return Pickup

This can happen when:

  • Courier cannot locate the buyer address
  • Buyer is not available
  • Package is not ready / rejected
  • Area not serviceable
  • Rider error or capacity issue
  • Wrong label / damaged packaging

🔍 Core Rule Analysis

Shopee's Core Rule:

1
Courier can cancel the pickup - Controlled by 🚚 Courier
2
Courier CANNOT finalize or cancel the return case - Controlled by ✅ Shopee system
3
Shopee still decides what happens next - Buyer action required

🚨 Critical Implementation Gaps

⚠️
No Courier Cancellation Distinction
Current webhook handler treats all cancellations the same way, whether initiated by courier, buyer, or seller. Shopee's model requires different handling for courier cancellations.
⚠️
Missing Retry Mechanism
No infrastructure exists for allowing buyers to retry pickup booking after courier cancellation. This is a core requirement for Shopee-like behavior.
⚠️
No Buyer Action Workflow
Current system doesn't provide clear next steps for buyers when courier cancels pickup. Buyers need guidance on re-scheduling and alternative options.
⚠️
SLA Monitoring Gap
No tracking of buyer response time to courier cancellations. System cannot auto-cancel returns or release funds based on inactivity.
⚠️
Alternative Shipping Methods
No support for drop-off alternatives when pickup fails. This limits flexibility compared to Shopee's approach.

🎯 Key Recommendation

Implement a "Courier Cancellation Recovery" system that mirrors Shopee's approach:

  • Detect courier-specific cancellations in webhook handler
  • Create retry window with buyer action requirements
  • Provide clear re-scheduling workflow
  • Implement SLA monitoring with auto-resolution
  • Offer alternative shipping methods (drop-off)

🔄 Recommended Courier Cancellation Flow

1
Courier Cancellation Detected - Webhook receives courier-specific cancellation status
2
Return Status Updated - Set to "COURIER_CANCELLED" (not "CANCELLED")
3
Buyer Notification Sent - Email with re-scheduling options and deadline
4
Retry Window Activated - 48-72 hours for buyer to take action
5
Buyer Action Required - Re-book pickup or choose drop-off alternative
6
SLA Monitoring - Auto-cancel return and release funds if no action taken
