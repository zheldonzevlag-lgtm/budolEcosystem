🔍 Phase 1: Webhook & Cancellation Detection Tests

TC-001: Courier Cancellation Webhook Processing
Critical

Test Steps:

1
Create a return request with status "BOOKING_REQUESTED"
2
Send webhook with event "CANCELLED" and reason "DRIVER_UNAVAILABLE"
3
Verify webhook signature validation passes
4
Check return status is updated to "COURIER_CANCELLED"
5
Verify cancellation reason is stored in database
// Sample webhook payload { "data": { "order": { "orderId": "TEST_ORDER_123", "status": "CANCELLED", "cancellationReason": "DRIVER_UNAVAILABLE", "cancelledAt": "2025-12-30T10:00:00Z" } }, "timestamp": "2025-12-30T10:00:00Z", "signature": "test_signature_123" }
Expected Result: Return status changes to "COURIER_CANCELLED", cancellation reason stored, retry window expiry set to 48 hours from cancellation time.
TC-002: Cancellation Type Detection
High

🔄 Rollback Plan

Emergency Rollback Scenarios:

  • Webhook Processing Failures: Revert to original webhook handler, disable new cancellation logic
  • Database Issues: Rollback migration, restore previous schema
  • Email System Overload: Disable new email templates, revert to basic notifications
  • Performance Degradation: Disable analytics tracking, simplify retry logic
  • Complete System Failure: Feature flag to disable entire courier cancellation handling

Rollback Commands:

# Database rollback npx prisma migrate rollback --preview-feature # Feature flag disable (add to environment) COURIER_CANCELLATION_ENABLED=false # Revert webhook handler to backup cp app/api/webhooks/lalamove/route.js.backup app/api/webhooks/lalamove/route.js
