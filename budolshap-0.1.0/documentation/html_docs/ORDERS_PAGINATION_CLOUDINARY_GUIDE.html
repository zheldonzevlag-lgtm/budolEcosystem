Successfully resolved the PrismaClientKnownRequestError (P6009) and optimized the entire image storage system.

Problem Solved

Solution Implemented

  • Pagination: Implemented server-side pagination for all order endpoints
  • Cloudinary Integration: Migrated image storage from database to Cloudinary CDN
  • Data Migration: Converted all 13 existing products from base64 to Cloudinary URLs
  • Performance Optimization: Reduced response size by 99.5% (5MB+ → 24KB)

⚙️ Technical Implementation

1. Orders API Pagination

File: app/api/orders/route.js

Key Changes: Implemented server-side pagination with configurable limits and metadata response.

Features Implemented:

  • Accept page and limit query parameters
  • Default limit: 20 orders per page
  • Maximum limit: 50 orders per page
  • Return pagination metadata (total, page, limit, totalPages)
  • Optimized field selection for related data

Code Example:

// Parse pagination parameters
const page = parseInt(searchParams.get('page') || '1')
const limit = Math.min(parseInt(searchParams.get('limit') || '20') || 20, 50)
const skip = (page - 1) * limit

// Fetch orders and total count in a transaction
const [orders, total] = await prisma.$transaction([
    prisma.order.findMany({
        where,
        skip,
        take: limit,
        include: {
            user: { select: { id: true, name: true, email: true } },
            store: { select: { id: true, name: true } },
            address: {
                select: {
                    id: true, name: true, street: true,
                    city: true, state: true, zip: true,
                    country: true, phone: true
                }
            },
            orderItems: {
                include: {
                    product: {
                        select: {
                            id: true, name: true,
                            price: true, images: true
                        }
                    }
                }
            }
        },
        orderBy: { createdAt: 'desc' }
    }),
    prisma.order.count({ where })
])

// Return paginated response
return NextResponse.json({
    orders,
    pagination: {
        total,
        page,
        limit,
        totalPages: Math.ceil(total / limit)
    }
})

🔧 Troubleshooting

Common Issues & Solutions

Issue: Orders not loading / 500 error

Symptom: API returns 500 Internal Server Error

Solution:

  • Check if pagination limit is too high
  • Verify all images are Cloudinary URLs (not base64)
  • Run migration script if needed
  • Check Vercel logs for detailed error

Issue: Images not uploading to Cloudinary

Symptom: Upload fails or images still stored as base64

Solution:

  • Verify environment variables are set in Vercel
  • Check Cloudinary credentials are correct
  • Ensure /api/upload endpoint is working
  • Check browser console for errors

Issue: Pagination not showing

Symptom: No Previous/Next buttons appear

Solution:

  • Check if pagination.totalPages > 1
  • Verify API is returning pagination metadata
  • Check frontend state management
  • Clear browser cache and refresh

Issue: 404 on product view

Symptom: Clicking "View" shows 404 page

Solution:

  • Verify fix is deployed: href={`/product/${product.id}`}
  • Check if product detail page exists at /product/[productId]
  • Clear deployment cache and redeploy
