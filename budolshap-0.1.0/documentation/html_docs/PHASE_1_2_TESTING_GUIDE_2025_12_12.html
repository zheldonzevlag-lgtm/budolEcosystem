Test 2: Simulate Delivery Failure (Phase 1 Core)

⚠️ Important: This test requires access to Lalamove Sandbox or using the manual cancellation API.
1 Option A: Use Lalamove Sandbox Simulator
  • Login to Lalamove Sandbox Dashboard
  • Find the booking you just created
  • Trigger "CANCELLED" or "EXPIRED" status
2 Option B: Use Manual Cancellation API
POST /api/shipping/lalamove/cancel/[orderId] // This will: // 1. Cancel the Lalamove booking // 2. Revert order to PROCESSING // 3. Set failureReason = 'CANCELLED_BY_SELLER'
3 Verify Failed State in UI
  • Refresh the Store Orders page
  • Expected: Order shows ❌ Failed badge
  • Expected: "🔄 Rebook Delivery" button appears
4 Check Modal Details
  • Click the order to open modal
  • Expected: Red box showing "Delivery failed" with reason
  • Expected: "Previous Attempts" list visible
  • Expected: "Rebook Delivery" button in modal
5 Test Rebook
  • Click "Rebook Delivery" button
  • Wait for new booking confirmation
  • Expected: New booking ID generated
  • Expected: Old booking archived in previousAttempts
