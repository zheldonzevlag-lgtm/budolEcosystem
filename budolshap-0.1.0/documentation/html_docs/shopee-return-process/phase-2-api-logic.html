3. Fund Management (Escrow)

Funds are held in a lockedBalance state within the store's wallet during the return process. This prevents the seller from withdrawing funds while a dispute or return is active.

  • On Request: Funds moved to lockedBalance.
  • On Approval (Refund Only): Funds deducted from lockedBalance and returned to Buyer.
  • On Completion: Funds finalized based on the return outcome.

4. Real-time Notifications

Every status change triggers a Pusher/Socket event to both the Buyer and Seller to ensure UI synchronization without page refreshes.
