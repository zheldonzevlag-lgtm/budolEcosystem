Troubleshooting Guide

Common issues encountered during the implementation of the Shopee-style return process and their resolutions.

Database
1. Prisma Enum Mismatch (BOOKING_REQUESTED)

The ReturnStatus enum was updated in schema.prisma but the Prisma Client or the database itself was not in sync, causing validation errors during updates.

Resolution:
  1. Ensure all schema files (schema.prisma, schema.local.prisma, schema.mysql.prisma) are synchronized.
  2. Run npx prisma db push to update the database schema.
  3. Run npx prisma generate to update the client.

Note: If prisma generate fails with EPERM, stop the development server and try again.

Lalamove
5. Courier Cancellation (Re-booking)

If Lalamove cancels a return booking (e.g., no rider found), the return status must revert to a state where the seller can book again.

Resolution:

The webhook handler in route.js has been updated to revert CANCELLED return bookings to BOOKING_REQUESTED instead of APPROVED. This ensures the "Book Courier" button reappears on the Seller's Return Management page.

Lalamove
6. New Tracking Number Assigned

Sometimes Lalamove assigns a new internal order ID or tracking number after a re-booking or driver reassignment.

Resolution:

The webhook handler now captures incomingLalamoveOrderId from every event. If it detects a mismatch with the stored trackingNumber, it automatically updates the database to ensure tracking links and waybills remain valid.

Lalamove
2. Failed to book return pickup

When the buyer clicks "Request Pickup", the status changes to BOOKING_REQUESTED, but the seller sees an error when clicking "Book Courier".

Resolution:
  1. Check if the LALAMOVE_CLIENT_ID and SECRET are correctly set in .env.
  2. Ensure the Store's pickup address and Buyer's delivery address are within Lalamove's supported regions.
  3. Verify the Lalamove balance (for production) or use the Sandbox environment.
API
3. Escrow funds not releasing

The return is marked as RECEIVED but the lockedBalance in the Store Wallet is not moving.

Resolution:

The refund only processes when the seller explicitly clicks "Receive" in the Return Management portal. This triggers receiveReturn() which handles the final escrow transfer.

// Ensure this function is called:
await refundFromLocked({
    walletId: store.wallet.id,
    amount: returnRequest.refundAmount,
    description: `Refund for Return #${returnRequest.id}`
});
API
4. Webhook status not updating Return

Lalamove delivery is completed but the Return status remains SHIPPED.

Resolution:

Check the isReturnBooking logic in app/api/shipping/lalamove/webhook/route.js. It relies on metadata being passed during booking:

const isReturnBooking = (data.metadata?.type === 'return') || 
                       (activeReturn && activeReturn.trackingNumber === lalamoveOrderId);
