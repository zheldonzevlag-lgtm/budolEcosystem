Phase 4: Frontend UI & UX

This phase focuses on the user interfaces for both buyers and sellers to manage return requests seamlessly.

1. Buyer Interface (Order Details)

Buyers need a clear way to initiate returns and track the progress of their requests.

Return Request Modal:
  • Selection of return type (Refund Only vs. Return & Refund).
  • Evidence upload for damaged or missing items.
  • Real-time status tracking via the OrderProgressTracker.
  • Return Summary Card: Displays refund amounts and escrow status directly in the payment section.

2. Seller Interface (Return Management)

Sellers require a dedicated dashboard to process incoming return requests and book couriers.

  • To Book Tab: Displays returns in BOOKING_REQUESTED status.
  • Book Return Button: Triggers the Lalamove booking API.
  • Receive & Refund Button: Finalizes the return once the item is received.

4. UX Considerations

  • Deadlines: Display countdown timers for seller responses and buyer shipments.
  • Transparency: Provide Lalamove share links for live tracking of return parcels.
  • Clarity: Use clear badges for Escrow status (Locked vs. Released).
