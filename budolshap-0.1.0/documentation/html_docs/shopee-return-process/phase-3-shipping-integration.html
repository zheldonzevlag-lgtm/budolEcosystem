3. Webhook Synchronization & Re-booking

The Lalamove webhook (/api/shipping/lalamove/webhook) is enhanced to handle both status updates and dynamic changes like new tracking numbers or cancellations that require re-booking.

Status Mapping

Lalamove Status Return Status Effect
PICKED_UP SHIPPED Notifies Seller that item is on the way
COMPLETED RECEIVED Starts the verification window for Seller
CANCELLED BOOKING_REQUESTED Allows Seller to re-book a new courier

Dynamic Tracking Number Capture

If Lalamove assigns a new tracking number (e.g., during driver re-assignment or manual intervention), the webhook captures the lalamoveOrderId from the payload and automatically updates the Return record's trackingNumber and bookingId. This ensures the tracking link remains valid for both Buyer and Seller.

Re-booking Workflow

When a Lalamove order is cancelled (e.g., no rider found), the system automatically reverts the return status to BOOKING_REQUESTED. This triggers the following:

  • The "Book Courier" button reappears on the Seller's dashboard.
  • The previous failed tracking number is preserved in history but replaced by the new one upon re-booking.
  • Notifications are sent to the Seller to prompt action.

4. Shipping Cost Handling

The platform determines who pays for return shipping based on the return reason:

  • Seller Fault (Damaged, Wrong Item): Seller pays (deducted from refund).
  • Buyer Fault (Change of Mind): Buyer pays (deducted from refund).
  • BudolShap Guarantee: Platform may subsidize costs.
