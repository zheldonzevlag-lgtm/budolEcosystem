BudolShap Lalamove Shipping Integration – TODO List

Core Architecture

  • Create services/lalamove.js – wrapper for Lalamove API (auth, quote, order, tracking, cancel).
  • Create generic services/shippingProvider.js interface to allow additional carriers.
  • Update Prisma Order model with shipping JSON field (provider, quoteId, bookingId, status, trackingUrl).

Frontend Checkout Flow

  • Add “Lalamove” option to the shipping selector on the checkout page.
  • Implement quote modal: collect address, package dimensions, preferred pickup time.
  • Call /api/shipping/lalamove/quote and display price & ETA.
  • On confirmation, call /api/shipping/lalamove/book and store booking ID.
  • Show order summary with tracking link and cancel button (if allowed).

Backend Endpoints

  • GET /api/shipping/lalamove/quote – returns real‑time quote.
  • POST /api/shipping/lalamove/book – creates Lalamove order.
  • GET /api/shipping/lalamove/track/:orderId – fetches current status.
  • POST /api/shipping/lalamove/cancel/:orderId – cancels order (if within window).
  • POST /api/webhooks/lalamove – receives status updates, validates signature, updates DB.

Additional Carriers (Future)

  • Implement services/jnt.js, services/2go.js, etc., following the same interface.
  • Add provider selector UI that dynamically loads carrier‑specific fields.
  • Update webhook router to handle multiple provider callbacks.

Testing & Deployment

  • Configure Lalamove sandbox credentials in .env.
  • Write integration tests for quote, booking, and webhook handling.
  • Run end‑to‑end flow on local dev server.
  • Deploy to Vercel and verify environment variables are set.
