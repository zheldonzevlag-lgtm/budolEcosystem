Phase 3: Delivery Proof & Archiving

Implementation Status: COMPLETED

This phase focused on accountability and auditability by ensuring all "Non-Happy Path" events (returns/disputes) are backed by evidence and all delivery data is archived.

1. Evidence Submission (Returns)

A new Evidence Modal has been implemented for buyers requesting returns or refunds.

  • Photo Uploads: Supports up to 5 high-resolution images.
  • Live Previews: Buyers can see images before submitting.
  • Reason Mapping: Standardized reasons (e.g., "Missing items", "Damaged") to streamline seller review.

2. Driver Information & Live Tracking

For integrated couriers like Lalamove, real-time driver data and location are integrated directly into the Buyer Order View:

  • Driver Info Card: Displays name, contact details, and vehicle information. During assignment, it shows "Waiting for Driver Assignment" as seen in the system UI.
  • Live GPS Tracking: An embedded map provides real-time location tracking for active deliveries via the courier's tracking URL.
  • Persistence: Driver name, phone, vehicle type, and plate number are saved to the shipping JSON object.
  • UI Integration: Vehicle icons (🏍️, 🚗, 🚐) in the progress bar update based on the actual driver's vehicle.
  • Archiving: Even if a delivery fails or is re-booked, the previous driver's info and tracking links are kept in previousAttempts for auditability.

3. Seller Review Portal

Sellers have a dedicated "Return & Refund" management interface:

  • Inspection Modal: Clickable image thumbnails that open in a full-screen viewer for detailed inspection of buyer evidence.
  • Decision Tools: One-click "Accept & Refund" or "Dispute" buttons that trigger automated wallet and status updates.

4. Success Criteria Validation

  • ✅ Buyers can upload multiple photos during the return request process.
  • ✅ Sellers can view and expand these photos in the Return Management dashboard.
  • ✅ Driver details are correctly displayed and archived in the order history.
  • ✅ All refund transactions are logged in the Wallet Audit Trail (DEBIT/CREDIT).
