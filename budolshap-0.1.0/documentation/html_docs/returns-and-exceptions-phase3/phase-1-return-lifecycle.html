Phase 1: Return Lifecycle Integration

Implementation Status: COMPLETED

The return lifecycle is now fully integrated into the OrderProgressTracker.jsx component and the backend returnsService.js.

1. Status Mapping Logic

When an order enters a return state, the 5-stage tracking bar pivots from the "Shipping" lifecycle to the "Return" lifecycle.

Order Status UI Stage Label Icon Description
RETURN_REQUESTED Return Requested ↩️ Waiting for seller review
RETURN_APPROVED Return Approved ✅ Seller accepted; waiting for parcel return
RETURN_DISPUTED Under Dispute ⚠️ Seller declined; Admin intervention needed
REFUNDED Refunded 💰 Funds returned to Buyer wallet / Deducted from Seller

2. Technical Implementation & Shopee-Standard Business Rules

The system distinguishes between Refund Only (e.g., item damaged, no return needed) and Return & Refund (item must be shipped back). This logic is modeled directly after Shopee's standard operating procedures:

Rule 1: The 72-Hour Response Window

Per Shopee standards, sellers have a 3-day (72-hour) deadline to respond to a return request. If the seller fails to respond, the system is designed to favor the buyer (Escrow Protection).

Logic: deadline: new Date(Date.now() + (3 * 24 * 60 * 60 * 1000))

Rule 2: Path A - Refund Only (Immediate Escrow Release)

If the buyer requests "Refund Only" and the seller clicks Accept, the escrow funds are immediately debited from the seller's wallet and returned to the buyer. This bypasses return shipping to save costs for both parties.

Rule 3: Path B - Return & Refund (The "Parcel-First" Release)

If "Return & Refund" is selected, the refund is locked until the seller confirms receipt of the physical item. Once the seller clicks Parcel Received, the system triggers the wallet debit and finalizes the refund.

Rule 4: The Shopee Guarantee Extension

Buyers can click "Extend Shopee Guarantee" once per order, which adds 3 additional days to the return request window. This is used when a buyer hasn't had time to test the product yet.

Rule 5: Return Shipping Fee (RSF) Responsibility

In a standard Shopee-style flow, if a return is approved due to "Seller Fault" (Damaged/Wrong Item), the Seller pays for return shipping. If the return is "Change of Mind" (if allowed), the Buyer pays.

Rule 6: NFR (Non-Fulfilment Rate) Impact

Any return approved under "Seller Fault" automatically contributes to the seller's NFR. High NFR leads to penalty points, mirroring Shopee's seller performance management.

The OrderProgressTracker.jsx dynamically switches activeStages based on the order status and path:

// From OrderProgressTracker.jsx
const isReturnRelated = ['RETURN_REQUESTED', 'RETURN_APPROVED', 'RETURN_DISPUTED', 'REFUNDED'].includes(order.status);

const returnStages = [
    { status: 'ORDER_PLACED', label: 'Order Placed', icon: '📝' },
    { status: 'DELIVERED', label: 'Delivered', icon: '📦' },
    { status: 'RETURN_REQUESTED', label: 'Requested', icon: '↩️' },
    { status: 'RETURN_APPROVED', label: 'Review', icon: '⏳' },
    { status: 'REFUNDED', label: 'Refunded', icon: '💰' }
];

const activeStages = isReturnRelated ? returnStages : stages;

3. Backend Service

The lib/services/returnsService.js handles the state transitions and wallet transactions:

  • createReturnRequest: Initiates the return and sets status to RETURN_REQUESTED.
  • respondToReturn: Allows sellers to ACCEPT, REJECT, or REQUEST_RETURN.
  • receiveReturn: Finalizes the process and triggers the REFUNDED status.
