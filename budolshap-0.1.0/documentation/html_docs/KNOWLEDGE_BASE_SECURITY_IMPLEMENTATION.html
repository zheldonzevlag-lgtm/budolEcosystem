🔒 Knowledge Base Security Implementation

⚠️ Restricted Documentation

This documentation contains sensitive implementation details about the security measures used in the knowledge base system. Access to this document is password-protected for security reasons.

Overview

This document details the implementation of security measures for the BudolShap Knowledge Base access system, specifically focusing on the obfuscation/hashing of sensitive values including email addresses and password expiry configuration.

🎯 Implementation Goals

Security Objectives

  • Obfuscate Sensitive Data: Prevent plain text visibility of email addresses and configuration values in source code
  • Maintain Functionality: Ensure system continues to work while values are obfuscated
  • Environment Variable Support: Allow override via environment variables for production deployments
  • Developer Privacy: Protect sensitive information from being visible to developers viewing the codebase

🔐 Email Address Obfuscation

Implementation Details

Location: app/api/knowledge-base/auth/route.js (lines 30-47)

Function: getTargetEmail()

// Get target email - prefer environment variable, otherwise decode obfuscated value function getTargetEmail() { // Check environment variable first (most secure) if (process.env.KB_TARGET_EMAIL) { return process.env.KB_TARGET_EMAIL; } // Obfuscated email (base64 encoded) - not visible in plain text // Decode: Buffer.from('cmV5bmFsZG9tZ2FsdmV6QGdtYWlsLmNvbQ==', 'base64').toString() const obfuscatedEmail = 'cmV5bmFsZG9tZ2FsdmV6QGdtYWlsLmNvbQ=='; try { return Buffer.from(obfuscatedEmail, 'base64').toString('utf-8'); } catch (error) { console.error('Error decoding target email:', error); return null; } } const TARGET_EMAIL = getTargetEmail();

📝 How It Works

  1. Environment Variable Check: First checks for KB_TARGET_EMAIL environment variable (most secure for production)
  2. Base64 Obfuscation: If env var not set, uses base64-encoded string: 'cmV5bmFsZG9tZ2FsdmV6QGdtYWlsLmNvbQ=='
  3. Runtime Decoding: Decodes the base64 string using Node.js Buffer API at runtime
  4. Error Handling: Returns null if decoding fails, with error logging

🔍 Decoding Process

The obfuscated email can be decoded using:

// Node.js/Browser console: Buffer.from('cmV5bmFsZG9tZ2FsdmV6QGdtYWlsLmNvbQ==', 'base64').toString('utf-8') // Result: 'reynaldomgalvez@gmail.com'

Note: While base64 encoding is not encryption, it prevents plain text visibility in source code, which is sufficient for this use case.

⏱️ Password Expiry Obfuscation

Implementation Details

Location: app/api/knowledge-base/auth/route.js (lines 8-28)

Function: getPasswordExpiry()

// Get password expiry - prefer environment variable, otherwise decode obfuscated value function getPasswordExpiry() { // Check environment variable first (most secure) if (process.env.KB_PASSWORD_EXPIRY) { return parseInt(process.env.KB_PASSWORD_EXPIRY, 10); } // Obfuscated expiry value (base64 encoded) - 10 minutes in milliseconds (600000) // Decode: parseInt(Buffer.from('NjAwMDAw', 'base64').toString('utf-8'), 10) const obfuscatedExpiry = 'NjAwMDAw'; try { const decodedValue = Buffer.from(obfuscatedExpiry, 'base64').toString('utf-8'); return parseInt(decodedValue, 10); } catch (error) { console.error('Error decoding password expiry:', error); // Fallback to default 10 minutes return 10 * 60 * 1000; } } const PASSWORD_EXPIRY = getPasswordExpiry();

📝 How It Works

  1. Environment Variable Check: First checks for KB_PASSWORD_EXPIRY environment variable
  2. Base64 Obfuscation: If env var not set, uses base64-encoded string: 'NjAwMDAw'
  3. Runtime Decoding: Decodes base64 string to get '600000'
  4. Type Conversion: Parses the decoded string to integer (600000 milliseconds = 10 minutes)
  5. Fallback: Returns default value (10 minutes) if decoding fails

🔍 Value Details

  • Obfuscated Value: 'NjAwMDAw' (base64)
  • Decoded Value: '600000' (string)
  • Final Value: 600000 (integer, milliseconds)
  • Meaning: 10 minutes in milliseconds (10 * 60 * 1000)

🔧 Environment Variable Configuration

Production Setup (Recommended)

For production deployments, use environment variables instead of hardcoded obfuscated values:

# Add to .env file or deployment environment variables: # Target email for knowledge base access codes KB_TARGET_EMAIL=your-email@example.com # Password expiry in milliseconds (optional, defaults to 600000 = 10 minutes) KB_PASSWORD_EXPIRY=600000

💡 Benefits of Environment Variables

  • No Source Code Exposure: Sensitive values never appear in codebase
  • Environment-Specific: Different values for dev/staging/production
  • Easy Updates: Change values without code modifications
  • CI/CD Friendly: Can be set in deployment pipelines

📋 Code Flow

Execution Flow

  1. Module Load: When the API route file is loaded, both functions execute
  2. getPasswordExpiry(): Called first, sets PASSWORD_EXPIRY constant
  3. getTargetEmail(): Called second, sets TARGET_EMAIL constant
  4. Constants Available: Both values are available throughout the module
  5. Runtime Usage: Values are used when processing authentication requests

🛡️ Security Considerations

Current Implementation

  • Obfuscation Level: Base64 encoding (not encryption)
  • Purpose: Prevent plain text visibility in source code
  • Limitation: Base64 can be easily decoded if someone knows where to look
  • Recommendation: Use environment variables in production for true security

Security Best Practices

  • ✅ Use environment variables for production deployments
  • ✅ Never commit sensitive values to version control
  • ✅ Rotate values periodically in production
  • ✅ Use secret management services (AWS Secrets Manager, Vercel Environment Variables, etc.)
  • ⚠️ Base64 encoding is obfuscation, not encryption - treat accordingly

📝 Additional Notes

Implementation Notes

  • Base64 Encoding: Used because it's simple, reversible, and sufficient for obfuscation
  • Error Handling: Both functions include try-catch blocks for robust error handling
  • Fallback Values: Password expiry has a fallback to default (10 minutes) if decoding fails
  • Email Validation: Email function returns null if decoding fails, which triggers validation in the API handler
  • Performance: Decoding happens once at module load, not on every request

⚠️ Important Reminders

  • Base64 is encoding, not encryption - it obfuscates but doesn't secure
  • For production, always use environment variables
  • Never commit actual email addresses or sensitive configs to version control
  • Consider using proper secret management for sensitive data
