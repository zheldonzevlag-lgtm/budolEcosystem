Problem Description

Error Message

HTTP Status: 400 Bad Request

Affected Order: cmikk91l7000411sr5967dqui

Root Cause Analysis

The booking failure was caused by an invalid phone number format in the store contact details.

Investigation Findings

Field Value Issue
Store Name Online Micro Sellers Multipurpose Cooperative ✓ Valid (45 chars, within limit)
Store Contact 01234567899 ❌ Invalid (dummy number)
Formatted Phone +6301234567899 ❌ Invalid PH format
Latitude null ⚠️ Missing (geocoding failed)
Longitude null ⚠️ Missing (geocoding failed)

Why Quote Succeeded but Booking Failed

The Lalamove API has different validation requirements for quotes vs. bookings:

  • Quote Request: Only requires coordinates and address (no contact details)
  • Booking Request: Requires valid contact names and phone numbers

This is why the user was able to get a quote but failed when attempting to book the delivery.

The Fix

Enhanced Phone Validation

Updated the formatPhone function in app/api/shipping/lalamove/book/route.js to:

  1. Detect Invalid Patterns: Numbers starting with 0 but not 09 (e.g., "01234567899")
  2. Validate Format: Check against PH mobile pattern +639xxxxxxxxx or landline pattern +632xxxxxxxx
  3. Use Fallback: If validation fails, use a default valid number +639170000000
  4. Add Logging: Warn when fallback is used for debugging

Code Changes

File: app/api/shipping/lalamove/book/route.js

const formatPhone = (phone, fallback = '+639170000000') => {
    if (!phone) return fallback;
    
    let cleaned = phone.replace(/[\s\-\(\)]/g, '');
    let formatted;
    
    if (cleaned.startsWith('09')) {
        formatted = '+63' + cleaned.substring(1);
    } else if (cleaned.startsWith('0') && !cleaned.startsWith('09')) {
        // Invalid dummy numbers like "01234567899"
        console.warn(`[Phone Format] Invalid phone: ${phone}, using fallback`);
        return fallback;
    }
    // ... other format handling ...
    
    // Validate final format
    const mobilePattern = /^\+639\d{9}$/;
    const landlinePattern = /^\+632\d{7,8}$/;
    
    if (!mobilePattern.test(formatted) && !landlinePattern.test(formatted)) {
        console.warn(`[Phone Format] Invalid format: ${formatted}, using fallback`);
        return fallback;
    }
    
    return formatted;
};

File: services/lalamove.js
