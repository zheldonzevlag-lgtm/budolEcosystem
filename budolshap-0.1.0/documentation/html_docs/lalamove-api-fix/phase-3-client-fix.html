Phase 3: Client-side Handling & Logging

Status: Completed

Objective

Improve the LalamoveWidget component to handle the new structured error responses and provide better debugging information in the console.

Implementation Details

Modified components/LalamoveWidget.jsx in the retryWithBackoff function:

if (!response.ok) {
    console.error('[Lalamove Quote API Error Response]', {
        status: response.status,
        statusText: response.statusText,
        data: data
    });
    
    if (response.status === 408 || response.status === 504) {
        throw new Error("Request timed out. Please try again.");
    } else if (response.status === 503) {
        throw new Error("Lalamove service temporarily unavailable.");
    } else {
        // Extract error message from various possible formats
        const errorMessage = data?.error || data?.message || (typeof data === 'string' ? data : null) || 'Failed to get delivery quote';
        throw new Error(errorMessage);
    }
}

Key Improvements

  • Contextual Logging: We now log the HTTP status and status text alongside the data, which helps identify infrastructure issues (like 502/504) vs. application errors.
  • Flexible Error Extraction: The logic now safely handles cases where the error might be a simple string, a { error: ... } object, or a { message: ... } object.
  • User Feedback: Standardized error messages are presented to the user via the setError state, with a "Try Again" option.
