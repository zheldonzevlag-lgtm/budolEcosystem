Implementation Summary

Lalamove API Error Reporting Fix

Project Overview

Successfully addressed the critical issue of empty error objects {} being reported in the Lalamove Quote API integration. The solution involved enhancing error serialization on the server-side and improving logging/handling on the client-side.

Technical Decisions

Decision Rationale
ESM Conversion Converted service classes from CommonJS to ES Modules to fix inheritance issues in a type: module project.
Explicit Property Extraction Ensures Error object properties (which are non-enumerable) are captured in JSON.
Structured Error Response Provides a consistent contract between frontend and backend for all shipping providers.
Status-based Logic Distinguishes between client errors, timeouts, and upstream server failures.

Impact

  • Reduced Debugging Time: Developers now see exact error messages from the provider.
  • Improved UX: Users receive more specific information about why a quote failed (e.g., "Service temporarily unavailable").
  • Increased Reliability: Better retry logic and timeout handling for unstable network conditions.
