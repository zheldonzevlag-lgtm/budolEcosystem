    5.2 handleLogout Function

    The logout function performs the following operations in sequence:

    const handleLogout = async () => {
  // 1. Sync cart to server if user is logged in
  if (user && cartItems) {
    try {
      await fetch('/api/cart', {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ userId: user.id, cart: cartItems }),
        keepalive: true
      });
    } catch (error) {
      console.error("Failed to sync cart:", error);
    }
  }

  // 2. Clear Redux cart state
  dispatch(clearCart());

  // 3. Remove cart from localStorage
  localStorage.removeItem('budolshap_cart');

  // 4. Execute logout (clears auth context)
  logout();

  // 5. Close user menu
  setShowUserMenu(false);
}

    8. Troubleshooting

    Common Issues and Solutions

    Issue: Navbar not rendering correctly

    • Cause: JSX syntax errors or missing closing tags
    • Solution: Check browser console for errors, verify all JSX tags are properly closed

    Issue: Links not working

    • Cause: Next.js Link component not imported or href attribute missing
    • Solution: Ensure import Link from "next/link" is present and all Link components have href attributes

    Issue: Logout not syncing cart

    • Cause: API endpoint not responding or authentication token missing
    • Solution: Check network tab in DevTools, verify /api/cart endpoint is accessible and returns 200 status

    Issue: Git script fails

    • Cause: PowerShell execution policy or git not configured
    • Solution: Run Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser and ensure git is installed

    10. Git Automation Script Usage

    Script Parameters

    • -BranchName: Name of the new branch to create (required)
    • -CommitMessage: Commit message for the changes (required)

    Example Usage

    # Create a feature branch
.\create_git_branch.ps1 -BranchName "feature/new-feature" -CommitMessage "Add new feature"

# Create a bugfix branch
.\create_git_branch.ps1 -BranchName "fix/bug-description" -CommitMessage "Fix bug description"

# Create a hotfix branch
.\create_git_branch.ps1 -BranchName "hotfix/critical-fix" -CommitMessage "Critical fix for production"

    Script Workflow

    1. Creates a new branch from the current branch
    2. Switches to the new branch
    3. Stages all changes (git add .)
    4. Commits with the provided message
    5. Pushes to origin and sets upstream tracking
