1. System Architecture

Key Concept: Virtual Escrow

The platform uses a single PayMongo account to collect all payments. Seller earnings are tracked in a virtual ledger (database) with two states: pendingBalance (held in escrow) and availableBalance (ready for payout).

    A[Customer Payment] -->|Instant| B[Platform Account]

2. Key Stakeholders

👤 Buyer

Protected by platform escrow. Money held until delivery confirmed.

Refund window: Up to 7 days post-delivery

🏪 Seller

Earns money → Pending → Available → Payout.

Payout processing: 1-3 business days

⚙️ Platform Admin

Monitors escrow, approves payouts, handles disputes.

Review time: Within 24 hours

export async function POST(request, { params }) {
  const { orderId } = params
  
  // Calculate net earnings (total - platform fee)
  const platformFee = order.total * 0.05 // 5% fee
  const netEarnings = order.total - platformFee
  
  // Move funds: Pending → Available
  await prisma.wallet.update({
    where: { userId: order.store.userId },
    data: {
      pendingBalance: { decrement: netEarnings },
      availableBalance: { increment: netEarnings }
    }
  })
  
  return NextResponse.json({ success: true })
}

export async function GET(request) {
  // Aggregate all wallet balances
  const walletStats = await prisma.wallet.aggregate({
    _sum: {
      pendingBalance: true,
      availableBalance: true
    }
  })
  
  // Get pending payout requests
  const pendingPayouts = await prisma.payoutRequest.findMany({
    where: { status: 'PENDING' },
    include: { user: true },
    orderBy: { requestedAt: 'asc' }
  })
  
  return NextResponse.json({
    totalPending: walletStats._sum.pendingBalance,
    totalAvailable: walletStats._sum.availableBalance,
    pendingPayouts
  })
}

7. Payment Gateway Agnosticism

🔌 Why This System is 100% Gateway-Agnostic

The escrow logic is completely decoupled from any specific payment gateway. Whether you use PayMongo, Xendit, Stripe, or any other provider, the core business logic remains unchanged.

Architecture: Separation of Concerns

graph TD
    A[Payment Gateway Layer] --> B[Webhook Handler]
    B --> C[Shared Escrow Service]
    C --> D[Database Wallet]
    
    E[PayMongo] -.->|webhook| B
    F[Xendit] -.->|webhook| B
    G[Stripe] -.->|webhook| B

Example: Adding Xendit (30 minutes)

Step 1: Create Shared Escrow Service

File: lib/escrow.js (Gateway-agnostic) 

export async function creditPendingBalance({ orderId, amount, gateway }) {
  const order = await prisma.order.findUnique({ 
    where: { id: orderId },
    include: { store: true }
  })
  
  const platformFee = amount * 0.05
  const netEarnings = amount - platformFee
  
  // Update wallet (same for ALL gateways)
  await prisma.wallet.update({
    where: { userId: order.store.userId },
    data: { pendingBalance: { increment: netEarnings } }
  })
  
  // Create transaction record
  await prisma.transaction.create({
    data: {
      type: 'ESCROW_HOLD',
      amount: netEarnings,
      gateway, // Track which gateway was used
      orderId
    }
  })
}

Step 2: PayMongo Webhook (Existing)
