Phase 1: Analysis & Planning

Current State Analysis

The MapPicker component used fixed sizes for map controls and the central pin icon, which felt oversized on mobile devices compared to standard apps like Shopee.

Component Current Size (Desktop/Mobile) Target Size (Mobile) Rationale
Center Pin Icon h-12 w-12, White Fill h-8 w-8, Pulse Reduced size and removed fill to create a lightweight pin that doesn't obstruct the map.
Address Overlay Floating Card Removed Obstructed map view. Address info now syncs directly to the search input above the map.
Zoom Buttons p-3, Icon: 20px Hidden Pinch-to-zoom is standard on mobile; removing buttons maximizes map visibility.
Direction Button p-4, Solid Green p-3, Pure Green Outline Removed background and shadow to create a pure outline style, maximizing map visibility.

Technical Strategy

  • Utilize Tailwind CSS responsive prefixes (md:) to differentiate mobile and desktop sizes.
  • Adjust absolute positioning offsets for the pin icon to ensure the "point" remains centered on the map regardless of size.
  • Maintain accessibility by ensuring tap targets remain reasonably large (min 40x40px for buttons).
