Phase 2: Refactoring & Fix

Implementation of robust state management and side-effect handling.

Key Changes

1. Decoupling Side Effects from State Updaters

The toggleProvider function was refactored to perform checks and side effects before calling the state updater:

const toggleProvider = (id) => {
    const isEnabled = settings.enabledMapProviders.includes(id);
    if (isEnabled && settings.enabledMapProviders.length === 1) {
        toast.error("At least one provider must be enabled.");
        return; // Exit early
    }
    setSettings(prev => ({ ...prev, ...newData }));
}

2. Safer Effect Hooks

Added isMounted tracking to useEffect to prevent state updates on unmounted components and handle race conditions during data fetching.

useEffect(() => {
    let isMounted = true;
    fetch('/api/...').then(res => {
        if (isMounted) {
            setSettings(data);
        }
    });
    return () => { isMounted = false; };
}, []);

3. Structural Optimizations

  • Moved static data (MAP_PROVIDERS) outside the component to prevent unnecessary object re-creation on every render.
  • Implemented proper error handling in fetch calls with res.ok checks.
  • Switched all form input onChange handlers to use functional state updaters (prev => ...) for better reliability.
