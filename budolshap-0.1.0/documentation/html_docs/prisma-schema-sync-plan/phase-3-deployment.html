Step 1: Generate Prisma Client

After synchronizing the schema files, the Prisma client must be regenerated to include the new fields in the TypeScript/JavaScript types.

npx prisma generate --schema=prisma/schema.local.prisma

Note: If multiple schemas are used, ensure you generate from the one active in your environment.

Step 2: Push Changes to Database

Update the local database schema to match the updated schema.local.prisma.

npx prisma db push --schema=prisma/schema.local.prisma

Alternatively, if using migrations:

npx prisma migrate dev --name sync_return_model --schema=prisma/schema.local.prisma
