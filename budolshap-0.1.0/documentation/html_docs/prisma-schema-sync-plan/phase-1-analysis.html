Root Cause

The development workflow likely involved adding features that required schema changes, which were only applied to the main schema.prisma. Since the local development environment is configured to use schema.local.prisma (as indicated by the "Using Database (Local Development)" log), the generated Prisma client does not recognize the new fields.
