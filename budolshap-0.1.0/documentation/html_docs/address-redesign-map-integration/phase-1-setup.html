Phase 1: Infrastructure & Setup

The foundation of the redesign involves ensuring all map providers are correctly configured and accessible to the frontend components.

1. Map Provider Settings

We will utilize the existing SystemSettings to determine the active map provider. This ensures consistency across the platform.

// Current supported providers:
// - OSM (OpenStreetMap)
// - GEOAPIFY
// - GOOGLE
// - RADAR

2. Library Dependencies

The project already has the necessary libraries installed:

  • leaflet: Core mapping library.
  • react-leaflet: React wrapper for Leaflet.
  • lucide-react: For UI icons.

3. Entry Criteria

Existing map settings in database.
API routes for system settings (/api/system/settings).
Leaflet libraries installed.

4. Implementation Steps

  • Create a shared useMapSettings hook to easily fetch active provider and API keys.
  • Verify API keys for Google Maps and Geoapify in the environment.
  • Establish the data structure for storing coordinates (latitude/longitude) in the Address model.
