Phase 1: Multi-Party Shipping Absorption

Implementing the core logic for platform vouchers, seller subsidies, and transparent cost breakdowns.

Executive Summary

This phase focused on the "Absorption Logic" where shipping costs are split between three parties: the Platform (via Vouchers), the Seller (via Subsidies), and the Buyer (Net Fee). We implemented calculation safeguards to prevent negative shipping costs and created hierarchical UI components for total transparency.

Technical Specifications

1. Calculation Engine

The core calculation is handled in OrderSummary.jsx (Frontend) and ordersService.js (Backend):

netShipping = Math.max(0, rawShippingCost - sellerSubsidy - platformVoucher);
totalAmount = subtotal + netShipping;

2. Data Storage

The Order model in Prisma was updated to store the netShipping value in the shippingCost field, ensuring financial records match actual payments.

3. UI Components Updated

  • OrderSummary.jsx: Added dynamic discount application and hierarchical breakdown.
  • return/page.jsx: Added post-payment transparency for buyers.
  • store/orders/page.jsx: Added detailed breakdown for store owners to see their subsidy contributions.
  • OrderItem.jsx: Enhanced buyer order list with a detailed tooltip breakdown.

Risk Assessment & Mitigation

  • Risk: Negative shipping costs leading to payment errors.
    Mitigation: Implemented Math.max(0, ...) wrapper on all shipping calculations.
  • Risk: UI confusion regarding "Free Shipping".
    Mitigation: Hierarchical breakdown shows exactly how the discount was applied (e.g., "↳ Seller Subsidy").
