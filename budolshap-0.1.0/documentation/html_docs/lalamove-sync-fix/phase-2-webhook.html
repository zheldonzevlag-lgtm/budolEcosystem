Phase 2: Robust Webhook Implementation

The webhook handler in app/api/webhooks/lalamove/route.js was updated to include security verification and enhanced logging.

HMAC Signature Verification

We implemented HMAC SHA256 verification to ensure that incoming requests are genuinely from Lalamove. 

// Signature Verification Logic
const lalamove = getShippingProvider('lalamove');
const isValid = lalamove.verifyWebhookSignature(rawBody, signature);

if (!isValid) {
    console.error('❌ Invalid Lalamove webhook signature');
    return NextResponse.json({ error: 'Invalid signature' }, { status: 401 });
}

State Machine Updates

The webhook now correctly updates the shipping JSON field in the database, capturing driver info, location updates, and delivery timelines.

  • Updates order.status based on Lalamove events.
  • Archives failed booking attempts in shipping.previousAttempts.
  • Triggers realtime events via triggerRealtimeEvent.
