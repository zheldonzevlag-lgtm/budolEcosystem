Phase 3: UI Auto-Sync & Realtime Updates

To solve the "stuck" UI issue, we implemented client-side auto-synchronization for both Seller and Buyer interfaces.

Seller UI Auto-Sync

The Store Orders page now identifies active Lalamove orders and polls the synchronization endpoint every 15 seconds. 

useEffect(() => {
    const activeLalamoveOrders = orders.filter(order => 
        order.shipping?.provider === 'lalamove' && 
        ['PROCESSING', 'SHIPPED', 'IN_TRANSIT'].includes(order.status)
    );

    const interval = setInterval(async () => {
        for (const order of activeLalamoveOrders) {
            await fetch(`/api/orders/${order.id}/sync-lalamove`, { method: 'POST' });
        }
        mutate();
    }, 15000);

    return () => clearInterval(interval);
}, [orders]);

Buyer UI Auto-Sync

Similar logic was added to the Buyer's Order List page, ensuring that status updates are visible without requiring the user to navigate to the order detail page.

Realtime Propogation

  • SWR Mutation: Calling mutate() after sync ensures the UI reflects database changes immediately.
  • Pusher/Socket.io: The server triggers events on store-{id} and user-{id} channels after every sync or webhook update.
