Phase 1: Analysis & Status Mapping

The first phase involved analyzing how Lalamove statuses correspond to Budolshap's internal order statuses. We identified that the "stuck" UI was caused by a lack of synchronization triggers on the Seller side.

Status Mapping Table

Lalamove Event/Status Budolshap Order Status Description
DRIVER_ASSIGNED / ON_GOING PROCESSING Driver has been assigned to the order.
PICKED_UP IN_TRANSIT Package is with the driver.
COMPLETED DELIVERED Order successfully delivered.
CANCELLED / EXPIRED PROCESSING Booking failed; order returns to processing for re-booking.

Root Cause Identification

  • Webhook Reliability: Lack of signature verification made webhooks potentially insecure or ignored.
  • UI Dependency: Seller UI was passive and only updated if a manual sync was triggered or the Buyer UI detail page was open.
  • Realtime Propagation: While Pusher was implemented, it required a database update to trigger.
