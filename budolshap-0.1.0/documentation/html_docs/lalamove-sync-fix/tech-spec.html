Technical Specification: Lalamove Status Synchronization

Overview

This document specifies the synchronization logic between Lalamove (external provider) and Budolshap (internal system). The goal is to maintain high consistency across the Seller UI, Buyer UI, and backend database.

System Architecture

The synchronization relies on three primary components:

  1. Push (Webhook): Lalamove sends real-time updates to /api/webhooks/lalamove.
  2. Pull (Auto-Sync): Seller and Buyer UIs poll /api/orders/[id]/sync-lalamove every 15 seconds for active orders.
  3. Realtime (Pusher): Server triggers Pusher events to update the UI immediately upon status change.

Status State Machine

The following mapping is enforced during synchronization:

Lalamove Status Budolshap Order Status Realtime Channel Triggered
ASSIGNING_DRIVER PROCESSING store-{id}, user-{id}
PICKED_UP IN_TRANSIT store-{id}, user-{id}
COMPLETED DELIVERED store-{id}, user-{id}
CANCELED PROCESSING (Reset) store-{id}, user-{id}

Data Persistence

Synchronization updates the shipping JSON column in the Order table. The shipping object structure: 

{
  "provider": "lalamove",
  "bookingId": "...",
  "status": "PICKED_UP",
  "driverInfo": {
    "name": "...",
    "phone": "...",
    "plateNumber": "..."
  },
  "location": {
    "lat": 14.5995,
    "lng": 120.9842
  },
  "updatedAt": "2025-12-27T..."
