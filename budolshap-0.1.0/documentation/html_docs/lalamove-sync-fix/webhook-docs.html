Webhook Documentation: Security & Flow

Security: HMAC SHA256 Verification

Lalamove webhooks include an X-Lalamove-Signature header. Budolshap verifies this signature using the webhookSecret configured in the Lalamove service. 

// Verification Method (services/lalamove.js)
verifyWebhookSignature(payload, signature) {
    const hmac = crypto.createHmac('sha256', this.webhookSecret);
    const content = typeof payload === 'string' ? payload : JSON.stringify(payload);
    const expectedSignature = hmac.update(content).digest('hex');
    return signature === expectedSignature;
}

Webhook Handling Flow

[Lalamove Event] -> [POST /api/webhooks/lalamove]
    |
    V
[Verify Signature] -> (Fail: 401 Unauthorized)
    |
    V
[Log Event in DB] (WebhookEvent table)
    |
    V
[Lookup Order by bookingId]
    |
    V
[Execute State Machine Transition]
    |
    V
[Update Order & Shipping Data]
    |
    V
[Trigger Realtime Events (Pusher)]
    |
    V
[Return 200 OK to Lalamove]

Retry & Error Strategy

The webhook handler is designed to be idempotent and robust:

  • Immediate Acknowledgement: Returns 200 OK even if non-fatal processing errors occur to prevent Lalamove from retrying unnecessarily.
  • Error Logging: Fatal errors are logged in the WebhookEvent table with the FAILED status and the error message.
