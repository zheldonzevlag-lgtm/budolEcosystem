Phase 1: API Update

The first phase of the optimization involves updating the product API to support batch fetching by multiple IDs. This reduces the number of network requests from N (where N is the number of items in the cart) to 1.

Changes

File: app/api/products/route.js

Added support for ids query parameter to fetch multiple products at once.

// Modified GET function
const ids = searchParams.get('ids'); // comma separated ids

if (ids) {
    where.id = {
        in: ids.split(',')
    };
}

// Only filter by inStock if we're not fetching specific IDs
if (!ids) {
    where.inStock = true;
}

Rationale

  • Performance: Batch fetching is significantly faster than multiple individual requests.
  • Stability: Reduces the likelihood of "Failed to fetch" errors caused by concurrent request limits in browsers.
  • Functionality: By disabling the inStock filter when specific IDs are requested, we ensure that the cart can still display items that have recently gone out of stock.

Verification

Tested by calling /api/products?ids=id1,id2,id3 and verifying that it returns the expected products regardless of their stock status.
