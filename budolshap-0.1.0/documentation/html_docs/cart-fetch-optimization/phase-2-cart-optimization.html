Phase 2: Cart Component Optimization

In this phase, the Cart component was updated to utilize the new batch fetch API.

Changes

File: app/(public)/cart/page.jsx

Replaced individual fetch calls inside Promise.all with a single batch fetch.

// BEFORE
const productPromises = productIds.map(async (productId) => {
    const response = await fetch(`/api/products/${productId}`);
    return await response.json();
});
const fetchedProducts = await Promise.all(productPromises);

// AFTER
const response = await fetch(`/api/products?ids=${productIds.join(',')}`);
const fetchedProducts = await response.json();

Key Improvements

  • Reduced Overhead: One HTTP request instead of many.
  • Graceful Error Handling: Added explicit checks for response.ok and logging for missing products.
  • Data Mapping: Implemented a product map for efficient O(1) lookup when rebuilding the cart array.

Impact

The "Failed to fetch" error reported by the user is resolved because the browser no longer attempts to fire multiple concurrent requests which were likely being throttled or failing due to network instability.

Verification

Verified by navigating to the cart page with multiple items and observing a single successful API call in the Network tab of the browser dev tools.
