Phase 2: Implementation Details

Step-by-step technical execution of the return status workflow corrections.

1 Schema Update

Modified the ReturnStatus enum to include the new BOOKED value.

prisma/schema.prisma 
enum ReturnStatus {
  PENDING
  APPROVED
  BOOKED    // Added for courier booking confirmation
  SHIPPED   // Now triggered by actual driver pickup
  RECEIVED
  REFUNDED
  CANCELLED
  DISPUTED
}

2 Service Layer Modifications

Updated the shipping service to use the BOOKED status upon successful Lalamove booking.

lib/services/shippingService.js 
// Line 167 update
await prisma.return.update({
  where: { id: returnId },
  data: {
    status: 'BOOKED', // Set to BOOKED instead of SHIPPED
    trackingNumber: booking.orderId,
    returnShipping: shippingUpdate
  }
});

3 Webhook Logic Enhancement

Updated the Lalamove webhook to correctly identify and transition returns from BOOKED to SHIPPED only upon the PICKED_UP event.

app/api/shipping/lalamove/webhook/route.js 
// Line 125: Include BOOKED in the query
returns: {
  where: { status: { in: ['APPROVED', 'BOOKED', 'SHIPPED'] } },
  orderBy: { createdAt: 'desc' },
  take: 1
}

// Line 268: Handle PICKED_UP event
if (newStatus === 'PICKED_UP' || eventType === 'PICKED_UP') {
  returnStatusUpdate = 'SHIPPED';
}
