Budolshap – Deploy to Vercel & Configure PayMongo (GCash)

📦 Prerequisites

  • Vercel account (free tier works fine)
  • PayMongo account with a Secret Key and optional Webhook Secret
  • PostgreSQL database (Vercel Postgres, Supabase, Railway, etc.)
  • Git repository (GitHub, GitLab or Bitbucket) linked to Vercel

🚀 1️⃣ Push Your Code to Git

Initialize a repo (if you haven’t already)
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin <YOUR_REPO_URL>
git push -u origin main
Make sure .gitignore excludes .env (already present).

🔧 2️⃣ Connect to Vercel

  1. Log in to vercel.com
  2. Click **New Project → Import Git Repository** → select your repo
  3. Leave the default **Framework Preset** as **Next.js**
  4. Click **Deploy** – Vercel will run npm install and npm run build

🔐 3️⃣ Add Environment Variables

Open Settings → Environment Variables in your Vercel project and add the following:

PAYMONGO_SECRET_KEY # your PayMongo secret key (Secret)
PAYMONGO_WEBHOOK_SECRET # optional, for webhook verification (Secret)
NEXT_PUBLIC_SITE_URL # e.g. https://your-app.vercel.app (Plain)
DATABASE_URL # PostgreSQL connection string (Secret)

Set each for Production (and optionally Preview).

💡 Never expose secret keys as public variables. Vercel masks them in logs.

🛠️ 4️⃣ Verify the Setup (Optional)

Create a quick test endpoint to ensure the variables are loaded:

/* app/api/debug/env/route.js */
import { NextResponse } from 'next/server';
export async function GET() {
  return NextResponse.json({
    paymongo: !!process.env.PAYMONGO_SECRET_KEY,
    webhook: !!process.env.PAYMONGO_WEBHOOK_SECRET,
    siteUrl: process.env.NEXT_PUBLIC_SITE_URL,
  });
}

Deploy, then visit /api/debug/env. You should see true for the keys you added.

⚙️ 5️⃣ Build & Deploy

Vercel automatically rebuilds on every push. If you need a manual trigger:

vercel --prod   # from the CLI, forces a production deployment

Or click **Redeploy** in the Vercel dashboard.

💳 6️⃣ GCash (PayMongo) Integration

  1. Make sure lib/paymongo.js contains the lazy client (already committed).
  2. Use the createGCashSource function in app/api/payment/gcash/create/route.js to generate a checkout URL.
  3. Redirect users to the returned checkoutUrl.
  4. After payment, PayMongo redirects to process.env.NEXT_PUBLIC_SITE_URL/payment/success (or /failed).

If you use webhooks, set the PAYMONGO_WEBHOOK_SECRET and configure the webhook URL in PayMongo to https://your-app.vercel.app/api/webhooks/paymongo.

🧹 7️⃣ Clean‑up & Future Steps

  • Remove any local .env from the repo (keep it in .gitignore).
  • Enable Automatic PR Deployments in Vercel for preview environments.
  • Monitor logs via **Vercel → Functions → Logs** for any runtime errors.
  • Consider adding a custom domain in **Settings → Domains**.
