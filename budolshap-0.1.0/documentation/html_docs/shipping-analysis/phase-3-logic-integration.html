1. Lalamove Quotation Flow

This flow occurs when a user enters their shipping address in the checkout page.

1. Frontend sends [Pickup_LatLong, Dropoff_LatLong, VehicleType] to Backend.
2. Backend calls Lalamove API: POST /v3/quotations
3. Lalamove returns: {
     "totalFee": "150.00",
     "distance": {"value": "12.5", "unit": "km"},
     "surge": "1.2"
   }
4. Backend applies BudolShap Business Rules (Subsidies/Buffers).
5. Backend returns final display_fee to Frontend.

2. Shopee-Style Cost Allocation Engine

A central service determines the final cost distribution based on active promotions and store settings.

Scenario Logic Calculation Who Pays?
Standard Buyer_Pays = Quote_Fee Buyer
Free Shipping Voucher Buyer_Pays = max(0, Quote_Fee - Subsidy) Platform (Subsidy)
Seller Discount Buyer_Pays = Quote_Fee - Seller_Discount Seller
Capped Shipping Buyer_Pays = min(Quote_Fee, Max_Cap) Platform/Seller (Difference)

3. Handling Price Variance (Post-Trip)

Lalamove's final bill may differ from the initial quote due to real-world conditions.

Variance Resolution Policy

  • Policy A (Platform Absorbs): Platform covers the difference if Actual - Quoted < ₱50.
  • Policy B (Escrow Adjustment): Deduct the difference from the Seller's payout if the error was due to incorrect pickup location.
  • Policy C (Buyer Surcharge): Charge the buyer's saved card (requires prior authorization) for significant toll/waiting fees.

Phase 3 Success Criteria

  • Successful Lalamove quotation API integration.
  • Logic engine correctly calculates Buyer/Seller/Platform splits.
  • Webhook listener implemented for ORDER_COMPLETED to update final fees.
