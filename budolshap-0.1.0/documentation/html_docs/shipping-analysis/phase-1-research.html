1. Lalamove Shipping Computation Logic

Lalamove provides real-world estimates based on live data. The quotation is dynamic and subject to change during the actual fulfillment process.

Factor Technical Detail
Distance Calculated via road-routing (Map-based), not straight-line.
Vehicle Type Base Fare + (Distance * Rate per KM). Rates vary by vehicle class (Motorcycle vs. Sedan).
Dynamic Pricing Surge pricing applied during peak hours, bad weather, or high demand.
Add-ons Fees for multi-stops, cash handling, or driver assistance.
Tolls Usually excluded from initial quotes; added to final billing.

2. Shopee Shipping Absorption Model

Shopee uses a multi-tiered approach to determine who shoulders the delivery cost. It is rarely a single-party expense.

Cost Allocation Scenarios

  • Buyer Pays (Default): Standard checkout behavior where the shipping fee is added to the order total.
  • Shopee Absorbs: Platform-funded vouchers or site-wide campaigns (e.g., 11.11).
  • Seller Absorbs: Seller-funded programs (e.g., "Free Shipping Special") or manual discounts set by the store.
  • Hybrid: Split between Shopee (voucher) and Buyer, or Seller and Buyer.

3. Key Requirements for BudolShap

  • Quote Accuracy: Use precise pickup/drop-off pins to minimize road-distance variance.
  • Transparency: Clearly label fees as "Estimated" and provide tooltips for potential surcharges (tolls, traffic).
  • Accounting: Track the difference between quoted_fee and actual_fee to manage platform margins.
  • Buffer Logic: Optionally add a % buffer to the initial quote to cover minor surge variations.

Phase 1 Success Criteria

  • Full understanding of Lalamove API pricing components.
  • Defined business rules for shipping absorption (Buyer vs. Seller vs. Platform).
  • Identification of technical risks (Surge pricing, Toll road adjustments).
