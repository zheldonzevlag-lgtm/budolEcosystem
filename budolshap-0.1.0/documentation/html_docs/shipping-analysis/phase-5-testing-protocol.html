1. Testing Strategy

Logistics testing requires both sandbox API simulation and real-world scenario modeling.

Unit Testing (Logic Engine)

  • Validate cost split logic (Buyer/Seller/Platform).
  • Test buffer calculation formulas.
  • Verify coordinate-to-distance conversions.

Integration Testing (Lalamove API)

  • Happy Path: Successful quote request and booking.
  • Network Failure: System behavior when Lalamove API is unreachable.
  • Webhook Validation: Simulating ORDER_COMPLETED and ORDER_CANCELLED payloads.

2. Real-World Test Cases

Test Case Expected Outcome
Peak Hour Booking System correctly captures surge pricing in the quote.
Multi-Stop Delivery API returns correct fee; UI displays breakdown clearly.
Toll Road Usage Final charge > Quoted charge; System logs variance correctly.
Driver Cancellation Webhook triggers order status update and notifies buyer/seller.

3. Rollback & Contingency Plans

Emergency Procedures

Scenario: Critical failure in shipping computation during checkout.

  • Step 1: Disable Lalamove dynamic pricing via Feature Flag.
  • Step 2: Revert to static "Flat Rate" shipping profiles.
  • Step 3: Notify users that live tracking is temporarily unavailable.

Phase 5 Success Criteria

  • 100% pass rate on core logic unit tests.
  • Successful sandbox booking flow with Lalamove.
  • Documented rollback procedure verified by the dev team.
