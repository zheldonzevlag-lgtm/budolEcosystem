🎯 Objective

To implement a comprehensive, user-friendly Order Tracking UI that supports both Lalamove (Live GPS) and Standard (COD) deliveries, and to enhance the Store Order Management system with proper status validation rules.

📱 Order Tracking UI

The Order Detail page (/orders/[id]) was completely redesigned to provide a premium, app-like experience.

Key Features

🗂️ Tabbed Interface

Content is organized into three clean tabs: Tracking History, Order Items, and Delivery Details. This reduces clutter and improves navigation.

📍 Lalamove Live Tracking

For Lalamove orders, an embedded map shows the driver's real-time location. A Driver Card displays the driver's name, vehicle, and contact info.

🗺️ Standard Delivery Map

For non-Lalamove orders, a static Google Map confirms the delivery destination, providing visual reassurance even without live tracking.

📱 Mobile Responsiveness

All components are fully responsive. The Progress Tracker, Timeline, and Details Grid stack vertically on mobile devices for optimal viewing.

🏪 Store Order Management

Enhanced the Seller Dashboard (/store/orders) to enforce business logic and prevent invalid status updates.

Validation Rules Implemented

  • Sequential Flow: Sellers can only move orders forward:
    • ORDER_PLACED → PROCESSING
    • PROCESSING → SHIPPED
    • SHIPPED → DELIVERED
  • No Manual "PAID": The option to manually mark an order as PAID was removed to ensure payment integrity.
  • Lalamove Protection: If an order is booked via Lalamove, the status dropdown is disabled and replaced with an "Auto" badge. This prevents manual interference with Lalamove's automated updates.
