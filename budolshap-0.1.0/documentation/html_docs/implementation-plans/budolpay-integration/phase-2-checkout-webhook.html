Phase 2: Checkout & Webhook Integration

This phase focuses on the asynchronous completion of the payment cycle, ensuring that users are redirected to the gateway and that BudolShap automatically processes payment notifications.

1. Checkout Redirection Flow

When a user selects budolPay in the Order Summary, the system creates a Payment Intent and redirects the user to the provided checkoutUrl.

// OrderSummary.jsx implementation
if (paymentData.checkoutUrl) {
    window.location.replace(paymentData.checkoutUrl);
}

2. Webhook Implementation

BudolPay notifies the ecosystem via webhooks. We have implemented a dedicated listener to handle these events.

Endpoint: POST /api/webhooks/budolpay

Supported Events: payment.success, payment_intent.succeeded

// Webhook Processing Logic
async function handlePaymentSuccess(data) {
    const orderId = data.metadata.orderId;
    await prisma.order.update({
        where: { id: orderId },
        data: {
            isPaid: true,
            status: 'PAID',
            paymentStatus: 'paid'
        }
    });
    // Credit escrow, send email, clear cart...
}

3. Transaction Verification

For added security and redundancy, BudolShap can manually verify a transaction status using the BudolPay Transaction Service.

Method: BudolPayAdapter.verifyTransaction(transactionId)

Gateway Endpoint: GET /transactions/:transactionId
