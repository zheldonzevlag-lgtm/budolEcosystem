BudolPay Service Requirements

Technical specifications for the BudolPay team to support BudolShap integration.

Overview

To successfully integrate BudolShap with BudolPay, the BudolPay team must provide a functional API Gateway and several backend services. This document outlines the expected endpoints, data formats, and communication protocols.

Base Configuration:
- Gateway URL: http://localhost:8004
- Protocol: HTTP/JSON
- Authentication: Header x-api-key: bs_key_2025

1. API Gateway Requirements

The API Gateway must act as the central entry point and route requests to the appropriate microservices.

Endpoints to Expose:

POST /payments/create-intent

Description: Creates a new payment intent and returns a redirect URL for the customer.

Request Body:

{
  "amount": 100.00, // Decimal format (e.g. 100.00 for PHP 100)
  "currency": "PHP",
  "description": "Order from budolShap",
  "paymentMethod": "gcash", // Optional
  "metadata": {
    "orderId": "clp1234567890",
    "app": "budolShap",
    "customer_email": "user@example.com"
  }
}

Expected Response (201 Created):

{
  "id": "pi_8yH2j9K...",
  "status": "pending",
  "checkoutUrl": "http://localhost:8000/checkout/pi_8yH2j9K...",
  "amount": 100.00,
  "currency": "PHP"
}
GET /transactions/:id

Description: Verifies the status of a specific transaction or payment intent.

Expected Response (200 OK):

{
  "id": "pi_8yH2j9K...",
  "status": "success", // or "paid", "failed", "pending"
  "amount": 100.00,
  "currency": "PHP",
  "paymentMethod": "gcash",
  "updatedAt": "2025-12-31T12:00:00Z"
}

2. Webhook Notification Service

BudolPay must notify BudolShap when a payment status changes. This is critical for updating order records in real-time.

Target URL: http://localhost:3000/api/webhooks/budolpay
Method: POST
Event Type: payment.success

Payload Format:

{
  "event": "payment.success",
  "data": {
    "id": "pi_8yH2j9K...",
    "amount": 100.00,
    "currency": "PHP",
    "status": "paid",
    "metadata": {
      "orderId": "clp1234567890",
      "app": "budolShap"
    }
  }
}

3. Checkout UI (Redirection)

The checkoutUrl returned by the Gateway should lead to a hosted payment page where the user can complete the transaction.

  • Must handle the payment intent ID passed in the URL.
  • Upon successful payment, it should trigger the Webhook mentioned above.
  • Should provide a "Return to Merchant" button pointing back to BudolShap (e.g., http://localhost:3001/auth/callback or order success page).

4. Security & Compliance

  • API Key Validation: The Gateway must verify the x-api-key header for all incoming requests from BudolShap.
  • Data Integrity: Future phases will require HMAC signatures for webhooks to prevent spoofing.
