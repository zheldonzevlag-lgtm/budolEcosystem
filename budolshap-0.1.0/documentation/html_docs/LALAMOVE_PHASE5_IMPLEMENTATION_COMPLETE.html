💡 Key Learnings

1. StopIds Must Come from Quote Response

The most critical fix: Lalamove requires using the actual stopIds returned in the quote response, not hardcoded values. Each quote generates unique stopIds that must be used when creating the order.

2. Request-ID Header is Required

Lalamove API v3 requires a unique Request-ID header for each request. Use crypto.randomUUID() to generate it.

3. Phone Number Format

Philippine phone numbers must be in format: +63 followed by exactly 10 digits (e.g., +639171234567). Remove the leading 0 from mobile numbers.

4. Minimal Payload Works Best

Start with minimal required fields (quotationId, sender, recipients) before adding optional fields like metadata and remarks.
