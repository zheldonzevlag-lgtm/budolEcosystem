Phase 3: Driver Info Persistence

The Issue

Driver information was only provided by Lalamove during the DRIVER_ASSIGNED event. Subsequent events (like PICKED_UP or ON_GOING) often omitted the driver object, causing the BudolShap system to clear the previously saved driver info when updating the status.

The Solution

The webhook handler in app/api/webhooks/lalamove/route.js was modified to prioritize existing driver information and fetch fresh data if needed.

1. Passive Merging

Modified the updatedShipping object construction to ensure the driverInfo is preserved unless explicitly replaced by newer data:

const updatedShipping = {
    ...order.shipping,
    // Preserve existing driverInfo if the current payload doesn't have it
    driverInfo: data?.driver || order.shipping?.driverInfo || null
};

2. Failure Persistence

In cases where a booking fails or is canceled (e.g., CANCELED, EXPIRED), the driver info is now archived into the previousAttempts array rather than being deleted, allowing sellers to see who was assigned before the failure.

3. Fresh Data Fetching

Added a proactive fetch using the Lalamove trackOrder service within the webhook route to ensure the latest driver details are always available during state transitions.
