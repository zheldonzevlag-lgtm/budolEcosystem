Phase 2: Provider-Agnostic Hooks

Overview

To prevent vendor lock-in and simplify component logic, we introduced a suite of reusable React hooks. These hooks encapsulate the complexity of switching between Polling, Pusher, and Socket.io.

Newly Created Hooks

Hook Target Data Realtime Channels
useRealtimeReturns Store return requests store-{storeId}
useRealtimeUser User-specific updates (status, etc.) user-{userId}
useRealtimeOrderDetail Specific order status/tracking user-{userId}
useRealtimeOrders Seller order list store-{storeId}

Example Implementation Pattern

export function useRealtimeReturns({ storeId }) {
    const { data: config } = useSWR('/api/system/realtime', fetcher);
    const { data, mutate } = useSWR('/api/returns', fetcher, {
        refreshInterval: config?.provider === 'POLLING' ? 5000 : 0
    });

    useEffect(() => {
        if (config?.provider === 'PUSHER') {
            const channel = pusher.subscribe(`store-${storeId}`);
            channel.bind('return-requested', mutate);
        }
        // ... Socket.io logic
    }, [config, mutate]);

    return { returns: data, mutate };
}

Testing Protocol

  • [ ] Polling Test: Set provider to POLLING and verify 5s refresh in Network tab.
  • [ ] Pusher Test: Set provider to PUSHER and verify WebSocket frames for return-requested.
  • [ ] Socket.io Test: Set provider to SOCKET_IO and verify event propagation.
