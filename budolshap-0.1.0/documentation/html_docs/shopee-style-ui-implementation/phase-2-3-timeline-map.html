Phase 2.3: Shipping Timeline & Map Integration

Objective

Align the TrackingTimeline and LalamoveTracking components with the standardized 5-stage milestone model for a cohesive user experience.

Technical Changes

TrackingTimeline.jsx

  • Renamed "Order Placed" to "Placed".
  • Renamed "Payment Confirmed" to "Paid".
  • Renamed "Delivered" to "Completed".
  • Updated internal checks to use the new "Paid" title for payment verification status.

LalamoveTracking.jsx

  • Updated live status messages to use Shopee-style terminology (e.g., "To Ship", "To Receive", "Completed").
  • Integrated consistent emojis for better visual recognition of stages.
  • Standardized fallback status messages for non-Lalamove specific events.

Code Snippets

Updated Lalamove live status logic:

{order.shipping?.status === 'ASSIGNING_DRIVER' && "📝 Finding a driver..."}
{order.shipping?.status === 'ON_GOING' && "📦 Driver assigned & on the way to pickup"}
{order.shipping?.status === 'PICKED_UP' && "🚚 Package picked up - on the way to you"}

Updated fallback status mapping:

{order.status === 'SHIPPED' && "📦 To Ship"}
{order.status === 'IN_TRANSIT' && "🚚 To Receive"}
{order.status === 'DELIVERED' && "✅ Completed"}

Validation Criteria

  • ✓ Timeline entries match the progress bar stages exactly.
  • ✓ Lalamove tracking overlay correctly shows "Completed" instead of "Delivered".
  • ✓ Live location status badge reflects the current Shopee-style milestone.
