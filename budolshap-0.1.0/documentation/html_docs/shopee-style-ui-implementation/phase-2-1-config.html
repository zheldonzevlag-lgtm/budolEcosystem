Phase 2.1: Core Configuration & State Alignment

Objective

Standardize the internal PROVIDER_STAGES configuration to align with Shopee's 5-stage tracking model: Placed, Paid, To Ship, To Receive, Completed.

Technical Changes

File: lib/shipping/config.js

Modified the stage definitions for both Lalamove and Standard shipping providers. Removed redundant stages like "Verifying Payment" and simplified labels.

Lalamove Provider Stages:

[SHIPPING_PROVIDERS.LALAMOVE]: [
    { label: 'Placed', status: 'ORDER_PLACED', icon: '📝' },
    { label: 'Paid', status: 'PAID', specificStatus: 'ASSIGNING_DRIVER', icon: '💳' },
    { label: 'To Ship', status: 'PROCESSING', specificStatus: 'ON_GOING', icon: '📦' },
    { label: 'To Receive', status: 'IN_TRANSIT', specificStatus: ['PICKED_UP', 'IN_TRANSIT'], icon: '🚚' },
    { label: 'Completed', status: 'DELIVERED', specificStatus: 'COMPLETED', icon: '✅' }
]

Validation Criteria

  • ✓ Stage labels are consistent (Placed, Paid, To Ship, To Receive, Completed).
  • ✓ Lalamove specific statuses (ASSIGNING_DRIVER, ON_GOING) are correctly mapped to primary stages.
  • ✓ Standard shipping stages match the new simplified model.

Rollback Plan
