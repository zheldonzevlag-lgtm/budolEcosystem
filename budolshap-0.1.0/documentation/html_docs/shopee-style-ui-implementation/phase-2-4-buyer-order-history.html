Phase 2.4: Buyer Order History UI

Objective

Update the OrderItem component in the buyer's order history to use Shopee-style status labels and ensure the shipping breakdown is clear and transparent.

Technical Changes

OrderItem.jsx

  • Status Mapping: Updated the desktop and mobile status badges to use "TO SHIP", "TO RECEIVE", and "COMPLETED" instead of "PROCESSING", "IN TRANSIT", and "DELIVERED".
  • Shipping Breakdown: Verified and enhanced the tooltip logic to show a detailed breakdown of shipping costs, subsidies, and vouchers.
  • Responsive Badges: Standardized badge colors and text across desktop and mobile views for consistent branding.

Code Snippets

Updated status mapping in OrderItem.jsx:

order.status === 'SHIPPED' || order.status === 'IN_TRANSIT'
    ? 'TO RECEIVE'
    : order.status === 'DELIVERED' || order.status === 'COMPLETED'
        ? 'COMPLETED'
        : ['ORDER_PLACED', 'PAID', 'PROCESSING'].includes(order.status)
            ? 'TO SHIP'

Mobile status badge update:

{order.status === 'DELIVERED' || order.status === 'COMPLETED' ? 'Completed'
    : order.status === 'SHIPPED' || order.status === 'IN_TRANSIT' ? 'To Receive'
    : ['ORDER_PLACED', 'PAID', 'PROCESSING'].includes(order.status) ? 'To Ship'
    : ... }

Validation Criteria

  • ✓ Buyer order list shows "To Ship" for orders being processed.
  • ✓ Buyer order list shows "To Receive" for orders in transit.
  • ✓ Shipping fee breakdown tooltip correctly displays subsidies and net fees.
