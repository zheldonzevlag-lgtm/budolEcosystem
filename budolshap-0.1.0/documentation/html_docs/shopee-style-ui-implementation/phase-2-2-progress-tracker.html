Phase 2.2: Order Progress Tracker UI

Objective

Update the OrderProgressTracker component to reflect the new Shopee-style milestones and ensure consistent behavior across all payment methods (COD, BudolPay, QRPH).

UI Enhancements

  • Stage Labels: Updated from "Payment Confirmed" to "Paid", and "Driver Assigned/In Transit" to "To Ship/To Receive".
  • Payment Validation: Refined logic to ensure the "Paid" stage only shows as completed when payment is verified (or for COD orders).
  • Icon Logic: Dynamic vehicle icons (🏍️, 🚗, 🚐) now correctly trigger for "To Ship" and "To Receive" stages based on Lalamove driver info.
  • Return Flow: Updated return-specific stages to match the new "Placed" and "Completed" terminology.

Code Snippets

Updated payment stage check:

const isPaymentStage = stage.label === 'Paid';
const isPaymentPending = isPaymentStage && !hasPayment;

Updated vehicle icon mapping:

(['To Ship', 'To Receive'].includes(stage.label) && order.shipping?.driverInfo?.vehicleType)

Validation Criteria

  • ✓ Desktop view shows 5 clear stages with "Placed, Paid, To Ship, To Receive, Completed".
  • ✓ Mobile view correctly lists current and completed stages with matching icons.
  • ✓ "Unpaid" warning shows correctly on the "Paid" stage for pending payments.
