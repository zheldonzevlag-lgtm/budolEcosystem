🐛 Lalamove Booking Error: ERR_INVALID_SCHEDULE_TIME

Issue Date: 2025-11-30
Status: RESOLVED

Problem Description

Error Message

ERR_INVALID_SCHEDULE_TIME

Lalamove API Response: { errors: [ { id: 'ERR_INVALID_SCHEDULE_TIME' } ] }

User-Facing Message: "Shipping service error. Please try again."

Root Cause Analysis

Timeline of Events

  1. 2025-11-29 17:23 - User generated a Lalamove quote
  2. 2025-11-30 01:52 - User attempted to book delivery (next day)
  3. Result: Lalamove rejected the booking because the schedule time was in the past

Why This Happened

The Lalamove API requires a scheduleAt field when creating an order. This field specifies when the delivery should be picked up. Our implementation was missing this field, causing Lalamove to either:

  • Use the quote's original scheduleAt time (which was from the previous day)
  • Reject the request due to missing required field

Lalamove Requirement: Schedule time must be at least 15 minutes in the future for sandbox environment.

Investigation Process

Step 1: Initial Hypothesis (Phone Number)

Initially suspected invalid phone numbers based on dummy data in the database (01234567899). This was fixed but didn't resolve the issue.

Step 2: Enhanced Logging

Added detailed logging to capture the exact Lalamove API error response:

console.log('[Lalamove Book] Contact details:', {
    pickup: pickupContact,
    delivery: deliveryContact
});

Step 3: Root Cause Identified

Vercel logs revealed the actual error:

Lalamove API error (POST /v3/orders): { 
    errors: [ { id: 'ERR_INVALID_SCHEDULE_TIME' } ] 
}

The Fix

Solution: Add scheduleAt Field

Updated services/lalamove.js to include a scheduleAt field in the order creation payload.

Code Changes

File: services/lalamove.js

Change 1: Calculate Schedule Time

// Calculate scheduleAt if not provided 
// (must be at least 15 minutes in the future for sandbox)
const scheduleAt = payload.scheduleAt || 
    new Date(Date.now() + 15 * 60 * 1000).toISOString();

Change 2: Add to Order Payload

const orderData = {
    data: {
        quotationId: quotationId,
        sender: { ... },
        recipients: [ ... ],
        scheduleAt: scheduleAt,  // ✅ Added this field
        isPODEnabled: false,
        metadata: metadata || {}
    }
};

Change 3: Update Error Map

'ERR_INVALID_SCHEDULE_TIME': 
    'Invalid schedule time. The delivery must be scheduled at least 15 minutes in the future.'

How It Works Now

Booking Flow

  1. User requests a quote → Quote generated with scheduleAt
  2. User clicks "Book Delivery Now"
  3. New: System calculates fresh scheduleAt = current time + 15 minutes
  4. Booking request sent to Lalamove with valid future schedule time
  5. ✅ Booking succeeds

Important Notes

  • Sandbox Requirement: 15 minutes minimum lead time
  • Production: May have different requirements (check Lalamove docs)
  • Quote Expiration: Quotes are still valid even if schedule time has passed, but booking requires a new schedule time
