Deploying Budolshap to Vercel

A step-by-step guide to deploying your Next.js e-commerce platform with Vercel Postgres.

0

Prerequisites

Before deploying, make sure you have the following ready:

  • Node.js (v18+) and npm installed.
  • Git for version control.
  • A Vercel account (free tier works fine).
  • Vercel CLI installed globally: npm i -g vercel
  • A PayMongo account with API keys (Secret and Public).
  • Access to the .env file (or copy from .env.paymongo.example) and fill in the required variables.
  • Basic knowledge of Prisma and the ability to run migrations.
1

Prepare Codebase for Vercel Postgres

Since Vercel works best with its native Postgres database, we need to switch your Prisma schema from MySQL to PostgreSQL.

1. Update prisma/schema.prisma

Replace the datasource block and fix data types:

// 1. Change provider to postgresql and add directUrl
datasource db {
  provider  = "postgresql"
  url       = env("POSTGRES_PRISMA_URL") // Uses connection pooling
  directUrl = env("POSTGRES_URL_NON_POOLING") // Uses direct connection
}

// 2. Find and Replace all @db.LongText with @db.Text
// Example in User model:
model User {
  ...
  image String @db.Text // Changed from @db.LongText
  ...
}

// Example in Store model:
model Store {
  ...
  logo String @db.Text // Changed from @db.LongText
  ...
}

2. Update package.json

Update the build script to generate the Prisma client before building:

"scripts": {
  ...
  "build": "prisma generate && next build",
  ...
}
2

Push to GitHub

Ensure your latest changes (including the Prisma updates) are pushed to a GitHub repository.

git add .
git commit -m "Prepare for Vercel deployment"
git push origin main
3

Create Vercel Project

  1. Log in to your Vercel Dashboard.
  2. Click "Add New..." -> "Project".
  3. Import your budolshap repository from GitHub.
  4. In the Configure Project screen, DO NOT deploy yet. We need to set up the database first.
4

Setup Vercel Postgres

While still on the project configuration page (or in the Storage tab if you already created the project):

  1. Go to the Storage tab in your Vercel project dashboard (you might need to create the project first without deploying, or just add it during creation if available).
  2. Click "Create Database" and select Postgres.
  3. Accept the terms and create the database.
  4. Once created, Vercel will automatically add the necessary environment variables (POSTGRES_PRISMA_URL, POSTGRES_URL_NON_POOLING, etc.) to your project.
  5. Important: Go to Settings -> Environment Variables and ensure these variables are present.
5

Configure Environment Variables

Add the remaining environment variables in Settings -> Environment Variables:

Variable Name Value / Description
JWT_SECRET Your secret key (generate a long random string)
NEXT_PUBLIC_APP_URL Your Vercel domain (e.g., https://budolshap.vercel.app)
NEXT_PUBLIC_SITE_URL Same as above (e.g., https://budolshap.vercel.app)
PAYMONGO_SECRET_KEY Your PayMongo Secret Key (Live or Test)
PAYMONGO_PUBLIC_KEY Your PayMongo Public Key
SMTP_HOST smtp.gmail.com (or your provider)
SMTP_USER Your email address
SMTP_PASS Your email app password
6

Deploy & Migrate

1. Deploy

Click Deploy in Vercel. Wait for the build to complete. It might show a "500 Error" initially because the database tables don't exist yet.

2. Run Migrations

We need to push the schema to the Vercel database. The easiest way is to connect your local machine to the Vercel DB.

  1. Install Vercel CLI: npm i -g vercel
  2. Login: vercel login
  3. Link project: vercel link
  4. Pull env vars: vercel env pull .env.production.local
  5. Run migration: npx dotenv -e .env.production.local -- prisma migrate deploy

Note: Since you switched from MySQL to Postgres, this will create a fresh database. You will need to create a new Admin user and seed data again.
