Project Overview

The goal of this project was to implement realtime updates for the seller returns dashboard, ensuring that new return requests from buyers are visible immediately without requiring a page reload. This was achieved by integrating Pusher realtime events into the returns workflow.

Outcome: Sellers now see new return requests and status updates in realtime. The manual "Refresh" dependency has been removed, improving the overall seller experience and responsiveness of the platform.

Technical Specifications

  • Realtime Provider: Pusher (Active), with fallbacks for Socket.io and Polling.
  • Frontend Stack: React, SWR, Pusher-JS.
  • Backend Stack: Next.js API Routes, Prisma, Pusher Server SDK.
  • Channel Strategy: Store-specific private channels (store-${storeId}).

Future Recommendations

Optimization: Consider implementing browser notifications (Web Push) in addition to UI updates to alert sellers even when they are not actively looking at the returns page.
  • Event History: Implement a "Notifications" sidebar to track historical realtime events.
  • Performance: Monitor Pusher message volume to ensure it stays within plan limits as the platform scales.
  • Error Handling: Add explicit UI feedback if the realtime connection is lost (e.g., a "Disconnected" indicator).
