Objectives COMPLETED

  • Replace manual fetch() logic with the new useRealtimeReturns hook.
  • Remove redundant loading states and manual refresh triggers.
  • Ensure UI components react instantly to incoming return requests.

Refactoring Details

The StoreReturns page was refactored to eliminate the dependency on a manual "Refresh" action. The page now subscribes to the realtime hook on mount.

Before:

useEffect(() => {
    fetchReturns(); // Manual fetch on mount
}, [storeId]);

const fetchReturns = async () => {
    setLoading(true);
    // manual API call
    setLoading(false);
}

After:

const { returns, isLoading, mutate } = useRealtimeReturns({ storeId });

// UI automatically updates when hook receives Pusher event

UX Improvements

  • No Loading Spinners on Event: SWR performs a background revalidation, providing a seamless transition.
  • Responsive Refresh Button: The manual refresh button now triggers mutate(), ensuring it stays in sync with the realtime cache.
