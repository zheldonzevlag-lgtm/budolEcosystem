Objectives COMPLETED

  • Create a reusable React hook for return-specific realtime events.

  • Support multiple realtime providers (Pusher, Socket.io, Polling).

Technical Specification

The useRealtimeReturns hook centralizes the logic for listening to return-related events on the store's private channel. It uses the store-${storeId} naming convention to ensure sellers only receive notifications for their own store.

Key Events Handled:

  • return-requested: Triggered when a buyer submits a new return request.
  • return-updated: Triggered when a return status changes or a counter-offer is made.
  • dispute-resolved: Triggered when an admin resolves a return dispute.
