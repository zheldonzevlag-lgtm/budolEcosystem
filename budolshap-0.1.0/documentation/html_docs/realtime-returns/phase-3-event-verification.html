Backend Audit Results

A comprehensive audit of the backend codebase confirmed that return-related events are being triggered in the following locations:

Location Event Name Trigger Point
returnsService.js return-requested Initial return request creation
returnsService.js return-updated Status changes / counter-offers
returnsService.js dispute-resolved Admin resolution of disputes
return/respond/route.js return-updated Buyer response to return offer

System Configuration

Realtime events depend on the SystemSettings configuration. The lib/realtime.js module ensures that events are only sent to Pusher if the provider is set to PUSHER and credentials are present.
