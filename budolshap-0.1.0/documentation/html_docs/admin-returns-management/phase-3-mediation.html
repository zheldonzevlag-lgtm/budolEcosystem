Phase 3: Mediation & Resolution

The final decision-making layer where admins resolve conflicts and finalize transactions.

1. Mediation Actions ✅ Completed

Admin-specific controls to resolve disputed return requests.

Full Refund to Buyer

Accepts buyer's claim. Funds released from escrow to buyer.

Reject Return Request

Siding with seller. Funds released from escrow to seller.

Arbitration Notes

Admin records reasoning for final decision.

2. Escrow & Refund Logic

Integration with the core payment system to handle money movement.

  • Automatic Triggers: When an admin "Executes Refund", the lockedBalance in escrow is moved to the buyer's walletBalance.
  • Seller Payout: If rejected, the lockedBalance is released to the seller's pendingPayout.
  • Transaction History: Every admin resolution creates a detailed entry in the OrderHistory and EscrowLog.

3. Communication & Closing

Notifying all parties of the final decision.

  • Pusher Events: return-resolved event triggered for real-time UI updates for buyer/seller.
  • Email Notifications: Formal summary of the resolution sent to both parties.
  • Order Status: Update main order status to REFUNDED or COMPLETED based on the outcome.
