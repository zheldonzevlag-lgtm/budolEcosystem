🔍 Issues Identified

Issue #1: Wrong Property Name in Component

File: components/LalamoveTracking.jsx

Problem: Component was looking for driverInfo property but webhook stores data in driver property

// ❌ Wrong
const { driverInfo, shareLink } = order.shipping || {};
const driver = driverInfo || { /* mock data */ };

Issue #2: Missing Vehicle Type Capture

File: app/api/webhooks/lalamove/route.js

Problem: Webhook handler wasn't capturing the vehicleType field from Lalamove driver data

// ❌ Missing vehicleType
updatedShipping.driver = {
    name: driver.name,
    phone: driver.phone,
    plateNumber: driver.plateNumber,
    photo: driver.photo
    // vehicleType was missing!
};

Issue #3: Test Script Not Sending Vehicle Type

File: scripts/trigger-pickup-webhook.js

Problem: Test webhook payload didn't include vehicleType

✅ Solutions Implemented

Solution #1: Fixed Property Name

File: components/LalamoveTracking.jsx

// ✅ Correct
const { driver, shareLink } = order.shipping || {};
const driverData = driver || {
    name: "Juan Dela Cruz",
    phone: "+63 917 123 4567",
    plateNumber: "ABC 1234",
    rating: 4.8,
    vehicleType: "Motorcycle"
};

Updated all references from driver to driverData in the JSX to avoid variable name conflicts.

Solution #2: Added Vehicle Type Capture

File: app/api/webhooks/lalamove/route.js

// ✅ Now includes vehicleType
if (driver) {
    updatedShipping.driver = {
        name: driver.name,
        phone: driver.phone,
        plateNumber: driver.plateNumber,
        photo: driver.photo,
        vehicleType: driver.vehicleType // ✅ Added
    };
}

💡 Key Learnings

  • Always verify property names match between data source and consumer
  • Webhook payloads should be fully captured, not just partial data
  • Local testing with production databases requires careful environment configuration
  • Mock data should match the structure of real data for easier testing

📝 Notes
