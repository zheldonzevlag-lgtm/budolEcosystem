WYSIWYG Editor Implementation & Deployment Fixes

Date: November 30, 2025

Status: Completed & Deployed

1. WYSIWYG Editor Implementation

Replaced the standard textarea description field with a rich text editor using react-quill.

Changes Made:

  • Installed Dependency: Added react-quill to the project.
  • Add Product Page: Updated app/store/add-product/page.jsx to use the ReactQuill component.
    • Implemented dynamic import for SSR compatibility.
    • Restored missing "Name" and "Price" fields that were accidentally removed during the initial edit.
  • Product Lists: Updated app/store/manage-product/page.jsx and app/admin/products/page.jsx to strip HTML tags from the description column to prevent raw HTML from showing in the table view.
  • Product Details: Updated components/ProductDescription.jsx to render the HTML content using dangerouslySetInnerHTML so customers see the formatted text.

2. Deployment Fixes

Resolved build and deployment issues encountered during the update.

Issues & Solutions:

  • Peer Dependency Conflict: react-quill has peer dependency conflicts with React 19.
    Fix: Created .npmrc with legacy-peer-deps=true to enforce legacy peer dependency resolution during Vercel deployment.
  • Build Error (Recharts): The build failed with Module not found: Can't resolve 'react-is' originating from recharts.
    Fix: Explicitly installed react-is as a dependency.
Deployment Status: The application has been successfully deployed to Vercel production.
