Phase 1: UX Audit & Design Specification

This document analyzes the current address form implementation and outlines the transition to a Shopee-like UI/UX to improve shipping accuracy and user experience.

1. Current Implementation Audit

Feature Current State Shopee Standard Impact
Selection Flow Manual text input or limited hardcoded dropdowns. Cascading Selection (Region > Province > City > Barangay). HIGH GAP
Map Integration None (Visual) / Post-submission geocoding (Background). Interactive Map Picker with Geoapify Autocomplete. CRITICAL GAP
Data Source Hardcoded CITY_BARANGAY_MAP in frontend. Geoapify Address Autocomplete + PSGC fallback. HIGH GAP
Coordinate Accuracy Estimated via Nominatim API on submission. Precise pin-drop + Geoapify Reverse Geocoding. HIGH GAP

2. Proposed Shopee-like UI/UX Design

2.1 Cascading Address Selector

  • Step 1: Select Region/Province (Dropdown)
  • Step 2: Select City/Municipality (Dropdown - Filtered by Step 1)
  • Step 3: Select Barangay (Dropdown - Filtered by Step 2)
  • Validation: Ensures only valid combinations of address parts are submitted.

2.2 Interactive Map Picker

  • Visual Confirmation: A map widget (React Leaflet/OSM) that updates as the user selects the address.
  • Manual Pinning: Allows the user to move the pin to their exact doorstep.
  • Reverse Geocoding: Moving the pin can suggest the nearest address components.
  • Lalamove Compatibility: Directly captures lat and lng for precise shipping fee calculation.

3. Technical Strategy

To implement this, we will move away from fragmented logic in AddressModal.jsx and StoreAddressModal.jsx towards a Unified Address Component.

  • Component A: AddressSelector.jsx - Handles the cascading logic and data fetching.
  • Component B: MapPicker.jsx - Handles the Leaflet map and coordinate state.
  • Data Layer: Integrate a standardized JSON or API for PH addresses (Region/Province/City/Barangay).

4. Success Criteria

  • Users can select their exact barangay from a standardized list.
  • The latitude and longitude fields in the database are populated with 100% accuracy.
  • Shipping fee discrepancies (e.g., 73 vs 102 PHP) are eliminated by providing Lalamove with precise coordinates.
