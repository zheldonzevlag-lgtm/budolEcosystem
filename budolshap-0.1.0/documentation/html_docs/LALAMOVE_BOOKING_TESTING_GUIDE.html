🧪 Complete Testing Procedure

Phase 1: Place Order with Lalamove Shipping

1 Login to Budolshap

Navigate to https://budolshap.vercel.app

Login with test credentials:

Email: tony.stark@budolshap.com
Password: budolshap
2 Add Product to Cart

Go to /shop page

Select any product (e.g., "Apple Earbuds")

Click "Add to Cart"

Navigate to /cart

3 Select Lalamove Shipping

In the cart page, under "Shipping Method":

  • Select "Same Day Delivery via Lalamove"
  • Wait for the Lalamove widget to load
  • Click "Refresh Quote" if needed
  • Verify quote appears (e.g., "₱94.00")
  • Verify ETA is displayed
4 Select Payment Method

Under "Payment Method":

  • Select "GCash"
  • Verify total amount includes shipping cost
5 Place Order

Click "Place Order" button

You will be redirected to PayMongo GCash payment page

Note: The URL will be similar to:
https://secure-authentication.paymongo.com/sources?id=src_...
6 Complete Payment (Manual)

Since this is a test environment:

  • Note the payment URL
  • Manually mark the order as PAID in the database (or use admin panel)
  • Alternatively, complete the GCash test payment flow
⚠️ Important: The order must be marked as PAID before you can book Lalamove delivery.

Phase 2: Book Lalamove Delivery

7 Access Store Orders Dashboard

Navigate to https://budolshap.vercel.app/store/orders

You should see the newly created order at the top of the list

8 Open Order Details

Click on the order row to open the order details modal

Verify the following information:

  • Customer name and contact details
  • Product details and quantity
  • Payment method: GCASH
  • Paid status: Yes
  • Lalamove section with Quote ID
  • Service Type: MOTORCYCLE
9 Book Lalamove Delivery

Click the "Book Delivery Now" button

Expected behavior:

  • Loading toast appears: "Booking Lalamove delivery..."
  • System automatically refreshes expired quote (if needed)
  • Success toast appears: "Lalamove delivery booked successfully!"
  • Button changes to show tracking link
10 Verify Booking in Lalamove Portal

Open Lalamove Partner Portal

Login with your Lalamove sandbox credentials

Verify the booking appears in the "Records" page:

  • Status: "Assigning" (looking for driver)
  • Pickup location matches store address
  • Delivery location matches customer address
  • Contact phone number is in E.164 format (+639...)
  • Service type: Motorcycle

Phase 3: Verification & Cleanup

11 Verify Order Updates

Back in Budolshap Store Orders:

  • Refresh the order details
  • Verify booking ID is displayed
  • Verify tracking link is available
  • Click tracking link to verify it works
12 Test Edge Cases (Optional)

Additional scenarios to test:

  • Place another order to verify consistency
  • Wait 5+ minutes and book (tests quote refresh)
  • Try with different delivery addresses
  • Test with multiple items in cart

🔧 Troubleshooting

Issue: "Book Delivery Now" button not responding

Solution:
  • Refresh the page (Ctrl+F5 for hard refresh)
  • Check browser console for errors
  • Verify order is marked as PAID
  • Ensure Lalamove quote exists in order.shipping

Issue: "Cannot read properties of undefined (reading 'order')"

Solution: This was a database client import issue. Ensure the latest code is deployed with the correct named import: import { prisma } from '@/lib/prisma'

Issue: "Invalid phone number" error

Solution: Phone numbers must be in E.164 format. The system now automatically formats Philippine numbers (09... → +639...). Ensure the latest code is deployed.

Issue: "ERR_INVALID_SCHEDULE_TIME" error

Solution: The quote has expired. The system now automatically refreshes expired quotes. Ensure the latest code is deployed with the quote refresh logic.

Issue: Booking doesn't appear in Lalamove Portal

Solution:
  • Verify you're logged into the correct Lalamove account
  • Check the date filter in Lalamove Portal (set to today)
  • Verify the booking API returned success (check browser console)
  • Check Lalamove API credentials in environment variables
