Phase 1: Analysis & Identification

Problem Identification

During Lalamove booking operations, specifically for returns, the following error was encountered:

Invalid `prisma.return.update()` invocation: { data: { returnShipping: { customerOrderId: undefined, shareLink: undefined } } }

Prisma JSON fields cannot accept undefined values. If a field is optional or may not be returned by an external API, it must be explicitly set to null or a fallback value.

Root Cause Analysis

The root cause was identified in the shippingService.js file within the bookShipping and trackShipping functions. The code was directly mapping response data from the Lalamove API to Prisma update payloads without ensuring that all properties were defined.

Additionally, a property name mismatch was discovered in trackShipping, where trackingInfo.driverInfo was incorrectly referenced as trackingInfo.driver, leading to missing driver data in updates.

Affected Components

Component Type Impact
lib/services/shippingService.js Core Service Initial error in bookShipping and trackShipping.
app/api/shipping/lalamove/webhook/route.js API Route Potential failures during webhook processing for both orders and returns.
app/api/webhooks/lalamove/route.js API Route Potential failures during standard Lalamove webhook events.
services/shippingOrderUpdater.js Service Failures during background status synchronization.
app/api/orders/[orderId]/sync/route.js API Route Failures during manual status sync triggered by users/sellers.

Technical Specifications

  • Environment: Node.js with Next.js App Router
  • ORM: Prisma with PostgreSQL/MySQL
  • External API: Lalamove Delivery API
  • Solution Strategy: Implement null coalescing fallbacks (|| null) for all optional fields in JSON payloads.
