PHASE 1

Analysis & Planning

This phase involves analyzing the current layout of the AddressFormManager component and planning the restructuring required to ensure the Map is displayed below the Contact Information on mobile devices, while maintaining the side-by-side layout on desktop.

Current Layout Structure

The original implementation used a nested flex and grid structure:

  • Parent Container: flex flex-col lg:grid lg:grid-cols-2
    • Left Column (div): Contains Contact Information, Address Location, and Settings.
    • Right Column (div): Contains the Map.

On mobile, this resulted in the Map being displayed after all the sections in the Left Column (Contact, Address, and Settings).

The Problem: The Map was logically grouped with the right column on desktop, but this grouping forced it to the bottom of the stack on mobile.

Requirements

  • Mobile View: Order should be Contact Information -> Map -> Address Location -> Settings.
  • Desktop View: Maintain the two-column layout with Map on the right.
  • Responsiveness: Ensure no layout shifts or broken styles across different screen sizes.

Proposed Solution: Flattening & Grid Positioning

Instead of grouping sections into "Left" and "Right" columns using intermediate div elements, we will flatten the structure so that all four main sections are direct children of the parent grid container. We will then use Tailwind's responsive grid classes to position them:

  • Contact Info: Row 1, Col 1 (Desktop) / Order 1 (Mobile)
  • Map: Row 1-3, Col 2 (Desktop) / Order 2 (Mobile)
  • Address Location: Row 2, Col 1 (Desktop) / Order 3 (Mobile)
  • Settings: Row 3, Col 1 (Desktop) / Order 4 (Mobile)
