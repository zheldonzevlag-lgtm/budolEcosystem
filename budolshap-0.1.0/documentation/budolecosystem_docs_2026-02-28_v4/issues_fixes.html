Issues & Fixes Log (v5.3.4)

1. "Failed to update category" (Foreign Key Violation) FIXED

Error: Foreign key constraint violated on the constraint: 'Category_parentId_fkey'

Issue: Selecting "None" in the parent category dropdown sent an empty string "". Prisma rejected this because it wasn't a valid Category ID.

Fix: Implemented a transformer in the API route to convert empty strings to null, which correctly represents a top-level category in the database schema.
