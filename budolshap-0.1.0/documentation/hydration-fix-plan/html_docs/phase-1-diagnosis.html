Phase 1: Diagnosis & Preparation

Problem Statement

Error: Hydration failed because the server rendered HTML didn't match the client.

This error occurred on the /orders page of the BudolShap application. The discrepancy was found between the server-rendered "My Orders" shell and the client-rendered "Loading" spinner.

Root Cause Analysis

The root cause was identified in app/(public)/orders/page.jsx and hooks/useRealtimeBuyerOrders.js:

  • The OrdersContent component uses the useRealtimeBuyerOrders hook.
  • This hook relies on getUser() from lib/auth-client.js to get the user ID.
  • getUser() returns null on the server because it accesses localStorage, which is only available in the browser.
  • Consequently, useRealtimeBuyerOrders returns isLoading: false on the server (since no URL is generated for a null user).
  • On the client, getUser() finds the user, SWR starts fetching, and isLoading becomes true.
  • The mismatch occurs when the server renders the full page structure while the client renders the <Loading /> component.

Technical Specifications

Affected Files:

  • app/(public)/orders/page.jsx

Proposed Solution:

Implement a mounted state in the OrdersContent component to ensure that the initial render on the client matches the server render. By forcing a "Loading" state on the server and the first client-side pass, we align the hydration trees.
