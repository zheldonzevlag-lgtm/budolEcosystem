Developer Manual: Mobile Scrolling Enhancement

Purpose

To enable and optimize horizontal scrolling for critical UI elements (Product Thumbnails and Category Icons) on mobile devices, ensuring a modern, native-feeling e-commerce experience.

Rationale

E-commerce users on mobile expect to be able to swipe through product images and categories. The previous implementation used a virtualized "4-at-a-time" logic even on mobile, which hindered natural touch interaction and discoverability of content.

Technical Explanation

The implementation shifts from a strictly virtualized rendering (using slice) to a responsive hybrid approach. All items are rendered in the DOM to enable native overflow-x-auto on mobile, while Tailwind's sm:hidden and range-based logic maintain the paginated view on desktop.

Scroll Snapping: Added snap-x snap-mandatory to parent containers and snap-start to child items. This ensures that when a user swipes, the scroll position snaps to the nearest item, providing a clean and deliberate mobile UI.

Changes

1. ProductDetails.jsx

Modified the thumbnail loop to render all galleryItems. Introduced isVisibleOnDesktop logic to apply sm:hidden to items outside the current desktop view window (controlled by startIndex).

Reference: ProductDetails.jsx

2. CategoriesSection.jsx

Converted the mobile grid (grid-cols-5) to a flex container (flex overflow-x-auto). Added flex-shrink-0 and explicit mobile widths to category cards to ensure they don't collapse during horizontal scroll.

Reference: CategoriesSection.jsx

Risks & Mitigations

  • Risk: Potential layout shift on desktop if range logic is flawed.
    Mitigation: Used sm:hidden which preserves the manual startIndex pagination used by the scroll arrows.
  • Risk: Performance impact of rendering many small thumbnails.
    Mitigation: Limited to image metadata; images use Next.js Image component for lazy loading.

Compliance

N/A - This is a pure UI/UX enhancement and does not affect data processing or security controls. Performance optimizations align with general web accessibility standards.
