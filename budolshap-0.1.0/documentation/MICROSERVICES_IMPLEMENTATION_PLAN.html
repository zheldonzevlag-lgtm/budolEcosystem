Phase 1: Gateway & Foundation

Duration: 2 Weeks Focus: Routing, Logging, Standards
1.1. Setup Middleware Gateway Est: 4 Hours

Objective: Intercept every request to tag it with a unique ID and handle CORS before it hits any logic.

File: middleware.ts
TYPESCRIPT import { NextResponse } from 'next/server' import type { NextRequest } from 'next/server' export function middleware(request: NextRequest) { const requestHeaders = new Headers(request.headers) // 1. Tag Request (Traceability) requestHeaders.set('x-request-id', crypto.randomUUID()) // 2. Strict CORS (Shopee Standard) const response = NextResponse.next({ request: { headers: requestHeaders }, }) response.headers.set('Access-Control-Allow-Origin', process.env.NEXT_PUBLIC_APP_URL || '*') response.headers.set('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS') return response } export const config = { matcher: '/api/:path*', }

CLI Execution:

npm install uuid vercel env add NEXT_PUBLIC_APP_URL production https://budolshap.com

Phase 2: Identity Decoupling

Duration: 2 Weeks Focus: Auth Isolation, Schema Split
Free Tier Task: Use Prisma Multi-Schema to simulate isolation. Use schema="identity".
Pro Tier Task: Create a new Vercel Project budolshap-auth and link a dedicated Postgres Store.
2.1. Configure Prisma Schemas Est: 1 Day

Objective: Physically or logically separate User tables from Product tables.

File: prisma/schema.prisma
PRISMA generator client { provider = "prisma-client-js" previewFeatures = ["multiSchema"] } datasource db { provider = "postgresql" url = env("DATABASE_URL") schemas = ["public", "identity", "catalog", "orders"] } model User { id String @id @default(cuid()) email String @unique // ... @@schema("identity") // Crucial: Moves table to identity schema } model Product { id String @id // ... @@schema("catalog") }

CLI Execution:

npx prisma db push # Output should show: "Applying migration... to schema 'identity'"
2.2. Switch to Stateless Authentication Est: 3 Days

Objective: Stop hitting the DB for session checks. Use signed JWTs verified at the Edge.

File: lib/auth/edge-verify.ts
TYPESCRIPT import { jwtVerify } from 'jose' export async function verifyAuth(token: string) { try { const secret = new TextEncoder().encode(process.env.JWT_SECRET) const { payload } = await jwtVerify(token, secret) return payload // Contains { userId, role } } catch (err) { return null } }

Phase 4: Async Transactions

Objective: Offload "Send Email" and "Update Inventory" to background jobs to prevent checkout timeout.
