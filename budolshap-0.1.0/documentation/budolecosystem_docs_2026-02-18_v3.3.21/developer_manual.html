Developer Manual: User Profile Audit & Forensic Logging

1. Purpose & Rationale

This implementation extends the forensic audit trail to cover User Profile Modifications. Capturing changes to identity information (Name, Email, Address) is crucial for detecting account takeovers and ensuring compliance with KYC/AML regulations.

2. Technical Explanation

The system uses a centralized utility createAuditLog in lib/audit.js which:

  • Creates an immutable record in the AuditLog table.
  • Enriches the log with geolocation data (if available).
  • Calculates a SHA-256 hash of the log content for tamper evidence.
  • Triggers realtime events via Pusher for immediate admin visibility.

Webhooks are logged in a separate WebhookEvent table to track raw payloads and processing status (PENDING -> PROCESSED/FAILED).

Strict Validation Implementation

To ensure data integrity, backend services now enforce strict validation rules:

  • Names: Cannot be empty strings or whitespace only.
  • Emails: Must match standard email regex pattern.

This is implemented in:

  • budolpay-0.1.0/services/auth-service/index.js (Mobile API)
  • budolshap-0.1.0/app/api/users/route.js (Web API)

5. Compliance

  • PCI DSS: Tracks all access to cardholder data (via payment logs) and critical system changes.
  • Data Integrity: SHA-256 hashing ensures logs cannot be altered without detection.
  • Audit Trail: Complete history of user actions and system automated responses.

6. Future Enhancements

  • Implement automated alert thresholds for high-value transactions or suspicious patterns in audit logs.
  • Create a dedicated forensic dashboard for compliance officers to search and filter logs efficiently.
  • Archive old logs to cold storage after 1 year to maintain database performance.

4. Critical Logic Updates

Cart Cleanup Robustness

File: lib/services/ordersService.js

Problem: `CartItem` variationId might be stored as `""` (empty string) while `OrderItem` (or product definition) uses `null`. Strict equality checks failed to clean up cart items after checkout.

Solution: Implemented robust matching logic for `deleteMany` operations.

// Robust Cart Cleanup Logic
const cartCleanupConditions = orderItems.map(item => {
    // If variationId is null or empty, match both null and empty string in CartItem
    if (!item.variationId) {
        return {
            productId: item.productId,
            variationId: { in: [null, ''] } 
        };
    }
    return {
        productId: item.productId,
        variationId: item.variationId
    };
});
