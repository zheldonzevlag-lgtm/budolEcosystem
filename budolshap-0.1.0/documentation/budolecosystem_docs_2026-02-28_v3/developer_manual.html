Developer Manual (v5.3.3)

Auth Logic Refinement

Previously, getAuthFromCookies relied on the isAdmin field being present in the JWT payload. Some legacy tokens or tokens generated via the mobile-compatible login routes might only contain accountType: "ADMIN".

The fix in lib/auth.js now normalizes this:

const accountType = decoded.role || decoded.accountType
const isAdmin = decoded.isAdmin === true || accountType === 'ADMIN'

Standardizing Admin Routes

The Category Admin API was using a "light" auth check (cookie only). To align with PCI DSS and BSP security standards for administrative actions, it has been migrated to requireAdmin, which performs:

  1. JWT Verification
  2. Database account lookup
  3. RBAC role check (ADMIN/MANAGER/LEAD)
  4. Audit Logging of access attempts
