Risk Register: Geocoding System

Risk ID Risk Description Impact Likelihood Mitigation Strategy
GEO-01 External API Abuse (Nominatim) High (Service Block) Medium Implemented User-Agent header and client-side rate limiting (1.5s delay). Consider server-side queueing.
GEO-02 API Key Exposure (Geoapify) High (Financial Cost) Low API Key is now stored in backend settings and never exposed to the client. Only proxy calls allowed.
GEO-03 Service Outage (Provider Down) Medium (Map unusable) Low Current fallback is manual address entry. Consider implementing provider failover (Nominatim -> Geoapify).
GEO-04 Google Maps Tile API Dependency Medium (Tiles not loading) High (If JS API missing) Implemented fallback to OpenStreetMap tiles when Google Maps provider is selected but JS API is not fully integrated.
GEO-05 Commercial Provider Rate Limiting Low (Cost increase) Low Implemented logic to bypass 1.5s rate limit for commercial providers (Geoapify, Google, Radar) to ensure smooth UX while respecting quotas.
