AWS Strategic Implementation Plan

Serverless Architecture: Free Tier vs. Enterprise Scale

Phase 1: API Gateway Core

Weeks 1-2
Goal: Establish a unified entry point. Unlike Next.js Middleware, AWS API Gateway is a dedicated service.
Free Strategy (HTTP API)

Use Generic HTTP API (v2). It's cheaper ($1/million) and includes a 12-month free tier.

  • Lower latency than REST API.
  • Native JWT Authorizers included.
Scale Strategy (REST API)

Use REST API (v1) for advanced traffic management.

  • WAF Integration: Block SQLi/XSS attacks at the edge.
  • Usage Plans: Monetize API keys or enforce strict rate limits per tenant.

Phase 2: Compute & Identity (Lambda/Cognito)

Weeks 3-4
Goal: Decouple logic into micro-functions. Authentication moves to Cognito.
Free Strategy (Cognito User Pools)

Cognito has a massive free tier (50k MAUs). Use it for all auth.

Authorizer: JWT (Cognito)
Scale Strategy (Custom Authorizer)

Use a Lambda Authorizer to validate third-party tokens or implement complex ABAC (Attribute-Based Access Control) logic.

⚡ Compute Strategy

Start with AWS Lambda (Node.js 20.x runtime). It scales to zero, meaning $0 cost when idle.

Phase 3: Database (DynamoDB vs RDS)

Weeks 5-6
Free Strategy (DynamoDB)

DynamoDB (On-Demand). 25GB storage is free forever. Use Single-Table Design to minimize capacity usage.

Scale Strategy (Aurora Serverless)

Aurora Serverless v2 (Postgres) if you strictly need SQL or relational data. Auto-scales capacity units (ACUs) instantly.

Phase 4: Async & Event Driven

Weeks 7-9
Goal: The "Shopee Standard" reliability. Never loose an order.
Free Strategy (SQS Standard)

SQS Standard Queues are virtually free (1M requests/month). Use for email jobs/inventory sync.

Scale Strategy (EventBridge)

Use EventBridge for complex routing rules (e.g. "If order > $1000, send to Fraud Detection Service").
