Developer Manual - Version 427

Authentication Normalization

Fixed the getAuthFromCookies function to return a flattened user object. Previously, it returned { token, decoded }, which required nested access (user.decoded.id) and broke RBAC checks expecting user.id.

File: lib/auth.js

// Before
return { token, decoded }

// After (Normalized)
return {
    ...decoded,
    id: decoded.userId || decoded.id || decoded.sub,
    accountType: decoded.role || decoded.accountType,
    token
}

Microservice Connectivity

Corrected the hardcoded port for budolAccounting. It was pointing to 8002 (the Wallet service) instead of 8005 (the Accounting service), causing JSON parsing errors when the Wallet service returned HTML/errors for accounting requests.

File: lib/accounting.js

Ecosystem Verification Fallback

Updated BudolPay's auth service to handle tokens that use id or sub instead of email for identification, ensuring SSO users from different parts of the ecosystem can be verified correctly.

File: services/auth-service/index.js
