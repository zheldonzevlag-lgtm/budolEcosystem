🚨 Sentry Error Tracking Setup Guide

Complete guide to setting up Sentry for budolShap error monitoring

📋 Overview

Sentry is a powerful error tracking and performance monitoring tool that helps you identify, debug, and resolve issues in your application. This guide will walk you through setting up Sentry for your budolShap application.

💡 Why Sentry?
  • Real-time error notifications
  • Detailed stack traces and context
  • Performance monitoring
  • Release tracking
  • User feedback collection
  • Free tier available (5,000 events/month)

🎯 Step-by-Step Setup

1

Create a Sentry Account

Visit sentry.io/signup and create a free account. You can sign up using:

  • Email address
  • Google account
  • GitHub account
2

Create a New Project

After signing up, you'll be prompted to create your first project:

  1. Select "Next.js" as your platform
  2. Enter a project name (e.g., "budolShap Production")
  3. Choose your organization (or create one)
  4. Click "Create Project"
💡 Tip: You can create separate projects for development, staging, and production environments to better organize your error tracking.
3

Get Your DSN (Data Source Name)

After creating the project, Sentry will show you a setup wizard. On the first page, you'll see your DSN. It looks like this:

https://xxxxx@xxxxx.ingest.sentry.io/xxxxx

Copy this DSN - you'll need it in the next step. You can also find it later by:

  1. Going to Settings → Projects → [Your Project]
  2. Clicking on "Client Keys (DSN)" in the left sidebar
  3. Copying the DSN value
4

Configure in Admin Panel

Now configure Sentry in your budolShap admin panel:

  1. Navigate to Admin → Settings → Error Tracking
  2. Toggle "Enable Error Tracking" to ON
  3. Paste your Sentry DSN in the "Sentry DSN" field
  4. Select your Environment (development, staging, or production)
  5. Set Traces Sample Rate (default: 10% = 0.1)
  6. Click "Save Error Tracking Settings"
⚠️ Important: After saving, you need to restart your Next.js application for the changes to take effect. The configuration is synced to your .env.local file.
5

Restart Your Application

After saving the configuration, restart your Next.js development server or redeploy your application:

# Development npm run dev # Production npm run build npm start
✅ Verification: Once restarted, check your browser console. You should see [Sentry] Error tracking initialized if everything is configured correctly.
6

Test the Connection

In the admin panel, click the "Test Connection" button. This will:

  • Verify your DSN is valid
  • Send a test error to Sentry
  • Confirm the connection is working

Check your Sentry dashboard - you should see a test error appear within a few seconds.

⚙️ Configuration Options

Environment

Set the environment name to distinguish between different deployment stages:

  • development - Local development
  • staging - Staging/pre-production
  • production - Live production environment

Traces Sample Rate

Controls what percentage of transactions are sent to Sentry for performance monitoring:

  • 0.0 - No performance monitoring (errors only)
  • 0.1 - 10% of transactions (recommended for production)
  • 1.0 - 100% of transactions (use for development/debugging)
💡 Performance Impact: Lower sample rates reduce Sentry's impact on your application performance and help stay within free tier limits.

🔧 Advanced Configuration

Manual Environment Variable Setup

If you prefer to configure Sentry via environment variables directly, add these to your .env.local file:

# Error Tracking Configuration ERROR_TRACKING_ENABLED=true SENTRY_DSN=https://xxxxx@xxxxx.ingest.sentry.io/xxxxx SENTRY_ENVIRONMENT=production SENTRY_TRACES_SAMPLE_RATE=0.1 # Client-side (NEXT_PUBLIC_ prefix required) NEXT_PUBLIC_ERROR_TRACKING_ENABLED=true NEXT_PUBLIC_SENTRY_DSN=https://xxxxx@xxxxx.ingest.sentry.io/xxxxx NEXT_PUBLIC_SENTRY_ENVIRONMENT=production NEXT_PUBLIC_SENTRY_TRACES_SAMPLE_RATE=0.1
💡 Automatic Sync: When you save settings in the admin panel, the sync-sentry-config.mjs script automatically updates your .env.local file.

Custom Error Context

You can add custom context to errors in your code using Sentry's API:

import * as Sentry from '@sentry/nextjs'; try { // Your code } catch (error) { Sentry.setContext('order', { orderId: order.id, userId: user.id, total: order.total }); Sentry.captureException(error); }

📊 What Sentry Tracks

🚨 Errors & Exceptions

JavaScript errors, unhandled promise rejections, and API errors with full stack traces.

⚡ Performance

Page load times, API response times, and transaction performance metrics.

👤 User Context

User ID, email, IP address, and browser information for debugging.

📱 Session Replay

Video-like replay of user sessions when errors occur (on client-side).

🔔 Alerts

Email, Slack, or webhook notifications when errors exceed thresholds.

📈 Trends

Error frequency trends, affected users, and resolution tracking.

🐛 Troubleshooting

Error: "Sentry DSN is required"

Solution: Make sure you've entered a valid DSN in the admin panel and saved the settings.

Errors not appearing in Sentry

Possible causes:

  • Application not restarted after configuration
  • Error tracking disabled in admin panel
  • Invalid DSN format
  • Network/firewall blocking Sentry API

Environment variables not syncing

Solution: Run the sync script manually:

npm run sync:sentry

Exceeding free tier limits

Solution: Reduce the traces sample rate or upgrade to a paid plan. You can monitor your usage in the Sentry dashboard under Settings → Usage.
