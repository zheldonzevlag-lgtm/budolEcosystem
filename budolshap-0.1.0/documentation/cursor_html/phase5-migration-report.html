4. Architecture Changes

4.1 Before Phase 5

Public API Route → Direct Shipping Factory → Lalamove Provider
                ↓
           Database (Order Update)

4.2 After Phase 5

Public API Route → Shipping Service Layer → Shipping Factory → Provider
                ↓
           Database (Order Update)

Internal API Route → Shipping Service Layer → Shipping Factory → Provider
                ↓
           Database (Order Update)

Orders Service → HTTP → Internal Shipping API → Shipping Service → Provider
         (via serviceClient)
💡 Key Insight:
Shipping operations are now abstracted through a service layer, making it easy to:
  • Call shipping operations via HTTP (internal API)
  • Test shipping logic independently
  • Extract shipping service to separate deployment in future
  • Maintain consistent shipping handling across the application
  • Support multiple shipping providers (Lalamove, manual, future providers)
  • Integrate with orders service via HTTP

6. Implementation Details

6.1 Shipping Quote Flow

Quote Request Process:

1. Validate input (pickup, delivery, package)
2. Get shipping provider from factory
3. Prepare stops array with coordinates
4. Call provider.getQuote()
5. Store quote in order (if orderId provided)
6. Return formatted quote response

6.2 Shipping Booking Flow

Booking Process:

For Manual Shipping:
1. Validate tracking number
2. Check for existing active booking
3. Update order with shipping details
4. Send email notification
5. Return booking confirmation

For Lalamove Shipping:
1. Check for existing active booking
2. Get or refresh quote (if expired)
3. Create booking with Lalamove
4. Update order with booking details
5. Send email notification
6. Trigger real-time event
7. Return booking confirmation

6.3 Shipping Tracking

Tracking Process:

1. Get order from database
2. Validate shipping provider (Lalamove)
3. Call provider.trackOrder()
4. Update order with latest tracking info
5. Return tracking information

6.4 Shipping Cancellation

Cancellation Process:

1. Get order from database
2. Validate shipping provider and booking ID
3. Check if order can be cancelled
4. Call provider.cancelOrder()
5. Update order with cancellation info
6. Archive previous attempt
7. Return cancellation confirmation

10. Shipping Flow

10.1 Complete Shipping Flow

  1. User initiates checkout → Frontend calls POST /api/shipping/lalamove/quote
  2. Shipping Service → Gets quote from Lalamove provider
  3. Quote Storage → Quote stored in order.shipping JSON field
  4. Order Creation → Order created with shipping quote
  5. Seller Books Shipping → Frontend calls POST /api/shipping/lalamove/book
  6. Shipping Service → Creates booking with Lalamove
  7. Order Update → Order updated with booking ID and tracking URL
  8. Email Notification → Customer notified of shipping
  9. Real-time Update → Shipping status updated via Pusher/Socket.io
  10. Tracking → Customer can track via GET /api/shipping/lalamove/track/[orderId]

10.2 Service-to-Service Communication

Orders Service → HTTP → Shipping Service
  (order created)      (getShippingQuote)
                      ↓
                   (store quote in order)
                      ↓
                   Database Update

Orders Service → HTTP → Shipping Service
  (ready to ship)     (bookShipping)
                      ↓
                   (update order status)
                      ↓
                   Database Update
