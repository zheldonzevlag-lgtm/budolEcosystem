Issues & Fixes Log (v5.3.7)

1. Products not showing in Shop category filter

Status: FIXED

Issue: User added "Men's Socks" but it didn't appear under the "Socks" category in the shop.

Cause: 1. Product was accidentally assigned to "Men's Fashion" (parent) because the category selector allowed intermediate selections. 2. API caching prevented immediate updates.

Fix: Corrected product data in DB and added leaf-selection enforcement in UI.

2. Stale Product List

Status: FIXED

Issue: Shop page sometimes showed old product lists even after new products were added.

Fix: Disabled server-side caching and added client-side timestamping to product fetches.
