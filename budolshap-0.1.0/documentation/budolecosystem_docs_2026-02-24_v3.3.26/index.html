Date: February 24, 2026

Release Highlights

  • User Address Validation Fix: Resolved "Missing required fields" error in User Address creation by removing mandatory zip code check in API.
  • Store Address Fixes: Fixed issues with Store Address validation (false mandatory errors) and persistence of Label/Default settings.
  • Optional Barangay: The barangay field is now optional across all address forms, API endpoints, and database models.
  • Zod Validation: Replaced toast-based validation with robust Zod schemas in the Address Manager.
  • Prisma Client Update: Updated Prisma client output path to @prisma/client-custom-v4 to resolve file locking issues on Windows.
  • Pre-fill Logic: Enhanced address form to automatically pre-fill user details from the profile.
  • Optional Zip Code: Made zip code field optional in database and address forms.
  • Address Labels & Default: Added address labeling (Home/Work) and default address management with auto-unset logic.
  • Form UX: Disabled save button for invalid forms to prevent submission errors.
  • Store Address Validation Fix: Resolved "Missing required fields" error by making district and zip optional for store addresses.
  • Migration Deployed: Applied database migration 20260224000000_make_store_address_optional to ensure optional fields and new columns (label, isDefault) are synchronized in production.
