Shopee-like Forensic Audit Implementation Plan

Objective: Implement a forensic-grade audit trail system that records, preserves, and analyzes system activities to detect fraud, disputes, abuse, and regulatory violations, compliant with BSP and NPC standards.

Phase 1: Data Capture Layer (Event Logging)

Implementation of a centralized `AuditLogger` service to capture high-fidelity event data.

Event Type Required Fields Status
Authentication User ID, IP, Device Fingerprint, Geo-location, Auth Method (OTP/Password) PARTIAL
Financial Transaction ID, Wallet Balance (Pre/Post), Ledger Ref, Hash Chain PENDING
Data Changes Entity ID, Previous Value, New Value, Diff Snapshot PENDING

Phase 2: Centralized Log Storage & Security

  • Immutable Storage: Use database constraints to prevent `UPDATE` or `DELETE` on audit tables.
  • Encryption: Sensitive fields (PII) encrypted at rest using AES-256.
  • Retention Policy: Automated archival script for logs > 5 years (Data Lake offloading).

Phase 3: Risk & Fraud Monitoring Layer

Real-time analysis engine triggering alerts based on velocity and pattern matching.

Trigger Rules

  • Multi-Account Collusion: >3 accounts sharing same Device ID / IP within 1 hour.
  • Rapid Cycling: Order -> Cancel -> Refund cycle < 5 mins.
  • Velocity Checks: >5 failed login attempts or >3 payout requests/day.

Phase 4: Forensic Investigation Workflow

Admin tools for Case Management and Evidence Reconstruction.

  • Timeline View: Chronological reconstruction of user actions across sessions.
  • Export Evidence: CSV/PDF export with cryptographic proof (implemented).
  • Case Linking: Ability to group logs under a "Case ID" for disputes.

Regulatory Compliance (BSP/NPC)

Adherence to Philippine regulatory standards.

  • AMLA: 5-year retention for financial transaction logs.
  • Data Privacy Act: Redaction of sensitive PII in general admin views (RBAC).
  • Non-Repudiation: Digital signatures on critical logs.
