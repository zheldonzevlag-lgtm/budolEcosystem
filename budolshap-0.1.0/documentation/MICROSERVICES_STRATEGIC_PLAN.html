Shopee-Scale Strategic Plan

Phased Migration Roadmap: Free Tier vs. Pro Tier

Phase 1: Foundation & Gateway

Weeks 1-2
Goal: Establish the "Front Door". Smart routing before splitting services.
1
Setup Central API Gateway (Days 1-3) Configure `middleware.js` to act as ingress. Implement Request ID tagging.
2
Logging & Tracing (Days 4-10) Setup centralized logs.
Vercel Execution
vercel env add GATEWAY_SECRET vercel redeploy

Phase 2: Identity & Auth Decoupling

Weeks 3-4
Goal: Isolate Identity. If Catalog fails, Auth must survive.
Free Strategy (Schema Split)

Use 1 Postgres DB, split logical data into `schemas`. Free, but resources are shared.

datasource db { provider = "postgresql" schemas = ["public", "identity"] }
Pro Strategy (Physical Split)

Create a **New Vercel Project** (`auth-service`) + **New Store** (`auth-db`). True isolation.

  • Create `auth.budolshap.com` project.
  • Link dedicated Postgres Store.
  • Enable "Password Protection" (Pro) for staging.

Phase 3: Catalog & Caching

Weeks 5-6
Free Strategy (Aggressive Edge)

Rely heavily on Vercel Data Cache (shared bandwidth). Use standard ISR.

Pro Strategy (High Performance)

Use **Vercel Secure Compute** for private DB access. Higher Bandwidth (1TB) for product images.

  • Enable **Concurrent Builds** for faster iterations.
  • Use **Preview Comments** for team review.

Phase 4: Orders & Transactions

Weeks 7-9
Goal: Complex checkout flows with high reliability.
Free Strategy (Queue Dependent)

Warning: Max timeout is 10s. You MUST use queues (QStash) for email/shipping calc, or the request will time out.

Pro Strategy (Long-Running)

Relaxed Constraints. Increase functions to 60s (or 900s). Handle complex synchronous shipping calculations without queues if needed.

// functions/checkout.js export const maxDuration = 60; // Pro Only
Execution

Integrate shipping providers. On Pro, enable **Firewall Rules** to block non-PH traffic from hitting checkout endpoint.

Phase 5-6: Payments & Security

Weeks 10-14
Free Strategy

Basic DDoS protection. Public Git interaction.

Pro Strategy (Enterprise Grade)
  • WAF: Custom Firewall Rules to block bots.
  • SAML SSO: For your Admin Dashboard login (optional).
  • Code Owners: Enforce review on payment files on GitHub.
