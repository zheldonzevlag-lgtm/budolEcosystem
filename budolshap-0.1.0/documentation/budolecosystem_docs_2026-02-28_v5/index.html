v5.3.5 Release Notes

Cache Synchronization Fix

Date: 2026-02-28

Executive Summary

This release fixes an issue where newly created categories (such as "Socks") were visible in the Admin portal but missing from the "Add Product" category picker. The root cause was identified as aggressive browser/Next.js caching of the public category list. We have implemented multi-layer cache busting to ensure instant synchronization.

Key Fixes

  • Route De-caching: Forced /api/categories to render dynamically (force-dynamic) on every request.
  • Frontend Cache-Bust: Added t=${Date.now()} and no-store headers to the CategorySelector fetch logic.
  • Slug Uniqueness: Verified that the "Socks" category uses a timestamp-appended slug to maintain global uniqueness across hierarchies.
