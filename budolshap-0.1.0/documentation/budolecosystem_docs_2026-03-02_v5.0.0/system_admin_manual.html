Purpose

This manual provides essential information for system administrators responsible for deploying, configuring, monitoring, and maintaining the BudolEcosystem application. It covers operational procedures, troubleshooting guidelines, and security considerations.

Deployment

Prerequisites

  • Node.js (LTS version recommended)
  • npm or Yarn
  • Database (PostgreSQL recommended)
  • Environment variables configured (see Configuration section)

Steps

  1. Clone the repository: git clone [repository-url]
  2. Navigate to the project directory: cd budolshap-0.1.0
  3. Install dependencies: npm install
  4. Configure environment variables (see Configuration).
  5. Run Prisma migrations: npx prisma migrate deploy
  6. Build the application: npm run build
  7. Start the application: npm start

Configuration

The application relies on environment variables for sensitive information and configuration settings. A .env.local file is used for local development, and appropriate environment variables should be set in the production environment.

Key Environment Variables:

  • DATABASE_URL: Connection string for the PostgreSQL database.
  • NEXTAUTH_SECRET: Secret for NextAuth.js.
  • NEXTAUTH_URL: URL of the application.
  • PAYMONGO_SECRET_KEY: API secret key for PayMongo.
  • BUDOLPAY_API_KEY: API key for BudolPay.
  • PAYMENT_GATEWAY_URL: URL for the BudolPay API Gateway.
  • GEOAPIFY_API_KEY: API key for Geoapify geocoding service.
  • PUSHER_APP_ID, PUSHER_KEY, PUSHER_SECRET, PUSHER_CLUSTER: Credentials for Pusher for real-time functionalities.

Monitoring

Key areas to monitor for application health and performance:

  • **Application Logs:** Monitor application logs for errors, warnings, and critical events.
  • **Database Performance:** Track database connection pool, query performance, and disk usage.
  • **Server Resources:** Monitor CPU, memory, and network usage of the application server.
  • **Payment Gateway Status:** Regularly check the status of integrated payment gateways (PayMongo, BudolPay).

Troubleshooting

  • **Application Not Starting:**
    • Check environment variables for correct values.
    • Verify database connection.
    • Review build logs for errors.
  • **Payment Failures:**
    • Check payment gateway logs for specific error codes.
    • Verify API keys and endpoint URLs.
    • Ensure webhook endpoints are correctly configured and accessible.
  • **Real-time Updates Not Working:**
    • Verify Pusher credentials and connectivity.
    • Check network connectivity to Pusher servers.
    • Ensure `realtimeProvider` setting in the database is correct.

Security Considerations

  • Regularly update dependencies to patch known vulnerabilities.
  • Ensure all sensitive data (API keys, database credentials) are stored securely using environment variables and not committed to version control.
  • Implement strong access control policies for administrative interfaces.
  • Conduct regular security audits and penetration testing.

Compliance

The system adheres to the following compliance standards:

  • **PCI DSS:** For secure handling of payment card information.
  • **BSP (Bangko Sentral ng Pilipinas):** Regulations for financial transactions in the Philippines.
  • **BIR (Bureau of Internal Revenue):** Requirements for financial record-keeping and reporting.
  • **NPC (National Privacy Commission):** Data privacy regulations in the Philippines.
  • **DTI (Department of Trade and Industry):** Regulations for e-commerce and consumer protection.
  • **Cybersecurity Law:** Adherence to national cybersecurity frameworks.
