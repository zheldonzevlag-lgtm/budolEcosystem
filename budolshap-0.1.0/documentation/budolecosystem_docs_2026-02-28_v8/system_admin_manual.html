Administrator Action: Data Consistency Check

The "Starts from ₱0.00" issue was triggered because some products were uploaded with a price of 0. System administrators should periodically audit the Product table to ensure that active products have non-zero prices if they are intended for sale.

Audit Query

Run this query via Prisma Studio or Database Console to identify products that might cause the issue:

SELECT id, name, price, inStock FROM Product WHERE price = 0 AND inStock = true;

Corrective Action

  • Update the price of the product via the Seller/Admin portal.
  • Alternatively, set inStock to false for test or incomplete products.
  • Ensure that the storefront's process.env.NEXT_PUBLIC_CURRENCY_SYMBOL is correctly configured in Vercel/Local .env file.
