Mitigation Approach: Hero Section Price Fix

The primary mitigation for Risk R001-PZ (Incorrect Price Calculation) involves a multi-layered approach to ensure data integrity and display accuracy.

Frontend Mitigation

  • Logic Guard: The Math.min() operation is restricted to a filtered array of products where price > 0.
  • Initial Guard: The system defaults minPrice to 0 and shows a placeholder during loading to prevent flickering with 0.00.
  • Type Casting: Prices are explicitly cast to numbers using Number(product.price) before calculation to handle any string-stored values from the database or API.

Testing & Verification

  • Unit Tests: Jest-based testing in __tests__/hero-price-logic.test.js covers edge cases (0 prices, all 0 prices, empty lists).
  • Data Audit: Manual check of the database for products with 0 prices confirms the logic is working against existing test data.

Future Proofing

A data-entry level mitigation is recommended where sellers are warned or prevented from setting prices to 0 unless specifically marked as "Free Promotion" items.
