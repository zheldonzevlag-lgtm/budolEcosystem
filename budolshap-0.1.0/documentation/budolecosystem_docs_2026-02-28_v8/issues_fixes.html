Reported Issues and Applied Solutions

Issue: "Starts from ₱0.00" Display in Hero Section

Description: The Hero Section of the shop displayed a starting price of ₱0.00 even when products with higher prices were present. This was caused by the Math.min() function picking up products with a value of 0 in the product.list state.

Root Cause: Lack of price filtering in the frontend calculation logic within Hero.jsx. Incomplete test data (e.g., "Smart Watch", "Men's Socks") with price 0 triggered the issue.

Fix: Updated Hero.jsx to filter out prices equal to 0 before calculating the minimum price.

Issue: Currency Display Sync

Description: Ensuring that the currency symbol in the "Starts from" price matches the system-wide currency symbol (Philippine Peso ₱).

Fix: Ensured that currency state in Hero.jsx is initialized from process.env.NEXT_PUBLIC_CURRENCY_SYMBOL upon mounting.
