Proposed Future Enhancements

1. Data Validation at Creation

Implement a validation rule in app/api/products/route.js and the frontend's add-product-wizard to prevent products from being saved with a 0 price unless it is explicitly marked as a "free" product.

2. Dynamic "Starting from" by Category

Enhance the Hero section to show "Starting from" prices based on the most active or featured category instead of a global shop minimum, providing more relevant entry points for specific user interests.

3. "Free" Badge

If a product is intentionally 0, it should be displayed with a "FREE" badge instead of a ₱0.00 price tag across the marketplace.
