Developer Manual: Google Maps Integration

Purpose

To provide users with a native Google Maps experience, replacing the OpenStreetMap (OSM) fallback tiles when the Google Maps provider is selected. This ensures better accuracy, familiar UI, and access to Google's Places data.

Technical Explanation

The integration involves two main components:

  • GoogleMapView.jsx: A wrapper around the Google Maps JavaScript API. It handles script loading, map initialization, and exposes methods (zoom, flyTo) via a forward ref.
  • MapPicker.jsx: The main container that conditionally renders either Leaflet (for OSM/Radar/Geoapify) or GoogleMapView based on the selected provider.

Configuration & Troubleshooting

Ensure that the map provider is correctly set in the Admin Panel. The system uses standardized IDs:

  • OSM: OpenStreetMap
  • GOOGLE_MAPS: Google Maps (Note: previously 'GOOGLE', now standardized)
  • GEOAPIFY: Geoapify
  • RADAR: Radar

The useMapSettings hook handles fetching these settings. Ensure the hook correctly parses the API response (direct object vs nested settings object).

GoogleMapView Component

Responsible for rendering the Google Map. It uses useImperativeHandle to expose control methods to the parent.
