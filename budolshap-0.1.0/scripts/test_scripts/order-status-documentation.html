The Order Status Dropdown

Understanding the seller-side manual control system for order fulfillment in BudolShap.

🎯

Purpose of the Dropdown

The status dropdown is a manual state-transition tool. While modern couriers (like Lalamove) provide automated webhooks, many critical business steps require human confirmation from the seller before the system can engage the next logistics layer.

It acts as a safety valve, ensuring that automation only happens when the physical package is actually ready.

⚙️

Transition Logic (The Workflow)

→ Mark Processing

Available when an order is PAID or ORDER_PLACED.

Signals to the buyer that the seller has seen the request and is currently picking/packing items.

→ Mark Ready for Pickup

Available when status is PROCESSING.

The "Fulfillment Trigger". This enables the delivery booking UI (Review & Book) where sellers can choose couriers.

→ Mark Delivered

Available for SHIPPING or TO_SHIP.

A manual override for manual riders or cases where the courier's automatic API fails to update.

💎

Seller Benefits

✓
Buyer Transparency

Reduces "When will you ship?" messages by providing real-time granular updates to the buyer UI.

✓
Error Prevention

The dropdown is context-aware. It prevents marking an order "Delivered" if it hasn't even been packed yet.

✓
Logistics Readiness

Standardizes the data flow so that external APIs (Lalamove/BudolShap Shipping) receive valid requests.

🔒

Finalized States

Once an order reaches Delivered, Cancelled, or Refunded, the dropdown is removed and replaced by a static badge. This protects the integrity of the transaction data and prevents accidental "resurrecting" of closed orders.
