🔄 Lalamove Integration - Phase 3 Processes

Detailed Process Flows & Implementation Guide

📋 Phase 3 Process Overview

Phase 3 focuses on integrating the Lalamove backend API into the frontend user interface. This document outlines the key processes and workflows that need to be implemented.

  • Checkout Integration: Allow users to select Lalamove and get real-time quotes
  • Order Tracking: Display real-time delivery status and driver information
  • Admin Management: Enable admins to manage and monitor Lalamove deliveries
  • Store Integration: Allow store owners to prepare orders for pickup

🛒 Process 1: Checkout Flow with Lalamove

1

User Enters Delivery Address

Customer fills in their delivery address in the checkout form.

  • Validate address fields are complete
  • Ensure street, city, postal code are provided
  • Use address autocomplete for accuracy (if available)
  • Store coordinates if available from geocoding
2

Display Shipping Options

Show available shipping methods including Lalamove.

  • Display standard shipping option
  • Display Lalamove same-day delivery option
  • Show Lalamove logo and description
  • Add "Get Quote" button for Lalamove
3

User Clicks "Get Quote"

Fetch real-time shipping quote from Lalamove API.

API Call:

POST /api/lalamove/quote
// Request payload { "pickupAddress": { "street": "123 Store St", "city": "Manila", "coordinates": { "lat": 14.5995, "lng": 120.9842 } }, "deliveryAddress": { "street": "456 Customer Ave", "city": "Quezon City", "coordinates": { "lat": 14.6760, "lng": 121.0437 } } }
4

Display Quote to User

Show the quoted price and estimated delivery time.

  • Display shipping cost (e.g., ₱150.00)
  • Show estimated delivery time (e.g., "2-3 hours")
  • Cache quote for 5 minutes to avoid repeated API calls
  • Handle errors (out of service area, API failures)

⚠️ Error Handling

If the address is outside Lalamove's service area, show a friendly message: "Sorry, Lalamove delivery is not available for this address. Please choose standard shipping."

5

User Selects Lalamove & Proceeds to Payment

Customer confirms Lalamove shipping and completes payment.

  • Add Lalamove shipping cost to order total
  • Store quote ID for later booking
  • Proceed to PayMongo payment
6

Payment Success - Book Lalamove Delivery

After successful payment, automatically book the Lalamove delivery.

API Call:

POST /api/lalamove/book
// Request payload { "orderId": "order_123", "quoteId": "quote_abc", "pickupAddress": { ... }, "deliveryAddress": { ... }, "customerName": "John Doe", "customerPhone": "+639171234567" }
7

Store Lalamove Order Details

Save Lalamove order ID and details in database.

  • Update order with lalamoveOrderId
  • Set initial status to "ASSIGNING_DRIVER"
  • Store tracking URL
  • Send confirmation email to customer

✅ Success State

Show order confirmation page with Lalamove tracking information and estimated delivery time.

ℹ️ Implementation Notes

  • Implement loading states during quote fetching (show skeleton loader)
  • Add retry mechanism for failed API calls
  • Validate that pickup address (store location) is configured in environment variables
  • Consider adding a "Refresh Quote" button if quote is older than 5 minutes

📍 Process 2: Real-Time Order Tracking

1

Customer Views Order Details

User navigates to their order details page.

  • Display order information
  • Check if order has Lalamove delivery
  • Show Lalamove tracking section if applicable
2

Fetch Current Delivery Status

Get latest status from Lalamove API.

API Call:

GET /api/lalamove/track?orderId={lalamoveOrderId}
// Response { "status": "ON_GOING", "driver": { "name": "Juan Dela Cruz", "phone": "+639171234567", "photo": "https://...", "plateNumber": "ABC 1234" }, "estimatedArrival": "2025-11-27T14:30:00Z", "currentLocation": { "lat": 14.6500, "lng": 121.0300 } }
3

Display Delivery Status

Show current status with visual timeline.

Order Placed

Your order has been confirmed and payment received.

Driver Assigned

A driver has been assigned to pick up your order.

Picked Up

Driver has picked up your order and is on the way.

Out for Delivery

Your order is currently being delivered.

Delivered

Your order has been successfully delivered.

4

Show Driver Information

Display driver details when available.

  • Driver name and photo
  • Contact number (click to call)
  • Vehicle plate number
  • Estimated arrival time
5

Real-Time Updates via Webhook

Receive automatic status updates from Lalamove.

  • Webhook receives status change notification
  • Update order status in database
  • Send push notification to customer (optional)
  • Update UI if customer is viewing the page

ℹ️ Webhook Processing

The webhook endpoint (/api/lalamove/webhook) automatically processes status updates. Frontend can poll the API every 30 seconds for updates, or implement WebSocket for real-time updates.

6

Delivery Completion

Handle successful delivery.

  • Update order status to "DELIVERED"
  • Show delivery confirmation message
  • Request customer feedback/rating (optional)
  • Send delivery confirmation email

⚙️ Process 3: Admin Order Management

1

View Lalamove Orders

Admin views all orders with Lalamove delivery.

  • Add filter: "Lalamove Orders Only"
  • Display Lalamove status badge on each order
  • Show quick stats (total Lalamove orders, in-transit, delivered)
2

View Order Details

Admin clicks on an order to view full details.

  • Show all order information
  • Display Lalamove tracking section
  • Show driver information if available
  • Display delivery timeline
3

Cancel Delivery (If Needed)

Admin can cancel Lalamove delivery if necessary.

API Call:

POST /api/lalamove/cancel

⚠️ Cancellation Rules

Lalamove deliveries can only be cancelled before driver picks up the order. After pickup, cancellation may incur fees or may not be possible.

  • Show confirmation dialog before cancellation
  • Call cancel API endpoint
  • Update order status
  • Notify customer of cancellation
  • Process refund if applicable
4

View Analytics

Admin views Lalamove delivery analytics.

  • Total Lalamove orders this month
  • Success rate (delivered vs cancelled)
  • Average delivery time
  • Total shipping costs
  • Revenue from Lalamove orders

🏪 Process 4: Store Owner Preparation

1

Receive New Order Notification

Store owner is notified of new Lalamove order.

  • Email notification with order details
  • In-app notification (if implemented)
  • Highlight that it's a Lalamove same-day delivery
2

Prepare Order for Pickup

Store owner prepares items for driver pickup.

  • View order items and quantities
  • Pack items securely
  • Prepare invoice/receipt
  • Wait for driver assignment notification
3

Mark Order as Ready

Store owner marks order as ready for pickup.

  • Click "Ready for Pickup" button
  • Update order status in database
  • Notify customer that order is being prepared

ℹ️ Optional Feature

This step is optional but helps improve customer experience by providing more accurate status updates.

4

Driver Arrives for Pickup

Lalamove driver arrives at store location.

  • Store owner receives notification of driver arrival
  • View driver details (name, phone, vehicle)
  • Hand over package to driver
  • Confirm pickup (if required)
5

Track Delivery Progress

Store owner can monitor delivery status.

  • View real-time delivery status
  • See when order is delivered
  • Receive delivery confirmation

⚠️ Error Handling & Edge Cases

Common Errors & Solutions

Error Scenario User Message Action
Address outside service area "Lalamove delivery is not available for this location" Disable Lalamove option, show standard shipping only
Quote API timeout "Unable to get quote. Please try again." Show retry button, log error for debugging
Booking fails after payment "Payment successful, but delivery booking failed. Our team will contact you." Create order with pending status, notify admin, attempt manual booking
No drivers available "No drivers available at the moment. We'll keep trying." Retry booking automatically, notify customer of delay
Delivery cancelled by driver "Delivery was cancelled. We're finding another driver." Attempt to rebook automatically, notify customer and admin
Webhook signature invalid N/A (internal error) Reject webhook, log security alert

⚠️ Critical Error Handling

Always ensure that payment is processed BEFORE booking Lalamove delivery. If booking fails after payment, the order should still be created with a "pending_delivery" status, and the admin should be notified to manually arrange delivery or refund.
