  • Task 2.1: Implement Webhook Receiver MEDIUM PRIORITY

    File: app/api/shipping/lalamove/webhook/route.js (Create new)

    Objective: Receive real-time status updates from Lalamove.

    • Create POST endpoint at /api/shipping/lalamove/webhook
    • Verify webhook signature using LALAMOVE_WEBHOOK_SECRET
    • Parse webhook payload and extract order status updates
    • Update order status in database based on webhook data
    • Handle different update types: ORDER_STATUS_CHANGED, DRIVER_ASSIGNED, ORDER_AMOUNT_CHANGED
    • Return 200 OK to acknowledge receipt
    // app/api/shipping/lalamove/webhook/route.js import { NextResponse } from 'next/server'; import crypto from 'crypto'; import prisma from '@/lib/prisma'; export async function POST(request) { try { const body = await request.text(); const signature = request.headers.get('X-Lalamove-Signature'); // Verify webhook signature const webhookSecret = process.env.LALAMOVE_WEBHOOK_SECRET; const hmac = crypto.createHmac('sha256', webhookSecret); const expectedSignature = hmac.update(body).digest('hex'); if (signature !== expectedSignature) { console.error('[Lalamove Webhook] Invalid signature'); return NextResponse.json( { error: 'Invalid signature' }, { status: 401 } ); } const payload = JSON.parse(body); const { updateType, data } = payload; console.log('[Lalamove Webhook] Received:', updateType, data); // Handle different update types switch (updateType) { case 'ORDER_STATUS_CHANGED': await handleStatusChange(data); break; case 'DRIVER_ASSIGNED': await handleDriverAssigned(data); break; case 'ORDER_AMOUNT_CHANGED': await handleAmountChange(data); break; default: console.log('[Lalamove Webhook] Unknown update type:', updateType); } return NextResponse.json({ received: true }); } catch (error) { console.error('[Lalamove Webhook] Error:', error); return NextResponse.json( { error: 'Webhook processing failed' }, { status: 500 } ); } } async function handleStatusChange(data) { const { orderId, status } = data; // Find order by Lalamove orderId const order = await prisma.order.findFirst({ where: { shipping: { path: ['bookingId'], equals: orderId } } }); if (order) { await prisma.order.update({ where: { id: order.id }, data: { shipping: { ...order.shipping, status: status, updatedAt: new Date().toISOString() } } }); } }

📚 Lalamove API Reference

Order Status Flow

Order Status Progression

Understanding the order lifecycle is crucial for proper status tracking:

  1. ASSIGNING_DRIVER - Order created, searching for driver
  2. ON_GOING - Driver assigned and heading to pickup
  3. PICKED_UP - Driver has picked up the package
  4. COMPLETED - Package delivered successfully
  5. CANCELED - Order cancelled by user or system
  6. REJECTED - No driver accepted (max rejections reached)
  7. EXPIRED - Order expired before driver assignment

⚠️ Common Issues & Solutions

Issue #1: Invalid stopId Error

Error: ERR_INVALID_STOP_ID or ERR_UNKNOWN_FIELD

Cause: Using hardcoded stopIds ('0', '1') instead of actual stopIds from quote response.

Solution: Always extract stopIds from the quote response (quote.stops[0].stopId, quote.stops[1].stopId) and use those exact values when creating the order.

Issue #2: Quote Expired Error

Error: ERR_INVALID_QUOTE

Cause: Quotes expire 15 minutes after creation.

Solution: Check quote.expiresAt before creating order. If expired, fetch a new quote.

Issue #3: Insufficient Credit

Error: ERR_INSUFFICIENT_CREDIT (HTTP 402)

Cause: Lalamove wallet balance is too low.

Solution: Top up wallet in Partner Portal. Implement balance monitoring and alerts.

Issue #4: Invalid Phone Number

Error: ERR_INVALID_PHONE

Cause: Phone number not in correct format (must include country code).

Solution: Ensure phone numbers are in format: +639171234567 (PH: +63 prefix)

Issue #5: Webhook Not Receiving Updates

Cause: Webhook URL not configured or not publicly accessible.

Solution:

  • Verify webhook URL is configured in Partner Portal
  • Ensure URL is publicly accessible (use ngrok for local testing)
  • Check webhook endpoint returns 200 OK
  • Verify signature validation is correct
  • Check Partner Portal → Developers → Webhook Attempts for errors
