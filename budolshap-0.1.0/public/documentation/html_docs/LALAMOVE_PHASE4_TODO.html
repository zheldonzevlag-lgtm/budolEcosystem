End-to-End Quote API Testing
Critical
Test the complete quote flow with real Lalamove Sandbox API using various address combinations and service types.
Endpoint:
POST /api/shipping/lalamove/quote
Test Cases:
Valid addresses, missing coordinates (geocoding fallback), different service types, invalid addresses, API errors
Expected Results:
Accurate quotes, proper error handling, geocoding success, reasonable delivery times

  • Test with addresses missing coordinates (trigger geocoding)
  • Test different service types (MOTORCYCLE, SEDAN, MPV, VAN)
  • Test with invalid/incomplete addresses
  • Verify quote response format and data accuracy
  • Check error handling for API failures
  • Measure response time (should be < 3 seconds)
2
Booking API Testing
Critical
Test the booking flow from store dashboard with real Lalamove Sandbox API to ensure deliveries are created successfully.
Endpoint:
POST /api/shipping/lalamove/book
Prerequisites:
Valid order with Lalamove shipping, store owner authentication, valid quote ID
Verification:
Booking ID returned, tracking link generated, order updated in database, idempotency working
  • Create test order with Lalamove shipping
  • Book delivery from store dashboard
  • Verify booking ID and tracking link returned
  • Check database update (Order.shipping field)
  • Test idempotency (prevent duplicate bookings)
  • Test booking with expired quote ID
  • Verify error handling for booking failures

  • Verify status updates (ASSIGNING_DRIVER, ON_GOING, etc.)

⚠️ Important Notes & Warnings

🔴 Critical Warnings
  • Sandbox vs Production: Always test in Sandbox first. Production API creates real deliveries with real costs!
  • Webhook Security: Never disable signature verification in production. This prevents unauthorized access.
  • Rate Limiting: Nominatim API has rate limits (1 request/second). Implement caching to avoid bans.
  • Idempotency: Always use unique client reference IDs to prevent duplicate bookings.
  • Error Logging: Monitor error logs closely in production. Set up alerts for critical failures.
💡 Best Practices
  • Test with various address formats and edge cases
  • Keep detailed logs of all API interactions for debugging
  • Implement graceful degradation if Lalamove API is unavailable
  • Cache geocoding results to improve performance
  • Provide clear user feedback for all operations
  • Document all configuration changes and deployment steps
  • Set up monitoring and alerting for production issues
