Audit Trail Enhancement: City Geolocation

PLANNED

Date: 2025-12-12
Objective: Capture and display the City level geolocation for accurate audit tracking.

Feature Overview

Currently, the Audit Trail captures the Country using Vercel headers. The goal is to increase granularity by capturing the City as well, providing admins with precise location data for security events (login/logout).

Implementation Plan

Step 1: Database Schema Update

Goal: Add a storage column for the city data.

Action: Modify prisma/schema.prisma to add the city field to the AuditLog model.

model AuditLog {
    id        String   @id @default(cuid())
    // ... existing fields
    country   String?
    city      String?  // [NEW] To store "Manila", "Cebu City", etc.
    // ... existing fields
}

Step 2: Backend Logic Update

Goal: Capture the header data.

Action: Update lib/audit.js to extract the x-vercel-ip-city header.

// Inside createAuditLog function
const city = request.headers.get('x-vercel-ip-city') || 'Unknown';
await prisma.auditLog.create({
    data: {
        // ...
        city,
        // ...
    }
});

Step 3: Admin UI Update

Goal: Display the new data point.

Action: Update app/admin/settings/audit-logs/page.jsx table structure.

  • Add a "City" column header.
  • Display the log.city value alongside the Country.

Technical Considerations

  • Accuracy: Relies on the x-vercel-ip-city header provided by Vercel's edge network.
  • Fallback: If the header is missing (e.g., local development), it will default to "Unknown".
  • Data Loss: Existing logs created before this update will have a null or empty City value.
