PayMongo Local Redirect Fix

Goal

Ensure PayMongo redirects to localhost:3000 (or the configured local URL) when running locally, instead of the production Vercel URL. This fixes issues with Maya, GrabPay, and GCash payments in development environments.

Proposed Changes

Backend

MODIFY app/api/payment/checkout/route.js

  • Modify the baseUrl determination logic.
  • Explicitly check if request.headers.get('host') contains 'localhost' or '127.0.0.1'.
  • If it does, use http://${host} as the baseUrl, IGNORING process.env.NEXT_PUBLIC_BASE_URL. This guards against cases where production env vars are loaded locally.
  • Keep the fallback to NEXT_PUBLIC_BASE_URL or VERCEL_URL for non-local environments.

MODIFY app/api/paymongo/create-intent/route.js

  • Apply the same logic: Prioritize host header check for localhost over NODE_ENV or NEXT_PUBLIC_BASE_URL.

Verification Plan

Manual Verification

  • User to initiate a payment (GCash, Maya, or GrabPay) locally.
  • Verify that successUrl passed to PayMongo (visible in logs) is http://localhost:3000/payment/return.
  • User validates the actual redirect after payment works correctly and leads to the local application.
