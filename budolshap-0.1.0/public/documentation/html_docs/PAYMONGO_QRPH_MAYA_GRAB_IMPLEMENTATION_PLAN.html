🚀 PayMongo Integration Plan: QRPh, Maya, Grab

⏱️ Total Estimated Duration: 3-4 Days

This plan updates the previous strategy to use a Gateway Agnostic Architecture (Adapter Pattern). It is broken down into safe, testable phases to ensure the existing GCash integration remains 100% functional throughout the process.

🛡️ Safety First Strategy

CRITICAL Protecting Existing GCash Flow

To guarantee no downtime for the current GCash payment method:

  • No modifications will be made to app/api/payment/gcash/create/route.js initially. It will continue to use the legacy lib/paymongo.js.
  • The new Unified API (/api/payment/checkout) will be built alongside the existing one.
  • We will first implement the new methods (Maya, Grab, QRPh) using the new architecture.
  • Only after the new architecture is battle-tested will we (optionally) migrate the GCash flow to use it, or simply leave the working legacy flow as is.

📅 Phased Implementation Timeline

Phase 1: Foundation & Adapter Core

1 Day

Goal: set up the generic payment structure without touching existing code.

  • Create lib/payment/adapters/base-adapter.js (Interface).
  • Create lib/payment/adapters/paymongo-adapter.js (New Logic).
  • Create lib/payment/service.js (Service Layer).
  • Create app/api/payment/checkout/route.js (New Endpoint).

Phase 2: Frontend Integration (Maya, Grab, QRPh)

1-2 Days

Goal: Allow users to select new payment methods.

  • Create components/payment/PaymentMethodSelector.jsx.
  • Implement UI for selecting methods (Card/Radio style).
  • Connect this component to the New Endpoint (Plan 1).
  • Test: Verify Maya/Grab/QRPh redirects work correctly.

Phase 3: Verification & Polish

1 Day

Goal: Ensure seamless UX and verify webhooks.

  • Ensure payment/success and payment/failed pages handle the new source types correctly.
  • Verify /api/webhooks/paymongo can handle events from these new methods (PayMongo events are generally consistent).
  • Deliverable: Fully functional QRPh, Maya, and Grab payment options alongside the existing GCash button.

✅ Deliverables

  • New Files: lib/payment/*, components/payment/PaymentMethodSelector.jsx
  • Unchanged Files (Safety): lib/paymongo.js, app/api/payment/gcash/create/route.js
  • Outcome: A robust, extensible payment system that adds 3 new payment methods without risking the stability of the primary revenue channel (GCash).
