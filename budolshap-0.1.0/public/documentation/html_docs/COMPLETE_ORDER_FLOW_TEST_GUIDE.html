🧪 Complete Order Flow Test Guide

GCash Payment + Lalamove Delivery Integration Test

📋 Prerequisites

⚠️ Important: Make sure all prerequisites are met before starting the test!
  • Development server is running (npm run dev)
  • Valid Lalamove API credentials configured in .env.local
  • Valid PayMongo API credentials configured in .env.local
  • Test user account created (ui_tester_2@example.com or similar)
  • At least one active store with products
  • User has a saved delivery address
  • Lalamove wallet has sufficient credit (for sandbox/production)

🔄 Complete Test Flow

1 Login to Application

Navigate to your application and login with test credentials:

http://localhost:3000/login

Test Credentials:

  • Email: ui_tester_2@example.com
  • Password: password123
2 Browse and Select Products

Browse available products and add items to your cart:

http://localhost:3000/products
  • Select at least one product
  • Click "Add to Cart"
  • Verify cart icon shows item count
3 Proceed to Checkout

Navigate to cart and proceed to checkout:

http://localhost:3000/cart
  • Review cart items
  • Verify quantities and prices
  • Click "Proceed to Checkout"
4 Select Delivery Address

Choose or add a delivery address:

  • Select existing address or add new one
  • Ensure address has complete details (street, city, phone)
  • Verify address is within Lalamove service area (Metro Manila)
Note: For testing, use addresses within Manila/Makati area for best results.
5 Select Lalamove Delivery

Choose Lalamove as shipping provider:

  • Click "Get Lalamove Quote" or similar button
  • Wait for quote to load (shows price and ETA)
  • Verify quote displays correctly
  • Note the shipping cost
⚠️ Quote expires after a certain time. If checkout takes too long, request a new quote.
6 Select GCash Payment

Choose GCash as payment method:

  • Select "GCash" payment option
  • Review order summary (items + shipping)
  • Verify total amount is correct
7 Place Order

Submit the order:

  • Click "Place Order" button
  • Wait for order creation
  • Should redirect to GCash payment page
Expected Result: Order is created in database with status "pending" and payment status "pending"
8 Complete GCash Payment

Complete payment through PayMongo/GCash:

  • You'll be redirected to PayMongo checkout page
  • Click "Pay with GCash"
  • Complete payment in GCash app or test mode
  • Wait for payment confirmation
⚠️ For Testing: Use PayMongo test mode. Payment will be simulated.
Test Card Numbers: PayMongo provides test credentials for sandbox testing.
9 Verify Payment Webhook

Check that payment webhook updated the order:

  • Navigate to your orders page
  • Find the newly created order
  • Verify payment status changed to "paid"
  • Verify isPaid is true
http://localhost:3000/orders
Expected Result: Order shows as "Paid" with payment confirmation
10 Book Lalamove Delivery (Store Action)

Store owner books the Lalamove delivery:

  • Login as store owner/seller
  • Navigate to Store Orders page
  • Find the paid order
  • Click "Book Delivery Now" or similar button
http://localhost:3000/store/orders
⚠️ Real Order: This creates an actual Lalamove order and charges your account!
11 Verify Lalamove Order Creation

Confirm order appears in Lalamove Partner Portal:

  • Open Lalamove Partner Portal
  • Navigate to Records/Orders section
  • Verify new order appears in list
  • Check order details match
https://partnerportal.lalamove.com/records
Expected Result: Order visible in portal with status "ASSIGNING_DRIVER" or "ACCEPTED"
12 Monitor Driver Assignment

Wait for driver to accept the order:

  • In Lalamove portal, watch order status
  • Status should change from "ASSIGNING_DRIVER" to "ACCEPTED"
  • Driver details will appear (name, phone, vehicle)
  • In your app, order status should update via webhook
Note: In sandbox mode, driver assignment may be simulated. In production, wait for real driver.
13 Track Delivery Progress

Monitor delivery in real-time:

  • Use Lalamove tracking link in your app
  • Watch status updates: PICKING_UP → ON_GOING → COMPLETED
  • View driver location on map (if available)
  • Receive notifications for status changes
14 Verify Delivery Completion

Confirm successful delivery:

  • Lalamove order status changes to "COMPLETED"
  • Webhook updates your order status to "delivered"
  • Customer receives delivery confirmation
  • Store receives completion notification
Test Complete! Order successfully placed, paid, and delivered.

🔧 Troubleshooting

Issue: Order not appearing in Lalamove Portal

Possible Causes:

  • Missing stopIds in booking request
  • Invalid API credentials
  • Insufficient wallet credit
  • Address outside service area

Solution:

  • Check browser console for API errors
  • Verify .env.local has correct Lalamove credentials
  • Check Lalamove wallet balance
  • Use addresses within Metro Manila
Issue: Payment not updating order

Possible Causes:

  • Webhook not configured in PayMongo
  • Webhook endpoint not accessible
  • Webhook signature verification failing

Solution:

  • Check PayMongo dashboard for webhook events
  • Verify webhook URL is publicly accessible
  • Check webhook handler logs for errors
  • Use ngrok for local testing
Issue: Driver not being assigned

Possible Causes:

  • Using sandbox mode (simulated drivers)
  • No drivers available in area
  • Order placed outside service hours

Solution:

  • In sandbox: driver assignment is simulated
  • In production: wait for available drivers
  • Check Lalamove service hours
  • Try different pickup/delivery locations
