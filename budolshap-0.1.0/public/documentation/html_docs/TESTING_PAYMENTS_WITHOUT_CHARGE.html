🧪 Testing QR Code Payments

WITHOUT Being Charged Real Money

Option B: Using PayMongo Webhook Simulator

This guide shows you how to test the complete payment flow without actually paying money.

Prerequisites

  • You have a PayMongo account
  • You're using test API keys (starts with sk_test_)
  • Your webhook endpoint is accessible (for local testing, use ngrok)

Step 1: Set Up Webhook Endpoint (Local Testing)

1.1 Install ngrok (if not already installed)

# Download from https://ngrok.com/download
# Or install via chocolatey (Windows)
choco install ngrok

1.2 Start your Next.js dev server

npm run dev

Your app should be running on http://localhost:3000

1.3 Start ngrok tunnel

Open a new terminal and run:

ngrok http 3000

You'll see output like:

Forwarding    https://abc123.ngrok.io -> http://localhost:3000
📋 Copy the HTTPS URL (e.g., https://abc123.ngrok.io)

Step 2: Register Webhook in PayMongo Dashboard

2.1 Go to PayMongo Dashboard

  1. Visit https://dashboard.paymongo.com/
  2. Login with your account
  3. Make sure you're in TEST MODE (toggle in top-right corner)

2.2 Navigate to Webhooks

  1. Click Developers in the left sidebar
  2. Click Webhooks
  3. Click + Create Webhook

2.3 Configure Webhook

Fill in the form:

  • Webhook URL: https://abc123.ngrok.io/api/webhooks/paymongo
    (Replace abc123 with your ngrok subdomain)
  • Events to listen to: Check these boxes:
    • ✅ payment.paid
    • ✅ payment.failed
    • ✅ source.chargeable
  • Description: "Local testing webhook"
  • Click Create Webhook

2.4 Save Webhook Secret

After creating, you'll see a Webhook Signing Secret like:

whsec_abc123xyz...
⚠️ Important: Copy this and add it to your .env.local: 
PAYMONGO_WEBHOOK_SECRET=whsec_abc123xyz...

Restart your dev server to load the new environment variable.

Step 3: Generate a Test Payment

3.1 Create an Order

  1. Go to your app: http://localhost:3000
  2. Add a product to cart
  3. Go to checkout
  4. Select QRPh (or Maya/GrabPay)
  5. Fill in shipping details
  6. Click Place Order

3.2 Get Payment Intent ID

When the QR modal appears, copy the Payment ID shown in the modal.

It looks like: pi_abc123xyz...

📝 Note: DON'T close the modal yet - keep it open or note down the Payment Intent ID.

Step 4: Simulate Webhook Event (The Magic Part!)

4.1 Go to PayMongo Dashboard

  1. Still in TEST MODE
  2. Click Developers → Webhooks
  3. Find your webhook in the list
  4. Click on it to open details

4.2 Send Test Event

  1. Scroll down to "Send a test webhook" section
  2. Select event type: payment.paid
  3. Click Send test webhook
🎉 Success! PayMongo will send a test payment.paid event to your webhook endpoint!

4.3 Alternative: Use API Explorer

If the above doesn't work, you can manually trigger it:

  1. Go to Developers → API Explorer
  2. Select POST /payment_intents/{id}/attach
  3. In the request body, you can simulate the payment completion

Step 5: Verify Order Status Updated

5.1 Check Terminal Logs

In your Next.js terminal, you should see:

✅ Payment successful: pi_abc123xyz Amount: 1000
✅ Order marked as paid and funds held in escrow: cmj5...

5.2 Check Database

Run this to verify:

# In your project directory
npx prisma studio
  1. Open Order table
  2. Find your order
  3. Check that:
    • isPaid = true
    • paymentStatus = 'paid'
    • paidAt has a timestamp

5.3 Check Orders Page

  1. Go to http://localhost:3000/orders
  2. Find your order
  3. Payment status should show "Paid" ✅

Step 6: Testing Different Scenarios

Test Failed Payment

  1. Create another order
  2. Get the Payment Intent ID
  3. In PayMongo Dashboard → Webhooks
  4. Send test event: payment.failed
  5. Verify order shows as "Payment Failed"

Test Webhook Signature Verification

Your webhook handler should verify signatures. Check logs for:

Invalid webhook signature

If you see this, your PAYMONGO_WEBHOOK_SECRET might be wrong.

Troubleshooting

Webhook not receiving events?

Check ngrok is running:

curl https://abc123.ngrok.io/api/webhooks/paymongo

Should return: {"error":"Invalid signature"} (this is good!)

Check webhook URL in PayMongo:

  • Must be HTTPS (ngrok provides this)
  • Must end with /api/webhooks/paymongo
  • Must be accessible from internet

Check terminal logs:

Look for incoming requests in your Next.js terminal

Order not updating?

Check webhook handler logs:

console.log('Received webhook event:', eventType)
console.log('Payment successful:', paymentId)

Check database connection:

Make sure your DATABASE_URL is correct in .env.local

Check Payment Intent ID:

The webhook must match the Payment Intent ID from your order

🚀 Quick Testing Workflow

  1. ✅ Start dev server: npm run dev
  2. ✅ Start ngrok: ngrok http 3000
  3. ✅ Register webhook in PayMongo with ngrok URL
  4. ✅ Create order with QRPh/Maya/GrabPay
  5. ✅ Copy Payment Intent ID from QR modal
  6. ✅ Go to PayMongo Dashboard → Webhooks
  7. ✅ Click "Send test webhook" → payment.paid
  8. ✅ Verify order status updated to "Paid"

You just tested the complete payment flow without spending a cent! 🎉

Next Steps

Once you're confident the flow works:

  1. Deploy to production (Vercel)
  2. Update webhook URL in PayMongo to your production domain
  3. Switch to live API keys when ready for real payments
  4. Test with small amount (₱1) before going live

Important Notes

⚠️ Test Mode vs Live Mode

  • Test keys: sk_test_... - No real money
  • Live keys: sk_live_... - Real money!

🔒 Webhook Security

  • Always verify webhook signatures
  • Never expose your webhook secret
  • Use HTTPS only (ngrok provides this)

💡 ngrok Free Tier

  • URL changes every time you restart ngrok
  • You'll need to update webhook URL in PayMongo each time
  • For permanent testing, consider ngrok paid plan or deploy to staging

Questions?

If something doesn't work:

  1. Check ngrok is running and accessible
  2. Check webhook is registered in PayMongo
  3. Check terminal logs for errors
  4. Check database for order updates
  5. Verify PAYMONGO_WEBHOOK_SECRET is correct
