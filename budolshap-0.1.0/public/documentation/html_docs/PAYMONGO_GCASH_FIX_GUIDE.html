🔧 PayMongo GCash Return URL Fix

Complete Guide to Fixing Vercel SSO Login Issue

BudolShap E-commerce Platform

📋 Problem Description

⚠️ Issue Encountered

After authorizing a GCash payment through PayMongo, users were redirected to Vercel's login page instead of the payment confirmation page.

Expected Behavior: Redirect to https://budolshap.vercel.app/payment/return

Actual Behavior: Redirect to vercel.com/login with SSO parameters

🔍 Root Cause Analysis

The issue was caused by PayMongo's return URL pointing to a protected Vercel preview/deployment URL instead of the public production domain.

URL Type Example Access
Production budolshap.vercel.app ✅ Public (No login required)
Preview budolshap-git-main-xxx.vercel.app 🔒 Protected by SSO
Deployment budolshap-xxx-projects.vercel.app 🔒 Protected by SSO

✅ Solution Implemented

🎯 Fix Applied

Hardcoded the PayMongo return URL to use the public production domain (https://budolshap.vercel.app) instead of relying on environment variables that might point to protected URLs.

📝 Code Changes

File Modified: app/api/paymongo/create-intent/route.js

Before (Problematic Code):

// Relied on environment variable which could be a protected URL const productionDomain = process.env.NEXT_PUBLIC_SITE_URL || 'https://budolshap.vercel.app'; let cleanDomain = productionDomain.trim(); // ... complex URL cleaning logic returnUrl = `${cleanDomain}/payment/return`;

After (Fixed Code):

// Strictly hardcoded to public production domain if (isDevelopment) { returnUrl = 'http://localhost:3000/payment/return'; } else { // Hardcoded to prevent SSO issues returnUrl = 'https://budolshap.vercel.app/payment/return'; }

Before testing, clear your browser cache to ensure you're not seeing cached redirects:

  • Click "Clear data"

⚙️ Environment Variable Configuration

📌 Is This Required?

No, it's optional. The PayMongo return URL is now hardcoded, so the payment flow works without this. However, setting it is recommended as a best practice for future features.

Method 1: Via Vercel Dashboard (Recommended)

1 Go to Vercel Dashboard

2 Select your budolshap project

3 Navigate to Settings → Environment Variables

4 Click "Add New" or edit existing NEXT_PUBLIC_SITE_URL

5 Enter the following:

Name: NEXT_PUBLIC_SITE_URL
Value: https://budolshap.vercel.app
Environment: Production (optionally Preview & Development)

6 Click "Save"

7 When prompted, click "Redeploy" (optional)

Method 2: Via Vercel CLI

# Add the environment variable vercel env add NEXT_PUBLIC_SITE_URL production # When prompted, enter: https://budolshap.vercel.app # Redeploy (optional) vercel --prod

🔧 Troubleshooting

Issue: Still seeing Vercel login page

Possible Causes & Solutions:

  1. Browser Cache: Clear cache completely or use incognito mode
  2. Old Deployment: Verify you're on the latest deployment in Vercel dashboard
  3. Wrong URL: Check the URL in browser - should be budolshap.vercel.app, not budolshap-xxx-projects.vercel.app
  4. DNS Cache: Flush DNS cache: ipconfig /flushdns (Windows) or sudo dscacheutil -flushcache (Mac)

Issue: Payment verification fails

Check the following:

  • Verify PAYMONGO_SECRET_KEY is set correctly in Vercel
  • Check Vercel function logs for errors
  • Verify payment intent was created in PayMongo dashboard
  • Check if webhook is configured (if using webhooks)

How to Check Vercel Logs

  1. Go to Vercel Dashboard → Your Project
  2. Click on Deployments
  3. Select the latest deployment
  4. Go to Functions tab
  5. Find /api/paymongo/create-intent
  6. Look for the log: 🔗 PayMongo Return URL: https://budolshap.vercel.app/payment/return
