🗄️ Database Configuration

PostgreSQL Connection (Vercel Postgres)

PRISMA_DATABASE_URL CRITICAL
Main database connection URL for Prisma (pooled connection)
Get from: Vercel Dashboard → Storage → Your Database → .env.local tab → Copy POSTGRES_PRISMA_URL
POSTGRES_URL CRITICAL
Direct database connection URL (non-pooled, for migrations)
Get from: Vercel Dashboard → Storage → Your Database → .env.local tab → Copy POSTGRES_URL_NON_POOLING
DATABASE_URL REQUIRED
Fallback database URL (same as PRISMA_DATABASE_URL)
Use the same value as PRISMA_DATABASE_URL

🔧 Troubleshooting

Common Issues & Solutions

Build fails with "Prisma Client not found":

  • Ensure prisma generate runs before build
  • Check that DATABASE_URL or PRISMA_DATABASE_URL is set

Database connection errors:

  • Verify both PRISMA_DATABASE_URL and POSTGRES_URL are set correctly
  • Check that database is created in Vercel Storage
  • Ensure migrations have been run

PayMongo payment fails:

  • Verify PAYMONGO_SECRET_KEY and PAYMONGO_PUBLIC_KEY are correct
  • Ensure NEXT_PUBLIC_SITE_URL matches your deployment URL
  • Check that you're using the correct keys (test vs live)

Email not sending:

  • Verify SMTP_USER and SMTP_PASS are set
  • For Gmail, use App Password, not regular password
  • Check SMTP settings (host, port)

Images not uploading:

  • Verify all Cloudinary environment variables are set
  • Check Cloudinary API key and secret are correct
  • Ensure cloud name is correct

Lalamove quotes not working:

  • Verify LALAMOVE_CLIENT_ID and LALAMOVE_CLIENT_SECRET are set
  • Check LALAMOVE_ENV is set to correct environment
  • Ensure API credentials are for the correct region (Philippines)
