1. Recent Incident Report (Issues & Fixes)

Issue 1: Production Login Failure ("Invalid Credentials")

Problem: Users could not log in to the live site. The database connected to the app was different from the one visible in the Vercel dashboard.

Fix: We aligned the Vercel DATABASE_URL to the visible database instance (c0999...) and explicitly created the admin user there.

Issue 2: Domain Redirection Loop

Problem: Logging in at budolshap.vercel.app redirected users back to the homepage.

Fix: We configured the NEXT_PUBLIC_APP_URL environment variable in Vercel to https://budolshap.vercel.app, allowing the middleware to authorize the correct redirect.

Issue 3: Deploy Failures (Missing Variables)

Problem: Vercel builds failed with Prisma errors.

Fix: We ensured DIRECT_URL was present in the production environment variables, as required by Prisma for migrations.

2. Database Management System

We implemented a robust system to switch your local environment between the Local Database and the Production Database safely.

Config Files:
Production credentials are stored in: .env.connection.prod
Local credentials are stored in: .env.connection.local

How to Switch

Use the following commands in your terminal:

Goal Command Effect
Develop Locally npm run db:local Updates .env.local to point to your local PostgreSQL (localhost:5432). Safe for breaking changes.
Debug Production npm run db:prod Updates .env.local to point to the live Vercel Database. USE WITH CAUTION.

3. Database Synchronization

To pull all data (Users, Orders, Stores, etc.) from Production into your Local Database for testing.

Prerequisites

  • Ensure local PostgreSQL is running.
  • Ensure you have the latest credentials in .env.connection.prod.

Sync Command

node scripts/sync-db.js
Warning: This script performs a Clean Sync. It will delete existing data in your local database tables to avoid conflicts before inserting the fresh production data.

4. Development Workflow (Git)

Follow this standard procedure to save your work and update the repository.

Step 1: Check Status

See which files have changed.

git status

Step 2: Stage Changes

Prepare files for commit.

git add .

Step 3: Commit

Save changes with a descriptive message.

git commit -m "feat: description of your changes"

Example messages: "fix: login redirect issue", "style: update navbar colors"

Step 4: Push to Remote

Upload your changes to GitHub.

git push origin [your-branch-name]

Example: git push origin implement-database-switching

5. Deployment Guide (Vercel)

Deploys are typically automatic when you push to the main branch, but you can trigger them manually or update environment variables.

Updating Environment Variables

If you add a new API key or config in your code, you must add it to Vercel:

echo VALUE | vercel env add VARIABLE_NAME production

Manual Production Deployment

To force a new deployment immediately from your CLI:

vercel --prod

Verifying Deployment

  1. Wait for the command to finish.
  2. Visit https://budolshap.vercel.app.
  3. Check the "Deployments" tab in your Vercel Dashboard for logs if issues arise.
