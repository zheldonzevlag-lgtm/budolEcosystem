Session Timeout UX/UI Improvements

Implementation Plan • December 13, 2025

Goal Description

Improve the user experience during session timeouts and subsequent logins.

  • Session Timeout: Provide a more distinct and clear "Session Expired" modal state instead of just a banner inside the generic login modal.
  • Login Success: Prevent the "Unauthorized" page from flashing briefly after a successful re-login by maintaining the loading state until the page reloads.

User Review Items

ℹ️
UI Update Note: The "Session Expired" state will now look cleaner and more intentional, replacing the "Welcome Back" header with "Session Expired".

Proposed Changes

Context

MODIFY AuthUIContext.jsx context/AuthUIContext.jsx

  • Update showLogin function to accept a type argument (e.g., 'session_expired').
  • Add loginType to the context state to track why the login modal is shown.

Components

MODIFY LoginModal.jsx components/LoginModal.jsx

  • Session Expired UI:
    • If loginType is 'session_expired', change the header to "Session Expired".
    • Update the subtext to clarify why the user needs to log in again.
    • Hide the "Create Account" link in this mode to reduce clutter.
  • Success State Fix:
    • In handleLoginSuccess, introduce a local isSuccess state.
    • When isSuccess is true, show a "Redirecting..." loading spinner inside the modal instead of closing it immediately.
    • Do not call hideLogin() if a full page reload or redirect is imminent.

MODIFY AdminLayout.jsx components/admin/AdminLayout.jsx

  • Update handleLogout to call showLogin(null, 'You have been logged out due to inactivity.', 'session_expired').

MODIFY StoreLayout.jsx components/store/StoreLayout.jsx

  • Update handleLogout to call showLogin(null, 'You have been logged out due to inactivity.', 'session_expired').

Verification Plan

Manual Verification

1. Test Session Timeout

  • Log in as an Admin or Seller.
  • Manually trigger the timeout (or wait for it).
  • Expectation: The "Session Expired" modal appears with the new specific UI (Title: "Session Expired").

2. Test Re-login

  • In the "Session Expired" modal, enter valid credentials.
  • Click "Sign In".
  • Expectation: The modal should show a loading/redirecting state and stay open until the page reloads.
  • Expectation: The underlying "You are not authorized" screen should NOT be visible during this transition.

3. Test Normal Login

  • Log out manually.
  • Click "Login".
  • Expectation: The standard "Welcome Back" modal appears.
