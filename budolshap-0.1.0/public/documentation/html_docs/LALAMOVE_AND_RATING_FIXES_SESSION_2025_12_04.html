🛠️ Issues Resolved

FIXED Rating Button Visibility

Issue: The "Rate Product" button was only visible for orders with status PAID. Once an order transitioned to DELIVERED or COMPLETED, the button disappeared, preventing users from rating delivered items.

Solution: Updated components/OrderItem.jsx to allow rating for all post-payment statuses.

// Before
className={`${order.status !== 'PAID' && 'hidden'}`}

// After
className={`${!['PAID', 'PROCESSING', 'SHIPPED', 'DELIVERED', 'COMPLETED'].includes(order.status) && 'hidden'}`}

FIXED Lalamove Booking UI

Issue: The "Book Lalamove" button was missing for orders paid via GCash because they lacked initial shipping data. Also, the booking API failed for these orders.

Solution:

  • Added conditional rendering in app/store/orders/page.jsx to show the button for paid orders without shipping.
  • Updated app/api/shipping/lalamove/book/route.js to auto-generate quotes for orders without pre-existing shipping info.

FIXED Webhook Connectivity (Workaround)

Issue: Lalamove Sandbox environment could not reach the Vercel-hosted webhook endpoint ("Destination host unreachable").

Solution: Created manual webhook simulation scripts to verify the flow and update statuses.

  • scripts/trigger-pickup-webhook.js (PICKED_UP → SHIPPED)
  • scripts/trigger-completed-webhook.js (COMPLETED → DELIVERED)

💡 Recommendations for Marketplace Enhancement

Based on standard multi-vendor marketplace architectures (like Shopee, Lazada, Amazon), here are recommended enhancements for BudolShap:

1. Enhanced Rating & Review System

Current State: Basic rating modal accessible only from the order list.

Recommendation:

  • Order Detail Integration: Add the "Rate Product" button to the Order Detail page (OrderItemsList.jsx) for better accessibility.
  • Media Uploads: Allow users to upload photos/videos with their reviews (crucial for "budol" culture).
  • Seller Response: Allow sellers to reply to reviews to manage their reputation.

2. Robust Seller Payout System

Current State: Payments go to the platform/admin account.

Recommendation:

  • Implement a Wallet System for sellers where earnings are credited after order completion (e.g., 7 days after delivery).
  • Add Withdrawal Requests feature for sellers to cash out via GCash or Bank Transfer.
  • Automate commission deduction (platform fee) at the source.

3. Formalized Dispute & Refund Logic

Current State: Basic status tracking.

Recommendation:

  • Add a "Return/Refund" button for buyers within the guarantee period (e.g., 7 days).
  • Create a dispute resolution dashboard for Admins to mediate between Buyer and Seller.
  • Implement logic to freeze seller payouts when a dispute is active.

4. Advanced Logistics & Tracking

Current State: Lalamove integration (Same-day).

Recommendation:

  • Multiple Couriers: Integrate standard couriers (J&T, NinjaVan) for nationwide shipping, not just same-day.
  • Waybill Printing: Generate printable waybills for sellers to stick on packages.
  • Bulk Booking: Allow sellers to book multiple orders at once.

5. Marketing Tools for Sellers

Current State: Basic coupons.

Recommendation:

  • Flash Sales: Allow sellers to nominate products for time-limited sales.
  • Bundle Deals: "Buy 2 Get 1 Free" or "Add-on Deals".
  • Store Decoration: Allow sellers to customize their store page banner and featured products.
