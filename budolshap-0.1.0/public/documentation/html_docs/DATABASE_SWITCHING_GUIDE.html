Database Switching Guide

This guide explains how to switch between your Local PostgreSQL Database and the Production Database.

Note: Both environments are configured to use PostgreSQL. The switch only changes the connection strings in your .env.local file.

How to Switch

1. Switch to Local Database

Use this when developing features or testing locally.

npm run db:local

This copies configuration from .env.connection.local to .env.local.

Default Connection: postgresql://postgres:password@localhost:5432/budolshap_v3

2. Switch to Production Database

Use this when you need to inspect real data or debug production issues.

npm run db:prod

This copies configuration from .env.connection.prod to .env.local.

Configuration Files

  • .env.connection.local: Stores your Local DB credentials. Edit this file if your local password changes.
  • .env.connection.prod: Stores your Production DB credentials.
  • scripts/env-manager.js: The script that handles the file copying.

Workflow Example

  1. Start work: npm run db:local
  2. Run app: npm run dev
  3. Need to check prod data? Stop server.
  4. Switch: npm run db:prod
  5. Run app: npm run dev

Troubleshooting Vercel Deployment

Deployment Failed?

If your Vercel deployment fails with database errors, ensure you have added DIRECT_URL to your Vercel Environment Variables.

Go to Vercel Dashboard > Settings > Environment Variables and add:

  • Key: DIRECT_URL
  • Value: Same as your POSTGRES_URL (or POSTGRES_URL_NON_POOLING if available).
