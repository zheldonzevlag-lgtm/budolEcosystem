Payment Process Redesign: Agnostic & Robust

This document outlines the new, Payment Method Agnostic flow for BudolShap. The goal is to decouple the "Place Order" action from the specific payment implementation, ensuring a consistent and safe experience whether the user pays via COD, GCash, Maya, or QRPh.

Current Issue: The "Place Order" button currently handles payment logic with specific if/else checks for each method. This led to QRPh orders being treated as "Complete" before the QR code was even scanned.

The New Agnostic Flow

1. Create Pending Order (Backend)
Frontend calls /api/orders.
  • Status: PENDING
  • Payment Status: AWAITING_PAYMENT (or COD_PENDING for COD)
  • Cart: NOT CLEARED yet (except for COD).
  • Email: NOT SENT yet (except for COD).
2. Initiate Payment Strategy
Frontend calls a unified processPayment(orderId, method). This function acts as a dispatcher.
  • COD / BudolPay: Returns { type: 'IMMEDIATE_SUCCESS' }.
  • GCash / GrabPay (Redirects): Calls API, receives checkoutUrl. Returns { type: 'REDIRECT', url: ... }.
  • Maya / QRPh (Direct QR): Calls API, receives qrCode image. Returns { type: 'QR_DISPLAY', data: ... }.
3. Handle User Interface (Frontend)
Based on the Result Type from Step 2:
  • IMMEDIATE_SUCCESS: Clear Cart (Client), Redirect to Success Page.
  • REDIRECT: Redirect window location to checkoutUrl. (Cart clearing handled on return or webhook).
  • QR_DISPLAY: Open QRCodeModal. Start Polling.
    User stays on Cart Page until payment confirms.
4. Payment Confirmation (The "Gate")
The order is ONLY finalized when payment is confirmed.
  • Client-Side Polling (QR): When API reports paid -> Clear Cart -> Redirect to Success.
  • Return Page (Redirects): Verifies payment -> Clear Cart -> Show Success.
  • Server-Side Webhook (Backup): If user closes tab, Webhook receives payment.paid -> Clears Cart (DB) -> Sends Email.

Implementation Logic: OrderSummary.jsx

const handlePlaceOrder = async () => {
    // 1. Validation
    // 2. Create Order (Status: PENDING)
    const orderData = await createOrderAPI(...);

    // 3. Process Payment (Agnostic)
    try {
        const result = await paymentService.process(orderData.id, paymentMethod);
        
        if (result.type === 'QR_DISPLAY') {
            setQrCodeData(result.data);
            setShowQRModal(true);
            // DO NOT REDIRECT. DO NOT CLEAR CART.
        } 
        else if (result.type === 'REDIRECT') {
            window.location.href = result.url;
        }
        else if (result.type === 'SUCCESS') {
            handleSuccessCleanup(); // Clear cart, redirect
        }
    } catch (error) {
        showError(error);
        // Order remains PENDING. User can try again.
    }
}
