🐛 Issues Encountered and Solutions

Issue #1: Missing Stops Data in Order

Order shipping data incomplete
CRITICAL

Symptom: Lalamove booking API failed because order.shipping.stops was undefined.

Root Cause: components/OrderSummary.jsx was not including the stops array when creating orders with Lalamove shipping.

Impact: Booking API couldn't retrieve stopIds required by Lalamove API, causing all booking attempts to fail.

Solution: Add stops to order payload
FIXED

File Modified: components/OrderSummary.jsx

Change: Added stops: lalamoveQuote.stops to the shipping object when placing orders.

// Before (Missing stops) shipping: shippingMethod === 'LALAMOVE' && lalamoveQuote ? { provider: 'lalamove', quotationId: lalamoveQuote.quoteId, serviceType: lalamoveQuote.serviceType, eta: lalamoveQuote.eta } : null // After (With stops) shipping: shippingMethod === 'LALAMOVE' && lalamoveQuote ? { provider: 'lalamove', quotationId: lalamoveQuote.quoteId, quoteId: lalamoveQuote.quoteId, serviceType: lalamoveQuote.serviceType, eta: lalamoveQuote.eta, stops: lalamoveQuote.stops // ✅ Added } : null

Issue #2: Database Client Import Error

Cannot read properties of undefined (reading 'order')
CRITICAL

Symptom: Booking API crashed with "Cannot read properties of undefined (reading 'order')" error.

Root Cause: Used default import for prisma client instead of named import. lib/prisma.js exports prisma as a named export, not default.

Impact: prisma variable was undefined, causing prisma.order to throw immediately.

Solution: Use named import
FIXED

File Modified: app/api/shipping/lalamove/book/route.js

Change: Changed from default import to named import.

// Before (Incorrect - default import) import prisma from '@/lib/prisma'; // After (Correct - named import) import { prisma } from '@/lib/prisma';

Issue #3: Invalid Phone Number Format

ERR_INVALID_FIELD: Phone must be in E.164 format
HIGH

Symptom: Lalamove API rejected booking with error: "'09876543212' is not valid 'phone'. Phone must be: in e.164 format"

Root Cause: Philippine phone numbers stored as 09... format, but Lalamove requires +639... (E.164 format).

Impact: All booking attempts failed validation at Lalamove API level.

Solution: Automatic phone formatting
FIXED

File Modified: app/api/shipping/lalamove/book/route.js

Change: Added formatPhone helper function to convert Philippine numbers to E.164.

const formatPhone = (phone) => { if (!phone) return '+639170000000'; let cleaned = phone.replace(/[\s\-\(\)]/g, ''); // If starts with 09, replace 0 with +63 if (cleaned.startsWith('09')) { return '+63' + cleaned.substring(1); } // If starts with 9 and is 10 digits, prepend +63 if (cleaned.startsWith('9') && cleaned.length === 10) { return '+63' + cleaned; } // If starts with 63, ensure + if (cleaned.startsWith('63')) { return '+' + cleaned; } return cleaned.startsWith('+') ? cleaned : '+63' + cleaned; };

Issue #4: Expired Quote Error

ERR_INVALID_SCHEDULE_TIME: Quote expired
HIGH

Symptom: Booking failed with "Shipping service error" when attempting to book 25+ minutes after order placement.

Root Cause: Lalamove quotes expire after 2-5 minutes. The stored quote from checkout was no longer valid.

Impact: Store owners couldn't book deliveries if they didn't do it immediately after order placement.

Solution: Automatic quote refresh
FIXED

File Modified: app/api/shipping/lalamove/book/route.js

Change: Implemented logic to detect expired quotes and automatically request fresh quotes before booking.

// Check if quote is expired or missing const isQuoteExpired = storedQuote?.expiresAt && new Date(storedQuote.expiresAt) < new Date(); const needsRefresh = !storedQuote || !storedQuote.quotationId || !storedQuote.stops || isQuoteExpired; if (needsRefresh) { // Get fresh quote from Lalamove const freshQuote = await lalamove.getQuote(quotePayload); // Update order with fresh quote await prisma.order.update({ where: { id: orderId }, data: { shipping: { ...order.shipping, quotationId: freshQuote.quotationId, stops: freshQuote.stops, expiresAt: freshQuote.expiresAt } } }); }

Issue #5: Service Instantiation Order

Failed to refresh quote: lalamove is not defined
MEDIUM

Symptom: Quote refresh failed because lalamove service was used before being instantiated.

Root Cause: getShippingProvider('lalamove') was called after the quote refresh logic that needed it.

Impact: Automatic quote refresh feature didn't work, causing booking to fail for expired quotes.

Solution: Move service instantiation earlier
FIXED

File Modified: app/api/shipping/lalamove/book/route.js

Change: Moved getShippingProvider call to before the quote refresh logic.

// Get Lalamove provider (needed for quote refresh) const lalamove = getShippingProvider('lalamove'); // Now we can use lalamove in quote refresh logic if (needsRefresh) { const freshQuote = await lalamove.getQuote(quotePayload); // ... }

🔧 Technical Implementation Details

Phone Number Formatting Logic

The system now automatically converts Philippine phone numbers to E.164 format:

  • 09876543212 → +639876543212
  • 9876543212 → +639876543212
  • 639876543212 → +639876543212
  • +639876543212 → +639876543212 (no change)

Quote Refresh Strategy

The booking API implements a smart quote refresh mechanism:

  1. Check if stored quote exists and has expiresAt timestamp
  2. Compare expiresAt with current time
  3. If expired or missing, construct new quote request with:
    • Pickup: Default Manila store location (14.5505, 121.0260)
    • Delivery: Customer address coordinates from order
    • Service type: From stored order or default to MOTORCYCLE
  4. Update order with fresh quote details
  5. Proceed with booking using new quote

Error Handling Improvements

  • User-Friendly Messages: All errors now display clear, actionable messages via toast notifications
  • Automatic Recovery: System attempts to recover from expired quotes automatically
  • Detailed Logging: Console logs provide detailed debugging information for developers
  • Graceful Degradation: If quote refresh fails, user gets clear error message instead of silent failure

💡 Lessons Learned

Key Takeaways

  • Data Persistence: Always ensure critical data (like Lalamove stops) is persisted in the database during order creation, not just passed in API calls.
  • Import Consistency: Pay attention to named vs default exports. TypeScript would have caught this error at compile time.
  • API Validation: External APIs (like Lalamove) have strict validation requirements. Always format data according to their specifications (e.g., E.164 phone numbers).
  • Quote Expiry: Time-sensitive data like quotes need refresh mechanisms. Don't assume stored quotes will remain valid.
  • Error Messages: User-friendly error messages are crucial. Technical errors should be logged, but users should see actionable messages.
  • Testing in Production: Some issues only appear in production environment. Thorough testing is essential.
