🚀 Budolshap Local Development Guide

Complete guide for running your app locally with MySQL and deploying to Vercel with PostgreSQL

📋 Overview

This guide explains how to switch between local MySQL development and Vercel PostgreSQL deployment for your Budolshap application.

Why Two Databases?
Your local development uses MySQL (via WAMP), while Vercel deployment uses PostgreSQL for better cloud performance and scalability.
Feature Local (MySQL) Vercel (PostgreSQL)
Database MySQL via WAMP Vercel Postgres
Connection localhost:3306 Cloud connection pooling
Environment .env file Vercel Environment Variables
Schema File schema.prisma (MySQL) schema.prisma (PostgreSQL)

💻 Local Development Setup MySQL

Step 1: Configure Prisma Schema for MySQL

Update your prisma/schema.prisma file:

datasource db { provider = "mysql" url = env("DATABASE_URL") }

Step 2: Configure Environment Variables

Your .env file should contain:

DATABASE_URL="mysql://root:@localhost:3306/gocart" # Authentication JWT_SECRET=your-super-secret-jwt-key-change-this-in-production JWT_EXPIRES_IN=7d ADMIN_EMAILS="admin@budolshap.com" # Email Configuration SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=your-email@gmail.com SMTP_PASS=your-app-password SMTP_FROM=your-email@gmail.com NEXT_PUBLIC_APP_URL=http://localhost:3000 NEXT_PUBLIC_CURRENCY_SYMBOL= ₱ # PayMongo Configuration PAYMONGO_SECRET_KEY=sk_test_your_secret_key PAYMONGO_PUBLIC_KEY=pk_test_your_public_key NEXT_PUBLIC_SITE_URL=http://localhost:3000

Step 3: Generate Prisma Client

1 Generate the Prisma client for MySQL:
npx prisma generate

Step 4: Run Database Migrations (if needed)

2 Apply schema changes to your local database:
npx prisma db push

Step 5: Start Development Server

3 Start the Next.js development server:
npm run dev
Success! Your app should now be running at http://localhost:3000

☁️ Vercel Deployment Setup PostgreSQL

Step 1: Configure Prisma Schema for PostgreSQL

Update your prisma/schema.prisma file:

datasource db { provider = "postgresql" url = env("POSTGRES_PRISMA_URL") // Uses connection pooling directUrl = env("POSTGRES_URL_NON_POOLING") // Uses direct connection }

Step 2: Set Vercel Environment Variables

In your Vercel project dashboard, add these environment variables:

1 Database Variables (from Vercel Postgres):
  • POSTGRES_PRISMA_URL - Connection pooling URL
  • POSTGRES_URL_NON_POOLING - Direct connection URL
  • POSTGRES_URL - Standard connection URL
2 Application Variables:
  • JWT_SECRET
  • JWT_EXPIRES_IN
  • ADMIN_EMAILS
  • SMTP_HOST
  • SMTP_PORT
  • SMTP_USER
  • SMTP_PASS
  • SMTP_FROM
  • NEXT_PUBLIC_APP_URL - Your Vercel domain
  • PAYMONGO_SECRET_KEY
  • PAYMONGO_PUBLIC_KEY
  • NEXT_PUBLIC_SITE_URL - Your Vercel domain
  • NEXT_PUBLIC_CURRENCY_SYMBOL

Step 3: Update Build Configuration

Ensure your package.json has the correct build script:

{ "scripts": { "dev": "next dev --turbopack", "build": "prisma generate && next build", "start": "next start", "lint": "next lint" } }

Step 4: Commit and Push Changes

1 Commit your changes:
git add . git commit -m "Configure for Vercel PostgreSQL deployment"
2 Push to GitHub:
git push origin main
Important: Make sure your GitHub repository exists and you have push access before running this command.

Step 5: Deploy to Vercel

Vercel will automatically deploy when you push to GitHub. You can also manually trigger a deployment from the Vercel dashboard.

Deployment Complete! Your app will be live at your Vercel domain.

🔄 Switching Between Environments

From Local to Vercel

1 Update schema.prisma:
# Change from: datasource db { provider = "mysql" url = env("DATABASE_URL") } # To: datasource db { provider = "postgresql" url = env("POSTGRES_PRISMA_URL") directUrl = env("POSTGRES_URL_NON_POOLING") }
2 Regenerate Prisma Client:
npx prisma generate
3 Commit and push:
git add . git commit -m "Switch to PostgreSQL for Vercel" git push origin main

From Vercel to Local

1 Update schema.prisma:
# Change from: datasource db { provider = "postgresql" url = env("POSTGRES_PRISMA_URL") directUrl = env("POSTGRES_URL_NON_POOLING") } # To: datasource db { provider = "mysql" url = env("DATABASE_URL") }
2 Regenerate Prisma Client:
npx prisma generate
3 Start local server:
npm run dev

✨ Best Practices

1. Use Separate Schema Files (Recommended)

Create two schema files to avoid manual switching:

Create schema.mysql.prisma:
// prisma/schema.mysql.prisma datasource db { provider = "mysql" url = env("DATABASE_URL") } // ... rest of your schema
Create schema.postgres.prisma:
// prisma/schema.postgres.prisma datasource db { provider = "postgresql" url = env("POSTGRES_PRISMA_URL") directUrl = env("POSTGRES_URL_NON_POOLING") } // ... rest of your schema
Copy the appropriate file before deployment:
# For local development: cp prisma/schema.mysql.prisma prisma/schema.prisma # For Vercel deployment: cp prisma/schema.postgres.prisma prisma/schema.prisma

2. Environment-Specific Configuration

  • Never commit .env files to Git
  • Use .env.example to document required variables
  • Keep production secrets secure in Vercel dashboard
  • Use different PayMongo keys for test and production

3. Database Migrations

Warning: Always test migrations locally before applying to production!
# Local (MySQL): npx prisma db push # Production (PostgreSQL) - via Vercel CLI: vercel env pull npx prisma migrate deploy

4. Git Workflow

  • Keep schema.prisma in PostgreSQL mode for Vercel
  • Temporarily switch to MySQL only for local development
  • Don't commit MySQL schema changes to main branch
  • Use feature branches for development

🔧 Troubleshooting

Issue: "Environment variable not found: POSTGRES_PRISMA_URL"

Solution: You're trying to run locally with PostgreSQL schema. Switch to MySQL schema as described above.

Issue: "useSearchParams() should be wrapped in a suspense boundary"

Fixed! This has been resolved in all payment and authentication pages by wrapping components in Suspense boundaries.

Issue: Prisma Client errors after switching databases

Solution: Always run npx prisma generate after changing the schema provider.

Issue: GitHub push fails with "Repository not found"

Solutions:
  • Verify the repository exists on GitHub
  • Check your GitHub authentication
  • Ensure you have push access to the repository
  • Verify the remote URL: git remote -v

Issue: Build fails on Vercel

Checklist:
  • ✓ Schema is set to PostgreSQL
  • ✓ All environment variables are set in Vercel
  • ✓ Build script includes prisma generate
  • ✓ All Suspense boundaries are properly implemented
