🎯 Quick Start

📌 One-Command Deployment
The fastest way to deploy your application to Vercel.

Windows (PowerShell)

.\scripts\deploy.ps1 -Production

Linux/Mac (Bash)

chmod +x scripts/deploy.sh && ./scripts/deploy.sh --production

Using NPM Scripts

npm run deploy:prod

📋 Pre-Deployment Checklist

  • Node.js v18+ installed
  • Vercel CLI installed (npm install -g vercel)
  • Vercel account created
  • Vercel Postgres database created
  • All environment variables set in Vercel Dashboard
  • Code committed to Git
  • Local build test passed (npm run build)

📜 Deployment Scripts

PowerShell Script (deploy.ps1)

Comprehensive automated deployment script for Windows with full validation and safety checks.

Production Deployment
.\scripts\deploy.ps1 -Production
Skip Pre-deployment Checks
.\scripts\deploy.ps1 -Production -SkipChecks
Skip Database Migration
.\scripts\deploy.ps1 -Production -SkipMigration
Quick Deploy (Skip All)
.\scripts\deploy.ps1 -Production -SkipChecks -SkipMigration

Bash Script (deploy.sh)

Cross-platform deployment script for Linux/Mac with identical features.

Make Executable & Deploy
chmod +x scripts/deploy.sh && ./scripts/deploy.sh --production
Skip Checks
./scripts/deploy.sh --production --skip-checks
Skip Migration
./scripts/deploy.sh --production --skip-migration

🔐 Environment Variables

⚠️ Important: All environment variables must be set in the Vercel Dashboard before deployment.

Dashboard URL: Environment Variables Settings

Critical Variables (Required)

Variable Value Description
DATABASE_PROVIDER postgresql Database type
POSTGRES_PRISMA_URL Auto-provided by Vercel Prisma connection URL
POSTGRES_URL_NON_POOLING Auto-provided by Vercel Direct database connection
JWT_SECRET Random 64-char string JWT token encryption
NEXT_PUBLIC_BASE_URL https://your-app.vercel.app Application base URL
NEXT_PUBLIC_CURRENCY_SYMBOL ₱ Currency symbol

Cloudinary (Image Storage)

Variable Description
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME Your Cloudinary cloud name
CLOUDINARY_API_KEY Cloudinary API key
CLOUDINARY_API_SECRET Cloudinary API secret

PayMongo (Payment Processing)

🔴 Use LIVE keys for production!
Test keys (sk_test_..., pk_test_...) will NOT work in production.
Variable Value Format
PAYMONGO_SECRET_KEY sk_live_xxxxxxxxxxxxx
PAYMONGO_PUBLIC_KEY pk_live_xxxxxxxxxxxxx
NEXT_PUBLIC_PAYMONGO_PUBLIC_KEY pk_live_xxxxxxxxxxxxx

Lalamove (Delivery Service)

🔴 Use production credentials!
Sandbox credentials will NOT work in production.
Variable Value
LALAMOVE_API_KEY Production API key
LALAMOVE_API_SECRET Production API secret
LALAMOVE_ENVIRONMENT production

Email (Nodemailer)

Variable Value
EMAIL_HOST smtp.gmail.com
EMAIL_PORT 587
EMAIL_USER your-email@gmail.com
EMAIL_PASSWORD App password (not regular password)
EMAIL_FROM noreply@budolshap.com

🗄️ Database Setup

Step 1: Create Vercel Postgres Database

  1. Go to Vercel Project Dashboard
  2. Click on Storage tab
  3. Click Create Database
  4. Select Postgres
  5. Enter database name: budolshap-db
  6. Select region (choose closest to your users)
  7. Click Create

Step 2: Configure Database Connection

After creating the database:

  1. Go to the .env.local tab in Vercel Postgres
  2. Copy the environment variables (automatically added to your project)
  3. Verify these variables are set:
    • POSTGRES_URL
    • POSTGRES_PRISMA_URL
    • POSTGRES_URL_NON_POOLING

Step 3: Run Migrations

The deployment script will handle this automatically, or you can run manually:
Pull Environment Variables
vercel env pull .env.production
Run Migrations
npx dotenv -e .env.production -- npx prisma migrate deploy
Or Use NPM Script
npm run migrate:deploy

Step 4: Seed Initial Data (Optional)

Create Admin Account
npm run create-admin
Seed Test Data
npm run db:seed

🔧 Troubleshooting

Build Errors

Error: Module not found or Cannot find module

npm install npx prisma generate npm run build

Error: Prisma Client not generated

npx prisma generate

Database Connection Issues

Error: Can't reach database server (P1001)

vercel env ls vercel env pull .env.production

Verify POSTGRES_PRISMA_URL is set correctly

Environment Variable Issues

Error: Variables not loading

vercel env pull .env.production vercel --prod

Always redeploy after adding/updating environment variables

Migration Errors

Error: Migration failed

⚠️ Warning: This will delete all data!
npx dotenv -e .env.production -- npx prisma db push

PayMongo Integration Issues

Error: Payment not processing

  • ✅ Verify using LIVE keys (sk_live_..., pk_live_...)
  • ✅ Check NEXT_PUBLIC_BASE_URL matches deployment URL
  • ✅ Verify webhook URLs if using webhooks

Lalamove Integration Issues

Error: Booking failed

  • ✅ Ensure LALAMOVE_ENVIRONMENT=production
  • ✅ Verify API credentials are for production
  • ✅ Check phone numbers are in E.164 format (+639...)

Image Upload Issues

Error: Failed to upload to Cloudinary

  • ✅ Verify Cloudinary credentials
  • ✅ Check NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME
  • ✅ Ensure API key and secret are correct

🔄 Deployment Workflows

First-Time Deployment

  1. Create Vercel Account
    • Sign up at vercel.com
  2. Install Vercel CLI
    npm install -g vercel
  3. Create Postgres Database
    • Dashboard → Storage → Create Database → Postgres
  4. Set Environment Variables
    • Dashboard → Settings → Environment Variables
  5. Run Deployment Script
    .\scripts\deploy.ps1 -Production
  6. Verify Deployment
    • Test all functionality on live site

Regular Deployment (After Changes)

  1. Test Locally
    npm run build npm start
  2. Commit Changes
    git add . git commit -m "Your changes" git push origin main
  3. Deploy
    npm run deploy:prod

Database Migration Workflow

  1. Create Migration Locally
    npx prisma migrate dev --name your_migration_name
  2. Test Migration
    npm run dev
  3. Commit Migration Files
    git add prisma/migrations git commit -m "Add migration: your_migration_name" git push
  4. Deploy to Production
    .\scripts\deploy.ps1 -Production

    The script will automatically run migrations

Rollback Workflow

  1. Identify Issue
    vercel logs
  2. Rollback via Dashboard
    • Deployments → Find working version → Promote to Production
  3. Or Rollback via CLI
    vercel rollback
  4. Fix Issue Locally
    • Debug and fix the problem
  5. Redeploy
    npm run deploy:prod
