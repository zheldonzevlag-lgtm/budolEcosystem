🔧 User Deletion Fix

Resolving Foreign Key Constraint Violations

Fixed: December 12, 2025

❌ Problem Statement

Issue

Admin users were unable to delete users from the system. When attempting to delete a user, the operation would fail silently or return an error.

Root Cause

The DELETE endpoint at /api/admin/users/[userId] was not properly handling foreign key constraints in the database. Specifically:

  • OrderItems were not being deleted before Orders
  • The Order model has a relation to User without onDelete: Cascade
  • Products owned by the user's store were not being deleted before the store
  • The deletion order was incorrect, causing foreign key constraint violations

Database Constraint Issue

From the Prisma schema (prisma/schema.prisma):

model Order {
    // ...
    user    User    @relation("BuyerRelation", fields: [userId], references: [id])
    // ⚠️ Missing: onDelete: Cascade
}

This means when trying to delete a user, PostgreSQL would throw a foreign key constraint error if the user had any orders.

✅ Solution Implemented

Enhanced Deletion Logic

Implemented a comprehensive deletion strategy using Prisma transactions to ensure all related records are deleted in the correct order:

Deletion Flow

Step 1: Find all orders where the user is the buyer
Step 2: Delete all OrderItems for those orders
Step 3: If user has a store, find all store orders
Step 4: Delete all OrderItems for store orders
Step 5: Delete all store orders
Step 6: Delete all products (cascades to ratings and cart items)
Step 7: Delete the store (cascades to wallet, transactions, payout requests)
Step 8: Delete user's orders as buyer
Step 9: Delete the user (cascades to ratings, addresses, cart, chats, messages, audit logs)

🔨 Technical Implementation

File Modified

app/api/admin/users/[userId]/route.js

Key Changes

// DELETE user (admin only)
export async function DELETE(request, { params }) {
    try {
        const { userId } = await params

        // Check if user exists
        const user = await prisma.user.findUnique({
            where: { id: userId },
            include: { store: true }
        })

        if (!user) {
            return NextResponse.json(
                { error: 'User not found' },
                { status: 404 }
            )
        }

        // Use a transaction to ensure all deletions succeed or fail together
        await prisma.$transaction(async (tx) => {
            // 1. Delete OrderItems for user's orders (as buyer)
            const userOrders = await tx.order.findMany({
                where: { userId: userId },
                select: { id: true }
            })
            
            if (userOrders.length > 0) {
                await tx.orderItem.deleteMany({
                    where: { 
                        orderId: { in: userOrders.map(o => o.id) }
                    }
                })
            }

            // 2. If user has a store, delete store-related records
            if (user.store) {
                // Get all orders from this store
                const storeOrders = await tx.order.findMany({
                    where: { storeId: user.store.id },
                    select: { id: true }
                })

                // Delete OrderItems for store orders
                if (storeOrders.length > 0) {
                    await tx.orderItem.deleteMany({
                        where: { 
                            orderId: { in: storeOrders.map(o => o.id) }
                        }
                    })
                }

                // Delete store orders
                await tx.order.deleteMany({
                    where: { storeId: user.store.id }
                })

                // Delete products (cascade to ratings and cart items)
                await tx.product.deleteMany({
                    where: { storeId: user.store.id }
                })

                // Delete the store (cascade to wallet, transactions, etc.)
                await tx.store.delete({
                    where: { id: user.store.id }
                })
            }

            // 3. Delete user's orders (as buyer)
            await tx.order.deleteMany({
                where: { userId: userId }
            })

            // 4. Delete user (cascade to other records)
            await tx.user.delete({
                where: { id: userId }
            })
        })

        return NextResponse.json({ message: 'User deleted successfully' })
    } catch (error) {
        console.error('Error deleting user:', error)
        return NextResponse.json(
            { error: 'Failed to delete user', details: error.message },
            { status: 500 }
        )
    }
}

🎯 Benefits

Transaction Safety

Using prisma.$transaction() ensures that either all deletions succeed or none do. This prevents partial deletions that could leave the database in an inconsistent state.

Proper Deletion Order

By deleting records in the correct order (child records before parent records), we avoid foreign key constraint violations.

Comprehensive Coverage

The solution handles all related records:

  • Orders (as buyer and seller)
  • OrderItems
  • Products
  • Store
  • Wallet & Transactions
  • Ratings, Addresses, Cart, Chats, Messages, Audit Logs (via cascade)

Better Error Reporting

The error response now includes the error message details, making debugging easier if issues occur.
