Budolshap – Vercel Deployment Guide (Free Tier + MySQL via Railway)

Project: budolshap (Next.js – App Router)

Date: 2025‑11‑23

1️⃣ Prerequisites

  • Git installed on Windows.
  • Node.js (v18+) and npm (or Yarn).
  • A GitHub/GitLab/Bitbucket account (Vercel pulls code from a Git repo).
  • A Vercel account (free tier).
  • A free MySQL instance on Railway.

2️⃣ Create a free MySQL instance on Railway

  1. Sign‑up or log in at railway.app.
  2. Click **New Project** → **Create a New Service** → choose **MySQL**.
  3. Give the service a name (e.g., budolshap‑db) and click **Create**.
  4. Once the database is ready, open the **Settings** tab and copy the **Connection URL**. It looks like: 
    mysql://username:password@containers-us-west-123.railway.app:5432/database_name?sslmode=require
    Keep this URL handy – you will add it as an environment variable on Vercel.

3️⃣ Wire the MySQL DB into the Next.js app (Prisma)

  • Add the connection string to .env.local (do NOT commit this file): 
    DATABASE_URL="mysql://username:password@containers-us-west-123.railway.app:5432/database_name?sslmode=require"
  • Update your Prisma schema to use the MySQL provider (if it still says PostgreSQL): 
    datasource db {
  provider = "mysql"
  url      = env("DATABASE_URL")
}
  • Install the MySQL driver (if not already present): 
    npm install mysql2
  • Run a local migration to verify the connection works: 
    npx prisma migrate dev --name init
    This will create the tables in your Railway MySQL dev branch.

4️⃣ Prepare the repository for Vercel

  1. Initialize Git (if not already): git init
  2. Create a .gitignore (if missing): 
    # .gitignore
node_modules/
/.next/
.env.local
.DS_Store
  3. Commit all files: 
    git add .
git commit -m "Initial commit – ready for Vercel with MySQL (Railway)"
  4. Push to a remote (GitHub example): 
    git remote add origin https://github.com/<YOUR_USER>/budolshap.git
git branch -M main
git push -u origin main
  5. Update package.json so Prisma runs on Vercel. Add a postinstall script: 
    {
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "postinstall": "prisma generate && prisma migrate deploy"
  }
}
    Commit the change.

5️⃣ Create the Vercel project

  1. Log in to vercel.com.
  2. Click **New Project** → **Import Git Repository** → select the repo you just pushed.
  3. Vercel auto‑detects **Next.js** (Framework Preset: Next.js, Build Command: npm run build, Output Directory: .next).
  4. Deploy – the first deployment will fail until the DB env var is added.

6️⃣ Add DATABASE_URL environment variable on Vercel

  1. In the Vercel dashboard, go to **Project Settings → Environment Variables**.
  2. Click **Add** and fill in:
    • Key: DATABASE_URL
    • Value: paste the Railway MySQL connection URL (including ?sslmode=require)
    • Environment: Production (and optionally Preview)
  3. Save – Vercel will automatically redeploy.

7️⃣ Prisma migrations on Vercel

Because of the postinstall script, each deployment runs:

prisma generate && prisma migrate deploy

If you prefer a manual one‑off migration, use the Vercel CLI:

npm i -g vercel
vercel login
vercel exec -- npx prisma migrate deploy

8️⃣ Verify the live deployment

  1. Open the Vercel‑provided URL (e.g., https://<project>.vercel.app).
  2. Test a feature that hits the DB (view products, place an order, admin dashboard).
  3. If data persists, the MySQL connection via Railway is successful.

9️⃣ Troubleshooting

Symptom Likely cause Fix
404 on API routes Incorrect DATABASE_URL or missing ?sslmode=require Re‑enter the env var with the exact Railway connection string.
Build fails – missing mysql2 module Dependency not installed Run npm install mysql2 locally, commit package-lock.json, push.
Prisma migration pending Postinstall script missing or not executed Ensure postinstall is present in package.json or run manual migration via Vercel CLI.
Env var not found Added only to Preview, not Production Add it to Production environment as well.

🔟 Optional: Custom domain

  1. Project Settings → Domains → Add your domain (e.g., www.myshop.com).
  2. Follow DNS instructions (CNAME to cname.vercel-dns.com or A record to Vercel IPs).
  3. After propagation (<30 min), the domain points to your Vercel deployment.
