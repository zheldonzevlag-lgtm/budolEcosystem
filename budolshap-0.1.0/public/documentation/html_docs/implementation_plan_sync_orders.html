Sync Buyer Order Status with Seller View

The Buyer's "My Orders" page displays stale status information (e.g., "Paid") and uses different formatting than the Seller's "Store Orders" view (e.g., "PROCESSING", "IN TRANSIT"). This plan addresses both the data freshness issue and the visual inconsistency.

IMPORTANT: This change will force the /api/orders endpoint to be dynamic, ensuring buyers always see the latest status from the database.

Proposed Changes

1. Fix Data Freshness (API Layer)

Prevent caching of the orders API to ensure the status is always up-to-date.

  • File: app/api/orders/route.js
  • Change: Add export const dynamic = 'force-dynamic' to opt out of static caching.

2. Sync UI Presentation (Buyer View)

Update the Buyer's order list to match the Seller's status display logic, including colors and text mappings.

  • File: components/OrderItem.jsx
  • Logic Update: Adopt the Seller's status logic:
    • SHIPPED: Display as "IN TRANSIT" with purple styling.
    • DELIVERED: Display with green styling.
    • CANCELLED/REFUNDED: Display with red styling.
    • PROCESSING/BOOKED/Others: Display with blue styling.

Verification Plan

Manual Verification

  1. Login: Open two browsers/tabs. Log in as Buyer in one and Seller in the other.
  2. Action: As Seller, change an order status (e.g., "Mark Shipped").
  3. Check: Refresh Buyer view. Verify:
    • Status text updates to "IN TRANSIT" (matching Seller view logic).
    • Status color changes to Purple (matching Seller view styling).
