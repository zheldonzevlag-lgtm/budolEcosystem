Phase 2: Data Restoration & Fix

Initial Issue: Restoration of Orders failed (0/4 restored). Debugging confirmed that the backup JSON for orderItems included the orderId field.

Technical Cause: Prisma's nested create logic prohibits specifying the foreign key orderId explicitly in the child records, as it is inferred from the parent.

Applied Fix: Modified scripts/restore-database.js to clean the orderItems array and remove orderId before insertion.

// Clean up orderItems for nested create
const cleanOrderItems = orderItems.map(item => {
    const { orderId, ...rest } = item
    return rest
})
