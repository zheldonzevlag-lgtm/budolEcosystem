Architecture Overview

The core architecture follows a provider pattern that makes it easy to add new shipping carriers in the future:

  • ShippingProvider - Base interface that all carriers implement
  • Lalamove - Concrete implementation for Lalamove API
  • ShippingFactory - Factory to instantiate the correct provider
  • Database Schema - Flexible JSON field to store any provider's data

To add a new carrier (e.g., J&T), simply create services/jnt.js extending ShippingProvider and register it in the factory. No changes needed to the database or core logic!
