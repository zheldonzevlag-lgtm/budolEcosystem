Implementation Plan - Fix QRPH Premature Redirection

The user reported that choosing QRPH redirects to "Order Complete" immediately without showing the QR code.

My analysis suggests that the logic in OrderSummary.jsx falls through to the "Success" block (intended for COD) because it fails to recognize QRPH as a QR payment method in some cases, or simply because the fall-through logic is too permissive.

User Review Required

IMPORTANT: I am changing the success redirection logic to explicitly check for COD and BUDOL_PAY. If QRPH fails to match the isQRPayment check, it will now throw an error "Unhandled payment method" instead of incorrectly marking the order as successful. This is a safety mechanism.

Proposed Changes

Components

[MODIFY] OrderSummary.jsx

  • Update handlePlaceOrder function regarding the success/redirect block.
  • Refactor the "Success" block at the end of the function.
  • Instead of if (!isQRPayment && !hasQRCode), use if (['COD', 'BUDOL_PAY'].includes(paymentMethod)).
  • Add a final else block to catch unhandled payment methods and throw an error/toast, preventing silent failures or premature redirections.

Verification Plan

  1. COD Order:
