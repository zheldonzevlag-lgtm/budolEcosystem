🐛 Issues Encountered

Issue #1: 500 Internal Server Error

ERR_AUTHENTICATE ERR_INVALID_LOCALE

Symptom: The POST /api/shipping/lalamove/quote endpoint consistently returned a 500 error when requesting shipping quotes.

Initial Error: ERR_AUTHENTICATE - HMAC signature authentication failed

Root Cause: The HMAC SHA256 signature was being generated incorrectly due to escaped CRLF characters (\\r\\n) instead of actual carriage return and line feed characters.

Issue #2: Incorrect Market Code

ERR_INVALID_MARKET

Symptom: After fixing authentication, requests failed with ERR_INVALID_MARKET when using PH_MNL as the market code.

Root Cause: The market code was set to PH_MNL (a region code) instead of PH (the country market code). Lalamove API v3 requires the country code for the Market header.

Issue #3: Invalid Payload Structure

ERR_UNKNOWN_FIELD ERR_INVALID_LOCALE

Symptom: Requests failed with ERR_INVALID_LOCALE or ERR_UNKNOWN_FIELD errors.

Root Cause: Multiple payload structure issues:

  • ❌ requesterContact field was included in quote requests (only valid for order creation)
  • ❌ Missing language: 'en_PH' field
  • ❌ Using location instead of coordinates in stops array

Issue #4: File Corruption During Editing

Parsing Error

Symptom: route.js file became corrupted with syntax errors during multiple edit attempts.

Root Cause: The file replacement tool introduced duplicate closing brackets and malformed code structure.

Resolution: Deleted and recreated the file from scratch to ensure clean syntax.

✅ Solutions Implemented

Fix #1: HMAC Signature Generation

CRITICAL FIX

File: services/lalamove.js

Problem: Template literals were escaping CRLF characters, producing \\r\\n (4 characters) instead of actual CR+LF (2 bytes).

Solution: Used String.fromCharCode(13, 10) to generate actual CRLF characters.

// ❌ BEFORE - Incorrect (escaped characters) const rawSignature = `${timestamp}\r\n${method}\r\n${path}\r\n\r\n${body}`; // ✅ AFTER - Correct (actual CRLF bytes) const CRLF = String.fromCharCode(13, 10); const rawSignature = timestamp + CRLF + method + CRLF + path + CRLF + CRLF + body;

Fix #2: Market Code Configuration

VERIFIED

File: services/lalamove.js

Solution: Changed market code from PH_MNL to PH based on API testing and Google AI verification.

// ❌ BEFORE - Region code (incorrect) this.market = 'PH_MNL'; // ✅ AFTER - Country market code (correct) this.market = 'PH'; // Philippines (Verified working in Sandbox) this.region = 'PH_MNL'; // Manila

💡 Key Learnings

1. HMAC Signature Generation

When generating HMAC signatures that require CRLF line endings, always use String.fromCharCode(13, 10) instead of template literals with \r\n. Template literals escape these characters, producing incorrect signatures.

2. API Documentation vs Reality

The Lalamove sandbox environment uses PH as the market code (country level), not PH_MNL (city level). Always verify with actual API testing when documentation is unclear.

3. Payload Structure Differences

Quote endpoints (POST /v3/quotations) have different payload requirements than order endpoints (POST /v3/orders). Fields like requesterContact and item are only valid for orders, not quotes.

4. Comprehensive Logging

Adding detailed logging at every step of the request process is crucial for debugging API integration issues. Log the exact payload, headers, and market codes being sent.
