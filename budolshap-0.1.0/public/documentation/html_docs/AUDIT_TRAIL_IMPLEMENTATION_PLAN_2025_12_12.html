Login & Logout Audit Trail Plan

PLANNED

Proposed Date: 2025-12-12
Objective: Implement a comprehensive logging system to track user access events (Login/Logout) for security auditing.

Feature Overview

This feature will record detailed metadata for every successful login and logout event. Administrators will be able to view these logs via a new specific submenu in the Admin Settings.

Target Data Points:

  • Timestamp: Exact date and time of the event.
  • User Identity: Full Name and/or User ID.
  • Action: LOGIN or LOGOUT.
  • IP Address: Source IP of the request.
  • Location: Country (derived from IP/Headers).
  • Device Info: Operating System (OS) and Hostname/Browser User Agent.

Phase 1: Database Schema Design

Goal: Update the Prisma schema to support audit log storage.

Estimated Time: 0.5 - 1 Hour

Steps:

  1. Define a new model AuditLog in prisma/schema.prisma.
  2. Create relation to the User model.
  3. Run migration to update the database.
model AuditLog { id String @id @default(cuid()) userId String action String // "LOGIN" | "LOGOUT" ipAddress String? country String? device String? // e.g., "Windows 10 / Chrome" userAgent String? // Raw string for backup createdAt DateTime @default(now()) user User @relation(fields: [userId], references: [id]) }

Phase 2: Backend Logic Implementation

Goal: Capture data automatically during authentication events.

Estimated Time: 1.5 - 2 Hours

Steps:

  1. Utility Function: Create lib/audit.js to handle data extraction (IP, OS parsing, Country) and database insertion.
  2. Login Route: Update app/api/auth/login/route.js to call the logging utility after a successful password check.
  3. Logout Route: Update app/api/auth/logout/route.js to call the logging utility before clearing the cookie.

Note: We will use Vercel headers (x-vercel-ip-country) for accurate geolocation in production.

Phase 3: Admin UI Implementation

Goal: Visualize the logs for administrators.

Estimated Time: 2 - 3 Hours

Steps:

  1. Sidebar Update: Add "Login & Logout Tracker" to the Admin Settings submenu in AdminSidebar.jsx.
  2. API Endpoint: Create app/api/admin/audit-logs/route.js to fetch logs with pagination and filtering.
  3. UI Page: Create app/admin/settings/audit-logs/page.jsx.
  4. Data Display: Implement a data table with two tabs: Login Logs and Logout Logs.

Phase 4: Testing & Deployment

Goal: Ensure accuracy and reliability.

Estimated Time: 1 Hour

Steps:

  1. Login with different accounts/devices to verify data capture.
  2. Check if "Country" is correctly populated in production (vs localhost).
  3. Verify pagination logic for large log volumes.
  4. Deploy changes to production.
