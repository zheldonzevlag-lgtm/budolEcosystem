Deployment & Account Migration Report

Technical documentation covering the migration of BudolShap V3 to a new Vercel account, resolving database synchronization issues, and fixing admin authentication loops.

Date Generated: December 12, 2025

Overview

This session focused on migrating the budolshap-v3 project from the jons-projects-9722fe4a Vercel account to the jon-galvezs-projects-f5372ed2 account. The process involved fixing critical deployment issues related to environment variables, database schema synchronization (Prisma), Vercel authentication protection, and admin user privileges.

Issues & Fixes

1. Vercel Authentication "Redirect Loop"

Issue: Accessing the application redirected users to a Vercel login page instead of the app's login. This blocked the automated tests and would block public users.
Fix: Disabled "Vercel Authentication" (Deployment Protection) in the Vercel Project Settings to allow public access to the production deployment.

2. Missing Database Tables (500 Error)

Issue: After switching accounts, the new deployment pointed to a database that had not been initialized. Logs showed P2021: The table `public.User` does not exist.
Fix: Executed npx prisma db push against the production database to synchronize the Prisma schema and create all necessary tables.

3. Admin Login Redirect Loop

Issue: Logging in with correct credentials redirected back to the login page. This was caused by inconsistent JWT_SECRET keys between the Middleware (Edge Runtime) and API Routes (Node Runtime).
Fix: Forcibly unified the JWT_SECRET environment variable across all Vercel environments and redeployed to ensure consistent token signing and verification.

4. Admin Permission Denied

Issue: Even after valid authentication, the admin was denied access.
Fix: Updated the database to explicitly set the isAdmin flag to true for the user admin@budolshap.com.

Technical Deep Dive

Environment Variables Configuration

The following environment variables were critical for the fix. Note the use of Prisma Accelerate/Data Proxy URLs.

Variable Purpose
DATABASE_URL Connection string for Prisma Client (points to Prisma Data Proxy/Accelerate).
DIRECT_URL Direct connection string to PostgreSQL (used for migrations/db push).
JWT_SECRET Secret key for signing and verifying JSON Web Tokens (Session Cookies).
NEXT_PUBLIC_APP_URL The canonical URL of the application, used for auth redirects.

Step-by-Step Guide: Transferring to a Different Vercel Account

Follow this exact process to move your project to a new Vercel account without downtime or configuration errors.

1

Prepare the Environment

Ensure you have the latest environment variables from your current working setup.

vercel env pull .env.backup
2

Switch Accounts

Log out of the old account and log in to the new one.

vercel logout
vercel login

Follow the browser prompts to authenticate.

3

Unlink Local Project

Remove the local Vercel configuration folder to force a re-link.

# Windows PowerShell
Remove-Item -Recurse -Force .vercel

# Mac/Linux
rm -rf .vercel
4

Initialize & Deploy

Run the deploy command. Vercel will detect it's a new project context and ask to set it up.

vercel --prod
  • Set up and deploy? Yes
  • Which scope? Select your new account/team
  • Link to existing project? No (unless you already imported it)
  • Project Name: budolshap-v3 (or your preferred name)
  • In which directory? ./
5

Configure Environment Variables

CRITICAL STEP: The new project will NOT have your environment variables. You must add them.

Go to the Vercel Dashboard for the new project -> Settings -> Environment Variables.

Manually copy the values from your .env.backup file, specifically:

  • DATABASE_URL
  • DIRECT_URL
  • JWT_SECRET
  • NEXT_PUBLIC_... variables

Alternatively, use the CLI:

vercel env add DATABASE_URL production
# (It will prompt you for the value. Paste it from your backup)
6

Database Synchronization

Since this is a "fresh" deployment context, ensure the database schema is in sync.

# Ensure your local .env has the correct DATABASE_URL first!
npx prisma db push
7

Disable Vercel Authentication (Optional)

If you want the site to be public:

  1. Go to Project Settings -> Deployment Protection.
  2. Find "Vercel Authentication".
  3. Set it to Disabled.
  4. Save.
8

Final Redeploy

Trigger one last deployment to ensure all environment variables are active.

vercel --prod

Recommendations

Security Note: We hardcoded the JWT_SECRET to a known string during this fix. For better security, you should generate a new random string (e.g., using openssl rand -base64 32) and update it in the Vercel Dashboard Environment Variables.

1. Automate Deployments: Connect your GitHub repository to the new Vercel project to enable automatic deployments on git push.
2. Database Backup: Create a backup workflow for your production data, as migrating accounts usually implies changes in responsibility.
3. Domain DNS: If you use a custom domain (e.g., budolshap.com), remember to update the DNS records to point to the new Vercel project, as the CNAME/A records verify ownership per account.
