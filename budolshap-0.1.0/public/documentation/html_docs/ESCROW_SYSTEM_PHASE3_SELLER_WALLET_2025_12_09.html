Escrow System Phase 3: Seller Wallet & Payouts

Date: December 9, 2025

Status: DEPLOYED

Version: Phase 3 Final

1. Overview

Phase 3 completes the Escrow System by enabling Sellers to view their earnings, differentiate between "Pending" (Escrow) and "Available" funds, and request payouts. This phase also included critical backend fixes to ensure seamless authentication.

2. Seller Features

2.1 Seller Wallet Dashboard

Access via Store Dashboard > Wallet. It displays:

  • Available Balance: Funds ready for withdrawal (Green).
  • Pending Balance: Funds currently held in escrow (Orange). Shows "Held in escrow" status.
  • Info Banner: Educates sellers about the 7-day protection period and platform fees.

2.2 Payout Requests

Sellers can request withdrawals directly from their wallet:

  • Input Validation: Prevents requesting more than available balance.
  • Immediate Feedback: Shows success/error toasts.
  • Transaction History: Logs the "Payout Request" as a pending debit.

3. Technical Fixes & Improvements

Critical Auth Logic Update: Identified and fixed an issue where the Store API routes were checking for decoded.id but the login token provided decoded.userId. This caused "Unauthorized" errors for sellers attempting to access their wallet.

3.1 API Updates

GET /api/store/wallet

Modified to use correct token payload (userId) and create a default wallet if none exists.

POST /api/store/payouts

Standardized auth logic and ensured atomic transactions (Prisma $transaction) when deducting balance and creating request.

3.2 Schema Relationship

Ensured PayoutRequest is correctly linked to Store in Prisma Schema, enabling the Admin to view store details for each request.

4. User Flow (End-to-End)

  1. Order Delivered: 7-day timer starts. Money is in "Pending Balance".
  2. Auto/Manual Release: Admin (or Cron) releases funds. Money moves to "Available Balance".
  3. Seller Withdrawal: Seller clicks "Withdraw" on Wallet page.
  4. Admin Approval: Admin approves usage on Admin > Payouts page. Status becomes "APPROVED" and money is effectively paid out (offline transfer).

5. Conclusion

The Escrow System is now fully functional, end-to-end, from Order Payment to Seller Payout.
