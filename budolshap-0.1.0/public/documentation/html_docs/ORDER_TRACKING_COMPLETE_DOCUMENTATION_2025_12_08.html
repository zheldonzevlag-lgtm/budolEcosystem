🎯 System Overview

The Budolshap order tracking system provides real-time visibility into order fulfillment and delivery status for both buyers and sellers. The system integrates with multiple delivery providers and maintains a unified tracking experience.

Key Features

🗺️ Live GPS Tracking

Real-time driver location tracking via Lalamove's embedded map for active deliveries.

📊 Dynamic Status Updates

Granular status messages that update automatically via webhooks from delivery providers.

🎨 Visual Overlays

Context-aware overlays that provide clear delivery confirmations and status indicators.

🔄 Auto-Refresh

Automatic page refresh for active orders to ensure buyers see the latest status.

📱 Multi-Provider Support

Factory pattern architecture supporting Lalamove, Standard delivery, and future providers.

📈 Seller Analytics

Comprehensive dashboard with delivery statistics and cancellation tracking.

// Status Message Logic
const getStatusMessage = () => {
    const status = order.shipping?.status;
    
    switch(status) {
        case 'ASSIGNING_DRIVER':
            return { text: 'Finding a driver...', color: 'text-orange-600' };
        case 'ON_GOING':
            return { text: 'Driver assigned & on the way to pickup', color: 'text-blue-600' };
        case 'PICKED_UP':
            return { text: 'Package picked up - on the way to you', color: 'text-purple-600' };

        case 'CANCELLED':
            return { text: 'Delivery Canceled', color: 'text-red-600' };
        default:
            return { text: 'Tracking your order...', color: 'text-slate-600' };
    }
};

🔄 Status Synchronization

The Challenge

Lalamove's embedded map iframe updates on its own schedule, which can lag behind webhook updates. This created confusion when:

  • Budolshap shows "Delivered" but map shows "Driver is nearby"
  • Status changes aren't immediately reflected in the UI
  • Users see conflicting information

The Solution

1. Dynamic ShareLink Updates

Modified app/api/webhooks/lalamove/route.js to capture updated tracking links:

const updatedShipping = {
    ...order.shipping,
    status: status,
    lastEvent: event,
    updatedAt: new Date().toISOString(),
    // Update shareLink if provided in payload
    shareLink: data?.order?.shareLink || order.shipping.shareLink,
    trackingUrl: order.shipping.trackingUrl,
};

2. Status Priority System

The UI prioritizes order.shipping.status from webhooks over iframe content:

  1. Webhook Status (Most Authoritative) - Direct from Lalamove API
  2. Database Status - Persisted order.status
  3. Iframe Content (Least Authoritative) - May lag behind

3. Visual Overlays

Overlays provide immediate feedback regardless of iframe state:

  • Delivered Overlay: Covers map when status = DELIVERED
  • Status Header: Always shows current webhook status
  • Color Coding: Visual indicators match status severity

3. Auto-Refresh Mechanism

For active orders (not delivered/cancelled), the page auto-refreshes every 30 seconds:

useEffect(() => {
    const isActive = order.status !== 'DELIVERED' 
        && order.status !== 'CANCELLED' 
        && order.status !== 'COMPLETED';

    if (isActive) {
        const interval = setInterval(async () => {
            // Sync Lalamove status first
            if (order.shipping?.provider === 'lalamove') {
                await fetch(`/api/orders/${params.orderId}/sync-lalamove`);
            }
            // Then fetch updated order
            fetchOrder();
        }, 30000);

        return () => clearInterval(interval);
    }
}, [order.status]);

🔗 Webhook Integration

Webhook Endpoint

POST /api/webhooks/lalamove

Supported Events

Event Trigger Action
DRIVER_ASSIGNED Driver accepts delivery Update driver info, send notification
PICKED_UP Driver picks up package Set status to SHIPPED, update timeline
COMPLETED Package delivered Set status to DELIVERED, schedule auto-complete
CANCELLED Delivery cancelled Set status to CANCELLED, notify seller
ORDER_STATUS_CHANGED Any status change Update shipping.status, refresh UI

Webhook Processing Flow

1. Receive webhook → Verify signature
2. Extract order ID → Find order in database
3. Parse event data → Extract status, driver info, location
4. Update database → order.shipping object
5. Send notifications → Email to buyer/seller
6. Return 200 OK → Acknowledge receipt
⚠️ Important: Always return 200 OK to prevent Lalamove from retrying webhooks, even if processing fails. Log errors for manual investigation.

🔧 Troubleshooting Guide

Issue 1: Map Not Loading

Symptom: "Waiting for driver location..." message persists
Solution:
  1. Check if order.shipping.shareLink exists in database
  2. Verify Lalamove booking was successful
  3. Check webhook logs for DRIVER_ASSIGNED event
  4. Manually sync: GET /api/orders/[orderId]/sync-lalamove

Issue 2: Status Not Updating

Symptom: Order shows old status despite delivery completion
Solution:
  1. Check webhook endpoint is accessible: /api/webhooks/lalamove
  2. Verify webhook signature validation isn't blocking updates
  3. Check database for order.shipping.status value
  4. Review webhook logs in Vercel for errors

Issue 3: Cancelled Orders Not Showing

Symptom: Cancelled tab shows 0 orders despite cancellations
Solution:
  1. Verify getDeliveryStatus() function includes cancelled check
  2. Check if order.status === 'CANCELLED' in database
  3. Ensure webhook handler sets status to CANCELLED on cancel events

Issue 4: Auto-Refresh Not Working

Symptom: Page doesn't update automatically for active orders
Solution:
  1. Check browser console for JavaScript errors
  2. Verify useEffect dependency array includes order.status
  3. Confirm interval is set to 30000ms (30 seconds)
  4. Check if order status is considered "active" (not delivered/cancelled)
