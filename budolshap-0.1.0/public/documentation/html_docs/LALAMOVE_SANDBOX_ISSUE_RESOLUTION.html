Lalamove Sandbox Insufficient Credit Issue

Issue: Order creation fails with ERR_INSUFFICIENT_CREDIT (HTTP 402) in the Lalamove Sandbox environment.

Diagnosis

We have verified the following:

  • The application correctly implements the Quotation → Order flow.
  • A valid quotationId is obtained from the /v3/quotations endpoint.
  • This quotationId and the corresponding stopIds are correctly passed to the /v3/orders endpoint.
  • Despite following the standard flow which is supposed to bypass credit checks for price checking, the Order Creation step in Sandbox still enforces a credit check.

This confirms that the Lalamove Sandbox environment requires a positive wallet balance or a specific account configuration to allow order creation, even for test orders.

Resolution Steps

1. Code Improvements (Completed)

We have updated the codebase to handle this error gracefully:

  • Enhanced Error Parsing: Updated services/lalamove.js to correctly parse Lalamove V3 error formats (including the errors array).
  • Friendly Error Messages: Mapped ERR_INSUFFICIENT_CREDIT to a user-friendly message: "Insufficient credit in shipping account. Please contact support."
  • API Response Handling: Updated app/api/shipping/lalamove/book/route.js to ensure this specific error message is returned to the frontend instead of a generic "Internal Server Error".
2. Action Required: Contact Lalamove Support

Since the code is correct, the issue is account-related. You must contact Lalamove Support to provision your sandbox account with test credits.
