🎯 Objective

The goal of Phase 5 was to implement the actual Order Creation flow with Lalamove API v3. This involved moving from simple quote retrieval to full order placement, ensuring that orders created in our system are correctly reflected in the Lalamove Partner Portal.

🚀 Implementation Process

1

Quote API Enhancement

Modified getQuote to return the stops array containing stopIds. This was critical because Lalamove requires the exact stopIds generated during the quote to be used when creating the order.

2

Database Integration

Updated the Quote API route (/api/shipping/lalamove/quote) to store the full quote details, including stops, into the Order record in the database.

3

Order Creation Logic

Rewrote the createOrder method in services/lalamove.js to accept dynamic stops instead of hardcoded values. Added validation to ensure stopIds are present.

4

Booking API Update

Updated the Booking API route (/api/shipping/lalamove/book) to retrieve the stored quote from the database and pass the correct stops to the order creation service.

5

Webhook Implementation

Created a webhook receiver (/api/shipping/lalamove/webhook) to handle real-time status updates (e.g., Driver Assigned, Completed) and update the local order status.

🔧 Issues & Fixes

Issue 1: Invalid Stop IDs

Problem: Initially, the createOrder method was using hardcoded stopId values ("0" and "1"). Lalamove API rejected these because it requires the specific UUID-like stopIds generated during the quotation phase.
Fix:
  • Modified getQuote to return result.data.stops.
  • Updated createOrder to accept stops in the payload.
  • Ensured the Booking API retrieves these stored stops and passes them to the service.

Issue 2: Missing Request-ID Header

Problem: Lalamove API v3 requires a unique Request-ID header for every request. Without it, requests were failing or returning generic errors.
Fix: Added 'Request-ID': crypto.randomUUID() to the headers in the apiRequest method in services/lalamove.js.

Issue 3: Insufficient Credit (Sandbox)

Problem: Received ERR_INSUFFICIENT_CREDIT (HTTP 402) when attempting to create orders in the Sandbox environment.
Resolution Path:
  • Confirmed code is correct (API is reachable and validating payload).
  • Identified that Sandbox might require initial setup or credit.
  • Drafted support email to Lalamove to resolve account-specific sandbox configuration.

📱 Phase 6: Order Tracking UI

Implemented a comprehensive, responsive Order Tracking interface that supports both Standard (COD) and Lalamove deliveries.

Key Features

  • Tabbed Interface: Clean separation of Tracking History, Order Items, and Delivery Details.
  • Universal Progress Tracker: 6-stage visual progress bar that adapts to the shipping provider.
  • Real-time Updates: Auto-refresh logic (every 30s) for active orders with a "Live Updates" indicator.

Lalamove Integration

Live GPS Tracking:

For Lalamove orders, the UI embeds the official Lalamove tracking page via an iframe, allowing users to see the driver's real-time location on a map.

Includes a Driver Information Card displaying the driver's name, photo, vehicle details, and a call button.

Standard/COD Integration

Destination Map:

For non-Lalamove orders, a static Google Maps embed confirms the delivery destination, providing a premium feel even without live driver tracking.

Responsive Design

The interface is fully optimized for mobile devices:

  • Stacking Layouts: Components like the Progress Tracker and Delivery Details grid stack vertically on small screens.
  • Touch-Friendly: Tab navigation and buttons are sized for touch interaction.
