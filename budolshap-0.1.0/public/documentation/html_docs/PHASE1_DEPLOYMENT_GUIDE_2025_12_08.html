🚀 Deployment Steps

1

Wait for Vercel Deployment

⏱️ 2-3 minutes

Monitor deployment progress:

🔗 https://vercel.com/dashboard

Look for: feature/escrow-implementation-phase1

Expected: Deployment completes successfully with preview URL
2

Backup Database (CRITICAL!)

⏱️ 1-2 minutes

⚠️ DO NOT SKIP THIS STEP!

This backup is your safety net if anything goes wrong.

Run in Vercel terminal or locally with production credentials:

node scripts/backup-database-escrow.js

What gets backed up:

  • • All Wallets with current balances
  • • All Transactions
  • • All paid Orders
  • • All Payout Requests
  • • All Store information
Expected Output: 
🔄 Starting database backup...
💰 Backing up Wallets...
   ✓ X wallets backed up
📊 Backing up Transactions...
   ✓ X transactions backed up
✅ Backup completed successfully!
📄 Backup saved to: backups/escrow-migration-backup-[timestamp].json
3

Run Database Migration

⏱️ 30 seconds

After backup is complete, run:

npx prisma migrate deploy

What this does:

  • • Adds pendingBalance column to Wallet table
  • • Sets default value to 0 for all existing wallets
  • • Preserves all existing balance data
  • • Creates migration record in database
Expected Output: 
Prisma schema loaded from prisma/schema.prisma
Datasource "db": PostgreSQL database
1 migration found in prisma/migrations
Applying migration `20241208_add_pending_balance_to_wallet`
Migration applied successfully ✓
4

Verify Migration

⏱️ 1 minute

Verify the new field exists:

npx prisma studio

Or check directly in database:

SELECT id, balance, "pendingBalance", "createdAt" FROM "Wallet" LIMIT 5;
Success Criteria:
  • ✓ All wallets have pendingBalance = 0
  • ✓ All existing balance values unchanged
  • ✓ No errors in Vercel logs
  • ✓ Application loads correctly

🔄 Rollback Plan (If Needed)

If something goes wrong during migration:

Option 1: Restore from Backup

node scripts/restore-from-backup.js backups/escrow-migration-backup-[timestamp].json

Option 2: Revert Migration

npx prisma migrate resolve --rolled-back 20241208_add_pending_balance_to_wallet

Option 3: Redeploy Previous Version

In Vercel dashboard, redeploy the previous working commit
