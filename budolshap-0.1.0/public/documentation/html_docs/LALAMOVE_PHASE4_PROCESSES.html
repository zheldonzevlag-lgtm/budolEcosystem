🧪 Process 1: API Endpoint Testing

1
Quote API Testing Process
Comprehensive testing procedure for the Lalamove quote endpoint to ensure accurate pricing and delivery estimates.

Prerequisites

  • Lalamove Sandbox API credentials configured in .env.local
  • Development server running (npm run dev)
  • Test user account with valid delivery address
  • Test store with valid pickup address

Testing Steps

  1. Prepare Test Data
    Create test addresses in database with known coordinates:
    // Example Manila addresses for testing Pickup: SM Mall of Asia, Pasay City (14.5352, 120.9818) Delivery: Bonifacio Global City, Taguig (14.5547, 121.0480)
  2. Test Valid Quote Request
    Navigate to checkout page, add items to cart, select Lalamove shipping. Verify quote appears within 3 seconds with accurate pricing.
  3. Test Geocoding Fallback
    Use address without coordinates. Verify geocoding process shows loading state and successfully retrieves coordinates.
  4. Test Different Service Types
    Request quotes for MOTORCYCLE, SEDAN, MPV, and VAN. Verify pricing differences and availability.
  5. Test Error Handling
    Use invalid address (e.g., incomplete street name). Verify user-friendly error message appears.
  6. Measure Performance
    Use browser DevTools Network tab to measure API response time. Should be < 3 seconds.
  7. Document Results
    Record all test results, response times, and any errors encountered in testing log.
✅ Success Criteria
  • Quote returned successfully for valid addresses
  • Geocoding fallback works for addresses without coordinates
  • All service types return appropriate pricing
  • Error messages are clear and user-friendly
  • Response time consistently under 3 seconds
2
Booking API Testing Process
End-to-end testing of the booking flow from order creation to delivery confirmation.

Testing Steps

  1. Create Test Order
    Complete checkout with Lalamove shipping selected. Verify order created with shipping.provider = 'lalamove'.
  2. Access Store Dashboard
    Log in as store owner and navigate to Orders page. Locate the test order.
  3. Initiate Booking
    Click "Book Delivery Now" button. Verify loading state appears and button is disabled.
  4. Verify Booking Response
    Check that booking ID and tracking link are returned. Verify order updates in database.
  5. Test Idempotency
    Attempt to book the same order again. Verify error message prevents duplicate booking.
  6. Check Tracking Link
    Click tracking link. Verify it opens Lalamove tracking page with correct order details.
  7. Verify Database Update
    Check order in database. Verify shipping.bookingId and shipping.shareLink are populated.
⚠️ Important Notes
  • Quote IDs expire after 15 minutes. Book delivery promptly after getting quote.
  • Each booking generates a unique client reference ID using UUID.
  • Sandbox bookings are simulated and won't create real deliveries.

🔔 Process 2: Webhook Testing & Verification

3
Webhook Signature Verification
Test HMAC SHA256 signature validation to ensure webhook security.

Testing Steps

  1. Set Up Testing Tool
    Use Postman or curl to send test webhook requests to https://your-domain.vercel.app/api/webhooks/lalamove
  2. Generate Valid Signature
    Create HMAC SHA256 signature using webhook secret:
    const crypto = require('crypto'); const payload = JSON.stringify(webhookData); const signature = crypto .createHmac('sha256', process.env.LALAMOVE_WEBHOOK_SECRET) .update(payload) .digest('hex');
  3. Send Valid Request
    Send POST request with signature in X-Lalamove-Signature header. Verify 200 OK response.
  4. Test Invalid Signature
    Send request with incorrect signature. Verify 401 Unauthorized response.
  5. Test Missing Signature
    Send request without signature header. Verify 401 Unauthorized response.
  6. Check Server Logs
    Review server logs to ensure failed verifications are logged for security monitoring.

⚡ Process 3: Performance Optimization

6
Geocoding Optimization Implementation
Implement coordinate caching to reduce geocoding API calls and improve quote response time.

Implementation Steps

  1. Create Address Update Endpoint
    Create PATCH /api/addresses/[addressId] to update coordinates:
    // app/api/addresses/[addressId]/route.js export async function PATCH(request, { params }) { const { addressId } = params; const { latitude, longitude } = await request.json(); const updated = await prisma.address.update({ where: { id: addressId }, data: { latitude, longitude } }); return NextResponse.json(updated); }
  2. Update LalamoveWidget
    After successful geocoding, save coordinates to database:
    // In LalamoveWidget.jsx const geocoded = await geocodeAddress(fullAddress); if (geocoded && deliveryAddress.id) { await fetch(`/api/addresses/${deliveryAddress.id}`, { method: 'PATCH', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ latitude: geocoded.lat, longitude: geocoded.lng }) }); }
  3. Implement Session Caching
    Cache geocoded coordinates in component state to avoid repeated calls during same session.
  4. Add Rate Limiting
    Implement debouncing for geocoding requests (min 1 second between calls).
  5. Measure Performance
    Compare quote response times before and after optimization. Target: 50% reduction for cached addresses.

🚀 Process 4: Production Deployment

8
Production Environment Setup
Configure production environment variables and Lalamove production API credentials.

Setup Steps

  1. Obtain Production Credentials
    Contact Lalamove to upgrade from Sandbox to Production API access. Obtain:
    • Production API Key (Client ID)
    • Production API Secret (Client Secret)
    • Production Webhook Secret
  2. Add to Vercel Environment Variables
    Navigate to Vercel Dashboard → Project Settings → Environment Variables:
    LALAMOVE_API_KEY=prod_xxxxxxxxxxxxx LALAMOVE_API_SECRET=prod_xxxxxxxxxxxxx LALAMOVE_WEBHOOK_SECRET=prod_xxxxxxxxxxxxx LALAMOVE_ENVIRONMENT=production LALAMOVE_API_URL=https://rest.lalamove.com
  3. Update Webhook URL
    In Lalamove Developer Portal, update webhook URL to: https://budolshap.vercel.app/api/webhooks/lalamove
  4. Test API Connection
    Create test endpoint to verify production API credentials work correctly.
  5. Verify Webhook Accessibility
    Use Lalamove webhook testing tool to verify endpoint is reachable.
⚠️ Critical Warning
Production API creates real deliveries with real costs! Always test thoroughly in Sandbox before switching to production. Consider implementing a feature flag to enable/disable Lalamove in production until fully tested.

💡 Best Practices & Tips

✅ Testing Best Practices
  • Always test in Sandbox before production
  • Document all test cases and results
  • Test edge cases and error scenarios
  • Measure and record performance metrics
  • Take screenshots for documentation
  • Keep detailed logs of all API interactions
💡 Deployment Best Practices
  • Always create backups before deployment
  • Deploy during low-traffic periods
  • Monitor error logs for 24-48 hours post-deployment
  • Have rollback plan ready
  • Communicate deployment to stakeholders
  • Set up monitoring and alerts
⚠️ Common Pitfalls to Avoid
  • Don't skip Sandbox testing
  • Don't disable signature verification
  • Don't ignore rate limits
  • Don't hardcode credentials
  • Don't deploy without backups
  • Don't ignore error logs
