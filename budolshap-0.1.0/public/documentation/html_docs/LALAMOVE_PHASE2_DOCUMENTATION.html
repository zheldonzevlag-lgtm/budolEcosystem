📦 What We Accomplished in Phase 2

API Endpoints Created

POST /api/shipping/lalamove/quote
Get shipping quotes from Lalamove based on pickup/delivery locations and package details. Returns price breakdown, estimated delivery time, and distance.
File:
app/api/shipping/lalamove/quote/route.js
Authentication:
Required (Session-based)
Input:
pickup address/coordinates, delivery address/coordinates, package weight/dimensions, serviceType
Output:
quoteId, price breakdown (base, surcharge, total), distance, ETA (pickup & delivery)
Features:
Input validation, error handling, formatted response
POST /api/shipping/lalamove/book
Book a Lalamove delivery order and store booking details in the database. Generates unique client reference ID for idempotency to prevent duplicate bookings.
File:
app/api/shipping/lalamove/book/route.js
Authentication:
Required (Session-based)
Input:
orderId, quoteId, pickup/delivery details, contact information, package details
Output:
bookingId, status, trackingUrl, shareLink, price, estimated times
Features:
UUID-based client reference ID, database update (Order.shipping), duplicate booking prevention
GET /api/shipping/lalamove/track/[orderId]
Retrieve real-time delivery status, driver information, location updates, and delivery timeline for a specific order.
File:
app/api/shipping/lalamove/track/[orderId]/route.js
Authentication:
Required (Owner or Admin only)
Input:
orderId (URL parameter)
Output:
status, statusText, driver info (name, phone, plate, photo), location (lat/lng), timeline, ETA
Features:
Authorization checks, database sync, status text conversion
POST /api/shipping/lalamove/cancel/[orderId]
Cancel a pending Lalamove delivery order with proper validation and error handling for cancellation window restrictions.
File:
app/api/shipping/lalamove/cancel/[orderId]/route.js
Authentication:
Required (Owner or Admin only)
Input:
orderId (URL parameter)
Output:
cancellation confirmation, updated status, timestamp
Features:
Status validation (prevents cancelling completed/cancelled orders), database update, error handling
POST /api/webhooks/lalamove
Receive real-time delivery status updates from Lalamove with HMAC signature verification for security. Automatically updates order shipping data.
File:
app/api/webhooks/lalamove/route.js
Authentication:
HMAC SHA256 Signature Verification
Events Handled:
ASSIGNING_DRIVER, ON_GOING, PICKED_UP, IN_TRANSIT, COMPLETED, CANCELLED, REJECTED, EXPIRED
Features:
Signature verification, timeline tracking, driver info updates, location updates, automatic database sync
Health Check:
GET endpoint available for testing webhook availability

🔐 Technical Implementation Details

Authentication & Authorization

Session-Based Authentication
All endpoints (except webhook) use Next-Auth session authentication:
const session = await getServerSession(authOptions); if (!session) { return NextResponse.json({ error: 'Unauthorized' }, { status: 401 }); }
Authorization Checks
Tracking and cancel endpoints verify order ownership:
// Verify order belongs to user (unless admin) if (session.user.role !== 'admin' && order.userId !== session.user.id) { return NextResponse.json({ error: 'Forbidden' }, { status: 403 }); }
Webhook Signature Verification
Webhook endpoint uses HMAC SHA256 signature verification:
const rawBody = await request.text(); const signature = request.headers.get('x-lalamove-signature'); const lalamove = getShippingProvider('lalamove'); const isValid = lalamove.verifyWebhookSignature(rawBody, signature); if (!isValid) { return NextResponse.json({ error: 'Invalid signature' }, { status: 401 }); }
