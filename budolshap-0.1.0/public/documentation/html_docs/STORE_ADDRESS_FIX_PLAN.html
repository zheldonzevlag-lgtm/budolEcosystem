Store Address Fix Implementation Plan

Ensuring robust, courier-agnostic address handling with automated geocoding.

🎯 Objective

Fix the issue where new store addresses are saved as unstructured text ("legacy"), causing failed courier pickups. We will implement structured address storage with geocoordinates to support Lalamove and other providers.

🛠 Implementation Steps

1

Frontend: Implement Geocoding & Structured Data

Est. 30 mins

Modify components/StoreAddressModal.jsx and app/(public)/create-store/page.jsx.

  • Integrate OpenStreetMap (Nominatim) API to fetch Lat/Long automatically when saving an address.
  • Pass the full structured address object (Street, City, Zip, Lat, Long) to the parent component.
  • Update the store submission payload to include this structured data.
Critical: Geocoding is required for Lalamove to calculate shipping quotes accurately.
2

Backend: Save StoreAddress Record

Est. 15 mins

Modify app/api/stores/route.js.

  • Parse the structuredAddress from the request body.
  • Update the Prisma `create` call to create a linked StoreAddress record.
  • Ensure latitude and longitude are saved to the database.
data: {
  // ... store details
  addresses: {
    create: {
      addressObj...,
      latitude: lat,
      longitude: lng,
      isDefault: true
    }
  }
}
3

Verification & Testing

Est. 15 mins
  • Create a new store and verify the address is geocoded (no "Legacy" tag).
  • Check database for Lat/Long values.
  • Verify in Admin Panel.

⏱️ Total Timeframe

Approximately 1 Hour

This includes coding, testing, and verifying the fix working with the courier system.
