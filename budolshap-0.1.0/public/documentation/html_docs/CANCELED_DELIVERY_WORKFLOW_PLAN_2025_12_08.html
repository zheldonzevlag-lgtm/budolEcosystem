Canceled Delivery Workflow - Phased Implementation Plan

📋 Goal Description

Implement a robust canceled delivery handling system aligned with industry best practices from Lazada, Shopee, Zalora, and TikTok Shop. Allow sellers to re-book deliveries when couriers cancel, without requiring buyers to re-order.

Date: December 8, 2025

Alignment: Follows standard e-commerce platform practices for 3PL cancellation handling

🎯 Industry Best Practices (Lazada/Shopee Standard)

✅ What Major Platforms Do When Courier Cancels

1. Auto-Revert to "To Ship"

Order automatically returns to shippable state. Previous tracking becomes invalid.

2. Seller Re-Arranges Pickup

Seller books new pickup/drop-off with same or different courier.

3. New Shipping Label

System generates new AWB/tracking number. Old label cannot be reused.

4. Buyer Notification

Automated message: "Courier canceled, new delivery arranged. Updated tracking: XXX"

5. Dispute System

If rider picked up but system shows canceled, seller can file dispute with evidence.

6. Courier Switching

Seller can choose different 3PL provider if repeated cancellations occur.

🏗️ Future-Proofing: Multi-Provider Architecture (J&T, Grab, Gogo)

✅ Loosely Coupled Design Strategy

The system is designed to be provider-agnostic. Core logic does not know or care which courier is used.

Layer Strategy Benefit
Database Using generic shipping JSON field for all provider data. Zero Schema Changes: Adding J&T or Grab requires NO database migration.
Frontend UI Status checks look for generic flags: order.shipping?.failureReason. Zero UI Changes: The "Rebook" button appears automatically for ANY provider that reports a failure.
Backend Logic Standardized "Contract": All failures revert status to PROCESSING. Plug-and-Play: To add J&T, you only write one file (the webhook handler) that adheres to this standard.

🔍 Comprehensive Cancellation Scenarios Analysis

🤖1. System-Triggered Cancellations

Event Trigger Condition Solution
EXPIRED No driver accepted within time limit ✅ Revert to PROCESSING, allow rebook
REJECTED Multiple drivers rejected (hit limit) ✅ Revert to PROCESSING, suggest address fix

🏍️2. Rider-Initiated Cancellations

Event Trigger Condition Solution
CANCELLED Driver cancels within 5-min window ✅ Revert to PROCESSING, notify seller
ORDER_REPLACED Cancel-and-Clone (waiting fees) 🔄 Auto-handled by Lalamove

🏪3. Seller-Initiated Cancellations

Action Trigger Solution
Cancel Delivery Only Seller clicks "Cancel Delivery" ✅ Shipping cancelled, order stays valid, can rebook
Cancel Entire Order Seller cancels order ❌ Order invalid, no rebook

🛒4. Buyer-Initiated Cancellations

Action Trigger Solution
Cancel Order Buyer requests cancellation ❌ Order invalid, no rebook
Cancel Delivery Buyer cancels via tracking link ⚠️ Revert to PROCESSING, seller reviews
