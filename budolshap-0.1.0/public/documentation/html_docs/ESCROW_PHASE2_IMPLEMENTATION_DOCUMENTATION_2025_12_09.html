Key Design Principle:

The escrow service is completely decoupled from payment gateways. Adding a new gateway only requires creating a new webhook handler that calls the same creditPendingBalance() function.

4️⃣ Issues Encountered

❌

Issue #1: ES Module Import in Test Script

Error: Cannot use ES6 import syntax in Node.js test scripts without module configuration.

SyntaxError: Cannot use import statement outside a module

Impact: Could not directly import escrow service functions in test script
⚠️

Issue #2: Platform Fee Calculation Clarity

Consideration: Need to clearly document whether platform fee applies to total (including shipping) or just product cost.

Decision Made: Platform fee applies to order.total (includes shipping cost)

Added comprehensive documentation explaining platform fee calculation and what it applies to.

Documentation Added:

  • • Includes shipping cost in calculation
  • • Net earnings = total - (total * 0.05)

8️⃣ Lessons Learned

🔧 Technical Lessons

📌
Gateway-Agnostic Design is Powerful

By separating payment gateway logic from business logic, we can add new gateways (Xendit, Stripe, etc.) in minutes without touching the core escrow system.

📌
Transaction Logging is Essential

Creating detailed transaction records for every balance change provides an invaluable audit trail for debugging and reconciliation.

📌
Atomic Operations Prevent Race Conditions

Using Prisma's increment/decrement ensures wallet updates are atomic and prevent race conditions when multiple payments occur simultaneously.

📋 Process Lessons

📌
Test with Real Data

Testing with actual production database data revealed edge cases that wouldn't have been caught with mock data.

📌
Incremental Implementation Works

Breaking the escrow system into phases (Phase 1: DB, Phase 2: Payment, Phase 3: Release) made implementation manageable and testable.

🎨 Design Lessons

📌
Single Responsibility Principle

Each function in the escrow service has a single, clear purpose. This makes the code easier to test, maintain, and extend.

📌
Clear Naming Conventions

Using descriptive names like creditPendingBalance and releaseFunds makes the code self-documenting.
