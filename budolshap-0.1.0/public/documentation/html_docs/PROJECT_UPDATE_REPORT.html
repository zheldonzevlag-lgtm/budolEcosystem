🚀 Project Update Report: Issues, Fixes & Process

Date: December 1, 2025

Version: Budolshap V2.0

1. WYSIWYG Editor Implementation

Objective: Integrate a professional, feature-rich, and user-friendly text editor for product descriptions.

🔴 Issues Encountered

SSR Compatibility Errors
Initial attempt with react-quill caused "document is not defined" errors during server-side rendering (SSR) in Next.js.
UI/UX Limitations
Default editors were either too large (taking up the whole screen) or fixed-size, making it hard for users to manage the form layout.

🟢 Fixes Implemented

Dynamic Import Strategy
Switched to react-draft-wysiwyg and implemented it using next/dynamic with ssr: false. This ensures the editor only loads on the client-side, eliminating build errors.
Resizable & Compact UI
  • Reduced default height to min-h-48 (compact).
  • Added CSS resize: vertical to the container.
  • Users can now drag the bottom-right corner to expand the editor as needed.
Advanced Toolbar
Configured full toolbar options: Fonts, Sizes, Colors, Emojis, Alignment, and History (Undo/Redo).

2. Repository V2 Setup & Migration

Objective: Establish a clean, secure GitHub repository (budolshap-v2) for the new version.

🔴 Issues Encountered

Authentication Failures (403)
Git push failed with "Permission denied" because the local environment was caching old credentials for a different user account.
Secret Scanning Block
GitHub rejected the push with GH013: Repository rule violations.
Cause: The .env.local.example file contained real API keys (Lalamove/PayMongo) and was being tracked by git.

🟢 Fixes Implemented

History Cleansing
Re-initialized the git repository (git init) to remove any hidden history of secrets.
Removing Secrets
Identified tracked environment files and removed them from the index:
git rm --cached .env.local.example .env.local .env
Secure Authentication
Generated a new Personal Access Token (PAT) with repo scope and configured git to use it via a temporary credential helper.
