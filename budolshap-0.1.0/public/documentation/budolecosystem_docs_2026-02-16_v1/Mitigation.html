R-001: Data Loss during Logout

Description: Race condition where empty cart sync overwrites server data.

Mitigation Plan:

  • Implement Logout State Check: Add `prevUserIdRef` in `PublicLayout.jsx` to detect logout transition.
  • Conditional Sync: Prevent API calls if `userId` is null immediately after a state change.
  • Verify with E2E Tests: Run `test_cart_persistence.mjs` to confirm fix.

Status: Implemented & Verified

R-002: SQLite Database Locking

Description: Concurrent writes cause "database is locked" errors in development.

Mitigation Plan:

  • Process Management: Ensure only one Next.js dev server is running. Kill stale processes.
  • Sequential Tests: Run test scripts sequentially, not in parallel with user traffic.
  • Migration to PostgreSQL: Plan for production database migration.

Status: Ongoing Monitoring

R-005: Authentication Token Expiry

Description: Users may be abruptly logged out or actions may fail silently if token expires.

Mitigation Plan:

  • Refresh Token Logic: Implement silent refresh using HTTP-only cookies.
  • UI Feedback: Show "Session Expiring" modal before forced logout.
  • Graceful Failures: Ensure API clients handle 401 errors by attempting refresh or redirecting to login.

Status: Planned for Next Sprint
