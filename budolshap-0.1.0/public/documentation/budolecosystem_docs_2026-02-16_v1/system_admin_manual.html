System Administrator Manual

Guide for deployment, maintenance, and troubleshooting.

← Back to Index

1. Environment Setup

Ensure Node.js 18+ is installed. The project uses npm for package management.

# Install dependencies npm install # Setup environment variables cp .env.example .env # Edit .env with your database URL and secrets # Initialize database npx prisma generate npx prisma db push

2. Database Management

We use Prisma ORM with SQLite (Dev) / PostgreSQL (Prod).

Viewing Data

Use Prisma Studio to inspect the database:

npx prisma studio

This opens a GUI at http://localhost:5555.

Backups

For SQLite, backup the `dev.db` file in the `prisma` folder. For PostgreSQL, use `pg_dump`.

3. Deployment

Deploy using Vercel or any Node.js hosting.

# Build for production npm run build # Start production server npm start

4. Troubleshooting

  • Database Locked (SQLite): Stop all running node processes and restart. Avoid heavy concurrent writes in dev mode.
  • Cart Sync Issues: Check browser console for network errors on `/api/cart`. Verify `userId` is present in local storage/cookies.
  • Image Failures: Ensure `next.config.js` allows the image domains you are using.
