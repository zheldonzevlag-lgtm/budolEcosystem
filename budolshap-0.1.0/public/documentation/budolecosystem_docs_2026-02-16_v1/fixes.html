1. Post-Logout Empty Cart & Race Condition

Issue: Users experienced an empty cart or 0 badge count upon re-login. This was caused by a race condition where the frontend would sync an empty cart (due to local storage clearing) to the server during the logout process, effectively wiping the user's saved cart in the database.

Resolution:

  • Modified app/(public)/layout.jsx to detect user logout (transition from valid userId to null) and clear the Redux state locally without triggering a server sync.
  • Removed explicit dispatch(clearCart()) from components/Navbar.jsx to prevent the race condition.
  • Added logic to app/(public)/layout.jsx to disable auto-sync if the initial cart fetch fails, preventing overwrite of server data with an empty state.

Code Change in PublicLayout.jsx:

// Handle cart clearing on logout
useEffect(() => {
    // If we had a user ID before, but now we don't, it means we logged out
    if (prevUserIdRef.current && !userId) {
        console.log('[Layout] User logged out, clearing cart state');
        dispatch(clearCart());
        localStorage.removeItem('budolshap_cart');
    }
    prevUserIdRef.current = userId;
}, [userId, dispatch]);
