📝 How to Configure Sentry in budolShap

Step-by-step guide to input DSN and configure error tracking

🎯 Quick Overview

You can configure Sentry directly from the budolShap admin panel. No need to edit files manually! All configuration is done through a user-friendly interface.

📍 Step 1: Access Error Tracking Settings

1

Login to Admin Panel

First, make sure you're logged in as an admin user:

  1. Go to http://localhost:3000/admin/login (or your production URL)
  2. Enter your admin email and password
  3. Click "Login"
2

Navigate to Error Tracking

Once logged in, follow this navigation path:

Admin Dashboard → Settings (in sidebar) → Error Tracking

Direct URL: http://localhost:3000/admin/settings/error-tracking

💡 Tip: The "Error Tracking" option is located in the Settings submenu in the left sidebar of the admin panel.

⚙️ Step 2: Configure Sentry Settings

3

Enable Error Tracking

On the Error Tracking page, you'll see a form. First, enable error tracking:

Check this checkbox to enable Sentry error tracking for your application.

Once you check this box, additional configuration fields will appear below.

4

Enter Your Sentry DSN

What is a DSN? DSN (Data Source Name) is a unique identifier that tells your app where to send error reports.

Your Sentry Data Source Name (DSN). Found in your Sentry project settings.

How to get your DSN:

  1. Go to your Sentry dashboard
  2. Select your project (or create one if you haven't)
  3. Go to Settings → Projects → [Your Project] → Client Keys (DSN)
  4. Copy the DSN (it looks like: https://xxxxx@xxxxx.ingest.sentry.io/xxxxx)
  5. Paste it into the DSN field in budolShap
💡 Quick Actions: The admin interface has buttons to help you:
  • "Get DSN" button - Opens Sentry DSN settings in a new tab
  • "Copy DSN" button - Copies your entered DSN to clipboard
5

Select Environment

Choose the environment where your application is running:

Environment name to tag errors in Sentry (e.g., development, staging, production).

  • Development - For local development
  • Staging - For pre-production testing
  • Production - For live production environment (recommended)
6

Set Traces Sample Rate

This controls what percentage of transactions are monitored for performance:

Percentage of transactions to trace (0.0 to 1.0). Lower values reduce Sentry usage. Recommended: 0.1 (10%).

  • 0.0 - No performance monitoring (errors only) - Best for staying within free tier
  • 0.1 - 10% of transactions (recommended for production)
  • 1.0 - 100% of transactions (use for development/debugging)
⚠️ Free Tier Limit: Sentry free tier allows 5,000 events per month. Lower sample rates help you stay within this limit.

💾 Step 3: Save and Sync Configuration

7

Click Save Button

After filling in all the fields:

  1. Review your settings
  2. Click the "Save Error Tracking Settings" button at the bottom of the form
  3. Wait for the success message
✅ What happens when you save:
  • Settings are saved to the database
  • You'll see a success notification
  • Next, use the Sync Configuration button to apply changes
8

Select Sync Target Environment

NEW FEATURE: You can now choose which environment to sync to!

After saving, you'll see a "Sync Target Environment" dropdown with two options:

🖥️ Development (Local)

Use this for: Local development on your computer

  • Automatically updates .env.local file
  • No manual configuration needed
  • Just restart your dev server after syncing

☁️ Production (Vercel)

Use this for: Production deployment on Vercel

  • Shows environment variables to set in Vercel Dashboard
  • Validates your configuration
  • Provides step-by-step instructions
  • Requires manual setup in Vercel Dashboard
💡 How to choose:
  • If you're developing locally → Choose "Development (Local)"
  • If you're deploying to Vercel → Choose "Production (Vercel)"
9

Sync Configuration

After selecting your target environment:

  1. Select the appropriate environment from the dropdown
  2. Click the "Sync Configuration" button
  3. Wait for the sync to complete

For Development (Local):

When you click "Sync Configuration" with "Development (Local)" selected:

  1. The script automatically updates your .env.local file
  2. You'll see a success message: "Configuration synced to .env.local successfully!"
  3. Restart your local development server

For Production (Vercel):

When you click "Sync Configuration" with "Production (Vercel)" selected:

  1. An alert dialog will appear with all environment variables
  2. Copy each variable from the alert
  3. Go to Vercel Dashboard → Your Project → Settings → Environment Variables
  4. Add or update each variable in Vercel
  5. Select the appropriate environment (Production/Preview/Development)
  6. Save and redeploy your application
10

Restart Your Application

Important: After syncing, you must restart your application for changes to take effect.

For Local Development:

# Stop your current server (Ctrl+C) # Then restart: npm run dev

For Production (Vercel):

After setting environment variables in Vercel Dashboard, redeploy:

# Via Vercel Dashboard: # Go to Deployments → Click "Redeploy" on latest deployment # Or via CLI: vercel --prod
⚠️ Don't Skip This Step! Sentry won't work until you restart/redeploy the application.

✅ Step 4: Verify Configuration

11

Check Status

After restarting/redeploying, go back to the Error Tracking page and check the status card at the top:

  • 🟢 Active - Sentry is working correctly
  • 🟡 Not Configured - DSN is missing or invalid
  • ⚪ Disabled - Error tracking is turned off
12

Test Connection

If status shows "Active", you can test the connection:

  1. Click the "Test Connection" button in the status card
  2. This will send a test error to Sentry
  3. Check your Sentry dashboard - you should see a test error appear within a few seconds
✅ Success! If you see the test error in Sentry, your configuration is working perfectly!

🎨 Using the Setup Wizard

💡 Interactive Setup Wizard: If this is your first time setting up Sentry, the admin interface includes a helpful setup wizard that guides you through the process!

The setup wizard provides:

  • Step-by-step instructions with visual progress indicators
  • Quick action buttons to open Sentry signup, project creation, and DSN settings
  • Auto-navigation through the setup process
  • Helpful tips at each step

To use the wizard:

  1. Go to Error Tracking settings page
  2. If not configured, you'll see a "Show Setup Guide" button
  3. Click it to reveal the interactive wizard
  4. Follow the steps and use the quick action buttons

📋 Complete Configuration Checklist

Before You Start:

  • ☐ You have a Sentry account (sign up at sentry.io)
  • ☐ You've created a Next.js project in Sentry
  • ☐ You have your DSN copied to clipboard
  • ☐ You're logged in as admin in budolShap

Configuration Steps:

  • ☐ Navigated to Admin → Settings → Error Tracking
  • ☐ Enabled "Enable Error Tracking (Sentry)" checkbox
  • ☐ Pasted Sentry DSN in the DSN field
  • ☐ Selected appropriate Environment (development/staging/production)
  • ☐ Set Traces Sample Rate (recommended: 0.1)
  • ☐ Clicked "Save Error Tracking Settings"
  • ☐ Selected sync target environment (Development or Production)
  • ☐ Clicked "Sync Configuration" button
  • ☐ For Local: Restarted development server
  • ☐ For Vercel: Set environment variables in Vercel Dashboard and redeployed
  • ☐ Verified status shows "Active"
  • ☐ Tested connection successfully

🐛 Troubleshooting

Status shows "Not Configured"

Possible causes:

  • DSN field is empty - Make sure you've entered a valid DSN
  • DSN format is incorrect - Should start with https://
  • Application not restarted - Restart after saving settings

Test Connection doesn't work

Solutions:

  • Make sure you've restarted the application after saving
  • Check browser console for errors
  • Verify DSN is correct in Sentry dashboard
  • Check if Sentry is enabled in the checkbox

Settings not saving

Check:

  • You're logged in as admin
  • Browser console for error messages
  • Network tab to see if API request succeeded
  • Try refreshing the page and checking again
