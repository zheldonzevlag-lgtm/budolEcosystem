# Deploy Budolshap to Vercel

This workflow outlines the steps to deploy the **Budolshap** Next.js application to Vercel, using the migrated Postgres database.

## Prerequisites
- Vercel account (https://vercel.com)
- Vercel CLI installed (`npm i -g vercel`)
- Git repository initialized and pushed to a remote (GitHub, GitLab, etc.)
- Environment variables set in Vercel (see below)
- Prisma migrations applied to the Postgres instance

## Steps
1. **Login to Vercel**
   ```bash
   vercel login
   ```
   Follow the prompts to authenticate.

2. **Link the project** (run inside the project root)
   ```bash
   vercel link
   ```
   Choose the existing Vercel project or create a new one.

3. **Set environment variables**
   ```bash
   vercel env add DATABASE_URL production
   vercel env add DIRECT_URL production
   vercel env add JWT_SECRET production
   # add any other vars required by the app (NEXT_PUBLIC_*)
   ```
   Use the values from your Vercel Postgres Quickstart.

4. **Generate Prisma client** (ensure schema is up‑to‑date)
   ```bash
   npx prisma generate
   ```

5. **Run migrations on the remote DB** (optional if already applied)
   ```bash
   npx prisma migrate deploy
   ```

6. **Deploy to Vercel**
   ```bash
   vercel --prod
   ```
   This will build the app, run `npm run build`, and deploy the production version.

7. **Verify deployment**
   - Open the URL provided by Vercel.
   - Check that product listings load without the previous `P6009` error.
   - Test any admin functionality.

## Post‑deployment checklist
- ✅ Environment variables are correctly set in Vercel.
- ✅ Prisma client generated and migrations applied.
- ✅ Application runs without oversized Prisma responses (pagination is active).
- ✅ All pages render correctly on the live URL.
