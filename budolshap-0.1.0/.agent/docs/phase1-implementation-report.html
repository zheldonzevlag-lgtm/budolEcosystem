🔧 Issues Encountered & Fixes

❌ Issue #1: Cannot Run Migrations Locally

Problem: Prisma migrate commands fail with connection error

Error Message:

Error: P1001: Can't reach database server at `db.prisma.io:5432`

Root Cause:

  • DATABASE_URL in .env uses Prisma Accelerate format
  • Accelerate URLs (prisma+postgres://...) are for query acceleration, not direct access
  • Migration tools need direct PostgreSQL connection

✅ Fix #1: Vercel Auto-Migration

Solution: Let Vercel handle migrations automatically during deployment

How it works:

  1. Created migration SQL file manually
  2. Vercel detects new migration during deployment
  3. Vercel runs prisma migrate deploy with direct database URL
  4. Migration applies automatically to production database

Benefits:

  • ✅ No need for local database access
  • ✅ Migrations apply safely during deployment
  • ✅ Vercel uses direct connection URL automatically
  • ✅ Zero-downtime deployment

⚠️ Important Note

For future local development:

If you need to run migrations locally, you'll need to:

  1. Get POSTGRES_URL_NON_POOLING from Vercel Dashboard
  2. Add it to .env as DIRECT_URL
  3. Run migrations with direct URL

For this implementation: Not needed! Vercel handles it automatically.
