🔄 How It Works

1

Order Delivered

Trigger: Lalamove COMPLETED webhook

Action: System sets autoCompleteAt = current date + 7 days

File: /app/api/webhooks/lalamove/route.js

const autoCompleteDate = new Date() autoCompleteDate.setDate(autoCompleteDate.getDate() + 7)

Find Eligible Orders

Criteria:

  • Status = DELIVERED
  • autoCompleteAt ≤ current date
const ordersToComplete = await prisma.order.findMany({ where: { status: 'DELIVERED', autoCompleteAt: { lte: new Date() } } })
4

Complete Each Order

For each eligible order:

  1. Update status to COMPLETED
  2. Set completedAt timestamp
  3. Send completion email to customer
  4. Log the completion
await prisma.order.update({ where: { id: order.id }, data: { status: 'COMPLETED', completedAt: new Date() } }) await sendOrderStatusEmail( order.user.email, order, order.user, order.store, 'COMPLETED' )
5

Return Results

Cron job returns summary:

{ "success": true, "timestamp": "2025-12-03T00:00:00.000Z", "results": { "totalOrders": 5, "completed": 5, "failed": 0, "errors": [] } }

⚙️ Environment Setup

Required Environment Variable:

CRON_SECRET=your-secure-random-string-here

⚠️ Important: Set CRON_SECRET in Vercel

  1. Go to Vercel Dashboard → Project → Settings → Environment Variables
  2. Add CRON_SECRET with a secure random value
  3. This prevents unauthorized access to the cron endpoint
  4. Vercel will automatically include this in cron job requests

Then trigger the cron endpoint to complete the order.

Phase Trigger Status Change Timestamp Set Email Sent
Payment PayMongo webhook ORDER_PLACED → PAID paidAt ✅ Payment confirmation
Phase 3 Lalamove booking PAID → PROCESSING - ✅ Order processing
Phase 2 PICKED_UP webhook PROCESSING → SHIPPED shippedAt ✅ Order shipped
Phase 2 COMPLETED webhook SHIPPED → DELIVERED deliveredAt, autoCompleteAt ✅ Order delivered
Phase 4 Auto-completion (7 days) DELIVERED → COMPLETED completedAt ✅ Order completed
